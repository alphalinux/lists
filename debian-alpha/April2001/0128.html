On Fri, Apr 27, 2001 at 11:05:16PM -0400, Tom Vier wrote:
> is it safe to use pci_device_override on a miata, with a matrox millennium
> II in one of the 64bit slots? i've build kernels without trouble, so it
> seems safe, unless it's just very rarely triggered. i've read that this card
> should work fine in a miata, and i was surprised when srm complained about
> it.

Did you upgrade to the latest version of SRM? That may help.

If it still complains, it should be safe enough to use
pci_device_override; that SRM console message is really only concerned
with controllers for SCSI, Ethernet, and such that might do things
that will tickle the PYXIS in inappropriate ways.
