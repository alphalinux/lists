A new gzip has been installed in the archive, so this problem should
not occur anymore. For those who have been victim of it in the
meantime: here is a possible solution:
 
As soon as you have upgraded to the latest libc in the archive, you
will be safe of any trouble. The problem is that you cannot upgrade
without gzip, so you need to install manually a new gzip (you could do
this for libc, but upgrading libc manually is a more dangerous
experience).

Take a "neutral" gzip (means should run with
every libc) on http://www.ens-lyon.fr/~lprylli/axp/

Install manually (link to it gunzip, zcat, uncompress).

Then dpkg should work, upgrade libc _first_.

After libc is upgraded, nothing wrong can happen to your system
anymore, feel free to upgrade to any package in the archive.

You can also boot from the rescue disk and copy the gzip there into
your normal root filesystem.
