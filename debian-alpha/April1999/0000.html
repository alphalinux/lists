If you are booting via MILO, be aware that there currently are *no* MILOs
which supply PALcode that includes the "wrperfmon" PALcall, which iprobe
needs to function. It should work fine if you boot from SRM console via
aboot.

In addition, iprobe only works on EV5 CPU based machines; not EV4 and not EV6.
