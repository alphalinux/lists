I had an idea...

In the past (2.0.x kernels), we've had to specify the system type during
kernel configuration, thereby making the kernel-source packages kinda dumb
for us since we would have to upload one deb per machine type (noname,
alcor, jensen, etc).

Well, after MUCH testing on my part with UDB's and a noname, so far I've
found the new "generic" machine type kernels to work nicely. Because of
this, I would like to try compiling a vanilla kernel-source package up
into a deb and have someone try booting with the kernel on as many machine
types as we can get.

If anyone would like to try this, let me know. As a slight confidence
booster, I exclusively compile my new kernels as "generic" rather than
noname now with no problems.

Thanks!
