Hi!

>fat office? Is it that bad?

That's what we called the version which is running locally.
The package is 65MB big (compressed!), some graphics & so
included, but the libraries take a lot of space. And the sources
to build the libraries... you get the picture.

> How does it (the program not the license) computer to applixware?

I only read some tests about applixware, and as far as I can tell
StarOffice has more features with all bells and whistles,
is more bloated because of that and a bit more unstable.
You get a whole suite of software, though: wordprocessor (lot of filters),
spreadsheet, graphic suite (which I can't comment on - I never used it),
time planner, browser, mailer, newsreader, IMAP4-client, database, etc.
Applix concentrates more on the "core": wordprocessor and spreadsheet.

On the legally side, I've checked my contract and I could not find
any paragraph in it which would forbid active work on a port of
the hopefully-soon-released-sources - but I still will have to check
the licence carefully, maybe they want to rule out former employees,
you never know.
