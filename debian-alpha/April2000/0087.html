Hi all...

Over the past week, I've been attempting to find a version of ldconfig
that actually works as advertised on Alpha. Well, what I concluded
was...there isn't one. The current implementation in the ld.so package
loves 32-bit ELF, but has no clue about 64-bit ELF. This is what's been
breaking the ncpfs package compilation all of this time (ldconfig can't
read the SONAME out of the library, so it doesn't set up links). Doing
ALOT of debugging on ldconfig shows that it's really not doing anything
for us at all.

So, I started trying to patch it for 64-bit ELF support (or actually, more
generic ELF support), but it just isn't going well. I'm lacking the time
to properly handle this one (even though I'm probably the best one to
handle it, unfortunately). I'm leaning towards a complete rewrite at this
stage, which would probably benefit the entire Linux community...

I would file a bug report, but I remember that not going over so well in
the past (thus the split and the ldconfig package awhile back).

Anyone wanna lend a hand and take a look at this for me? I'd greatly
appreciate it.

Think I will file a bug in the meantime, but I doubt that the maintainer
can do much about it ATM.

Thanks a ton!
C
