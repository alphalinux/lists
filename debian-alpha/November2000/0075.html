scr (whoknows@onlinehome.de) wrote:

> Hi,

Hello;

> I'm stuck while trying to boot/install potato using
> ARC/Milo on a PC164.
>
> After loading Milo I'm trying to boot into the
> installation system, but so far with no success.
> I tried out the following commands on the Milo
> command line:
>
> boot fd0:/linux.gz root=/dev/fd0
> to boot from boot/root floppies
>
> boot scd1:/linux root=/dev/scd1
> to boot directly from the CD
>
> In either way I run into a kernel panic:
> "Warning: unable to open an initial console.
> Kernel panic: No init found. Try passing init=
> option to kernel"

How about:

boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1

Replace vmlinux.gz with the kernel image if needed: i.e. vmlinux.gz,
vmlinuz.gz, linux, etc.

> So what can I do now? Any advice appreciated. Sorry
> if this is a FAQ, but I'm new to debian (and this
> list).

There are serveral FAQ's and other documentation on
<http://www.alphalinux.org>
