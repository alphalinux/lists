On 25 Sep 2000, James Troup wrote:

> Just a heads up: at some point dinstall is going to start rejecting
> packages with screwed up time-stamps as the number of them currently
> in the archive (a lot from binary-alpha, it seems) make our cron.daily
> output virtually unreadable. Now would be a good time to install
> ntpdate/chrony/ntpd/whatever on your porting machines :->

Already did....in fact, ntpdate has been running on my system for
months. There is a kernel problem, IIRC, with the RTC code on
Alpha. It's getting fixed upstream, but it prevents ntpdate/ntpd from
working properly.

Actually, I'm kinda anxious to see the archive reject the debs like
that. Unfortunately, I don't check my system time often enough, so this
would give me a good idea when I'm uploading a load of debs that are
broken like that.
