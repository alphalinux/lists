Hello,

I have a dec alpha system running the unstable debian tree. (kernel 2.3.99).

I have downloaded the binaries that were provided at xfree86's ftp
site: XFree86 version 4.0.1

However, this xserver makes my machine crash very badly: it simply
starts rebooting.

Because of this hard-reset, I cannot examine any logs on my filesystem:
they never got the chance of being flushed to disk.

Some details:

- I use the mga driver on a Matrox g200

- Starting the server with -configure flags works: the system nicely
  examines my machine and created a config file.
  when running the server with this config file, the machine reboots.

I expect this to be far more than a bug in the xserver.
Surely this must be a bug in the linux kernel?

My question:
Has anyone managed to binaries, provided by xfree86.org, work on
a dec alpha.
If so, with what gfx card?
Also, has anyone else had severe crashes like this?

I would try a pre 2.4 kernel, but read some messages about
2.4 not working 100% for alphas.
Also, my alpha is stable, and only crashes on this specific xserver,
and its not overclocked. (lx164 at 533)

many thx,

   Bram Stolk
