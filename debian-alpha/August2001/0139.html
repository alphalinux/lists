On Fri, 17 Aug 2001, Laurent Jacques wrote:

> I'm a debain newbie and I have a litle problem when I try to upgrade my
> debian potato distribution to the woody one. More precisely, it seems that
> there is an error with 'cp' (unrecognized/invalid -L option) on the package
> libc6.1 (version 2.2).

See bugs #106120, #106096 and #108679 in the Debian BTS
(http://bugs.debian.org)

> Any idea to solve this problem ?

I solved the problem for myself using a little wrapper for cp that removed
the -L option from it's commandline and then fixing the symlinks for
/lib/libdb.so.3 myself, but the best option is to wait for
version 2.2.3-8 or higher, which fixes this problem.
libc6.1 version 2.2.3-11 is now available in the unstable distribution, so
if you are impatient, you could try downloading libc6.1 from sid and use
that one.
