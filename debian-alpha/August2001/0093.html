Falk Hueffner wrote:
>
> Hi,
>
> I've built new boot floppies from the current boot-floppies CVS and
> put them on ftp://134.2.15.134/boot-floppies/disks-alpha/. I think
> they are now in a state to be added to ftp.debian.org at the next
> boot-floppies release. I've used MILO 2.2-17 this time, since 2.2-18
> was reported to be broken on at least two subarchitectures.
> Unfortunately, I have only my SX164 to test, so I'd appreciate any
> reports. I have not yet tested the complete installation on my
> machine, since I have pppoe for internet access which isn't supported.
>

I tested the new floppies on my Alpha XLT with the following results:

1. The "rescue" and "root" floppies work fine using the milo provided
with Potato built around the 2.0.35 kernel. I booted all the way to the
initial config screens (keyboard, etc) and stopped there. I have not
yet tried a full install due to Internet connectivity problems.

2. The new milo (built against the 2.2.17 kernel) hangs at "Swapping to
PALcode at 0x80000" and will not proceed. I don't have a clue about a
"fix", but using the Potato milo is a work-around for me.
