   I've Cc'd alpha list to ask about the package name currently involved
   (libc6.1) will that need to change to 6.2? What about libc0.2 for hurd?

Keep it libc0.2 for the Hurd. There will be a dependency issue
though. People will need a recent enough version of the hurd
package for glibc 2.2. AFAIK there isn't a Debian package yet that is
recent enough. It's probably best to coordinate this with Marcus
Brinkmann.

Oh, and by the way, the libstdc++.so versioning madness isn't applied
for the Hurd :-). There's definitely no reason for a seperate
libstdc++ package for glibc 2.2 on the Hurd.
