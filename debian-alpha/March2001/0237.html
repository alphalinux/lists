My hardware:

Digital "Multia" Alpha workstation
64MB RAM
500 MB Quantum HD

The problem:

Since I have no floppy drive and no way to hook a CD-ROM drive up, I'm
attempting to netboot the Debian installer.

The ARC and SRM consoles load fine (afaict). The SRM console loads the
kernel fine via bootp & tftp (again afaict). I'm using the command

        boot ewa0 -p bootp

to boot.

What happens is that after the kernel starts to run, I see a message
referring to some PAL code, get a warm reboot, and find myself back at the
SRM console. This happens reliably.

NetBSD does the same. I haven't tried anything else yet, mostly because
I'm pretty sure I'll get the same results.

Does anyone have any suggestions?
