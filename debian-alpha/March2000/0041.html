I've seen those with XF86_3Dlabs as well. The Oops (which seems to point
to a kernel bug somewhere in the KDxxxxx ioctls) only happens when the X
server finds out it can't run (not configured properly) and switches back
to the text mode. As soon as the server is configured, the Oopses should
cease.
