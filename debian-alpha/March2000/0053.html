David Huggins-Daines <dhuggins@linuxcare.com> writes:

> This also makes the directory layout a lot cleaner. Since we don't
> have any disk sizes other than 1.4M on Alpha, in 2.2.9, I think I will
> remove the intervening images-1.44/ directories (all we need to do is
> make dbootstrap not default to looking in these directories on Alpha -
> Sparc may want to do this too),

I don't like this idea. For one, I don't know why you couldn't have
2.88 disk images, for instance, for a combined boot/root. For two,
even if you only have one floppy size, it's still interesting to have
the image-1.44 dir since it is clear what files are the disk images.

Finally, deviating from having it be the same or generally the same on
all architectures makes it all that much harder to document and
maintain stuff.
