>> When I try to install libc6.1_*-9, ldconfig gets run, which tanks on
>> an illegal instruction still.

I encountered this problem the other day. It seems that the ldso 1.9.11-5 was
corrupted. Shortly after this, however, I found that there was a new package on
the main Debian FTP (may not have reached all mirrors yet), version
1.9.11-5.0.1. I downloaded this with ftp and used dpkg to install it. After
that, ldconfig was fine.

Get this package at ftp.debian.org, as it's certain to be there. It was only
updated last night (which is interesting, because 1.9.11-5 has a timestamp that
was four days old when it was pulled.)
