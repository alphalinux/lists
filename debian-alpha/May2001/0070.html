Hello Alex,

On Tue, 22 May 2001, Alex Slok wrote:

> Hi all,
>
> I am trying to install Debian 2.2r3 on a DEC2000 300 (Jensen). I used
> the jensen17.img from
> http://www.alphalinux.org/docs/jensen-install-0.7.shtml as the boot
> image on my first partition. When switching to the OSF console the
> machine automaticallly boots into aboot ??? Well don't think that
> should be a problem, but still.
        BTW if you type "q" in aboot 0.7 you will get back to the srm
prompt.
>
> At the aboot promt I will type :
>
> aboot> b 0/dists/potato/main/disks-alpha/current/jensen/linux
        This will not work. The kernel from jensen17.img has to be started
by giving aboot the following line (sans stuff in double quotes):
aboot> b 0/a "put_your_boot_options_partitions_etc_here"
The reason is that aboot only boots kernels from the same medium it
resides on (aka the same scsi device as jensens only boot from scsi
devices).

Ahoi & good luck
