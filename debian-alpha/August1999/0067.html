Hi Paul,

I got some troubles with a my system clock some weeks ago as well. The new
milo on a SX with a kernel < 2.2.{8??} causes it to speed up time (hehe)...
So every time I booted to linux my clock would be screwed. Setting it back was
a struggle. So here is my remedy for all those who encounter it.

turn off hwclock.sh (/etc/init.d/hwclock.sh stop)
set date (date -s ..)
move the /etc/adjtime file to e.g. /etc/adjtime.old
turn on hwclock.sh

And all should be fine again. So maybe this is bothering you here as well.
When you leave the /etc/adjtime file in place the next boot will screw your
time again. (and again...)

HTH,
