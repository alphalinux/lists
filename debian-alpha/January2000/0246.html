Hi -

I had similar problems a while ago with our alpha and a DEC tape drive. I
could not get it working reliably enough to use on our machine which we use
as a file server (and backup device), so I swapped to a new ethernet card.
Apparently, the qlogic driver _does_ support disconnect/reconnect according
to Matthew Jacob who wrote the new combined qlogic driver (although this is
from memory so sorry if I'm wrong). It may be worth trying his driver
instead as it definately made a difference on our alpha box - see
http://www.feral.com Its been in the Free/Net/OpenBSD releases for quite a
while. Unfortunately we'd already bought the new card so I couldnt say for
definite it fixed our tape/scsi problems.

>
> Not sure how to interpret this since the tape drive does work, the only
