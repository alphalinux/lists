> this has been happening for a while.. was waiting to see if it got updated
> and fix itself but that has not happened. I've even tried having apt/dpkg
> build the source package, but that failed to finish the compile. Here is
> the error
>
> Warning: /boot/System.map not parseable as a System.map
> Warning: /usr/src/linux/System.map not parseable as a System.map
> PID TTY TIME CMD
> 20731 pts/1 00:00:00 bash
> 4122 pts/1 00:00:00 ps
>
> Any ideas? the source package was generating a bunch of cast problems
> etc.. before it failed.

It's really a warning, not an error. To fix it, copy the System.map file
from your compiled kernel source tree to /. It just needs a reference to
map the kernel stuff to (which that file provides).
