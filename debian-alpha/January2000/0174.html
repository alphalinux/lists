On 21 Jan 2000, Ron Farrer wrote:

> With the help of "Jaldhar H. Vyas" <jaldhar@debian.org> I have been
> trying to build pine deb's for one of my systems running potato. After
> getting quite a ways I get:
>
> <SNIPPED>
> dpkg-shlibdeps bin/pine
> dpkg-shlibdeps: failure: cannot exec ldd: No such file or directory
> dpkg-shlibdeps: failure: ldd on `bin/pine' gave error exit status 2
> make[1]: *** [binaryPine] Error 35
> make[1]: Leaving directory `/home/rbf/pine-tmp/deb/pine-4.21'
> make: *** [binary-arch] Error 2
>
> ldso _IS_ installed and the current version. Is this something that is
> a known problem? If anyone wants a complete log of what happens contact
> me in private and I'll send it to you, it is long.

Unlike i386 (in the past), ldso does not provide ldd on Alpha-potato.
>From the looks of the error, it seems that you don't have the libc6.1-bin
package installed (which provides ldd on Alpha and other archs). Try
installing that first, but if it still barfs on you, please mail back and
we can try a few more things.

FYI, pine's attachments on my Alpha are severely broken. I'm working on
debugging that hopefully this weekend, so beware on that front...
