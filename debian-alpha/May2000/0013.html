"Marcus Williams" <marcus@quintic.co.uk> writes:
> The server froze completely and had to be rebooted.

On a Multia, I would worry that this was hardware, not
software---they're finnicky little boxes. I have one here that will
run for long periods of time, but if I cycle power, it won't start
again until I pull out the motherboard, wiggle the power connection
and whisper sweet nothings in its ear.

> However, this week cvs/ssh/samba requests started to block. After
> checking on the server the only thing that could point to this was a
> lot of 'NFS: task XXXX can't get request slot' messages in the logs
> (some of the cvs roots/home drives etc are served via NFS). The
> network load appears to be normal and the server itself was fairly
> idle. Once these messages start occuring the server seems to go off
> on a spiral of death and can only be resurrected by a reboot.

I've seen these on i386, though I seem to remember having seen them on
the _client_, not the server.
