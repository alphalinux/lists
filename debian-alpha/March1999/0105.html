Background to explain motivation for the changes:

1)
For the change of the base2_1.tgz files:
It appeared that the master file that have been used for profiles and
tasks on the Alpha root disks was a bit obsolete. In particular X11
packages were not selected as appropriate for some profiles, leading
to incomplete selection at installation. The user had to resolve
the conflict/dependency problems manually after selecting a profile.
I have tested two different install with the new base files.

2)
About the nis package:
 I know the binary compiled from hamm source is rock solid, I am
running it on several productions machines for some months, on an
other machine where I have tried to run the current nis, it has never
lasted more than a few days before segfaulting. I have try to upload
new recompiled package that seems more stable, but finally even the
last nis_3.3.1-1axp3.deb is not stable.
