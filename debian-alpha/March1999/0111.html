With avanti-s, there is no MILO, so you need to use:

>>> boot dva0 -fl i
aboot>linux root=/dev/fd0 load_ramdisk=1 (or whatever you want).

> What do I need to generate my own set of boot floppies using the
> MILO I have?

Start from the avanti rescue disk (not the avanti-s), this is an msdos disk,
replace the milo on it by your milo. If you want to be able to boot it
from SRM, do srmbootfat /dev/fd0 milo afterwards.
