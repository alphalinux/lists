> - Is this problem known?

No, not to me at least...

> - Why is that? I read about the UART chip on the board; seems fairly standard.
> - How to fix it? Update/rcompile kernel? Which version, which patches?

You can try the latest setserial from potato. Download the *source* and
compile it. You have to install dpkg-dev and fakeroot/sudo to do so.
dpkg-source -x -sn setserial[..].dsc
dpkg-build -B -muser@localhost -uc -rfakeroot

Install the result and see if this solves it. If so, or if not, please report
back ;-)
