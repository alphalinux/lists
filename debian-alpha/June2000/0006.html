Hi,

I have to comment my own mailing.

It seems that the problem is solved. At first I tried some things using
the --test option.

hwclock --test --directisa --debug --show

told the correct hardware time, but still wrong system time. Adding
--localtime gave correct results. So I changed the entry in
/etc/defaults/rcS again to UTC=no, did a "hwclock --directisa
--localtime" and a "hwclock --hctosys".

So alphons has the correct time setting now, but as one consequence the
last entries of the .bash_history are gone.
