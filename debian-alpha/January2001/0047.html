Happy new year to everyone !

I have some troubles to use my PWS500 on a 100 baseTX network.
Under SRM, I have forced the network card to use the FastFD mode. But
when the kernel boot up, the card switches to 10 baseT/Full duplex mode.

Following, there is some logs :

Under SRM, the 'sh conf' command says :
>Digital Semiconductor 21143 Network Controller
> ewa0.0.0.3.0

and 'sh ewa0_mode' says :
> ewa0_mode FASTFD

'dmesg' says :
>tulip.c:v0.91g-ppc 7/16/99 becker@cesdis.gsfc.nasa.gov
>eth0: Digital DS21143 Tulip rev 48 at 0x8000, 00:00:F8:76:40:75, IRQ 24.
>eth0: EEPROM default media type Autosense.
>eth0: Index #0 - Media 10baseT (#0) described by a 21142 Serial PHY (2) block.
>eth0: Index #1 - Media 10baseT-FD (#4) described by a 21142 Serial PHY (2) block.
>eth0: Index #2 - Media 10base2 (#1) described by a 21142 Serial PHY (2) block.
>eth0: Index #3 - Media AUI (#2) described by a 21142 Serial PHY (2) block.
>eth0: Index #4 - Media MII (#11) described by a 21142 MII PHY (3) block.
>eth0: MII transceiver #5 config 2100 status 784b advertising 01e1.
>eth0: Advertising 0061 on PHY 5, previously advertising 01e1.
>eth0: Advertising 0061 (to advertise is 0060).
>eth1: region already allocated at 0x8000.
>(...)
>eth0: Setting full-duplex based on MII#5 link partner capability of 41e1.

If someone has an idea ?
