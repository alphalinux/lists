Lorenzo,

You can test your serial cable with NT's hyperterminal. Set the line to
9600, 8N1, no flow control (on both sides, if possible).

If MILO is alive, you should be able to see the MILO> prompt after
pressing Enter on the terminal. If it's dead, you won't see anything... :-(

You may want to enable MILO's early serial output, which is disabled by
default because it used to confuse modems. This can be done with the
miloctl tool on any little-endian machine in a manner similar to:
        miloctl -v /dev/fd0 +S +V
(or by mounting the floppy and specifying the actual milo image on the
command line - see the miloctl manpage).

Miloctl is available from genie.ucd.ie (an Alpha binary and the manpage).
If you want the sources, they are in the MILO source tree under
tools/boot.

Good luck,
