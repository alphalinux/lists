Hi..

I played around with milo a bit. The next release will at least compile on
all platforms and is tested on LX164 and Ruffian.
I got the idea that it would be nice to boot other OSes using Milo, like
for example NetBSD. The kernel of NetBSD is quite huge, so I had to set
the maximum kernel size to 8 instead of 4 megs. Now the Kernel (elf binary
using buggy binutils) gets loaded but after clearing out BSS memory in
boot.c (boot_main.c) load_image_from_memory() It seems to hang at the
return kernel_entry; (last command of function)

What happens here? Would it be hard to fix this? Does anyone have a hint
about the Milo problems with newer binutils?

I guess there will be much more problems with NetBSD like i.e. missing pal
code functions. But maybe the BSD guys would participate in Milo
development if we have a good base.
