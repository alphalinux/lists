On Monday, 19 Apr, Jeff Garzik wrote:
> Where are the latest? Should we set up sources on a CVS server
> somewhere? A versioned release with all these new features would be
> nice...

The latest official sources are on gatekeeper in the Kernels directory.

The branch I currently maintain, which can be considered as advanced
when compared to the official version, can be found at:
        ftp://genie.ucd.ie/pub/alpha/milo
This is the version I would personally call "the latest".

There are also other branches with their own "latest" sources.

>
> FWIW, I am very interested in evolving MILO long-term into a featureful,
> and maintainable piece of code. I even thought about making MILO simply
> a set of kernel configuration parameters, so that it would be easier to
> track kernel sources.

That is, merge MILO with the kernel sources? This way it will be much
easier to maintain it, but most people will not appreciate having to
download extra 1.5 Mb... On the other hand, people use only a very
small amount of all the existing drivers, and MILO can be counted as yet
another kernel feature.

I actually have a bunch of ideas on re-organising MILO. One of them is
to teach MILO to use kernel sources rather than pre-built libraries and
object files, and to build whatever it (MILO) needs on its private
premises. It should be possible to make a single configuration process,
which would include config.in from different (relevant to MILO) parts of
the kernel tree and of course one for MILO itself. The resulting
configuration would then be used for both kernel and MILO compilation.

I shall probably elaborate more on the ideas I have in a couple of
days...

Nikita
