Your message brings up a meta-issue I consider every now and then:
support for older Alpha machines.

Is working on MILO the best route? Maybe we can convince Compaq to let
us get SRM going for older machines. Then we could standardize on SRM
for firmware, and aboot for the boot loader. The bigger problem (IMHO)
is that the various alpha platforms have such specialized needs. I am
such a geek that I want to support them all :) but from the perspective
of a distro builder, supporting SRM/alphabios/milo/aboot/etc/etc are a
real pain.

Maybe combine linuxbios+linux[+milo+openbsd?] code to create GPL'd alpha
firmware?

Comments welcome. I'm already putting some personal time into keeping
the alpha kernel and Mandrake's alpha distro going. I would be willing
to help out here... I would much rather like milo to morph into
firmware proper, than to have to support aboot and milo both as
bootloaders...
