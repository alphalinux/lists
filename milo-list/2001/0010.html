> arjan pointed me to eCos: http://sources.redhat.com/ecos/

> I think linuxbios+eCos is better than linuxbios+linux, for open source
> alpha firmware, because the end result will be much, much smaller.

note it comes with the penalty that it doesn't have linux's driver
support. maybe we can hack a linux driver API into eCos, if we don't
use linuxbios+linux for the firmware...

Also, Tigran A has created a "bcp" patch for the Linux kernel. BCP is
Boot Command Processor, ie. an SRM-like command line. This would be
useful in creating firmware also.
