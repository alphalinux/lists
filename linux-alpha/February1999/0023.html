Distribution: RedHat Linux 5.2/alpha
Machine: Digital AlphaStation 200 4/233 192MB RAM
Sotrage: NCR53C810 controller, RZ28M disk, RRD45 cdrom, TLZ07 tape
Video: Digital TGA (ZLXp-E1, 8 bit only)
Monitor: Digital VRT17-H4 (good for at least 1280x1024)
Upgrades: modutils 2.1.121, net-tools 1.50, procinfo v15 & all from
redhat's update site
Bootup: SRM from e2writeboot diskette, using aboot from redhat 5.2
kernel diskette
Firmware: V5.3 CD from Digital, that's ARC V4.57 and SRM V6.9

Bootup string is >>> boot dva0 -file vmlinux.gz -flags "root=/dev/sda1"
If the kernel is compiled with TGAfb, the screen will go blank and the
system flies to some other world.
The kernel without TGAfb seems to boot ok, but with serial console only.
I've attached my config file. I've also tried CONFIG_SRM_SETUP=Y with
the same results.
Kernel (2.0.35) reports an Alpha PCI BIOS32 revision 0.03.
It also warns "bios32 WARNING: slot 13, function 0 request 0x8000000
bytes of contig address space--don't use sparse memory accesses in this
device!!"
And also "bios32.sio_fixup: weird, device 8086:0484 coming in on slot 7
has no irq line!!"
Compiler is redhat's included egcs-2.90.29 980515 (egcs-1.0.3 release)
with default kernel Makefile optimizations.
I've tried ARC+MILO without success, crashing hard (the machine needing
a hard reset) when i insert the kernel diskette at the "milo> boot
floppy" prompt just when installing, with both avanti.img and
avanti-s.img.
X works just fine with redhat's kernel, tested at 1280x1024 8 bit color.
Any hints? Should i trash the TGA and get some plain S3?
Maybe i'm missing something in the kernel config?
TIA!
