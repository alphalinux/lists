Hi, I'm new to the list but an alpha veteran,

    I have an alphastation 500/500, that's happily been running 2.0.34
with the ipchains patches for about a year. It has never crashed.
However
on compiling 2.2.5 with and without the patches from redhat, with both
gcc 2.7.2.3 and egcs 1.1.2, trying both SRM and MILO.* My kernel locks
hard
with no message. The easiest way to get this to happen is

ping -f another_machine
dd if=/dev/fd0 of=/dev/null

which usually does the job. This doesn't crash things under 2.0.34. I
think
it's an interrupt related problem as I can improve the reliability by
adding
a few cli()s here and there. 2.1.99 seems stable, but under MILO
enabling
interrupts on the printer port seems to stop all EISA irqs happening
ever at all.
(Ie no keyboard or mouse, but everything else is ok). I'll try to pin
down the
2.1.xx at which it breaks totally, ARE THERE ANY 2.1.X kernels I should
avoid?

*I've seen some MILO distns which think the bret and the maverick have
21064
and have palcode that doesn't enable the byte/word ops.

Thanks,

James.

Configuration of the AS500/500 is as follows

QLogic ISP 1020
DEC TGA2
S3Trio64/V2 (Added by Me)
DC21040 Tulip
82375EB off which hangs a super io controller, and a soundcard.
