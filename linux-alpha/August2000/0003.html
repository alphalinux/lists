Hi Warren,

Per your disk layout and per the default RH Alpha 6.2 install, you should
simply be able to boot your system as follows:

boot dqb0 -fl 0

The new RH 6.2 uses the latest aboot. It uses a new format.

For this to work, the assumptions RH makes are as follows:

1.) The /boot directory can be found in the 2nd disk partition. As you
laid your disk for us below, the /boot is not in a separate partition, and
therefore will be in the / partition which as you say is the 2nd partition.

2.) The flag == 0, means your boot kernel is the first entry in your
/etc/aboot.conf file.

3.) RH 6.2 places aboot in the beginning of your disk for you.

So, if this does not work....

1.) Boot the RH CD while mounting the hard drive

boot dqa0 -fi kernels/generic.gz -fl "root=/dev/hdc2" (you may need to
change the 2 to another number to get this to work).

2.) Determine what partition the /boot directory really exists in by
viewing the /etc/fstab file.

3.) Tell aboot what partition to use at next boot:

abootconf /dev/hdc X

4.) Shutdown

5.) boot using the "boot dqb -fl 0" scheme again.

Please remember I am writing this from memory, and I do not have an Alpha in
front of me to confirm this (at this time).
