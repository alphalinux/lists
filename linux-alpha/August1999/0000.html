I've got a new DP264 board and I've had some issues with Linux.
I'm wondering if anyone else has had these problems and whether
anyone has fixed them...

(1) If I don't boot off the floppy drive the floppy controller isn't
    found:

    Floppy drive(s): fd0 is 2.88M
    floppy0: no floppy controllers found

    Under some conditions I can get the kernel to find the controller
    but for the most part it's not found. Is this a H/W problem?

(2) Time is very messed up. The value of HZ is 2048. I've been told
    that someone wrote HZ=1024*(# of processors) in the C library.
    Also, as reported before the "time" program returns crazy values:

    575.468u 5.296s 1:17.82 746.2% 0+0k 0+0io -350116377303073892pf+0w

    The "date" command seems to work fine, but lots of things that
    rely on HZ are broken.

(3) Programs that aren't compiled with "-mcpu=ev6" get unaligned
    access warnings. This seems like a egcs compiler bug...no?

(4) tcsh 6.08 history is broken. Even if I do "source -h .cshrc" the
    history will still be blank with "set history=1000", If I "set
    history=10" it will remember what I "source" but not any commands
    I type.

Anybody know how to fix any of these or have any info?

        Thanks,
        Rick Niles.
