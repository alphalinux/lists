I had similar problems installing Linux to Alpha Servers previously.

However, following this howto at CompaQ/digital:
http://www.digital.com/alphaserver/linux/install_guide.html
I managed to install RedHat Linux 6.0 (kernel 2.2.5-xx) on 3 different
machines, ie. AlphaServer 4100, AlphaServer 1000A and an AlphaStation 500/333.
All installed through the SRM console. I could have installed it with less
headaches if I just heeded the fact that aboot requires the boot partition
requires a BSD disklabel (hence normal fdisk routine is insufficient)

Having said that, all of the machines had a qLogic scsi controllers, but none
of them had any RAID controllers, so I am not sure it will work on your
config. But to me, it is worth a try coz all 3 machine is running A-OK since
install (Tested and Running Apache + PHP Module, MYSQL, and Qmail)

Hope this helps in making you reach your decision [or god forbid, confuses the
issue even more :-( ]
