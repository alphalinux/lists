Hi,
   It seems that the problem is partly due to my switch after all. I am
using a cheap manual switch and what happens when I switch from one system
to the next is that some residual "noise" causes the mouse and/or keyboard
to become dysfunctional. The problem is only apparent when I switch between
different Opsys'. For example going from NT to Linux or Win98 to NT or
anyone of the above to Linux. What I did to somewhat compensate is to
connect Linux to port 4, NT to port 1 and Windows to Port 2. That way I
have a buffer of no signal on port 3 between the Linux box and the Windows
boxes. The only thing is that in switching from windows to Linux I have to
change from the X console to a text only console and then back (say 5 to 1
to 5) to reinitialize the mouse. That is what I think is strange b/c at
that point the effects of the KVM should be removed and it should be like I
just removed the mouse and stuck it back in. I hope this is clear.
