On Tue, Jan 23, 2001 at 03:17:13PM +0100, Markus Klappenbach wrote:
> Hi all,
>
> i have a problem with my 164LX board. i try to install a pci card in the
> last pci slot (DEC NIC 21140 which works fine in any other slot). the
> configuration is as follows: slot0; Matrox Millenium, slot1; HBA Symbios
> 8751SP, slot2; WintTV PCI and slot3;?. if i install any card in any
> combination in the last slot the system does not boot with SRM-5.8-1 (no
> VGA Signal) and with AlphaBios-5.70 the system boots but WINNT4 crashes
> with NMI: Parity error. if i remove the card from slot3 the SRM Console
> boots and i am able to boot Redhat-7.0

Hmmm.. I've got an old AXPpci33 system which shows, well, comparable
behavior: one of the PCI slots can't trigger interrupts there. I was
told that IRQ13 is routed to that slot, which gets ignored after
because IRQ13 was reserved for FPU on ia32. For my board, I can find
the table in ./linux/arch/alpha/kernel/sys_sio.c:343. (IRQ13 gets routed
to last slot). You'll find sth like that for your board you may want
to play with...
