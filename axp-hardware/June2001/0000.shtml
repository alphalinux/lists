On DEC^WCompaq's Jumpstart CD-ROMs there is a patch to access SRM
environment variables through procfs. I've taken that code and
forward ported it to recent 2.4.x kernels. However - I don't have
any chance to test it because my Alpha system (AXPpci33) is miles
away:-(

Would anybody like to test this (completely untested;-] patch and
send me a ksymoopsed trace^W^W success report? Patch is against
2.4.6-pre1, but should apply cleanly (maybe except
./Documentation/Configure.help) to all recent 2.4.x kernels.

However - there are some glitches.
        - I don't know if copy_from_user() is right at the point
        - I don't know if all return -ENOMEM, -EFAULT etc. are
          correct
So it would be nice to have some feedback...
