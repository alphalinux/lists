Hi,

I've successfully assembled my first alpha last friday.
I bought a used 164LX-board with a 533Mhz 21164a and 128MB ECC-SDRAM.
It is now in a 300W ATX-case together with a Western Digital 10,2GB HD and a
40x Teac-CDROM, both E-IDE. The graphics-card is a Voodoo 3 2000PCI with
16MB-SGRAM, because I haven't found another PCI-card.
My Alpha boots without problems into AlphaBIOS and I've already installed
Debian/GNU Linux 2.1, which runs nicely.

Now I want to copy my home-directory from my Pentium 120-PC to my new alpha
by using plip over a parallel cable.
The PC is set up correctly, because he was connected with a 386 over my
parallel cable.

But my Alpha's parallel port does not seem to be allright:
In AlphaBIOS LPT1 is shown as SPP at 0x3bc and IRQ 7, but Linux (2.0.36,
2.2.5 and 2.2.10) doesn't recognize the IRQ. When I do "cat > /dev/lp0" dmesg
says "lp0 off-line", so I assume that the driver works still correctly,
because no printer is attached. This also seems to be the case why cat has
to be killed and doesn't exit on Ctrl-D.

But PLIP doesn't work, because it needs an IRQ. When I set the IRQ
for parport0 to 7 (as shown in AlphaBIOS) by cat-ting it to
/proc/parport/0/irq, the lp-driver still works for some time and again says
"lp0 off-line". But when I then kill "cat > /dev/lp0" with Ctrl-C, my alpha
hangs *hard*. Neither Ctrl-Alt-Del(for init) nor Alt-SysRq works at this
moment. The cursor still blinks.
This also happens when I try to ping over the PLIP link, either from
the alpha or from the PC. According to ifconfig the PC has received some
packets, but wasn't able to send some back (ifconfig on the PC):
RX packets:29 errors:0 dropped:1 overruns:0 frame:0
TX packets:0 errors:68 dropped:0 overruns:0 carrier:68

I can't try remote-shutdown, because the plip-link is the only
net-connection the alpha has.
I hope some of you on this mailing-list can help me.

Besides this problem, my first experiences with alphalinux are very well.
In the mailing-lists on alphalinux I've read that the Sb64 doesn't run on
alphas because of some PCI-confusion. Now my question is, which soundcards
do work on the alpha ??

I hope my english isn't too bad and I'd be happy about a solution for my
problem.
Greetings and happy alpha-linuxing,
Marius Hillenbrand
