>
> Hi Folks:
>
> Well I finally got my cheesy AXP noname 166 mobo up and running.
> Of course the problems have not ended. It seems as if Red Hat 6.1 wants
> more than the 1.2 gig of drive space I have for it. Am I missing
> something? Is there really a minimum / size > 1Gig? There seems to be no
> way to pick and choose packages before partition. Can anybody recommend a
> distro of Linux that has a base install, root, X-apps (no server needed),
> C and maybe Fortran, that I can fit on a 1.1 gig partition? It would be
> nice to be able to pick and choose packages as well. I find these unit
> install options (Kde, Gnome, Wkstation etc) to be madding.

You've got about 1GB *and* you've trouble to install what you want?
Those little apps shoult fit 4 times (or more often;) on that hard
disk... Try to install Debian Potato. I'm quite happy with Debian...

Hint: Begin with a "minimal" installation. Then, afterwards, install
whatever you miss. A simple "apt-get install gnapster" for example
will install anything (including X base files, X libs etc.) you
*need* for gnapster). It's fun;)
