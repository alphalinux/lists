Hello again,

Sorry for all the messages- I keep finding new problems in the rpms. :-(

There's an error in the fastmath top-level makefile so that libffm.so
comes with no symbols defined (except for pow, which depends on powr, so
it's not very helpful). It seems that all of the .o files except pow.o
are deleted before the shared library is created.

There are two possible fixes: modify the libffm makefile so that it
doesn't remove the object files (or so that it creates the shared
library itself), or expand line 21 of the makefile replacing "make
pow.o" with "make asincos.o atan.o atan2.o exp.o log.o powr.o sincos.o
tancotan.o sinhcosh.o sqrt.o sqrti.o dsqrtv.o dsqrtiv.o pow.o". Kinda
ugly, but I don't see a way around one or the other. Of course, since
libffm is LGPL, you can modify and redistribute the makefile with no
copyright problems.

One other question: why call it libffm.so.1.0.0 when it's 0.28?

Finally, a question to the libffm authors: as of version 0.28, aside
from log32 and sqrt32, I don't see any symbols in libfm that are missing
from libffm. Can we now link -lffm -lm and remove libfm from our
systems?
