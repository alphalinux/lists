Hello,

Now a new problem: I know how to fix the makefile, how do I build a new
binary RPM?

I tried editing the Makefile in the BUILD/BLAS dir, and all of the tests
worked fine, then any rpm -b command killed the whole BUILD/BLAS tree,
making the build die in the tests.

So, I tried to back out, construct an entirely new lapackrpm directory,
and re-construct the blas-alpha.tar file. Then while building the RPM,
the patch failed!

How can I re-create the blas_patch to reflect my changes in the
top-level makefile? How can I make the RPM? At least I can copy
BUILD/BLAS/libblas.a and .so* to the right places, but then lapack.rpm
won't install unless I can build and install the blas rpm!!

Finally just added . to my path and built successfully, but but didn't
create the .rpm in RPMS! Oh well.

This RPM system seems to make installation negligibly easier ("make" is
planty easy for me), but modification extremely difficult. Since
parts of the library aren't LGPL, we can't distribute the RPMs, thus
negating much of the motive for making an RPM in the first place. As a
general principle, for something like this under heavy development,
might it make more sense to have a source .tar.gz with a makefile and a
place to put the non-LGPL part, patch only that part, and distribute the
.spec file separately? It seems this would make modifying and
rebuilding far less painful.

Sorry to flame, I'm just frustrated after a couple of hours' worth of
repeated fix/rebuild attempts.
