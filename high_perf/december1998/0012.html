Jonathan,

I posted that you may try the BLOCKINGSIZE from 16 to 1000, but
I checked the program again and now I think that the parameter
will hit the best performance with value 4 or 5.
(with that small value, the unrolling method of the subroutine
might be changed, but I will not mention for simplicity.)

The MAF has only 6 entries, and all the prefetch may use an entry,
then when it reaches to the 6th loop there may be some stall.
So, if you set the BLOCKINGSIZE to 5 the program will call the vectorlized
sqrti before the stall and the pending prefetch will be executed during
the calculation.

The last MAF entry can not merge same line load, then the parameter
of 4 may hit higher performance. Or if we can fully control load order
from high level language, we still can use 5 for best performance.
