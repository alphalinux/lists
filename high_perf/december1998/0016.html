When I checked the generated code of the find_jastrow_sub,
I found the gcc generates unnecessary stores.
In this paticuler problem, we don't need to use software pipelining.
Because:
 1. The vector will be divided into a block which will fit in Dcache.
 2. The block size may be very small.

Then replace the subroutine with following code.

-------------------------------------------------------------
double find_jastrow_sub(double *vtemp, double b, int count) {
  int i;
  double tmp0,tmp1,tmp2,tmp3;

    dsqrtiv(vtemp, vtemp, count);

    tmp0 = tmp1 = tmp2 = tmp3 = 1.0;
    for(i=0;i < count-4; i+=4){
          tmp0 *= (1.0 - b * vtemp[i+0]);
          tmp1 *= (1.0 - b * vtemp[i+1]);
          tmp2 *= (1.0 - b * vtemp[i+2]);
          tmp3 *= (1.0 - b * vtemp[i+3]);
    }
/*
The rest of the loop will be here.
*/
    for(i=i;i < count; i++)
          tmp0 *= (1.0 - b * vtemp[i]);
    return tmp0*tmp1*tmp2*tmp3;
  }
