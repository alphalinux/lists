Hi, folks

I coded the optimized DGEMM with benchmarking facility.
The routine is TLB and cache friendly and you may have
good performance on a machine with smart memory system.

I designed it to run on any architecture, and it may be useful
for people without Alpha chip( where an assembler version exist).
Though it is coded by C, it also have interface for FORTRAN users.
Especially the matrices is row majored to be used with FORTRAN.

Some results with my local machines are here.

Machine OS C RESULT
===========================================================================
Pentium-133MHz FreeBSD gcc(-O2) 11.593 MFLOPS(172.523 sec)
21064A-300MHz(AlphaPC64) Linux EGCS(-O2 -Dev4) 39.312 MFLOPS(50.875 sec)
PA-8000-180MHz(D series) HPUX C89(+O4) 183.993 MFLOPS(10.870 sec)
R10K-200MHz(NEC UP4800) UNIX_SV CC(Kopt=2) 35.965 MFLOPS(55.610 sec)
UltraSparc-143MHz(Ultra5)SUNOS CC(-O5) 26.178 MFLOPS(76.400 sec)
---------------------------------------------------------------------------
C=alpha*(A*B)+beta*C where m=k=n=1000, lda=ldb=ldc=1001, alpha=2.0, beta=0.5
