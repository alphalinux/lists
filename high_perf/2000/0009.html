Franco Broi <franco@robres.com.au> wrote:
> Anybody had any success in making FFTs and convolutions go fast on the Alpha?
> The CXML libraries perform very badly on my XP1000 500MHz when compared to
> an HP 9000 240MHz.
>
> Normal FORTRAN code runs about 2-3 times faster so I was expedting big things
> from CXML.

If you time the FFT code in CXML as a function of the grid size N,
you will find that it performs very well for powers of 2, and
up to 5 times worse for nearby points :-( Compaq doesn't seem
to have the resources to tune the FFT in CXML for general grid sizes.

We have therefore chosen to use the Open Source FFTW library
available from http://www.fftw.org/ because it has a very
consistent high performance on the Alpha EV6/EV67. On powers of 2
and a few other grid sizes, the CXML performs a little better, though.
