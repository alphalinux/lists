For my own selfish reasons, I would love to see an accelerated dger, the
level 2 BLAS function which calculates the outer product of two
vectors. I think it is the last unoptimized part of dgetrf (which I use
a lot), and I think it is what makes dgetrf slow for large block sizes
(e.g. faster for 32x32 than 64x64, which I think should be faster for
dgemm).

By the way, I am finishing up a modified dgetrf which takes an extra
argument which is a function pointer to a user-supplied progress bar
function. It gives progress in approximate Flops, so it gives a pretty
good estimate of the fraction of time completed. User-supplied
functions can be fancy GUI bars (e.g. using Qt/gtk+/motif) which
calculate processor throughput and time remaining, or something as
simple as:

     fprintf(stderr, "Completed %d out of %d\r", done, total);
     fflush(stderr);

One can usually estimate how long something will take, but it's nice to
see it sometimes- especially to tell whether another job is slowing down
this one.

Anyone interested? It's pretty simple...
