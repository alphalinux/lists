Piotr wrote:

>But, when reading Digital Unix Assembly Programmers Guide (1996) one can
>learn about several integer division instructions (divl, divlu etc) on
>page 3-15. What is going on?

The architecture itself does not provide an integer division instruction.
The assembler treats these as macro instructions, and converts those
macro instructions into calls to libc routines.

>And if my computer has sqrt instruction why my compiler (egcs 1.1) does
>not use it (or how to insist on using it)?

The hardware sqrt instructions don't show up until the 21264 series chips.
Unless you specify which target (at gcc/egcs configure time) to generate
code for, the default is to generate the instructions which are currently
available for the machine you are building the compiler on ('native').
You _can_ trigger this with -mcpu=ev6. Also, if you want relatively
fast sqrt() behavior for machines which do have a hardware sqrt()
instruction, you'll need to use -ffast-math as well. Otherwise,
there is a bunch of extra code generated.
