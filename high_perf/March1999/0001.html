The following uuencoded compressed tar file contains several
variations on coding the double square root equation from
reflection seismology. The times are rather counterintuitive.

The run_"n" files contain data which may be plotted w/ the gnuplot
script, "plt" enclosed.

All examples were run using DU 4.0E and the software development
tools from the most recent ASAP SDK.

In particular, the Digipaq Extended Math Library vsqrt() has
the poorest performance of all. (run_2 & run_5 using sub2.f
and sub5.f respectively)

Interestingly, the straight forward loop unrolled twice w/
explicit temporaries matches the 25 cycles per result given
by Cray in "The Benchmarker's Guide to Single-processor
Optimization for CRAY T3E Systems" for the Cray vector libm
routines.

I am unable to assemble either of the sqrt routines posted
by Kazushige Goto <goto@statabo.rim.or.jp>. Some pathetic
attempts on my part to modify the code for the DU assembler
succeeded in producing a .o file w/o any error messages, but
a simple "nm -og sqrt.o" causes nm(1) to dump core. I have
a Linux box waiting to be assembled, but am trying to get some
programming done first.

Reducing the cycle count per result by even a little bit would
be of great value to me. The 12 cycles quoted by Goto for his
code would be staggering. Using Linux would be even better :-)
