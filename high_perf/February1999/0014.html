Hi,

Seems there are a number of us interested in real-life performace
of square root calculations.

I'm doing molecular dynamics on biological systems consisting of
10000-100000 atoms. The computational heavy part is the inner
loops (written in fortran of course) where I have to calculate the
inverse square root of the distance between pairs of particles.

The problem is reduced by using neighbourlists for each atom, but
this might trash the cache. We're currently using a fortran
implementation
of a table look-up & iteration routine directly calculating 1/sqrt(r).
The sqrt routine is still taking about 25% of the total runtime.
However, we're not using any vectorized version... There might
be something to win by not having to reload the table.

I've got access to both a DU box and a linux one and will be happy
both trying to contribute and test any new ideas... I'll send a piece
of code tomorrow if anyone's interested!

Regards,

Erik
