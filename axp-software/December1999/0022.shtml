On Wed, 15 Dec 1999, Raffaele Cicchese wrote:

> Hi Lee,
> I ended up this situation:
>
> 1) I made a cd-rom with the iso-image downloaded from:
> ftp://ftp.alphalinux.org/redhat/
>
> 2) I made the two boot floopy disk from an intel linux this way:
> dd if=vmlinux.gz.2.2.10 of=/dev/fd0 bs=1024
> dd if=ramdisk.img of=/dev/fd0 bs=1024
>
> 3) I boot the Alpha from SRM whit this command:
> boot dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1"
>
> 4) The istallation goes fine until the last screen shot when a message
> say the system is rebooting ... at this time the macine blocks and I
> need to shut it down.

OK, what's on the screen at this point? I don't know where you got that
install kernel (Compaq ?) but it maybe that it can't reset the machine.
This shouldn't be a too much of a problem.
 
> 5) Where is the problem?
> My idea is that the iso-image I downloaded does not contain the right
> kernel (vmlinux.gz.2.2.10) and the system block at reboot (it looks for
> the right kernel and don't find it).

Have you tried restarting the machine at this point? Hit the reset button
and try to boot using the harddisk as the root filesystem? You'll probably
need to use the floppy for the kernel as aboot won't be setup on the
harddisk yet.
