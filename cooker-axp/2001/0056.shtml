On Thursday 08 November 2001 11:12 am, you wrote:
> I subscribed yesterday to this list, to see if I could install Cooker
> on my old Alpha.

Welcome. You will find that it is very quiet and peaceful here :-)

> My alpha axp has only 16 Mb ram.
> Is that enough to try Cooker on, or should I better go with Mdk 7.1

I don't think that it makes any difference. 16 MB isn't very much at all,
but you might squeeze something in if you don't want to run X. What sort of
machine exactly do you have?

> then, and upgrade to Cooker once it's running.

The thing you need to grasp is that since 7.1 there hasn't been a working
install suit for Mandrake Alpha. You should be able to install 7.1 OK, but
upgrading to Cooker involves replacing packages by hand and it is mighty
difficult to do this without breaking your working 7.1.

What I did was to install a minimal 7.1 on one hard disc and then spend many
happy hours downloading Cooker and installing it onto another drive with
creative use of the chroot command. It did work, but by God it was a
character building experience!!!

If the installer is now working and I am talking drivel please stop me.

> I'm planning on an nfs install here.

Good Luck!

> According to this message I could best wait for the mirrors to update
> with this kernel:

I think that what Stefan and Jeff have done here is to get a kernel with all
the Mandrake patches working OK. This is something I don't bother with since
I like to stay on the bleading edge and applying Linus's latest 2.4.Xprex
patch against a heavily patched distribution kernel just causes unnecessary
pain so I just stick with the pure untainted Linus kernel (with an LVM patch)
which seems to work just fine. I guess that I don't get things like XFS, but
I manage to live quite happily without.

Have fun.
