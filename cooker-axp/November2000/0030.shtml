George Hamer wrote:
>
> I'm having trouble installing on a DEC Personal Workstation 500a using
> srm. The
> install goes great and all hardware seems to be detected correctly and
> it
> terminates the install successfully and there the trouble starts. When
> I
> attempt to restart the system using "boot DKC0 -fl 0" it starts up and
> very
> quickly I get the message:
>
> sh: error in loading shared libraries:libreadline.so.4.1 cannot open
> shared object file: no such file or directory
which version are you using? the 7.1 based iso's? Perhaps it was built
in the time when mandrake moved libreadline from /usr/lib to /lib.
Perhaps you can restart the install (until you get a usefull shell where
you can access the disk) and then move the files over manually. On my
alpha (running nearly up2date cooker, interested in the packages??) I've
got the following:

[stefan@alpha /lib]$ ls libread*
libreadline.so@ libreadline.so.4@ libreadline.so.4.1*
[stefan@alpha /lib]$ ls libread* -l
lrwxrwxrwx 1 root root 18 Jun 12 04:47 libreadline.so
-> libreadline.so.4.1*
lrwxrwxrwx 1 root root 18 Jul 1 10:16 libreadline.so.4
-> libreadline.so.4.1*
-rwxr-xr-x 1 root root 246664 Aug 30 08:42
libreadline.so.4.1*
[stefan@alpha /lib]$ pwd
/lib

[stefan@alpha lib]$ ls libread* -l
-rw-r--r-- 1 root root 617060 Aug 30 08:42 libreadline.a
lrwxrwxrwx 1 root root 24 Jun 13 04:15 libreadline.so
-> ../../lib/libreadline.so*
[stefan@alpha lib]$ pwd
/usr/lib

Hope this helps.
