Wolfgang Dreyer writes:
>
> at this time NEVER use partitions bigger than 2 GB (trie always to stay
> below this value)

May I ask why? Boot partitions cannot be bigger than that, but on
Linux/Alpha personally I created, mounted and used (on a short term
basis, unfortunately) partitions of order of 8 Gig. The biggest
one I heard reported in use was 54 Gig but this was on Linux/Intel,
this is a second hand knowledge, and I never had enough of disk on
hand to try something like that. None of these was on IDE drives,
AFAIK.

It is true that trying to make such big partitions from Red Hat
installation menu may run you into troubles; it is also more likely with
kernel 2.0.30, as it showed up in RH 4.2, then with previous kernels to
run into bugs in a virtual memory handling when running mke2fs. But none
of these obstacles cannot be worked around with a bit of knowledge.

Never say NEVER.

Michal
