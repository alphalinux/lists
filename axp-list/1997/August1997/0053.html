I'm trying to do an ftp install of RedHat 4.2 onto a UDB which up until
recently has had 4.0 installed on it. I decided to do a complete new
install as I thought it might be easier...

The system already has MILO installed (built against 2.0.18) and I've got
new noname and ramdisk floppies. I start milo and insert the floppies at
the appropriate times and get into the RedHat install. I setup to talk to
my ftp server and things go swimmingly. I get to the stage where it detects
the two disks (a 540Mb internal which I use for NT) and a 1Gb external.

The external disk is already partitioned with 3 partitions which I plan to
use for /, swap and /usr. (/dev/sdb1, /dev/sdb2 and /dev/sdb3
respectively). I create the swap partition and then in the mount disks
dialogue I setup sdb1 as /, sdb3 as /usr and /dev/sda1 as /dos. Filesystems
are created on sdb1 and sdb3 and then the install fails with "invalid mount
point". I thought this might have been related to the /dos partition so I
tried again without specifying this, but to no avail.

I've never had this problem before in installing 4.2 on my XL300, so I
guess I'm asking to be told what important thing I forgot :-) I've even
checked the UDB installation HOWTO and reread the RedHat Installation Guide
- things are getting desperate! The real problem is that I promised that
this system would be up tomorrow to allow a number of other users to
install 4.2 on their new XL300s - me and my big mouth.
