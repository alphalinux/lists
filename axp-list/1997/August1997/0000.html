Hi, GCC gurus,
I would like to report the following problem with GCC on the alpha,
which manifests at link time when I go over some invisible internal limit
on the number/size of binaries to link together. There is no problem with the
code on the Pentium or 486 platforms, also running Linux.

Symptom: If I try to link too many binaries together (over about 12, totalling
about 3.7 Meg) I get a huge number of "relocation truncated to fit" messages.
Under this limit of 13 binaries, I can compile and run the code. If I add
a 13th library it fails to link. If I now remove a different library, so I
am back to 12 again, it compiles, links and runs.

I would greatly appreciate any hints on what to do to get my code to work on
this machine, which I specifically bought to run it.
