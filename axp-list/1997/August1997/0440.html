On Thu, 21 Aug 1997, Roman Martonak wrote:

> gcc -D__KERNEL__ -I/usr/src/linux/include -Wall -Wstrict-prototypes -g -O2
> -pipe -mno-fp-regs -c FlashPoint.c -o FlashPoint.O
> cpp: output pipe has been closed
> gcc: Internal compiler error: program cc1 got fatal signal 6
>
> Does anybody have an idea what might be wrong ?

Try lowering the level of optimization. First, change -O2 for -O and if
that doesn't work, try omitting it. There are some bugs in the current gcc
(particulary in the optimizer).
