Hi all,

Due to some problems with the clock on the PC164-LX (aka LX164), and some
NCR875-based SCSI controllers, there's a new set of MILO sources located at:

gatekeeper.dec.com:/pub/DEC/Linux-Alpha/Kernels/jays-milo-2.0.30-06aug97.tgz

These are *temporarily* the latest sources, and supercede the ones located
in the standard location (Miniloader/v2.0/src/milo-2.0.30.tar.gz). They fix
the above-mentioned problems (the SCSI problem could occur on any Alpha).
As before, these MILO sources are happiest being built against kernel sources
linux-2.0.30 that have been patched with alpha-patches-2.0.30-0.1 (available
in the same Kernels/ directory).

Also on gatekeeper in the Kernels/ directory are the following binaries:

milo-2031-pc164 # PC164 MILO with NCR875 support
milo-2031-lx164 # LX164 MILO with NCR875 support

These are based on kernel sources linux-2.0.30 plus alpha-patches-2.0.30-0.1
plus pre-patch-2.0.31-2 (from Dave Miller, available on www.linuxhq.com).
The latter patches were used primarily to bring the NCR53C8XX driver up to
revision level 1.18i, which includes working support for the NCR875 SCSI
chip.

So, if you've had any problems with the clock running at half speed on
LX164, or flakiness when running NCR875-based SCSI controllers (for example,
Intraserver's), these MILOs will help (if you have either of those two
platforms). If you have a different platform, I'm afraid you'll have to
roll-your-own, but that should no longer be as big a production as it has
been in the past.

As always, please report *any* problems (or kudos :-) back to me.

Good luck!
