I was about to flash MILO on my UDB the other day, but Maddog suggested that
I take another approach.
Set up a small (1 or 2MB) MSDOS partition, and copy loadlin.exe and MILO to
it.

Then, from the ARC console, set up the boot selections:
>From the boot menu, go to the Supplementary menu/Set up the system/Manage
Boot selection menu.
Select Add or Change a boot selection.
Set the variables as appropriate:
SYSTEMPARTITION=scsi(0)disk(0)rdisk(0)partition(n)
Where n is the partition number where loadlin and milo are.
OSLOADER=scsi(0)disk(0)rdisk(0)partition(n)\linload.exe
OSLOADPARTITION=scsi(0)disk(0)rdisk(0)partition(n)
OSLOADFILENAME=\
OSLOADOPTIONS=boot sdax:vmlinux.gz root=/dev/sdax

The osload partition does not need to be the first partition. So, even on an
existing system, you can shrink one partition and insert or add a new
partition. I used arcinst.exe to format the partition, but you can also copy
the milo dd image. When I copied the dd image, I did something to the
partition table. Rather than try to figure out what I screwed up, I grabbed
arcinst.exe from the NT public archive.

One of the nicer things with ARC is that you can have multiple boot
selections. I have one for normal multi-user, 1 for single user, and 1 for
MILO.
