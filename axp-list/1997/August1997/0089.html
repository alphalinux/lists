> I just noticed that it does not seem to be possible to include <sys/ipc.h>
> into a C--File that you want to compile with gcc's "-ansi" option.

Except that -ansi is intended to denote a strict ISO C environment, that
is, in the headers you only get what ISO says should be in them. But
<sys/ipc.h> is not an ISO header, so you can not expect that to work.

Why were you wanting to use -ansi anyway?
