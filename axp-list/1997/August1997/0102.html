Actually, I ran into the same problem with fdesign as it came with version
0.81 of XForms. -- The fdesign binary that comes with version 0.86 of
XForms works fine on my box (UDB running RHL 4.2).

If you're looking for a GUI lib to write free software with, may I
suggest having a look at Qt too... -- Qt 1.2 compiles and runs straight
out of the box on Linux/Alpha. Troll Tech does not (yet) provide a GUI
builder for Qt, but there is "dlgedit", a GPL'ed dialog editor for Qt that
is quite useable.

dlgedit is also available from Troll Tech's ftp server (ftp.troll.no).

I've done a bit GUI Programming under NeXTStep some time ago and Qt's
signal--slot mechanism somehow reminds me to NeXT's concept of event
handling (which I liked, but of course YMMV...).

hope this helps
