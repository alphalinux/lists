>>> Huw Davies said:
>
> 2) I built a new kernel 2.0.30 with the latest greatest patches from
> gatekeeper and it all worked well except that the PCI memory address of my
> video card changed. Is this expected, has anyone else seen this???

Yes, and it was expected (at least by me :-).

Sorry I didn't mention that as a side-effect of using those patches; it
became necessary to rework the start addresses and allocation of PCI bus
memory and I/O space to work around some (unrelated to video cards) problems
found recently.

Most times this would be transparent to the user, but I forgot that there
might still be people using the older TGA server that requires a MemBase
option. The newer TGA servers from XFree86, starting with 3.2, no longer
needed such a configuration option.

Again, sorry if this has caused any problems, and thanks for bringing it up,
as I'm sure others may run into this as well...
