Ryan Kirkpatrick <rkirkpat@nag.cs.colorado.edu> wrote:
> I assume that would be a SCSI zip drive? I have tried (not very
> hard) and failed to get my Parallel port ZIP drive to work with
> Linux/Alpha. I would assume that it is not possible, due to the fact it
> was designed for Intel.

There's no technical reason why a Parallel ZIP couldn't be made to work
on an Alpha machine that has a PC-style parallel port (like the UDB).
However, it might be the case that the PPA3 driver doesn't currently
support the Alpha.

> It is possible to boot a Linux kernel from the ARC
> console with out MILO?

No.

> If the hard disk crashes, then one no longer has
> milo on the hard disk present to boot a linux kernel. Would one need a
> floppy with milo on it, boot that from the ARC console?

Just put a small FAT partition on the ZIP, and put Milo on that. It works
just like booting from the internal hard disk, except that you have to
add a different entry to the ARC console's boot list.

PC-formatted ZIP disks just have one FAT partition, which for some strange
reason is partition 4. However, you can FDISK it and set up a normal
Linux partition table.

I have 88M in my UDB, so I don't really need any swap space for my recovery
disk. Thus, with a little bit of care I was able to set it up so that I
could write-protect the ZIP.
