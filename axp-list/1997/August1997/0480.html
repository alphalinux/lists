On Aug 26, RHS Linux User wrote:

> i can't seem to remember what i did to make this work before...
> anyone have any ideas??

installing the X fonts and/or running "mkfontdir" on the font directories
like this (as root):

mkfontdir /usr/X11R6/lib/X11/fonts/misc
mkfontdir /usr/X11R6/lib/X11/fonts/75dpi
mkfontdir /usr/X11R6/lib/X11/fonts/100dpi

> (**) FontPath set to "/usr/X11R6/lib/X11/fonts/misc/,/usr/X11R6/lib/X11/fonts/Type1/,/usr/X11R6/lib/X11/fonts/Speedo/,/usr/X11R6/lib/X11/fonts/75dpi/,/usr/X11R6/lib/X11/fonts/100dpi/"

> failed to set default font path '/usr/X11R6/lib/X11/fonts/misc/,/usr/X11R6/lib/X11/fonts/Type1/,/usr/X11R6/lib/X11/fonts/Speedo/,/usr/X11R6/lib/X11/fonts/75dpi/,/usr/X11R6/lib/X11/fonts/100dpi/'
> Fatal server error:
> could not open default font 'fixed'

XFree86 3.1.2 *very* ancient, I seem to remember that there was a bug in some
very old versions that setting the font path didn't work if not all directories
did exist and contained valid "fonts.dir" files.

a) get XFree86 3.3.1 !
b) comment out all nonexisting or empty font dirs in you X config file...
