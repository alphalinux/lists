On Tue, 5 Aug 1997, Richard Henderson wrote:

> > I just noticed that it does not seem to be possible to include <sys/ipc.h>
> > into a C--File that you want to compile with gcc's "-ansi" option.
>
> Except that -ansi is intended to denote a strict ISO C environment, that
> is, in the headers you only get what ISO says should be in them. But
> <sys/ipc.h> is not an ISO header, so you can not expect that to work.
>
> Why were you wanting to use -ansi anyway?

Thanks for the reply! -- Actually, it wasn't me who originally decided to
use "-ansi"... ;-) -- I ran into the problem when I tried to compile
Mesa-2.1 (a GPL-ed Op*nGL lookalike) on my Alpha.

BTW: When compiling Mesa-2.1 I also got one of the "internal compiler
errors". The file was "src/readpix.c" -- Using "-O" instead of "-O2" did
not fix the problem, I had to compile the file without any optimization.

Richard, if you are interested I could try to generate a small C--File
that still reproduces the problem and mail it to you. -- gcc is 2.7.2.1 as
shipped with RHL 4.2.

Anyway, the OpenGL demos that ship with Mesa-2.1 are now happily running
on my Linux/Alpha box...

thanks for your help!
