Hi,

Newbie here, with my first posting. I've a hunch this has been
covered, but I haven't been able to find the answer looking at the
mail archives.

I'm running Redhat 4.2 on a Microway 500Mhz Alpha. RH4.2 doesn't come
with ypbind, it's supposed to be built into glibc, but I'll be
dog-gonned if I can figure out how to turn it on. Commands like "ypcat
hosts" return:

No such map hosts.byaddr. Reason: Can't bind to server which
serves this domain

I've set up /etc/nsswitch with lines like:

hosts: files nis dns

And /etc/yp.conf has in it:

domain hip.atr.co.jp
ypserver rokkoh.hip.atr.co.jp

which would seem to me to be correct.

I've filed a problem report with Redhat, but they say that this is not
covered, since it's not an installation problem (though they say it
works "out of the box" if everything is properly configured ...?)

I've also tried using ypbind, which I've gotten from:

>With H-J's gracious permission, we are making it available via
>anonymous ftp:
> ftp://www.dsm.fordham.edu/~pub/misc/ypbind-linux-alpha.gz

I realize this is for RH4.1, but after starting it NIS clients
subsequently start crashing with:

nfs_stat_to_errno: bad nfs status return value: 35
ypwhich(3443); unaligned trap at 00...0001f: 0000011ffff7dc 23 1
crond: memory violation at pc=155556c6c58 rp=155556e62510 (bad address = 616d006f6b696873)

and login no longer works. I have to reboot my machine without it.

So, my brief question, nay plea: how to get NIS clients working under
Redhat 4.2? And how can I quickly check if NIS clients work, is "ypcat
hosts" the best test command?

Thanks for any help!

Sincerely,
Rick Woudenberg
