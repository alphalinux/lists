On Aug 13, Brian Lund wrote:

> Regarding: xfree86 with S3-Trio64V+ - problems running in more than 8bpp.
> XFree86 Version: 3.3.1 (happens under all version of Xfree86 that I have tried)
> Server: XF86_S3
> S3-Trio64V+ Rev 54, RamDac: s3_trio64
>
> The machine is an alpha (21066A 233Mhz in a AXPpci 33 board)
>
> When ever I try to run X in >8bpp it crashes after some time,
> The crash happens sooner if I run program like nxterm, irc-clients etc, if I just run xlock it can run for hours without crashing.
> Before the crash I get some errors in my syslog, after about 10 of those, the machine hangs totally (doesn't respond to anything, ping, telnet etc)
> The errors look like this:
> Aug 12 14:32:49 caffeine kernel: lca: machine check (la=0xfffffc00002084d0,pc=0x120108838)
> Aug 12 14:32:49 caffeine kernel: Reason: access to non-existent memory (longframe):
> 4:32:49 caffeine kernel: car: 6150dc51
> 4:32:49 caffeine kernel: CPU initiated PCI Memory Read cycle toaddress 14095398 failed due to no device.
> Then a lot of numbers/addresses.
> Option "no_pci_disconnect" didn't change anything.

I know about such problems but I never have been able to reproduce/see them myself
and I have no idea what's wrong here. my guess is still that this is a hardware
problem with DEC chipset for this mainboard but AFAIK no one really knows.

some people already debugged/traces these problems and showed that these
machine checks and lockups happen for perfectly valid frame buffer memory accesses,
so it's some weird hardware (timing) stuff...

Harald
