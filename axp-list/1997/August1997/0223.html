Sometime last week (time flies when you're having fun :-) I posted a
question about problems I was having installing RedHat 4.2 on my UDB. I
eventually solved the problem by changing the way I'd partitioned the disk.
I don't know whether it was just rewriting the partition table or changing
the layout that fixed the problem, but my 1Gb disk now has two partitions
(swap, /) rather than three (/, swap, /usr).

OK, onto a couple of other little problems :-)

1) I want to use lpd to allow various systems to print to the DJ400
attached to the UDB. I got the latest rpm (0.19) but I still get memory
violations from lpd. A quick search of the web indicates that this is a
known problem and that the only solution suggested was to use the lpd from
the 3.0.3 distribution. I can't find a copy but before looking into the
code (and fixing it) I thought I'd check to see if anyone else had already
done this.

2) I built a new kernel 2.0.30 with the latest greatest patches from
gatekeeper and it all worked well except that the PCI memory address of my
video card changed. Is this expected, has anyone else seen this???
