Hi, everybody,

I don't know if you were following this problem, but the status now
is that the driver, in the kernel level, is already working for all
boards (PCI and ISA, Cyclom-Y and Cyclades-Z).

My problem now is with the Cyclades-Z firmware loader, which loads
the firmware file in the Cyclades-Z on-board memory, in order to
allow the IDT (the Cyclades-Z on-board processor) to run.

Since this is an application, I can't use the kernel functions writel,
readl, writeb, readb (which are the functions that solved the problem
in the kernel level). As far as I know, the applications should not
notice the difference in the platform, since the kernel is changed to
accomodate these diferences.

In the Intel platform, I just use mmap to map the memory and control
windows (which give me access to the Cyclades-Z hardware, more
specifically to the PCI bus master interface chip registers and to the
on-board RAM) and, after that, I can access the memory as I was
accessing local RAM.

In the Alpha, this is not working. I'm not able to neither read the
hardware registers nor write to the on-board RAM.

Do you guys have any idea on how to map a PCI device in Alpha, in
application level ??

Any help would be highly appreciated.
