After installing a BusLogic BT958 SCSI controller, linux seems to be
unwilling to boot from an IDE disk. When I try to boot the machine:

-MILO reports that it is unable to determine the SCSI host ID.
It reports this when MILO is first loaded, and subsequently if
any attempt is made to set SCSI0_HOSTID manually.
-MILO will load the kernel from /dev/hda2
-Everything will progress normally until the kernel messages report
that the BusLogic driver has been initialised (immediately after
reporting `FDC 0 is a post-1991 82077', when it should in
theory be reporting the number of SCSI hosts detected), at which
point it gets wedged.

The system in question is a PC164 running 2.0.27, with (as far as I
can tell) all the appropriate options compiled in (i.e., support
for BusLogic SCSI cards). The only SCSI device I've had connected
to the BT958 is an Exabyte 8200, and it exhibits the same behaviour
when no devices are connected to the SCSI card.

There's an active terminator at the end of the SCSI chain (when anything
is connected to the card, that is), and the card, cabling and drive
work fine when hooked up to a linux/intel machine.

Although I couldn't locate anything in the archives, I'll still ask:
Known Problem?

Also, the card has a built-in menu-driven BIOS configuration widget
that comes up at boot on an intel-based system. Is it possible
that something or other needs to be diddled in there? And if so,
are there any utilities to do that sort of thing on an alpha?

Yours etc.,

SubGenius
