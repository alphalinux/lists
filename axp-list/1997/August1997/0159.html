> I had an old milo/linload sitting in /dev/sda1. sda1 was a small
> 2 meg partition that ARC loaded linload and milo from. Since
> upgrading milo, i tried putting the new one in the same way as
> i did the old milo. It will not work. ARC says there is a problem
> with the boot entry. I'm currently booting my new milo/linload
> from a DOS FAT floppy. If i dd this floppy onto sda1 ARC
> complains.
Ahem - dd-ing would put a DOS floppy-filesystem to a HD-Partition, which
has a slightly different format (other media-ID). I suppose this is what
ARC complains about. You should have copied the files instead of using dd.
You probably have to make a dos-filesystem on sda1 again and copy the files
from the disk. I am yet unsure, if mformat supports making HD-filesystems.
