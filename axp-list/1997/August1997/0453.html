After the suggestion to try Alt+F2, I retried the install.
I tried to run fdisk from a command prompt, but no luck, the device was
either busy or unavailable.
However, switching to other virtual consoles I noticed that there were
quite a few complaints about

/proc/scsi/scsi having the wrong revision or incomplete version. Hmm....

There were no errata yet posted for 4.2, however there is one for 4.1 that
provides a new ramdisk.img that supposed to cure faulty makedev during the
install. On the hunch that this might be the problem, I dld and made the
new ramdisk.img (There is place in the world for dumb Windoz boxes....)

This seems to have fixed the problem, the install is currently running.

the package for XF_86 Install or XF_86 Config even though I selected all
the X options. Again I am using the Linux PowerTools 6 CDROM disk. from
RH.
