The confusion here is the origina of the clibs.  I don't remember what the
history of the Intel C-Libs is, but they are not produced by GNU.  The alpha
is using the gnu (cygnus?) clibs and thus the different version numbers.
So, you do want the so called glibc libs for the alpha.

I can't check from my dialup at home as easily, but I think most of
the alpha stuff you need is in binary form Red Hat's contrib directory.  
Check


ftp.redhat.com:/pub/contrib/alpha/
or
ftp.infomagic.com:/pub/mirrors/linux/RedHatContrib/alpha


For modutils, I recommend the 2.1.43.tar.gz version found in either
of the mirrors.  


Oh, your should also check the SRPMS directory for rpm'd source
packages that will probably ease your life a bit as well if there isn't
a binary package available.  I used, for example, ld.so.1.8.4 for
the source rpm.


Also, for these kernels, more recent versions that described in the
kernel documentation is probably not a bad thing.
