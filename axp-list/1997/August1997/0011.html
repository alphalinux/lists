Hi,

I'm changing the Cyclades Linux driver in order to work on the Alpha
platform. I'm using a Alpha PC164 366Mhz with Linux RedHat 4.2
(Biltmore).

I could put the ISA boards to work fine under kernel 2.0.30, but when
I have the Ethernet card in the machine, it does not complete the
boot, halting as follows:

Enabling swap space.
Initializing random number generator...
INIT: Entering runlevel: 3
eth0: enabling 10TP port.
CIA machine check: reason for machine-check unknown (0x311508)
+ 0 0000000000000008 0000000000000010

The normal boot messages (without the Cyclades driver included in the
kernel) at this point are as follows:

Enabling swap space.
Initializing random number generator...
INIT: Entering runlevel: 3
eth0: media is TP.
Mounting remote filesystems.
(...)

First of all, I have no idea what CIA stands for (obviously it's not
the Intelligence ...). It looks like a hardware conflict being
trapped by the kernel, because if I remove the Ethernet card from
the machine, it works just fine.

Also, if I have the Ethernet card in the machine and the Cyclades
driver is built in the kernel, even if I have *no* Cyclades cards in
the PC, the crash happens.

So ... any ideas ???

Thanks in advance.
