We are running RedHat 4.2 on a DCG PC164 500Mhz SCSI Alpha machine.

We are having scattered reports of problems with pine 3.95. I have
found very little on this subject, except that it seems like noone has
been able compile pine under alpha Linux using "build lnx". I am wondering
if anyone has had this problem or knows of any fixes. ALSO, I'm wondering
if the redhat 3.0.3 or 4.0 or 4.1 full ALPHA distributions are online
anywhere?

I have gotten the reports second hand, but it sounds almost like
terminal emulation probelms with pine 3.95. When someone goes to compose,
then hits ctrl-t (to addressbooks) the screen gets messed up, with the
addressbook screen half on the screen. These problems happen sometimes on
some telnet clients (trumptelnet, EWan, procom) but seems to be ALMOST
fine all the time via linux to linux. with term=linux, from another linux
box, when I go back after choosing an address, I get an extra left over
line from the "key help menu"at the bottom of the screen, so now there are
three lines with keys I can use, the bottom one is wrong...

Could pine 3.93 help this? Thank you in advance.
