Jacques Leisy <jleisy@bw.webex.net> writes:
> It looks like the Axp 150 doesn't like using an internal modem (ISA)
> in an EISA slot.
>
> My previous problem (?06 DBL MCHK crashes ) is gone since I replaced
> the internal modem by an external one using the MB serial ports

Hm, that's no good solution either, because the internal serial ports
are 16450 compatible, not 16550, which means they don't supply a
buffer, so connections above 9600 may get huge error rates.

I use an additional 16550 FIFO serial card with no problems. I can't
see an reason for internal modems to fail because of the Jensen
architecture.

OTOH, do you use an S3 card? I heard that S3 configuration registers
use the same addresses as internal modems, commonly refered to COM4, I
believe. So maybe you need to reconfigure your modem, to use other
addresses. Hmm, hopefully it isn't a PnP modem, or is it?

> Maybe this is due to the fact I didn't run the ECU configuration (I don't
> know where to get the software from!)

Hmm, I guess that's unlikely. You probably got an ISA card, and the
ECU just makes sure, that you don't use the same resource for two
cards. If you do a careful setup, and you do know the resources of the
cards in use, you don't need the ECU for ISA cards.

You may try to use the proc file system to find out, which resource
are already in use.

> Too bad it is so difficult to find **any** documentation on this machine.

Yes, that's true. I found out most stuff by reading the kernel source,
especially <asm/jensen.h>.
