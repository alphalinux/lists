After talking some with Jay Estabrook I got the machine booting with
the card installed. I hadn't posted a fix yet because I don't properly
have one yet. What I did was something like you suggest above:

-Removed the card
-Rebooted the system
-Built a kernel with SCSI support but without support for the BusLogic
card. Included the driver for a card I didn't have, because the
subsequent MILO compile with fail with no SCSI drivers enabled
-Made MILO
-Rebuilt the kernel with both SCSI support and support for the
BusLogic card
-Installed both MILO and the new kernel
-Rebooted

Interestingly, although both the kernel and MILO should be using the
same drivers, whenever I attempt to build MILO with the same BusLogic
driver that works in the kernel, MILO will get wedged upon booting
as I described. Booting with a BusLogicless MILO to a BusLogic-savvy
kernel works without any hiccups.

Since I'm currently booting the machine from an IDE drive and so
can initialise the SCSI devices post-MILO in the kernel, this works
for Your Humble Narrator for the time being. Seeing as I've got
a SCSI hard drive coming in in roughly (three days + [UPS strike constant]),
I'm rather interested in getting MILO talking to the BusLogic card.
