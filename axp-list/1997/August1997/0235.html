On Aug 12, Thaddeus A. Wilson wrote:

> ** we now have a more current issue pertaining to the stealth graphics
> card Trio64 and our X server
>
> every time we try to run startx we get hung up. jay had suggested adding
> the following "option" to the "device" section of the /etc/X11/XF86Config:
>
> Option "no_pci_disconnect"
>
> however, this is not recognized as a valid option.

this option is new in XFree86 3.3 !

> XFree86 Version 3.2 / X Window System
> (--) S3: PCI: base address not correctly aligned or address conflict
> base address changed from 0x01800000 to 0xf8000000

XFree86 3.3.1 (just released this week) finally should avoid this one
for plain Trio64 chips...

so please get XFree86 3.3.1 !
