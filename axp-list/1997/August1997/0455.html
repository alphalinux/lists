All,

I've been having little success in building a working milo from the latest
sources on my UDB. For starters, this needs to happen in the top-level
source directory before 'objstrip' will build:

*** Makefile.orig Mon Jul 28 06:43:53 1997
--- Makefile Sat Aug 23 17:20:09 1997
***************
*** 563,568 ****
--- 563,573 ----
# The tools that we need to build the miniloader.
# ----------------------------------------------------------------------------

+ $(OBJSTRIP): $(KSRC)/arch/alpha/boot/tools/objstrip.o
+ $(CC) $(CFLAGS) -o $(OBJSTRIP) \
+ $(KSRC)/arch/alpha/boot/tools/objstrip.o \
+ $(KSRC)/arch/alpha/lib/memset.o
+
tools: $(TOOLS)

tools/bin/clist: tools/list/clist.c tools/common/commlib.a

Not sure how it ever built for anyone. The __constant_c_memset entry
point is in memset.o - which wasn't getting linked.

When I try to start the resulting 'milo' image with linload.exe, it gets
as far as "Swapping to PAL Code at 0x0" and locks up solid.

Suggestions are welcome...

Steve
