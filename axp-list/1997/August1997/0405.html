> Is there a possibility to get both, a fast disk throughput and a
> accurate system time?

Sure. By not using IDE disks (with prefixes or not).

One of problems with IDE is that these disks eat system resorces
like crazy (not only interrupts but CPU time as well). So you pay
for cheaper disks by making other components busy doing I/O instead
of a useful work. This is unimportant on a single user single-tasking
DOS (and derivatives, like '95) system, but shows up when you really
start multitask.

> Is it secure to use `cron' (maybe every 5 to 10 minutes) to
> synchronize
> the system time with CMOS time? (the differences in time are in the
> order of
> several seconds per minute, if there is intensive hard disk access)

It depends what you are doing on this machine. If it is used
practically as a single user workstation then results of doing that
should be relatively harmless.
