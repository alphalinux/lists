however I still have the pipe problem.

# > Piping (the | command in pine), Spell checking (Ctrl+T), and display
# > filters don't seem to work very well in 3.95 on my two UDB's. I tried
# > compiling 3.96 from the pine FTP site and after a edit here, patch there,
# > I still had the same problems.

# What exactly does it do when you try to use each of those features?

Well, I use this display filter:

display-filters=_BEGINNING("-----BEGIN PGP")_ /usr/bin/pgpdecode _TMPFILE_

which catches PGP signed/encrypted messages and runs a perl script which
in turn runs PGP. The filter works, *but* if I try to pipe a message that
is caught by this display filter, i end up with 'Error Piping Message.

As an example, I run nearly a dozen Majordomo mailing lists. If someone
posts to one of the moderated lists, I have to approve the message which
I do by piping the message into an 'approve' script (makes life easier).
If the message posted to the moderated list is a PGP signed/encrypted
message and I attempt to approve it, I end up with the error message. I
doesn't matter what I try to pipe to. I even tried /bin/cat with the same
luck.'

# Lastly, what kind of patches have you applied? From my brief analysis of
# the code, it shouldn't need patching really....

The only things I changed were fixing some of the incompatible pointer
warnings and also edited the os-lnx.h file. But, I didn't do anything to
the 3.96L I found.
