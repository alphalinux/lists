optimistically we seem to have solved our problem with a suggestion made
by jay. by downloading the current milo for pc164 i.e. ver 2.0.31 we were
able to start the machine with the new ncr53c8xx driver at ver. level
1.18i rather than 1.12d which we formerly had on milo 2.0.30. we still
need to get a current kernal built with this newer driver, but our system
still seems be be able to force a disk check following and unclean
dismounting of the drives.

** we now have a more current issue pertaining to the stealth graphics
card Trio64 and our X server

every time we try to run startx we get hung up. jay had suggested adding
the following "option" to the "device" section of the /etc/X11/XF86Config:

Option "no_pci_disconnect"

however, this is not recognized as a valid option. We also cannot use the
XF86Setup command, but by using either Xconfigure or XF86Configure we can
set up our system.

Diamond Stealth 64 S3-Trio64 Chipset PCI 2MB
