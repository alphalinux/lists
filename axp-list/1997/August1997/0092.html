All,
I have something strange happening with ip aliasing that I can't figure out.
I've tried everything and have resolved it must be something in the kernel.

I assign an IP address to my alpha, say XXX.yyy.zzz.80, then alias 81 82 and
83. From the outside world, I can ping, telnet to, ftp to, httpd to any of the
IP address with no problem. I have a nameserver running on the machine and I
can only use the 80 (real ethernet) address in resolv.conf on other machines on
the same network that don't have named running. If I try any of the alias addres
s it hang and will not resolv.

I moved the nameserver over to my intel box, did the same thing with the
aliasing and it work perfectly, I can use any of the alias address for the
nameserver on my remote machines and they work.

I am using kernel 2.0.29 on both machines. I tried several other kernels on the
alpha, including 2.1.47, but it still won't work.

Any help or confirmation would be appreciated.
