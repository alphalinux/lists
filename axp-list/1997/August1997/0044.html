After doing a full install of the RH4.1 distribution (kernel version
2.0.27), the first thing I went to do was reconfigure the kernel to support
my ZIP drive as a module. I have spent the last week pulling my hair out
trying to figure out what I've done wrong. (I've done this before on an
Intel machine with no problems)

First problem: After rebooting with the new kernel, I do a insmod ppa and
get a file not found error. (Might be a problem with my MODPATH?)

Second problem: try giving the command with the full path to ppa and get an
error saying that the module was not linked properly ("did you use -dc when
linking")

Third: Went to Red Hat's website and read some of the tech notes and help
docs. One of them said to do a "depmod -a" after "make modules_install".
Tried it and found that the depmod binary was not on the system. So I
installed the modules-2.0.0 source and tried to recompile the programs and
got more errors than I would care to list.

Now I've decided to swallow my pride and ask for help.

Any and all help is greatly appreciated.
