>>> Les Weber said:
>
> We're trying to get redhat linux on a avanti 200 with 64 Mb of ram and 2
> scsi hard disks. We're trying to boot from the cd-rom,version as of a
> month ago. We use the command -
>
> boot dka2 -file /images/avanti-s.gz
>
> It seems to work and then we get a kernel panic. Is it trying to mount
> the cd-rom as a writable volume? If so how do we boot from the cd so that
> it uses one of the hard drives?
>
> When we try the boot the last x lines from the console are ->
>
> VFS: Cannot open root device 08:02
> Kernel panic: VFS: Unable to mount root fs on 08:02

1. you'll need to make the RAMdisk floppy from the file on the CD, and
tell the kernel to use it via:

boot dka2 -file /images/avanti-s.gz -flags "load_ramdisk=1 root=/dev/fd0"

The error message above is because the kernel defaults to /dev/sda2 for
root unless you tell it different, and you didn't tell it different.

2. at the end of the install, the scripts will ask you to insert the kernel
floppy; if you don't, the install doesn't finish, so you might as well
make that one as well before you start.

3. booting the kernel from the CD does usually work, but it very likely won't
work to try to mount it as root.

Good luck.
