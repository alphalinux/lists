>>> James Jin-shea Kuo said:
>
> I've managed a AlphaServer 1000A running NT. It's with very poor in
> performance throughput. Is it possible to run Linux for Alpha on this server?
>
> I visited RedHat homepages and found RedHat don't support Linux for Alpha
> for this machine type. I've hearded that DEC support MILO loader. Is MILO
> available for any machine with Alpha chip like AlphaServer 1000/1000A?
>
> Would someone out there give me some ideas ?

The 1000A is known internally as NORITAKE; it can have either EV4 or EV5
based CPU daughtercards; which does your have?

NORITAKE is a close cousin to MIKASA, but there are enough differences so
that a MILO or kernel for one will most likely not work for the other, so
don't bother to try the MIKASA stuff that's available from Red Hat.

I believe that there are some kernels available on gatekeeper.dec.com, for
booting via SRM console on NORITAKE with either EV4 or EV5 CPUs; look for
files in the /pub/DEC/Linux-Alpha/Kernels/ directory. Note that you will
have to change from ARC console to SRM console to use them; most of your
machine class come with both ARC and SRM, but some may not. You can find
the appropriate FW updates on gatekeeper as well, in:

pub/DEC/Alpha/firmware

or somewhere below there.

Good luck.
