I am not an expert by any means but I have a similar system (Avanti 400)
running Redhat 4.2 happily for a while. First, the MILO image for Avanti
was not shipped with the distribution and I had all kinds of problems
trying to boot up the kernel. I downloaded the appropriate MILO image from
DEC's FTP site. I built a boot floppy instead of trying to boot from the
CDROM during installation. You may want to try it out and see if that helps.
(It is very possible that MILO does not suppport the CDROM device).
Once the system is installed, Redhat will copy the boot image directly to
the / (root) directory and from then onward will be able to boot from the
hard disk.
Also, in the boot coomand line, you have to specify a root device. This is
what is, most likely, causing the panic. I use the following command at the
MILO boot prompt :

boot sdb2:vmlinux.gz root=/dev/sdb2

Hope this helps.
