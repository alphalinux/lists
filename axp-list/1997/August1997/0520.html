On Tue, 26 Aug 1997, Les Weber wrote:

| We're trying to get redhat linux on a avanti 200 with 64 Mb of ram and 2
| scsi hard disks. We're trying to boot from the cd-rom,version as of a
| month ago. We use the command -
|
| boot dka2 -file /images/avanti-s.gz
|
| It seems to work and then we get a kernel panic. Is it trying to mount
| the cd-rom as a writable volume? If so how do we boot from the cd so that
| it uses one of the hard drives?
|
| When we try the boot the last x lines from the console are ->
|
| Ramdisk driver initialized : 16 ramdisks of 4096K size
| Floppy drive(s): fd0 is 2.88M
| Started kswapd v 1.4.2.2
| FDC 0 is a National Semiconductor PC87306
| ncr53c8xx : at PCI bus 0, device 6, function 0
| ncr_attach: unit=0 chip=810 base=11001000, io_port=b400, irq=11
| ncr53c810-0: assu;ming 40 MHz clock
| ncr53c810-0: initial value of SCNTL3 = 03, final = 13
| ncr53c810-0: requesting shared irq 11(dev_id=0xfffffc000015c0a8)
| ncr53c810-0: restarting (scsi reset)
| scsi0 : 1 host.
| Vendor: TOSHIBA Model: CD-ROM XM-3501TA Rev: 3384
| Type: CD-ROM ANSI SCSI revision: 02
| Detected scsi CD-ROM sr0 at scsi0, id 6, lun 0
-----------------------^^^
| scsi : detected 1 SCSI cdrom total.
| PPP: version 2.2.0 (dynamic channel allocation)
| TCP Compression code copyright 1989 Regents of the University of
| California
| PPP Dynamic channel allocation code copyright 1995 Caldera, Inc.
| PPP line discipline registered.
| eth0: DE434/5 at 0xb0000 (PCI bus 0, device 11), h/w address
| 08:00:2b:e6:11:1a, and requires IRQ15 (provided by PCI BIOS).
| VFS: Cannot open root device 08:02
| Kernel panic: VFS: Unable to mount root fs on 08:02
|
| This is for a non-profit which is promoting a good cause.

It is apparently trying to mount the major 08, minor 02 as root fs. What
you need is to mount /dev/sr0 as root device, by appending a
"root=/dev/sr0" to your boot command line.

Hope this helps,
