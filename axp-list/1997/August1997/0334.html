In linux.redhat.axp gb141@columbia.edu writes:

| It is my understanding that SPECfp_rate95 is the only one of the
| SPEC benchmarks that can compare a multi-cpu system to a single
| cpu system. Is that correct? Can the above result be interpreted
| as an ~70% performance increase of the Alpha system over the dual
| Pentium II under this particular set of benchmarks?

SPECrate runs several independent copies of the SPEC benchmarks.
It's not the only way to compare multi-cpu and single-cpu systems,
but it is the way that makes the multi-cpu system look best.
It would be appropriate, for example, for a multi-user system that's
always heavily loaded, so there are always threads that want to run.

Another way to compare them is to compare uniprocessor performance,
and guess or measure how utilized the extra cpu(s) will be, and use
SPECrate to gauge how much extra compute power each extra cpu adds.
(It's less than a full cpu's worth because of memory contention and
kernel interlocks and so on.) (Usually.)
