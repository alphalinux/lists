I'm hoping to get Red Hat 4.2 installed on an AlphaStation 200 4/233, but
have run into a number of rather nasty problems:

According to the manual, the system should have either ARC or SRM firmware
in 512K of flashROM. When I first turned on the system, I did get the ARC
console...but wasted no time in messing things up in such a way that now
when I turn things on, after the "BIOS Emulation" messages and stuff I just
get a blue screen. Frozen. However, if I boot with the keyboard
disconnected, then after a minute or so it complains and reverts to console
output on the serial port...running the SRM console. Go figure. On a whim
I tried running the firmware update software from floppy...it reports the
presence of 1 meg of flashROM and proceeds to update both ARC _and_ SRM.
I'm really not picky...whatever it takes to get Linux started, that's
fine...but I'd just like at least ONE of them to work properly at start-up,
keyboard attached or otherwise. Any ideas?

My Red Hat 4.2 CD doesn't arrive until tomorrow at best, so I've been
fiddling around with some of the boot disk images in the meantime...but
can't seem to get any one of them to work. Tried avanti.img, avanti-s, and
XL. I forget which one eventually started doing something other than
freezing (avanti-s, I'm assuming, since I was in the SRM console*), but
even with that one, the farthest along it got was to a seemingly infinite
loop of "SCSI bus reset" messages. I found a reference to this error among
the archive but no resolution to it.

* And then at this point, console I/O switches back to the monitor and
keyboard...which had to be detached to get SRM started. Argv!

Finally...assuming I DO eventually get this all installed and running...one
of the tasks I have in mind for the system is IP masquerading. I'll need
to add a network card for a second 10Base-T port...just wondering if there
are any recommendations or compatibility issues to be aware of, or if
something like the $20 no-name special will work. Both an ISA and a PCI
slot are free, so whatever is required or people think is best, there's
room. I'm sorta new to this whole thing, so any pointers, clues, or
words-o-wisdom would be immensely appreciated. Thanks much!

-- Phil
