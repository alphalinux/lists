On 31 Jul 1997, Eric Smith wrote:

> Ryan Kirkpatrick <rkirkpat@nag.cs.colorado.edu> asks:
> > Before I go about re-inventing the wheel, I would like to know if
> > anyone out there has made a rescue disk set for their Alpha box before?
> > And if you have, how did you go about doing it?
>
> I use a rescue ZIP disk. It was really easy to create; I just did a
> somewhat minimal install of Red Hat 3.0.3 onto a ZIP. I haven't got around
> to doing a new one for 4.2; maybe I should. But I've never gotten any
> kernel version newer than 1.3.90 to run reliably for more than half an hour
> anyhow. I get tons of those "unable to handle kernel paging request"
> messages.

I assume that would be a SCSI zip drive? I have tried (not very
hard) and failed to get my Parallel port ZIP drive to work with
Linux/Alpha. I would assume that it is not possible, due to the fact it
was designed for Intel.
Though that does give me a good idea of how to
make some "nicer" rescue disks for my Intel boxes. You just use a floppy
to boot the system, with the root device set to the zip disk?
Also, the part of a rescue disk that is the trickiest is the
booting of the machine. It is possible to boot a Linux kernel from the ARC
console with out MILO? If the hard disk crashes, then one no longer has
milo on the hard disk present to boot a linux kernel. Would one need a
floppy with milo on it, boot that from the ARC console? Then instruct milo
to boot a linux kernel off another disk? Keep in mind I am using a UDB 166
here, with the SRM console booting the ARC console, and the ARC console
booting Milo, and Milo booting Linux (boy, what a mouthful!). Am I
understanding this correctly?
Thanks to everyone's suggestions so far. Esp the ZIP drive idea!
Now if I can just figure out the booting sequence, I should be able to
make my rescue disks.....
