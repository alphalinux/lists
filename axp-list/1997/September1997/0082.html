> > Hello,

> > Is anybody doing anything on this front ?
> > So far I have understood (from experience) that just
> > loading any kind of modules modules at an Alpha is
> > sure way to invite kernel panic..
> >
> > I have modules-2.1.34 at my linux-2.1.51+ system, and
> > chronic lack of time to hack them to work...
>
> I am experimenting now with modules on kernel 2.0.30 with a UDB..the main
> reason was to try the comercial version of oss.
>
> Until now I never succeed to load oss because I got undefined
> symbol reference, but I've been able to load kernel modules without a
> problem (nfs.o, lp.o..etc).
>
> The kernel never panics, I get only undefined references..

Undefined references are mostly due to that some kernel symbols are
missing in the alpha architecture since noone is (was?) checking if
modules are working on the alpha. Simply add the missing references
to your kernel in for example arch/alpha/kernel/ksyms.c and recompile
the kernel.

> so...I keep working on it.

Kerneld works fine for me too.

I am using linux-2.1.36 (long time no upgrade), newer binutils than
redhat-4.2 and modutils-2.1.13. Upgrading binutils is maybe the thing you
don't come to think of when the modules aren't working, but it is needed.
I've had working modules since I was using something around linux 2.1.24.
