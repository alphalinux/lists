OK.. I have seen some strange behavior with regards to the /dev directory entries.

If I un-tar a LINUX image to be used as a root filesystem from a LINUX system to an NFS mounted drive
on a Digital UNIX system, Then look at the /dev directory, the Major - minor codes appear to be
corrupted..

--------------------
Example A
should be-> /dev/tty0 should be crw--w--w- 1 root tty 4,0
but is -> crw--w--w- 1 root tty 0,0

But, if I look at the same /dev dir being served from Digital UNIX side, the codes are correct.

Digital UNIX sees /linux/dev/tty0 crw--w--w- 1 root tty 4,0

--------------
Example B
to make it more confusing, if I then do a ./MAKEDEV from the Linux box on the nfs mounted
file system, the codes are corrected..

Linux box sees /linux/dev/tty0 crw--w--w- 1 tty 4,0
but Digital UNIX now sees /linux/dev/tty0 crw--w--w- 1 sec 0,1024

------------------
Example C
Now if I only reboot the Digital UNIX system, when it comes back up,

Digital UNIX sees /linux/dev/tty0 crw--w--w- 1 tty 4,0
Linux box sees /linux/dev/tty0 crw--w--w- 1 tty 0,0

The same as I started out in Example A?? confusing???

I used tty0 in my example, but all the /dev entries behave the same way...

Anyone know whats going on?
