Hi,

The state and structure of linux-alpha development has me confused.
I've tried to build a number of kernels with various patches from
gatekeeper and kernel.linux.org and had little or no success. I think
the only bootable kernel I've managed to build was the redhat 2.0.30-3
update. It wedged every ~30m, and was unusable.

Unfortunately, the bug it was supposed to fix (disk buffer cache
corruption when swapping) bit me yesterday, and my filesystem was
sufficiently damanged fsck was unable to repair it. fsck filled
lost+found with unattached inodes, and segfaulted when it tried to
expand lost+found.

Could have been worse (nothing critical was lost except time to
reinstall the machine). Could have been better, I still have not been
able to get my scsi dds2 tape drive to work (mostly becuase I have
been unable to build a stable kernel).

What I'd like to know is:

Who is doing what development? Where can I find their work? (ie.,
kernel patches?) What do these patches do? Which patches
interoperate? When do these patches get integrated into the mainline
linux kernel sources? Any advice if I want a stable machine?

If I cannot make this machine stable under linux/alpha, I'll have to
install digital unix, which would be a shame.
