Hi there,

the axp-list seems to have been down for almost a week. I just mailed
RedHat about it and half an hour later it was working again... Coincidence?

Well, onto the troubles,

I just got an AlphaPC 164LX, 533Mhz. Slick machine... Twice as fast as a
PPRO when running gcc and three times as fast running dhrystone (yes a silly
benchmark, I know). 530.57 BogoMips...

Anyway, we bought it to try it out as a server running our ASIC-simulators,
but of course we didn't get the kind of speed we wanted. In fact the first tests
showed it to be a tad slower than our PPROs. Sad. We knew beforehand that all
the byteaccesses in the simulator might mess up the Alpha and it seems we were
right. So we changed all bytes into ints and tried again. 20 percent faster, although
the code got half the size of the byteversion and even smaller than the PPRO one.
Cache trashing or something? I don't know quite what is happening.

I'm not sure but I think the CPU has instructions for byte access, but
gcc doesn't use them yet. I've heard that gcc for Alpha makes bad code
so I would like to see if DEC cc does a better job.

Could someone with DEC UNIX pick up http://www.axis.se/neteye/scripts/dectest.c
and run it through their cc (-S?) and mail me the assembler?

More questions... I seem to remember someone disabling the external
cache but cannot find any info about it. The external memory are 10ns
statics so it might be slower running through the 2meg cache.
Any experience with this?

Thanks,
