On Wed, 3 Sep 1997, chrismcc wrote:

> I am participating in the Bovine RC5 Cracking Effort (
> http://rc5.distributed.net , team: linux@linuxnet.org ) . After running
> the client on several linux boxes ( 486/66 , P133 , PP200 , Alpha300 ) I
> was disappointed in the alpha. The alpha does 350K keys/sec where the
> PP200 does 500K keys/sec. The problem is probably the source code not
> taking advantage of the alpha's fp routines/64 bit arch.
>
> Is anyone else doing the rc5 thing?
> Is someone capable of tweaking the source code to run better on the
> alpha?

Me! I'm participating in the Cyberian effort, though. (see:
http://www.rc5.cyberian.org). I'm running the RC5 client
(among many others) on an AlphaStation 500/500 and get about 550kKeys/s,
though. This is because: Cyberian have a much better RC5 implementation
and had a german computer scientist improve the code for several
RISC machines.

The major problem of the Alphas in contrast to the X86 breed, though
is that it does *not* have any ROLL instructions which are needed
for a fast RC5 implementation (at least for the cyberian algorithm).
So the ROLL instruction needs to be worked around on the Alpha.

I suggest you switch to cyberian to be faster :-)
