Well, folks, I'm back 8-)

No, I haven't been ignoring you, I've just been busy doing other things
and working on em86 whenever I have time (which isn't as often as I'd
like,
sad to say...)

Anyhow, I'm now in the process of releasing a second beta version of
EM86.
The bits themselves seem to have already migrated out to
ftp.digital.com,
and the updated docs should make their way out there in the next day or
so...

The new version features more system call coverage, more ioctl coverage
(RealAudio player now works!), and calls to native shared library
functions. Performance is better, but not rip-roaring; the shared
library
calls have to be implemented and validated one at a time, and I was only
able to do some of the more important libc and libm calls. I am going
to
put out a document on how to implement the library thunks so anybody can
write as many as they want (in particular, if someone can do libX11,
performance should go through the roof! 8-) )

I've also packaged it up as a set of rpm's so as to make installation
easier and more robust.

Here's a quick-start guide for those folks who absolutely can't wait for
the
docs to appear:

Pull down the following from
ftp://ftp.digital.com/pub/DEC/Linux-Alpha/em86/v0.2:

kernel-patches/alpha-patches-2.0.30-0.2.gz
RPMS/em86-0.2-1.alpha.rpm
RPMS/em86-x86-0.2-1.i386.rpm
RPMS/em86-baselibs-0.2-1.alpha.rpm
RPMS/em86-X11libs-0.2-1.alpha.rpm

Apply the kernel patch to an *unmodified* 2.0.30 source tree
(this patch set includes David M-T's patches and Jay's patches
as well as EM86 patches)

Install the RPMs in the order specified above. You may have to
use --ignorearch on the "em86-x86" package; this is the EM86
stuff that has to be built on x86, which is why it generates an
i386 rpm.

Give it a whirl!

Pointers to more docs and a webpage will be forthcoming.
