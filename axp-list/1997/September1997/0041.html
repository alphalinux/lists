>>> Johan Almqvist said:
>
> I have a problem installing RHL 4.2 on my Highscreen Alpha 5000
> (ie. pc164 500 MHz 64M IDE VIRGE/VX) from ftp.redhat.com with a 3c905 EN
> card.
>
> I can boot into milo and get milo to load the disk from the pc164 image
> and the ramdisk image. Then the installer says "hi" and asks how I want to
> install. I say "ftp". Then I get to the disk partitioning bit,
> everything's fine. Then it asks "what kind of network adapter you have"?
> and I tell it a 3Com 90x (or 509, for that matter) and tell it to
> "autoprobe" whereupon the bugger says it can't find /bin/insmod.
>
> There is a binary called install2 that most stuff links to, so i guess
> /bin/insmod should just be a link to that too -- but there is no link, an
> ln doesn't exist either. Furthermore, I can't seem to find the 3Com
> module either.

The kernels that Red Hat distributes should have the 3c509 driver in, but
prolly not the 3c90x (since it's not reliable on Alpha), IIRC. Maybe the
NE2000 is present as well; send a query to Red Hat support, they'll know.
Also, those kernels aren't currently set up for modules, so if you get to
the install question about which module to load, you know you're in trouble.
This is true not only of Ethernet cards, but SCSI cards as well.
