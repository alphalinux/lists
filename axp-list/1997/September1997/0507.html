The reason for my message is that Redhat will not ordinarily
update their packages unless there is a documented security hole or
critical bug. Like you state below, the one in netstat could be called
"critical" and yet there is no fix issued by Redhat. Not to mention
"most" Redhat users are loathe to go out to every site to get the latest
and greatest version of software unless they have to (leave that to
Debian). Nor is the ftp site below mentioned in the README for the
net-tools package. So frankly I was not sure where to go. So, I took the
bull by the horns and fixed it myself.

Like I said, this kind of bug should have been fixed a long time ago.
Netstat isn't performance critical and a system could work perfectly fine
without using it. But if you are running an ISP, tools like netstat are
ESSENTIAL. And if it were just cosmetic, I could live with that, but
I personally have seen our alpha crash due in-part I believe to the
unaligned trap errors in the kernel code. (reading a large core file)
Whether that also had to do with netstat, I do not know.

Thank you very much for the response, hopefully the latest version
of net-tools will be in the Redhat 5.0 release (tbird), and in the
meantime, Redhat will release an update RPM for Alpha users so that
this problem is corrected for them.

Stephen Zedalis
System Administrator - Exis Net, Inc.
