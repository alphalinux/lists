Certainly, the SRM PALcode (used by Digital UNIX, OpenVMS) can be used by
LINUX; that's how Linus got started on the JENSEN.

ARC/AlphaBIOS PALcode, ie that intended to be used by NT, is *NOT* usable
by LINUX at this time. This is where MILO comes in; it provides PALcode that
is very similar to SRM PALcode, so that LINUX can run.

The PALcode appearing in MILO is based on the so-called "sample" PALcode
produced and distributed by Digital Semiconductor as part of our Evaluation
Board System Developers Kit (EBSDK). This was initally produced specifically
to give to third party hardware houses, to help them to build and sell systems
based on the Alpha CPUs.

The example PALcode sets have been modified, over time, to support some of
the systems not actually built by Digital Semiconductor, systems such as
AVANTI/XL, ALCOR/XLT, MIATA. Until the introduction of the LX164, *all* the
EV5-based platforms supported were using EB164-based PALcode.
