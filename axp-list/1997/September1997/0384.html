At 05:03 PM 9/19/97 -0700, Richard Henderson wrote:
>> Where can I find a version of MILO that can boot 2.1.55
>> kernel so I can try out the IPv6 feature included in the 2.1.x kernel?
>
>You can use whatever milo you already have. The version numbers
>for MILO is the version of the kernel it can build against, not
>the version it can boot.

Well, unfortunately, this isn't always true; the 2.1.xx kernels can sometimes
be too big for an older MILO to boot, what with their funky "init" code/data
pages that (later) get returned to the system.

Best bet is to find the newest MILO for your platform on gatekeeper.dec.com.
Look in the /pub/DEC/Linux-Alpha/Miniloader/v2.0/ directory tree at the dates,
and give the latest a try. Also note that there are MILOs for LX164/PC164/EB164
in the /pub/DEC/Linux-Alpha/Kernels/ directory, named "milo-2031-xxxxx". These
should work with the 2.1.xx kernels, if you have one of these platforms.

Another thing to try is to configure a minimal 2.1.xx kernel for your machine,
using only the drivers you absolutely need. Or possibly to put as much as
possible in modules, but that, of course, is a horse of a different color... :-)

Good luck.
