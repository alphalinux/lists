I don't think so; only byte/word load/store.

> Sometimes, however, unaligned loads are necessary. In particular,
> the Ethernet packets, which are delivered by hardware as a binary block,
> have internal parts that aren't aligned. If one just grabs them,
> one ends up with an unaligned load. Of course it is possible to copy
> them byte by byte to an aligned buffer and access it there, but this
> requires special-case code in the driver, which wasn't done.

Actually it is IPX not Ethernet itself that is the culprit
here, I beleive. Certainly IP does not have this problem.

When the subject came up sometime back about putting in explicit
unaligned load sequences (which, btw, don't require a separate
buffer) to handle this, it was vetoed by On High.

The rationale? "I won't slow down the important case (IP) for
the unimportant case (IPX)."
