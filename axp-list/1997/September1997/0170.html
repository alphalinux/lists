> I'm trying to port a project from linux/ix86 to linux/axp, and on two
> of the C source files, I'm seeing the error message:
>
> Error: macro requires $at register while noat in effect
>
> when trying to compile.

It means that someone used an assembler macro that requires a temporary
register ($at). Gcc has no wish to give up control of a register to
macros it does not use and so issues ".set noat" at the top of the file.

The only way you could be seeing this is if the program contains inline
assembly and someone is using such a macro.
