Thanks for the suggestion, Daryll, but I'm simply using
normal '/etc/hosts' for IP addresses. I have a hosts
file of about 55 host addresses. The strange thing
is rlogin from PC Linux to SGI is quick, and so is
SGI to DEC Linux and DEC linux to PC Linux, but
PC Linux to DEC Linux is agonyzingly slow! The
DEC is using the RH 4.2 distribution and the PC's
are using RH 4.1. The security setup appears a
little different (RH 4.2 has libpwdb and RH 4.1
does not) and there are probable several other
differences (pam.d vs pam.conf, etc.), but I
just can't nail anything consistent down. All
machines have the same 'hosts' file--no
nameserver being used. I've done the netlib update
and the pwdb updates from the RH errata, also,
but no difference in behavior.

My weak knowledge of this stuff is at its limit,
and my frustration grows daily. Perhaps upgrading
all my PC's to RH 4.2 will cure the problem, but
I'm reluctant to invest the time at the moment
(especially when my PC's appear so stable--except
for the rlogin-to-DEC problem).
