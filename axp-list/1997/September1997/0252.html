I have source and binary RPMs at
ftp://gallifrey.uchicago.edu/pub/users/dgc/linux

There are notes on the circumstances of the build in the description
section of the RPMs. Particularly, the alpha binary was built against
2.1.36, and probably requires a 2.1-series kernel. You might be able
to rebuild from the SRPM, though; and even if not, the patches should
be helpful in getting it built on 2.0.x.

You need scsi-generic support in the kernel.

Here's the summary of what I changed:
My patch cleans up a number of compiler warnings, fixes byte-
ordering macro definitions, makes several System V semaphore
fixes, and repairs a few 64-bit issues.

This was my first RPM, and I won't object if you find it tastelessly
done -- I certainly do.

FWIW, I've exchanged information with cdda2wav's author about getting
these fixes into his source tree. A lot of it's taken care of as of
the 0.9-beta3 non-release, but we haven't spoken in quite some time,
and I'm unsure what the release status is.

-D.
