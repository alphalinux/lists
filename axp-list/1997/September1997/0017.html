>>> "Eduardo A. Trettel" said:
>
>
> I'm trying to install a 2.1.43 kernel on my AS255/233 Mhz
> running RH 4.2.

I think you'd be much better off trying a later version, say 2.1.51.

IIRC, around 2.1.4[34] the kernels wouldn't boot, or if they did, might
trash disks or similar nastiness.

> The compiling ends without any errors, except for
> several warnings about some simbols being used initialized.

Those are known, and can be safely ignored, I believe.

> The configuration in use:
> -AlphaStation 255/233Mhz
> -64 Mb RAM
> -Using Milo 2.0.22 to boot the kernel

You need a recent MILO to be able to boot a 2.1.xx series kernel; 2.0.22
is too old and cannot boot 2.1 kernels. Unfortunately, I don't believe there
are any available for AVANTI at this time, that are capable of booting 2.1.
This situation will be rectified soon, I believe, but until then, you're
out of luck... :-(

Oh, I take that back; you *may* be able to boot a 2.1 kernel with the 2.0.22
MILO, but you might need to strip down your kernel size to the minimum, ie
don't add any drivers/options you don't absolutely need.

> In the kernel configs I have:
> -commented out the SMP=1 flag on Makefile
> -selected "Avanti" as the system type
> -disabled support for the booting trought SRM
> -left all other options in their defaults.

Doesn't the 255 have a builtin NCR SCSI controller? If so, note that
the SCSI defaults to QLogicISP (what Linus has :-).

Good luck.
