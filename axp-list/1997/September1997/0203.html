multiple shared libraries, some having the same module/routine name!

say for instance i have a main(), and a private .so ( private because I created it for
this pgm ) . in the private .so there is a global open() routine. ALSO in the private
.so, in another routine, there is a call to dlsym(RTLD_NEXT, "OPEN"). the main()
routine also has use of libc.so which also contains an open() routine.

ergo:
private.so -> open()
libc.so.6 -> open()

What address should be returned - the open from my private .so, or the one from libc.so
?? if dlsym was called from private.so ??

DONT ANSWER SO FAST. The program I am debugging, codewise, suggests that the address
should be from libc.so. remember that the dlopen occures within the private .so.

I was debugging on the alpha/axp box - but my intel seems to have the same
unpredictable behaviour.
