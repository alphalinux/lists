I'm managed to get MILO booting linux off a SCSI drive attached to an BT958.

I'd earlier reported the odd behaviour of the machine hanging when a boot
command was issued, but booting `normally' if the `show' command was
used prior the boot command. My `fix' for the problem is
inelegant enough---I just basically added part of the show_parse()
routine to the top of milo() in milo.c . Specifically, adding a
call to show_devices() after the (existing in the distribution) call
to milo_whatami() appears to be sufficient.

The source that I've been using is jays-milo-2.0.30-06aug . I observed
similiar behaviour with older versions of MILO, but haven't attempted
applying the `fix' to see if it works with them.

- -SubG
