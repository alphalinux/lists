I'm continuing to observe some odd behaviour with my BT958 on my PC164.

I recently purchased a Seagate ST32107WC, which is a 7200 rpm 2.1 GB
drive with a SCA interface builtin. The vendor shipped it with a
little SCA converter board with 50 and 68 pin SCSI connectors.

My problem is that both MILO and the ARC console appear to have
trouble identifying the BusLogic card and the devices attached to
it.

The SCSI drive is partitioned in a fairly conventional manner: sda1,
a wee DOS partition containing MILO and linload.exe; sda2, /; sda3,
/usr; and sda4, swap.

When I attempt to add a boot selection from the ARC console, it only
offers:

SCSI Bus 0 Hard Disk 0 Partition 2

as an option. If I try to select `New system patition', and then
under `Select media:' select `SCSI Hard Disk', and then attempt to
enter a SCSI bus, it is unwilling to accept any number (and so
the only way to exit the `Enter SCSI bus number' prompt is to
use [ESC]).

If I accept the `SCSI Bus 0 Hard Disk 0 Partition 2' option, add
it as a boot option, and then attempt to edit the options (i.e.,
changing SYSTEMPARTITION, OSLOADER and OSLOADPARTITION to
reference scsi(0)disk(0)rdisk(0)partition(1) instead of
scsi(0)disk(0)rdisk(0)partition(2), it reports:

Error: Bad device name/number or partition number

...when I try to boot the machine using that boot option.

I can boot MILO from a floppy without any complains. Using a MILO
compiled from jays-milo-2.0.30-06aug, I get the following messages:

dev_config: don't know how to set hostid of SCSI device (104b:1040).
dev_config:fixing up latency from 32 to 32 for 104b:1040
dev_config:fixing up latency from 0 to 32 for 8086:404

If I try to boot linux from the SCSI drive, using something like:

boot sda2:vmlinux.gz root=/dev/sda2

...MILO will either hang immediately after the BusLogic driver
initialisation banner, or will report finding no SCSI hosts and reboot
the machine.

If I swap the MILO floppy for a floppy with a linux kernel, I can

boot fd0:vmlinux.gz

...and the machine will load the kernel with no problems, the BusLogic
driver will initialise without hanging, and the SCSI devices will
be properly identified. It will mount root from the SCSI drive with
no problem, and the machine will come up and go about its merry way
without any furth complaints.

Ideas?

Yours etc.,

SubGenius
