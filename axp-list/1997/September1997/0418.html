I have a Sony SCSI recordable CD-ROM drive that I hooked up to my UDB. Before
I worry about writing CDs, I want to get it working as a reader.

In /etc/fstab, I have the line:

/dev/scd0 /cdrom iso9660 ro,user,noauto,unhide 0 0

After I mount it, df (with the Red Hat Linux CD) reports:

Filesystem 1024-blocks Used Available Capacity Mounted on
/dev/scd0 658966 658966 0 100% /cdrom

Doing ls -a /cdrom, reports:

COPYING TRANS.TBL dev images misc tmp
CREDITS U doc kernels mnt usr
README Un dosutils lib proc
RPM-PGP-KEY X etc live rr_moved
RedHat bin home milo sbin

Here is the strangeness, "." shows as "U" and ".." shows as "Un" in every
directory on the disk.

[root@Alpo /]# cd /cdrom
[root@Alpo /cdrom]# ls -a
ls: .: No such file or directory
[root@Alpo /cdrom]# ls -a RedHat
RPMS U alpha rpmcontents.gz
TRANS.TBL Un base
[root@Alpo /cdrom]# cd RedHat
[root@Alpo /cdrom/RedHat]# ls -a
ls: .: No such file or directory

When I try to mount any other CDs (such as some Windows program disk) I get:

mount: wrong fs type, bad option, bad superblock on /dev/scd0,
or too many mounted file systems

I have an MS-DOS program, SCANCD (a "Corel CD-rom Information Utility v1.0")
which reports that all of the disks are ISO 9660.

Anyone have any idea what is going on here? Can't a CD recorder be used as a
reader?

My system is RH 4.2 with modutils-2.1.34-1. The ISO 9660 driver is running as
a module.
