> Hi all,
>
> I have a UDB 133Mhz, running RH 1.1. I did the base installation,
4.1 this is, right?

[...]

> If I try to FTP from a UNIX machine to the UDB I get output like this:
>
> jove:/home/tech/rzook% ftp babylon5
> Connected to babylon5.na.informix.com.
> 220 babylon5.na.informix.com FTP server (Version
> wu-2.4.2-academ[BETA-11](1) Wed

> Name (babylon5:rzook): rzook
> 331 Password required for rzook.
> Password:
> 530 Login incorrect.
> Login failed.
> Remote system type is UNIX.
> Using binary mode to transfer files.
> ftp>
>

[...]

Another Pluggable Authentication Modules (PAM)-Victim, it seems :-/

The pam-config entries for anon-ftp in RH4.1 seem incorrect. I don't
recall if installing a newer anon-ftp package or newer pam-rpms solved
the prob for me, but that's definitely the way to go.

If you ever expierience trouble with authentication, your first thought
should be to blame it on pam ;-)
