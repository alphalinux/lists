Hi there. I'm the guy trying to do the TFTP/NFS thing. I found some
documentation on MILO which has helped a lot (hadn't been able to find any
initially) so now I'm getting closer ... looks like I can't TFTP boot from
MILO directly. I was wondering if this support was being planned/added
for future MILOs.

Failing that, I have two other options that I can see. One is to flash
MILO into the boot loader, boot from a single floppy, and hack that floppy
to automatically TFTP the new kernel (possibly with linload?) and NFS
mount the root partition. This sounds extremely difficult.

My last option that I see is to setup SRM to TFTP MILO and NFS mount the
root partition, etc, if possible (is it? The UDB has virtually no
documentation). Should it be this difficult to set up a diskless client?
The biggest drawback I see so far, is that it appears that things such as
remote reboots simply won't work - booting would require user
intervention. This would also be problematic in the event of power loss.

I was able to boot the UDB using the three disk setup (SRM boot dva0, MILO
disk, BOOT disk, ROOT disk), but what a hassle! Completely unacceptable
for 99% uptime. Next question: It seems that most people use the ARC
console to boot from floppy - is there a mechanism to create a 'diskless
client bootdisk'? The three disks I created would all fit together on a
single 2.88MB floppy, so it seems it should be possible. Unfortunatley,
I'm not a kernel-hacker type of guy, and don't have the time to
investigate it.

Also, I would like to apologize for my previous message; I wasn't able to
find the documentation initially - I was just following RedHat's rather
sketchy instructions...

Thanks for your time,
