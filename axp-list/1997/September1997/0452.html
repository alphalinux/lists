Most risc machines can only accept a datum on a natural boundary,
thus a 32 bit integer's address has to be evenely divisible by 4.

This is not true with the Power PC, BTW The alpha design team didn't
really know what they were doing. In any event to answer your question,
its the job of the compiler to properly align things. If you get this
message then its really a compiler bug, i.e. the compiler that was used
to create the object module, which when excuted causes the trap. These
are a pain in the ass but can usually be fixed (If you don't want to fix
the code generation phase of the compiler) but forcing alignment by one
means or another, such as, declaring a variable of appropriate size either
above or below the variable which is misaligned. If it is gcc, ask
Stallman or Cygnus to fix it once you have isolated the module and the
variable. Use nm sorted by address.
