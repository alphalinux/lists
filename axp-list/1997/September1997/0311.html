Once upon a time, Eric Woudenberg said:
>
> Hi,
>
> I'm a happy Alpha Linux 4.2 user but I'd like to get "normal" Unix
> behavior on floating point overflow. This statement:
>
> double a = 1.0/exp(720);
>
> on OSF1, SunOS, and HP-UX quietly sets a to 0.0. On Linux is causes
> an FPE trap, even if I set SIGFPE to SIG_IGN.

compile with gcc -mieee. This will probably result in slower code.

See the axp-list archives at
http://www.redhat.com/mailing-lists/axp-list/
for more on this topic; also do a dejanews search in comp.os.linux.alpha
regarding floating point.
