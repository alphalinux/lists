Hmm, I don't think that this version has ever worked for anyone else
than Richard Henderson. The problem in building XEmacs from scratch
is a linker bug. This bug is not yet fixed in the current RedHat
distributions.

With a little trick it is possible to build a 19.15 version. I provide
some binaries via my little ftp site

ftp://franck.pc.uni-koeln.de/pub

(you have to get the "common"-file from a XEmacs mirror site).

Be careful in choosing the proper version since the XEmacs binary is
*very* sensible to the installed libraries (I think that this is the
biggest problem with the 20.2 binary, too).
