"Colorado/CSC Digital Unix,Motif,CDE Support" <t_sullivan@csc32.ENET.dec.com> writes:
> I'm trying to install RedHat Linux 4.2 on a jenson here in the office and
> not having much luck. So far, I've successfully partitioned the disks
> using minload.exe and copied the bootdisk image (bootdisk.1.3.72.jensen.srm)

The 1.3.72 kernel is outdated, it doesn't support ELF binaries. Try the stuff from

ftp://ftp.azstarnet.com/pub/linux/axp/jensen/install

> Q. How is the pseudo-bootdisk.gz image supposed to be used ? I have not
> found any information other than the statement "RedHat requires it for
> installation" found in the jenson-HOWTO.

The RedHat install usually is initiated by booting from a floppy,
using the MILO boot loader, I believe. Later, the installation process
copies the kernel from this floppy onto the hard disk, makeing it the
normal boot kernel.

However, with the Jensen you cannot boot from a floppy, but the
installation process still needs a disk to copy a kernel from. Thus it
was called 'pseudo-bootdisk'. Because your Jensen will still be booted
from the image you copied with 'copyboot', you should be able to use
any other bootdisk (created for different machines) from RedHat. You
may delete that kernel (found in /) afterwards.
