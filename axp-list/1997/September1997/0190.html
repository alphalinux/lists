Looks like it's just taking a long time to link the largest executable
in the GNU compiler system -- f771, the Fortran compiler module.

Perhaps your system doesn't have enough memory to do this operation
efficiently. I'd say it's almost certain you need more than 24MB
to build g77, so upgrade your machine to 64MB (or 80MB, which means
buying 64MB and continuing to use the two 8MB SIMMs your system
probably already has, and just storing away the two 4MB SIMMs).

>but already before that, the compiler had complained about missing files

Don't worry about that; it isn't relevant at all in this case.
(BTW, it's not the compiler that's complaining, it's the *build
procedure* for the compiler; and the complaint comes from moving
a bunch of files that are generally considered part of a particular
stage's build, when the first stage generates only a small subset
of them. So it's okay, and always happens to everyone doing a
bootstrap build.)

>Am I doing something wrong ? I have exactly followed the instructions for
>Quick Start as contained in the INSTALL file.

Good, just make sure you have enough memory, at least 64MB RAM and
3 to 4 times as much swap.
