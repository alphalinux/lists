I have a network composed of four architectures:

PC Linux (RH 4.1)
DEC Alpha Linux (RH 4.2)
SGI IRIX .3
SGI IRIX 6.2

The 'r' commands work fine between
the PC's and the SGI's, and fine between
thr DEC and the SGI's, but between the
PC's and the DEC there is a LONG time
delay before a rlogin is accepted (and
sometimes the connection is refused).

I have compared the pam stuff, the 'hosts' file,
the '.rhosts' files for the various users, and
can find no difference. The 'rpc' files are
identical between the DEC and the PC's. The only
system differences I have been able to find
between the PC's and the DEC are:

1. In the '/etc/inetd.conf' file, the alpha has an additional line:

auth stream tcp nowait nobody /usr/sbin/in.identd in.identd -l -e -o

and I, so far, have found no documentation about the line.

2. The alpha has an additional file: '/etc/pwdb.conf'. Again, I
can find no documentation on the file.

If anyone can suggest an area to find the cause of the problem, I
would be grateful.
