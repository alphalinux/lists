I have tried to compile and run a program package for ab-initio molecular
dynamics simulations on a UDB. It compiles without problems, but crashes when
I try to run it, with a segmentation fault (no core being dumped, however).
When I compile it with the -g option and run it under gdb, I get the following
message:

Program terminated with signal SIGSEGV, Segmentation fault.
The program no longer exists.
(gdb) where
No stack.
(gdb)

My question is first, why does the program crash on Alpha, when it runs
without problems on a Pentium Pro machine, compiled with g77-0.5.18 (in fact,
the version of the program is adapted for Linux, so it simply should work).
Are there perhaps some particular flags which have to be set on alpha to make
it work properly ? Second, is there any way at all to debug the program
compiled by g77 on Alpha ?
