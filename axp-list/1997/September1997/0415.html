Greetings...

Recently, I got a AlphaPC 164LX 533 overclocked to 600, with 128 megs of
memory, a big ole' 4gig drive hanging off an Adaptec 2940, and a Digital net
card.

Basically, after talking to the tech support of the people we got it from, I'm
still having a whole lot of trouble making the system run stablely.... stable
as in for more than 15 minutes at a time.

So, here's some symptoms and some ways I get the thing to bomb.. Basically,
the big error I'm *ALWAYS* getting is
``Memory Violation at pc=xxx rp=xxx (bad address = xxx)''

Now, first off.. does anyone else here run a similar configuration that is
working reliably?

As for software, I've tested the machine from how they installed it, which is
claimed to be redhat 4.2, with all the latest and greatest patches....

What I've tried.. installing redhat 4.2, with no nifty patches or anything.

Okay, how do I kill it.. well, I've booted the thing, and left it sitting idle
for a few momements, and then inn starts to bomb (the config'ed machine they
sent has everything on it, basically.)

Also, just trying to do a make config causes the errs..

Now, after I get a bunch of memory violations, I start to get other fun stuff
like paging errs, and problems with the inode tables... which is bad, to say
the least..

even the times when I've gotten make config to go all the way through, it dies
durring the make (using both make boot and make -j boot)

Now, the best sure fire way I've found to make it get the errors, is simply by
attempting to run tcpdump.. insta-err...

So, can anyone help me out here? If ANYthing, I'd like to hear from someone
successfully running a similar system, in a stable fasion..

Email me on or off list.. whichever you deem as appropriate..
