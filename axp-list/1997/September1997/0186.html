Ran into this back in July, dump version is 0.something and the test (
s_rev_level > DUMP_CURRENT_REV ) is causing the error message.
I recompiled dump (with patches installed) and used a version of 1 then
ran it. It is backing up without complaint, but I wonder if there's
actually something different between the file systems or is it just a rev
reporting problem.
BTW, there was also a problem with the source refering to one incorrect
function ext2fs_llseek, which I fixed. It looks like there may be some
