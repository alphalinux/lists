Hi,
I'm having a rather odd problem with my DE425 adaptor under linux
2.0.30 with alpha-patches-2.0.30-0.2.gz applied on a Jensen.
I get some warnings in the kernel compile in de4x5.c in the
de4x5_alloc_rx_buff function.. the error messages follow..

the kernel then continues to compile just fine, but when I reboot, the
DE425 adaptor initializes properly, and it links fine with my hub.. BUT I
can only transmit data.. if I try to receive data, ifconfig never
registers a packet as coming in... if I send data out, however, the
machine gladly sends it, my hub blinks, and tcpdump shows it on the lan..
hence, I think my problem is related to the alloc_rx_buff stuff..

The other problem I run into, is at the very end of make boot, I get
undefined references to pci bios calls, I simply commented the pci
functions out in de4x5.c since I have a jensen which only has an EISA
bus.. could that have broken the alloc_rx_buff function? has anyone else
had these problems?
thanks in advance
Pat
