By the way... one of the reasons this version of em86 took me so long
was that I was using Netscape as my primary testbed. Big mistake.
If Netscape crashed, I figured it must be an emulator problem and went
back to tear my hair out some more. It took me a long time to realize
that Netscape on Linux/x86 is just plain *broken*.

Okay, so the question I have is: just *how* is it broken? I've seen
different explanations floated as to how it does funny things with
memory
allocations or pixmaps or whatnot, but none of them hold up under close
scrutiny. I've tried a couple of fixes that have been posted on the Web
(use gnumalloc, or use the "nets" script which loads older versions of
shared libraries), and all they do is move the failure point around. It
may not crash in exactly the same way as before the fix, but it will
crash
somewhere else (especially if I'm using Netscape Gold to *edit* Web
pages!)

So: has anybody come across a *definitive* answer as to why Netscape
crashes on Linux/x86? If we can get to the bottom of this, it might be
possible to tuck a fix in to em86 so it will *not* happen on Alpha 8-)
