> Cache trashing or something? I don't know quite what is happening.

Not likely; original Alpha CPUs had no byte/short operations; all byte
access is done with int/long ops to load into registers, and then byte
extract/insert ops as appropriate. Currently, GCC utilizes this scenario
even on CPUs which do support the byte/short load ops. V2.8 of GCC purports
to do this, but its not been seen publicly, AFAIK...

> More questions... I seem to remember someone disabling the external
> cache but cannot find any info about it. The external memory are 10ns
> statics so it might be slower running through the 2meg cache.
> Any experience with this?

On the MIATA (PWSa series), it's been observed for apps that create a lot
of memory<->PCI bus traffic that the cache *does* get in the way.

The LX Tech Ref Man at:

http://ftp.digital.com/pub/Digital/info/semiconductor/literature/dsc-library.ht
ml

says that there are jumpers that allow specifying 0Mb of Bcache, which would
prolly do what you want, but I don't know if its safe to do that when there
*is* cache installed. I'll find out and let the list know...
