Roman,

> I have tried to compile and run a program package for
> ab-initio molecular dynamics simulations on a UDB. It
> compiles without problems, but crashes when I try to run
> it, with a segmentation fault (no core being dumped,
> however). When I compile it with the -g option and run
> it under gdb, I get the following message:

> Program terminated with signal SIGSEGV, Segmentation
> fault. The program no longer exists. (gdb) where No
> stack. (gdb)

Please see The Fine Manual, Chapter "Known Causes of Trouble with
GNU Fortran", Section "Bugs not in GNU Fortran", Paragraph "Stack
Overflow", and Paragraph "Strange Behaviour at Run Time".

If, in spite of trying out the options offered in those paragraphs,
your problem persists, mail fortran@gnu.ai.mit.edu with a complete
bug report (again, see the manual for what this means).
