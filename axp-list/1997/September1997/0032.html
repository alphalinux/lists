On Wed, 3 Sep 1997 jestabro@amt.tay1.dec.com wrote:

> Not likely; original Alpha CPUs had no byte/short operations; all byte
> access is done with int/long ops to load into registers, and then byte
> extract/insert ops as appropriate.

Is there a list what Alpha CPUs support which extensions?

I played around a bit retargetting a compiler to the Alpha and got a bit
confused. Neither the Architecture Handbook nor the Alpha compiler
cookbook seemed to be very clear about which instructions are only
available on certain CPUs.

I then tried the ldbu, ldwu instructions which seemed to work on my 21164.
However after a while I discovered that GNU as obviously treated them as
macros and generated a series of instructions using an additional
register. :-/

Also I tested some instructions which I had to convert to hex by hand
because GNU as didn't accept them.

Can I determine the available extensions with the amask-instruction?
GNU as says it's only supported on 21164A. The Architecture Handbook seems
unclear to me. It says on 21164 amask just copies and on 21164A, 21164PC
and 21264 it correctly indicates extensions. Does that mean the 21164 has
no extensions or does it mean that amask doesn't indicate them?
