On Sep 11, lebrun@lasa13.astrsp-mrs.fr wrote:

> Dear all,
> I am currently trying to port some binaries from DEC-UNIX to alpha-linux. I have
> ftp'd the binary, and put the correct protections for execution. However, the
> systems answers for example:
> ./pi: Command not found.
> whan a 'ls' command gives :
> > ls pi
> -rwxr-xr-x pi
> >
> This is a very simple 2 lines C++ program, whithout any external calls.
>
> Do you have any idea of what's happenning here ?

it can't load the shared library loader (/lib/ld.so for Linux binaries,
/sbin/loader or similar for DU).

strace ./pi

should be able to show what's missing...

either you have to copy all the shared library stuff from DU (but of course
only if you have a DU license for this CPU...) or link your program static
with DU's cc (using "cc -static" or similar, RTFM), then you can run them
with no problem on Linux...

Harald
