>>> Christopher Curtis said:
>
> My last option that I see is to setup SRM to TFTP MILO and NFS mount the
> root partition, etc, if possible (is it? The UDB has virtually no
> documentation). Should it be this difficult to set up a diskless client?
> The biggest drawback I see so far, is that it appears that things such as
> remote reboots simply won't work - booting would require user
> intervention. This would also be problematic in the event of power loss.

The latest Alpha patches (alpha-patches-2.0.30-0.[12].gz) contain support
for building *kernel* images that can be used with SRM console and BOOTP.
Simply do an additional "make bootpfile" after the "make boot", and look
for arch/alpha/boot/bootpfile as the image. You MUST build the kernel to
be booted directly from SRM for this to work, as MILO is not involved.

At some future time, AlphaBIOS on the newer platforms will be updated to
support some kind of network boot, but I know nothing about it yet. At that
time, I expect we'll find a way to package MILO and kernel together for the
image to be downloaded, as the LINUX kernel will need the PALcode that MILO
brings to the party...

Also note that I am working on patches for SRM booting that (gracefully :-)
support remote/unattended/automatic (re)booting. They're not quite ready
for prime time at the moment, but will be Real Soon Now (tm)...
