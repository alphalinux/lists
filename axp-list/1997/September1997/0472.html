I had this problem a while back (immediately after upgrade from RHL
4.1 to 4.2) with several programs... sync included. I first discovered it
while trying to backup to a tape using tar. The entire process would hang
and if I tried to stop it, I got zombies. The logs contained:

Whoops: unlock_buffer: async io complete on locked page

I upgraded my MILO and used a different self-compiled kernel and the
problem went away.

Hope this helps,
