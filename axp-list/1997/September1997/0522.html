James R. Shipman wrote:
>
> Be careful about caching with memory mapped I/O. Many times reading
> or writing registers causes side effects (like clearing status bits).
> If you don't turn off caching (or turn on guarding) for these
> registers you can get hit by cache line fetches for nearby address'.
> ie) the program accesses 0x0100 and the cache fetches a line from
> 0x100 to 0x120 or such. If you have memory mapped registers in that
> range which do something when you read them you get burned. Same for
> writes.
>
> Power PC has a "guarding" attribute which tells the cache not to
> touch memory which you didn't specifically ask for. I don't know
> about alpha.

Ok, that's a possibility. I have two questions:

- When I declare a variable as volatile (which I do for all pointers
to hardware), shouldn't caching be disable for this piece of code?
- Does anybody there know about the Alpha specs in regards to the
caching issue Jim is talking about ?
