>>> DKS@hep.anl.gov said:
>
> when I tried to install the Setup
> package, it complained that it couldn't find XFree86-VGA16. However, there
> was no such package in the update directory on your ftp site.

That's because the 16-color VGA server hasn't been ported to Alpha, and that's
the "lowest common denominator" server that XF86Setup uses on INTEL.

> In addition, when I
> tried to update the various servers, SVGA, TGA,... this failed because of
> cross dependencies with other server packages in the 3.2.9 sequence. The
> result is that I now have a partially upgraded version of XFree86, and hence
> no X capabilities. Where did I go wrong?

Yes, those complaints *are* annoying, but don't know who to blame them on.
Workaround is to simply do:

rpm -U --nodeps --force <whatever>.rpm

and that'll make it install.
