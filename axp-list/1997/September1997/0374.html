Assuming you are talking about running with the DU 4.x libraries, there
is a problem with java applets working. It has to do with the fact that
the DU 4.x libXt.so uses poll() calls instead of select().

You can get a workaround from DEC that consists of a libXt.so that uses
select() and a shell script to detect the OS and change the
LD_LIBRARY_PATH environment variable to point to this modified lib. You
will have to change the script if you are running Linux as the script
will want to see DU as the OS.

If you used a Netscape that came from DEC that would explain why it
worked.

Sorry I don't have the exact location on DEC's system for this
workaround.
