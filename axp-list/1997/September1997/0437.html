>>> nm said:
> well i guess i dont really need arc except that i cant figure out
> how i should properly set the time? in srm? or thru redhat?

I believe SRM has a "date" command, you could set it there.

Also, you must remember to *not* set the ARC mode in /etc/sysconfig/clock.

If you want to set the clock from LINUX when booted from SRM, do:

date xxxxxxxxxxx
clock -w

and it should get set. Use "clock -r" to read the TOY back.
