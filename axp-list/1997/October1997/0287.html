Hi!

4 days ago I reported some problems I had with Linux on a AlphaPC 164LX
motherboard. Now I seem to have solved these problems and as promised I
report a short summary of how I got my system to work as expected.

> 1.) Originally the system came with AlphaBIOS 5.60-3, and MILO couldn't
> boot Linux at all (to be correct: MILO_LX) - it just hung after
> some SCSI controller startup messages.
> I then downgraded to AlphaBIOS 5.60, and MILO_LX was then able to
> boot Linux, but sometimes (I would say 1 out of 5 times) it still
> just hangs.
> This seems to be a problem with NCR based SCSI controllers, I
> noticed
> the same troubles with an NCR-based Tekram DC390, but with an
> Adaptec
> 2940UW it works fine.
>
> So my questions are: Is this a known problem? Is anyone working on
> a solution to this? Any hints or patches to try?
>
This problem was solved when I got the MILO Jay Estabrook suggested in
his reply.
For the AlphaPC 164LX you have to use the following MILO:
<ftp://gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels/milo-2031-lx164>
(file size: 404105 bytes, file date: 06-Aug-97, 14:37:00)

> 2.) The system looses time, the system clock just runs exactly half the
> speed it should! For 60 seconds in "real time" the PC164LX counts
> 30 seconds! (This has an interesting side effect on benchmarks: I
> got 1920000 Dhrystones/Sec, for instance :-)
>
> So I have the same question: Has anyone a solution to this problem?
>
This problem also was solved using the new MILO.

> 3.) I want to use Alpha-based Linux systems as file- and
> network-servers.
> For that purpose the machines don't need keyboards attached, but
> when
> I first tried booting without a keyboard, nothing happend, the
> screen
> just remained dark, the system hung!
> For a server this is not acceptable, and I can't believe the PC164LX
> needs a keyboard to boot. Maybe it switches to some "serial console
> mode" when it doesn't detect a keyboard, but I couldn't find any
> documentation about it and the CMOS setup doesn't offer any option.
>
> Can anyone comment on this topic, please?
>
This problem I couldn't reproduce after I wrote my mail. After I had a
working system, I tried to boot the machine without a keyboard
connected,
and it (almost) worked as expected:

When the AlphaBIOS doesn't find a keyboard connected to the keyboard
controller, it displays a short explanation on the VGA screen and
redirects all further output to serial port #1 (at 9600bps, 8N1)

This works fine. I connected a serial terminal to the port and I could
use the terminal to control the setup-program and so on.
But there seems to be one glitch: now the AlphaBIOS seems to ignore the
Auto Start option: I have enabled Auto Start (10 seconds waiting time),
but the AlphaBIOS just displays the OS selection menu and waits for
input
from the terminal.
Here I can select the OS manually, and the Linux kernel even echoes
all kernel messages to the serial port (I compiled with this option
enabled), but I want it to boot automatically!
Anyone knows if there is a new AlphaBIOS (newer than 5.60) which
doesn't show this behaviour? Where can I download a newer AlphaBIOS for
the PC164LX?

Anyway, I want to thank all who responded to my questions, first of all
Jay Estabrook, who provided the clearest solution :-)

I also want to thank Christian Czezatke who helped me setting up the
machine with different MILO's and other startup-parameters over and over
again, despite of his cold (Hi Christian, hope you're feeling a little
bit better now, maybe the large pizzas were the right medicine?! :-)
