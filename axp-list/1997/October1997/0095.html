>
> I've created a bootpfile and loaded it onto my bootp server, but when I try
> to use bootp to load my bootpfile, my server says it is "ignoring a request
> for server 0.0.0.0 from client 0.0.0.0". I've added server address' for
> every possible server and it still gives me the same message. When I load
> linux off of diskette and use nfsroot, the bootp requests are accepted, but
> nfsroot doesn't ask for a server.
>
> I know the bootpd server has a database entry for the client ethernet card
> because I use bootp from that very same server and client to load the
> nfsroot when I boot off of diskette.
>
> What do I need to either get SRM to not ask for a server, or to get linux
> bootpd to give a reply for that request?

Here's the settings of all the ewa0* envvars on my CABRIO that will boot
fine from my PC164 server; note that only a few of these were actually set
by me, and that some simply hold the last used value. I'll mark the ones
I *know* I set by +, and the ones I'm not sure I set by ?...

>>>show ewa0*
ewa0_arp_tries 3
ewa0_bootp_file
ewa0_bootp_server
ewa0_bootp_tries 3
ewa0_def_ginetaddr 0.0.0.0
ewa0_def_inetaddr 0.0.0.0
ewa0_def_inetfile
ewa0_def_sinetaddr 0.0.0.0
ewa0_ginetaddr 0.0.0.0
?ewa0_inet_init bootp
?ewa0_inetaddr 192.168.0.1
?ewa0_inetfile /tftpboot
ewa0_loop_count 3e8
ewa0_loop_inc a
ewa0_loop_patt ffffffff
ewa0_loop_size 2e
ewa0_lp_msg_node 1
+ewa0_mode Twisted-Pair
+ewa0_protocols BOOTP
?ewa0_sinetaddr 192.168.0.2
ewa0_tftp_tries 3

To set the "empty" string, do:

>>> set ewa0_bootp_server ""

Good luck.
