> On Tue, 28 Oct 1997 Jay.Estabrook@digital.com wrote:
>
> >
> > >>> Sprag Johnson said:
> > >
> > > Can milo boot Digital UNIX or OpenVMS? I'd love to hear if it does.
> > > I've got a hankerin' to run VMS :) Digital unix would be nice on an
> > > occasion too :)
> >
> > No, MILO does not (currently) boot either DU or VMS. It could certainly be
> > taught to, but the critical part is the PALcode it brings to the party...
> >
> > Neither DU nor VMS will ever run using the existing EBSDK PALcode that MILO
> > supplies.
> >
> > --Jay++
>
> What about NetBSD, or (shudder) NT booting from MILO? I would love to be
> able to forget about ARC totally and still be able to run NT if I
> absolutely have to. And, the last I heard, NetBSD has to be booted from
> SRM. If it's not possible to boot NetBSD from MILO, does anyone know what
> would be involved in adding SRM firmware to a PC164 with ARC console (MILO
> is *not* currently flashed in)?

WNT has it's own _totally_ incompatible palcode. MILO was taken by the alpha
NetBSD folks but they never did anything with it (don't forget that MILO uses
real Linux device drivers). That's why BSD is limited to SRM systems.

As far as I remember, there is an SRM console for DS produced boards. However,
you have to buy Digital Unix to get it.
