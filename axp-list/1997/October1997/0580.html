"ST" == Stephen Tow <stow@ssibm4.ssp.ameslab.gov> writes:
ST> Why doesn't rsh/rlogin work when logged in as root? It works fine for
ST> normal users, but not for root. I'm sure this is some sort of security
ST> measure, but is there some way around this? Without it it's pure hell
ST> to try and propagate a common file across a dozen machines.

Hi.

I have no problems with rsh to a linux box when I setup the appropriate
entry in /root/.rhosts. Hosts.equiv is not sufficient for root logins.

Obligatory plug for ssh....
I use it for sysadmining tasks every day. It is trivial and
quite secure to setup a key that unlocks root on every machine and use
it as a replacement for rsh. It is possible to setup keys to perform
certain tasks. I'll be happy to expand on this ad nauseum...
