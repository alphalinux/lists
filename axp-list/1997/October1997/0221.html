Vik writes
When trying to compile opie-2.3.1., I get the following:

opielogin.c: In function `main':
opielogin.c:835: unknown register name `di' in `asm'
opielogin.c:835: unknown register name `cx' in `asm'

and line 835 is around

memset(&timeval, 0, sizeof(struct timeval));

FD_ZERO(&fds);
FD_SET(0, &fds);

Clearly something injects asm() code, and it is 8086 assembly code
instead of AXP assembly. You will have to analyze #define and #include
directives in opielogin.c to see why this happens. The code doesn't
look like it relies on asm() so I expect it should be easy to force
it to compile without them.
