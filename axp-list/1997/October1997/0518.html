>>> "RAAIJMAKERS@rulxha.leidenuniv.nl" said:
>
> All goes well, until the system prompts for me to set the root password.
> After the password is set, the system prompts:
> "Insert boot floppy into first floppy drive, if it is not already there."
> I insert the ramdisk-floppy into the drive, and the system responds with:
> "Mount: failed, Invalid argument"

It asked for the floppy that had the *kernel*; the RAMdisk floppy has none.

> And from that moment on I cannot complete setup (while rebooting brings
> me back to the setup program, so everything starts again).
> What am I doing wrong? For one thing, I have 1 big native Linux partition,
> and I didn't specify any swap partition.

Yes, the installation doesn't finish up at all cleanly if you don't give it
a floppy with the kernel on it; it doens't *have* to be the correct kernel
for the platform, especially if you've booted the correct kernel off the CD,
but you *MUST* give it a kernel on an EXT2 floppy or it will have problems.

And yes, you'd better repartition your disk to create a swap partition...
