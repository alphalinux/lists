I installed David Mosberger's patches to libc_p & gprof. I also
rebuilt gcc with Toon Moene's profiling patch. Now profiling seems to
basically work:

hjstein@randy:~$ cat foo.c
#include <stdio.h>

main ()
{
int i;
double x = 0;
for (i=1; i<100; i++) {
x+= i*i/2;
}
printf("x = %f\n", x);
}

hjstein@randy:~$ gcc -pg -g -O2 foo.c -o foo
hjstein@randy:~$ ./foo
x = 164150.000000

gprof runs on the output & doesn't seg fault.

I also seem to be able to use the -a option for block counting:

hjstein@randy:~$ gcc -pg -g -O2 -a foo.c -o foo
ld: bfd assertion fail elf64-alpha.c:2312
ld: bfd assertion fail elf64-alpha.c:2312
hjstein@randy:~$ ./foo
x = 164150.000000

gprof also seems to work, except that with the -l --annotate options
it seems that nothing happens:

hjstein@randy:~$ gprof -l --annotate foo
hjstein@randy:~$

Firstly, does anyone know of any way to fix this? And secondly, the
two above failed assertions are worrying. Is this something I should
worry about?
