Ok, I'm stumped.

Machine: Alphastation 255/233 - pretty much vanilla
- standard Digital TGA & ethernet & 1 Gig HD & CD Rom & official $100
Digital Fd.
- also a Segate 4 Gig disk on SCSI Id 1 - which is where I want to
put Linux.
- Console is SRM 6.0-943.
- default OS is DEC UNIX 3.2d

Linux Distribution: RedHat 4.2 for Alpha.

Configuration I need to reach:
Digital UNIX on the Dec HD
Linux on the Seagate HD
either bootable. [I don't care if I need to floppy boot LINUX, as
long as I can boot both and know how to build a stand alone LINUX
machine at some point]

I *think* this means
1. I have to boot under SRM, not MILO
2. I don't think I can upgrade SRM based on what I saw on
gatekeeper. The release notes for SRM 6.4 and 6.5 appear NOT to
support DEC UNIX 3.2d, although they support 3.2c and 4.0 [I don't
think upgrading to 4.0 or downgrading to 3.2c is an option] Can anyone
confirm or disabuse this?
3. I need to find some way to boot an `avanti' kernel.

What I think is on the RedHat CD [think, because the RedHat book
doesn't track the distribution very well]:
- MILO does not support `avanti' - don't see one in ~cdrom/milo/...
- aboot for SRM may be supported someplace in there someplace, but
I'm not sure where. I had a floppy constructed which brought up aboot
and tried to boot a kernel, but it couldn't figure out what to do with
the kernel.

Can somebody clearly map out the CD, so I can try to find out what is
on there? What is the difference between avanti... and avanti-s...?
What is the difference between images/ and kernels/? Stuff like that.

Alternately, I could build everything from scratch under DEC Unix, but
would like some words of advice from `someone who has gone before'.
This is not acceptable in the longer term because we are aiming at a
commercial product which must have clear, easy to follow, correct
instructions which result in a working system with little effort and
in a short time. This finite-but-arbitrary-time stuff won't fly.

Eagerly awaiting any light for this dim corner of the world.
