Hello all;

We are having some severe problems with an XL266 Workstation running
RedHat 4.2. We have a kernel from DEC (vmlinuz-2.0.30-fixed) and a
MILO-2.0.22. The machine boots all right, but blows up out of a sudden
without any apparent reasons. Now we do have another equivalent
machine running RedHat 4.0 which is has been running for ages.

We have build an own kernel (2.0.30 with axp-diffs) and the same thing
happens on the RH4.2-machine. I should tell you that we updated
XF86_S3 and friends to XFree3.3 in order to make use of 1152x8xx
resolution. The X-server triggers tons of alignment traps and the
machine seems to like to hang when switching from X to another console
or back.

Also: ypbind crashes frequently with memory violation at
pc=15555817fc8 rp=120004c1c (bad address = 00000044). My question: Is
anybody out there running an XL with similar setup and does see some
of those problems? Did anybody solve some of them? Can the X-server
hang the whole machine? What is going on here?

Thanx in advance
