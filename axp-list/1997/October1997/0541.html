It doesn't happen on my 533mhz 164-LX:

hjstein@blaster:~$ cat hello.c
#include <stdio.h>
main ()
{
printf("Hello World!!\n");
}
hjstein@blaster:~$ type hello
type: hello: not found
hjstein@blaster:~$ echo $PATH
/usr/bin:/bin:/usr/X11R6/bin:/usr/local/bin:/home/hjstein/bin
hjstein@blaster:~$ export PATH=$PATH:.
hjstein@blaster:~$ type hello
hello is ./hello
hjstein@blaster:~$ hello
Hello World!!
hjstein@blaster:~$ ./hello
Hello World!!

Did you apply the em86 patches to the kernel that came with Redhat or
to a pristine 2.0.30? I applied alpha-patches-2.0.30-0.2a.gz to a
pristine 2.0.30, and then applied the patch ap-0.2-fixes-1.diff from
ftp://genie.ucd.ie/pub/alpha/kernel/alpha-patchset.tar.gz. One hunk
from the latter got rejected, and I applied it by hand (just added
result = -EIO; to the NOT_READY & ILLEGAL_REQUEST cases in
drivers/scsi/sr_ioctl.c). I also left /usr/src/linux pointing at the
old kernel source tree. If you tried patching the kernel that comes
with Redhat, you might have gotten rejects or reversals or bad
patches, because the redhat distributed kernel already has some of the
alpha patches mixed in.

I'll post some general patching info separately.
