Several months ago when em86 first came out, I had a problem with
the first em86 kernel that I compiled using em86-patch and
traparg-patch-2.0. Some commands would not run and gave a seg
fault. I recompiled with only em86-patch (i.e. without
traparg-patch-2.0) and the problem went away. So I've been
using this kernel with good stability.

Now when I apply alpha-patches-2.0.30-0.2.gz the kernel gives
a similar problem. Binaries of .c programs that I write
and also "tcsh" and "bash" run from the command line give a seg
fault when I run them in the current directory, but run fine from
other directories. The problem is associated with tcsh, as using
bash as my shell instead of tcsh removes the problem. Going back
to the old kernel described above fixes the problem. Binaries
are from RedHat-4.2.

> cd /bin
/bin> tcsh
Segmentation fault
/bin> bash
Segmentation fault
/bin> ../bin/tcsh (runs OK)
/bin> ../bin/bash (runs OK)

I imagine that the problem is associate with the "traparg"
stuff in the alpha-patches-2.0.30-0.2.gz patch file, as
removing this from the previous em86 kernel fixed a similar
problem.

Any ideas ?

Rob Smith
