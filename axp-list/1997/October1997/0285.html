Hi,

On Sun, 19 Oct 1997, nm wrote:

> the mouse is a standard two button SERIAL mouse
> (not a ps/2 mouse) is this the problem? does
> du not work on the axppci33 boards with serial
> mice rather than ps/2 mice? any info on a
> work-around, or is he sol?

Yes. Just modify the file /usr/lib/X11/Xserver.conf to include
the section:

input <
< _dec_xi_serial_mouse lib_dec_xi_serial_mouse.so XiSerialMouseInit
/dev/tty00:microsoft:1200:1 >
>

Note that this is already there. You just need to remove the comment
lines.

Restart the X server and it should use an MS compatible mouse on COM1
after that. Change /dev/tty00 to /dev/tty01 for COM2.

Also make sure that the com ports are detected and listed in the boot
messages like this:

ace0 at isa0
ace1 at isa0

If you can't change the file because you can't login in X mode, boot
the machine to single-user mode with ">>> boot -fl s", remount the
file systems with "/sbin/bcheckrc" and edit the file there.

Hope this helps -- Tom
