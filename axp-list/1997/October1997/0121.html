On 1997/10/06 13:12 -0400, Robert Williams wrote:
> I've lost the /etc/fstab file
> on an axp system following an erroneous
> nfs mount. (There is an fstab file,
> but it is empty.) The system will come up
> in repair mode, but the root filesystem
> is mounted as read-only, so I can't
> repair fstab this way. Any suggestions?

Once you're booted in single user mode, you need to remount the root
filesystem read/write. This is done with:

mount -n -o remount,rw /

Then you can edit your /etc/fstab, and then make your fs readonly again:

mount -n -o remount,ro /
