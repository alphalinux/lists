I cannot commet on the status of g77 on Linux/Alpha. Just a few words
about the UDBs:

+) The first UDBs came out in 1994 (or was it even 1993?), so the machine
and the CPU they're using is quite outdated compared to the current
21164A implementation of the Alpha CPU. (so it would be fair to
compare the performance of the UDB to a state-of-the art PC
dating from 1994...;-))

+) The UDBs were sold by digital under the name "Multia". They
were sold with WinNT bundled with some additional software:
WinNT + an X--Server running under NT + a 3270 terminal
emulation.

The idea was to provide a machine that allows you to
+) Access Un*x boxes (the X--Server)
+) Access Host Computers (the 3270 terminal emulation)
+) run some standard Windoze apps locally
with just a single box sitting on your desk.

So the UDB was never designed as a number cruncher. I'm writing this mail
on a 233MHz UDB. I use this box under X for doing textprocessing (with
LyX) and for programming. -- Using the digital X--Server the box is rather
fast concerning normal X--Operation. Unfortunately compiling on the UDB is
dog slow because these machines have a very slow memory interface. :-((

I've seen a 600MHz 21164 box (LX based) recently and I have access to a
couple of similar machines (altough I have to share them with other users
most of the time...). -- And believe me, these boxes are a _LOT_ faster
than a UDB... ;-)

Oh, and BTW: I'm not a digital affiliate even if this email sounds as if I
were one... ;-)
