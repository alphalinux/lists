I upgraded to kernel 2.0.31 on our Alpha and detected severe problems
when a process starts swapping.

System configuration is as follows:

21164/500MHz
PC164 board
256MByte RAM
4GByte EIDE disk, no SCSI (up to now)
ELSA Winner 2000AVI 3D graphics card

When I start a process that allocates 250MByte of memory, kswapd dies
immediately, remaining in the process table as a zombie.
