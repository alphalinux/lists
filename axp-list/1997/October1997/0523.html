Hi!

I'm very pleased over the mustang release in general, there have been very
few problems to solve. But I have one seriously annoying problem right
now, and that's rpc.nfsd. It works (flawlessly), but it eats memory even
more eagerly than win-dos. I have to restart it every day because it's
size has grown from about 1MB to about 20MB. Naturally I am using nfs a
lot, but still, this behaviour is totally unacceptable.

I am using Red Hat Mustang (4.9.1) and linux-2.1.56 (I had no problem with
this kernel and the old Red Hat 4.2 tools) on my noname machine.

Has anyone seen this problem or is it only affecting me? I probably won't
have any problem debugging it myself if that becomes necessary, but I feel
lazy today and want to hear if it's a known problem.

BTW: 'pine' wasn't working for me in the distribution (and 'more' has the
same problem). The input was four characters after my keystrokes (sort of
a 4 byte fifo on the stdin) so pine was essentially unusable. I was able
to work around this by forcing use of termios instead of termio while
recompiling.

/Andreas
