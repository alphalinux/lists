Jan Gryko wrote:

> Hello Alpha-Linux gurus:
>
> I am really scared by several postings to this list about
> very low speed of Alpha systems. I am seriously considering buying
> Alpha 533 MHz system to run large Fortran programs and after
> reading todays postings I think that maybe Pentium Pro 266 will be
> a safer investment? Any comments?

You shouldn't make the mistake and mix up the older (and now
obsolete) UDB with the next-generation chips & chipsets. The UDB
features a 166 to 233 MHz 20166 chip. It's hard to judge the speed of
those machines but my 233 MHz Multia has a hard time beating my 133MHz
P5. Integer-wise, that is. The P5 cannot compete in floating point.

Now, the 533 system features a 21164 and is a completely different
story. Better memory interface, better CPU. You won't get a faster chip
from any vendor at this time (and that includes Intel).

Of course, one should never forget a CPU doesn't exist on its own.
Linux AXP isn't really optimized for the Alpha CPU. And all the PD
software stuff uses the GNU C compiler whose code qualitiy is far from
perfect on the AXP.

Now, if only DEC wouldn't charge so much money for DEC Unix ......

Which means: If you need raw performance, go for the Alpha ......

Just my DM 0.02 ...
