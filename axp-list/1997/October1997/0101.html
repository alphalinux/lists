I'm booting from bootp w/nfsroot on my PC164 w/SRM now. I looked at my
ewa* variables and tried the ewa0_sinetaddr one. Suddenly all started
working. I then went back in and put each variable back to the default,
and it still works perfectly! Don't know what was wrong, but now its
working with all defaults.

The last bit I needed was how to pass a command line to the linux kernel
from SRM. I figured this one out by looking at the source code in bootp.c.
You need to pass the command line in the boot_osflags variable. All the
docs say you can only put a number or "i" or "h" here, but you can, in
fact, enter a complete kernel command line (including "root=nfsroot").

Working great now, thanks,
