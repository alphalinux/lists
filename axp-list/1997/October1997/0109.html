Hi,tell me why the double value returned from go_convert() is not 7.0???
If u step through ( via gdb ) the val l.dd = 7, x.dd = 7,

BUT d.dd is some bogus number. d.dd should have 7.0

Am I doing something wrong ( u cannot change the code, unless there is a "C"
violation somewhere!!!!)

GAT

PS, thanks for the SIGNAL/FPE syscall. u should note ( although I havent verified )

that a sigfpe is still possible on a double conversion to long.

*********************************************************
typedef long int64_t;
double go_convert ( int64_t );
main()
{
union {
long ll;
double dd;
} l, d;
l.ll = 0x401c000000000000;
d.dd = go_convert ( l.ll );
}
double
go_convert( int64_t v )
{
union {
long ll;
double dd ;
} x;
x.ll = v;
return(x.dd);
}
