> no, how ( of if FPE ) errors (signals ) are to be reported (or fixed and sent

__ieee_set_fp_control(0) masks all FPE signals. You can retrieve
the acrued exceptions via __ieee_get_fp_control(); see <asm/fpu.h>
for the bits to check there.

Glibc 2.1 will provide the ISO C 9x <fenv.h> access routines for
this purpose.
