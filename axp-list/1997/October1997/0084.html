It seems that binutils-2.7.0.2-4 conflicts with
glibc-devel-0.961212-5, in that both contain the file obstack.h, and
the files differ.

I tried to incorporate David Mosberger's patch to gprof into
binutils-2.7.0.2. When I tried to install it I got:

root@randy:/usr/src/redhat/RPMS/alpha# rpm -U binutils-2.7.0.2-4a.alpha.rpm
/usr/include/obstack.h conflicts with file from glibc-devel-0.961212-5
error: binutils-2.7.0.2-4a.alpha.rpm cannot be installed

Checking system integrity shows:

root@randy:/usr/src/redhat/RPMS/alpha# rpm -q binutils
binutils-2.7.0.2-4
root@randy:/usr/src/redhat/RPMS/alpha# rpm -q glibc-devel
glibc-devel-0.961212-5
root@randy:/usr/src/redhat/RPMS/alpha# rpm -V binutils
S.5....T /usr/include/obstack.h
root@randy:/usr/src/redhat/RPMS/alpha# rpm -V glibc-devel
.......T /usr/include/rpcsvc/rquota.h

It looks like even the original binutils contains obstack.h (so it's
not due to my minor mucking with the package). Evidentally,
glibc-devel must have been force-installed over binutils on this box
(I bought the box preconfigured).

So, which one's right? I'd bet on glibc-devel. Anyone know who
should be notified to correct the conflict?
