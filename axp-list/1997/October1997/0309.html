> Can't help much with the new kernel, I don't know of any reason to
> upgrade. Anything compelling in the new kernels aside from new hardware
> drivers?

The real reason for not using 2.0.30 are problems in a virtual memory
handling. With some pattern of usage a machine may lock up with
"out of memory" while a lot of memory is really available. For one
reason or another 2.0.30 was more likely to trigger this bug then
earlier kernels. If this is not a problem for you then you may
continue to with 2.0.30.
