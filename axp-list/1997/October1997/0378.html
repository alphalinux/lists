I don't know whether Redhat has a fixed kernel source distribution already
on the FTP server. -- I also haven't bothered to check out the 2.0.31
kernel, but there is a simple way to get a 2.0.30 kernel with working
shared memory support:

+) Grab a pristine Linux 2.0.30 kernel source archive (for example from
ftp.funet.fi or from ftp.kernel.org

+) Get the appropriate axp-diffs for 2.0.30. AFAIK the latest patches
for this kernel can be found at:

ftp://ftp.gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels

AFAIK the latest axp-diffs for 2.0.30 are "alpha-patches-2.0.30-0.1.gz"

+) Install the pristine kernel sources (they usually reside in
/usr/src/linux)

+) apply the axp-diffs by using the following commands:
cd /usr/src/linux
zcat alpha-patches-2.0.30-0.1.gz | patch -p1 -E

+) Build a kernel from the resulting sources

This procedure should give you a kernel with working shared memory
support.

BTW: Anyone out there knowing what's the "official way" to build a 2.0.31
kernel for Linux/Alpha? -- Do the 2.0.30 AXP diffs still apply to the
2.0.31 sources or are there already axp-diffs for 2.0.31 available?
