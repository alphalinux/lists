Hi:

I'm using the following new milo-for-miata, it has fixed the "fast
clock" problem, I've been really satisfied with that for these 2 or 3
days;-)

> gatekeeper.dec.com:/pub/DEC/Linux-Alpha/Kernels/milo-2030-miata

But today I found one minor problem with it:

when I boot my Linux kernel 2.0.30 from this milo and then I reboot,
Linux returns to the milo and milo goes into the 30-second-wait-loop
before re-excuting the last boot command.

In this 30-second-wait-loop, when I hit any key than Esc, the milo
prompt appears, it's OK. But when I hit the Esc key, milo displays
following messages and die:

> kfree of non-kmalloced memory: fffffc000044d870, next=0000000000000000, order=0
> kfree of non-kmalloced memory: fffffc000044d858, next=0000000000000000, order=0
> kfree of non-kmalloced memory: fffffc000044e178, next=0000000000000000, order=0
> idle task may not sleep
> idle task may not sleep
> idle task may not sleep
> idle task may not sleep
>
(Note: those messages are typed by hand, actually a lot of kernel messages
like "Oops 0" and register info are displayrd before them)

What's happend? Is there any cure fot it?
