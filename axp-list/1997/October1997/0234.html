Hi!

I just entered the Linux/Alpha universe and now I'm in the process
of trying and learning.
I have a PC164LX with a 600MHZ 21164 CPU, 128MB RAM, IntraServer
3140U Fast Wide SCSI Controller and Fast-Wide SCSI harddisks and
some other goodies.
So far this system is very powerful and works quite well - if it works
at all...

The PC164LX seems to be very new, so Linux (and software tools like
MILO) still have some problems:

1.) Originally the system came with AlphaBIOS 5.60-3, and MILO couldn't
boot Linux at all (to be correct: MILO_LX) - it just hung after
some SCSI controller startup messages.
I then downgraded to AlphaBIOS 5.60, and MILO_LX was then able to
boot Linux, but sometimes (I would say 1 out of 5 times) it still
just hangs.
This seems to be a problem with NCR based SCSI controllers, I
noticed
the same troubles with an NCR-based Tekram DC390, but with an
Adaptec
2940UW it works fine.

So my questions are: Is this a known problem? Is anyone working on
a solution to this? Any hints or patches to try?

2.) The system looses time, the system clock just runs exactly half the
speed it should! For 60 seconds in "real time" the PC164LX counts
30 seconds! (This has an interesting side effect on benchmarks: I
got 1920000 Dhrystones/Sec, for instance :-)

So I have the same question: Has anyone a solution to this problem?

3.) I want to use Alpha-based Linux systems as file- and
network-servers.
For that purpose the machines don't need keyboards attached, but
when
I first tried booting without a keyboard, nothing happend, the
screen
just remained dark, the system hung!
For a server this is not acceptable, and I can't believe the PC164LX
needs a keyboard to boot. Maybe it switches to some "serial console
mode" when it doesn't detect a keyboard, but I couldn't find any
documentation about it and the CMOS setup doesn't offer any option.

Can anyone comment on this topic, please?

Well, that's it for the moment. Apart from the problems described above
(some are quite serious, though), I'm quite happy with the PC164LX.
I hope, with your help I can resolve the remainig problems so I can
recommend Alpha-based Linux systems to my customers in the future.

Thanx!

- andreas
