Dear UG and the countless others that ask me this

>according to www.dcginc.com
>"Due to inherent problems with Linux and IDE drives on Alpha we do not offer
> this system with Linux"
>
>that was for a 533mhz LX. My noname/axp pci 33 does not have probs with the
IDE, or
>have I not caught the bug/problem yet ????

Thats because you have a different chipset on the AXPpci33. The PC164 and
the 164LX both use a CMD 646. The bug is in the CMD 640 driver which was
never upgraded.

We can however, get Linux on an IDE, but you need to format the disk on a
working Alpha and load the file system onto the drive. From there you can
take it to the system in question and load it but you MUST NOT FORMAT IT OR
LOAD THE FILE SYSTEM.

Can you live with that? Please note this requires two systems, one already
running Linux, most people don't have two!

It works just not clean.
