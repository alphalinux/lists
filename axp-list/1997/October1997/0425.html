I just did some kerenl building but I have some issue's I can't resolve
myself.

I use RedHat 4.2 (As in the Powertools kit) on a Alpha XL 300.

The installed kernel is 656,787 bytes compressed or 1,677,864 bytes
uncompressed. But If I build my own kernel it get's 8,028,277 bytes
(compressed 2,351,942 bytes) Blarge.

Is this logical? (I don't think so as I only removed some extra stuff and
added the NCR 53C810 variables to speed up the disk.)

All I did was:
make config (make menuconfig does not work too well!!!)
make depend
make clean
make

The main reason for this is to get rid of some unused code but above all
use the NCR53C810 chip at 10MHz SCSI speed! I think the default slow
setting is quite slow.

This was among the first things I did form the Intel system and it did
improve disk performance considerably.
