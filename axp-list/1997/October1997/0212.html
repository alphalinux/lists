I've compiled Mesa 2.1 on exactly this platform. Unfortunately I cannot
give you any figures on performance comparison since I haven't used Mesa
on any other platform so far.

Just two hints for compiling Mesa 2.1 on RHL 4.2:

1) Remove the flag "-ansi" from the compiler options or certain
files won't compile

2) Due to a bug in the gcc optimizer the file "src/readpix.c"
will not compile with optimization turned on. The easiest way
to get around this is to compile that file without optimization
(i.e.: remove the "-O2").

If you don't want to go through all that hassles I could provide Alpha
binaries of the Mesa libraries so that you don't have to recompile them
yourself.

hope this helps
