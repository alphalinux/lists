Sheer Panic <sheer@eni.net> writes:

> Is there any way to set up fsck so that it does not halt for single user
> even on a serious fault, but makes a attempt to repair the filesystem
> anyway? I have systems on a remote power switch that I can't access in any
> other way. I guess the other option would be to have some way that single
> user talked to the serial port instead of the console.

Well I got a similar system, and in case of emergency fsck, I just have
the network and a demon started to make it possible for root to log in,
do the fsck'ing, and reboot.

On RH-Systems you should check /etc/rc.d/rc.sysinit.

Hope this helps,
