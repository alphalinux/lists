On Mon, 6 Oct 1997, Michael Tibor wrote:

> I'm trying to do the same thing. I have an Alpha UDB that acts as an
> internet gateway for my home lan, and my roommates have been bugging me to
> run a quake server on it.
>
> If you figure out how to do it, I would very much appreciate hearing how.

Ya, I finally got it. Here's a list of steps.

I will assume that you have Quake running on a Win95 machine.

1) make a dir on your Linux-AXP box (we'll call it quake).

2) ftp the contents of your quake directory (from your Win95 box) to the
quake dir that you just created on the Linux box. All you REALLY need is
the id1 directory and its contents, but I if you have other things that
you want, you might want to copy them, too. I copied everything and then
deleted stuff I decided I didn't want later. You don't need any of the
stuff in the base directory, but it won't hurt to have it there.

3) Get alpha-ELF--xquake-1.06.gz from somewhere. If you can't find it,
tell me. It shouldn't be hard to locate, but I can mail it (or FTP it) to
you if you would like. Gunzip it into the quake directory.

4) This is the odd part. I guess the libc that you have will be too new
for the 1yr old binary that you just downloaded. Mine was, so I had to
find a new one.

I got the one I needed at ftp://yoyodyne.lawbulletin.com/pub/libc-1.98.so

FTP it into the quake directory.

5) Run the following command line from the quake directory.
LD_PRELOAD=./libc-1.98.so nohup ./alpha-ELF-xquake-1.06 -dedicated 16 -nosound -noipx &

You probablly don't NEED -nosound or -noipx, but it can't hurt to be sure.

I hope this helps. Mine is now running at 151.159.83.30 if anyone wants
to stop by. If you do, drop a line and tell me how it is.
