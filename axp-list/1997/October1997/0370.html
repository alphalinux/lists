Just a guess: If you are running a kernel based on the sources Redhat
shipped on the 4.2 CDs, it could be that you've run into a problem with
shared memory support.

These kernel sources seem to have the "axp-diffs" only partially applied
resulting in a non-working shared memory support. -- This usually breaks
"xanim" too when it is run locally.

Try to start gimp with

gimp --no-xshm

If that gives you a working gimp, you're suffering from the broken shm
support menitoned above.

hope this helps...
