On Sat, Oct 18, 1997 at 12:20:45PM -0700, Christopher J. Grayce wrote:
>
> Can anyone shed light on this? I get SIGSEGV when calling realloc
> with what I reall think is a legitimate pointer.

My guess is that you've corrupted the heap by writing outside the memory
you actually allocated. This would be consistent with the program
happening to work on DU, since the arena meta-data is stored at different
places.

I don't recall what version of malloc is in Red Hat 4.2's libc, but
you can try setting the environment variable MALLOC_CHECK_=1, which
will detect some problems and print messages to stderr.
