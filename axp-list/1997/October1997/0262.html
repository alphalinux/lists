>
> It is not a problem of Alpha Chip but a problem of Fortran compiler
> (not very optimized). You will have the same problem (and more) with a
> Pentium Pro 266, do not forget that Pentium is only 32 bits processor
> (bad performance in double precision computes) and has not secondary
> memory cache.
>
> The only advantage of linux Alpha System is when you have already
> bought an Alpha station with Digital Unix because you can compile
> with the DU Fortran compiler (which is very performant) and execute
> your program on the Alpha Linux system.
>
> Lionel Provost

In order to do so, which shared libraries from Digital Unix do I need ?
Concerning the speed, when compiling with g77 -O2 (so far I did not have
problems with this, although in the info it's written that -O2 should not
be used on Alpha machines), my Monte Carlo program (doing almost all the
time floating-point operations) executes on my 166 MHz UDB a factor of 2
slower than on a 180 MHz Pentium Pro, which is not bad taking into account
the difference in price between the machines. However, my experience is that
the weakest point of g77 on my UDB is debugging, which sometimes turns out
to be completely impossible. FORTRAN programs which cleanly compile and execute
without any problems on IBM RS 6000, Linux/Pentium, or Alpha with Digital Unix,
crash with strange segmentation faults on Linux Alpha. Even when running
under gdb, when they crash, the debugger says "the program no longer exists",
and it is not possible to find the place where it crashed and the reason why.
Since my main motivation in buying the UDB was to get a small home machine to
use for Fortran code development, I am a bit disappointed with this behaviour
of g77 on Alpha platform; when buying the UDB I believed that g77 would run
just as well as it does on Pentium.
