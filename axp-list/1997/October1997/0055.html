Hi,

A month ago, I posted here about the clock problem of Linux on DEC PWS
433a(Miata) : "The system clock goes too fast than the CMOS clock".
And with some follows from Jay A Estabrook and other members on this
list (I thank you a lot!), I found the main culprit was a so-called
"PALcode" of milo for Miata.

And now, does anybody know how the development for milo or PALcode for
Miata is going?

I think the latest milo image for miata resides on:

http://gatekeeper.dec.com/pub/DEC/Linux-Alpha/Miniloader/v2.0/latest/Miata/

but this milo has the same clock problem.

For a test, I compiled milo by myself using the source
milo-2.0.30.tar.gz on my RedHat 4.2 system, the complilation ended
successfully, and the new-milo looks working fine for bootloading, but
of cource it has the same clock problem.

As to the source tree derived from milo-2.0.30.tar.gz, my newly
complied milo seems to use /milo-2.0.30/palcode/eb164/osfpal.nh as its
PALcode. I wonder, if the 'true' PALcode for Miata is provided by
Digital Semiconductor or somebody, and you replace 'osfpal.nh' with it
and compile, does everything come to work fine?
