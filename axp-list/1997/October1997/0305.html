I got an alpha system from Microway with Red Hat 4.2 installed. I had
originally been able to NFS mount drives on this machine OK. I went to
install NIS and discovered some discrepancies with the Intel and Sparc Red Hat
4.2: Intel and Sparc do not have ypbind, whereas there is a ypbind utility
for the alpha. I was able to get NIS sort of working -- you could log in, but
there were a lot of delays and cron was sending root hundreds of mail messages
a day complaining of problems:

do_ypcall: clnt_call: RPC: Unable to send; errno = Resource temporarily
unavailable

also

YPBINDPROC_DOMAIN: No server for domain saintl.com

I decided to remove NIS and so I deleted the S links to ypbind and ypclient,
which I had originally made according to documentation for running NIS on
sparc/intel. I also re-named my machine and after rebooting, I also lost the
ability to mount drives via NFS. Other machines can mount drives on this
alpha, but it cannot mount drives from other machines. The errors are:

rpc mount: RPC: Timed out

If I try to get information from the portmapper using rpcinfo -p, I get:

rpcinfo: can't contact portmapper: rpcinfo: RPC: Timed out

The portmapper is running and there are no messages from it in /var/log/messages
rpc.nfsd and rpc.mountd are also running.

The network is running because you can telnet to and from the machine. Anyone
have any idea 1) how to restore the ability to mount NFS drives and 2) how to
get NIS running ? My NIS server is a sparc running solaris 1.1.x

Thanks,

Dirk
