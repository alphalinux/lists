> > In reading the IP Masquerading HowTo, I see that it seems to rely on

>
> It should work fine on alphas by now. I haven't tried it personally,
> though.

But I did - not so long time ago. Linux/axp machine, which is directly
hooked up to Internet via ethernet cable, masquerades for ppp
connections which use "internal" addresses. It is not required, but
I found it more convenient to have both ends of ppp connection on
an "internal" subnet. Nameservers are not bothered immediately
when a connection goes up. As far as I can tell everything works
like it is supposed to.

> and with a
> few tricks, modules will work fine on the alphas.

These "few tricks" consist of one command "strip --strip-debug"
to be performed on module object files. This makes insmod happy
and that is all.
