> whenever I try to access the 3,2 GB disk, I get severe
> problems.
>
> scsi: aborting command due to timeout: pid 9282, scsi0, channel 0, id1,
> lun0, Read(6) 00 00 02 0e 00
> scsi host 0 abort (pid 9285) timed out - resetting
> scsi bus is being reset for host 0 channel 0

This could be termination problems (the 1740 uses passive termination - you
could try active termination at both ends of the cable).

Another problem is that Quantum screwed up the SCSI interface on the
Fireball TM series - they do just 3MB/s (while the EIDE version does more
than double that), *and* according to the german c't magazine, the tagged
command queueing code in the firmware is broken. I am not sure if the 1740
driver uses tagged queueing and if it can be disabled, but my recommendation
would be to get rid of that drive somehow.
