> i'm installing red hat 4 on jensen.
> i've already downloaded the install packages
> and stored them in the /RedHat directory on
> /dev/sdb1. i want to install linux on /dev/sdba2
> since /dev/sdba1 is there for getting it up and
> /dev/sdba3 for swapping.

What do you mean with /dev/sdba? ? It's either
/dev/sda? or /dev/sdb? !

> _but_ the install script does _not_ seems to be
> unable to mount the disk with the install media.
> why?
>
> says .... mount failed: device not configured
> i say ... _what???_

AFAIK, this is output from the install program, not a
direct error from 'mount'. You can switch wie ALT-Fx to
other virtual consoles, there are usually some messages
on consoles 2,3,4, or 5. Try them all.

The install program checks, if it can find an textfile named
/RedHat/axp on that partition. It's an empty file, but it must be
there. You may use 'touch' if you lack it.

Your packages must be found under the directory /RedHat/RPMS. I guess
you'll also need the stuff under /instimage. I'am not sure, if there's
more stuff needed.

> now "show dev" in the srm console says it's everything
> all right!
>
> ... and which is even worse _is_ that i can't do
> it myself cause "mount" is "not found"

It must be somewhere. Maybe not in the usual location. Browse the
floppy contents. But that's probably not the problem, see above.
