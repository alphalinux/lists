On Fri, 21 Feb 1997, Donnie Barnes wrote:

> I've installed RH 4.1 on an Alpha UDB (Multia).
>
> Standard config for a multia.
>
> Well, when the network comes up: S10network
> The loop that goes through all the files in /etc/sysconfig/network-scripts
> hangs for half an hour. I'm not exaggerating.
> While it's hanging, it's hanging the WHOLE NETWORK.
> I took out parts of the loop and I'm just running 2 file:
> /etc/sysconfig/network-scripts/ifup ifcfg-lo boot
> /etc/sysconfig/network-scripts/ifup ifcfg-eth0 boot
>
> I also noticed that perhaps if ifcfg-eth0 was run first, it would cause the
> same problems.. if there wasn't a local interface already define.
>
> Very very bad.

I can't be of any help directly, but I can say that we have run 4.1
on several UDBs and we haven't seen that. I suggest taking this to
the axp-list@redhat.com since that is where the Alpha gurus hang out.

The problem arises if you have the updated glibc, with the
/etc/nsswitch.conf file. If you start the loopback before the other
network interfaces all works well.
