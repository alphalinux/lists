>
> One - one of our disks was terminated improperly which crashed the
> machine. When I go into the initial alpha setup screen and try to format
> it there it just hangs. Is there a way on the alpha to just do a
> straight low level format?

This is not a "low level format" but mk2efs running. Doing that
from a shell prompt (Alt-F2) may give you an extra information.

In my past experience booting from CD, or ram drive, so disks are not "in
use" gaves a better way to resolve such troubles. So far I did not run
into a situation, with Linux/axp, which fsck, possibly multiple times,
was not able to resolve. You may end up with things in 'lost+found',
but this is another story. :-) If you managed to start mk2efs already
it may be too late for such attempts. This assumes that the underlying
hardware trouble was fixed.

> Secondly, since this problem the machine refuses to reboot into multi
> user mode, is there a way to fix this?

Yes, sure, once the problem was identified and corrected.

Mucking around with a file system is done better from a single user
mode anyway and if confronted with such situation I boot "single" on
purpose.
