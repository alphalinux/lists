Hello world,
I've upgraded w/o any problems from 4.0 to redhat 4.1, and
wanted to compile a new 2.1.x kernel to use digital unix 4.0
binaries.
I had no trouble compiling the kernel, but couln't
boot it. precisely,
# make boot
# cp arch/alpha/boot/vmlinux.gz /2.1.26.gz
[boot]
MILO> boot sda2:2.1.26.gz
and MILO displayed a long sequence of sharps, followed by 'done.',
followed by a 1 second pause, and the machine rebooted
spontaneously.
It is a PC164, with nothing really fancy, and I had already
often compiled 2.0.x kernels, without experiencing this.

A problem I noticed with redhat is the xdm X manager.
The program xdm does not start, complaining about a missing
shared object /usr/X11R6/lib/X11/xdm/libXdmGreet.so
I haven't been able to locate this library.

Thanks to all,
