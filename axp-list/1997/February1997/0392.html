This is wrong for reasons repeated over and over again. It should
be /dev/ttyS0.

> mru 128
> mtu 128

These are way too low. The smallest packet you can get is 256 bytes
plus 40 bytes of a header - so minimal values for these are really 296.
Small packets are good for an interactive use, but not so good for
transfers.

If I remember all of this is described in PPP-HOWTO (or one of similar
documents which come with PPP distribution).
