>>> Robert Allen Barker said:
>
> I am attempting to write an alpha/linnux driver for the BT848 multimedia and
> frame grabber chip. To capture a frame the chip dumps the video data
> directly into the system main memory. To do this the chip must be provided
> with the physical address of an array in the driver program running under
> linux.
>
> Q: How can I get the physical address of a data array in my program? I am
> still using redhat 3, the function phys() does not seem the be implimented.

As Richard Henderson said, the functions you need are virt_to_bus() or
virt_to_phys(), in this case, probably the former.

Explanation:

virt_to_phys() takes a kernel virtual address and provides you back with
the location's physical address AS SEEN BY THE CPU.

virt_to_bus() takes a kernel virtual address and provides you back with
the location's physical address AS SEEN BY A DEVICE ON THE BUS.

On the Alphas, devices on the PCI/ISA/EISA bus are provided with a "window"
into main memory, the window having a range of addresses on the bus which are
then mapped to main memory addresses; this is NOT an identity mapping.

For example: on most of the Alphas, the "window" is setup for the range of
bus addresses from 1GB to 2Gb-1, mapping (upto) a total of 1Gb of main memory.
All the other bus addresses can be used fpr bus-local memory, such as frame
buffers for video cards, etc.

Examples of the use of virt_to_bus() can be found in the SCSI and net drivers
that have been ported to the Alphas.
