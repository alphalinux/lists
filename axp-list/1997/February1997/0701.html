I'm trying to port a device driver from Linux/Intel to Linux/Alpha, and
find for the most part it's pretty easy. However, I'm having trouble getting
a PLX9060 set up to do a DMA on the PCI bus to the Alpha memory.

Specifically, I'm having trouble getting memory addresses that I can give
to the PLX so that it can do a transfer.

I've allocated some buffer space with get_free_page, fine.

I've been able to manipulate the PLX registers just fine, from
both sides. (This is a Cyclone-911 board. I have existing software
on the 911 side.) I've verified this.

When I set up a DMA transfer, I use the address from virt_to_phys
as the transfer address. I'm getting addresses like 3fb4000 which
seem plausible, and 32bit thankyou.

No good. It appears that the PLX is not getting a response from the memory.
I'm stuck.
