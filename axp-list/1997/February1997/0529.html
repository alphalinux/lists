I have been having reliable results using a Diamond Stealth 64 Video VRAM
(3240, S3-968 chipset) with the XFree-3.2 (and now XFree-3.2A) X-server in
16bpp mode. This is in a VX-42 UDB. This has been rock solid for me, staying
up for days at a time, including when running remote-X applications like
Netscape. I get unaligned traps when using Netscape, but they seem to be
harmless.

I found the XFree-3.1.2 server to be problematic, so if I were you, I would
upgrade to Redhat 4.1 (which has the 3.2 server), or pull the XFree binary
distribution of 3.2 or 3.2A for linux-alpha-ELF from ftp.xfree86.org.

David
