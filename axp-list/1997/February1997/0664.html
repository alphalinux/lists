On Thu, 20 Feb 1997, Andrea Dell'Amico wrote:

> My machine hangs on boot, but if I boot in single user and do manually:
>
> cd /etc/sysconfig/network-scripts
> ./ifup ./ifcfg-lo
>
> ./ifup ./ifcfg-eth0
>
> exit (and going on to runlevel 3)
>
> then all works right.
>
> If in the same situation do manually:
>
> /etc/rc.d/init.d/network start
>
> the process hangs. I stop it after a while with ^C, and this is the
> ifconfig output:
>
>
> eth0 Link encap:10Mbps Ethernet HWaddr 08:00:2B:E4:06:16
> inet addr:131.114.6.137 Bcast:131.114.6.255 Mask:255.255.255.0
> UP BROADCAST RUNNING MULTICAST MTU:1500 Metric:1
> RX packets:5755 errors:0 dropped:0 overruns:0
> TX packets:35914 errors:2 dropped:0 overruns:0
> Interrupt:10 Base address:0xb000
>
> So the loopback interface isn't started. Shutting down eth0, starting
> lo and eth0 in the right order and going to runlevel 3 works.

I had this problem and I hacked the network startup script to force the
loopback interface set up first and it worked. It seems that some code
somewhere wants to talk to 127.0.0.1 and it ends up shipping packets out
the eth0 interface because lo isn't around to talk to.

I don't see why lo isn't set up first all the time...
