>
> Is there a better way to do it? Instead of reboot, is it possible to
> clean it up in the C-A-D script. I suppose the X server is dead

I don't know; the problem is that both X and the VC's get confused as to the
mode the TGA is in, so that *neither* work. The machine and server do not
typically crash, so orderly shutdown is best if one can do it. And
unfortunately, the kernel doesn't seem to restore text mode very well even
after the server is shut down.

Best hope is that the 3.2 version will gain accelerated functionality soon,
and make this moot. The 3.2A beta binaries are or should be available now at
ftp.xfree86.org, and there was talk of having added some new acceleration to
the new TGA server code, IIRC.
