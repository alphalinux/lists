I've installed 4.1 on a PC164
(several times now, using two different
ide drives). The ftp install process proceeds
without incident.

Upon booting for the first time,
the console stops at "amd" as follows:

Starting lpd: lpd
Starting named: named
Starting NFS services: rpc.mountd rpc.nfsd
amd

And that's all. The console becomes
hung. No virtual consoles are available.

The system responds to telnet from
a remote system, but root logins are not allowed.

There are no scsi drives on the system.
(I want to rebuild the kernel with the
newest Buslogic Ultrawide driver.)

The system was running fine under RedHat 4.0.
This problem started with 4.1. I first
tried an upgrade and had the same problem.
Subsequent full installs on several disks
always stop reliably at the same point.

So...Help!
