On Tue, 18 Feb 1997, Uncle George wrote:

> I can only say that the analogy is the same with fclose(). I expect
> fclose to give me an error return - something more gracefull than a
> segfault. Fclose should know that the ptr is invalid - points to no open
> channel - and give an appropiate error code.

The only error POSIX compliant fclose() can return is EOF, which is not
appropriate in the case of this type of programmer error. I suppose you
could make a case that your compiler should return a warning or error, but
to change fclose() to return what you think is an appropriate error at
runtime is at least as wrong as depending upon what is specified as
undefined behaviour.
