Forgot some things...

1. Some1 suggested to use -fno-common , this doesn't fix the "problem"
though. (And I really can't understand how it should either, from
reading the gcc info page, -fno-common just allocates space in bss
even for unitizialized vars, and to quote the info page:

"This has the effect that if the same variable is declared (without
`extern') in two different compilations, you will get an error
when you link them. The only reason this might be useful is if
you wish to verify that the program will work on other systems
which always work this way."

2. A newer binutils does indeed help. temacs compiles fine now. I'm not
sure what cygnus says about telling where their "secret" gas snapshot
area is though.. (I'm using gas (binutils) snapshot from 970210, but
some of the older snapshots (from januari) also compiled xemacs
