On Wed, 19 Feb 1997, Richard Henderson wrote:
> > I recently read somewhere that the Linux-axp Port is
> > not so fast as it could be, because gcc is not completely
> > optimized for axp CPU`s.
>
> As fast as it could be? When is anything as fast as it could be?
>
> Anyway, GCC gets within 3% of DEC's compilers for the EV45 (21064/21066).
> There is not a public release of GCC that can schedule for the EV5 (21164),
> so it's off by 15% or more there.
>
> The system as a whole doesn't suffer for it, though. No system programs
> are so compute-bound.

Well, this ofcourse depends on the application. My machine
participates in the rc5-56bit crack now going on (see
http://zero.genx.net, please join, $1k to whoms machine finds the key!).

This application is only cpu-bound (not depending on libraries, io) so
gcc is the only "important thing here".

On my 266Mhz noname I get these results:

rc5-client compiled with Digital CC: 8.4 secs for 1M keys
rc5-client compiled with gcc : 14.2 secs for 1M keys

A very noticable difference. I've tried every possible optimization gcc
has, but it really doesn't matter.

I'm gonna investigate (whenever I find free time..) why it
differs so much and report my findings to this list.
