> I recently read somewhere that the Linux-axp Port is
> not so fast as it could be, because gcc is not completely
> optimized for axp CPU`s.

As fast as it could be? When is anything as fast as it could be?

Anyway, GCC gets within 3% of DEC's compilers for the EV45 (21064/21066).
There is not a public release of GCC that can schedule for the EV5 (21164),
so it's off by 15% or more there.

The system as a whole doesn't suffer for it, though. No system programs
are so compute-bound.
