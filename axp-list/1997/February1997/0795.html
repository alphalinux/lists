Aaron M. Gowatch wrote:
>
> I am somewhat new to Linux/AXP and I have a few questions.
>
> I installed Linux 2.0.29 on a UDB which has a TGA video adapter. Is it
> possible to boot Linux in any other resolulution than 80x25? I've tried
> passing "vga=xx" parameters from MILO as you would in x86 Linux, but I
> havent had any luck. Perhaps this has been answered in some FAQ
> somewhere, but I havent been able to find it...

TGA is not VGA. TGA has some minimal textmode emulator for VGA textmode
emulation. If you want diffrent reolution look at tga.c in the kernel
source. Passing vga parameters does not work.

> The other question is related to kernel builds. I installed the 2.0.27
> axp kernel source rpm with RH4.1. When building the kernel, I receive a
> number of warnings, most of which end with "...might go uninitialized in
> this function". The kernels build, but I was just curious as to whether
> or not I should be concerned with the warnings or if I'm doing something
> wrong. I had similar results with the 2.1.X kernels.

Don't think it is a problem, I was running a kernel for couple of weeks
where the compilation of the kernel produced this kind of warnings,
mostly
when compiling drivers (e..g. network cards and sound cards drivers).
Running it caused no visible errors 8-).
