Hi,

Hope this is my final message about this subject. After several tries, I still
can't install a boot loader in the hd. Some sanity checks:

* I have SRM, I don't have ARC. SRM doesn't know about disk partitions and
filesystems, so if I wan't to boot from the HD I can't use fdisk at all (eg
no use in making a first partition and putting milo into it).

* It follows that I have to partition using the bsd disklabel program; the
first partition will have an apropriate offset, in which I will install
aboot.

* Life is much easier if you have ARC ;-)

I hope it's ok so far. Now here's where I'm finding problems.

When I enter fdisk, I use the 'b' option to go to bsd disklabel. Unlike fdisk,
it already knows my disk geometry (but it has its share of bugs, also).

After that, the installation procedure runs without problems until it tries to
make ext2 filesystems in my / and /usr partitions. At this moment, mount
complains about not being able to find a directory. I manually unmount
/tmp/sda1.

I click on 'Retry' and the installation goes on for some minutes, then
the machine freezes. Now the console is showing this error message:

init: memory violation at pc = 0000...

Any hints? Strange it doesn't happen with fdisk... This is on an AS255 with
Redhat 4.1, BTW.

Thanks,

Adriano
