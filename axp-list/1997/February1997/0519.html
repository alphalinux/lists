Hi!

I'm considering buying a dirt-cheap ET6000 card with 4.5MB ram to be the
sequel to my old, tired Trio-64 card which for some reason won't even
work with 2MB ram or 16 bpp. Also, 16 bpp implicates <= 80MHz dotclock on
Trio64's where 16 bpp works ok.

As the SVGA server (in which the ET6000 support is) doesn't yet support
any other cards than the Matrox Millenium, this will mean that I will have
to do the porting/fixing of the ET6000 code already available in XFree86.
Now, I personally don't belive that there should be any problem for me
doing this, but I thought I'd hear here first if anyone knows about
anything directly stopping me from doing so.

For example, will the card go through the MILO x86 emulator and initialize
in a usable textmode? This would certainly simplify the work... If anyone
has such a card lying around, prehaps you could try it and se if the linux
console will work with it?

If anyone knows about anything speaking against this project (except that
the builtin DAC in the ET6000 is only 135MHz, this I can live with),
please tell me before I go ahead and buy it instead of the more than twice
expensive Matrox Millenium.
