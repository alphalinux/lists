> I have two shared objects. The first one references symbols only in the
> wish program that loads it. It loads fine. The second object references
> symbols in the first loaded object, and fails to load.

You havn't given me quite enough information, but it doesn't sound
like a bug, it sounds like intended behaviour.

(1) In order for one dlopen'd object to see another, you must give
the flag RTLD_GLOBAL to dlopen.

(2) You could alternatively add the first object to the second object's
link line so that it will automatically be loaded along with
the second. This also adds the first object to the second object's
symbol search path.

Given that neither of these are the case, I have no idea why it would
work on Intel or Sparc, and would seem to indicate a bug in ld.so on
those systems.
