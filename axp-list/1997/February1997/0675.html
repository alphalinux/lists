Machine: PC164, 433 MHz

Being an adventurous type, I've been trying to get the
MILO loader into flash RAM. The recent MILO runs nicely
when loaded through the "fail-safe loader", and allows
fmu.gz to run and program the NVRAM. However, the image
would not boot when I set "bootopt". I erased the ARC
loader and put MILO in its place, but this still did not
boot, and gave a blank screen. Then I erased "Block 0"
which fmu did not label as a booter, and found that
that the "fail-save loader" was gone. The only software
that boots now is the SROM mini-debugger.

I imagine that there is a way to load the fail-safe loader
with the SROM debugger by sending it an image of a file
from another computer. If this is possible, could someone
help by telling me how to do it and where a downloadable
image of the fail-safe loader is, and what SROM commands
I need to set up to load the fail-safe loader into NVRAM?

Thanks,
