Hi!

I composed the src and axp packages of AfterStep 1.0 pre2 (based on
AfterStep-1.0pre2-4.src.rpm by khelekir@encamix.es I found at
oak.sfasu.edu). The bynaries are runing flawlessly on Digital's
Alpha/AXP Multia (a.k.a. UDB). Source package builds without errors
on my RH 4.0 AXP distribution.

Already present patches (from AfterStep-1.0pre2-4.src.rpm) are:
fixes2 patch, locking patch, Animate locking patch.

There is quite a few new patches in the source RPM:

as-10pre2-axp-libs.patch
Fixes the problem with identifying Alpha architecture within Makefile
files so that the right libs are included in linking sequence. This
is Q&D (quick&dirty) patch and it should be implemented in "cleaner"
way.

as-10pre2-cosmetic.patch
I haven't try to build AfterStep on i386 architecture, but there are
some warnings when I try to build it on AXP. This is mostly cosmetic
prototype/casting patch.

as-10pre2-afterstep_err.patch
The function afterstep_err is changed so that it accepts va_list
instead of three char * arguments. This can also be considered
cosmetic patch since there are no more warnings when passing int
as char * argument. MHO is that va_list is the right way for passing
arguments after format string to such functions.

as-10pre2-Animate-PI.patch
VQ&D (very quich & dirty) patch. I dunno where the constant PI should
be defined. So if there is no PI defined, define it form M_PI (form
math.h).

as-10pre2-Animate-XDrawLines.patch
Another prototype/casting patch for Animate. There was wrong type
definition for arguments of XDrawLine fonction (or atleast I suppose
so). Should be checked.

The files are in ftp://sunsite.unc.edu/pub/Linux/Incoming named
AfterStep-1.0pre2-5.*. There is README file, src.rpm and axp.rpm.
There is maybe a new. prefix infornt of them because I had some
problems with upload. They should be moved somewhere around
ALPHA/Alpha directory.

That's all folx!
