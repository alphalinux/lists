Due to negligence at 07:49 PM 2/26/97 -0800, Tony Galan let this roam
freely into my Inbox of Terror:
|>I installed red hat linux 4.0 in my alpha udb, but I don't know how to set
|>up the video. I know udb's have TGA (21030) graphics card, but which
|>Xserver do I use? Can I use Xconfigurator to set up my video? What specs
|>do I need to input?

For 4.0, you can drop by gatekeeper.dec.com/pub/DEC/Linux-Alpha/X11/ and
get XF86_TGA-3.1.2-bugfix-EV4.gz. This server is faster than the server
included with XFree86 3.2 for the TGA, but is not as stable (AFAIK). If
you'd prefer stability over speed, go grab the XFree86 3.2 RPMS that are
floating around various places on RedHat's site (I believe the latest
revision is *-5) and use the TGA server included. (there's also a
XF86_TGA-3.2.bugfix.gz on gatekeeper, you might want that..) Finally,
XFree86 3.2A is a beta version which also includes support for the TGA,
with a little more in the way of speed enhancements over 3.2. It's
available on ftp.xfree86.org and its mirrors, and isn't in RPM form (again,
AFAIK).

Oh, and btw, the version of TGA in the UDB only runs at 8bpp in all
resolutions.
