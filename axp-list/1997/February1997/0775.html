>>> Mark Hamstra said:
>
> I managed to boot up the Red Hat 4.1 install from AlphaBIOS without much
> trouble, but fdisk showed that my existing NT installation had done something
> weird with the disk partitioning.

Yes, I think I've noticed that sometimes the "extended" partitions can cause
problems under LINUX, but never looked into it...

> That's OK,' I thought; 'If NT can't manage
> to partition a disk correctly, it deserves to die. Goodbye.'

So, you ran "fdisk" on it and setup with just primary partitions?

Did you reboot after finishing "fdisk"? This *used* to be necessary, but
I've lately gotten out of the habit, as it seemed to be working OK without
a reboot.

> Red Hat seemed
> to install just fine after that, and rebooted without going all the way out to
> AlphaBIOS just fine. I didn't power cycle all the way back to AlphaBIOS and
> test boot Linux before re-installing NT. I kind of wish I had now, because I'm
> not sure if it would have worked. I can boot NT fine, but if I try to
> select Linux from within AlphaBIOS it gives me the 'Not a valid OS' message
> --or
> whatever it is... my notes are buried on my desk here somplace-- and refuses
> to boot :-(

Is there a chance that the LINUX boot selection still points at the
floppy and you've not got the floppy inserted? Or did you do a "full"
install of LINUX before rebooting, by copying MILO to a DOS partition
and then (re)setup the AlphaBIOS LINUX boot selection appropriately?

During the NT install, it didn't install "new" firmware, did it? The latest
linload.exe *requires* the very latest AlphaBIOS firmware from off that
"windowsnt" website mentioned before, and if it got accidentally overwritten,
prolly with an older version, it'd stop working.

>
> Is there a simple solution or newer software to try?

Dunno; if you got it to work at all with AlphaBIOS, youm prolly have the
latest linload.exe and AlphaBIOS firmware...
