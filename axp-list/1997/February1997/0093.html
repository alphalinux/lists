But even aside from this, there seem to be problems with 2.1.25. On a
more-or-less stock RH 4.0 system (gcc 2.7.2), I encountered the
following errors when linking:

kernel/kernel.o: In function `sys_umask':
sys.c(.text+0x6afc): undefined reference to
`__xchg_called_with_bad_pointer'
sys.c(.text+0x6b00): undefined reference to
`__xchg_called_with_bad_pointer'
net/network.a(ipv4.o): In function `tcp_do_sendmsg':
tcp.c(.text+0x10204): undefined reference to
`csum_partial_copy_from_user'
tcp.c(.text+0x10208): undefined reference to
`csum_partial_copy_from_user'

The latter error I traced (I think) to a typo in include/asm-alpha/checksum.h.
The former, however, seems to trace back to the file
include/asm-alpha/system.h, where it is accompanied by the message:

/*
* This function doesn't exist, so you'll get a linker error
* if something tries to do an invalid xchg().
*
* This only works if the compiler isn't horribly bad at optimizing.
* gcc-2.5.8 reportedly can't handle this, but as that doesn't work
* too well on the alpha anyway..
*/

Wether this points to a bug in gcc optimization code (suprise) or to
"an invalid xchg()", I don't know...

Fortunately, however, 2.0.28 compiled flawlessly for me.
