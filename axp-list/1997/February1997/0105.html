I recently purchased a UDB which I have outfitted with 64 Meg and a 2 G
SCSI drive. I installed Windows NT in the first partition (sda1 to
Linux, the C: drive to NT) because NT4.0 didn't seem to want to boot
from the second partition (D:) when I had it partitioned that way. I
then partitioned the rest of the drive 1440K DOS, ~900M Linux and ~100M
Swap. Now I can't get Redhat 4.0 to boot from the hard drive. I was
using boot sda3:vmlinux.gz which failed with VMS:Kernel Panic : can't
load root fs at 08:02, and after consulting the archives I tried booting
from the cd drive with boot -t iso9660 scd0:kernels/noname.gz
root=/dev/sda3. This failed with an error that the boot disk couldn't be
found. Should I have used the load_ramdisk=1 command on the cd boot?
what about the partitioning scheme? Any help would be much appreciated...
