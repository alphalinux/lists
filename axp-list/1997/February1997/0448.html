We are running a Cabriolet with two DE500 cards, one (eth0) connected
to a 10Mb/s network, the other (eth1) connected to a 100Mb/s
network. This setup did work under Digital Unix. We are using a 2.0.28
kernel with DE4x5 network drivers.

eth0 works fine. When we did "ifconfig eth1 192.168.173.27", it said
first something like "Mode = 100Mb/s". However, the output of
"ifconfig" is:

lo Link encap:Local Loopback
inet addr:127.0.0.1 Bcast:127.255.255.255 Mask:255.0.0.0
UP BROADCAST LOOPBACK RUNNING MTU:7168 Metric:1
RX packets:191894 errors:0 dropped:0 overruns:0
TX packets:191894 errors:0 dropped:0 overruns:0

eth0 Link encap:10Mbps Ethernet HWaddr 00:00:F8:30:2D:4A
inet addr:128.130.173.27 Bcast:128.130.173.127 Mask:255.255.255.128
UP BROADCAST RUNNING MULTICAST MTU:1500 Metric:1
RX packets:6995747 errors:2 dropped:0 overruns:0
TX packets:4330328 errors:0 dropped:0 overruns:0
Interrupt:17 Base address:0xb400

eth1 Link encap:10Mbps Ethernet HWaddr 00:00:F8:00:17:A1
inet addr:192.168.173.27 Bcast:192.168.173.255 Mask:255.255.255.0
UP BROADCAST RUNNING MULTICAST MTU:1500 Metric:1
RX packets:9268 errors:0 dropped:0 overruns:0
TX packets:11 errors:0 dropped:0 overruns:0
Interrupt:18 Base address:0xb800

so it appears to be in 10Mb/s mode. The machine does not talk to the
others on the 100Mbit/s network. Do you have any idea about the reason
for this problem and how to fix it?

Other questions:

1) Writing from a Linux-NFS-client to a Digital-Unix-NFS-Server is pretty
slow (30k/s with 1k blocks, 170k/s with 8k blocks). Is there any way
to improve this.

2) Apparently it is necessary to give the machine two IP adresses
under Digital Unix, depending on the interface. Is there any problem
with using the same IP address for both interfaces under Linux?
