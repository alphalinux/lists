> On Wed, 5 Feb 1997 08:09:45 -0500 (EST), Jim Nance wrote:

> >> libSM.so is a generic X library. XFree86 should have put that library in
> >> the /usr/X11R6/lib directory. It should not be one you need to take from
> >> an OSF (DU) box.
> >
> >I don't think this is correct. You can't mix DU and Linux libraries.
>
> Yes you can,
>
> in fact..... you MUST if you want to run Netscape on Linux-AXP.

What Jim meant was, the DU Netscape application will use *ONLY* the DU
libraries, loaded by /sbin/loader, the dynamic loader for DU. The Netscape
binary (which is ECOFF/AOUT) certainly isn't setup to work with ELF shared
libraries.

However, the X server, being LINUX ELF, will load the ELF shared libraries
that *it* needs.

If libICE and libSM are needed by Netscape, *NOT* by the Xserver, then you
need the DU ones.

It *is* a mix, but only as viewed from a total system aspect; each app
(Netscape and the Xserver) will have loaded only the libraries each knows
how to work with.
