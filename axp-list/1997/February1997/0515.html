Hi!
I got a DEC UDB with 166Mhz, 40MB of memory and everything else like
any UDB. I installed the newest Firmware (from ftp.digital.com),
RedHat 4.1 and Kernel 2.0.29 with NCR53C7xx and FAST-SCSI support. I
attached a HP-DAT-drive to the external SCSI-port. The SCSI-bus is
terminated (passiv).
Now I somemtimes run in one of two strange situations:

A) Everything seems to work, no errors, but when I try to shutdown (-r
or -h) nothing happens after the notify. Even the Bash-Prompt stays
alive. When I then try "sync" it hangs (the other VC continue to
work).

B) Sometimes I get a SCSI-error like eg.:
scsi0: disabled or bad target 0 lun 0
SCSI disk error: host 0 channel 0 id 0 lun 0 return code=25040000
scsidisk I/O error dev 08:02, sector 52894
I can't say anything about the order in which these messages are
printed, because they are being repeated several times.

Does anybody know what to do?

Greetings, Daniel
