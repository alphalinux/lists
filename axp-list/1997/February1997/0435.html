On Tue, 11 Feb 1997, Steven Kokinos wrote:

> When I try to upgrade the rpm, to install some packages from 4.1 I get
> the following error:
>
> rpm -Uvh --nopgp rpm-2.3-1.alpha.rpm
> package rpm-2.3-1 is for a different architecture
> error: rpm-2.3-1.alpha.rpm cannot be installed
>
> Has anyone seen this problem before?
>

I also had this problem. At that time, the installed distribution was
RedHat 3.0.3. I tried to use different switches to the the newer version
of rpm installed. Eventually, when I received my new version of RedHat
4.0, I upgraded my system by using the boot.img that has been rawrited in
a floppy disc. By doing this, I solved my problem.
