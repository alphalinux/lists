>I had 4.0 working about 90% but got greedy and tried installing(not
>upgrading) 4.1. The install seemed to go fine and the system works for the
>most part. But it won`t die! Whenever I try to shutdown the system it
>broadcasts it's warning message and things continue to run. Reboot, halt,
>telinit, and shutdown with the -n switch are all similarly inefective.
>
>Once I logout of a VC, that VC never gets a new login prompt and the only
>way to use the concole is to alt-Fn to a not-yet-used VC.

This all sounds like a problem with your init rpm. Might want to
verify it against the rpm file, or do a --force install. The 4.1
release init rpm is SysVinit-2.64-7.alpha.rpm, though there might
be a newer one in devel. (Doesn't anyone mirror devel?)

If you haven't rebooted yet, the problem is that you're running the
old init, which the new init cannot pass messages to. Shutdown calls
on init to pass messages to the running init process, so shutdown will
hang waiting for init to finish messaging init, which it won't. I'm
not sure why it's not respawning getty procs. cd to /, kill things
manually, umount everything you can, sync a couple of times and punch
it. When you reboot things should be OK after minimal fsck.

-D.
