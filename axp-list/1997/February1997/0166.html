At 12:28 hrs. 02-06-97 +1100, you wrote:
>>>using boot sda3:vmlinux.gz which failed with VMS:Kernel Panic : can't
>>>load root fs at 08:02, Have you tried:
>
>>boot sda3:vmlinux.gz root=/dev/sda3
>
>>If the root does not reside on the same partition as your boot, you need to
>>tell it where to find the kernel. For example, if your boot partition is
>>sda3 and root is on sda 4 then root=/dev/sda4.
>
>Now that's one confusing answer ;-)
>
>In fact, if your root is anything but sda2, you have to tell the kernel
>about it. The ALPHA port has sda2 compiled in as default.
>
You can put root on any partition you want, and boot on any as well. This
has NOTHING to do with the "ALPHA port", althjough it is the default option
for a Red Hat install.
The point is that you need to point to the root partition if it is differnet
than the defaults.
Many people who set up Linux/Alpha also have NT or Digital UNIX or both on
their systems as well, so things end up in all kinds of interesting places.
