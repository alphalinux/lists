After some discussion with David Miller, him relating edicts that
came down from Alan and Linus, I can report that unaligned accesses
related to IPX will not be fixed.

This is because IPX by nature misaligns the IP header within the
ethernet frame, and fixing it would impact everything else that uses
the IP stack, slowing things down about 10%.

So we take the traps and be thankful that it is merely IPX that
is slowed. As for the messages, they can be easily commented out.
