The RH4.1 Network Configuration tool in the Control-Panel under X bombs if
there is a static-routes file in /etc/sysconfig

If you put in the route(s) into the Network Config tool, it writes the file
and uses it correctly. However, the next time you try to run the tool, it
wont. No error message (that I can find)--it just exits. I see it running if
I do a ps but then it simply goes away.
If you rename static-routes to something else, the tool will run.
