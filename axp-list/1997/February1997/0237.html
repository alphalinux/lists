I noticed that on one Alpha I visit from time to time :-) - RH 4.0
with errata - 'locatedb' was not updated despite of a presence
of /etc/cron.daily/updatedb.cron script and proper entries in
crontable. Actually the situation was even worse - an "update"
run but it was leaving a file of a legth zero. Running this script
from a command line did not show up any problems.

Making the script "chatty" resulted in the following in mail:

shell-init: could not get current directory: getwd: cannot access parent
directories
find: cannot open current directory: Permission denied

Still do not know where this script was lost, but adding 'cd /' before
the command invocation solved the issue. Eric, please note for
the future!

All of the above in the case if you wonder why similar things are
happening to you. :-)
