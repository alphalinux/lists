I have a UDB networked to a P100.

The P100 is a gateway to my ISP and
uses IP masquerading. Everything
worked well with Redhat 3.03 on the Alpha.

After upgrading to 4.1 the UDB would hang
going to run level 3 after setting up the ethernet.
While the UDB is hung I can ping it, but no rlogin.

I changed the lo/ethernet so lo is set up first.
I got rid of amd and sendmail.
I added a nsswitch.conf to check hosts first, then dns.
No nis.

The UDB still hangs after settig up the ethernet.

The resolv.conf on the UDB has the ISP nameserver address.

I logged onto the ISP while the UDB was still hung.

After connecting to the ISP the UDB booted immediately,
and worked without a problem.

Therefore, the boot problem on my configuration
is caused by something trying to resolve an IP address
and taking a long time to time out.
