At 11:38 AM 02-26-97 -0500, you wrote:
>>
>> I'm trying to mount an ext2 partition from a linux-intel box
>> ...
>> linuxaxp tells me :
>> VFS: Can't find an ext2 filesystem on dev 08:11
>> mount: wrong fs type etc.
>
>Could this be an endian problem? I seem to remember reading something in a
>linux kernel internals book that indicated that data structures are directly
>written to and read from the disk.
No, I have prepped e2fs disks on an Intel Linux box, and mounted them on
Alpha and it works fine. However, some SCSI controllers do logical mapping
differently. This _might_ be the issue.
To tell:
Run fdisk on the drive on one box, and note the heads, sectors per track
and cylinders reported at the top of the screen. Do a "p" command to see
this listing.

Then mount the disk on the other machine and do the same.
If the configurations are seen differently you might have found the problem...
