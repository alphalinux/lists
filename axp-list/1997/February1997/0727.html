> It gets about 100 Meg of memory, then dies.
> The last address is out of bounds, and I didn't get the -1 error
> indicator.

Hmm. Yep. This is one case where, either through design or
carelessness, we followed Linux/i386 instead of OSF/1. What is
happening is brk(2) always returns the current brk address. If
this is not what you'd requested, you know an error has occurred.

I'll have to investigate what would happen to libc if this were
changed. I think we can get away with it without changes to libc.
