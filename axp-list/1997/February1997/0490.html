>>> "D. Champion" said:
>
> >Apparently MILO sends errors to com1? Can anyone confirm this.
>
> I don't think MILO itself does, but you can configure the kernel to
> do so. I think you can also make the SRM log to a serial device,
> but I don't know what variables you can set or commands you can run
> to alter that behavior.

Yes, the pre-built MILOs on gatekeeper.dec.com (and mirrors) all typically
echo messages and debug info to COM1 (in fact you get more info there than
on the console display). Kernels can also be configured to echo "printk"
messages to COM1 as well. You can build MILO without the echoing, but this
is only to be attempted (currently) under the ECOFF compilation system,
that is, Red Hat 3.0.3, for best results.

If you are using a serial mouse, try COM2 instead for that, unless of course
you are on a UDB without a serial port splitter, in which case you can't...

> I'd dearly love to see in-depth documentation of SRM operation. Does
> anyone know of a source?

Try:
ftp://ftp.netbsd.org/pub/NetBSD/misc/dec-docs/index.html

and look for the SRM users guide under the DS Evaluation Boards. Note that
this is an SRM console description of what is on the Evaluation Boards, and
*may* differ in some significant ways from SRM on other systems that are
not from Digital Semiconductor.
