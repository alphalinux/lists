On Tue, 4 Feb 1997, Adam Forsyth wrote:

> Here's whas happening to me. Whenever I reboot my alpha the arc console
> hangs and says that there are problems that need to be fixed. It
> highlites the time and the boot selections. I can never see anything
> wrong in the boot selections menu, but when I look at the time,
> everything is correct except the year is set to 2077. I then reset it
> and tell it to boot. Durring the boot process I see a line that says
> resetting the system clock, and then the next line states the date as
> being in the year 2077. After its booted if I set the time with the X
> control pannel, it seems to keep time beautifully. My first thought was
> the battery, but only the year is messed up. The hour, minutes and date
> are correct.
> Can anybody offer any suggestions?

Others on the list have already responded to the question on setting time,
so I won't belabor it. The other message drove me nuts for several weeks
(and ought to be in the FAQ!). It occurs when one or more of your boot
selections targets the floppy drive. If you look closely, it makes a
quick access to the floppy at startup. If a disk with the expected system
file(s) is not there, you get the complaint. Only Microsoft could make a
totally uninformative warning so obtrusive! At any rate, once your system
is installed and running properly from the hard drive, just delete any
boot selections which expect a floppy-based load. The warning will go
away.
