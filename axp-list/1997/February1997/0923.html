Sorry for the delay in answering folks...

I compared the performance of 3 FORTRAN (argh) programs between a ppro
200 and a DEC alpha station 1000/200 running DEC's unix 4.0. All
programs were compiled with DEC's fortran f77 or f90 with the -fast
option. In the ppro they were compiled with g77 -O2 or -O6.

One of them is a partial differencial equation solver, which uses a
rather small working set. It runs 3 times faster in the alpha. The
second calculates electron wave functions, and deals with matrix
operations. It runs in the same time in both machines. The third one
uses intrinsic functions (calculates the exp of a whole vector) as
well as arithmetic operations. It traverses a 5MB array both along
rows and columns, so it needs some memory. This one runs 2 times
faster on the PPRO!

It seems the performance of the alpha is very dependent on the amount
of memory used. If the program fits in the secondary cache it flies,
if not... This is a consequence of the high clock speed.

I also measured the time to compile zsh on both machines, and the ppro
is 2 times faster (using gcc -O6 on the ppro and DEC's cc -fast on the
alpha).

If your tasks can be paralelized, ie. distributed simultaneously among
several machines, and they use more memory simultaneously than it fits
in the secondary cache of the alpha, I think the ppro may have a
better price/performance ratio. Otherwise the alpha is faster.

We'll be receiving a 433 machine with 2MB cache from Microway soon and
I'll compare the performances again.
