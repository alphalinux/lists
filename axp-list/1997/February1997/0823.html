Anyone else having problems with ypbind? My machine is an AS255 running Redhat
4.1, kernel 2.0.29 with the latest glibc and netkit in updates.

When I start it, it works ok for about one hour, then the error messages start
to appear, and in three hours only root can log in. The messages are:

ypbind: RPC: Unable to send; errno = Resource temporarily unavailable
ypbind: clnt_create for server failed

and, eventually:

curry kernel: ypbind: memory violation at pc=155557ccd44 rp=155557ccc0c (bad
address = 15500000018)

My nis server is running SunOS 4.1.3, and works fine with Linux i386 and
Digital Unix (and other Suns, of course).

I compiled a new ypbind-3.0 (with the nistools patch for glibc) but the
results were the same, as expected.
