Adriano Nagelschmidt Rodrigues <anr@ime.usp.br> writes:
> Michael Schwingen writes:
> > Exactly this. aboot wants a disklabel, not a fdisk-style partition table.
>
> Meaning that I can't use aboot if I used fdisk?

Yes, at least not on the boot disk.

> How to put a boot loader in the HD, then? Do I create a dedicated
> first partition and dump milo into it?

No. You don't need MILO, once you got aboot installed.

> What's the best way?

In case you got two hard disk, use one with fdisk as usuall for
booting, and use 'minlabel' to partition the other one, then install
aboot accourding to the instructions on it. It will be bootable, if
you follow the instructions. (SRM-Howto?)

If you are able to boot you system from a floppy (with MILO), do it,
then install aboot, and you're OK.

In case you can't boot from an floppy for some reason, I'd guess the
same tools as for the Jensen install apply. (With a different kernel
image, of course)
