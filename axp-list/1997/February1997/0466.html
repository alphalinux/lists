> Advice on cross compile from alpha-dec-osf3.2 to
> alpha-unknown-linux and alpha-unknown-linux to alpha-dec-osf3.2
> is welcome. What includes and headers are needed? Do I have
> to patch on both directions? How to cross compile ld, as and
> especially ar?

Cross compiling with gcc generally works best if you do bits like

configure --prefix=/usr/local --host=alpha-dec-osf3.2 --target=alpha-linux

This will look for cross utilities in /usr/local/alpha-linux/bin,
system headers in /usr/local/alpha-linux/include, libraries in ...

Cross compiling to linux is easy -- you need all the glibc headers
and libraries. Cross to osf -- dunno. You shouldn't have any
problems if you grab all of /usr/include. Oh, binutils doesn't
understand dec's shared libraries, so you'd have to build static there.
