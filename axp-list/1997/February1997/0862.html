Hello List,

I got my hands on a PC164 at 433Mhz. We are running Red Hat 4.1 +
updates on this machine. I can crash the machine reliably by running:

rpm --verify -a -v

When I do this I get several minutes of hex display on the screen.

Sometimes the machine reports "CIA machine check" and halts. It
crashes a couple of times before being able to compile a complete
kernel.

I got the 2.0.29 kernel on that machine and I got an Oops that
only told me that the problem was an access to 0x0 from the schedule
function:

Unable to handle kernel paging request at virtual address 0xfffffc00009dbb29
PC = 0xffffc000031b020
ps = 7
vp = 0xffffc000033e530

Code: 47ff041f 47ff041f 47ff041f <a4690000> a5290008 e460002d a4230000

>From the System.map file, the PC points somewhere in the schedule
function.

Since I got a CIA machine check, I turned on debugging in the
arch/alpha/kernel/cia.c file and got this while booting (it does not
even boot with this kernel):

Probing PCI hardware
mk_conf_addr (bus=0, dev_fn=0, where=0xe)
pci_addr= 0xfffffc0000303f20
type1= 303f28
returning = 0xe
conf_read (addr=0xfffffc87000001c0, type1=0)
conf_read: CIA error was 0x0
cia_machine_check: vector=0x670 a_ptr=0x5a0 pc=0xfffff000003199b9 size=8

The machine has 2 SCSI disks on a QLogic card that also has the
ethernet board. This machine is the Durango model.

Any idea on what could be wrong with the machine?

cheers,
Miguel.
