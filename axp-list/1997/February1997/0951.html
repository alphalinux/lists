Before we rush out in mass and request that Netscape provide a port of
Navigator for Linux/Alpha, please consider that some basic tools they need may
be missing.

Currently, the only version of Motif available is the static ECOFF 2.0 from
Metrolink. Of course, Netscape *could* build an ECOFF version, but it would be
fairly large. However, since ELF/glibc is still a changing target (perhaps it
has settled down now), ECOFF may be the best way to produce a binary that would
be compatable with all Linux/Alpha distrbutions. No doubt most of us
are using ELF/glibc now, and we would probably prefer to see a shared library
version, which means a couple of things:

1. glibc must stabilize so that a shared binary has a reasonable chance of
working with the next version of the libraries.

2. With (1), Metrolink would need to release a more modern version of Motif
2.0 (I assume Netscape is using Motif 2.0) for Alpha/Linux that would
allow one to build ELF linked binaries, perhaps statically linked to Motif for
distribution, but compatable with glibc.

My gut feeling is that it is premature to start to pester Netscape about a
Navigator binary, unless we are willing to accept an ECOFF version. Let things
settle and stabilize a bit. I think getting Metrolink to produce an ELF Motif
is key to porting Netscape and many other commericial binaries.

Just my $0.02

David
