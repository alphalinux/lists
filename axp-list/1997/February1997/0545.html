ever since I upgraded to 4.0/4.1 the pgm called 'suck' has crashed with
a seg fault.

I finally got fed up( gdb/xxgdb doesn't exactly work well with stack
traces!) , and tracked it down to an fclose() call.

ie: fchan = fopen(); /* works */
fclose(fchan); /* works */
fclose(fchan); /* segfaults */

I suppose that there are other pgms that potentially follow this
process, and maybe there would be a fclose() that recognises a bogus
pointer/close channel.

Ok, so Who's in charge of this code, so I relay my findings?
