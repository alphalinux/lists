Robert Williams <bob@bob.usuf2.usuhs.mil>
> Still trying to upgrade a PC164 from 4.0 to 4.1,
> after I'm asked to identify the root partition,
> the install system attempts to look at the existing
> rpm data base and reports:
> "No RPM data base".
> Is this related to the way I've partitioned
> the disk, with "/" and "/usr" on separate
> partitions? Is there a workaround?

Did you move /var somewhere else and symlink it? If so, make sure that
you use a relative symlink rather than an absolute. For instance, on
my machine /var was symlinked to "/usr/var", which caused problems for
upgrades. The fix was to change the symlink to "usr/var".

The reason this is a problem is that during upgrades your root partition
is mounted under /mnt. Since the RPM database is in /var/lib/rpm, it
shows up in /mnt/var/lib/rpm.
