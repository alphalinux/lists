Hi.

Because I've been extremely busy lately, and I can't see it lowering soon,
I have to drop some stuff I've been "working" on.. xemacs on linux-alpha
being one.

Several ppl sent me patches when I asked about it a while ago. But they
still doesn't work with the new glibc. Some1 suggested for xemacs-20.0
beta coming. It came out but fixed nothing. I started debugging xemacs
and found out that the size field in (Lisp_Vector) Vobarray(symbol.c)
being uninitialized and when this is used, xemacs goes caboom..

If some1 else can give this a go, please do. Having xemacs would be
a nice thing.

ftp://nova.campus.luth.se/pub/linux/patches has:

xemacs-linux-alpha-elf64-20.0.diff
----------------------------------
Makes xemacs compile on linux/alpha using elf64 (and glibc). Though,
it "doesn't work, see above". It should apply on xemacs 19.14 also.
This doesn't need the unexelf1.c from emacs. A small gotcha though,
like some others have noticed, if you have MATH_ERR defined (which
configure does define) floatfns.c does not compile. A simple hack
is to have NO_MATHERR defined. I haven't really investigated what
the consequences of this will be.

xemacs-20.0.spec
----------------
A simple spec file for supporting linux/alpha. Hopefully it can save
you some mins from writing it yourself.

Good luck, and sorry I didn't have time to help out with this.

/Mikael
