I just recently installed the Redhat 4.1 Alpha/Linux
on a 166MHz Multia with 32Mb of memory and the standard
DEC hardware and once again the Linux world amazes me!
But I've found a condition that has me baffled and I've
not seen it discussed in the list archives; The
setup is to power down the alpha and start a ping
between from a Sparc/Linux box to a Intel/Linux
system and everything looks ok with the normal
ICMP exchanges. Then when I boot the alpha,
as soon as the eth0 device is configured and
activated, the other hosts start reporting exactly
4 duplicate packets. Now the real kicker..this
condition will persist until I run tcpdump on
the alpha which puts the interface into promiscuous
mode. Once tcpdump starts on the alpha, the other
hosts return to the normal ICMP echo request-echo
reply sequence. If I stop tcpdump...the dupe's
start again.

I've triple checked the address assignments and
masks (mask of 255.255.255.0) and it looks
ok to me.

Anybody noticed/experienced this characteristic or
care to shed some light what might be causing it.

Thanks!
