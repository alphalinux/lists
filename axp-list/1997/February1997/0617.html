Tom Dropka <dropka@aptec.com> writes:

> I've got an AS/255 with 32 MB of memory, but neither the SIMMs nor
> the Technical Guide say whether or not true x36 parity memory will
> work. I need to add at least 32 MB more, but the existing SIMMs
> have a 2Mx33 tag on them. Will the 255 use standard x36 SIMMs, and
> if so, does it just ignore the "extra" few bits? (The manual talks
> about a 128-bit data path + 4 parity bits.)

Yep, the AS/255 will happily accept any true parity 72-pin SIMMs, as long
as they're in groups of four (I think...I'm not 100% positive that two
won't cut it, as I've always added four at a time).

The machine I'm posting this from is an AS 255/233 with 4 16MB SIMMs
factory-installed and 4 8MB SIMMs from Kingston.
