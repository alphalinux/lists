Hi once again,

> 2) Apparently it is necessary to give the machine two IP adresses
> under Digital Unix, depending on the interface. Is there any problem
> with using the same IP address for both interfaces under Linux?

> Michal Jaegermann told me that each interface must have a separate IP
> address.

I would argue a bit - I am using the same IP on both eth0 and eth1 and
it seemes to be harmless... But I wondered a lot about this, so please
correct me if I am wrong.

The only slight problem is that RedHat net config tool is a bit
confused by two identical IPs, but once I set up proper routing manually
everything works fine.

I thing the only reason for 2 IPs is classical simple routing setup in
subnetted networks, but if one does not adhere to it, IPs can be the
same.

In fact I am not using routing but bridging, so there could be another
problem that both interfaces reply to one ARP query (which could
traverse the bridge), but in reality this is fine as well.

However my approach is questionable, so I would like to hear from
somebody willing to do it in similar way... :)
