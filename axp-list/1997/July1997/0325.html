And if you can get 1280x1024 X working with the original XF86Config from
the readme, you can get 160x128 on your UDB. I am not going to do MILO,
but you can extrapolate from this and the previous patches :).

There is probably enough in these patches to allow an ioctl for the
equivalent of SVGATextMode (also see XFree86 TGA driver source for which
bits go where), and even loading of fonts if the array holding the
default8x8 font is expanded, and the cursor is adjusted.

NOTE: XF86_TGA does not return to this mode correctly and I don't know how
to get it reinited properly, so once in X, stay there.
