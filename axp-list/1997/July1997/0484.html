I have a Diamond Stealth 64 working just fine in my alpha, a PC164/500
with Linux 2.0.30. However, when I add 3c509 ethernet board (as eth1)
the XF86_S3 server fails immediately with a message rather like

.... failed to set IOPL for IO ...

(sorry, it's not in front of me at the moment.)

This is odd. The 3c509 is an ISA board, and the video is a PCI board. It
seems hard to believe there are any resource conflicts between the two,
yet I cannot start X when I have the 3c509 installed.

There must be some way to make it work because the computer was orriginally
shipped to me (by DCG) with the 3c509 installed, and it seemed to work
then.
