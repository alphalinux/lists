recently Jay++ posted:

>Please try the MILO located on gatekeeper.dec.com as:

> pub/DEC/Linux-Alpha/Kernels/milo-2029-pc164

>This may help with the memory problems; it also will boot
>2.1.xx kernels that are too large for older MILOs.

>Give it a try, if possible, and let me know what happens.

>--Jay++

We're using a bunch of PC164s w. 256M (RH 4.2, kernel 2.0.30)
to run very large array computations. Until we tried the above
mentioned milo we got frequent and fatal memory violations,
unless we were running X (X presumedly reduces the per unit time
memory access). After we started using this milo all our memory
aberations thankfully vanished.
