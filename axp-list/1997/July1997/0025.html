At 03:42 PM 7/1/97 -0700, you wrote:
>O Linux-axp.Gurus!
>
>Having spent over two hours looking for information, I've no recourse but
>to throw my query out into the cyberwind, to wit:
>
> Does RHL 4.1 (or 4.2) have support for PCMCIA cards?
>
Both versions do with a bit of work. RH4.1 worked with pcmcia-cs-2.8.23 from
hyper.stanford.edu (although the xircom and smc drivers failed to compile).
RH 4.2 is working for me (at the moment) with pcmcia-cs-2.9.5 (although I
had to use the insmod binary from 4.1 because the stock 4.2 insmod is
broke). Both versions required doing a 'strip -x *.o' on the compiled pcmcia
modules when put in /lib/modules/2.x.y/pcmcia. Oh, the sources from
hyper.stanford.edu don't look in the right places for the xforms stuff with
RedHat's file setup but that's an easy fix. Also, I modified my network
startup script so that it would crank up the eth0 interface regardless of
whether pcmcia was set. I'm using the UDB as a gateway/firewall for my PC's
at home (so I dont have to give my 11 yr old my login passwd...:).

>As y'all know, the Multia comes with two PCMCIA card slots. I want to
>explore using a PCMCIA modem to hook up to my ISP, but I haven't a clue as
>to how to go about it. (Once you hear that it took me five installs to get
>everything going, you won't be surprised :-)) In the same vein, assuming I
>can get the PCMCIA modem working, are there any strong opinions about using
>it vs. using an external modem?
>
So far have been using 28.8 pcmcia modem card without too much hassle. The
network configurator tool RH packages with 4.1/4.2 is fairly painless to use
for doing ppp dialups.

Has anybody tried an ISDN pcmcia card in the slot?
