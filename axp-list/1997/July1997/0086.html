On Thu, Jul 03, 1997 at 12:04:44PM -0400, Matt Stocum wrote:

> I was wondering if the 433a can run linux yet. My understanding was (from
> the faq) that pretty much everything worked just milo hadn't been ported
> so it was impossible to boot the kernel.

The FAQ is outdated :-)

MILO has been ported and can be found on gatekeeper.dec.com in
/pub/DEC/Linux-Alpha/Milo (or so). You only need a "miata" kernel and the
AlphaBIOS linload to get it work.

As I understand, RedHat 4.2 ships with a suiting bootdisk and kernel which
does everything but support the onboard Ethernet chip. Support for it is
available, it just came a bit late for RedHat -- use a kernel from
gatekeeper or get the latest tulip.c from Donald Becker's page and compile
a kernel yourself.

> Also, would there be any problem
> having both windows nt and linux installed on the same hd?

No, I had a setup like this -- that is, before NT committed suicide :)

Parition your disk, install NT (which gives you two entries in the boot
menu) and install Linux then without touching the NT partitions. Then,
copy linload.exe and milo on the NT boot partition (or on some other FAT
formatted device), add a boot menu entry, make it the default :) and off
you go.
