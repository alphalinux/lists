> I'm trying to compile MILO-2.0.30 in AXP noname to be used
> in AlphaStation 600A 5/500 machine and have few questions:
>
> Do I first compile configure and compile kernel with
> drivers needed in AS600A?

Yes, build a kernel first.

> And what features and drivers are mandatory in that
> kernel?
>

It's really the SCSI drivers and the interrupt handling/machine
specific stuff that MILO wants/needs.

> Then compile MILO, but how to tell it that I want
> milo for AS600A, because it compiles MILO for
> nomame as default if I'm right?
>

Use the command

make KSRC=where_your_as600a_kernel_tree_is

> And MILO-HOWTO states, that i should answer NO to
> question:
>
> Use Digitals BIOS emulation code (not free) (MINI_DIGITAL_BIOSEMU)
>
> Is that correct?
>
>

Not anymore. The latest (2.0.30) MILO sources now build on ELF as the
DIGITAL BIOS emulation library is now in ELF object file format.
