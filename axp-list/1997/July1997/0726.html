Perusing the mailing list archives, I noticed a lot of activity about
my problem two months ago, but I seemed to have missed the resolution.

I've compiled my own copy of milo ( we want to hack on it, but first I
figured I should get the plain vanilla copy running ), version 2.0.30
from decs site. Compiled fine against the 2.0.30 kernel, with the
patches from the dec sight applied. During boot, however, it freezes
after

Swapping to PALcode at 0x0 ...

I saw that some thought it might be a video card problem, but I've
tried both a PCI S3 based card, as well as an ISA trident VGA only
card.
I also read that it might have to do with the version of linload that
you are using. I've tried both 1.3a, as well as 1.5. I've also
dd'd milo.dd onto a disk, and tried booting via SRM. There, it says
swapping, sets the cursor back at the top of the screen, and freezes.

Does anyone have any idea why it does this? Any help is greatly
appreciated.
