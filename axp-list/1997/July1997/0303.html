On Jul 10, Eric Hilton Jones wrote:

> Has anyone experienced flakiness running redhat 4.1 and fvwm2-95 with
> the s3 server? I have a multia 166 with a powergraph 64 (STB) card, and
> ever since I made the move from 4.0 to 4.1 I've had to use the built-in
> tga card instead of my s3. Here's some system information:
>
> Multia 166, 32mb ram
> Powergraph 64 s3 card
> kernel 2.0.30
>
> I've tried using the fvwm2-95 from 4.0 to no avail, and I've even
> installed the server and libs from XFree86-3.3. My machine locks up
> completely, no ctrl=alt-backarrow or ctrl-alt--del will fix it.

use the XFree86 3.3 S3 server and add

Option "no_pci_disconnect"

to the device section of the config file...

maybe 3.3 will work even without this option, but I don't think so.
please try and let us know...
