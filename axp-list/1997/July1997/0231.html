I was having the same partitioning problem as the other fellow, so I
followed his advice: I took the drive over to a Pentium. Surprise, the
disk geometry was entirely different, and I had no trouble
partitioning it. When I brought it back to the alpha, unfortuantely,
it is unable to reliably write to it for very long, i.e. if I mke2fs
on a big partition, it will freeze somewhere in the middle. If I
do a redhat install without having it run mke2fs, it starts installing
packages but again freezes after a few 10's of megabytes. So perhaps
I have a hardware problem as well; I'm taking it up with the system
vendor.
