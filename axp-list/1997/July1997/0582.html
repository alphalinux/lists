Hi,

I'm using a
pc164/256MB,
scsi NCR810,
ether ne2000,
kernel 2.0.30 (pristine w/ latest patches),
RedHat 4.1 w/ many packages from 4.2.

Whenever I reboot linux (shutdown -r now), the system goes down like
it should, but at some point just hangs, with some Milos (like the
current pc164 milo2030 from gatekeeper) right after the

'Please stand by while rebooting the system'

-Message, with other Milos (like the one I made from the newest
milo-2.0.30 sources) after going back to the Milo prompt, counting
down from 30 to 0 and then after

scsi-ncr53c7,8xx : NCR53c810 at memory 0x9000000, io 0xb400, irq19

it just stands there.

I never run into problems when starting up the machine, booting for
the first time, by e.g. doing a hard reset.

I tried to upgrade my milo to one I wanted to compile with the newest
ncr53c8xx-driver (Version 2c I think), but I didn't manage to get milo
to link with mm.o (ncr53c8xx seems to need that one, it calls vfree
etc...).

The lx164 milo (yes I even tried that one) seems to be compiled with
that driver code, so could someone possibly tell me how to do that?

Another problem I ran into, and it might be related to the above, is
that the kernel sometimes fails to detect my ne2000's irq-line...

I need to have a machine that I can reboot remotely (w/out laying my
hands on the reset taster), so I'd really be grateful for any helping
pointers.

thanks in advance,
