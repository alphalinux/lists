>>> Steve Borho said:
> The mailing list search feature doesn't seem to work today so I'll just
> post this question. Like the subject says, I've got a Matrox Millineum in
> an XL300 that works well in 8, 16, and 32 bpp modes but locks up tight as
> a drum if I try 24. I think I remember this as being a known bug but wasn't
> sure. AFAIK I'm using XF863.3, but it could possibly be 3.2 (whatever ships
> with RH4.2)

YES! This is a known limitation of the Millennium (and prolly Mystique)
support in the SVGA server from 3.2 *or* 3.3 (3.2 is on the RH 4.2 CD, but
3.3 RPMs are available in the updates/ directories). Note that the Mystique
is supported ONLY in 3.3.

There's too many instances of length assumptions in the 24-bit code at the
moment (ie sizeof(long) == 4, as on INTEL); it would take a *lot* of work
to fix them all, for relatively small payback. Sorry...
