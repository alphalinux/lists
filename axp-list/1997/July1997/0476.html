> I've been attempting to install the latest Enlightenment beta on my XLT300
> with Matrox Millinium and XFree86 3.3 with RedHat 4.2. I think I've located
> at least a part of the problem. The E graphics library imlib attempts to
> use the MIT-SHM extensions, which appear to be broken with the stock RH4.2
> kernel tree source.
>
> I have downloaded a pristine 2.0.30 kernel tree from sunsite and the latest
> axp-diffs from gatekeeper. My question for the list is this; Do these
> axp-diffs fix the MIT-SHM mismatch bug, or do I need to backup to some older
> kernel/XF86/libc combination?

I hate to reply to my own post, but I want to make this easy for anyone in
the future to look this up in the archives.

Compiling a pristine 2.0.30 linux kernel tree with the alpha-patches from
DEC's gatekeeper server did fix the problem. I can now use the shared
memory extensions in RH4.2 and XF86 3.3. Enlightenment now runs just great.
