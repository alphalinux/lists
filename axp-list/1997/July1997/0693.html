On Wed, 30 Jul 1997 wsb@VNET.IBM.COM wrote:

> I don't know much about firewalls but I did read the manual that comes
> with RH 4.2. It explicitly states modules are not supported. You must
> use a monolithic kernel. I know some people have gotten modules to
> work by using previous versions/patches of the relevant source code.
> If you have those fixes great, but if you're using the standard kernel
> source from RedHat, I don't think inserting modules works. Note this
> is a Alpha specific problem.

I am running Red Hat 4.2 on a UDB with the Masquerade modules functioning
perfectly. I had to upgrade to:

modutils-2.1.34-1 and (this is RH 4.2)
binutils-970127-1

to get them to work. I also don't run kerneld, I /sbin/modprobe each
module in /etc/rc.local Then set up my ipfwadm rules. So if you get the
new binutils and recompile your kernel it will work.

The Masquerading modules WILL NOT work built into the kernel. I tried it
first.

Cheers, Rick.
