I installed Redhat 4.2 on a PC164, because many directories and symbolic
links with Redhat 4.1 had a date with the year 2017. Unfortunately, the
kernel used to install Redhat 4.2 also thinks it is 2017 and installs the
directories and links with the wrong date. Does anyone else have this
problem with a PC164? Also, is there anyway to set the clock in the
install phase without getting a message like, "cannot access I/O port"?

The wrong dates wouldn't bother me, but I think they confuse RPM when
upgrading packages.
