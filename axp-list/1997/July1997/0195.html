Hello,

Here are my notes on getting RedHat 4.1 to run on a AlphaStation 500
Hope it helps.

Configuration Notes on DEC AlphaStation 500 / 266

Model PB540-A9
Processor Type 21164
SCSI Adapter Qlogic SCSI ISP1020 revision ID (2)
Video Adapter Cirrus CL-GD5424 512Kb Ram (checked with SuperProbe)
Network Adapter ? (On NT displays 21040 Chip)
Memory 32Mb

CDROM Drive:
Harddisk: 1.xGb
Floppy Drive: 2.88Mb

Installing RedHat 4.2 CD dated xx.xx.97
Kernel 2.0.27

- From DOSBox go to the Image directory on the Alpha CD.

- rawrite the ALCOR.IMG to floppy

- rawrite the RAMDISK.IMG to another floppy

- Configure the AlphaStation 500 machine to VMS or Digital UNIX, SRM boot

- >>> boot dva0 -file vmlinux.gz -flags "root=/dev/fd0 load_ramdisk=1"

- At remark "VFS Insert RAMDISK" do so

- Pop in the CD in the drive

- Start configuring.

- After install is complete use the ALCOR.IMG disk to boot, but with
different parameters
boot dva0 -file vmlinux.gz -flags "root=/dev/sda1"

sda1 depends where your root partition is.

- This makes my AlphaStation 500 / 266 run in terminal mode.
I tried to start the XFree XTga driver from the X directory
of the RedHat 4.1 CD.
X -probeonly gives no errors but when I do startx the monitor goes black
and I have to use the reset button to get the SRM prompt back.

Secondly I was unable to compile the Tulip driver into
a new kernel, due to missing symbols at link time.

At the moment I have given up to try to get this running, because I
get the feeling that the "resently" released 4.2 might do better.
