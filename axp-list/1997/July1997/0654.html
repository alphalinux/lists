Hi everyone,

I maintain a 500 MHz PC164 Machine from Quant-X with 128 MB, but only
an IDE disk (WDC AC3400L, 3815MB w/256kB cache, LBA, CHS=7752/16/63).
Getting a SCSI subsystem is high(est) on my wishlist, but I will have
to live with only the IDE disk till the fall. My question is simple:
How can I make my system more stable and, possible, speed up disk
access.

I finally rebuilt a kernel with the latest Alpha patches from
gatekeeper.dec.com and upgraded to the corresponding milo (I couldn't
compile milo myself, but that's a different posting), which I hope
will fix most instabilities of the system. However, since upgrading
the kernel I had already one system crash when a paging request could not
be fulfilled. (This happened while gzipping -9 a 25MB file)

While I can live with a slow disk system, I need to stabilize the
machine. Somehow, I suspect that the IDE disksystem misses
paging/swapping requests (sorry, this is probably not the correct
technical way of expressing what happens), and this leads to a
system crash.

What I am, therefore, wondering is whether during kernel compilation
and/or using hdparm I can improve the performance of the disk system
(or, alternatively, make it fail-safe even if it gets slower).

I tried

hdparm -u 1 /dev/hda

from the Alpha FAQ, with no noticeable difference, but this addresses
a different problem anyways. As hdparm has plenty of options, I am
curious whether someone has already experimented and could provide
some hints.

As to kernel compilation options, I only configured the new (E)IDE
driver without any of the options to handle known bugs (but these are for
chipsets used in Intel PCs, right).

Obviously, I'll try other suggestions as well (as I said, we'll get
a SCSI disk, but not until the fall :-()

Thanks,

Stefan
