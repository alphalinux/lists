At 06:03 hrs. 07-19-97 -0400, you wrote:
>Hello All,
> I have recently bought a DEC UDB. It has no hard drive in it..
<<snip>>

>I really want to try and stay internal, as this computer will be ported
>about quite a bit and so I would like to keep it all together. Also keeping
>the drive internally simplifies 240/120v conversion which I will need as I
>will use it both sides of the Atlantic.

Don't. It may be more portable, but you give up too much.

>I would like to keep the PCI slot free if possible so I can put my S3 card
>in there in the future.

Good plan.

>This leaves 2.5" drives:
>
>The highest capacity 2.5" SCSI drive I've found is a 800Mb Toshiba one
>(please let me know if anyone knows of any higher capacity 2.5" ones).
>Whereas I can get a 2Gb IDE Toshiba 2.5" drive for the same cost. Although
>SCSI is traditionally faster than IDE, does this hold with 2.5" drives?
>>From the specs both drives have similar av. seek times (about 13ms).

All the 2.5" SCSI drives are fairly slow, low RPM, have a tiny onboard
cache, and don't go very fast.

>The machine is going to be used as a personal workstation, doing mainly C,
>Latex, Postscript, telnet, Netscape type stuff. No heavy number crunching
>or disk IO.
>
>So what do you think? Is the SCSI drive going to be significantly faster
>than the IDE drive for my requirements? Will I fit RedHat 4.1 comfortably
>into 800Mb with room for 100Mb worth of data files etc? If I need to I may
>invest in an external Zip drive for archiving work on.

No, it isn't going to be much faster if you get a 2.5" one.
Get a 3.5" drive, in a small external SCSI case. There are lots available
that are quite tiny, and the difference in price and performance is
substantial.

My $.02 worh on the subject...
