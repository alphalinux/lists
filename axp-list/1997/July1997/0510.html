> Not really. If you avoid shoddy programming practices and include proper

The interface to accept() has a (int *) in the manpage which is a
(size_t *) in the include file.

Other OSes, not to mention Redhat 4.1/Intel, have int *. There is no
easy way to write compatible code for both cases.

This is a real issue. I'd bet that it was done on purpose as there
are many things in the bsd interfaces which ought to be size_t's.
But we need a graceful plan for migrating and some explanation, so
I can explain to my boss why my code breaks on this new OS revision
but it's not my fault.
