Hello,

I'm currently using my Alpha running Linux as a firewall between the
internet and my private network. Most things work, but I'm trying to get
FTP working. I activated my firewall using the following commands (my
real internet IP-address is a.b.c.d, my private network uses the addresses
10.0.0.x with a netmask of 255.255.255.0, the IP-adress at the private
network side of the firewall is 10.0.0.1):

/sbin/ipfwadm -I -p accept
/sbin/ipfwadm -I -f
/sbin/ipfwadm -O -p accept
/sbin/ipfwadm -O -f
/sbin/ipfwadm -F -p deny
/sbin/ipfwadm -F -f
/sbin/ipfwadm -F -a masquerade -P all -S 10.0.0.0/24 -D 0.0.0.0/0

This is run after both networking cards are configured using ifconfig and
route like this:

ifconfig eth0 a.b.c.d netmask 255.255.0.0 broadcast a.b.255.255
route add -net a.b.0.0 netmask 255.255.0.0 dev eth0
ifconfig eth1 10.0.0.1 netmask 255.255.255.0 broadcast 10.0.0.255
route add -net 10.0.0.0 netmask 255.255.255.0 dev eth1
route add default gw a.b.p.q dev eth0

Then, in order to get FTP working, I insmod the relevant module:

/sbin/insmod /lib/modules/`uname -r`/ipv4/ip_masq_ftp.o

But still, from within the firewall, FTP only works using passive mode,
but this module should enable active FTP as well.

It worked when the firewall was an Intel machine, but that changed
recently.

I'm using RedHat 4.2, with a self-compiled kernel with as few modules as
possible (all usually needed stuff is compiled in, only the ip_masq_*
modules as left, of which I only use the ip_masq_ftp module. The kernel
is a pristine 2.0.30 with axp-diffs applied.

Does anyone have an idea? Did anyone get this to work? I could use a few
hints :)
