Thus spoketh wsb@VNET.IBM.COM (wsb@VNET.IBM.COM):

> Henri,
>
> I have a very similar system to yours and I followed some advice I saw
> on axp-list a couple days ago and it seems to have fixed the problem
> you described and which I also had. When you get to the milo prompt,
> enter 'set MEMORY_SIZE=127' to reduce your effective memory size when
> Linux boots. I've been running a couple days now that way while testing
> my tulip Enet card and have not had any problems since I did this.
> Once the change is made, use your normal boot command to start up
> Linux. Note I'm using a 2940UW card so changing SCSI cards is not
> going to cure the problem.

I tried this at it seems stable now. But still, I don't consider this a
fix since I anticipate that I have to upgrade the memory to at least
256Mb in near future so I can run some large numerical codes...

So the questions remains: why doesn't it like the uppermost 1Mb?
