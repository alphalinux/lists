I tried to get my Linux/alpha machine to support 2 ethernet cards and
I ran into some difficulties.

The ifconfig for eth0 works fine.

The ifconfig for lo or eth1 hangs. I see by strace that it is sending DNS
messages full speed, not waiting for replies. It is doing a select with
aa zero timeout. Presumably it doesn't do this for eth0 because glibc
notices that there are no routes to any DNS server.

If I edit /etc/nsswitch.conf so that only files are searched, then
ifconfig complains that 127.0.0.1: host not found or whatever for all the
ip host and network addresses I supply.

RedHat Linux
2.0.30 kernel for alpha
net-tools-1.32.alpha-2
glibc-0.961212-4
The ethernet cards are a DE450 and 3c509. These seem to be OK.
