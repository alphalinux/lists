Cheers,

If this is an FAQ I apologize. I looked but couldn't find it. Any help
offline or online would be appreciated.

Been using linux for about 3 years on intels in a production environment.
Recently found a well endowed Alpha machine with no documentation -
willing to grovel for assistance. (The machine runs, and runs NT 3.51 well
- has one of those ARC consoles) On the front of the case it says "AXP 275
ACTion Systems". Net search found nil.

I opened it up and it looks like a smaller AT Type case, with 3 ISA and 3
PCI slots. There is a long row of jumpers along the left side of the
motherboard and 3 unjumpered jumpers to the right of that. A ROM chip
close to the back of the case says "Aspen Systems - Alpine (c)1995, V3.3".

The machine has what appears to be 2 SCSI controllers (QLISP 1020 and
NCRC810). I tried all the milo images till I got one that got past the
PALcode reload. Got to milo prompt with the "cabriolet" image. But there I
get a SCSI timeout error and a lockup. Any command at the milo prompt such
as "boot" or "ls" or "show" locks up with the following:

dev_config:fixing up latency from 0 to 32 for 1011:2
dev_config:fixing up latency from 0 to 32 for 8086:484
dev_config:don't know how to set hostid of SCSI dev (1077:1020)
dev_config:fixing up latency from 0 to 32 for 1011:2
dev_config:setting SCSI 0 hostid to 7
dev_config:fixing up latency from 0 to 32 for 1000:1
no environment present

# (milo built against 2.0.29)
# (so I give it an ls -t msdos fd0:)

floppy drive:fd0 is 2.88MB
floppy0:no floppy controller found
scsi-ncr53c7,8xx:at PCI Bus0, dev 5, function 0
scsi-ncr53c7,8xx:NCR53c810 at mem ox2001000, io 0xb800, IRQ 18
scsi1:burst length 8
scsi1:NCR code relocated to 0x4009ae14 (vitual 0xfffffc000009ae14)
scai1:test started
scsi1:driver test1 timed out due to lost interrupt.
please verify that correct IRQ is being used by your board,
and that the motherboard IRQ jumper matches the PCI setup on
PCI systems.
If you are using an NCR53c810 board in a PCI system, you should
also verify that the board is jumpered to use PCI INTA, since
most PCI motherboards lack support for INTB, INTC, and INTD.
scsi1: DSP = 0xfffffc000009b60c
scsi1: DSPS = 0x4000000

(THEN IT LOCKS UP TIGHT)

Any ideas on this? Where are these PCI INT jumpers and what should they be
set to? TIA.
