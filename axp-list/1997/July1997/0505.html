All,
I have found (and fixed) the problems that were stopping MILO
booting from the AlphaBIOS console on the LX164 motherboard that is now
starting to sell in quantities. It was a PALcode difference. I will tidy up
my code and build more MILO images tomorrow (later today) at work. These
will appear on gatekeeper tonight after about 6pm EST. I'll create a new
LX164 directory under 2.0.30 for them. You should note that you still need
the PC164 vmlinux image - there are no differences at the kernel level, just
one difference in the reset PALcode. If anyone is desperate I can send
them uuencoded .gz images via mail. Sources will appear later this week, as
will new 2.0.30 images for all of the commonly available boards. My only
problem now is that I cannot keep the LX for long (it's for customer demos) -
I'll just have to make do with a PC164...

Dave
