On Tue, Jul 04, 2017 at 10:45:38PM +0800, Lo Sheng wrote:
^^^^^^^^^^^^
first problem: your clock is misconfigured :-)

I don't remember the exact location of where you can change them (My
miata's switched off right now and I'm some kilometers away from it) but
it should be something in /etc/sysconfig where you can add "ARC" to some
clock parameters.

If you dig a bit in /etc/rc.d (search for "clock" and find out how you get
the RedHat script to pass "-A" to the clock command) you will find out
very quickly... if not, I'm shure some other mailing list member can help
you out.

> However, the next problem is the setting up of X windows, using the Matrox
> Millennium card and XFree3.3 on the newly installed RH4.2. I get a whole
> lot of errors when I tried to startx. The errors are attached for
> information. If anyone has make the card work on the machine, could you
> please let me know how you have done it? Thanks!

> Fatal server error:
> xf86EnableIOPorts: Failed to set IOPL for I/O

This is to be solved by

ln -s Alcor /etc/alpha_systype

(for an explanation, search for Jay's excellent article on this subject
which went through the list some days ago). And no, there don't need to be
any file calles Alcor.

> (**) SVGA: Graphics device ID: "My Video Card"
> (**) SVGA: Monitor ID: "My Monitor"

This is another problem. You have to explicitely select "Matrox
Millennium" as your graphics card during xf86config, elsewhere it won't
work.

Apart from that, I couln't find any mistakes... I think this should get
you going.
