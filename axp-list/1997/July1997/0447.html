On 18-Jul-97 B. Embrey wrote:
>4.0 on it. The kernel version comes up as 2.0.18. The kernel as installed
>from the CD does not have sound (Microsoft Sound System) or PPP support. I

It does have PPP support, it doesn't have any sound support :) (If you're
getting the 'This kernel does not have PPP support' stuff, it may mean
something else, too).

>I have consulted several sources and the last step differs. Some say make
>boot, make zImage, or just make.

make zImage is correct only if you're using Linux/Intel. make boot is correct
on all Linux platforms (AFAIK), and is the prefered method :) just 'make' gets
you close, but no cigar..

>I would really like to get this working so I can use my PPP account and
>play some sound files. I have just recently ventured into the Alpha world
>and would like to just this running.

I've found getting sound to work with my UDB a little tricky (I got digital
audio to work, which is all the UDB has, but there was constant white noise..).
You might want to check out Timidity, btw, which allows you to play MIDI files
via digital audio (with lots of Gravis Ultrasound patches).
