>>> "Steven N. Hirsch" said:
>
> Is there any means of convincing Milo to boot a Linux kernel on the
> second SCSI hard drive (/dev/sdb)? I've tried everything:
>
> Partition w/ fdisk using Linux/MS-DOS partition mapping
>
> Partition using BSD slices

Interestingly, I've seen similar situations, and it's because *both*
partition tables are present on the disk.

Try this:

dd if=/dev/zero of=/dev/sdb bs=512 count=2

to clear out those tables, then start over with "fdisk", since you're using
MILO to boot...

Also, sometimes it helps to reboot between running "fdisk" and actually
doing the install.

Good luck.
