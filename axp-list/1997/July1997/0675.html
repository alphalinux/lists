>>
>>Unable to handle kernel paging request at virtual adress 0000000000b64796
>>sort (1396):Oops 0
>>pc=.... ps=....
>>rp=.... sp=.....
>>r0=....
>> and so on...
>>
>>Have you got an idea of the problem ?

I myself had the same problems on a UDB since I upgraded to more than 64 MB.
A possible workaround is "reducing" the memory-size in MILO.
Try:
MILO> set MEMORY_SIZE 127
before booting the Kernel. This should fix it. The unpleasant fact is, that
with this workaround to be applied, it is impossible to boot up automatically,
because you have to enter this at the MILO> -prompt.
