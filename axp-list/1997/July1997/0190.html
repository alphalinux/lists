> Yes, when your memory is gone, then it is gone - real or virtual -

> more swap space you may add more swap partitions; on a new disk,
> for example. A default is eight swap partition total, but this

You can also add swap "files" to your swap space, if repartitioning isn't
convienient. Use

dd if=/dev/zero of=swapfile bs=1024 count=xxxx

replacing xxxx by the number of megs you want your file to be. This will
create the file, but it still needs to be initialized. To do that, use

mkswap swapfile xxxx

and then add the file to your active swap by

swapon swapfile

Be aware that you may incur a slight performance hit by using a swapfile,
rather than a swap partition, as the swapper must do some filesystem
translation to get at the file. However, if you can't (or don't want to)
repartition, and need the extra swap, this is the way to go.
