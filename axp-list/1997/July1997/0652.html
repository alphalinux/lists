> sheer@eni.net said:
> > Okay, so I'm being mildly sarcastic.. but how on earth do I have
> > procs that can't be killed by kill -KILL???
>
> These are likely caused by being stuck in some kernel function. There is
> no way to kill these processes because they are stuck and kernel mode,
> and therefore part of the kernel that would be responsible for killing
> them.
>
> Reboot.
>

These processes are usually stuck in kernel mode because there is some sort
of a deadlock, and it is waiting for access to a file and/or device to
become available while in kernel mode. If you can find what resource the
process needs to satisfy this request, then the process will die normally.
