Hi there.

I have a problem getting a PC164 to run (64MB, IDE CD-ROM, 3.8GB IDE HD, and since
yesterday, a 4.3GB SCSI HD on an Adaptec AHA-2940U). The idea was, after having
to get the original IDE drive replaced due to a head crash thru which I lost my
Linux installation :-(, to add a scsi drive in order to set up Linux/RH 4.1 more
easily.

The plan was to put some WNT on the IDE drive and give Linux the scsi disk and
the rest of the other disk. Things are not working out, tho.

This is what happens: The firmware registers the AHA-2940U as an AIC78XX right after
startup. Is this ok? Dumping hardware info gives

scsi(0)disk(0)rdisk(0) RAW IBM
scsi(1)disk(0)rdisk(0) (3partitions) WD AC 34300
scsi(2)cdrom(0)fdisk(0) CD-Rom

or something similar. I attached the IDE HD to IDE connector 0/1 and the CD-Rom
to IDE connector 2/3 on tne mainboard, so this looks okay. In order to get
WNT to run, I had to set the hostid for scsi bus 0 to 0. The CMD-646-driver
for the IDE stuff seems to require hostid 7. So this way I can get WNT to run.

Then, when I try to boot milo either from the FAT-partition on the IDE or from
floppy, milo tells me: Don't know how to set hostid of scsi device (9004:6178)
which is the Adaptec. Now I can't do anything more cause milo throws out lots
of garbage on the console (due to this serial thing which is configured into
standard milos!?).

I'm not sure if my problems are due to the fact that I'm using a 2940U NOT 2940UW
or because of conflicts with scsi hostids. Maybe I need to jumper something on the
Adaptec?

Any help or pointers will be appreciated very much!
I want the damn thing to do what I want...

Thanx and cu,

Jo
