Henri,

I just added another 128MB to my PC164 last night. Without setting
MEMORY_SIZE slightly smaller than the real memory size, I got
the errors again. So, I set it to 252 MB for the new memory and
it is now stable, with no paging errors. I'd say there is a bug in
the VM system in Linux but then why would no one but PC164 users
have the problem?? Wonder if there is something unique about the
PC164 setup. Maybe Milo for PC164 has a problem?? I don't know but
if someone has some idea how to debug it, I can run some tests and
send results.
With the memory size == real memory, I can crash my PC164 in under
5 minutes. With the MEMORY_SIZE fix, it runs for days.
