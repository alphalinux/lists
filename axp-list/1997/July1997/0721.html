Dear Folks,

I am bedeviled with gdb problems. Can anyone hint at what might be
going on here? I have a PC164 Alpha clone on which I installed RH 4.1
without significant incident, and then made a number of package
upgrades to from the RH 4.2 distribution using the rpm utility and
downloads from ftp.redhat.com. In particular, I got the latest gdb,
glibc, and binutils. I believe I'm already running the latest gcc
for which there is an rpm.
The kernel is 2.0.30 which built without error from the 2.0.30
source from ftp.funet.fi and the 2.0.30-0.1 patches from gatekeeper.
MILO is from the 2.0.30 source at gatekeeper, and also built without
error.
Below I append a detailed description of one of the problems. I
would be glad of any help. Essentially my only use for this computer
is reasonably complex scientific C programming -- I don't give a damn
about web serving or having a cool window manager. But not having a
reliable C debugger (or is it the compiler?) is an absolute curse.
