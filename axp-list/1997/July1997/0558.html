>>> Charlie Fenwick said:
>
> We have an Alpha Station 255/300. I just put a second scsi card in it,
> a BusLogic BT-950 PCI Wide Ultra SCSI Host Adapter.
>
> The Alpha is running kernel...
>
> Linux pippin 2.0.18 #11 Wed Jul 23 08:51:34 EDT 1997 alpha
>
> and has Leonard Zubkoff's driver, BusLogic SCSI Driver Version 2.0.9 of
> 29 March 1997
>
> I am successfully running this driver on my Intel-linux box.
>
> After a kernel rebuild and reboot, I feel that the devices is not recognized
> by the alpha itself....
>
> Jul 23 12:03:17 pippin kernel: Warning : Unknown PCI device (1011:d).

This error is for the TGA2 graphics adaptor, right? Later patches will
eliminate this...

> Jul 23 12:03:17 pippin kernel: bios32.sio_fixup: weird, device 8086:0484 co
> ming in on slot 7 has no irq line!!

This is a bogus, to-be-ignored warning about the PCI-ISA bridge chip that
the bios32.sio_fixup code didn't seem to identify as a bridge...

> ...
> Jul 23 12:03:17 pippin kernel: BusLogic: FlashPoint Host Adapter detected a
> t PCI Bus 0 Device 12
> Jul 23 12:03:17 pippin kernel: BusLogic: I/O Address 0xB400 PCI Address 0x3
> 00000
> 0, but FlashPoint
> Jul 23 12:03:17 pippin kernel: BusLogic: support was omitted in this kernel
> configuration.

Well, the BusLogic driver *FOUND* the card, but it was a FlashPoint card,
and apparently the support you configured into the BusLogic driver didn't
include the FlashPoint support. Not that it would have helped you, as I
can't get a kernel to build cleanly when I *do* select FlashPoint
support... :-( :-(

This may be *why* you don't have FlashPoint support in there? :-) :-)

All the rest of the messages look OK; if you've been successfully using
the NCR 53c7,x88 driver, continue to do so, but the 53c8xx driver ported
from BSD seems to be in better favor these days.

Also, if you want to use the latest kernel, 2.0.30, note that I've some
new patches available; please read my posting to axp-list dated last
Thursday, 15 July, IIRC...
