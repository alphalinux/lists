To those that responded, Thanks. I still don't understand why gdb SIGSEGV's
but I found the problem. Turns out there were some free()'s that were
attempting to release some char[]'s in the middle of an array of structures.
It would seem that the whole array of structures disappeared when free()'d.
which caused the symptom I described. Only way I could track it was by
adding prints into the code and running until I isolated where the code
failed. Sure wish gdb had worked....
Also, it's interesting to note I built the same code on a PPro and it
would fail the same way as my Alpha so that confirmed it wasn't a 32vs64
bit problem.
