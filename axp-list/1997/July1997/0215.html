>Glint is broken when trying to install/uninstall packages. I think it
>has something to do with some bugs in either Python or Tcl/Tk, but I
>haven't looked into it in any depth. It only seems to be broken on
>Alpha, so it's likely 64-bit inspired. :-) I wish Red Hat would get
>this fixed, because I used to enjoy using this tool for maintaining my
>RPMS.

Here is the patch for glint that seems to fix the problem with verification of packages.

diff -c glint/rpmmodule.c glint-new/rpmmodule.c
*** glint/rpmmodule.c Mon Dec 16 09:02:25 1996
--- glint-new/rpmmodule.c Fri Jul 4 04:59:47 1997
***************
*** 446,452 ****

fileNumber = (int) PyInt_AsLong(args);

! if (rpmVerifyFile("", s->h, fileNumber, &verifyResult)) {
return Py_None;
}

--- 446,452 ----

fileNumber = (int) PyInt_AsLong(args);

! if (rpmVerifyFile("", s->h, fileNumber, &verifyResult, 0)) {
return Py_None;
}
