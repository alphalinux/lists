Hi there.

Yesterday I wrote about some problems I'm having getting a PC164
to run with a scsi disk, an IDE disk and IDE CD-ROM. I got some
clues as to what could be wrong from Jay.

My machine is a PC164, 64MB RAM, Elsa Winner 2000AVI/3D graphics,
3.8 GB WD IDE drive, Samsung 8x CD-ROM and now an Adaptec AHA-2940
Ultra NOT Wide with a 4.1GB IBM DCAS 34330 SCSI drive.

The problem is that I can't properly install RH 4.1. Things start
going wrong when the setup procedure tries to install things (on
the SCSI disk). I get messages like

aic7xxx : (abort) Aborting scb 0, TCL 4/0/0
scsi : aborting command due to timeout: pid 15, scsi 0, channel 0, id 4, lun 0,
Read (6)...

This leads to a scsi bus reset. Also, I seem to 'lose' the partitions
I set up with either fdisk OR the Disk Partition Utility once in a while.

So, Jay told me someone at CERN had this problem with the non-wide
Adaptec. If you're this person and succeeded in getting the thing
to work, or anyone else who knows what to do, please send me a note!

Any help is appreciated!
