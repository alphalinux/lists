>I am trying to run Scotty/Tkined on my Alpha, and I get this error:
>
># tkined1.4.5
>invoalid command name "package"
> whiel executing
>"package require Tcl 7.5"
> (file "/usr/bin/tkined1.4.5" line 9)
>
>Anyone have any experience with this?

What version of Tcl are you running? If memory serves, the "package"
command was introduced in Tcl 7.5, so this is exactly the response one
expects if you are running Tcl 7.4, for example.

I don't know anything about Scotty/Tkined, but it is also possible
that that code redefines, or rather, attempts to redefine, the package
command.

I would suggest starting up a Tcl shell ('tclsh'), and at the prompt
first enter

% puts $tcl_version

to see what version you are running (the '%' here is just the command
prompt from the interpreter), or even

% info patchlevel

Then try the above command, i.e.,

% package require Tcl 7.5

If the interpreter responds with 'invalid command name "package"',
then you need to upgrade your Tcl/Tk installation.

Incidentally, if you decide to upgrade your Tcl/Tk, make sure you get
Tcl/Tk 7.6/4.2 patchlevel 2 (from January 1997), as opposed to the
patchlevel 0 release (from October, 1996--I think), which has several
serious bugs. I don't know what ships with RedHat 4.2, but 4.1
shipped with the buggy version.

I'm currently experimenting with the Tcl/Tk 8.0 beta release. It
compiles out of the box, though with multiple warnings about
int<->pointer conversions. I don't know if those are real problems or
not...my code runs ok, but I certainly have not exercised all of
Tcl/Tk.

Hope this helps,

-mike
