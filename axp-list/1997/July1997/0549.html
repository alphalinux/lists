>I have two xquake binaries from two different places. Both core dump if I
>try to get them to run as a dedicated server. I have just installed RH
>4.2 on a UDB with 32 meg of ram. No X server is running (nor is XFee86
>installed, except the libs). I used to have this working under 4.0 a long
>while back, but I can't get it now. It also dumps if I try to join a game
>from a full graphical window (remote xserver). I guess I'm searching for
>the RIGHT binary for this platform. Pointers? Thanks!

Quake works fine using alpha-ELF-xquake--1.06 here, *providing* it is used
with an older version of libc (1.98 works for me, whilst 1.93 and 1.99
don't). I think 1.98 is included in an older 4.x release of RH.

You'll need to

- copy it into quake/oldlibs
- cd quake/oldlibs ; ln -s libc-1.98.so libc.so.6
- cd ..
- create fixquake.sh:

#!/bin/sh
LD_LIBRARY_PATH=quake/oldlibs:/lib:/usr/lib/:/usr/X11R6/lib
export LD_LIBRARY_PATH
quake/alpha-ELF-xquake-1.06

- chmod a+x fixquake.sh
- fixquake

This makes ld use libc-1.98 instead on the default version installed in
/lib.
