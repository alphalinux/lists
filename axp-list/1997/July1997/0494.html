> I never used my udb as a dial-in server before, what package
>should be used, how to configure?

Assuming that you mean you want your machine to act as a ppp server
you'll need some sort of getty to run on the port and the same ppp
package that you would use for outgoing ppp.

Set up a user account and have its shell be a script something like this:

mesg n
stty -echo
exec pppd -detach silent modem crtscts

Now when a user logs in with the "special" account and password the
server side ppp will start on your machine.

mgetty is the most flexible (IMHO) as it allows you to receive faxes as
well. It needs some patching to compile on an Alpha (unless that's been
fixed).

>
> I have some external fast SCSI-II devices, can it be used by
>UW-SCSI controller? Will a 68-pin to 50-pin adapter do the job?

Sure will - works for me.
>
> I also want to connect another computer to the existing
>network, this means I have to use a T-connector? How to use the
>DNS from a UDB running linux?

Yes, assuming you are running 10-base-2 (coaxial cable) you'll need a BNC
"T" connector and another length of RG-58/U cable. Don't make the cable
shorter than about 5 feet or so.

I'm not sure I understand your "how to use the DNS from a UDB" question.
If you mean how do you get the UDB to use an outside DNS then you just
create an /etc/resolv.conf file and put

nameserver <IP address of your DNS server>

in it. You may have more than one such entry if you like.

If you are asking how to run a DNS server on your UDB then that's another
matter, and probably outside of my expertise (or lack thereof) and also
probably outside this groups topic. A caching only DNS server can be set
up fairly easily on a UDB but a full-blown DNS server requires a LOT of
work.
