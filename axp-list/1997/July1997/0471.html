Roberto,

I had a similar problem when I installed RH 4.2 and did not set a
domain name. A bunch of daemons were spawned and clogged up the
system.

When I reinstalled and set a domain name but no DNS server, the problem
went away. Something in RH 4.2 setup requires a domain name even
if you choose not to turn on DNS. If you can get in to the machine,
try fix up the domain name and reboot to see if the problem goes away.

Think you can set up things in /etc/sysconfig/network. With mine working,
it looks like this:

NETWORKING=yes
HOSTNAME=wsb4.paralleldata.com
DOMAINNAME=paralleldata.com
GATEWAY=x.xx.xxx.xx
GATEWAYDEV=eth0

Good Luck....
