>>> Robert Allen Barker said:
>
> This implies that Linux Redhat won't run on an LX as yet (?).

Yes, it does run, but only when booted from SRM console, as I said.
The normal PC164 MILO and kernel work; actually, its the "milo.dd"
that gets used to boot into MILO from SRM console.

And yes, you could boot a kernel directly from SRM console (via "aboot"),
but I think I've a few patches that are required to build such a kernel,
that aren't part of the sources or axp-diffs yet.

> Are the miata versions of MILO and the kernel eventually intended
> to work on the LX?

Um, actually, no; even though the LX uses PYXIS, as does MIATA, the
LX uses interrupt delivery similar to PC164. Fortunately, the PYXIS
registers that are accessed appear at the same locations as the CIA
registers, which is why the PC164 images work.

Also, there are some *major* differences, like MIATA has an onboard
PCI-PCI bridge, LX uses the SMC SuperIO chip, etc.
