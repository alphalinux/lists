This is really odd.

I have an AXPpci 33 motherboard, with a 166MHz chip (noname).
It has the AT style Keyboard connecter.

IT worked fine till today.
The machine has been on running linux for the past couple of weeks, and I
went to use it on console today (normally I just use XTerms). The screen
was blank, becuase of the screen saver. I taped the shift key to get the
screen back. Nothing happend, I checked the machine via an Xterm,
everything looked normal, but still the keyboard would not do anything.

So I su'd up and did a shutdown, then half way thru the shutdown the
screen poped on. Nothing out of the ordinary was on the screen. Once the
machine was halted, I turned the power off and let it sit for a few
minutes. I turned the power back on, and it boots into the ARC console
where it sits (I don't have it automaticly boot into linux). Still no
keyboard response. So I thought it was a bad keyboard, grabed another one
and tryed the same thing. Still no response.

Ok, now I think I have tried everything besides using com port console.
Because I need to goto the store tommarrow and buy a gender changer for my
cable I have.

Any Ideas?

I have already tried the following:
failsafe floppy boot -> no keyboard response
removed all hardware except floppy and video -> no keyboard response
switched video cards -> no keyboard response

Please help
