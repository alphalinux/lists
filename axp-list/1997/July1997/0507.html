I am running RedHat-4.2 an an Alpha box. The problem is with libc.
Whenever I compile something with socket code I get warnings about
incompatable pointer types in accept() and a few other important calls.
The reason is on an Alpha longs are 64 bit while on x86's they're only
32. The network functions in question use size_t rather then int as
they should. This is a bad thing because size_t != sizeof(int) on an
Alpha like they are on a x86 but rather sizeof(long). This causes some
network programs to be really broken. I guess what I'm asking is if
anyone knows of a non-broken libc or patches to fix libc on an Alpha.
Any info would be appreciated.

Thanks,
