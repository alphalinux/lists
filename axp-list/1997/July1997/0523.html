On Tue, 22 Jul 1997, Remco Hannink wrote:

> I just upgraded my DEC UDB with Redhat 4.1. I also installed network
> support, this is the ethernetcard (build in) from the UDB which is
> connected to an Intel Linux box through coax. After completing the
> installation, the last line of the bootscreen is
>
> eth0: unconnected media, and then nothing.
>
> When I remove networking from rc.d the machine boots fine. But with the
> networking active I do not get a login screen. The machine does only
> respond to CTRL-ALT-DEL to shutdown correctly.

That's cause there's a bug in Redhat 4.1's init.d/network config script.
Modify it to look like this:
....
interfaces=`ls ifcfg* | grep -v ifcfg-lo | grep -v :`

echo $interfaces
echo "shutting down inet devices"
# See how we were called.
case "$1" in
start)
./ifup ifcfg-lo
for i in $interfaces; do
./ifup $i boot
done

touch /var/lock/subsys/network
;;
....
