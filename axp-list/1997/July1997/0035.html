I'm wondering if anyone else out there running DU binaries on their UDBs
with 2.1.42/2.1.43 kernels is noticing the same problem I am. It seems
that after a couple days of running Netscape, the system gets into a weird
state where it thinks it has run out of memory and process ids. I keep
getting messages like "fork: unable to allocate memory" and "no more
processes"

This only happens after a couple of days running various DU binaries,
notably Netscape. As a test, I decided I'd run Netscape through em86 for
a while and see if the problems would go away. Well, it's been over a
week and I haven't seen one error.

Anyone else notice this? I'm sticking with em86 for now..
