I believe this is a problem associated with the modutils package with RH
4.2. I am have the same setup (using a UDB as a firewall), and was able
to get the ip_mask_* modules loaded *and* working properly by running
`strip -x *.o` on the modules in the ipv4 directory and using insmod
from the modutils package from RH 4.1.

David
