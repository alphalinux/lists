Once upon a time, Steve Borho said:
>
> > I would like to be able to run a rsh on my account from a
> > remote machine and from the same machine. I edited .rhosts in the
> > usual way but I get a "permission denied". If I try a rlogin
> > I get prompted for the password. I guess one has to set up
> > correctly the /etc/pam.conf - but how ?
> > If I change the line
> > rsh auth required /lib/security/pam_rhosts_auth.so
> > to
> > rsh auth sufficient /lib/security/pam_rhosts_auth.so
> > any user (even without .rhosts entry) can run a remote shell. Any
> > ideas ?
> >
> remove group read-write access to the .rhosts file

Well, as an additional data point, I have experienced similar problems with
rlogind ignoring my .rhosts files. Permissions were set correctly and all.
Oddly, rlogin worked correctly if the host I was logging in _from_ had a
hostname of at least (exactly?) 8 characters. I was very perplexed by
this.

I eventually straced rlogind and watched what it was doing. Instead of
doing a reverse lookup on the IP address (like SunOS's rlogind -- a very
bad thing to do) it iterates through the .rhosts file, checking the IP
address of each listed host in turn. Something in the PAM code was causing
it to fail to recognize hosts with names of fewer than 8 characters.

I never traced it further than that. That was also on RedHat 4.0, and
might be fixed in later releases. My workaround was to install ssh. :>

just my .02;
