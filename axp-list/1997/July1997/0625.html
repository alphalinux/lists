> Hi.
>
> I am having trouble with rebooting a UDB Alpha running RedHat 4.1 Linux.
> The problem is two-fold --
>
> 1 - When I execute /sbin/reboot instead of rebooting I am dumped to the
> MILO prompt which prevents remote rebooting. When I cold start the system
> it boots into Linux, start loading the kernel, etc. How can I configure
> things so that I can reboot remotely without having to power down and
> restart the system?
>

When Linux reboots it reruns MILO. It is more or less impossible to restart
the ARC console (different memory mode and the ~4Mbytes of memory that it
uses has been used by Linux). MILO will autoboot in this case, booting exactly
the way it booted last time...

> 2 - When booting, things freeze at:
>
> eth0: media is TP.
>
> The freeze can be anywhere from 10 or 15 minutes to indefinite.
> Sometimes the system gets by after 15+ minutes and continues booting and
> sometimes not. What would be causing this?
>

Bad network set up...
