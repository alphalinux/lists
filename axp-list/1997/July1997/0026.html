I have just installed RedHat 4.2 from the CD-ROM on my first Alpha
(Enorex Ultra PC with AlphaPC motherboard, Adaptec 2940UW SCSI
and Ensoniq Soundscape sound card). Unfortunately, I have had a number
of problems. Let me try to list the most anoying ones:

* I was not able to install networking from the usual installation menu
because when I selected 3c905 card from the menu, I got a message
that 'insmod was not found'; this looks like a bug in the installation script
because Alpha kernels are not supposed to support modules.

* When I tried to build 2.0.30 kernel to support 3c905 and Ensoniq sound card,
I discovered that I am not able to do this because time stamps get messed
up on the files (some of my files had futuristic looking dates of
February 13, 2017!) ; it looks like the Linux idea of time is wrong;
During installation I chose the 'ARC console option' for time;

* I recompiled the 2.0.30 kernel using an earlier 2.0.29 kernel provided by
Enorex at their site, and the compile went OK; however, when
I try to bring the networking up with 'ifconfig eth0 up', the machine freezes;
Someone suggested that the card gets fried by the x86 setup script but
I am using the card successfully under Windows NT, so this is probably
not the problem;

* Glint produces segmentation violation when trying to verify a package

* Glint freezes when trying to install a package; it looks like rpm fails
because rpm shows as a zombie process

* Games like xpilot cause a seg. violation; this may be caused by the
shm* from within X; as someone suggested, and is supposed to be fixed
in 2.1.x kernels

Has anybody experienced these problems? Has anyone fixed them?

Thank you,
-Marek Rychlik
