> Or is there any way to convert Mark's OSF-1 COFF objects to Linux ELF?

In theory, objcopy can do this. In practice, it has difficulties in
translating between ECOFF relocs and ELF relocs (and possibly some
other problems). Fixing objcopy (BFD, actually) to do this properly
definitely would be a useful project (I suspect this DEC UNIX-only
object-file problem will keep cropping up), but I'm not sure how
difficult it would be (it shouldn't be hard, but with BFD you never
know for sure... ;-/).
