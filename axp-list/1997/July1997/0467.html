In article <199707181557.AA10377@maxwell.meridiantc.com> you write:
>I have downloaded a pristine 2.0.30 kernel tree from sunsite and the latest
>axp-diffs from gatekeeper. My question for the list is this; Do these
>axp-diffs fix the MIT-SHM mismatch bug, or do I need to backup to some older
>kernel/XF86/libc combination?

The axp-diff fixes the shared memory problem. The problem is that the kernel
declares some uid records as "short" instead of something like
"uid_t", which is incorrect on an alpha (this "uid_t" is 4 bytes long
under linux-alpha).

But you may also take care when applying the axp-diffs, as the one I used
wanted to compile the kernel with the "-O -g" flags and not the
"-O2 -fomit-frame-pointers" which is better, in my opinion, as far as
speed is concerned.

The other way is using a recent 2.1.x kernel, where the problem has
been fixed. 2.1.36 seems to be rather stable.

--Thomas Pornin
