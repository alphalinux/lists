Hi there,

here's some progress w.r.t. my problems upgrading a PC164 w/64MB that
had only IDE drive and CD-ROM with a SCSI disk.

I grabbed an Adaptec AHA-2940 UltraWide to check if THIS would work,
as the AHA-2940 Ultra NOT WIDE seems to have timing problems (actually
Jay told me there's a new driver which might get it to work correctly).

Okay, things work out better but not yet perfect. Now I can set up
partitions w/fdisk on the 4.1GB IBM DCAS 34330 disk. Milo boots okay
and I can boot the kernel/ramdisk to start RH 4.1 setup.

I can check the partition table ('v' in fdisk), it's okay. Proceeding with
the installation, things work fine, ext2-fs are made and the installation
starts.

Now the problem: some time thru this part messages come up and tell me
that a package X cannot be installed, a script failed or such, rather at
random and not very reproducible.

Sometimes I get thru to the point where you have to reboot, sometimes not
(this is ridiculous: the machine behaves like an Intel PC under Windows
or DOS or some such crap, just starts to reboot!!!). Whatever the case,
after rebooting I cannot boot from the scsi drive. MILO tells me something
about Bad Magic Numbers and the partition table is messed up.

Anyone else experiencing this type of problem? I've seen something about
a Durango PC164 from Greg Lindahl, have you fixed things yet?

I can't remember, but I think there was some time ago a note that there's
a problem setting up swap...?

Help appreciated!

Thanx, and have a nice 4th of July, those of you sitting in the states!

Jo

P.S.: I haven't tried w/o the IDE HD, I'll be checking that!
