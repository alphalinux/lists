Those 'oops' messages look alot like what my PC164 did before I set
MEMORY_SIZE in milo before booting. At the milo prompt try:
milo> set MEMORY_SIZE xxx
Where xxx is your real memory size minus 1-4MB. On a 128MB, I used 127
and it worked and with 256MB, I used 252MB and the system remains stable.

There are new sources to fix the problem on gatekeeper too. I have them
but haven't gotten them installed. Too busy working on an application
code. I don't really miss the 4MB so it's no rush to apply the changes.

My PC164 routinely stays up, excluding power glitches, until I decide
to take it down.
