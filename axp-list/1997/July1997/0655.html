>
> However, since upgrading
> the kernel I had already one system crash when a paging request could not
> be fulfilled. (This happened while gzipping -9 a 25MB file)

Most likely you run into a known (this is not Alpha specific) memory
management bug which may lead to a page starvation even if there is
a plenty of free memory on your system. For some reasons this bug
is more likely to show up with 2.0.30 than with earlier kernels
(2.0.29 including).

The bug is very actively worked upon but problems are not so simple.
It seems that a resolution is in sight - we will see soon. As a
workaround for now you may try, for now, for memory intensive jobs to
hike up values which show up in your /proc/sys/vm/freepages by echoing
a new triplet there. This has to be done from root account.
"Downgrading" to 2.0.29 also may help (but does not have to).

With 128M of memory - if you have stability problems did you try
to tell milo that you have only 127M?
