On Thu, 10 Jul 1997, Fude Yao wrote:

> Hi, everyone,
>
> I am trying to compile an example C file (simple.c from John Ousterhout's
> book) that use the tcl library. I got the following msg:
>
> /usr/lib/libtcl7.6.so: undefined reference to `dlerror'
> /usr/lib/libtcl7.6.so: undefined reference to `dlopen'
> /usr/lib/libtcl7.6.so: undefined reference to `dlsym'
>
> I use the command
>
> gcc simple.c -ltcl -lm
>
> What is wrong?

You need to link against /lib/libdl.so:

gcc -o simple simple.c -ltcl -lm -ldl

Enjoy!

Steve

p.s. - A good trick to remember when you get missing symbols and are not
sure where they can be satisfied:

$ find / -name "lib*.so.*" -print | xargs nm | grep "<symbol>" - | more

Works every time (assuming you _have_ a library with the missing entry
points..).
