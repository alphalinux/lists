On Jul 21, Greg Lindahl wrote
> Other OSes, not to mention Redhat 4.1/Intel, have int *. There is no
> easy way to write compatible code for both cases.

Sure there is.

#ifdef __GLIBC__
# define sock_size_t size_t
#else /* __GLIBC__ */
# define sock_size_t int
#endif /* __GLIBC__ */

sock_size_t len;

accept ( ..., &len );

This is exactly what the perl people have done---you should look at
the code in 5.004, as it also knows about the proper defines for the
new spec that specifies this.
