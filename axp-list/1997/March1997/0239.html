>>>>> "JB" == John Banghart <john@future.success.net> writes:

JB> [root@web qmail-0.92]# make setup ./load printnumd
JB> libsubstdio.a liballoc.a liberror.a libstr.a libfs.a
JB> libsubstdio.a: could not read symbols: Archive has no index;
JB> run ranlib to add one make: *** [printnumd] Error 1

Yeah, John. You need to run command "ranlib libXXX.a" on each of the
.a files created. The "ar" command will stuff a bunch of object files
together, but on Linux the "ranlib" command must be used to generate
the symbol table index thingy that the linker needs (so it can figure
out where in the archive the symbol is).

That's probably really fast and loose, so I will appeal to Those Who
Know Better to correct my mistakes.
