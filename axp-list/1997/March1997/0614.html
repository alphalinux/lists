>>> Bryan Liles said:
>
> I have an axppci33 with 72MB of memory. Milo only recognizes 24 of it.
> When I placed a mem=72MB in the boot string, the boot process crashes with
> some sort of screen dump. The only way to fix it is to power cycle the
> machine. Has anyone seen this before?

First, assuming you boot MILO from the ARC console, how much memory *does*
the ARC console tell you is installed? If *it* says 72, but MILO says 24,
then you probably have a new MILO and an old linload.exe. Get the latest of
the latter and I think MILO will then get the right information.

If you must change memory size for some reason, it'd be better to use the
MILO command "set MEMORY_SIZE 72".

>>> And then Wolfgang Dreyer said:
>
> As i remember the system MUST have in each slot the same memory.
>
> I think your population is two Simms with 32 MB == 64 MB
> two Simms with 4 MB == 8 MB
> Total 72 MB
>
> Yoo must place there 4 x 4MB Simms == 16 MB if this does not reach your
> needs you must go with 4 x 32MB == 128 MB Memory.

No; from the docs, memory must be installed in pairs, with each SIMM of a
given pair being the same size; so 2 * 32 plus 2 * 4 should be fine, so
long as each pair is placed in the appropriate SIMM slots for a pair.
On the AXPpci33 (NONAME), the 4 SIMM slots are arranged in 2 banks, with
the top 2 slots bank #1, the bottom #2. A bank must be filled completely
with the same size SIMMs.
