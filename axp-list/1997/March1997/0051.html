>
>
> The RedHat install process gives two choices
> with respect to setting system time.
> Which one is appropriate for a PC164?
>
> I think that all ARC-based systems (including PC164) must use GMT-based
> time: the logic of setting time in /etc/rc.d/rc.sysinit is
> if [ ${CLOCKMODE} = "GMT" ]; then
> clock -u -a
> else
> if [ ${CLOCKMODE} = "ARC" ]; then
> clock -A -a -u
> else
> clock -a
> fi
> fi
> Therefore, teh /etc/sysconfig/clock must be "CLOCKMODE="ARC"
> and the ARC time should be set to GMC/Universal time.

The above is correct if you do no want touch provided scripts, but
"must" is a bit too strong. Another option is to edit obviously
buggy script if you are not using UTC for your time base.
In general the stuff above should read somewhat like that:

if [ ${CLOCKBASE} = "GMT" ]; then
GMT=" -u"
else
GMT=""
fi
if [ ${CLOCKMODE} = "ARC" ]; then
clock -A -a $GMT
else
clock -a $GMT
fi

but this requires changes in a setup process. ARC mode and GMT mode
are independent.
