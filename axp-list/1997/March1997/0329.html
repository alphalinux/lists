I have a problem... I sit in a lab with an Alphastation 200 and a couple
of 500s. The problem is that they all run Digital UNIX, and I want to
stick Linux on one to hack around on.

I can't make any significant changes to the machines; they are set up for
SRM to boot UNIX, and have to stay that way. I have an external 2GB drive
(DKA300) that I can use as I please.

I have the Yggdrasil Winter 1997 "Linux Internet Archives" set; disk 8 has
RH 4.0 on it. Booting directly from the CD starts aboot (ver 0.5), and I
get this:
------------------
aboot: loading plain ...
aboot: unexpected object file format 1034
aboot: loading compressed ...

unzip: unknown compression method
------------------

If I try to load one of the kernels in the kernels/ or images/ directory,
I get similar errors.

I downloaded a boot disk with kernel 2.0.18 (I believe), and I get:

------------------
>>> boot dva0 -file vmlinux.gz
[...]
aboot: loading plain ...
aboot: unexpected object file format 13
aboot: loading compressed ...

unzip: unknown compression method
------------------

Trying to load vmlinux.gz directly gives an "attempt to read past EOF"
type error. I have read the disk and unzipped the vmlinux.gz without
problem on an Intel-based Linux box.

First of all, I don't know what kernel to use on the Alphastations; none
of the docs mention it. Second, what few docs exist are for using MILO,
not aboot.

Once I get Linux running on the Alphastation, then maybe I'll attempt it
on the VMEbus Alphas that we're working with... <G>
