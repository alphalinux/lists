On Tue, 25 Mar 1997 JAMES.JOHNSON@bge.com wrote:

> I have a Digital AXP PCI33 "NoName" motherboard and I would like to
> know if the onboard IDE controller is supported under RedHat Linux for
> both booting and installing using IDE hard drives and IDE CD-ROM's.

Well, I can atleast tell you that you can't use your CD-ROM on the built
in IDE interface. This is not EIDE or anything cool, it's just plain old
ide. I have tried to use an IDE ATAPI CD-ROM, and this did not work at
all. On the other hand, the same CD-ROM worked fine when I connected it to
my soundcard's IDE interface.

I would advice against using the onboard IDE controller for harddrives
given that it doesn't support EIDE or any DMA mode. (Apparently it has
some strange own version of DMA and a couple of jumpers for controlling
this on the motherboard, but I don't believe this is supported in the
kernel driver).

> I've searched Redhat's web site and it seems to ignore this issue. The
> best I've been able to find indicates that Red Hat Linux supports IDE
> hard drives but you must install from SCSI CD-ROM drives, floppy, or
> network. Is this correct? The reason I ask is that I have extra IDE
> devices to use and competing Linux for Alpha products do mention that
> they do support the IDE controller.

Yes, I guess they should say atleast something about IDE.

> I've sent multiple messages to redhat sales and can't get more out of
> them other than "check our web site", which doesn't help and only
> serves to irritate an already frustrated possible customer. I won't
> buy Red Hat unless I find out if my setup is supported. I really

Sorry, either way, your hardware is stopping you here.

Perhaps you can get things going by installing an EIDE interface, but then
you will probably need a kernel which is compiled with IDE support to
install on it. (Don't know if IDE support is in the installation kernel)
But then you would only be able to use the second interface (I think),
since there is no way to disable the builtin interface.

Also I have no idea if the Alpha version of RedHat will let you partition
and install on an IDE drive. But since other Alpha motherboards have EIDE
onboard I see no reason for not having IDE support in the kernel and not
allow the user to install on IDE. There are certainly people which can
confirm or deny this.

> don't want to buy from the 'other guy' if I can help it;-) Thanks for
> whatever help you can offer.

Hope you are helped by this. Good luck!
