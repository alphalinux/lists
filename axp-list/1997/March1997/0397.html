David A Rusling wrote:

Hi all,
what are the best settings for this driver? Should I
include this instead of
the NCR 7 driver in MILO (it works just fine)?

imho, you should NOT include the 1.12 version of the driver, it has
shown vulnerable by
scsi termination problems and the like.
The 1.16e Version works very well for me, and is not crashing my box
even if playing around
with the scsi bus while the system is running. As far as i can tell it
works better than the NCR 7
driver if the system is heavy loaded and in critical situations.
The following settings work very well at my Alphaserver 400, while some
others do not. Particularly, setting the sync scsi speed to more than
5MByte per second looks like begging for trouble.

CONFIG_SCSI_NCR53C8XX=y
# CONFIG_SCSI_NCR53C8XX_TAGGED_QUEUE is not set
CONFIG_SCSI_NCR53C8XX_IOMAPPED=y
CONFIG_SCSI_NCR53C8XX_MAX_TAGS=4
CONFIG_SCSI_NCR53C8XX_SYNC=5
# CONFIG_SCSI_NCR53C8XX_NO_DISCONNECT is not set
