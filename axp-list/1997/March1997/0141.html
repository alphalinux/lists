Just installed redhat 4.1 on an alcor w/ 256mb ram.

Top reports seemingly correct process sizes, but ps reports ridiculous numbers.

For example...
top:
PID USER PRI NI SIZE RSS SHARE STAT LIB %CPU %MEM TIME COMMAND
480 eric 2 0 1336 1336 840 S 1272 0.0 0.2 0:00 login

ps:
FLAGS UID PID PPID PRI NI SIZE RSS WCHAN STA TTY TIME COMMAND
100 500 480 1 0 0 16880 1336 wait4 S 1 0:00 /bin/logi

ps appears to be using the data in /proc, which also report the excessive size.

In addition, top's assessment of total memory availability seems similarly
skewed:

Mem: 255088K av, 50160K used, 204928K free, 13472K shrd, 17320K buff
Swap: 123016K av, 0K used, 123016K free 22728K cached

As time goes by, even with the machine not doing much, the "used" field
increases fairly quickly to the maximum amount of memory in the machine.

I'd like to know whether there is a really bad memory leak happening, or
whether it's just the memory usage reporting that's wrong... and, of course,
whether anyone else has seen this and knows how to fix it.

Thanks...

eric
