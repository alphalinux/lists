With lots of reloction overflow errors? This isn't a glibc
problem but one of the version of ld that you have can't
handle really large programs.

There are two solutions, both of which have been used successfully.

(1) split some of the static libraries that xemacs uses out
into shared libraries.

(2) Use a newer ld -- though this has some caveats due an old
bug in as that didn't show up with the old ld.
