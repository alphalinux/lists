On Tue, 25 Mar 1997, Stephen Williams wrote:

> Try setting your coredumpsize to unlimited and running again. This *should*
> get you a core file that can be examined with gdb. Another approach, run it
> under gdb.

I have attempted to run POV from within GDB. After several attempts
it became clear that GDB has some problems of its own.
When attempting to issue a 'return' command at one of the points of failure
GDB was unable to return. When setting breakpoints within GDB, and performing
a 'bt' command, the frames are displayed, however when attepmting to view
a particular frame GDB does not select the correct frame. When using the
'info breakpoints' command the breakpoints are correct at the beginning of the
debug run, but later the 'info breakpoints' command displays incorrect
breakpoints.

As far as the core file is concerned, GDB is unable to process the core file.
