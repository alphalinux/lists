Hello;

I now have rpm version 2.3.6 running and the errors concerning
the RedHat 4.1 RPMS files have stopped. I have downloaded all
the latest updates from gatekeeper.dec.com .

Here is my game plan to upgrade the PC164 to RedHat 4.1.
BTW, I have no idea what version of RedHat this box is currently
running. It is whatever Microway pre-installed on it.

Option 1: Manually install RedHat 4.1 to a separate disk partition.
Actually build a shell script using only the latest files from
RedHat. Advantage here being that my original install remains
intact and I am able to go back to it if need be.

Option 2: Get a third harddrive, and copy the original Linux disk
to this third harddrive. Disconnect the original harddrive to ensure
that nothing will happen to it. Upgrade the copy.

Option 3: Backup the original harddrive to DAT, and hope and pray that
the if the upgrade fails, it will be usable enough to recover everything
from the backup tape.

Would anyone out there have any other suggestions?
Has anyone ever manually upgraded a previous installation of Linux to
RedHat 4.1 before?

BTW, has anyone out there ever been able to build milo from the source?
I have searched for the quickstrip program, to no avail.

The reason I am hesitant to just upgrade and hope for the best, is that
at the moment I am doing this literally single handed, the left shoulder
is broken. The doctors are talking about surgery next week if it does not
start healing. I would rather have the machine in its current somewhat
usable state than down and recovering from possible shoulder surgery.

BTW, I broke the shoulder when a 7 ft electronic rack feel on me at a
client site.
