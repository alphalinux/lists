As far as I know, the only solution nowadays is to use XFree-3.1.2.
(Plain 3.1.2 is actually not as good as 3.1.2G, at least for me, but
3.1.2G has expired and no longer wants to run; so I've got to provide it
with a fake library that convinces the X server that it's now October 1,
1996).

I have this problem myself, and I've seen several postings in this
mailing list from other people about it. The problem seems to only
affect AlphaStations 255 with S3 Trio64. It first appeared in XFree-3.2
release of S3 server and is still present in 3.2A. XFree development
team is notified about this, but they can't do anything, because on most
Alphas everything's OK so it's impossible for them to trace it down.
