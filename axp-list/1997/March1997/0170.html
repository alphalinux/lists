Hello;

I am still having probelms with installing
RedHat 4.1.

I have downloaded the source code for milo-2.0.23,
which matches my current running kernel.
In attempting to make milo-2.0.23, the make
kept erroring out in the devices.c file.
The declaration of block_read and block_write
were different from /usr/src/linux/include/linux/fs.h.
After modifying devices.c declarations to match fs.h
the make errored out looking for 'quickstrip'.

Would anyone know what 'quickstrip' is suppose to do?
Would anyone know where to find either the source to
quickstrip, or a ECOFF binary?

Going back to the declaritions of block_read which
declaration is "correct", fs.h or devices.c?
The declarations are below:

fs.h
extern int block_read(struct inode *, struct file *, char *, int);
extern int block_write(struct inode *, struct file *, const char *, int);

devices.c
long block_read(struct inode * , struct file * , char * , unsigned long )
long block_write(struct inode * , struct file *, const char *, unsigned long )

An int on the Alpha is 32bits whereas a long is 64bits.
