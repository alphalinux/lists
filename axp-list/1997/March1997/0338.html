At 02:44 AM 03-13-97 -0500, you wrote:
>I may have been too vague: I have a particular mouse (actually keyboard
>with a built-in touchpad, but let's call it a mouse) with a 9-pin serial
>adaptor and nothing in the documentation about using it with a PS/2 part
>(although the keyboard part of the cable comes with a PS/2 adapter). Is
>there any way to use the mouse part with a UDB? The keyboard part is
>working fine.
>
>Thanks a lot for any advice.
Well, let's put it this way. Certain mice have the characteristics of the
IBM PS/2 mouse, sufficiently that an Alpha, for example, recognise them and
can use them. The test is pretty simple if you have SRM console on your
system. If it is a PS/2 compatible pointing device SRM will boot through to
prompt happily. If it says " no mouse detected" you're out of luck.
Now, having said that, it may still be usable. If it is a Microsoft or
Logitech compatible serial mouse, you may be able to define in your
X86Config file for it and use in X-Free. Maybe...
Sounds like what you have is interesting.
Another test:
If you plug it in a Wintel box and use standard drivers for, let's say
Microsoft Mouse, does it work?
