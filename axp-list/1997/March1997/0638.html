Can anyone offer some help please ?
Hardware
Digital multia ( supposed to be same as noname ? )
48 MB RAM
2 SCSI HD
Disk 1 -- 2 partitions , Windows NT 3.51
Disk 2 -- Linux
1 CD-ROM
Video - TGA ( I think that's what comes on board )
Alpha firmware ver 4.33
MILO 1.5
Redhat Power Tools Linux 6-CD set from Feb '97 (doesn't have any
registration in the CD set though!! )

Here's a summary of what I did so far.

Load MILO from the floppy, reports the right amount of memory.
load NONAME.IMG from floppy ( from kernels/noname.img on redhat CD )
with command
boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1
this seems to load fine.
When prompted for root floppy disk to be loaded into ramdisk, I supply
it with disk with ramdisk.img. Reports compressed image found at block 0
and proceeds to happily load that. So far so good.

I get the Redhat installation screen.(:)

Selected packages to install from cdrom and the installation proceeds to
completion, at least in terms of loading the packages.
Installation goes through keyboard selection, mouse, video card,
monitor.
At this stage there is a problem running X - probeonly, reports and
error trying to exec X, please start over.
But then it takes me to network configuration( IP address,DNS etc.),
time zone, etc.
Asks for the boot floopy, I feed it the floppy with NONAME.IMG on it and
copies the kernel. Tells me "Congratulations, installation is complete.
Remove floppy from the drive and press return to reboot".
Then tries to reboot, MILO reports about to boot from fd0:vmlinux.gz,
this is where things go wrong.
When the system tries to reboot ( from floopy /dev/fd0 also tried from
sdb1:vmlinux.gz) it reports
"Unable to open an initial console"
and appears to stop. Alt-F(1-6) has no effect whatever. Am I leaving out
some step or messing up something inadvertently? Am I missing some step
in telling MILO where to boot from possibly, and if so, how can I fix
that? Right now I don't have another machine to check if I can remotely
connect to the alpha box. I am at a complete loss so any and all
suggestions will be very welcome.
Thanks to all in advance,
William
