Nicholas Alexander Bastin wrote:
>
> Well, I fixed my boot problems by disabling amd,...
> When I try to do an RPM upgrade of the updates that I downloaded from
> redhat, it seg faults with a memory violation....
> are other people having these problems as well?

Yes, other people...
You need to remove the distribution amd (rpm -e amd) and
replace it with the upgrade version. If you can't get
networking, do the amd update by floppy.

I've found that updating glibc starts YPBIND error messages.
netkit-b will not upgrade because it conflicts with a file
in the glibc update. If I use --force on it,
my machine will no longer boot to multi-user.
