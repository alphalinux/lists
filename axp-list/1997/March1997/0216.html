At 04:21 PM 3/7/97 -0500, you wrote:
>Sure.
>
>I have been running 4.1 on a UDB since 12/26/96 (pulled down off the devel
>directory). X, perl, and emacs work, and a new gdb that can handle ELF
>debugging was just updated. I don't know about gprof or f2c, although they
>are provided with 4.1. Networking works fine, I am running my UDB as a
>NFS and SAMBA server, as well as my personal workstation displaying remote X
>applications such as Netscape and Applixware from a Linux/Intel box. I have
>no problems with booting.
>
>Part of the reason you have heard "horror stories" about UDBs are because so
>many of them were dumped on the market, snatched up by first-time Alpha
>owners (like myself) who were wondering how Linux works on a non-Intel
>machine, and were stumbling about getting it up and running :-) And many of
>the items on the errata list are there as security updates, *not*
>because they are fundamentally broken (i.e., don't work).
>
And part of the reason is that they sometimes, in fact, *DON'T WORK*. A
number of the errata updates were put there because the orginals were just
plain broke. I'd been running on a UDB since last July upgrading from 3.0.3
to 4.0, building kernels, hacking around getting PCMCIA to work, etc. Not
exactly a 'first time user'. Thought I'd buy the Official 4.1 CD from RedHat
and install from scratch. Surprise! no workee. After going through
support/this list/errata/grief/wasted weekends/etc., I've gotten everything
working except a very stubborn named (which btw hangs the machine going into
rl3). Best I can tell, there are still gremlins in some of the networking
stuff. (There's even a little gremlin in RedHat's network configuration tool
such that the tool won't run if you have a static-routes file in
/etc/sysconfig. Unless they've fixed it by now....). Don't plan on
installing from scratch and having a functional system from the original
CD's (upgrading might work - can't say, didn't try it. For complete install,
don't do it unless you can install from a place that has the updates in place.)
