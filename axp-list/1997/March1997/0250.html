>>> Doug Chapman said:
>
> Another thing to worry about is if it will work without an intel CPU.
> I ready someplace that a big problem with a lot of these cards on the
> alpha is that they contain some intel code in their rom that is used
> to initialize the card. Without an intel cpu this code can not be
> interpreted.

DEC realized this a few years ago, which is why *all* the consoles (SRM, ARC,
and AlphaBIOS) have built-in BIOS emulation code.

Now, sometimes that emulation code was not up to the task, but this does
get fixed for cards that DEC intends to support. Cases in point were the
ARI Mach64's and the Matrox Millennium, which, under early versions of
the emulator, could hang the system before it ever came up... :-(

And it's also why our MILO has emulation code as well, both borrowed from
the DEC consoles (but distributed only in object format) or a freeware x86
emulator that's been tweaked a bit. When MILO is not FLASHed, that emulation
is a bit of overkill, as the first-up console (ARC/SRM/etc) will do what is
necessary. But it hasn't seemed to hurt anything, unless it's older and not
up to par, of course.

So, feel free to plug in any VGA and expect that the BIOS code will be run
to the best of the emulators' abilities. If you have problems, see if there's
an update to the FLASHed console that you are using.
