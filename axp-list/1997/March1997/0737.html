Hello;

The probelm with trying to debug this problem is that if you lower the
size to something smaller than 800x600, the scene renders with no problems.
This would seem to indicate that the problem is with memory getting trashed
at higher resolutions. I have not try this but it would seem to make sense
that if I would render the scene at say 1600x1200 it should fail sooner.

Rendering at 800x600 I have consistant failures at the following pixels.
row 377 column 227 <- little over two hours to get here
row 387 column 232
row 397 column 233
row 431 column 252

The line of code it is failing at is line 328 in super.c.
The values v0 and v1 are obviously invalid, if you work through
the functions and initial variables from which v0 and v1 obtain
their values, there is no way they should contain what they contain.
Also P0 and P1 have values which are obviously invalid.

I have turned optimization off when building POV on the Alpha.

I have rendered the scene to completion by using the +C option in
POV. This at best is a very crude work-around.

I have sent e-mail to the POV team member who supports the UNIX/Linux
port. I have offered him access to my Alpha if he needs it for debugging.
I have received no answer yet.
