Erik, here's the solution to your "QM_MODULES: Bad address" problem
with modutils built on 2.0.27. I now remember fixing this in 2.1.7,
but it never got moved back to the 2.0 series.

r~

--- linux-2.0.27/include/asm-alpha/unistd.h.orig Mon Nov 18 02:25:38 1996
+++ linux-2.0.27/include/asm-alpha/unistd.h Sat Mar 29 18:04:41 1997
@@ -214,7 +214,7 @@
type name (type1 arg1,type2 arg2,type3 arg3,type4 arg4,type5 arg5) \
{ \
extern long syscall (int, ...); \
- return syscall(__NR_##name, arg1, arg2, arg3, arg4); \
+ return syscall(__NR_##name, arg1, arg2, arg3, arg4, arg5); \
}

#endif /* __LIBRARY__ && __GNUC__ */
