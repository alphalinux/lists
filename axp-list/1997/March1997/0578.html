On 3/19, John Abella uttered the following other thing:
>
> Trying to do my first ALPHA/Linux install (i've done intel and sparc
> already) and I've run into a bit of a problem. The machine is a Jensen...
> I followed all of the instructions I could find, but when it starts to
> boot up, I get this: Kernel panic: VFS: Unable to mount root fs on 08:02
>
> A Few lines before that I get some kernel: unaligned trap errors. I can
> send these if they'll help.
>
> Someone please tell me I made some really dumb mistake, so I can get this
> thing going asap. Please reply to me, as I am not yet on this list.

Not sure if this will help, but I had similar problems installing, and
the reason was because I had created my filesystems the way I usually
do, which was boot then swap than root, but alpha-linux defaults
to mounting the second partition as root, which failed and caused the
trap (it looked like it thought it was a DOS drive). The work
around is to tell milo the filesystem to use as root, ie:
boot sda3:vmlinux.gz root=/dev/sda3

Or, it could be something completely different.
