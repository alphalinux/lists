ABSTRACT:

While upgrading RedHat 4.0 to RedHat 4.1 on Alpha XL-366
the installation program hangs up the system when it tries
to prepare a swap partition (if this step is omitted then
it hangs up the system when it upgrades packages).

SYSTEM DESCRIPTION:

System: Alpha XL 366
CPU: 21164 at 366 MHz
Memory: 128 Mb
Cache: 2 Mb
SCSI: NCR53C810 (embedded)
SCSI disk 1 (id=0,sda): SEAGATE ST32550N
SCSI disk 2 (id=1,sdb): SEAGATE ST15150N
SCSI CDROM (id=5,sr0): TOSHIBA XM-5401TA
Ethernet card 1 (eth0): PCI DE500-XA
Ethernet card 2 (eth1): DE434/5 (embedded)
Video: PCI VGA Matrox Millennium
Floppy: 1.44 Mb (detected by Linux as 2.88 Mb but works)
BIOS installed: Alpha BIOS Version 5.13 (May 8 1996)
ARC Multiboot
X86 BIOS Emulation

System work under Windows NT (boot from disk 1) and did work
under Linux 2.0.18 - RedHat 4.0 (booted from disk 2) (the last
system is killed by upgrading to RedHat 4.1).

DISKS PARTITIONING:

sda:
sda1 (2 Gb) FAT (used for Windows NT)

sdb:
sdb1 (200 Mb) FAT (used for booting Windows NT and MILO)
sdb2 (500 Mb) Linux native (root file system for Linux)
sdb3 (256 Mb) Linux swap
sdb4 (3 Gb) Linux native (mounted as /home)

INSTALLATION METHOD:

Installation from local hard drive (partition sdb1), boot using
MILO. We use MILO from `xlt' series, xlt.img and last ramdisk.img
(we also have tried MILO from `XL' series: it hangs up the system).
