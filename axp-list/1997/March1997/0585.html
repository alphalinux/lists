My AS255 is now running Redhat 4.1 Linux. Since I discovered an
important point that (I felt) wasn't clear in either RedHat's or the
SRM and MILO howto's I thought I'd write it up so that somebody else
doesn't trip over it.

If your alpha has the SRM firmware you cannot use fdisk style
partioning on the hard disk you want to boot from, i.e. you must use a
BSD style disklabel.

Why? Because fdisk since it is from the PC world writes it's
partition info in the end of block 0 which is eactly where the SRM
firmware expects to find the block number and number of blocks to
load.

Now for a question concerning nfs: I have a machine (pc w/linux) that
is exporting a 3.5GB filesystem and when I try to mount it on the
alpha I get an error with a message like "unable to get root inode,
too many mounts, etc". Mounting a smaller filesystem (500mb) works
fine. Have I tripped over a problem with linux's nfs?
