> What exactly is the status on the utmp problem situation? Is there a fix
> in sight? None of the apps that draw on utmp work properly due to the hosing
> of the utmp file.

My utmp file on a system thats almost but not quite RH 4.1 (Most of their
4.1 RPMS installed...) doesnt get corrupted - just not updated properly.
Telnet connections write and remove their entry fine, rlogin connections
stay forever, but except for size the file has no problem.

I had utmp corruption when I was running the first version of glibc
and had not managed to update all possible tools that wanted to fiddle
with utmp, but as I got more 4.1 RPMS installed, that problem went away.

I suspect my rlogin problem might be a program from a package thats
escaped updating - although since it isnt hosing utmp, it cant be that
old.
