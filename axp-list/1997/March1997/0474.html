I am trying to make those three work together,
but they don't.

First, I succeeded to compile PCMCIA 2.9.2 on my UDB
under kernel 2.0.27 and 2.0.29. It took some changes in
include/pcmcia/k_compat.h file.

Second, I compiled wavelan_cs.c and made insmod successfully insert the
module.
It required some insb()/outsb() hacking.

The card service seems to work fine, and it makes the new eth1
interface.
After that, the troubles begin.

The following three commands make the machine hang up forever:
ifconfig eth1 up
ifconfig eth1 down
ifconfig eth1 up

The command
ifconfig eth1 up addr 10.0.0.1 netmask 255.255.255.0
sometimes executes fast, sometimes it takes about a minute.
And ,on "ping" the "transmit" light is blinking (!).

This error is the same for 2.0.27 and 2.0.29 kernels.

Could anybody help me make it all work ?
