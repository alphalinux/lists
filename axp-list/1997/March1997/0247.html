Er...um...I retrieved modutils-2.1.23.tar.gz from FUNET a bit ago and
compiled it against an unmodified 2.1.23 kernel (just changed the
ARCH= line in the top-level Makefile to "alpha" and did a 'make
symlinks'). kerneld, depmod, modprobe, genksyms, et al, work just
fine with the modular 2.0.29 kernel I've been running since Saturday.

I ran into a few hangups with the 2.0.29 kernel itself (four modules,
- fs/binfmt_aout.ou (start_thread)
- fs/isofs.o (strncat)
- fs/nfs.o (__kernel_thread)
- net/slhc.o (ip_fast_csum)
complained about unresolved symbols [symbols in parenthesis]), and the
following patch to kernel/ksyms.c did the trick. I haven't had time
to catch up my reading on linux-kernel to see if this is an actual
bug, so my problem may be a fubar on my part.

@@ -47,6 +47,11 @@
