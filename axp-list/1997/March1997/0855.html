Stronghold support staff,

My client, Spatz Computer Graphics, is running stronghold on its
Alpha/Linux servers (Redhat 4.0, kernel 2.0.27). I've found a problem
with stronghold.
We're running two servers in a failover setup. Both servers are
running stronghold, but only at a time is aliasing the ip's for the sites
we're hosting. I've set up pages to access info from the servers. From
them, one can run a cgi to check the routing table of the server to see if
it's running the aliases. This works, but isn't necessarily the best way
to check that.
So, I thought of another idea. I set up this line at the top of
the page:

<CENTER><H2>Alphabits is now the <!--#exec cmd="/bin/cat
/var/run/WWW.STATUS | /usr/bin/tr A-Z a-z"--> server</H2></CENTER>

/var/run/WWW.STATUS contains either "Primary" or "Secondary", depending on
whether it's running the aliases or not.

If I switch quickly to and from this page with netscape, it causes
this error:

Mar 28 10:16:25 alphabits kernel: httpsd: memory violation at
pc=155556c52e8 rp=155556c5474 (bad address = 100000000)
Mar 28 10:16:38 alphabits last message repeated 7 times

The same page minus the "#exec" line causes no problems. Any
ideas on what's wrong?
