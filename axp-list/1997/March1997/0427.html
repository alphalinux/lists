carlos@fisica.ufpr.br said:
> That's not good enough! I'd expect it to be faster. If it's only 2.5x
> I'm not sure it's worth the extra cost...

Such things are very memory intensive and thus memory bound. The alpha
may be a hell of a lot faster then the PPro but is chopped down to size
by memory bandwidth. A friend of mine has an older alpha then mine. It
is 400MHz with 2M cache and *128 memory path*, mine is 500MHz with 1M cache
and *256 bit memory path*. His programs run twice as fast on my computer
(which has half the cache) and we believe the win is the memory path width.

<soapbox>
Personally, I kinda wish computer manufacturers would cool it with the
clock speed wars and solve the real problem, memory bandwidth. Data cache
sucks, adds useless complexity, and doesn't really solve the problem. After
all, the faster the computer, the more data it can process and the more
memory it needs to hold that data. Isn't that the point?

Anybody at DEC listening? 512bit wide SDRAM should get 3.2GBytes/sec, or
enough for 1 load/clock without wait states on a 500MHz alpha:-)

I believe that the instruction caches are useful.
</soapbox>
