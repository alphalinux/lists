"Jerry Feldman" <gaf@zk3.dec.com> wrote:
> The 166 Multia (which uses the ev4 chip) uses a dual issue strategy where
> it can execute 2 instructions simultaneously. The 233 Multia uses the ev5
> chip which can execute 4 instructions in parallel.

Thanks for trying to help, but unfortunately you are mistaken.

The 166 MHz Multia uses the DECchip 21066, which is known as LCA or LCA4.
It is based on the EV4 core, but is not an EV4 chip. It does, as you say,
have dual issue.

The 233 MHz Multia uses the DECchip 21066A, which is known as LCA45. It
is based on the EV45 core, however it does *not* have the larger cache
size of the EV45. And it *still* is only dual issue.

Some newer 166 MHz Multias might also use the 21066A, but would presumably
use the 166 MHz speed grade rather than the 233 MHz grade.

There is (unfortunately) not a 21164 (EV5) or 21164A (EV56) based Multia.
Those are the chips that have quad issue.

For more information see:
http://www.brouhaha.com/~eric/computers/udb.html
