Hi All,

I have been having on again off again problems with shadow passwords on
our pc164. Let me go through our modifications.

First (after updating the glibc rpm off redhat's site) we managed to get
shadow to work for root. Then, after rebooting it worked for users.
Next we updated to a newer version of glibc directly from one of its
distribution sites. This did not break shadow, as after this it still
worked (several reboots).

Next, I added a simple file to the /usr/lib directory, to try and get
linuxthreads going. Since the file wasn't there before I can't see why
it would affect anything:

/usr/lib/libc.so (text file)

Contents:

GROUP ( libc.so.6 ld.so.1 libc.a )

As per the instructions in linuxthreads. After rebooting this time,
shadow will work only for root and noone else. The proble is, removing
this file, and rebooting, or removing it and running ldconfig doesn't
seem to make any difference. If I am logged in as a specific user I get:

passwd: User not known to the underlying authentication module

which was the problem before the upgrade. Is it possible that our passwd
program is looking to the wrong libraries, or is something else going on?

Thanks,
