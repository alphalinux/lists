Sorry, clicked too soon. Left out an important step...

I installed the RH4.1 and althought there are still some problems related
(evidently) to named.init, I decided to try to get pcmcia working so that I
could use my pc-card modem. My config is UDB w/80Mb, Official RH4.1 with
updates installed from scratch.

First, the RH4.1/axp CD does not install the pcmcia stuff. IIRC, the pcmcia
is installed on the Intel side. <Hint: RedHat should do this on alpha as
well> In any event, I downloaded pcmcia-cs-2.8.23 from hyper.stanford.edu
(D.Hinds site). I went to do the make and it hiccupped on the smc and Xircom
ethernet card driver modules on what appeared to be duplicative includes of
functions or defines or something. Since I have neither card, I deleted
those two out of the makefile. All the rest went smoothly.

After the make install, I went to /lib/modules/2.0.27/pcmcia (stock kernel
is 2.0.27, use whatever is running) and did a 'strip -x *.o' on the modules
there. Turned on pcmcia in /etc/sysconfig/pcmcia. Commented out the if-then
test in /etc/sysconfig/network-scripts/ifup that exits after pcmcia (and
doesn't start ethernet or the rest of networking as a result). Rebooted and
all magically worked.

I also tried to download the pcmcia-cs-2.9.1 and compile worked but it puked
at run time with an undefined procedure 'Get_user'. I took this as a 2.1
kernel-ism and went back to 2.8.23. So far, things are working properly,
including being able to ppp to Netcom via the pc-card modem (kudos to RedHat
for the ppp auto-config option as part of their network configuration - jabs
for the fact that the network config tool bombs if you have a
/etc/sysconfig/static-routes file....)

Since a couple of people have asked about this, I thought I'd let folks
know. Again, sorry for the double post...
