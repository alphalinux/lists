On Mon, 31 Mar 1997, Steven Kokinos wrote:

> We have a number of programs with a predilection for going into state 'D'
> - uninterruptable sleep. It seems that tar, gz, and make when it is
> doing certain things will all do this on occassion. It is a pretty big
> problem because it means the machines will not come down quietly, and
> require a hard boot.

What motherboard or Alpha workstation do you have?
Other system specifics would be helpful also, like SCSI card, SCSI hard drives
(Manufacturer/Model), etc.

I have an Intel box that does this when you change the CD to quickly in the SCSI
CD Player.
Xmcd goes into 'uninterruptable sleep' because the SCSI bus is being reset.
Have to reboot the machine when it happens.

>
> Has anyone else seen problems like this? Could it be related to file
> system problems, or is it related to a bug in one of the libraries. Any
> input would be appreciated.

This sounds like the SCSI bus is being reset while these commands are running.
They will go into 'uninterruptable sleep' waiting for the SCSI bus.
Are you seeing any SCSI errors on the console or syslog file?
