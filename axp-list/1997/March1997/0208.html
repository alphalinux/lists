I've got another clue to the "Unable to handle paging request" problem
with 2.0.x kernels. I found that if I tried copying a large amount of
data from an NFS disk to my local disk I could get consistant crashes,
so I decided to try different kernel configurations to see if any
permutations would eliminate the problem. I found that if I turn off
the "IP: Allow large windows" options in the "Networking options"
window, the kernel doesn't crash (at least in my test). I can't say
that it's completely fixed, but it hasn't crashed yet.

I'm using kernel version 2.0.29. I've built two kernels with only that
one option different. If the option is turned on, my system crashes
consistantly withing a few minutes. With it turned off it doesn't crash
at all.
