I had the same problem with my UDB using 10base2. I found in *my* case that
there was a contact problem in the BNC connector. Monitoring the kernel
messages I could see the connection connect and unconnect as I twisted the
tee when attached to the connector.

When I rewired using 10baseT, all the problems went away.

The UDB by default uses 10BaseT, and only switches to 10Base2 if a
connection (i.e., live network) is detected through the interface. This is a
"feature" of the DEC autosensing adapter built into these machines. I have
the same adapter on a Linksys PCI ethernet card in my Pentium, and I get the
same message (eth0: on EXT SIA) if I disconnect the cable from the hub.
