On 3/13, Brandon Long uttered the following other thing:
> A common question on this list (judging by the archives) but I didn't
> see any reference to this problem, and now I'm wondering if I'm
> just plain missing something obvious.

I was missing something obvious. Like, /etc/sysconfic/pcmcia
said:
PCMCIA=no
so I changed it to
PCMCIA=yes

I also needed to strip -x the modules. Then ds.o wouldn't load because
of get_user undefined, so I hacked that one up (in Alpha, it seems
get_fs_long, get_fs_word, get_fs_byte are defined in terms of get_user,
so undef'ing get_user and then defining get_user in terms of get_fs_etc
doesn't work too well.)

Now, things still don't work, but in a way I'm much more familiar with.
My Adaptec 1460 causes a memory fault, which is better than panicing
the kernel on my latop under 1.2.13/intel. The network card complains
about missing symbol definitions which are in 8390.o, which is loaded,
so I don't get that. And depmod just plain doesn't work, and the code
is rather ugly, I don't even know where to start.

Ah, reminds me of the good old days. Interesting how my udb 166 with
32M compiles the kernel lower than my 486/33 with 16M used to. The
kernel has grown a bit.
