The subject says it all: How do I boot RH4.1 (from CD) on a 255
(avanti?) without a floppy drive?

Particulars:

Alphastation 255 with SRM,
64 MB RAM,
1 1 gig scsi with Digital Unix,
1 scsi cdrom
1 4 gig scsi drive
(for linux either using BSD or pc style partitions, i.e. whichever
works)

- No other alpha linux system is nearby.

- There is an Intel i386 linux system next to the alpha (if that helps).

- I have the latest source (and images) to aboot-0.5 and milo.

All the instructions with the redhat release assume that you have a
floppy to write the correct image too. They also appear to require
that you boot the floppy with milo (that has been previously installed
on a harddisk).

My Questions:

1. Is it possible to boot and install RH-4.1 without a floppy on
this system (255 with SRM)?

2. Would it be possible to 'dd' the floppy image to the 4gb drive
(on big mother floppy) and then install aboot on the 1gb drive
(wiping out DU) and then boot aboot and use that to boot the
image from the "big" floppy?

3. Have I missed something totally obvious? It wouldn't be the first
time! ;-)

Any thoughts, comments, etc.
