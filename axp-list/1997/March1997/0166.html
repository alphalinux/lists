I recently got the BYTE Benchmark program, and after some trouble, got it
to compile on my Alpha UDB 166. What is puzzling to me is the results.
Everyone has been saying that a UDB 166 is about as fast as a Pentium 90,
book look at what I get:

BYTEmark (tm) Native Mode Benchmark ver. 2 (10/95)
NUMERIC SORT : Iterations/sec.: 32.76 Index: 0.84
STRING SORT : Iterations/sec.: 2.79 Index: 1.25
BITFIELD : Iterations/sec.: 11116152.42 Index: 1.91
FP EMULATION : Iterations/sec.: 2.41 Index: 1.16
FOURIER : Iterations/sec.: 480.34 Index: 0.55
ASSIGNMENT : Iterations/sec.: 0.38 Index: 1.45
IDEA : Iterations/sec.: 94.46 Index: 1.44
HUFFMAN : Iterations/sec.: 51.77 Index: 1.44
NEURAL NET : Iterations/sec.: 0.22 Index: 0.35
LU DECOMPOSITION: Iterations/sec.: 19.13 Index: 0.99
...done...
===========OVERALL=============
INTEGER INDEX: 1.319
FLOATING-POINT INDEX: 0.575
(90 MHz Dell Pentium = 1.000)
===============================

And here is what I get on my pentium 120:

BYTEmark (tm) Native Mode Benchmark ver. 2 (10/95)
NUMERIC SORT : Iterations/sec.: 55.08 Index: 1.41
STRING SORT : Iterations/sec.: 4.13 Index: 1.84
BITFIELD : Iterations/sec.: 7360309.96 Index: 1.26
FP EMULATION : Iterations/sec.: 3.85 Index: 1.85
FOURIER : Iterations/sec.: 1099.94 Index: 1.25
ASSIGNMENT : Iterations/sec.: 0.48 Index: 1.81
IDEA : Iterations/sec.: 71.43 Index: 1.09
HUFFMAN : Iterations/sec.: 39.93 Index: 1.11
NEURAL NET : Iterations/sec.: 0.61 Index: 0.98
LU DECOMPOSITION: Iterations/sec.: 21.81 Index: 1.13
...done...
===========OVERALL=============
INTEGER INDEX: 1.448
FLOATING-POINT INDEX: 1.114
(90 MHz Dell Pentium = 1.000)
===============================

So over all the alpha performs much better than a Pentium 90 on Integer,
but everyone's been saying it's about the speed of a Pentium 66. Also, I
thought the Alpha's strong point was supose to be Floating point, so why
the terrible results? (i.e.
http://www.brouhaha.com/~eric/computers/udb.html For non-floating-point
tasks, the 233 MHz model seems to offer performance comparable to a 100 to
120 MHz Pentium. However, the floating point performance is fairly good.
) I was expecting just the oposite. Maybe better floating point results,
and worse integer than a pentium 90. Certainly the Alpha does seem slower
than my pentium 120 by quite a bit, and compiling takes forever... (slower
than these results would indicate, I think..) anyway, can anyone explain
these results?

ON Thu, 6 Mar 1997, Adriano Nagelschmidt Rodrigues wrote:

> gcc: Internal compiler error: program cc1 got fatal signal 6
> make: *** [nbench1.o] Error 1

I had the exact same problem. I think all I had to do was change the
DEFINES section in the Makefile and it compiled just fine. This is what it
should look like:

DEFINES= -DLINUX -DAXPPROC

> When I take out the -O6 flag, compilation works, but an ld assertion fails:

Yeah, put -O6 back in the Makefile :-)
