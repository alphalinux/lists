Hello everyone,

Earlier on this list I had remarked about troubles we were
having with our new 500MHz PC164 system from Microway.

The machine is now in stable operation.

When it arrived from Microway, it was crashing randomly,
so we installed RH4.1 on it. However, that still didn't work,
possibly because we didn't configure milo properly to use the
new kernel.

Steven Gaudet suggested that I compile kernel 2.0.27 and use
it with milo 2.0.27. I did that and the system has operated
very well for a week, despite heavy use.

Thanks, Steve, for the working suggestion.

Our system is configured like this:

500MHz CPU
PC164 MB
2M cache
Diamond Stealth 64 Video VRAM 2M
2G EIDE disk
ATAPI CDROM
128M of RAM
256bit memory interface selected
Kernel 2.0.27 (patch with axp-diffs 2.0.21)
Milo 2.0.27

It runs our scientific simulations 2.5x faster than a PPro 200MHz
system.

Thanks to all who suggested solutions to
the problems we were having with crashes.

Regards,

Peter Mardahl
