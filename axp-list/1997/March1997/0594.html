Sorry, but the current SABLE support is bootable from SRM console only,
and there is no MILO for it (and prolly never will be). "aboot" is machine
independent, you only need the appropriate machine kernel built to be
booted from SRM console.

You MUST partition your disk appropriately for booting from SRM and the
installation of the "aboot" bootstrap. In particular, that means:

1. you must have a BSD-style partition table; I recommend the use of
"minlabel" rather than the BSD option of "fdisk", but that is personal.

2. you must offset the start of the first partition by AT LEAST 1024 sectors,
in order to leave room for the "aboot" bootstrap to be installed.

Please refer to the SRM Booting HOWTO at: azstarnet.com/~axplinux/srm.html
for more details, especially the commands and data to use to install "aboot".
Note that Red Hat 4.0 (and I assume 4.1) has the necessary utilities and
data files for an "aboot" install. There's also information installed at
/usr/doc/aboot-0.5-1 in a 4.0 full install.

Now, assuming that /dev/sda corresponds to DKA0 under SRM, and that your
kernel is installed in sda3, you would boot a kernel from the harddisk like:

>>> boot dka0 -file 3/vmlinux.gz -flags "root=/dev/sda3"

Note the the "-file" option specifies the third partition by number.

If you'd like to be able to say simply ">>> boot", then setup the
following environment variables thusly:

>>> set bootdef_dev dka0 # default boot device
>>> set boot_file "3/vmlinux.gz"
>>> set boot_osflags "root=/dev/sda3"

Good luck. Keep me posted with any problems or successes... :-)
