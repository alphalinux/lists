Britt Bolen wrote:
The only problems I
> had with the build were of this type:
>
> warning: cast from pointer to integer of different size
>
> Which (if I'm not mistaken) is because of the alpha's 64 bit architecture,
> right? The program dumped core as soon as it ran.
pointers = 64 bits
long = 64 bits
int = 32 bits
short = 16 bits ( but should be avoided. fetches are 32/64 bits )
bytes = 8 bits ( also should be avoided ( if u can ) )

therefore:
int i;
char *p

i = p; /* will loose the upper 32 bits */

You should be carefull of longs, and how they are handled. the & masking
of longs may not get the upper 32 bits ( ie long k & 0xffff still leaves
some 48 bits and not the expected 16 )

hope this helps
