pete,

Having returned from my holiday I now have time to run your test to
completion. However, I understand the FP-exception still occurs. I'll leave
the debugging to other people (you ?), but let me know when a correct
version of PoVRay is available. BTW, this was sent to the axp-kernel-list,
by Richard Henderson:

<START of QUOTE>

There are two problems that showed their head while running
one of the recently added libc math tests.

(1) We fail because libm was built without -mieee, and so don't
pass muster on "regular" denormals and infinities.

(2) We fail because the EV5 has no switch to turn 0/0 into either
a software completion or a native NaN, and so the kernel
always sends SIGFPE.

the later is, imo, a kernel bug -- when we get an exception we
should run it through the kernel's ieee_get_fp_control and see if
it really applies.

For the former, I wonder if -mieee shouldn't be the default, with
the option to turn it off when you _know_ it isn't needed. Isn't
this the better way to go, with the fewer surprises?

Opinions from those that actually do math for a living?

<END of QUOTE>

Could item (2) possibly be related to our problem ?

For now, what I can do is render the picture at the largest size possible
without running into this problem, to get at least a fair comparison
between the Pentium and the Alpha. Right now I started one at 640x480 on
the Alpha. When it succeeds I'll try the same on the Pentium.

Memory used is less now (35M rather than 48M), probably due to the fact
that I started it on a freshly booted machine. I may also try the same test
with less memory (16M, like yours), and see what happens.

Some of the results submitted by others also seem to indicate that not only
memory size is an issue, but also memory bandwith. If that is so, you will
probably not want to buy one of the lower end Alpha's: memory bandwith is
known to be the weak point of the UDB.
This suggests another variation. Is there anyone out there with a PC164,
366MHz and 8 SIMM's installed, who would be willing to run the same test ?
(On the PC164, the data path between the L3 cache and main memory is as
wide as the number of SIMM's: 128 bits with four (like mine), 256 bits with
eight.)
