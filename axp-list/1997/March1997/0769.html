Shawn Dawson here,

I used Insure++, this product used to be called Insight. I
run it on SGI and a Sun machine, but not a Dec machine.

It is a good tool finding memory leaks, memory read/write errors,
and a bunch of other stuff. It is SLOW, so I don't use it
every day, but once a month or so, I'll run my code through it
to see if I've made any mistakes.

Insure basically replaces your C compiler, so I compile all of
my code, plus any libraries that I have the source code for, with
Insure, and make executables, then run them through my test suites.

I have a mix of Fortran and C in my code, and Insure doesn't
handle the Fortran. It gets confused and doesn't know what
the Fortran does to the memory or pointers. There is another
product called Purify (I have no idea if it's ported to Linux)
that instruments the .o and .a files directly. Purify works
fine with object modules which come from Fortran or C, so
for me, Purify is better.

However, if your code is all C, then Insure can be a big help.
