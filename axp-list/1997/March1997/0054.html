Michal Jaegerman writes:

..buggy script if you are not using UTC for your time base.
In general the stuff [in /etc/rc.d/rc.sysinit] should read somewhat like that:

if [ ${CLOCKBASE} = "GMT" ]; then
GMT=" -u"
else
GMT=""
fi
if [ ${CLOCKMODE} = "ARC" ]; then
clock -A -a $GMT
else
clock -a $GMT
fi

but this requires changes in a setup process. ARC mode and GMT mode
are independent.

Good point; I think this makes for a good patch suggestion for RedHat;
of course the config-panel time tool should then be changed
accordingly. In general, keeping time in UTC is better (because it is
time-zone independent, duh), unless one also uses a system that can't
deal with conversion from UTC CMOS clock (we know what those are :^)
