> I would like to flash milo so I can
> boot it directly (without Windows NT ARC)

I've gotten fmu to run on my machine by the following procedure:

1) Rename milo "fwupdate.exe" and place on dos floppy disk.
2) Change jumper so the "fail-safe booter" runs.
3) Insert floppy, reboot machine, milo (renamed fwupdate.exe) will run.
4) Run fmu.gz from HD.
5) Try flashing milo into position 3 (after WinNT ARC monitor). Test.
6) Try erasing WinNT ARC monitor, then flash milo in its place. Test.
(You can recover ARC by rebooting with real "fwupdate.exe" on floppy.)
7) Don't erase "Block 0" of NVRAM; it contains "fail-safe booter".
(It isn't labeled by fmu, but it does sit in Block 0).
(Yes, you can recover if you erase "fail-safe booter" but
it requires additional sofware; not fun...)

On the pc164, the SROM at boot time does not look for a "bootopt"
number so milo located in the third position cannot be run. One
might expect milo written over WinNT ARC monitor would run in
position 2 when the jumper is set to boot WinNT ARC, however I
tried this and it didn't work. Either the SROM looks specifically
for "WinNT ARC" or there's something wrong with milo so that it
doesn't run correctly in place of ARC. So I'm still using WinNT
ARC monitor to boot milo from a small DOS partition. Am I doing
something wrong ?
