Traffic from eric:
> Just installed redhat 4.1 on an alcor w/ 256mb ram.

Equally so, only 128mb ram.

> Top reports seemingly correct process sizes, but ps reports ridiculous numbers.

I had not tried top...

> As time goes by, even with the machine not doing much, the "used" field
> increases fairly quickly to the maximum amount of memory in the machine.
>
> I'd like to know whether there is a really bad memory leak happening, or
> whether it's just the memory usage reporting that's wrong... and, of course,
> whether anyone else has seen this and knows how to fix it.

I have seen this behaviour. I have not been able to debug nor repair it.

Logging in on multiple vtys gets a 16MB process reported by ps for each login
process on that vty. I'll note that it reports the 16MB *SZ*, and a 1.3MB
RSS. The RSS is what top is correctly advertising above.

At any rate, what I'm seeing in terms of memory usage is the set of buffers
slowly taking over all available RAM...when I start a ramhog, however,
the set of buffers decreases. I have not, as yet, swapped out. It just
seems that, on the DECs, Linux uses buffers more and thus allocates more
memory to that end...quickly relinquishing free buffers when memory is
needed.
