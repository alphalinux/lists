All,

For some reason, the current Alpha Linux amd is built without NIS maps
enabled (thanks to Miguel de Icaza for pointing that out!). Although
there is a remark in the linux makefile about "..not on Linux (yet)", I
built it with the defines turned on and it works fine.

For those among us feeling experimental:

1) Download amd-920824upl102-7.src.rpm

2) rpm -i amd-920824upl102-7.src.rpm

3) cd /usr/src/redhat/SPECS

4) rpm -bp amd-920824upl102.spec

5) cd /usr/src/redhat/BUILD

Apply the following patch from BUILD:

*** amd-upl102/config/Makefile.config.orig Sun Feb 9 10:10:57 1992
--- amd-upl102/config/Makefile.config Sat Mar 8 11:49:29 1997
***************
*** 53,58 ****
--- 53,63 ----
# for Hesiod and/or Named.
#
#RESOLV = -lhesiod -lresolv
+ #
+ # We don't need the resolver, but must specify libnsl.so to
+ # link yp functions in.
+ #
+ RESOLV = -lnsl

#
# Define XLIBDIR if you have libraries not on the standard
*** amd-upl102/config/Makefile.linux.orig-rh Sat Mar 8 11:45:52 1997
--- amd-upl102/config/Makefile.linux Sat Mar 8 11:47:19 1997
***************
*** 52,58 ****
YACC=bison -y
LEX=flex

! HAS_NIS_MAPS = -DDIRENT #-DHAS_NIS_MAPS -DHAS_NIS_RELOAD # not on linux (yet)

# The following syntax only works with GNU's make - which all linux users have.
# Linux can lose the segment padding to page boundaries for these binaries.
--- 52,58 ----
YACC=bison -y
LEX=flex

! HAS_NIS_MAPS = -DDIRENT -DHAS_NIS_MAPS -DHAS_NIS_RELOAD

# The following syntax only works with GNU's make - which all linux users have.
# Linux can lose the segment padding to page boundaries for these binaries.

6) cd amd-upl102

7) make

Now, I have only tested the actual 'amd' executable that is built in this
manner. I had previously installed the binary rpm, so my installation was
limited to copying the new one over it.

YMMV, but NIS maps now work just fine on my UDB!

Erik, why did you turn this feature off?

Steve
