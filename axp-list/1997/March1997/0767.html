> Mar 25 18:30:19 floyd kernel: sendmail: memory violation at pc=155558f6c70
> rp=155558e0248 (bad address = 1201b4772)
> Mar 25 19:30:20 floyd kernel: sendmail: memory violation at pc=155558f6c70
> rp=155558e0248 (bad address = 1201b4772)
>
> WOW! I didn't notice that till now, check out the time, it happens every
> hour exactly! This is even more strange than I first thought. Any ideas
> anyone??

The time isn't particularly weird. You probably have sendmail set up to
run the queue every hour (something like `-q1h' on the command line). It
looks like every time this happens, the queue-runner is segfaulting.

I'd expect bad memory problems to be a lot more random than this - unless
the machine is _very_ lightly loaded, you'd probably trip over them more
often. Strange.
