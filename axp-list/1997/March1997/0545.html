Hi.

A recap: I'm running Alpha RH4.1. The error msg
passwd: User not known to the underlying authentication module
happened when I did the following:

>> (1) Started ypbind running.
>> (2) Took a test user out of /etc/passwd, relying only on the NIS
>> passwd map for his information.
>> (3) Logged in as said test user.
>> (4) Tried to change his password, using "passwd".

One response:

> Unless Linux does something different then SUN you need use the
> yppasswd command to change a nis users passwd.
>
> I know that some systems make passwd work for files, NIS and NIS+ with
> one command, but some don't.

Oops. You're right. OTOH, yppasswd seemed to die with an RPC
mismatch. Also yppasswd and yppasswdd are only available as source,
not as binary RPMs; they're not part of the ypclients RPM.

Another data point I forgot to mention: chsh worked OK; I didn't need
ypchsh.
