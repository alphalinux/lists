All,

What exactly is required for the Alpha automounter to properly find its
maps using NIS? My 6x86 box has no difficulty at all finding the map on
the domain server when started as:

/usr/sbin/amd -a /amd -l syslog -- /net amd.master

where "amd.master" is exported as an NIS map. On the UDB, using the
latest glibc and amd from RH4.1, ypcat properly displays the map, and amd
does not complain at startup. However, it refuses to take any mount
action whatsoever. Copying the map physically to the UDB and specifying
the file by absolute path in /etc/sysconfig/amd _does_ work.

Granted that the amd documentation is far from a model of clarity (rivaled
only by the original DRI CP/M manual), but this does work on the other
boxes.

Any ideas??

Steve

P.S. - Yes, I did try placing

automount: files nis

in nsswitch.conf. No luck.
