On Mon, 31 Mar 1997, Richard Henderson wrote:

> > And: Can somebody give
> > an authoritive answer whether glibc includes threads and semaphores which
> > should otherwise be provided by libpthread???
>
> Glibc itself does not contain any of these, primarily because Xavier
> cannot sign the FSF copyright assignment though he still gives it away.
>
> So there is a linuxthreads add-on for glibc that should be configured
> when glibc is built so that the result is a thread-safe libc. But
> you need to use the version of the add-on that matches the version
> of glibc that you are building.
>
> It is unfortunate about the timing of events wrt the glibc release
> and the RH releases. The current glibc is, imho, more stable than
> the 961212 snapshot in RH4.1, but it is not binary compatible with
> the older version.
>

it appears the 961212 glibc rpm from redhat does contain linuxthreads 0.5.
# rpm -q glibc
glibc-0.961212-3
# rpm -ql glibc|grep pthread
/lib/libpthread-0.5.so
/lib/libpthread.so.0

am i missing something here?

has anyone compiled either Chris Provenzano's mit threads or fsu threads
under linux/axp?
