Did you by any chance use a long, complex (like a root passwd should be :)
passwd when you installed? RH's installer doesn't like them much...I
believe the cut-off is 8-9 characters. What are you using to boot? You
should be able to pass 'single' to the kernel thru any bootloader. For
instance, in MILO you would type: boot sda2:vmlinux.gz root=/dev/sda2
single -- or something similar :)

That ought to allow you to login and change the password (the bug isn't
present after the installer, don't worry)
