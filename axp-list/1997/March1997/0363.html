I have gotten gcc-2.7.2.2 / g77-0.5.20 to compile and work (so far) under
RH4.1. My system is RH4.1 installed from the RedHat PowerTools CD set
plus all the updates on ftp.redhat.com as of a few days ago.

I used gcc-2.7.2.2 with the patches from g77-0.5.20 applied. I had to go in
at tweek the configuration a bit though. It seems that while the patches with
g77 do add support for alpha-linux to gcc, it is support for an older, non
ELF, system.

The changes I made, as I remmeber them are:

Followed the install instructions that came with g77 up to the point where
there tell you to fun configure

Copied config/alpha/elf.h from the gcc SRPM from RH4.1 (after applying the
patches in the spec file).

Edited the configure script to use config/alpha/elf.h for tm_file with
the alpha-linux configuration.

Added "#define HAVE_STRERROR" to config/alpha/xm-linux.h

Changed the ifdef around "extern void *alloca ();" in config/alpha/xm-alpha.h
to "#ifndef alloca"

Run configure, using the parameters shown in the Rh4.1 gcc spec file.

make bootstrap

At this point the build proceeded normally. Gcc 2.7.2.2 was able to compile
itself.

I've tested it on a few small C and Fortran programs of my own, and on xmcd.
Everything seems to be working fine.

I tried to use it to compile octave. Octave compiled, but seg faults on startup
when you run it. This was the same behavior using gcc-2.7.2.1/f2c though.

If anyone wants more details or a better explanation, let me know. Be warned.
This is the first time I've ever looked at the inards of gcc. I don't really
know what I'm doing. It does seem to work though. *shrug*
