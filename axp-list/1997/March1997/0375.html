I eventually found by using strace that my locking up network init scripts
were locking up due to no name server. It would seem there is a bug in
the resolver somewhere, as strace show ifconfig sitting in a loop sending
messages to 127.0.0.1, port named, and timing out, and trying again, and
again. Of course named inst started to much later in the boot process,
and at any rate, there is a bit of a 'chicken or the egg' problem if its
going to try and talk to named via 127.0.0.1 before lo is bought up !

I would have though that sending numerical addresses to the resolver was
a bit pointless.

Putting loopback, my eth0 address and my network address in /etc/hosts
seems to have fixed it.
