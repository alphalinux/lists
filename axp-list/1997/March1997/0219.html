I have manually upgraded 4.0 from 3.0.3 and 4.1 from 4.0. The key is to pay
close attention to dependencies. I generally start off my upgrading
libraries, and then proceed to the other packages. When I get a failure to
install due to dependencies, then I go back and install the packages listed.
Some packages, I had to force install with --nodep.

It is a bit of a pain, generally taking a couple of hours to get
everything installed cleanly, and holding ones breath on the first reboot
that everything installed okay.

This is a fairly keyboard intensive procedure,
running rpm manually for individual packages. What you could do is to query
each of the packages in advance using the following, assuming you are in the
directory with the package files:

find . -name "*.rpm" -print -exec rpm -qpR {} \; > packagedeps

This should get you a list of the dependencies for each package. Then you
can review this list and write a script to install the packages in the
proper order.
