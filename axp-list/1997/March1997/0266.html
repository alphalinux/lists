Al,

I am in the same court.

1. Grail - Grail was developed using Python 1.3 using a Tkinter based on an
earlier version of Tcl/Tk than on Red Hat 4.1. The Grail site warns against
using a later version of Tcl/Tk, although no specific reasons are given.
Since the installed Python is 1.4, there may some problems related to this,
also. But if these work in Intel, then we have a alpha port problem. I
have been meaning to look at this, but have been distracted by item 2.

2. PGP - Something is seriously broken here. Compiling crypto.c results in
a gcc signal 6 fault. I have to reboot and
fsck after each fault, or else I end up trashing my filesystem with repeated
attempts at compiling this piece of junk. Sorry, I know PGP is important,
but I am also very frustated with the damage it is causing. Perhaps someone
who is *VERY* experienced with alpha/linux porting should look at this.

3. Kerneld - try using the latest modutils (modutils-2.1.28). Personaly,
I compile a monolithic kernel with everything I need compiled in, or install
modules with modprobe rather than kerneld. I really do not see much need for
kerneld on a machine with a standard hardware configuration like the UDB,
considering that the alpha does not suffer from a kernel size limitatation
like on Intel.

4. PCMCIA-CS - No experience here. I have been thinking about getting a
PCMCIA modem for my wife's laptop, just so I could borrow it to tryout
PCMCIA services on the UDB.

Sorry, this is not help, just a confirmation that you are not alone.

David.
