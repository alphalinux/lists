>
> I got my hands on a PC164 at 433Mhz. We are running Red Hat 4.1 +
> updates on this machine. I can crash the machine reliably by running:
>
> rpm --verify -a -v
>
> When I do this I get several minutes of hex display on the screen.
>
> Sometimes the machine reports "CIA machine check" and halts. It
> crashes a couple of times before being able to compile a complete
> kernel.
It's your RAM -- almost all the commodity parity
memory modules on the market has this problem.
It may work for a while, or for a long time with
light load, or with 128bit memory interface (using
four SIMMs). It's easy for linux to spot the
problem -- you can never finish compiling the
kernel, while it seems to take a bit longer
for NT to notice it.
