I think I may have some more information about the password problems.

When we reboot the machine, the /var/run/utmp file is new. This is why
when the first person logs in, everything is fine. Once the first user
logs out, then there are problems.

utmp does not get written to properly, as the number of users shown by
'w' will continue to increase. If I delete the /var/run/utmp file
completely, then passwords work again, and 'w', etc. show the correct
information until more users start logging in. This problem is also why
shadow doesn't work, since it can't get the proper user information.

Does anyone know of a bug with the libraries and utmp?
