> ... and ended up with linux-ld.so.2 having a segmentation fault while

Whee. You've had a close encounter with a bug in the binutils shiped
with RH. There are two ways to proceed.

(1) Add

ifeq ($(subdir),elf)
sysdep-CFLAGS = -DNO_AXP_MULTI_GOT_LD
endif

to the configparms file while building glibc.

(2) Get binutils 2.7.0.11.

I, as well as the glibc release notes, recommend the later. Though
be aware that you'll need to rebuild libgcc.a at the same time.
