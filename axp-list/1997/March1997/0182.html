Hello all,
I decided to try to set up a cross compile environment so I could
compile Linux under Digital Unix 4.0. This was trivial under the 2.0.X
because I could use gcc compiled for Digital Unix, but since the 2.1.X
kernels have to be copiled as ELF, I build binutils and gcc as cross
platform tools.
The first problem I ran into was that the arch/alpha/lib Makefile
detects if you are cross compiling and trys to use the Digital Unix ld
command. This workes if you are compiling and ECOFF kernel, I suspect
it will not work for ELF. I just commented out the check.
Here is the next problem I found:

make[1]: Entering directory `/group/jlnance/src/linux/arch/alpha/lib'
/group/jlnance/bin/alpha-dec-linux-gcc -D__KERNEL__ -I/group/jlnance/src/linux/include -c -o stxcpy.o stxcpy.S
stxcpy.S:22: alpha/regdef.h: No such file or directory

Does anyone know where to get this file? Its not in the 2.0.28 source.

Now that I think about it, I probably have to applay some diffs from
azstarnet dont I? I am downloading axp-diffs-2.0.21.gz now and will try
again.
