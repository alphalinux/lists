At 12:26 hrs. 03-01-97 -0600, you wrote:
>>
>>
>> I got my hands on a PC164 at 433Mhz. We are running Red Hat 4.1 +
>> updates on this machine. I can crash the machine reliably by running:
>>
>> rpm --verify -a -v
>>
>> When I do this I get several minutes of hex display on the screen.
>>
>> Sometimes the machine reports "CIA machine check" and halts. It
>> crashes a couple of times before being able to compile a complete
>> kernel.
>It's your RAM -- almost all the commodity parity
>memory modules on the market has this problem.
>It may work for a while, or for a long time with
>light load, or with 128bit memory interface (using
>four SIMMs). It's easy for linux to spot the
>problem -- you can never finish compiling the
>kernel, while it seems to take a bit longer
>for NT to notice it.
You can also get the same problem with an overclocked machine. I worked on
a 433 accidentally pushed to 500 with exactly the same symptoms.
Also, video card used can make this more sensitive. On the same overclocked
machine a Number 9 GXE330 it was almost unusable, while with a Diamond S64D
2201 ( Trio64+ 765 chip ) it ran almost fine. Compiles usually finished,
things ran OK.
SCSI termination can also make an Alpha twitchy too.
I have to say that it is almost impossible to say " It's the ____" (fill in
the blank with a prognosis). There are too many things which will make a
machine unstable while it still runs (sort of..).
