Hi.

We are running Redhat Linux 4.1 (kernel version 2.0.28) on a DEC Alpha
(Mikasa). Our NIS server is an RS/6000 running AIX 3.2. Both
machines are on the same ethernet segment. Prior to the release of
yp-clients for Alpha, we got a working version of ypbind custom-built
for us by H. J. Liu, when he was kindly helping us out with an attempt
to go to the new glibc. This version of ypbind, based on version 2.x,
works fine.

The version of ypbind from the current RedHat RPM (3.0.1), however,
causes trouble. So far as we can determine, the problem arises when
the NIS server, for reasons unknown, fails to respond for 30 seconds
or so to some of the periodic requests for NIS info from ypbind on the
Alpha. The requests are repeated at an ever-increasing rate until
they flood the network, jamming it.

Anybody know of a way to fix this, whether by setting some config
parameters or patching the source?

Thanks.
