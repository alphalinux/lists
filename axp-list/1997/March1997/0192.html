Though this leaves me a bit in the dark wrt the state of my system, it
certainly seems to imply that overclocking the PC164 is not as easy as I
thought.

So lets look at a more legitimate way to speed it up. Currently I'm using
64M of RAM in 4 x 16M SIMMS, thus using a 4 x 32 = 128 bits wide path
between cache and memory. If I had 8 x 8M (8 x 16M would be nice too,
unless problems with >64M memory still persist) the path would be 256 bits
wide. Can anyone comment on resulting performance gains ?
Does the L3 cache line size depend on the width of the data path, or is it
fixed ?
