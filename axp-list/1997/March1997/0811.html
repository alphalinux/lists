On Thu, 27 Mar 1997 19:32:08 -0500, Uncle George <gatgul@voicenet.com>
said:

> Steven Kokinos wrote:
>> Has anyone heard about linux having problems on the alpha with file
>> sizes over 2GB. Apparently this has something to do with the ext2
>> filesystem.

> Partitions can be greater that 2 gig
> files can be only up to 2 gig! - from previous dialogs - no change in
> sight!

True up to the last bit. Partitions can be pretty much any size, as
long as the device drivers support it --- ext2fs copes with filesystem
sizes up to about a terrabyte. Individual files are limited to 2GB.
There *used* to be a limit that your boot partition had to be <=2GB;
that was a limitation in MILO which I understand disappeared a while
ago.

As for there being no change in sight, that's not entirely true. :)
I'm back in the land of the living as of this Monday (thesis
submission day!!!), and the top three things on my todo list at that
point will be a new release of supermount, an updated defrag, and
proper 64-bit file support on linux/axp ext2fs. Watch this space.

Cheers,
Stephen.
