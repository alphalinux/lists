Michael Brennen wrote:

> Now: is it possible to build a linux kernel to talk to the serial port?
>
> I cannot get an OS to talk to the serial port. I know the 2.0.26 kernel
> has the option to copy the console output to the serial port, but it does
> not seem to want to listen for keystrokes there. Is there something I
> have missed along the way?

Yup, you need a pgm at the other end of the serial port listening. If u
want to login, then you should have the getty/login programs working for
that serial line/device. Editing /etc/inittab ( u be careful ) to
startup the getty pgm on ttyS0 should start u off. There is also
/etc/gettydefs - a set of tables used to set parameters on different
serial ports.
