All,
I've just built all new MILOs for 2.0.29. The changes are:

[1] PCI setup patch from David M-T (changes devsel for Saturn ISA bridge),

[2] A new command, SET PCI_LATENCY nn; where nn is 0-255, default is 32.
This command basically sets how long a PCI device will hang onto the PCI
bus once it gains control of it. Make it 255 and better transfer rates
across the PCI bus will occur but device drivers will have to wait longer
to get control of the PCI bus. I've been running my EB164 like this (255)
for a month and I've not seen any problems. I have seen better performance
from the system both in graphics and in SCSI transfer rates.

I have not included the sources, I will but they need tidying up
a little. Also, I have built ECOFF images (so that I can link in Digital's
BIOS emulation code) on an ELF system. I need to package that up before I
release it - this saves me booting my system as ECOFF to build MILO and it
will give me back a lot of disk space.

I have tested what images I can, but I really need you folks to
try this out and let me know what works and what does not. I want 2.0.29
MILO to be more or less the last stable build so that I can move over to
2.1.* and rearrange the code and add new features.

Dave

ps I am not porting the Digital BIOS emulation code over to ELF myself. That's
been done in the US. The port is part complete and I have been told that it
will not be too long. Meanwhile I did play with objcopy and the linker to
see if I could force a valid link, but that was a fruitless exercise.
