Hi,

A problem with `clock -A` on PC164 boards under RedHat 4.1. When you issue
the command a number of times in succession, you will find that sometimes
it reports something like:
Sun May 17 17:17:17 1998, rather than:
Wed Mar 5 20:17:29 1997.
(Note the recurring 17s)
This is particularly nasty when it occurs with the `clock -A -a` during
startup.

I had this same problem running 3.0.3. It was solved by a fix Jay Estabrook
submitted to this list. I've never noticed it under 4.0, but now I've
upgraded to 4.1, and it's there again.

Q.:
Did Jay's fix never make it into the official source,
or,
was it taken out again for some reason,
or,
is this a different problem altogether ?

- It fails less frequently than the 3.0.3 version, so maybe it is a
different problem.
- The binary Jay sent then now dumps core (Segmentation fault)
