I've tried it all...I think, please somebody give me some directions (has
this been solved / discussed when 4.2 was 'new'? I have not been able yet
to find anything in de axp-archive)

I am trying to install RedHat Linux 4.2 on my UDB. The UDB is a 166MHz with
32Mb RAM, 256kB cache and a 340Mb HD. There is a SCSI CDROM attached to the
UDB, which I intend to use for the install.
What I am seeing: after 'rawriting' the required images to three floppies,
I start up the UDB, insert the MILO-floppy in the diskdrive and select the
freshly made 'Boot floppy milo'-option from the menu. At that point is
starts reading the flop, here are (some) of the things I see passing by:
- Linux Loader v1.5
- opening file .......(0)....(0)....\milo
- (some start-up / boot type messages)
- ..... 810 SCSI...
- memory is 32MB
# Error .... Correctable ECC error... correctable read error to memory
occured at adress 55a918
# Several other errors occured
# xxxxxxxxxxx xxxxxxxxxxxx (x = hex number, I'm not quick enough to note
# xxxxxxxxxxx xxxxxxxxxxxx these down)
# xxxxxxxxxxx xxxxxxxxxxxx
# xxxxxxxxxxx xxxxxxxxxxxx
These last rows (with # in front) are being generated indefinitely it
seems. The memory adress reported starts of being random, after some time
it continues to give the adress given above.

I am puzzled. I've tried:
* Removing the second 16MB (2 x 8) of memory I added myself.
* Replacing the original 16MB with the the ones I bought.
* Updating the firmware from BL5 3.7-628 to BL5 3.8-36 (=the lastest from
DEC - ftp)
* Reformatting & Re-rawriting the milo-floppy.
* Downloaded the milo-image file from RedHat-ftp and made floppy once
again.
All this did not solve the problem :(
Please gimme a hint...
