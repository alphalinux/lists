Thanks to those who replied to my inquiry. Basically I received three
answers:

a) The 'gravis ultrasound ace' works. All PnP cards tried
didn't work.

b) SoundBlaster PnP cards work fine, by using the isapnptool
utilities.

c) SoundBlaster PnP cards programmed to non-PnP mode by
DOS/Intel utilities available from Creative's web side.

Well, I wasn't able to get the GUS card, and I don't own an Intel
(but I wasn't able to find those utilites, either), so I went for b).
That worked, but almost any tool needed some patches.

First problem, to compile the sound driver as a module on the Jensen,
it was necessary to include _bus_outb and _bus_inb in
linux/arch/alpha/ksyms.c.

Although downloaded from Redhat's devel tree, the isapnptool needed to be
patched, too. Simply comment out the 'ioperm' calls. Apart from that
the tools worked fine. I used 'pnpdump' to get a list of supported
features, edited that, and now 'isapnp' is used to initialize the
card during startup. Later the sound driver is loaded as a module.

The card I bought, advertised as "SoundBlaster 16 VALUE PnP", model
ct4170 -- not listed on Creative's web site yet -- apparently doesn't
support 16 bit DMA. So it was necessary to answer "1" to the related
question during sound setup. I guess this degrades performance, so I
felt sort of ripped off by that buy.

Various sound playing utilities needed patches to get them 64 bit
clean. I will try to make these patches, as well as the ones mentioned
above, available from my web page the next days. Also I am going to
send them to the authors. In case you like to try them at once,
just mail me.
