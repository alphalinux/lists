Like some of the previous RedHat releases, generally it's OK but it has one
or two little bugaboos which basically kill forward progress until
corrected. For example, the install script didn't put in the host name to
/etc/hosts. This had the effect of making things hang seemingly forever on
the inital bootup (if your system is stand alone - if your IP is in a
connected nameserver, it can resolv it and go on). I'm still working to get
ppp going. It takes a while to get some things squared away and generally
the fixes show up in RedHat's /updates directory fairly quickly. PCMCIA
compiling is almost always problematic no matter which release of RedHat
you have. In the latest, 2.9.12+RH5.0(2.0.30), there were boo-boos in
ftl_cs.c and netwave.c (neither of which I use so I took them out of the
Makefile).

"Pioneers" help make better products for the rest of the (supposedly 3000+
UDB) users. My setup is likely to be different than yours and both ours are
likely to be different than RedHat's. Is it worth it? I think so, but I
wouldn't want to make a living being a guinea pig^h^h^h regular paying
customer. (He says as he gets ready to shuck out the coin to RedHat for the
fourth RedHat release just to have the CD around...)

Hope this helps.
