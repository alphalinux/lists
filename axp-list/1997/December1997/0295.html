On Mon, 8 Dec 1997, C. Le Cannellier, Pdt LUG Normandie wrote:

> Hi,
>
> After upgrading Xfree86 in order to support a ViRGE DX board, I have
> lost the original X environment conf (no more fwm95, just twm, no more
> XF86Config, just X86Setup, ...)
> After copying a couple of executables and missing libraries from the
> original /usr/X11R6/... to the new onw, I finally had things working fine using
> xinit (startx from command line mode under init3).
> However, when I init 5 (xdm way), I have the twm environment. What am I
> missing ? BYW, what it the purpose of the xbanner program ? what si it supposed
> to display ? Thanks for any input.

xdm uses your .xsession file instead of your .xinitrc file. I have a link
to make it use my .xinitrc:

ajo@wopr ~ >ll .xsession
lrwxrwxrwx 1 ajo ajo 8 Apr 21 1997 .xsession -> .xinitrc*

If no .xsession file is available, there is a default fall-back twm
environment.

The xbanner program displays a "Red Hat Linux" logo and a penguin. Better
than the normal default X background I would say.

/Andreas
