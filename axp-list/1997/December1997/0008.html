>>> Amedeo Perazzo said:
>
> I have a DEC PBXGB-AA 8 planes board. Is it possible to make it work with
> XFree86 TGA server for linux on alpha?
>
> Bus 0, device 12, function 0:
> VGA compatible controller: DEC DEC TGA2 (rev 34).
> Medium devsel. Fast back-to-back capable. Master Capable.
> Latency=255. Min Gnt=8.Max Lat=64.
> Prefetchable 32 bit memory at 0x2000000.
>
> If actual XFree TGA server is not correct for my board (string TGA2 in
> /proc/pci seems suspicious...), shall I have any hope to make it work
> under linux? Do you know any workarounds? Is there any commercial server
> which could work for PBXGB-AA 8 planes?

Sorry, PBXGB-AA is TGA2, and is NOT supported by any of the TGA servers.
Unfortunately, AFAIK, there are no plans to implement such support in XFree86,
nor have I heard of any plans by commercial vendors of servers.

Trade it for a nice Millennium II... :-)
