I'm currently having some problems getting a 21140 working at 100 Mbps and I
was wondering if anyone has any clues....

Here's the background:
Multia (hence an onboard ethernet)
Generic 21140 clone.

I've tried tulip.c:v0.86A... No Joy.
I've tried tulip.c:v0.86I... Machine crashes (there's also a compile time
warning)
I've tried adding ether=0,5,eth0 on mile
I've tried half a dozen other ether= lines
I've tried loading 0.86 as a module and passing options... It wouldn't load
the module... It says resource busy.
I've tried de4x5.c:v0.52... The card lights up "100 HD", the switch reports
"100 FD", the card doesn't work, ifconfig shows TX errors. Locking the switch
at 100 HD (or 10 mpbs anything) doesn't work.

Here's what I'm running now:

tulip.c:v0.79 9/3/97 becker@cesdis.gsfc.nasa.gov
eth0: DEC DS21140 Tulip at 0x8000, 00 40 05 40 ea d3, IRQ 15.
The following verbose information is emitted for
bug reports on media selection.
eth0: EEPROM default media type Autosense.
eth0: Index #0 - Media MII (#11) described by a 21140 MII PHY (1) block.
eth0: ***WARNING***: No MII transceiver found!
eth1: DEC DS21040 Tulip at 0x8800, 08 00 2b e2 c8 b7, IRQ 10.

In this mode, it works at 10 Mbps half duplex just fine. I'm also having the
same problem in an AlphaStation 255.

Can anyone help? (please please please :)
