Thomas M. Langley wrote:

> I have the MILO image loaded for my processor and use the AVANTI.img file
> for the kernel. (The XLT.img
> will not boot even though I have a "true XL as defined by Digital".

For the Alpha XL233/266 you'll need the Avanti or XL images,
the XLT image is for the XL300/366/433 systems.

The XL300/366/433 have an AlphaBIOS instead of the ARC firmware
the XL233/266 systems have and are referred as XLT.

> I tried copying ramdisk.img to ramdisk.gz running gzip -d .... followed by
> gzip ... and the resulting file was not = in size to the original. Also got
> a 'trailing garbage ignored' message on the decompress.

You'll need to do some raw writing from a DOS or Linux machine
to create the disks from the image files, gzippin' won't work.

In MSDOS you'll do: rawrite myimage.img
In Linux you'll do: dd if=myimage.img of=/dev/fd0

Good luck,
Antoine
