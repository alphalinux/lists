Well, it's more a matter of things in there that aren't desireable in the
official sources, mostly header file incompatibilities (done because the
Alpha libc is different from INTEL libc, primarily), along with other things
like removal of the 2Gb file limit, use of 64-bit args/return values from
the common read/write device routines, etc.

One *could* remove all the "objectionable" stuff, ending up with essentially
bugfixes and platform support patches only, which might be acceptable.

However, would it be as useful any more under Red Hat releases <= 4.2, I don't
know... ???

I'm very reluctant to abandon support for RH 4.2 and prior; I'm even more
reluctant to take on maintenance of two sets of patches... :-\
