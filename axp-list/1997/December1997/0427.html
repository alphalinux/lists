I don't have a "corrective" algorithm such as you describe,
but I do have a fairly simple parabolic interpolation algorithm.
It requires that the abscissas of the points between which
you are interpolating be evenly spaced, and each interpolation
requires 3 points, rather than the two used in linear interpolation.
If the 3 points are collinear, it reduces to linear interpolation,
so it would probably give good results in both parts of your
curve. It might not be terribly clean at the transition,
but it would not do anything disastrous. Let me know if you
think this might be appropriate and I'll go digging through
my old notes. Somewhere I have the code in Pascal, which
p2c should be able to convert.
