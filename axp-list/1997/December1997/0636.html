On Thu, 18 Dec 1997, Johan Almqvist wrote:

> On Wed, 17 Dec 1997, Aaron M. Gowatch wrote:
>
> > Hello --
> >
> > I'm investigating two options, a 600Mhz 21164 based system from Microway,
> > or a dual Pentium II based system. Both machines would be equally
> > equipped with memory and disk, with the exception of a DIMM capable Alpha
> > motherboard. Memory latency would probably not be a critical issue in
> > this particular application. The Alpha based system is a little bit more
> > expensive than the Intel based PC, but in this case performance is more
> > important than price.
> >
> > I've had a bit of trouble finding information pitting these two platforms
> > against each other in real world, non-fp applications. Does anyone have
> > any opinions or experience with both platforms?
>
> Hmm. I'd go for the Intel platform in this case (if anything...) What you
> are looking for in this case is disk and network i/o performance, and the
> existing drivers are more tweaked on the intel side...
>
> Memory i/o and raw processing power may be bigger on the alpha side, but
> as mail translates into many parallel low-bandwith processes that are
> independent of each other, you should get substantial performance from a
> dual-CPU system.
>

Well, the machines tasks are not constrained just to handling mail, though
most of what happens has _something_ to do with mail processing.
Potentially, we'll have over 300 users running IMAP on that machine within
a few weeks. Obviously, a sizable amount of RAM is required. Either
machine will be configured with hardware based RAID-0. The machine that
we're currently running on is a Pentium 180 running BSDI, which frequently
becomes CPU bound. Once the load on a machine that runs sendmail gets so
high, it ceases mail processing until the load is back under control.
We're trying to avoid that with a hefty box that will scale easily.

Aa.
