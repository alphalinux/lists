I looked at some inline functions in /usr/include on Linux/Alpha
and <__math.h> there is bracketed by
#if defined (__GNUC__) && !defined (__NO_MATH_INLINES)
...
#endif

So far, so good. Only in <math.h> one can find this gem:

/* Get machine-dependent inline versions (if there are any). */
#if defined (__NO_MATH_INLINES) || defined (__OPTIMIZE__)
#include <__math.h>
#endif

Eh?? It seems to me that '!' vanished here somewhere. :-)

OTOH no big harm likely done in any case. If __OPTIMIZE__
is defined you will get your inline functions no matter what.
Still things look a bit funny. <__math.h> has nothing but
inline functions only. Or there are plans to put there something
else, not inlined, as well? If yes then what?

Comments? (This is RH 4.2 distribution, do not know how this
looks for other ones).
