> I'm wondering if anyone can help me out on this one. I recently installed
> a Graphics Blaster 3D (Cirrus Logic GD5464) 4mb video card into my UDB (it
> was in a Win95 machine previously). Text mode is working like a champ,
> but when I try to get X running, the server can't seem to find the card.

I thought that this card had a Verite 1000 onboard ?!?

> XFree86 Version 3.3.1 / X Window System
> (protocol Version 11, revision 0, vendor release 6300)
> Release Date: August 4 1997
> If the server is older than 6-12 months, or if your card is newer
> than the above date, look for a newer version before reporting
> problems. (see http://www.XFree86.Org/FAQ)
> Operating System: Linux 2.0.30 alpha [ELF]
> Configured drivers:
> SVGA: server for SVGA graphics adaptors (Patchlevel 0):
> mga2064w, mga1064sg, mga2164w, s3_virge
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

As you can see the precompiled SVGA-Server does only contain servers for
Matrox cards (Millenium and Mystique) and S3 Virge based chips.
Sigh! Either grab the binaries form XFree86.ORG or one of its mirrors
or recompile the Xserver yourself. (Given your UDB you don't really want
to recompile it...)

But why don't you use the builtin TGA of the Multia? Works like a charm...
