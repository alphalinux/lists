Funny.. ;-) Just yesterday I've configured a Multia as a diskless
Workstation. It is now basically used as an X--Terminal. -- Works great!

Here is a brief explanation on how to do it. If anyone is interested in a
more detailed description, please get in contact with me per email. -- If
the topic seems interesting enough for a few people, I could write a
document describing the process of setting up a diskless Mulita.

Here is a short description on what to do.. :-)

+) Set up bootpd (+tftpd) on your server box.

+) get the aboot sources (and the aboot faq)

+) Build a kernel image for your UDB. You must enable the "root filesystem
on NFS" option when building the kernel.

+) Use aboot's "make netboot" option to get a linux kernel that can
be net-bootet

+) Switch your UDB to SRM

+) Finally you should be able to net-boot your UDB with something
like the following command from the SRM--prompt:

boot -protocol bootp ewa0 -flags "root=/dev/nfs
nfsroot=<server-IP><server_export>"

The "nfsroot" points to an nfs export that is mounted as root--filesystem
on your diskless UDB.

The boot process can be automated (so that the UDB automatically net-boots
when turned on) by setting a few environment variables. Unfortunately I
don't have the necessary information at hand right now.

Feel free to get in contact with me for a more detailed explanation.
