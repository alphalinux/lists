I installed RH5.0 and after putting in the hostname to /etc/hosts, decided
to get my Internet link re-established. I set up the ppp inputs the same
way as they were confgiured on my RH4.2 (which still works when I boot from
my 4.2 partition). The /etc/sysconfig/network-scripts for ifup and some
others are very different than 4.2. However, with only the addition of a
IPV4_FORWARD=yes in /etc/network file, it's the same as my 4.2 config, with
ppp0 being my default gateway device and no gateway IP. The chat-ppp0 and
ifconfig-ppp0 config files in /etc/sysconfig/network-scripts are the same
as in 4.2. When I go to crank up ppp on 5.0, it initiates, modem dials,
handshakes with netcom, and gets IP addresses. As soon as that happens
though, I get:

ioctl(SIOCADDRT) device route: Network is unreachable

and the modem hangs up, etc. I checked to PPP-FAQ and it mentioned
potential problems with routed. I turned it off & rebooted and ppp still
fails at the same place.

I hesitate to go thru the hassle of replacing the new network config
scripts with the old ones one at a time to see where the problem is if
there's something in the RH5 setup that has changed - which would make me
have to do something different than I was doing. (It looks like ifup has
been re-written, for example.)

I seem to recall seeing this message before but neither RH's mail-list
archive search nor dejanews turned up anything. I can't use 5.0 as my home
gateway until I get ppp working...
