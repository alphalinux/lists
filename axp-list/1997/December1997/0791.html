Here's a trick that should help with this.
When NT does a disk partition on SCSI, it likes to do a 64 x 32 x number =
of
cylinders layout.
Translate what Linux fdisk is using, and stick to this.
Either prepare at least one partition on the disk in NT or from the NT =
cd
\Alpha\arcinst.exe first, then in Linux do your swap, / and/or other =
Linux
partions.
Or, if you don't want any NT partitions on a disk, the multiply the =
number
of cyls, heads, and sectors per track that fdisk reports in Linux. =
Divide
the result by 32, divide this result by 64, the write down the number =
you
get. reduce to an integer by dropping any remaining fraction.
In fdisk select "x" for expert mode, "s" for sectors per track, reply to
the prompt with "32". "h" for heads, reply with "64". "c" for cylinders,
reply with the result you calculated. =20
"r" for return, "p" to see if it's happy. Create your partitions, "w" to
write, "q" to quit. Reboot. NOW. Don't mke2fs or mkswap until after =
you've
rebooted!

This should help...
