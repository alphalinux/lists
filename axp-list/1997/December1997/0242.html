For the record, the AlphaPC164-SX uses:

"Cypress CY82C693U Hypercache / Stand-Alone PCI Peripheral Controller with USB"

to provide a PCI to ISA bridge along with some of the standard ISA devices:
IDE, RTC, keyboard/mouse, and DMA controllers (serial/parallel/diskette are
provided by a separate SMC chip, the FDC37C935).

Here's what the Cypress documentation says about the IDE on this chip:

"The CY82C693U contains an integrated, dual-channel PCI to IDE bridge. The
IDE controller conforms to ANSI modes 0, 1, 2, 3, and 4 for PIO (Programmed
I/O) transfers. Single word and multi-word DMA transfer modes 0, 1, and 2
are also supported. ATAPI (CD-ROM) protocols are allowed with pre-fetching
disabled. The controller allows for CHS (cylinder-Head-Sector) or LBA
(Logical Block Address) addressing. Each of the tweo channels support two
devices for a maximum of four IDE devices in the system. The controller
allows for PIO pre-fetching, post writing, and DMA PCI mastering in order to
increase overall system performance."

Also stated later in the document:

"IDE DMA is compatible with SFF-8038i (the Small Form Factor Committee
specification defining bus mastering on IDE). Only 16-bit DMA operation is
supported."

And it goes on to say that it supports scatter-gather as well.

Sounds like a case of "the hardware is willing but the software is weak" to me.

Now, as to prior motherboards, like PC164 and LX164, those use a CMD 646
PCI IDE chipset. These are also capable of DMA and bus mastering, I believe.
But there's no LINUX driver for it that I am aware of, probably because:

1. its predecessor, the CMD 640 was very broken, and was rightly shunned by
most INTEL motherboard manufacturers.

2. those same manufacturers were reluctant to employ the CMD 646, because of
how bad the 640 was, so few mobos were built with it, and so no driver.

The above is, of course, pure speculation on my part, but it seems to fit.
Now, as to why the DEC mobo designers chose the 646, I'm not going to guess,
but it prolly was influenced by the ease of implementation.

As to this being a problem only the Alpha has (poor IDE performance when
using the PIO driver), I think you should check the archives of the
linux-kernel list on vger.rutgers.edu. I can personally vouch for the fact
that within the last two months there was great hue and cry from some
posters there about how bad IDE was; I had to keep checking the Email
address to make sure it wasn't from axp-list... :-\
