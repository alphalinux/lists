According to alsani@acsu.buffalo.edu:
>
> Please help me in answering the following two questions:
> 1) Does the statement "an application is not ported to Alpha" mean that
> "an application can't be installed on Alpha"? If yes, then can some body explain briefly how FX!32 will run such application.
>

Yes, "not ported" is equivalent to "won't work" natively (see below).

FX!32 is an emulator: FX!32 decodes x86 instructions and "does the right thing"
on an alpha to make the x86 executable run. This is different than running
"natively" where the cpu decodes the instructions. The emulator runs more
slowly than native execution because many instructions are executed to
emulate a single x86 instruction. This is not necessarily a serious
problem on an alpha which is blazingly fast ...

> 2) Should I buy a computer with Linux preinstalled, or is it easy enough to
> install by my self.
>

If you're thinking of buying an alpha, you will have a harder time
finding help; therefor a linux install is a little bit more difficult than
on an x86 platform.

If you just want to use the computer and are unsure of your abilities,
buy pre-installed to save yourself a lot of time and grief.

Otherwise, do the install yourself. I don't know of any better way to
learn about linux than starting from scratch with an install.

Hope this helps. Your mileage may vary ...
