Hi all.

A week and a half ago I mentioned that I had a couple of Alphas
where certain programs like ypbind as well as programs we
compiled ourselves would display an arithmetic trap error
message. On occasion, there's also an unexpected function
code 0x33 from the alpha FP emulation module.

Someone suggested that I try patching pristine kernel sources
and compile and run a kernel from the patched sources.
I've now tried patching pristine 2.0.30 kernel sources
with the alpha-patches-2.0.30-0.4.gz that was announced here
a week ago, as well as patching pristine 2.0.32 kernel sources
with the alpha-patches-2.0.32-0.1.gz that was announced here
a week ago. In both cases, I still get the arithmetic trap
and unexpected function code error messages.

Does anyone have any more suggestions as to what to do now?
A lot of research is being held up because of this problem.
In case it matters, the Alpha we really need working is an
EB164 with I believe 32 Mb of RAM.
