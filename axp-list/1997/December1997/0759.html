Read the message "Booting Ramdisk!?" from Keith Baker. I have a similar
problem.

I have the MILO image loaded for my processor and use the AVANTI.img file
for the kernel. (The XLT.img
will not boot even though I have a "true XL as defined by Digital". When I
issue:
b fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1
at the MILO prompt, everything seems to go just fine until:

VFS: Insert root floppy disk to be loaded into ramdisk and press ENTER
RAMDISK: Couldn't find a valid ramdisk starting at 0.
VFS: Insert root floppy and press ENTER.

If I press enter at this point the system panics.

The ramdisk.img was originally loaded from the 5.0 CD set in the images
subdirectory. I subsequently downloaded the ramdisk.img from the AXP 5.0
errata. No joy.

I tried copying ramdisk.img to ramdisk.gz running gzip -d .... followed by
gzip ... and the resulting file was not = in size to the original. Also got
a 'trailing garbage ignored' message on the decompress.

The machine is:
Alpha XL 266 w/ 64MB Number 9 Video (Trio chipset) 2GB SCSI disk.
