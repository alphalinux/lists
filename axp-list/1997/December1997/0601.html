On Tue, 16 Dec 1997, Bill Broadley wrote:

> >Normally speculative execution refers to the ability to simultaneously
> >execute both the fall-through and the else path, and then to select the
> >right one (discarding the other ) when the truth value of the test is
> >known.
>
> Many many cpu's execute speculatively (P6, PII, R10k, Ultrasparc, Alpha
> 264, PA-8000, etc).
>
> Executing BOTH paths after a branch is very rare. Branch prediction is
> common, following both isn't. Much research, chip real-estate etc goes
> into branch prediction, following both is just too resource intensive.

Actually it needn't be. You obviously need 2 ALU's and 2 pipelines, but
remember that the pay-off comes from avoiding pipeline flush.
>
> Keep in mind that a branch usually comes up every 5-7 instructions,
> and more aggressive cpu's can speculatively execute up to 4 of
> them I believe. Keeping track of 16 states would be much harder
> then 4.
>
> I don't think any current superscalar risc follows both paths
> of a branch, I welcome corrections though.

I thought the 604 and 620
from IBM did, but I could be wrong.

>
> BTW I don't know of an if statement in asm language either. Just
> branch/jump in various flavors.

branch < condition> is effectively an if statment.
