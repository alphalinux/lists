I just installed RedHat 5.0 (2.0.30) onto a AS255/300 whithout too much
difficulty. I then recompiled the kernel sources that come with RH5 to
support the DEFPA fddi card. On boot I now see:

fddi0: DEFPA at I/O addr = 0x8800, IRQ = 9, Hardware addr = 00-00-F8-40-08-D8

/proc/pci shows:

Bus 0, device 12, function 0:
FDDI network controller: DEC DEFPA (rev 0).
Medium devsel. Fast back-to-back capable. IRQ 9. Master Capable. Latency=32.
Non-prefetchable 32 bit memory at 0x6000000.
I/O at 0x8800.
Non-prefetchable 32 bit memory at 0x6010000.

Everything works fine, until I try to ifconfig the card. At that
point, the machine hangs hard - power cycle required. Anyone have a
clue?

Also this machine has a TGA2 connected to a Sony 20" monitor, If I try
to startx using the TGA server, I get nothing but a flashing power
light on the Sony. Is the TGA2 card supported at all under X? The
server correctly identifies it...
