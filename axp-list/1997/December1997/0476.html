>>>>> "alsani" == alsani <alsani@acsu.buffalo.edu> writes:

alsani> Please help me in answering the following two questions: 1)
alsani> Does the statement "an application is not ported to Alpha" mean
alsani> that "an application can't be installed on Alpha"? If yes, then
alsani> can some body explain briefly how FX!32 will run such
alsani> application.

Not necessarily. FX!32 (for NT) and em86 (for linux) emulate the actual
x86 CPU. So a limited number of x86 binary applications (see the relevant
package for an incomplete list) will run on the emulator (transparently to
the user) which will (more or less) convert the instructions to the alpha
instruction set for actual execution. If source code is available, the
necessary modifications to port a x86 linux app to alpha linux are often
quite small, non-existent in the case of very clean code.

alsani> 2) Should I buy a computer with Linux preinstalled, or is it
alsani> easy enough to install by my self.

The installation process isn't too difficult if you are well versed in
dealing with BIOS and bootloader issues and have installed linux a few
times on different x86 systems already. If you are new to linux, and or
unix installation, I'd heavily recommend buying it preinstalled. I bought
my alpha with linux preinstalled by DGC, then redid everything and
reinstalled from scratch. Having the initial install there was extremely
helpful, particularly because I don't have an NT install on this machine to
assist in hardware investigation.
