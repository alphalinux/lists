> can do to it is echo cmnds to /dev/cua0...

DO NOT use /dev/cua? for anything. It is long obsolete, not really
supported, hangs there only for compatibility reasons and Ted Tso,
who put it there in the first place, regretted many times that he
provided these. I would not be much surprised if glibc would
skip these totally.

> any ideas?

Get some program which can talk both ways with /dev/ttyS0 (kermit,
minicom, put something together in perl, whatever) and try to
chat with modem to see what all of this is about. Also mgetty is
really good in logging all modem traffic - when asked for.
Internal modems are PITA.
