Thanks for all the input all. Generally seems like if your using alot
of disk i/o your better off with scsi (much more so then with an x86 box).

People report good overall performance with EIDE mainly because Linux
does a decent job of using memory as a disk cache, when theres enough
ram/not disk intensive jobs.

Apparently the LX motherboard was either designed with a poor eide chip,
or doesn't take full advantage. Doesn't seem fixable.

Thanks for all the input.

Anyone know if the SX is similiarly low performing in the area of EIDE?
Seems like EIDE would be more important on the more price sensitive
SX configurations. Any ideas, experiences etc?
