Hello:

We have just obtained an Alpha system running RedHat 4.2 and
everything has configured hunky-doory just like the Intel systems.
The only problem (and I have seen this repeated on the mail archives)
is that nis is not working. What happens is the ypbind runs just fine
for a while, then it starts flooding the yp server with packets which
effectivly ties up our network. I have configured everything the same
as the Intel system (which uses ypbind, not the kernel NYS) and I have
even tried the fixes mentioned on the mailing list:

o upgrade the ypbind package to 3.3
o edit the /etc/nsswitch.conf file
o make sure the /etc/yp.conf file has "domain" and "ypserver" entries
o make sure the domain name is set before ypbind is started

Can anyone recoomend an fix for this? If anyone is running RedHat
5.0/Alpha does NIS work out of the box? Thanks for your time.
