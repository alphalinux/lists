On Tue, Dec 16, 1997 at 08:42:30AM -0500, Mike Howard wrote:
> "Unable to handle kernel paging request at virtual address %016lx\n"
>
> From which I deduce you have a bad spot on your disk. I would try:
> A. a new paging disk

No, this is no hardware problem. A bug in the kernel tried to
access memory that does not exist, and faulted.

What to do with this is to get hold of all the juicy numbers
describing the state of the processor at the time. Then look in
the System.map file that was built with the kernel you are now
running, and for each [<...>] number, find the map entry just
before that.

With a report like this, it becomes possible for someone to look
into the problem and perhaps find an answer.

An additional note for 2.0.x kernels -- there are quite a few
patch sets floating about, and it will be important to know
what flavour you are using at the time.
