David Alan Gilbert wrote:
>
> Hi,
> Has anyone got xf86quake to run under RH5.0 ?
> the standard xquake runs but the xf86quake complains a bit and then
> quits with the screen partially scrolled up.
> I have a Matrox Millenium 2 card which probably doesn't help :-)

There is an assumption made in quake for memory allocation wich
doesn't hold for glibc.

Read the following quoted from a former posting (maybe in axp-list):

-----------------------------------------------------------------------

Quake broke with glibc 1.99. If you're running RedHat 4.1 or higher you
need
to get a copy of glibc 1.98 to use with Quake. I have a copy at:

ftp://yoyodyne.lawbulletin.com/pub/libc-1.98.so

Here's the command I use to run Quake:

LD_PRELOAD=./libc.so.6 ./xquake

Or, to run as a dedicated server in the background:

LD_PRELOAD=./libc.so.6 nohup ./xquake -dedicated 16 -nosound &

libc.so.6 is just a symlink to libc-1.98.so in my Quake directory.
Good luck!

-jrh

------------------------------------------------------------------------

Concerning speed:
quake on intel boxes sucks it's speed out of assembler core functions.
For the alpha everything is plan C.
