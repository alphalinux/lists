On Tue, 16 Dec 1997, Andrew J. Scott wrote:

> With 64 meg _100_ meg swap, the udb will run into memory problems trying
> make -j vmlinux!! On an intel box you can make -j to compile the kernel
> with 32 megs ram and 32 meg swap, with little trouble.

Yes, RISC binaries are larger, they take longer, AND they take ALOT more
memory to compile. While making the *entire* at once, in parallel may
be amusing on a single CPU machine (and very useful on a SMP box), WHY,
are you doing it on a UDB?

The entire larger binary thing, + the harder to optimize thing, +
the gcc is not tweaked out for compiling on the alphas, geez.

Compiling bigger binaries (~ 30%), with a compiler that is itself larger
(~ 30% again), doing a parallel compile, with compile options that make
for faster binaries but use alot more memory and time optimizing things...
And doing it on one of the slowest alphas available. :)
