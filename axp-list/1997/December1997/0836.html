Michael W. Chalkley wrote:

> Trying to remote print from Solaris 2.5.1 to a RedHat Linux Alpha 4.2
> box running lpd. It doesn't seem to work. Anybody have remote printing
> through lpd working?

If your alpha lpd dies constantly with a memory violation (or something
like that), then you've got the same problem as I did.

Some time ago, someone in this list (could't find the original mail,
sorry...) suggested to re-compile the lpr package and add

-Dgetline=get_line

to the definition of CFLAGS in the Makefile. This helped for me.

If you don't want to go through all that SRPM hassle, I can send you my
lpd binary.

If this doesn't help, please provide us with some more information (like,
does printing work locally? via cat >/dev/lp0 and/or via lpr? are you
sure that the Solaris box is configured correctly? etc? :-) ).
