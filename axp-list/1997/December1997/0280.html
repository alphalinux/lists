> A new set of alpha-patches based on 2.0.30, which includes SX164 support,
> and preliminary support (untested and perhaps not working) for the
> Deskstation RUFFIAN (aka RPX164-2, but soon to be aka AlphaPC164-UX or BX).

We tried this patched 2.0.30 on our Ruffian/RPX164-2 (RedHat 5.0):
compilation was no problem, but booting using the milo.ful from
Deskstation gave a flood of error message

init: memory violation at pc=15555832010 rp=1555555cfd4 (bad address=00006c71)

> A new set of MILO sources based on 2.0.30, which includes some new EBSDK
> PALcode for certain platforms (EB164/PC164/LX164/SX164). There are no binaries
> available at this time, though the ones on gatekeeper in Test-Milos-2.0.30+
> are very close to what you can build with these sources. I will make a more
> comprehensive release of MILO at a later time, with proper binaries and updates
> to documentation for things that changed (very minor).

We then tried the MILO source together with osfpal.nh and ldmilo.exe
from Deskstation: it just hangs when trying to load milo.ful.

Should we use the original linload.exe and/or different PALcode?

Thanks for the effort though!
