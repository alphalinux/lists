Some time ago, there was a discussion on this list about getting
Xemacs working and a number of patches were written by various
people. The upshot being that Xemacs 19.16 was persuaded to work on
the alpha. (I remember compiling and running it successfully.)

I've recently downloaded the new Xemacs 20.3 release and have been
looking through the source. It seems that some of the patches made it
to the Xemacs team and have been incorporated into the main
source. However, the darned thing still doesn't build properly! There
are lots of warnings about UNEXEC being defined twice (which is a good
sign something is wrong) and then it stops in the inet code
complaining about conflicting types for ntohl. Whilst I'm sure I could
hack at this, I'd rather consult the people who worked on xemacs
before, to find out what needs doing. (I'm pretty sure the problems
are identical with those that happened before, although the old
patches clearly won't work with the newer sources.)

Has anyone yet put together a set of patches for the 20.x Xemacs
sources? (I'm aware that it is a seperate source tree from the 19.x
Xemacs. I know it's definitely possible to get it going, because
people sometimes release compiled alpha/linux binaries. For some
reason these always like to have shared files in strange places, and
so on, though, and it would be nice for people to be able to build
their own version.

I'd quite like to build a proper Redhat xemacs package, as we've
waited long enough for it on the alpha! But without knowing how I need
to mutilate it to make sure it works *properly* (I can't test
everything!) this would be an unwise thing to do.

Can anyone help?

Best wishes,
