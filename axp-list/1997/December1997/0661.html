First, thanks to all of you who have replied to my pleas for help on
the "Unable to handle kernel paging request" issue. I've seen a
number of different suggestions posted, and I'll try to respond to all
of the points made there. If I've missed something, or if you think
some other bit of info would be helpful, please let me know.

Thanks again for all of your help...

1) Possibly a problem with the swap partion.

I don't _think_ it's a problem with the swap partition. I tried
running 'badblocks -w' on it and got none back (no suprise -- it's a
SCSI device, and I think they hide those from you, yes?) and
re-mkswap-ing it with the -c option. No help there. I also threw
together a small program to hammer memory+swap moderately hard and
experienced no problems. Besides -- the problem has occured when the
system is practically unloaded. This is a machine with 96Mb of
memory, and as I don't often run X (monitor problems -- unrelated), it
usually isn't terribly loaded. Though it does occur more often when
doing things like kernel compiles.

2) Try 2.0.32 or higher.

Haven't yet, will do. I'll let you know my experiences when I do.
Though because this problem is somewhat sporadic and I have yet to
figure out a guaranteed way of replicating it, it'll be a bit hard to
determine whether it's fixed immediately.

3) Tweak the kernel freepages and swap parameters via /proc

I'm not familiar with these and they aren't documented in my proc man
page. Is there documentation outside the kernel sources, or do I have
to delve into that? (Please understand, while I've hacked my share of
low-level code, these days I do AI and prefer to stay as far from the
bare metal as I possibly can. User-level docs preferred.)

4) Find the running function from the System.map file

<sigh> Sorry. I don't happen to have a map for any of my kernels
laying around at the moment. The last one I had got trashed when I
reconfigured my kernel to use the BSD NCR driver in the vain hopes
that _that_ would cure my problem. (It might have, except that it
introduced a _new_ set of problems... <sigh>) When I build the .32
kernel, I'll make sure to save one.

5) In general, provide more info on your system configuration.

Glad to.
