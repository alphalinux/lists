On Fri, 26 Dec 1997, Mike Howard wrote:

> I'm confused [not unusual, unfortuantely]. I'm attempting attempting
> build a kernel for an Alphastation 255 and am having trouble with the
> SCSI host adaptor.
>
> The stock kernel in images/avanti-s.gz boots and works fine -
> i.e. finds the ncr53c810 chip, talks to it, and comes up fine.
>
> Every kernel I build finds the chip, but goes into an infinite loop of
> reset-the-bus:timeout:reset-the-bus:...
>
> The &*(*^%&()&(*()* of it is that I had a configuration which built a
> kernel which worked fine. Fool that I am I didn't back it up and when
> mkswap ate the disk ...
>
> Anyone have any clues? [I've check the alpha-linux faq, redhat axp
> knowledge base, the last couple of months of axp-list archives. I've
> also checked in the SCSI-HOWTO, but - how much of this applies to
> Alpha's?]

This isn't necessarily an AXP only issue. I had the same problem on an
intel/RH-linux box, and I was stumped for quite awhile. The default
kernels load almost everything, including the scsi driver as modules. When
you make your new kernel you have to make sure you rm -rf the old modules
directory. This was what finally worked for me, it seems like a
no-brainer, but there you go. =)
