Hi,
OK - I've finally got PPP and mail going on my LX; here are some
soloutions and some problems I hope people can help me with:

1) PPP - if your PPP is stuck upgrade your kernel! I built a 2.1.63
kernel from standard unpatched 2.1.63 sources and it works fine.
(Its doing this!!). My old PPP scripts worked with very minor
tweaking using the RH5.0 distribution pppd/chat.

If you have an LX tell the 2.1.63 config you have a PC164 and its
fine.

2) Compilation giving multiply defined FD_SET and the like. Upgrade
upgrade your kernel - problems disappeared with 2.1.63; glibc doesn't
like 2.0.30??

3) C++ programs crash with seg faults in system functions (typically
fclose). This problem seems to be caused by libio being in both
libstdc++ and glibc - you get conflicting symbol errors if you try
-static - has anyone got a fix for this????

4) Has anyone got a webbrowser binary for Linux/Alpha? Other than Lynx!
I'm trying to build the KDE stuff, but due to (3) and the fact that
the old version of EGCS I have crashes if I try that instead I'm
failing.

5) Soundcard - I have a soundblaster 16 value; true creative labs -
works fine on Linux/x86 - how do I get it to work on Alpha? I tried
building the sound drivers (in 2.0.30) in the same way as for intel
and one of my EIDE channels stopped working - presumably due to
an interrupt screw up??? How to ISA interrupts map to the interrupts
and addresses seen by the kernel??

6) Beep - I haven't heard my machine beep from its motherboard speaker
at all - I presume this is a fault in the hardware (for which I'm NOT
sending it back!). Can anyone suggest any other problem which could
cause silence.

7) Matrox Millenium II - get corruption on your xterms? Turn off
acceleration using the 'Option "noaccel"' line in the Section
"Device" part.

8) Any one had any look with EGCS and -mCPU=21164? I get assembler
errors (I'm on EGCS 971127).

9) If you add an IDE device after installing onto SCSI the AlphaBIOS
moves the device numbers and you'll have to change the OS loader
settings. Even when adding just a CDROM (which doesn't constitute
another disc) this happens.

10) When building the 2.1.x kernels note that the VGA console and TGA
console options have been merged into one 'TGA instead' option.
So where as on 2.0.30 you could define both, if on 2.1.x you define
the TGA you get no kernel output displayed.

11) On 2.1.63 the Adaptec 2940 drivers have a small piece of x86
assembler in - but the routine isn't called anywhere; comment the code out
and your fine.

12) Remember to comment out the SMP =1 line in the Makefile. Note:
setting SMP=0 screws up!

13) 2.1.63 feels a lot more responsive and memory careful - and you get
the option of setting the SCSI bus pause period after reset :-)

Finally:
* NOTE: I have had mail problems over the last few days - things may *
* have bounced *

I'm not sure about this - but after 1.5 days of trying to get PPP going
I only had 6 messages - most of which disappeared! I am using fetchmail
to get mail via PPP but I forgot that my service provider will deliver
mail via SMTP automatically - and sendmail was running but badly
configured.
I haven't seen any errors, messages or anything which would indicate
lost mail - but neither have I seen the mail; I'm guessing there were
bounces with user unknown errors but I do not know. If those came
to you - sorry!!
