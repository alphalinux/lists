> I have successfully compiled a linux kernel 2.1.72. But up to now, it
> was not possible to me, to boot the kernel (neither milo, aboot nor

I'm betting that you'd built 2.0.xx kernels, either as installed by Red Hat
distributions, or using alpha-patches on top of pristine sources?

If so, then be aware that only the patched 2.0.xx kernels allow *both*
TGA and VGA to be present in a kernel; the pristine 2.0.xx and 2.1.xx
kernel sources ask you "Use TGA console" (or some such), and default to Y.
If you take the default (or say Y), you will get a kernel that has ONLY
TGA support, and, if your system has only a VGA, you won't have the
proper text-mode console support built in; because, TGA != VGA.

If this is not the case, then send me along a bit of hardware description
(cat /proc/pci) along with the ".config" file from the kernel source tree
that you built in, and I'll have a look...
