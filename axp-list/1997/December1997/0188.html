I decided to break down and get a nice fast external SCSI disk for my trusty
UDB. I then proceeded to install RH5.0 from scratch. All went well until it
went to reboot. The reboot got hung at the sendmail init for about 10
minutes. Then again at the httpd init. Then again at the smbd init. After
about an hour, it finished booting. Thinking the problem might have been the
(bug that was fixed in a previous release) /etc/nsswitch.conf file, I copied
that to the new /etc and rebooted. No joy.
All the wondermous new system utils were worthless because X windows
wouldn't come up. I'd get the pattern background and X cursor but nothing
else. X would just beep at a mouse click. Had to bail using ctl-alt-bs.
Tried xdm, it's REALLY cool looking now. Didn't help, though.

A little more digging, however, turned up that the /etc/hosts file only had
one line in it, the localhost line. The 5.0 install script had failed to put
the machine's own host name in the /etc/hosts! Adding this made the boot
come up like magic. Sendmail, httpd, smbd, and X evidently really wanted to
know what the hostname is of the machine they're being run on. Not sure if
the 'fix' I used is the right one, but it did the trick.

Most folks prolly don't see the problem since they're connected to the
Internet directly or have a nameserver running on their local net and thus
can resolve their own hostname. Since I use my machine AS the
gateway/nameserver to dial-up, it's stand-alone. Therefore, there was no way
to resolve its own host name since the RH install script didn't file it...

If I find any more minor boo-boo's that cause major heartaches, I'll let
folks know...
