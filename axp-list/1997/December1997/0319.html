Richard Henderson <rth@dot.cygnus.com> writes:

>
> On Tue, Dec 09, 1997 at 11:40:06AM +0200, Harvey J. Stein wrote:
> > It seems that linux/alpha's default behavior is to signal a SIGFPE on
> > divide by zero, whereas linux/intel's default behavior is to work with
> > NANs, INFs & MINFs.
>
> Yep, a mistake in RH4.2.
>
> > What's the appropriate way to switch between these two behaviors?
>
> ieee_set_fp_control(0);
>
> gets you to no exceptions signaled. See <asm/fpu.h> for what to
> put otherwise.

This isn't working for me:

hjstein@blaster:~/software/xlispstat-3-50$ cat test-zero.c
main()
{
double num = 1;
double denom = 0;

ieee_set_fp_control(0);
return( (int) num/denom);
}
hjstein@blaster:~/software/xlispstat-3-50$ gcc test-zero.c -o test-zero
hjstein@blaster:~/software/xlispstat-3-50$ ./test-zero
Floating point exception (core dumped)

Could it be a RedHat 4.2 problem, or a problem with one of the patches
that's been circulating?
