With regards to your message at 05:35 PM 12-08-97 +0100, Ronny Ranerup.
Where you stated:
>
>me:
>> What's the experience on tape drives on Alphas? The SCSI howto
>> says that all SCSI-compatible tape drives should work. Is this true?
>
>sfo@sanetwork.on.ca:
>>I use the:
>>Host: scsi0 Channel: 00 Id: 03 Lun: 00
>>Vendor: ARCHIVE Model: Python 28849-XXX Rev: 5.AC
>
>Funny, when I checked Altavista for Python 28849 I found a SGI 6.2
>page where it says 'The assumption is that the SCSI commands,
>capabilities, and request sense info are constant within a type. This
>is rarely completely true, but is often close enough to allow a drive
>to work.', and then there is a long list of the different 'types' of
>drives which apparently work differently.
>
>Then I checked the Linux Hardware-HOWTO where it says 'Drives using
>both fixed and variable length blocks smaller than the driver buffer
>length (set to 32k in the distribution sources) are
>supported. Virtually all drives should work.'
>
>So as usual it seems you have to hope that someone else has tried
>exactly the hardware you're interested, otherwise your gambling when
>you buy it.
>
>So I repeat my question, is anyone using a new high capacity
>(10-20gig uncompressed) tape-drive on Alpha/Linux?
IF you are using Digital Linear Tape (DLT) from Quantum, Digital, or
Seagate, I can say "YES".
I have used these with no difficulties. And they are VERY fast.
