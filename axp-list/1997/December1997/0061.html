>>> nm said:
>
> the only disappointment i have with milo is that (to my knowledge)
> it cannot boot a piece of 'raw media' like the dec firmware cd
> or just a plain old srm boot floppy...

That's only because YOU haven't written it yet! :-)

> now, anyone have an idea how much work would be involved in modifying
> milo to do this?

It may be more work than you realize; SRM console provides more than just
PALcode for the OS, it provides some "callback" routines which the OS booter
can use to do things like get/set environment variables, read/write blocks
on the boot device (disk, floppy, tape, Ethernet, whatever). MILO doesn't
have any of that stuff (now).

> anyone know who is really incharge of milo devel?

Well, up to this point, there's been none other than us in DEC that have
been interested enough... :-\

Seriously, though, certain pieces of MILO could only have come from DEC
(the PALcode sets and the proprietary BIOS emulator); the rest, however,
could have been done by anyone with access to sufficient documentation
and the appropriate hardware. Everything EXCEPT the DEC BIOS emulator is
available as source in the MILO kits; that includes all the PALcode.

Other folk from outside DEC have contributed bits and pieces in the past;
most recently I've been working on integrating the required patches to
support the Deskstation RUFFIAN (aka RPX164-2) motherboards, initially done
by someone there (Dean Hall?).

> is this something that can be done? and can be done legally?

Absolutely. Sources are currently available in the Miniloader/ directory
tree on gatekeeper, though they are a bit out of date at the moment, as
new platforms and updated PALcode sets have recently been integrated into
the working sources. I expect to have a new test kit available within the
next few days (stay tuned to axp-list for any announcements), though it will
(initially) only be the sources (I don't have Dave Rusling's scripts yet
for the full deal :-).

But be warned; the current MILO sources build best against a 2.0.30 kernel
modified with the latest alpha-patches, and I've not attempted to build
either kernels or MILOs under RH 5.0 with updated GCC, GLIBC, etc.
