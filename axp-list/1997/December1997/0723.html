Hi!

I've just been sitting hacking for a couple of hours in order to get my
TextMode improved on my Miata. I've got it now working as far as standard
VGA modes (80x25x8 upto 132x60x9) are concerned, using the Chipset "VGA"
entry in /etc/TextConfig.

However, if I tell SVGATextMode that I have a Matrox Millennium, it
doesn't believe me and tells me it "Could not get Millennium PCI info".
Digging and hacking in the source code revealed that pcibusSetup() inside
SVGATextMode-1.6-src/XFREE/common_hw/xf86_PCI.c don't even find find any
PCI bus.

Opening the case of the computer and pointing at the PCI slots shouting
"see, I *do* have a PCI bus" didn't impress it, strangely enough :-)

Before I dig through all the XFree86 source code and look how *this* beast
finds the PCI bus and try to backport this somehow into the SVGATextMode
sources, I'd like to ask if anyone has done things like this before -- I
think it should be quite trivial, even though it might get longish :)

So: Anyone staring at some flashy 160x100 text mode on his Alpha box using
a Matrox card right now who wants to share the secret how he's done it?

If not, I'd see if I can squeeze out some more spare time over the
christmas holidays and either burn down my monitor or get it working.
