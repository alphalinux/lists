On Wed, 3 Dec 1997, Andrew Smith wrote:

> yep me too :( I have made it through the config for sound but it always
> bombs on make boot. has any one had any luck with this on a udb? i didn't
> persue it because i have a mac on my network for doing sound work but it
> would be nice to have the option.

i have compiled a 2.0.30 kernel with sound support for the built-in
hardware of a UDB. the driver however only properly deals with output.
sampling/recording is broken.
here is the relevant section of /usr/src/linux/.config:

#
# Sound
#
CONFIG_SOUND=y
# CONFIG_PAS is not set
# CONFIG_SB is not set
# CONFIG_ADLIB is not set
# CONFIG_GUS is not set
# CONFIG_MPU401 is not set
# CONFIG_UART6850 is not set
# CONFIG_PSS is not set
# CONFIG_GUSMAX is not set
CONFIG_MSS=y
# CONFIG_SSCAPE is not set
# CONFIG_TRIX is not set
# CONFIG_MAD16 is not set
# CONFIG_CS4232 is not set
# CONFIG_MAUI is not set
CONFIG_AUDIO=y
# CONFIG_MIDI is not set
# CONFIG_YM3812 is not set
MSS_BASE=530
MSS_IRQ=9
MSS_DMA=3
DSP_BUFFSIZE=65536
# CONFIG_LOWLEVEL_SOUND is not set

hope this helps.
;)

i also recall a problem after applying some set of patches, a seemingly
incomplete addition of a second DMA channel or something, which caused
compilation to stop with an error. to fix that i simply undid the patch to
that file by hand. grep around for something like MSS_DMA_2 in
approperiate areas of the src, or in the diffs you have applied.
