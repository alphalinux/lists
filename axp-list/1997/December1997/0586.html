Hello:

We just upgraded to RH 5.0 from 4.2 in order to use NIS on our
system. Everything has gon fine except that we ran in to the
following problems:

(1) We have a small C++ program which hangs or crashes consistently the
first time it calls fclose().

(2) Intermittently, when doing a lot of disk I/O (twice during a cvs
checkout, once during a tar xvf, other times randomly) to a disk that
was local but accessed through NFS, the disk access would hang,
requiring a reboot.

(3) After a disk hang (cf. (2)), we sometimes noticed that the clock was
crazy, showing a date that was off in either direction by decades. I
can't tell if there was any consistency in how far off the date was, or
whether it was always off when the disk hang occurred.

Problem (1) is the killer right now. Problems (2) and (3) will have to
be solved before Linux 5.0 is useful to us.

I have read through a lot of docs and to no avail so far. Does
anyone have any fixes for this stuff? Thanks.

-danimal
