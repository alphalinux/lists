Hi !

> Trying to remote print from Solaris 2.5.1 to a RedHat Linux Alpha 4.2
> box running lpd. It doesn't seem to work. Anybody have remote printing
> through lpd working?

Hmm. Maybe you should replace your bsd-lpr environment with LPRng.
ftp://ftp.astart.com/pub/LPRng

I don't know if it even works on Alphas. It simply solves all my network
printing problems with Linux/i386.

But there could be other problems also. ("It doesn't seem to work" is
not very productive)

I'm not sure but old Solaris versions use ATT and not a BSD printing model.
The following may be outdated...
In case of ATT, you must tell "lpsystem" on Solaris via "-t bsd"
that the remote system is to be treaded as BSD system.
Try something like:
lpsystem -t bsd linuxhost
lpadmin -p rpr -s linuxhost!lp -I and -T unknown
accept rpr
lpstat -t

But please forget bsd-lpd for remote printing under Linux.
(this warning excludes samba - samba prints local (relative to lpd).)
