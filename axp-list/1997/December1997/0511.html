Installed 5.0 from scratch on a DEC UDB. Got most of what I need going. Set
up the ppp configuration stuff the same way as my (still working on a
different partition) 4.2 ppp setup. Did diffs on actual files to verify
settings. The scripts crank up, modem dials, handshakes, logs in, and gets
IP addresses. The I get a message like:

ioctl(SIOCADDRT): device route: Network is unreachable

in /var/log/messages. The modem immediately hangs up and the whole thing
tries again (until I stop it like I started it - via control panel). The
routing configuration is identical to the working 4.2 config - no default
gateway IP and gateway device = ppp0, static-routes having only the eth0
interface masking only the local lan, ipfwadm from local to remote (although
whether on or off it still dies). The click box in control panel for IPv4
forwarding can be off or on, the behaviour is the same. Kernel is supplied
RH5 recompiled to make sure all the ppp options and ip-masq stuff is on.

I tried doing things like replacing the 5.0 pppd binary with the 4.2 pppd
binary and it wouldn't execute (I made sure the permissions were correct). I
tried replacing the ppp related scripts with their 4.2 versions. Still no
joy. Also downloaded source RPM and did a rpm --recompile. Still didn't fix
the problem.

Is there something new about 5.0 that requires a change to the config files
or something? And/or is there an update to pppd in work that will correct
this problem?

Thanks for the help.
