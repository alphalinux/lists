I have an NCR53c875 SCSI controller in my Aspen/Durango II based machine.
This machine is configured for DEC unix (not installed yet) so I am using
the SRM console. When I try to boot a kernel (2.0.30), from either floppy
or CD rom (RH5.0) the thing hangs after detecting the controller. It
detects a weird interrupt and other settings:

ncr53c875-0: rev=0x01, base=0x19001000, io_port=0x9000, irq=18

when I give as boot flags
max_scsi_luns=0
it gets a little further, but then gives me a kernel trap...

Is there a newer kernel out there with better support for this SCSI
controller?
