>>> Tad Grosvenor said:
> I have installed redhat-4.2 on my udb, and have been happy with it so far.
> It works with my fixed frequency sony monitor (gdm-1601) except for one
> thing: the image hangs off of the edge (right side) of the screen by about
> 10-15 pixels (1/8-1/4 of the afterstep icons). This is not a problem, but
> is really really annoying.
>
> I tried to start xvidtune, but all I get is an error stating that my
> Xserver is running an old Xfree86-VidModeExtension (version 0.4 as opposed
> to the 0.5 that is required) I can't find this anywhere. I am running
> the TGA server that is on ftp.redhat.com in
> /pub/redhat/redhat-4.2/alpha/X. Any ideas?

"xvidtune" will not work with the DEC TGA; it assumes VGA-style video
registers, which the TGA does NOT have.

If you feel adventurous (and don't mind taking a chance on frying the monitor),
you could try modifying some of the clock-programming values on the Modeline
you're using, as the latest XFree86 TGA server does use those to program the
clock on the TGA (AFAIK).

My suggestion? Move the AfterStep icons to the left side... ;-}
