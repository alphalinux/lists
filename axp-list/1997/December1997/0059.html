>>> nm said:
>
> hmm... the ability to upgrade firmware to arc or srm via booting from a floppy
> or cdrom would be really nice... but this would mean that milo would have to
> install some routienes that code from the booting media would call?

Well, the firmware *is* currently upgradable from floppy or CDROM, under
either ARC/AlphaBIOS or SRM, AFAIK. I'm speaking of ARC/AlphaBIOS or SRM
here, of course.

Also, there was a utility (FMU?) which Dave wrote that could be run from
MILO (IIRC) which would update the MILO firmware in the FLASH. But it was
not available on all the platforms, since many have their own peculiar ways
of updating the FLASH memory.

What I was speaking of, WRT the "callback" routines, is the way that DU is
booted. The primary bootstrap is quite small (a sector or two?), and all it
really knows how to do is read in the secondary bootstrap from the same
device. It and the secondary bootstrap (which understands about UFS
filesystems and kernel loading) do the image reading using callbacks into
the SRM console for the driver code necessary to access the appropriate
device, be it disk, floppy, tape, CD, or Ethernet. These are what you'd need
to provide, in MILO, in order to be able to boot DU, or the firmware update
program, the same way they are now.
