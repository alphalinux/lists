> > how to capture the output from the make just to locate the first errors.

> The best way to run such things is from within emacs. The reason is
> that if you will get any errors/warnings then C-x ` (i.e. next-error)
> will walk you through all this spots without a need for a manual search.
> A command is "M-x compile" and you can define your own key for that.
>
> Another possibility is to use 'script' command. 'man script' for
> details. You will have to hunt for errors yourself.
>
> Last, but not least, you can redirect an ouutput to a file or a pipe but
> you need to capture both stderr and stdout. As you still likely want
> to see what is going on you should use 'tee', so things are getting a
> bit involved. That is a reason for the first two suggestions.
>
> How to redirect is described in docs for a shell you happen to use.
> 'make 2>&1 >capture' or 'make 2>&1 | less' will work with Bourne
> derived shells (like bash).
