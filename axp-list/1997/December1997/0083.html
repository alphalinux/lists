>
> I'm not sure - has anyone had any experience with 2.0.32 here?

I am running for quite a while on Alpha 2.0.31pre10 - patched -
which is actually pretty close. :-) 'pre10' instead of 2.0.31
release to avoid mentioned here problems with a filesystem.
As for now I did not have any troubles with this kernel but this
machine did not played a role of a heavily loaded news server either,
for example. I did not hear any complaints from an owner of
another Alpha machine running the same kernel vintage but with
a different SCSI controller (Adaptec, a driver with Doug Ledford
"abort/reset" modifications).

> 2.0.32
> came pretty quickly after 2.0.31, and consists mainly of bug fixes to
> solve the fs problems in 2.0.31, and a few other instability issues.

I patched 2.0.31 using 2.0.30 assorted Alpha patches (more or less
like in a Harvey Stein writeup) and resolving few ensuing conflicts
"by hand". I would expect a similar deal for 2.0.32 but I did not
try it myself yet.
