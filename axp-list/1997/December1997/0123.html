>
> The Alpha box set up as SMB server used SAMBA as SMB server software. As
> far as I can remember it compiled cleanly.=20

Samba server does not need any changes in a kernel if TCP/IP networking
is supported. It will work happily with kernels of 1.2.x vintage
(I run it on Intel boxes).

> In order to import SMB shares on your Linux box you have to compile
> support for SMBFS into your kernel

This is an independent, although related, thing. This goes another
way around and indeed, as a new file system, needs a kernel support.
In 1.2.x days this was a separate "outside" patch. Later it got
integrated. Instead of "posting" shares you may mount shares posted
by somebody else as a part of a file system of your Linux box.

> "smbount" is part of the SAMBA package, btw.=20

Well, no, not in a general Samba distribution (which is not Linux only).
Various other variants of Unix use "Rumba" which does something similar
to Linux smbfs. "Rumba" is not a part of "Samba" server either.

> So if nothing has been broken in the kernel since then SMB should work
> quite fine on Linux/Alpha. ;-)

Agreed. :-)
