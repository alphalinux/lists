>
>...therefore errors are not strange at all.

Well, they seemed so to me at the time :-D.

>> [rzook@babylon5 cgi-bin]$ cc -o rsvp rsvp.c
>> rsvp.c: In function `main':
>> rsvp.c:20: warning: assignment makes pointer from integer without a cast
>> rsvp.c:25: warning: assignment makes pointer from integer without a cast
>
>strtok() is not declared, so compiler assumes a default 'int' as a type
>of a return value. You try to assign this to a pointer. This is a bug
>in your code.

But why didn't I get some kind of link error? Why did it produce an
executible at all if strtok was not declared or defined in my code?

>> /tmp/cca197261.o: In function `main':
>> rsvp.c(.text+0x8c): the `gets' function is dangerous and should not be
used.
>
>Indeed! Anybody feeding your program a line longer than your buffer
>will crash it or worse (manipulating your stack one can make your
>program to execute any code at all). Big fun in a CGI context when
>you apparently are heading.

I know, gets() can produce dangerours results, and I plan to add a
dynamic allocation for postline based on the results of CONTENT_LENGTH,
but not while I'm testing.

Besides, just using a pointer in c can cause dangerous results, so

personal problem to me.
