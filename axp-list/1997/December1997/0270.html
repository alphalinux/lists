Eric Hilton Jones <ehjones@whale.st.usm.edu> writes:
> Has anyone with a UDB installed 5.0 yet, and does it appear to be
> stable? I'm deliberating over FTPing and installing it or not.

I don't have an UDB, (Jensen instead) but there are some general
problems I noticed. E.g. any C++ program using X output dumps
core. Thus, if you need packages programmed in C++ (e.g. KDE), you
should wait until this is fixed.

Some of the admin tools don't work properly, but the didn't in RH-4.2
either (e.g. Glint).

Upgrading with the install disk didn't work (the install program
catches a signal while updating the base package). I had to upgrade
manually. I later installed a fresh RH-5.0 on a spare partition to see
it makes a difference, but I got the same problems as with my updated
system.

The really nice new thing for Jensen owners is, that X works out of
the box :-). 'Clock' is still brocken(*), though :-(

Martin

(*) Install the SRPM package, #define USE_DEV_PORT, and change
'#include <asm/fcntl>' to '#include <sys/fcntl> to get it working.
