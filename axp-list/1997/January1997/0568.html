Hi all,

I just received a PC164 based machine and am at a deadend
with respect to installing Linux on it. I am able to get
loadlin running from the WinNT ARC firmware using the
instructions in the Miniloader HOWTO but MILO stops dead in
its tracks after printing the "Swapping to PALcode at 0x0..."
message. The machine just sits there -- I need to hit the
hardware reset switch to get anything to happen.

Here are more specifics:

All the firmware messages below will be transcribed by
hand since I have no way of capturing them to a file...

I have no mouse attached since all I have right now is a
serial mouse and I read in the mailing list archives that
they can cause troubles on these boards. When I did have
the serial mouse attached the machine did detect it.

I have a Fujitsu harddrive and Creative Labs CD-ROM attached
to the onboard primary IDE controller and a 1.44mb floppy on
the onboard floppy controller.

I have tried with two Diamond S3 based PCI video cards: a 2meg
DRAM one and a 4meg VRAM one.

Booting with a an NCR 810 board and SCSI drive installed
resulted in the machine recognizing the board but still
hanging after swapping to PALcode.

On powerup the firmware first reports:
"Initializing drivers: ATAPI...OK."

Then I get the boot manager or the setup menu depending upon
what I had been playing with previously.

The Hardware Config shows:

Processor ID 21164
Processor Revision Unknown
System Revision 0x1
Processor Speed 433.08MHz
Physical Memory 256MB
Backup Cache Size 1MB

Extended Firmware Information:

Version 4.48 960603.1113
NVRAM Environmental usage: 16%
(468 of 2894)

Video Option detected:
BIOS controlled video card

Devices detected:
multi(0)video(0)monitor(0)
multi(0)key(0)keyboard(0)
multi(0)disk(0)fdisk(0) (Removable)
multi(0)serial(0)
multi(0)serial(1)
scsi(0)disk(0)rdisk(0) (RAW) Fujitsu M1638 TAU
scsi(0)cdrom(1)fdisk(0) (Removable) CreativeCD....

It's strange that it's reporting the IDE devices as being on the
SCSI controller??? Can I fix this somehow??? Do I need to???

PCI Slot Information:

Number Number Number ID ID ID vector type
------ ----- ------- ------ ------ -------- -------- ------
0 5 0 5333 88d0 0 0 old unknown type

Finally, during the boot process here is what happens:

Opening file multi(0)disk(0)fdisk(0)\milo...

Loading file at 0x80000000...

Getting firmware parameters...

...Memory size is 0x10000000 bytes

Setting up arguments for Milo @ 0x801FFC00
Setting up memory size for Milo @ 0x801FFE00

Swapping to PALcode at 0x0

and then it freezes. I've tried Milo 2.0.25 and 2.0.27 for PC164
from gatekeeper as well as the Milo from the Miniloader/archives/PC164
directory on gatekeeper and Milo-2.0.8-EB164-ide from the kernels
directory all with the same result.

I'm very new to Alpha hardware so any and all help is appreciated.
Hopefully, I'm missing something obvious and simple!!!

Thanks for reading all this and hopefully for secret that will
get my machine up and running!!!
