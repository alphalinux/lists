On Sun, 26 Jan 1997 14:47:37 -0900, Michael Tibor
<tibor@lib.uaa.alaska.edu> said:

> I'm running into something now that doesn't seem to make much sense.
> When I try to boot the kernel off sda2 my hard disk at the Milo prompt,
> I get the following error message:

> ext2_mount: too many groups (307) max is 256

> or something similar anyway. I was thinking it might mean that my root
> partition is too big (about 2.5 gigs), but that doesn't seem to make
> sense to me.

There's definitely no such limit in ext2fs itself, but MILO contains a
stripped down, readonly version of ext2fs which doesn't cater for as
many block groups per partition. So I'm afraid that yes, you are
stuck with 2GB as the largest size for the partition which contains
your kernel.

I always set up my machines with a small (typically 2MB) partition
reserved for /boot, and I throw all the bootable images into that. It
generally saves a lot of grief.

Cheers,
Stephen.
