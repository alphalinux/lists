>>> Topi Kanerva said:
>
> > > Why is the binary 3.1.2 Xserver faster than the 3.2 on the TGA? (DEC
> > > cc compiled, maybe?)
> >
> > Nope; the 3.2 version uses dumb framebuffer ("cfb") code, the 3.1.2 version
> > uses highly optimized (and proprietary) code to make it scream.
> >
>
> Well, is the 3.1.2 version still compiled with Digital cc or is it just gcc?
> It should make some difference to some CPU intensive sections in the X server
> code, since gcc does *not* have a scheduler, whereas Digital cc does a good
> job in scheduling the alpha instructions to execute in parallel in the 21064
> dual pipeline.
>
> Shouldn't it make a difference?

It's just GCC; it *would* make a difference, I'm sure, building with DEC cc,
but it'd be ECOFF and statically linked, and a pain in the neck to get all
the Imakefiles and Makefiles and configuration files straightened out. It's
really just the XFree86 top-end with DEC TGA bottom-end, so it was *much*
easier to stuff the DEC code into the XFree86 structure and continue to use
GNU Make and GCC, as that stuff was well-tested under Linux/Intel.
