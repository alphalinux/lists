You are setting your machine as a gateway; are you sure that the kernel has
IP forwarding/firewalling/gatewaying options compiled in?

When debugging network connection I usually proceed as follows:

- /sbin/ifconfig (check the IP address/netmask/broadcast)
- ping myself and other hosts on the LOCAL net
- check the routing (tricky if more than one interface)
- ping my point-to-point partner on the other end of the line and other hosts, by number
- check DNS, ping computers by name

The gatewaying/routing adds another dimension to it; I would make sure
that the gateway can network on its own first, before I tried the
aliasing, etc. You sound like you understand the issues and the above
is a superfluous information, but I will post it just in case you are
overlooking some simple problem.
