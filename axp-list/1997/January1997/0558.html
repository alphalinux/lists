Once upon a time, Scott Manley wrote
> The Error messages all take the Form of
> Corrected (Or Uncorrectable) Bit errors which certainly could be the result of
> dodgy parity simms, but since the P5 manged it then there's nothing I can
> think of to explain away this problem.

Does the Pentium system you were using to check the SIMMs out do just
parity or ECC? If it is error-correcting, you would never know if the
Pentium motherboard got a single bit error and corrected it (there is
not a way for the hardware to report a corrected error back to the OS).

Also, if you are putting the memory in a UDB, it could be a heat related
problem. Things are pretty crammed into the UDB box. How you could
test this I don't know.
