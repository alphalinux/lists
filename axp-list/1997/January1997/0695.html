David C.Tuttle <dct@odin.mdacc.tmc.edu> wrote:
> I've got a Multia box running RedHat 3.0.3. I inherited an HP 35480A
> DAT drive, but the Multia's firmware doesn't support it (it recognizes
> it fine as a drive, it just doesn't support it :-P). I downloaded an
> ARC firmware update from Digital's Web site but it did no good.

ARC firmware doesn't "support" any tape drive. What exactly do you
expect ARC to do with it?

If you want to use the tape drive with Linux, you need to make sure you
have the SCSI tape drive in your kernel. Then you just use mt, tar, cpio,
etc.
