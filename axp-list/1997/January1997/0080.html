I wrote previously on this list telling everybody that I can boot
Alpha (UDB and eb164) telling milo something like this:

boot -t iso9660 scd0:kernels/noname.gz root=/dev/scd0

This is indeed correct. But yesterday I tried the same trick with
Cabriolet type (EV4) machine and I was consistently getting a panic
during a root mount. Kernel itself was loading fine. This was the same,
physically, CD and I tried that with two different CD drives, one of which
was actually the same one which served me well on two previous occasions.
Making boot and ramdisk floppies and booting from that (I had a floppy
drive, lucky me) solved the problem. Mounting the same CD and reading it
was not an issue once kernel was booted. Maybe some kernels on this CD
(RH 4.0, Archives) can perform CD boot and some don't? Or maybe there
are some differences in milo (for different modeles they are not the
same)? I was using the latest milo from gatekeeper.dec.com on all three
occasions.

Just another data point.

Michal
