I'm having some problems with my Linux/Alpha system, and would like to
solicit suggestions from you folks. Here's a list of my woes:

(By the way, the system is an AlphaStation 200 4/266, running 2.0.18 RedHat
4.0.)

1. Setting date/time via the SRM console results in a year value of 2017.
Setting the time via 'date -s' fixes that, but when I try to write into the
CMOS with 'clock -w' the next time I reboot I get a "date/time needs to be
fixed" error.

2. I tried patching the source tree up to 2.0.28, then compiling the
kernel. It errored badly. Attempts to compile xntpd and other things have
caused compilation errors too. I suspect that the source tree is hosed
badly, but I don't know enough to debug it myself.

3. I've used rpm to install anon-ftp services and upgraded the wuftp
package, but for some reason most ftp clients don't see anything via 'ls'
or 'dir'. Certain ftp clients CAN see things just fine, though, and others
on the same system (like ncftp) don't work at all.
