errrr. i struggled with building a kernel for a long time on this same
machine. i've posted this before, but here we go again (should this be
in a faq? is it already in a faq? does someone know?)...

grab the kernel source - the latest stable release i know of is:
ftp://sunsite.unc.edu/pub/Linux/kernel/v2.0/linux-2.0.27.tar.gz

the latest development release is:
ftp://sunsite.unc.edu/pub/Linux/kernel/v2.0/linux-2.1.20.tar.gz

you'll need the patch for axp-diffs. grab:
ftp://ftp.azstarnet.com/pub/linux/axp/sys/axp-diffs-2.0.21.gz

if you decide to build a 2.1.x kernel, then grab:
ftp://ftp.azstarnet.com/pub/linux/axp/sys/axp-diffs-2.1.1.gz

in /usr/src/linux, do a:
patch -p1 < axp-diffs-2.0.21

if you're going to build module support for the kernel, you'll
need to grab modutils (somewhere at ftp://ftp.redhat.com)

i was able to build everything i needed for my udb except sound
support. (i believe you need glibc for this - don't grab glibc-1.99 -
it has problems on linux/axp). if you want my config settings, hollar
and i'll mail them to you.

do a make config;make dep;make clean;make boot;

if you're compiling in module support, i believe you must also do
make modules;make modules_install;
