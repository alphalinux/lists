> Hello all,
>
> Has anyone tried putting a 28.8 (or similar) PCMCIA modem into
> a UDB ?? If you have then could you answer the following
> questions?
>
>
> Q1: Were you successfull? (if no then ignore the rest of these)

Yes.

> Q2: How much effort was involved?

It took much effort the first time, but with the help of this list and
it's archives at "http://www.redhat.com/support/mailing-lists/archives/
axp-list/index.html" it's easy now!

I'm running with a 166 UDB with Redhat 4.0 (2.0.27 kernel), Megahertz
28.8 pcmcia modem, pcmcia-cs-2.8.23 package. I couldn't get the latest
pcmcia-cs-2.9.x to work.

> Q3: Are you satisfied with the end result?

Yes, but I wish it were as simple as a RPM install (I'm spoiled!)

> Q4: Would you recommend the PCMCIA route versus an external modem?

I had the modem from my previous notebook computer, so I didn't have to
buy an external modem.
I have no trouble using setserial to bump speed to 57600, but I've seen
some messages on this list with people complaining about getting speeds
greater than 38400 on their external modems. (Not sure is this is
related to the serial ports in the UDB or what.) I haven't tried an
external modem on my machine.

> Q5: Can you summarize the steps involved?

It's been a while since I did it, but...

- Read the PCMCIA-FAQ

- Get pcmcia-cs-2.8.23 package

At the next to last line in the file modules/k_compat.h from the
PCMCIA package add

#include <asm/termiobits.h>
#undef __KERNEL_STRICT_NAMES
#include <linux/resource.h>
#include <linux/types.h>

You may also have to change the Configure script which wants to use
/sbin/ksyms. I don't have this on my UDB for some reason, so
change
this to cat /proc/ksyms. (This is only a problem if you choose to
have the Configure script look at the running kernel.)

- The last thing I had to do, once all compiled and installed, is
change
to the modules directory and execute "strip -x <modulename.o>".
Without this, I was getting file size errors with insmod.

- I also had to do some fiddling with the startup scripts to get eth0

working again...
