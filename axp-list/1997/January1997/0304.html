Hi!

I'm currently tinkering at the latest development version of LyX on
Linux/Alpha. gcc 2.7.2 (as it comes with RH 4.0) fails to compile one
file (lyxrc.C) when the "-O2" flag is specified:

[ceci@udb src]$ make lyxrc.o
gcc -c -g -O2 -Wall -Wno-unused -ansi -I/usr/X11R6/include -I. -I.
-I../images lyxrc.C
lyxrc.C: In method `int LyXRC::Read(class LString)':
lyxrc.C:717: internal error--unrecognizable insn:
(insn/i 474 471 5005 (set (reg:SI 160)
(reg/v:DI 165)) -1 (insn_list 471 (nil))
(expr_list:REG_DEAD (reg/v:DI 165)
(nil)))
lyxrc.C:717: confused by earlier errors, bailing out
make: *** [lyxrc.o] Error 1

Compiling without the "-O2" works fine. Is this a "known bug" already? Any
advices on how to work around the bug? (gcc dies at the end of a function
that contains a very long "switch" statement, BTW).
