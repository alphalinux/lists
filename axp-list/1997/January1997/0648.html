> I have this problem too, and I seemed to have solved it by not unmounting
> the /usr partition when I reboot, but instead remount it readonly.
> in file: /etc/rc.d/init.d/halt

The problem is that one of the libs used by bash lives in /usr/lib.
Another way to solve this problem is to make the rc.idontremember script use
ksh, or the move the library to /lib.

> It looks like it is working for me, I would be curious as to what others
> think of this idea. I'm pretty sure this is safe and doesn't corrupt
> files or anything otherwise I would see the error when rebooting and I
> don't. This doesn't solve the mystery of why you cannot simply unmount
> /usr, but it fixes the long fsck :)

I think it is a great idea. Not only as a solution for this particular
problem, but as a superior solution to unmounting filesystems at shutdown
in general. It just seems cleaner. Is there a way to do tell mount
to go through the /etc/fstab file and remount everything ro. Something
like:

mount -a -remount ro
