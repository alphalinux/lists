Hi

> On another topic: Does anyone have any
> comments on the stability of g++ and libg++ on the alpha? This is my
> workhorse on the pentium, and I have already encountered some problems in
> porting to the alpha, which admittedly may be due to my code (some 200K
> lines), but I am not sure.

I'm using a stock Redhat 4.0 system and have successfully compiled and run
several C++ apps on my UDB. -- The biggest C++ program I've been
compiling so far was LyX v. 0.10.7 (a _really_ nice WYSIWYG like frontend
to LaTeX). Sources are about 1300KBytes. It compiled virtually without any
warnings and after about one month of using it I dare to say that it is as
stable on the Alpha as on my Intel box.

That's just my personal experience. -- Maybe some "library guru" could
give some more detailed information. Anyhow, I think that the FAQ saying
"there is no working libg++" is definitely wrong in that point...

BTW: Which problems did you experience when porting to the Alpha?
