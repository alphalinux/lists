Erik Lindahl wrote:
>
> I've learned from the archives that certain OSF binaries
> can be run under alpha linux, but that it takes a digital unix
> license to use some files from DU, right?

Correct. That means purchacing DEC Unix since the
required libraries are in the Base. Even statically
linked (to DEC Unix libs) executables which do not
require any runtime DEC libraries cannot be run without
a license.

> Provided we get the licensing right, what's the status; will
> some/most/all OSF binaries run?

Most will. However, if you require the BLAS, LAPACK (DXML)
from DEC, you are out of luck. Those libraries require
an awareness of the correct DEC license in the kernel.

> Would it even be possible to run digitals compilers?

Haven't tried that one.

> (I haven't checked if we have site licenses for these, but if not, does
> anybody have any experience of digitals attitude, can we buy a license
> of DU and possible the compilers to use on a linux box?)

Hmmm......Let us know how that turns out.
