I'm not sure if this is an Alpha problem, but this list is full of
knowledgable people, so I would will it a try.

First a description of the settings.

I have stock RH 4.0 with a modular 2.0.28 running on a stock 166 UDB.
/home is mounted (with rsize=8192,wsize=8192) from a generic DX2/66
with 8MB memory, 32MB swap, 2 IDE drives and an AT1500 Ethernet card.
It is also running modular 2.0.28 on top of a stock RH 4.0.

I got a quite a few "NFS server not responding, still trying" messages
lately (after I upgraded kernels on both machines?) when I was doing
light compiling on the UDB (I/O through the Intel box). The Intel box
complains about "Can't get free page" while there were about 16MB free
swap space left. I noticed that this was more likely to happen when
the UDB was complaining about memory violations. So there could be an
Alpha connection.
