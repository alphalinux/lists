In message <Pine.LNX.3.93.961212101822.30990E-100000@redhat.com>, Erik Troan wr
ites:
>On Wed, 11 Dec 1996, Maurice Hilarius wrote:
>
>> I have created and used a single 500MB swapfile on an EB164. It works fine.
>> The machine has 256MB RAM.
>
>The limit on swap file size for Linux is based on the page size, right?
>Wouldn't that mean that swap files could be twice as big on the Alpha
>as on Intel?

Actually, I think it should be based on the *square* of the page size
(the limit is a page worth of references to other pages), so 512MB
makes more sense than 256MB. I've got about a gig free on my UDB's
/usr filesystem, so I guess I can try building a 600 MB swap file...

[root@envy src]# ls -l tmpswap
-rw-r--r-- 1 scott users 629145600 Dec 12 16:17 tmpswap
[root@envy src]# /sbin/mkswap tmpswap
/sbin/mkswap: warning: truncating swap area to 523648kB
Setting up swapspace, size = 536207360 bytes
[root@envy src]# free
total used free shared buffers cached
Mem: 21408 21120 288 6256 8960 2416
-/+ buffers: 9744 11664
Swap: 139232 12256 126976
[root@envy src]# /sbin/swapon tmpswap
[root@envy src]# free
total used free shared buffers cached
Mem: 21408 20920 488 5672 8928 2464
-/+ buffers: 9528 11880
Swap: 662872 12240 650632

I guess that answers the question -- there's a ~512 MB limit on
swapfiles. Like someone else said, there's a #define in the kernel
that limits you to 8 or so swap spaces at a time, but I think you can
just change that and recompile.
