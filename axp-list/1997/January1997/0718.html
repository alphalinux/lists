All,

I recently purchased a used UDB FX-40 (166mhz.), which has been running
just fine under RedHat 4.0+ (the plus means I'm using glibc-1.99, and with
good success).

However, a few annoyances remain:

- Whenever the machine is cold-started in the ARC console, it complains
about incorrect CMOS information which will prevent me from running NT
(not that I'd ever want to), and suggests that I use the Boot setup menu
to correct it. Unfortunately, there is absolutely nothing wrong in the
boot setup. I've redone it several times, and Linux starts up perfectly!
What is causing this time wasting message? How can I rid myself of it?

- I am unable to update the console firmware. Using an MS-DOS diskette
with FWUPDATE.EXE (latest and greatest), it will boot all the way to the
update utility. When I key in "update arc", something like this ensues:

APU-I VERIFY LOADED ROM IMAGE
0x337d8
APU-I LOADED IMAGE VERIFIED
APU-E ROM DEVICE JUMPER ON?

>>>>_

Same thing if I try to update SRM. I suspect that the previous owner may
have diddled with jumpers internally, but I cannot find any documents
which mention EEPROM write enable (although I'd guess this is the
problem). Where is the magic jumper to make this work?

Many thanks for any input.

Steve
