gcc problem? gcc would have a big problem if it would produce stripped
binaries when -g flag (debug) is in use. Anyway, Unix compilers **never**
produce stripped binaries unless explicitely asked (-s flag). If you
would find one which does not behave like that then I would classify
that as undoubtely broken.

'file', on the other hand, is less than forthcoming. :-) This should
be repaired, but this is really minor. Use 'size' for now.
