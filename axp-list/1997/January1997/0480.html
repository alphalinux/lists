> First there is no -lcrypt.

In RH4, crypt is in libc. It is back out again in 4.1.
This was primarily due to waffling on the part of the
glibc maintainers.

> Second, is file extension for shared libraries in RH 4.0 so(such
> as libc.so)?, is C library for RH 4.0 libc.so?

.so is what the linker looks for; .so.N[.M] is the binary.
You've discovered a neat trick glibc uses -- libc.so is a
linker script not a symlink to a binary as is the norm.

> Has anyone complied glibc-1.09.1 before?

No. It probably more than two years old. The current beta
glibc is 1.99 or 1.100 and can be found mirrored at

ftp://ftp.redhat.com/pub/glibc/
