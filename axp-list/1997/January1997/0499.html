Hi everyone,
I have a problem that may not be my own, but I wanted to run it by
you as my ISP is taking its own sweet time working on this with me.
My ISP gave me a subnet (net. add: 207.60.199.160, netmask
255.255.255.248) to connect with. My ppp0 connection connects up,
establishes the connection, gets assigned 207.60.199.161 for an IP, gets
the proper netmask, and sets up the deafult route (to the modem on their
end, 206.119.1.180).

I set up the following route:
207.60.199.160 0.0.0.0 255.255.255.248 U 1 0 91 eth0

for the internal network. This typically works, with a default set like
this:
0.0.0.0 206.119.1.180 0.0.0.0 UG 0 0 17 ppp0

right?

Maybe I'm missing something. Anyways, eth0 is set up for the same IP as
the PPP connection, and I made an aliased IP (eth0:2) with the IP
207.60.199.163 to test with (if I can ping it from outside, then it works,
right??) and it doesn't come up as anything from the outside world.

I can't ping my gateway (the 206.119.1.180) IP from any of my internal
machines, and a traceroute ends up looping forever.

Anyone have any ideas??? This one has blown my mind, on Linux/Intel I
have done this successfully several times, am I missing something?
