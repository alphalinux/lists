Hello,

I'm trying to install Redhat 4.0 on an Alpha Station 255. As the axp pages
suggest, I'm using the Avanti images (and the updated ramdisk).

The kernel boots ok, recognizes all devices (including the correct disk
models). In the install program, after entering IP address, netmask etc. and
selecting nfs image install, when I enter the remote computer's address and
directory path, I get "wrong argument to mount". If I use the IP address, it
apparently works. Now, my DNS server is ok (the Alpha has a record in it),
when I boot Digital UNIX there's no problem resolving names (and I can nfs
mount the redhat directory).

The second problem is with fdisk. It doesn't know the number of heads, sectors
and cilynders (all set to 0). The BSD disk label option shows the partitions,
but has some quirks (it asks the last cilynder and uses the number for size
instead when creating partitions).

Any hints? I'm looking forward to running Linux on my Alpha...

Thanks a lot,

Adriano
