On Thu, 23 Jan 1997, Mikael Nykvist wrote:

> On Thu, 23 Jan 1997, Steven N. Hirsch wrote:
> > After coaxing the latest Alpha glibc-1.102 into compilation, I am stuck
> > with regard to rebuilding all the existing system software. Numerous
> > existing utilities are (apparently) hard-coded into requiring libc.so.6,
> > while the new glibc is libc.so.6.1.
>
> Usually there exists symlinks to differnt names, like
>
> libc.so.6.1 -> libc.so.6
> libc.so -> libc.so.6
>
> would probably fix it (and a ldconfig ofcourse)

No, don't do this.

lib.so.6.1 is not just a minor release of libc.so.6. The soname is
different because incompatible changes have happened. Your binaries will
not work if you try to use libc.so.6.1 as libc.so.6, or vice versa.
