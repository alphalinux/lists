Yes, it is possible to install a different video card. I am currently using a
Diamond Stealth 64 Video VRAM (3240) in my UDB. However, there are a few
things you will need to pay attention to:

1. You must install using the TGA adapter. TGA support is built into the
kernel, and it excludes any VGA device. The UDB will boot fine with the card,
but as soon as you boot the kernel image, the screen will blank.

2. You must compile, and install a kernel *without* TGA support prior to
installing the video card.

As I see it, there are two primary advantages associated with installing a
video card. One is virtual console support with X up and running (this is only
if you have been using DEC's accelerated TGA X-server, which doesn't support
virtual consoles, the unaccelerated XFree86 driver supports VC on the TGA), and
the other is higher color depth. The TGA is a very fast display adapter, albeit
only capable of 8-bit color, and is faster than most of the lower and middle
tier video cards available for the PC. Windows NT comes with TGA support
built-in. With the card you stated you are using, you may be better served
using the TGA adapter built into the UDB rather than the Cirrus Logic board.
Another issue is whether there is X-Free86 support for Cirrus-Logic boards on
Alpha/Linux, most of the work has been getting drivers for the Mach64, S3, and
Matrox up and running. I personally do not know of support for Cirrus Logic.

I hope this clarifies things a bit for you.

Good Luck.

David
