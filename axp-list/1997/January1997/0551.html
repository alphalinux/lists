Once upon a time, Scott Manley wrote
> Well after much expeerimenation we concluded that it was definately a problem
> with the new memory. So we tried them in my friend's Pentium box and it worked
> fine, we recomplied the kernel twice and never found any glitches, sig 11
> faults or anything else unexpected. In fact nothing I try will generate any
> errors related to the memory.

Did you try the memory in a Pentium motherboard with parity checking
turned on? Are you _sure_ the memory is "true parity", not logic
parity? Remember, most Pentium motherboards do not check parity (at
least not by default), so you don't really _know_ there is no problem
with the memory (there could be a problem with the parity chips).

> So to say the least I'm a bit ticked off, everyone on this list always reporst
> that UDB's use just bog standard 36 bit true parity 72 pin 70ns ram. And
> indeed that's the spec fo the 12 meg simms alreadyu in my machine. So
> evidently there's something else which people should be aware of when trying
> to upgrade their boxes.

Not that I am aware of. Everyone I know that has bought true parity
SIMMS has not had any trouble with them.
