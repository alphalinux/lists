I'm not sure if this is the place to complain about this or not, but under
Redhat 4.1 on my alpha, I had to change the network init script to
configure lo first instead of going through ifcfg-* (where * is the
interface name in the config directory) because it would try sending
packets to the loopback interface but they would end up routed out the
ethernet card.

I realize this isn't very specific, but if more information is needed, I
can find it relatively quickly.

The bottom line is: I think lo needs to be configured first and it isn't.
