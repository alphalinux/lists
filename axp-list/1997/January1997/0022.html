> Can someone comment on the speed (anyone running gaussian or some other
> fp intensive task and/or code development) of 21164 based systems. My
> research advisor has asked for some input on hardware/OS solutions and
> I'm itching to go with Linux, but if Digital UNIX is a superior OS for
> software development I'll go with that.

Well its sort of hard to make blanket statements, but I use both
operating systems, and I will try to answer your question.

Digital Unix is a VERY nice OS, but it is somewhat of a resource pig.
As an example one of the new features of Digital Unix 4.0 is that it will
actually work on a system with "only" 24 M of ram. I run it on multi-cpu
machines with between 1 and 4 G of ram, so I really don't care how much
memory the kernel uses, but this may be important to you. The development
tools are nice, but gdb does not work particularly well. Digital has a nice
debugger called ladebug, but it uses dbx syntax which is somewhat of a pain
if you are used to gdb. I was pleased to see that gzip, CVS, emacs, and
tcl/tk come with the OS.

> I'm trying to convince our research group to go with UNIX (either Linux
> or Digital UNIX) instead of VMS and nobody here wants to monkey with the
> operating system, they just want to code.

Linux/Alpha is not yet a mature system. For me this is a plus because
it means that I might actually be able to contribute to or learn from its
development. I learned a tremendous amount about operating systems by
following alt.os.linux, and it was great fun. I can get some of the
same enjoyment I got from the early days of Linux/x86 development by
following Linux/Alpha development.

However, your goal is NOT to learn/hack the OS, but to use it. Assuming
that you can afford the software and ram, I would go with Digital Unix.
The libraries are more optimized, the compiler produces better code,
the compiler runs faster, and the debugger is faster. I hope that
this is not true forever. One of the nice things about Linux/Alpha
is that it will run Digital Unix binaries. If you start out with
Digital Unix, you can switch to Linux at some point in the future,
and still run your binaries (and you will have a license for all the
libs you will need).

Hope this helps,

Jim

PS Netscape runs under Digital Unix, but java applets dont work.
