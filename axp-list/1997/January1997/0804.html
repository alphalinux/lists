Michal Jaegermann wrote:
>
> AFAIK Linux does not boot from partitions over 2 gig (M$ strikes
> again?). On the other hand data partitions can be considerably bigger.
> Personally I tried something over 4 gig. I am aware of at least
> one claim (Stephen Tweede) of using, without problems, 52 gig (or so)
> partition.
>
> Root partition does not have to be the same as a boot partition.
> I boot my Intel Linux, which lives nearly all on the second drive,
> from a small partition, which keeps only boot images and some lilo
> stuff, on the first drive. I did not really check if 2 Gig limit
> really applies to a boot partition or this is a magic number for
> a root partition (as well?).
>
> Michal

I messed around with it and I was able to boot like this:
boot fd0:vmlinux.gz root=/dev/sda2

It seemed to like that, so I guess I'll make two 1440K partitions, one
for Milo and then one to boot from, and see how that works.
