>>> Robert Williams said:
> Installing on my PC164:
> The Buslogic BT-958D ultrawide
> SCSI adaptor appears to have IRQ=4,
> conflicting with another device.
> It doesn't appear to auto-configure
> itself, and the procedure recommended
> for running the AutoSCSI ROM after the
> BusLogic BIOS banner doesn't work.
> There is no BusLogic BIOS banner.
>
> Must I first install the adaptor on
> an Intel based PCI board for configuration?
> Any other advise?

If it's a PnP device, and you can set it up via some DOS utility, you *will*
have to do so, as there's no way currently to do PnP or BIOS setup on Alpha.

As it's a PCI device, it ought to get an IRQ > 15 on the PC164, IIRC.

Keep a SCSI controller that's supported by the console (ARC/SRM) for booting,
as the BusLogic is not supported for that. Or you could boot from floppy or
install an IDE disk for that purpose (at least ARC supports booting from IDE,
I believe).

Anyone out there with a idea for how we can do PnP on Alpha? This is starting
to become a worry for SCSI and soundcards... :-( :-(
