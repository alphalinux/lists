AFAIK gcc/Alpha (as it comes with RH 4.0) has a few bugs in the optimizer.
I'd try to compile the offending file without the "-O2" switch. Up to now
I haven't had any problems with gcc/Alpha after removing the "-O2" switch.
(but YMMV.. ;-) )

Again, AFAIK the Alpha port of gcc is maintained by Richard Henderson. I
think he will have less a hard time when looking at it if you could
provide him with a C++ source file that is as short as possible but still
allows to reproduce the bug.
