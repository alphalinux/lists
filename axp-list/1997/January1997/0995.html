On Fri, 31 Jan 1997, Richard Henderson wrote:

> > Has anybody compiled and installed glibc-2.0 yet ?
>
> For the record, 2.0 will not compile without minor patches on the
> Alpha. I was late getting changes back to Ulrich. But there were
> other problems as well, so there will be a 2.0.1 soon.

Yep with the patch it compiles for me. I have it installed now.
It does render your box pretty useless though, since a lot of
things are still going wrong for me (for example /bin/login stopped
working ;)

> > The announcement tells that a binutils-snapshot is needed for compilation,
> > and i'm wondering which one and where to get it.
>
> H.J's binutils 2.7.0.7 has everything what is needed, and it can
> be found on tsx-11 and sunsite and their mirrors.

I used stock redhat 4.0 with all the updates applied for this. I
just got a new kernel tree (2.1.24), the new modutils snapshot and
it worked. Then recompiled ncurses to get those headers back.
I'm still missing some netinet headers, though I am not sure if
programs using those should be fixed for glibc (Like not use
<netinet/protocols.h> or if I need to install bind for the new
headers.

And I should track down some redhat sources so I can recompile
/bin/login ;)

The guide posted on c.o.l.a was kinda Intel based (It described
how to park libc-5 etc). Since the alpha uses glibc already, it's a lot
easier then it might seem at first. Though I am still puzzled why login
is segfaulting (Might also be 2.1.24) but it's kinda hard to debug
at this moment without X, and a fixed 1024x768x60hz with the linux
kernel doing either 800x600 or 640x480 ;)
