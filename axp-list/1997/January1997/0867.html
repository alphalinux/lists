Sigh!. Try to boot your machine with the following command
(this is from the top of my head and I do not have docs handy,
but this IS in Milo docs; corrections anybody?):

boot -t iso9660 scd0:kernels/noname.gz root=/dev/scd0

(kernel has to the one for your machine, of course). It this
will work (on some models does not) then mount your hard drive,
as most likely it can be mounted - maybe after fsck, fix whatever
should be fixed and continue.

If booting from CD will turn out impossible, then try to mount
your root from a floppy:

boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1

Actually you may specify as above any reasonable combination
of a boot device (it can have ext2, iso9660 or msdos file system
type) and a root device. It is hard to believe that every possible
combination will fail. Once you will boot, even in an installation
screen, then you have a console - maybe with Alt-F2.

You have to be careful what you are using and what you are booting.
If you have yet an old Milo on NVRAM, and you try to boot ELF kernel,
and you did not switch to ARC console to load a new milo from a floppy,
then you will have problems. The same if you will mix up various
boot floppies.

There is a remote possibilty that something catastrophic did happen
with your hardware. This would be very unfortunate, but it is not
really likely.
