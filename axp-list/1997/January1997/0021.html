On Jan 1, 7:39pm, Steve Giles wrote:
> Subject: Red Hat loader partition problem
> Greetings,
>
> I am having a problem getting Red Hat 4.0 on my UDB to boot from
> partition 1. I have used dd to move the MILO floppy to a 1.5 meg
> partition (as per instructions). I can boot from the floppy but
> booting from partition 1 says "Bad device or invalid partition..."
> or something like that.
>
> I can mount the partition 1 in Linux and can see the linload.exe
> and milo files. Everything else works perfectly.
>
> This problem occurs identically on both of the UDBs I have so it
> is probably not a problem with hardware.
>
> I could just boot from the floppy however that would become tedious.
> Any ideas?
>
No ideas, just guesses. 1. is fdisk happy with the partitions as they stand
now. Tell it to print out the map and see if any of the sizes look fishy. 2. Is
partition 1 and active partition? 3. What format is partition 1 formatted with?
It had best be DOS; Milo knows ext2, but ARC only knows DOS FAT partitions.

(Which is funny. You'd think it might know about NT's file system)

Oh. And this configuration DOES work (that's a ray of hope, not a tease). It
works on my UDB fine...
