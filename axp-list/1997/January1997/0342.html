after finally figuring out how the alpha deals with timer and
real-time clock interrupts, i've gone ahead and added bits to various
files to enable the safe and sane use of CONFIG_RTC on the
alpha. before i send the patch on to linus, i would like people to see
if it works on their machines as well. if anyone's using
an eb166, i would love to hear if it works on one of those. its
palcode differs from all the others w.r.t. handling of the real-time
clock. please email me if you want the patch (against 2.1.20).

here's a little more detail. the following changes only take effect
when CONFIG_RTC=y:

1) sets up the interval timer to do ~ 1024HZ.
2) turns off interrupts from the rtc at startup.
2) interval timer takes over timer_interrupt.
3) rtc interrupts get handed to the rtc interrupt handler.
4) turn rtc back on upon shutdown (to make milo happy).

i've also added a little more space to /proc/interrupts to deal with
the jillions of interrupts that the timer reports. finally, i've added
a fudge to rtc.c. so that ARC time gets properly converted to linux
time. i have a question here. i'm assuming that all alpha's use ARC
time. is this true? do people not using ARC time get the right time by
default? i'll just add a config thing if ARC time isn't universal.
