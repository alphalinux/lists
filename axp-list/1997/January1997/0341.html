Hello Roger

>I am running a alpha machine w/ SCSI 1GB HDD. the verison of the linux I am
>using is Red Hat Linux 4.0 for Alpha. The problem I have is about fdisk
>utility comes with Red Hat. I have 1GB SCSI Hard Drive. and the
>specification of the hdd is
>2700(cylinders),
>9(heads)
>84(sectors).
>The old fdisk utility for linux only allow you input as high as 63 sectors
>for the hard drive. Could anyone tell me how to put 84 sectors in the fdisk
>utility? any comments and suggetions are welcome.
>
>PS: I tried to reduce number of sectors by use LBA calculation. I changed
>specification of HDD to 1012(cylinders), 32(heads), and 63(sectors). I
>don't know if this is the right way to do it or does this way will harm the
>hdd?

Heres what we always do with new drives. First, under fdisk run option "B"
for the BSD label. Then write down the heads, sector count and cylinder
count. Always never matches drives specs. Second, quit out and run fdisk
again and type "x" for expert mode and load in the information as follows:

h= heads
c=cylinders
s=sectors

The highest sector count you can use is 63, and then Linux shows the heads
and cylinders to be bogus BUT, it works everytime. In addition, if you
multiply out the sector x cylinders x heads will equal your drive space.
