On Jan 1, 8:32pm, Daniel Gruner wrote:
> Subject: Big problems with 4.0 installation
> Well, I thought that after all the good advice I got on this list I
> would be home free with the upgrade to redhat 4.0 on my machine.
> Needless to say, foolish me! I am really annoyed.
>
> The situation is as follows: I finally had a working machine with
> a kernel (2.0.23) that supported my hardware (AHA2940UW), network
> properly working, etc. The machine was flaky, sometimes panicking on
> the scsi controller, and I couldn't use the serial ports (builtin on
> a pc164), so "foolish me! ! !" I made boot floppies and took the plunge
> to upgrade. It all seemed to go smoothly until reboot time, when I got
> a message:
> /lib/modules/st.mod: No such file or directory

Tape drives? /etc/rc.d/rc.sysinit, line 168 reads,
insmod st
Which I changed to,
echo SCSI Tape Found! How nice...

That fixes that (and actually that's an Intel bug also). On the other issue
I dunno. If you have interrupt conflicts you likely would have seen them before
the upgrade, too
