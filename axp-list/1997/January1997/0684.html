I am having the same problem. I tried using the clock command first,
and then on reboot, ARC forced me to go in a change it there. However,
I am back to the year 2017.

shouldn't this be in the FAQ? anyways, the ARC console uses a
different starting epoch than linux (and most everything else)
does. as a consequence, dates get shifted upwards by 20 years. if
you're using redhat, the clock program should have a -A option which
accounts for this. make sure that your /etc/sysconfig/<something
related to time> is set so that you start up in ARC mode.

oh yeah, a personal plug: if you want to try the 2.1.21 kernel, i
have a patch to it which should just make your date problems
disappear. you do have to use hwclock instead, but you also get the
wonders of /dev/rtc. i'm not sure how well it works on a pc164
yet. send me email if you want to be adventurous.
