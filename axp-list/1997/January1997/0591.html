I wonder if this is known problem with milo.

I have a PC164 installed with Redhat develop (4.1) distribution. I use
milo-2.0.27 to boot the kernel. Both lnload and milo-2.0.27 resides in a
dos partition on my scsi harddisk. Everything works fine except that when I
try to reboot the machine, either by shutdown -r or cltr-alt-del. Milo
seems to regain control but hangs after it prints out its banner and the
amount of memory present (256M). One would expect the next printout from
milo is its boot prompt.

I'm using the latest firmware and a AHA2940 SCSI adapter.

Any ideas?
