I never really found out what.. but it looks like the kernel routing table
is now manipulated differently, and may need a new route binary. I posted
a message a week or two ago and got no response. Does anyone know of a fix?

At 07:54 AM 1/8/97 +0800, you wrote:
>I compiled the new 2.1.20 kernel on both our Jensen box
>and our 486. The compiles succeeded without problems.
>On booting with the new kernel, the following 'route add'
>commands in the network setup scripts produced warning messages
>
> /sbin/ifconfig lo ...
>>>> /sbin/route add -net ...
> /sbin/ifconfig eth0 ...
>>>> /sbin/route add -net ...
>
>I am using RedHat 4.0 on our Jensen box, and Slackware 3.1
>on the 486. The warning messages are:
>
> SIOCADDRT: no such device
> route forgot to specify route netmask
>
>for each of the two 'route add' commands. So I decided to revise
>the route add commands as follows:
>
> /sbin/ifconfig lo ...
> /sbin/route add -net 127.0.0.0 netmask 255.0.0.0 dev lo
> /sbin/ifconfig eth0 ...
> /sbin/route add -net a.b.c.0 netmask 255.255.255.0 dev eth0
>
>Now the warning messages disappeared on both the Jensen and the 486.
>But now I can only access hosts on our local net, but not hosts on
>the internet.
>
>I must be doing something wrong here. Please tell me. Thanks.
