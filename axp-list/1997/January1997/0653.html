Note that I submitted the results for the 500MHz Alpha, and then was
told (by the owner of the web page these results came from) that the
version of the benchmark I used was broken (on the Alpha) for the FP,
hence the bad result. As someone else said on this list, I couldn't
let running this benchmark become a "giant time sink", so I never
looked into fixing the FP code for the Aplha. Long story short -
ignore the Alpha's FP result.
