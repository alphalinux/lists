> "conversion from pointer to integer of different type."
>
> I guess this has to do with the fact that Alpha's pointers are
> 8 bytes but ints are 4 bytes only. Can I safely ignore these
> warnings (my machine is only 96MB anyway) or must I correct
> the ints to longs?

In general it is NOT safe to ignore warnings like these. These warnings
are usually caused by missing prototyes. For example you will get one
if you do something like:

char *str = fgets(buffer, sizeof(buffer), stdin);

and don't include string.h. The problem is that even though fgets()
returns a char*, if there is no prototype the compiler assumes that
it returns an int.

I don't know about Linux, but Digital Unix puts most things above 4G to
force a segmentation violation if you try something like this. I
suspect that Linux does too.
