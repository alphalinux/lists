Question for anyone willing to listen. I am running redhat 3.03 on a UDB
and not having too many problems but one of the most annoying thigs is the
speed of the ppp connection I get dialing out on a 288 modem.

I am being assigned an IP by my provider and I dial out using the
following command :

pppd connect 'chat -v "" ATDT*70,(*the #*) CONNECT ""
ogin: (*the acct*) ord: (*the Password*)' /dev/modem crtscts 57600 debug
defaultroute

Routing ends up as follows :

dialup-1.tmn.co * 255.255.255.255 UH 0 0 0 ppp0
loopback * 255.0.0.0 U 0 0 16 lo
default dialup-1.tmn.co 0.0.0.0 UG 0 0 25
ppp0

the port ends up :

ppp0 Link encap:Point-Point Protocol
inet addr:205.177.137.159 P-t-P:205.177.137.3
Mask:255.255.255.0
UP POINTOPOINT RUNNING MTU:1500 Metric:1
RX packets:2043 errors:0 dropped:0 overruns:0
TX packets:2150 errors:0 dropped:0 overruns:0

Would just love to speed this up a bit.
