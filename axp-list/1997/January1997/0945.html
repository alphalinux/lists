Hi,

let me first say that the upgrade to Red Hat 4.0 has been done on my
system month ago. Everything works flawlessly.

Today I struggled a bit against disk space and looked for packages
which aren't needed on my system.

I was really surprised as I saw that the old libc is still installed
on the system:

# rpm -qa | grep libc
libcaxp-debug-0.43-1
libcaxp-profile-0.43-1
glibc-profile-0.960906-1
libcaxp-0.43-1
libcaxp-include-0.43-1
glibc-static-0.960906-1
glibc-info-0.960906-1
glibc-0.960906-1
glibc-devel-0.960906-1

Is this really intentional? Also the old gcc (2.7.1) was still
installed. What I'm now asking myself is how to get rid of these
packages. If I remove them with:
rpm --erase libcaxp-0.43-1
I will certainly lose for example "/usr/lib/libbsd.a" which is a
part of "glibc-devel-0.960906-1", too.

# rpm -qf /usr/lib/libbsd.a
libcaxp-0.43-1
glibc-devel-0.960906-1

To be precise: I didn't upgrade "by hand" but used the update
procedure provided by Red Hat.

Did anyone experience the same? Any help appreciated. Thanks,
Markus
