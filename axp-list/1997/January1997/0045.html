>> I have created and used a single 500MB swapfile on an EB164. It works fine.
>> The machine has 256MB RAM.
>
>The limit on swap file size for Linux is based on the page size, right?
>Wouldn't that mean that swap files could be twice as big on the Alpha
>as on Intel?

Actually, four times as big. Linux uses one page of memory as a
bitmap to indicate which swap areas are in use. So you have:

Linux/i386: 4096 * 8 * 4k = 128MB
Linux/alpha: 8192 * 8 * 8k = 512MB

>As for 500MB, I woulnd't believe the kernel was using all of that unlesss
>/proc/meminfo supports you.
