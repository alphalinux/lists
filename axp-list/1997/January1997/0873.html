>Eagerly awaiting g77 version 0.5.20, which should have Alpha
>support...

Make that "will".

However, it's not clear that g77 0.5.20 will be noticably better
at generating optimal code than f2c|gcc in any particular case.
There _are_ some performance improvements in 0.5.20 that will
make it better for some kinds of code, but I don't think any of
them will improve things like division and square root on the
Alpha.

So, while g77 is going to support Linux/Alpha soon, it won't be
giving _everyone_ a reason not to use Digital UNIX and f77.

However, I am pretty sure g77 _will_ also get better at producing
decent Alpha code in the future. 0.5.20 will just be the first
release of note to Alpha users. For example, if it's true that
gcc 2.8 or some new version of gcc offers improved code generation
for Alpha, then that almost certainly means a subsequent version of
g77 will be made available that supports that new version of gcc.

tq vm, (burley)
