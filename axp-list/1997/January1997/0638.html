Hi everyone,
It looks like Digital UNIX 4.0 has this new system call:
set_program_attributes() (syscall number 43), and I'm having problems
figuring out what it actually does in order to emulate it.. (Actually, it
looks like the system call might be there already in 3.2, but nothing uses
it until the 4.0 /sbin/loader).

My current guess is that it sets a new base for the text and BSS sections
or something like that. It should be simple enough to emulate, but I just
want to find out what it actually should do. Unlike the other system calls
that we have no problems emulating, I can find no manual-pages for this
thing, nor even a function declaration to find out what the argument types
actually are.. It looks like it is used only inside the /sbin/loader, and
as usual the loader functionality is totally undocumented.

My detective work has so far uncovered that it has 4 arguments: that's
from looking at the Digital Unix sysent structure for the damn thing.
Looking at the use of this system call, it _looks_ like the third and
fourth argument is the base and length of the new BSS section, and the
first and second argument just _might_ be the base and length of where the
text should be loaded, but that's pure speculation.

Anybody have any information on this thing?

Linus
