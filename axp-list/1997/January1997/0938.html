> That worked for me :) *cheer* One small screen problem is gone,

> I still have a problem with ARC, MILO and Linux VC's ;)

> Guess it's time to hunt for ARC and MILO updates which respect the

ARC will never do such; it does its own thing and *only* its own thing... :-(

MILO could be made to do whatever you want, as its code closely resembles
the TGA driver code in the kernel, both of which you could certainly hack on.

You could avoid the ARC problem altogether by booting directly from SRM
console, which leaves only mucking about in the kernel to get the TGA driver
to do as you want. Make sure to read the FAQ section re: SRM booting; its
biggest drawback is that the disk partition table must be BSD style, and
the partitions must be setup with space left unused for the bootstrap code.
