>>> Bill Broadley said:
> I found a great deal on a sdram Alpha.
>
> 433 Mhz, 2.5 GB eide, NT4.0, 32 mb ram, matrox NT under $5k.
>
> Not bad, considering it's getting better specfp95 scores then the 500 Mhz
> clones costing the same or more.
>
> I BELIEVE it has a cdrom but not sure. I'm unassociated with the
> vendor btw.

Pretty sure that the one with EIDE disk comes with EIDE CDROM, but YMMV...

> In any case this purchase is to run Linux. I've heard that EIDE, and
> the on board ethernet are unsupported at this time.

The onboard Ethernet, built around the DEC DC21142 chip, has no LINUX driver
at the moment. Plugging in an EIDE disk caused the kernel to crash, but that
was early code, and may not be a problem now.

> I was considering buying a 4 GB scsi disk, Buslogic 958 scsi controller,
> and SMC etherpower 10/100 Ethernet card. I was planning on using
> the Matrox card (any matrox/alpha/linux users out there?).

Neither SRM nor AlphaBIOS consoles will, AFAIK, recognize the BusLogic, so
you may not be able to boot from it. The 21142 is capable of 10 or 100 Mbit,
but be careful as to which MAU (10) or MII (100) you buy. The driver will
be forthcoming; perhaps it'd be better to get by with an NE2000 or 3C509
before this happens. And the Millenium server, available now (XF86_SVGA from
gatekeeper.dec.com:pub/DEC/Linux-Alpha/X11) is quite stable, although 32-bit
depth has a few glitches.

> Anyone using these with Alpha/linux? Any ideas on these working on the
> new personel workstations?

It's being contemplated; so far access to test machines is problematic, and
no tests have been tried with the NT/AlphaBIOS boxes that are available now.

> I've heard that so far Linux will only boot with the SRM console thats
> not available yet. Is that still the case?

Not available yet, but soon. Don't know what the price differential will be,
but the hope is to support the NT machines with our MILO as soon as we can.

> BTW I've heard it gets 300-350 MB/sec on the stream benchmarks which
> is approximately twice the alpha 500/500.

Well, I've heard more like 320 vs 180, so twice is a little high... :-)

> Any recommendations? Suggestions? Warnings?

The cheapest box (433a) comes with no cache, but is still a reasonable
performer, because of the bandwidth. Also comes with IDE peripherals and
32Mb of memory in DIMMs, which are a bit pricier than SIMMs, I think.

Buy it now, and have fun trying out all the INTEL apps under FX!32, the
binary translator system another group in my department has done. Check it
out at:

http://www.service.digital.com/fx32/

By the time you run out of things to try, LINUX will be running on it... :-)
