On Mon, 6 Jan 1997, Steven N. Hirsch wrote:

> Would someone "in the know" please comment on this? I'm not aware that
> it is ever permissable to directly execute the loader.. This will not
> be the end of your problems, however. You will find that a handful of
> the tests fail. I haven't had time to dig into this at all - maybe over
> the next weekend.

It _is_ possible to directly execute the loader. If you run it with no
parameters it tells you not to, but if you give it a program name as an
argument it runs that program using the loader you specified, rather than
the default (/lib/ld.so.1).

There was a problem with ld.so in glibc 1.99 for the Alpha. Wait for
glibc 1.100, which is due to be released shortly.
