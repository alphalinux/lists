For one, you don't have a default gateway (see route). It looks like
you may be trying to config your local eth interface as a gateway, your
default gateway should be next hop on that interface.

As far as your named stuff (I just quickly glanced over everything)
typically in the resolv.conf file I'd put the real IP instead of the
localhost IP, don't really think that'd cause ya a problem though.
The above route problem COULD, I suppose, be causing your dns problem if
your machine is for some reason trying to hit a root server for a domain
name
server lookup??

Gerald
