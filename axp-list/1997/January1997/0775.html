Hi,

I'm trying to build a new MILO for my machine (PC164, RH4.0, kernel
2.0.28, glibc-0.961212, gcc-2.7.2.1, binutils-2.7.0.2-4). I've been
able to build an ELF MILO but it bombs out in booting and I suspect
the x86 emulator (excerpt of bootlog below). So I'd like to try
building an a.out MILO so that I can use the emulator provided by
Digital which, if I understand correctly, can't be linked to ELF
code. Can one build an a.out kernel and MILO on an RH 4.0 system? From
some of the switches in the Makefile for the a.out MILO, I'm almost
suspecting it is built under Digital Unix. Also, I read somewhere in
the archives about being able to build a MILO that uses the ARC
firmware emulator rather than one it is usually built with -- does
anyone have any more details on how to do that? I think that would be
the coolest solution!

Thanks for any help...

--erik
