Hi all,

I doubt many of you have been following my saga of getting my PC164 to
boot milo... Jay Estabrook was very helpful and patient but I think
both he and I were stumped. To recap: I have a PC164 board with 256
meg installed and couldn't get it to but MILO, various versions of
MILO exhibited the same behaviour of hanging when swapping to the
PALcode. Attaching a serial console to watch MILO's progress more
closely showed that it was getting to probing the PCI hardware, giving
up and trying to restart -- it was getting stuck in an infinite loop
doing this.

After swapping around and eventually removing all PCI cards I still
got the same behaviour so on a lark, I removed half the memory and
set the jumper on the motherboard to 128-bit data access. Guess what?
The machine boots and I'm running the RedHat installer right now...

So my question is: are there known problems with running with 256meg
and/or 256-bit datapaths under Linux/AXP? If it is possible, what is
necessary?

Thanks for any help and thanks again, Jay.

--erik
