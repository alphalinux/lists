> I have no mouse attached since all I have right now is a

The latest kernels, 2.0.27 and above, 2.1.15 (?) and above, have code
to fix the serial port problems (arch/alpha/kernel/bios32.c: SMCInit).

> I have a Fujitsu harddrive and Creative Labs CD-ROM attached
> to the onboard primary IDE controller and a 1.44mb floppy on
> the onboard floppy controller.

These are fine; I believe the latest Red Hat kernels have IDE CD-ROM support
built in, as well as SCSI CD-ROM, but I might be mistaken; Erik?

> I have tried with two Diamond S3 based PCI video cards: a 2meg
> DRAM one and a 4meg VRAM one.

The one shown below is an S3-964-based card; that's fine...

> Version 4.48 960603.1113

You could try updating the ARC console; see:

http://ftp.digital.com/pub/Digital/info/semiconductor/alpha/dsc-software-alpha.
html

for the file to download.

> scsi(0)disk(0)rdisk(0) (RAW) Fujitsu M1638 TAU
> scsi(0)cdrom(1)fdisk(0) (Removable) CreativeCD....
>
> It's strange that it's reporting the IDE devices as being on the
> SCSI controller??? Can I fix this somehow??? Do I need to???

Yes, it is bizarre, but quite normal (maybe the latest ARC console fixes
this?).

> Swapping to PALcode at 0x0
>
> and then it freezes. I've tried Milo 2.0.25 and 2.0.27 for PC164
> from gatekeeper as well as the Milo from the Miniloader/archives/PC164
> directory on gatekeeper and Milo-2.0.8-EB164-ide from the kernels
> directory all with the same result.

You've done all the right stuff, and those are the latest MILOs; do you
have the latest "linload.exe" to go with them, also from gatekeeper?

Also, you could try hooking up something to the COM1 serial port, see if
any MILO messages come out on there; the MILOs from gatekeeper are built
to echo messages to COM1 as well as to the screen...

Good luck.
