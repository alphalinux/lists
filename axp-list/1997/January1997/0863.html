Hi Claus,

> for (i = 2; i <= i__1; ++i) {
> gam += 1. / (doublereal) i;
> }

> Now, the Fortran-program compiled with the Digital Unix (f77) compiler executes
> TWICE as fast as the gcc executable (I found this factor also in other
> programs). I have read that the gcc math libraries are not very fast,
> but I do not understand why this simple loop is that slow with gcc. So my
> questions:
> 1) Any ideas why ? (This is Redhat 4.0).

I am not supprised that it is faster, but the factor of 2 does supprise me.
I would find it interesting if you could compile the output of f2c with the
Digital cc compiler to see if the difference is related to the f2c translation
or a problem with gcc. I think the g77 people are working on making their
stuff run on 64 bit platforms. You might see if you can get hold of some
of that code and see if it runs better.

> 2) Will this problem be eventually resolved some day (in the nearer future)?

People are aware that both the Linux libraries and the gcc optimizer need
to be improved. As far as I know, no one has taken this project on.

> 3) Has anyone successfully installed Digital Unix f77 on a Linux system ?
> 4) What would be the license $ for f77/DU if 3) works ?

It is probably possible to get Digitals f77 to run under linux but its going
to be a lot of work. You would probably be happier setting up your machine
to dualboot Linux and Digital Unix.

Sorry I don't have better news,

Jim
