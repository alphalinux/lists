On Mon, 27 Jan 1997, Miguel Bettencourt Dias wrote:
>
> Well .. I noticed a problem ... I can't get any Netscape Server running on my
> system because it says root is not a user on this system, during instalation.

Ok. This kind of problem is probably not a kernel problem but a user-level
database problem. You might try stracing the server and see what files it
tries to open: Digital UNIX has more user information than just the
/etc/passwd file (a user security database).

It _may_ be that it also wants to use some special system calls, but in
that case your system logs should log that (the kernel prints out
something like "<sc(xxx,xxx,xxx,xxx)>" for unimplemented OSF/1 system
calls - there aren't all that many and most of them are really obscure).
