>>> "Todd M. Shrider" said:
>
> I have a UDB (Multia) and had some questions about adding a video card
> to replace the TGA 8 plane video adapter built onto the motherboard.
> Specifically I was going to add an ET4000. I tried this but when I turned
> the computer back on it didn't display anything. Are there jumpers on the
> board that have to be set to disable the TGA adapter? Also, I'm guessing
> that I should be able to find an Xserver for the card that has already
> been ported to the alpha platform. Thanks for any info...

Are you sure you plugged the monitor cable into the correct graphics adapter?

:-) :-)

Seriously, the console does try to choose the VGA over the TGA, if it sees
a VGA present. Did you try swapping the monitor cable over to the TGA when
you got nothing on the VGA, to see if it was trying to come out there?

If you got nothing on either, then perhaps the BIOS code on the ET4000 gives
the emulator a rough time; or perhaps it just takes a very long time to do
the BIOS init code (early emulators could take forever, on such cards as
Matrox Millenniums). In this case you may be out of luck... :-(

Regardless, you won't have an XFree86 server that'll work on that card,
unless you port the code yourself. The only part of the SVGA server that
runs on Alphas and is currently part of the official code base is Millennium
support, although I do have patches for the PVGA1/WesternDigital driver.
And the XF86_W32 server has also not been ported to Alpha, AFAIK.

Good luck.
