>>> Shane Anderson said:
>
> I've been (fairly) happily running the XF86_TGA-3.2 server on my UDB for
> some time and I just caught a tidbit saying that the 3.1.2 version had the
> proprietary (and faster) code in it. So off I stampede to get 3.1.2 and
> it doesnt work! Same XF86Config file that works with 3.2... Hmmm..

Well, stampede off and read the README that came with it... :-) :-)

You won't be able to use the exact XF86Config file, as 3.1.2 had no XKB.
And you need to provide a "MemBase" so the server knows where the TGA
hardware framebuffer and registers appear on the PCI bus; 3.2 can determine
it for itself.

Please, consult the README for details:

gatekeeper.dec.com:pub/DEC/Linux-Alpha/X11/axpbin-Xtga-BETA-0.2.README

Note that this README is appropriate for *all* the 3.1.2 TGA servers there,
but not for the 3.2 version.

Please also pick up the one appropriate for the Alpha CPU in your box; EV5
for EB164/PC164/ALCOR/AS500/AS600; all others, especially UDB, use the EV4...
