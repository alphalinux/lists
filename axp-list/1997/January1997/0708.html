> I'm downloading the RedHat 4.1 stuff now. I wanted to get any opinions on if

RH4.1 is NOT released, but I asume you mean the _unsupported_ devel tree.
I installed the 970117 snapshot (complete reinstall, I did not try to
upgrade), and here is my "log of errors" I always have on my machines.

axp/README says 4.1/Intel ;)
"scanning packages" during install gives some strange output.
util-linux installed twice
In /etc/rc.d/init.d/halt awk and xargs are executed after all partitions
in fstab are unmounted. if /usr is on a different partition, this will
fail.
inetd doesn't respond to -HUP
identd doesn't like -o option.
Wrong permissions on /var/spool/mail. RedHat creates its as 775, but this
makes procmail to timeout each time a mail is to be deliverd. 755 and
1777 works ok. though, elm does only work with 1777
/usr/sbin/tmpwatch has wrong permissions, should be 755.
Add /usr/games to path, or xlock -mode marquee can't find the fortune
program.
Any chance of useing gpm in repeater mode (-R) if you have a ps2 mouse?
(And using MouseSystems instead of ps2 in XF86Config (Xconfigurator's
job?) Else you can't have gpm and X11 running at the same time..
Shadow passwords does not work. pwconv5 works, but you can no longer
login, pwunconv fails to restore the normal passwd (it does not create
a correct npasswd either)
Bug in logrotate, all entrys in logrotate.status gets the 'date' 1900-1-0,
which gives an error later on.

(Erik, are you insterested in me keeping on sending in bugreports? I
didn't get any feedback on the last reports, maybe you are busy? :-)

So, should you install or not? The only reasons to install the devel tree
today must be to either help redhat find bugs, or you have a big need of
several programs in the tree.

Oh btw, incase you didn't know there has been some "big" changes in libc
lately, so all programs needs to be recompiled to work against them. So
incase RH4.1 will be based on glibc1.102+ you will have to download all
packages again and reinstall them.

/Mikael
