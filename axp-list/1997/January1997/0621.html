Hi Miguel,

For once, I hope I can help you out :-). The glibc version distributed
with RedHat/Alpha does not support NIS. It is possible to install
glibc-1.99 from the devel directory at ftp.redhat.com and obtain NIS
function, but it requires the simultaneous upgrade of numerous other
packages. I struggled through it last month and posted a long, rambling
howto on this list. You can probably dig it up by searching the message
archives on their website.

Since then, I've discovered a mistake in my configuration of the
nsswitch.conf file, so here is an update on my scripts:

#
# /etc/nsswitch.conf
#
# An example Name Service Switch config file. This file should be
# sorted with the most-used services at the beginning.
#
# The entry '[NOTFOUND=return]' means that the search for an
# entry should stop if the search in the previous entry turned
# up nothing. Note that if the search failed due to some other reason
# (like no NIS server responding) then the search continues with the
# next entry.
#
# Legal entries are:
#
# nisplus or nis+ Use NIS+ (NIS version 3)
# nis or yp Use NIS (NIS version 2), also called YP
# dns Use DNS (Domain Name Service)
# files Use the local files
# [NOTFOUND=return] Stop searching if not found so far
#

passwd: files nis
group: files nis
hosts: dns [!UNAVAIL=return] files nis
# hosts: db files nis

In that particular version of glibc, the dns module is the only one that
can handle IP addresses in "presentation" format. This has been fixed
(alledgedly) in later versions of the library, but I have yet to be
successful in installing glibc-1.102.

Good luck, and let me know if you need any more input?

Steve
