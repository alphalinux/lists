This sounds like an infamous "missing nsswitch file" problem, which
was rehashed on this list over and over again. Your machine is
trying to contact a nameserver and timeout on this is rather long.
Add to your /etc/ directory something like that (boot "single"
to do that - you do NOT want network services yet :-).

# /etc/nsswitch.files:
#
# An example file that could be copied over to /etc/nsswitch.conf; it
# does not use any naming service.
#
# "hosts:" and "services:" in this file are used only if the
# /etc/netconfig file has a "-" for nametoaddr_libs of "inet" transports.

passwd: files
group: files
hosts: files dns
networks: files
protocols: files
rpc: files
ethers: files
netmasks: files
bootparams: files
publickey: nisplus
netgroup: nisplus
automount: files
aliases: files
services: files

>
> I did not have these problems when upgrading my pentium box...

You will have the next time. :-) AXP is using newer, in a sense,
libraries and that is likely the reason Red Hat forgot about this
file. It is not needed on Intel yet, but it will be - soon.
Make sure that you have a valid /etc/hosts file before you will
try multiuser boot again.

> And this machine doesn't seem to want to continue booting (still stuck at
> smbd, after 20 minutes or so) - hey! now it is trying nmbd!

The general principle is that if you have problems booting you boot
in "single user" mode and try to resolve them. This is why this mode
is needed.
