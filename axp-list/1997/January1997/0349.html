> As a new Alpha owner I am struggling to set up perl 5.003 with

Instead of struggling just run the current distribution with
shared libraries. It does dynamic linking by a default. Here
is an output of 'perl -V' from Alpha with RH 4.0 distribution:

Summary of my perl5 (5.0 patchlevel 3 subversion 0) configuration:
Platform:
osname=linux, osver=2.0.18, archname=alpha-linux
uname='linux judy.redhat.com 2.0.18 #1 tue sep 10 17:44:31 edt 1996
alpha '
hint=recommended, useposix=true, d_sigaction=define
Compiler:
cc='cc', optimize='-O2', gccversion=2.7.2
cppflags='-Dbool=char -DHAS_BOOL'
ccflags ='-Dbool=char -DHAS_BOOL'
stdchar='char', d_stdstdio=define, usevfork=false
voidflags=15, castflags=0, d_casti32=define, d_castneg=define
intsize=4, alignbytes=8, usemymalloc=n, randbits=31
Linker and Libraries:
ld='cc', ldflags =' -L/usr/local/lib'
libpth=/usr/local/lib /lib /usr/lib
libs=-lgdbm -ldb -ldl -lm -lc -lposix
libc=/lib/libc.so.6, so=so
Dynamic Linking:
dlsrc=dl_dlopen.xs, dlext=so, d_dlsymun=, ccdlflags='-rdynamic'
cccdlflags='-fpic', lddlflags='-shared -L/usr/local/lib'

@INC: /usr/lib/perl5/alpha-linux/5.003 /usr/lib/perl5
/usr/lib/perl5/site_perl/alpha-linux /usr/lib/perl5/site_perl .

Personally I would insert **at least** -Wall in compilation flags and
would be more agressive with optimizations. These are details, but
important, as MakeMaker will use these as default flags for all modules
you will try to install. Paying attention to what -Wall says can guard
you against many nasty surprises on 64-bit machine. You can still do
that by recompiling perl from scratch (you may want to wait with that
a bit for 5.004 - RSN :-).
