> After coaxing the latest Alpha glibc-1.102 into compilation, I am stuck
> with regard to rebuilding all the existing system software. Numerous
> existing utilities are (apparently) hard-coded into requiring libc.so.6,
> while the new glibc is libc.so.6.1.

Indeed. Programs remember what version of the library that they
were compiled against. As of the 970104 snapshot, several of the
basic types like sigset_t and dev_t changed size, and sanity
required that the library version be changed.

> My question is, how can I rationally rebuild bash, login and friends on an
> Alpha which only boots and functions under libc.so.6, yet have the
> resulting object files correctly link and run against libc.so.6.1?

You pretty much have to do this in a chroot'ed environment. You'll
need to install glibc to a test directory, build staticly linked
versions of bask, make, gcc, binutils, and other utilities and install
them in there as well. Then you chroot to the test directory and
you have an environment only using libc.so.6.1.

Once you've verified that everything seems to be working fine after
having rebuilt the basics of your system, you can overwrite your
main system and start rebuilding the rest.

It isn't particularly complicated, but it is tedious and unless you've
got some time and patience don't even try.

> BTW, "test-canon" still fails on the completed libc.so.6.1.

>From previous examination, I think the failure is in test-canon rather
than libc.
