I've got a PCI network card that I'm putting in the
PCI slot in a UDB machine. The syste will boot to MILO
prompts, but whenever I try to access the SCSI disk after that
point, the machine hangs. I can watch the PCI bus with
an analyzer and it shows that there are just a series of
I/O accesses to 0xb4xx which is the I/O space for the SCSI
disk/card. I believe that the "new" card is NOT getting the
same IRQ as any other cards (by watching config space accesses).
The new card does not have any I/O space, so there's no conflict there
and the memory space allocated to the new card appears to
be in a different region of memory than all the other devices.

I'm stumped at the moment. The card works fine in x86 machines
(and others) running linux and it works in a Dec Alpha running
DecUnix.

I've tried to boot MILO and the kernel from a floppy, but the machine
always seems to want to probe the SCSI bus since it sees the controller,
and the machine hangs at the point. I've pulled the disk, but there's
seemingly no way to disable the controller even for a quick test.

Anyone else seen problems like this?

Help!
