I concur with John. In fact, I have just done that, where I have one
linux box as the only real link to the outside (using slip), and another
linux box and an old pc running windows being masqueraded by the
gateway. It works like a charm, once you get the MTU values sorted out.
I suggest you look at the article on masquerading that appeared in
Linux Journal, Issue 27, July 1996. It is very enlightening, especially
about the fragmentation problems.
Also, make sure you load the modules that belong to the masquerading
software, such as "ip_masq_ftp", if you want ftp to work.

Daniel

>
>
> I would use IP Masquerading in this instance. Set up your home
> LAN to use a private Ip addressing scheme, such as 192.168.x.0 for
> you home subnet, and let the linux box do NAT (network address
> translation.) This way you don't eat up a whole 30 device subnet
> for 3 boxes. Also, doing it this way, you can handle dynamic IP
> addressing on the PPP side.
