On Thu, 23 Jan 1997, Steven N. Hirsch wrote:

> Immediately after posting my previous response, I remembered what the
> "gotcha" was! With the new libraries installed in /lib, getpwent() and
> friends become completely broken. Many tools fail because they are
> unable to map uid to user name (and vice-versa). Because of this, I was
> not actually able to build anything.

Oh dear. It sounds as if libnss* could perhaps have done with a soname
bump as well.

> How shall I go about dealing with this problem?

If there are incompatibilities, we should change glibc so that the nss
libraries also have sonames like 'libnss_files.so.1.1' or something.
Richard, do you agree? There is not much point bumping the soname on libc
if binary compatibility has been broken in other places too.

As an interim solution, it's worth remembering that the libraries that are
linked against are /usr/lib/lib*.so, and don't have to be the same as the
ones you're running. So, theoretically, with a bit of care you could
rebuild everything against new libraries while still running the old ones.
I suggest you hang on to see what Richard has to say though.
