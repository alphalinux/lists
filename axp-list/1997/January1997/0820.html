> You forgot some bugs in gdb,mtools etc etc. I have a feeling the alpha
> distribution was made with 'If it yields a binary after compile, add
> it to the distribution'. Some things just are not tested, like mtools
> etc which always dump core. I can understand they can't find all the subtle
> bugs like the one(s?) in gdb which only show up debugging something
> else, but some very obvious easy to find bugs are a bit disappointing.

I think the problem is that they want the distribution for alpha to
contain as many of the x86 packages as prossible, and they haven't
got enough people to check if things are actually working on alpha.
Then all the users wants new distribtions out as fast as possible, and
there isn't get enough time to test what they are releasing.
Therefore they are relying a bit on users reporting bugs.

But I think it's ok that the bugfixes shows up in updates after a while.
It's very easy to upgrade to them.

> Btw. I am not bashing redhat, they've mostly done a good job, esp.
> considering the lack of tools they had to develop this compared to
> the intel platform. The alpha is just a lot less stable.

Well, I'm not sure about all the bugs you are talking about, but atleast
joe and mtools are supposed to be fixed in the updates directory. When I
installed 4.0 myself, I made simple patches to them, and a few weeks later
similar patches showed up in updates. Are you sure that you are using the
packages in updates? (I just tried the joe and mtools updates, and they
appear to be working here)

aumix seems to be made for the console, it works in a vt (TERM set to
linux), but not in xterm. I think they rely on the different characterset
used in the console.

By the way, I have just installed 4.1 from the devel directory, and it
seems that there are a lot fewer buggy packages than there were in 4.0.
xplaycd and tracker still coredumps, though. I hadn't tried any of them
myself actually.
joe and mtools are definitely working except joe won't resize if an xterm
is resized, and it picks the wrong size in console and remote login.
