Hi!

Thanks to all the peope who have provided help on that issue. -- The good
news is that the machine is now happily running RH 4.0. ;-)

Finally we ended up opening the box and really found that the SCSI bus was
only terminated on one end! -- Lesson learned: To err is human. -- Even
people assembling big, expensive hardware make errors... ;-)

That ended almost all of our problems: Copying 1.5GB via NFS to a local
disk while building a kernel, whilst runninx X... did not crash the box.

I've got an "I/O Read Error" once when trying to read a larger amount of
data from a certain CD-ROM, but I suppose that the problem was indeed an
unreadable CD-ROM. (It was reproducible with that CD-ROM but did not do
any harm -- like crashing the box...)

Oh, BTW: Milo still complains about "NCR dead, restarting", but up to now
Milo has never failed to load the kernel (off the harddisk) and the
kernel itself does not complain. I've even been able to enable tagged
command queuing and disconnect/reconnect (and it works)! ;-)

Concerning the DEC-Tulip ethernet adapter I've followed the advise you've
been giving me and stayed with the "DE4x5" driver which happily
transferred the 1.5GB NFS data... -- Suppose I'm not missing that much
when not runnin the "Tulip" driver?

And BTW: the graphics adapter in this machine is a Diamond Stealth 64 with
4MB VRAM and S3-986. -- XFree86 3.1.2 works like a charm... ;-)

Thanks!
