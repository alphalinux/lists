Dave Golden wrote:
>
> IRQ 4 is historically the interrupt for COM1. I don't recommend
> mucking with that. I'm very surprised the buslogic would default
> to that IRQ...
> If all the settings are "soft" you will have
> to use an Intel based system to run the config utilities and set
> it to something reasonable. Oh, and I'm assuming this is an ISA card
> right?

PCI.
It appears now that the PC164 ARC Console shows SCSI device 7
(the card) has IRQ 4, but a check of the card on an Intel system
shows that it is set to IRQ 9. I think this is probably a driver
problem, because system boot doesn't complete. I doubt now
that anyone has a working BusLogic BT958D (Ultrawide) on a PC164.
I'm pursuing the problem with the driver author and BusLogic.
