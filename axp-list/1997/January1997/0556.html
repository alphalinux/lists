Regarding the quality of simms, if you can get them with the same =
manufacturer for the chips and the simm board they're mounted on you =
traditionally end up with a higher quality product. True parity is a =
must in RISC based environment as you well know and Intels are more =
forgiving. Pentinums do not normally require parity so if your chips are =
marginal on the parity bit testing them in a Pentium isn't a true test =
of the functionality of the chip. If the Pentinum you have allows you to =
turn parity on it would be a much better indicator if you have quality =
memory.

Hope this helps. Most of this info is based on personal experience so =
you can take it or leave it since i don't have any technical references =
to backup these observations.

Bruce

Once upon a time, Scott Manley wrote
> Well after much experimentation we concluded that it was definitely a =
problem=20
> with the new memory. So we tried them in my friend's Pentium box and =
it worked=20
> fine, we recompiled the kernel twice and never found any glitches, sig =
11=20
> faults or anything else unexpected. In fact nothing I try will =
generate any=20
> errors related to the memory.

Did you try the memory in a Pentium motherboard with parity checking
turned on? Are you _sure_ the memory is "true parity", not logic
parity? Remember, most Pentium motherboards do not check parity (at
least not by default), so you don't really _know_ there is no problem
with the memory (there could be a problem with the parity chips).

> So to say the least I'm a bit ticked off, everyone on this list always =
reports=20
> that UDB's use just bog standard 36 bit true parity 72 pin 70ns ram. =
And=20
> indeed that's the spec of the 12 meg simms already in my machine. So=20
> evidently there's something else which people should be aware of when =
trying=20
> to upgrade their boxes.=20

Not that I am aware of. Everyone I know that has bought true parity
SIMMS has not had any trouble with them.
