At 02:58 hrs. 01-28-97 -0800, you wrote:
>On Tue, 28 Jan 1997, Stephen J. Gaudet wrote:
>
>> You need an SRM license and a UNIX license. The UNIX license allows you to
>> unlock and load Digital UNIX. Without the license it will not load. As far
>> as the UDBs are concerned it comes with it so the license is built in.
>
>This is strange. I recently got an AlphaServer 1000 with DU installed.
>It came without:
>
> the liscense
> a power cord for the monitor
> a key for the hard drive bays
> the hardware documentation
> the EISA config disk
> and a couple of other things
>
>But, DU still booted. It only allowed root logins, but I could get it
>up, run a couple benchmarking utilities, and figure out that the Compaq
>QVision card that came with it sucks. Then, I put in a Diamond Stealth
>and now DU won't boot. But, my point here, is that DU will boot without
>a liscence. The liscence wasn't installed at the factory because I had
>to reinstall DU (installed Win NT over it...oops...damn ARC listed the
>drives in the reverse order. Who would'a thought disk3 would be the
>first in the list?) Perhaps you meant that you need an SRM liscense to
>boot DU?
This is normal. To install Digital UNIX you need SRM console in the system (
usually installs into flash ROM on most models) On most machines this SRM
console code bears a license cost. Notable exceptions to this rule are the
UDB/Multia machines and the AXPpci33 (noname) machines. Next you install
UNIX. Now, UNIX runs, but supports NO users. You can log in as root, and do
some, but not all things with it. Next, you run lfm to install the license
information, and then you are set. Note that on most models it is possible
to have both the SRM console firmware and the ARC NT loader firmware at the
same time, and toggle between them. On some models there is even enough room
in the flash to also burn MILO in there as well.
