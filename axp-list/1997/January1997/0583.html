Forgot one of the main points I wanted to make in my last post:
When I got the Enorex I went through my normal new harware ritual of
opening it up to see what was inside. Unfortunately, I found that the
front fan was not hooked up to the power supply, and the scsi cable was
*just* about to fall off the CD-ROM drive. That, unfortunately, doesn't
say too much for Enorex's quality control. I'm guessing those guys
can't make these things fast enough, so they're not getting the checking
they should be, but I might be wrong.

Also, when I first had the machine up running NT(oooh..the fun part
comes to see if I can dual boot this guy later this week...) a few times
when the machine was idlely sitting there running an OpenGL screen
saver, I'd suddenly hear this loud POPWHACK and the monitor(MAG 17")
would go blank...power monitor off and back on, it'd work ok. Called up
Enorexs sales line to get a number for their tech support, lady was nice
enough to deal with my problem there and have a techie call me back.
Looks like the Matrox card was overdriving the monitor, which was
shutting down as a result.
Got the beast running X almost painlessly once I got the SVGA-test
server, then realized all my ssh connections had died...looks like there
might be a conflict between the matrox and the 3com.
Anyways, I'm beginning to blab on and on, so I'm going to go to sleep. ;)

one last note: recompiled povray with some fp switches turned on for
gcc, got the raytrace time for the previously mentioned frame down to 86
seconds, a bit more like what I wanted to see. :)
