Hello,

>install (not an upgrade), and the same thing. Eventually it timed out,
>but it took about 15 minutes, and kept getting stuck when initializing
>almost every service (at least those that had to do with networking
>(rpc.mountd, rpc.nfsd, sendmail, smbd), and I am still waiting for it
>to finish booting...
>

Not from my own experience, but from reports on this list, I believe this
phenomenon is related to a missing /etc/nsswitch.conf files from the 4.0
distribution. Without it the new library routines attempt a DNS lookup
first, and uses your /etc/hosts file only after this times out.

Here's one that may work (supplied to this list by David Mosberger-Tang):

---------
#
# /etc/nsswitch.files:
#
# An example file that could be copied over to /etc/nsswitch.conf; it
# does not use any naming service.
#
# "hosts:" and "services:" in this file are used only if the /etc/netconfig
# file contains "switch.so" as a nametoaddr library for "inet" transports.

passwd: files
group: files
hosts: files dns
networks: files
protocols: files
rpc: files
ethers: files
netmasks: files
bootparams: files
publickey: nisplus
netgroup: nisplus
automount: files
aliases: files
services: files
sendmailvars: files
------------
