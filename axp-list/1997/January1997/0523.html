At 08:02 AM 01-20-97 -0500, you wrote:
>
>IRQ 4 is historically the interrupt for COM1. I don't recommend
>mucking with that. I'm very surprised the buslogic would default
>to that IRQ. I would expect it to use 14 or 15 (the primary and
>secondary IDE IRQ's). If all the settings are "soft" you will have
>to use an Intel based system to run the config utilities and set
>it to something reasonable. Oh, and I'm assuming this is an ISA card
>right?
>
Actually these are PCI cards, and on most Alphas they by default install the
PCI SCSI at IRQ 3 and 4. I am looking right now at an EB164 with NT 4.0
server on it, and the 2 installed SCSI cards are at 3 and 4. This works.
