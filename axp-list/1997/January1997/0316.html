> NEVER, EVER tell fdisk or LILO or the kernel about it. It is strictly a
> business between the SCSI controller and the disk. Let me repeat that:
> only silly people tell fdisk/LILO/kernel about the true SCSI disk
> geometry.

Only silly people with very old disks can do that - the geometry of modern
disks is not fixed, but uses a variable number of sectors per cylinder
depending what zone the cylinder is in. Even the SCSI controller does not
know about this - SCSI only uses sector numbers, starting at zero.

You are right that if you need a geometry due to some old (eg.
MSDOS-inherited) partition info format, you can pick any geometry that gives
a correct number of sectors - you won't be able to tell fdisk about the real
geometry the disk is using internaly.
