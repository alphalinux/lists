On 09-Jan-97 Topi Kanerva wrote:

>umm, i'm not really into this msdos stuff. what is this 'extended'
>partition and why it has to be there? i suppose you can't have bigger
>disks than 500MB with msdos partitioning. but on the other hand, the root
>disk has to have at least one msdos partition (sda1) where to boot milo
>from... i guess this situation is quite annoying? i don't want to use
>msdos partitioning on my hd.
>
>

Only four primary partitions can be created per disk. If you need more
partitions, you must create an extended partition, and add logical partitions
as needed. I believe this is a Linux fdisk restriction. If you recall, under
MSDOS, you can only have 1 primary partition per disk, any additional
partitions must be logical partition within an extended partition. Also, the
~500MB BIOS limitation per partition (only with older BIOSes that do not support
logical block addressing) do not apply for Alpha.

The reason you need an MSDOS partition is that MILO uses the ARC firmware for
booting Linux on the Alpha. The ARC firmware was developed by (or for)
Microsoft in order to setup and boot NT on supported RISC machines. The ARC
firmware includes Intel emulation to run the DOS executable residing on a
MSDOS filesystem that loads NT. Linload.exe is a DOS executable that loads
the milo image used to boot Linux.

Of course, you can always boot from the SRM console using aboot to avoid your
system being tainted by Microsoft code ;-). But the ARC console + MILO is the
prefered method to boot Linux, if available.
