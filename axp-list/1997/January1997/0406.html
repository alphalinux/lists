>>> Andy Sparrow said:
>
> I purchased a UDB recently, and wish to set the resolution of the
> TGA to 1024x864 @ 60Hz, so I can use a fixed-frequency DEC workstation
> monitor (a VR-299 from a DECstation 5000/200, I have several of these).
>
> Two problems:
>
> i) I know I need to set jumpers W6-W3, and what the settings should
> be.
>
> However, there are two groups of four jumpers on a VX-40 board,
> one near the PCI riser, and the other waaay over by the SIMM
> sockets.
> Does anyone know for sure which is the right set (I think it's
> the one by the PCI riser) and which jumper is which, e.g. which
> way the labelling convention goes?
>

The appropriate four jumpers are in a "box" shape, just above the 8 VRAM
chips making up the TGA memory, and near the TGA (21030) itself. With the
back of the MB to your right, the values of the jumpers are:

8 o-o o-o 1
4 o-o o-o 2

Also, for 1024x864 @ 60Hz, the jumpers should be set to 7 (seven).

> ii) Errr, there's some mode the UDB jumps into, 640x480 with a
> blue screen. I think this is the SRM console mode(?).

Nope, ARC is blue (last), SRM is black (first)... :-)

SRM will use what the jumpers tell it; ARC will *always* force 640x480.

The LINUX kernel TGA driver will also *always* force 640x480, unless you
change the table it uses to program the video mode, in drivers/char/tga.c.
Try looking at the XFree86 3.2 release TGA server code to get an appropriate
set of values for the video mode you want. Then you'll simply need to change
the lines and columns numbers in arch/alpha/kernel/setup.c for the new screen
resolution, and make sure that the code in tga.c actually uses those values
instead of using hardwired 25 rows by 80 columns. Also, you'll be stuck with
an 8x16 (but 8x18 as rendered via tga_blitc()) font, which may be kinda small
at that resolution. And unless you change over to boot from SRM, you'll still
pass through ARC to boot MILO, and ARC can't be changed, and you won't be able
to see anything while in ARC mode.

>
> Once I've loaded the firmware etc, do I need this mode at all,
> or can I boot from the other mode all the time?

As mentioned above, you can setup the system to boot from SRM; see the info
about this in the FAQ at http://www.azstarnet.com/~axplinux/srm.html, IIRC.

Good luck!
