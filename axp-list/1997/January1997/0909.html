Hmm... it seems to me this is going in the wrong direction. MILO is
used for booting--that's all right? I think MILO is already a little
too big myself. But it is quite impressive that I can do directories
and such, and boot a kernel 'by hand'. So I suppose some simple
utils would be nice here, but it seems to me that it'd be turning
into it's own full blown 'shell' under a linux kernel. LILO seems to
be the opposite extreme. Remember which blocks on an accessable
drive the kernel is in, load it, and run it.. simple and effecient.

Anyways, just my $0.02..
