Ximenes <ximenes@netset.com> writes:

> There's a patch on www.linuxhq.com to fix a problem where swap is
> never freed...perhaps this is what you need? It's for the .30
> kernel, but it might apply to the .29 kernel as well.

Unfortunately that's not quite correct --- the patch fixes *reporting*
of free swap space, changing the way in which swap-cached pages are
accounted (pages which are both in memory and on swap). It might be
enough to account for some significant difference between reported "ps
-max" and "free" output, but it's not a memory leak and physical
memory is not affected by it.

It's still worth applying to see if it makes a difference, though.
