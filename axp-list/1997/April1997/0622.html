On Sun, 13 Apr 1997, Thomas Peter Burg wrote:

> Partitioning the disk worked fine, but I cannot create a filesystem
> (It says there was no linux-native partition on the device) and the machine
> seems to hang when it says "scanning hard disks ...". Could this be a problem
> of my Adaptec SCSI adapter?

I don't think so as I have the same setup as you (Miata 433a, Adaptec
2940UW, Millennium).

However, I did have some problems installing RedHat from CD (RH complained
"this is not a RedHat CD", probably because it was a self-made CD where
the RedHat directory was not where the install tool looked for it) so I
copied the relevant part (the RedHat directory in the alpha tree) onto my
NT partition. Using this kludge, even RedHat would install.

(btw: Can anyone tell RedHat that it's a shame they don't even give you
an opportunity to mount the CD yourself and tell it where the files are) :)

The only other "non-standard" thing I did was partitioning the HD using
the utility in AlphaBios... I just used linux fdisk to set the types
then. I don't think this matters, though.

> I`ve encountered similar problems when installing
> WinNT 4.0 before I installed some strange "CMD IDE ..." driver or something
> like that which was contained on the WinNT-HAL disk for my 433a.

Hmmm... never had any problems with NT. I however did install the CMD IDE
driver right from the beginning and I didn't use NT for anything but
running the X server supplied on the DEC CD and copying things onto the
HD.
