On Apr 15, Gabriel TAMURA wrote:

> Has anyone solved the problem with X in the AlphaStation 255 with S3
> Trio64 (2 Mb vram) and 32 MB (or 64MB, I have tested on both
> configurations) ?

I've seen some similar reports now for these AlphaStation 255 but I don't
have any realy idea and even worse I don't have such a machine for
reproducing and locating the real problem :-(

I'd really like to understand and fix this one but I need your help!

whoever has such a AlphaStation 255, problems with XFree86 3.2 S3 server
and a possiblity for network login (or serial terminal) to be able to
debug the Xserver while running, PLEASE do this:

please get the XFree86 3.2 server-only sources

ftp.XFree86.org:/pub/XFree86/3.2/source/X32servonly.tgz

(read README in the same directory about necessary changes to xc/config/cf/xf86site.def!!!).
you'll need to create xc/config/cf/host.def saying

#define BuildServersOnly YES

#define XF86S3Server YES

#define XF86MonoServer NO
#define XF86MonoDualServer NO
#define XF86I8514Server NO
#define XF86Mach8Server NO
#define XF86Mach32Server NO
#define XF86Mach64Server NO
#define XF86P9000Server NO
#define XF86AGXServer NO
#define XF86W32Server NO
#define XF86I128Server NO

#define XnestServer NO
#define XVirtualFramebufferServer NO

the easiest way to to get a server with limited (but all necessary) debuging stuff
is to compile a first S3 server using "make World". then do

cd ..../xc/programs/Xserver/hw/xfree86/accel/s3
rm *.o */*.o
make "CDEBUGFLAGS = -O -g3"
cd ../../../.. (or cd .../xc/programs/Xserver)
make XF86_S3 SUBDIRS=mi

then login remotely and start this server in gdb after setting some breakpoints:

cd ...../xc/programs/Xserver
gdb ./XF86_S3
b s3Probe
b s3Init
b s3Initialize
b mmio928_s3Initialize
b newmmio_s3Initialize
r

the 3 critical routines when starting X11 are in s3Probe, s3Init and s3Initialize.
so stepping through these routines (using 's' for step or 'n' for next)
should finally either crash your machine at a very specific point in code
(sync your disks before starting...) or the server will startup fine.

knowing where it crashes hopefully gives enough information to find/fix this problem.

any takers ??? *please*

Harald
