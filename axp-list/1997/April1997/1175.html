Try with:

MILO> boot sda3:vmlinux.gz root=/dev/sda3 init=/bin/sh

(with whatever is your root instead of sda3, of course).

Then the init scripts are never called and the only userland
process you have running is a lonely shell. You will probably have the
filesystems mounted readonly, so remember to do something like "mount
/dev/sda3 / -o remount,rw -n" to remount them rw. Just edit
/etc/sysconfig/network again, sync, back to milo and you're done.

>
> Just what is the correct way to deal with the problem of booting in two
> different locations, one location is on an network and the other
> location is not networked?

As long as you plan to use X-windows you are "networked", if only
on a loopback interface. Even more so if you think about a possible
use of ppp and/or slip (this includes diald).

To correct booting problems make sure that loopback is started before
any other connections, if any. If you do not have any ethernet cards
then simply do not start them and there will be no problems with
"eth0: TP hang". Why this series of "y"s with NETWORKING=no I have
no idea, and this should not happen, but I do not see many situations
when NETWORKING=no is a correct answer for a multiuser operation.

Michal
