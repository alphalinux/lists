On Fri, 25 Apr 1997, Tim Miller wrote:

> I have read (although now I can't remember where) that
> the Linux/Alpha kernel will run native Dig Unix binaries.
> Is this true? (I *beg* that it is). Anyone have any
> direct experience with Linux/Alpha and Digital binaries
> or, more specifically....with the DEC Fortran compiler
> and Linux/Alpha.

I've been experimenting around with Digital Unix binaries on axp/linux for
several nights now using kernel version 2.0.29 on a DEC PWS433a. I'm using
loader/libs from OSF/1 3.2A. Running OSF binaries doesn't seem to me to be
as easy as I first thought it would be. I've tried installing maple, but
it segfaults already during the installation process. I experienced the same
problem with khoros2.1 binaries for OSF/1. This seems to be due to an
incompatibility between several linux- and digital unix - kernel structures,
(please correct me if I'm telling something wron). I would be interested if
anybody else has made any better experience with DU-binaries under linux.
(exept running netscape, which seems to be quite stable).
