According to the systems architecture classes I had in college, above a certain
cache size there was a point where system performance would start to decrease.
It has to do with the looking up and accessing the wanted data in the cache.
When the cache is too big the time it takes to find the data in the cache is
longer than simply retrieving it from main memory. This point is dependent on
the cache architecture, type of cache and memory addressing modes, cpu design,
system bus design, ect. On Intel type designs there is very little performance
increase above 512k caches with <64 megs of main memory as the communications
overload starts to make itself felt. Second generation and later alpha CPU's
have a very big and efficient data and addressing pipeline which means that they
can effectively use very large caches. PentiumPros might benefit from bigger
caches, but I don't know enough about their architecture to tell.
