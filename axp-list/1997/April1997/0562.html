I'm trying to get linux running on a DEC PWS433a with a PowerStorm4D50T
and a cirrus logic 543x graphics adapter and an adaptec 2940UW SCSI adapter.
I have successfully installed miata-milo-2028 and now I'm trying to boot
the miata-kernel-2029.gz kernel found on gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels. I made a root disk from the image
fdisk_root_disk at gatekeeper.dec.com/pub/DEC/Linux-Alpha/ARCHIVES/BLADE_0.3/
ROOT_FLOPPIES. Then I tried to boot the kernel (after the MILO> prompt) with
boot /dev/fd0 root=/dev/fd0
I have a terminal connected to COM1. After unzipping the kernel there's no
output on the local screen anymore, but I can see the kernel messages
on my terminal. After the kernel initialisation I'm asked to provide a root
floppy. When I put in my root disk and hit return, everything seems to work
but there's still no output on my local screen and the output to com1 stops
after the message "VFS mounted root (ext2 filesystem) readonly". There seems
to be kind of a shell running since I can enter commands, and theres a curser
at the bottom of my local screen moving as I type and after hitting return
there's some disk activity. But except of the curser, I only have a blank
screen. Has anybody already experienced a similar problem or does anybody
know what I can do to fix it?
