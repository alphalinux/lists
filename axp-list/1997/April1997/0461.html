> >Alvin Starr wrote:
> >
> >Just gotta love C for its ability to do address calculations
> >
> >in "ip_fw.c" you will find
>
> [snip]
>
> >The problem is that the address calculation multiplies the ihl value by 8
> >due to the fact that it points to an "unsigned long". This calculation
> >should be infact
> > ((char *)ip)+(ip->ihl<<2)
>
> Is this only a problem on Alpha? What problems does this cause, bogus
> filtering? I'm setting up firewalling on 2.0.27 Intel box and the code
> is the same. Should I apply your patch to it?

The problem will only show itself on systems where sizeof(long unsigned)
!= 4. You should be safe on an intel platform.
The problem is that the person who wrote the code took advantage of the
fact that a unsigned long was 4 bytes in size. This is not true on all
platforms.
