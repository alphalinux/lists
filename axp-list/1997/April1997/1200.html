I have been using Linux on x86 for almost 2 years now, but yesterday I
started using Red Hat 4.1 with 2.0.30 kernel on an alpha. I appologize
in advance if this is a stupid question.

If I compile a file that includes <sys/resource.h> (which includes
<resourcebits.h>) I get tons of error messages of the form:

In file included from /usr/include/sys/resource.h:27,
from test.c:1:
/usr/include/resourcebits.h:31: `RLIMIT_CPU' undeclared here (not in a
function)
/usr/include/resourcebits.h:31: enumerator value for `_RLIMIT_CPU' not
integer constant
/usr/include/resourcebits.h:37: `RLIMIT_FSIZE' undeclared here (not in a
function)
/usr/include/resourcebits.h:37: enumerator value for `_RLIMIT_FSIZE' not
integer constant

etc...

With RPM I have determined that these files are part of the
glibc-devel-0.961212-3 package. It appears that something is mismatched
in the header/kernel include headers, but I can't figure it out quickly.
I have perused the axp-list archives to no avail so far. I believe that
I have Red Hat 4.1 with all updates loaded, as well as a 2.0.30 kernel.

Does this ring a bell with anyone?
