I am having the same problem and it has got drastically worse. My UDB
was setup as dual boot between NT and Linux and two weeks ago I
started using the NT extensively. I found that if I left the machine
on overnight it would crash with a PAL error. Like you I suspected a
heat problem and took measures to relieve it but it did not help and
finally I called the vendor who suggested possible SIMM problems but
I have good quality DEC memory supplied by the vendor and swapping out
pairs of SIMMS didn't help as the problem occured with any combination
of the available SIMMS. The vendor also suggested trying to repair NT
in case it was an NT problem as I had not been having any problems with
Linux.

After the NT repair option (which identified some problems)I was very
busy with real work but would turn my monitor periodically to verify
that NT was still up. Unfortunately I didn't try jiggling the mouse or
typing on the keyboard and the machine had in fact frozen as you
describe:-(

This Friday I rebooted into Linux intending to get some useful `work'
down and cursing MS for an unreliable operating system and of course
the mouse/keyboard lock up then happened in Linux. As I had had the
machine for several months without problems I was convinced that it
was some form of corruption caused by incompetence on my part and
decided to do a clean install of both operating systems and to reset
the UDB to its default state. Big mistake! The machine froze up twice
during reformatting of the system partition using arcinst.exe. Finally
I got past that problem and went to bed late regreting not having
waited to call the vendor on Monday. This morning I attempted to
install NT but the keyboard/mouse lockups started happening but I got
enough installed such that the machine will boot into NT but not far
enough that NT is fully configured. However the machine stability has
collapsed to the point that when I turn it on nothing happens - my
monitor (Nanao F2-17EX) indicates no signal from the UDB although the
(external) drive spins up and the green led next to the networking
ports on the back comes on.

I do not think this is a static problem ( i live in AL - an area
renowned for humidity although it has been dry lately) but will see
what the vendor says on Monday. Any and all suggestions welcome in the
interim.

Gary Spiers
