Jim> I assume that
Jim> this runs on the other platforms because v0*v1 will generate INF or NaN
Jim> while it generates an FPE on the alpha. There is a compiler switch to
Jim> control this:

Jim> -ieee
Jim> Ensure support of all portable features of the IEEE Standard for Binary
Jim> Floating-Point Arithmetic (ANSI/IEEE Std 754-1985), including the
Jim> treatment of denormalized numbers, NaNs, and infinities and the han-
Jim> dling of error cases. This flag also sets the _IEEE_FP C preprocessor
Jim> macro.

Jim> If your program must use IEEE signaling features that are not portable
Jim> across different IEEE implementations, see the ieee(3) reference page
Jim> for a discussion of how to access them under the Digital UNIX operating
Jim> system.

Jim> but the people from Digital told me not to use it unless I had to
Jim> because it slowed things down a lot. I don't think gcc has a flag like
Jim> this, but I dont know this for certain.

gcc supports -mieee and -mieee-strict when generating code for the
Alpha. The functionality corresponds to the relevant DEC Unix cc
flags. It's not recommended when you care about performance but it's
extremely useful for debugging (since with -mieee you'll see the
SIGFPE where it happens and not a dozen or so instructions later).

Jim> On a related note, does anyone know if glibc will compile under Digital
Jim> Unix? I would love to be able to make statically linked DU binaries that
Jim> I could distribute w/o having to worry about Digital Unix license
Jim> problems. Actually, if I knew where to get it I would just try to
Jim> compile it, so I am really asking for a pointer to the source.

GNU libc used to build on DEC Unix but I'm sure we (I) broke minor
bits and pieces when doing the Linux/Alpha port (we also fixed a thing
or two... ;-). It shouldn't be hard to fix whatever problems there
may be, but I seriously doubt glibc would compile out of the box as
is. Maybe it's best to give it a shot and see how far you get (be
sure to use GNU make and gcc, since both are required to build glibc).
