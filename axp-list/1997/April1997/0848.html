Hello Everyone;

I am writing this using Dad's account. Dad is having surgery on his
left shoulder on Monday morning, so I am working on taking care of
his e-mail.

I had previously posted a request for help in building the latest
current compiler, and all glibc libraries awhile back. Mr. Richard
Henderson provide an excellent reply which I have been working with.

This evening (Sunday 20th April 1997) while attempting to build the
gcc-2.7.2.1 compiler is when problems started.

I was able to build glibc-2.0.2 all right. It did fail two tests.
The one test failed was tst-strtol at test 20.
I do not remember the other test.
I did install the glibc using the command 'make install'.

After running the configure command I started to build the gcc compiler
in stages. The make failed on genattr. The program genattr fails with
an unaligned trap error, immediately followed by a segmentation fault.

I decided to investigate further so with the help of my sister I wrote
a small program to try. The ELF version fails in the same way, the static
version runs fine.

Below is the program.

main()
{
printf("hello world\n");
}

This program was built twice with the following commands.

gcc static -g3 -o pnr-static pnr.c
gcc -g3 -o pnr-elf pnr.c

I debugged both programs with gdb. The output of the debugging session is
below.

Based on this, I went back to the gcc-2.7.2.1 directory and ran genattr
in gdb. Genattr fails at the exact same spot.

Based on this I went to the povray directory where Dad and I have been working
on debugging POV-RAY. I attempted to build povray, and make complained that
zlib, and libpng conflicted with libc.so.6.1.
I went to rebuild zlib, and the example program in zlib failed in the exact
same way as the previous programs.

According to Dad the problem is in the loader code from glibc, since
statics run ok.

He also attempted to explain that the programs are failing even before the
are beginning to execute any actual program code. He did attempt to explain
about the loader code which 'wraps' each program, that went over my head.

My questions are:

Has anyone else seen this when installing binutils-2.7.0.9, glibc-2.0.2,
and gcc-2.7.2.1?

Is there a patch to fix this problem?
