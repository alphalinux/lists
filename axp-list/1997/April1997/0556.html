David A. Waltermire writes:
> More specifically I need to know how to partition disks and how the
> offsets work. I just tried to partition my HD as follows:
>
> Size Offset Fstype Size (MB)
> 1 4096 1 unused 2.0 (cyl. 0*- 1*)
> 2 131072 4897 swap 64.0 (cyl. 1*- 33*)
> 3 3971402 135169 ext2 1939.1 (cyl. 33*- 1019*)
>
> When I try to install 3.0.3 I get unaligned_trap errors. Is this
> related to the way I have it partitioned?

I don't think so, the table looks ok. If you are planning to install aboot,
maybe it's better to start the first partition at an offset x > 1, instead of
overwriting it. Here's my partition table for reference:

4 partitions:
# size offset fstype Size(Kb) Size(Mb)
1: 262144 10240 swap 131072 128.0 # (Cyl. 6*- 171*)
2: 262144 272384 ext2 131072 128.0 # (Cyl. 171*- 337*)
3: 1679360 534528 ext2 839680 820.0 # (Cyl. 337*- 1397*)
4: 1896592 2213888 ext2 948296 926.0 # (Cyl. 1397*- 2594*)

Offsets work like this:

offset of partition (y) = offset of part. (y - 1) + size of part. (y - 1)

Hope this helps,
