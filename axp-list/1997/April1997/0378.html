I too, am having kernel build problem. I have built many (x86) kernels in the past
but for some reason I can't get the right combination of header files, and libraries
to go from 2.0.27 to 2.1.27-32 alpha. I have spent many hour with this I didn't get a
reply to my post so I am assuming that either no one else is experiencing problem
or that no one else is trying to accomplish this.

However, I had no problem building a new kernel 2.0.27 from 2.0.27. The only thing
Here what I did,

1) Move to the /usr/src/linux directory.
Use one of the configurer makeconfig or xmenuconfig. I use the old method
make config ( this is the one I'm most familiar with )
2) Modify the Makefile. It builds the makefile with ARCH = x86.
I changed this to alpha.

After this the compile went smooth. It takes about an hour on a UDB with 24 megs
of memory. I been told that this is due to the gcc compiler not be optimized fo
alpha ( 64 bit ) and that I need more cache ( which I can't upgrade since I have
vx40a)

Otherwise I think it should go rather smooth once you change the ARCH line in
the Makefile.
