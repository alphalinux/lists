>>> Thomas Peter Burg said:
>
> I'm trying to install redhat-4.1 on a DEC PWS433a. I have AlphaBios5.25, a
> Matrox Millenium and an Adaptec 2940UW SCSI controller. I have come until the
> redhat-install tool asks me to select the installation partition. I have
> made a linux-native partition with the linux fdisk tool on /dev/sda2. (First
> is WinNT). Partitioning the disk worked fine, but I cannot create a filesystem
> (It says there was no linux-native partition on the device) and the machine
> seems to hang when it says "scanning hard disks ...". Could this be a problem
> of my Adaptec SCSI adapter? I`ve encountered similar problems when installing
> WinNT 4.0 before I installed some strange "CMD IDE ..." driver or something
> like that which was contained on the WinNT-HAL disk for my 433a.

Any chance you have an IDE CDROM? If so, be warned that the very latest
ramdisk.img is necessary for avoiding a bug in earlier install scripts.

Look on the Red Hat FTP site, in the pub/linux/redhat/devel/axp/images
for the ramdisk.img, dated March 10.
