Hello!

I would appreciate if those guys at RedHat listen to the following problems
and comment them shortly because they did *not* appear with RH4.0! It would
be nice to hear a bit more from them on this list anyway...

The problems occured on two installations (Jensen and PC164). On the Jensen
I compiled a 2.0.18 kernel while the RH-kernel was broken. I already used this
kernel on my own self-made Linux-System (a.out) without any problems. On the
PC164 we compiled a 2.0.27 from the delivered source code tree.

Now the problems:

- Jensen:
- when doing a strip on the 9.5MB xemacs-19.14 from gatekeeper the kernel
oops with a memory violation. After that the system is completely instable
(sync destroys filesystems, etc.). This error is reliably reproducable.
Is it possible that the filesystem cache gets being destroyed by anything
else outside the kernel???
- xinit always looses connection to the Xserver even if I use my own
XF_S3/XF86Config/xinit/xinitrc/xserverrc/startx from my old system.
Reliably reproducable, too.
- PC164/128MB mem:
- after starting large jobs (>50MB) within bash and suspension of them, they
have execution errors when resume them with 'fg'. If you do not suspend
them they run very fine. We noticed this problem because the jobs check
every anomaly by asserts.
reproducable.

I would be glad if anyone knows how to fix the problems.

Thanks,
Stephan
