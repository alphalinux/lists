I just got RH Alpha Linux 4.1 up and running on an Alphastation 500 that
is on loan from DEC. Thanks to an earlier suggestion from someone on this
list, it is working using a 2.0.29 milo image for an xlt board and the
xlt kernel. I am using a Diamond Stealth 64 PCI. I downloaded and
installed all the updates from RedHat. With those installed I was able to
run ypbind and get NIS working. If I run from the console or log in
remotely everything seems to work o.k. However, if I run X I get periodic
freezes. It almost seems as though I am running out of some kind of
resource. The system can't be pinged from remote systems, but I am able to
switch between virtual consoles and even get through the login/password
phase of a root login. I can then reboot with a ctrl-alt del. However, ths
system then hangs with various kinds of RPC errors. I'm not sure if this
is because the network is down, etc. I deleted the rc scripts for my
runlevel for things I don't need like gpm, http, news, smb, ... Should I
try a more recent kernel? Is it this particular video board? I have a ATI
Mach 64 that I could try if that would be better. Thanks for any input.

BTW, did I mention that it is fast :-) Very nice. I am running my
benchmarks at better than 3 times as fast on our old alphstation 200/233s,
even when compiling with gcc. I tried it under Digital Unix with the
DEC cc and it was a little faster (about 17%). I haven't tried compiling
and linking the static code under DU then running under linux.
