Hello,

I've got a 48Mb VX-40 UDB running RH4.0, 2.0.18 kernel. Last weekend
I tried to upgrade the 3.1.2 XF86 X-server to v3.2 using RPM's. I
downloaded
the rpm's from the RH ftp-site and started upgrading them. The first
problem was the architecture that rpm wouldn't accept. The answer to
this problem was using the -ignorearch switch. That worked. But XF86
had some dependancies that it needed. So I installed them:
glibc-0_961212-4_alpha.rpm, xinitrc-1_1-1_alpha.rpm and while I was
at it also amd-920824upl102-7_alpha.rpm and rpm-2_3_2-1_alpha.rpm.
Then things turned hairy. First I couldn't get past the "famous" eth0
hang.
I went in single user mode and deleted the "/etc/nsswitch.conf file.
This got the machine running past the eth0 hang. But now it didn't want to
jump into multi user mode. When I tried to log in the system kicked me
out, right after hitting the "enter" after entering the username (I didn't
get
to the password)... I wasn't able to get the system running again, and
decided to re-install (I needed the machine through the week....).

Is there a way to get my system on XF86 3.2 with using RPM's (in a nice
way)?? did I do something wrong / forget something during last weekends
attempt?? Or should I just accept that it isn't possible and do a manual
install (grab the tar.gz files from XF86.org and install manually)??

(Why do I want XF86 3.2?? My crap monitor only does 640*480@60Hz,
800*600@56Hz, 1024*768@I. XF86 3.1.2 only supports the 640*480
mode, and i'd like to use the 800*600 mode in 3.2.)

Thanks in advance...
