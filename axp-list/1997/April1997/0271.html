On 4 Apr, Jack Snedecor wrote:
> I've been searching for an answer to this one myself. It is a common
> question on this list, and I've not seen an good answer. The ARC
> console will boot either one, but I believe the disk boot blocks
> conflict, i.e., you can have only one set of boot code in the boot
> sector at a time. Perhaps someone with dual boot experence can
> elaborate on this?

If you boot Linux from ARC, then there should be no problem - install
NT, leave space for Linux, install Linux and let MILO be started from
the ARC menu.

> boot block and the other with a NT boot block. I haven't heard of
> anyone trying it.

I did this some time ago on a Jensen - two disks, one with NT and one
with Linux. Be sure to unplug the Linux disk during NT installation,
as NT will wipe the boot block. Works fine, only it's a bit annoying
that you have to switch between ARC and SRM each time.
