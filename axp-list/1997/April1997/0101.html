Thomas Mason writes:
>
> There was another thing i did not mention. ldconfig now
> dumps core on my system. I have no clue as to why this would
> happen.

I have see that when in an installation process for Red Hat 4.1 decided
to install one of '-devel' libraries without bothering to put on a disk
an underlying "regular" stuff. A resulting link "libxxx.so" to nowhere
made ldconfig very unhappy - core dump roughly half way through library
list when trying mmap a nonexistent file. I eventually traced that
down with a help of strace. Adding missing libraries, so the link was
no longer orphaned, resolved the situation. Smells like a bug in
ldconfig.

Please check if something similar does not happen to you.

BTW - adding MEMORY_SIZE=128 to a boot command was not what Jim
Paradis suggested. He talked about setting Milo environment. This
is not the same thing.
