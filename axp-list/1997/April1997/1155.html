On Mon, 28 Apr 1997, Adam Biskobing wrote:

> > > I am trying to compile Apache 2beta8 on my udb running redhat
> > > 4.0. I do not know all that much about C. It is erroring out on the
> > > following section:
> > >
> > > if ((shmid = shmget(shmkey, SCOREBOARD_SIZE, IPC_CREAT|SHM_R|SHM_W))
> > > == -1)
> > > {
> > > perror("shmget");
> > > fprintf(stderr, "httpd: Could not call shmget\n");
> > > exit(1);
> > > }
> > >
> > > It errors when it hits the SHM_R and SHM_W. I do not know what they are
> > > and they are not defined anywhere else in the distribution. I tried
> > > compiling it on an i386 linux system and it worked beautifully. Can
> > > anyone shed some light onto this problem or tell me what include file
> > > these are in?

The protection bits for semaphores are the same as the protection bits for
normal files. If you want owner readable/writable semaphores (as in the
example above) you can use S_IRUSR|S_IWUSR or even the octal number
0600 instead of SHM_R|SHM_W which seems not to be defined in our shm
include file.

If you are going to use S_IRUSR and S_IWUSR you need to include
<sys/stat.h>
