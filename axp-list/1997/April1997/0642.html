I wonder if anyone has seen any work being done for Linux to use a
"dynamic" swap of any useful kind? Soem OS's have implemeted this, to

Two comments: first, the disk access is still at most 5MB/sec for many
people, so if you put in 100MB swap, it takes at least 20 seconds to
actually use it (in other words if you truly use it, you are in trouble).

Secondly, it is possible to swap to a file, and turn on swapping on the fly:

dd if=/dev/zero of=/tmp/swapfile bs=1k count=5000 (create a 50MB file)
/sbin/mkswap /tmp/swapfile 5000
/sbin/swapon /tmp/swapfile

It is slower than swapping to a partition, but it's great in a pinch. I now
routinely set up 40MB swap regardless of the memory size (just to avoid getting
in a bind because 2MB of memory was missing), and rely on temp swap for short
term solution, and SIMM swap for long term :^)
