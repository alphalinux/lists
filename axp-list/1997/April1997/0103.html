On Thu, 27 Mar 1997, Steven Kokinos wrote:

> Hi All,
>
> we have a PC164 wth 128MB of RAM. We have had weird issues from day one,
> but this is what we have been told - Memory >64MB causes the machine not
> to reboot properly, kernel paging request errors, etc. My question is,
> can we specify somewhere on the command line for linux to not use all of
> that memory? I recall somewhere a way that could be done.
>
> We see strange disk problems, and generally unreliable behavior. I'd
> like to see if it has something to do with the memory situation.

I've just caught up with my axp-list reading (I was behind by
several hundred messages), so this response is a bit late, but....

Have you resolved this problem? We had a lot of instabiliby
problems on our EB164's. We solved them (with much help from Steve
Gaudet) by updating the SROM and firmware, and rolling back to a 2.0.27
kernel. This was on RH 4.0. I'm just today upgrading one of our systems
to 4.1.
