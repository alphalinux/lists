>>> Stephan Schroepfer said:
> After installation of RHL4.1 on a PC164-based machine 'date'
> reports 'year 2017' (the rest of the date is o.k., during
> installation even the year was right).

Sounds like you didn't tell the installation script that you were booting
from ARC console. Change /etc/sysconfig/clock to: CLOCKMODE="ARC".

>
> Beeing curious whats happening during startup I found a call
> to 'clock' somewhere in the /etc/rc.d/ files.
> Calling clock the first few times from the shell I was able to
> get the right date with some command line options I can't remeber.
> But now (after the first few calls) I only get a message
> like 'missing I/O port address'.

If it says: "clock: unable to get I/O port access", its probably because
you're not running as "root". If it says something else, check that the
output of "cat /proc/cpuinfo" has EB164 for the value of "system type".

If not, that could be a problem; let me know if such is the case. Also,
you could try the work-aound by doing:

ln -s EB164 /etc/alpha_systype

which should allow "clock" to do the right thing.

BTW, if booting from ARC console, do "clock -r -A" to read the hardware
TOY clock, and "clock -w -A" to write it.

Note also that the standard "clock" command has a problem with the clock
chip implementation on the PC164; look at the axp-list archives for a late
November posting by me about a fix for it, but don't try to use the "clock"
binary that's part of the posting unless you're running RH 4.0, as it won't
work (for some reason) under RH 4.1... :-(

Good luck.
