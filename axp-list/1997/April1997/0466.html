> With the state of the kernel source as it stands at 2.0.30, what *needs*

> patches still need to be applied, or have these been integrated into

I _just_ upgraded to 2.0.30 today (haven't rebooted to it yet, but...)
I applied no patches, just changed the Makefile's ARCH = to alpha, and
commented out SMP. It DOES help to have the latest glibc installed,
though. I'm running redhat's glibc 961212 rpm... that fixed a lot of
problems for me.

I also applied all the kernel patches for em86... no bad patches, no
compilation errors... sooo..... =shurg=
