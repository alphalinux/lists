Answers to your problems mostly have to do with the fact that you have:
a) Probably chosen the wrong video card, and
b) Don't have as many "colour bit planes" as X apps tend to expect to =
get.

If you're using the built in card, you should select the 'TGA' X server. =
(It=20
is there amongst the options).

Unfortunately, it is limited to 8 bit planes, which will result in =
somewhat=20
grainy results. If fvwm95-2 is allocating lots of colors to different =
things=20
at start-up, you may have nothing left for (say) xv to work with.

I ran "AfterStep" on my S3V (x86)-based system *once,* and found that =
the=20
"cool icons" chewed up the colour map completely, meaning that Netscape =
then=20
displayed in black and white.

You should expect that xv will be somewhat grainy on the UDB due to =
limited=20
numbers of colours.

There is a web page or two out there dealing with fvwm95 configuration; =
I'd=20
suggest doing an AltaVista search on that. I may have a reference on =
one of=20
my web pages; I don't much use fvwm95 and so haven't kept much info on =
that.
