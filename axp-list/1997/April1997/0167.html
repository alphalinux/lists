wrote:

>> Has anyone out there been able to get netatalk to compile successfully
>>under linux-axp? If so is there some place I can get the source, patches,
>>binaries? I have tried compiling versions 1.3.3 and 1.4b2, but always get a
>>bunch of warning messages about pointers being cast to ints of different
>>size, and while the resulting executable loads ok, it gives many memory
>>errors when I try to use it. I was thinking of going into the source and
>>changing the ints to longs to see if that works, but I figured I'd check to
>>see if someone else had fixed it first.
>>

if you haven't already done it, i can send you my patches. note: you don't
want to change the ints to longs. you want to change the unsigned longs to
u_int32_t's and the longs to int32_t's.

one warning though: my netatalk code is heavily patched right now as i've
been going through and trying to get things to work properly. in its
current state, my code uses bits of code that won't appear in the kernel
or glibc headers for a little bit as well as fixing the bitsize
assumptions. i think my changes are clean enough to compile without
problems on less hacked machines, but i can't state that with a certainty.
