> When I apply the patches for em86 (both em86-patch and traparg-2.0 with
> enabled Linux/Intel Elf config option) my system only recognizes
> 32 meg ram when I really have 128. Has anybody else out there seen
> this behavior? When I reverse the patches Linux recognizes the full 128.
>
> I can run em86 with the patches applied or not. I ran Netscape 3.01
> without a hitch.
>
> My system: PC164 500mhz, 1mb cache, 128meg ram, running RedHat 4.0,
> kernel 2.0.28, gcc 2.7.2, Adaptec 2940W.

Well, I have nearly the same configuration that you do and I have no
problems at all. The only difference between my configuration and
yours is that I'm using 2.0.29 + axp_diffs and my SCSI controller is
a Qlogic ISP. Neither of these should make a difference. I've tried
my patches on kernels as early as 2.0.18 and they work just fine. I see
all 128Mb of memory in my machine

Just for yuks, try explicitly telling MILO what the memory size is
and booting (i.e. MILO> setenv MEMORY_SIZE 128).

Also, try power-cycling the machine... MILO has been known to mis-size
memory across a warm reboot...
