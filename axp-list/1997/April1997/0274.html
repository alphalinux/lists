OK,
I got the 3.01 POV source and compiled and ran. I encounterd the SIGFPE
that I posted about earlier, and fixed it by changing line 328 of super.c
like this:

- if (v0 * v1 < 0.0)
+ if ((v0>0 && v1<0) || (v0<0 && v1>0))

I think that will be faster anyway.

It now runs fine and the rendered image looks good too. I assume that this
runs on the other platforms because v0*v1 will generate INF or NaN while it
generates an FPE on the alpha. There is a compiler switch to control this:

-ieee
Ensure support of all portable features of the IEEE Standard for Binary
Floating-Point Arithmetic (ANSI/IEEE Std 754-1985), including the
treatment of denormalized numbers, NaNs, and infinities and the han-
dling of error cases. This flag also sets the _IEEE_FP C preprocessor
macro.

If your program must use IEEE signaling features that are not portable
across different IEEE implementations, see the ieee(3) reference page
for a discussion of how to access them under the Digital UNIX operating
system.

but the people from Digital told me not to use it unless I had to because it
slowed things down a lot. I don't think gcc has a flag like this, but I dont
know this for certain.

On a related note, does anyone know if glibc will compile under Digital Unix?
I would love to be able to make statically linked DU binaries that I could
distribute w/o having to worry about Digital Unix license problems. Actually,
if I knew where to get it I would just try to compile it, so I am really
asking for a pointer to the source.
