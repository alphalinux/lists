I've been having a few problems making IP masquerading work on my
system. I've thoroughly read the IP masquerade HOW-TO, and implemented
it's instructions. I can set up the ethernet and ppp connections fine,
but ipfwadm breaks on the last necessary line in the HOW-TO. I enter:

ipfwadm -F -p deny

first, and it does this just fine returning me to the prompt. Then I do:

ipfwadm -F -a m -S 192.168.1.0/24 -D 0.0.0.0/0 (yes, I'm aware of
the potential security problem here...)

and the computer returns:

ipfwadm: setsockopt failed: Invalid argument

I'mnot sure whether this is an AXP-only problem, but I can't figure it out
in any case. I've even changed -a m to -a masquerade as shown later in
the HOW-TO and I get the same message. Is there something wrong with
ipfwadm for axp, or is it becoming blatently obvious to the members of the
list that I can't read a simple HOW-TO? Thanks in advance.
