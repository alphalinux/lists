Alpha-meisters,

A little while ago someone on the list mentioned having written
some scripts to automate the application of new updates to RedHat Linux. I
was thinking of developing something like that, but why rewrite the wheel,
eh? Has anyone else written such a tool? Could anyone who has package it
up (in .rpm or .tar.gz format), upload it to ftp.redhat.com, and announce
it's availablity here?

I think the inclusion of such a tool into rhl would add great
value to the distribution. In OS/2 there is an update tool that works
over the Internet, contacts one of IBM's servers and gets a list of
available updates. You then click on any of the updates you want, and it
downloads and applies them automatically. The only thing missing from
that tool is a database of the updates you've already applied so you'll
only see new updates listed.

I would like to suggest that RedHat host an update management tool
writing contest. This would motivate the user community to develop a new
tool that would _greatly_ increase the ease of administration of a RedHat
system. One of the specifications should be that it work on all the
architectures that RedHat provides a distribution for.
