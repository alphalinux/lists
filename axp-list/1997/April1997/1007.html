T. S. Horsnell wrote:
> If someone with the knowhow could write a sub-linker to take a DU COFF
> image which has been linked -call_shared and link it to the DU .a libraries,
> would that not enable us to run all sorts of .exe's on Linux/Alpha
> for which we only currently have the DU dynamic images? If I had the faintest
> idea where to start, I might even have a go myself...

I thought about doing this myself, but when I ran the idea by MadDog, he
told me this wouldn't solve the legal problem. Turns out that legally
you need a DU license to run even the library code in *static*
executables.
This isn't so much Digital's doing as it is due to the fact that some of
the library code is royalty code for which the *original* authors want
their due.

Up until now this hasn't been a problem because DU binaries could only
run
on DU platforms; therefore, if you were able to run the code AT ALL that
was proof that you'd bought the license. Now, of course, this is no
longer
the case.

On the other hand, if someone wanted to be REALLY adventurous, one could
hammer away at various GPL sources and come up with a "clean-room"
reimplementation of various of the DU shared libraries and distribute
THAT for Linux/Alpha 8-)
