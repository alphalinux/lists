On Apr 9, Ben Scott wrote:
>
> First of all, we've been building a monolithic kernel because we're under
> the impression that loadable modules don't work on an Alpha. Is this right?
>
Not quite. Correct modutils should be able to load modules. I use
rather old snapshot 970116, and it works. However, there are problems
with ksyms that prevent some modules from loading. Generally, to avoid
hassle, it is better to build a monolithic kernel. Sound module is
known to load OK even with insmod-bfd, so I think it is safe to build it
as a module (and get the advantage of being able to test different versions
without rebooting).

> Second: This computer has an NCR SCSI chipset of some sort; however, I'm not
> really sure which of the several NCR SCSI drivers in "make xconfig" to pick.

Look into /var/log/messages to see what your generic kernel detects.
AlphaStation 255 that I use has NCR 53c810 (driver NCR53c7,8xx).

>
> Third: We've been picking just "DEC Tulip" for Ethernet adapter, is that
> right? Because (when it gets this far), every time the Ethernet driver inits,
> we get this stuff:
>
[...]
>
> The hex numbers seem to be the same, always. Once that is done, the net
> generally works okay, but randomly spews this:
>
> > eth0: enabling AUI port.
> > eth0: enabling BNC port.
> > eth0: enabling unknown type port.
> > eth0: enabling 10baseT port.
>
> ... over and over again, sometimes messing up the net, sometimes not.
>

This is really annoying with that driver. I just chose DE4x5 instead of
Tulip driver (although the adapter is said to be DECchip Tulip)
and have had no problems since then.

> Lastly, does anyone know how to get the AlphaStation's built-in sound card
> to work with Linux? Is it even possible?
>

Quite possible. The standard driver supports it under the name MSS.
There is no hardware for things like FM synthesis or MPU-401 built in,
though. And, if you want to take full advantage of it (say, full duplex
capabilities), you need to patch 2.0.x kernel with my patches found at
ftp://genie.ucd.ie/pub/alpha/kernel/sound-* (or switch to 2.1.x).
