David,

The problem is the network card defaults to the 10baseT connection and takes
a little while to autosense over to 10Base2 or Thickwire. I had the same
problem as any network connections created before the autosense had
completed were inavlid.

The network card starts initialisation when the 'ifconfig' is issued and can
take up to 8 seconds to complete initialisation. To get around this problem
I added a little delay to the network startup to allow for the autosense to
complete before any services were started.

eg. Add

> sleep 10s

to /etc/rc.d/init.d/network

after the interface configuration.

Hope this helps
Jason.
