Hello,

I failed to set up the NIS client on my Alpha Linux box with RedHat 4.1.
It seemed that RedHat4.1 used NYS(NIS+) which had no need to run "ypbind"
for connecting to the NIS-server in the network. But only have to write some
configuration information in "nsswitch.conf" and "yp.conf" files (According to
the NIS-HOWTO). The following is the contents of my "nsswitch.conf" and "yp.conf".

1. Where was it wrong in these configuration files?
[In nsswitch.conf file]
hosts: file nis dns
passwd: files nis
group: files nis

[In yp.conf file]
domain netlab
ypserver graf.comm.eng.osaka-u.ac.jp

"netlab" is the NIS domainname in my network and "graf.comm.eng.osaka-u.ac.jp" is
the NIS server.

2. I may use the normal NIS instead of NYS but there is no "ypbind" in this
RedHat4.1/Alpha. Where can I find the axp-patch for the ypclient-package?
