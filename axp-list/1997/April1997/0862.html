Hi,

There are some 'incompabilities' between NFS on Linux (which goes by the specs)
and NFS on other platforms (which include 'performace-hacks'). There are two
ways to go by them;

1) Simple:
Mount the filesystems with
server:/path /path nfs rsize=8192,wsize=8192
This should speed up things.

2) The better way:
Run Linux on the Server.
We are happily running RH 4.1 on a Dec Alpha Server 2000 (4/200).

As the second option is not yet supported by redhat, I would go by the first.

Hope this helps,

Edwin

> I'm having trouble with NFS over FDDI.
> my config:
> NFS client: PC164 with 128Mb, 400MHz, DEFPA UTP FDDI., RHL4.1, kernel 2.0.27.
> NFS server DEC Alpha 2100 running DEC UNIX 4.0 (NFS3)
>
> I get reproducible NFS-read rates of ~ 1.2 Mb/sec, (not exactly
> wonderful, DU client to the same server achieves 3.5 Mb/sec)
> Whereas the NFS write-rates are abysmal. 0.04 Mb/sec between Linux
> client and DU server (Yes, 40 mins to write 100Mb), compared to
> 0.5 Mb/sec between DU client and DU server. I've taken care to ensure
> that system caches are flushed everywhere before each test.
>
> One really curious thing is that if I monitor the disk-traffic on the
> server for the appropriate disk (using iostat), I see disk rates which
> agree with the the NFS read and write rates for the DU client, and read rates
> for the Linux client, but for Linux-client writes, I see about 50 x more
> disk traffic than I would expect for the 0.04Mb/sec NFS rate.
>
> I also saw on the Linux client, klogd and syslogd with 100% cpu *each*
> (how you get this on a single-cpu machine I dont know..) but this went
> away after I restarted syslogd.
>
> 'netstat -i' logs no error on either client or server.
>
> One final curio: I takes me about 10 secs to logout on the Linux client
> when my home disk is NFS. I have no .logout file...
