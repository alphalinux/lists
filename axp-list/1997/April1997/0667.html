I had the same problem when I used minicom to talk to a remote system using
7-E-1 serial settings, and then forgot to tell Minicom to switch back to
8-N-1. I fixed it by entering minicom and telling it to configure the
serial port for 8-N-1, but there is probably a way to do it with stty:
something like

stty cs8 -parenb -parodd < /dev/ttyS0 # an example only; probably won't work

I fixed a similar problem -- ttyS0 was set up to 9600 baud on startup -- by
putting a big nasty stty line in my rc.local (right after the line that
turns down the noisy sound mixer line using rexima). If you'd like a copy,
mail me back (off the list!) and I'll get it for you (it's at home and I'm
at school right now.)
