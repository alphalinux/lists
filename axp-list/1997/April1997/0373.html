Here are the results running on an Alphastation 500/500 with a DU compiled
binary. Actually its a DU system, but I think this program spends all its
time in user space so that should not matter. I used the command:

time ./unix/povray +a0.3 +w640 +h480 +iKALEID5.POV +v

instead of +d +SP32 because I wanted to see the line counter at the bottom.
This is POV-Ray 3.01 with a few fixes to get rid of the SIGFPE problem.

Time For Parse: 0 hours 0 minutes 2.0 seconds (2 seconds)
Time For Trace: 1 hours 45 minutes 57.0 seconds (6357 seconds)
Total Time: 1 hours 45 minutes 59.0 seconds (6359 seconds)
6333.781u 0.627s 1:45:58.95 99.6% 0+37k 189+408io 5pf+0w

This is not too shabby! I think the difference between my run and yours is
the 8M cache on this machine. This means that POV and all its data should
fit inside the cache.

I profiled the program and looked at the areas where it was spending a lot
of time. It looks like things have been optimized in a way that might
(or might not) actually be worse for modern processors. I might play around
with seeing how fast I can make this run, but I dont know the source very
well so it might take a while. I will also recompile this with gcc tonight
and run it again so we can quantify the difference the compiler made.

Hope this is of interest.

Jim
