On Sun, 27 Apr 1997, SurferBoy wrote:

> Hello,
> Well I've figured out my Samba problem...it's not Samba it's
> all services, and not just when I connect to myself. I tried to
> telnet into my Linux box from a Win95 box (which I did all the time
> when I had 4.0 installed) and I get connected; but then nothing
> happens. For telnet the login doesn't run, for FTP the logon
> process works (I assume because it's the ftpd doing it) then nothing.
> I also noticed that my /var/log/messages contains the message
> from named sysquery: finds error (3) and the name of my machine (FQDN).

This sounds somewhat familar to a problem I have hit a time or
two... Have you ever tried letting the telnet connection sit there for a
long time (like 10-15mins), waiting for the prompt?
What I have found causes problems like this is when the machine
you are loging into (the linux box) can not resolve the IP address it
recieves by the client into a full hostname. From your later follow up
message it sounds like you have a DNS server on your local net. Are all of
the computers on your network (that communicate with the Linux box)
entered into the DNS database? That is, is there an entry for converting
the hostname of each machine to a IP adress AND an entry for converting
the IP address of each machine into a hostname?
What happens is the Linux box recieves a request for a telnet or
ftp session and the IP address of the client (your win95) box. The linux
box then takes that IP address and tries to determine the hostname that is
associated with that IP address, by querying the DNS server. If it can not
get back the associated hostname, it fails, and tries again. This will go
one for quite some time, until it finally gives up and possibly lets you
in.
This problem seems to come and go, and each Linux distrib has this
configured differently. Overall it is a rather tricky one to track down,
but I have seen the same thing happen as is happening to you, and this
seemed the only explaination. Upgrading to RH 4.1 is probably what caused
the change. I have never figured out how to disable this, only
I know it is possible. The simplest way to solve this is either to put a
full entry for each client in the DNS or put a complete /etc/hosts file on
the linux box.
Hope this is of help! Feel free to email me if I lost you anywhere
or you have any other questions.
