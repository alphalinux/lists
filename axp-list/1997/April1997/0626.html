Maurice Hilarius wrote:
>
> I wonder if anyone has seen any work being done for Linux to use a
> "dynamic" swap of any useful kind?

Yes, sure, on Linux you can see everything. :-) First of all
there is 'swapd' daemon available for quite some time. Check
usual places. Fairly recently there were also annoucements
with entries like this:

Begin3
Title: real-demand-swap
Version: 0.1
Entered-date: 02MAR97
Description: A patch to the demand swap daemon ond the kernel to
communicate via the kerneld IPC mechanism, making swapd useful
at last.
Keywords: demand swap memory
Author: Ian T Zimmerman <itz@rahul.net>
Maintained-by: Ian T Zimmerman <itz@rahul.net>
Primary-site: sunsite.unc.edu /pub/Linux/kernel/patches/misc
4395 real-demand-swap-0.1.tar.gz
565 real-demand-swap-0.1.lsm
Alternate-site: ftp.rahul.net /pub/itz
Platforms: swapd-1.3
Copying-policy: GPL
End

I am not sure what "making swapd useful at last" really means, as
I did not try either swapd or these patches. Also, as you can see
from a version number, these patches are "early beta" although currently
they are at 0.3, or something like that, if I remember correctly.
Anyway, if you are running out of swap - try this out.
