I will rewrite what he said (with own words):
We've got a PC164 from Vobis (a large and cheap computer discounter here in
Germany) for testing purposes for more than one week now. We deinstalled
Windows NT first (formatting the harddisk) and installed RH4.1 on it. It was
not easy do install it, but at the end we managed it by simply copying the
CD contents to harddisk, unplug the CDROM drive and install from HD.

However, standard RH4.1 for alphas is ***very*** buggy and we had to fix several
problems after installation. We installed it on an old Jensen, too, and the
system runs with the same problems, so they do not depend on EIDE/SCSI or
other system hardware.

Performance is quite good. Kernel compilation takes about 4 minutes and the
largest speech recognition system from our labratory has a speedup of up to 7
measured against SGI R4400/5000 systems and a Pentium 166 which is comparable
to R5000/180MHz.

As slightly mentioned above the system is equipped with EIDE harddisc and
CDROM drive which leads to installation problems of RH4.1 (it tries to run
fdisk on the CDROM drive and hangs after that).

Harald Koenig asked which memory type is used with that machine. Our system
uses 8 70ns standard PS/2 SIMMs with parity and thus runs with one 256bit
memory bus. It is as well possible to use only 4 SIMMs, but then the bus
will be reduced to 128bit.

Stephan
