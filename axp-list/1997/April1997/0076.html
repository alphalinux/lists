Hi,

On Wed, 2 Apr 1997, Dave Riesz wrote:

> When I try to compile Mesa (v2.2) on an Alpha, I get the following errors
> when the makefile gets to "readpix.c":
>
> (make linux-alpha)
> gcc -c -I../include -O2 -pedantic -L/usr/X11R6/lib readpix.c
> gcc: Internal compiler error: program cc1 got fatal signal 6
>
> (make linux-elf)
> gcc -c -I../include -O2 -ansi -pedantic -fPIC -funroll-loops -ffast-math
> -DSHM readpix.c
> gcc: Internal compiler error: program cc1 got fatal signal 6
>
> I know Mesa can be compiled on an Alpha, so I suspect something is wrong
> with my compiler or development libraries. Any suggestions where to look?

I had the same problem, and I think it's the optimizer of gcc that's at
fault here. I compiled this file by hand without optimizations, i.e.

gcc -c -I../include -pedantic -L/usr/X11R6/lib readpix.c

in the src directory worked fine for me.
