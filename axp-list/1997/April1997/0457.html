I was just helping someone, via email, install RedHat on a UDB that
has a VGA card in it -- you know, going through swapping which video
adapter the monitor is connected to when using the boot/root images --
and, shockingly enough, a thought occurred to me: "Hey, I've got two
nice display adapters in this thing (my UDB)... Is it possible to use
both at the same time?"

I know that when the TGA driver is in the kernel then that is the display
that is used instead of a VGA card/driver. Would it be possible to write
the TGA driver (if this is the part that would need to be rewritten)
so that you could use both the TGA and a VGA card at the same time?
I'm thinking that you should be able to at least assign one of the
vty's to the TGA card and be able to hot-key out of X to that monitor
and still have the X display running on the other. Possibly even run X
on the two different displays? (I know that they'd be different servers.)

Would this be possible, or am I just flapping in the wind here?
