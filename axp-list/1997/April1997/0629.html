At 09:57 AM 04-13-97 -0000, you wrote:
>> For Linux/x86 twice the memsize is recommended for swap. I believe that
>> this is generally assumed to be valid for axp as well, but I doubt if
>> anyone ever investigated this.
>
>Gee, so on my Intel box which has 192M of physical RAM, I should have
>384M of swap? And on my Alpha with 88M of RAM I should have 176M of swap?
>I don't think so.
>
>The problem with this sort of rule of thumb is that..

You are right of course. BUT, as you said, it's just a "rule of thumb"
Nobody else knows how much memory you are going to need for your purposes.
Thing is, if you allow lots, you are safe. If you run out, oh well,
increase your swap. No big deal (unless you run out in the middle of a 3
day process that needs more RAM, and you are in a hurry!) The thing is, it
depends what you are doing. If the machine is a generally unattended
server, which is mission critical, you BETTER have enough swap defined, or
some users will be having conversations with you... ;-}

I wonder if anyone has seen any work being done for Linux to use a
"dynamic" swap of any useful kind? Soem OS's have implemeted this, to
varying degrees of success. Downside to this is that if you use up too much
filespace, you may be running too little swap. Double-edged sword.
