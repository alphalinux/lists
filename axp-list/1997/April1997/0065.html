Well I have a Multia 166MHz with only 24MB of ram and 32MB swap and
I was able to run Netscape last night under em86. But it was very slow,
which is understandable 8-). I mean, netscape is even slow under
Linux/Intel. One thing to note though is that I did not install
netscape in that sence. I just copied the executable from a Linux/Intel
machine and ran that. So all nls information (??) and X-appdefault (??)
were not installed on the Alpha system. I did not notice though how
much ram/swap it was using, but all the CPU was used while downloading,
Specially when loading www-pages with lot of pictures, and of course
when netscape was starting up.

I like the idea (no longar an idea 8-)) of getting the Digital 80x86
emulation environment ported to Linux/Alpha. I think it is the second
best move since Linux/Alpha and will help many undiceded souls to make
up there mind about going Linux/Alpha. Altough em86 is not fast it
atleast gives the owner of an Alpha system the chance to run binary
only programs from the a Linux/Intel system. On a decent machine the
performance might even be good (Comment from somebody running some of
