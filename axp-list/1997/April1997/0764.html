>Since so many codes were written with this assumption, one would assume
>that some body would come up with a C compiler that forces 32bit
>compitablity you know like a compiler flag or something. Sure, would save
>me a lot of time fixing idoitic bugs. All I really care is something
>that works better than emulations. Does it exists? Are there any reason why
>not?

Well, I don't know about gcc, but DEC C for VMS on Alpha will generate the
appropriate fetch-shift-mask combinations to get it with aligned accesses if it
knows that something isn't aligned. If it doesn't know, then it will assume it
is aligned and the (slower) exception handler will fix up the exceptions.

There is a __unaligned delaration you can use, even.

There are also 'fetch-long' instructions, but I believe that they use a full
bus cycle to do it.
