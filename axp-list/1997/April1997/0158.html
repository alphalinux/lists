Hello;

I am typing this with one hand, so please forgive any typo's.

The include file timespecs.h was introduced by the axp-kernel patch.
There is really no longer any need for this patch on the later Linux kernels.

The choices are either apply the axp-kernel patch to the 2.0.29 kernel
source, or work through the sound source files and remove dependencies
on this kernel patch.

I would suggest that we coordinate a plan of attack to rid ourselves
of having to rely on this axp-kernel patch, and phase it out.
This will require changing source code that has come to rely on the
patch being there, but if we do not do it sooner it will just keep getting
worse and more difficult later.
