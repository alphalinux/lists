On Wed, 23 Apr 1997, T. S. Horsnell wrote:
> If someone with the knowhow could write a sub-linker to take a DU COFF
> image which has been linked -call_shared and link it to the DU .a libraries,
> would that not enable us to run all sorts of .exe's on Linux/Alpha
> for which we only currently have the DU dynamic images?

Well, while that might be possible, it wouldn't get around the license
issue. You'd still be using program code written by Digital on a machine you
don't have a license for. The fact that the code in question has been copied
out of a shared library file into an executable program file doesn't change
that.
