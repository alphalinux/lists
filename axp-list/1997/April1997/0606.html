1. I have an NT-version DEC PC164/21164 running RH4.1. Is it possible
to run DEC Unix on this? I would like to make sure the system works
before parting with money for a license but:
a) I cant find the SRM anywhere on gatekeeper
b) I've been told that the NT processor chips have fusible-links
blown in them which makes it impossible to run DEC Unix
Has anyone suceeded in running DU on an NT PC164/21164?

2. I want to build a small farm of such machines to act as a compute server.
We have a DEC 2100 running DU, and the idea is to use precompiled
binaries from this machine and run them on the Linux farm. So far
this works - 3 out of 3 jobs run, so long as they are linked -non_share
on the DU. I'm sort of surprised that it was so easy. Am I missing
something? Does anyone know of a utility that will convert dynamic
executables to static ones without needing the original source
code or object code? There is a distinct speed advantage (I've
seen 25% speedups) in building -non_shared from scratch but in
many cases I'd be prepared to sacrifice that. (I'm told the slowdown
in -shared is partly due to the use of position-independant code.)

3. I've seen references to UDB boards with 2Mb cache. (The PC164 only
has 1Mb). Can someone tell me more? So far, my 3 test jobs show
a 2 to 3 speedup over the 2100 (433MHz EV5 for my 21164 as against
275MHz EV4 for the 2100) but I realise cache sizes can be a significant
factor in this and would like as big a 3rd-level cache as possible..

Thanks

--Terry.
