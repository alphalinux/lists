We are running a PC164 successfully with RH4.1. We originally had problems
similar to yours, though not exactly.

We did something which fixed it. What EXACTLY we did I don't know, but
what I DO know is that RH4.1, Milo 2.0.27 AND a kernel 2.0.27 [compiled
from the 2.0.27 source after applying axp-diffs-2.0.21] works satisfactorily
for us.

We are using a Microway PC164 with 2M cache, 128M RAM, Seagate EIDE disk,
DEC Ethernet card, ATAPI CDROM, Diamond Stealth 64 VRAM video board.

Regards,

PeterM

I've just got RH4.1 going on a DEC PC164. MILO wont see more than 32Mb
of my 128Mb mem, so I use 'setenv MEMORY_SIZE 128' to force it (until
I can find the fix), but I then got similar random crashes during
kernel rebuilds (but a noddy to access all memory worked OK...).
However, these crashes went away when I used 127 instead of 128
in the setenv command.
