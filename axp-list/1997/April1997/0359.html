> OK,
> I got the 3.01 POV source and compiled and ran. I encounterd the SIGFPE
> that I posted about earlier, and fixed it by changing line 328 of super.c
> like this:
>
> - if (v0 * v1 < 0.0)
> + if ((v0>0 && v1<0) || (v0<0 && v1>0))
>

>
> It now runs fine and the rendered image looks good too. I assume that this

Running POV version 3.00b on both platforms yields similiar results, and
does work on both platforms.

Jim's fix above does work, but I think that we are fixing the symptom and not
the cause.
