Hello

>Why are all Alpha clone makers advertising with misleading benchmarks?
>
>For example, they say their 500 MHz systems have SPECfp95 of 20.4. That
>is not true. They must be aware of the fact that this score came from
>DEC's Alphastation with a different motherboard and 8 Mb of L3 s-cache!
>It's base price is $26,000.
>
>When I inquired about the discrepancy, almost everyone answered that
>these are the scores which were provided by Digital. Not true, again,
>Digital Semiconductor said the PC164/500MHz board/cpu configuration is
>capable of 13.7 SPECfp95 at the most. They also deny having provided
>clone makers with these benchmarks as relating to the PC164 motherboard.
>
>Would anyone like to offer any answers?

I will. Digital Semiconductor gave me along with any other OEM a set of
manuals that state the SPEC numbers on the PC164 motherboard, not an
AlphaStation. If you have a fax I'd be more than happy to fax it to you and
you can read what Digital put out, not us. I can only fax it to you so you
will see the numbers we post are in fact from Digital not resellers.

Furthermore, you should be aware when a vendor such as Digital and Intel
derive SPEC numbers its from an optimised OS and complier to achieve high
numbers. Unless your a ISV you'll never see the numbers they post.

Anyway here's what we have found from the net to be the TRUE SPEC nummbers
of the PC164 motherboard.

++++++++++++++++++++++++++++++++++++++++++++++++
=I did some research with our chips and performance folks and here is
=the performance story about the PC164 board. People have complained about
the ="marketing" numbers they use (i.e. chip numbers for a machine with very
fast
=and large cache, not what they have). So here are measured numbers
=for the 500 Mhz variant, I have included linear downscaled (i.e.
=pessimistic) numbers for the 433 and 366Mhz variants (will replace
=those with the probably better measured numbers once I get them).
=
=The cache is small, but pretty fast (Timing for the 500 Mhz one is 22/16/16/16\
,
=hence cache read bandwidth is 872 MByte/sec.). The 1MB size doesn't hurt
=SPECint95 as badly as it does hurt the SPECfp95 numbers:
=
= SPECint95 SPECfp95
=PC164/1MB L3 14.6 13.9
=PC164/2MB L3 14.9 16.0
=AlphaStation 500/500 15.0 20.4
=
=The AlphaStation has 8 Mbyte cache. So, you loose about 3% in SPECint95,
=but about 32% in SPECfp95 with the smaller caches. The cache RAMs are
=usually soldered in, it is hence very difficult to upgrade that.
=
=So the (measured for the 500 Mhz, linear clock-downscaled, pessimistic
=for the 433 and 366) results are:
=
= SPECint95 SPECfp95
=1MB/500 14.6 13.9
=1MB/433 12.6 12.0
=1MB/366 10.7 10.2
=Dell PentiumPro 200 8.20 6.21
=IBM 43P PPC604e 2002 7.22 6.91
=
=The system chip set and memory system is otherwise identical to the
=AlphaStation 500 (i.e. the same 21172 chip set with the same parameters).
=Memory bandwidth is (virtually independent of CPU Mhz) as measured by
=STREAM:
=
= Machine ID ncpus COPY SCALE ADD TRIAD
= ------------------------------------------------------------------
= PC164/21172 1 166.4 166.4 177.5 177.5
=
=People usually quote the COPY number as the STREAM bandwidth.
=
=So, the performance of these boxes for integer code *with the same
=characteristics as SPECint95* should be nearly indistinguishable
=from an AlphaStation 500/500. For floating point you need to
=know the memory footprint of your data. If it fits in 1 Mbyte,
=great. If it fits in 8 Mbyte but not in 1 Mbyte, you loose at
=least 32 percent. If it doesn't fit in 8 Mbyte then performance
=should again be nearly identical. I will try to get the individual
=SPEC95 results so that you can compare for yourself.
=
=Since Mac users seem to have some applications that relate well with
=Bytemarks:
=
=Bytemark Integer Index Bytemark FP Index
=PC164/1MB/500 8.41 8.35
=PC164/1MB/433 7.29 7.24
=PC164/1MB/366 6.16 6.12

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

Now until someone comes along and runs these test themselves we really have
to go by what we told from Digital and Intel.
