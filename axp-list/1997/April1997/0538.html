Random patching in a random order what not a list of hints, as far as
I recall.

> 1. I grabed linux-2.0.30.tar.gz from ftp.cs.helsinki.fi and applied the
> axp diff from 2.0.21.

You had to end up with rejects. It means, in particular, that one
header was patched (include/linux/fs.h), but a corresponding patches
to a code were rejected.

> When I tried compiling it I got the same error that
> other people got.

This is not surprising if you did not resolve collisions "by hand".

>
> 2. Next, using the same linux-2.0.30.tar.gz, I applied then axp diff from
> 2.0.21, and then applied a patch that someone on the list sent out.

That "someone" probably would be me. These patches were clearly
described as NOT for that situation. You were bound to get rejects
once again.

> It puked the compile at the same spot.

See above.

>
> 4. I don't know why I thought this might work, but here's what I tried
> next. I had the linux-2.0.28.tar.gz. I untarted that, applied the patch to
> bring it up to linux 2.0.29. Then I applied the axp diff patch.

There were no rejects this time, as the stuff which collides was
modified between 2.0.29 and 2.0.30

> Then I applied the patch to 2.0.30,

You got rejects once again. Posted patches were exactly for this
situation (current 2.0.29-axp, patched to 2.0.30 and some leftovers had
to be fixed). BTW - it is not hard to see from a posted patch contents
how modified code should look like if you had chosen a route like in
your attempt 1.

> This time it worked!

It works without resolving remaining conflicts because, as luck has it,
what needs to be done is a removal of few variables which are not used
anymore in a patched code of block_dev.c and changes to a code miracuosly
survived. So, if you left things alone, you got a few "dead variables"
in a kernel code and maybe some warnings from a compiler. Other time,
when a "real code" will be involved, your luck may not hold.

Moral: random trials are not a substitute for paying attention to
'patch' warnings and error messages (and acting on a received
information).
