Hello all,
I have been looking at POV-Ray some more, specifically the speed
differences between compiling with the Digital Unix compiler and gcc.
I have been playing around with a utility called om which is a link time
optimizer. Here are my runtimes so far using the following command
line:

time ./unix/povray +a0.3 +w640 +h480 +iKALEID5.POV +v

Compiler Seconds User Time
cc 6333.781
gcc 7210.081
cc+om 5629.090
gcc+om 6122.983

The thing that interests me is that gcc+om is faster than the Digital Unix
cc command. From a practical standpoint, the number of people with the
knowledge, desire, and time to optimize gcc is very small. I think its
about 2 (I am not one of them). It also probably involves a lot of
unexciting and difficult work. On the other hand, I would expect writing
something like om to be much more interesting and Digital has published
papers describing how it works:

http://www.research.digital.com/wrl/techreports/abstracts/92.6.html
http://www.research.digital.com/wrl/publications/abstracts/86.3.html
http://www.research.digital.com/wrl/publications/abstracts/89.17.html
http://www.research.digital.com/wrl/techreports/abstracts/94.1.html

I mention this because it would be great if someone looking for something
to do wanted to work on this. It would probably also be useful to the
people doing the Ultrasparc port. In any event, I wanted to let people
know that om did work well with gcc generated code.

Thanks,

Jim
