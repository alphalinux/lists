>>> "Eric C. Frey" said:
> I have an Alphastation 500 that I am trying to install Linux in. It has
> a Diamond Stealth64 PCI video. It seems to be running ARC, and says
> something about Alpha Firmware 4.49. I made the milo boot disks using both
> ALCOR.IMG and EB164.IMG images from the cdrom in the /MILO/IMAGES
> directory. I set things up too boot according to the instructions in
> the Linux Users Guide. When I boot, linload loads, then it says something
> about loading milo, then it says it is switching to the PALCODE and
> everything stops. Any suggestions? Are the MILO images in these images
> files appropriate for a VGA card such as the diamond? The system is on
> loan from DEC and came with some fancy 4D60 adapter. I originally tried
> with it and got the same result. Thanks.

For MILO, please try using from gatekeeper.dec.com:

pub/DEC/Linux-Alpha/Kernels/milo-2029-xlt

or the similarly named image file.

For a kernel, please use the Alpha-XLT kernel (ie xlt.gz) from the CDROM.

Apparently, the newer AS500's look (hardware-wise) more like an XLT than
an ALCOR.

It all ought to work fine with the VGA instead of the 4D60 (what's that got
on board, 32Mb VRAM or something? Whatever, its useless under LINUX... :-)

Good luck.
