On Mon, 7 Apr 1997, Jonah Giacalone wrote:

> Could someone explain to me how a 233mhz 64bit system compares with only
> a P90? Aren't they both PCI? Where are all those Hz going? Is it a
> cisc/risc thing? What would allow the UDB to reach more of it's
> potential? More cache? If this is really the case, why would anyone want
> to go with the Alpha other than for a reasonably asy port to faster
> Alpha platforms?

It's the UDB hardware that usually limits it... and the 21066 processor.
Yes... Alphas scream... if you have the super-pipelined and optimized
21164's. ;) The UDB contains 1st generation Alpha chips. the 21164 is
_3rd_ generation alpha. It's basically the difference between a 486 and a
pentium. Case closed. :)

Also... gcc for linux AXP is NOT as efficient as DUNIX's cc... which also
leads to slowdown (but without the $1000's pricetag). gcc produces code
about 50-85% the speed of DUNIX's cc compiler.
