On Tue, 1 Apr 1997 provost@iap.fr wrote:

>
> 1) Is there g77 software for Alpha ? Because fort77 (which is in
> the reality f2c) has very bad performance.

Unfortunately, no, not yet. Although the developers are working on the
port, there are many issues still unresolved. Maybe in a few months.

>
> 2) Some messages are displayed on my console screen :
>
> compress[22414]: unaligned trap at 00000000120003a38: 0000000.....
>

The Alpha has to have data align on word boundaries, while most Intels
have no alignment requirements whatsoever. This message appears when the
Linux kernel traps an unaligned access (usually a legacy from ix86 code).
The traps do no harm, although they cause a register flush, and therefore
a large number of wait-states (in other words, they're a real performance
killer). The various developers and the guys at RedHat are slowly stamping
out the causes of these traps, with the result that I see only about 10%
of the number I used to.
