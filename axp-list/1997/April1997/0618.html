Hi, all,

I've scanned the last four months of axp-list archives and have seen
no references to the problems I've been having.

Ever since I upgraded my UDB Alphas to Vanderbilt and kernel 2.0.27, I've
been having trouble with my SCSI CD-ROM and tape drives.

When I try to mount the CD with 'mount /dev/scd0 -t iso9660 /mnt/cdrom',
I get:

mount: block device /dev/scd0 is write-protected, mounting read-only
mount: wrong fs type, bad option, bad superblock on /dev/scd0,
or too many mounted file systems

I know the CD is ok, because I've mounted it before the upgrade.

When I try a 'tar -tf /dev/st0' for my tape drive, I get:

tar: Cannot open /dev/st0: Operation not supported by device

Both are recognized upon boot according to 'dmesg'.

Both of these commands worked with RH 4.0 and kernel 2.0.14.

Has anything changed significantly? I noticed mount is now in its
own rpm...
