I thought I would upgrade to the 2.0.30 kernel now that there is a set of
axp-diffs for it. However, I found (much to my chagrin) that (once again)
the headers/includes don't match up with something. When I try to compile
pcmcia-cs-2.8.23, I get scads of "_(function name) redefined" messages and
the make fails. As I recall, there was a band-aid fix for this problem
(something about fixing k_compat.h or some such), but I've since lost what
it was. The stock RH4.1 kernel RPM and the stock pcmcia-cs-2.8.23 work
relatively well together (there were only two errors, Xircom and one of the
ethernet cards, neither of which I use).

Three questions:
(1) Is this the same libc/glibc mismatch that plagued RH4.0?
(2) Has anybody re-compiled pcmcia to work with 2.0.30 yet?
(3) What fixes, patches, include file pointers do I need to adjust to be
able to re-make pcmcia with the "standard" (source+axp-diffs+em86) kernel
sources?
