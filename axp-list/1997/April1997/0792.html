I'm having some trouble getting a 3c503/16 card working with my
alpha (more complete system information below).

Initially, networking appears to function fine: I can telnet/ftp/ssh
in and out of the machine without any noticable problems. However,
whenever I attempt to transfer large files, the machine locks up.
Once it gets itself wedged, it won't respond to anything short of
a power cycle.

Interestingly, the amount of data that will successfully transfer
prior to the machine locking up appears to be fairly consistant in
transfers with the same host, but appears to be vary from host to host.
It does not appear to vary according to network latency or machine
type (i.e., I get different results in transfers with similarly
configured SPARC 10s sitting in the same machine room on the same
subnet). The problem seldom manifests itself before about 30 Kb have
been transferred, or after more than about 300 Kb have been transferred.
I've successfully transfered files of ~10 Mb, consistantly, with
a single host.

I've tried this with several 3c503 cards---several 3c503/16s and
one eight bit card. I've tried jumpering all the I/O base addresses,
and all the memory addresses (including none), and a considerable
number of the possible permutations of the two. I've also tried
the cards in both of the ISA slots in the motherboard. None of this
seems to affect the behaviour.

The system is a PC164 with a 21164a 500MHz, and it's running Red
Hat 4.1, with a 2.0.27 kernel.

I've tried compiling support for the 3c503 into the kernel using
the source off the Red Hat 4.1 media, and I've also tried it with
the 2.0.27-6 kernel RPMs. I've also tried installing the following
RPMs:

NetKit-B-0.09-1.alpha.rpm
glibc-0.961212-4.alpha.rpm
initscripts-2.91-1.alpha.rpm
netcfg-2.15-1.alpha.rpm
python-1.4-4.alpha.rpm
pythonlib-1.16-1.alpha.rpm

...in addition to the versions on the Red Hat CD. I've tried
configuring it by hand, and using the netcfg GUI.

The machine has 32 Mb physical memory, and 65 Mb of swap. The system
disk is an IBM 2 gig IDE drive.

So: Anyone have any ideas or suggestions[1]?

Yours etc.,

SubGenius

1 Apart from buying a different network card, that is.
