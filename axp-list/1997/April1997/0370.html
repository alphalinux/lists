Hi,

I had the problem of loading MILO on my AlphaStation 500 with
one 2-GByte-SCSI-HardDisk, one SCSI-CDROM, one 10-Based-T-Ethernet-Adapter
and one SCSI-Controller(QLogic ISP1020). I used Red Hat Linux/Alpha 4.1
and booted MILO by the \MILO\Alcor\milo image.

When I boot MILO, I get the following error messages:
"dev_config: don't know to set hostid of SCSI device (1077:1020) " and
"NVRAM is not supported on this system". Then, there are some initialization
messages of MILO following with the MILO prompt. I can use some commands such as
"set" or "help", but when I try to use the "ls" commamd to access the CDROM,
I get the following messages and after that the system is hanged up.
" Ramdisk driver initialized: 16 ramdisks of 4096K size.
Floppy drive(s): fd0 is 2.88M.
floppy0: np floppy controllers found.
qlogicisp: now isp 1020 revision ID(2).
scsi0: QLogic ISP1020 SCSI on PCI bus 0 device 9 irq 28 base 0xb000.
scsi: 1 host. "

Does someone knows what the problem is and how to fix it?
Thanks in advance for any suggestions.
