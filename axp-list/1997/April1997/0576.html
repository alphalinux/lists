I have a 2940UW, which is auto-terminating, and defaults to ID7 hooked into
a seagate UW which is not terminated and is on ID4, which is followed by
another UW seagate at the end of my chain, which is teminated and set as
ID0. As for a cdrom, I don't have one in this system.

Could the hardware-savvy people explain how does the auto-termination
work? It can't possibly replace the termination at the far end of the
cable with attached devices??? I thought that auto-termination only
dispenses the need to attach the terminator to the external connector
if it is not being used (or internal, if external only is used), but can't
replace the second terminator----unless this something very fancy
with active reflection cancelling (which I don't think it is).

Moreover, would active termination work if you have wide bus with
narrow devices? What if you terminated the narrow part only--would it
autoterminate half of the bus?

The relevance to axp-list: many problems people seem to be having
result from SCSI configuration problems. Is there a lucid explanation
available somewhere?
