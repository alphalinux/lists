> I am trying to build some fairly large Fortran programs with f2c/gcc on
> a UDB running RedHat 4.1. The code seems to compile ok, but when I do
[...]
> zmatrx.c(.text+0xa918): relocation truncated to fit: LITERAL iip.533

That is a limitation of the version of the linker that RH4.1 has.
There are two solutions:

(1) Break your program up into multiple shared libraries. This is
most convienient if you are already using static libraries to
partition your code, but in reality it can even be done arbitrarily.

(2) Try a new version of the linker. A few days ago I announced on
axp-list binaries of binutils, glibc and gcc, all of which must be
upgraded simultaneously. In addition, you must recompile all of
your existing object files.
