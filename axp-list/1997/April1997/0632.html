> > For Linux/x86 twice the memsize is recommended for swap. I believe that
> > this is generally assumed to be valid for axp as well, but I doubt if
> > anyone ever investigated this.
>
> Gee, so on my Intel box which has 192M of physical RAM, I should have
> 384M of swap? And on my Alpha with 88M of RAM I should have 176M of swap?
> I don't think so.
>
> The problem with this sort of rule of thumb is that it has no objective
> basis. The fact that I have x megabytes of physical memory doesn't in any
> way suggest that I need 3x megabytes of virtual.

On the contrary, there is an objective basis. Just not for Linux.

BSD variants have long required that swap be at least twice the size of
physical RAM. This allowed kernel memory allocation code to make certain
assumptions (that I don't recall).

At one time, the Linux kernel may have laboured under similar assumptions, but
that's definitely not been true for several years; swap>2*real was not
required when I first installed 1.0.9 back in 1994. At that time, the
limitation I had was that I couldn't have swap partitions *larger* than
2*real, because I had 8MB of RAM, and Linux only supported 16MB swap
partitions. The limits have, of course, grown since then.
