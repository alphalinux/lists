The version of glibc with RH4.1 _requires_ that the kernel headers
that it sees have the axp-diffs found on ftp.azstarnet.com applied.o
Later versions of glibc do not have this restriction.

I don't know that there exists a version of the axp-diffs for 2.0.30,
but there is no particular reason you should have to replace the
2.0.27 kernel headers that Red Hat distributed. The kernel is quite
happy being compiled somewhere other than /usr/src/linux (say,
/usr/src/linux-2.0.30? what an idea.) and if you leave libc's
headers alone you'll be fine.
