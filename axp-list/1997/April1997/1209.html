Hiya!

I'm having Linux running now pretty stable on my Miata (PWS 433a), and
overall it works quite well. However, there are still some quirks which
bother me a bit.

First: My syslog is full of "unaligned traps". According to
grep "unaligned trap" /var/log/messages | wc, I have had now way over
20'000 unaligned trap errors during the last two days. The total uptime
during these days was about ten hours.

/proc/cpuinfo tells me there were even 832165 "kernel unaligned acc" and
1138 "user unaligned acc" during an uptime of 3:19. All I was doing was
reading news and using my Miata as a 433 MHz x-terminal to use the
Linux/i386 version of maple :)

I know that "unaligned" accesses can happen from time to time, especially
beacuse Alphas are not very wide spread among the Linux developers, but the
number I have seems to be rather high.

Second: The clock runs quite fast -- after one "real" hour, the system
time has advanced about one hour and ten minutes. The CMOS time reported
by clock -Au -r is correct, however.

I knew I've got a fast machine, but I thought the clock would make an
exception to this :-)

Third: The driver for the parallel printer port seems to be quite broken.
It seems that some characters get lost, since the printer (an old HP
Deskjet 510 which has been running faithfully during the last few years on
an Amiga 500, an Amiga 2000 and a Pentium System) randomly "throws in"
some text characters and sometimes even form feeds while printing
graphics. The errors are different every time, and since a PCL file which
works flawlessly when I "copy file lpt1:" under NT produces "random"
errors when I "cat file >/dev/lp0" under Linux, I think the driver is
to blame. Nothing special, just normal "parallel printer support" in the
Linux kernel.

The first and the second problem occur whether I use
miata-kernel-2029-complete.gz from gatekeeper.dec.com or a self-made one
(linux 2.0.30 + axp-diffs 2.0.21 + miata-diffs 2.0.29 + manual cleaning-up
of some rejects + tulip.c v0.73 4/16/97).

The third problem occurs only using the self-made kernel because
miata-kernel-2029-complete.gz obviously lacks parallel printer support.
However, I don't think my pathches have broken something in the parallel
printer driver.

So, the goal of this rather lengthy message is to ask if someone has
experienced similar problems on a Miata (or on a different Alpha), and
ideally if someone has already come up with some kind of fix.

cu,
Andi
