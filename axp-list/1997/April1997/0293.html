>
> Michael Callahan and I are trying to install 4.1 onto an AlphaStation 255
> that has a CD-ROM but not a floppy drive. We're a bit baffled by the
> forest of MILO, aboot, SRM, etc etc. Here's what we know:
>
> The system definitely has SRM. We can say
> boot dka400 -file kernels/avanti.gz
> and then we see the "initializing HWRPB" etc messages and this:
>
> jumping to bootstrap code
> aboot: Linux/Alpha SRM bootloader version 0.5
> aboot: switching to OSF/1 PALcode version 1.46
> aboot: no disklabel found.
> aboot: booted_dev=`scsi 0 6 0 4 400 0 0', guessing boot_device=`scd0'
> iso: Max size: blah blah
> iso: First datazone: blah blah
> aboot: loading plain...
> aboot: unexpected object file format 1024
> aboot: loading compressed...
>
> unzip: unknown compression method
> Enter kernel name and arguments:
> aboot>
>
> Can anyone tell us what's going on? Thanks.

I am willing to make a bet that you have an old version of SRM. I
seem to remember having the same problem with an AlphaStation 200
with SRM 5.3 on it. I upgraded the SRM firmware to 6.4 and everything
works like a champ.

It looks like SRM 6.4 is the latest for the AlphaStation 255 too. I
would suggest that you upgrade and try again.
