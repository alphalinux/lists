My first bug is that the size of the NSS_BUFLEN_GROUP in grp.h is too
small. In our case we have a group with 75 people in it. The 1024 byte
buffer isn't big enough and lots of program die when they attempt to
do getgrpbynam and get back a zero. I doubled the size of that buffer.

The second bug is that if you only use files for gethostbyname it fails
to resolve dotted decimal names. It seems that the dns entry in the
nsswitch.conf does handle dotted decimal, so having both files and dns
listed works fine, because the dns catches it. If you only list files,
then it fails.

Both of these were relatively easy to work around once located, but I
hope I can save someone else the pain of finding them.
