The article below is a Request For Discussion for the creation of the
unmoderated newsgroup comp.os.linux.alpha. I recently submitted this
request to news.announce.newgroups where the original copy of this
article can be found. Note that this is _not_ a request for
voting---if you simply want to vote on this issue, wait for a couple
of weeks until the call for votes (CFV) for this group appears. If
you want to discuss issues relating to the creation of this group,
please subscribe to news.groups (in other words, don't bother sending
me mail directly---if it's not in news.groups, it didn't
happen... ;-).

Cheers,

--david

davidm@web.azstarnet.com (David Mosberger) writes:

>
> REQUEST FOR DISCUSSION (RFD)
> unmoderated group comp.os.linux.alpha
>
> This is a formal Request For Discussion (RFD) for the creation of the
> unmoderated Usenet newsgroup comp.os.linux.alpha. This is not a Call
> for Votes (CFV); you cannot vote at this time. Procedural details are
> below.
>
> Newsgroup line:
> comp.os.linux.alpha Linux on Digital Alpha machines.
>
> RATIONALE: comp.os.linux.alpha
>
> Linux running on Digital Alpha processors is quickly gaining
> popularity and is being discussed actively in a number of forums.
> This list currently includes a mailing list run by Red Hat Software
> and various newsgroup such as comp.sys.dec or comp.os.linux.sys.development.
> The amount of interest in this topic can be demonstrated easily by
> considering recent traffic statistics for the Red Hat operated mailing
> list. For example, in the 22 days between April 1-22, there were over
> 950 mails sent to the axp-list@redhat.com mailing list. Those mails
> were originated by over 200 different persons. The traffic volume on
> Usenet is harder to estimate since there is no specific group on this
> topic yet, but a increasing portion of comp.sys.dec traffic is now
> related to Linux on Alpha.
>
> The creation of this newsgroup would provide a focused and
> vendor-neutral place for discussing issues related to Linux on Alpha.
> Since a newsgroup is more easily managed than a high-volume mailing
> list, such a group would allow the casual Linux/Alpha user to touch
> base and keep track of developments without being inundated by a
> flood of email messages (or having to set up email filters).
>
>
> CHARTER: comp.os.linux.alpha
>
> The comp.os.linux.alpha newsgroup should be used for all discussions
> relating to buying, installing, running, maintaining, and developing
> Linux on Alpha processor based systems.
>
> The posting of commercial information to this group is permitted so
> long as the information is directly of interest to Linux on Alpha
> users and provided the postings are kept within reason (no frequent
> reposts, exaggerated or misleading claims, etc).
>
> END CHARTER.
>
>
> PROCEDURE:
>
> This is a request for discussion, not a call for votes. In this phase
> of the process, any potential problems with the proposed newsgroups
> should be raised and resolved. The discussion period will continue
> for a minimum of 21 days (starting from when the first RFD for this
> proposal is posted to news.announce.newgroups), after which a Call For
> Votes (CFV) may be posted by a neutral vote taker if the discussion
> warrants it. Please do not attempt to vote until this happens.
>
> All discussion of this proposal should be posted to news.groups.
>
> This RFD attempts to comply fully with the Usenet newsgroup creation
> guidelines outlined in "How to Create a New Usenet Newsgroup" and "How
> to Format and Submit a New Group Proposal". Please refer to these
> documents (available in news.announce.newgroups) if you have any
> questions about the process.
>
>
> DISTRIBUTION:
> This RFD has been posted to:
> news.announce.newgroups
> news.groups
> comp.os.linux
> comp.sys.dec
>
> After the RFD has been officially posted to news.announce.newgroups
> it will be forwarded to:
> axp-list@redhat.com
>
