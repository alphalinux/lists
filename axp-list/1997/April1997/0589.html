At 12:19 AM 4/13/97 -0700, you wrote:
>I'm trying to upgrade my system to Redhat 4.1 and being new to Linux I'm
>encountering some problems. The system came preinstalled with 3.0.3 and I
>could never get root for it so I've decided to do a clean install.
>Following the instructions I've decided that I'm going to have to boot
>into milo and start the install from there.
>
>Using rawrite I created a floppy disk to hold the noname-udb version of
>Milo. I created the proper entry in the boot menu (along side the already
>present Redhat linux entry) and tried to boot. However it gives me the
>error "File system not recognized." What could be the problems? I've
>tried recreating the disk many times and even formating the disk clean
>before installing the milo image. Could it be that my floppy drive (fd0)
>is not setup correctly? I noticed it was setup for 2.88 format and set it
>to 1.44 but this did not work.

I had the same problem. I found that if you use your linux system to
create the disks it works much better. Use "fdformat /dev/fd0H1440"
first, then the "dd if=<path to milo.img> of=/dev/fd0". That finnally
worked for me.
