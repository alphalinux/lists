Robert G. Smith wrote:
>
> When I apply the 2 em86 patches to the 2.0.30 kernel, most things
> run fine, including netscape and applixware. However "xfig"
> (3.14 or 3.2) crashes the X server after a few seconds. This
> doesn't happen when the em86 patches have not been applied to
> the 2.0.30 kernel. This is on an EB164 running the XF83_S3
> version 3.2 server.
>
> Any ideas about how to fix the problem ?

First question: are you using an x86 version of xfig when this
happens, or the native alpha version?

Next: Try isolating the problem by applying only one patch set
at a time. The two patch sets are independent:

em86_patch gives you the ability to run x86 binaries
transparently; without it, you have to explicitly invoke
em86 to run an x86 executable.

trapargs-patch-2.0 provides additional information to
the trap handler so that it can fill in the "sc_traparg"
fields in the sigcontext structure. Without this
patch, programs with large stacks won't work (this
patch is used by the automatic stack-growth code). A
number of executables (including netscape) work just fine
without this patch (I didn't need it until I started
running Applixware)

Let me know which patch is the culprit 8-)
