Yesterday Michael Callahan and I asked for help installing Red Hat 4.1
onto a floppy-disk-less AlphaStation 255. Thanks to everyone who wrote.

Since then, we've upgraded the SRM console to v6.4 (using bootp/tftp to
pick up the new flash image over the net). After that, we could use the
aboot image on the CD to boot a kernel, but then we couldn't figure out
how to get that kernel to read a ramdisk image off the CD-ROM (is it
possible?).

At this point, we gave in and cannibalized a PC's floppy drive. By
loading ramdisk.img onto a floppy, we could now get into the install
program.

In fact, we could install a Red Hat image onto the disk that boots
correctly, _provided_ you boot it by going via aboot on the CD: that is,
SRM> boot dka400 -fl i
...
aboot> kernels/avanti-s.gz root=/dev/sda1
[Linux kernel from CD boots, mounts /dev/sda1 as root]

Now, we'd obviously like the hard disk to boot itself.

[[Aside: where does it explain the difference between avanti.gz and
avanti-s.gz?]]

As far as we can tell, the approved way of doing this is to use aboot (is
it just us, or is the Alpha documentation very confusing on the question
of how to set up a bootable hard disk?). So, we install the aboot RPM and
do
swriteboot /dev/sda /boot/bootlx
(Actually, this comes after reinstalling the whole system from scratch in
order to leave room for the aboot boot loader at the front of the disk...
it would be awfully nice if the documentation would mention this at some
point _before_ the point where, having installed your machine, you
discover that aboot is what you apparently need to make it reboot
properly.)

In any case, running swriteboot now appears to have trashed the partition
table. This is judging from running fdisk -l just before the swriteboot
(which looked fine) and comparing it to fdisk -l just after the swriteboot
(gibberish).

We're now going to try doing this whole thing over again once more, using
a BSD-style disklabel.

Can anyone who has a working AlphaStation 255 with SRM console please
explain how they have set their machine up, especially including how they
did it so that it would boot straight off the hard disk into Linux?

Thanks,

Peter and Michael
