I'm working on GNU Fortran (g77) version 0.5.21, which will fix
a bunch of bugs in 0.5.20.

One thing I've heard on this mailing list is that g77-0.5.20
doesn't compile cleanly "out of the box", so to speak, on
Linux/Alpha/ELF systems.

I use an ancient Linux/Alpha/ECOFF system (still), and that
will probably change (it was supposed to last week, not sure
what's going on, but that's another story). I've been
intending to put off alpha-testing 0.5.21 until after
my current Alpha system is exchanged for one that has newer
system software (presumably ELF-based). But it is not
clear that I'll personally be running ELF here when
doing putting 0.5.21 through alpha testing prior to public
release.

In any case, it could be helpful to have, among the g77 alpha
testers, one or two people who are willing to alpha-test
g77 specifically to make sure it builds cleanly on canonical,
contemporary Linux/Alpha systems (and maybe old ones as well),
such as RedHat, Craftworks, whatever.

Send me (<burley@gnu.ai.mit.edu>) email asking about becoming
an alpha testers if you're interested in doing this. (Though
this e-mail is intended to rope in Linux/Alpha users, g77
alpha testing is still open to all sorts of people willing
to help out during the alpha-test phase of development.)

If you have some patches or at least some info on what needs
to change in g77 to support Linux/Alpha/ELF better, please
at least send those along to <fortran@gnu.ai.mit.edu>. Even
if you can't become an alpha tester, maybe others will, who
will be able to test out the g77 I produce by using the
patches/info you provide!

(Alpha testing is still done on every single g77 release
_before_ it goes to the public as a "beta" release. While
that has not yet led to the perfect beta, I'm pretty sure
there's been no perfect alpha release either, so without
the g77 alpha testers, public releases wouldn't even be as
good as they have been!)

tq vm, (burley)
