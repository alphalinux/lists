On 04-Apr-97 CHIH-HSIANG HENRY YAO wrote:
>I am thinking of moving to Digital Alpha systems. Here is a few questions:
>
>1. How often could you port a Intel application software without much
>problems?
It depends. Sloppy programming that assumes that pointers are the same size as
integers is the most common porting problem. Alpha uses glibc2, so there may
be a few header file conflicts. Generally, not much a problem with most
software.

>2. Berkeley SPICE for Alpha precompiled?
Pass on this one.

>3. Magic for Alpha precompiled?
Pass again.

>4. Netscape for Alpha?
With the brand-new EM86 code from Digital, Alphas can run Linux/Intel binaries
such as Netscape. Otherwise, you either had to have a copy of Digital UNIX
(OSF/1) and install the libraries and loader on the Linux/Alpha system to run
Netscape compiled for Digital UNIX, use Mosaic 2.7.3beta or Chimera, or have a
spare Intel machine around to run Netscape via remote X.

>5. Knews or good news reader for Alpha?
Knews compiles out out of the box. Slrn is in Redhat 4.1 and is my favorite.

>6. how hard is it to install NT 4.0 with Linux simutaneously?
Very easy. You use the ARC console to boot either Linux or NT.

>7. Motif library precompiled for Alpha?
>From MetroLink. Static (ECOFF) only. $300. I am waiting until they support
ELF and lower the price a bit.
