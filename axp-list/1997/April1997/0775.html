Well, exactly what it said...

The kernel ran out of places to stick information on open files. It doesn't
dynamically grow those structures because this could indicate a rogue
process on your machine getting far too greedy. If it's just a busy
machine (i.e. a server), add the following to /etc/rc.d/rc.local:

echo "5000" > /proc/sys/kernel/inode-max

Which will increase the number of inodes the kernel will let itself open. You
can 'cat' that same file to see the current limit btw. A related /proc
entry, /proc/sys/kernel/file-max, controls the number of files which may
be opened at a time (inodes are used for directory operations and other
miscellaneous jobs so the numbers aren't quite the same though they seem
like they should be at first glance).
