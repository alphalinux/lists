>>> Brian Langdon said:
>
> What's the likelyhood of the TGA version being extended to cover the 24-bit
> card (a ZLXp-E3 ??) please.

What's the chance of finding someone willing to do the work?

This is not as facetious as it sounds; there are so few Alphas out there
running Linux with an -E2 or -E3 installed that I think you could prolly
count them on the fingers of one hand. To us, it doesn't seem to make sense
to devote any of our already scarce (2 full-time developers) resources to
this project.

Now, a great deal of pretty good code supporting 8 and 24 plane versions of
the TGA was donated by DEC to XFree86. To this point very little of that code
has been incorporated into the existing XFree86 TGA server, which remains
predominantly based on dumb framebuffer code (ie "cfb"). Also, I don't think
that any of the Alpha-endowed Xfree86 contributors actually *has* an -E2 or
-E3, making it pretty tough to develop.

If you have the need for basic support, and the hardware, of course, I'd say
donate your services to XFree86. Start by extending their current server to
support the 24 plane versions, using the "cfb" code for quickness of
implementation. Then, if you really want speed, go back and start
integrating in some of the DEC-written code.
