For what it's worth, I see this problem on both my home system and the
systems I set up for customers at work to use as SMTP and UUCP mail
routers. These are 'clean' installed; I installed RH 4.1 from the
PowerTools CD set and then immediately installed all of the updates from
the FTP site.

Running 'screen' definately causes the problem, but it isn'tt a
requirement. Logging in from remote seems to cause it too; in fact if you
log in locally you can change your password, but logging in on the same
account from remote gets you the 'User not known to underlying
authentication module' message when trying to run passwd. Is that weird or
what?

removing the utmp file does _not_ solve the problem as someone else
suggested. It just gets corrupted again.
