I don't need a 'sleep 10' either. It seems that

#network start

is reading network values and then does :

for i in ifcfg-*[0-9o]; do
./ifup $i boot
done

FOR-loops sort the arguments in alphabetical order.
In my case the loop is equivalent to:

#/etc/sysconfig/network-scripts/ifup ifcfg-eth0 boot
#/etc/sysconfig/network-scripts/ifup ifcfg-lo boot

this is very, very bad.

Solution:

try to start the loopback before the network interfaces,
- it works perfect - without waiting 10 seconds or deleting
any file !
