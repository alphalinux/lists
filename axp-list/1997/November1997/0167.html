I second Stefan's comments. I use Linux PPro's that use DMA mode
transfers and are faster than low end SCSI cards (NCR810 for example).
(not faster than a 2940UW though)
My 164LX which is EIDE only, get's about 1.8MB/sec with 80-90
percent CPU utilization. I believe the problem is that DMA is not
implemented for the Alpha EIDE control chip. I saw someone earlier
mention they had better performance with EIDE but I tried their
command on my system and all it did was slow the system clock down
so that it appeared to run faster but it in fact didn't. You have
to unmask interrupts with hdparm to see that things don't change.

Anyone have any idea how to get DMA going on DEC's EIDE chips??

My motherboard book says the chip is a CMD PCI0646 and my PC164
says CMD646.

Wes
