I'm trying to install redhat 4.2 on an alphastation 255. It's currently running
digital UNIX and has SRM, and I don't see a way to switch to AlphaBIOS.

I've figured out that as255 is an avanti, XL is too, and all avantis are
supposed to be the same, but redhat even lists the 255 on their hw list. I'm
using dd on intel linux system that works. And there were no errors with dd
(ie, no bad disks)

First of all, I'm been trying to boot milo. I have found milo stuff in three
places:

1) milo/images/XL.img and noname.img. they boot as invalid boot blocks.

2) milo/XL/milo.dd which has (apparently) a valid boot block but hangs when
SRM jumps to the code.

3) milo/noname/milo.dd, which produces the same result.

I've tried images/xl.img, xlt.img, avanti.img and avanti-s.img. All give the
same result - they load aboot but do not have a valid compressed (or
uncompressed) kernel behind aboot. Once I have aboot running I've also tried
telling it to fetch the kernel from disks made from the .img files under
milo/images, just to have tried all combinations, but that didn't work either.
Is the vmlinux kernel supposed to be behind aboot? Here's what I'm getting
(much to my amazement, I didn't think I'd get anything)

(boot dva0.0.0.0.1 -flags a)
block 0 valid
reading 172 blocks...
boostrap code...
init HWRPB...pagetable...machine state
set affinity...
jumping to bootstrap code...
aboot: Linux Alpha SRM bootloader version 0.5
aboot: switching to OSF/1, PALcode version 1.46
aboot: no disklabel
aboot: booted_dev dva0.. guessing boot_device fd0
aboot: loading plain
aboot: unexpected object file format 13
aboot: loading compressed
unzip: unknown compression method
Enter kernel name and arg

at that point I've tried 3/vmlinux.gz, 0/vmlinux.gz, and a
few other things but nothing's working. with 3/vmlinux.gz:

3/vmlinux.gz single
aboot: booted_dev= ...
aboot: loading compressed vmlinux.gz
unzip: attempted read past eof
aboot: loading plain vmlinux.gz
aboot: unexpected object file format 137165

Does anyone have any idea what's going on? If this means that milo hasn't been
ported/just doesn't work on the 255, then what about vmlinux? Since I have a
working aboot, which is the "preferred" method for booting from SRM (according
to the SRM howto), shouldn't I be able to load the kernel, and if it were an
incompatability shouldn't I be getting just about any error but a compression
complaint?
