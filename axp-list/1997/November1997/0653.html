Dave,

Occasionally, (more like frequently) Digital adds support for devices into
the firmware. The firmware has drivers for devices and system
initialization code. Sometimes Digital finds better ways to initialize
the system than they had been doing previously, or they find gross errors
which need to be corrected. In the same way that your house or place of
business needs to rest on a firm foundation, so must your operating system
boot from a solid firmware. Fortunately for computers, and unfortunately
for Real Estate, firmware can be upgraded whereas foundations ... well
it's much more difficult.

One of the early AlphaBIOS versions had a problem in how it set up the
memory subsystem... so any system with more than 128 MB or RAM had
problems. This firmware also had problems with Glint chips running OpenGL
applications. The safer AlphaBIOS is anything after or including 5.60.

To upgrade your AlphaBIOS, you need a file called "lx164nt.rom". Drop
that file on a floppy disk, then go to the "setup menu" and select
"Upgrade AlphaBIOS". The rest is very easy. For ARC, the steps are very
similar. ARC has been stable on all platforms for quite a while. The
only real reason to upgrade your ARC is to add bootable support for any
new devices.

The firmware image files can be obtained from whomever you purchased your
system.

There is not a Windows-NT only ARC nor AlphaBIOS ... they can both run
Linux.
