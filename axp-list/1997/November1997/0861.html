>>> Dragisa Duric said:
> When trying to run any Linux/Intel binary on my Alpha box, I am getting
> lots of these:
>
> applix(22068): unaligned trap at 0000000000000024: 000000004001d712 28 27
> or:
> yd(22095): unaligned trap at 0000000000000024: 000000004001d712 28 27

These are only warnings, things should operate correctly, albeit a bit more
slowly because of the exception handling that has to take place.

> Any hints on this, and is it common to have slower running apps as I
> have? I remember reading about emulated NT/Intel binaries running on
> slower Alphas than mine, but faster than on PPro/200.

What you are referring to here is FX!32, which is NOT just an emulator; it
can actually translate INTEL machine code into Alpha machine code. It runs
ONLY under Alpha NT 4.0.

em86 does NOT do the translation part, it uses ONLY the emulator from FX!32,
and therefore you'll not see the same overall performance from it.
