>>> Kelvin Lunsford said:
>
> This discussion makes me curious. Isn't the SRM console firmware available
> at:
>
> ftp://ftp.digital.com/pub/DEC/Alpha/firmware
>
> along with the firware load utility? I know it's for booting DEC OS's, but
> can it boot LINUX? Can it boot NT? Will it solve any of Michal Jaegermann's
> problems? Also, will it allow remote boot?

1. it is available, but it is legal only for people who have a license for
Digital UNIX or OpenVMS on the specific box they wish to install.

2. it can run LINUX; that is, it provides appropriate PALcode to support
the LINUX kernel.

3. it CANNOT boot NT directly.

4. it has its own idiosyncracies; it understands only BSD partition tables;
it doesn't understand MSDOS or EXT2 filesystems (that's for "aboot" :-).

5. it does support "remote boot", if by that you mean serial console control
of the boot process.
