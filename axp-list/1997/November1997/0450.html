> i believe that the IDE on the motherboard works with ide drives. it
> might not be as fast or efficient as one would like it to be. but it is
> supported, and working.

Quite so. The CMD646 chip works pretty much "by the spec" (of IDE),
and thus it has not yet attracted need for any fixup code, nor
apparently has so wide presense in iX86 boards that it could have
attracted performance tuning code.

OTOH, CMDIO646 (as it is called) documents speak of PCI IDE spec
version 0.9, which may mean that this chip could be treated with
generic IDE code just fine. Towards the end of the document
there is paragraph:

DMA Programming

See Revision 1.0 of Intel's "Programming Interface for Bus Master
IDE Controller" for information about bus master DMA programming.

I personally have SCSI disks, and IDE cdrom on CMD646, but as the
cdrom is not performance critical item, I have not had much interest
at coding the driver either...
