> I need some really specialized help now, though: I need testers who
> have Deskstation's RUFFIAN board (aka RPX164-2, I believe).

We got three of them:

RPX164-2, 500Mhz
256MB, 2MB L3
Linux version 2.0.30 (RedHat 4.9.1 Mustang)
gcc version 2.7.2.3 / glibc
ide: i82371 PIIX (Triton)
ncr53c875-0: ID 7, Ultra SCSI (onboard)
IBM DCAS-34330W 4.1GB
eth0: DEC DS21142 Tulip (onboard)
Pionerr CD-ROM DR-U24
Diamond Stealth 64 4MB VRAM

> /pub/DEC/Linux-Alpha/Kernels/ruffian-2030-24nov.gz

This kernel gives:
init: memory violation at pc=00000000 rp=1555555cfb4 (bad address = 00000000)

We got the kernel from www.deskstation.com and we're able to compile
it using RedHat's gcc.

Now wer're running NIS, NFS, amd on it. There seem to be several
problems still:

- swap spce dosen't seem to be set up propperly:
# swapon -s
swapon: /proc/swaps: No such file or directory

- the amd automounter doesn't unmount filesystems after a while
(allthough amq doesn't show them, df does!)

- there seem to be some memory problems (in freeing).

- ypbind sometimes shows weird behaviour

- f2c/fort77/g77 seems to be a bit tricky

Do you have the src code for the kernel? The one from
www.deskstation.com does work, but ony by editing .config and
autoconfig.h by hand ...

Hope that helps
