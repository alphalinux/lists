Paul Anderson wrote:
>
> Let me say that I love Red Hat and I love my Alphas ( I have quite a
> collection.) That being said, I would not use Linux for an ISP on an
> Alpha. There are too many pieces missing for me (NFS would be one, for
> instance).

Eh??? Just last night I came back from COMDEX when, on the floor,
I used NFS there and back again between two Alpha machines running
Linux which were displayed in our booth. Not that I advertised
that fact specifically. We needed that as a tool to help us with a
setup and for me this was a matter of course. If you think about amd
(automounter daemon) then every time I have Alpha on my "home network"
of few machines amd is heavily used on it as this allows me not to worry
if some particular machine is on-line. I crossmount file systems in
every possible direction and then some. :-)

I can think of few holes or things not working as advertised in current
Red Hat distributions on Alphas but none would affect seriously an ISP
business - AFAIK. Actually I would think that Alpha with its memory
access capabilities and I/O is a great machine for ISP (not UDB though,
unless you run something rather lightly loaded).
