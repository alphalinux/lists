On Fri, 21 Nov 1997, Christopher Curtis wrote:

> I have a little UDB which I finally got RedHat 4.2 insatlled on. I'd like
> to configure it as a masquerading firewall, and have everything setup,
> except I needed to recompile my kernel to support firewalling. Using the
> sources from the updated RedHat RPMs, I recompiled 2.0.30. The BusLogic
> driver, which I tried to compile as a module, aborted GCC with a signal 6.

A few people I spoke to who also own multia's, and my own setup, showed that
2.0.30 with a multia/alpha will not work correctly ask firewall/masq host.
I am not sure if this has been fixed, but I don't think so.

>
> My second question concerns memory. The UDB had two 16MB true parity
> matched SIMMs, but upon booting only reported 24MB. Following the
> instructions on the list, I got the latest MILO from gatekeeper today and
> installed it onto my 4MB FAT partition. linload.exe is not easy to find.
>
> This did not help. I took two additional 16MB true parity non-matched
> SIMMs and put them in the UDB (in slots 0&1) and moved the original SIMMs
> to slots 2&3. Now, upon booting, it reports 32MB. Adding the kernel
> option "mem=64M" from the ARC OSLOADOPTIONS didn't help, nor did adding it
> from MILO.

It is a feature of ARC, that when you run milo by naming it updatefw.exe
and picking "Install new firmware", to boot in a sort of special mode. In
this mode, the machine will ALWAYS report 24MB of RAM, nothing you tell
milo will change that later on. So don't boot it like this, but use a proper
boot entry in the ARC menu.

I think they compile well, though I am running stock rpm's here. (But if
