I have written optimizing fortran compilers and from my experience
a hand unrolled Newton-Raphson (I think I last used 3 times for double
with table lookup seed worked very well. Michal is also right about
the index calculations. If you don't unroll, the next best thing is to
use shadow variables, i.e. use the address as the induction variable.
As I recall, the LDA instruction on alpha works well for this and
probably avoids pointer load delay stalls in the pipeline.

Obviously, you can always improve any single program, but you don't want
to do it every time, which is why I insist that it should be done in
the code generator. Lean on cygnus to implement the optimization. That
backend is too messy for me!
