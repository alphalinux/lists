On Wed, 12 Nov 1997, nm wrote:

> hmm, well what i am calling an AT keyboard is what plugs into the AT keyboard
> port on my axppci33 board ;)

Actually, you may use a converter if the plug is incompatible. ;-) We
have some keyboards here, which are identical both externally and
internally, only the plugs differ.

> im not so sure that they are the same, just for the fact that i can unplug my
> 'AT' kbd and plug it back in while the system is running without any problems
> where as on many systems with ps/2 kbd's once you unplug it, blamo, you
> have to
> reboot to get it to recognize it again.

Yeah. That's because of stupid software. Actually, some time ago, I
wrote a program for the Intel PC that used the keyboard in the native,
PS/2 mode. As the keyboard default mode is AT, I had to switch the mode
upon startup. And yes, the program was able to detect the fact of
keyboard reconnection, so it got reinitialized and even the LED states
were restored -immediately- after, without any user intervention. So it
is possible; just some developers do not bother to implement it.
