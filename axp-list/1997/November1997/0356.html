On Fri, 14 Nov 1997 10:09:05, Clemens Vonrhein wrote:

> We just got some new RPX164-2 based alpha systems (boards from
> deskstation). I'm just curious if anyone else out there has one of
> them too or if we're the first ones (well, I had this impression from
> some difficulties in getting things working and all that).

We got one RPX164-2 based system from Deskstation Technology, Inc. (DTI)
this past monday. There are indeed difficulties in "getting things
working and all that" for DTI, since, as they told us, they don't have
any Linux expert right now.
One of the things they couldn't get to work was running 3 Ethernet
cards (the one on motherboard, and two additional, Tulip-chip based ones)
simultaneously. Has anyone been able to get this to work on such a
system, or can give us some suggestions?

> The machines:
> RPX164-2 500Mhz
> 256Mb
> 4 Gb disk
> SPEA V7 Mercury P64V (4Mb)
> Eizo FlexScan T67S
> RedHat 4.2

Very similar system, with RH 4.2, kernel 2.0.30.

> We got them working now more or less with NFS, NIS, amd and all
> that. There are still minor problems in parsing OSLOADOPTIONS from the
> ARC BIOS to milo and the workaround hack for using NIS maps with amd
> isn't as nice as I want it to be.
>
> We're planning in installing DQS 3.1.* on them: any experience with
> that?

No.

I think this is sort of a FAQ; but since this is a new system, maybe
it's at least a new variant of the problem. When we boot up the system,
it always comes up with the year=1925. (This prevents NFS mounting, so we
have to reset the date, and then can mount NFS file systems -- quite
annoying.) Does anyone else have this problem -- and maybe a solution?
