Where you stated:
>On Nov 22, Maurice W. Hilarius wrote:
>
>> Oh, I don't know if CPU is all of the story. From the amount of HD I/O you
>> produce I think disk subsystem performance is pretty important to the
>> figures, as well.
>

This is a "critical path" type of issue. If you are running a system
deficient in either RAM or disk I/O it shows up very clearly. For example,
compare a UDB running the stock internal HD to one with a fast external
disk, the difference is about 50% on a compile.
Same goes for memory and swapping. At around 32MB or less an Alpha will
LIVE in swap on a kernel compile.
