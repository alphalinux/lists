>
>
> I have recently read in a few Linux books (based on Intel Linux) that you
> can have upto a 128MB swap partition,

This limit does exist but is 512MB on Alpha. It applies not only to
swap partitions but to all "swap areas" - which can be partitions or
files.

> but multiple swap partitions.

A "regular" kernel supports up to eight swap areas. If you are willing
to dive into kernel sources this number can be changed.

> One
> book claims that if your swap partition exceeds 128MB, it will only use
> the first 128M.

This limit is caused by mapping swap pages to bits in maps. A page
which is unmappable will be not used. Different mapping on alpha
makes for a different limit.

Books could be a bit more precise in what they say.
