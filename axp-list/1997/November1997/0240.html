Yue-Feng Sun (sunyf@lamont.ldgo.columbia.edu) writes

Question and Background: I mostly use MPI on IBM SP2 to run
parallel jobs. I also laterly tested the MPI on my single
Pentium. Wes's results show that Alpha 533 MHz 21164 with
ALphaPC 164LX is 3.5 times faster than a 200MHz PPro ( so 4.5 times
the speed of the PPro), so one Alpha 533 MHz has the speed about a
300 MHz Quad Pentium II. Then the choice between the two depends
upon the price and the real speed of parallel computing on a 300MHz
Quad Pentum II. What are the intercommunications between processors
on a Quad Pentium? Does it have High Performance Switches like IBM
SP2 or it uses only Ethernet to do the intercommunications? If
anyone knows the intercommunication mechanisms on multiprocessor
Pentiums or can direct me to related newgroups, I will greatly
appreciate your help.

The quad pentium motherboard has four pentia sitting on a shared bus
to memory. It is a UMA (Uniform Memory Access), i.e. all CPUs see all
of the memory as local); this is in contrast to Non-UMA (NUMA) setups
like SP2 or Network-Of-Workstation, where CPU has its own local memory
and goes to other CPU's memory via some kind of communication channel
(ethernet or dedicated switches).

MPI was really designed to deal with NUMA; you don't need MPI on an
UMA machine (the data worked on by CPU 1 is already accessible to CPU
2---you don't need to send it). In other words, on an UMA machine MPI
is reduced to a simple synchronization mechanism---the message doesn't
need to be passed, it's already there.

If you are lucky, your MPI implementation on quad-pentia will be written to
avoid this unnecessary sending of messages, and you'll get a 4x speedup.
That is, if you aren't bandwidth-limited, which you very well may be.

The bottom line is that you have to benchmark and see. It really depends
on how your program is written and on the MPI implementation.
