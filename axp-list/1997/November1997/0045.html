you need a dos FAT disk partition of at least 1Mb holding milo and
linload.exe. if this exists (or you create it) you can use the following
steps from the arc console (your paritions may differ, of course. also,
most notable is that if your root partition is not /dev/sda2 then for #14
below you should append the argument 'root=/dev/sda(whatever)'
if you boot multiple os's you should skip 16, 17, 18. you can still have
linux as the default which will only add a (human intervention) delay time
before booting...

1. select 'supplementary menu'
2. select 'set up the system'
3. select 'manage boot selection'
4. select 'add a boot selection'
5. select 'scsi buss 0 hard disk 0 partition 1
6. enter osloader directory: 'linload.exe'
7. is the os in the same partition as the osloader? yes
8. enter os root directory 'milo'
9. enter name for boot selection
10. initialize debugger at boot time? no
11. select 'change boot selection'
12. select the linux boot selection
13. get to 'osloadoptions'
14. replace text with 'boot sda2:vmlinux.gz' hit return then esc
15. select 'setup menu'
16. select 'autoboot'
17. should the machine autoboot? yes
18. set delay time
19. save changes
20. esc to arc menu
21. select boot linux
