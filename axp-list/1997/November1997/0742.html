Roger Hill-Cottingham <eesrjhc@ee.bath.ac.uk> wrote:
> > > I was hoping for a fortran compiler that worked, as f2c is broken (as
> > > installed from RH4.2 CD-Rom):
> > >
> > If you just compile f2c from the sources at netlib, it will work.
>
> OK, just done that, and it is fine for little things, but large code
> I get millions of
>
> relocation truncated to fit: LITERAL
>
> (seems like one for every assignment). Do I have to fix something in
> f2c/src before making it (like definitions of integer in f2c.h)?

This actually seems to be a problem with the linker; or at least it manifests
itself during the linking. I checked that by doing f2c, gcc, ld manually.
The C code generated by f2c at least *looks* o.k. I also get many thousands
of these error messages

beckw2.c(.text+0x14f0): relocation truncated to fit: LITERAL j1.20
beckw2.c(.text+0x14f8): relocation truncated to fit: LITERAL j2.21
beckw2.c(.text+0x157c): relocation truncated to fit: LITERAL xfac.53
beckw2.c(.text+0x1598): relocation truncated to fit: LITERAL yfac.54
beckw2.c(.text+0x15b4): relocation truncated to fit: LITERAL zfac.55
[etc.]

They seem to come from local variables only, as far as I can tell
(which are declared static by f2c).

Does this help anyone explain what's the problem here? And what a
fix, or at least workaround, would be?

Marc

P.S. Versions used: f2c (version 19970805)
gcc version 2.7.2.1
ld version 2.7 (with BFD 2.7.0.2)
RH 4.2, kernel 2.0.30
