I have been installing Linux/RH on a variety of Intel boxes, and it
has been very smooth. For some reason, the alpha install isn't going
as well. I am doing this on a pc164lx. Here's a step by step
description of what happens:

1. I go through the RedHat install process up to the partition table
editing step.

2. I create the partitions using fdisk. Here I notice some strange
behaviour: RedHat says it's partitioning /dev/sda, which is correct,
but when I enter fdisk, it says /tmp/sda. I create four partitions
(tmp/sda[1..4]), the first a small dos partition (for milo and
linload), then the swap, then the root partition, and then the usr
partition.

3. Then, it formats the swap space, and asks me which packages to
install. I select the packages I want to install.

4. Then it says "making ext2 filesystem on /dev/sda3 and /dev/sda4
(which are my root and usr filesystems) (but it's actually doing it on
/tmp/sda3 and /tmp/sda4).

5. Once it finishes this, I get a "mount failed: Invalid argument"
error from the RedHat. Looking though the other consoles, I see this
error:

EXT2-fs error (device 08:03): ext2_check_descriptors: block bitmap for
group 100 not in group (block 6)!
EXT2-fs: group descriptors corrupted

6. Also I note that it's trying to mount /tmp/sda3 on /mnt. Is this
the correct behaviour?

Thanks in advance for your help.
