Roman Martonak wrote:

> So is g77 going to be further developed and will there
> be new versions, apart from the egcs project, improving
> in particular the behaviour on Alpha's ? When trying to
> get information on the g77 status via finger -l
> fortran@gnu.ai.mit.edu, it tells me Unknown host
> gnu.ai.mit.edu(few months ago it worked this way).

Yep, try finger -l fortran@alpha.gnu.org

> I mentioned some time ago on this list problems with a
> program compiled with g77 running fine on Intel Linux
> but crashing with segmentation fault on Alpha, being
> apparently impossible to debug and find out where the
> problem really is. I did not have time to fight with it
> more but it seemed that the debugging support of g77 on
> Alpha is not yet very good.

Or the program is buggy and it only shows on Alpha (yes, I know
this sounds like a cheap chick-out, but this often happens on the
g77 bug mailing list).

Your best chances are sending the failing program to
fortran@gnu.org according to the recipe in the g77 manual (Chapter
"Reporting Bugs" and subchapters).
