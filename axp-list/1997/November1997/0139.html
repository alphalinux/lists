Hello,

I try to get RedHat4.2 function with a Alpha PC-164. The main problem
I have is NIS and in some cases the r-Commands (rlogin)!

There seems no possibility to get NIS running. I tried all a human
creature can do. All new ypbind versions, different network-cards(SMC
Ultra, WD800* and 3COM 3C509) and different yp-clients.

If I start ypbind with the -debug Option there is no problem, but if I
try to connect to the Alpha-PC (terminal-login) I get the following error:

memory violation at pc=155556c6c58 rp=155556e2610 (bad address
= 74726f6b610072)!

I can also execute "ypcat hosts/passwd etc" and after the correct
answers are displayed the ypcat core-dumps with the following message:

yp_all: clnt_call: RPC (unknown error code); s1 = 0 s2 = 0
Illegal Instruction (core dumped)

Is it a memory problem? - I have on;y 64MByte!

I use the latest offical kernel with Redhat 4.2, all patches and the
latest milo!

At last I tried a DEC-Card with the TULIP-Chipset and now the systems
stops while entering runlevel 3 (after sendmail is executed and the
network card has been set to 10Mbit/s) with an error message!

Please help!
