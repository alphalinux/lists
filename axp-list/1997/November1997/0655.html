Hello All,

I've got an Intel Linux system running RH 4.0 which connects via 115200
bps ISDN (two simultaneaous lnes) to an ISP... works fine! I'm trying to
achieve the same with an Alpha-Linux system running RH 4.2. The probm I'm
seeing is that The Alpha serial port, although 16550 compliant, appears to
hose the modem at speeds higher than 38400 bps. Giving the modem this:
#setserial /dev/modem spd_vhi
allows minicom to establish a connection at 115200 bps, but does not dial
out to the provider properly. The following command does minicom & dial:
#setserial /dev/modem spd_normal
but does not connect properly because it's only 38400 bps...

Question: Does anybody out there have an Alpha connecting via ISDN to an
ISP at 115200 bps? If yes, what did you have to do to get there?

All of you running ISPs on Alphas should know this! :-)
