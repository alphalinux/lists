I've forgot to mention that this patch is against cdrecord-1.6a5.tar.gz
from ftp://ftp.fokus.gmd.de/pub/unix/cdrecord/alpha

Some other notes I've forgot in my first mail:

You need to increase the maximum size of a shared memory segment to
be able to use the default fifo size of 4BM (increase SHMMAX in
asm-alpha/shmparam.h; I'm using 0x2000000, which I took from
asm-i386/shmparam.h). You can also reduce the fifo size by using
fs= as cdrecord option.

You need the patch for mlockall(), which is included in 2.0.31 (neither
2.0.31 nor 2.0.32pre5 worked for me without flaws on my Alpha, so I took
the patch from 2.0.31 (only for file mm/mlock.c) and put it into my
2.0.29 kernel.

And finally I had to upgrade the NCR538xx driver to the version included
in 2.0.31 or higher (I'm using 2.6b), otherwise the scsi driver couldn't
interpret the respones from my JVC XR2010.
