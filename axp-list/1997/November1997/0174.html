>On Fri, 7 Nov 1997, Don wrote:

>> Does anyone have any idea what's going on? If this means that milo hasn't been
>> ported/just doesn't work on the 255, then what about vmlinux? Since I have a
>> working aboot, which is the "preferred" method for booting from SRM (according
>> to the SRM howto), shouldn't I be able to load the kernel, and if it were an
>> incompatability shouldn't I be getting just about any error but a compression
>> complaint?

> If you want to boot via SRM, then grab a "-s" image, put it on a floppy
>and put the ramdisk image on another one. Then enter SRM and type:

>boot dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1"

>Once the kernel is booted, switch the disks (it will ask) and proceed. If
>the kernel complains about hardware and is unable to proceed then you may
>need another image (I succeeded at the third time when booting on an
>AlphaServer 300).

> HTH.

And then what? I used that procedure in order to get the RH 4.2
install to work and succeeded in installing RH 4.2 on a second hard
drive. Unfortunately, I haven't figured out the proper incantation to

HD so SRM can't find it.

Looks to me like I need to re-partition the HD using the BSD
[in]compatible partitioning buried in fdisk, install aboot on the HD,
re-install RH 4.2 on the HD, ... but I'm only guessing.
