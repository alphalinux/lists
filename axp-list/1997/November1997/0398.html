For an Alpha, you'd probly want to try to get 64MB RAM, as RISC code tends
to be rather large (and memory hungry). For normal operations, the 166
UDB will probably feel about as fast as a Pentium-90 or -120. If you're
doing a lot of FP intensive tasks, though, it will be signifacantly
faster. Sorry, I don't have real numbers handy. But I'm sure you can
find loads of benchmarks on the 'net.

> I've seen the DEC Multia 166Mhz systems being auctioned off on
> www.onsale.com. I'd appreciate any more good advice or honest opinions
> one whether these things are any good. The site doesn't have much info on
> the system (like whether IDE is available, whether or not the CPU is
> upgradable, how good the power supply is, etc).

You can think of the UDB as a first generation NC. It's actually very
slick, with integrated Ethernet, PCMCIA, Video, etc, but also very
limited. The first UDBs are not upgradeable, and use an older technology
chip. The onboard cache is also constraining. The newer UDBs can be
upgraded to a 233MHz chip, but I think that's about the end of life. The
power supply is about 75 watts, and any hard drive will have to be a 2.5"
half height notebook-sized drive. It does have a SCSI-II port on the
back, though, so you can hook up externals. Internally, it has one PCI
slot. Also, the video is limited to 256 color, but I think supports the
1600x1200 range resolution. Will do 1280x1024 at a minimum, I'm pretty
sure does the next highest level too.

You can actually get the spec sheet (I think from Digital) on the 'net in
PDF format. Has all these details and more. Lost that URL, too. =)
