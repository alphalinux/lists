>>> Jurriaan Saathof said:
> Hi,
>
> I'm fairly new with Alpha's but I've managed to get my Linux to run on
> my XL-233. I have compiled this new kernel ( version 2.0.30 which came
> with RH4.2 ) to meet my configuration. But when I booted the system with
> this new kernel I get the following error once every +/- 5 min :
>
> Unaligned trap at fffffc000038b530 : fffffc0001a13029 29 2

I'm betting that you configured the tulip.c driver, rather than the de4x5
driver, for the TULIP Ethernet card you have in there. If so, please try
the DE4X5 one instead, as the tulip.c one in 2.0.30 needs to be upgraded
to a much later rev to eliminate those unaligned accesses.

> I also have this problem with my Matrox Millenium which came with the system.
> The bios says "Bios32_fixup - weird, unknown PCI device found in slot 7
> has no IRQ line... "

Please try the latest XL kernel you can find on gatekeeper.dec.com; ie:

/pub/DEC/Linux-Alpha/Kernels/xl-2030-fixed.gz

This will probably fix *both* problems. If you wish to build kernels,
grab the alpha-patches-2.0.30-0.3.gz file from that same directory, and
apply it to "pristine" 2.0.30 sources, ie NOT the Red Hat kernel sources
from their RPM or in /usr/src/linux. Build the kernel in a different place
using something along the lines of (with your own names for "someplace"
and "archive"):

cd /someplace-not-usr-src
tar xzf /archive/linux-2.0.30.tar.gz
cd linux
zcat /archive/alpha-patches-2.0.30-0.3.gz | patch -p1 -E >& ../log
make config
make dep
make boot

Good luck.
