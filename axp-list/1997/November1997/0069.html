Jay.Estabrook@digital.com wrote:

> >>> "Jim. Shipman" said:
> >
> > I have a PC164 system which uses bootp under SRM. This took kernal patches
> > and a special build to create the boot image which goes on the bootp
> > server. Now I am trying to do the same with AlphaBIOS. Can anyone give me
> > a hint as to where to find the info to build a boot image and make it work
> > on a PC164LX machine?
>
> Well, here's a hint: what are you going to do for PALcode that the LINUX
> kernel depends on?
>
> SRM console can provide the OSF PALcode that the LINUX kernel can use; MILO
> can provide the EBSDK PALcode (similar but not the same as the OSF stuff).
> LINUX can't (at least currently, maybe never) run using the NT PALcode that
> ARC or AlphaBIOS provide.
>
> This is not insurmountable, but you may have to roll your own solution. I
> suggest looking into building a MILO image with embedded LINUX kernel; this
> was done many moons ago by Dave Rusling, for testing purposes only, I
> believe, but the build options and compile-time switches are still present
> in the sources, IIRC. That way, MILO will bring along the appropriate
> PALcode for the LINUX kernel to use.
>
> Once you've got the MILO plus embedded-kernel capability, then you'll need
> to figure out the BOOTP image format that will work when loaded by AlphaBIOS,
> as opposed to the image that is BOOTP-able from SRM; these may be different,
> I'm thinking.
>
> Good luck, be sure to send me the patches when you get it working... :-)
