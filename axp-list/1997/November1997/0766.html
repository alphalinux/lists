On Mon, Nov 24, 1997 at 10:06:12PM +0100, Alexander Volovics wrote:
> 1) How stable is the Alpha Linux kernel, does this really approach the Intel
> kernel now in stability?

I think that for home use (as opposed to server use) Alpha/Linux is quite
stable enough (and probably for server use as well).

> 2) How is the hardware compatability: memory, SCSI, disks, zip drives, printers,
> and especially soundcards (AWE64?)

I think I heard some complaints about sound support on this list.
Otherwise a lot of hardware just works as it would on Intel. Be careful
about video-cards, though, as AlphaBIOS isn't always as good at emulating
an x86 for executing the video-bios. (I know from personal experience...)
I've got a built-in NCR3c815 and an Adaptec AHA-2940UW which run just
fine. SCSI works smoothly. So SCSI ZIP-drives shouldn't be a problem. I
don't know about parallel ZIP.
Memory on an Alpha usually has to be true parity SIMMS or ECC SDRAM, from
what I hear. This means you can't always buy standard memory.

> 5) As I do not want to have to compile my own software for the Alpha (or
> (re)compile the kernel if at all possible) it is nice to see more RPM's and
> tarballs available. But how about commercial software like Stata, Applixware,
> CDE, Netscape, to name the ones I use. Do these run at all comfortably under
> EM86, or do they even run at all. Is there software that has problems with EM86?
> (Stata for example still uses a.out).

AFAIK Applixware and CDE are available for native Linux/Alpha. You also
might want to give KDE a try. Some website's got Alpha rpms available,
but I forget which one.

> 7) Does EM86 have problems with Linux (RH 5.0 say). Is EM86 just as good as
> FX!32 or is it a shabby port?

I haven't used em86 myself, but AFAIK em86 is better than FX!32. I think
FX!32 only runs DOS-programs, no Windows apps. em86 runs Linux/Intel
apps.
I've heard em86 is based on FX!32 btw.

> 8) Where do you get EM86, from Digital? Is it expensive, do you buy a license or
> software?

No ! It's available for free ! But it isn't too obvious, from what I
hear.

> 10) How about installation of Linux (RH4.x or 5.x). Does this, in general, go
> just as smooth as RH on Intel Pentiums or are there peculiarities that really
> distinguish the Alpha from the Intel. I would not be interested in a machine
> that I would have to spend 1 to say 8 weeks coaxing along just to get Linux
> installed and then again maybe another couple of months just to iron out the
> kinks and get it running smoothly. I use my machine to work and not to wrestle
> with. I don't want especially to be a bleeding edge pioneer (or is the Alpha
> past that phase).

The difficult part of installing Linux/Alpha is getting Milo to boot the
kernel. If that's dealt with, you probably won't have too much trouble.
I've had quite a lot of trouble getting milo to work, but I've tested milo
for RedHat 5.0, and it worked perfectly.

> 11) In the light of 9) above how about MILO. Does MILO have any quirks or
> peculiarities that distinguish it from LILO (BIOS, disks, controllers, etc).

Milo uses the same drivers and filesystems Linux does, so no need to run
lilo every time you recompile your kernel any more. :-)

> 12) How about ISP connectivity (modems, ISDN, cable modems, etc?).

Haven't tested that out.
