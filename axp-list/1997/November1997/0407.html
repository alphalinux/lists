>> generally it can be said that cisc code is faster to comile than risc code
>> (which is an *advantage* of the risc architecture, not a drawback!).

>Why? Being slower to compile surely seems like a drawback to me. Usually
>it doesn't matter but why should it be an advantage?

The more you cut down the code into little pieces (load/store/add...) before
running it, the less time is needed by the CPU to interpret the commands at
runtime. And, yes, CICS-CPUs are heavily micro-coded (not hard-wired), and
thus behave like an interpreter.

Having to perform only some basic operations, the RISC can take much more
advantage out of syncronous pipes. Ever heard of a CISC-CPU with a 17-stage
pipeline?

You also would not want to say:
"BASIC is the best coice, because it doesn't need *any* compile-time..."
