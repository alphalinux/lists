> I am playing right now with pc164lx machine. It has AlphaBIOS 5.60-30
> on it. This AlphaBIOS has this "nice" property that it does not
> allow me to set a milo location to a floppy or to a small FAT partition
> on a hard drive.

Sigh; yes, I dislike this as well; it's a hold-over attitude of the group
that did AlphaBIOS ("there's only one operating system that will ever be
booted from AlphaBIOS: and that's NT!").

Be glad that they did listen to reason, though; it had even *more*
restrictions on it when it first started appearing on the Alpha platforms
(you couldn't even specify A: for the location of the bootstrapper!).

> This AlphaBIOS also does not allows me to set a name of a program
> to load on the second stage. Only a location is allowed. Luckily
> 'linload.exe' has some defaults so this location better be '\'
> and you load something called 'milo' or nothing else. Sigh!

WRONG! You can put *anything* you want there for a name; yes, AlphaBIOS
*will* complain if it's NOT a path (directory), but it *will* allow you
to save that setting, which is all that is important.

"linload" will then use that path as a filename, and assume it is located on
the same device that it ("linload") was booted from. So you can have
multiple versions of MILO out there, even in different directories if you
want to; I often have 2 or more versions floating around out there for debug
and stability purposes.
