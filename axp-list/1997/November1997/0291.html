Tom/David,

Actually the divide may be as much trouble as the sqrt() function.
Some tests I did indicate any Alpha divide (float/int) is slow.
The table lookup for sqrt() is probably the fastest technique for
an approximation. Maybe you could modify your code to move the
divide/sqrt() out of the loop?? Can it be pre-computed and reused
repeatedly? Maybe you can eliminate both the divide and sqrt()
with a table lookup/interpolate?? That works if you "know" the
range of values the code will use and you can tolerate the
error in the calculations. Stuff I do can tolerate such errors
if the table is reasonably finely sampled.

Wes
