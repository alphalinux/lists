> Shouldn't some /usr/x86/lib entries be coming up with ldconfig -p?
>
> [disco] /usr/local/netscape: ldconfig -p

No, ldconfig -p should NOT show /usr/x86 libs. ldconfig86 should:

[root@silas /root]# ldconfig86 -p
106 libs found in cache `/usr/x86/ld.so.cache' (version 1.7.0)
1 - ELF libz.so.1 => /usr/x86/usr/lib/libz.so.1.0.4
etc...

You might need to run ldconfig86 to update your x86 lib cache.
