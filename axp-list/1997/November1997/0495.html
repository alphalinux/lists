On Tue, 18 Nov 1997, Jeremy Sieminski wrote:
% I have a UDB 166 with 96megs of ram and ~128megs of swap running RedHat
% 4.2. I often use a util called Top which basically displays some stats
% from /proc/meminfo, /proc/uptime, a ps, and a few other things... a very
% handy util if you haven't used it. Anyways, my question is why am I using

I think a lot of people, including myself, use that "handy little
util" about as much as 'ls'.. it's often the first command I type on a
new system ;) The Linux top is actually missing some rather cool
features found in FreeBSD, etc.. That said, you might want to look at
'vmstat' it seems to be better suited for what you're trying to
diagnose anyway.

What happens if you load up Netscape or some other memory intensive
apps? The Samba cache should get out of way in favor of real
applications. If it doesn't, and stuff starts swapping, then you've
got a problem.
