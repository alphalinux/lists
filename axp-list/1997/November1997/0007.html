>
> There is a name server on the network (2 machine network, alpha & one
> other), otherwise the UDB hangs forever also.

"Forever" means here around 20 minutes (a timeout for DNS querries).

Doing a little cleanup in a startup scripts solves that problem as well.
I do not remember details as this was quite a while ago, I do not have
anything on hand at the moment, and this may slightly differ between
distributions as well, but I did such modifications on quite a few Alphas.
Making sure that loopback is started before you attempt to initialize
your ethernet interface is a good first step. You may want to turn off
sendmail before you clean up everything (on a small "table" network,
where sendmail is used just for an occasional error message, you may
want to run it from inetd in any case or maybe even not at all).

In any case, it is perfectly feasible to run UDB on a small local
network without any nameserver in sight. Just keep this /etc/hosts
table up to date. Your routing table needs two entries total -
a loopback interface and a network IP as a default through eth0.
