>
> I think this is sort of a FAQ; but since this is a new system, maybe
> it's at least a new variant of the problem. When we boot up the system,
> it always comes up with the year=1925. (This prevents NFS mounting, so we
> have to reset the date, and then can mount NFS file systems -- quite
> annoying.) Does anyone else have this problem -- and maybe a solution?

Yep, well kind of: for some reason the date isn't picked up properly
from the BIOS. So we use xntp as a time daemon. During booting we use
a ntpdate call to several xntp servers to get the right date.

Hope that's clear (I'm not familiar with the whole xntp philosophy
though ...).
