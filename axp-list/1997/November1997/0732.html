martonak@nero.physik.uni-mainz.de (Roman Martonak) writes:

> > What kernel are you running? If you're using the stock redhat one, it
> > has broken misalignment trap handling, which could cause misalignments
> > to crash your machine. The same holds of
> > 2.0.30+alpha-patches-2.0.30-0.2a.gz. You should get a 2.0.30 & patch
> > it according to my kernel patch notes.
>
> I read your kernel patch notes, and found that I am not quite sure about
> my English: to make sure I understand what you mean, I would like to ask
> what precisely is meant by "pristine" and "stock" kernels, respectively ?
> Is the "stock redhat one" the precompiled one which is installed during
> RH installation or the one which one obtains by compiling the RH kernel
> RPM ?

I see that you're refering to the compiled kernel image, whereas I'm
refering to the source tree from which it is built.

The stock redhat kernel is the source tree that comes with Redhat 4.2,
which is a heavily patched version of 2.0.30. It really shouldn't be
called 2.0.30, except that it has to be called 2.0.30 so that the
modules can load. The pristine 2.0.30 is the original and
unadulterated 2.0.30 source tree which can be gotten from
ftp://ftp.kernel.org/pub/linux/kernel/v2.0/linux-2.0.30.tar.gz.
