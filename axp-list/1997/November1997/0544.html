I don't think that you read my post closely enough. I ftp installed to
make sure that the cd WAS worth buying.
I have no problems buying redhat on CD. CD is a extremely useful media
because it helps turn installs that would take hours to complete into
installs that take minutes. I _have_ problems buying software that isn't
troubleshot well, and personally I can't stand code that is poorly written
or stupidly implemented.
If I remember correctly (sorry to drudge up any past) redhat's first elf
release was 4.0, and everyone in the Alpha community rejoiced. Yay. I even
went so far as to order a copy before the release date. I was not pleased
when it decidedly refused to install on a machine that was clearly stated
to have been supported. And this is the reason why I think that having the
product on ftp BEFORE it is stamped onto cd is an intelligent idea,
because people like me can try it out and decide as to whether it is worth
a crap or not.
But hey, how you run your business is your own business. I just would
like to see linux's "messiah of distributions" have the intelligence
to avoid making the same mistakes that Bill Gates does.
Peace.

Eric.
