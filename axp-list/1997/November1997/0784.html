Don't make libF77 & libI77. Use libf2c.zip & just make libf2c.a &
libf2c.so. Did you use:

CFLAGS = -O2 -DNON_UNIX_STDIO -DIEEE_drem -DWANT_LEAD_0 -DAllow_TYQUAD

Only the -DNON_UNIX_STDIO is necessary. The -DIEEE_drem might help
efficiency sometimes, and the last two are purely options.

You're doing this under RH4.2?

You might try getting the RH5.0 source rpm for f2c, and building from
that. It includes a newer version of f2c. You won't be able to
install the RH5.0 binary package (since it's compiled against very
different libraries), but you should be able to build the source rpm
(assuming that RPM hasn't changed too much).

I can also email you f2c.h, f2c.1, f2c & libf2c.a, if you'd like.
