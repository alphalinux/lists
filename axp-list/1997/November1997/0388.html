> I have an EXABYTE DAT tape drive. Linux recognizes (in that it
> acknowledges that there is an EXABYTE tape backup device attached to the
> SCSI card) when it boots.
>
> I'm not sure if it is truly working, since attempts to
> mt -f /dev/st0 rewind don't work, and the /dev/tape device doesn't even
> exist. Can anyone send me their backup script so I can see how it's done?

First point (going from the "easy stuff"), /dev/tape is normally a symbolic
link to the "real" tape device. Thus, if the proper device is /dev/st0, you
would use the command:
# ln -s /dev/st0 /dev/tape

You could alternatively use an appropriate mknod command to define /dev/tape
with appropriate major/minor numbers, but I wouldn't recommend that; it's just
not "standard" behaviour.
