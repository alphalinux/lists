No solution, but description of a potentially similar problem (BTW, it
would certainly help if you gave some more technical details)

My machine is a pc164 (500MHz), 128MB RAM, only IDE subsystem. I have
complained about the last issue a lot, but the bottom line is that
this is my fault. It makes for a sometimes slow, unresponsive system
that was nevertheless stable (uptime 2 months+) during the last 6
months. During that uptime the machine was used in heavy duty
numerical simulations, including some disk I/O intensive
post-processing (that's how I truly noticed the shortcomings of the
IDE subsystem). The machine is running RH 4.2; however, I got a clean
2.0.30 kernel source plus the famous axp-diffs from June (July) that
managed to clean up a lot of problems; this is what I am still
running.

Since about a week, I can pretty much at will kill the machine
with an "Unable to handle kernel paging request at virtual address"
when transferring lots of data (ftp of large files, tar to
a remote tape drive, cp via nfs etc.) It's not fully
reproducable, but after a while the machine dies. I should add that
increasing /proc/sys/vm/freepages so far did not help at all as far
as I can tell (As I said, the problem is not completely
reproducable, but I have seen a crash with the default values during
e.g. the fifth 60MB file to ftp; rebooted (power off/on, as a
reset leads to even stranger things on this machine), try again
after increasing /proc/sys/vm/freepages by a uniform factor
of 3, and it dies during the fourth file ??? That's why I doubt
that it has to do with these parameters)

So, the only thing that changed recently is that I brought all
rpms that are of interest to me to the lastest level (update
directory from the local redhat mirror). Could e.g. the latest
NetKit suddenly cause trouble with the kernel that I use (how
would I go about finding that out...)? Personally, I am more
worried that I am seeing signs of some developping hardware
problem.

Nevertheless, has anyone suggestions which kernel + axp-diff (+
possible latest milo?) to try at this point and to see whether
this changes anything. As I said, the 2.0.30kernel + axp-diffs from
June served me well until a week ago, so I saw no need to try out
the latest new stuff...

Thanks for suggestions!
