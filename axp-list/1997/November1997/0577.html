>>> BELABBAS Mohamed Ali said:
>
> I wanted to try Xfree with more than 256 colors, so I inserted my old
> trident AGi 9440 in my multia VX-42 overclocked to 266Mhz. Doing so, the
> UDB simply don't start up. No signal from the TGA nor the Trident !
>
> Got an idea ?

Yes, probably the BIOS emulator built into the UDB console, is not up to
the task of emulating the BIOS code on that card. This is not uncommon, as
the version of BIOS emulator in there is quite old, and has shown itself
to be very fragile with the more "adventurous" BIOS code-carrying cards...

> I supposed it came from the firmware but when i tried to update it with
> the update files for the DEC AXP PCI33, it turned in some kind of srm
> console. OK . Then, I typed update srm and it reported a jumper problem.
>
> Are there any jumper to permit an update of the rom ?

Uh, that wasn't the right thing to do; the UDB is *NOT* exactly like the
NONAME (AXPpci33), hardware-wise, and one should not attempt to run the
former's console code on the latter. Please go to the "interim" directory
on gatekeeper.dec.com, ie /pub/DEC/Alpha/firmware/interim/, IIRC), to get
the last version of UDB code that will EVER be available.
