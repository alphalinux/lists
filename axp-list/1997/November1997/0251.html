Bryan B. Thompson wrote:
>
> Ok. What is the story concerning gcc performance and optimization for
> Alpha? How does this tie in with the egcs story?

The folks working on the egcs project were aware of some problems
in the way the g77 frontend worked with the gcc backend and they
made matching changes to both to fix code generation for loops.
That change created dramatic performance improvements for my code
and also a CFD code from France they were using to test with.
If you want to read about it, use:

http://www.dejanews.com/

and search for 'g77 and comp.os.linux.alpha'. Look for a post
from a person who's first name is Toon (I think).

I haven't run my C linpack test yet to see if the change works
for the C frontend or not. I assume if the g77 performance has
improved that much, they also did the change for C. Don't know
for sure though. One warning though. Simple DO loops do not
benefit much from the change (25%). Only reasonably complex
loop start flying, on the order of 20+ operations in the body
of the loop. That means they haven't gotten results as good
as the real DEC compiler, which can actually pipeline from
one loop iteration to the next. But, it is a lot faster than
it was.
