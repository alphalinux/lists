I ran into a similar bind a while back with one of our UDB's. One thing
you'll probably want to double check is the SCSI id of your hard disk. I
believe the alphabios assumes 0 to be the default (correct me if I'm
wrong please) so if you set the physical jumpers on the drive to
something else you need to change the number in the string of boot
parameters from within the console. I believe the reason that this
doesn't matter once milo loads is because linux will always look to the
first hard drive on the scsi chain, and doesn't really care what the id
is explicitly.
