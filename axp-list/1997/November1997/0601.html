Hi!

I have been using only milo for a long time in my noname. Recently, I
decided to try the srm (v1.6) as well (to investigate NetBSD progress).
This resulted in a very weird problem. I have two network cards in my
machine, one 3c509 and one NE2k clone (Eagle Technologies). Here's the
thing:

If I boot linux via milo only, no problem, both cards work. If I boot
linux via srm and then milo (milo.dd on a disk), the ne2k probe code hangs
the machine completely on module load (yep, 8390.o and ne.o are modules on
my system). The 3com still works ok.

The ne card is using irq 5, and I got the system to print an infinite
number of 'interrupt 5 for unknown device' or the like upon module load.

So, the only difference being the srm, I figured it was screwing with the
interrupt controller somehow (even though I think linux should reset any
such actions on boot). Therefore I tried to use the 'isacfg' utility in
the srm to define that the card was really there. I don't know if this
really has any effect besides on digital unix?

Anyway, when I tried to add an entry for the ne using

>>> isacfg -slot 3 -dev 0 -mk -iobase0 340 -irq0 5

I got an error like 'All ISA entries are used' no matter what I tried. The
standard devices shows up with isacfg -all, and these may not be removed.

Ok, so the question got a little long. Does anyone know what's causing the
problem and/or what I should do?

How about the isacfg utility, why can't I add any entries?

I have no problem using only milo for linux, but the ne won't work under
NetBSD either, and I figure the problem is related.

Thanks,
Andreas
