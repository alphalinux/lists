On Tue, 18 Nov 1997, Jeremy Sieminski wrote:

> I have a UDB 166 with 96megs of ram and ~128megs of swap running RedHat
> 4.2. I often use a util called Top which basically displays some stats

> Normal:
> 11:05pm up 2 days, 12:30, 1 user, load average: 0.13, 0.04, 0.01
> 30 processes: 29 sleeping, 1 running, 0 zombie, 0 stopped
> CPU states: 3.1% user, 5.3% system, 8.3% nice, 91.7% idle
> Mem: 94152K av, 25424K used, 68728K free, 15592K shrd, 6696K buff
> Swap: 131344K av, 0K used, 131344K free 9344K cached

> Abnormal:
> 1:17am up 1 day, 14:26, 1 user, load average: 0.00, 0.00, 0.00
> 30 processes: 29 sleeping, 1 running, 0 zombie, 0 stopped
> CPU states: 0.7% user, 1.2% system, 1.8% nice, 98.1% idle
> Mem: 94152K av, 91904K used, 2248K free, 15584K shrd, 58440K buff
> Swap: 131344K av, 0K used, 131344K free 10752K cached

I'd say you have the labels backwards. The top one ought to be
'abnormal'. The only time I can think of when this would be normal is
immediatealy after bootup. Free memory is wasted memory; whatever free
memory you have should be used as disk cache, and since this appears to be
where the memory is, and as you're not using any swap, this looks
perfectly normal to me. Linux always keeps some memory 'free' for quick
loading of programs (is this tunable? I imagine it should be a /proc
setting in the development kernels). Anyhow, I see nothing wrong here.
If you start seeing swap appear, I'd start to get concerned. My system
uses some swap anyway (not sure for what, about 2MB worth) even under no
load. I assume I exceeded the physical memory sometime and it the system
decided it wasn't really that important and just left it on disk (an Intel
system w/20MB RAM). Util that point, or if you're noticing dignificant
perforance loss, don't worry. It's normal.
