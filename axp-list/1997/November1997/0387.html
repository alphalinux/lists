Well I hate to keep whining to the list about my install problems, but I've
made progress but I'm at an even more unusual spot. To recap:

I have an as255 running SRM. Redhat was ->NO<- help but I got the as255-srm.img
from gatekeeper to boot from.

Then it wanted a ramdisk. The Redhat 4.2 CD image images/ramdisk didn't work
(are we seeing the pattern yet?) but the one I got via ftp (from images/ on
redhat ftp) does. So I'm into the install program now. But fdisk wigs out with
my disk.

On bootup, this is what the linux kernel says:

scsi0:ncr53c8xx (rel 1.12c)
scsi: 1 host.
Vendor: DEC Model RZ26F ... Rev 630J
Type: Direct-Access ANSI SCSI rev: 02
Detected scsi disk sda at scsi0, channel 0, id 0, lun0
Vendor: DEC Model RRD45 ... Rev 0436
Type: CD-ROM ANSI SCSI rev:02
Detected scsi CD-ROM sr0 at scsi0, channel 0, id 4, lun0
scsi: detected 1 SCSI cdrom 1 SCSI disk total
ncr53c810-0-<target 0, lun0>: FAST SCSI-2 100ns (10MB/sec) offset 8
SCSI device sda: hdwr sector-512 bytes. Sectors=2050860 [1001MB] [1.0 GB]

...
Partition check:
sda: sda1 sda2 sda3 sda7

So there's obviously a disk there and since digital unix boots, it obviously
works and the partitions are for real. But even though the linux kernel
recognizes the disk, once I get into the fdisk program it says the disk has 0
heads, 0 cylinders, etc, and shows a heading for the partition table but no
entries.

It said it was using /tmp/sda, so I switched VT's and looked around. There's no
/dev/sda, but /tmp/sda has major number 8 and minor number 0, in case anyone
knows what the correct numbers are supposed to be.

Any suggestions on what to try? There's only one ramdisk.img but I assume that
since the kernel claims to understand the disk, there's really no problem once
I get to the kernel whether I'm using SRM or ARC or AlphaBIOS to get there. And
it seems that the ramdisk image should work for any install kernel I can get
going, but obviously I'm not an expert on this stuff.

Don

PS: many thanks to the people who've gotten me this far. I probably should

added to the FAQ/howto's. RedHat certainly isn't making much of a contribution
as to how to do any of this stuff other than hosting the list itself.
