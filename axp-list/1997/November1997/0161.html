Red Hat 4.2 distribution for Alpha comes with XF86Setup program.
Very nice, I think; I used this thingy on Intel and it worked pretty
good in all cases I tried. There is a catch, though. The program
requires VGA16 server to run and I do not recall seeing an example
of that for Linux/Alpha.

Is this program presence in the distribution a mistake or there is
some workaround to make it run? I tried a bit of "cheating" without
much luck (with SVGA server). XF86Setup starts but later falls
flat trying to connect to a server socket.

Oh, I can make X windows to behave in my current configuration. I did
that already too many times. :-) But it would help to have
XF86Setup operational.
