On Mon, 10 Nov 1997, Mike Howard wrote:

> > If you want to boot via SRM, then grab a "-s" image, put it on a floppy
> >and put the ramdisk image on another one. Then enter SRM and type:
>
> >boot dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1"
>
> >Once the kernel is booted, switch the disks (it will ask) and proceed. If
> >the kernel complains about hardware and is unable to proceed then you may
> >need another image (I succeeded at the third time when booting on an
> >AlphaServer 300).
>
> And then what? I used that procedure in order to get the RH 4.2
> install to work and succeeded in installing RH 4.2 on a second hard
> drive. Unfortunately, I haven't figured out the proper incantation to
> boot it from floppy and don't have `aboot' properly installed on the
> HD so SRM can't find it.
>
> Looks to me like I need to re-partition the HD using the BSD
> [in]compatible partitioning buried in fdisk, install aboot on the HD,
> re-install RH 4.2 on the HD, ... but I'm only guessing.

Obviously you did read the information provided at
'http://www.azstarnet.com/~axplinux/' and left a few starting cylinders of
the disk reserved for booting purposes, didn't you?

So if you have a free space at the beginning of the disk (100 kB or so),
all you need is to have the disk BSD-disklabeled.

If your disk is partitioned and you have no extended partitions and no
more than 8 partitions, then you may easily convert it on-line. Just:

1. Start `fdisk'.
2. List your partition table and note all boundries.
3. Delete all partitions.
4. Enter the BSD disklabel menu.
5. Create partitions using the noted boundries.
6. Write, quit and reboot.

If you have extended partitions, then the conversion is still possible but
you need to skip the amount of sectors per track at the beginning of the
partitions which were previously contained within the extended one (not
tested, though).

Now that you have just run `swriteboot' with appropriate parameters (read
the man page first) and, optionally `abootconf'.

That's all. HTH.
