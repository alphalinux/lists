On Sun, 23 Nov 1997, Dave Wreski wrote:

> Anyone know where there is a HOWTO? If not, can someone give me some ideas on
> how to do it? ldd tells me libm.so.6 is missing. I have libm-2.0.5 currently.
>
> Is this the difference between glib and our old lib? Which is which?

You have confused them, they are the same and you have glibc. The old
libc was called libc.so.5 (and libm.so.5). Your problem is that you
have a newer version than the one your program expects/was compiled
against. All glibc libraries version 2.x should have a link which ends
with .6.1 to them.

On my system, libm looks like this:

ajo@wopr ~ >ll /lib/libm*
-rwxr-xr-x 1 root root 773393 Oct 9 17:50 /lib/libm-2.0.5.so*
lrwxrwxrwx 1 root root 13 Oct 24 03:37 /lib/libm.so.6.1 ->
libm-2.0.5.so*

If someone is looking for libm.so.6 this probably means that the program
is compiled against glibc 1.99. Either recompile the program or get the
old libraries (libm-1.99.so and link libm.so.6 to it). The later solution
is not so good since this means you will have two copies of the libm
library in your memory at the same time.

> Are there any other things I should know before trying to get this to work? If
> there isn't a HOWTO currently, and people think we need one, I'll write up what
> I did to make it work, once I do..

Well, the right thing to do is simply to recompile.
