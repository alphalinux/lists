I'm using Redhat 4.2 on an Alpha (multia) system. Since this machine is
used as a masquerading router, I wanted to use firewalling to block
certain 'dangerous' ports from accessing them from the Internet.

But, firewalling rules don't seem to work. For example when I test the
following :

$ /sbin/ipfwadm -Il
IP firewall input rules, default policy: accept
acc tcp 127.0.0.0/8 anywhere any -> sunrpc
acc tcp HomeNetwork/24 anywhere any -> sunrpc
rej tcp anywhere anywhere any -> sunrpc
acc udp 127.0.0.0/8 anywhere any -> sunrpc
acc udp HomeNetwork/24 anywhere any -> sunrpc
rej udp anywhere anywhere any -> sunrpc

Which should disallow access to the portmapper from outside, but I can
still talk to the portmapper from wherever.

I tested with setting the default policy to deny and trying to set
'accept' rules, which didn't work either. The 'deny' worked perfectly, but
what I set to accept wasn't accepted.

The machine is a Multia, Redhat 4.2, kernel :
Linux morris 2.0.30 #1 Sun Nov 9 01:36:52 CET 1997 alpha unknown

(from the linux-source rpm with rh4.2).

Any idea what is going wrong and how to fix this ?
