It looks like I am following my own posting, but....

I did a bit of digging on my hard drive and checking sources.
It turns out that all underlying stuff for XF86Setup is written
in Tcl, so one can look there. Turns out that it is possible
to run that utility without VGA16 server IFF one runs it from
an already running X.

Contrary to what you may think that would be very useful as this
would allow expanding and tuning up "barely there" installation
of X. But there is one big BUT. As it is well-known Tcl is
hopelesly broken on Linux/Alpha (vide an ongoing Grail fiasco
which can be traced back to Tcl; apart of that Tcl fails on Alpha
its own test suite). The net effect is that XF86Setup goes
through all motions, whining from time to time, and locks up
when trying to write an new configuration file. Big bummer!!
Not a single byte is written.

Michal
