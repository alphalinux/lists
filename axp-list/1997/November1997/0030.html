Hi folks - Having just upgraded from RH4.1 to 4.9.1
(upgrade was completely smooth, kudos to RH!) I am trying
to get g77 recompiled for it.

(I used to use g77-0.5.20, the compilation of which
required lots of adjustment.)

I got g77-0.5.21, gcc-2.7.2.3, patched the tree as per
g77 instructions, and it died in the "stage 2" compilation,
because the short programs that it uses (genattr, genemit, etc)
wouldn't run (they compiled OK, but the system just complained
that it couldn't find them.) So, I applied a single patch
from the RH gcc-2.7.2.3 modifications,
"gcc-2.7.2.3-glibc2.patch" - which changes the ld library.

Then, the whole compilation finished OK.
I can use it to compile little "hello world" type programs.
However, when I try it on my own (bigger) programs,
I get more of that funny error, "command not found" , even
though its sitting right there!

FYI, I'm using the pc164.img distribution kernel (haven't made a new one
yet) box is a 433MHz PC164, 64Mb memory, from DCG.

Has anybody seen this sort of thing before? Any suggestions
what to do about it?

(BTW, I did try applying the full set of RH patches and then
the g77 patch, that's a screaming disaster...)
