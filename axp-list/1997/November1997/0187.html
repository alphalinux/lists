On Tue, 11 Nov 1997, Mike Howard wrote:

> Do you, by any chance, have the incantation necessary to boot the hard
> drive -after- I've installed RH 4.2?

If you have the root filesystem installed on the hard disk and no way to
boot from it, then you may always use the boot floppy, e.g.:

boot dva0 -fi vmlinux.gz -fl "root=/dev/sda3 ro"

if your root filesystem is located on the third partition of the first
hard disk. What you specify as flags is actually the kernel command line.
You may find more information about it in "BootPrompt-HOWTO", in
'/usr/doc/HOWTO' (where RH installs it).

> reliable guide to the magic spells. Is there a real document or is
> this an `oral tradition'?

If you read the RH installation manual, then you must have seen a
reference to 'http://www.azstarnet.com/~axplinux/'. If you have not, then
I still recommend it to you.

> BTW, I'm still not convinced that I should use fdisk to do it rather
> than using the Dec UNIX format utility [name forgotten right now]. I
> need to retain Dec UNIX on /dev/rz0 and put RH on /dev/rz1. Do you
> have any experience one way or the other? It seems to me that I can
> build the correct disk label under DEC UNIX, but then risk the
> possibility that Linux won't be able to read it. Contrariwise, the
> documents I've read hint that either approach works, and using the Dec
> UNIX tools is preferable.

I have no idea. I used `minlabel' upon first installation. Later, one
of our DU disks in another machine had to be replaced and the only disk
available was the one containing Linux/Alpha. I was told that DU
recognized the label correctly. Later, I installed Linux on another disk.
I used `fdisk' at this time. I experimented a little and noticed that if
a "legal" DU label is present (i.e. with the third partition covering the
whole disk), the kernel ignores it and all partitions after it. So there
actually may be a problem if a disk is partitioned within DU but I did not
test it.

HTH.
