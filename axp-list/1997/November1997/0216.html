About 2 months ago I posted here a description of the signal 11 problem
appearing from time to time on my UDB, and got no answer at that time.
I am running RedHat 4.2 Linux on a UDB (166MHz, 32 MB RAM, 2.1 GB external
SCSI hard drive, 96 MB swap). When compiling programs with gcc, compiler
crashes with signal 11 sometimes occur (not often), in an unpredictable way,
and after typing 'make' again, compilation continues O.K. The problem can be
triggered to occur with a probability close to 1/2 when compiling a larger
Fortran program (about 150 kB of source code) with g77-0.5.21, and in that
case the crash produces a huge core file (about 20 MB) which corrupts the
filesystem making it impossible to delete the file and recover back the
disk space unless the filesystem is repaired. The probability of crash appears
to strongly increase with increasing optimization level (the above value of
1/2 corresponds to -O2).
In the GCC-SIG11-FAQ file as it comes with RH Linux 4.2, precisely
this sort of problem is described as being typically hardware related;
however, the whole discussion applies to Linux on Intel platform. I have
contacted the author of this FAQ, and he says that he has NEVER heard about
anybody having this problem on Alpha. So my question now is: has anybody
experienced such or a similar problem on Linux/Alpha, and is there perhaps
any knowledge whatsoever about the origin and possibly a way around ?

Thanks a lot for any suggestion,
