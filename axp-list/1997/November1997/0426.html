>which works fine when booting manually. But is there a way to set
>MEMORY_SIZE in the OSLOADOPTIONS of the ARC Console boot selector,
>so that this will be set when autobooting? It seems that in MILO,
>one can manually put two commands on one input line by separating
>them with " ; ", but the semi-colon didn't show up when I tried to put
>it in OSLOADOPTIONS. In other words, is there another way to put the
>following two MILO commands into OSLOADOPTIONS?

We've run into the same problem at times. I'd suggest two things -
First, one of the newer milo's/kernel combinations probably won't require
you to set this memory parameter anymore. As we've found they clean up a
lot of the flakiness in earlier releases.

Second, if you are happy with how the machine runs and don't want to
change anything, you should be able to reboot automatically, meaning this
will only be a problem when powering up. I believe setting the memory
down from 128 (I've found even 127 will do the trick) should stop the
machine from hanging on reboot.

That said, no, there isn't much you can do. I've discussed this a couple
of times with Jay Estabrook (Jay - any new developments please correct
me) and there isn't any way to pass multiple commands from the ARC
console to milo, and there is not way to preset what milo will do from a
cold boot.
