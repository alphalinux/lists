Wait 5 more minutes. :-)

It looks like your networks is misconfigured and/or nonfunctional.
Most likely something (amd?, sendmail?) waits for a response from
a DNS server and not getting it. A timeout for that is 20 minutes.

Once you booted turn off these services (and maybe net in general).
It is enough to rename various links in your /etc/rc.d/rd?.d
directories which start with 'S' not to start with S. Like this,
but I making it up "mv S60amd +S60amd". These things will not
start until you sort out things and restore original link names.

The other way is to boot "single" and do this renaming before
you continue with a multiuser boot but you may need to remount
root partition read/write if it will be not that way.
