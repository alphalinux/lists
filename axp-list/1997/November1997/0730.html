"Jonathan D. Sturges" <jonathan@usgcc.odu.edu> writes:

> I've been trying, casually, to get em86 running. I've got a 2.0.30
> kernel, which I tried patching with alpha-patches-2.0.30-0.2a.
>
> However, unless I'm *really* missing the boat, these patches do not seem
> to mesh with the stock 2.0.30 kernel code. I got about 15 rejects, and
> more than half of the patches seemed to be in "reverse" order (ie, the
> destination file is the source). Patch stops you and asks, "Assume -R?"
> Finally, I noticed that it changed the value of ARCH in the top-level
> kernel makefile to be "i386", which completely hosed everything!

Yes, you're really missing the boat. Download & untar a pristine
2.0.30. You must be patching against the kernel supplied by Redhat,
which is a heavily patched 2.0.30. See my writeup of how to properly
patch the kernel for alphas.
