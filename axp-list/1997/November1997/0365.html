> so i can compile a new kernel with the fix/patch/whatever??

Those changes (to allow XLs to run Millennium Xserver) are included in the
new alpha-patches-2.0.30-0.3.gz that are in:

gatekeeper.dec.com:/pub/DEC/Linux-Alpha/Kernels/

The above includes the EM86-related ones (0.2 and 0.2a), along with updates
to the NCR53C8XX driver (rev 2.4a, supporting NCR875), and tulip.c-083.

These are applied to a pristine 2.0.30 source tree in the following manner:

cd /somewhere
tar xzf /archive/linux-2.0.30.tar.gz
cd linux
zcat /archive/alpha-patches-2.0.30-0.3.gz | patch -p1 -E >&../patch.log

PLEASE, make /somewhere NOT BE /usr/src; that will cause confusion over the
correct header files that the system will use.

Good luck.
