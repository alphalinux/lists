Where you stated:
>This discussion makes me curious. Isn't the SRM console firmware
available at
>
> ftp://ftp.digital.com/pub/DEC/Alpha/firmware
>
>along with the firware load utility? I know it's for booting DEC OS's,

>LINUX? Can it boot NT? Will it solve any of Michal Jaegermann's problems?
>Also, will it allow remote boot?

I work with Michal, and while, yes, SRM and aboot may be _AN_ answer, in
this case it's not THE answer. If you wish a machine to boot both Linux and
NT, you need an AlphaBIOS firmware installed.
Really the bitch is as follows:
AlphaBIOS doesn't allow us to set up a loader entry that points to the
floppy disk.
What this means ( as far as I can see so far) is that in a setup like this
one HAS to install NT first, so as to create and access a FAT partition as
the first one on the root hard disk. While we can certainly do this, it has
some limitations:
What if the partition table or master boot record gets slightly hosed, and
we REALLY want to boot from an "emergency" floppy? Bummer.
Of course, as AlphaBIOS does allow us to use a CDROM for this we could
always create a custom boot CD, this is kind of a pain in the ass...
