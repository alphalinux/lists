interestingly enough there is enough of a worry that DEC has a toggle bit in the FPCR
( floating point controll register ) of the 21164. on the underflow, u can set the
hardware to either plug in a 0x00, or a denormal number. The 21064 does not have this
feature.

Linux, the kern, does not care too much about the FPCR, and doesn't even bother with
the new bits in the FPCR that controll this behavior. ergo u get what ever someone
else has set if u dont bother to ( or know how to) change the FPCR register to suit ur
application.

thanks for the interest.
PS, the denormal is used by the solaris/java/alpha port of java 114. a true zero is
detected, and rejected by the java compile scripts as unacceptable!!!!!! a denormal, I
should say a IEEE denormal is needed.
