On Thu, 6 Nov 1997, Steven Kokinos wrote:

> >A search in some newsgroups and mailinglist-archives found suggestions
> >to "echo my_max > /proc/sys/kernel/file-max" in some rc-file.
> >
> >Is this in your opinion a Safe Thing To Do [tm]?
>
> Absolutely, this is part of what we modify on all kernels, both intel and
> alpha. The standard open files is set *very* conservatively and can be
> much higher without causing any particular difficulty, and in fact cleans
> up several problems, especially when using a busy web server in
> conjunction with databases such as MySQL, or msql.

I do this to all my system, otherwise I run out on many of em.
Only problem is the per user file limit (for non root users) is hardcoded,
and is an issue for non-root servers (httpd, mysqld, etc).

I missed the start of this so am not shure if it has been said
but.. When you increase file-max you should also increase inode-max.

If you increase file-max by a factor of 3 you should also do the same with
inode-max, otherwise you may max out allowed inodes before reaching you
file limit.
