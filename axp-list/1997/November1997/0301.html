>
> The following (stupid) program:
[calculates a lot of square roots and stores them in an array - snip]
> takes 21 seconds when compiled with g77 -O2 -funroll-loops and 11
> seconds when compiled with Digital f77 -fast on our 500 MHz 21164.
> This difference should be mainly due to the efficient implementation
> of the Digital sqrt library function.

Actually if this would be really true that would be very surprising.
It is really hard to slow down any sane implementation of sqrt to
that extent. Newton-Raphson converges very fast and there are some
shortcuts (Alpha does not have a sqrt in a hardware, does it?).

In a light of recent annoucements about egcs based g77 I would strongly
suspect that this is array index calculations which kill here g77
performance. Anybody with egcs-g77 handy to check this hypothesis?

Fixing sqrt is likely a magnitude times easier than fixing a compiler
code generator and optimizer. :-)

Michal
