On Tue, 25 Nov 1997, Benjamin J. Greslick wrote:

>
> Regarding the DEC Alpha PC164 128MB RAM that I have and the crashes I
>hav ebeen experiencing, I also have a 256M swap partition (one partition).
>I have recently read in a few Linux books (based on Intel Linux) that you
>can have upto a 128MB swap partition, but multiple swap partitions. One
>book claims that if your swap partition exceeds 128MB, it will only use
>the first 128M.
>
> My question is could the larger partition be causeing problems, or does
>Alpha Linux support larger swap areas? (I'm running kernel 2.0.30).

It does... and you are right at the limit. The formula for maximum swap
size takes into account page size. The page size of the Alpha is twice
as large as Intel so the maximum swap area is twice as large. So the
maximum is 256M which you are right at. You may want to lower the swap
space to the amount of RAM you have (That is the usual thumbrule over
about 32 MB) and see if that helps.
