Just thought I should post a summary of the answers to my question
about the Cogent em110. Well, not answers really, as I just got one.
Makes you wonder about how many Alpha/Linux users there really are.
Feels like we're a select bunch...

Anyway I got the em110 to work. It has problems detecting what kind
of ether is flowing through the net, i.e. 10 or 100mbit. If I have
it plugged in on boot it will detect 100mbit correctly about 50% of
the time, otherwise it will just sit there looking stupid until you reboot.

If you twiddle the driver a bit, you can hardcode it to 100mbit and it
will work everytime. This was driver 'tulip.c:v0.83'.

capel@crim12b.nsls.bnl.gov also had problems with memory alignment errors,
but this was with driver 0.77 and I don't get that with the 0.83, it
includes an Alpha specific hack to get around that.

Summary: Add this to 0.83 if you use 100mbit. Seems to work. Change the
0 to 1 if on 10mbit.

#define TULIP_DEFAULT_MEDIA 0
#define TULIP_NO_MEDIA_SWITCH

If 0.77, also increase size of TX_RING_SIZE and RX_RING_SIZE to 64 to avoid
alignment errors (courtesy capel@crim12b.nsls.bnl.gov).
