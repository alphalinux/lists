First you (or someone else) need to figure out how to fix it.

> > I think you should add the -mno-fp-regs compile flag. This is used on all
> > modules in the kernel, and I have had modules malfunctioning without it.
> > This does not solve your problem though.
>
> Shouldn't matter... I suspect that all -mno-fp-regs does is speed
> things up a bit, rather than functionally change anything.

Please, do not assume to much about things you have not checked. I am
telling you that while porting a device driver for my sound card, this was
the most hard-to-track bug. The driver occasionally killed programs around
the system when a particular part of it was used.

This seems to be due to the kernel not saving any fp-regs at the entry
point. I guess this would be too time consuming. Instead it relies on no
part of the kernel using the fp regs. Therefore, you need to instruct gcc
NOT to use the fp regs. Even if there is no fp math in your code, gcc can
sometimes use an fp reg as temporary storage instead of allocating stack
for it.
