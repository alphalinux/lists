> Hi everyone,

> I am trying to figure out how the select is working on my alpha. I have a
> UDB 166 (that's right the one that cannot be upgraded :-( ) running

> around 10 (elapsed time) to return. The systems is not running anything

Ok, try this source. When I run it, it gives me the correct timeout of one
second.

Here is my result using glibc-2.0.3:
ajo@wopr ~/src/test/select >time ./select
0.016u 0.032s 0:01.05 3.8% 0+0k 0+0io 60pf+0w
^^^^^^^
One second + startup time

And this is using glibc-1.99 from RedHat-4.1,
RPM package version 0.961212-4:

ajo@nova ~ >time ./select
0.015u 0.026s 0:01.06 2.8% 0+0k 0+0io 51pf+0w

Both of these machines are running Linux-2.1.36 though.

--------------------------- select.c ----------------------------
#include <sys/time.h>
#include <sys/types.h>
#include <stdio.h>
#include <unistd.h>

int main (int argc, char *argv[])
{
struct timeval timeout = { 1, 0 };

select (0, NULL, NULL, NULL, &timeout);
}
----------------------------------------------------------------

If this is not working for you, you need to tell us what type of system
you have, what is installed on it, etc. I can also make my precompiled
binary available to you if you wish to check if it is the kernel or the
compiler/glibc which is misbehaving.
