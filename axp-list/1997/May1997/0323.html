Running the tcl test-suite, i found undefined mathematical operations like log(-1) always
generate SIGFPE at my avanti. Looking inside the log() function, if found that log(x) for all
x < 0 executes the following:

static double zero = 0.0;
return (x-x)/zero;

This should result in NaN, but it doesn't. Instead, the division by zero raises SIGFPE.
Surprisingly, leaving the generation of NaN's or infinite values to the compiler works:

isnan(0.0/0.0) returns 1 and isinf(1.0/0.0) returns 1, as expected.

Does the alpha's FPU not support infinities and NaN's ?
