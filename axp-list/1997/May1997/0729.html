I installed redhat-4.2, on a miata-433a, it's been kinda unstable, with lotsa
of lags, and very poor net performance.

I'm trying to build a miata 2.0.29 kernel to update the tulip
driver.

I downloaded 2.0.29. Patched it with:
-rw-r--r-- 1 root root 180267 May 23 16:40 miata-patches-axp2029-29mar97

Dropped in tulip.c:
-rw-r--r-- 1 root root 66639 May 23 16:54 tulip.c-076

Edited the makefile for ARCH=alpha.

Did make menuconfig, selected miata.

Then said make depend:
make[2]: Entering directory `/usr/src/linux/fs/ncpfs'
if [ -n "dir.c file.c inode.c ioctl.c mmap.c ncplib_kernel.c ncplib_kernel.h sock.c" ]; then \
/usr/src/linux/scripts/mkdep *.[chS] > .depend; fi
make[2]: *** [fastdep] Error 138
make[2]: Leaving directory `/usr/src/linux/fs/ncpfs'
make[1]: *** [fastdep] Error 2
make[1]: Leaving directory `/usr/src/linux/fs'
make: *** [dep-files] Error 2

Seems kinda strange....

This happened shortly before a crash last night:
May 22 22:31:28 pulsar kernel: ^?ELF^B^A^A: memory violation at
pc=1555555d6b0 rp=1555 555b3f4 (bad address = 00000008)

I believe the above was caused by a stock 1.2.20 sshd.... not sure.
Should be easier to tell once I compile a kernel and get a system.map.
