On Fri, 30 May 1997, Rick Windham wrote:

> Hello,
> I've noticed that at certian times (unknown why) netstat will
> print headers and then hang. Sometimes after a very long time the output
> will come. This is also accompanied by the standard slew of unaligned
> traps that netstat creates. This also occurs on ipfwadm -Mlex. I tried
> to cat the respective /proc/net files and I got the raw output instantly.
> Anyone have any ideas? Esp. now that I'm masquerading I NEED this
> output to check the usage from the internet net.

I haven't seen this problem myself, but the default operation of netstat
is to lookup the name of the ip addresses it is printing. If there is any
trouble with the name server, this may take some time. It depends very
much if the looked up values are in any nameserver caches. If the
nameserver is even unable to lookup the name, it will certainly take
some time. This name lookup can be overridden with the -n flag. I don't
know if this is your problem, but you can atleast try it in a simple way.

ipfwadm has the same flag (-n) to disable name lookup.

/Andreas
