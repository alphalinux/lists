> I ran some numerical code on a 500 MHz 21164 Alpha system (Microway Screamer,
> 2 MB cache) and on a P166. Both systems are running Linux and the code
> was compiled using GCC. Most surprisingly, the P166 seems to be twice

first problem :) gcc doesn't produce the best code for the alpha..

> as fast than the Alpha workstation!! How can this possibly happen?

it depends on the code you made. if you recoded things to make the most
use of the alphas 64bit regs and took the descrepancy between memory and
cpu speed into account, i'm sure you'd get much better results.

if you send the code we might be able to make some suggestions.

as an example, i took a piece of code that was reading in 32bit ints,
converting them to 16bit (with clipping..) and writing them out. i hacked
it a bit to read in 2 64bit words and write out 1 64bit word.. it was 30%
faster despite having to do the unpacking / packing. not bad at all
for a relativly trivial (no real algorithmic difference in the code)
optimization :)
