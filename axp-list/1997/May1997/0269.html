> 1) /var/run/utmp never gets cleaned up after someone logs out. In
> other words, 'who' will generate a list of all those users who
> have logged on since the machine last shutdown. As such, according
> to 'who', there are several users using the same tty.
>

I had the same problem as you. After I upgraded to the following 3 packages:
1. NetKit-B-0.09-1.1.alpha.rpm
2. util-linux-2.5-34.1.alpha.rpm
3. passwd-0.50-2.1.alpha.rpm
from the "updates" directory at RedHat-site, "utmp" correctly cleaned up
the person who loged out from the "/var/run/utmp" file.

But don't forget to delete the past wrong-entries in the "/var/run/utmp".
If you don't want to reboot you Linux box, you may choose to delete all
contents of the "/var/run/utmp" file. I did the second choice and it worked,
but be sure that no one is using the "/var/run/utmp" file.

> 2) Any user who attempts to change their password while logged on
> to a repeated tty (according to 'who' as discussed above) get the
> error "User unknown to the underlying authentication module."
> Changing passwords seems to work flawlessly as long as the tty
> they are on is not already listed when 'who' is issued.
>

The solution of the problem (1) should solve this problem.

Hope this will help you,
Komwut.
