On Mon, 5 May 1997, Uncle George wrote:

> Got a seg fault within alpha/axp version of dlsym(). Tried to get man
> pages for dlsym/redhat 4.1 system - not there.

Yes, it seems as if the man-pages for the dl functions are forgotten. On
redhat/intel they come from the ld.so package, but this package is not
used on the alpha.

> Tried pulling off (--rebuild ) ld.so src from the SRPMS cd - invalid
> architecture!. Installed it instead, then tried to build it - still
> wrong architecture. read the spec file - exclusive i386.

ld.so is part of the glibc rpm:

ajo@nova ~ >rpm -qf /lib/ld.so.1
glibc-0.961212-4

> So where, or is there, a specific libdl for the alpha/axp system??????

It's supposed to come with glibc:

ajo@nova ~ >locate libdl
/lib/libdl-1.99.so
/lib/libdl.so.2
/usr/lib/libdl.a
/usr/lib/libdl.so
ajo@nova ~ >rpm -qf /lib/libdl-1.99.so /lib/libdl.so.2 /usr/lib/libdl.a \
/usr/lib/libdl.so
glibc-0.961212-4
glibc-0.961212-4
glibc-static-0.961212-3
glibc-devel-0.961212-3

Hope this straightens things out for you.
