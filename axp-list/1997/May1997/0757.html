On Sun, 25 May 1997, Michael Tibor wrote:

> I've been trying to update some of the RH 4.0 stuff on my UDB, and I'm
> getting some strange messages when I try to install the rpms:
>
> =========================
> [root@alpha rpms]# rpm --checksig --nopgp rpm-2.3.11-1.alpha.rpm
> rpm-2.3.11-1.alpha.rpm: size md5 OK
>
> [root@alpha rpms]# rpm -U rpm-2.3.11-1.alpha.rpm
> package rpm-2.3.11-1 is for a different architecture
> error: rpm-2.3.11-1.alpha.rpm cannot be installed
> =========================

Your old rpm doesn't know that the .alpha.rpm extention is equal to the
old .axp.rpm extention. You can override this with the option --ignorearch
on the rpm command line.

When you have upgraded rpm you will probably not see this problem anymore.
