Hi.

I had a similar problem installing RH 4.1 on the cheap Vobis 164PC,
which also has a 4GB IDE HD plus CDROM. After getting an updated
ramdisk image from
ftp.tu-chemnitz.de/pub/linux/redhat-mirror/redhat/devel/axp/images
things started to work out.

Still, fdisk failed pretty badly. I had to start out making swap, /
and /usr partitions that summed up to less than 1GB (plus 5 MB FAT
for milo). Then the errors formatting and checking for bad sectors
didn't happen anymore and the installation script could be completed
with no further problem.

I then proceeded to partition the rest of the disk using cfdisk, made
file systems and so forth.

Summary: There seems to be a problem fdisk'ing large IDE drives. You
can work around, but it's not a nice affair. Anyone know any nice
answers?
