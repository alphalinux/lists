On Sun, 18 May 1997, Rick Windham wrote:

> I do a rpm -bp <somename>.src.rpm to install and prep the package. I'm
> running RH 4.1 on a UDB. Anyone know why this might be happening?

"rpm -bp" takes a spec file as its argument, so you must do something
like:

# rpm --install foobar-1.2.src.rpm
# cd /usr/src/redhat
# rpm -bp SPECS/foobar-1.2.spec

Alternatively, you can say

# rpm --recompile foobar-1.2.src.rpm

to install, prep and compile, or replace recompile with rebuild to build a
binary RPM as well.
