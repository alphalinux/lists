>>> Tony Galan said:
> I startx X on my UDB i get a gray background & mouse pointer, about 5
> minutes later everything else loads up. Why is X it taking so long?

The stock X setup with RH 4.0 and beyond, IIRC, uses fvwm95, which has the
option of passing its config files through the M4 macro processor. Those
config files probe the installed packages and custom build the menus for
what the user has installed. EACH TIME! :-(

Look into the config files under /etc/X11/TheNextLevel; IIRC, one of them
has a little set of comments about how to shorten the startup time by
copying the result of the M4 pass to a local file, and indicating *that*
as the config file for fvwm95 to start with (instead of its default).

I did this with a different (but nearly as) slow machine, and it helped
a lot.

Good luck.
