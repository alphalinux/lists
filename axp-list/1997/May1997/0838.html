On May 29, Avinash Shenoy wrote:
> Hi,
> Novice question....I'm trying to install RH 4.1 on a AlphaStation 255
> 300 Mhz. It runs the SRM console. I'm able to complete the install fine...
> however, on rebooting, the hard disk is identified as having a unknown "magic".
> Any ideas as to why this might be happening ?
>
> Avinash Shenoy
>
> PS : I'm using BSD style partition tags as normal fdisk is unable to see
> the head/sector/cylinder nos of the SCSI HDD.~
>

You might have occasionally overwritten your BSD disklabel: unlike
PC-style partition tables, BSD disklabel usually resides inside the
first partition, so it does not survive when you make a filesystem or
install a boot loader with plain cp or dd (special tool is needed).

My experience tells me that fdisk does not work very well with BSD
disklabels. PC-like partition table may be better for you, once you
have figured out the drive geometry (e.g., from the disklabel).
I'm afraid you'll have to re-install Linux anyway... If my guess about
your problem is correct.

As for disklabels, my approach is to make the first partition small, just
to hold MILO (or aboot, but I don't like it) and the disklabel, and partition
the rest as usual for real filesystems.
