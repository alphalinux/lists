Stop. Most of this is configureable or should be.

The thing with LILO/MILO "boot linux s" or "boot linux 1" can be inhibited by
LILO configuration (i don't know for MILO yet) :

LITERAL=<string> Like APPEND, but removes all other options (e.g. setting
of the root device). Because vital options can be removed unintentionally
with LITERAL, this option cannot be set in the global options section.

Moreover anything you do on this boot-options commandline is only an
environment variable passed down to the kernel and lateron init.

Init can be configured, too on what to do if launched in a particular
runlevel. Setting /bin/sushell for single mode there would be a good idea.

Booting from external devices can be configured out of the BIOS/Startup
System Console for almost any modern system and access to this configuration
can normally be protected with a password.

What cannot be done away with in software is the possibility to ovveride
such protections by going into the hardware.

BIOS-restrictions can be overcome by exchanging the BIOS, zapping the Data-
NVRAM (CMOS RAM on PCs), ... booting-restrictions are eliminated this way.

Another way is to attach the harddrive to another system and access it from
there, ...

The only things that keep you secure against this kind of attack is :

- protect physical access to the system. Use special screws that need
specific tools to turn them. Use mechanical locks, if available.
Screw the system to the desk it is on.
- Use a crypted filesystem. The data needed for decryption should be
held seperate from the harddisk it is on and be trapped against
unauthorized access. There are hw-solutions for this.

And if your data is really sensitive. Lock away the box which holds it.
Put the computer into a locked room and only pull the network-connection
through a small hole ;-).

People sitting at the console of a system are normally considered to be
trustworthy quite a bit more than those coming from the network, because
they can normally do more serious damage by wielding a baseball bat than
by hacking into the system.
