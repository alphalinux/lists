>>> Rick Windham said:
>
> Well I delved into the source code and found that the block_read
> and block_write functions simply give a message that they were called. So
> I modifed them in device.c of milo to fit the fs.h in the kernel tree.

This is because some of the MILO sources have assumed that the kernel sources
have been patches with the axp-diffs from David M-T. His patches, among other
things, update some of the basic read/write routines to use "long" instead
of "int" for return values and some arguments.

> Oh, does anyone have any info on the different files that are made
> by the milo make (e.g. milo.exe, milo.full(.exe,.dd, etc.) and what they
> do? I'm assuming they are different types of milo, like milo.exe might
> run from the ARC console; but I doubt it.

Some of the milo.* files are intermediate results used to make the "final"
versions. However, I know of the following:

milo # booted by linload.exe from ARC/AlphaBIOS
milo.dd # dd to a floppy, boot from SRM by "boot dva0"
milo.rom # FLASHable image on the supported platforms
