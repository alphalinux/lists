At 05:02 PM 5/28/97 -0400, you wrote:
>> I was able to remove amd from the startup scripts (rc3.d) and the machine
>> booted properly. I checked the nsswitch.conf file (the culprit in 4.1) and
>> it seems to be the same as that from the fixed/updated 4.1. I don't know
>> enough about amd to find where it's getting tangled up this time.
>>
>> (As a side qustion, what am I actually missing if I don't have amd running?)
>>
>> On another item, the insmod in 4.2 appears to be somewhat hosed. It simply
>> refused to load any modules. It also went looking for (and didn't find)
>> /lib/modules/2.0.30/modules.dep which I thought was an intel (only) burble.
>> However, replacing the /sbin/insmod binary in 4.2 with the one from 4.1
>> works well as a temporary fix. I was able to load all the pcmcia modules by
>> doing so.
>
>The reason that it doesn't find /lib/modules/2.0.30/modules.dep is that
>most likely on your system, it doesn't exist. RH4.2 doesn't include
>(at least, its mirrors don't) kernel-modules (check your rpm database.
>chances are you prob.. have an rpm from your previous version which can
>be erased).
>
>To make depmod happy, all you have to do is make the directory
>/lib/modules/2.0.30. Then depmod -a will automatically create the
>modules.dep file the next time you reboot (which, of course, will be empty
>because you won't have any modules).
>
>Jeff Odom
>System Administrator
>The Johns Hopkins Krieger Mind/Brain Institute
>
