>For whatever reason, Linux thinks my floppy is a 2.88 floppy drive. I have
>it setup in ARC at a 1.44. Is there anything I need to do specifically in
>Linux? I'm running RedHat 4.1, Kernel 2.0.27.
>TIA.

I seem to recall reading a while back that the UDB floppy is supposed to
be a 1.44, although at one point DEC ran out of them and put 2.88 units
in without mentioning it.

In any case, I also read that for some reason Linux will report them as
2.88s no matter what they actually are. The kernel does not derive this
information from ARC so once you have jumped to the kernel the ARC value
means nothing.

It should work OK (mine does) but you might run into some problems
formatting (I've never had occaision to format a disk in my UDB).
Probably be OK as long as you specified the correct device though.
