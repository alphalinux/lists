The company I work for is currently doing virtual domain hosting off of a
Pentium 200 Linux box, however, do to recent increases in customer base and
the increased load on our server, we are looking at upgrading it.
Currently, we are running RH Linux 4.1 with Apache 1.2b10.

This wasn't a bad to configuration to start out with, but currently, we are
looking for a faster and flexable solution. I was wondering if somebody
could give me their opinion on what would be a fast web server running off
of an Alpha 500MHz machine with Linux.

I have been looking at Zeus(http://www.zeus.co.uk) and it seems to be one
of the fastest web servers avaliable(according to the SPECWeb96 results),
but does anybody know how flexible it is? For example, one of the beat
features of Apache(IMHO) is the fact that you can create and install custom
modules to do specialized functions. Is this possible in Zeus?

Along the same lines, would anybody have any clue as to how fast Zeus would
run on a Linux Alpha machine?

That's about it.... any information would be much appreciated.
