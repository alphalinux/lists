Hello,

I hope this is the right mailing-list, but I found no better one.
I have some problems running Quake on my UDB (Redhat 4.1, linux kernel
2.0.30). I could find three binaries : an xquake and an xf86quake in COFF
format, and an xquake in ELF format. The three of them fail on a strange
error.

I have installed the id1/ directory with all the required files (from the
shareware version) unpacked. When I run one of my binaries, it says the
following:

FindFile: can't find gfx/pop.lmp
Playing shareware version.
FindFile: /tmp/./id1/gfx.wad
Console initialized.
UDP Initialized
Exe: 16:35:22 Aug 4 1996
8.0 megabyte heap
FindFile: can't find gfx/palette.lmp
Error: Couldn't load gfx/palette.lmp

I assume the two first lines are correct (it just means I have only the
shareware). Strangely, xquake finds the gfx.wad file and copies it into
/tmp/id1. I don't know why, and only the COFF binaries do that. The ELF one
just finds the file and reports that fact.
Anyway, the process follows, and ends on the "can't find gfx/palette.lmp".
But this file exists. Moreover, strace shows that xquake did not attempt
any open() or stat() on this file, as it does with gfx/pop/lmp and
gfx.wad. It just decides the file does not exist (I find this rather
frustating).

I have searched the archives of this mailing-list and could not find any
answer. Many thanks to whoever spends some time reading this message.
