Some weeks ago I was having a problem connecting via ppp after upgrading
from RedHat Linux 4.0 to 4.1. The line kept disconnecting with a message
saying "not 8 bit clean". Turns out this did not indicate the problem,
but was only a result of it. The problem was that Linux had suddenly
started assigning both tty00 and tty02 the same IRQ... Fixed this by
using

/bin/setserial -b /dev/cua2 irq 5

in /etc/rc.d/rc.local, and changed the IRQ on my modem. I seem
to recall someone else having a similar "not 8 bit clean" error
message. Maybe this will help if it hasn't been fixed already.
