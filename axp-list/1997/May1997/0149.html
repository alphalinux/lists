> I'm puzzled on two issues. 1) Is this a serious problem? so
> far the compiled program still seems to run ok after compiling with
> these warnings.

Possibly. Several fundamental types were changed to accomodate
expected needs in the near future: dev_t is now 64-bits wide,
and sigset_t is now 1024 bits wide.

I'm actually surprised to hear that it works.

> 2) I thought with the above links the libX11 would
> link to libc-1.99.so and have no problems. Why does it use
> libc.so.6.1 even tho it is asking for libc.so.6?

Because the program itself is linking to libc.so.6.1 while X11
is bringing in libc.so.6 -- a situation that _can_ cause problems.

> And lastly is the only way around this basically to compile
> Xfree for glibc?

Yes.
