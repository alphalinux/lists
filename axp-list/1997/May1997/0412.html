Why is X required to used diald? There is nice control utility
for diald that requires X and tcl/tk, but diald can easily be run with out
X. I run diald on 386DX16 and DX33 boxes, that are headless "black boxes"
with no problems. Diald can be completely controled via a shell, and is
IMHO the best solution for dial on demand ppp. There are a couple of other
solutions, but none quite as nice and simple to use as diald.
Though, I have never run diald on an Alpha, but I don't see
anything that would stop it from doing so. Is there some one out there
that has done this? With or with out X of course. :)
