Hey,

I have had nis and amd rpm's installed for a while but do to lack of time,
I've not done anything with them. Some days ago I noticed that there is
an updated rpm for ypbind and I installed it.

And after next reboot, whole neighbourhood started to complain that
my Alpha disturbs them, with log messages such as

May 13 21:45:42 bart portmap[18756]: connect from 193.166.80.64 to callit(ypserv): request from unauthorized host

(here 193.166.80.64 is my machine and bart is another nearby host).

I.e. for some reason, it starts to query other machines with no real reason.
Or at least I have always thought I have to tell domain name with 'domainname'
for NIS to be usable. Now domainname returned "(none)" which is right,
I hadn't set it.

Anyway, I removed ypbind and now others are happy but I'm still wondering...
Is this really how NIS is supposed to work and I should set a domainname
before I even think about starting ypbind? After all, I had had ypbind
for a long time (since RedHat 4.1 or at least when Alpha version came out)
and it didn't cause these problems. Or was this really the reason for
the new ypbind, i.e. the older one really did nothing?
