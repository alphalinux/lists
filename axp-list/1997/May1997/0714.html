On Fri, 23 May 1997 dave_tucker_@gtech.com wrote:

> I have been running 4.1 with all the updates on a udb. I ftp'd the
> 4.2 rpm's and tried to rpm them.
This is exactly what I'm doing now.

> 1) There were a lot of failures due to dependencies so I said
> --nodeps and --force
I was expecting this...couldn't you just install the packages that the
dependancies required first? Yes I'm sure some are 2 or 3 levels deep in
dependencies. :-)

> 2) I also said -U thinking that it gets rid of the old stuff?
>
> There were a lot of loops, hangs, and reboots (library problems I
> suppose) but finally I'm guessing 99% of the 4.2 rpm's are installed.
> I couldn't find the kernel, so I have tried some I had around that had
> been working well for me (2.0.27 and 2.1.25).
The kernel source and headders are in the RPMS directory:

kernel-headers-2.0.30-2.alpha.rpm kernel-source-2.0.30-2.alpha.rpm

If you are looking for the compiled kernel it's in the images directory
and you have to dd the floppies (or mound with the loop device which I
haven't been able to make work yet) mount then kernel floppy and then copy
it to your root directory.

> Assuming I can get back to where I was and I still want to go to 4.2,
> how should this have been done?

The other method is to dd the ramdisk.img and kernel .img for your box and
boot them and run though the install. You'll have to provide a direcory
tree for the location of the RPMS that the install scripts will accept.
Once I've tried this I can provide information as to what it should be.

Hope this helps, Rick.
