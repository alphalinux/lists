At 14:43 09.05.97 +0200, you wrote:
>Some thoughts and questions about the high end motherboards from Digital
>Semiconductor (DS). Please arrest me if wrong. ;-)
>
>The EB164 motherboard has a 256-bit data bus to main memory and 64-bit data
>bus to the cache. Cache sizes supported are 2MB, 4MB and 8MB. It uses
>two Alpha cache SIMMs. This motherboard was build using the 21171 chipset
>supporting the 21164 processor (266-333 MHz). Was the 2MB and 4MB Alpha
cache
>SIMMs ever available?

In my expierence there were no 4 MB Cache modules avaible as i am looking
for a customer of mine for these since mounth.

>The PC164 motherboard DRAM memory subsystem can be configured as
>either 128-bit or 256-bit. Cache is soldered onboard, size 1MB (9ns).
>Microway and DCG sells the same motherboard* with 2 MB (9ns). The L3
>cache subsystem have a 128-bit data path (doubles the theoretical
>memory bandwidth compared to the EB164 64-bit cache-bus). A great board,
>pity it don't uses replaceable cache modules like the AS500/AS600. It is
>build on the 21172 chipset supporting the 21164A (366-500MHz).

several vendors sell it with 2 MB Cache as the specs from DS are mention 2
MB possibility also.

>The PC164LX motherboard have 4 DIMM slots supporting SDRAM DIMMs rated at
>10ns (this is an advertising gimmick, they really more like 40ns...??).
>Compared to the PC164^ and EB164 256-bit memory bandwith, this
>motherboard has only 128-bit..?? The cache size is 2MB (soldered),
>128-bit data path. The motherboard is based on the 21174 chipset
>supporting the 21164A rated at 366-1000MHz (the 1000MHz is theoretical,
>66.66MHz*15).
>
The new chipset speeds up about 3 times of the old chipset. generaly the
access is 3 times faster on the new 128 bit Bus as on the PC164 with
256-bit Bus.

>All in all it seems like DS must make motherboards with some kind of
>drawbacks. 64-bit cache access for the EB164, soldered cache chips on the
>PC164 and PC164LX, and only 128-bit data path on the PC164LX.
