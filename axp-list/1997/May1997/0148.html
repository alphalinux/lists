Kari Davidsson wrote:
>
> Britt Bolen wrote:
> >
> > logging in a then
> > doing an su root doesn't let me access shutdown

This is, of course, not the case. If you are root you have an access
to everything.

> This is because how the $PATH variable is preserved when doing su.
....
>
> Either add /sbin/ to the path

Don't! There are reasons why root and regular accounts do not have
the same path. Moreover, if you care a tiny bit about security, path
for root should be tightly controlled, without any environment variables
in the middle to expand, and without stuff like '.' anywhere.

Do 'su -' instead and you will get all root environment. You can
also always execute root startup files "by hand" - if you forgot this
'-'. BTW 'su - <name>' applies to other accounts as well.

> or just give
> the absolute path /sbin/shutdown -r now

This is always a simple option. :-)
