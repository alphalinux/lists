> On Thu, 29 May 1997, Avinash Shenoy wrote:
>
> > Novice question....I'm trying to install RH 4.1 on a AlphaStation 255
> > 300 Mhz. It runs the SRM console. I'm able to complete the install fine...
> > however, on rebooting, the hard disk is identified as having a unknown "magic".
> > Any ideas as to why this might be happening ?
> Does the AS255 have the ARC console? MILO gets information about memory
> size from ARC. I would think it would be easier.
>

A lot of the AlphaStation 200s and 255s do not have ARC. They can be
upgraded with some flash chips and the firmware disk, but by default
they do not have ARC loaded.

Actually I find an Alpha with SRM far more flexible and easier to use
than one with ARC. Booting multiple configurations and checking the
hardware is far better under SRM. And you don't need to create
another stupid partition to hold milo and linload.

> > PS : I'm using BSD style partition tags as normal fdisk is unable to see
> > the head/sector/cylinder nos of the SCSI HDD.~
> I've never tried BSD partition tags, I thought Linux supported them. The
> reason fdisk can't see your geometry is because it's a SCSI hard disk.
> The formula that most people have been using is h = 64, s = 32, c = <size
> of disk in megabytes> My Seagate Barracuda is 64,32,4095 in fdisk.

I doubt that the geometry is the problem. Most nearly every disk will
report back the geometry without any conflicts. There are two basic
things that I think are happening here. One, that the disk did not
get the initialization label written to the disk. When you
partitioned the disk, did you use fdisk or minlabel? I have always
have far better results using minlabel. I think that I have only
gotten an install to work once when I use fdisk. I have a disk image
with minlabel and other required programs at
ftp://ftp.alphapower.com/pub/linux/minlabel.img. Grab this before you
do the install.

The second thing that may be happening (although I have never had
time to confirm this) is that not all 8 partitions were filled in. I
created only 3 partitions with one of the first installs under SRM
that I did. The other 5 were blank (not zeroed); I used 'd' to delete
the partition entry. If I remember correctly, I had some bizzarre
results in trying to get the system to boot. I ended up reinstalling
Linux on the machine to get things working.

I am currently writing an Application Note about installing Linux on
Alphas with SRM. If you would like to get a copy, the give me a yell.
It is still in a first draft form, but it is sufficent to get an
install done.
