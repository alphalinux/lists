>
> Hi anybody,
>
> I've got a problem trying to reconfigure an alpha kernel on an AlphaStation 200
> running RH4.2 .
> After I did 'make configure' (I chosed kernel name 'avanti', with SRM boot
> loader), then I executed 'make dep' and 'make clean'.
> So I build all with a single 'make', and obtained the new kernel vmlinux (not
> compressed).
> I've compressed it with gzip, but when I try to boot it aboot tells me that
> there are two unexpected entry points for the new kernel.
>
> Some idea ?
>
You need to do a make boot, which creates a vmlinux.gz in ./arch/alpha/boot
The makefile creates the compressed image by

strip vmlinux
gzip -fv vmlinux

Not sure what the fv args do.
