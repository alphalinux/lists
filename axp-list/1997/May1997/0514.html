The following trivial C program does not run correctly on a MicroWay
500MHz Alpha running Red Hat Linux 4.1 when compiled under DEC OSF/1
3.2 and linked statically (-non_shared):

#include <stdio.h>

int main()
{
printf("before call to system\n");
system("echo inside call to system");
printf("after call to system\n");
return 0;
}

Compiled with "cc -non_shared -o systest systest.c" and run under DEC OSF/1
it produces:

before call to system
inside call to system
after call to system

When run under Red Hat 4.1 it produces:

before call to system
after call to system

Compiled with "gcc -o systest systest.c" and run under Red Hat 4.1
it produces:

before call to system
inside call to system
after call to system
