> > /sbin/insmod: module is 617240 bytes long (expected 605514)
> > /sbin/insmod: forgot to specify -dc when linking libafs.o?

You've got a bogus version of insmod-bfd -- probably 0.2.

Get modutils-2.1.34 and build that.

One thing though -- the 2.0.27 kernel headers have a bug in
<asm/unistd.h>. _syscall5 fails to pass the fifth argument
on to syscall. Fix that and things will work fine.
