Heikki Virkkunen <tesno@ellu.vaala.fi> writes:
> wondering, which gcc flags make alphas slower. I have a Multia 21066A
> 233MHz, and i've really noticed that axp-stuff runs much less fast than
> same code compiled w/ intel/linux. So, which flags can one remove and/or
> which ones to add?

Usually just use '-O2'. You may try to use '-O3' (same as '-O2
-finline-functions' and/or add '-funroll-loops'. It depends on your
program wheater things get better or worse with this.

However, I wouldn't expect a 21066A to get much faster than a 100/133 MHz
Pentium on most applications anyhow. Its memory bandwhith is too low.
