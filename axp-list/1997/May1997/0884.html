I'd like to report the following problems that I haven't encountered
with Intel-x86 RH 4.1/4.2 Linux:

RH 4.1 xl kernel 2.0.27 doesn't seem to find the lp ports.
Is there lp port support compiled in the kernel?

[root@hep3 /proc]# cat ioports
0060-006f : keyboard
02f8-02ff : serial(auto)
03c0-03df : vga+
03f0-03f5 : floppy
03f7-03f7 : floppy DIR
03f8-03ff : serial(auto)
b000-b07f : DE434/5 (eth0)
b400-b47f : ncr53c8xx

I'd also like to report that alpha RH 4.1 network configuration
tool does not work for me. Actually, it refuses to start at all.

RH 4.1/4.2 xl installation diskettes can't find the net.
I find this strange, because the same xl 2.0.27 kernel
works fine after the installation is done.

Offtopic:

RH 4.2 Intel, SMC/WD 8003 8-bit card, network hangs after
some 15-60 minutes from boot. ISA bus is not over-clocked,
card works OK with MS Windows 3.1. Ideas?

When trying to install or upgrade alpha RH 4.1/4.2 from a
hard drive partition, I get lots of comps error messages.
I've downloaded the files from ftp.funet.fi mirror:
rpmcontents.gz, base/ and RPMS/.
I've taken all the files in those directories.
