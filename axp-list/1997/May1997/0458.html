> Is this a known bug with gcc ?

Yes. The reload pass has problems with stack frames larger
than an immediate offset off the stack pointer, i.e. 32K.

While the problem is worked on, workaround by allocating big
automatic arrays with alloca instead, i.e:

#include <alloca.h>
void write_image_bin(double **image)
{
int *imageint2, j, k;
imageint2 = (int *)alloca(sizeof(int)*IMGSIZ*IMGSIZ);
for (j=0;j<IMGSIZ;j++) {
for(k=0;k<IMGSIZ;k++) {
imageint2[k*IMGSIZ+j]=(int)image[j][k];
}
}
}
