>
> I think that DU, as well as a lot of (most?) other "proprietary,
> license-encumbered UNIX ports" have features which Linux doesn't have,
> and won't have for a least a few years. Certainly this will be less
> and less of an issue as time goes by, but at the moment I can't see
> any major Unix OS producer ditching their product for Linux.

Indeed.

Values of DU over Linux include that it offers:

- C2 security, which it is unlikely that Linux will ever offer. For some
markets, this is a show-stopper.

- AdvFS, with the combination of a LVM, Journalling FS, and dynamically
resizable disk partitions.

At the meeting, some "Linux-philes" commented that Linux was likely to have
this Real Soon Now.

Yes, there's a project to build a journalling file system. (See:
<http://hp.cso.uiuc.edu/~c-cook/prof/lfs/>.) Linux will probably have a
full-featured file system significantly before Windows NT has a half-decent
one. It's not clear that this project is progressing anywhere fast. The
original prediction was availability in about July '97 of "alpha" code. The
latest info (last updated in Feb. 97) indicates an availability date "in 1998."

In short, what people believe may become available and what is actually likely
are probably two different things. Linux doesn't have AdvFS, and certainly
won't in the next year unless Veritas donates code or something of the sort.

- DU supports some of the hardware required for building high-availability
systems that just plain aren't in the mainstream of Linux development.

(Mind you, I've seen the DU kernel crash and burn far more times than I've
seen Linux kernels do so; I'm more than a little dubious of the quality of
some of the code that Digital has released; but it must be said that this was
~2 years ago, and represents the earlier "bad" versions of DU...)

These three major things are the fundamental stumbling blocks of things not
available for Linux that are needed for large scale "modern" UNIX systems.
