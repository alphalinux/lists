On Tue, 13 May 1997, Avinash Shenoy wrote:

> Is anyone out there using one such ?? My machine config is :
>
> 433a, 64 MB RAM, 2 GB Ultra SCSI, IDE CDROM, Adaptech controller.

ftp gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels/miata-milo-2028 and
miata-kernel-2029-complete.gz. You'll need linload.exe from the AlphaBios
directory and and a root floppy (RedHat's *latest* install image will do --
that's the one in the developement directory at ftp.redhat.com) as well.

Put the linload.exe and the milo on a fat formatted floppy (or a fat
formatted harddisk partition) and enter it in 'os selection setup' inside
AlphaBios (there should be a howto lying around on gatekeeper), and boot.
>From then on, everything should work as usual ;)

Note that you'll need the axp-diffs, the miata-diffs *and* the latest beta of
tulip.c (to be found at http://cesdis.gsfc.nasa.gov/linux/drivers/tulip.c) to
build a working kernel yourself, and you should set SKBUFF_RX_COPYBREAK in
tulip.c to something large (16384 worked for me) if you don't want your
syslog to be filled with unaligned traps ;)

Contact me (or the list) if something's unclear.
