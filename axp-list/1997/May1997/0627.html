You need the 'ls' program.

Cut and paste is wonderful. Here is the whole process. :

Yes it is possible. Read the FAQ from wu-ftpd on setting up grestgroups.
Basically the steps are:

One: edit the /etc/ftpaccess file and add a guestgroup entry
two: make the user a member of said guestgroup
three: make the directory structure you want them to stay in along with any
libs and progs (ls) from the anonamous ftp area. Look in the bin and lib
derectorys.
i.e. /home/ftponly/etc
/home/ftponly/bin
/home/ftponly/lib
/home/ftponly/home/userdir
/home/ftponly/home/other_user_dir
four: make their shell something else (if you don't want shell access) like
/dev/null or /bin/passwd . With /bin/passwd they can shell in the the
passwd program and change their own passwd. :)
five: IMORTANT put the shell program they use in the /etc/shells file
IMPORTANT
six: edit their home directory entry and put a /./ in it
i.e /home/ftponly/./home/userdir
seven: TEST
eight: edit the passwd and group files in their new /etc/ directory to
reflect their uid/gid 's .
nine: IMPORTANT you don't need their encrypted passwords in the new passwd
file so DON'T put them there IMPORTANT

Hope this helps
