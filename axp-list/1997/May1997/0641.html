> Would it really be a good idea to upgrade to 4.2? - I mean, with the
> number of patches needed before the 4.1 was up running OK (the system was
> not even stable with the supplied kernel on my alpha), why should anyone
> believe that the 4.2 is better? I would like to hear the experiences
> from those who volounteer to try ...

Well, since I never upgraded to 4.1 from 4.0, I decided to give 4.2 a try.
I did an ftp upgrade over 10Mbit twisted pair. As far as I can tell, it
went mostly flawlessl. I did have a problem where every now and then the
upgrade would complain that it couldn't get one of the rpms from the ftp
server. I haven't verified what the prblem is but I suspect that it was
from where the network was too slow and the ftp connection timed out.

My machine is a 166Mhz UDB with 40Mmb RAM and its up and running fine.
The only real problem was the printer rpms but Erik already posted about
that :-)
