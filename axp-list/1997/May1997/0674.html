Hello, all!

I have to second the question about NFS from a week ago that was never
really answered. NFS is working for me except that created files gets
created but no data can be written to them. I mounted like this:

ajo@wopr ~ #mount -t nfs attic:/u1/home/ajo /mnt/attic
ajo@wopr ~ #

This is written to the messages file at mount time:

May 23 13:52:36 wopr kernel: nfs warning: mount version older than kernel

This is the output of mount:

ajo@wopr ~ >mount
[...]
attic:/u1/home/ajo on /mnt/attic type nfs (rw,addr=130.240.192.254)
ajo@wopr ~ >

I have a user on my system which has the same user id as my 'ajo' user on
attic, called ajo2. I can read, mkdir and remove files/dirs ok. Now, I try
and write to a file with the ajo2 user:

ajo2@wopr /mnt/attic >echo >file Hello
ajo2@wopr /mnt/attic >ls -l
total 2
drwxr-xr-x 2 ajo2 users 512 May 20 01:01 bin/
-rw-rw-r-- 1 ajo2 users 0 May 23 14:57 file
drwxr-xr-x 3 ajo2 users 512 May 23 14:57 src/
ajo2@wopr /mnt/attic >

The filesystem is exported as rw from a FreeBSD machine. Am I doing
something wrong or is nfs writes not working ok (atleast against x86,
haven't tried against another alpha) ?

The same thing seems to work fine on a Red Hat 4.1 / x86 machine.

Oh, another thing. When I umount on my 2.1.36 kernel (with the entry.S
patch for nfs to work) I get two oops:es. The system seems to work fine
after, but still, I don't like it. I am sending the details to Richard
Hendersson.
