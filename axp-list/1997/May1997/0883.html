Rick Windham wrote:
>
> Hello,
> I noticed today that if I boot <kerel> emergency I get asked for
> the root password, or the system will come up normally. I've also noticed
> that if I boot single or 1 it simply comes up with a root shell.
> This is a very large security hole (IMHO) requiring me to
> physically lock up all Linux boxes behind lock and key.
> Is there a way or patch to make the single ask for the password?

(Borrowed from debian /etc/inittab)
Add the line
~~:S:wait:/sbin/sulogin
in /etc/inittab. This will ask for root password when entering
single user mode on bootup (at least on my Linux/Intel box). I have
not tested this on my Linux/Alpha box.

But also consider the answer from Michal Jaegermann. No password
protection will replace a locked room and big key 8-).

Hope this helps,
