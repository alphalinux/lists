> 4. The message "YPBINDPROC_DOMAIN: No server for domain: (none)" came up
> about six times.
>
This happens also in RH4.1 with the latest ypbind and such. The reason
is that you haven't set your NIS domainname and therefore it is not
found (and no machine in your subnet is willing to be a NIS server for
you).

It's a bit strange that no-one asks one to specify the domain and still
it should be set. Maybe the startup script could be changed a bit...
a line echoing "Set your NIS domain here" and the actual ypbind line
in comments (and perhaps a comment stating one to remove that comment
after setting domain).

Of course this is mentioned on the man page (as someone kindly told me
when I was asking about it a week ago or so), but I don't think it's
quite correct to assume that everyone selecting "Install all" reads
all the hundreds of manual pages immediately. At least I didn't read.
