>>> Shane DeRidder said:
>
> We recently purchased two Diamond Stealth 3D 2000's w/4mb of EDO memory
> for a pair of 166mhz Alpha UDB's we're going to be using as Linux
> workstations. I grabbed the S3 ViRGE XFree86 3.2a server as it says it's
> the only one that will support 24-bit color with this card, but anytime I
> try to start the X server, it coredumps and leaves the console unusable. I
> can still telnet in, however. I can get it to work just fine in 8, 15 and
> 16 bit color, but anything higher doesn't work. Has anyone with this card
> or server been successful?

If you actually tell it to run at 24-bit depth, it *will* bomb. The 24-bit
code is *very* INTEL-specific, and won't work on the Alphas.

Try specifying 32-bit depth and see if it works any better...

Good luck.
