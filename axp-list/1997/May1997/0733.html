> Well I've discovered that the RH install script is very picky about the
> missing RPMs even though they are NOT needed for the update. I thought it
> would go through but then it died with an error message I couldn't
> capture.
>
> My only other alternative to downloading the ENTIRE distribution is to
> find the RPMs that have the same version numbers as the ones from 4.1 and
> move them from the 4.1 CD-ROM to my 4.2 install directory.

There should be an easier way;

Do symbolic links for *all* RPMs in 4.1.

something like:

$ zsh (or csh)
% cd /mnt/cdrom/RedHat/RPMS
% foreach i (*)
> ln -s /personal/RedHat/RPMS/$i /mnt/cdrom/RedHat/RPMS/$i
> end
[machine chuckets away for a couple minutes...]
% cd /personal/RedHat/RPMS-I-downloaded/
% foreach $i (*)
> rm -f /personal/RedHat/RPMS/$i
> cp $i /personal/RedHat/RPMS/$i
end
%

The idea is that this first symbolic links for *all* the old RPMS in
/personal/RedHat/RPMS, linking them to your CD-ROM copy of v4.1.

You then copy in all the RPMs that you ftp'ed from Red Hat.

If there's a problem with simultaneously having (say) XFree86-3.2.0-axp.rpm as
well as XFree86-3.2.1-axp.rpm, you might wish to write a Perl script that
reads in the items in the "new" directory, uses a regex to strip off the
[version].[rpm] part, and tries to delete all corresponding files in the
"link" directory. This task isn't too hard; left for now as an exercise for
the reader. (I don't feel like writing the script this evening.)
