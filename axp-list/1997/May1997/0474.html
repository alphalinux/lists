On Mon, 19 Jan 132, Johan Grape wrote:

>
>
> I'm interested in getting the hfs_fs module working on
> my alpha, so that I can use a zip drive to shuttle stuff
> back and forth between a mac and the alpha (I can use dos, but
> hfs would be more elegant). having file system access is
> also nicer than the hfs_utils.
>
> The moduls compiles cleanly, but when I try to install the
> module:
>
> /sbin/insmod hfs.o
> /sbin/insmod: module is 158026 bytes long (expected 152140)
> /sbin/insmod: forgot to specify -dc when linking hfs.o?
>
> well - -dc has been specified.. What do I need to do to
> fix this?

You are using old modutils. Either get newer ones (and I think new
binutils too) or try strip -x on your module. Insmod seems to
dislike any symbols it has no use for, so the workaround it to strip them.
