Hi

I'm straying a long way from my usual haunts today. You may know that I
work on drivers for parallel port devices. I've had a couple of questions
about using those drivers on the Alpha, but haven't been able to do much
about them. Apparently my EZ135 driver doesn't compile on the Alpha ...

I obtained a copy of RHS 4.1 for the Alpha (thanks Donny :-)) and finally
managed to "borrow" a machine. The machine is an ALPHAstation 500 with
a 333Mhz CPU, 320 Megs of memory, 2.1 G of disk on a Qlogic ISP1020
controller, and a Cirrus video adapter (according to NT anyway). It has
both the SRM and ARC (not ALPHAbios) consoles. It's actually a loaner
from Digital and has not been altered in any way.

According to the RedHat manual, this is an ALCOR machine and I should
be using an ALCOR Milo image. I can find no other place where I should
look for a mapping of machines to Milo images. There is an ALCOR image
on the 4.1 CDrom, but it doesn't work for me. I've searched gatekeeper
for later images - but can't find any. I've also gone through the
RHS 4.2 development tree, and can't find any help there. I also tried
to follow the pointer about booting via the SRM console but it leads
to instructions about how to build bootable media and no useful
information about installation ... (I suspect some information has moved
around and the pointers haven't been updated.)

The one thing I've not done (yet) is to try to hook something to the
serial port - it's not too easy to do in this environment.

Using linload v1.5 and the ALCOR Milo, I get to the message

Swapping to PALcode at 0x0

and nothing further appears to happen. The keyboard is dead (numlock
doesn't do anything), so I don't think it's sitting there waiting
for input ...

I've noticed a few questions on this subject in the archives, but the
responses appear to have been by private mail ...

Is there anything I can do to get this machine working ?
