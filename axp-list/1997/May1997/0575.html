>stuck in state DW until reboot, keeping the load avg at 1. Nothing is
>actually read or written. "mt status" shows the following before hanging:

I don't have any ideas, but we experience similar problems (process going
into state D, load average sits at 1) with an Archive DAT autoloader
SCSI2. The only thing we found that helped was rebuilding the kernel
while disabling all options from the driver, i.e. no tagged command
queueing, no iomapped, etc. However, now though we don't frequently see
processes drop into D, we will see "unexpected CIA machine check, io
error" at some point during use of the tape drive. It works without
incident on other machines.
