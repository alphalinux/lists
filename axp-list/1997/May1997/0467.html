There was some discussion on this list previously concerning running the
Digitial Unix version of Maple under Alpha/Linux. I was wondering if
anyone had had any success doing this? I've been playing with this under
both kernel version 2.0.29 and 2.1.36 and get the same behaviour. Maple
starts up fine and many commands work, but certain commands kill the maple
"compuational engine". For example, Maple has no problem with 2*2, 2^2,
and 2.2^2, but 2^2.1 drops one back to the command line from maple, and
xmaple gives the popup window "The Maple Computational Window is no longer
executing. No further compuations are possible in this session"

If no one has been able to get Maple to work I was wondering if someone
knows why it "can't" work? To help I'll include a diff between an strace
-f when I do a 2.2^2.2; and one where I do 2.2^2. One should note that
the lines such as

open("/home2/allanb/maple//lib64/type/positive.m", O_RDONLY) = -1 ENOENT
(No such file or directory)

don't seem to be the problem. The files don't exist on a DEC Unix box
which runs Maple with out a hitch and one can generate such lines when
running commands that appear to function fine. Likewise the calls to
SYS_112 appear for both commands that work and those that don't.

It would be nice if someone could figure out the problems. We'd love to
buy Maple for all our Alpha's, but our e-mail to Waterloo Maple Inc.
yielded a reply that they at that time had no intentions of releasing a
version of Maple for Alpha/linux.
