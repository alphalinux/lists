Some thoughts and questions about the high end motherboards from Digital
Semiconductor (DS). Please arrest me if wrong. ;-)

The EB164 motherboard has a 256-bit data bus to main memory and 64-bit data
bus to the cache. Cache sizes supported are 2MB, 4MB and 8MB. It uses
two Alpha cache SIMMs. This motherboard was build using the 21171 chipset
supporting the 21164 processor (266-333 MHz). Was the 2MB and 4MB Alpha cache
SIMMs ever available?

The PC164 motherboard DRAM memory subsystem can be configured as
either 128-bit or 256-bit. Cache is soldered onboard, size 1MB (9ns).
Microway and DCG sells the same motherboard* with 2 MB (9ns). The L3
cache subsystem have a 128-bit data path (doubles the theoretical
memory bandwidth compared to the EB164 64-bit cache-bus). A great board,
pity it don't uses replaceable cache modules like the AS500/AS600. It is
build on the 21172 chipset supporting the 21164A (366-500MHz).

The PC164LX motherboard have 4 DIMM slots supporting SDRAM DIMMs rated at
10ns (this is an advertising gimmick, they really more like 40ns...??).
Compared to the PC164^ and EB164 256-bit memory bandwith, this
motherboard has only 128-bit..?? The cache size is 2MB (soldered),
128-bit data path. The motherboard is based on the 21174 chipset
supporting the 21164A rated at 366-1000MHz (the 1000MHz is theoretical,
66.66MHz*15).

All in all it seems like DS must make motherboards with some kind of
drawbacks. 64-bit cache access for the EB164, soldered cache chips on the
PC164 and PC164LX, and only 128-bit data path on the PC164LX.

On the EB164 and the PC164 the processor speed, the PCI-speed and the
memory speed are controlled by the crystal oscillator and some jumpers.
If I changed the division ration, using these jumpers, giving for
instance 40MHz on the PCI-bus, well inside the PCI2.1 standard, will this
give a performance increase of approx. 20% on I/O? Probably.
Since this also decreases the access time to main memory (requiring 50ns
SIMMs), problems will first arise here. I know its possible to get 50ns EDO
RAM, what about 50ns 36-bit parity RAM?? The PCI-cards can also create
problems. Read the proper news-groups to get peoples experience with
increasing the PCI-bus speed on Intel-based PC's.

Then to the DIMMs of the PC164LX. The 430TX from Intel supports both
SDRAM DIMMs and EDO SIMMs. Peoples experience with motherboard based on
this chipset is: for memory speed increase to 80MHz (max 40MHz on PCI-bus)
50ns EDO RAM is prefered (price), for 100MHz system (max 50MHz on PCI-bus)
the 100MHz SDRAM DIMMs are the only alternative. Problem with systems
using 50MHz on bus is the PCI-card. Very few manage to operate under this
speed. Back to the PC164LX, 40ns 128-bit memory access gives theroretical
transfer-speed of 0.4 GB/s, the EB164 with 60ns 256-bit bandwith gives approx
0.5 GB/s.... A EB164/PC164 with 50ns 256-bit memory access gives 0.64 GB/s,
that is 60% more than the PC164LX...

Everything I know about the PC164LX is based on:

http://ftp.digital.com/pub/Digital/info/semiconductor/literature/164lxntu.pdf

I hope some people from Digital or else will prove me wrong, but hell,
this is what learning is about. ;-)

And, at last, the final question. Both the EB164, PC164 and the PC164LX
have 2 64-bit PCI slots. Anybody delivering products using these slots
to full extend? I've heard a rumor saying Intel don't want to support it
because they have to base it on Digital's solution...

* If I'm not to wrong, Microway swapped the cache chips. Great job!

^ Can be configured as either 128-bit or 256-bit.
