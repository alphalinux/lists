> (As a side qustion, what am I actually missing if I don't have amd running?)
AMD is an AutoMountDaemon. It will mount NFS volumes (or even other
filesystems - but NFS is what it was designed for) on the fly.

While this is a nice thing to have, configuration can be hairy and it is
seldomly really needed.

> On another item, the insmod in 4.2 appears to be somewhat hosed. It simply
> refused to load any modules. It also went looking for (and didn't find)
> /lib/modules/2.0.30/modules.dep which I thought was an intel (only) burble.
No. modules.dep is dependency information on what module uses which other
module. This takes care to load the infrastructure needed when loading a higher
level module. E.g. fat.o is loaded before msdos.o.
