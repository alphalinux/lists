Hi.

I had observed two strange things in my RH 4.1+ (glibc 0.961212-5) installation. The
first thing is when I debug a program with gdb (gdb 4.16-6) it dumps a core when I
try to step or next over a instruction like:

data = (char *)malloc((size_t)(size * sizeof(char)));

The same with calloc.

Another strange behaviour I encountered was when I use function
like:

double my_fkt(double input)
{
/* some calculation with input */

return output;
}

When I follow the instructions in the function the results are ok,
but when I leave it the function I receive a different result which is
constant during the whole program session. Any suggestions what might
the problem here?

And at last I had encountered real problems to create a global
variable through modules, but it doesn't work. On other systems like
i386-Linux, DEC/OSF, SunOS, HPUX and MS-DOS it work well there. Any
suggestions what I made wrong or is this just a inconvenience?

Would be nice to solve these problems, because I can't be sure if a
runtime error is real or just a problem on my system.
