I'm having problems with adding two external SCSI devices to my UDB. One
is a Seagate 1.2gb harddrive and the other is a Pioneer 4x CD-ROM. I'm
beginning to wonder if I have a parity or termination problem. If I only
attach the hard drive, everything is alright. The moment I add the CD-ROM
to the chain, I get random SCSI resets and sense key errors. I have the
Multia Service Info booklet, but it doesn't say anything about wether the
SCSI bus or internal drive is terminated or parity enabled/disabled.
Anyone know? I do have proper termination on the last external device and
none of my ID's conflict (internal ID 0, external HD ID 1, CD-ROM ID 4).

BTW, as you can probably see, I hacked my way around the PGP problem, but
instead of having pine send to the filters via STDIN, I have it dump a
temp file which the pgp program reads. It's a kludge, but it'll work for
now.
