Hi everyone,

we just received a 500 MHz Alpha (21164A CPU) machine from Quant-X,
ARC firmware, with RH 4.0 (!) pre-installed. Since unfortunately we
had to save money, the machine only has a 4GB IDE drive and an IDE
CD-ROM. Basically, I am not able to get the machine to work and
start to suspect hardware problems. I appreciate hints / thoughts
from folks more experienced with Linux on the Alpha platform.

To make a long story short:

(1) The machine arrives, boots nicely; I attach it to the network --
this works, so N.B. the machine was up and running for app. 30
minutes. Then I realize that there are app. 2GB of unused diskspace,
so the next thing I do is to attempt to format this partition (the
partition was already there, but no filesytem). Immediately after
hitting the RETURN key, the machine freezes, and I had to perform a cold
reboot.

(2) Upon rebooting, the machine hangs during fsck -- since I
didn't have a 4.0 CD or rescue disk I never got back into the
system.

(3) I have, however, a RH 4.1 CD, so I try to boot off the CD -- it
simply doesn't work (or, rather, it hangs in a quite early stage) (The
Austrian repr. of Quant-X installed 4.0, shipped me a 4.1 CD, but
warned me that 4.1 doesn't work -- go figure!)

(4) At this point I decide that I might go directly to 4.1 anyways,
so I try a full install. The installation always hangs when the
screen says 'checking partitions' (i.e., quite early on) What
surprises me is that (i) the system mounts the CD-ROM on some
temp. device hdb (I would have expected hdc), then (ii) it
scans disks for partions on hda (i.e. the hard-drive) and this
hdb device; there it hangs.

(5) In a final attempt, I got 4.2 boot/root floppies and thought
if it works at all, I'll do a ftp install. This install also
fails, but it fails reproducible either when preparing the
swap space or when formatting the / partition.

BTW, I always used the pc164 kernel on the boot floppy.

At this point, I gave up and am beginning to suspect either a
hardware problem in the IDE subsystem or some incompatibility
between the software and the hardware. I have a reasonable amount
of expertise with Linux on Intel, but none with the Alpha platform,
so it may well be that I did/do something very stupid. Also,
I have not really read yet the manual, but the 4.1 manual
does not strike me as very detailed.

In a quick search through the FAQ I didn't discover anything special
regarding IDE drives; the vendor said something that IDE partitions
have to be smaller than 2GB -- even if this were the case, it
doesn't explain the install failures as the root partition was the
first one and had app. 500MB.

So, I am wondering whether someone has hints or had / has similar
problems, or if you think that I may not be so wrong with my
suspicion of faulty hardware.

Many thanks in advance,

Stefan Boresch
