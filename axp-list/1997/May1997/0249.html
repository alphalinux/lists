I've recently installed (not upgraded) Linux/Alpha Version 4.1 on
my Alpha Station. I have a few problems which I believe are related:

1) /var/run/utmp never gets cleaned up after someone logs out. In
other words, 'who' will generate a list of all those users who
have logged on since the machine last shutdown. As such, according
to 'who', there are several users using the same tty.

2) Any user who attempts to change their password while logged on
to a repeated tty (according to 'who' as discussed above) get the
error "User unknown to the underlying authentication module."
Changing passwords seems to work flawlessly as long as the tty
they are on is not already listed when 'who' is issued.

3) Logging on remotely from X-terminals is impossible. The xdm
login widget seems to work, but just before the session is begun
the user gets booted out. xdm-errors gives a cryptic error like
connection from client rejected.

I've been in contact with Red Hat support for a few weeks on this,
and they seem baffled. They referred me to this list. Has anyone else
experienced these problems when installing Vanderbilt (4.1) from scratch?

-Bob
