I have upgraded my Cabriolet from 64-128Mb. It seems to be working OK for
the most part but it will crash quite frequently now. The same kernel had
been running for several months without a crash(2.0.27).
When the system crashed I seem to just end up with the last part of a
register dump on the screen so I am unable to tell what is wrong. If I
reset the box it reboots and goes through the fsck process and starts
booting to runlevel 3. Or at least that is where I think it is getting
and than it will hang immediatly or continue but if I do not power the
machine on and off it most times will hang immediatly.

I have some sytems running with both 64 and 128M I only have one 128M
system that is running reliably the rest seem to have crash problems.

I am booting through the ARC console to run lilo.

I have a feeling that the problems are related to the networking code. My
default configuration is to turn on most of the networking code and on
the one reliable system I have significantly reduced the running network
code and are running with almost all the ports disabled with ipfwadm.

My thoughts/questions

There is a problem still to be found in the networking code?.

Is is possible that memory is not completely zeroed when the system boots?

Is there a way to limit the memory size to 64M in lilo?
