I have a Tecmar DAT drive which works just fine on a Sun machine running
Solaris 2.5.1. When I tried to hook it up to my UDB the machine hung when
it initially interrogated the device (something about cannot allocate
offset) and the boot failed.

The UDB is properly set up for a SCSI tape device as I have been using a
Conner CTMS-3200 SCSI/QIC drive with no problems for over a year. Cables,
termination and addresses are OK.

I am running an older kernel (1.3.89) but the failure is occurring before
the machine even jumps to the kernel code.

Is there something funny about the 53c810 or its driver as related to DAT
drives or is this just another instance of "Sun hardware is different" a
la CD-ROM drives? I would have thought that scsi is scsi, and might have
expected problems with block sizes or the like but not boot failures.

Since these DAT drives are available for $225 I would think there would
be a lot of interest among Linux users if they could be made to work.

Any thoughts appreciated, thanks.
