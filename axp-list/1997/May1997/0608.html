True enough. What you present above looks more or less what the script
in the most virginal install I have, which is Red Hat 4.1 linux/SPARC.
It, interestingly, has had no problems configuring its interfaces at
boot.

On my Red Hat 4.1 linux/axp, the relevant bit of the script looks like:

interfaces=`ls ifcfg* | grep -v ifcfg-lo | grep -v :`

# See how we were called.
case "$1" in
start)
./ifup ifcfg-lo
for i in $interfaces; do
./ifup $i boot
done

...which actually -shouldn't- cause the same sort of hang-on-boot
problems. This leads me to suspect that indeed the problem has
been fixed and is in initscripts-2.91-1 (and presumably later), which I
installed after applying my beforementioned workaround to the
problem in the /etc/rc.d/init.d/network from initscripts-2.81-1 .

And so, before this already over-ornate solution to a trivial
problem becomes any more prolix than it already is, I would
commend (dig this) upgrading to the most recent initscripts
as a fix.
