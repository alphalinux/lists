A> You have a UDB with one SCSI drive. The formatted size of the drive is
A> 345,090,000 bytes. You want to use it as a firewall and IP masquerade
A> platform. Knowing that those two activities are more memory intensive
A> than disk intensive, you put in 40 MB of RAM. You are using RH 4.1. You
A> will connect to the Internet via your ISP with a US Robotics 33.6
A> Sportster. The local LAN (2 more UDBs and a PC) is 10BaseT.
A> This machive will do nothing else except firewalling and IP masquerading.
A> What is the best partitioning for this drive?
A> Bear in mind that another drive is NOT an option.

A> Device Filesystem Type Size
A> --------- ----------------- ------
A> /dev/sda1 16-Bit MSDOS <32M 2 MB

Shouldn't this partition have a 12 Bit FAT?
For MS-DOS compatibility it's best to set the correct value,
you don't know which utility eventually will destroy your boot partition.

A> /dev/sda2 Linux Native 101 MB
A> /dev/sda3 Linux Swap 40 MB

I don't do firewalling and masquerading, so I can't give any advice, except my
experiences with a Jensen, 32 MB RAM (29630464 bytes total, BTW, anyone knows
where the rest has gone?), 32 MB swap results in 8 MB swap being used and
approximately 500 kB memory staying free while compiling a kernel on one
virtual console and running X with xemacs and two xterms and so on on another.
(According to /proc/meminfo in one of the xterms.) So perhaps you can keep swap
a bit smaller. If you really need much memory, it's always better to use /real/
memory rather than swap space.

A> /dev/sda4 Linux Native 194 MB
A> ------
A> 237 MB

I'd put all needed programs and things that stay mostly unaltered into one
partition which would be as small as possible (enough to do a bit of
maintenance as compiling a kernel or upgrading the system with new software),
and use the rest for /var, /tmp symlinked to /var/tmp.
