Hi there!

I've had some trouble with the "SoundBoostar" sound card in the Vobis Alpha
(PC164, 64 Meg RAM, 3.8 GB IDE drive, IDE CDROM). My machine is running RH 4.1
but I've upgraded to kernel 2.0.30, which seems to be working fine.

Initially, I couldn't get thru make boot. It seems that the BusLogic SCSI support
conflicts with the sound card (???), so I threw it out. Then compilation succeeds.

With this kernel, I tried "cat bong.au > /dev/audio", which tells me "device not
configured". "cat /dev/sndstat" spews out the following:
----------------------------------------------------------------------------------

Sound Driver:3.5.4-960630 (Wed Jun 18 11:01:47 MEST 1997 root,
Linux localhost.localdomain 2.0.30 #3 Mon Jun 16 22:20:39 MEST 1997 alpha unknown)
Kernel: Linux localhost.localdomain 2.0.30 #2 Wed Jun 18 11:01:58 MEST 1997 alpha
Config options: 0

Installed drivers:
Type 10: MS Sound System
Type 24: MS Sound System (AXP)
Type 27: Compaq Deskpro XL
Type 1: OPL-2/OPL-3 FM

Card config:
(MS Sound System at 0x530 irq 9 drq 3)
(OPL-2/OPL-3 FM at 0x388 drq 0)

Audio devices:

Synth devices:

Midi devices: NOT ENABLED IN CONFIG

Timers:
0: System clock

Mixers:

----------------------------------------------------------------------------------

In the sound part of kernel configuration, I selected MSS/WSS, generic OPL, /dev/dsp
and /dev/audio support and changed the MSS irq from 11 to 9, as suggested.

I've tried the isapnptools-1.8.src.rpm, which compiles after minor modifications in
the Makefile, but to no effect. Stephan Kanthak told me this was to be expected, as
he couldn't get isapnp to activate anything even on a Wintel-PC.

Am I missing something obvious? I don't understand why no audio device is listed in
the cat /dev/sndstat!
