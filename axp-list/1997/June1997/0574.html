whoevery has hangup problems with S3 chips (not ViRGE) for XFree86 3.2
and thus is still using 3.1.2 or similar, please get the XFree86 3.3 S3 server
and give it a try. if it still locks up (only then!) please add
Option "no_pci_disconnect" and try again (and report to me; thanks!)

this seems to fix these lockups for a number of AXP reports...

I'd like to know too if this Option "no_pci_disconnect" might harm
any now working installation or if we can make this default for AXP
in the future (please compare performace too...)

thanks,

Harald
