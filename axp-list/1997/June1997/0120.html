Hunter A. Ware wrote:
> I just upgraded to glibc 2.0.3 and as far as I can tell EM86
> has ceased to function. netscape suddenly has started dying with a
> "Signal 7 received". I haven't had time to test this extensively but
> I was wondering if this was a known problem?

Yes, I too ran into problems like this when I upgraded. The
"solution", which will come out along with v0.2, is to do some
things in the kernel (e.g. keeping mmap's in the low 2Gb) that
really *should* be done in the kernel. In V0.1 I tried to do
these functions in user space so that em86 would run on stock
kernels. I finally decided that the hacks I had to do (e.g.
custom versions of ld.so on both the Intel and Alpha sides)
made for a compatibility nightmare. Better to try and get my
changes into the "official" kernel tree (some of which are
already there in 2.1)
