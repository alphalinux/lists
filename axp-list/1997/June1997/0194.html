> I'm installing Redhat 4.0 on a Noname and have a similar problem.
> sda1 : DOS (for MILO ...)
> sda2 : ext2-fs
> sda3 : ext2-fs
> sda4 : extended partition
> sda5 : ext2-fs
> sda6 : ext2-fs
> sda7 : ext2-fs

> sda4 is not a *real* partition. It's only an extension that contains
> sda5-sda7 (well, did I do this right?? I used fdisk to configure the
> partitions).
Yes. This is how it works.

> However, the installation process recognizes: sda2, sda3, sda4, sda5, sda6
> as valid ext2-partitions. The number is right, but it should use sda7 instead
> of sda4. The installtion hangs when it tries to mount sda4.
> How can I fix this?
This sounds like either partition IDs being wrong, the kernel counting
wrong or something like it.

This is an example on how it should be :

/dev/sda1 * 1 1 296 614761 6 DOS 16-bit >=32M
/dev/sda2 297 297 716 872340 5 Extended
/dev/sda3 717 717 1017 625177 83 Linux native
/dev/sda5 297 297 508 440293 6 DOS 16-bit >=32M
/dev/sda6 509 509 716 431985 6 DOS 16-bit >=32M

I.e. the IDs should be 5 for the extended pseudopartitions [here sda2]
(which is probably not done by fdisk per default) and corresponding to
the FS on it otherwise (DOS for sda1/5/6 and Ext2 for sda3).

Please note also, that sda5/6 take up the same space that is reserved using
the sd2 pseudo-partition.

> For test I did an install with sda2 and sda3 only. This succeded up to
> the "login:" prompt. But it's not possible to login - I do enter "root"
> and the prompt appears again (impossible to enter a password). Why?
This sounds like /bin/login being broken ...
