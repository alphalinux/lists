On Sun, 1 Jun 1997, Michael Tibor wrote:

> What's the best method of upgrading glibc on a RH 4.0 box (UDB)? I've found
> that if I just install the rpm from 4.2 it breaks half a dozen things
> related to authentication.
>
> I then tried to upgrade to pam-0.57 and it complained about a bunch of
> things missing (pwdb, correct version of pamconfig, etc).

The trick is to upgrade lots of things. Install everything it complains
about and when it complains some more, install more. Once you can get a
single command to upgrade all of the packages you'll be all set.

Of course, it's probably easier just to upgrade to 4.2.
