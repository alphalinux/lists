greetings:

We're trying to get the adaptec ANA6944A TX (formerly the
Cogent EM400) to operate with linux 2.0.30 running on a DEC PC-164-
based alpha. We're using the 0.76 release of the tulip driver, with
a single alpha-mandated modification:

#define SKBUFF_RX_COPYBREAK 16384

When using the cogent EM100 (single port 10/100 Mz card using the
same DEC chip set) we have no problem booting and getting the tulip driver
to operate stably...no luck with the ANA6944A. The alpha sees all four
ethX's (the bios hardware inventory lists all the devices and their interupt
vectors, as well as the PCI-PCI bridge). When lauching the tulip driver
(kernel or module-based) we get the following:

tulip.c:v0.73 4/16/97 becker@cesdis.gsfc.nasa.gov
eth0: DEC 21140 Tulip at 0xc000,, EEPROM not present, 00 4c 69 6e 75 79, IRQ 0

THIS IS AN ALPHA TEST DRIVER.
The following verbose information is emitted for
bug reports on media selection.
eth0: Missing EEPROM, this device may not work correctly!
eth1: DEC 21140 Tulip at 0xc400,, EEPROM not present, 00 4c 69 6e 75 7a, IRQ 0
eth1: Missing EEPROM, this device may not work correctly!
eth2: DEC 21140 Tulip at 0xc800,, EEPROM not present, 00 4c 69 6e 75 7b, IRQ 0
eth2: Missing EEPROM, this device may not work correctly!
eth3: DEC 21140 Tulip at 0xcc00, 00 00 92 a7 41 9c, IRQ 18
eth3: EEPROM default media type Autosense.
eth3: Index #0 - Media MII (#11) described by a 21140 MII PHY (1) block.

Only eth3 was connected to a 10baseT line...the other ports were not
connected. Anyone have any experience with this beast successful or otherwise??
