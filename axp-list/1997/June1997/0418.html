>Kelvin Lunsford wrote:
>>
> >Does anybody out there know of a C compiler (DEC/GCC/NT) for the
>>Alpha processor which will let you set the default int size to 64
> >bits. This is for an embedded application, the resulting binaries
> >would not be running under an operating system.
>
>How about:
>
> % gcc -Dint=long "-Dunsigned=unsigned long"
>
Wouldn't this convert an "unsigned char" to "unsigned long char"?
I assume this has about the same effect as typedef's.

What I'm looking for is a compiler that will use 64 bit integral promotion
when expressions are evaluated. The Alpha is a native 64 bit machine
and the C language is supposed to have an int size which is optimized
to the target machine so that the most efficient code can be generated.
Unfortunately too much C code has been written which uses 32 bit
int's, so rather that fix the C programs the Alpha compilers were made to
generate extra code which does all of the conversions and testing to make
the int's look like they are 32 bits. I think it is great that a
compatibility mode exists so that older programs can be supported, but
it's not a compatibility mode, IT'S THE ONLY MODE.

Kelvin Lunsford, kelvin@qantel.com

Qantel Technologies, Inc.
