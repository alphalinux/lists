I'm trying to install RedHat 4.2 onto a UDB ("noname" alpha) with 32MB
of memory and 2 1gb SCSI disks.

I made all the disks, booted the RedHat install, and started installing
over FTP just fine. Then about 50M into the install, I got impatient
and aborted it with Ctrl-Alt-Del, thinking to come back, select less
packages, and begin again.

Doing exactly the same thing, I now cannot get to the package install
point at all. I get to this point:

- partition disks
- input network info
- FTP base/hdlist (ie. networking works), and
- select packages
- format drives, with or without bad block checking
(doesn't work even if I skip formatting)
- get the error "mount failed: invalid argument"

Here are the last few lines from the install console (any typos are
mine):

/dev/sda is fdisked
running: /usr/bin/fdisk /usr/bin/fdisk /tmp/sda
found partition sda1
found partition sda2
found partition sda3
found partition sda4
found partition sda5
found partition sda6
/dev/sdb is fdisked
running: /usr/bin/fdisk /usr/bin/fdisk /tmp/sdb
found partition sdb1
making device sda3 (8, 3) as /tmp/sda3
setting up swapspace, device = sda3, size = 134692864 bytes
making symlink from /tmp/rhimage to image
psFromHeaderListDesc read 417 headers
mounting sda2 in /mnt/ as type ext2
creating directory /mnt rc = 0
making device sda2 (8, 2) as /tmp/sda2
calling mount(/tmp/sda2, /mnt/, ext2, 3236757504, (nil))

Could the problem be the trailing '/' on /mnt/?

I did a search on DejaNews for this, and came up with several hits
relating to RedHat, so it seems I am not alone with this problem.
