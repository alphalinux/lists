>>> Jon Brinkmann said:
> Help!
>
> I've been trying to put MILO in flash ROM on my EB164 using FMU without
> success. I've tried booting MILO from both ARC and SRM, with the same
> result. After FMU unzips and starts I get the error message:
>
> CIA machine check : reason for machine - check unknown (0x5001b0)
>
> repeated continuously for about five seconds. The error message then
> stops for about five seconds and then starts repeating more slowly.
>
> I've tried both the 2.0.29+ and 2.0.30 versions of FMU and MILO from
> ftp://ftp.digital.com/pub/DEC/Linux-Alpha/Miniloader/v2.0/, although
> milo.rom is only available for 2.0.29+.

Jon,

We don't recommend FLASHing MILO, as a general rule, any more, partly
because it doesn't always work, even on the platforms it (ostensibly)
is supported on (as you've found out).

Another reason is that this facility has (probably) not kept pace with
other changes in MILO and the ARC/SRM consoles, so it may be more prone
to problems with moire recent versions of (any of) them.

One final reason: the MILOs we build on certain platforms depend implicitly
on the correct setup of some pieces of the hardware by the underlying console,
be it ARC, AlphaBIOS, or SRM. By FLASHing MILO, you may be inviting that
hardware to be incompletely set up

Convinced yet? :-) :-(

Now, if you want, we can enter a discussion as to which console to pick... :-)
