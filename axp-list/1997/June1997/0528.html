I'm using a copy of strip compiled from the binutils-2.8.1 sources
against glibc-2.0.4, and it seems to have some memory consumption
problems.

Specifically, when trying to run strip -g on a static library of any
size (like libc.a), it rapidly tries to allocate many dozens of
megabytes of memory.

Has anyone else seen this? I don't follow linux-gcc these days, so
I'm a bit out of the loop. I haven't heard about it happening on i386
