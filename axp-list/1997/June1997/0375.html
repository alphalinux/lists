  • In reply to: Arthur Lobo: "Making full use of 128 MB for application"
> I have a PC164 433 MHz Durango motherboard with 128 MB (Linux 2.0.30) and am
> using it to run a finite element simulation. When I boot the system and
> do a "free" to find the amount of free memory in my system, I get a low figure
> sometimes (eg. 2000 K) and am unable to run my code which requires a large
> amount of memory (eg. 35 MB for the global stiffness matrix). Sometimes I
> find that the amount of free memory is adequate (~110 MB). How can I control
> the amount of free memory available for running my code?
>
You should not worry if the amount of free memory is small. This is
because Linux uses the free memory as a disc-cache. So if you start a
large job the disc-cache will be made smaller dynamically.

However, I also have a 128Mb PC164, and have similar problems. I compare
with a IBM machine also with 128 MB and on my alpha I am only able to run
job at around 75% of the size as I am on the IBM machine. Curious.
