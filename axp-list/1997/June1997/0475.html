On Mon, 16 Jun 1997, Jonathan M. Skanes wrote:

> I have been following the thread discussing compile problems with the
> latest apache on alpha. I applied the patch that was posted, but still no
> luck. The only difference I can see is I'm running RedHat 4.2. Here is
> the error message:
>
> gcc -c -Iregex -DLINUX=2 -DSTATUS -D__alpha__ http_core.c
> http_core.c: In function `set_rlimit':
> http_core.c:1034: dereferencing pointer to incomplete type
> http_core.c:1045: dereferencing pointer to incomplete type
> http_core.c:1058: dereferencing pointer to incomplete type
> http_core.c:1065: dereferencing pointer to incomplete type
> http_core.c:1067: dereferencing pointer to incomplete type
> make: *** [http_core.o] Error 1

In apache_1.2.0/src/conf.h, there is a line that looks like this:

#include <linux/resource.h>

Change it so it looks like this:

#include <sys/resource.h>

You'll also need to add -lcrypt to EXTRA_LIBS in the Makefile.
