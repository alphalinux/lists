On Mon, 2 Jun 1997, Lee Burnside wrote:

>On Mon, 2 Jun 1997, Alex Krowitz wrote:
>
>>
>> Has there been any resolution to this issue? I'm thinking of buying a
>> 500 MHz 21164 to run neural network simulations, and I need something
>> faster than what Pentiums can give me now.
>>
>
>Haven't been impressed with the 500MHz under DEC UNIX so far, bnut our old
>EB164 333MHz beats a P-6 200MHz about 2-1 in floating point (after
>optimization, of course).

Just another data point: I run micromagnetic simulations that are
heavily dominated by FFT's. On a standard sized problem, the
21164/333-Linux box is 2x faster than our PentiumPro/200-Linux, with a
3-1 ratio in the FFT section. (---after upgrading the Alpha box to
RedHat 4.1 from 3.0.3. In particular, the square-root code in the old
libc can be a real performance killer.)

You need to take some care with the code, though. See, for example,

http://www.linuxexpo.org/1997/after/papers/davidm/paper-html/

I don't pretend to be an expert on this, but I would guess that if
your data access pattern is such (or can be made such) that cache miss
rates are low, then the 21164/500 would be a big win for you. OTOH,
if every other datum is coming from a random location, then the poor
Alpha will just be stuck in idle. Perhaps someone more knowledgeable
can comment?
