On Jun 20, Michael B. Schmitz wrote:

>
> The following function comes from the XFree86-3.3 source code. I have
> a Matrox millenium which fails to work with the X server. The server loops
> forever at the lines marked with an 'X' below. Again, I am puzzled. Could
> this have anything to do with the preceding message?

I don't know the "preceding message"

> /* Loop here forever -- mbs */
>
> /* Wait for PCLK PLL to lock on frequency */
> X while (( inTi3026( TVP3026_PIX_CLK_DATA ) & 0x40 ) == 0 ) {
> X ;
> X }

which mode line and exact pixel clock are you using ? does this happen
only for one specific dot clock rate ? to to use a slightly different clock rate
and see if the PLL correctly locks then.

I've never seen this happen for correct dot clock rates on any S3 chip.
no real idea, maybe power supply problem on the card or within the chip
might be one reason for not locking correctly for correct PLL programming...

if this doesn't help try to add a small timeout to this loop (say, after 1e5-1e6 iterations...)
and see what's displayed on your screen...
