Hi

After getting my XL300 up and running, and installing
Redhat 4.0 I did a reboot, the message I got was invalid
operating system. The machine runs AlphaBios. I tried
playing with some of the settings to find myself getting
deeper and deeper in the mire.

Does anyone have any ideas what is going on. An approximate
transcript/settings is:

AlphaBios 5.11

My AlphaBios settings are:

Boot Name: Linux
Boot File: a: \linload.exe
OS Path: Disk 0, Partition 2 \WINNT
OS Options:

When I did the initial installation the hard drive had Windows
NT 4.0 so I guess the OS Path entry was valid (not sure about the
Partition number though, the disk only has one partition). At
this stage everything was OK and the milo prompt came up and I
booted from floppies and carried on with the installation (using
xlt.img and ramdisk.img). The installation went successfully and I
rebooted, the message returned from the AlphaBios was:

Invalid operating system

The thing is none of these settings had been changed since I
first booted into milo.

If I change OS Path from \WINNT to / then the machine dies
instantly not even looking at the floppy disk:

Fatal machine check, CIA state follows: our choice
HalpMasterAbortExpected=0
CIA_ERR: 0x0
< snip >

NT firmware Monitor
Press H for help, Q to quit
** Machine Check occurred.
PC=0xffffffff80705e7c, VA=0x7e1437d2bdb2a5a3

I can provide the snip'ed bit if it will help.

My hard drive is a Seagate 1GByte SCSI with an NCRC810
controller. Before installing Linux it had one FAT partition
and after:

Partition 1 3 MByte FAT
Partition 2 943 MByte Linux native
Partition 3 64 MByte Linux swap

I partitioned the disk using fdisk.

Can anyone give me any ideas as to how I can remedy
my problems?
