Kelvin Lunsford wrote:
>
> Does anybody out there know of a C compiler (DEC/GCC/NT) for the
> Alpha processor which will let you set the default int size to 64
> bits. This is for an embedded application, the resulting binaries
> would not be running under an operating system.

How about:

% gcc -Dint=long "-Dunsigned=unsigned long"

This works even in cases of "unsigned int" and "unsigned long"
because on Linux/Alpha GCC sizeof(long)==sizeof(long long)==8.

This is the only solution I can think of. The default programming
model for both DEC C and gcc on Alpha is sizeof(int)==4 and
sizeof(long)==8. There's no easy way to change this (hell it was
a major feat back in the early days of my 32-bit Linux/Alpha port
to convince gcc to generate code where sizeof(void *)==4)

Does this help?
