I can understand the desire of Rick to have a password on single user
boot. We have a number of systems running in rack mount enclosures. Each
of these enclosures has a key lock on the front panel so that the drives
and switches are not accessable. At the same tiime we have theses systems
connected to a keyboard/monitor switcher so that the consoles of all the
systems are available. A single user password protection scheme increses
the difficulty of gaining control of the systems by at least one order of
magnitude possibly more. Not a bad security enhancement for a line or two
of code changes?
