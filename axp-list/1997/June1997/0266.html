> > I'm trying to install over FTP, but the ethernet interface never seems to
> > come up. When booting, the kernel sees the card and recognizes that I'm
> > using the twisted pair interface. After I specify my IP, subnet mask,
> > etc, I get link status on the switch its plugged into, but not on the box
> > itself.
> >
> > The weird thing is that I can go to another machine, and ping the
> > interface just fine. This works from any subnet.

This is strange. I suspect it is some routing- and/or NS-related problem.
I have run into a bunch of them while using the RH-provided configuration
files in /etc/sys*/network* .

I will report them here, so maybe others can avoid them :

With the standard configuration which first brings up eth0 and then lo (due
to the ascending alphabetical order) bringing up lo resulted in a hang where
the machine was desperately trying to resolve things like the netmask (surely
given numerically ...) which our local Server couldn't resolve what caused
long timeouts.

I got it working then by first bringing up lo (which probably works because
there is no route to the NS then and it doesn't try that weirdness) and eth
later.

Everything worked fine, until I noticed I had forgotten to set the gateway
so I could reach the outside-world. Setting the route manually worked fine,
so I just set the GATEWAY parameters in /etc/sys*/network and bang - system
hangs for about a minute as soon as INIT goes to the default runlevel.

Being fed up of this I put it in rc.local (where it shouldn't be - right)
until I find the time to port the "handmade" special init scripts which
work fine for all the other machines - they don't do fancy variable
extraction from some config-file, but they work ;-).
