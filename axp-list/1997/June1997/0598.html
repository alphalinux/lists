This looks like interrupts are being lost; if this was the kernel file
whose name has "srm" in it, be warned that that WILL NOT work booted
from MILO...

> I've upgraded to Alphabios 5.29. May be installing miata-srm-myri-fwup.img
> from gatekeeper would help, but how ? I've installed it and it didn't seem

All the following file names are from:

gatekeeper.dec.com:pub/DEC/Linux-Alpha/Kernels/

miata-kernel-2030-test.gz # boot from MILO, tulip.c WILL run 21143
miata-kernel.gz # boot from MILO, tulip.c won't run 21143
miata-kernel.img # floppy image, EXT2 FS with miata-kernel.gz
miata-milo # MILO for booting from AlphaBIOS
miata-milo.img # DOS FS floppy image with MILO and linload.exe
miata-srm-myri-2030.gz # boot from SRM via aboot

Please DO NOT USE the following file; it contains BETA test software that
is ONLY of use when needing SRM console to boot a MIATA with Myrinet card
installed.

miata-srm-myri-fwup.img

It should NOT harm anything if installed, but unless you have need, do not
bother installing it, it won't buy you anything.
