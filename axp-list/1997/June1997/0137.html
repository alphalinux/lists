> I'm don't know why it fails, but the reason I made that patch is so you
> can do stuff like:
>
> gdb> p printf("Hello World\n")
>
> (passing the "Hello World\n" string constant causes gdb to implicitly
> call malloc()). It certainly used to work with ECOFF. With ELF, I
> probably always had debug info available while working on these
> things, si I never noticed. In other words, it's quite possible that
> the problem got introduced during the ELF->ECOFF switch.

Looks like something got changed in the switch. Without the patch, and
with just a simple program like

main(){printf("blah");}

gdb> p printf("Hello World\n")

produced

evaluation of this expression requires the program to have a function
"malloc"

Adding a simple malloc the program, and doing the same, produces

Cannot access memory at address 0x20100c30.

A static binary produces:
The program being debugged stopped while in a function called from GDB.
When the function (malloc) is done executing, GDB will silently
stop (instead of continuing to evaluate the expression containing
the function call).

Which I'm guessing is because of a lack of debugging symbols??
