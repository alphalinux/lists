> > Either this, then somone should look into it, or bad RAM which I doubt, if
> > your Alpha uses ECC logic. On Intel the most common cause for internal
> > compiler errors is bad RAM.

> Then it wouldn't be the same file that was giving the compiler error every
> time.

No. I have seen this happen several times. It reproducibly bombed on the
same file. Once the same machine bombed on ls (!) with "illegal instruction".
I suspected RAM problems, so I started X (hoping it would cause the VM
to be moved around), but ls in an xterm gave exactly the same effect.

As long as you have plenty of RAM (no heavy swapping) it is very likely
that a RAM error will occur in the same place over and over.

I finally pulled out two of the SIMMs and voila - system stable again.
