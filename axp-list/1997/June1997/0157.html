Last time I bother you, I hope.

I post this message for the people who might one day search this
mailing-list just like I did. I finally succeeded in playing Quake on
my UDB with the shareware version.

These are the caveats I found :
-- install the shareware on a dos (or linux + dosemu) x86 system; then
create a directory for Quake, put in it a subdir named "id1" and in
this subdir the pak0.pak file. It is useless to unpak this file with
the qutils you might find on some sites.
-- get the alpha-ELF-xquake-1.06 executable. There is a "Linux Quake
Homepage" somewhere, use Altavista. Beware of the version, do not get
the 1.01 but the 1.06. Put the executable in the Quake subdir.
-- use a 2.0.x kernel WITH THE AXP-DIFF or the latest 2.1.x kernel.
Without the axp-diff, there is a bug in the ipc part of the kernel,
which prevent from using shared memory under X (with XFree servers).
The bug has been corrected somewhere in the 2.1.x kernels (at least,
it has disappeared in 2.1.42).
-- at this point you should be able to watch the demo and dump a core
when you try to play yourself. This seems to be a libc problem; the
libc in RedHat 4.0 has been reported to work, and the one in 4.1 not
to work. I found on a friend's Alpha an old libc-1.98 which works
with Quake. No need to be root to use it, just adjust the
LD_PRELOAD variable before launching the Quake binary. I also tried
a libc-1.99 but this one does not work.
-- to have the sound, ensure you have read AND write access to /dev/dsp.

So I use the following shell script to run Quake:

#!/bin/zsh

QUAKE_DIR=/usr/local/games/quake

cd $QUAKE_DIR
LD_PRELOAD=$QUAKE_DIR/libc.so $QUAKE_DIR/alpha-ELF-xquake-1.06 $*

where $QUAKE_DIR/libc.so is a symbolic link to the libc-1.98.so file.

Now I will try to set up a 320x240 video mode to have full screen and
smooth animation (640x480 is too much for my UDB, although it runs at
300 MHz).

Anyway, many thanks to all who helped me.
