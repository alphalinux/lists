OK,
since I do not know if this is publically known and since some other
reported problems with the X11 MIT Shared Memory extensions and I had those
problems too, I just wanted to report that it is now working!
When applying the axp-diffs-2.0.30 to the vanilla 2.0.30 kernel everything
works how it should.
But why are these changes not yet integrated into the 2.0.30 mainstream?
I had the SHM problem for at least the last 20 releases, because I never
applied the axp-diffs. I remember someone saying that those diffs would only
be needed for debugging the kernel with a remote gdb. Obviously this was not
fully correct.
But now everything works fine and I'm happy again ;)
Programs now working properly: XAos, Virtual Game-Boy and XAnim.
CU
nils
