On Jun 18, Michal Jaegermann wrote:

> So what is wrong with milo-2.0.26? As it was said many times
> on this list a version of milo must match a version of kernel
> ONLY when you MAKE it, and not when you are using it.

right now, there is nothing wrong with milo-2.0.26, but there will be
a time... it's not possible to load 2.0.30 kernel with pretty old
MILO (don't remember which version, I'd guess even milo-2.0.0 can't
load kernel-2.0.30).

and once this happes (or I need some new hardware driver for MILO)
I'm in trouble so I prefer to ask for help in advance and not when on fire;)...

> OTOH in the past I was using milo-2.0.30 (one even which I personally
> hacked and recompiled :-) without problems. Shrug!

XL266 too ? which firmware release ?
milo-2.029 is no problem for my UDB166, only for XL266...

> > beside the installation problems from witch i can write books the Xserver
> > witch comes with 4.2 also does not recognice my Number nine 330 (trio64+)
> > card anymore.
>
> Number nine on Alpha made my Xmas pretty "interesting", so this is not
> the first time something like that happens. Harald knows something
> about it. :-)

it's not #9 specific, it's a problem with S3 chips and some AXP boxes
(like AlphaStation255, UDB, but e.g. not for my XL266;)

whoevery has hangup problems with S3 chips (not ViRGE), please get
the XFree86 3.3 server and give it a try. if it still locks up (only then!)
please add Option "no_pci_disconnect" and try again.
this seems to fix these lockups for a number of AXP reports...
