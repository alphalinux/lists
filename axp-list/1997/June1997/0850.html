Hi!

I just noticed that libm 1.99 as it is shipped with RH Linux 4.2 seems to
have all the various long double math functions implemented (like sqrtl,
powl, etc...).

Unfortunately, I cannot find them being declared in any includefile, so I
just wanted to ask what the status of these functions is.

Are their declarations missing from "math.h" because they are not (yet)
fully functional and shouldn't be used or is it save to use them and
supply the missing function declarations yourself?

The problem with the current situation is that Autoconf--scripts may fail
in the current situation. -- For, example doing an "AC_CHECK_FUNCS(powl)"
yields the result that the function "powl" is available on Linux/Alpha. --
Unfortunately the compilation of C++--sources using "powl" fails with an
error after that because a missing function prototype is treated as an
error when compiling C++--code.

Of course this is only a minor problem, but I just wanted to make you
aware that there is such a problem at all... ;-)
