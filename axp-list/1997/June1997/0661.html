Hi,

I have problems in installing RedHat 4.2 Package on an AlphaStation
600 5/266. I can boot the system with the alcor.img followed by
ramdisk.img with the SRM console by typing on it:

boot dva0 -file vmlinux.gz -flags "load_ramdisk=1 prompt_ramdisk=1
root=/dev/fd0"

Then, I come in the RedHat Installation and set everything for
NFS installation. I set the partitions on the disk, set the swap
space and choose all the packages to install in the root partition.

The installation process hangs up at the formatting level
of the root partition. The virtual console 5 (ALT-F5) is
stopped at "Writing superblocks and filesystem accounting
information:" line.

I have no idea of what is happening? if somebody has an idea,
please, help me !!!

I must say that when I partitionned the disk, the fdisk program
had not seen the Cylinders,heads,sectors setting of my drive.
I did the setting by myself. The drive is a Seagate Barracuda
2 Gig (ST32550W). The setting I put is:

255 Heads, 63 Sectors, 261 cylinders
Units = cylinders of 16065 * 512 bytes
