On Thu, Jun 19, 1997 at 11:38:31AM +0200, D. Unger wrote:

> > Does anybody know why the sync command (RH 4.2 on PC164, AH2940UW,
> > 4.3 Seagate 34371W) hangs forever and can't be killed (kill -9
> > then also hangs)?
> >
> it's even worse: after having given the (hanging) "sync" command, I can't
> shutdown the machine, the command is just ignored after telling
> "the system is going down", kill -9, or init 0 is also ignored

I had this problem as well. amd seemed to be the cause -- somehow it made
any program inquiring all mounted filesystems (including sync) to go into
"uninterruptable sleep".

Rebooting (the hard way) into single user mode (so amd was not started)
and "rpm -e"ing it fixed it. If you need amd, look for a upgrade or
perhaps a configuration problem.
