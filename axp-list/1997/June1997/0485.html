I have had a frustrating time trying to boot Redhat 4.2 onto an
AS 200 (4/233.) I am new to Linux on Alpha and this will probably account
for most of my confusion. I hope you can help me regardless, perhaps
by pointing me to other posts or documentation. I have already scanned
the axp-list archives (and voted for comp.linux.alpha:)

- There is no MILO (at least, no pre-compiled MILO) for this machine.
- I have no Linux/Alpha system (this is to be the first) so I cannot
compile MILO and/or aboot.
- I have no root file-system, other than perhaps the one on CD, or the
ramdisk, if I could manage to use them.

Luckily, Redhat provides a floppy image to boot Avantis (images/avanti-s.img)
via aboot from SRM.

I have tried (always from SRM):

>> boot dva0 -file vmlinuz.gz -flags load_ramdisk=1

The kernel is decompressed and loaded, but the root file system is
not found and the machine hangs after printing:

[MS DOS FS Rel. 12,FAT 12,check-n,conv=b,uid=0,gid=0,umask=022]
kernel: unaligned trap at fffffc0000393314: fffffc0001ff0011 29 1
[me=0x5a,cs=4050,#f=90,fs=57810,fl=57810,ds=17830,de=23130,data=19315,se=23130,ts=1515870810,ls=23130]
Transaction block size = 512
Kernel Panic: VFS: Unable to mount root fs on 08:02

Ok, I figure it is trying to find a root file system in the DU disk.

>> boot dva0 -file vmlinux.gz -flags root=/dev/fd0 load_ramdisk=1

Now it gets as far as it froze before, but VFS asks me to insert the
root floppy and press RETURN. At this point I insert:

- the ramdisk, which causes an error similar to the above VFS panic.
- the boot disk, in which case the machine freezes after printing:
"Unable to open an initial console"

OK, I don't know where else to get a valid root partition, so I try to
boot directly from CD.

>> boot dka300 -file kernels/avanti.gz -flags root=/dev/scd0

Now it gets as far as before (notably it finds and loads the kernel
from CD,) but then prints the following ad infinitum:

ncr53c8xx_abort : abort call
ncr53c810-0: abort ccb=fffffc0001fe8f30 (skip)
SCSI host 0 abort (pid 1) timeout - resetting
SCSI bus is being reset for host 0 channel 0
ncr53c810-0: restart (scsi bus reset)

It may be obvious that I don't have a good grasp on the Linux/Alpha boot
sequence. I have looked through azstarnet and redhat sites pretty
thoroughly, and have looked for axp-list archives. Forgive me for
possibly missing the obvious. Thanks in advance.
