Since xemacs 20 is so amazingly big that it resists the dll-ization
that previous versions succumbed to, and really requires being built
with the new linker and all that portends, I've made binaries available.

It is just a tarball not an rpm since I had to do funny things to get
it built with the new linker against stock RH4.1 libraries. It
expects to be unpacked under /usr/local.

Get it at ftp://ftp.redhat.com/pub/alphabits/xemacs-20.2-axp-bin.tar.gz
or any of a number of mirrors that have sprung up.
