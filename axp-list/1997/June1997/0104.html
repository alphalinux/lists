If you're writing the code specially for alpha, as well as optimizing for 64-bit code, your simulations should be much faster than pentium systems. It's when you take C code written for i86/Linux that's 32-bit and recompile for alpha that it isn't as optimized, and gcc is just slow for compiles on the alpha; a) because gcc isn't optimized for the alpha yet and b) 64-bit code takes more time to turn out that 32-bit code. Your applications should see considerable performance gains over pentium systems once compiled, however.
