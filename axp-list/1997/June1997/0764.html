Maybe someone reading this list has a solution or maybe a
work-around for my problem: at the Chair of Process Control
Engineering, we're developing an open communication system for
process data. The source run's fine on Linux/Intel, HP-UX, and
Solaris using g++ 2.7.2.x. However on the Alpha, I can't get the
whole thing to compile because gcc 2.7.2.1 just dumps core whenever
it sees a header file belonging to the RPC stuff.

An example of how to throw the error is quite simple:
#include <rpc/rpc.h>
#include <rpc/pmap_clnt.h>
int main(int, char **)
{ return 42; }

I'm sure it isn't the fourty-two that makes gcc fail :-)
