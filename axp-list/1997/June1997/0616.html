On Thu, 19 Jun 1997, florit wrote:

> Hello again. I followed the advice of several people, attempting to boot
> the cabriolet with milo (from NTarc), then booting with the cab.img, and
> when I'm prompted with "Insert root floppy and press enter", I put the
> disk created with ramdisk.img and it dies (kernel panic)
> kernel: unaligned trap <blah blah blah>
> <stuff with hex codes and stuff>
> Transaction block size=512

I'm not 100% sure, but it seems to me that a while back I was getting
unaligned traps from the kernel at boot up when it came time to fsck the
filesystems, thus kicking me into a "repair mode". This kernel was the
pc164.img file from the rh 4.1 CD -- _supposedly_ for my system. I ended
up booting an old kernel that I had and then recompiled the new kernel for
my system. It solved my problem. Maybe this is yours as well?
