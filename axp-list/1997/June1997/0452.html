Mikko Torni writes:
> If this happens only with the installation disks, why not try mem=64 as
> boot parameter?

I did a fresh install of RH 4.2 on my Avanti system (AS 255, 128MB) and I am
having big kernel (2.0.30) instability problems. My Alpha has SRM and I
usually use aboot (which recognizes all my memory, as opposed to milo).

When I try to compile glibc, at some point the system halts out of the blue
(actually in the blue SRM console ;-), no messages in /var/log/*, nothing. I
have tried several combinations with milo (2.0.25), the installation kernel
and a custom 2.0.30 one (built from the updated rpm kernel source package) and
found out that:

* if I boot without setting MEMORY_SIZE (the variable defaults to 32), I can
compile glibc without problems.

* if I set MEMORY_SIZE to 128, the system hangs while compiling glibc.

* if I set MEMORY_SIZE to 64, gcc dumps core while compiling glibc, but
the system doesn't hang.

I don't think it's a hardware problem, because this Alpha used to run DU
without problems (but I could be wrong, of course).

I compiled a 2.1.42 kernel to check if the problem would go away, but I can't
boot it with aboot (I included the SRM boot option). Anyone knows how to boot
a 2.1.x kernel?.

I also compiled the latest milo (2.0.30) and dumped the milo.dd image to a
disk, but when I boot and see the familiar "Jumping to code...", instead of
getting the milo screen, I'm back to the SRM console with the message "Halted
CPU 0 etc".

Just for the record, but any hints are extremely welcome ;-)
