> Does anyone have the take on the difference between things found in the
> "alphabits" directory and the "contrib" directory? Are the files in
> alphabits done by Red Hat themselves?

No, the files in alphabits are done by me.

Red Hat was kind enough to give me ftp space since I am behind a
firewall here at A&M and quite a few people have problems getting
though.

As for the 2.8.0.1 binutils in the contrib directory, all that should
be fine _except_ if you are upgrading straight from a stock RH
release. The version of binutils RH has been using since 4.0 has a
bug in the assembler which was exposed by linker fixes. Thus all of
the static libraries that you use must be upgraded at the same time.

That goal is what the files in alphabits accomplish. Upgrade all of
glibc, gcc, and binutils simultaneously and everything works.
