I am having problems with pppd and chat.
I have an AXPpci 33 (Noname) 166MHz, I have RedHat 4.2 installed on it.
The install went smoothly *thank god*.

So now I am trying to setup the machine to run ppp full time over a 28.8
dialup. As I have read on this list there is something weird with these
serial ports. I have not been able to get an external modem or a mouse to
work on either of the two serial ports. So I put in a 28.8 internal USR
modem on cua3 (com4) IRQ 5. I have run setserial to set it to the correct
IRQ.

If I run minicom I can dial out with out any problems.
If I run minicom and connect to the remote modem, then quit without reset
(from minicom) and run pppd manually, it just dies with this error
"Jun 17 16:20:59 alpha pppd: Exit."

Now if I use a script (which works correctly on an x86 machine with the
same modem and any other modem I have) I get these errors

Jun 17 16:27:08 alpha pppd[3714]: pppd 2.2.0 started by root, uid 0
Jun 17 16:27:09 alpha chat[3716]: send (ATZ^M)
Jun 17 16:27:09 alpha chat[3716]: expect (OK)
Jun 17 16:27:54 alpha chat[3716]: ATZ^M
Jun 17 16:27:54 alpha pppd[3714]: Connect script failed
Jun 17 16:27:54 alpha pppd[3714]: Exit.
Jun 17 16:27:54 alpha chat[3716]: alarm
Jun 17 16:27:54 alpha chat[3716]: Failed

I have compiled pppd into the kernel (not a module). The modules which I
compiled were not loading correctly.

Has anyone got pppd working? If so what did you do, any pointers in
helping me get this working?

My ppp scripts are not using the redhat scripts, because I can't get X
working due to the non-funtional serial ports ( and I don't have a PS/2
connecter). I am using similar scripts to what was published in Linux
Journal a few months ago. And As I stated before the same scripts work on
x86 machines.

Thank you in advance if you can help out here.
