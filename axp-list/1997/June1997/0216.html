I have a f77 program which contains the following non-standard (but
usable on AIX, Solaris, Intel/linux, DU) lines of code:

...
call system('hostname > temp123456')
call system('whoami >> temp123456')
call system('pwd >> temp123456')
call system('date >> temp123456')
open(unit=11,file='temp123456',status='old')
rewind(unit=11)
read(11,'(a80)') host
...

After compiling/linking statically (so that all run-time libs are
bound into the executable) on a Digital Unix Alpha, this code
runs fine on the DU Alpha, but when PORTING THE EXECUTABLE to an
Alpha running Linux, I get the error message
forrtl: severe (29): file not found, unit 11, file temp123456
Command exited with non-zero status 29
and the program stops. Apparently the file temp123456 hasn't been created
and the system call didn't work,

but why?

Any ideas are greatly appreciated,
Ditmar.
