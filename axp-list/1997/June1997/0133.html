This is of course not really Alpha specific, but important and perhaps
unappreciated, so here it goes. Linux uses a secure X model where
only specific computer/account pairs have access to your display. This
is important security precaution, because if, like, say, SGI, your X
server allows wide access, you can read its keyboard, contents of the
screen and all (try 'xev -display SGImachine:0' or
'xwd -root -display SGImachine:0 | xwud' sometime).

You have to propagage the session key from file .Xauthority
on your desk box to all accounts from which you will be running
programs. I do it by having something like

rsh hostA xauth merge - < .Xauthority &
rsh hostB xauth merge - < .Xauthority &
...

in my .Xclients file (of course I make sure that host{A,B...} are
rsh-enabled via .rhosts or /etc/hosts.equiv or otherwise.
