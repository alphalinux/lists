Kirk Saban wrote:
>
> Stephan, by upgrading to 256 bit access, I am assuming that you
> mean you added more ram to fill up the entire 8 RAM slots on the 164
> board?
>

Upgrading from 128 Bit MA to 256 Bit MA on a PC164 is done by filling
all SIMM sockets with memory modules of the same size and by pulling
out a certain jumper on the board (see PC164 manual).

BUT:
My system running RH4.1 (kernel 2.0.27) sometimes shows problems
resulting in
- cc1plus (the c++-compiler proper called by gcc or g++) falling into
interruptible sleep (STAT 'D' in ps).
- 'Unable to handle kernel paging request' messages (if so the machine
is 'dead' and must be rebooted with reset button.

My current workaround is to run gcc on a small c++ source file
immediately after booting. If this comes back, everything seems
o.k. and no kernel problems appear so far.
