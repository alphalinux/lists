I agree with all of the above...except for the "instead" part. ;-)

>In case anyone does not know what Ted is doing, he has been instrumental in
>writing and debugging filesystem tools and utilities for E2fs.

Seriously, I remember how important ext2fs was for my work years ago
when I was first using Linux to do g77 work. In fact I remember being
happy just having extfs with its long filenames, because when I first
started using Linux, it supported only 14-character names or some such
thing. Quite painful. It's quite important to ensure that the base
software we all depend on is well-supported.

BTW, if I do get both a donated 21164 _and_ enough funding to provide
half of what's necessary, I'll match the funds to set up the UDB so it serves
well as another Alpha to build and test, and especially benchmark, g77
on. That way, I'll more easily be able to avoid introducing optimizations
that help the 21164 but hurt the 2106x processors. Plus, it might
be easier to identify performance improvements (in particular, spot slow
code) on the UDB. Doing that would mean adding a pretty large disk, RAM,
and such to the UDB, plus I'd have to buy an extra monitor for it, which
is why I'd expect to spend some serious money.

(This is one reason I'm happy to have my 486/33 kicking around, because
it's often instructive to see how one's code runs on older technology.
Though, I'm wondering if I could accomplish much the same thing by building
and benchmarking g77 entirely under em86 on the UDB. :)
