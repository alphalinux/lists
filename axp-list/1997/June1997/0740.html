>>> John Banghart said:
> I have an AS255 with a video card it calls a PBXGB-AA/CA.

This is a TGA2, aka Powerstorm, prolly the 8-plane version.

> The chip is a Cirrus Logic CL-GD5424 but when I select that option in
> Xconfigurator, X won't run. It complains the Digital is a unknown chipset
> and that none of the specified devices can be found.

Not surprisingly, on two counts: the VGA on board is just for text-mode
console purposes, and prolly has little memory to work with, and the XFree86
XF86_SVGA server only has support for Millennium/Mystique and Trident, IIRC.

And you're out of luck WRT an X server for TGA2, unless you decide to run
Digital UNIX or NT, of course... :-) :-(

> I am going to try an STB Lightspeed 128 and see if I have any more luck,
> but I would like to use the card that came with the AS255 if I can.

You'll have much better luck checkiong the XFree86 docs to see what cards
are supported on Alpha; IIRC, the STB is not one of them.

> I am running RH 4.1, kernel 2.0.27.

The latest and greatest XFree86, ie release 3.3, is available via RPMs from
Red Hat, but they are be built against RH 4.2 and might have problems run on
top of RH 4.1; give 'em a try and see...

Good luck.
