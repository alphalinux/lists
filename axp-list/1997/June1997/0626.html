I'm preparing g77 0.5.21 for release to alpha testing, and am integrating
patches and suggestions people have offered to improve the likelihood that
g77 (gcc) will compile out of the box for the typical Alpha user.

I think I've got all the info I need on "modern" ELF systems, and if I'm
wrong, the several new g77 alpha-testers who've volunteered as a result
of reading this list will catch it.

What I'd like to also get "right" the first time, so I don't need
so many iterations of alpha testing, is out-of-the-box compiles for
obsolete systems (like on the Cabriolet I'm using, and have to return
soon).

There seems to be two issues, at least on my system. One, ELF support
requires generating assembler directives my `as' doesn't support, so
my system, for example, shouldn't default to ELF, but rather to ECOFF
(broken though it is, apparently). Two, C++ initializer support
also requires unsupported assembler directives.

For now, I've modified the config.guess on my gcc, along with configure
script, to identify as "alpha-unknown-linuxoldas" any system whose
`as --version' prints the string " version 2.6 ". Does anyone know
if this is right, wrong, etc., or have a more definitive way of identifying
a version of `as' that doesn't support the `.section' directive?

Presumably there will be ECOFF systems with a newer `as', however.
Is that the case? If so, is there a definitive way to tell whether
someone is likely to want ECOFF to be built?

Keep in mind I'm looking for fairly definitive info -- in particular,
please don't respond to this list with speculation, just to keep traffic
down, though if you want, you can e-mail it to me directly. I'm sure
there's one or two people who really know the history of ECOFF and ELF
on GNU Linux/Alpha!

Thanks in advance!

tq vm, (burley)
