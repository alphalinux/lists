I've managed to get hold of an XL-300 which I plan to use to replace my
Multia. I've got both NT and linux installed, and have the latest 'stable'
kernel 2.0.30 with the axp patches that Jim Paradis provided. I'm running
the milo from 2.0.29+ and have AlphaBios 5.24.

On a cold start (ie, turning the system on) or on a linux reboot I get the
following errors. On the other hand, if I boot into NT first, and then
reboot into linux, it all works. Given this, I doubt that I need to reset
and IRQs, but I'm happy to be proved wrong. Comments, suggestions etc welcome.

I should point out that the XL300 flies. I'm planning to make it my
everyday box to replace my DU Alphastation :-)

Please verify that the correct IRQ is being used for your board,
and that the motherboard IRQ jumpering matches the PCI setup on PCI
systems.
