Michal Jaegermann wrote:
> > How about:
> >
> > % gcc -Dint=long "-Dunsigned=unsigned long"
>
> This would convert 'unsigned int' into 'unsigned long long'
> (or we would recurse on unsigned? but this is easily avoidable)
> not something you likely would want to see. You can probably
> control a code you write to avoid such surprises but headers
> will be a fun.

Actually, I tried this out before posting it and it works just
fine. It does not recurse on the "unsigned" bit. While it
*does* turn "unsigned long" into "unsigned long long", that is
no problem on Alpha; "long long" is a recognized datatype in
GCC, and on Alpha it just *happens* to be exactly the same size
as "long" (in the way that "int" and "long" used to be interchangeable
on 32-bit architectures).

In short, the solution I propose above (while a bit ugly and
abhorrent to language purists) should solve the user's problem.
