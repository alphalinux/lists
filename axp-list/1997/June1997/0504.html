Hello all,
A friend of mine has an AlphaPC board (cabriolet), and we've been
trying to install linux on it with much difficulty. There isn't much
documentation in the RedHat 4.2 tree, and we'd like to install from the
network. We managed to get a milo console from the box (using the
milo/cabriolet/milo program) using the milo/arc/linload.exe program. Our
problems came when trying to boot the redhat installation disks.

I put the images/cab.img and the images/cab-s.img onto two
seperate floppys, thinking that they were two different boot/supplemental
floppies. I also put the ramdisk.img onto a floppy simply because of the
similar naming convention on the i386 install. I tried booting from the
cab.img with the cab-s.img as the second RAMDISK (when prompted). The boot
line was as follows:

MILO> boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1

This was taken from the installation FAQ in the alpha tree. I was hoping
the same FAQ would tell me which images were the proper ones to use, but I
couldn't find that information.

Using the two disk as I mentioned above, the kernel would boot and detect
the hard drives and floppy, but it would die with the error

"Cannot initialize console device"

Or something very similar to that. When I tried using the ramdisk.img
floppy as the second disk, it gave me an error to the effect that it
wasn't the proper floppy. I am not sure what I am supposed to be doing. I
would very much like to get this alpha running linux. This is a beta
board, and any help would be appreciated.
-peace
