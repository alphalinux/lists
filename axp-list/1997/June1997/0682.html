I have to admit that I cannot help you with gimp but I just wanted to report
that I have the same problem with _any_ X11 program that tries to use the
MIT shared memory extension.
With some programs it helps to set the DISPLAY environment variable to
DISPLAY=hostname:0
instead of
DISPLAY=:0.0
but that does not really help. You do not use MIT SHM then :(
BTW: I tried this with up to XFree 3.3 on my Jensen, installation based on
RH4.0. Programs that show this effect: XAos and 'Virtual GameBoy'. In the
case of XAos setting the DISPLAY variable helps (s.a.) and VGB has a
commandline option to disable MIT SHM.
Since MIT SHM is needed for some applications, seems GIMP is one of them, it
would be really very helpful to know where the problem is.
One thing that came to my mind, may be fully wrong :) , is that there may be
problems with memory access with the X11 libs and the Linux kernel for the
shared memory? I know that SHM is a very critical thing...
