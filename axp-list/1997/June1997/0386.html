On Fri, 13 Jun 1997, Ken Haste Andersen wrote:
> I have a 500 MHz PC164. I run floating point intensive calculations
> compiled with g77 (-ffast-math -funroll-loops -fno-f2c -O3). These run
> approx 2 times the speed of a 180 MHz Ppro, which is a bit slower that
> what is to be expected. Therefore I tried to compile the programs under
> Digital Unix, using their f77 compiler (-O4 -arch ev5), and transfering
> the code to my Linux station. Surprise! - a factor of two in speed
> increase!
>
> Is g77 (and then gcc) really that bad for floating point apps.?
>
> Ken
>
> -----------------------------------------------------------------------

I did the same thing with a 433 MHz PC164, and my own
floating-point intensive code. I found that g77-optimized
code ran at about 85% the speed of DEC-f77-optimized code
(and nearly twice the speed of f2c+gcc optimized code!).

I think that these kinds of compiler performance questions are
very dependent on the program involved.
(BTW, my compiler options for g77 were
"FFLAGS = -ff77 -ffast-math -O2 -C", and for DEC f77 the same
as yours, except with the "statically linked" option.
