Hi,

I have got myself into a bit of a tangle over setting up SRM on my
AlphaStation 255 to boot linux without the need for a milo floppy & am seeking
help. This is what I did :

Machine is a AlphaStation 255 300 Mhz, 32 MB RAM, 2 GB SCSI II, NCR53c810
SCSI controller, de425 Ethernet adapter.

What I did :
- Used the RH 4.1 distribution, avanti.img, ramdisk, milo.
- successfully installed RH4.1(used fdisk, not minlabel,with params 64heads,
32sectors, 2007cylinders), machine on the network but no X due to
unsupported Video card. Partition table starts with cylinder 3(first 2 MB
left out as specified), sda1 is swap..32 MB, sda2 is / ...remaining disk.
- got fed up of booting from floppy & decided to set SRM up to boot linux
automatically. Did the following :

at SRM prompt >>>

boot dva0 -file vmlinux.gz -flags "root=/dev/sda2"

/sbin/swriteboot /dev/sda /boot/bootlx

reboot

at SRM prompt >>>

set bootdef_dev dka0

set boot_file 2/vmlinux.gz

set boot_osflags "root=/dev/sda2"

set auto_action restart

Now, on powering up the machine SRM finds the kernel, starts booting, and then
says

aboot:Linux/Alpha SRM bootloader version 0.5
aboot:swatching to OSF/1 PALcode version 1.46
aboot:valid disklabel found: 0 partitions
aboot:booted_dev = `scsi 0 6 0 0 0 0 0', guessing boot_device=`sda2'
aboot:invalid partition 2
aboot:mount of partition 2 failed
Enter kernel name and arguments:
aboot>

I'm stuck at this point & do not know what to do next.

Any help is appreciated.
