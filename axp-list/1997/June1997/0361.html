>
> I am trying to compile an assembly language program using the GNU
> assembler gas on RHL4.2 and I keep getting Segmentation Fault (core
> dumped) errors. I have checked my code a zillion times and just can't
> figure out what's going wrong. Anybody have any suggestions or good
> references for Alpha Assembly Language?
One common error is on function calls. Generally, the first 6
parameters are passed as registers (eg. 16-21). If you are passing a
floating point parameter, it is passed by a corresponding fp register.
Also, register 31 (and f31) always contains zero. Also, you should be
aware that some registers must be preserved over calls where some can
be trashed.
