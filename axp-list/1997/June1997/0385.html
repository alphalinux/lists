Hi all,

I have a 500 MHz PC164. I run floating point intensive calculations
compiled with g77 (-ffast-math -funroll-loops -fno-f2c -O3). These run
approx 2 times the speed of a 180 MHz Ppro, which is a bit slower that
what is to be expected. Therefore I tried to compile the programs under
Digital Unix, using their f77 compiler (-O4 -arch ev5), and transfering
the code to my Linux station. Surprise! - a factor of two in speed
increase!

Is g77 (and then gcc) really that bad for floating point apps.?

Ken
