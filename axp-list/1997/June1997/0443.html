On Fri, 13 Jun 1997, Hexadec wrote:

> I recently aquired an old Multia to use as a IP-Masq gateway for my LAN.
> The problem is that when I attempt to compile a kernel it dies on the
> same file, with the same error each time:
> gcc -D__KERNEL__ -I/usr/src/linux-2.0.30/include -Wall
> -Wstrict-prototypes -g -O2 -pipe -mno-fp-regs -c FlashPoint.c -o FlashPoint.O
> cpp: output pipe has been closed
> gcc: Internal compiler error: program cc1 got fatal signal 6
> make[3]: *** [BusLogic.o] Error 1
> make[3]: Leaving directory `/usr/src/linux-2.0.30/drivers/scsi'
> make[2]: *** [first_rule] Error 2
> make[2]: Leaving directory `/usr/src/linux-2.0.30/drivers/scsi'
> make[1]: *** [sub_dirs] Error 2
> make[1]: Leaving directory `/usr/src/linux-2.0.30/drivers'
> make: *** [linuxsubdirs] Error 2

When you want to include support for Buslogic adapters in a Linux kernel
for the Alpha, you should say no to the support for Flashpoint adapters.
So, when the config scripts ask: "Omit Flashpoint support?", say yes. The
flashpoint support is 32 bit only, and produces a signal 11 on my system
if I try to compile it.
