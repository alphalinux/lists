I hate to bring this up, but none of the threads I have
checked had a happy ending.

Never a single memory problem with 4.0 and 2x8+2x4=24M
on the Multia 166. I upgraded to RedHat 4.1 with
2x32+2x8=80M, and now have serious memory problems ...
>kfree of non-kmalloced memory: fffffc000046xxxx
>Unable to handle kernel paging request at virtual address 87xxxx
>random(36): Oops 0
>idle task may not sleep
>Aiee, killing interrupt handler

These problems seem to go away when I switch to 2x32=64M.

-They are true parity, 70ns.
-Latest MILO(built on kernel 2.025)
-latest linload.exe that came with redhat 4.1 cdrom
-latest(3.8) firmware from DEC

Has anybody solved this problem?
