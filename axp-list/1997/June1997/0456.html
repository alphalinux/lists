Hi!

As I'm getting prepared to go 21164PC (its getting close!) I've read
through the 164SX docs (ftp://ftp.digital.com
/pub/DEC/DECinfo/semiconductor/literature/dsc-library.html) I've noticed
it will be using the Cypress CY82C693U, which is non-supported chip
by Linux. I guess it will be working in a "ide-compatible" mode, but
supporting EIDE DMA-2 would greatly increase the perfomance.

(I really hope PCI-ISA bridge,keyboard,mouse and rtclock features of the
cypress chip will work without any Linux support, and I can live wo
the USB bus for a while..)

Any guys inside digital working on this? If not, I might see if I can
get a data book from Cypress and give it a go myself. I think great
ide support is a good thing to have, because this is supposed to be
a lowprice solution, and going scsi will make a huge impact on the
price (and in a 'non server' enviroment, dma-2 is almost as good
as ultra scsi, imho)

Btw, any information about the speed the memory will be running at?
(66,83 or 100Mhz is figures I've been hearing).

Keep on hacking!

/Mikael
