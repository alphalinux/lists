>
> ln -s Alcor /etc/alpha_systype
>
> There don't need to be any file called alcor, the libc just reads the symbolic
> link and acts upon the name it points to.

Simple explanation: the Alpha LIBC needs to know which Alpha platform
it is running on, in order to correctly provide support for inb/outb
and mmapping of the framebuffer. The PWSa series (aka MIATA) is not
currently included in the table which LIBC uses to identify the Alpha
systems it knows about. Creating the above symbolic link will give LIBC
the information it needs to choose the proper support for user apps
which wish to access bus I/O and memory space.
