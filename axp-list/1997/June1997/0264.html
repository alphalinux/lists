On Fri, 6 Jun 1997, Aaron M. Gowatch wrote:

> I'm having some trouble installing RedHat 4.2 on a UDB.
>
> I'm trying to install over FTP, but the ethernet interface never seems to
> come up. When booting, the kernel sees the card and recognizes that I'm
> using the twisted pair interface. After I specify my IP, subnet mask,
> etc, I get link status on the switch its plugged into, but not on the box
> itself.
>
> The weird thing is that I can go to another machine, and ping the
> interface just fine. This works from any subnet.
>
> I've tried booting from SRM and ARC and I've updated to the
> latest rev. I dont think its a hardware problem because its had OSF/1 on
> it for the past few months which has worked fine. I've also had RedHat
> 4.1 installed in the past.

I answered my own question with a slightly odd answer...

For some reason, installing over FTP would fail if I used a machine name,
like ftp.foo.org as the host. However, if I used an IP address
instead, it worked fine. For some reason, the install was unable to
perform any name lookups, including the reverse lookup it tries to perform
to determine your hostname.

I'm pretty sure it wasnt a configuration error since I've setup countless
Linux boxen using an almost identical configuration on x86 machines.

Perhaps something didnt make it from one of the .img files onto my
floppies...

Aa.
