Alvin Starr wrote:
>
> I have a noname that when I run it configured with a memory size of 64
> Meg Most all is ok. When I enable the full 128 Meg all hell breaks
> loose.
> I have seen similar problems on a cabriolet that we have also.
> The kernel is built with a virgin 2.0.30 + axp-diffs. I am convineced
> that the problems are related to the network code in some way but I am
> unsure where to start looking.
>

I have the same (or at least similar) problems --- and a workaround,
which works o.k. for me so far. My system is a PC164 running RH 4.1
(e.g. kernel 2.0.27), which I upgraded to 128 MB a few weeks ago.

My workaround is:
Immediately after booting and starting X I try to start emacs.
If emacs does not start correctly --- going into 'uninterruptible
sleep' before showing a window on display --- I simply shut down
and boot another time.
If emacs starts correctly no subsequent errors occur so far.
