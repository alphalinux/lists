John Banghart wrote:
>
> We are stuck with an error condition in aboot. Is there any way to
> get out of aboot, or keep it from running so that we can get our
> SRM prompt back and correct the error?

Try hitting the reset button after aboot starts. When you bring the
system up, SRM first goes through its self tests. Then it checks
the "auto_action" environment variable; if it's set to "boot", it
immediately starts to boot. Once it's started to boot, hitting the
reset button should bring you back to the SRM prompt.

If you set "auto_action" to "halt", you get the SRM prompt right
after the self tests. The disadvantage is that you have to type
the boot command yourself. The advantage is that you don't get
stuck in situations like yours 8-)
