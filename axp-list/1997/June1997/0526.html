Juan Leon writes:
> My problem now is that minlabel does not let me set a partition type to
> 83 (Linux native) yet the install script expects the root partition to
> be a Linux native partition.

Which is very reasonable. minlabel uses this numbers:

FS type numbers:
0: unused 1: swap 2: Version 6
3: Version 7 4: System V 5: 4.1BSD
6: Eighth Edition 7: 4.2BSD 8: ext2
9: boot 10: resrvd10 11: resrvd11
12: resrvd12 13: resrvd13 14: resrvd14
15: resrvd15 16: ADVfs 17: LSMpubl
18: LSMpriv 19: LSMsimp 20: (null)

> How can I convince installation to accept a non-Linux-native root, or
> minlabel to set the root partition type to 83? Also, should teh swap
> partition be of type 1 (as implied by minlabel) or should it be
> 82 (as is in Linux/Intel)?

For swap partitions, use 1
For Linux native (ie ext2), use 8
