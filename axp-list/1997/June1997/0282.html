Ray <ray@gamma.ray.camelot.de> writes:
> it seems as the bootimage for jensen Systems (jensen.img on redhat 4.0) is
> again missing in 4.2 (it was already missing in 4.1). Can anybody tell me
> whether I can use the /kernel/jensen file to create a working bootdisk by
> gzip-ing it and then do the cat aboot.lx ... >/dev/fd0 thing?

No, you need to make an ext2 filesystem on that disk, an copy the
compressed kernel there, as vmlinux.gz, I believe. Take a look at the
RH-4.0 boot disk for details.

> Or should I
> better compile a 2.0.30 kernel myself using my working 4.1 system and make
> the bootdisk with that one? The image in the /kernel directory of the CD
> (I've got the black box with the book) is the only one that has not been
> g-ziped so I'm not sure whether it's 100% OK... The third alternative would
> be to use the 4.0 bootdisk, that is what I did for the 4.1, did someone try
> this?

No, though it will probably work. The bootdisk isn't needed for any
other purpose then keeping the installation process happy. You can
just copy that /kernel/jensen later to your hard disk, and adjust
aboot accordingly.

The kernel isn't gziped, because when RH-4.0 was released, some
systems weren't able to handle commpressed images, reason unkown.

But actually, the best way for you would be to boot from cd. You *need*
a CD-ROM drive which is jumpered to a 512 bytes block size, then you
should be able to boot with

>>> b dk<x>00

where <x> is the scsi id of you cd drive. Use 'show dev' to get that
displayed.

You'll get an aboot prompt (or at least you'll get it, if the cd isn't
blown in any way ;-), where you continue with

aboot> kernel/jensen root=/dev/scd0

I hope I recalled that stuff correctly. In case it doesn't, take a
look at the SRM HOWTO, which is accessible from the FAQ at azstarnet.

> Does anybody know /why/ the jensen images are missing since 4.1?

No. But maybe the installation process no longer instist on using the
boot disk? (' doubt it ...)

I'am also wondering why the README says, that there's information
about booting from the SRM console at www.redhat.com, while I wasn't
able to find it ever since RH-4.0 was released ...
