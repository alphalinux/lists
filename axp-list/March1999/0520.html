I'm getting a CIA machine check, vector 0x660, reason = 0x21E = corrected
ECC error (this is with pc164 MILO, which uses the eb164 codes, yes?).

The trouble is I get an infinite number of them in a row, so apparently,
the error is not getting cleared before the handler returns.

The symptom is the system locks up hard, but if you happen to be in the
console window (i.e. not running X), you can see the machine checks going
by (but they never get written to the log and the keyboard is dead).

What is the proper way to clear the error? Do I reset the bit and
write it back to the CIA_IOC_CIA_ERR register (or clear all the bits)?
Just doing
        wrmces(rdmces()); /* reset machine check pending flag */
in core_cia.c doesn't appear to cut it.

Perhaps interrupts are not getting turned back on correctly?

(I noticed a FIX ME in the palcode where it suggests distinguishing
between corrected errors and uncorrected ones. Perhaps that's the place
to ignore this, but going back to the kernel so a printk() can be done
to log that the error occurred is good.)

I'm assuming of course that this machine check is safe to ignore?
Oddly, it's semi-reproducible -- it happens while writing to SCSI disk
through an adaptec controller, so perhaps I have a problem there, although
I swapped the adaptec for an identical one and the machine check is still
reproducible with dd if=/dev/zero of=/tmp/moo count=large over and over.
(I only have one internal disk.)
