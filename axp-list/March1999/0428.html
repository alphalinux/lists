> It seems -O5 option does
> something wrong regarding subnormality issue as
> a program generates FPE printing variable which must
> be exactly 0.0.

It's worth poiting out that the g77 I/O libraries can't print out the
value of a denorm, they get a FPE. They would probably work (and not
get much slower) if compiled with -mieee.

As for your problem, remember that g77 + libffm or libcpml will never
generate a denorm. So if you see one, there's a serious bug. If you
are using libm, on the other hand, denorms can creep in from the math
library. The best fix for you is probably to use libcpml, since you
don't seem to want those denorms. Obviously there are a lot of
particular issues with your code...
