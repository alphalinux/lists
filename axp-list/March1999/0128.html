Some time ago I posted a question regarding the DMA buffer alignment
and the tulip chipset on the alpha platform related to:

> src/linux/net/de4x5.c (from 2.2.1)
> the following comment:
> I have removed the buffer copies needed for receive on Intels. I cannot
> remove them for Alphas since the Tulip hardware only does longword
> aligned DMA transfers and the Alphas get alignment traps with non
> longword aligned data copies (which makes them really slow). No comment.
> ...

> Is this still an issue with tulip-like cards on the Alpha?

This "problem", if you can it that, still exists so far as I can tell.

However, it exists only on the receive side and whether it is really
a performance hit depends on the kind of data you are transmitting.

The gigabit yellowfin driver implements some tradeoffs between various
size incoming packets and "copying" the packet data.

As it happens, in the application for which I am considering the use
of linux/alpha I would not be impacted by the extra copy.
