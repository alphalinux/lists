>>> Hyung Min SEO said:
>
> I found this code in file "linux/arch/alpha/kernel/sys_dp264.c" line 64 on
> the kernel 2.2.1.
>
> static void dp264_device_interrupt(.....)
> {
> #if 1
> printk("dp264_device_interrupt: NOT IMPLEMENTED YET!! \n");
> #else
> .......
> #endif
> }
>
> I'd like to know that the code is modified on after version of kernel 2.2.1.
> If not when can I expect to modify the code ?

That routine is for handling interrupts delivered by the EBSDK PALcode for
the DP264; ie when booted via a MILO, which doesn't (yet) exist for that box.

The code will necessarily have to examine the interrupt summary register(s)
to determine which device is interrupting; it is this code that is missing.

Look at some of the other sys_xxx.c files for examples of how this is done
on other boxes (EV4 or EV5 based).

In addition, the code should look a lot like the interrupt mask register(s)
management code, only with the interrupt summary register(s) as the targets.

Good luck.
