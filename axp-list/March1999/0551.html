Hi,

I have a very specific problem when linking a executeable on Alpha/Linux
I try to describe it as easy as possible :-)

My environment (all packages from ftp.alphalinux.org)
RedHat 5.2
binutils-2.9.1.0.19a-2
egcs-1.1.1

I am working on a large C++ software project.
In our code we are overwriting the C++ "operator new"
This operator is in a object file which goes into a library
libFoundation.a.

With this library an executeable prnew is produced

When applying "nm libFoundation.a" the result is
0000000000000000 T __builtin_new

Which is fine.

But after linking the executeable a "nm prnew" gives
0000000120290e94 ? __builtin_new

Which means that our operator new will be never called
(the ? means it is an unknown symbol) :-(

The linker somehow is "loosing" the symbol.

Wenn linking the object file containing the operator new
directly to the executeable it works.
"nm prnew" gives

000000012000af00 T __builtin_new

The same works on an i386/Linux with a nearly equal environment
egcs-1.1.1
binutils 2.9.0.1-15

I have already tried tu use the original 5.2 environment
egcs-1.0.3
binutils-2.9.0.1-15

Without any difference

Any Ideas ?
