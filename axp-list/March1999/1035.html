I have attempted to build a linux 2.2.3-ac2 kernel for a Ruffian (UX)
system using the linux 2.2.3 sources and the following patch:

ftp://ftp.linux.org.uk/pub/linux/alan/2.2/patch-2.2.3-ac2.bz2

It all builds successfully (with egcs-1.1.1) but fails during boot with
the following:

[earlier messages scrolled off top of screen ...
ncr53c875-0: rev=0x03, base=0x9100000, ioport=0x9000, irq=20
ncr53c875-0: ID 7, Fast-20, Parity Checking
ncr53c875-0: on-chip RAM at 0x9101000
CACHE TEST FAILED: script execution failed.
start=80753648, pc=80753648, end=80753674
CACHE INCORRECTLY CONFIGURED.
ncr53c875-0: detatching...

As the root filesystem is on the single disk attached to this controller
this is pretty much a show stopper!

I have also tried applying the following patch (before patch-2.2.3-ac2)
and configuring for the sym53c8xx driver:

ftp://ftp.tux.org/pub/roudier/misc-stuff/ncr-8xx-896-bundle-for-linux-2.2.3.patch.gz

As before, all builds successfully but fails during boot in a similar
manner with the following:

[earlier messages scrolled off top of screen ...]
sym53c875-0: rev=0x03, base=0x9100000, ioport=0x9000, irq=20
sym53c875-0: ID 7, Fast-20, Parity Checking
sym53c875-0: on-chip RAM at 0x9101000
CACHE TEST FAILED: script execution failed.
start=80751cf8, pc=80751cf8, end=80751d18
CACHE INCORRECTLY CONFIGURED.
sym53c875-0: giving up ...

MILO has no problem booting using the same controller and disk
combination, and earlier kernels (eg. 2.2.1) ran without any obvious
problems.

Any suggestions?
