> Hi,
>
> It seems I'm getting close to get my stuff working on my SX.
> But in fact this task requires much more RAM which I need
> and I can afford for my own machine. ...
...
> ... I'm writing this message from a PII350,
> which isn't too bad but we have hard time trying to get its RAM
> at least above 512MB (prices for 256MB modules I've
> got from European suppliers are about $1K each).
>
> So, if anyone has any info about systems with 4-8 GB RAM
> I'd be very happy to hear about it.

        We have some subtle problems at exceeding about 1G RAM in
        Alpha systems. The subtleness issues come from the IO-
        subsystem, and how PCI bus-mastering access reaches the
        main memory thru the PCI/CPU interface.

        These issues have been mentioned before (like up to 2 years ago),
        but as far as I understand, nobody has made necessary tweaks to
        PCI bus device drivers to do sort of 'bounce buffers' scheme we
        already use for the ISA-bus devices. (GFP_DMA, and the new
        GFP_DMA_PCI32 ?)

        Yes, large memory systems are difficult animals, and rare...
        (Thus we don't yet have the necessary support for them.)

        I would bet the necessary addition of these GFP_DMA_PCI32
        flags would be quite wide-spread ('GFP_DMA' is for ISA limits.)

        Most of the necessary ground work has already been done for
        some SPARC64 devices, so it shouldn't as difficult as when
        ploughing into a new field...

> Thank you very much for your help,
> Rgds,
