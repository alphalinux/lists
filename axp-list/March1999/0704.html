Hello,

First the system.

System: Multia 166Mhz with 96Mb of memory 256Kb cache
        Elsa Quickstep 1000Pro PCI Passive ISDN adaptor
        RedHat 5.1 with some updates from the redhat ftp site
        

Then the problem/bug:

Downloaded last night the Vanilla Linux 2.2.3 kernel and and the latest
ISDN subsystem from oldhades.de and compiled.
I decided to be adventurus and selected the sound option. The builtin
sound card is MSS at IO 530, IRQ 9 DMA 3. I also select the new
persisten DMA audio buffer.

The compilation went just fine. And everything seem to work. Both
ISDN and sound card. After a while when listening at mp3 with mpg123 version
0.59p (its been a while since I downloaded that one) the system locks up.
It just seem to hang, not accepting any input and the screenimage is fixed.
After powecycle and some fsck-ing of the disk the system comes up again.
I repeat the experiment. The song is played for a while and then the system
locks up. Powersycle. Analysis of /var/log/messages shows the a lot of the
errors happening at the PCI bus (at the end of this message). The last
message is the /var/log/messages is the same kind of error.

Hypothesis:

I believe this is something connected to the audio driver. I have been using
the system a long time now (couple of weeks) with 2.2.0.final and the
ISDN drivers from oldhades but no sound. I also seem to be able to use the
system without crashes, just if I don't use the sound system.

Question:

Is this a known bug? or maybe broken hardware?

I can supply the full error message if that helps as well as the detailed
configuration of the kernel.
