This IS related to processes on different machines renaming and/or
deleting and then recreating the file control/rcpthosts. We regenerate
this file on a periodic basis. I tried using 'noac', but the problem
still persists. Regardless, I would assume that if the fh becomes stale
that the kernel should try to re-aquire it. Instead, it seems to hold
onto it.
