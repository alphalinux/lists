>
> This is not how the current SCSI system does it
> (linux/drivers/scsi/sd.c).

Sorry for the misinformation, I didn't look at the details... :-\

> For writes from an un-DMAable buffer, it
> allocates in low memory, copies, and then writes. If I'm reading the
> code correctly, for reads, the code simply allocates a buffer in
> DMAable memory. No copy, no use of high memory. This has the bad
> side-effect of limiting your disk buffers to that area in memory, and
> increases pressure on that memory.

Indeed; not as bad pressure as on the low 16Mb on an ISA machine, but
pressure none the less.

> But it doesn't waste any time
> copying. It would be fairly simple to extend this code to use a bounce
> buffer if the system was running low on DMAable memory.

But it still doesn't handle direct read to a user-defined buffer in
non-DMAable space, does it?
