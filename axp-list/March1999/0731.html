On Sun, 14 Mar 1999, Hirsim{ki Mika wrote:

> Is anyone running the 2.2.3 kernel on a 164LX?
>
> If I compile the kernel specifically for 164LX it crashes
> right at the start of the boot process (no messages at
> all). If I compile it using the Generic-setting, the
> system boots up normally, but the kernel binary is huge
> and dmesg shows:
>
> Linux version 2.2.3 (root@sauron.orthanc.fi) (gcc version 2.7.2.3) #2 Sun Mar 14 18:09:12 EET 1999
> Booting on EB164 using machine vector EB164
> Command line: bootdevice=fd0 bootfile=vmlinux.gz root=/dev/sda2
> Console: colour VGA+ 80x25
> ...
>
> Why does the kernel think it is running on an EB164, when
> it is actually runing on a 164LX?

Check your .config. It must be configured wrong. Notably:
        CONFIG_ALPHA_LX164=y
and
        # CONFIG_ALPHA_EB164 is not set
should both be in the file.
