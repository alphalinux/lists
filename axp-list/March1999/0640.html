>>> philip.chandy@infortech.com said:
>
> I've installed Alpha version of Linux 5.2 on Alphastation 200 4/233
> (Avanti). I had the character mode working great. Then I went ahead and
> configured the system to boot in X. The system will boot okay and come to a
> login prompt. However if I login as root, the system is not stable, the
> system will lock up while I am doing some tasks such as changing mouse
> configuration or xedit-ing a file, etc. However this problem does not exist
> if I login as one of the user names that I created. When this lockup
> happens, it locks up solid, I cannot break out of it. All daemons(such as
> wwwd) also seem to freeze. The only way to bail out to is to power down.
> The problem is getting worse and worse after each boot. Now if I login as
> root at the xWindow login prompt, the GUI will start up half way and lock
> up.

If your box has an S3 Trio64 based card in it, add the following to the
Device Section of /etc/X11/XF86Config:

        Option "no_pci_disconnect"
        Option "sw_cursor"
