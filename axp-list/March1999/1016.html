OK. I have Linux mostly installed on the XP1000
that arrived today. I'm running into a couple problems.

1. The main board has a 21142 built in that tulip
   seems unable to run properly. In particular, tulip
   wants to believe that my 100Mb hub is 10Mb. Also,
   since this is a kernel from gatekeeper, it appears
   tulip is built in, not a module since lsmod doesn't
   show anything. So, I'm off downloading the latest
   2.2.3 kernel to see if I can get this puppy to behave.
   Has anyone gotten 21142's to behave and was it tulip
   or de4x5? Also what version of the driver and any parms
   to set. If I can't find a working version, I will see
   about disabling the onboard 21142 and installing a 21140
   based adapter.

2. I seem to have either heat problems or the memory dimms
   are not up to standard. I can't get the latest egcs1.1.2
   to go thru a full build. It builds fine on my 164LX. I
   get either core dumps at what appear to be random places
   or I get a message something like "Unable to dup fd xxxxxxxx
   to fd 0" while trying to copy the files to the installed
   location. The first location looks obviously bogus.
   Anyone seen that before??

3. I added a Symbios 875J based UW SCSI to the system and
   it appears SRM does not see it or any devices attached to
   it. Is there something I can set to get that recognized by SRM?
   I'm currently booting off the floppy until I get that fixed.
   I'd like to boot from that card. Note AlphaBios does see the
   adapter. It is on Bus 0, so I guess it's not sitting across
   a PCI bridge or anything odd like that.

For your amusement:

996.15 BogoMIPS

Any info appreciated...
