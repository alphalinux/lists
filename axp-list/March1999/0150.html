I'm trying to make some emergency boot disks, but I fail to get the system
running from them.

The kernel boots OK (from MILO):
boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1

the kernel gets loaded and prompts for a ramdisk. I enter it and then receive
the following:

[...]
Freeing unused kernel memory (72k) //-> this still is the kernel
: Can't open

Afterwards I have to power-cycle (Sigh...)

I found out that /bin/ash, which is executed instead of /sbin/init prints the
"Can't open" message
('strings /bin/ash | grep open' shows it)

What am I missing?

Konrad
