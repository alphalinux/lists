It is a pain in the butt to do a 'install everything' in debian - it
takes an hour or so to select everything that looks interesting then also
resolve package conflicts. Many packages have conflicting dependencies,
like emacs and emacs with X (just an example, this particular one may have
been improved), in which it must be tediously resolved, almost
recursively. The people that I know that love debian curse the install
process. I think debian follows GNU's (implicity?) suggested file
structure better, so if you tar.gz recompile and install many packages, it
will install in the 'correct' places versus having to edit makefiles.

It is far easier to install base plus a couple packages, then later
install the other stuff that you need.

I have no better experience on with debian; I simply stuck with an easier
install - RedHat - for better or worse.
