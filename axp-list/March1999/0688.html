No. I would rather just allocate memory that has the properties that
the device needs, and these silly (and many not-so-silly) cards want
memory that can be address with only 16 bits, or 32 bits, ... etc.

can avoid it, and frankly, I think an allocator can be fixed up to
handle the problem at hand.
