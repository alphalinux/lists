On Wed, 10 Mar 1999, Matt Nelson wrote:

> well, the thing is asking for something like 750M of memory on a 1G
> system (physical+swap). it is the only active user job when it core
> dumps.
>
> could the memory get too fragmented to allow for declaration of such a
> large array? (i don't know the mm internals, so i'm clueless on this
> point).
>
> -matt

I don't think that that is the problem. When you ask for 750 MB of memory,
you are asking for 750MB of contiguous VIRTUAL memory, not physical memory.
The virtual memory can have pages from any position in physical memory. They
need not be contiguous. It might be some kind of stack limitation.

However, the fact that it works sometimes and not others is very suspicious.
