On Thu, Mar 11, 1999 at 08:14:23AM +0100, Danilo Godec wrote:
> Well, I tried to compile my own mozilla. It compiles quite fine (as far as
> I can tell) with gtk toolkit, but when I try to start it, I get this:
>
> Going to create the event queue
> Floating point exception
>
> What does that mean?

The Alpha FPU in its native mode is not fully IEEE754 compliant. When
denormals, NaNs ... are produced and used as input to another FP operation,
you'll see a FPE, where the IEEE754 would require no exception to occur.
This can be be avoided by compiling with the -mieee flag (gcc). It causes
20% for some FP operations, so it's not the default. Most programs can be
fixed to avoid denormals.

Now for netscape, lesstif, GTK+, I would just use -mieee, as you don't have
too much FP involved in these programs, anyway, so your netscape might be 1%
slower.
