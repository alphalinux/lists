On Fri, 5 Mar 1999, Greg Lindahl wrote:

> When you use assembly with gcc, it runs it through the optimizer. In
> some cases this makes the code better; if the code is great to start
> with, it often makes it worse. The complaint about egcs is that you
> can't turn off the optimizer.

Have you tried an egcs without the Haifa optimiser enabled? This is most
likely the optimisation that is causing you the degraded performance. For
C, F77, and C++, it tends to do well (although I have seen exceptions).

Try recompiling egcs with the --disable-haifa switch passed to configure
and try recomparing. Unfortunately, too little progress has been made on
gcc 2.8.x for Alphas of late for me to switch.

> For C or Fortran code on the Alpha, egcs probably always produces
> faster code.

I would say "almost always". There have been exceptions in the past,
although I haven't run tests in awhile.
