On 14 Mar 1999, Martin Ostermann wrote:

> On Sun, 14 Mar 1999 01:15:00 -0800 (PST), "B. James Phillippe" <bryan@terran.org> said:
> > IO functions are defined in <asm/io.h>.
>
> Well, those define the kernel io functions. They will only work on the
> architecture on which you compile your stuff, if they compile at all.

The kernel IO functions are the only IO functions, unless you write your
own architecture-dependent version for every CPU you plan to support in
your code. There are no IO definitions in the standard library, and it
doesn't make sense for there to be any; IO is inherently
architecture-dependent. And what do you mean "if they compile at all"?

> Rather use the glibc include file, maybe <sys/io.h>? Try a
> 'find /usr/include -name io.h -print' to find them, Or a
> 'grep outb /usr/include/{*,*/*}'

All <sys/io.h> does is #include <asm/io.h>; there is no such thing as a
non architecture-dependant IO definition. If you only care about
compiling on systems after glibc, <sys/io.h> is the header to use. But
there is no <sys/io.h> for libc5, so YMMV.
