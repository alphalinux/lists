lseek from glibc-2.0.7 is broken on Alpha. It is documented to
have the following prototype:

   off_t lseek(int fildes, off_t offset, int whence);

Here is a fragment of a trace from a new version of 'lsmod':
....
open("/dev/kmem", O_RDONLY) = 4
lseek(4, 18446739675790475552, SEEK_SET) = 127435040
....
where the program attempts to seek to a location with a module
information in a memory. 18446739675790475552 happens to be
in hex 0xfffffc0007988120 and the return value is 0x7988120
so it is returned as 'int' and not 'off_t' - which, at least on
Alpha, are of different sizes.

What is more lseek() does not seem to be searching to a required
location as an attempt to read from there results in obviously
bogus values (all zeros).
