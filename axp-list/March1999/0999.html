Hello all!
Quiet some time ago I reported that the MPEG-I player from MpegTV (MTV)
keeps crashing on my 164SX 533MHz machine using various versions of kernels
and libc. Up until now I could not reproduce the effect with any other
application so I thought, OK, live with it. Some time later other users
reported exactly the same oddities with 164SX boards and MTV. On the
baseline it seems that some kind of memory corruption occurs when mtv (or
mtvp) and some other load sucking application were running. My test was and
is to run mtv and compile a kernel. This results in crashing of mtv
(seg-fault or illegal instruction) or compiler (typically sig 11, sometimes
illegal instruction).
This description allone does not tell much about the problem and since I
could not reproduce the effect I kept quiet until now!

Today I tried x11amp on my 164SX and see ... it causes exactly the same
effect! Either x11amp itself crashes at random times or other processes
suddenly vanish! And tell you what: kernel compile crashes again!
So where is the connection between mtv and x11amp?

Both use X11.
Both use the audio device.
Both use threads <- my favourite cause for faults!

I then tried x11amp on a remote display and the effect is still there so the
XFree server cannot be the reason. Still in the game: audio device and
threads. Since I never had any similar problem when using any kind of audio
application I think we can eliminate that too.
So the last guess are the threads.
My current setup (the IMHO relevant parts):
  kernel 2.1.131
  glibc-2.0.7-29 from RedHat 5.2

So here we are again ... any ideas?
Is anyone else able to report problems with threads on 164SX boards?
Or maybe this is a CPU problem and all 21164PC users could be affected?
Are there any other programs out there that toggle this effect?
And last but not least: Is there a fix? (and don't tell not to use x11amp,
mtv or such multimedia applications ...)
Has anything relevant to threads changed from kernel 2.1.131 to 2.2.3?

OK. Thanks for your attention ;)
