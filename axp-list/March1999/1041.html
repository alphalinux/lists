I've tried running the dhcpcd-1.3.16-0.alpha.rpm that was part
of the updates/kernel-2.2/alpha directory on updates.redhat.com.

It doesn't work. All I get is a set of unaligned traps and a
timeout message.

I know that dhcpcd version 0.70 and below don't work on a 2.2
kernel; what I want to know is (a) whether anybody else has
a working version of dhcpcd; or (b) if anybody knows what's broken,
so that I can fix it. I did fix one bug in 1.3.16 already;
it was claiming that the checksums were invalid, and I was able
to fix that (it was doing it 16 bits at a time, and changing it to
a byte at a time fixed THAT problem, but didn't fix the response
problem in general).
