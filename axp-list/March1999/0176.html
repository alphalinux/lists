>Hi, I've read that you can flash MILO to your multia so that it
>pops up every time you boot (w/o a floppy). The docs I read
>suggested this >may be bad because it limited future
>hardware upgrades. My multia has probably
>seen it's last hardware upgrade (short adding some
>ram). Do any of you have experience with this. If so is there
>a better (for novices) howto on this than the MILO Howto?

When you set up the root drive, make three partitions. A DOS
FAT-16 of about 10 meg, a LINUX SWAP (type 82) of at least
1 1/2 times your RAM amount, and the rest for your root.

Once you get the OS loaded, run "dd if=/dev/fd0 of=/dev/sda3"
(assuming that your DOS partition is there - run cfdisk to be
certain where it is. Do not send it to your root or SWAP
partitions!). Reboot the system.

Once ARC is up, select:

Supplementary Menu
Set up the system
Manage boot selection menu
Add a boot selection

It will ask you some questions:

Assuming that your LINUX partition is sda1, your SWAP partition is sda2, and
your DOS FAT-16 partition is sda3 -

For the first question answer:

SCSI BUS 0 HARD DISK 0 PARTITION 1

Then, in order (notes in parenthesis):

enter the osloader directory and name: \linload.exe
is the operating system on the same partition as the osloader: no
enter the location of the osloader: SCSI Hard Disk (default)
enter the SCSI bus number: 0 (default)
enter partition: 1 (default)
enter the operating system root directory: /milo (trust me)
enter a name for this boot selection: Linux
do you want to initialize the debugger at boot time: no

After it's done, you want to edit boot selection, select Linux
Arrow up and down to chose, enter to select

You need to edit the SYSTEMPARTITION to point to the partition you dd'd MILO
to.

Change:
SYSTEMPARTITION=scsi(0)disk(0)rdisk(0)partition(1)
to
SYSTEMPARTITION=scsi(0)disk(0)rdisk(0)partition(3)

And change
OSLOADOPTIONS=debug
to
OSLOADOPTIONS=boot sda1:vmlinux.gz root=/dev/sda1
so it knows what to boot, and where to boot it from

That's about it (ie - that's the short version!)
