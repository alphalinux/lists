I have the same AS 200 4/233. Couldn't get it to work with 16MB, end up
upgrading to 128M, the parity memory was cheap, could only find the ones
with gold contacts (vs. tin). I got it from a mail order shop
(Micromatics - (408) 970-8566 Ask for Fred). Don't ask for Alpha Memory,
just ask for cheap parity memory.

Also I want to note another issue that I ran into after the install. I've
installed Redhat Alpha version 5.2. I had the character mode working great.
Then I went ahead and configured the system to boot in X. The system will
boot okay and come to a login prompt. However if I login as root, the
system is not stable, the system will lock up while I am doing some tasks
such as changing mouse configuration or xedit-ing a file, etc. However this
problem does not exist if I login as one of the user names that I created.
When this lockup happens, it locks up solid, I cannot break out of it. All
daemons(such as wwwd) also seem to freeze. The only way to bail out to is
to power down. This causes the system to force a disk check when it boots
again. The problem is getting worse and worse after each boot. Now if I
login as root at the xWindow login prompt, the GUI will start up half way
and lock up.

I have a ticket open with Redhat, haven't heard anything, I'll post it if
get an answer that fixes the problem.
