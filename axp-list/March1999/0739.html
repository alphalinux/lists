On Sun, 14 Mar 1999, Greg Lindahl wrote:

> I talked to Jakub Jelinek about the sparc64 support and it seems that
> the following hack will get AlphaLinux working with big memory and
> bounce buffers today:
>
> 1) Increase CIA's memory window to as large as possible
> 2) set unchecked_isa_dma = 1 in your SCSI drivers
> 3) increase ISA_DMA_THRESHOLD to the size of the CIA memory region.
>
> Recompile and enjoy - 2&3 mean that existing SCSI drivers will start
> using bounce buffers. This breaks ISA card support (no big deal) and I
> haven't looked at the appropriate ide driver to see if it uses bounce
> buffers. This would also break my favorite network driver (myrinet).

Is this going to work for drivers which never expected to have ISA
versions or need bounce buffers? Or for example ones which push lumps of
machine code to the SCSI controller for them to execute?

I dont think it will ?
