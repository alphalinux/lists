On Sun, Mar 14, 1999 at 06:28:26PM -0500, Rich Payne wrote:
> No, No, No.... this looks like the older TurboChanel machines. This will
> NOT run Linux, and it (or ones just like it) have been on Ebay for months.

True. The TurboChannel is not supported, although some work was spent on it,
IIRC. And even if it was, you have strange graphics adapters, and you will
have trouble to find an X-Serve to run on it.

> If you get the machine you'll need to run NetBSD?? (One of the BSDs will
> run on this, not sure which one).
>
> X will run fine on any supported Alpha (not the 3000 series of course).

Your VGA/MGA/TGA/whatever has to be supported by XFree86 or Accel-X.

> StarOffice is not on Alpha, though ApplixWare is, WP8 is Intel only, though I have
> read reports from a couple of guys within Compaq that they have it running
> under em86, I've also heard that Corel might port it to Alpha.

... and while Linux-Alpha runs Digital Unix binaries, you still have trouble
to get them working, cause you need a license for the shared libraries. Note
that you need a 2.2 kernel for DUX-4.x binaries.

> As for speed of the Multia (166MHz) it think a P90 is about right. I have
> a P100 and it keeps up with that most of the time. Kernel compiles take
> longer, but I don't see that as a problem. The Multia is a good box to
> learn/play with, and the best part is they are cheap :)

My feeling is that the alphas are as fast as their intel counterparts when
compared with the same MHz (This was a comparison of a axp21164-266 against
a PII-266). For floating point, the alpha is much stronger. On the other
hand, I don't know about the old 21064 chips.

Compiling is always slower on the alpha. It's a RISC and the compiler has
much more work to do to get reasonable code produced. (More instructions to
be generated, less dense code, more issues wrt instruction scheduling, ...)
Also your binaries are larger by a factor of 1.5, so your memory and disk
subsystem have to perform a little bit better to keep up with intel ...
