Well,
        If it's broken, let's fix it. How does the complier tell the library
that the code was compiled with -mieee? What isn't it doing the right
thing, and what exactly IS the right thing?

In an ideal world:

1) program compiled with -mieee
Slow, but can handle denorms correctly.

2) program compiled without -mieee.
Fast, but rounds to zero on small numbers.

To get rid of the FPEs associated with denorms, we can either
1) Never Generate them
2) Once generated, handle them with kid gloves.

libffm, and libcplm does the first.

It seems that libm does the second, assuming gcc/egcs will generate codes to
handle the denorms. If this is really the way to go, it looks like a compiler
bug.

Comments?
