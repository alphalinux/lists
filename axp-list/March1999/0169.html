> I decided to upgrade this machine to the new 2.2.2 kernel and
> installed all the nessisary suport software. On the surface this
> server seems to run fine, but after a short time I start to get the
> following errors:
>
> nfs_revalidate_inode: control/rcpthosts getattr failed, ino=529984,
> error=-70
> NFS: bad fh
> 000983c601cb11c9000000000008164001cb290d00000000000983c601cb11c9

I see 2 likely options:

1) 2.2 has a botch
2) the netapp has a botch, and 2.2 sees it because it does
   attribute caching differently.

In either case, the error "-70" is coming from the netapp. You should
be able to look it up in the netapp documentation and see what it means.

Is it always the same inode?
