> before I buy a new Adaptec 2940 U2W card, I would like to be sure it is
> cool with MILO and all the other drivers. Anyone have good luck with this
> adapter y/no?

    I've been asking questions about U2W for some time now,
    and I finally bought it yesterday. Getting it to boot
    Linux was a bit tricky, but it can be done.

    Problems:

      -AlphaBIOS 5.66 doesn't recognize U2W.
      -SRM doesn't recognize U2W.
      -RedHat 5.0 MILO (2.0.30) doesn't recognize U2W.

    This means that in order to boot using AlphaBIOS or
    SRM, you'll either have to have another SCSI-controller
    that is recognized or an IDE bootdisk. These are your
    only options, unless you're willing to mess with your
    flash rom or to wait for an AlphaBIOS upgrade.

    I didn't want to buy an IDE drive just for the sake of
    getting my alpha to boot nor did I have an empty expansion
    slot for another SCSI-card. I solved the problem like
    this:

    -Burned a MILO (2.0.30 unfortunately) into the flash rom.
    -Booted a 2.0.35 kernel from a floppy disk with an option
     "root=/dev/sda2" (replace sda2 with your boot partition).

    This seems to be the only way to boot until a 2.0.35+
    based fmu.gz (to flash the MILO) becomes available.

    Note that even if you have the latest RedHat or Debian
    (with a MILO that can talk to the U2W) you can't install
    it using AlphaBIOS, because the hardware is not properly
    detected at power up.

    In spite of these problems it seems to be rather a
    nifty card. You can connect both LVD and FAST-scsi
    drives to it without losing its Ultra2 capability
    and Bonnie did show some very nice data transfer
    rates, too.
