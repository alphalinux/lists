hi,

> The patch is now distributed in the beta ide-udma driver for 2.2.x
> (http://www.dyer.vanderbilt.edu/server/udma/) and it gives me ~6MB/s vs.
> ~1.5MB/s (hdparm -t) on my 164sx. Integer performance on a pc164sx/533
> should be about the same as a pII/350 (floating point much higher on
> anything but linux).

i get ~8MB/s with my patch and an IBM DTTA (hdparm -t) and ~9MB/s using
time dd if reading through the file system (ext2).
by the way does anybody know why reading through the file system is faster
than reading /dev/hda or hda1 direct? (i tried several diferent block sizes
and got almost the same results every time).

one of the things i use my sx for is mpeg video encoding (only integer no
floating point). i use the stanford mpeg encoder with a modified motion
estimation routine for alpha and intel mmx.
my pc164SX/533 is about 2 to 3 times faster than my PII/333.

the alpha version uses my own MVI assembler (if you have a pc alpha use it :-) )
optimized motion estimation and is build with the digital unix compiler.
the intel version using intel's mmx code for motion estimation and build with
Visual C++ 5.0 and running under NT(i don't have linux on my intel box!).
 
since it takes hours to transcode large mpeg the alpha realy makes a
difference !!!!
