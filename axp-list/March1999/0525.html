On Tue, 9 Mar 1999, Joakim Roubert wrote:

Any thing within the /proc file system is not real -- it is a reflection
of the various process's running.

        "/proc -- this file system type provides access to the address
         space image of each active process in the system. It is typically
         used by debugging and tracing utilities"

 -- "The Magic Garden" Benry Goodhart & James Cox

Although the above quote is from a book about SVR5 the proc file system is
modeled on this. And whatsmore is a particulaly good implementation.
Compare it to OSF.

The proc filesystem is a means of interrogate the kernel about any process
that is running on the system. Try it. Look at /proc/pci or /proc/meminfo
or cd to /proc/[0-9]{3,4} and cat any file in there.

The proc filesystem is a very good thing.
