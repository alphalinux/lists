On Fre, 05 Mär 1999 Greg Lindahl wrote:
>> >Note that he is talking about inline assembly, not actual C code.
>>
>> So why is there a difference in the code generated from the assembly between
>> gcc-2.8.1 and egcs-1.1.x ?
>
>When you use assembly with gcc, it runs it through the optimizer. In
>some cases this makes the code better; if the code is great to start
>with, it often makes it worse. The complaint about egcs is that you
>can't turn off the optimizer.
>
>For C or Fortran code on the Alpha, egcs probably always produces
>faster code.

Well, probably I should ask this the egcs people, but still:
What does gcc -O0 do ? Still going through the optimizer ?

Moreover as I understood the missing feature to turn off optimization was meant
for DU cc ?

Btw: I am happy with egcs (+libffm, testing libcpml right now) for my mixed
C++/FORTRAN project, there are several pros so I wouldn't even think about
switching to gcc-2.8 or any other suite, still I am wondering why gcc-2.8
should be so much faster (590 vs 520) ?
