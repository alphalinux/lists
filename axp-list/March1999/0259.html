> >Note that he is talking about inline assembly, not actual C code.
>
> So why is there a difference in the code generated from the assembly between
> gcc-2.8.1 and egcs-1.1.x ?

When you use assembly with gcc, it runs it through the optimizer. In
some cases this makes the code better; if the code is great to start
with, it often makes it worse. The complaint about egcs is that you
can't turn off the optimizer.

For C or Fortran code on the Alpha, egcs probably always produces
faster code.
