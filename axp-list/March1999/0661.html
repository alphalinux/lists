On Thu, Mar 11, 1999 at 01:08:38AM +0100, Thomas Dargel wrote:
> Hi,
>
> till monday I used several programs compiled on DU 3.2d and running
> very good on my PC164 RH4.2 kernel 2.0.30.
>
> For several reasons, especially due to the system upgrade on the DU
> box to 4.0D, I had to upgrade to 2.1.1xx or 2.2.x for running the
> DU4.0 compiled programs on my machine, that's right, isn't it?

Yes, correct, you need 2.1/2.2 to run DUX4 binarties.
I think runnning a 2.1/2.2 kernel on a RH4.2 based system is a very bad
idea. A lot of system tools have to be upgraded to properly work with 2.2.
Check /usr/src/linux/Documentation/Changes.

> I transferred the DU40 libs to my box, put the /sbin/loader to the
> right place -- but the binaries don't run.

I have succeeded in running most of the DUX4 binaries on a linux-alpha
machine.
Unfortunately, neither netscape nor the CXX compiler do work.
