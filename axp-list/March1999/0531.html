> > > - declare some dummy arrays: real dummy1(60000000) (three of em)
> > > - save an I*8 pointer to the array: pointer=%loc(dummy1(1))
> > > - pass the pointer to subroutines that do the meat of the work via
> > > %val(pointer)

You should print a debugging statement right after you declare those
arrays (i.e. at program start, if they are in the main program); that
will tell you if you ran out of memory and immediately dumped core.

Then you should print the I*8 value of pointer. You can see if it is
misaligned or not.

There may also be a weirdness in the way %val() interacts with
subroutine arguments on Alpha g77. Hm.

> could the memory get too fragmented to allow for declaration of such a
> large array?

I don't think Linux has this "feature".
