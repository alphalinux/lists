  Hi Ron,

    I have also been experiencing strange lock ups on a Multia. The lockups
only occur when there is a large amount of network traffic. I use my multia
to IP Masquerade 2 linux boxes and a Windows box over a cable modem.
I can surf the web for
hours through the multia as long I'm just accessing small text/graphics
pages, but if I try to download a large stream of data from a fast site,
my multia hangs after a few megabytes. Another program which causes it to
hang is Netperf. Netperf will send a continuous stream of around 10Mbits and
the multia will hang after a few seconds of running the Netperf benchmarks.
Netperf consistently hangs the machine!

 I have only a shell open on the multia, and a few daemons. X is not running.
When it hangs, I can type a single character on the console, after that it
won't echo any other output, but the LEDs on the keyboard flash after
several keystrokes. There are no messages in /var/log/messages that report
any kernel messages.

 This happens on my system with kernel 2.0.34 and 2.0.36.

 I think it's a network problem maybe IP Masquerade is involved. I was also
suspicious of the Tulip driver and my network card which is a Netgear FA310TX
card that used the Lite-on PNIC chip instead of the Digital 21040a. Do you
have one of these cards by chance?

My system:
Multia UDB 166Mhz
24MB RAM
340MB SCSI hard drive in external case.
Netgear FA310TX with Lite-on PNIC chip.
Kernel 2.0.36
RedHat GNU/Linux v5.1 with syslogd patch.
