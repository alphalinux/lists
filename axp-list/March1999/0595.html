I had a look in the DP264 users manual, and I checked the pinout of the
DIMM modules. As fare as I can tell, it seems that there's only 12 address
pin's [a0:a11]. On top of that there are two bank select pins [ba0:ba1]
To my knowledge this means that it is only possible to select
[2**(2*12)]*[2**2] or 64Meg. addresses onto the DIMM's.
Knowing that the DIMM's data bus width is 8 bytes wide, the maximum
addressable bytes will therefore be 512 MB/DIMM.
This means that the max useable bytes will be 16 * 512MB or 8GB.

Now, pin #54 is on some DIMM modules referred to as A12/CS2 but
only as CS2 in the DP264 DIMM pinout. Anyhow if the Tsunami
chipset can use this A12, this would mean that 2GB is accessible
on a single DIMM and thus a total of 32GB.
Well, one can only dream, but I wouldn't count on more that 8GB.
