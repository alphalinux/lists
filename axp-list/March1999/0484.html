> Pentium class is out of the question. 32 bit address space limits you to
> 4GB. In practice, because of how the memory is segmented, you can only put
> 1GB on any Intel system. There are patches to hack it up to 2GB, but at the
> expense of larger per-process memory usage (IIRC). Apparently the Xeon
> class processors support 36-bit addressing, but in a stupid and archaic
> (typical Intel) method of segmentation/bank switching. There was discussion
> yesterday on the kernel list about adding support for this to the kernel,
> and the response was generally negative. You never know though, someone
> might write it...
>
> I'm interested to hear a response on the question of using only 64-bit PCI
> slots on alpha... PCI is another Intel design. People over there have
> their brains hardwired to 32-bit...
>
> Also take a look at UltraPenguin Linux on Sparc 64's. I don't know if they
> support more memory, but it's worth a look since it's theoretically
> possible.

At the pisk of getting flammed for posted this on an RedHat Alpha list.
Check out Solaris. It is now free for personal/non comercial use and
it will be "almost open source" very soon. It does support the memory
size you are looking for and it will do large scale SMP. I think support
for 8GB RAM is a hardware issue as well as a software issue so you will
have to spend the bucks for the big iron hardware. Solaris isn't really
all that bad. I've managed to build a GNU/Solaris system that is almost
RedHat-like execpt for the Kernel and the X server.
