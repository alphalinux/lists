On Mon, 15 Mar 1999, B. James Phillippe wrote:

> program compiles and executes normally). All that I can see in <sys/io.h>
> is that ioperm and iopl are declared as extern. This doesn't even make
> sense for Alpha, so I assume if those functions are defined in the glibc
> that they're noops geared only to aid in porting(?)

 Ioperm() and iopl() use mprotect() to permit or deny access to I/O port
ranges. These functions as well as libc's _in[bwl]() and _out[bwl]()
make use of /dev/mem mmap()ed in a way depending on contents read from
/proc/cpuinfo or /etc/alpha_systype (in this order of preference).

 From applications' point of view all these functions perform in exactly
the same way as their ia32 counterparts which make use of CPU features.
