This confirms my guess -- ffm and cpml never return denormals,
so the g77 wrapper doesn't have a problem with them. Fortunately or
unfortunately, we don't have to worry about a proper fix, since
libffm and libcpml never return denormals. Ideally there would
be a compiler switch which made user code and the libraries all
deal with denormals.
