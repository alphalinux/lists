Thanks for the tips about inb/outb on Alpha. You were right about it
not making sense for a architecture-independent inb/outb. I guess I
should have thought it out a little more.

Anyway, let me back the truck up a bit, and ask my real question. I
got one of those Connectix QuickCams that interfaces to the parallel
port, and wondered if it would work on the Alpha. I tried compiling
one of the packages that supports it under Linux (cqcam), and it is
(apparently) *very* x86.

>From what I can tell (I haven't spent too much time examining the code),
it uses port IO (from user space) to access the camera and "do it's thing".
I was gonna try to get this working under Alpha when I met these inb/outb
problems.

So, before I launch into a code review and make someone else's project
"Alpha-clean", maybe it would be better to ask first, "Has anyone used
a QuickCam on their Alpha?"
