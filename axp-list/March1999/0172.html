Daniel J. Frasnelli wrote:
>
> > To what extent to standard server daemons like Sendmail, Squid, telnetd,
> > wu-ftpd, etc. use floating point math? How about bash? I'm not a
> Little to none would be my guess.

Correct. You can try running "ldd" on the executables. You can
prove to yourself that sendmail does not even reference libm.
So there is not even one call to the math library. Still this
is not to say that it does not do any floating point multiplies
or divides which would not require a call to the library.

I tried running ldd on the X server. It does use libm. You'd
have to look at the source code to figure out why. Font rendering
could make use of FP math. I think Postscript fonts require
FP math. I don't have time to look into it but an interesting
idea.
