> I don't think this is going to happen however, as we are seeing that the XP

I agree with the QC/repair issue. It's tough getting a service tech who
works on PC's to understand what you're talking about for the XP1000.
Still waiting on them to respond to my request for a new mainboard.

So far, I've found the memory bandwidth problem to be two edged.
First, it is tied to smart compilers. egcs isn't quite up to the
task for certain code styles. egcs can however do quite well with
the "right" style of code. I've already mentioned some of those
problems in other posts. The second issue is the bandwidth of the
XP1000 memory subsystem. It is not as fast as the DP264, which the
SPECFP/stream and my own tests show to be true. It is however, pretty
fast in general so overall, I'd say it's a reasonable box. (now that
the kernel is stable!!)

> And in a couple of months we shall see the newer SlotB designs from Alpha
> Processor and AMD affiliates coming to market, at which time the 21264 will
> become an affordable alternative for the market.

I hope API does some work on the memory subsystem to get it
faster, hopefully staying with commodity parts.

AMD in general is a bit worrisome since they keep losing money
every time they start up a new chip series.
