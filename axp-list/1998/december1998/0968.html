In message <5518FE2C95E6D111B1A500104B72113B02042061@alfexc8.alf.dec.com>, Bob
Segrest writes:
> Greetings,
>
> After you install RedHat Linux and start up the default window manager
> several applications (nxterm, control-panel, console...) are automatically
> started. During the last few days I have spent several hours trying to
> figure out where this is done so that I can customize my desktop
> configuration at startup.
>
> What I have discovered is...
>
> The /etc/X11/xinit/xinitrc never gets executed.
> As a result, the user configuration file ~/.xinitrc never gets
> executed.
> When the /etc/X11/xinit/Xclients script executes all it really does
> is run /usr/X11R6/bin/RunWM
>
> Can anyone tell me...
>
> Where and in which script the default nxterm application is launched
> ???

I'm not up to what Red Hat is currently doing. However, the file you are
looking for *should be* ~/.Xclients.

> What is the prescribed method for launching additional applications
> at user login ???

The same (generally). You can also start stuff in /etc/X11/xdm/Xsession.
