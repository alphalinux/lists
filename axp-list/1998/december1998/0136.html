I was trying to build the Postgresql database server on my
DEC Multia under RH5.2 It chugged away for hours (Multias
are slow) then stopped with a compiler error I've not seen
before (see below) Looks to me like a problem with egcs.
I am using this version from the RH5.2 CD:

[[chris@owl ~]$ gcc --version
egcs-2.90.29 980515 (egcs-1.0.3 release)

This is not a "show stopper" problem. If I change the optimizer from
-O2 to -O1 it compiles OK. Looks like a egcs optimizer bug.
Maybe there is a better compiler I could try? Is there someone
who would like to see the bug report?

gcc -I../../../include -I../../../backend -O2 -mieee
   -Wall -Wmissing-prototypes -I../.. -c datetime.c -o datetime.o
