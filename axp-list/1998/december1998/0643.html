On Tue, Dec 15, 1998 at 08:57:52PM -0500, Jonathan L Dubois wrote:
> can 2.0.36 be compiled without patches?
> I changed ARCH = i386 to ARCH = alpha in the makefile but
> make config gives:
>
>
> make: *** arch/alpha: Is a directory. Stop.

I think 2.0.36 needs patches as does every other kernel I have
compiled. I don't quite understand it, but I suspect the problem is
that the kernel folks don't test the kernel very thoroughly for each
patch. Say they release #7. We get it and make patches so it can
compile and these get incorporated into #8. At the same time, other
people submit patches to #8 that don't compile on alpha. Et cetera,
et cetera.

I'd expect that there will always be an alpha patch until they have QC
for kernel releases.
