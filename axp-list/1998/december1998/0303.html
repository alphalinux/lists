  I'm attempting to build a cross compiler on my Red Hat 5.2 Alpha XLT
to let me make i386-linux-gnu binaries. Following the instructions in
the GNU CC manual I am able to successfully compile/install a
cross-assember and cross linker (everything in binutils) but glibc
fails with..

# ./configure --target=i386-linux-gnu
# make
(successful compilation output)
make -C sunrpc others
make[1]: Entering directory `/usr/local/src/glibc-2.0.6/sunrpc'
LD_LIBRARY_PATH=..:../elf:../nss ../elf/ld-linux.so.2 ./rpcgen -c
# rpcsvc/bootparam.x -o xbootparam.T
./rpcgen: Error in loading shared libraries
../elf/ld-linux.so.2: undefined symbol: _dl_default_scope
make[1]: *** [xbootparam.stmp] Error 127
make[1]: Leaving directory `/usr/local/src/glibc-2.0.6/sunrpc'
make: *** [sunrpc/others] Error 2

and then gcc fails with ...

# ./configure --target=i386-linux-gnu
# make
(successful compilation output)
toplev.c: In function `get_run_time':
toplev.c:939: storage size of `rusage' isn't known

These are all of the newest GNU packages straight off of
prep.ai.mit.edu. Has anyone on this list built cross compilers for
x86 or other machines? What other issues do I have to look out for
regarding floating point (axp isn't ieee), etc.. Ideally I'd like to
build onecompiler that lets me ouput code for i386-linux,
i386-freebsd, axp-linux, and possibly other platforms.

I checked the FAQ and list archives but couldn't find anything useful.
