Jeff DeMaagd wrote:
>
>
> As far as
> I can determine, the linker isn't finding the directories those files are
> in.

There is a bunch of directories which linker will visit "always"
(this is really determined by your 'specs' file for a compiler).
You can always add extra directories to this list by listing them
with -L option in compilation flags. Compiler docs have something
to say on the topic.

BTW - what loader will consider when looking for shared libraries
is listed in /etc/ld.conf.so (once again - there are some defaults
like /lib and /usr/lib). Just rerun ldconfig is adding something
new to the list.
