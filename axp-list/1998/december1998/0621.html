I've had the same problem booting MILO from the ARC console on my Multia,
ever since I installed a Matrox Millennium card. ARC recognizes and
initializes the card (with some strange interference while it's at it) and
I get all the output on the Matrox card. MILO, however, locks up at the
same point where yours does: Swapping to PALcode at 0x0...

The MILO installed is the one that came with RedHat 5.0. I haven't tried
booting any newer MILO (if there is any) this way yet. However, I tried
booting from disk the latest (UBD) milo.dd image from ftp.digital.com.
Booted this way, MILO recognizes and initializes the Matrox card, without
any interference unlike when ARC does the job. Everything goes fine up
till I get a MILO> prompt, but then any command I give results in PCI
errors flooding past the screen and the machine eventually locking up.

So, it quite certainly is about the graphics card. Like you, I'm eager to
any advice on how to fix this. Oh, and booted this way MILO recognizes
only 24MB of the 56MB of memory I have, can anyone help with that?
