On Wed, Dec 09, 1998 at 07:11:11PM -0500, José Pedro Sousa do Amaral wrote:

> Has any one managed to get the CDDB function of KSCD (KDE CD player) to
> work? I read the FAQ and they recommend the upgrade to a newer version to
> those that are running version 1.20 (such as I). Where can one find a
> newer version? I thought that 1.20 was the newest but apparently I am
> wrong.

Kscd was orphaned by its original author Bernd Wübben and is now
maintained by Sam Maloney <thufir@illogic.ml.org> (though this address
probably doesn't work anymore since ml.org stopped their service).

Anyway, the latest kdemultimedia package (in the 'unstable' directory on
any recent kde mirror) contains the post-1.20 versions of kscd. I compiled
a more-or-less recent snapshot of the whole kde on my Alpha and have no
problem using it.

If you only want to be able to access cddb in read mode (i.e. if you don't
regularly submit new cddb entries), you can also install kscd 1.2.0 and
change evey occurence of "Kscd" to "kscd" in it, because it's the
identification of Kscd rather than kscd that suddenly seems to be blocked
by cddb servers.

You can change this in the source (using find, grep and sed or any normal
editor), or in the kscd binary (using hexedit).
