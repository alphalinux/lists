Greetings,

Well I made some progress today... I discovered that my RedHat kernel was
not built with SCSI tape support and then managed to build a new kernel.
Now I can tar and dump files to my tape drive...

However, the dump command fails to recognize that a DAT tape writes at much
higher density (61K bpi) than a mag tape transport and appears to be using a
default density of 1600 bpi. The net result is that dump needs more than 8
tapes to backup about 315MB of data on a tape transport capable of storing
2GB (uncompressed).

Just as a test I used tar to backup the same 315MB of data to the same tape
transport. It worked without any problems, but tar is not a great utility
when it comes to partial restoration of a large archive...

I looked into the taper utility, but when I issued the "taper -T /dev/st0"
command it just came back with a "Illegal tape type" message....

Is anyone using a SCSI DAT tape transport with Linux ???

If so,

        What kind of DAT tape are you using ???

        What archive tool are you using ???
