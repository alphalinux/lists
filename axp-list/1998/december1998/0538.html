Hi.

I've been using the 2.1.131 kernels with Alan Cox's patches the last
week or so...

As I was reading some of the stuff in the kernel source tree.. I noticed
that the Makefile under /usr/src/linux/arch/alpha ignores the config var
CONFIG_ALPHA_EV5 ... my system is a 21164a alpha PWS500a.. Miata
family.. so I should really be using

-mcpu=ev56 -Wa,-m21164a ....

so I changed the Makefile to do this.. and the resulting kernel was
smaller than it was without it. This I suppose is a good thing, since
the new kernel compiled with the options above works and hasn't shown
any problems.

The odd thing is that most of the time, when I compile something
(afterstep, enlightenment, etc) with the -mcpu=ev56 and -Wa,-m21164a
options... the resulting code is usually bigger... so what is up? does
this make sense?
