On Sun, 13 Dec 1998, Steve Frampton wrote:

> H1 d00dz:
>
> In attempting to upgrade XFree86 to 3.3.3, I noticed that I'm currently
> only using SVGA support. My machine, an old XL-233 has a Matrox
> Millenium 3D card. Is the SVGA server the best I can use?

The SVGA server includes support specifically for Matrox cards. It doesn't
fall back to "VGA compatability" or anything...

> Also, I applied a lot of the RPM's already, but am having problems with
> some:
>
> # rpm -Uvh XFree86-3.3.3-10.alpha.rpm
> failed dependencies:
> libttf.so.2 is needed by XFree86-3.3.3-10
>
> What is "libttf"?

TrueType font library. It's in the "freetype" package.

> Also, I installed the 100dpi-fonts, 75dpi-fonts, etc. but I notice a bunch
> of fonts, "XFree86-ISO8859-2-100dpi-fonts-1.0-1.noarch.rpm" (there are a
> bunch of these "XFree86-ISO8859-..." files), as well as an
> "XFree86-cyrillic-fonts-3.3.3-10.alpha.rpm" which I've never seen in the
> x86 world. Do I need these? What are the differences between these and
> the "regular" fonts?

You only need cyrillic if you want to read cyrillic languages... :) You
probably don't need the ISO8859-2 set either.
