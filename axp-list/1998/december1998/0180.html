>> >decompressed and the message "jumping to kernel 2.1.131..."

I think you just did not compile console support into your kernel, so it
starts to boot, but you can't see it.

#
# Character devices
#
CONFIG_VT=y
#
# Console drivers
#
CONFIG_VGA_CONSOLE=y
# CONFIG_FB is not set

Be sure those are set in the .config, or by using make config just say yes
to Config VT's and Config VGA Console. After you do that, when you reboot
after it jumps to the kernel you will have a console. If its not that,
then I have no clue what it is :)
