I compiled xcdroast-0.96d with egcs-1.1b on a RH51 installation
with kernel 2.0.34 (with SHMMAX set to 8MB).
My first attempt to master and write a CD failed, but I could work
around the problems by calling mkisofs and cdrecord manually from
the shell with exactly the same command line options as are used
by xcdroast (I actually copied and pasted from xcdroast.log).

Problems:

1)
mymkisofs-1.12b3 (as shipped with xcdroast-0.96d in source code)
creates _not_ the same output if called by xcdroast as it does
if called from command line (a `diff --binary -r' of the original
files tree and the CD finally written showed me that it was the
right way to go on with the CD created from command line).

2)
cdrecord-1.6 failes when called from xcdroast, but works o.k
when called manually (only veryfying the burned image seems
to work from xcdroast).

Are these known (compiler version related) issues?

Regards, scr
