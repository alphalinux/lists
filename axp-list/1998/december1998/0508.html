Wai-Sun Chia <waisun@msa.dec.com> writes:

> Themis Lazaridis wrote:
> > /sbin/loader: Error: unresolvable symbol : _OtsDivide32
> > I suppose I need to transfer a library from DEC to linux.
> Brute force method ;-)
> cd to the directory containing the shared lib, typically in /usr/shlib
> nm *.so | grep _OtsDivide32

Even easier, under DU run "odump -Dl <file>" on the binary you plan to
move to Linux, and it will show what libraries it is linked against.
I believe certained shared libs can be linked against other shared
libs, so you might run "odump -Dl" on the shared libs you copy over to
make sure you get all of the ones that you need.
