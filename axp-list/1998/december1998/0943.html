Yep, but the point is that with x86 kernels >= 2.0.36, there is no
need to use the lilo 'fix'. The kernel will detect the correct amount
of memory automatically. If you were to change the amount of memory
(up or down) on those systems, you'd have to remember to change the
lilo settings. If you added memory and forgot to change 'mem=', you'd
be in the same situation you described above (your system wouldn't be
using all its available memory). If you removed memory and forgot to
change the 'mem=' setting, the system would most likely not boot.

One can still use the lilo 'mem=xx' feature for testing, for example,
low memory conditions [ie. setting mem=8M to see if the vm subsystem
breaks when running X, quake2, netscape, xemacs, a kernel compile, and
an egcs compile simulutaneously ;-> ].
