On Wed, 9 Dec 1998, Vignaud Jean-Baptiste wrote:

> I would like to know if someone is actually porting Mozilla with GTK support
> to linux-axp?

I've actually gotten the "viewer" (which is the testbed for the core
NGLayout engine) to run. The full browser segfaults somewhere in GTK code
before displaying any windows.

"viewer", unfortunately, was Slower Than Death(TM), but I understand this
is common to all Linux platforms. They're also getting ready to rewrite
the whole browser front end outright.

In other words, yes, Mozilla is being developed in such a way that porting
it to the Alpha should be fairly painless, but right now is not a good
time to try: there is no fully functional browser on ANY platform, AFAIK.

Wait a couple months and then see what's up.

www.mozilla.org/status and www.mozillazine.org are good for news, and
someone already mentioned the port page at unix-ag.org.
