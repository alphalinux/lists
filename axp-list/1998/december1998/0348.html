> ghostview still crashes, but I D/L xpdf and it works great. Thanks for

xpdf is the best option that I am aware of; I haven't tried ghostview
for pdf's. One word of warning with xpdf though: I haven't been able to
get it to work with pdfs that have a frame of categories that link into
other pdf's. This feature is used often on doc CD's .. which I use a lot
for reference to chipset specs and the like.. so consequently, I use
the Acrobat Reader under Linux/x86. Not a solution that I particularly
like, but it beats Windows. All my linux boxen are used via remote X
sessions using my Multia as the console, so it isn't too bad .. but
I'd still prefer an Alpha solution since my Multia owns the SCSI box
with my good CDROM drives. My Pentium just has a little 2x deal carried
over from days of yore. NFS works, but I'm using 10b2, and that requires
a lot of shells all over the place to do the set up and mounting.

If anyone knows how to get xpdf to deal with the frames/links (the ones
with the little triangle buttons to navigate the categories) I'd like
to hear about it.
