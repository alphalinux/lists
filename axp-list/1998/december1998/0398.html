Well, first of all, this isn't that scary... this is probably some old
32 bit code.... on 64bit machines, pointers are 64 bits wide, and the
original code was probably casting a 32bit integer (signed/unsigned?) to
a 32 bit pointer on a 32 bit architecture. That would have been ok. Now
it's trying to cast a 32bit integer to a 64bit pointer. I'm not sure how
this situation is handled, usually you get the upper 4 bytes padded...
but depending on what the pointer is used for, (and you'll have to look
at the code very carefully)... you might be able to get around those
warnings if you cast the 32bit integer to a 64bit integer (__64, __64 or
int64_t.. or something like that).... or use a 64bit integer from the
start if you intend on casting it to a pointer. Well... these are just
some of my thoughts, I'm no professional programmer... so if anyone has
comments, I'd be happy to learn from them.
