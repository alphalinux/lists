Kevin.Brown@asu.edu wrote:
>
> The multia has the NCRC810 SCSI-2 controller, yet under the SCSI low-level
> drivers section there is no listing for NCR8xx.

It is located in drivers/scsi/ncr53c8xx.{c,h}. There also an extensive
README.ncr53c8xx (around 50-60k, depending on a version) there;
most definitely a recommended reading material.

BTW - if you are not sure what you have on hand then a peek
into Documentation/Configure.help may make wonders. In general -
checking docs is **really** something that should be done before
you venture on web.

> There is an NCR53c406a and a
> Generic NCR5380/53c400.

If you are not using 'menuconfig' (a mistake unless you know very
well what you are doing) then you are picking up a wrong driver. Some
choices are mutually exclusive. NCR53c8XX is the last choice in
NCR family. Just run 'make config' or 'make oldconfig' (the later if
you already have some .config file and if you want to modify it; but
before that you have to **delete** all entries there which are relevant
to things you want to modify - does not matter if they appear to be
commented out or not).

> I searched the web (yes I tried something useful) and
> found some mention of people using it with the 2.0.7 and 2.0.10 kernels.

It is one of the most popular SCSI drivers on Alpha so "some" is a bit
of understatement.

> Do I need to download this and compile it to get
> support for this controller

Once you have have your machine up and running you may want to
experiment with the current version of the driver
(ftp://ftp.tux.org/pub/roudier) but what you have on hand should
be fine for a start.
