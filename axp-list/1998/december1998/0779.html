> I've got an AlphaServer 1000 4/266. I'm using the MIKASA-s.gz from
> ftp.digital.com in the Test-2.0.34pre? directory. That one worked. I've
> got an Alpha Station that I'm trying to build a SRM kernel for MIKASA
> using the RPMS that RedHat just issued for the 2.0.36 Kernels. I've
> compiled it with SRM=y and Mikasa as the machine type. It doesn't work.
> I also used the mikasa-s.img file from redhat that came with RedHat 5.2

Please try using, from gatekeeper.dec.com:

        /pub/DEC/Linux-Alpha/Kernels/alpha-sources-2.0.35-0.2.tar.gz

as your sources. The 2.0.36 set, either pristine or from Red Hat, does
not have all the Alpha patches that the gatekeeper version does.

Your config file, as well as the config file apparently used by RH to
generate their 5.2 MIKASA-s kernel, look fine. And the kernel in the
mikasa-s.img is the same as the mikasa-s.gz in kernels/.

So, perhaps there's something broken in 2.0.35-0.2 WRT MIKASA? If you
build from those sources, and still have trouble, let me know...
