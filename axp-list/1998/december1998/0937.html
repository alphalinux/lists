Hi,

As a practical matter, we cannot support every platform that exists.
Every port to a new platform requires testing (with each release) and
continued support. As a result, we try to support the platforms that
will bring the largest number of customers (well, actually,
largest amount of revenue).

We have been tracking requests for KAI C++ on Alpha/Linux. Given the
number of requests that we have received for KAI C++ on Alpha/Linux, we
have been considering the possibility of extending support to this
platform. However, no final decision has been made yet.

As for your question regarding performance of KAI C++ generated code
versus gcc or egcs generated codes, the answer depends partly upon the
style of a particular piece of code. The optimizations of KAI C++ are
aimed at object oriented C++. So for object oriented C++ codes, you
should, in general, expect to see performance improvements, while for
plain C codes you will not see much improvement. One caveat with
comparing gcc and KAI C++, is that gcc's C++ library is not as standard
compliant as the one included with KAI C++. In some areas of the
standard library, such as I/O, standard compliance comes at a
performance
cost. A discussion of how KAI C++ improves performance can be found
in
the KAI C++ User Guide at
http://www.kai.com/C_plus_plus/online_doc/UserGuide/chapter_3.html .

KAI C++ does support Ansi C and K&R C as compilation modes.

Off hand, I am not aware of any C++ compilers for Alpha/Linux besides
gcc/g++ and egcs. If we do decide to support Alpha/Linux, we will let
you know when the product is available.
