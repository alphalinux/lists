>>>>> "justinbailey" == justinbailey <justin_bailey@brown.edu> writes:

justinbailey> Hello all. I am looking to set up my a mail server on my
justinbailey> as200, serving no more than 5 pop accounts with average
justinbailey> mail loads, and before I go through the pain of
justinbailey> configuring sendmail, I wanted to ask all of you how
justinbailey> qmail compares. I have heard from some that it's better
justinbailey> in almost every way (security, speed, ease of
justinbailey> configuration, etc.) Is this true? If so, does anyone
justinbailey> know of better documentation than the mail-HOWTO on the
justinbailey> LDP? Again, thanks for sharing your wisdom with a
justinbailey> relative newbie.

For the task your are talking about, I don't think there would be much to
choose between sendmail, qmail or exim for that matter. I would use whichever
MTA you are most familiar with. The configuration of any of for this task
is essentially very simple, but having said that, if you are not familiar
with the software that makes these tasks appear more difficult than they
are.

More than a few people find that they have some problems with getting
outgoing mail working for pop accounts attached to a qmail server. This is
not because it is difficult (it's actually very straightforward), but
because they fail to realise that they need to consider this as
"relaying". (This crops up on the qmail mailing list *all* the time, much
to the annoyance of a number of the regulars.)

If you go this route with qmail, make sure you also grab Dan Bernstein's
tcpserver software as well.
There is plenty of information in the qmail documentation and FAQ to help
you set up qmail. There is also a relatively new "Qmail-HOWTO" which you
might find useful (http://www.flounder.net/qmail/qmail-howto.html).

I run qmail here, and on my other machine which is also a small pop
server. It works well, but then so did sendmail.
