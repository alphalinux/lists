Cornelius Creedon wrote:
>
> one of 2 things:
> use 'make config' rather than 'make menuconfig'

'make oldconfig' is usually the most convenient one; but indeed
doing 'make menuconfig' is a bad idea.

> and/or
> you shouldnt be changing the permissions on the resulting vmlinux.gz.

Permissions on vmlinux.gz are totally irrelevant. When milo is
loading this file it just has to be able to read it. That is all.

> mine is 775 or 755 and i did not change it after compiling...
> 644 is probably bad since it removes the executable attribute.

Which may actually be good idea in case somebody would try to run
this thing as a program - not a nice thing to do. Execute permissions
on a resulting file are an artifact; this is how linker works and
you want that most of the time. :-)
