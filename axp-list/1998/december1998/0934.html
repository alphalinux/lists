Stefan Boresch wrote:
>
> Strange, strange -- on my Ruffian, ARC=false and the system
> is quite happy... (and yes, the time is correct, too)

On your Ruffian you probably have the latest version of 'clock'
utility. It was rewritten to be "smart" and it discovers by
itself if ARC correction is required or not. Flags like '-A'
are still present to keep a command line backward compatible
but they are doing nothing. It follows from that that if you
are using this new 'clock' then settings of an 'ARC' configuration
constant are totally irrelevant; this is not true if your
'clock' was not updated yet.
