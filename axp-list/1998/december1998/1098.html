Quoting Tristan Colgate (colgate@spods.dcs.kcl.ac.uk):
> I know the dhcp client does alter the network file, but if you are just
> using ppp to connect to an isp dhcpcd doesnt come into play. The IP is
> negotiaited by the pppd itself.

Well, actually the /etc/HOSTNAME file is changed by
/etc/sysconfig/network-scripts/ifup-post
whenever the process of bringing up the interface changed the IP, whether it
was done by dhcp or ppp. Since I had a static IP on my dial up account, and
dynamic IP on my cable modem, I assumed it was dhcp that did it, but I was
wrong.

I assume that if it bugs you, you can comment out the calls to set_hostname in
ifup and ifup-post.
