One of those "weird-build-options-(what-a-freak!)" things...no
reply/response is necessary.

Seeing as I'm a masochist (I'm also attempting to build sendmail 8.9.1
from sources, including several brand new configuration files), I
tried to build an SMP-aware Linux out of the following:

        one (1) AlphaStation 200 (Avanti)
        one (1) Linux kernel 2.1.131-ac13 (sources)
        one (1) egcs-1.1b-2

The kernel boots from SRM, hanging after the line "Console: colour
VGA+ 80x25". Here's the exact kernel boot message:

Linux version 2.1.131 (Linux version 2.1.131 (xenophon@nemesis) (gcc version egcs-2.91.57
 19980901 (egcs-1.1 release)) #1 SMP Sun Dec 20 20:03:38 EST 1998
Booting on Avanti using machine vector Avanti
Command line: root=/dev/sda4 bootdevice=sda4 bootfile=vmlinux.gz
apecs_init: using Window 2 settings
apecs_init: PB2R 0x40080000 PM2R 0x3ff00000 TB2R 0x0
setup_smp: 1 CPUs probed, cpu_present_map 0x1, boot_cpu_id 0
Console: colour VGA+ 80x25

Rather than post the kernel config here, I've made my config available
at <URL:http://www.irtnog.org/~xenophon/.config>. If I turn off
CONFIG_SMP, everything builds and boots happily.

Of course, I doubt SMP is supported on the Avanti (and in any case is
wholly unnecessary for uniprocessor systems, such as the AS200). Call
me crazy, but it was either play with SMP Linux or clean the kitchen.
Note that the kitchen is still a mess. :)
