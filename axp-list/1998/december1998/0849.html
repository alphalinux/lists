>
> However, the dump command fails to recognize that a DAT tape writes at much
> higher density (61K bpi) than a mag tape transport and appears to be using a
> default density of 1600 bpi.

'dump' command does not know anything about density but has a default
tape length - which is rather short one and which can be overriden
('man dump' for details). A simple way which allows not to bother
with length and densities is to dump on stdout and redirect output
to a tape. This will work as long as your whole dump **really**
fits on one tape.

>
> Is anyone using a SCSI DAT tape transport with Linux ???

Yes. For a long time. On different machines and different processors.
It is not essentially different from using DAT drive on any other Unix box.

With many DAT drives you want to 'mt -f /dev/st0 setblk 0' in order
to use variable block size. Your drive manual may have something
to say about that (otherwise run few experiments to see if this makes
any difference to you).
