I'm not sure if this would be "errata" or not but:

Installing on a RedHat 5.0 system the installation script expects libstdc++ to
have been installed via rpm. Since I had uninstalled the gcc RPMs in order to
compile egcs from scratch this caused a dependency failure. Re-naming
/etc/redhat-release caused the system to install with --nodeps and everything
was fine. Obviously there are other workarounds possible as well.

Other than that - a *great* job. Many many thanks.
