OK, the problem seems to be with my old versions of linload/milo. I'm
using the same versions I used with 2.0.32, which are blown into flash.
Somebody else loaded these for me and I'm loathe to experiment lest I
screw them up. Also, the old ones worked with 2.1.129 and 2.1.130.

/proc/cpuinfo used to say (under 2.1.130 configured for an EB164) that
my machine is an EB164, LX164 variation; the book that came with it says
AlphaPC164/LX.

I downloaded the latest linload.exe/milo from the AlphaPC164LX directory
on gatekeeper, copied them to an mformatted diskette with mcopy, and set
up the appropriate boot menu record. The Generic 2.1.131 kernel (with all
patches applied) then booted successfully.

/proc/cpuinfo now says this is an EB164, variation 0.

So now I have N questions:

Should I continue to use the Generic kernel or not?

What's the difference between Generic and (say) EB164 or LX164?

How do I set up the flash ROM with the new firmware? I realize
this is in the HOWTO but it's still not clear to me exactly how
to do an upgrade; in particular, where does the new linload.exe
go and how do I get it there? I guess I need to use fmu to
write the new milo into the flash... There aren't userspace
utils for doing that, are there?
