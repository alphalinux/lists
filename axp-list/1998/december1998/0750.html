After posting, I realized that if the file is on an NFS mount, the
'1k effect' takes place, otherwise not, and I get ff...... counts, too,
with any argument when on a device directly hooked to the machine.

> The question is if the correction value works correct with
> system calls.

I've instrumented about every libc I/O routine used by cpp, cc1, as and ld
in my test runs; fclose is the only routine that gives me a headache.
e.g. doing a write() and a close() instead of fwrite() and fclose() works
nicely and produces decent numbers. I haven't yet looked into libc sources
to see if some particular syscall fclose uses causes the odd behaviour.
