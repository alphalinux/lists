Hi,

Just in case anyone else had the same problem I did, what was causing my
2.1.131 kernel to not boot was the frame buffer stuff for the matrox
millenium II. Once I got rid of that, I was able to boot my system.
However... now I have other problems....

But first, just a list of some observations I've made....

With kernel 2.0.35 I was getting very slow disk access via SCSI 3940UW
controller... and mounting the root partition took several minutes.
Also, if I try to reboot without resetting... basically, going back to
MILO and then booting again, the system would freeze.

With kernel 2.0.36 disk access was getting faster.... however, mounting
the root partition still took several minutes. On the good side,
rebooting from MILO (without hard reset) started working.

With kernel 2.1.131.... wow... SCSI disk access for the 3940UW was
greatly improved... from what I can tell, the new aic7xxx seems to take
into account expiration times of some sorts... in anycase, disk access
was GREATLY improved... mounting root partition took 6 seconds, starting
X took 2 seconds (took 36 seconds with the 2.0.35 and 36)...

However, that's about where all the good things stop.

With 2.1.131 I am now having problems with my tulip driver/ethernet
card. First I noticed the 131 kernel comes with a very old version of
tulip.c ... I replaced it with the newer version that was with 2.0.36
and recompiled... yet, same problem.

The symptoms of the problem are:
when I try to use ipfwadm, I get the error message I got when I used the
default ipfwadm that came with RH 5.2. Which is as if it wasn't compiled
into the kernel... but I check that, I do have the correct options
enabled for IP masquerading.

also, my routing table is awkward. Every entry is duplicated. But if I
shutdown all the interfaces (3 of them, eth0,eth1, eth2) and restart
them manually, the routing table is fine.

the third problem I think I just figured out... for some odd reason eth0
and eth1 got swapped... they both use the tulip driver. So thats
probably why the I couldn't get out to the net using eth0.... as my net
connection was switched to eth1.

Anyways... anyone trying 2.1.131 on Miata,... these are some things to
watch out for... and if anyone knows why ipfwadm isn't working... please
let me know. Thanks!... I heard rumors that I need a new ipfwadm to work
with the newer 2.1.131 kernels... confirm?
