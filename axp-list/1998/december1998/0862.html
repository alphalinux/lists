Hi All,

Yesterday, I downloaded the following sources from
ftp://ftp.digital.com/pub/DEC/Linux-Alpha/Kernels/:
 
alpha-sources-2.0.35-02.tar.gz and milo-sources-2.0.35-02.tar.gz

I unpacked the kernel sources in /usr/src/linux-2.0.35-02 and made a
symbolic link to it with /usr/src/linux. I configured the kernel options
with the exact ones I used with the 2.0.34 version of the sources,
compiled and linked the kernel using:

===
make mrproper
make menuconfig
make dep
make clean
make boot

mv /vmlinux.gz /vmlinux.gz.old
mv /usr/src/linux/arch/alpha/boot/vmlinux.gz /vmlinux.gz
chmod 644 /vmlinux.gz
chown root:root /vmlinux.gz

make clean
===

If I'm not mistaken, all this is correct. There are no modules yet so
that does not have to be considered.

I then tried to boot the UDB without recompiling milo and it failed
miserably. Why?

Anyway, I then unpacked the milo sources in /usr/src/milo-2.0.35-02.
I attempted to compile the new milo sources at that failed as well.

What am I doing wrong?

TIA! :-)
