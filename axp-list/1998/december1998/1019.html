On Wed, 23 Dec 1998, Peter Petrakis wrote:

> Hi Guys,
> After reviewing my courses for the next semester I have found that I can
> live without NT for the whole semester so I deleted NT and my Linux
> partition and reinstalled from scratch. I installed 5.1 like I always do.
> The zip drive which is a scsi ext on the back of a 2940UW is detected by
> disk druid like it always is and the rest of the install is uneventful.
> Reboot,login, first thing I do is unmount the zip then I got into fstab and
> change the fs it reads from msdos to ext2, that works. I it got to mount
> an ext2 disk and it works , I do my thing, unmount it. Now when i try to
> mount another disk I get "mount: /dev/sdb4 is not a valid block device".
> What's going on? I've tried everything I know and I'm getting nowhere.
> Thanks alot.

Run 'fdisk /dev/sdb'. Display the partition table by typing 'p'. The
new cartridge may not using the 4th partition. Because I find using
the 4th partition on zip/jaz disk unintuitive, I always repartition
disks before use to use partition #1.
