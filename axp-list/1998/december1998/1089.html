With regards to your message at 01:37 AM 12-27-98 -0700,
Kevin.Brown@asu.edu. Where you stated:
>I'm trying to configure the kernel for the alpha (stupid newbie that i am).
>The multia has the NCRC810 SCSI-2 controller, yet under the SCSI low-level
>drivers section there is no listing for NCR8xx. There is an NCR53c406a and a
>Generic NCR5380/53c400. I searched the web (yes I tried something useful)
and
>found some mention of people using it with the 2.0.7 and 2.0.10 kernels. I
>then found a link to tsx-11.mit.edu that contains the source code for the 810
>driver for the alpha. Do I need to download this and compile it to get
>support for this controller (it is built into the MB). I know I'm probably
>annoying some of you with my questions, but I am trying to learn so please be
>patient. Thanks in advance for any help I can get.
>
If you are booting from the 810, then yes, I suggest building it into the
kernel.
Current sources for the 53c8xx driver is at:
ftp://ftp.tux.org/pub/roudier/
And version 3.1C is the current one. There are some later subvariants (
3.1D) in the 2.1.131 kernel sets, but I believe that is unnecessary for
your purposes..
