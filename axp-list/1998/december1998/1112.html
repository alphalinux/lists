yesterday I attempted to get 2.1.132 running.
- downloaded from my favorite mirror.
- included some patches to softirq.h (posted on this list by Peter Rye.)
- make config
- make dep clean boot
- tried to boot new kernel: hung at "jumping to kernel"
- changed -O2 to -O in Makefile
- make dep clean boot
- tried to boot new kernel: hung at "jumping to kernel"
- copied newer linload.exe and milo from RedHat CD
- tried to boot new kernel:
after "jumping to kernel" the screen went black, but I noticed some
disk-activity, so it seemed the system was actually booting. Sure enough,
after a while the X logon screen came up. Now everything is fine as long as
I'm in X, but when I switch back to a textconsole with Ctrl-Alt-F1, the
screen is a dark blue, with brown characters: hardly legible. Using color
apps (like Midnight Commander) also shows the colors completely mixed up.
Ideas anyone ?

My configuration:
PC164
S3 based graphics
RedHat, recently upgraded to 5.2
.config file attached
