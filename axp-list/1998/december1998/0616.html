Seriously, though, folks...

Dave Rusling, the original author of MILO (and since gone from DEC to work
on ARM Linux, IIRC), put this in just as a way to fill up the HWRPB (aka the
HardWare Reset Parameter Block).

The HWRPB is normally created by SRM console, and contains a lot of info WRT
CPU, system type, etc, which is very useful to an OS. Digital UNIX depends
on it; when LINUX was first ported to the Alpha, it seemed a good idea for
it to depend on it as well.

When MILO came along, it was necessary to create an HWRPB with (at least)
the needed information, and some other fields were filled just for fun. I
believe this all dates from the earliest MILO work, which was done for the
NONAME boards (JENSEN was the first to run Alpha Linux, but it's been
impossible to put together a MILO for it); we needed a way to run LINUX on
them without requiring SRM console (and its license fee).

How about starting a contest to replace "Linux_is_Great!" with something
better? I'll contribute a prize: an autographed floppy containing the
original NONAME MILO, bootable only from the Debug Monitor, with no user
interface... ;-}
