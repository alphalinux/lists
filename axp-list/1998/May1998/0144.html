> Second... how can I fix passwd... I think its and RH5.0 problem... Root
> can update passwds just fine... When a user does, it seg faults

I have seen this problem recently with 5.0 on **some machines** and with
**some software** configurations. It appeared to be a trouble with
opening and reading some files - /etc/passwd in particular. Kind of
hard to track it down when gdb crashes after 'attach' when trying to read
symbol tables for 'passwd' process. :-( Updating a library to the current
version of glibc from 5.0 updates caused the passwd problem to disappear.

Everything was also ok when I used my copy of glibc-2.0.6 (NOT 2.0.7)
which I modified and recompiled in the past. But this had another
known gotchas so I preferred to use the current 2.0.7.

How this behaves in 5.1 distribution I have no idea. I did not try
it (and 5.1 was not officially released yet :-).
