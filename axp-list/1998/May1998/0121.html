>
> I'm trying to load linux on a personal workstation 500au, but am having
> problems.

Ah, so it's an "au", then you must be booting from SRM console, right?

> Using redhat 5.0, with miata image. This works until the scsi probe, where
> we get some kind of timeout and it goes into a loop with scsi timeout
> messages.

Red Hat's MIATA images is for booting from AlphaBIOS *only*.

> Found a later miata image from about 7th of Jan. This gets past the scsi,
> and correctly identifies the scsi drives, but does not get past the prompt
> for the ramdisk floppy. It looks like it is either hung, or the keyboard
> is not working.

Noe of the MIATA kernels directly in the /pub/DEC/Linux-Alpha/Kernels
directory is bootable from SRM, but go down one level into the
Test-pre34-11b/ directory and grab MIATA-s.gz, that one I know is, however
you will have to find a way to get it onto an SRM bootable floppy. Use one
of the NORITAKE*.img files in Kernels, and simply replace the vmlinux.gz
with the MIATA-s.gz one.

Good luck.
