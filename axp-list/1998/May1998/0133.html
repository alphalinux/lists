Willi Schiegel writes:
> I have a DPW 500au with 2 MB cache and 512 MB memory. When using
> nbench2.1 (a Linux/Unix-port of BYTE Magazine' BYTEmark from Uwe F.
> Mayer) the system is not faster than an IBM PC with an AMD K6 233 Mhz
> with 512 Kb L2 cache and 32 MB of memory.
>
> Under the preinstalled Digital Unix 4.0D (again with nbench2.1) the
> machine is twice as fast as the IBM PC (floating point performance even
> factor 5.5).

Is it mainly the fourier test that is slow? This is a libm problem.
The fdlibm that comes with gnu libc is a generic libm written in C
while Digital has an Alpha optimized one. Linux on intel has a libm
that contains a lot op intel specific optimizations (including
assembler stuff).

Recent traffic on comp.os.linux.alpha suggest there is hope (look for
cray libm in the subject).
