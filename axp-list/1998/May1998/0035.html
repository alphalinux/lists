On Fri, 8 May 1998, Wes Nakamura wrote:

> I'm planning to upgrade to gcc 2.8.1 or egcs 1.0.2 (will probably try
> both of them). Either should work with glibc-2.0.7 and libstdc++ 2.8.1,
> right? (I'd have to compile libstdc++, as there doesn't seem to be any
> rpms...) Would c++ work (as I understand it, the Redhat 5.0 c++ problem
> was a lib problem, not a compiler problem)? I can't say I'm 100% clear
> on the exact relationship between the compiler and lib projects. Any
> problems compiling kernels (I need to compile a 2.1.x kernel)?

Just a summary:

1. gcc 2.8.1. Compiles kernels (2.1.[89]x series) but does not compile
glibc (2.0.[567]).

2. egcs 1.0.2. Compiles glibc (2.0.7) but does not compile kernels
(2.1.9x series). Includes libstdc++ 2.8.0.

3. egcs 19980502 (snapshot). Compiles glibc (2.0.7) and kernels (2.1.99
and 2.1.100). Includes libstdc++ 2.8.0.

Currently I stick with the last option.

"Normal" programs, including shared libraries seem to compile well with
any of these three. I haven't compiled any C++ program since some time so
I cannot make comments on C++ behaviour.

If you like to experiment with binary RPMs, you may get the ones I use --
they are at 'ftp://ftp.amg.gda.pl/pub/OS/Linux/people/macro/'. There are
source RPMs, too, but beware their recompilation is quite
resource-consuming.
