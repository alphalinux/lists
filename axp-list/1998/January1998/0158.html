hi folks,

i am working on a pc164lx platform, which i have updated from redhat 4.2 to
redhat 5.0 and from kernel 2.0.30 to 2.0.32. up to this point, everything works
fine. now i have tried to install em86 but encountered some problems doing that.
i got the em86 v0.2 from gatekeeper and started making the modifications as
indicated in the description 'downloading and installing em86'.

patching and rebuilding the kernel has been no problem, but installing the
packages needed for em86 doesn't work. when doing
rpm -ivh em86-0.2-3.alpha.rpm
i get the error message
failed dependencies:
libm.so.6 is needed by em86-0.2-3
libdl.so.2 is needed by em86-0.2-3
libc.so.6 is needed by em86-0.2-3
what is going wrong here? i have noticed that in /lib i have libraries which are
called for example libm.so.6.1. so i made a link libm.so.6 to libm.so.6.1 but
the error message remains. then i tried out a libm.so.6 which is included in the
redhat distribution (under misc/src/trees/skeleton/lib), but also the effect is
the same. i read in a posting that for em86 libraries from the intel-linux are
needed, so i took the three 'missing' libraries from intel-linux, but also with
no effect. the package manager keeps on saying that he needs those three
libraries, even if they exist (or is the placing for the libraries in /lib wrong
- but i also tried out /usr/lib, with the same effect).

then i tried installing with
rpm -ivh --ignorearch --nodeps em86-0.2-3.alpha.rpm
to supress the checking of dependencies, which worked - but the third package,
em86-baselibs-0.2-1.alpha.rpm failed during install with the error message, that
ldconfig86 is missing - which is part of em86-x86-0.2-1.i386.rpm, that has been
installed as second package, without error messages, but also, as it seems,
without success. but i think the errors already come from the problems that the
first package has with the libraries (and that were supressed by --nodeps).

so, can anybody help me out here, at the moment i have no more ideas regarding
this problem...
thanks in advance,
ciao,
