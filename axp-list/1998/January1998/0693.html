On Wed, 21 Jan 1998, Randy Carpenter wrote:
> Right. And on a 600Mhz alpha, it might perform like a 286-10 :) Bochs is
> far from being usable, but I can install and boot Win95 on it. On a
> Pentium 133, for example it took about 8 hours to install Win95...
I would imagine that this could be greatly improved by hacking the code to
take advantage of the Alpha architecture, i.e., using specific Alpha
registers as PC registers rather than depending on gcc optimization. It
would also seem plausible to change instruction implementations to
actually use sets of Alpha instructions, again rather than trusting gcc.
This is certainly not a small task, and wouldn't be distributable, but if
you really need Win95, it might not be a bad idea.
