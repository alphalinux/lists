OK, here's where you can help test the latest alpha-patches... :-)

After choosing the "Prompt for development and/or incomplete code/drivers"
option (do "make config" with the latest alpha-patches installed), choose
the "Use SRM PCI setup" option.

The problem you've run into is caused by the LINUX kernel doing its own PCI
bus setup; choosing the above options will attempt to keep the PCI bus setup
as close as possible to identical to the way the SRM console does it. That
way, when the SRM console is re-entered during the "reboot" operation, it
will find things approximately the same as it originally set up.

But be aware, these patches apparently don't work on certain machines (I have
a negative report from some NORITAKE folk), although I've had fair success
on a number of others.
