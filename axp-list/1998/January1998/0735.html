Nicholas Moffitt wrote:
>
> On 20-Jan-98 Maurice W. Hilarius wrote:
> > FX!32 first runs applications in an emulator, while it builds it's own
> > binary for the application. After a few passes and some "idle" time one day
> > it runs your shiny new optimized binary instead.
>
> Does anyone know if there are AXP-Linux emulators that do this? If so,
> why aren't people distributing translated Netscape binaries and the like?

No, there are no Linux/Alpha emulators that do this. If there were, I
would probably have been the first to know about it 8-)

Hammering the x86 execution engine from FX!32 into em86 was difficult
enough. Trying to get the FX!32 incremental translator running would
be an order of magnitude greater difficulty, and we simply don't have
the resources to do so. In fact, I'm thinking of ripping some of the
translator hooks *out* of the emulator to increase performance.

There was a fellow talking about doing a *static* translator and using
em86 as a runtime system, but I haven't heard from him in a long time.

As for "why people aren't distributing translated Netscape binaries and
the like"; the main issue involves legalities. As it is, translating
applications you've already licensed falls into a legal grey area; do the
license restrictions against disassembly or reverse-engineering apply
if it's all done automatically without the user's knowledge? Hard to say.
*distributing* such code would certainly require the permission of the
copyright holder, and very few are willing to do so because of support
issues (if a translated program breaks, is it the translator's fault or
the a bug in the original program?)
