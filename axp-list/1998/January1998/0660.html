>Hi,
>
>Sometime ago i installed MILO to flash, now i would like to go back to SRM.
>(I'am using a AXPpci33 Noname board.)
>How do i create a SRM firmware boot floppy from linux (intel)
>
>Greetings Eric.

hey hey! i did the same thing a while back and i posted
to this list with no luck :(

eventually i found the solution... (sorry but i cannot give credit to
the source of the solution as i have forgotten)

--PART-A--
ftp over to this address and snag the udbfs138.img file
and rawrite it to a floppy.

ftp://ftp.digital.com/pub/DEC/Alpha/firmware/interim/udb/

--PART-B-
now you may need to either have 1) a serial terminal connected to the first
serial port OR 2) run a null modem cable from your first serial port
to the serial port of another running machine and use some kind of terminal
software (telix, minicom, etc...) and that way the other computer will
serve as the serial terminal...

you also have to flip a jumper on your board so that it will boot from a
floppy disk instead of the firmware...

on the noname boards this is jumper is J29
the position "1-3" will set it to boot from the fail safe floppy that you make
in part A

there... that should fit the bill... if not mail me
and we'll get it worked out...
