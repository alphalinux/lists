Well, I'm getting closer and closer to releasing a new version of EM86
that'll work under RH5.0.

Turns out the shared-library problem had to do with dueling library
versions. If the name of the Alpha libc that libc-thunk asks for
is not *exactly* the same as the one that em86 is currently using,
you end up mapping libc twice and the two versions don't get along
well (e.g. static data in the malloc module says that memory is
available when it actually isn't... nasty stuff).

I've got a fix for this, and the previous race-condition fixes.
Now I have to clean up some of the debug scaffolding that I put
in to track these problems down.

Two new debug features will be added that are kindof neat. The first
is a disassembler that works with instruction tracing. Now if you have
X86_TRACE_INSTRUCTIONS set, you get not just the PCs of all the
instructions,
you get full disassembly and register state too. This was what I had
in the old, slow BOCHS version and missed sorely until I figured out how
to do it with the FX!32 interpreter.

The second related feature is the ability to remember the last 64
instructions executed, along with x86 register state at each
instruction.
If you seg-fault, it dumps out disassembly and registers in a manner
similar to X86_TRACE_INSTRUCTIONS. This is handy for tracing down the
"how did that register get that way?" bugs... (I may turn the "64" into
a configurable value, I dunno...)

I hope to have a new release next week. Hang in there!
