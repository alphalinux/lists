The 'user' part of NFS (client and server) are to old for the kernel side
of NFS. Some where along the 2.1.x series, there was a rewrite of the NFS
code in the kernel.

I get these messages all the time on both my Alpha and x86 system
due to being unable to comple the new user space side of NFS with glibc
(at least I think its glibc - the build fails in a similar way on both
platforms).

However NFS seems to still work, although im having strange unreported
read errors between my P166 and a Solaris sparc at work, that might be
a result of not having the correct user side code.
