> figured that it would, and that I could run netscape and applixware with

I successfully installed em86 and applixware on RH4.2.
As I got the alpha I tried with RH5.0 but I got problems
with the libraries.
Somebody on this list told me that one could solve the problem
using egcs. On the other hand I have read on a egcs faq that by now
egcs still does not compile KDE (the new desktop environment
for unix) because some parts
of the KDEs source don`t comply with certain c++ ansi/iso norms

By now I am quite happy with RH4.2.
I can make a list of problems/solutions that I found,
(actually the solutions came mostly from this mailing list,
so thank to everybody) hoping that it might be useful to somebody:

The fortran package given with the RH4.2 gave me problems. I solved
this problem with egcs-1.0.1

I had problems creating a very-very big library (lapack library).
This problem is actually solved (they told me) on RH5.0.
As I wanted to stay in RH4.2 I broke the library in two smaller parts
(single and double precision) and it works.

My programs did not comply with C++ ansi/iso norms. Instead of
changing them to be able to use egcs c++; by now (lack of time)
I compile the c++ part by gnu gcc, the fortran part by egcs g77,
and everything works happily toghether.

I had hard times trying to install em86 but thanks to the help
of people of this mailing list I successfully installed it.
To do that, I used the kernel 2.0.33 with the alpha-patch 2.0.32
and the correction to the file a.out.h that
Jay.Estabrook@digital.com provided

About netscape, I still haven`t tested it.
KDE has a filemanager with browser capabilities, so
I will install it with a lower priority, but I am
confident that it should work

Best regards
