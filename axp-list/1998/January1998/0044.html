On Fri, Jan 02, 1998 at 12:16:34PM -0600, Stephen Wolff wrote:

> I have been told that binaries that run under UNIX
> can be made to run under Linux with the help of some kind
> of ``UNIX emulator''. If so, then I am interested in doing this for
> our Alpha.

Umm... forgive me this question, but what is a UNIX binary?

As far as I know, there is no such thing as *the* UNIX. There are plenty
of unices available, and almost as many binary formats. Not to mention
the different processor architectures etc.

So far, as far as I know, Linux/Alpha is "only" able to run

* native Linux/Alpha binaries (in ELF or COFF)

* Digital UNIX binaries

and

* binaries for Linux/i386.

The Linux/Alpha and DU binaries can be run without any external
"emulators". But note that you need the Digital Unix shared libraries
in order to run DU binaries, and there is no way to get DU libraries
without buying a full DU license. Which is not cheap.

Linux/i386 binaries can be run (or at least made walk :) ) using em86, a
program which basically emulates an intel processor. That's not exactly
fast, but it works for about everything you can get for Linux/i386
(including RealPlayer and Netscape :-) ). You can get it from
ftp://gatekeeper.dec.com/pub/DEC/Linux-Alpha/em86.

You also need some Linux/intel shared libraries, but as they are free
this is not a problem.

As a last thing, the people which told you there is "some kind of UNIX
emulator" probably meant iBCS, a thing that makes binaries for SCO UNIX
run under Linux/i386. While it may technically be possible to make this
also work under Linux/Alpha (perhaps using some code from em86), I doubt
this exists already.
