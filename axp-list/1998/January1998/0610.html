> >
> >Speaking for prices/performance, I have one additional question.
> >
> >Which variant gives better performance and less troubles:
> >SCSI configuration of SX164 or IDE configuration of LX164?
>
> Both use the same ISA based CMD 646 controller.
>

No, docs from digital say that the SX uses a Cypress chip for
IDE (the LX board uses --- as the PC164 board --- a CMD 646).
Both of these controllers are PCI based, not ISA.

Speaking about the CMD 646 chip (on PC164 and LX, but not SX):
this one is slow under Linux, I think because of missing
driver support, you will get about 1.5 MB/s throughput.
So, if your intended apllication needs high disk throughput,
you need SCSI with these boards (but I never had troubles
besides bad performance with IDE on a PC164).

Isn't there anyone with a SX board and an IDE configuration
out there who can say something about performance?
