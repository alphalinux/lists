Ive been trying to upgrade my kernel from 2.0.30 to 2.0.32 or 2.0.33 with
very little luck. For some strange reason whenever the kernel has been
installed, it normally freezes and/or crashes right after the Ramdisk driver
initialization. So I had removed FDD support in the kernel, which didn't
work, so I removed the ramdisk support, and that solved the boot-up problems
I had. I have installed 2.0.32/2.0.33 source and ran the patch from
gatekeeper (2.0.32-0.1) patch so I'm confused whether this is a known
problem and/or the best solution to fix this.

The other problem I have is with Xwindows and my Matrox Millennium II --
sometimes when I try to run startx, the machine completely dies -- it goes
to a black screen and sits there forcing me to do a hard reboot. This
happens about every other time that I run startx.
