On Wed, 14 Jan 1998, Jeremiah Cornelius wrote:

> A couple other questions for the UDB experienced:
>
> (1)
>
> I haven't yet checked the file on gatekeeper. Is there an
> experimental config file for this adaptor with 16 or 32 bit colour
> support? Is this even possible on a UDB/Multia?

No, the adapter, while it can handle 1600x1200 resolution, maxs
out at 8bits. There are other models of the TGA card (not found in UDBs)
that can do higher, but no Xserver has ever been written for them (at
least last I heard).

> (2)
> I suspect that the only real solution is just to buy a cheap PCI card,
> like the S3 Vision series... If this is the case, can I make the
> Multia a "two-headed-dog?" Or, does the inclusion of a video adapter
> in the PCI slot automagically disable the on-board TGA, as in
> home-market PC machines?

Yes, the solution is to get a cheap PCI video card, just make sure
it is short, there is not a lot of room in the UDB. Also make sure you can
return it no questions asked, the Alpha is a little picky about what video
card it will work with. I have tried a Trident and a Diamond 64 Stealth to
no avial. I ended up running X on my P100, and use my UDB as database/web
server, so I can't tell you what card will work. Maybe some one else has
an idea.
And yes, installing another video board, automatically disables
the internal TGA adapter. No double headed dog mode in this machine that I
know of.
Hope this helps!
