On Wed, 21 Jan 1998, Chad Campbell wrote:

> > Would it even be possible to do something like this? IE: by running our
> > imaginary program as follows:
> > ix86toalpha binary-ix86-src binary-alpha-dest
>
> There is a group doing this sort of thing for Windows->OS/2 translation.
> The last report i saw said that they could completely translate simple
> programs like minesweeper and solitare. I don't have the address at hand,

I think this is incorrect. OS/2 and Windows use different file formats
for executables. Windows uses the "PE" layout, while OS/2 uses ("LX"?
"LE"?). What I believe the OS/2 folks are doing is using IBM's Open32
APIs as a starting block, then adding additional Win32 APIs. Ultimately,
though, the file format has to be changed for OS/2 to run it. This is, I
think, the only part that involves any translation; they are not
converting Win32 calls into native PM calls, but distributing a PM DLL
that acceps the Win32 calls. Then you just rewrite the executable into
the OS/2 "executable" format. More like how FreeBSD is able to run Linux
binaries; especially if one used GlibC and the other libc5.
