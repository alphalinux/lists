I don't like it when mailing lists prepend the list name; I like the
labelling generally, but I prefer to use a shorter mnemonic than the
full list name. I use procmail recipes similar to the following to
achieve this. (Mine is more complex than this; I hope this
simplification actually works.) I have a pattern similar to the
first stanza for each mailing list to which I subscribe; the second
section files all my lists separately, and retitles them for my
viewing pleasure. `formail' comes with `procmail'.

-D.

--Dxnq1zWXvFF0Q93v
Content-Type: text/plain; charset=us-ascii
Content-Disposition: attachment; filename=procmailrc-extract

### Match the AXP mailing list
:0
* ^Resent-From: axp-list@redhat.com
{
DROP=axp
}

...

### If DROP was set to "null", just dump it and end
:0
* DROP ?? null
/dev/null

### Otherwise, if DROP was set, file; continue processing regardless.
:0
* DROP ?? .+
{
# Save, with lock
:0 c:
lists/$DROP

SUBJECT=`formail -b -z -xSubject:`

# Use $DROP to retitle.
:0 f
| formail -i "Subject: [${DROP}] ${SUBJECT}"

# Mark "Old" to indicate saved state, if no status was set
:0 f
| formail -a "Status: O"
}
