On Sat, 17 Jan 1998, Hugo van der Kooij wrote:

> On Sat, 17 Jan 1998, Dave Gilbert wrote:
>
> > 1) A 32 bit address environment where the C library is mapped
> > below the 2^32 limit and all things are frigged to produce
> > low addresses. Combined with a, #define long int, perhaps
> > this would allow seriously broken bits of code to be made
> > to run.
>
> It's better to have the code fixed. As thes assumptions will break on
> other platforms as well. We should not endorse an environment that allows
> crap code to work ;-)

Sure - but there is some code which is so well broken that its probably
beyond repair. I'm actually thinking of the recently released Doom
source. Sure if its your own code you go and fix it, sure if its
just being a little naughty in someone elses code you go and fix it.
But when its reading binary data files in using longs, passing pointers
and integers back and forth, etc. etc. then its not easy.

>
> > 2) Wine/Alpha - run Alpha NT binaries under Linux.
> > including:
> > a) FX!32
> > b) Device drivers??
>
> As wine is stil a long way from working mainstream anyway this is a very
> sweet dream that you can keep for quite some time. Much of wine may need a
> full revamp to work on Alpha.

OK - what are the technical issues there?

I'd certainly like to know whether its possible to run NT/Alpha
binaries - and although I'm not sure of the issues I think it would
be nice to be able to use device drivers (although I realise its
a lot harder!).

> > Are these possible or are there any show stoppers in the way?
>
> Lack of ime, lack of money, bad concepts and lack of interrest are usually
> good showstoppers. I think you will find plenty of those will apply to
> these proposals.

Time, money sure - I wouldn't have suggested them if I thought either
the concepts or interest was a problem.
