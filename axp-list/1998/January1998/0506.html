Hmm...guess I was not clear, the print file is from a previous session.
What appears to have happened is that when you had the system up
one time. You were logged in with Xwindows running and had either
requested a print screen or some application to print something. This
was then stuffed into the Xprinter queue. When you reboot the system
is telling you there is a report in the queue to print but that it can not do
so. When the system starts up it already has something in the Xprint
queue from a previous session.
