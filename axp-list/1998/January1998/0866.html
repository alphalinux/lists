Richard B. Kreckel wrote:
>
> Hello there;
>
> We need to compile a 2.1.x-Kernel (on RH 5.0) due to some problems with
> NIS. I once have read about AXP-diffs for 2.1.x but could not find such
> anywhere and am not sure about that any more. Now I was wondering what a
> stable Kernel (for UDBs and XLs) might be and what we need to compile it.
> Yes, I have read Dominique Quatravaux's FAQ but this applies to
> 2.0-Kernels only. Can anybody please give some short instructions on how
> to make a stable as-new-as-possible kernel?

Generally you just build the standard source. On some of the kernels
there are screw ups which stop them working on Alpha (I seem
to remember 2.1.81 having problems because someone had screwed up somewhere).
I built 2.1.63 with very little hastle - I think the only problem was the
Adaptec driver which had a function in which had a piece of inline x86
assembler;
the function wasn't called at all - so I commented it out and all is happy.

The 2.1.x kernels still don't have support for the SX (21164PC) in;
but they have support for things which have been around for a while
(if you have an LX, like me, then just choose PC164).
