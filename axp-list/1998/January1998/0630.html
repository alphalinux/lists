Hi,
Ive got a Personal Workstation 433au with 128Mb
ram/Adaptec 2940 scsi, powerstorm(cirrus piggback) graphics
and SRM console firmware. I cant get it to boot, and would
appreciate some suggestions. The system boots Digital unix ok.

Here is what Ive done. (I know DEC unix and Linux/1386 well).

1) read the RedHat install manual
2) read (several times) the MILO faq (surprising eh?)
3) Decided that my machine was a "miata".
4) made a milo.dd milo disk from the RH4.2 cdrom I purchased earlier
(milo/images/miata) and tried booting it.
- result system hangs after "jumping to bootstrap"
5) repeat above with RH5.0 image.
- result system hangs after "jumping to bootstrap"
6) repeat above with different floppy disks amd with latest
stable images from gatekeeper (as in FAQ)
- result system hangs after "jumping to bootstrap"
7) Search axp-list archives for similar experiences/read SRM console
Howto.
8) Try using a boot disk made from the miata image in images/miata.img.
>>> set bootdef_dev dva0
>>> set boot_osflags "root=/dev/fd0 load_ramdisk=1"
>>> boot -file vmlinux.gz

- Success! System loads aboot, uncompresses linux.
Ok, now starting kernel.
Screen goes white on red, and then scrolls up irregularly
printing black on black. (which I guess are boot messages)
9) Repeat above using RH 5 - same result.
A) >>> set console serial
>>> init
(wire up a vt420 to serial port)
Console comes up on the serial terminal ok.
using rh4.2 aboot miata.img
>>> boot -file vmlinux.gz
Graphics screen goes red and black as before.

(Anyone from DEC UK Workstations div. please look away now)
B) Replace power storm card with a generic S3 868 32bit PCI card
Console diagnostics (still on serial terminal) print
"illegal device found in slot 5"
"power down system and remove device"
-- hey DEC, that's not nice. (and it used to work in an
alpha station 200 with DU, btw)
Move S3 card to a 32bit slot. Console diagnostics freeze after
".eb."

Ok, so now what?
People have the 433a running (NT firmware)
Ive seen one message saying the 433au wont work.

Anything dumb I've overlooked? (and thanks for reading this far!).
Any kernel parameters i can pass to at least *see* the messages
that I guess it is printing? This is the first linux/alpha system
in the place, so rebuilding kernels is going to be tricky.

Yrs,
Puzzled, Andy.
