Hello everybody:

I'm trying to compile xemacs-20.3 on an UDB with linux ( RedHat
5.0 Release (Hurricane) and kernel version 2.0.30 ).

I apply the pathes that come with the source and type

configure --with-gcc --with-database=gnudbm --with-x11=yes

in the xemacs-20.3 directory. My computer is identified as an
alpha-unknown-linux.

It starts compiling, get several warnings but most of them
related to the UNEXEC variable being redefined in config.h and alpha.h.
And then after a while i get:

gmalloc.c:1291: conflicting types for `__sbrk'
/usr/include/unistd.h:774: previous declaration of `__sbrk'
make[1]: *** [gmalloc.o] Error 1
make[1]: Leaving directory `/usr/src/redhat/SOURCES/xemacs-20.3/src'
make: *** [src] Error 2

ok... edit the gmalloc.c file and comment out the line where
__sbrk is defined and let unistd.h supply the definition. Make again.
Compiles for a while and when it starts linking it chokes again, and
says:

sysdep.o: In function `end_of_data':
/usr/src/redhat/SOURCES/xemacs-20.3/src/sysdep.c:2483: undefined
reference to `_EDATA'
make[1]: *** [temacs] Error 1
make[1]: Leaving directory `/usr/src/redhat/SOURCES/xemacs-20.3/src'
make: *** [src] Error 2

In the sysdep.c file (and somewhere else as well) there are two
variables edata and etext that are referenced but never defined. However
if i try to define them somewhere it doesn't like it, i've tried placing
the definitions in diferent parts and also in a separate .c file but
nothing has worked... and is driving me up the wall!....

Any ideas???
