>
> I have some applications (xinit, xf86config... ) which I
> can not execute (the message "command not found" is dis-
> played ). The path seems ok and when I try a "whereis" com-
> mand the good path is displayed. File Permissions seem ok.

Try first 'file `which xinit`' on these. Maybe they are binaries
from a wrong architecture (accidents are known to happen)?
If they are ok then what about 'ldd `which xinit`'. Do you have
required libraries and 'ld' can really find them? 'ld' may
have problems if its config file is incorrect. If everything
looks ok then what 'strace' has to say (use -o option or you will
have trouble)?
