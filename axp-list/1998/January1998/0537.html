---Dan Weeks <danimal@blueskystudios.com> wrote:
> Barnett Hsu writes:
> >
> > Hi.
> >
> > On a DEC Alpha EB164 running Red Hat Linux 5.0 plus all updates,
> > but with kernel 2.1.72 (recommended by someone on the list for
> > some problems I was having), I get a bunch of messages at bootup
> > similar to the following:
> >
> > portmap: RPC call returned error 61
> > RPC: task of released request still queued!
> > RPC: (task is on xprt_pending)
> > lockd_up: makesock failed, error=-61
> >
> I have been getting this error too. The only thing I can trace it
too
> so far is some sort of NFS request that is still trying to happen on
> either itself or another host. Beyond that I have yet to see what it
> is. I have it happening on RH 5.0 and 4.1/4.2
>
Based on what someone on the list said, I reversed the order of
the nfs and portmap scripts in /etc/rc.d/rc3.d and the messages
disappeared on bootup. Now I just get them on shutdown.

However, I've now observed that on occasion I get messages
starting with nfs_safe_delete and then inode numbers and so
forth. These messages did not appear prior to upgrading
to Linux kernel 2.1.72.
