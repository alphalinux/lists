> I'm having some weird problems with a few Alphaserver 1000 4/233 at a
> clients site. We're attempting to install Redhat 4.2 from cd, and are
> running into a problem when we attempt to mount the root filesystem on the
> CD.

There's two ways to attack this problem, I believe:

1. don't try to use the CD as root; put the provided "ramdisk.img" on a
floppy and go from there (it's my main technique). This works from either
ARC/MILO boot or from SRM boot.

dd if=/cdrom/images/ramdisk.img of=/dev/fd0H1440 bs=10240

Tell MILO (using the mikasa.img kernel floppy):

boot fd0:vmlinux.gz load_ramdisk=1 root=/dev/fd0

and wait for it to prompt you for the floppy to load into the RAMdisk.

Tell SRM (using the mikasa-s.img kernel floppy):

boot dva0 -file vmlinux.gz -flags "load_ramdisk=1 root=/dev/fd0"

and wait for the prompt.

2. you have to provide a couple of additional bootline parameters in order to
convince the RH install script that it has all the info it needs to use the
CD as root. This probably works only from MILO, BTW...

If I remember correctly, the bootline should be something like:

boot -t iso9660 scd0:kernels/mikasa.gz root=/dev/scd0 bootdevice=scd0
bootfile=kernels/mikasa.gz

The latter (CD root) *is* a bit faster, but I've found the former (floppy
RAMdisk) to be more reliable, perhaps because my memory isn't reliable WRT
what those bootline parameters need to be... ;-}

Good luck.
