>
>On Mon, 29 Dec 1997, mark wrote:
>
>> I just purchased a udb multia 166. After it loads arc it still does not
>> sync with the monitor. I have tried three monitors ranging from vga to
>> multisync. All I get is jibberish. You can make out some of the menus
>> but most of the text is jibberish.
>
>I had this problem too. I had an old 14 inch Svga monitor hooked up, but
>when I booted SRM or NT (pre installed), all it says was 'click' (the
>relais in my monitor). A friend of mine found some info on the UDB
>motherboard on the net. It said that somewhere on the motherboard (it was
>under my harddisk) you could switch a set of jumpers (set all the jumpers
>in the right lane). After I had done this, it worked like a charm.

Here's some info that you might find usefull:

From: "Travis W. Johnson" <tjohnso1@eecs.wsu.edu>

>W6 to W3 Under SRM, these jumper determine the resolution driven to the
>graphics display
>
>W6 W5 W4 W3 Standard Res VF (Hz) DOT (MHz)
>0 0 0 0 VGA 640x480 59.94 25.175
>0 0 0 1 ERGO VGA 640x480 72.79 31.50
>0 0 1 0 SVGA 800x600 60.32 40.00
>0 0 1 1 ERGO SVGA 800x600 72.19 50.00
>0 1 0 0 VESA 1024x768 70.07 75.00
>0 1 0 1 Digital 1024x768 72.033 74.367
>0 1 1 0 Digital 1024x864 60.00 70.00
>0 1 1 1 S3 86C928 1280x1024 60.10 110.15
>1 0 0 0 1280x1024 66.51 119.843
>1 0 0 1 1280x1024 72.556 130.808
>1 0 1 0 1280x1024 72.05 130.08
>
>(This data is from DEC's Multia MultiClient Desktop Service Information
> Part Number: EK-MULTS-IN.)
>
>NOTE: I have not tried to change the SRM video mode just passing on info
>
>For some more sources of UDB info:
> Digital UDB Information -- http://www.brouhaha.com/~eric/computers/udb.html
> http://www.annex.co.uk/systems/udb.html
> Digital DOCs -- ftp://ftp.netbsd.org/pub/NetBSD/misc/dec-docs/index.html
