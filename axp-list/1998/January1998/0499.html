>>> Joakim Roubert said:
>
> When I compiled, I first applied 2.0.30-patch and then 2.0.32-patch ,but
> didn't get successful. So I just applies the later one, and it seemed to
> contain everything that was needed. It worked allright, at least.

The alpha-patches-2.0.30-0.?.gz are meant to be applied to pristine 2.0.30.

The alpha-patches-2.0.32-0.?.gz are meant to be applied to pristine 2.0.32.

Only one set of patches need be applied: for the latest patches against
2.0.30, apply only the latest alpha-patches against 2.0.30 (currently 0.4).
The same goes for 2.0.30 and alpha-patches against it.

According to reports I've received, the alpha-patches-2.0.32-0.1 will apply
successfully against pristine 2.0.33, for those wishing the latest in stable
kernels.
