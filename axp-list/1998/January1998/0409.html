In case you're wondering, no I haven't been ignoring your pleas with
regard to EM86... in fact, I've been working on it late into the night
all this week.

Turns out that the problems encountered all would have happened sooner
or later... the biggest problem involved race conditions in the
signalling
path. Changing the OS can tweak the race conditions such that they
occur
or not...

So far I've fixed the following bugs:

- Fixed setsockopt/getsockopt for real for sure this time 8-)
I actually laboriously flowcharted the kernel's implementation
thereof so I could accurately handle all the cases... This is
not a straightforward system call because its semantics depend
on state information carried in the socket itself.

- Fixed the signal-handling path so that signal handlers are now
run correctly... previously there were cases where I was only
running the first instruction of a signal handler. Bad move!

- Added more state-saving entries to the interpreter interface.
It turns out that some of the condition code information was
not getting preserved across signals. This would manifest
itself as possible incorrect results if a signal handler was
run just before the execution of a conditional branch. Definitely
a hard one to pin down!

I still have some more work to do; I'm still getting a mysterious crash
with
Netscape, but at least it's consistent.

I'll keep you all posted!
