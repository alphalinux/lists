Hi folks,

I am running a LX164PC with a 533MHz Alpha under RedHat 5.0 and Kernel
2.0.32. I have successfully installed em86 (after patching the kernel),
but now I encounter some problems. Some programs, that I copied to test
em86 (e.g. gzip) work fine, but programs that need X always crash. When
starting them, I get a couple of error messages, but nonetheless the
program starts (as an example: xarchie, which runs without problems on
an Intel Linux machine):

.//xarchie: '/usr/X11R6/lib/libXaw.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libXmu.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libXt.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libSM.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libICE.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libXext.so.6' is not an ELF executable for
386/486
.//xarchie: '/usr/X11R6/lib/libX11.so.6' is not an ELF executable for
386/486
Warning: Unable to load any usable fontset

After this, The first 'real action' (in this case e.g. requesting
information) causes the program to hang up with the error message:

EM86: axp_dlopen of /usr/x86/lib/libc-assist.so failed: libresolv.so.2:
cannot open shared object file: No such file or directory

What is going on here? Can anybody help me out? Thanks in advance,

Dirk !!
