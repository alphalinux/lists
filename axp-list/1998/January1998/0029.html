On Thu, 1 Jan 1998, Steven Filling wrote:

> Apologies for the moderately off-topic post.
> I've just transferred a bunch of web pages from an NT system to an alpha
> running apache 1.2.4 on RH 5.0, and am having a great deal of trouble
> getting things back up. For some reason the .htm extension of the legacy
> pages causes those pages to be displayed as text rather than html. This
> seems to be alpha specific, as I've transferred sample pages to both i386
> and Sparc systems running apache and things display fine. Since there is a
> significant amount of cross-referencing embedded in these pages, I'm
> hoping not to have to rename all to .html

This has absolutely *nothing* to do with the server being an
alpha. Go edit your mime.types. This file defines your basic extension
to file type mappings. Mine has the line (included with many others)

text/html html htm

if yours only has

text/html html

then .htm files will not be treated as html.
