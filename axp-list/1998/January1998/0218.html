tom@kednos.com said:
> A cross-compiler is just a program that reads an input file and
> produces an output file. The fact that the output file is object
> code and maybe for a 64bit architecture is irrelevant.

Yes, one would think that. In practice, though, there are a lot of cases
where expressions need to be evaluated from the source being compiled,
and the compiler (gcc, I presume) uses native arithmetic to do it. And
believe it or not, gcc has bugs in it.

Sure, this is not "correct" but is is done for performance reasons. This is
why 64->32bit cross compiles are more likely to work then 32->64bit compiles.
Gcc is getting better, though. I'm sure if you (or anybody) finds a bug
or a fix, the gcc and EGCS people would love to hear about it.
