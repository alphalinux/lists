I don't know if this is Linux/Alpha specific but I have a problem on
L/A RH 5.0 that doesn't occur on any other UNIX we have. We have a
filesystem mounted and within it we do builds for our internal apps.
The make system touches some files of zero size. If the file does not
exist everything goes fine and the file is created. If the file
exists and looks like so:

-rw-rw-r-- 1 devel1 rnd 0 Jan 21 12:07 ALL_DEPS
-rw-rw-r-- 1 devel2 rnd 0 Jan 21 11:57 ALL_IIOS
-rw-rw-r-- 1 devel1 rnd 0 Jan 12 19:15 ALL_IIOSg

and user devel2 tries to "touch" ALL_DEPS then we get the following:

touch: ALL_DEPS: Operation not permitted

Both users are in the rnd ggroup and the directory's permissions are
775 with group rnd. Does anyone know why this is happening?
