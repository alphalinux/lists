On Sun, 4 Jan 1998, Dave Gilbert wrote:

> Hi,
> Has anyone got 2.1.77 to run? It fsck's on mine and then strange things
> start to happen - like a lot of the echo'd console messages from the rc
> scripts don't appear and then it just hangs after 'initialising random
> number generators' (I said a LOT of the messages don't appear - some of
> them do - like the random number message!).

The exact same thing happens for me. Hitting ctrl-scrllock and shift-scrllock
show that there are just three or four processes running. The kernel
spawned stuff, init, rc.sysinit and kerneld. This is on RH 4.2. same happens
on 2.1.76. The Changes file said that you need modutiles-2.1.71, which I
got and found out you need a 2.1 krnel to compile. Since i have 2.0.32,
I cannot do that. I made a non modular kernel, which failed at the same
stage.
Commenting out the random stuff or kerneld isnt helping. It seems it's
stuck trying to end the rc.sysinit script and goingto runlevel 5. Upgrading
sysvinit didn't help me. Booting in single user mode won't help me either.
Adding a /bin/bash call at the end of rc.sysinit, AFTER the call to /dev/random
gave me a shell. But when i typed 'lsmod', i get the same hang situation,
probably cause the shell crashes (causing rc.sysinit to end, causing the
hang again).

I briefly spoke to alan, who said to make sure AF_UNIX is compiled in.
But at the time, I had already rm'd the kernel tree. I'll compile a new
kernel at some later time to see if that was my error.
