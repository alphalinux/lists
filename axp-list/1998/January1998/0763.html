>>> Stefan Schroepfer said:
> Jay.Estabrook@digital.com wrote:
> >
> > I *think* that those messages just indicate that the BIOS code on the
> > Adaptec card has been run through the x86 emulator that is built into
> > the consoles (ARC/AlphaBIOS/SRM).
> >
> > This is the way VGA BIOSes have been run for a while; it's only recently,
> > IIRC, that the SCSI BIOSes have joined them...
>
> Hmm, I think that this cannot be a generally true answer.
> The Intraserver adapters show messages (and are configurable)
> with AlphaBios (on the LX), but not with ARC (on a PC164).

Ah, yes, you're right, I should have qualified that statement to be specific
to AlphaBIOS, but then, we *were* talking about AlphabIOS originally... ;-}

I believe it's the same emulator code that is included in the latest versions
of *all* the consoles (ARC/AlphaBIOS/SRM), but it may take more than just the
emulator code to perform the appropriate initializations needed by a SCSI
controller (as opposed to a VGA, IMHO).

So, it most definitely does depend on whether you're running AlphaBIOS;
it may be rather hard to back-port into ARC. Whether that's really because
of the emulator versions employed by those consoles, or in the specific
console code itself, I don't know.
