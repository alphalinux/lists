Hello:

I installed RedHat 5.0 (Alpha) on a large SCSI drive, 2.0 Gb. It
is partitioned as follows:

Device Boot Begin Start End Blocks Type Filesys
/dev/sda1 1 1 2 2032 DOS 16-bit <32M (MILO img)
/dev/sda2 3 3 334 339668 Linux native /
/dev/sda3 335 335 463 132096 Linux swap (swap)
/dev/sda4 464 464 2047 1622016 Extended
/dev/sda5 464 464 1715 1282032 Linux native /usr
/dev/sda6 1024 1716 2047 339952 Linux native /home

I was a little concerned about this, because the "v" (verify) flag
shows that partition 5 overlaps partition 6 -- because partition 6
"Begin"s on block 1024. However, this is much the same type of
configuration as produced by Disk DRUID (but I wanted better control
over where things went so I redid the table by hand with fdisk) so
it should be okay, right?

Anyways, things looked pretty much "okay" after installing and booting
RedHat for the first time. I don't have X-Windows working (I opened a
separate ticket for that) but I do not know if the problem is related, or
is completely separate.

I then went to create an account for myself. I typed "adduser" and
got:

SCSI disk error : host 0 channel 0 id 0 lun 0 return code = 2
scsidisk I/O error: dev 08:05, sector 786458
Kernel panic: EXT2-fs panic (device 08:05): ext2_read_inode: unable to
read i-node block - inode=98339, block=393229

Message from syslogd@flant at Sat Jan 13 14:30:42 2018 ...
flant kernel: Kernel panic: EXT2-fs panic (device 08:05): ext2_read_inode:
unable to read i-node block - inode=98339, block=393229

And then things hang up. I can switch to other consoles though, and try
other commands -- they work. But if I try "adduser" that console hangs
too (no messages). I tried using "kill -9" on the processes that were
hung up but they refuse to die.

This seems rather serious -- I created the partitions with
bad-block-checking and didn't see any problems. AND, I was using the same
machine for Windows NT for several months, never a problem with the drive.

Any ideas?

Thanks...
