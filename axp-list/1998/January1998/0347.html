David Gilbert wrote:
>
> > What does -mieee do? Does it contain a library of exception handlers or does it
> > turn the
> > exceptions off. (I would kind of expect the kernel to handle denormals, although
> > NaN's and Infinities
> > I can imagine the user getting).
> Specifically, this tells the compiler to generate floating
> point instructions with the /s (software completion) option.
> (eg. multsu $f10, $f11, $f12 # multiply registers $f10 by $f11
> (t-floating format) with software completion, underflow
> enabled). If either input contains an invalid value (denormal,
> NaN, ...) or the result underfloaws or is invalid, generate a
> trap. The kernel should contain the appropriate code to perform
> the fixup. (Note: I am not 100% sure of Linux. I am responsible
> for the Digital Unix kernel floating point completion).
>
> So what happens without -mieee? He is seeing traps without -mieee.

The point is that -mieee allows code to gracefully handle NaNs and
denormalized numbers. Without it, the floating-point exception handler
cannot produce a reasonable result and let the program continue. A
reasonable result would be a NaN if one of the operands is a NaN, and it
would probably be a 0 if a denormalized number is multiplied by
something.

You might have noticed that -mieee implies some other options which
increase the granularity of exception context information to the
instruction level. I don't know what other effects it has on the actual
generated code.
