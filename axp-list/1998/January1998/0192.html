A cross-compiler is just a program that reads an input file and produces
an output file. The fact that the output file is object code and maybe
for a 64bit architecture is irrelevant.

On Thu, 8 Jan 1998, Hugo van der Kooij wrote:

> On Wed, 7 Jan 1998, Jim Paradis wrote:
>
> > David Alan Gilbert wrote:
> >
> > > Hmm - you SHOULD be able to; I'm pretty sure I've seen a comment in the GCC
> > > source
> > > which says that it can't do certain optimisations when going from a 32 bit
> > > machine to a 64 bit machine; so I presume that means that you can do
> > > that direction but not as efficiently.
> >
> > True, but there's another wrinkle: you can get away with this on some
> > architectures but not on others. The Intel architecture, in particular,
> > is so register-poor that when you combine registers to make 64-bit
> > pseudo-registers, you don't have enough to do even basic operations.
> > That's why you can *build* a 64-bit Alpha compiler on an Intel system,
> > but when you try to run it on any but trivial sources it'll frequently
> > die with an "illegal register spill".

This is a bug in the cross-compiler. You have no need to build 64bit
"pseudo-registers" in the cross-compiler. If you do, you need to rebuild
the code generator.
