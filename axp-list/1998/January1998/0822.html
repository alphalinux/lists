>
> Hey all:
>
> Okay, I am running RH 5.0 on a PC164 setup. When I log in from
> the console or via rlogin/rsh to the system and I am root I have no
> problems, everything is very snappy. When I try to login on the
> console as a user it hangs forever and when that same user tries to
> rsh/rlogin to the system they get repeated error messages saying:
>
> do_ypcall: clnt_call: RPC: Timed out
>
> and after about 10 of those the person is logged in, but it takes
> forever to get there. YP appears to be working correctly as I can cd
> to any ~user directory and I can finger the user on the box to get the

> started happening?
>

An interesting development. I was messing around with this trying to
fix it. When I changed to a slave server for NIS any user can log in,
albeit slowly, without getting the RPC errors. I also upgraded to
the latest glibc, ypbind, amd, and portmap off of the Red Hat site. The
RPC call didn't go away until I changed the ypserver.
