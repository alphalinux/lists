Greetings.

My group is purchasing an Alpha box from SWT
which will come pre-configured with X11R6 under
Red Hat Linux.

I know Red Hat ships a Motif for Linux/Alpha, but
I have not yet been able to find an implementation
of OpenGL (commercial or otherwise) for Linux/Alpha.

Metro Link OpenGL is apparently Intel-specific. From
what I've seen, it is possible to compile Mesa for
Alpha systems, but I'm not sure Mesa will have the
functionality I need.

Basically, I need to be able to render in overlay
planes so I can display images in the 0-level frame-
buffer and draw on top of the images without
having to redraw the images. I need multiple colors
in the overlays, which is why I'm not using XOR
drawing.

I'm not sure how Mesa deals with hardware-specific
functionality. It may be that Mesa only deals with
whichever visuals are obtainable from the X server,
in which case all I should have to do is use an overlay
visual for my rendering context, and Mesa won't care,
but as I said I'm not sure how this will work.

If anyone out there knows about other OpenGL
implementations for Linux/Alpha, I'd appreciate
hearing about them.

Thanks in advance ....
