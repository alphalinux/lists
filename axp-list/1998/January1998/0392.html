>>> Matti Aarnio said:
>
> It seems to me that PYXIS driver is indeed unrealiable
> to do PCI configuration space register lookups.

Well, I must say that it's been pretty reliable for me under 2.0.xx sources,
with pretty much the same hardware and appropriate alpha-patches, although
I've never tried one of those SMC boards.

However, there have been a couple of problems found with the latest release
of alpha-patches-2.0.30, namely the 0.4, and, since the RH 5.0 AXP kernel
sources are based on those, them as well.

1. kernels wouldn't run correctly on the newer MIATAs, due to incorrect
PCI-PCI bridge handling code. (by "new", I mean if the PCI-PCI bridge
shows up as device 20, rather than device 8, in /proc/pci).

2. kernels wouldn't run correctly on 600MHz MIATAs, due to incorrect PCI
bus probing code at that speed.

Both problems have been fixed, but the new alpha-patches hasn't yet been
released; soon, I promise...

> To review the PCI programming details et.al. of pyxis.c file,
> I tried to figure out which chipset the machine has and if I can
> find documents for it, but I could not figure out which Digital
> Semiconductor's chip id refers to PYXIS.

Docs for PYXIS have not been released to the public, AFAIK; part of the
problem is that the chip is not actually made by Digital Semiconductor, but
was designed by (another part of) DEC and founded by IBM, IIRC. I believe it
has a part number of 21174, FWIW.
