the byte benchmark suite, on

Linux 2.0.30 alpha unknown (RedHat 5.0 kernel, built with egcs-1.0.1)
OSF1 V4.0 386 alpha

Fairly identical machines -

Linux: AlphaStation 255/300 w/ 192M RAM, nbench compiler = gcc v2.7.2.3
OSF1: AlphaStation 255/300 w/ 128M RAM, nbench compiler = gcc v2.7.2.1

Bitfied, fp emulation, idea and huffman are similar. Assignment,
neural net, LU decomp and the sorts are all of the same order as each
other but what is going on with fourier test?

The fourier test ran (4040.8/765.8) 5 times faster under OSF1 than
Linux. Does anyone know why this happens? It completely wipes out any
floating point advantage that Linux/Alpha may have over Linux/x86
platforms. (my users want fast fp...)

btw. results are very similar with egcs-1.0.1 compiled nbench.
