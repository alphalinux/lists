I'm afraid MILO is going to have a hard time figuring out what the cycle freq
needs to be set to. AlphaBIOS probably knows but I don't know how MILO could
get that info from it, and SRM usually sets the field accurately (and no MILO
is needed using SRM console).

Sorry if this is naive, but doesn't alpha have a cycle counter? On Pentium, I
have done things like
cnt=RDMTR();
sleep(1);
cnt = RDMTR() - cnt;
where RDMTR is an assembler inline reading the pentium cycle counter register.
For a 166 MHz pentium this returns pretty close to 166,000,000.
