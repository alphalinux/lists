There are at least two issues here:

- You must use "make install" (or equivalent) to actually copy
the g77 Info documentation to the appropriate Info directory.
(Note that, for this and other reasons, having already configured
gcc+g77 with the `--prefix=/usr' option is best for systems that
don't have their own "native" compiler other than gcc -- like
GNU/Linux.)

- You must update the "master" list of Info files to include an
entry for g77. This file is usually /usr/info/dir. Doing
"make install" *should* do this automatically, but it doesn't
yet in g77's case, because g77 doesn't yet use the `install-info'
command, which is part of the Info subsystem. I hope to change
this soon.

So, after you build gcc+g77, to install the products *and* the
documentation, use "make install" in the build directory. Then
edit the `dir' file in your system's Info directory to add
the appropriate entry for g77 in the appropriate category
("programming", I would think).

I'm pretty sure egcs is packaged with the Info files intact,
i.e. not requiring you to build them yourself. If that's
not true, you need a recent version of `makeinfo' and such
(see gcc/f/INSTALL for more details). However, I still have
not yet tried egcs myself (strange but true).

Otherwise, `info -f gcc/f/g77.info M GLOBALS' should suffice
to find the pertinent portion of the g77 docs, assuming `gcc'
is the name of your gcc+g77 source directory.

Hope this helps. Packaging is still not something we advertise
as being "excellent" in the GNU world, I'm afraid. ;-)
