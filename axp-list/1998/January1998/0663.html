>>> "Jon E. Mitchiner" said:
> For some strange reason whenever the kernel has been
> installed, it normally freezes and/or crashes right after the Ramdisk driver
> initialization. So I had removed FDD support in the kernel, which didn't
> work, so I removed the ramdisk support, and that solved the boot-up problems
> I had. I have installed 2.0.32/2.0.33 source and ran the patch from
> gatekeeper (2.0.32-0.1) patch so I'm confused whether this is a known
> problem and/or the best solution to fix this.

This is not a known problem, and those patches have been out for a while
with no similar problems reported.

Do you have a working 2.0.30 kernel that you built yourself? You could start
by looking at the differences between the ".config" files of the two, see if
there's anything that looks odd.

Oh, and you are using "make config", right? I don't trust "menuconfig" or
"xconfig" to get all the Alpha-related options correct, so this could be
your problem, if you used one of those...

> The other problem I have is with Xwindows and my Matrox Millennium II --
> sometimes when I try to run startx, the machine completely dies -- it goes
> to a black screen and sits there forcing me to do a hard reboot. This
> happens about every other time that I run startx.

I've had this happen a few times, too; XFree86 3.3.1 support for Mill II was
deemed "EXPERIMENTAL and largely UNTESTED" (in their own words :-). They are
working on a 3.3.2 for sometime not too far in the future, which should fix
a number of problems, among them this one, IIRC. In the meantime, you could
see if running at 16bit or 32bit depth is any better, or try putting in the
"no_accel" option (which will slow it down, but may be more acceptable than
crashing... :-).

Patience will be rewarded, I believe... ;-}
