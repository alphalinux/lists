> What about 'rdate'? Does it not enable one to get time information
> from one of the internet atomic clock sites?

Yes, rdate wil "get time information". So will timed. So will reading
a CMOS clock. So will reading the wall clock. All will *set* the
time with accuracies (assuming careful work) of a second. If that's
all you care about, then use whatever and be happy.

However, the issue of time is *lots* more complicated than getting time.
XNTP provides (amongst other things)

1) monotonicity (e.g. time will not shift backwards, possibly
causing database inconsistencies, or duplicating cron driven
tasks, or failed makes on network file systems, or ...)

2) "best guess" estimates of time averaged across several
servers to improve accuracy.

3) continuous error estimates and adjustments.
