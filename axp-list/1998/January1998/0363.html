Stefan Traby <stefan@sime.com> wrote:
>
> gcc handles NAN well under i387+ because FPU-exceptions on an intel
> can be masked. I don't know it, but your posting makes me think
> that masking is not possible on AXP, so this info-file is simply
> outdated because it depends on FPUs where exceptions may be masked.
> Is this assumption correct ?
> If it's correct than "-mieee" means libm installs a SIGFPE handler itself and
> fakes errno, but without "-mieee", SIGFPE is passed to the app ?
> I'm in panic for now. If SIGFPE cannot be avoided at all
> (ok, it can, SIG_IGN, -mieee), fp-pthreads programming must be near to hell.)
This assumption is NOT correct. The Alpha has a fast floating point mode which is the default. The -mieee causes the compiler to generate floating point instructions with the /S modifier. It is generally the kernel that handles software completion, not the math library.
