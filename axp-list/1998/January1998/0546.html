If I were to attack this, I would first ask the question "what format is
the binary in?" Answering this question allows one find a way to load the
binary in an address space.

Next, one would need to figure out all the interfaces to the external
environment, and how those interfaces are accessed, and guess a way to
provide them.

It seems to me that NT, being a proper kernel, may be emulatable by Linux.
One would guess that an application makes system calls that can be implemented,
etc., much the way OSF/1 is done.

Just my wild speculation, here.
