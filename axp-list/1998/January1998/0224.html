> I posted a message about this a few weeks ago, and I'd certainly like to
> see it fixed. These are defined in <resourcebits.h> using constants it
> expects to find in <asm/resource.h>, RLIMIT_CPU, RLIMIT_FSIZE, etc.
> However, when using Alpha patched kernel sources, these constants are no
> longer defined in <asm/resource.h>. Instead it defines
> __KERNEL_RLIMIT_CPU, __KERNEL_RLIMIT_FSIZE, etc. As a result, anything
> that uses <sys/resource.h> will fail. I modified <resourcebits.h> to be
> compatible with the new namespace and I havent had any problems since.

Yes, you did, and I think I posted a message about this as well, indicating
it is NOT A GOOD THING to replace the stock RH kernels sources in the
/usr/src/linux directory with other sources that have been alpha-patched.

Red Hat is *very* careful to (try to) make the headers consistent between
Alpha and INTEL versions, so that packages will compile on both.

The alpha-patches-2.0.30-whatever DO NOT try to maintain this consistency,
leading to the problems mentioned above.

Again I'll say, PLEASE do not replace /usr/src/linux with alpha-patched
sources! Build your custom kernels either from the RH versions there, or
move to someplace like /usr/local/src to unpack, alpha-patch, and build
your custom configured kernels.
