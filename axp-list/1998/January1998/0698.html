Not really; when you load MILO, you replace the NT "firmware" with different
stuff, much more in tune with running a UNIX-style OS.

Let me explain (you knew I would... :-):

ARC, AlphaBIOS, SRM, *and* MILO are what I'll call "consoles". They are made
up of (roughly) two distinct parts, the user-interface/device-support code,
and PALcode. PALcode (Privileged Architecture Library) can be thought of has
high-level microcode, tailored toward what the OS and user code levels want
to do.

ARC and AlphaBIOS provide PALcode tailored to run NT; SRM and MILO provide
PALcode tailored to run UNIX-like OSes. That's not to say that the LINUX
kernel couldn't be modified to depend on NT PALcode, just that it would be
a very bad fit and not easy to do, IMHO.

Again, the PALcode reference is in the Alpha Architecture Reference Manual at:

http://ftp.digital.com/pub/Digital/info/semiconductor/literature/dsc-library.html
