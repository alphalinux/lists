Jim Paradis wrote:

> True, but there's another wrinkle: you can get away with this on some
> architectures but not on others. The Intel architecture, in particular,
> is so register-poor that when you combine registers to make 64-bit
> pseudo-registers, you don't have enough to do even basic operations.
> That's why you can *build* a 64-bit Alpha compiler on an Intel system,
> but when you try to run it on any but trivial sources it'll frequently
> die with an "illegal register spill".

Are you saying that the problem is that 'long long's on Intel just don't
work correctly?
I can imagine that if the compiler was really broken with these you could
have problems *building* an Alpha compiler, and that if the implementation
of these was duff then you could have problems using large values in the
resulting compiler, but I don't understand why these would screw up something
like register allocation in the compiler built on Intel.
