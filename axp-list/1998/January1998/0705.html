Hi,
This is mostly for anyone else with the same problem searching
the archives in the future. I know its bad form to follow up ones own
posts, but I didnt find this intuitive, and it stalled me for 2 days.

If you have a Digital Personal Workstation 433au, and want to
install Linux.

First, check you have a supported VGA card. I used a S3 Trio 64
from a pc. Second - at the SRM console;
>>> set console graphics
>>> set os_type NT
Power down
When you restart you should get messages about NVRAM data being
invalid. Answer yes to the rather scary messages.
It will hang after the scsi bios initialisation for a long time.
Be patient. Eventually you will end up at the AlphaBios.
Congratulations - You have just converted a $8000 PWS 433au to a $4000
PWS 433a. Install linux as in the install-guide.

Networking seems a little flaky, but then thats probably due to
the ancient cogent 21040 ethernet adapter I put in when I couldnt get
the on board 21142 working (no MII interfaces found).

As always YMMV.
