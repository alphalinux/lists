>Hi, I use to follow the evolution of my Fortran programs redirecting the
>output. The system does not write the buffer on disk until the amount of
>information reaches certain size or at a regular interval, wich looks a
>long time for me. i would like to do it anytime. How can I force the
>system to write on disk? if I well understand, the update daemon is
>started in inittab and it's risponsible of this. I tried with sync and
>update but it doesn't do what i am expecting.

Have you looked into whether your Fortran compiler supports
"CALL FLUSH(unit)"? g77 does, for example. Anyplace in your
code where you want to make sure the output is flushed, in a
"semi-portable" way, insert this call, with unit being the output
unit number. (Note that I think g77's implementation of CALL
FLUSH() happens to always flush *all* units regardless of number,
but that this is probably going to be changed someday, so it is
best to explicitly flush just the units you care about...but read
the docs to be sure.)
