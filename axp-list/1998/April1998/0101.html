On Tue, 14 Apr 1998 Jay.Estabrook@digital.com wrote:

>
> >>> "Michael J. Donahue" said:
> >
> > FWIW, I have found that I can get 115200 if I use 'setserial' with
> > spd_vhi, and then don't specify a speed in my dip script.
>
> >From the "setserial" manpage:
>
> spd_hi Use 57.6kb when the application requests 38.4kb.
> This parameter may be specified by a non-privileged
> user.
>
> spd_vhi
> Use 115kb when the application requests 38.4kb.
> This parameter may be specified by a non-privileged
> user.
>
> So, it looks like you *should* set the speed in the dip script to 38.4K.
> Of course, you may not need to if this is somehow the default... ;-}

Awkwardly enough this is not a requirement for Intel based systems. So
does the Alpha handle the serial port differently? Or does the XLT contain
other hardware for the serial interface?
