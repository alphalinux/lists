> As all libraries detected by ldd seem to be from RH 5.0 set I am not
> sure how did you managed to get this "ld.so.1" there. Smells like you
> managed to misconfigure something badly. Do you have some links going
> nowhere in your /lib/? And how did you managed to include a reference

*I* didn't do it. These dependencies are there. And that's not the only
program so confused. I did notice that there was a ld.so.1 in /lib that
was a link to ld-2.0.6.so, even after I upgraded to 2.0.7. Of course,
ld-2.0.6.so doesn't exist. Other then that, all the links seem OK.

[root@icarus /lib]# which xpdf
/usr/bin/xpdf
[root@icarus /lib]# ldd /usr/bin/xpdf
libX11.so.6 => /usr/X11R6/lib/libX11.so.6 (0x0000020000006000)
libXpm.so.4 => /usr/X11R6/lib/libXpm.so.4 (0x0000020000204000)
libm.so.6 => /usr/lib/libm.so.6 (0x0000020000318000)
libc.so.6 => /lib/libc.so.6 (0x0000020000448000)
ld.so.1 => not found
/lib/ld.so.1 => /lib/ld-linux.so.2 (0x0000020001000000)
libXext.so.6 => /usr/X11R6/lib/libXext.so.6 (0x000002000062e000)
[root@icarus /lib]# rpm -q xpdf
xpdf-0.7-2

Adding a symbolic link seems to fix it for now.
