What? This is an old loader from the previous distribution.

> /lib/ld.so.1 => /lib/ld-linux.so.2 (0x0000020001000000)
^^^^^^^
And this is the current one (which should be able to load, at least
in theory, older libraries as well - if required).

> Ack!

As all libraries detected by ldd seem to be from RH 5.0 set I am not
sure how did you managed to get this "ld.so.1" there. Smells like
you managed to misconfigure something badly. Do you have some links
going nowhere in your /lib/? And how did you managed to include
a reference to an old loader in your executables?

Loaders themselves are statically linked, for obvious reasons :-), so
missing libraries are not a problem for a loader.

If you happen to really need both current and old libraries (some
commercial stuff in a binary format requires these, for example) then
you will be well advised to separate them in their own directories
and to adjust your /etc/ld.so.conf.

If a new loader indeed does not have hiccups with old libraries (I did
not try that myself) then doing, in lib, 'ln -s ld-linux.so.2 ld.so.1'
should take care of that. Otherwise you need a copy of ld.so.1 itself.
This still does not address the quoted example which looks like
some installation error. Do you have a mixture of old and new libraries
by any chance?
