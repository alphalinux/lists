David Hamm wrote:
>
> Whenever I boot this PC164LX I'm likley to get the wrong time and date. I
> re-installed RedHat several times due to this problem and finaly set the date
> to 4/14/2078 current time +2 hours to get the correct system time. Is their a
> fix for this madness? If not I'll be forced to move to another os for this
> server.

There are 2 fixes depending which of two problems you are suffering from:

1) The date is always 20 years ahead - if you see this then you need
to set the 'clock' program to use the Arc BIOS mode; its a flag
to 'clock' which is called somewhere in your rc files; I think you
can change it from one of the RedHat panels.

2) Random dates/times when you boot up - get a new version of the clock
binary; sorry can't remember where.
