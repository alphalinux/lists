> archive download contains support for my arch (CONFIG_ALPHA_LX164) but
> another won't (the latest (2.0.33) of course.

This is a continuing sore spot -- can someone please write something
for the FAQ? BTW, Harry, I believe that the Right Way to build a
kernel for a PC164LX and 2.0.30 is to use "make config" and use PC164
as the machine type; be sure to get rid of broken drivers like
BusLogic. That's how I made mine, and all 66 of them boot...

I think someone posted a cookbook a while back for building 2.0.33 but
it would be nice if there were a source rpm for it like the i386
source rpm. Ah well.

In addition, I could use a clue about 2.1 series kernels. I am running
2.0.30 and have various applications which crash the machine with
floating point exceptions. I hear they're much better in 2.1. So which
2.1 should I run, and can we put a nice cookbook for how to compile it
in the FAQ?
