The MILOs shipped with RH 5.0 (based on 2.0.30) ought to boot both older and
newer kernels fine, including the 2.1.xx development ones.

> 2.0.33, when loading with the default-shiipped-with-RH5 milo, goes thru the
> boot process fine and unzips the new kernel then hangs, while the 2.0.30
> kernel proceeds fine.

That's the pre-built 2.0.30 kernel from RH 5.0, right?

> I had thought it was this same problem - that (as in
> the manual/HOWTO - see I did read them :) milo had to be built vs the
> target kernel version.

NO! THIS IS NOT TRUE AND HAS NEVER BEEN TRUE!!!

The MILO version indicates which kernel it was BUILT with, not the only one
it will boot.

> If this is not the case, is there an explanation about why it hangs like
> this? I thought that there might be a mem offset difference that would not
> be passed correctly unless the two were matched, but this is not the case
> apparently...

I'd be much more likely to believe it's a configuration problem with your
custome-built 2.0.33 kernel. Care to share the .config file so we can see
what might have been gone wrong?

Oh, and please ONLY USE "make config" and not either of "make menuconfig" or
"make xconfig", as the latter two often don't get all the options correct.
