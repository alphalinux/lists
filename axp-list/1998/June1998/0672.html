On Fri, 15 May 1998, Andrew Smith wrote:

> ./java: error in loading shared libraries
> libc.so.6.1: cannot map file data: Bad file descriptor

You said that this was for RH5 but you're on 4.2? Download the glibc2
RPMs from RedHat and install them. Hopefully that's all you need, but
that 'bad file descriptor' notice is a little unnerving.

> question: Will it be easier to update to 5.0 or debug this?

You can have both the new libc and old libc around at the same time. I
would just make sure that libc.so (if it exists) points to the old one.

I must warn you that I haven't tried this though, and things could get
messy, though they *shouldn't*.
