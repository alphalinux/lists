Michal Jaegermann wrote:
>
> >
> > The second is what I think larger system problem, as apparently
> > the codes in connection.c, and WaitFor.c assume select(2)
> > semantics with components of the fd_set's array to have size of
> > 32 bits
>
> >From what I see Xpoll.h header is broken in the first place.
> It overrides macros, like FD_ISSET, defined elsewhere in header
> files and does that in a manner which is incorrect on Alpha.
> Patches by Dave are of a "ho-hum" kind. "1L" happens to work at
> this moment but we have seen the same bug before on other occasions
> and there is a fd_set typedef to deal cleanly with the issue.

Yes I agree. What is the correct (as far as Posix?) place for
fd_set etc. ?

Incidentally I think you'll have a hard time convincing the X
guys that they should change one of their headers. There is a glaring
use of a #define where a typedef should be used in one of the
headers which can cause havoc; I mailed them about this
a few years ago and got a response which basically said that the
problem was known about since earlier systems (W?) and that
basically they can't change it now if they wanted to because
it had gone through standardisation mechanisms.
