>>> Stefan Reinauer said:
>
> I just checked out /proc/cpuinfo, and I noticed
> that the system is recognized as an Evaluation Board
> EB164. Actually, I have an 164LX Board. I know, these
> two boards are very similar, but is it impossible to
> find out, which one is built-in or did just nobody do it
> yet?

Your situation is, I believe: using an older MILO with a newer kernel.

Newer MILOs provide a non-zero "system variation" field, which, in your
case, that kernel would report as "LX164".

> Generally, how can one determine, which board is built-in
> via software? it could be fun to have one big "milo" for
> all system types that takes the values it needs from
> his database, depending on what board it recognizes..

Determination is not done dynamically; the lowest level code, either SRM
or ARC/AlphaBIOS, is built knowing only the hardware on which it will run.

SRM console provides a "generic" kernel with information, in the form of the
HWRPB (HardWare Restart Parameter Block; see the Alpha Architecture docs for
the format/contents of this data structure). From this info, dynamic
"config" of a generic kernel can take place; Digital UNIX's "genvmunix" will
run on *all* the Alpha platforms for which there is an SRM, from laptop to
"mainframe".

ARC and AlphaBIOS, however, are different, and don't provide an HWRPB, as
such; that task is taken on by MILO, which is why there's one for each
platform.

And yes, a "generic" MILO would be great, but there are technical issues of
significance that would need solving first, primary among them being the
PALcode that would be utilized (see the Alpha Architecture docs for more info
on this as well).
