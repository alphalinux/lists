> I'm still having problems (missing 'net-pf-4' and 'net-pf-5',

Read /usr/src/linux/Documentation/modules.txt. This stuff is at the
very bottom. :-)

Actually reading some documentation before embarking on some new
adventure is an idea with a merit - even if shunned by some for
reasons not entirely clear.

> and a
> complaint about unresolved symbols in 'ncr53c8xx.o',

Is this, by any chance, your controller? If yes, then you try to use
it built-in and load later as a module, or you would not be that far.
If not, then then this makes sense only if you have two controllers.
Trying to load a module "by hand" usually gives a pretty clear indication
what it is unhappy about. It happens from time to time that some symbols
were not exported in kernel sources and a minor edit is in order.

> What I'd like to do is take the
> kernel that was installed by default from the RedHat CD, remove the
> modules I *don't* need, and install the one I *do* need (SCSI tape
> support).

Compile and install **only** modules you need - and that is about it.
But with scsi tape you will not gain much. :-) Some tape drives do
not like their drives to show up and disappear dynamically. So Red
Hat plays it safe and in startup scripts loads such module with
'insmod' for it to stay. You may compile tape support into a kernel
as well and it will actually take less memory. Or, if your drive
does not mind driver games, you may modify scripts in /etc/rc.d/.

> Maybe if I had the original .config file that RedHat used to create the
> default kernel

Kernels distributed by Red Hat on Alpha do not use modules.
