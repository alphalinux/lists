This should have work if nothing went screwy in the process and your
diskette did not have have any bad bits and if /dev/sda6 was big enough
to hold the whole image (and they you would likely to waste some disk
space unless /dev/sda6 was **precisely** of a size of a floppy).
You just cloned the whole file system from a floppy.

> OSLOADER=\linload.exe can't be found

Why you will not, instead, use a program which runs from a menu entry
on ARC to recreate and format that partition? After that use either
'mcopy' to copy what you need from a floppy or mount both and use
a regular 'cp' to do that job. You will have guarantee that this file
system looks exactly like ARC wants to find it. MS-DOG has too many
undocumented variations to be sure any other way.
