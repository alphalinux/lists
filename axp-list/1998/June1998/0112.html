On Mon, 22 Jun 1998, Tim Jenkins wrote:

> long foo;
> int *bar;
>
> foo = (long) bar;
>
>
> Is this an acceptable solution to making programs 64 bit clean and is it
> portable across 32 bit and 64 bit platforms?

Basically, no. If you need a pointer, use a pointer. If you need an
int/long use an int/long. The only really portable way to do this is to
union them and use it as such. The only time it's really "safe" to use
casts that change types (eg, excluding signed/unsigned) is when you have a
variable argument list or void pointers. The bummer is that function
pointers can't be converted into void pointers portably. :(
