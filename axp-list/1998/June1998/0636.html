Hello,

Has someone experience with the bttv driver under Linux Alpha? I just got a
Miro PC-TV tv tuner/frame grabber and got the bttv-0.56 driver (I ve just
downloaded the 0-58 version, have to test it this week end), and tried to have
this work, it seems that the code isn t 64 bits clean since I got warning
about casting data from incompatible types in one module (bttv.o, the others
(videodev.o and i2c.o) seemed to compile fine and can be loaded, while the
bttv.o hangs on a sigsegv). Looking closely at the places where the warnings
where issued, it looks like it tries to affect a pointer to a double word and
since I guess the memory addresses are 64 bits on the alpha, I imagine this
can cause trouble), I started trying to fix that, but before I spend my week
end doing that I think it could be valuable to ask if someone has already got
this working on his system.
Note: Compiling an application that use this driver (xtvscreen or xwintv),
just lead me to a can't open (or read I don't remember) /dev/bttv (which exist
and seems to have proper rights), and then segfaults.
