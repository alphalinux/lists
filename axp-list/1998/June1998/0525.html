I am trying to install RedHat on a PC164LX system. I have
the latest SRM console installed. I would like to be able
to install RedHat 5.1 to this system. The standard boot
floppy does not seem to work as I have the Qlogics ISP1020
SCSI card. When I boot with this floppy the system hangs at
the point where the SCSI bus gets reset. I tried making a
new boot disk on an identical system by editing the Makefile
to set the rootdev to FLOPPY and ramdisk to 1440. When I
boot with this disk the SCSI bus has no problems, but the
disk tries to add swap (there is a SCSI disk with a swap
partition defined) and hangs there. Even if I boot from a
bootp server I experience the same hang.

I tried "boot ewa0 -fl nfsroot=ipnum:root_dir" and the
machine booted from the bootp server and then mounted the
nfs directory, but then it failed with a message telling me
that it could not open a virtual console. I am trying to
use the serial port as the console as I have no graphics
card installed on this machine (it is one of several
machines for a research project here that runs headless).

The vendor (MicroWay) told me that I would have to put in a
grphics card and connect a keyboard to do the install. I do
headless installs of Digital Unix on these machines all of
the time with no problem, so I am hoping to do the same with
Linux. I realize that I could do an install to one machine
with a graphics card and then clone the disks for the other
machines, but I wuold rather not install a video card at
all.

If anyone has a tip or three please reply and maybe we can
generate a complete procedure in a summary.

Thanks,
