On Tue, 30 Jun 1998, Matthew Geier wrote:

> > Also, don't you think that 107 patches is a bit excessive for "stable"?
> >
> 107 patches isnt excessive for a development tree. Many of those
> patches were rewrites of significant kernel parts, then a flurry of
> patchs to get it right!.
> Ive been running 2.1.x on my AXP for 12months. Not all versions tried
> worked, but those that do are 'stable'.
>
> Development doesnt always equate to unstable.

If there's a "flurry of [patches] to get it right" then it is unstable.
"unstable" does not mean that it is going to crash it you try to run it;
it means that the code is in a state of flux and subject to major changes
without notable cause or reason. Old code may no longer work.
Significant rewrites may be needed. APIs may change. Unstable.
