On Mon, 29 Jun 1998, Steve Frampton wrote:

> I'm having a strange problem with X. I've got my system configured to run

No you're not...

> as runlevel 5, bootstrapping xdm. If I log in as a normal user, I can do
> whatever I wish in X. But if I, for example, "su" to root, and then try
> to run 'make xconfig', I get:
>
> "Xlib: connection to ":0.0" refused by server
> Xlib: Client is not authorized to connect to Server

That's correct. Only you as the logged in user have access to the display.

> Now, I certainly can understand why I would want to prevent other users
> from accessing my display while I am using it, but in this particular
> situation, *especially* while root, it seems strange that I get this.

X grants root no special privelege. I suppose it *could* for root on the
local machine, but what about a root user from another machine? I guess
the thinking is that root can access the framebuffer directly if they
really wanted to, so X needn't make any special provisions above and
beyond those imposed by the kernel.

> The solution to this is to run 'xhost +myhost.mydomain' and then run 'su'
> again -- *then* things work properly. :-)

While you've su'd, simply open another xterm and "xhost +root@localhost".

Need you be reminded that "make xconfig" doesn't work reliably on Alpha?
