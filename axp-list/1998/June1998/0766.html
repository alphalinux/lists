Meanwhile I discovered something interesting about MS PPTP:

We have it bound to an ethernet interface card, and even
though the bindings are correct for the WAN/LAN interfaces,
PPTP seems to 'take over' the NIC card so that no other traffic
(ie pings, sharing, etc) can occur on the adapter LAN
interface IP address - ie only the adapter WAN pptp
interface works with the NIC card.

We have proved this by hanging up the VPN pptp connection
and the LAN connection comes good again. I suspect this
is a security "feature"; anybody know for sure?

[BTW I realize only the first sentence above
gives some semblance of relevance to the axp-list
but VPN should be a big growth area soon...]
