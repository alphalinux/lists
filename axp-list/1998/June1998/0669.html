Adam Fritzler wrote:

>
> BTW, is there any way to store SRM and ARC on this system at the same
> time? (without having to flash over SRM everytime you want to boot NT or
> Linux?) I'm currently using SRM, then doing a 'boot ewa0' and loading an
> ARC image via bootp/tftp.

You can't load ARC and SRM at the same time (like the UDB) because there
is not enough room in Flash.

These machines do boot very nicely from SRM though, using aboot. The
only problem is that you can't get an unattended re-boot because you
will hang at the SRM prompt, then have to do an init and then boot. I'm
told this will probably be fixed in 2.0.34.
