> I should explain. :)
>
> The three files would be:
> ../app-defaults/fixed - settings that can never be overriden
> ../app-defaults - default settings that may be overriden
> ~/. - user overrides

Programmatically, you can force a value to not be overridable, or to set
defaults that can be overridden by the database. If you arrange the
XAPPLRESDIR path and permissions on the directories correctly, you can have
all sorts of control over what's overridable and what's not. For instance,
make the ../app-defaults/fixed NFS mounted read-only so that only the system
administrator can change it on a site-wide basis, make the ../app-defaults
directory only writable by the local machine owner, and ~/.Xdefaults writable
by the user. If you make the "fixed" directory come last in the XAPPLRESDIR
path, it will override anything the users override.
