> can someone at lesstif tell me:
>

> MenuShellPopdownDown()
>
> and the breakpint gets hit, my X stops taking mouse input. Is the X server
> waiting for something to complete?
> My first attempt at this froze the gdb session ( focus on mozilla ), and i
> coildn't continue.
>
> does this happen on any of ur X servers?
> I tried to close the mozilla composer window using the "file" menu

If netscape is grabbing the mouse then you're going to be stuck until
it lets go. But, in this situation the mouse pointer will still move
around. And, you'll be able to Cntrl-Alt-F1 to a console & kill
Netscape, but by that time X might be somewhat hung & you'll have to
kill the X server. I'm not sure how close this is to you're
situation.

BTW, I often have this problem with Netscape on linux/intel. If my
ppp session is hung & I start netscape & pull down a menu, it grabs
the mouse & then seems to try to do some name resolution, during which
it just freezes up waiting for a response because the networking is
down. At this point I have to kill netscape & often have to kill X
too.
