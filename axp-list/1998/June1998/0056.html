Hi all,

I am trying to install RedHat 5.1 for the first time on a DEC Multia. When
making the install disks, the documentation told me to create a MILO disk
from an image in /milo/images on the RedHat CD. However, the /milo/images
directory is empty. A check on ftp.redhat.com revealed that it was empty
there as well. So, I attempted to make a MILO disk by rawriting the file
/milo/udb-noname onto a floppy. Unfortunately, I can't boot off this
floppy--ARC says it has an unrecognized filesystem, and SRM says it does
not have a valid boot block. I suspect that the problem is that
/milo/udb-noname is not the correct image to use. If this is the case,
where can I find a suitable MILO image? Is it somewhere else on the CD, or
on the net? Or is /milo/udb-noname the correct image after all, and
there's some other problem with my install?

BTW, as a sanity check, I tried booting aboot from the boot disk I created
from /images/noname.img, and that worked fine. I would rather use MILO,
however.

Any suggestions would be much appreciated. Thanks!
