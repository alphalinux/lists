A bit more information about the kernels available and things to look for:

The kernels with "-s" appended to the name, eg ALCOR-s.gz, are built to be
booted from SRM console only. Files without the "-s" are intended for ARC
or AlphaBIOS boot only.

The kernels with "-5" appended are intended only for the EV5-based models
of that platform, eg NORITAKE-s-5.gz is for SRM console and the AS1000A 5/xxx models.

If there is a "-5" version, the non-"-5" version is for the 4/xxx models,
eg MIKASA-s.gz is for the AS1000 4/xxx models.

There's a text file with all the supported platform types, aliases, and typical
MILO/kernel names, located on gatekeeper.dec.com:

/pub/DEC/Linux-Alpha/Kernels/systypes.txt

AVANTI family members booting from ARC console should use the XL.gz kernel.

The RAWHIDE-s.gz kernel is actually based on 2.1.95 (or thereabouts), and is
built for uniprocessor only. The 2.1.xx kernel was used because RAWHIDE family
has multiple PCI busses, and porting such support into 2.0.xx would be a lot
of work.

Specifically to make sure that these kernels would work for installation of
LINUX, please check out CD and floppy usage, as well as network operation.
Be aware that the DE4X5 driver that the kernels are configured with seems
to have somewhat slow autosense switching, so it may take a while for that
driver to get the the media type you actually have connected. Slowness is not
the critical issue here, correct operation is...
