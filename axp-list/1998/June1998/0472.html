On Mon, Jun 29, 1998 at 12:40:52PM +0800, Mikolaj J. Habryn wrote:

> Sigh. Okay, what I'm seeing is basically MILO messages saying
> jumping to kernel, and then nothing. I've seen a few other people
> report this - is this indicative of something really obvious that I'm
> too dense to catch? Has anyone resolved this - the only thing that
> occurs to me is some MILO/firmware incompatibility, but I've upgraded
> both (separately) with no change. Currently running the latest of
> each.

I also had this 'black screen problem' when I thought I was running the
latest MILO. I then found out that the newer MILOs (of which the latest
one works fine for me) reside in the Kernels directory instead of the
Miniloaders directory on gatekeeper.dec.com.

So please double-check that your MILO really is the latest one.

If this is the case, it could also be that you misconfigured your kernel.
In 2.1.x, you explicitely have to enable the "virtual terminal support",
and -- as usual on alpha -- you have to make sure that the VGA console and
not the TGA console is selected.

> Other things that I see, just in case it's relevant, are thing like
> 'PYXIS machine check NOT expected' at odd moments - I managed to
> compile up a .106 kernel (I believe), which at least starts running
> and then does some hardware autodetection, right down to SCSI, before
> rebooting. The PYXIS messages come up when I next try to boot anything
> without a power cycle.

I also see these messages. The 2.0.34 kernel, the 2.1.107 kernel and the
MILO I'm using always output such things -- but I didn't notice any
problem related to it...
