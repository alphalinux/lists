Bruce, Stephen writes:
Hello,

If I setup a ppp0 interface using the netcfg command,
when I activate the interface it then automatically dials
out and can use PAP.

Problem: netcfg dials out, connects, sends username/password
and then aborts after about 30 seconds without a PPP session
being setup

.....

If I have debugging enabled, where can I find the log file?

First, make sure you have ppp-2.3.3-2.alpha.rpm installed. You can
get it from ftp://ftp.redhat.com/pub/redhat/redhat-5.0/updates/alpha.
The PPP that was shipped with 5.0 is broken.

Whatever errors you are encountering, they should be logged under
/var/log/messages. If it was failing with an error something like 'no
route to host', then the updated PPP rpm will likely fix your
problem. It fixed it for me anyway and the symptoms I saw were very
much like the ones you describe.

If that's not it, the message should hopefully point you toward the
right direction.

Good luck...
