On Mon, 11 May 1998, Brian E.W. Wood wrote:

> Right now I'd settle for just finding out how avanti-s was compiled (what
> patches etc.). It works fine but needs to have a few more things compiled in.

If you get a different I/O address for the network interface, then it has
SRM_SETUP switched off. You should get all patches (applied?) in an Alpha
binary RPM for the kernel sources, which should be available from RH
mirrors.

> I don't know how long 2.0.34 will be and I don't know if it is a good idea to
> give my boss a developmental kernel, although I will probably try out 2.1.x at
> home. Does 2.1.101 work on an SX machine? I heard that the
> 2.1.x series didn't work on the SX, but that was some time ago. Does 2.1.101
> need patching for the Alpha? I think I heard it did not.

If 2.0.33 + patches from 'gatekeeper' work for SX, then 2.1.101 should
work too. The patches were merged into 2.1.92 and since `kmod' was fixed
for Alphas some versions later, 2.1.x work very well.

> The kernels I compile that do not work properly are from 2.0.33 sources with
> alpha-patches-2.0.32 applied. Are there other patches I should have applied?

Just SRM_SETUP switch off. Patches for it do not exist, though they
could possibly be backported from 2.1.x.
