>> Hm, I did it! But the problem still remains

>By default cdwrite is using 4 Megs of shared memory. This is a tad
>too much on Alpha without modifying a kernel. Dropping that to
>4000000 bytes works ok in my experience. There is an environment
>variable you can set which implies that I call xcdroast via a small
>shell script which sets everthing which needs to be set and runs
>"the real thing". Check manual page as I do not want to qoute names
>from a faulty memory (mine wetware, that is :-)

You can try the following patch (which worked for me):

--- linux/include/asm/shmparam.h.orig Sun May 10 10:12:12 1998
+++ linux/include/asm/shmparam.h Sun May 10 10:09:40 1998
@@ -36,7 +36,7 @@
* SHMMAX <= (PAGE_SIZE << _SHM_IDX_BITS).
*/

-#define SHMMAX 0x3fa000 /* max shared seg size (bytes) */
+#define SHMMAX 0x2000000 /* max shared seg size (bytes) */
#define SHMMIN 1 /* really PAGE_SIZE */ /* min shared seg size (bytes) */
#define SHMMNI (1<<_SHM_ID_BITS) /* max num of segs system wide */
#define SHMALL /* max shm system wide (pages) */ \
