On Thu, Jun 25, 1998 at 10:00:49AM -0400, Jeff Sturm wrote:
> So what good is -mieee? Is it really intended for ill-behaved fp code?
> (Sorry if this has been covered before. I've seen a lot of comments
> about programs that must be compiled with -mieee without explaining WHY...)

The Alpha does not handle the trickier end-cases of strict IEEE arithmetic
in hardware, so as to streamline the pipeline. So if any of the inputs or
the expected result of an operation are one of Inf, NaN, or denormal, the
processor will signal an arithmetic exception.

If the insn that trapped had the software completion bit set (/s) this fact
will be reported to the kernel. Which then knows that the code was written
respecting trap shadow rules and that the operation may be emulated and
restarted.

If /s was not set, we know right away there is no hope and send sigfpe.

What -mieee does is to set /s and insert trapb insns as necessary to
follow trap shadow rules.
