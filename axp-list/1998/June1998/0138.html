Hello:

I just received the "Official" Red Hat 5.1 boxed set (kudos to Red Hat for
getting it to Canada *next day*).

I created the boot, ramdisk, and milo images by following along in the
"Alpha Installation Addendum" manual. Interestingly, there was nothing in
the 'milo/images' directory (the directory exists on the CD, the book says
to get the image from there), but there were images in the 'milo'
directory.

I chose the 'xl' image (I have an Alpha XL 233), and wrote it to disk:

dd if=xl of=/dev/fd0

When I try to boot from it, I get "unrecognized filesystem" and the
milo image refuses to boot.

So, I decided to grab a new XL image from the ftp.digital.com site. I
downloaded the milo.dd image from
/pub/DEC/Linux-Alpha/Miniloader/latest-Images/Alpha-XL and wrote it to
floppy:

dd if=milo.dd of=/dev/fd0

I tried to boot, and got the same error as before.

So, I'm booting from my 5.0 milo image (can't remember where I got it,
because I jumped through these hoops <but even moreso ;-) > when I set up
version 5.0) which works fine.

I suppose I *could* use the 5.0 image, problem-solved, but I was
wondering:

- Why I'm having these difficulties
- Where I can get an up-to-date milo image for my machine that works

Thanks in advance.
