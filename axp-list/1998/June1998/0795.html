> Hi gang, probably a bit off topic but need some serious newbie help on XDM.
> I have set up Linux 5.0 on a UDB within my home. My Desktop of choice is
> KDE.
[...]
> this works as well. My question is this. Is the there any way to start XDM
> on startup, without having to log into the box and start it manually? I am

I don't know if RedHat has this setup properly, but typically run level 4
is for X-Windows; the default is 3 - multiuser w/NFS. *If* it is setup
right, simply edit /etc/inittab and change the default runlevel from 3 to
4. You can test if it works by typing "init 4" as superuser, though I'm
not sure if it works correctly from runlevel 3 (I don't trust init
scripts). You should be able to pass it the parameter "4" from the milo
boot prompt, but I haven't tried any of this on Alpha. The only problem
being that if you change the default before you test it, you could render
your box dead, save recovery by using a boot floppy (change 4 back to 3).
