When it first starts to boot, hit ctrl-C, it'll break out.
If I might suggest an easier alternative while you're in the development
mode:
Set the rest of the boot options in SRM, but leave auto_action set to
"halt", then at the prompt >>> you only need to type b, and you're on
yer way. I've set mine up like this:

auto_action halt
bootdef_dev dka0 (my harddrive)
boot_file 2/vmlinux.gz (2nd partition, and the kernel)
boot_osflags "root=/dev/sda2"

Then at the prompt >>> I just type b, and it boots.
The good part to this is, if I want to experiment with a different
kernel, I can boot it with any, none, or all of the regular flags. So a
new kernel would look like:

>>> b -fi 2/vmlinux.new.gz

Hope that helps!
