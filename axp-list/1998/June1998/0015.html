Yes, those are quite concerning messages. Looks like you have an old or
broken CD-ROM. Try recompiling one more time, but set
ALLOW_SCSI_DISCONNECT to "no" (if you used "yes").

If that doesn't work, try to force synchronous transfers and failing that,
turn off the fast option. You may want to make everything safe (hit '?'
when prompted y/n) and then turn things on until you find the right one.

On a side note, and you probably shouldn't do this too often, but seeing
as these are simply options for the controller, there shouldn't be any
dependencies across kernel subdirectory boundaries and you can probably
get away with doing a "make dep" without a "make clean" to save yourself
some compile time. You should be able to use the same modules as well,
but this is less likely. Worst case you may have to recompile the
modules for the new kernel.
