Yes, we've managed to make it work here. Thanks to all those who helped.

Our system: LX164, RH5, kernel upgraded to 2.0.33, gcc 2.7.2 (C compiler),
egcs 1.0.2 (C++ compiler + libraries), lesstif-current, ns (first
release).

Steps:
1. Get lesstif-current (compile and install according to Uncle George's
instructions... it's in the archives together with his mozilla patch)
2. Apply the patches from Uncle George (see 1) to the Netscape source (I
had to apply it manually since the mail reader broke it).
3. Read my comments below, and see if it's applicable (ignore otherwise).
4. Last problem I encountered (probably my fault, too) was Grid.c looking
for the includes in the wrong place. I just edited the first 5 #include to
point to/usr/local/include/Xm/the_header_filename).
5. set the variables in the README.

Comment:
On Thu, 23 Apr 1998, Artimage wrote:

> pruthr.c:1242: `JB_SP' undeclared (first use this function)
> pruthr.c:1242: (Each undeclared identifier is reported only once
> pruthr.c:1242: for each function it appears in.)

Encountered exactly the same problem here. Changing the ff. lines (89-95)
in ns/nsprpub/pr/include/md/_linux.h allowed mozilla to compile and run
(I'm a novice in this area so I don't know if I broke anything by forcing
a different definition for _MD_GET_SP(_t)). Anyone has any idea what I
did?

----
/* Alpha based Linux */

/*#if defined(__GLIBC__) && __GLIBC__ >= 2
#define _MD_GET_SP(_t) (_t)->md.context[0].__jmpbuf[JB_SP]
#else*/
#define _MD_GET_SP(_t) (_t)->md.context[0].__jmpbuf[0].__sp
/*#endif*/ /* defined(__GLIBC__) && __GLIBC__ >= 2 */
----
