Hello,

I've got a PWS 433au, and i'm having some problems getting
the network interface going...

The network interface is a 21143 DEC chip, and I've got a
8-port 10BaseT non-full duplex hub.

I've complied my kernel with the patched 2.0.34
which is available at digital's FTP site, thus
my Qlogic 1040b is working (Thanks to Jay!!).

I compiled the tulips driver als a module. I put
alias eth0 tulip
options tulip options=12

in the /etc/conf.modules file. I tried to get the
interface working with the module I compiled, but it
didn't go. I then downloaded the tulip module with
Loic's patch. But this one also woes. It does say
that it's using the user specified media 10baseT(forced)
and that Loic's patch is active, but I still can't
ping it...

Windows NT works fine with this configuration, no
problems at all.

Trying to get it working with a UTP cross cable
also isn't succesfull... (My other machine is
an intel with a 3com 905 interface).

I've had this problem before on an intel system
of my friend. We also couldn't get the tulip
driver working on a 21142 chip. In the end
we swapped the network card for a 21042 chip one.
It works fine now.... the problem with the PWS 433
is that the network card is on board ;-).

If anybody can help me, thanks in advance!!!

greetings,

Stefan van der Eijk

PS: An other question: since I'm thinking about buying
a 100Mbps hub, does anybody have any suggestions for
me?? I've seen some cheap ones at onsale.com. Is
there anything I should look out for / stay clear of??
