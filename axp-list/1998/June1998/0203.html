Christopher Curtis writes:
> However, if you do recompile, you're better to support the new
> hardware non-modularly.

Actually, if only feasible, the opposite is really true. If you have
problems you have a much bigger flexibility with a module (and a better
chance that you will get a meaningful error message on an attempt to
loade instead of a total crash) and is faster to try some variation,
option or a new module instead of recompiling the whole kernel from
scratch over and over.

There are also some things (VFAT support, or iso9660 - maybe, or
"scsi-generic" for writing CDs) which are used only from time to time
and are much better of as modules. Also initializing Plug&Pray sound
card, for example, without a module could be quite nasty and you may
need initrd for that. In a module form this is not a problem if only
you can talk to this card.
