On Mon, 29 Jun 1998, Steve Frampton wrote:

> Yet most things appear to be functional. What I'd like to do is take the
> kernel that was installed by default from the RedHat CD, remove the
> modules I *don't* need, and install the one I *do* need (SCSI tape
> support).

RedHat does not use stock kernels. What you'll have to do is download the
SRPM they created, rerun "make config" and say "m" to SCSI tape. Then
simply do a "make dep clean modules modules_install". If you're lucky you
can compile without the "dep clean" and still get a working LM. You'll
also have to be running the kernel they shipped, not any you made.

My personal opinion is that there's little need for modules; they're handy
to support and useful for probing installations, but when you compile your
own the majority of the stuff should not be compiled as a module. The
only time you need modules support is if you're *really* short on memory,
or for when you install new hardware and don't want to recompile the whole
thing. However, if you do recompile, you're better to support the new
hardware non-modularly. The RAM increase is rarely worth the trouble,
imo. However, make sure you enable module support as I think PPP requires
it to run at all.
