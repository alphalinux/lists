This is sort of a followup to the thread "malloc oddity ?"

I am interested in this subject because I'm trying to port
some FORTRAN code to Linux that uses structures and dynamic
memory allocation. AFAIK, egcs/g77 does not directly support
dynamic memory allocation, but Mike Tripicco let me look at
their code for dynamic array sizing. I tried a slight modification
of Mike's code, then managed to get "real" dynamic memory
allocation working!

This is not as clean as using compiler-supported functions,
but it seems to work for me in my test program. If you're
interested, you can get the details at:
http://walleye.gonefishing.sk.ca/laz/linux/falloc.html

My test program, etc. are accessible through this web page.

Thanks to Mike and others who commented on Linux-Alpha memory
allocation for helping to solve one of my key porting problems.

Now if the g77 guys would just move structures higher up the priority
list!!

Larry Z.
