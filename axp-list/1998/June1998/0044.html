On Sun, 21 Jun 1998, Ric Tibbetts wrote:

> No good. running /sbin/swriteboot -v /dev/sda /boot/bootlx destroyed the
> partition table leaving only partition sda1 & sda2. all others were
> gone. So I re-installed 5.1, and tried this (per the manual)
>
> Leave a 2mb space at the begining of the disk to give space for bootlx.
> ... ok.
> So I started sda1 at sector 4 (rather than the usual sector 1). When
> the install finished, I ran swriteboot again, and "bam" partition table
> blown again.

Are you sure you partitioned the disk using a BSD style disklabel instead
of a IBM PC style partition table?

If not, then repartition the disk using fdisk's <b> command or using
minlabel.
