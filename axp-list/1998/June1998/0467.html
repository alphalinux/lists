> Linux is really much more capable than NT as an Internet gateway. You
> might consider dialing out from your UDB with pppd rather than NT RAS.

I have an entire network of NT/WIN95 machines which all have hidden
addresses (non-routable) and one linux box which acts as a gateway.
The linux system handles all the translations from internal addresses
to the external address going both ways. There is only one ip address
assigned dynmically from the ISP every time it establishes communication
using ppp. It was easy to setup.

I've even setup squid (httpd cacheing) on it and the users of the
NT/WIN95 systems (semi-sophisticated users) are astounded at how fast
their browsers load up at start time.

I know the advertising calls NT the "network operating system," but
Linux is better suited for the task you describe.

As far as NT is concerned, you might want to check out the security
features and make sure you're set to pass packets. I think you need
to click on network while in the settings->control panel.
