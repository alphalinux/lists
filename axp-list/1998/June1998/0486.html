>No, the simplest thing to do is NOT select "CONFIG_ALPHA_SRM_SETUP" from
>the experimental code. This is intended to preserve, as much as possible,
>the hardware setup done by the SRM console, ie IRQ's, PCI bus addresses
>of cards, etc, in hopes that it will allow more graceful "reboot". It seems
>to work on some platforms better than others, which is why it's still in
>the "experimental" options.
