On 2 Jul 1998, Weiping Hu wrote:

> I recently 'successfully' installed RH5.1 on my DEC Alpha station, by
> using 3 floppy disks and a local CD ROM. But after I login to the
> system (as root), I cannot mount a floppy disk, or using the DOS
> utilities such as "MDIR" on a floppy disk. Whenever I do this the
> system dies, and I can't even open another virtual terminal to kill
> the process. The floppy indicator is on, but there no sound of
> reading or disk rotation.

I tried to install RH5.1 recently and has to install MILO on a FAT
partition on the harddisk for it to work (from floppy the keyboard went
mad on the 'a'-key). MILO would boot the floppy with the kernel on it, but
would refuse to read the ramdisk floppy. In the end I copied the files
from the uncompressed ramdisk image to an external hardisk on a Intel
machine and connected that to the alpha to boot.
I know someone else who has not been able to install RH5.1 on a similar
machine to mine, although for him only the ramdisk failed.

The floppy does not work in the running system. Using mdir just makes it
switch on the floppy light and wait, removing the floppy creates a read
error. Luckily the system does not crash.
When I try fdformat the format works but the verify fails, so maybe only
the reading does not work.

Another interesting thing I found was that MILO has a option to read a
block of data off a device, so I tried reading the first block off the
floppy and that failed.

Any ideas?
