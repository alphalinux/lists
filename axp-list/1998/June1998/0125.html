I've had similar, but not identical problems using the 2.0.30 kernel,
right off of the RH 5.0 distribution CDROM and compiling the sound
module from the SRPM on the CDROM. I'm using a SB 16, which is not
PnP and an EB164 300 mHz motherboard.

Using SoX (12-12, with the RH5.0 CDROM), the AU driver always works
with mono, 22.05 kHz wave files, but randomly fails with stereo 44.1
kHz wave files, producing a "buzz" instead of coherent sound. I
downloaded the latest SoX, 12-14, and compiled in the OSS driver. The
OSS driver reliably works with all the files, so I don't believe the
intermittent problem is a kernel problem. The one problem remaining is
that 44.1 kHz stereo files always have a "vibro" effect when played,
while the 22.05 kHz mono files don't. I thought might be an
initialization problem and I corresponded with Chris Bagwell, the
current author of SoX. He didn't think it was an initialization
problem and suggested I upgrade the device driver/kernel. His 2.0.33
kernel worked properly (Intel???).

I tried installing the patches in ftp.redhat.com:/pub/sound for the
sound driver, but they fail for Makefile and Config.in. These patches
appear to be cumulative, i.e., the later ones include the earlier
patches, so I've been trying to install only linux-2.0.30-sound-21.patch.
Has anyone had this problem and have the solution?

So, I'd recommend getting the latest version of SoX and using the OSS
driver. I don't know if 2.0.34 has the sound patches mentioned above.
