>>> Bryan Gomes said:
>
> I have just obtained a DEC AlphaStation 200 that is completely blank, new
> Hard-drive, new cdrom drive. I can see all of the hardware at the SRM
> prompt but when I try to boot off of the boot-disk I get to the aboot>
> prompt, but that is as far as I get (it ask for the kernel and arguments,
> do I put in the compressed kernel disk, what arguments does it want? I'm
> lost)and I don't know what to type in from there. Is it possible to boot
> directly off of the cdrom drive? I get a bootstrap error when I try that.

Use the following boot command line for SRM boot from floppy:

>>> boot dva0 -file vmlinux.gz -flags "load_ramdisk=1 root=/dev/fd0"

You don't need to get to the aboot prompt to get the kernel booted, this
command will do it all...

> I can't boot off of the MILO disk image! forgive me I'm used to booting
> off of the cdrom drive and jumping into the install. I'm new to DEC's and
> LINUX I'm used to Sparc stations, but I'm dying to move on.

You don't use MILO, normally, when booting a kernel directly from SRM console.
