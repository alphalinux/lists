There doesn't seem to be that option for the NCR53C8XX driver. (I presume
that's the correct one to be using on my UDB, since dmesg reports:
ncr53c8xx: at PCI bus 0, device 6, function 0). There is a similar one,
though, for setting the SYNC speed (defaults to 5MHz).

I am currently compiling a new kernel with the following options:
CONFIG_SCSI_NCR53C8XX=y
CONFIG_SCSI_NCR53C8XX_NVRAM_DETECT=y
# CONFIG_SCSI_NCR53C8XX_TAGGED_QUEUE is not set
# CONFIG_SCSI_NCR53C8XX_IOMAPPED is not set
CONFIG_SCSI_NCR53C8XX_MAX_TAGS=4
CONFIG_SCSI_NCR53C8XX_SYNC=0
# CONFIG_SCSI_NCR53C8XX_NO_DISCONNECT is not set

I'm hoping that setting "SYNC=0" means that it won't try to sync, and if that
works I'll try doing an "echo 'setsync 0 10' > /proc/scsi/ncr53cxx/0" to set
the sync on the first hard drive, and then later I'll try the same thing on
the second one, to see if that helps.

I reported earlier that I was having success with the 2.0.30 kernel. Well, a
while after that I got a SCSI error of some sort on the serial console, and
the whole damn machine locked up tight. Unfortunately I can't find the error
in my log files, so I'm not sure what it was.
