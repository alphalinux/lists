Quoting Tim Jenkins (tim@rnaworld.princeton.edu):
>
> I have several programs which cast pointers to ints, which I am hoping to
> clean up. It appears that if I redefine the integer variables as longs
> and then cast the pointers to longs, I am converting a 64 bit pointer to a
> 64 bit long, rather than a 64 bit pointer to a 32 bit int.
> e.g.

According to (my memory of) the ANSI standard, converting from a pointer to an
int is not safe. Period. If you want to be safe (i.e portable), use a void*
as your data type to store pointers, not ints or longs. One way to do this
would be

typedef union int_or_ptr
{
long intval;
void* ptrval;
} int_or_ptr_t;

int_or_ptr_t var;

var.ptrval = some_pointer;
or
var.intval = 12;

It would then be up to you to make sure you only used var.ptrval when you'd
stored a pointer then, and use var.intval when you'd stored one of the
integral data types.
