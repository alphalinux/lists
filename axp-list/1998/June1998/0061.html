On Mon, 22 Jun 1998, B. James Phillippe wrote:

> On Mon, 22 Jun 1998, Darren Kruger wrote:
>
> > my local network is 10.0.0.0/24, the alpha's eth0 interface has an ip
> > address of 10.0.0.113 (which i'm using as the gateway for my windoze 95
> > machine.)
> >
> > to enable masquerading, i'm running the following commands:
> > ipfwadm -F -p deny
> > ipfwadm -F -a -m -S 10.0.0.0/24 -D 0.0.0.0/0 -V 10.0.0.113
>
> There must be a HOWTO out there with some information bass-ackwards. Try
> using the interface IP address of your EXTERNAL connection (your PPP
> device) with the -V flag, or better yet, do it like this:
>
> ipfwadm -F -a accept -m -S 10.0.0.0/24 -W ppp0
>
> Also, I had a conversation a while ago with someone about the use of
> implicit rules (omitting the "accept") in filter rules. My opinion is that
> filter rules are best left with all details explicitly defined. BTW there
> is a mailing list better suited to this problem if you're still having
> trouble. Try linux-ppp@vger.rutgers.edu
>
