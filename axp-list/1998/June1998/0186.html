On Sun, 28 Jun 1998, Matthew Woodworth wrote:

> I am trying to run a program (compiled executable) from the current
> directory but I keep getting the following error message:
> bash: ./rc5des: No such file or directory .
>
> I also have the same problem with a shell script in the same
> directory. However, I've copied other programs into this directory
> and they work fine.

These problems are one of two things: missing exectuables for shell
scripts or missing libraries for executables. If you have problems with a
shell script, type "head -1 MyScript" and make sure that the program
specified on that line exists. For binary executables, type "ldd
MyProgram" and see which library is missing. Then you'll either need to
obtain that library or modify your LD_LIBRARY_PATH to point to the
directory with that library. Alternately, you can edit /etc/ld.conf to
include it always, after running /sbin/ldconfig -v
