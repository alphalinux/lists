> Bob Taylor wrote:
> >
> > A possible problem I'm having getting linux running on my LX164 (I
> > haven't received *any* responses from my recent post) *may* be a
> > TGA/VGA problem. When I make xconfig, I'm shown a line which gives me
> > an option for TGA. The *next* line gives me the option for VGA. What
> > I see happens is:
> >
> > 1. If TGA is set to no, then the VGA option is dead.
> > 2. If TGA is set to yes, then the VGA option is alive and selecting
> > VGA 'y' is not possible.
> >
> Which video card are you using? If you're using the ZLXp-E[123] (a.k.a.
> TGA) series, then you should:
>
> #
> # Console drivers
> #
> CONFIG_VGA_CONSOLE=y
> CONFIG_FB=y
> CONFIG_PCI_CONSOLE=y
> CONFIG_DUMMY_CONSOLE=y
> CONFIG_FB_TGA=y
> CONFIG_FBCON_ADVANCED=y
> CONFIG_FBCON_CFB8=y
> CONFIG_FBCON_CFB32=y
> CONFIG_FBCON_VGA=y
> CONFIG_FBCON_FONTS=y
> CONFIG_FONT_8x8=y
> CONFIG_FONT_8x16=y
>
> This is on my Avanti (AS200 4/166) on 2.1.127 with a ZLXp-E1 (TGA/2MB).
>
> If you don't have a TGA card; i.e. either a Matrox or an S3, then just
> just pick the:
> CONFIG_VGA_CONSOLE=y
