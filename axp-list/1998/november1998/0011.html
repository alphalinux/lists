Hello,

        I'm reading the section on interrupt handling in Alessandro
Rubini's "Linux Device Drivers", and in one spot he describes that the
timer interrupt does not show up on Alpha because it reaches the processor
separately from other interrupts and has no IRQ number assigned. When I
cat /proc/interrupts on a PC164, the output agrees with the statement:

% cat /proc/interrupts
 1: 3751 keyboard
14: 10502883 + ide0
16: 676565340 DC21140 (eth0)
17: 274309 + ncr53c8xx
%

However, on my personal AlphaStation 200 Avanti box, I do see IRQ 0/timer
listed:

% cat /proc/interrupts
  0: 9856919 timer
  1: 2791 keyboard
  3: 210515 +serial
  5: 2398 DE434/5 (eth0)
  8: 0 +rtc
  9: 1 MSS audio codec
 11: 11311 +ncr53c8xx
 12: 0 PS/2 Mouse
%

Is this a processor specific difference (the 21164 versus my 21064) or a
motherboard difference?
