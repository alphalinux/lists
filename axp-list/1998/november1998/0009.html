I found the problem with sox. I just couldn't drop it. I printed out
what the program thought was the MAGIC number and found it was indeed
"dns.". The file (.au) has it ".snd". In the source the MAGIC is checked
for endianess:

fprintf(stderr, "MAGIC=%x\n", magic);
fprintf(stderr, "SUN_INV_MAGIC=%x\n", SUN_INV_MAGIC); /* sanity check */
.
.
.
        else if (magic == SUN_INV_MAGIC) {
                ft->swap = 1;
                report("Found inverted Sun/NeXT magic word");
        }

According to the fprintf, this SHOULD be "0x646e732e == 0x646e732e" which
FAILS!. Glibc is not an issue and neither is the kernel. I am therefore left
with egcs. Perhaps this is why the kernel I downloaded from ftp.digital.com
(axplinux-2.0.35-0.2) hangs after the jumping to message?

I'm really lost if I can't trust my compiler! Any recommendations are
greatly appreciated!
