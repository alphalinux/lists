On Mon, 23 Nov 1998, Mike Larkin wrote:

> I decided to partition the thing with a 30mb "boot" partition, offsetting
> it about 5-10mb into the drive as the docs said (who cares about waste),
> and the rest is just set up as an extra swap partition. (My machine
...
> I partitioned the drive using fdisk and selecting the BSD drivelabel
> option. Everything went smoothly, I mke2fs'ed the 30MB "boot" partition
> and all went without problems. Then, I used swriteboot and copied the
> kernel to the boot partition. No problems.
>
> ... But... Booting gives a "UFS mount -- bad magic" message at the aboot
> stage. Is this to mean that the "boot" partition I created has to be
> formatted UFS?

No, UFS filesystems do not need to be involved unless you're dual-booting
an operating system other than Linux (which requires them).

A possible problem.. the 30MB partition; does it start on the beginning of
the disk? And did you put the kernel on that partition and run swriteboot?
I believe that will corrupt your filesystem. I think you would have needed
to use e2writeboot for that case. But there is a more preferred method,
and that is to donate an entire partition (or void space, before a
partition) to aboot. If you want to "donate" your 30MB partition, you
would just blast it, put the kernel on some other partition (you'd need to
split your 30MB in two) and then run swriteboot on the disk.
