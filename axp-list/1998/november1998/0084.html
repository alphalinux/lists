In message <199811142331.SAA17352@ra.lexis-nexis.com>, Larry Snyder writes:
> It's not exactly what you were asking for, but touch sets the date
> stamp on files. You might could try something like:
>
> find / -exec touch {} \;
>
> This is a bit brute-force-ish, and should probably be done single-user,
> and may produce migraines in some things (rpm?). I'm sure someone else
> will have a more elegant solution. OTOH, look at the good side:
> You have an excellent y2k testbed!
> Good luck!
> -ls-
>
>
> Michael <michael@area54.net> wrote:
> >
> > As some of you know the install program won't read the correct date
> > from some Alpha's like my XLT-300. If left unchanged, all files are
> > written with a year of 2018. The last time I upgraded (reinstalled) my
> > XLT-300 I went in to the shell prompt before the install program started
> > copying files and fixed the system date to the current date. This time I
> > did a complete reinstall that went very well but I forgot to change the
> > date. Does anybody know a easy way to do a global file & directory date
> > change from 2018 to 1998. Only the year is affected, the month & date is
> > ok.

I have just experienced this little problem on my LX164. I discovered that
cron doesn't like this and refuses to run its jobs. I did the following
(as root of course):

touch junk;find / -newer junk -exec /bin/touch \{} \;

Now to see if it worked!
