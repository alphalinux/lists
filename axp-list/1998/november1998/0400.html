Nicholas Sallembian wrote:
>
> Being accustommed to compiling things under Linux/x86, I have always
> used the -m486 switch to enable the 486 optimizations to be performed.
> When I compiled the kernel on my Multia, I noticed that it was using the
> switch -Wa,-m21164. Quite promptly I changed the Makefile so that it
> says -Wa,-m21064 instead, but I don't know what this means. Is it just
> to enable the 21164 optimizations to be performed or does it have
> another meaning ?

gcc switches, are described in info docs, which you can read
via emacs or the standalone program info (you can find docs
for most, if not all, GNU software there).

Options related to your question (examples):
-mcpu=21164 : optimize for a 21164 CPU (this is new in egcs,
                gcc 2.7.2.1 didn't have such an option for
                Alphas)
-Wa,[option] : pass [option] to the assembler

Unfortunately I can't find any documentation for the assembler
option -m????? in RH 4.2 I'm currently running, but it has
something to do with which opcodes will be generated (and which
not).
