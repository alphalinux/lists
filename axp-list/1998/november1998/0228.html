That would be nice to have; I prefer to stick to the official variants when
possible. Can anyone confirm if the RedHat-5.2 RPM for aboot will
successfully decompress and execute 2.1.1xx kernels?

While on the subject of aboot, I have observed one small annoyance that
remains with the latest version I've tried; if a bad filename (or bogus
file) is given on the SRM boot line, aboot will fail (naturally) and
request you to re-enter the filename. However, it is not able to load the
re-entered filename, nor any file at any point after the failure. I must
hard reset to retry. Can anyone comment on whether they too have this
problem with the RedHAt-5.2 aboot RPM?
