>>>>> "MT" == Michael Tibor <tibor@lib.uaa.alaska.edu> writes:

MT> Unfortunately, when I use this [new DHCPD] with 2.1.129 (and
MT> also 2.1.128) I get the error "can't locate module net-pf-17"
MT> which appears to be AF_SOCKET or something like that.

MT> Does anyone have any idea how I might fix that?

Yeah, I ran into that problem with mopd-linux and the newer kernels.
It seems that the packet socket code is no longer compiled into the
kernel by default. You must say yes to CONFIG_PACKET when you build
the kernel.

>From the config help for CONFIG_PACKET:

  The Packet protocol is used by applications which communicate
  directly with network devices without an intermediate network
  protocol implemented in the kernel, e.g. tcpdump. If you want them
  to work, choose Y.
