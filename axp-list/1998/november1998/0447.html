A portability problem I have tried to figure out a good soloution for is
that of structure sizes in arrays - e.g.

struct frob {
  void *a;
  void *b;
  int x;
  int dummy;
}

Now that isn't unreasonable is it? Size on 32 bit machine 16, size on
64 bit machine 24.
The problem is that say I made a big array out of those, and you want to
make sure that the compiler uses shift rather than multiply to address it.
This will happen on the 32 bit machine but not the 64 because of the
non-power of 2 size.

How does one write a structure of a mixture of pointers and ints etc while
guarenteeing it will be a power of 2 in size?
