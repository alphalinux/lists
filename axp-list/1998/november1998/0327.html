I have seen the same thing on the 3940UW. I see that you have a Wide Drive (I
assume that the W means wide) and a slower tape drive. This is exactly what I
had. I had the UW Drive at the end of the chain and had an external terminator
(not cheap when the drive can do it for free!) I saw:

Nov 8 02:40:17 server kernel: scsi : aborting command due to timeout :
pid 69119, scsi2, channel 0, id 0, lun 0 Write (6) 04 00 4d 02 00

Nov 8 02:40:17 server kernel: scsi : aborting command due to timeout :
pid 69120, scsi2, channel 0, id 0, lun 0 Write (6) 04 24 49 04 00

Nov 8 02:40:17 server kernel: scsi : aborting command due to timeout :
pid 69121, scsi2, channel 0, id 0, lun 0 Write (6) 07 80 55 02 00

I also saw the same 'trying harder' and 'card must have gone back to Adaptec
messages' that you mention above.

I had to move the CD and tape drive onto another bus altogether. It is a pain
but I can now backup 3 gig of Data onto the DAT with no errors!
