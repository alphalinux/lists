> Use Q-mail, it compiles cleanly, and is much easier to work with than
> sendmail. I even have it working with Majordomo :)

I understand that qmail is a great piece of software, but I am
afraid that a change would be rather expensive to us.
We are using every feature of sendmail (virtual user table,
user database, mailertable, queueing, various strange check_* rules,..),
using a two-layer setup (some servers talking only with inside,
other only with outside), and many features are integrated with
user administration procedures (so that we do not waste technicians
time to, for instance, add/modify/delete aliases etc).
I frown upon the idea of changing the foundation of this
baroque cathedral..

I know that qmail has better performance, but we have already
solved the sendmail performance problem by dedicating two alphas
to do exclusively SMTP and POP :-)
