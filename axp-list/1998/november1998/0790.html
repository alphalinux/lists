I got the impression from the above that the "Out of Memory" errors happen
while running the RH install script. I can understand that. It uses a RAM
disk and a few virtual consoles, a curses based (I assume) interface and
so on. Remember that the install scripts runs for a while with no swap
space. RH claims you need 16MB for an Intel install (true) and the Alpha
Kernel is bigger as are all the Alpha binaries. So I'd assume you'd need
at least 24MB to run the install. More then just the Kernel loads between
the "boot:" prompt and that first blue screen.

I have 32MB in my Alpha and find that I should have more. I built
the ddd debugger and lesstif yesterday and it took about 8 hours of disk
thrashing. watching the 'top' display I found that if I had had 80MB of
RAM it would not have swapped.
>
> So what is consuming such a lot of memory even before the kernel gets loaded /
> running ???

As I said above, nothing. I think the problem is after the kernel loads.
