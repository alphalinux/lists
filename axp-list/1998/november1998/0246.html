Since the libraries are apparently NOT compiled with
-mieee, which is how this thread started, it seems
the addition of -mieee is creating the extra labor.
In fact, I believe this is the first piece of SW
that someone complained was having FPE's in the G77
libraries on this list. That statement in itself
indicates there must be FP calculations in the library,
which in turn means -mieee is not "free".

Be that as it may, the solution for DgncppB is to either
determine how the code generated data that causes the FPE
in LIBI77 or, since it's a problem specific to that code,
rebuild egcs with the -mieee flag active for the library
and see if that helps. Note using -mieee does not guarantee
FPE's go away.
