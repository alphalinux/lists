> I am trying to build sendmail 8.9.1 with Berkeley db 2.5.9 (from
> SleepyCat) on a stock RH 5.2 system (2.0.35, glibc 2.0.7-29,
> egcs 1.0.3).
>
> It apparently works, but I get messages
> kernel: sendmail: unimplemented OSF mmap flags 0201
> in /var/log/messages, apparently every time sendmail starts.

My previous mail contains a diagnosis, here is a workaround on Berkeley DB
which effectively suppresses the spurious error messages.
Just comment out the lines:

#ifdef MAP_HASSEMAPHORE
        if (!is_rdonly)
                flags |= MAP_HASSEMAPHORE;
#endif

in file os/os_map.c in the Berkeley db sources (tested on 2.5.9),
and then rebuild db and sendmail.
