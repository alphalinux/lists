Hey,..

I had the same problem just a week ago....

>From what I found/understood... ipfwadm version 2.3.0-6 was compiled
with the wrong libs. I've heard that this problem exists both on RH5.1
and RH5.2 (I was using RH5.2).

Some kind person on this list told me that you have to change some
u_int32_t to unsigned long in some header file... can't remember exactly
now.... and recompile ipfwadm. I tried that but couldn't get it to
work.... there is something wrong with my system.... the include files
are conflicting....

So I didn't want to fix a problem to fix a problem... and just
downgraded to ipfwadm version 2.3.0-5... you can get the rpm from the
redhat site under old releases. Once I did that, it worked fine. Now I
will try to figure out what happened with the conflicting includes.

I would recommend that you uninstall ipfwadm 2.3.0-6 first, and then
install version 2.3.0-5. To figure out what version you're using you can
do:

rpm -qa | grep ipfwadm

to uninstall:

rpm -e ipfwadm-2.3.0-6

and to install:

rpm -i ipfwadm-2.3.0-5

I hope this helps...
