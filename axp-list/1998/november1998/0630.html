On Thu, 26 Nov 1998, Sabrina Minshall wrote:

> The firware has found that some information in the machine NVRAM is inconsistant.
> The problem areas are:
> Boot Selection

Well, if it says this, something that you specified in your boot selection
menu is wrong.

Typically, this message appears when the computer can't stat the file
pointed at in the selection menu.

I have a small FAT-partition, where my milo stays. Then I set the ARC to
boot milo, with the OSLOADOPTIONS boot sda3:vmlinux.gz root=/dev/sda3.

It then launches my linux from sda3.
Did you try with milo on a floppy? I'd recommend that as step one; create
a DOS-floppy with the right milo on, and set a boot alternative to point
at the floppy's milo. Then boot it. Once having milo working, get it to
boot linux.

If this works in the right way, then try to make this FAT-partition, put
your milo in there, and configure the boot alternative you wanted.
You can always use the test boot partitons command (what is it called
again?) to see that the file you specified is found by the computer.
