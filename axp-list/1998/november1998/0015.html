>>> Heinz Ulrich Stille said:
>
> How do I build milo for the ruffian? I got the sources from gatekeeper
> (milo 2.0.35-2), compiled the corresponding kernel (2.0.35 with alpha-patch
> es),
> ran `make' in the milo directory, and ended up with a milo binary that
> just hangs the machine without any output. I tried gcc-2.8.1 and egcs-1.0.3.
>
> What am I doing wrong?

If you configured it to use the DEC BIOS emulation code, you MUST build it
under RH 4.2 (ie glibc 1, gcc 2.7.2.3). Building under RH 5.x does not work,
because of some library/C++ issues, I believe.

Are you running the latest RUFFIAN MILO that's there, ie milo-ruffian-981020?

What do you need to build a MILO for, newer drivers? Is something broken?
