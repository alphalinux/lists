Heinz Ulrich Stille <Heinz.Ulrich.Stille@mbi-os.de> writes:
>On Fri, 13 Nov 1998, Brian E.W. Wood wrote:
>> {
>> _wd * wd = (_wd *)m_menuFactoryItems[k].callback_action;
>> if (wd && (wd->m_id==id))
>> return m_menuFactoryItems[k].path;
>> }
>
>What type is callback_action? 32 or 64 bit?

Ulrich's question is the first thing to check. Anyone with experience
porting 32-bit apps to the Alpha knows that pointer casts are red flags.

Easy thing to try: remove the pointer cast, i.e.,

    _wd * wd = m_menuFactoryItems[k].callback_action;

and compile with -Wall. If callback_action is a void*, then you won't
need the cast. If not, then dimes to doughnuts it is a 32-bit int.
Passing a * through an 'int' works on a 32-bit OS (like Linux/x86), but
is sure to segfault on Linux/axp.

In any event, the compiler warning message should inform you what conversion
is going on here, without having to read through the code.
