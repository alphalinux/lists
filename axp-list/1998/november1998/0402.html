Also, don't forget that running an integer benchmark on two
CPUs essentially means letting them do the same work.
But compiling for two different CPU architectures is in no
way doing the same work.

BTW: While egcs generates faster code than gcc 2.7.2.1 in
many (but not all) cases, compiling is slower at least by a
factor of 1.5 (comparing compilation times for my code on
Alphas using gcc 2.7.2.1 and egcs 1.1b).
