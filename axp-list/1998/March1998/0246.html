> Unfortunately not C-Kermit, does anyone have C-Kermit working on RedHat 5.0

Sure! With few minor adjustments everything seems to work just fine.
This file was transferred just with this kermit. "Telnet mode"
also does not create any troubles so far.

This is for a release version 6.0.192. On kermit.columbia.edu is
available alpha of 6.0.193 - which apparently requires even less
of intervention.

Following an advice of Tim Murphy I used -DNOCOTFMC flag in a
compilation. Really do not know if this necessary but it seems to
work that way so I left it alone. :-) The only issue is that glibc
seem to support max 38400 on a serial port; a big leap backwards. I
guess that one would have to use again 'setserial' kludge to get
higher speeds.

Flags which I used for a compilation look like follows:

--- makefile~ Thu Nov 28 09:39:48 1996
+++ makefile Wed Mar 11 12:47:19 1998
@@ -2839,9 +2839,10 @@
@echo 'IMPORTANT: Read the comments in the linux section of the'
@echo 'makefile if you get compilation or link errors.'
$(MAKE) wermit "CC = gcc" "CC2 = gcc" \
- "CFLAGS = -O -DPOSIX -DDYNAMIC -DCK_CURSES -DCK_POSIX_SIG \
- -DBIGBUFOK -DTCPSOCKET -DLINUXFSSTND $(KFLAGS)" \
- "LNKFLAGS = $(LNKFLAGS)" "LIBS = -lcurses -ltermcap"
+ "CFLAGS = -O3 -Wall -DPOSIX -DDYNAMIC -DCK_CURSES -DCK_POSIX_SIG \
+ -DBIGBUFOK -DTCPSOCKET -DLINUXFSSTND $(KFLAGS) \
+ -DNODEBUG -DNOHEBREW -DKANJI -DNOCYRIL -DNOCOTFMC" \
+ "LNKFLAGS = $(LNKFLAGS)" "LIBS = -lncurses -ltermcap"

#This version was used for Linux prior to C-Kermit 6.0.192.
#Now the "Linux File System Standard" is considered standard, ditto TCP/IP.

Adjust to your taste. Few other patches attached. That's all.

Michal
