I didn't mean to stir up so much dust.
This does appear to be a sin of omission,
something I'm guilty of from time to time.

On a more productive note:
I've run 50 hours of Gaussian 94
calculations (quantum mechanics code)
using Kazushige Goto's assembler
dgemm code and am glad to report no errors.
His code is developing quickly. Unaligned
cases now run around 700 MFlops on a 625 MHz
21164A, and I suspect that cache congruence
will not be a problem soon.

Cases requiring a transpose are still
somewhat slower, but (not speaking for
Dr. Goto) I suspect that problem
will go away soon also.

His work is much appreciated.
