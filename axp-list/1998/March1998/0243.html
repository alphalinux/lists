I posted a while ago on this list a warning that Perl supplied on in
RH 5.0 is shot. It has more problems than just beeing obsolete.
I was told by some that it works for them (TM) ergo is fine. :-)
RSN I will post spec file and config which will allow to recompile
something saner.

When compiling perl ***pay attention*** to comments in hints/linux.sh
and uncomment what they say to uncomment for glibc. This is important!
Note that most likely you do NOT want a binary compatibility with
Perl 5.0003 unless you have a big pile of modules compiled for it
and you are do not plan to recompile/replace them. If you would ever
be doing some embedded perl then you will be glad that you picked
up to be NOT binary compatible with an old version.

Also all these '*.ph' files produced automatically by specs from Red
Hat are pure garbage. Perl docs say so. :-) If you nees some of them
then this automatic conversion supplies only a skeleton which has to
edited by hand as header parsing in h2ph is simple-minded at best.
Besides you may need one or two headers if you have some "legacy" Perl
4 code which cannot live without these. Then you have to grab h2ph and
an editor and provide what is necessary. The current Perl is not using
these files at all as it has better and more reliable means.
