When's 5.1 due out? :-)

I have a few hundred thousand lines of g++. As you folks have
heard, there are several problems with RH 5.0 on the alpha
and C++ programs:

1) header files. fd_set is declared in inconsistant ways that seem to
bother C++ compilers much more than C compilers. I fixed it via changing
/usr/include/linux/posix_types.h to read:

/* typedef struct fd_set {
unsigned long fds_bits [__FDSET_LONGS];
} __kernel_fd_set; */

typedef struct __fd_set __kernel_fd_set;

2) g++ library problems

2a) link with libc. Works with a toy, but my big programs still dump
core. See (3) to see why I don't know where.

2b) use egcs. Chokes on my C++ grammar. Bug report submitted.

2c) fix the library. Shinji Sumimoto gave the hint for that, so I
got the library source and added a #define _G_USING_THUNKS to
libio/libioP.h. Works for toy programs; my big C++ program goes into
an endless loop. See (3) to see why I don't know where.

3) Insult to injury section:

/tmp/cca23354.s: Assembler messages:
/tmp/cca23354.s:754: Fatal error: String too big (4134 bytes)

That's what happens when I use -g. Looks like ecoff binaries can't
have any string bigger than 4096 bytes, and my doubly-nested templated
classes or whatever generate debugging strings which are too large. I
could "fix" this by fiddling with the source, which might let me find
my endless loop... but...
