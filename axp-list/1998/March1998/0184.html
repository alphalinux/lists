First:

My SX164 (21164PC-533, 128M memory, NCR875, NE2k ISA-bus network adapter)
dies on heavy disk write-I/O, the machine just locks up without any
warnings. First I though that the problem was only with the IDE-drives,
but further tests showed that any continous write-I/O to any drive locks
up the machine.

A simple 'dd if=/dev/zero of=file bs=1k count=100k' kills the machine
in about ten seconds (i.e. probably then when the kernel starts to
flush buffers to disk), same thing happens if I'm untarring a large
.tar to the drive or copying files with cp. The lock-ups get less
frequent if I use only 5 second sync interval and 1 second flush
interval, but they still occur.

I'm using RedHat Linux 5.0 with 2.0.33 kernel patched with
alpha-patches-2.0.32-0.1 from gatekeeper.

Linux reports that the SCSI-controller is NCR875-J (Diamond FirePort 20
in PCI-bus). I have to use 2.1.x -series SCSI-driver (ncr53c8xx.c,
version 2.5f) because the version 2.4 of the SCSI-driver does not work at
all with my controller. The drive connected to this is a HP C3724S,
which is FastSCSI2 device. Does this have something to do with my problems?
Should I change the controller to a better supported one? Recommendations?

Second problem:

Because the 2.4 version of the NCR SCSI-driver does not work at all, I
have to use the IDE-drive as a boot drive (only the root partition is
on IDE). I tried to compile MILO, but the my version just hangs during
MILO boot. I followed the instructions in
<URL:http://www.azstarnet.com/~axplinux/milo-4.html>
with no success. I used milo-sources-2.0.30-0.4.tgz from gatekeeper and
pure 2.0.30 kernel with alpha-patches-2.0.30-0.4. I did replace the
drivers/scsi/ncr53c8xx.c from 2.1.88 (it didn't get any further in boot
with the original ncr -driver). Can anyone help with this matter?
