Hello,

I have an AlphaStation 600 5/333, but do not have either video or
monitor for it, however it does have a VT420 terminal connected to
COM1 which has been sufficient to boot and use Digital UNIX. In
attempting to install RH Linux 5.0 from the SRM (using the alcor-s.img
boot image) I get the impression that I'm going to need to purchase
video hardware to install:

Environment:
SRM console V6.7-2
ARC console 4.56
PALcode VMS PALcode V1.20-0, OSF PALcode V1.22-0

The SRM environment variable 'console' is set to 'serial'

Floppies have been created with rawrite on an NT box.

>>> boot dva0 -file vmlinux.gz -flags "load_ramdisk=1 prompt_ramdisk=1"
[...]
jumping to bootstrap code
aboot: Linux/Alpha SRM bootloader version 0.5
aboot: switching to OSF/1 PALcode version 1.22
aboot: no disklanel found.
aboot: booted_dev='dva 1 0 0 0 0 0 0', guessing boot_device='fd0'
aboot: loading compressed vmlinux.gz...
aboot: ok, now starting the kernel...

& that's all she wrote. I note that there is a 'linux serial' command
that can be given to SPARC machines, so I have put 'serial' in the
flags as well as having removed the 'prompt_ramdisk=1' and simply
switched disks as soon as the kernel has finished loading. No further
console output in all cases. Bottom line question then: do you need
to have video hardware to install Linux/Alpha?

I have found a MILO on gatekeeper.dec.com that purports to work with
ALCOR based Alphas but have not yet tried it, nor have I tried the ARC
route with the alcor.img boot image. Wisdom would be welcome!

Replies cc'd to fahnoe@FahnoeTech.com are especially appreciated!
