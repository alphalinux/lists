Quant-x support writes:
>
> The SVGA server seems to be working fine on RedHat 5.0, but you'll have to
> use the following files from RedHat 4.2 for proper operation:
>
> ld.so.1
> libdl.so.2
> libc.so.2
> libm.so.6
> libnss*.so.1
>

These would indeed work but instead of that gross hack, not required here
as you do have sources, I would rather recompile X.

I know that this is on a biggy side but playing a bit with configuration
you may cut it down only to this server you need/want. I know, I was
doing that a while ago when I needed an extra X support for a laptop. :)

> Installing older shared libraries also proved useful on other occasions, when
> we
> tried to install binary packages built for RedHat 4.2.

Excluding commercial software, where you indeed do not have a choice,
that means that you have in memory usually two different sets of shared
libraries. As we all know too well you never have too much memory
on Alpha. This is not a cheap solution in terms of resources.

BTW - if you do decide to go that way install old libraries in a
separate directory (like /usr/lib/old/), add that location to
/etc/ld.so.conf and rerun ldconfig. It will help you stay sane
in administration tasks.

Oh, it looks like that RSN we will recompile that stuff anyway.
Then we will have to find a bandwidth to upload it somewhere accessible
- which may be a harder problem.
