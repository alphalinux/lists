KDE Beta-3 is *mostly* working for me under 5.0. I'm having a problem,
however, with some of the network-related utilities, an example of which
is knu.

Knu fails to run due to missing dependencies thus:

ldd knu
libkdeui.so.0 => /opt/kde/lib/libkdeui.so.0 (0x0000015555556000)
libkdecore.so.0 => /opt/kde/lib/libkdecore.so.0
(0x000001555575c000)
libqt.so.1 => /usr/X11R6/lib/libqt.so.1 (0x0000015555940000)
libXext.so.6 => /usr/X11R6/lib/libXext.so.6 (0x0000015555c6e000)
libX11.so.6 => /usr/X11R6/lib/libX11.so.6 (0x0000015555d80000)
libg++.so.27 => not found
libstdc++.so.27 => not found
libm.so.6 => not found
libc.so.6 => not found
ld.so.1 => not found
libstdc++.so.2.8 => /usr/X11R6/lib/libstdc++.so.2.8
(0x0000015555f7e000)
libm.so.6.1 => /lib/libm.so.6.1 (0x00000155560da000)
libc.so.6.1 => /lib/libc.so.6.1 (0x000001555620a000)
/lib/ld.so.1 => /lib/ld-linux.so.2 (0x0000000000000000)

In an attempt to correct this problem I made the following links:

libstdc++.so.2.7.8.2. => libstdc++.so.27
libg++.so.2.7.2.8 => libg++.so.27
libc.so.6.1 => libc.so.6
libm.so.6.1 => libm.so.6
ld-linux.so.2 => ld.so.1

This of course makes ldd happy but the program still fails to run.

I assume this has something to do with the new glibc but I am too stupid
to pursue the matter further without some outside help.

I have also had some other problems with compiling which are related to
syserrorlog and errno.h, but I've so far been able to get around these
troubles with some very ugly hacks.

Any ideas?
