Installing all the errata and rebuilding emacs fixed the ^G problem
when running emacs in an X window.

However, I still have some problems with emacs in text mode wrt
suspending and interrupting. This problem was also reported by Michal
Jaegermann <michal@ellpspace.math.ualberta.ca>.

It can be reproduced as follows:

1. Start emacs in an xterm:
emacs -nw

2. Give it an interrupt (^G) followed by suspend (^Z). You should get
back to the shell prompt.

3. Bring emacs back to the foreground:
fg

4. Give emacs an interrupt (^G) followed by suspend (^Z) again. This
time, emacs doesn't suspend! This is a bug.

5. Give emacs an interrupt again (^G). Now it suspends! This is also
a bug.

6. Bring emacs back to the foreground:
fg

7. Now suspend (^Z) does nothing, and interrupt (^G) as well as many
other control characters get printed instead of getting executed,
and emacs is thoroughly hosed. ^C now suspends it!
