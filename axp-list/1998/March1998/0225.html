Here it is promissed kit for a recompilation of Perl 5.004_4.
Watch out! This kit, "as is", assumes that you are using glibc.
It means that you should be ok on Alpha does not matter if you
are using RH distribution 4.2 or 5.0.

If you would like to use it on Intel with libc5 then remove
perl5-hints.patch and edit specs file accordingly. Anybody knows
how to make operations conditional for rpm depending on a library
version?

This kit DOES NOT compile, on purpose, sperl ("suid perl") and all
.ph files. It also creates executables which are NOT binary
compatible with perl5.003. The later in only an issue if you have
a big pile of library modules compiled for perl5.003 which you still
want to use. My advice in such case - recompile. You will do that
sooner or later anyway.

If any of these is an issue for you then create a new config.sh
with settings you want and use that one for a compilation. The
easiest method would be to replace perl5.004_04-config.sh.patch
with your own version and proceed as described below.

Instructions. Get perl5.004_4 sources from your closest CPAN
node (one address in in specs). Drop these sources into
/usr/src/redhat/SOURCES directory. Unpack included tar.gz file
in /usr/src/redhat. If you do not want to do any changes then
type, as root, 'rpm -ba SPECS/perl-5.004_04.spec' and wait for
a while for a compilation to finish. Once done, just to make
sure, switch to BUILD/perl5.004_04/ directory and run 'make test'
there. If everything is OK (it should be) then install binary
rpm archive you will find in a subdirectory of RPMS. Save
somewhere created binary and source rpms (the later in SRPMS
subdirectory).

This process is nice with one flaw. While making new archives
it forces you to work from a root account and there is really
no good way around it. I prefer usually to keep my root presence
to a minimum.

Included spec file specifies explicitely all files installed. That
way, when getting a new version you will not clobber all these
files you installed in subdirectories of /usr/lib/perl5, in site_perl
in particular, by adding various extra modules and libraries.

Enjoy,
Michal
