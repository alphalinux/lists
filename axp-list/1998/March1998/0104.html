>Stefan Boresch wrote:
>=20
>> I have the following question regarding the compilation of kernels
>> under Redhat 5.0. I guess this is an FAQ, but I am slightly confused.
>
>[...]
>When building a new kernel, make sure /usr/src/linux points
>to linux-2.0.33, or whatever you called this dir.
>(This is the directory where you've untarred the prestine
> kernel sources + alpha-patches).
>
>When building applications, make sure /usr/src/linux points
>to linux-redhat (i.e. the RedHat distribution sources).
>
>When linking /usr/src/linux to linux-2.0.33 (i.e. the prestine
>kernel sources), all curses based apps (like BitchX) and many
>others normal ones (like nxterm) miserably fail to compile...

I just link /usr/src/linux to whatever directory the real (used)kernel is=
in.
Then there are links in /usr/include into that directory.
So for any compilation I am using the actual headers.

This is with RH-5.0, glibc-2.0.6, gcc-2.7.2.3/gcc-2.8.1/egcs-current on a=
n
Alpha,Station 500 - it's working with all.
