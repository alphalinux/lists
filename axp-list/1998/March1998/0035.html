The system:
DEC Alpha, UDB/Multia
RedHat 4.2
Kernel 2.0.32 + Alpha Patches-2.0.32-1

PCMCIA utilities 3.0.0 from hyper.??.edu

Symptoms:
depmod -a reports "TYPE /lib/modules/2.0.32/pcmcia"
?? What the h*ll does that mean ??

I've recompiled the utilites and installed them in
/lib/modules/2.0.32/pcmcia/lib/modules/2.0.32/pcmcia (whoops. heh-heh.)

# cd (that directory)
# depmod -a .
Segmentation fault (core dumped)

Peculiar.

# gdb `which depmod` core
(gdb) where
#0: ...
(gdb) info threads
* 1 process 1140 0x1555582ccd0 in __curbrk ()
(gdb) quit

Hmm. Okay, no biggie. We don't NEED to depmod stuff...

# insmod serial_cs.o
Unable to reoslve dependency ... (there were two of them, plus:)
Unable to resolve dependency CardServices

# for FILE in *.o ; do
> echo $FILE:
> strings $FILE | grep CardServices
> done

I see it is in pcmcia_core.o - woohoo!

# insmod pcmcia_core.o
#

Success?

# lsmod
Module Pages Used by
pcmcia_core 6 0
ip_masq_vdolive 1 0
...

# insmod serial_cs.o
serial_cs.o: get_kernel_syms: Invalid request code

Hmm. Let's unload the core module, then.

# rmmod pcmcia_core.o
rmmod: module pcmcia_core.o not loaded

Really?

# lsmod
Module Pages Used by
pcmcia_core 6 0
ip_masq_vdolive 1 0
...

# rmmod pcmcia_core
Segmentation fault

Not even a core dump. Okay, let's try individual dependencies:

# depmod pcmcia_core.o
modprobe: get_kernel_syms: Invalid request code
pcmcia_core.o: unresolved symbol(s):
pcmcia_core.o:

(Same for serial_cs.o)

This is about as far as I'm getting. I now have a module stuck in memory
that I cannot remove, and that I cannot add to. Worse yet:

# cd
[ /root]
# insmod minix
/lib/modules/2.0.32/fs/minix.o: get_kernel_syms: Invalid request code

Doh!

Advice, pointers, cookbooks?

tia,
