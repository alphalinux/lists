I have the same machine and you seem to be encountering the same
problems I had.

I haven't been keeping up with mail on the group lately, I have had
to drop work on the alpha for now to get other things running.
I was just catching up on my mail and noticed your posting so sorry
for the delay.

It sounds like you are running the SRM firmware (dieing while
building file systems etc).
The first thing to do is switch from 'au' to 'a' series ie boot with
alphabios.
Do this by setting the os_type env variable to NT - with the SRM
console.
While SRM-Milo-... path appears to work and will build it doesn't
recognize the hardware properly. For one thing it seems to think it
has a 32 bit memory bus so it will think you have a quarter of the
memory you actually have.

As someone else has already pointed out the is a bug in the kernel
with regard to the Miata and there is a patch - it speeds things up
astronomically.

Xfree86 only supports 2MB TGA cards. I have a 2MB powerstorm but
it turned out to be TGA2 which also isn't supported (I pulled a Matrox
out of another machine and it worked fine).

In short it does work but there are lots of traps! It took me so long
to get it running that I ran out of time and have had to set up other
servers to do the job it is supposed to do as I haven't yet been
able to get it to run as a SAMBA NT Domain controller for our student
lab (don't let Digital's "Personal Workstation" epithet fool you
I intend to use the machine as the departments main server).
Consequently I have been in panic mode for the last week but hope to
get back onto the alpha soon.
