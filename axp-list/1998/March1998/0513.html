I tried to compile jade (some SGML parser written in c++) on my system but
get an awfull lot of warnings and errors like:

primitive.cxx: In method `class ELObj * MemberPrimitiveObj::primitiveCall(int, class ELObj **, class EvalContext &, class Interpreter &, const class Location &)':
primitive.cxx:347: warning: unused parameter `int argc'
primitive.cxx:347: warning: unused parameter `class EvalContext & context'

and like:

nsgmls.cxx(.text+0xf8): undefined reference to `Vector<String<unsigned short> >::~Vector(void)'

On a clean RedHat 5.0 system with updated kernel (2.0.33).

These messages do not occur on my clean RedHat 5.0 i386 system.

Any change we can get a clean, working glibc2 system on Alpha one of these
days? At present the Alpha has an unpleasant smell regarding c++ which
makes me wonder how redhat got their pieces of code compiled anyway.
