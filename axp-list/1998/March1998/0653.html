Hellos,

Yesterday I downloaded the latest SAMBA source via Lynx (there was no RPM
for Alpha RedHat 4.2, just 5.0) and I think that is when I noticed it - my
system load started to grow, and never stopped. It eventually went to
around 35 (yes, thirty-five) before I rebooted the machine. The wierd
part was that trying to copy a file resulted in a hang. I was in X at the
time (standard Fvwm'95) and the xterm would simply not respond. It did
not matter if I did the copy as myself or as root, both would hang, but I
could open new xterms, memory swap didn't seem to be affected, and I could
navigate the directory tree. There were NO processes consuming more than
about 12% of CPU, and that was typically X/Top screen updates. I did not
try to untar the file or do any other sort of disk activity (aside from
the copy). The problem seemed to be trying to copy across a partition - I
was able to 'mv' the file from /tmp into / (though after the hard reboot
it went back into /tmp with its original name). Oh yeah, I couldn't
reboot, either. I tried "reboot", "shutdown -r now", "shutdown now",
"init 0", and "init 5", all with no effect besides a broadcast message.
Attempting to remount the filesystems read only gave me an error about not
being able to create /etc/mtab~. There was nothing in the logs, either.
Any ideas? This is a 166 UDB w/64MB RAM, 32MB of which is recognized, and
an IBM 4GB SCSI in an external case, running Kernel 3.0.33 with
Alpha-patches and firewalling.

Thanks,
Christopher.

only need pnm2gif and pnmscale if anyone has them handy...
