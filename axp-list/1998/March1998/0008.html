Hi,

I'm currently suffering a rude awakening to Linux on alpha. We have 3 new
533Mhz beasts to be dedicated to running Linux. They came preinstalled
with RH5.0, quite well done it seems.

Since getting them I have so far solved some instabilities by making my
own kernel (the originals were non-RH and based on 2.0.31, hence suffering
the "inode still has mappings" etc). I've even managed to get to grips
with MILO :) All updates are applied of course.

Only two serious problems seem to remain;

1) X instability. The machines have Matrox cards (spit), and the SVGA X
server looks ok (bit of gfx corruption sometimes), but alas will sometimes
hang the entire machine upon "startx".

I'm thinking the dedicated XSuSE Matrox server has much more stable
support. Does anyone have an alpha RPM for this? If not, some pointers to
the source and a bit of assistance and I could try and knock one up for
the benefit of the alpha Linux community.

2) g++ instability. This one is _nasty_. Could someone please get an alpha
Linux RH5.0 machine and verify if this crashes or not:

compile with "g++ -o test test.cc"

==== cut, begin test.cc ==

#include <stdio.h>
int main(void) {
FILE* pFile;
pFile = fopen("a file", "w");
if (pFile != NULL) {
fclose(pFile);
}
}

==========================

Note!! The exact same program runs fine (no segfault) if compiled with
"gcc" rather than "g++". Also!! If I build egcs-1.0.1, "egcs-g++" will
build the little program fine. Unfortunately, egcs-1.0.1 on the alpha
appears to have other problems that causes some of our academic research
software to crash in other places (that 2.7.2.3 gets right).

So we need a hybrid of egcs and 2.7.2.3!! Seriously though, stdio in a c++
program appears to not work too well on alpha RHL5.0. Personally I suspect
libg++, and I would _love_ to try libg++-2.8.0b6(?), but I can't get it to
build :(

Does anyone have any pointers, or perhaps an rpm for a recent libg++?

Cheers for any help,
Chris
