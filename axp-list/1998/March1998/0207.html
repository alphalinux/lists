> 2) The libraries created have the same version number as the ones which are
> already there on RH5.0; whats going on? How are these supposed to work?

I had a quick peek in diffs, not in the whole thing, and library changes
seem to be minimal and of a rather cosmetic nature. I would fully expect
that everything would work without library changes. The same likely
applies to xterm. I may possibly missed something, as this was a very
quick scan, but I do not think so.

Note that libraries of X as supplied in RH 5.0 distribution may have
select problems due to incorrect headers in glibc (later fixed in
updates); so it may be your advantage to replace libraries with new
copies anyway. This assumes that you applied updates before venturing
into X recompilation.
