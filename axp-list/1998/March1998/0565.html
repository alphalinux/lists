"No such file or directory" in this case usually means that the program c=
an't
find one or more of the libraries that it needs. Try ldd netscape and see=
what
happens (although I'm not sure ldd would work with a DU binary).

Do you have the required DU libs installed? You need:

LinXm.so
linXmu.so
libm.so
libX11.so
libXt.so
libdnet.so
libc.so
linXext.so

All installed in /usr/shlib.

You also need /sbin/loader

All this assumes you are running the DU 3.2 libs. Things get uglier if oy=
u're
trying to use the 4.0x libs.

Check the FAQ at azstarnet for more info.
