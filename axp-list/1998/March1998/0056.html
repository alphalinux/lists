>
> After installing 5.0, I need to recompile my kernel to enable loop-back
> device support and generic SCSI device support. However, I'm running
> into a continual problem with gcc generating a sig 6 here:
>
> gcc -D__KERNEL__ -I/usr/src/linux/include -Wall -Wstrict-prototypes -g -O2 -pipe -mno-fp-regs -c FlashPoint.c -o FlashPoint.O
> cpp: output pipe has been closed
> gcc: Internal compiler error: program cc1 got fatal signal 6

Not taking anything from your compiler :-) - do you really need this
Flashpoint stuff? This hack with linking with an extra binary
files is a known "trip point" for gcc. Switching it off likely
will allow you to compile.

Please note that much revamped BusLogic support, with saner Flashpoint
code and a lot of improvements, is available from www.dandelion.com
(thanks, Leonard!).
