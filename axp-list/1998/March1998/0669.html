Hi,

I been running the alpha with a network adapter connected to the
MII connector on the motherboard running to a hub and then to our
thinwire net (this all worked). I have finally received a 10base2
adapter card with a BNC connector on it from DEC. This card plugs
into the MAU connector on the motherboard and linux does not seem
to know about it. The network stuff still all gets configed fine
but linux thinks it is not connected to a wire.

The driver messages indicate that it is looking at the MII interface,
it says 'no MII transceiver found' - which is quite reasonable but
not what I want it to look for!
There are settings in the driver tulip.c to say use only 10base2 so
I enabled these and recompiled but still get the same message but
with another message a few lines later saying 'using user specified
media 10base2', still no working net. I don't know if this situation
is what these settings are for.

Does anyone know if this is a driver problem or is it something
earlier? The booklet with the 10base2 adapter doesn't mention any
jumpers on the motherboard and there is nothing to set in the
AlphaBios (there are settings for connector type in SRM
- and if set to BNC the adapter does physically work from the SRM
console)
