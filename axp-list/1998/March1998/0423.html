Oh, correct me ifI am wrong on these. I will be installing a generic S3
4mb card in my multia (hopefully).

1. grab kernel sources from sunsite.unc.edu (2.0.33) and the patches from
gatekeeper.dec.com (alpha-patches-2.0.32-0.1.gz,
alpha-RLIMIT-patches-2.0.33.gz, alpha-em86-patches-2.0.33.gz)

2. extract the kernel source
tar -xzvf kernel-source.tgz

3. link the directories
ln -s linux old
ln -s linux new

3. run patch on the patch file (patch -
gzip -d < alpha-patches-2.0.32-0.1.gz | patch -p0
gzip -d < alpha-RLIMIT-patches-2.0.33.gz | patch -p0
gzip -d < alpha-em86-patches-2.0.33.gz | patch -p0

4. remove extra links
rm old
rm new

5. run make config and select normal stuff, plus VGA (no TGA support)

6. recompile kernel
make clean; make dep; make clean; make boot

7. move new kernel to suitabvle location (/boot)

8. shutdown and install S3 card.

9. Pray :)

10. Turn it on and hope that all goes well.

11. Presuming it boots ok, get the S3 X server from gatekeeper.dec.com

12. Link X to the new Xserver

13 Fire up X

14. go to bed

Anything I am missing or confused about?

Thanks
Tad
