Dave Gilbert wrote:
> Has anyone got XFree 3.3.2 Xservers going on RedHat 5.0? They seem to be
> linked to an ld.so.1 and be using .6 libraries; on my RedHat 5.0 system
> there is an ld.so.2 and the libraries are 6.1.
>
> Does this indicate the new binaries are built for RH 4.2???
> (This is just trying the new SVGA server for my matrox card)

The SVGA server seems to be working fine on RedHat 5.0, but you'll have to
use the following files from RedHat 4.2 for proper operation:

ld.so.1
libdl.so.2
libc.so.2
libm.so.6
libnss*.so.1

These are available from the usual RedHat mirror sites, for example:

ftp://ftp.funet.fi/pub/Linux/images/RedHat/redhat-4.2/alpha/lib

Installing older shared libraries also proved useful on other occasions, when
we
tried to install binary packages built for RedHat 4.2.

It is probably a good idea to upgrade the other parts of the XFree86
distribution
as well, since 3.3.2 contains many important security-related fixes.

(if you can't find one of the files or still have problems running the new
SVGA server, drop me a line)
