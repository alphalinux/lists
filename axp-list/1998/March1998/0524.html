I know, this is an old one, but thought you might be interested in
the fix.

This morning I found tons of log messages like the following
in my /var/log/messages

Bounty kernel: Cannot read proc file system: 9 - Bad file descriptor.

while klogd and syslogd were happily swallowing 50% CPU each.
The system is a PC164LX with a 21164 @ 533MHz, running a 2.0.32 kernel.

Sending a SIGTSTP/SIGCONT pair to klogd did not help initially, however
sending the signals again with a delay of a few minutes in between
brought the system back to normal.

>From what i found scanning the mailing lists logs, it seemed that the
medicine usually applied was rebooting, so I thought that a less invasive
method might be of help to somenone with the some occasional problem.
