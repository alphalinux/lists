On Sun, Mar 29, 1998 at 09:08:19PM -0500, adisaacs@trurl.resnet.mtu.edu wrote:
> I'm getting the following message when I attempt to boot 2.0.34pre4:
>
> lca: machine check (la=0xfffffc00002084d0,pc=0x12003c5dc)
> Unknown errorlog size 408
>
> It looks like this happens when X attempts to come up (I'm running XDM.)

Turns out that the new kernel had apparently moved my TGA to a different
memory location, but I still had the old MemBase in my XF86Config.
I commented out the MemBase, and everything was fine. (Make sure you're
running the latest XFree86, though, because old versions <3.3.2 weren't
able to probe for a TGA correctly in all situations.)
