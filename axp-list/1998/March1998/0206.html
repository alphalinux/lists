Grr ... still trying to compile MySQL. I figured that if GCC kept
crashing, I'd use EGCS. So I grabbed the latest snapshot (980308) and am
trying to compile that. However, I came across the infamous RLIMIT_CPU
not defined errors. I *have* updated my {linux,asm}/resourcebits.h as per
another requirement.

I modified /usr/include/resourcebits.h to reflect these changes and is
attached. This might fix the problem others were having with 4.2.
However, big disclaimer: I make no intention of even suggesting that I
know what I'm doing with this stuff, so use at your own risk. It allowed
EGCS compilation to continue, though, so ...

Basically, the problem is that /usr/include/linux/resource.h changed the
definitions to __KERNEL_RLIMIT_* when I upgraded, but
/usr/include/resourcebits.h was still looking for _RLIMIT_*. I'm sure my
modification broke something else, if not simply put my system back the
way it was originally. :)
