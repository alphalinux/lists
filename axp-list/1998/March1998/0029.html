We just received a 533MHz lx164 machine from TRANSTEC, 2MB cache,
256MB RAM, Adaptec 2940AU controller (ultra, not ultra-wide) and
a 9.1GB Barracuda drive. The machine came without OS and I
bothered the list last week with some stupid errors that turned
out to be misconfigured items in the AlphaBIOS. BTW, thanks
to all for pointing out that the lx milo image will hang after one
has changed anything of importance in the BIOS.

Unfortunately, the machine is not as stable as I would expect it to
be. During the install of RH 5.0, I already received two Sig 11s
which terminated the installation (during the copying of stuff from
the CD (Plextor 12/20x SCSI). On the next day, however, everything
worked miraculously. I have made all upgrades from the RH update
directory and recompiled the kernel (just the Redhat one for the time
being, I know that I probably should upgrade soon to 2.0.33, right?).
Everything was fine. Next, I had a go at egcs-1.0.1. Make bootstrap
died several times with Sig11, when restarting the process it compiled
the 'offending' file without problem. The sig11s give an entry of the
form "cc1 caused memory violation at ...". In addition, the machine
rebooted once over the weekend (all other machines were not affected,
so no, it was no power outage!), unfortunately, no message in the
syslog that I can find. I finally finished the compilation of
egcs-1.0.1, and for what it's worth g77 compiles and runs a hello,
world program.

My question now is: Should I worry that I bought a machine with a
hardware problem. Two add. observations: There is only a 235W
power-supply, which seems low. The box is a minitower and the airflow
strikes me as suboptimal. The hot Barracuda sits pressed against the
CD-ROM, but it's the only bay where it actually fits. If anyone (in
Europe) happens to have a similar machine from transtec, please let me
know your experiences. Additional tests (of what to try) appreciated.
Transtec doesn't support Linux natively, they only have NT, so I want
to have a firm case before I start to make noise (although it's
absolutely clear from several little details that the machine was not
tested, let alone stress-tested before shipping.

Thanks,

Stefan
