Ive been trying to get an 164SX Alpha system configured with RedHat Linux
v5.0 on a system that has an IDE hard drive and IDE CD-ROM drive. I am able
to get into the RedHat Linux setup screen and then at that point, RedHat
tells me that I don't have any hard drives installed and that I probably
"forgot" to configure a SCSI card. Since this system is IDE and has no SCSI
cards, this leaves me puzzled.

The hard drives have been recognized in the firmware and even on the RedHat
bootup disk (sx164.img) so it seems Redhat does not have a problem
recognizing them during boot up but does not allow me to configure the
drives. I have tried going into the shell (alt-ctrl f2) and manually
fdisking the drive but it can't seem to find the drive as hda/hdb/hdc/hdd.

I know that RedHat supports IDE drives on the Intel platform so it makes me
wonder if somehow it is not supported on the Alpha platform, though I would
think that this is very unlikely. :) The system also has Windows NT
installed on another partition of the hard drive and it is working fine.

If anyone has any suggestions or tips, then I would appreciate it.

Thanks!
