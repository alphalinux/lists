Hello to all,

I've just started getting RH5 running smoothly on an alphastation200
4/100. I've updates all rpms in the update directory and also netools.

I've been trying to tailor the kernel to suit my needs but I've run into
serveral problems. I am using the SRPMS directory and rpm to get
kernel-alpha-2.0.30-2. I applied the alpha-patches distributed by
linking the "old" and "new" directory to the "linux-2.0.30" directory.

i)make mrproper and make config always works fine.
ii)make dep froze the alpha yesterday
iii)make boot crashed once or twice. It also issues a vast amount of
warnings when compiling the network files such as

tcp.c: In function `do_tcp_sendmsg':
tcp.c:1138: warning: `seglen' might be used uninitialized in this function

ip_output.c: In function `ip_build_xmit':
ip_output.c:560: warning: `fraglen' might be used uninitialized in this
function

etc etc. Also when I try to compile with all SCSI drivers it
systematically crashes when compiling those.

I am making with only the 53c810 SCSI driver since that is what MILO
recognizes as my SCSI adapter. I have tried various settings
pertaining to that driver but none make any difference. Also I am
requesting that all console messages be sent to COM1. That's about all
there is special to my config.

Once the kernel is created, I copy it to /vmlinux2.gz, go back to milo
and reboot on that kernel. It always hangs the alpha after it has loaded
the kernel and displayed various things. It stops when it has displayed
the current date and time.

I really don't know what's wrong !! Any clues?
