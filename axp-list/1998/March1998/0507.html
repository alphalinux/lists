The first version of cdrecord I tried to use was 1.6a7 and as you know,
it was very buggy concerning other than intel platforms...

I reduced SHM usage to 2 MB, rewrote these strange fill- and movebytes
completely and used HZ instead of 100 (ticks per second) to correct for
the wrong calculation of SCSI timeouts.
Also rewrote the swap code (which should really be called "swap" instead
of "swab" all the time) but that was more an aesthetic concern.

Since then (last december) it runs very reliable.
I had a look on version 1.6a11 an found most of my changes implemented,
but as there are no major new features, I didn´t try it.
