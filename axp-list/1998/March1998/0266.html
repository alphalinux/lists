Christopher Curtis writes:
>
> Why does this program:
>
> #line 3823 "configure"
> #include "confdefs.h"
> #include <stdio.h>
> main()
> {
> FILE *f=fopen("conftestval", "w");
> if (!f) exit(1);
> fprintf(f, "%d\n", sizeof(char));
> exit(0);
> }
>
> tell me that the sizeof(char) is 0 ?

Because it is buggy.

C standard says that type returned by sizeof() operator is of an unsigned
integral type and is implementation defined ("integral type" does not
mean, of course, 'int'). But it also says that when sizeof is applied
to any object of structure, union or array then the result gives a total
number of bytes in such object, including any internal padding if present.
On 64-bit Alpha this requirement obviously makes a choice of 'unsigned
int' for a type of value of sizeof operator unfeasible.

On 32-bit machines sizeof(sizeof_t) happens to be the same as sizeof(int)
and the code above does accidentally what you expect. If you would use
-Wall to a compiler gcc would tell you loud and clear what it thinks
about this code.
