Harvey Stein wrote:
> I have an application which builds & runs fine on linux/alpha RH 4.2,
> but when I try to build & run it on linux/alpha RH 5.0 it fails. It
> seems to build fine, but when I run it I get an arithmetic trap reported
> by the kernel, and the program crashes with a floating point
> exception.
>
> Anyone have any ideas what might be wrong and/or how to fix it?

Depends on what the application is doing: There were some problems im Linux and GLIBC
WRT to sqrt() and under/overflow. Richard Henderson submitted a patch for both which
appears to fix it, only one IEEE DEFECT is still reported by "paranoia" and i have already
submitted an informal bug-report. My last test was done with linux 2.1.91 and the glibc
snapshot from 03/15/98.
