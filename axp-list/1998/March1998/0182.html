Hello Jason and others,

Jason wrote:
} I just got a 164sx recently, and I've noticed some real weirdness with the
} clock. It seems the computer clock gains two seconds for every real
} second that passes.

Yep, me too. I already pointed this out to this list,
and various potential solutions were offered, unfortunately
still to no avail. But....

There is one 'solution': Boot the PC164SX with the milo file
gatekeeper.digital.com/pub/DEC/Linux-Alpha/Kernels/milo-sx164-31oct-2030+
and then run the pertaining kernel sx164-31oct-2030+.gz in the
same directory. For a still unknown reason this one combination
is rock solid.

I tried myself: other milos, including building my own, and
various kernels (2.0.30, 2.0.33, 2.1.85, 2.1.88) with various
alpha-patches, with varying effects: Up to 1050 BogoMips (Hey,
Jay, I beat your Pass-1 :-), and time skew from 100% (as Jason's)
to about 2% (still unacceptable).

Sofar I have come to the idea that apparently something for
that timing stuff should be set up in milo (or kernel booting)
that apparently is different for the SX. What I don't know,
but I do know that Jay Estabrook succeeded in doing so while
building the aforementioned milo/kernel combination. Thus, it
can be done. Now only to find out how.

I'm open for all suggestions.

Met vriendelijke groeten, Wim van Dorst
