I noticed that RedHat issued a new errata for glibc. Hoping that it
would solve some of my problems I grabbed the files.

The first 5 seemed to install OK (the ones that are glibc-*). The last
two initially gave me problems.

portmap-4.0.9.alpha.rpm Unable to locate file (from the ftp server via
the web site).

texinfo-3.12-1.alpha.rpm This file was 295 bytes and gave the "not an
RPM" error.

The five glibc files do seem to have solved one of the dependency
problems I had been having. The system now finds libc.so.6 where it did
not before. Unfortunately it has not solved similar problems with other
lib files.

libg++.so.27 => not found
libstdc++.so.27 => not found
libm.so.6 => not found
libc.so.6 => /lib/libc.so.6 (0x0000015555f7e000)
ld.so.1 => not found

Should I be making links from these "missing" files to existing ones?
I've tried but so far without success.

At least we're going in the right direction.

Any input on this from RedHat?
