Hi:

I have a problem with the combination of the QLOGIC SCSI adaptor and
the new Miata (DEC PWS433a) motherboard.

In this combination, when doing mke2fs, or writing to a dos
partition on the SCSI disk, following errors occurred:

> SCSI aborting command due to timeout...
> SCSI host 0 abort (pid xxxxx) timed out - resetting
> SCSI bus is being resetmfor host 0 channel 0
> ...

And the RedHat 5.0 installer also hang when mke2fs.

o The QLOGIC's chip is:

ISP1040B 2405101
MCL7L11/N9704D <- is this number important?

o The new Miata motherboard means the one which PCI-PCI bridge is in
PCI slot 20 (old one's is in slot 8), and the PCI bridge uses 21052-AB
(rev 2) chip.

o I'm using plane 2.0.33 kernel with alpha-patches-2.0.32-0.1 and the
patch posted in :

http://archive.redhat.com/axp-list/1998-February/0249.html

o The old Miata motherboard and the ISP1040B QLOGIC card has no problem.
And the new Miata and the AHA2940UW card also has no problem.

Any idea ?
