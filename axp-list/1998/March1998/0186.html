Hello,

For those of you who read my previous messages, I have finally solved =
the problems. Now I get to the install part.
I'm using an IBM DCAS-32160 2.1 GB harddrive and I'm installing through =
FTP. I have a tulip ethernet adaptater and a Miro PCI VGA card. Every =
time I install RedHat 5.0 or 4.2 the alpha (station200/100 with latest =
firmware) hangs or crashes at some point.
I'm trying to install RH4.2

First I had to specify heads/cyl/sectors but I didn't have the info so =
I used RedHat5.0 and diskdruid and it set them to 67/1017/62. Next I =
returned to Redhat 4.2 and partitionned and started the ftp download.
This systematically results in hanging the computer (either during the =
progress indicator or before) or, better, in a message "RPM Install of =
... : failed Unpacking of archive failed". It also crashed "sending =
termination signals" one or two times.

I've tried repartitionning every possible way. Including making the =
root partition over 1GB just in case the installation process was =
running out of space (where does the installer put everything by the way =
if I only set up a 60MB / and the rest on /usr ?).
I've used DiskDruid and fdisk. DiskDruid often hangs after I ok the =
configuration. I hung also when formatting the swap space (160 MB). =
fdisk hung once after making the io call.

I'm going crazy !! Anyone have a clue to solve my problem ?
Thanks
Philippe Girolami

*~*
Answers to my previous questions:
----------------------------------------------------

I had the impression the kernel hung after MILO juimped to it because I =
was using COM1 and apparently the kernel isn't built to dump to it. =
Using a monitor and a PS/2 keyboard solved the problem

Putting the "Parity Check" jumper on the Hard drive also rid me of the =
"bios32.sio_fixup: weird, device 8086:0484 coming in on slot 7 has no =
irq line !!"

To go into ARC on this alphastation (it has two Flash ROM slots): >>set =
os_type NT and reboot

As for the time problem, well I tried it the next dat and it worked !
