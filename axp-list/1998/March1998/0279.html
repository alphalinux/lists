> Uncle George wrote:
> > I have been unable to reliably to get GDB to work with C routines that

> As a temporary work-around (until this problem is fixed in gdb, or wherever
> it comes from), you could try to print the value of i using:
>
> (gdb) p *(&i-8)
>
> This will print the contents of the memory address 8 bytes below where
> gdb thinks i should be, which, in this situation, is the correct address
> for i.
>
> > can some ont tell me where i can get current info on alpha/gcc
> > produced/ asm directives
>

> look at
