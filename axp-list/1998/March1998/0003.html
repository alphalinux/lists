This is my favorite bug. I can't speak for redhat prior to 5.0 but, the 5.0
release serial driver is broken because of the definitions for CBAUDEX and
CBAUD. You can't set the serial port speed above 38,400. Trying to set it
to 56k gets you 0 baud and 115200 gets you 50 baud. I sent mail on the
hurricane list about a couple of months ago about it.

As it turns out the differences in the definitions of CBAUDEX and CBAUD
aren't in themselves fatal since I think the only code that should care is the
kernel serial driver. The part that breaks the serial driver is that the
serial driver is trying to be common code between axp and i386, but the
numbering of the baud rates are different. i386 numbers the baud rates from
0 to 15, then sets the CBAUDEX bit and goes on with 1 and 2, but the alpha
headers go 0 to 15, then sets CBAUDEX and uses 0 and 1. Guess what a
'common' source code serial driver does with this. Nothing good.
