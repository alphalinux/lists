On Tue, 17 Mar 1998, Buddy Cook wrote:

> On a recently purchased Multia MILO goes into a loop right after
> displaying the total memory. Loop starts with:
>
> lca: machine check (ls-0xfffffc0000208490,pc=0xfffffc000054bf20)
> Reason: correctable ECC error (short frame, retryable, dc_stat=0):pn
> Correctable read error to memory occurred at address 54bf30
> Several other correctable errors occurred.
> + 0 800000000000040 ffffffff00000018
> + 10 000000100000086 67f4daf900000213
> + 20 fffffff8054bf37 0000000000000000
> + 30 000000200000002 0000007100000071
>
> Loop repeats with a couple other memory addresses. The machine has
> 32MB of brand new parity RAM. I've swapped RAM with another machine,
> reloaded firmware twice, tried two different MILOs, and booted from
> floppy and hard drive. No change. Any suggestions? Thanks...

I think it's a memory problem. I had the same problem. Try reseating
the RAM, rearranging. Also check to see if the problem occurs at the
same location or similar places. The memory may work fine in a Pentium,
but I think Alpha Linux is too picky about hardware. Stuff that works
perfectly in AlphaNT AlphaLinux rejects.
