davet> If the method used by Goto is anything like Strassem's Algorithm, for n x n
davet> matrix multiplication, n should be > 100 to see any real benifit.

I uses unrolling, blocking, and taking into consideration of 2'nd cache
on my routine. I do not use special technique. Thus, if n is under
100, you can get faster result(e.g. if n is 40, it takes about
980 MFlops(600MHz 21164A machine)).
