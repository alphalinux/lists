While doing some backups with cpio, I noticed the following:

> cpio --version
GNU cpio version 2.4.2
> echo XFree86-3.2-5.src.rpm | cpio -o --format=crc > test.cpio
76538 blocks
> cpio -i --only-verify-crc <test.cpio
cpio: XFree86-3.2-5.src.rpm: checksum error (0x2b59b90c, should be 0x12b59b90)
76538 blocks

Is this a known problem? The archived data appear to be fine, only
the CRC is bad. This only happens on some large files. Given the
similarity of the bit patterns could this be related to the shift
problem brought up on this list some time ago?

This is on a RH 4.1 system.

TIA,
