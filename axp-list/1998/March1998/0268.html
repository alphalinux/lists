Okay, since at least one other person is trying to get MySQL to compile, I
figured I'd post a log of how to fudge it ...

First, in client/get_password.c, you must:

#define alpha_linux_port

I did this immediately before the #ifndef __WIN32__ line.
Furthermore, after "#include <asm/ioctls.h>" :

#include <asm/termiobits.h>

Second, in include/my_pthread.h, you must:

line 75:
#if !defined(HAVE_SIGWAIT) && !defined(HAVE_mit_thread) &&
!defined(HAVE_rts_threads) && !defined(sigwait) &&
!defined(alpha_linux_port) <-= add this

and in the folllowing files:
mysys/my_init.c
mysys/mf_keycash.c
mysys/my_lock.c
mysys/mf_brkhant.c
mysys/my_alarm.c
[...] and I get sick of .c files [...]
mysys/mysys_priv.h
mysys/thr_alarm.c

#define alpha_linux_port

The first 5 .c files probably don't need to be changed...

Third, in the file mysys/my_pthread.c around line 190

Create something that looks like this:

/*int sigwait(sigset_t *setp, int *sigp)*/
int sigwait __P ((__const sigset_t *__set, int *__sig))
{
#define setp __set
#define sigp __sig

... And that's about as far as I've gotten. I'm sure these hacks aren't
proper for the full source, but they make it further. Alpha-Linux is
listed as a beta platform, but at least it's listed. Once this gets set
by the configure script (sorry, I don't like messing with configure)
things should be okay. One word of warning, though, it will not compile
clean. I got a couple errors about TCP_NODELAY and such being redefined
(which doesn't really bother me as Samba says it can't set it anyway) but
those are probably trivial. Overall, though, it is a _very_ clean
compile until is stops.

Here's the final error message:

gcc -O3 -g -O2 -DDBUG_OFF -rdynamic -o test_fn test_fn.o -L. -lmysys
-L../dbug -ldbug -L../strings -lmystrings -lnsl -lm -ldl -lpthread
/usr/lib/libpthread.a: could not read symbols: Archive has no index; run
ranlib to add one
collect2: ld returned 1 exit status

... and ...

# nm libpthread.a | more
nm: attr.so: File format not recognized
nm: cancel.so: File format not recognized
nm: condvar.so: File format not recognized
nm: join.so: File format not recognized
nm: manager.so: File format not recognized
nm: mutex.so: File format not recognized
[...etc...]

So it looks like libpthread just ain't going to work here. Unless someone
can get a working pthread library on RH 4.2, only the folks who have made
it to RH 5.0 (or equiv) will be able to compile MySQL.

Shucks...
