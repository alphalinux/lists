On Thu, Feb 19, 1998 at 12:47:22PM -0700, Neil Henson T-12 wrote:
> Hmmm. I am finding that none of the exectables that I created under
> RedHat 4.2 are working any more under 5.0.

The libc shipped with RH 4.x was not an official release of glibc,
and is not binary compatible with the 2.0 release of glibc. Those
are the breaks for using snapshots.

In order to catch problems that this incompatibility would cause,
the sonames of the libraries were changed from e.g. 6.0 to 6.1.

It should be possible to arrange to be able to run RH 4.2 binaries
and RH 5.0 binaries at the same time. You'd create a directory
somewhere (/usr/lib/rh4.2 perhaps) and copy the libraries needed
by your application that resided in /lib, /usr/lib, and /usr/X11R6/lib.
You'd copy /lib/ld.so.1 from rh4.2 into /lib. You'd then invoke
your application like so

LD_LIBRARY_PATH=/usr/lib/rh4.2 myapp (bash)
env LD_LIBRARY_PATH=/usr/lib/rh4.2 myapp (tcsh)
