Gerald Brandt <gbr@hvdc.ca> writes:

> Hi there,
>
> I have some more f2c problems. Again, all this works on an Intel/Linux
> system.
>
> Alpha/Linux RedHat 4.2
>
> brand new f2c from netlib.
>
> Compiling a small(ish) program works like a charm.
>
> Compiling a VERY large program results in the following errors:
>
> /emtaux.f:3691: relocation truncated to fit: LITERAL c__100
> /emtaux.f:3691: relocation truncated to fit: LITERAL c__300
> /emtaux.f:3691: relocation truncated to fit: LITERAL c__40

I believe that's the problem with older versions of ld not being able
to cope with certain large sections in the executable file.

I think there're 2 solutions:

1. Split the code into a set of shared libraries, or
2. Upgrade binutils.

I don't know which version of binutils is sufficiently new to deal
with this.

In any case, the problem is unrelated to f2c in the sense that very
large C programs will cause the same problem (I had this problem with
bigloo).
