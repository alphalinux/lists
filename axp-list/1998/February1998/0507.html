Hello Richard et al,

David Woodhouse wrote:
} > I've ported the support for SX164, LX164 and Ruffian platforms
} > to 2.1.85, and included the em86 patches which were also on the site.

Richard Henderson wrote:
} I've cleaned these up a bit and sent them to Linus. I would be
} interested in hearing that SX164 and especially Ruffian still work.
} This is against 2.1.88.

I patched a new 2.1.88 kernel with it. After that I had to
do some little things to get it to compile:

- Makefile: comment out the SMP=1
- drivers/scsi/sr.c: about line 470: close the comment string, add */
- arch/alpha/kernel/bios32.c: ~8 times: remove __initdata

It booted up to the place where the Adaptec 2940 driver (aic7xxx)
apparently hung on 'scanning the A channel' where it normally doesn't
even stop for a second. I tried to locate the problem in this driver,
but it changed so much since 2.0.30 that I don't see an obvious error.

The conclusion is that on my SX164 with redhat 4.2, gcc 2.7.2.1, and
a working sx164-31oct-2030+ kernel from gatekeeper.dec.com, I can't
make a working kernel from 2.1.88 sources yet.

Met vriendelijke groeten, Wim van Dorst
