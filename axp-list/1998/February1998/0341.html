On Sun, 22 Feb 1998, Hugo van der Kooij wrote:

> 2.3. Patching the kernel
>
> Now we are going to apply that patches.
>
> But before we do we have to make some extra links so the patches won't
> hurt us.

:) Would you please elaborate?

> cd /usr/src
> ln -s linux old
> ln -s linux new
>
> Now those are in place we can patch the lot.
>
> cd /usr/src
> gzip -d < alpha-patches-2.0.32-0.1.gz | patch -p0
> gzip -d < alpha-RLIMIT-patches-2.0.33.gz | patch -p0
> gzip -d < alpha-em86-patches-2.0.33.gz | patch -p0

Not to bitch about your fine kernel compilation document, but really, why
don't you check out 'man patch' rather than trying to work around to
problem of only using patch -p0 all the time?

The "right" way to apply patches that include the original directories
'old' and 'new' would be to cd into /usr/src/linux. There, use patch -p1
instead of -p0 to strip the 'old' and 'new' directories away from each
file to patch.

So, you should have done like this:

cd /usr/src/linux
gzip -d < ../alpha-patches-2.0.32-0.1.gz | patch -p1
gzip -d < ../alpha-RLIMIT-patches-2.0.33.gz | patch -p1
gzip -d < ../alpha-em86-patches-2.0.33.gz | patch -p1

> And now we can remove those ackward links.
>
> cd /usr/src
> rm old
> rm new

... and we don't need to do this! :)
