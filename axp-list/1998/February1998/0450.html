I'm trying to use the snmpd daemon for Linux/Alpha.

When I connect to it with snmpwalk, running it either locally:

snmpwalk localhost public

or from another (Linux/Intel) system:

snmpwalk -v 1 alpha public

I get the response: No Response from alpha

On the Alpha, when I run snmpd in debug mode it spits out:

[root@alpha /root]# snmpd -d
received 46 bytes from 199.85.17.7:

1D 02 04 17 01 5C D6 02 01 00 02 01 00 30 82 00
0D 30 82 00 09 06 05 2B 06 01 02 01 05 00

over and over, but no MIBs are passwd back to
snmpwalk. I get similar responses when I use
SNMP clients such MRTG or Scotty.

I'm running Redhat 4.2 with a 2.0.30 kernel
and alpha-patches-2.0.30-0.3.gz. Also
using a recent milo.

I've tried this with cmu-snmp-3.3-2 which was on the 4.2 CDROM
and with cmu-snmp-3.3-3, hot off the Redhat FTP site.

Has anyone been able to use snmpd on Linux/Alpha
successfully, and if so how?
