> Coul it be the fact that I used swriteboot /dev/sdd jensen.img instead of
> copyboot.exe from the arc console, that make it fail ?

Yes, that won't work. copyboot.exe will merge an existing partition
label (bsdlabel, or minlabel style) with a boot image. jensen.img is
only needed when the redhat install is asking for the boot disk to
be inserted. Prior to that, follow the instructions given by
Michael Schwingen in his Jensen-INSTALL instructions. Use
jensen.gz as kernel, and ramdisk.img as root disk.

If your using swriteboot, which means you got already some AXP
system, I guess, you should follow the instructions of ther SRM
howto. This also involves using jensen.gz instead of jensen.img.
