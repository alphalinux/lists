> > You are comparing (compiling :-) apples and oranges. Compiling a kernel
> > for a 64-bit RISC machine is a totally different animal from making such
> > kernel for a 32-bit CISC. And optimizer is far from the same.
>
> Why is compiling kernels for different architectures so different
> from compiling user land programs for different architectures?
> (OK - sure there are different drivers, memory managment code etc. for
> the different architectures - but surely they wouldn't make that
> much difference?)

Um... The internal arrangement of Pentium is far different from that of
Alpha. Pentium is a RISC processor using hardware to convert any given
CISC instruction into a RISC, from complex to simple. The hardware does
much optimization on the fly without the program's knowledge, so the
compiler may not have as much to work at. With RISC, the compiler may
have to deal with register optimization. Also, because of the installed
base of Pentiums, more people spend time making the compiler more
efficient. The UDB uses a first generation Alpha, which I believe was
used to prove that the architecture works. Digital has gone very far with
the next two generations in terms of handling instruction parallelism,
adding more functional units so more instructions can be operated on
simultaneously, and increaseing clock rate spectacularly.

I'll admit I don't have enough info on the first gen. Alpha chips to give
better info, so if anyone knows better than I, please pipe up in this
discussion
