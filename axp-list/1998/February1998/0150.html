> I replaced the hard disk that came with the machine with a new bigger
> one on which I installed Red Hat 4.2. I defined the mouse as a PS/2
> mouse and double checked that /dev/mouse is linked to the PS/2 mouse
> port. When I fire up X, the mouse acts really strange. The mouse
> cursor jumps to the top line and kinda stays there but it will move
> horizontally.
>
> Has anyone else seen this problem? Does anyone have any suggestions as
> to how to go about fixing this problem?

This is a problem (for me) on RH4.2 on intel too. When the mouse acts
funny like this (hugging edges and corners of screens and jumping
significantly), this is an indicator of using the wrong mouse protocol.
The RedHat mouse configure program doesn't seem to work for PS/2 mice
(this on intel, I have not used 4.2 for alpha). I have not had this
problem on 5.0 (intel or alpha). I'm sure you should be able to fix it
without upgrading, but I don't know how. Perhaps there is no kernel
support for /dev/psaux protocol compiled in. Perhaps RH messed
this one up, mind you that setting PS/2 mouse support has been problematic
in Linux in the past.
