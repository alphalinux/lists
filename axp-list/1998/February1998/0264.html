On Wed, 18 Feb 1998, Mika Kortelainen wrote:

> > With 128M RAM (plus 128M swap), I didn't have any problem compiling (and
> > running) any version of kde I tried, including Beta 3. Perhaps you should
> > try to temporarily set up a swapfile for the kde compilation?
> >
> That's what I did. It seems to require about 150MB at one file,
> apparently that file contains some complicated classes or such...
>
> I have 96MB RAM and had at that time only 64MB of swap, no go.
> Then I added a swapfile of about 100MB and compilation finished
> nicely (and looking top showed that egcs took that 150MB).

Well. I have created the RPM's for the Alpha and did Beta2 on my XL-300
with only 32MB RAM and 100MB swap. It did swap but swapping took about 10
to 15 MB at that time.

Beta3 was done after an upgrade to 64MB. It still swapped a bit ( < 9MB )
but I was working on the machine as well at that time.

Although GCC C++ will require quite some memory it is workable. You
problem might be related to the fact that there is so much swapping going
on that the system may be to slow to perfomr other tasks at that time and
slowly crawl to a halt.

Too little memory can't properly be compensated with swap. marginal
memory can be compensate but will result in a slow system.
