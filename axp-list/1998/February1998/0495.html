I seem to have a big problem with my RH-5.0/AXP login program here. xdm
authentication works fine, but console login dies at after printing the
password with the following message (echoed on /var/log/messages)

Feb 27 11:23:27 Bounty kernel: login: memory violation at pc=15555b0e910
rp=15555eb6a3c (bad address = 00000000)

Running /bin/login from an xterm produces the following instead:

Feb 27 11:25:40 Bounty kernel: X(381): unaligned trap at 00000001201c89e4:
0000000120b8d38c 2d 21
Feb 27 11:25:40 Bounty kernel: X(381): unaligned trap at 00000001201c89e4:
0000000120b8d394 2d 21
Feb 27 11:25:40 Bounty kernel: X(381): unaligned trap at 00000001201c89e4:
0000000120b8d39c 2d 21
Feb 27 11:25:40 Bounty kernel: X(381): unaligned trap at 00000001201c89e4:
0000000120b8d3a4 2d 21

Should I worry about bugs or trojans? AFAIK /bin/login is taken straight
from util-linux-2.7-15.alpha.rp. However, running rpm --verify on
util-linux give the following:
% rpm -q util-linux
util-linux-2.7-15

% rpm --verify util-linux
..5..... /bin/login
..5..... /usr/bin/chfn
..5..... /usr/bin/chsh
..5..... /usr/bin/newgrp

% rpm --verify -p util-linux-2.7-15.alpha.rpm
..5..... /bin/login
..5..... /usr/bin/chfn
..5..... /usr/bin/chsh
..5..... /usr/bin/newgrp

Which seems to indicate failed MD5 checksum. Note that the last test
is done on a *fresh* rpm package just downloaded frpm ftp.redhat.com.
Can someone run the same and see if I am allucinating or it's for real?
