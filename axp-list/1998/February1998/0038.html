On Tue, 10 Feb 1998, Greg Lindahl wrote:

> > > [ I also sent in an installation bug to RedHat that they don't ship
> > > config's on the CD which match the kernels they ship on the CD... ]
> >
> > That is not a bug.
> > I have never seen a .config file shipped on any distribution.
>
> That depends on what you consider a "bug"... I consider it to
> be very annoying that it is not easy to change one small thing
> about my kernel. And you can see from the number of people who
> can't set their machine type properly that recreating .config
> isn't trivial. In fact, if you "make menuconfig" in RH5.0,
> LX164 is not even an option...

Which makes it obvious that `make menuconfig` is not working well. So your
bug report should report that. `make config` will allow you to create a
proper .config file and at present is the only safe way of doing it.

I have seen similar problems with `make xconfig` on the intel machines. It
seems to work but it is not failsafe (at best).
