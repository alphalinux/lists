Anyone have kermit running on redhat 5.0. I downloaded the redhat rpm
from columbia but had no end of problems, but it seg faults on connect.

First it wouldn't install because it required ld.so.1 and libc.so.6.
So I forced the install. Then since I had libc.so.6.1 I thought it was
worth a try so I linked libc.so.6 to libc.so.6.1, well to libc-2.0.6.so.
Next I tried linking ld.so.1 to ld.2.0.6.so. With this it almost worked.
kermit would come up, and I can set the line and baudrate, but as soon as I
say connect it segmentation faults. Looking at i386 redhat, it seems that
libc.6 was libc-2.0.5.so (maybe close enough) but ld.so is linked to an older
ld.so.1.9.5 which I don't have on the alpha.

I even tried rebuilding kermit, but the source rpm chokes badly with the new
headers.
