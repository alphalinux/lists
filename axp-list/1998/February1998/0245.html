Hi,

Having discovered that there is no difference between the Personal
Workstation 'au series' and 'a series' - it comes with both BOTH SRM
and AlphaBios installed and changing an environment variable in either
switches to the other - I am now booting with AlphaBios.

However the scsi disk is extemely slow (4.1GB Quantum UW with Qlogic
(PCI) controller). A process sits around for ages apparently waiting
for the disk to respond - there is no load on the system.
eg to dd a 100MB file to /dev/null takes <2 secs process time but
about 135 secs real time!! The same operation on the same box with
Digital Unix took <10secs real time (and about 1 sec total process
time).

This is slower than linux on a 120MHz pentium with an old IDE disk.

More importantly dumping from the SCSI disk is much slower than dumping
from the IDE CDROM on the Alpha! by a factor of 3!!

This is using RH5.0, it is the same using the Miata kernel from the
RH CD and with one I compiled (from the RH sources).

Other possibly relevent info:
There is a long pause (about 30secs) during start up after the line
'Mounting local file systems' which is not very surprising given the
slowness of the disk. It then complains about /proc being already
mounted for some reason when it tries to mount it here via the last line
of /etc/fstab.
The only other other strange thing I see during startup is

Feb 18 11:44:28 angst kernel: kernel: unaligned trap at
fffffc000042a5c4: fffffc000029e4e1 29 16
Feb 18 11:44:28 angst kernel: kernel: unaligned trap at
fffffc000042a5c4: fffffc000029e4e1 29 16

I don't know what this is about?

Does this behaviour sound familiar to anyone?
Any suggestions?

Mark Brown.
