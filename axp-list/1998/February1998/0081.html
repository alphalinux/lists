Apropos of FP weirdness.
Can someone tell me why the following program will get
a floating point exception (SIGFPE) ?

#include <stdio.h>
#include <math.h>
#include <assert.h>

int main(int argc, char* argv[])
{
int i, j;
double v,m; /* works fine with m=0.0 */

assert(finite(m)); /* passes this test */

for (j=-10; j<=0; j++) {
v=exp(-(double)j);
if (v > m ) { /* dies with SIGFPE here */
m=v;
}
}
}

System config is
kernel 2.0.32
libm-2.0.6.so
libc-2.0.6.so
cpu : Alpha
cpu model : EV56
PC164LX
