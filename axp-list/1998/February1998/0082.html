Francesco Callari wrote:
>
> Apropos of FP weirdness.
> Can someone tell me why the following program will get
> a floating point exception (SIGFPE) ?
>
> #include <stdio.h>
> #include <math.h>
> #include <assert.h>
>
> int main(int argc, char* argv[])
> {
> int i, j;
> double v,m; /* works fine with m=0.0 */
>
> assert(finite(m)); /* passes this test */
>
> for (j=-10; j<=0; j++) {
> v=exp(-(double)j);
> if (v > m ) { /* dies with SIGFPE here */
> m=v;
> }
> }
> }

Since "m" is uninitialized, it's anybody's guess as to what its initial
value actually is. In most cases, it will be some stack pointer
address.
This gets interpreted as an IEEE *denormal* number. Since this is not
an "infinity", it passes the "finite" test.

Note that if you build the program with "-mieee", you avoid the SIGFPE.
But in general, uninitialized variables are a bad idea.
