On Wed, 25 Feb 1998, Uncle George ( Garlic Pizza Specialist ) wrote:

> as of yet there are no char fetch/store instructions for the alpha gcc compiler ( for the
> 21164 processor there are instructions, but compiler hasn't been educated yet )

Well, gcc 2.7 doesn't have them, but egcs does, and probably gcc 2.8 too
(haven't tried it as people keep telling me how buggy it is).

Anyway, to tell egcs to compile a program with the byte/word load/store
instructions, add these options:

-mcpu=21164a -Wa,-m21164a

The first is to make egcs produce the instructions, and the second is to
make the assembler accept them instead of making macro expansion of them
(which doesn't even work as this used the $at register in a noat
environment). I would like it if egcs automatically sent -m21164a to the
linker when -mcpu=21164a is present (makes very good sense to me atleast).

You won't need a newer binutils for this support, Red Hat's 5.0
binutils package (version 2.8.1.0.1-1) does the trick. egcs is available
at Red Hat's contrib directory for hurricane as an alpha RPM.

I have had one other problem with egcs and that's profiling. Apparently it
expects the profiling code to be in a library called gmon, but it's
somewhere else. When you link with -pg it says this:

/usr/bin/ld: cannot open -lgmon: No such file or directory

So I made a bogus (empty) library called libgmon.a and linked it with
that. Worked like a charm! But it's not a very beautiful solution, this
should be fixed.
