Hmm, I have actually found that the linux client is faster on my
PC164-500. One strange thing that I have noticed though is that I have
to start the client several times to get the 'optimum' performance out
of it. I can start the client and have it process a 2^30 block in,
say, 7:24, block after block, like clockwork. I can then stop it and
start it again, and have it process the same size blocks in 6:56
consistently. My best effort in this department was about 6:45. My
_guess_ is that this might have something to do with the code not
'lining up' in the most efficient manner in the cache at certain
times.

Anyone care to take a guess at what might be happening?

And, no, there is nothing else running on the system at the time,
other than the 'usual' stuff. :)
