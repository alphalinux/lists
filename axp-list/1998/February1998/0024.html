This milo code compiles very cleanly, even on RH 5.0 (not using the dec
vga bios library, but I'm just using a TGA card, anyway), but I'm having
half-speed clock problems with the osfpal.nh in this distribution. I
don't recompile the palcode, I just use what's distributed. Also, I
didn't install the patch, since it's already incorporated in RH 5.0.

Watching xclock (w/ "-update 1"): it takes nearly two seconds for the
clock to tick.

I'm running on a 400MHZ PC164.

The last time I had this problem was when I accidentally recompiled my
own palcode (with the wrong gas). I switched to the distributed
palcode, and it worked. In that distribution of milo, there was not a
seperate directory for the pc164 palcode; it would just use the eb164
version.

The new palcode, in the distribution Jay mentioned above, does contain a
seperate palcode directory for the PC164. So, now I'm using the palcode
meant for my machine, but it acts like the palcode I mis-compiled in an
earlier version.

I copied my old eb164 palcode (that has been working) to the new 164
palcode directory, compiled it in with the new milo (from the
distribution Jay described, above), and time seems to be in step with
reality.

After all the recurring clock troubles I've had, I'm not taking anything
for granted, and need to know a good way to assure the clock is working
correctly. Any ideas?
