The UDB should auto-sense the network connection on boot-up. The order
it checks in is:

twisted-pair
thinwire (coax)
thickwire (AUI)

To check this, go to the SRM console

Type SHOW DEVICE

Check that ewa0.0.0.11.0 EWA) 08-00-2B-E2-C0-3B is there

Type

SET EWA0_MODE AUTO

This will make sure the network is in the auto mode. You could also type

SET EWA0_MODE TWISTED

To force TP mode.

If you have loopback connectors for the various network connectors you
could install them and use the

TUNET

command to check the operation of the port

Basically it is correct that if you have no link light nothing is going
to happen. If everything is configure correctly as above it should work.

Also, I know it is basic but check that you are going into a "normal"
port on your hub and not one set up with reversed connections to
"daisy-chain" to another hub.
