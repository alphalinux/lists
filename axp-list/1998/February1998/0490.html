Stephen Williams wrote:
>
> gatgul@voicenet.com said:
> > GTK? what this/that
>
> GIMP ToolKit (I think.) GIMP uses it. control-panel also uses it to make all
> the pretty little buttons you see. Well, you would see them if it didn't
> crash.
>
> It is a library that does X stuff. It seems to determine that the X server
> can do MIT-SHM, but gets an error when it tries to use it. The kernel update
> causes the SHM to work correctly, allowing all the GTK-base programs to
> start running.

Yes, btw, you can get the most recent versions of GTK @
ftp://ftp.gimp.org/
The GTK libs that are shipped with RH5 are rather old and outdated,
most recent version is 0.99.3.
While you're there, I'll suggest you'll grab a copy of the GIMP too,
as GIMP is a way cool PhotoShop clone (for free!), most recent
GIMP version is 0.99.18 (again the RH5 version is rather old).

You can get them all as plain src.tar.gz, src.rpm & bin.rpm
(all RH-5) for alpha as well PPC en x86.

p.s. unfortunally GTK is still rather buggy, best way to start
the GIMP is running it from an xterm like: 'xterm -name Gimp -e gimp'
GTK _can_ take your whole X session down, and this is a nice way
to avoid it ;-)
