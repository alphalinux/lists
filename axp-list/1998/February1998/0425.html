>does anyone know how to get gcc to do char/short instructions?

Pick up a copy of gcc-2.8.0, gcc-2.8.1 (released real soon now),
or egcs (any), and build it for your system. The binutils on RH 4.2
do _NOT_ recognize the new instructions, so you'll need to upgrade
those to be able to assemble code that uses said instructions.
Last time I checked, you have to explicitly tell egcs that you want
the default to be to generate those instructions. gcc configures seem
to assume that it should try to generate code using those instructions
if your machine will support them. In general, expect to see somewhere
between a 5-20% performance improvement in code which utilizes these
instructions.

>maybe the alpha 21164 has the 8/16 bit instruction - should i build two
>products - one for 21064, and another for the 21164?

I would say "don't bother with a 21064 version", but then, my job
is to create software to move _new_ chips/systems, so to me, the 21064
is irrelevant. To be perfectly honest, I think the PC164SX machines
are inexepensive enough that you can reasonably tell commercial customers
that they really need to have a machine with a 21164 in it, and then
charge extra for the 21064 version (after all, if you have to do
extra work to support it...)
