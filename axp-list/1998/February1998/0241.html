Christian,

I have not had the pleasure to set up diskless workstations. :)
However, using my knowledge of NFS and sysadmin here's my crack at
the problem. FWIW

First you need to set up a directory for each diskless workstation and
export this as read,write for only that workstation. Next, the workstation
will need to mount this as it's root directory structure. You can not use
the servers root for your workstations. Your workstations will have
different devices and configuration files, all of which is controled in the
root directory structure. If you tried to use your servers directory
structure they would all have the same configuration files, which for
example would make them think they all have the same hostname and ip
address. You could potentially set them all up to have the same /usr
directory, although this would depend on how you set up applications.

Hope this helps.

An idea I'd like to implement... please tell me if I'm off course (or
there's a better way).

I'm trying to boot diskless alpha/linux nodes.

It looks like vmlinux is already setup (given it's configured for an
NFS root) to mount the root partition across the network. My tests
actually failed, but I think that was because the bootp server was
running DUX... and the /dev part of the file system didn't sit well on
DUX. Anyway, it seems to correctly negotiate with the bootp server
and mount an NFS root before failure.

So, if I could get milo, vmlinux, and some bios/console all into
NVRAM, then this might work.

But, I was looking at DEC's debug monitor, and it looks like it will
netboot too! It seems limited to the DEC 21040 (tulip) and am79c960
NIC cards, but if that's not a problem, then it should be able to
retrieve milo & vmlinux to boot.

I have had success booting milo/vmlinux from the debug monitor,
although I've been told to only boot milo from the ARC console (that's
all DEC is supporting). This might be a problem for the future of
this idea.

I'd change the debug monitor so that it would timeout if no user
characters were entered, and go directly to trying to boot from the
net.

It would need to retrieve a file into memory that had both milo &
vmlinux images combined, then boot milo.

I'd change milo to find vmlinux in memory, then unzip the memory copy
of vmlinux, and boot it.

Any better way to do this? Any "gotcha's" you can think of?
