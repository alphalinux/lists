Hello, I've been trying to compile the example PAM app, but I'm having
trouble that I think is coming from /usr/lib/libdl.so.

First off, the program seg faults before ever reaching main (and gdb
complains crypticly about heuristic-fence-post. Further, strace seems to
indicate that the segfault occurs after closing libc.

If, instead of using -ldl, I explicitly reference the static
/usr/lib/libdl.a (oh, the irony), the program at least makes it as far as
main (though gdb still complains about heuristic-fence-post along the way)
and dies in the PAM call that does the dynamic loading.

But I'm determined to solve one problem at a time, and am sure (hoping)
that fixing the former will make the latter disappear.

Actually, when I started writing, I thought that libdl was the culprit,
but strace has shown that when the static libdl is used, libc isn't
dynamically loaded, so I'm now leaning toward libc.

The Errata page for 4.2 points at a new version of libc, and upgrading that
sounds like a good idea in any case, but that file seems to have
disappeared from the FTP site. Any clues on where it's gotten off to?

Just for the record the versions I'm using:
libpam: pam-0.57-4
libc & libdl: glibc(-devel)-0.961212-5 ** I'm not sure that's right...
Redhat Alpha 4.2:
% uname -mrspv
Linux 2.0.30 #1 Tue Apr 22 16:15:17 EDT 1997 alpha unknown
