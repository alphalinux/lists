>>> David Alan Gilbert said:
> Jonathan L Dubois wrote:
> >
> > I have an LX164 running stock redhat 5.0 + updates.
> >
> > When I defined ARC=false in /etc/sysconfig/clock, rebooting caused the
> > date to jump ~ 30 years into the future.
> >
> > With ARC=yes however, the date jumps ~30 years into the past.
>
> There are TWO problems here - and I think your fixing the wrong one!!
>
> Problem 1) There are two different standards for the format of date
> held in the hardware clock; thats what ARC changes. If you
> are exactly 20 years out then thats your problem.

If you're MILO from AlphaBIOS, then ARC=true is the appropriate setting.
Make sure that you set the correct date/time using the AlphaBIOS facility,
or set it under LINUX using "date" and then write it *in ARC format* using
"clock -w -A".

> 2) The RedHat distribution 'clock' program is broken and
> sometimes gives a broken value - there is an updated version
> somewhere which fixes it.

This is usually very noticeable; you get a real bogus date once in a while
while the rest of the time it's OK. Try:

while true; do clock -r -A; done

and if you see some bogus dates, get:

gatekeeper.dec.com:/pub/DEC/Linux-Alpha/Kernels/clock-pc164-rh50

and replace /sbin/clock with it.
