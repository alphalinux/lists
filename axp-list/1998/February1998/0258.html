On Wed, 18 Feb 1998, Jim Paradis wrote:

> Hugo van der Kooij wrote:
> > Following up on my own message I found that the binary RPM's list contains
> > one more RPM named ...... em86-fx-0.4.alpha.rpm! So where is the source
> > RPM for this driver? Could the maintainer put it on gatekeeper.dec.com or
> > at another good public spot?
>
> No. 8-)
>
> More specifically: em86-fx-0.4.alpha.rpm is the rpm for the x86
> emulation
> library. It's the one part of EM86 that I *cannot* distribute in source
> form. In addition to the library itself, it contains the include files
> necessary to build an application against the library. You must install
> this rpm in order to build em86 from source.
>
> I used to bundle the library and includes in with the em86 source
> itself... for the 0.4 release I separated the fx component out as a
> separate piece to make it easier to maintain both components separately.
> There *does* exist a src rpm for em86-fx, but it's strictly internal
> to Digital.
>
> I'll get around to writing the documentation for this at some point...

OK. Thanks for illuminating this. I will try if em86 builds well on RHL
5.0 sometime later.
