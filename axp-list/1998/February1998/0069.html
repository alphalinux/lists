(still no Java). We run Debian based on glibc2, and I guess there should be no
problem also on RedHat5.0?

I need only a small patch to em86.cc, the check for the "31 bit"
kernel patch is IMHO wrong, so I disabled it (the check is made on
stack variables, but we do not care as a special stack is created for
the x86 code, (and the kernel 2.0.33 with latest alpha-patches does
not try to apply the 31 bit limit to the initial stack).
