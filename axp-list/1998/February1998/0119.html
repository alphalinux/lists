Hi !

I want to thank the nice person who changed (AXP patches 2.0.32)

struct file_operations in linux/include/linux/fs.h

< long (*lseek) (struct inode *, struct file *, off_t, int);
< long (*read) (struct inode *, struct file *, char *, unsigned long);
< long (*write) (struct inode *, struct file *, const char *, unsigned long);

---
> 	int (*lseek) (struct inode *, struct file *, off_t, int);
> 	int (*read) (struct inode *, struct file *, char *, int);
> 	int (*write) (struct inode *, struct file *, const char *, int);


because of this my machine crashed completely (with my own kernel driver).


The change is ok; but my drivers must run on intel _and_ alpha.
it's no problem with "#define" but a hidden type like off_t would
be really nice.


Maybe the changes are really made in 2.1.xx but I can't switch
(and I don't look at) 2.1.xx because I have a SX-board.


BTW, it's great that the count-fields are unsigned now.
It's only theoretically good because read returns signed.
(it _must_ return signed, to return -ERRNO)


Ahhm, I haven't tried it; but is it possible to run a kgdb-client
on an intel-machine ? I currently have only one AXP-machine but
a lot of intel power.
