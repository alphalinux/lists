Friends,

Please help me. I've been attempting to get my new UDB on my home network
for about six weeks and I'm running out of ideas. I don't mind working to
get my first Linux machine operational but six weeks of failure can be
pretty frustrating. Here are the details:

1. I have one other machine on the network (Windows NT) and I'm quite sure
it is set up correctly.

2. The light on the hub does not light up when I boot my Alpha machine (the
NT machine does)

3. As I boot up the machine I get an "unaligned reference" error while
routed initializes. (I've seen many newsgroup postings on this and they
tell me that I shouldn't worry about unaligned references)

4. The machine can ping itself but not the other machines on the network.

5. The output of ifconfig that pertains to the eth0 card is as follows:
Link encap:Ethernet HWAddr: 08:00:2b:e4:e1:08
inet addr:10.1.1.2 Bcast:10.1.1.255 Mask 255.255.255.0
UP BROADCAST MULTICAST MTU:1500 Metric:1
RX yada, yada
tx yada, yada
Interrupt: 10 Base Address:0x8000

(Please forgive the fact that the tables are not lined up, my mailer stinks)
6. the output of netstat -r -n is:
Destination Gateway Genmask Flags MSS Window
irrt Iface
10.1.1.0 0.0.0.0 255.255.255.0 U 1500 0
0 eth0
127.0.0.0 0.0.0.0 255.0.0.0 U 7168 0
0 lo
0.0.0.0 10.1.1.1 255.255.255.0 UG 1500 0
0 eth0

7. After reading hundreds of newsgroup postings I found something about the
tulip driver not working for somebody in a similar situation. I tried
recompiling to kernel to use the de4x5 driver but the new kernel would not
boot.

8. Otherwise the machine works fine and I feel very cool running Linux on
an Alpha. However, coolness alone will not help me. I need this to work.

Thanks in advance,
Matthew S. Woodworth
