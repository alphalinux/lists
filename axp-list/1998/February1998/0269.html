A useful calendar program ical has a number of places with
a cavalier usage of types and suffers from using the current g++
for linking. Here is a fix.

Instructions:

- find ical-2.2-3.src.rpm on Red Hat 5.0 distribution
- do 'rpm -i ical-2.2-3.src.rpm' and go /usr/src/redhat directory
- uudecode and feed into 'patch' diff file below; it should apply
cleanly
- type 'rpm -ba SPECS/ical-2.2.spec'
- go to RPMS/alpha directories and install resulting .rpm files.
- save these, and a corresponding .src.rpm from SRPMS for a future
reference

Now it should behave.

Cheers,
Michal
