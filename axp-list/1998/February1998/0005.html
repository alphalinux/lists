Michal Jaegermann wrote:
>
> >
> > Compilation on Alphas for Alphas seems to be slow. Thus it takes
> > my 21164a/533MHz on LX about 6-7 minutes to compile a Linux kernel,
>
> Hm, your machine seems to be doing quite a few other things as well.
> On few various LX machines I tried a compilation time for a kernel
> consistently varied between 4 and 4 and half minutes - depending
> on options and few details. Do you have IDE drives?
>
> 'make -j 3 boot' may help a bit.

Thats worrying; thats the second benchmark which has come out low for
my Alpha against other peoples lately! (The other is Dhrystone - I'm
getting ~900K with good EGCS, Richard Henderson is well over the 1M).

I suspect the kernel compile time is due to memory; I only
have 64MB; could this be the problem? (Only! - 64MB on an Intel box
and you could compile a kernel with -j 5 and it still wouldn't swap
much). The hard drive is a Seagate Barracuda (its number starts
with 15....N) - gives about 6MBps on hdparm -t. The compile time is
using the standard RedHat gcc - I think it was:

make dep && make clean && make && make boot

(but it was some time ago that I did it).

> > which is about the same time it takes to compile on a Pentium-II 266MHz.
>
> You are comparing (compiling :-) apples and oranges. Compiling a kernel
> for a 64-bit RISC machine is a totally different animal from making such
> kernel for a 32-bit CISC. And optimizer is far from the same.

Why is compiling kernels for different architectures so different
from compiling user land programs for different architectures?
(OK - sure there are different drivers, memory managment code etc. for
the different architectures - but surely they wouldn't make that
much difference?)
>
> > Other native compilation tests are similarly disappointing.
>
> I would rather worry about a quality of a produced code. This may
> use some improvements.

It depends - if you spend a lot of your time tweaking code and
recompiling large chunks then the speed of compilation is important.
If you actually do something 'useful' then it may be very different.
