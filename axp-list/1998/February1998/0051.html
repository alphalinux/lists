Hello,

We have a PC164/533MHz on which we run linux 2.0.30. From
that machine I can ftp to any machines without any
problem. However, I can't ftp to this machine from
anywhere. I always get 'Login incorrect' (even root
or locally defined users - not using yp - fail):

Connected to Bounty.McRCIM.McGill.EDU.
220 Bounty.McRCIM.McGill.EDU FTP server (Version wu-2.4.2-academ[BETA-15](1) Fri Dec 12 10:42:56 EST 1997) ready.
Name (bounty:soucy):
331 Password required for soucy.
Password:
530 Login incorrect.
Login failed.
Remote system type is UNIX.
Using binary mode to transfer files.

I suppose that this must be a simple matter but I could
not find the problem. Anybody else had that problem?

We have the following entry in /etc/inetd.conf:

ftp stream tcp nowait root /usr/sbin/tcpd in.ftpd -l -a
