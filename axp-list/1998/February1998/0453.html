1. I installed RH4.2 with a struggle, but managed to get the system up.
I'm really a beginner at Linux, especially sysadmin stuff.
I consider myself lucky to have completed the installation and gotten
as far as I have. Now I have a few questions for all the gurus...
I've gone way past my 30-day installation period and can't get support
from RedHat. I wasn't far enough along to ask these questions before.
I have the 166MHz Alpha in the form of the Multia UDB, 32M mem, 2G SCSI
disk, and SCSI CDrom.

2. I just ordered a 56K PCMCIA modem for my multia (that has two PCMCIA
slots). Can you tell me step-by-step instructions for setting it up
so I can dial my ISP and do TCP/IP? How do I make /dev/modem be alive?
How do I tell it to dial out on /dev/modem to establish PPP with
the ISP that I have been dialing from my Win95 system (via PPP)

3. The console clock on my Multia won't let me save the correct date.
It forces it to the year 2018 - and then when my system boots, the
system time on machine is always off by a couple of decades.

4. My X-server runs, but the screen size seems to be wrong - it thinks
the screen is bigger than it really is, resulting in windows that hang
off the edge. Any idea what I did wrong or how to fix it?

5. I'm hoping to setup my Alpha as a gateway for my LAN to connect to
the real world (via dialin to my ISP) - Any suggestions on how to
get it right? Can I setup my alpha as a DNS server for the other machines
on my LAN?

6. I'm a bit nervous about upgrading to RedHat 5.0 - should I be? or
might it make things easier rather than harder? My concern is not
for the reliability of the newer software - it is probably better than
4.2. My concern is that my system works now, and "if it ain't broke,
don't fix it".

I'm really looking forward to getting my alpha on the net so I can get
updates by downloading -- that'll be so cool. I'll also be able to
have a web server going. Big changes about to happen here... I hope.
