Peter Cox wrote:

> But it should at least be able to perform as well as a Pentium 166 yes?
> I was under the impression that even without the fast floating-point it
> should outperform a similarly clocked pentium...

Well I think I'd expect a UDB to be similar to a P100 or P120 for
general stuff; a lot faster for floating point (unless your using
fortran and standard GCC in which case it will be abysmal).

> I have a P200 sitting next to it with less RAM (64MB as against 80MB)
> which has much faster compile times...

Compilation on Alphas for Alphas seems to be slow. Thus it takes
my 21164a/533MHz on LX about 6-7 minutes to compile a Linux kernel,
which is about the same time it takes to compile on a Pentium-II 266MHz.
Yet my Alpha will cross build a Linux/ARM kernel in about 3.4 minutes.

Other native compilation tests are similarly disappointing.

I'm presuming that the problem is that since Alpha binaries are bigger
the compiler has to do more work to generate and optimise them - and
what with the Alpha actually having some registers (unlike x86) it has
to do more work optimising the use of them.

Yes - I found this disappointing - I think I'll have to take gprof
to the compiler some day.
