>>> Eric van Dijken said:
>
> I'am trying to use a shared scsi bus, using 2 noname boards (both with an
> extra ncr810 controller) If set the scsi_id to "0" in ARC / MILO all, is well.
> As soon as milo boots, it will reset the ID back to 7.
> How can i make sure the kernel/milo won't touch the scsi_id on
> my controller.

We hard-wired MILO to default to setting the SCSI ID to 7 for any NCR
controller found; this was because certain ARC consoles set (or left) the ID
set to 0 on some platforms, which caused a lot of trouble (as you can
imagine).

However, you can try using the following to set the host ID to whatever you
want:

MILO> set SCSI0_HOSTID 2

You should get a message that says something like:

dev_config: setting NCR SCSI0 host id to 2

if it worked correctly.

The general format is SCSI<number>_HOSTID, where <number> is the index of
the desired controller (ie, if you do a "show" it'll number the SCSI
controllers scsi0, scsi1, scsi2, etc).

Good luck, let me know if this doesn't work.
