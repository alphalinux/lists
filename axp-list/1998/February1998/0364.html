I've got an EB164 system that is dead, i.e., I can no longer access the
SCSI hard disk. The system hung during a partitioning session with "fdisk"
to install RedHat Linux 5.0. It hung upon the "w" command, with a message
which said something like "issuing ioctl() to re-read partition table".
Upon rebooting the system, I got the following behavior:

ARC Executing the "Display Hardware Configuration" in the Supplementary
Menu hangs at the point where it would display the SCSI devices.

SRM: Behaves normally

Linux: (booting from floppy). MILO boots normally. Issuing a "show"
or "boot" command causes MILO to hang at the partition check,
where it says:

Partition check:
sda: sda1 < sda5 sda6 sda7 sda8scsi0 channel0 : resetting for second half of retries.
SCSI bus is being reset for host 0 channel 0.

Win NT: Normal boot until after the banner page which gives the version
SP and build number. Then dumps with the error message
"INACCESSABLE BOOT DEVICE"

Now, I was sure the disk hardware had failed or I had trashed the low
level format. So, planning a low level format and media check, I moved
the disk and SCSI controller to a PC, running Windows 95, and ran the
controller internal BIOS. I did a media check and got no errors! I
then installed the Qlogic Windows 95 driver. The system mounted the
DOS partitions (1 and 3) with no errors! Running scandisk on these
partitions produced no errors! I returned the disk and controller to
the Alpha, and got the same problems as mentioned above. I checked +5
and +12 volt power at the disk, both fine. I disconnected the narrow
devices, no change. I reconnected the narrow devices and disconnected
the hard disk, everything worked fine (except of course I couldn't boot
NT).

I'd run the SRM media diagnostics, except it turns out that you have to
have the Motherboard SDK to run them, which I don't have. I could also
install the disk and controller on another NT machine, which I don't
have. I'm about out of ideas... has anyone got suggestions?

The system particulars:

EB164 300 MHz motherboard
64 MB memory

PCI: Qlogic QLA1000 (ISP1020) fast/wide SCSI controller
Number 9 Vision 771 graphics controller
Cogent EM960 ethernet controller

ISA: Soundblaster Vibra 16 sound card

Storage:
Seagate ST410800W fast/wide hard disk
Exabyte EXB-8500 tape drive
Sony CDU76S 4X CD-ROM drive

Software:
BIOS Emulation V2.02
ARC Version 4.49-7
SRM Version 4.8
Linux Loader Version 1.5
MILO / Linux Version 2.0.30
Windows NT 4.0 SP3

1 Boot partition: 10 MB DOS
2 System partition: 250 MB NTFS
3 Swap partition: 200 MB DOS
4 User partition: 500 MB NTFS
