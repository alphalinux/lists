Hi Folks,

I´ve no problem to use unsigned numbers for UIDs and GIDs because we enforce a
uniform name space in our whole net. SuSE is using -2, but also the UCX NFS
server under VMS uses this sort of Nobody UID. The best is to use UIDs/GIDs in
the range from 0 to 32767 only. In my opinion 99/99 for nobody/nogroup is not
the best:
I´ve created as root on a Solaris 2.6 machine a file on a non-root exported
filesystem from a RedHat-Linux Alpha. Root squashing occurs and I´ve read:

#touch xxx
# ls -l
total 4
drwxrwxrwx 2 dat hmiduser 1024 Feb 12 1998 .
drwxr-xr-x 3 dat hmiduser 1024 Feb 12 1998 ..
-rw-r--r-- 1 99 other 0 Feb 12 09:04 xxx
#

In /etc/group on the Sun:

other::1:
nogroup::65534:

On the Alpha directly:

dat@dalph1.hmi.de-116>ls -l /chmi/dalph1/hmiduser/dat/nfstest/
total 0
-rw-r--r-- 1 nobody bin 0 Feb 12 09:04 xxx

On a SuSE 5.1 system over NFS:

dat@dixtat-78>ls -l /chmi/dalph1/hmiduser/dat/nfstest/
total 0
-rw-r--r-- 1 99 bin 0 Feb 12 09:04 xxx
dat@dixtat-79>

Very confusing.

The NFS server will use a name (nobody) but not a number for root_squashing,
but uses numbers only over the net. Using the number 65534 for nobody would be
the best here, I think.

Andreas
