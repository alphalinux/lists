Gerald/Harvey,

Do you really need shared libraries or just smaller subroutines
so the number of constants generated is smaller per routine?.
I build quite large binaries but most routines don't exceed
500 lines of code.
(I stick with egcs now that it has a reasonable g77 but did
use f2c to start with before g77 got the right answers)

One other thing Toon mentioned. If the code requires promoting
REAL's to double precision, why not just run something like sed
on it and just replace the definitions and add the standard FORTRAN
variable definitions explicitly to the code? I have probably 100K
lines of code I work with regularly and although all my code uses
explicit variable definitions already("IMPLICIT NONE" helps find
errant variables), it wouldn't take long to add that sort of thing.

Just a thought. I've had to move code between systems/compilers
before and that sort of thing was minor compared to re-doing the
I/O software (multi-GB file accesses, re-writing user interfaces)
since I just setup scripts to edit the source the way I wanted it.

Wes
