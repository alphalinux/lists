On Sun, 22 Feb 1998, Robert Williams wrote:

> Stephen Oberski wrote:
>
> > According to Robert Williams:
> > > When will the kernel support a packetfilter
> > > device, as does DEC Unix in /dev/pf/pfiltnn?
> > > Can a pfilt device be cludged to work now?
> >
> > Try "man 4 ipfw".
> > This is a network level packet filter system.
> > Works fine on the 2.0.X kernels in i386.
> > Never tried it on alpha.
> >
>
> I don't think ipfw creates a device in /dev.
> That's what I'm looking for. Some
> network utilities available only as binaries
> (etherman) open this device
> under Dec Unix. /dev/pf/pfiltnn must
> exist and be functional under Linux for
> these utilities to work there.

But then there is not just the case that you need the device but the
device also has to work exactly the same as with Dec Unix.

We have some nice tools under linux to control this so why rely on
binaries for other systems? After all this is a O.S. related issue that we
don't want to mess about.
