I have seen several messages regarding the issue that c++ does not work
well on RedHat Linux 5.0 but not seen a good soulution to cure this.

I must add the restriction to this cure that it will be restricted to
working RPM's or the possibility to creat working RPM's out of the code.

I did give EGCS RPM's a brief try but then the configure scripts don't
recognize it as a valid C++ environment.

So I would welcome suggestions that will keep as close as possible to a
clean RedHat Linux 5.0 system as possible.

(As I want to keep a clean system to build clean packages.)
