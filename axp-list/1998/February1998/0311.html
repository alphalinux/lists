Received a 164lx machine today, but fail in the first step
of installing Linux (RH 5.0, from Powertools CD-set). To be
more precise, the AlphaBIOS doesn't even switch to linload.exe/milo
but tells me that the OS selection is not valid...

In more detail: I have my milo - boot floppy (FAT), which contains
linload.exe and milo (lx164), generated from the lx164.img on the CD.
(Will check against gatekeeper whether there is anything more up to
date, but I believe my problem is one step earlier)

Following the latest MiloHOWTO I am aware of, and which appears
to be identical to the one in the RH 5.0 User's Guide, I have
set the following options for the 'Primary OS'

Boot name: Linux
Boot File: A: linload.exe
OS Path: CD: \
OS Options:

And yes, I have saved them. With the floppy in place, I get upon
trying to boot: "The highlighted OS is not valid. To modify an OS,
..."

Now the OS Path: CD: looks fishy to me, but I am not allowed to
change this. (When setting the options, I get the error as
described in the HOW-TO (Sect. 5.2), but note that this happens
later, when I actually try to boot. The system briefly searches on
the floppy and CD, then gives me this error. I don't have NT so
that I can verify whether all of the hardware is truly inctact.

Finally, the AlphaBIOS is 5.62-1a Rev. 971006-1435.

Hints etc. appreciated. Please don't tell me that I can run
only Windows NT on this box. (If it were absolutely required I
can get Windows NT via the campus license of the university, but
I have no need, neither desire for it...)
