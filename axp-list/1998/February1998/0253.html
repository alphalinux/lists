>>> Mark S Feldman said:
> I have a UDB (166MHz) running RedHat 4.2 (kernel 2.0.30 w/em86
> patches) and just installed a 2MB S3Trio64V+ card that I purchased
> from onsale.com.  I recompiled my kernel to ignore the TGA.  I
> downloaded the latest S3 X server (3.3.1 -- was running 3.3) and have
> no trouble whatsoever with 1024x768x8.  Rock solid.  When I run
> 1024x768x16, however, the machine hangs (dead quiet -- X cursor stops
> moving, no response from keyboard, no disk activity) after an
> arbitrary amount of time.  Power switch time.  I bought the card so I
> could have 16bpp.  And it looks good right up to the point at which it
> hangs.  Any ideas?


Try adding:


	Option "no_pci_disconnect"


to the Device Section of your XF86Config file.


Alphas and S3s seem to be particularly sensitive in this area... :-\
