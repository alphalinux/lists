I honestly suspect this is your problem. :(

I've become very distrusting of RedHat; unfortunately, RH4.2 isn't "up to
snuff" with running MySQLd (it may work, I'm not terribly sure - I haven't
been able to compile it for a while); evidently it needs more robust
threading present in glibc2. I've been running 4.2 on a UDB with no
problems, and this is running as a masquerading firewall with web, ftp,
telnet, mail (smtp and pop) and various other (finger, ident) services
running. Same is true for a couple other UDBs I know running RH4.1 If
you can hang in there for RH5.1 you may be better off, until then you're
either stuck trying to track this down yourself or dealing with it. If
you've been compiling your own daemons, make sure that you're replacing
the existing ones (RedHat likes to move things around) and that you leave
debugging symbols in. This makes it much easier for you to degug when
problems do arise.
