On Mon, 6 Jul 1998, Tim Rowley wrote:

> B. James Phillippe wrote:
> > But, to retrack the original topic: anyone here using a ZLXp-E2 on
> > RedHat-5.1 Linux text console in 640x480 have their red and blue colors
> > swapped?
>
> Perhaps... but all command-line tools I use output in shades of
> gray. Got anything good for displaying a palette in console
> mode?

Sure, this is what I use to test it (but DIR_COLORS and dialog have the
same problem):

#!/bin/sh
# - colortest
echo "Showing ANSI colors..."
echo [30mColor 30[0m
echo [31mColor 31[0m
echo [32mColor 32[0m
echo [33mColor 33[0m
echo [34mColor 34[0m
echo [35mColor 35[0m
echo [36mColor 36[0m
echo [37mColor 37[0m
echo [38mColor 38[0m
echo [39mColor 39[0m
echo [1\;30mColor 30[0m
echo [1\;31mColor 31[0m
echo [1\;32mColor 32[0m
echo [1\;33mColor 33[0m
echo [1\;34mColor 34[0m
echo [1\;35mColor 35[0m
echo [1\;36mColor 36[0m
echo [1\;37mColor 37[0m
echo [1\;38mColor 38[0m
echo [1\;39mColor 39[0m

An excerpt from DIR_COLORS showing what's what:

# Attributes
# 00=none 01=bold 04=underscore 05=blink 07=reverse 08=concealed
# Text color codes:
# 30=black 31=red 32=green 33=yellow 34=blue 35=magenta 36=cyan 37=white
# Background color codes:
# 40=black 41=red 42=green 43=yellow 44=blue 45=magenta 46=cyan 47=white
