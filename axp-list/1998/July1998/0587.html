On Sun, Jul 05, 1998 at 07:52:51PM -0500, William Bradford wrote:
> I'm running plain-vanilla Redhat 5.0 on an AlphaStation 200 4/166 with
> 80mb RAM and a #9 Vision 330 (S3 Trio64) video card.. however, when running
> anything that does a lot of DNS lookups (it seems), such as Mosaic 2.7b2
> or the compile of Mozilla (with Lesstif) mentioned on here recently, the
>machine will seem to block on a DNS call,Netscape will become a "blank" window,
> and the network interface on the machine (built in 10baseT) will seem to go
> "dead" - link light still on (on both ends, the machine and the hub) but
> I cant ping my own IP or the gateway of my router (Cisco 1003) on my local
> LAN, much less get anywhere else. Basically, all TCP/IP traffic is shot to
> hell until I reboot, which solves the problem (albeit temporarily).
> Any suggestions? I'm waiting until the RH5.1 Toolkit cd-rom set comes out
> to upgrade to 5.1.. if theres a newer kernel or patches I can apply to keep
> rh5.0 working until then, that would be a great help... now that we've got a
> mostly-working web browser for Alpha/Linux, I'd hate to have to move to DEC
> Unix if I can help it...
> Bill

Ok, found out that when this happens, I can run /etc/rc.d/rc3.d/S10network start

to bring the network back "up" - what happens is that the system loses all of
its route information (including default route) and doesent know how to get it
back until I restore it by running the network startup script.

Still, its annoying. Anybody know if a newer kernel solves this?
