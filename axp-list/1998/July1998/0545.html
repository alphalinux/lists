On Wed, 8 Jul 1998, Dan Roscigno wrote:

> Under heavy NFS load the automounter on my PC164LX machines
> generates so many messages that klogd and syslogd are using
> all of the CPU!
>

My take is that it's a /proc filesystem bug.

My fix (it works for me, YMMV) is to force klogd to use the
system call interface ( syslog(2) ) to the log ring, insted
of accessing it via /proc, by starting it with the -s option.

If you are using RH, apply the following medicine to
/etc/rc.d/init.d/syslog:

@@ -19,7 +19,7 @@
start)
echo -n "Starting system loggers: "
daemon syslogd
- daemon klogd
+ daemon klogd -s
echo
touch /var/lock/subsys/syslog
;;

Hope this helps
