I'm new to Alpha Linux and I'm having trouble installing my AlphaStation
200 4/233. I've looked for information, but so far nothing has worked.
I even tried RedHat tech support, but they haven't answered me yet. So
here goes...

System configuration:

CPU/System: AS 200 4/233
RAM: 32MB RAM
Video: ZLXp-E2 TGA, 8MB, 1024x768 @ 72Hz
CD-ROM Teac 532S 32x SCSI-2 CD-ROM (device 6?)
Hard Drive: Seagate ST15150N SCSI-2, 4GB (device 1?)
Console: ARC with latest DEC firmware (as200_v6_8.exe)

I set up a boot selection according to the directions in RedHat's Alpha
Installation Addendum. Here is what it looks like:

LOADIDENTIFIER Linux
SYSTEMPARTITION multi(0)disk(0)fdisk(0)
OSLOADER multi(0)disk(0)fdisk(0)\linload.exe
OSLOADPARTITION multi(0)disk(0)fdisk(0)
OSLOADFILENAME \milo
OSLOADOPTIONS boot fd0:vmlinux.gz

I created the three disks (MILO, BOOT, and RAMDISK) according to the
RedHat instructions, using the appropriate images for my system:

MILO: dd if=/mnt/cdrom/milo/avanti of=/dev/fd0 bs=1440k
BOOT: dd if=/mnt/cdrom/images/xl.img of=/dev/fd0 bs=1440k
RAMDISK: dd if=/mnt/cdrom/images/remdisk.img of=/dev/fd0 bs=1440k

When I try to boot the MILO disk, I got "File system not recognized." I
copied the avanti image (renamed "milo") and linload.exe to a DOS
formatted floppy, but I try to boot this MILO disk, a wierd flashing
green pattern covers the screen after the "Swapping to PALCode at 0x..."
message. The system appears to hang at this point.

I tried the install with the latest versions of the MILO image and
linload.exe obtained directly from DEC, but the install still hangs at
the same point. I tried the install with the video set to different
resolutions, but I still get the wierd green pattern. I also tried the
installation with OSLOADOPTIONS blank, but I still have the same
problem.

So here are my questions:

1. What is the correct MILO disk?

2. Are my settings correct?

3. How do I stop the screen corruption and the system from hanging?

4. Are there any other pitfalls I should be aware of?

Thank you.
