> Think about what you're implying here. Somehow, one of the most
> heavily used pieces of SW on a system is broken, glibc. That is
> indeed possible. Not the most likely thing though. Most likely, the
> applications you're using are passing bad arguments to the glibc
> routines which causes them to get errors. I run both a PC164 and
> a PC164LX and rarely see these sorts of problems. Unfortunately,
> I don't use mysqld or ipop3d so I can't comment on them. My
> bet is you have pointer/int bugs in your apps, whether you wrote
> them or not. If you can get a consistent failure, use gdb to run
> the code and see where it's failing (after building the thing with
> debug flags, -g, and making sure they aren't stripped). Then fix
> the code. Intel is only a temporary solution. When Merced comes
> along, it will have the same problems in 64 bit mode... Fixing the
> code is the only long term solution.
