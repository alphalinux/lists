The problem of the RH 5.0 UDB kernel and some kinds of network connections
has been well known for quite some time.

The "tulip.c" driver was chosen for the 5.0 kernels because, at that time,
it was (considered to be) the only driver that would work on all the DEC
TULIP chip-based cards and builtins. We know that to be wrong, now... :-\

The accepted work-around is indeed to utilize the 4.2 UDB kernel, which has
the "de4x5.c" driver which will work with all the possible connection types
of the UDB hardware.
