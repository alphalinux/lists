Any SCSI gurus out there?

I have a 164SX system, running RH5.0 with kernel 2.0.33 with the gatekeeper
patches.

I'm using an NCR53C860 controller with a hard drive, a Jaz drive, a scanner and
a Conner CTMS-3200 SCSI tape drive. The buss has an active terminator on the
external chain. Other than the following I have had no scsi-related problems
(that I know of).

The problem is that when tarring to the tape device I will get a "can't write
to device" error. This occurs at seemingly random places, never the same place
twice. Sometimes it gets almost all the way through the backup and sometimes it
errors out right away. I clean the drive before each backup. The tapes are new.

I'm using the NCR53C8XX driver.

I have read that when using a tape drive you must be sure to enable device
disconnects but make config never asks me if I want this or not. I have enabled
tagged command cueing and the synchronous speed is set to 20Mhz.

Any suggestions? I'm planning to add an NCR53C875 controller for a new
ultra-wide drive but I would keep the tape drive on the old controller.

I've thought of trying the NCR53c7,8XX driver to see if I still have the
problem, does this make sense?

I have not ruled out a hardware problem with the tape drive, I've had it for
quite a while.

Any suggestions appreciated.
