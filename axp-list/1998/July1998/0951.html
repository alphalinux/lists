> My system needs to have file-max set very high. What is usually a good
> number to set that to? Does the file-max have any relations to other
> things needed by the kernel? I'm just asking this because when I have my
> file-max left at default (1024) the machine crashes after a few days, but
> when I set it to 4096 or higher it will be up for about 5 to 10 days. It
> always crashes with the same error on console "no free inodes, contact
> Linus".

As I think someone's mentioned before, you really need to up the
inode max as well:

cat 4096 > /proc/sys/kernel/inode-max

There may be a kernel restriction that if you have N files open, you
need a minimum of N inodes. If we're seeing a bug where the kernel
doesn't actually properly close open files, then you're getting an
ever-increasing # of open files, and the inode limit is what you're
hitting. Once you've upped the inode limit, you'll go back to running
into the fd limit after being up for a while longer.
