This reminds me of the stuff under gcc/config/alpha/alpha.md.
That file describes the instruction set of an Alpha processor.
Maybe your flag asked g77/gcc to use an instruction mode that
is not defined correctly? Alpha instructions have several
suffixes that can be added to an instruction for FP exception
handling. I've been looking at Alpha assembler the past couple
days trying to figure out just how egcs is slowing things down
and what, if anything can be done to alter the scheduling rules.
Course this from a guy who writes C/FORTRAN for a living, not
compilers... Think I can write better assembler though than
egcs is generating for the important loops in my code by hand
if I can't sort out what the egcs scheduler is doing. Haven't
had to do ASM programming for a long time though.
