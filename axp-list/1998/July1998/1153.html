Hi,

I created a patch which allows current 2.1 kernels built with binutils
2.9.x to be booted from aboot. With the patch applied, aboot correctly
boots both compressed and plain ELF images. I have currently no way to
test ECOFF images, but hopefully they work right, too.

I created a RPM package which contains this patch as well as another one
which is needed to compile the program in glibc2 environment. The source
package is available at:

ftp://ftp.amg.gda.pl/pub/OS/Linux/people/macro/SRPMS.glibc21/aboot-0.5-5.src.rpm

The binary package may be recreated using the --rebuild option of rpm.
There is also the binary package available at:

ftp://ftp.amg.gda.pl/pub/OS/Linux/people/macro/RPMS.glibc21/alpha/aboot-0.5-5.alpha.rpm

but this one requires glibc 2.1 to be installed (I've already get rid of
glibc 2.0) and as such it may prove not to be very useful at the moment.

Brave testers, comments, etc. are welcomed.

Maciej
