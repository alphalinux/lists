> Is that hardware or software related?

SCSI drives normally have a few spare cylinders and they can
be told to remap duff sectors onto the spare cylinders if one
goes dodgy. However this is a selectable option which you can
flip with a SCSI command or two. I've never done it under
Linux.

> > (get the scsiinfo program)
>
> Ok. Do you know where to do this?

Probably on sunsite. Has a pretty Tcl/Tk interface - and
a command line....for those whose X won't work because of the
afformentioned flaky drive!

I think you use the 'badblocks' program to map duff blocks
out under Linux; but thats normally used on older non-SCSI
drives which didn't know about mapping sectors out themselves.
