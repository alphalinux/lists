On Tue, 4 Aug 1998, Christopher Curtis wrote:
> On Tue, 4 Aug 1998, LSL Information wrote:
> > >I think the POSIX support is fairly well documented (don't ask me
> > >where...) but the biggest problem is that it is especially weak.
> >
> > This "weak" subsystem is good enough to pass POSIX tests and for almost any
> > POSIX.1 code to port directly to it and run. Several companies have done
> > this and none of their products (admittedly not Open Source) suffer from
> > [g]libc problems, internationalisation issues and alike. They just work on
> > Alpha and Intel.
>
> Yes, I should have said "incomplete". Naturally I forget many details
> having little interest in the whole thing but NT's POSIX subsystem only
> meets POSIX levels (eg) 1.1 - 1.3 maybe, and nothing from 1.4 - 1.9 (which
> are primarily the shells, etc). Many vendors are producing full fledged
> POSIX tools that build on and enhance this subsystem from the shells and
> POSIX threads all the way to X11 iirc.
>
> I'm not really sure the jist of this thread -- is it NT's growing POSIX
> support threatening UNIX/POSIX from vendor perspectives? If so I suppose
> it's very real. NT can now (with 3rd-party tools) essentially run an
> entire GNU subsystem in its POSIX segment. Very Be-ish, but without all
> the coolness that Be.

Actually I know of no third-party tools built on NT's POSIX subsystem.
The popular posix-ish tools like cygwin32, uwin, mks, etc. are actually
built on the win32 subsystem, which is at least well-documented.

Softway's tools (Interix, formerly OpenNT) is built on a true POSIX
subsystem they developed to *replace* Microsoft's. Apparently the MS
POSIX stuff wasn't adequate, but I'm not sure why. The Softway product
is very nice, until you realize that it lacks symbolic links and shared
libraries. At least they support Alpha :|

Anyways... this is straying a bit off-topic. Just wanted to mention that
Windows will never be Unix any more than Unix can be Windows. We're
talking open vs. proprietary and they can never really merge...
