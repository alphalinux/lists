> Section 6.1.1 (page 90) of the "Official Redhat 5.0 Linux Installation
> Guide" says:
>
> "Only Red Hat Linux/Intel and Red Hat Linux/SPARC support
> modular kernels; Red Hat Linux/Alpha users must build a
> monolithic kernel (see Section 6.1.3 below)."
>
> which is plain enough.

But it is also plain wrong and it was wrong for a long, long time.

> However, I find that the kernel distributed
> with Red Hat Linux/Alpha has modular support compiled in, e.g., the
> file /proc/modules exists.

Actually currently even kerneld works just fine - which was a problem
in the past.

> On the negative side, I've built a module for my sound card, a SB 16,
> following the directions in the Readme files in
> /usr/src/linux-2.0.30/drivers/sound/ but cannot get it to function.

Some dependencies may be not resolved correctly. Sound modules were/are
always the most twitchy ones. Try loading first via insmod and see what
is unhappy and why.

> In addition to the sound module, I'd
> like to install the SCSI tape module,

If you will look at startup files then you will find out that Red Hat
always loads tape module directly, i.e. autoclean will not work. The
reason for that is that **some** tape drives do not like to have their
drivers loaded and unloaded dynamically. In such situation you do not
gain very much by loading tape as a module. Actually a memory usage
will be slightly higher (modules always use multiples of pages - which
is 4k on alpha).
