For those of you tinkering with bleeding edge kernels, has anyone
gotten a kernel with support for frame-buffer devices (2.1.107+) to run?

I can compile it fine by patching linux_logo.h (to include the mda
image), but any kernel I compile with fbdev support doesn't correctly
boot.

The bootstrap code seems to work. It correctly uncompresses the
kernel, then enables virtual memory mapping, then jumps... and that's it.
Not a single thing pops up after that.

This is on an SX164 using a Matrox Millennium II vid. card.
(533MHz, IDE [ick], 64MB SDRAM, etc.)

If I disable the framebuffer device entirely, the kernel boots and
runs just fine. There's no console paging, but I think that may have been
intentionally changed due to adding the fb stuff. I can't seem to find
anything in the initialization code that even calls the framebuffer stuff
(though since it switches _first_, it probably does really early on
somewhere).

Thanks.
