I am running RH4.2 on a lx164, the installation is clean with no changes
I have retrieved the latest srcs for svgalib from the SRPMS dir at redhat and
did the following:

1. installed with 'rpm' <no problems>
2. did *not* apply patches as they seemed
unrelated to alpha/lx164
2. altered Makefile.cfg to
- cutout unwanted cga chips
- turn on static libs
3. make clean <no problems>
4. make static <no problems>
5. use gcc --static -O2 prog.c -lvgagl -lvga <no problems>
prog.c =

#include <vga.h>
#include <vgagl.h>

main()
{
printf("hello\n");
getchar();
vga_setmode(G1024x768x256);
cga_setmode(TEXT);
}
6. run 'prog' and then:

Printf and getchar work, the video mode changes (to something) and the
computer either hangs or reboots. I have tried to put a gl_line statement
after the vga_setmode in an infinate loop but do not see the line. It seems
the computer goes into left field after setting the mode.

Do I need to do something special for alpha or lx164? I have been building
building svgalib and earlier redhat versions and hardware platforms.

Any tips much appreciated.
