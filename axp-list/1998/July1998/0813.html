Randy Carpenter wrote:
>
> I'm still getting the following whe I run a tape backup:
>
> DUMP: short read error from /dev/sda6: [block -557397338]: count=1024,got=0
> DUMP: bread: lseek2 fails!
> DUMP: short read error from /dev/sda6: [sector -557397338]: count=512,got=0
> DUMP: bread: lseek2 fails!
> DUMP: short read error from /dev/sda6: [sector -557397337]: count=512,got=0
> DUMP: bread: lseek fails
> DUMP: bread: lseek fails
> DUMP: bread: lseek fails
>

I use 8mm's locally without problems so I don't think it's
a tape problem. Maybe you found a bug in rdump? Odd that
it isn't repeatable but then again, I doubt your file system
is identical between runs so it could be data dependent.
Possibly something is not defined as a long and should have
been. Negative blocks/sectors are probably not good. Try
load the rdump sources and look where those numbers came
from and how they were defined.
