On Sat, 11 Jul 1998, B. James Phillippe wrote:

> I'm running 2.0.35pre6 with the Alpha patches from
> ftp.uk.linux.org. My system is Avanti (AS 200 4/233). I have determined
> that if I compile with CONFIG_ALPHA_SRM_SETUP, the de4x5 driver does not
> work. It detects my card (a DEC 21040) but with the wrong IRQ and IO
> address. The card is always detected at 0x8000 and IRQ15 normally (and
> works like a champ), but with ...SRM_SETUP it is found at 0x10100 and IRQ5.

The addresses are OK, it's the way SRM sets them. Maybe the IRQ routing
setup is still wrong with 2.0.35 -- the 2.0.32 Alpha patches used to be
incorrect in this matter. It was fixed in the 2.1 line some time ago.
