I just tried using "swriteboot -f1 -f3", which successfully destroyed the
partition table. I re-wrote the table with fdisk.

Does swriteboot not work with fdisk partitioned disks? The docs imply it
does. They apparently are reading different tables, though, because swriteboot
thinks partitions 1 and 3 begin at offset 0, while fdisk shows partition 1 doesn't
exist and partition 3 is offset 84 Meg from the begining of the disk.

This disk used to have digital unix on it, so perhaps there's some left
over data that's confusing swriteboot??
