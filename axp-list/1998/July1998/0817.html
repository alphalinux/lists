Michael Engel <engel@numerik.math.uni-siegen.de> writes:
> The first thing I tried was compiling aboot. The most recent version
> I found was 0.5 - could someone please confirm that this is the latest
> version?

Yes it is. I believe there is a version labeld 5.1, but the binaries
actually differ.

> After some tweaking aboot compiled fine on my AXPpci33 (the
> definition of memset in include/asm-alpha/string.h used __constant_c_memset
> which wasn't defined anywhere. Btw., I'm using kernel 2.0.34 at the moment).
> When booting aboot from a SCSI floppy drive, I get the following output:
>
> aboot: Linux/Alpha SRM bootloader version 0.5
>
> ?02 KSP INVAL
> PC= 00000000.00000000 PSL= 00000000.00001F00
>
> >>>

If I recall correctly, this is the same error I got on my Jensen when
combining aboot with firmware version more recent than version
1.7. You can search the axp-list archive, I posted about it a long
time ago. I also believe that this didn't happen with aboot-0.3, so
you might want to compare what changed between these versions.

In case you have an old firmware version (v1.7 or lower) available,
you could also try it immediatly.

HTH, Martin

PS.: I never bothered to look into this myself, because my video card
also hat problems with newer firmware versions.
