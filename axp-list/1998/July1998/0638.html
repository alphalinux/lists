When I get FPE's on my own code, I run it in gdb and see
what array is screwed up and fix it... Most FPE's are not
normal parts of an algorithm. Underflows are the only
exception I've found in my own codes that are expected.

If your povray run isn't too big, might try gdb and see
what got screwed up. From what I can tell, by default,
underflows are handled in HW. You use -mieee to insert
suffixes and trapb's on/around instructions with possible
error situations. Then, the kernel gets control and
fixes it up. Someone with more compiler knowledge
is welcome to correct me if I'm mis-stating what is
going on.

If you're using RH5.0, and you can make a tarball of the
povray program and your input to it, I can run it on
my PC164lx and see what's wrong. Actually, a tarball of
povray source and your test case, if not too big would
work better. Send it off line though.
