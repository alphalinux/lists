I've got binaries where I mix C/FORTRAN and am trying
to gprof them on a RH5.0 system. Strictly C programs
appear to profile fine after adding the '-pg' flag to the
compile/link. However, when I add this flag to g77,
it seems to convert it to '-p'. Not what I want to
have happen. Is this what it does?? If so, why??

My command from the make printout:

g77 -c -pg -O2 -fschedule-insns -fschedule-insns2 -v t31int.f
Reading specs from /z/home/alpha/lib/gcc-lib/alphaev56-unknown-linux-gnu/egcs-2.90.27/specs
gcc version egcs-2.90.27 980315 (egcs-1.0.2 release)
/z/home/alpha/lib/gcc-lib/alphaev56-unknown-linux-gnu/egcs-2.90.27/f771 t31int.f -fset-g77-defaults
-quiet -dumpbase t31int.f -O2 -version -fversion -p -fschedule-insns -fschedule-insns2 -o
/tmp/cca08161.s
GNU F77 version egcs-2.90.27 980315 (egcs-1.0.2 release) (alphaev56-unknown-linux-gnu) compiled by
GNU C version 2.7.2.3.
GNU Fortran Front End version 0.5.22-19970929
as -nocpp -o t31int.o /tmp/cca08161.s

Is there a way to stop this from happening??

Thanks.
