Jordan <jordan@sydney.healey.com.au> writes:
> As the SRM-Howto says, I have left space (about 2.5 meg) at the start of my
> scsi disk for aboot to be installed into.
>
> I have made my kernel and aboot binary and tools, but when I attemp to
> write the boot block to the disk, swriteboot says:
>
> error: bootcode overlaps with partition #1. If you really want this, use -f1
>
> I though that I would give this a go the first time, but as I expected it
> killed the partitions.

You probably used fdisk to partition the disk. You need to use a
minlabel style label. Either you the minlabel program, or the
bsdlabel compatibility mode of fdisk.

> I have now gone through the process of installing again, but this time I
> have read all of the installation notes and documentation, but it still says
> the same thing.

It semms that more and more people fall about this, so it would be
nice if you could contact the author of the SRM-Howto or Aboot, to
include a section that this all will only work with bsdlabel style
partitions.
