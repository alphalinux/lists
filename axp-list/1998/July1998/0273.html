On Sun, 24 May 1998 mgraffam@mhv.net wrote:
> On Sun, 24 May 1998, Christopher Curtis wrote:
> > On Sun, 24 May 1998, Scott wrote:

>>> this works as well. My question is this. Is the there any way to start XDM
>>> on startup, without having to log into the box and start it manually? I am

>> I don't know if RedHat has this setup properly, but typically run level 4
>> is for X-Windows;

> I don't know if RH has that set up properly or not, but why not save
> yourself the headache of finding out and just add "xdm &" to some
> startup script, like rc.local?

1) RedHat does have it set up properly. (At least, it's pretty much the
same as what I've seen on other Linux distribs :)
2) The run level for xdm is 5. Run level 4 is unused.
3) Use "telinit <level>" to switch run levels from the command line.
4) Use the settings in inittab because xdm will be restarted
whenever it exits...just like *getty.

> Makes more sense to me, and if xdm fails or X bombs or whatever you will
> still have a virtual console to fix up the system without rebooting.

Check out /etc/inittab. Boogy on down to where the tty's are spawned.
You'll notice that all 6 tty's are set to run at run level 5. If you
set 5 as your default, you won't lose your virtual consoles.
