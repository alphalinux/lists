>Can SRM be used to replace Alphabios/ARC? I thought there had to
>be more flash memory to do that which means only certain systems
>can have SRM on them. If it fits, can one swap between them some

>What's the scoop?
On some systems there is enough room in flash for both, and on some only
for one.
For example, the 2100 server comes with flash big enough ( and maybe SRM
and ARC code small enough) for both to fit.
A modern LX motherboard will NOT take both. It's one or the other.
By using the appropriate fwupdate.exe and the .rom file for the firmware
one can flash in one or the other BIOS. From Alphabios or arc it is done
with the "update firmware" menu entry. From SRM it is done with the
fwupdate command.

So, you get a floppy ( or CDROM) with the needed files, run the update, it
asks you for confirmation, then it reprograms the flash, verifies it, and
then it tells you to power-cycle the machine.

That is basically "it".

If you lose the image somehow ( cosmic rays, error, whatever) it gets more
complicated, as you then generally have to set some jumper(s) on the board,
hook up a terminal, do a failsafe load from a floppy, and flash the
firmware from that. Ugly, but doable.

One nice thing about the newer AlphaBIOS boards, like the LX, is that you
can set one jumper ( J7 on the LX) stick in a floppy with the .rom file,
and it will bootstrap from the floppy file. Makes life easier.

One other note: Sometimes when you flash SRM in to a machine, after you
power-cycle it, there is no video output. If htat is the case, hook a
terminal ( or term program on another machine) to the srial prt, and if you
see the output there, then from that command line you do a "show co*" and
see if the console mode is set to "serial". If so, then simply type "set
console_mode" to see your choices, and then pick the right one with a "set
console_mode" command. Usually this is "vga", although it varies on some
models of Alpha.

Hope this all helps you... <whew>.
