> Is there a way to monitor the number of open files . I am familiar with
> commands like pstat or sar found in some implementations of unix. In the
> past, on other systems I have had to increase the following:
>
> number of open files (system wide)

cat /proc/sys/kernel/file-max
echo 8192 > /proc/sys/kernel/file-max

> number of open files (per user)

Can't be usefully changed -- limited to 256. glibc doesn't dynamically
allocate that data structure, so if you increase it in the kernel and
recompile it _and_ glibc, all processes get bigger :-(

> Max number of processes per user

/usr/include/linux/tasks.h

Change NR_TASKS, recompile kernel. Look in limits.h to see if
CHILD_MAX needs to change; it defaults to NR_TASKS/2
