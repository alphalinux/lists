Christopher Curtis wrote:
>
> What _actually_ happened was that I
> was installing Linux on a Sparc with an internal drive set to SCSI id 3.
> This was called 'sda'. I then attached an external drive at id 1.
> Woohoo, the system now will not boot because it cannot mount the root

There is 'devfs' patch in an active and constant development
which provides a Solaris-like scheme for numbering devices.
Much less intrisive, but with similar effects for SCSI devices,
is 'scsidev' set of tools which can be found on tsx-11 as an ALPHA
software (no, not for Alpha processor, but as an ALPHA-release).
