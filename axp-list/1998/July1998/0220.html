On Thu, 24 May 2018, K. Spoon wrote:

> 1) RedHat does have it set up properly. (At least, it's pretty much the
> same as what I've seen on other Linux distribs :)
> 2) The run level for xdm is 5. Run level 4 is unused.
> 3) Use "telinit <level>" to switch run levels from the command line.
> 4) Use the settings in inittab because xdm will be restarted
> whenever it exits...just like *getty.

I'm curious ... is there a standard for inittab layout? Some of you may
recognize my suggestions as being very Solaris-y. For those unaware,
SunOS/Solaris do:

0 halt (go to firmware)
1,s single user
2 multiuser
3 multiuser w/NFS
4 X (+multiuser w/NFS)
5 power off
6 reboot

What is the "Linux standard", or is there one?

0 halt (power off?)
1,s single user
2 multiuser
3 multiuser w/NFS
4 *unused* ?
5 X ?
6 ?
