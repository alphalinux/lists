Greetings,

I run a Digital "Multia" UDB as a low-end web server. I've run into a
problem, though - on boot the machine hangs/stalls during the network init
process. Oddly enough, this never happened when I was running the stock
2.0.30 kernel... the machine was a very reliable web server... but since
upgrading to 2.0.33 the network init completely fails.

I've spent a great deal of time going through the axp-list archives for
clues on how to solve this problem. As it happens, I thought I'd found the
solution when I came across an old post regarding this matter which
suggested terminating the BNC ethernet port. This trick, to my surprise,
actually worked for a time... there would only be a short pause during
network init, and then the machine was up and running fine. However, this
fix wasn't the solution to the problem. Yesterday evening I added 3 new IP
routes, (which I properly did via the appropriate eth0:0, eth0:1, and
eth0:2 config files in /etc/sysconfig/network-scripts), and proceeded to
reboot the machine to ensure that these new routes were activated at boot.
As you can probably guess, it stalled during the network init.

Last night I came across some other vague references to this problem in
the axp-list archives... a couple of suggestions revolved around the the
order in which interfaces are initialized... which is apparently
alphabetically? A couple of people suggested renaming the loopback config
file in /etc/sysconfig/network-scripts so that it would be read and
initialized before the ethernet interfaces.

Does anyone know if the above solution works? I would test this myself,
but unfortunately the machine was recently shipped to a remote network,
and I've been running it from afar. The folks at the ISP where it's
located have no experience with this problem either, so I'm hoping someone
on this list has a solution.

Thank you in advance,
Jason Kelly
