>>> Secret Asian Man said:
>
> I boot into the SRM console, and it tells me that I have 32M of RAM (true)
> When I load MILO (via floppy made through milo.dd from Digital) MILO tells
> me that he only has 24M of RAM (yes, I do believe that Milo is a male
> name) and when I load up the Kernel, I get 22M of RAM (I know that the SRM
> takes some of the RAM...I anticipate this loss of RAM) But I'm still
> wondering where that 8M went!

Normally on the UDB, MILO is booted by linload.exe from ARC console. MILO
expects linload.exe to pass it the correct memory size.

By using milo.dd from the SRM console, you are not using linload.exe to run
MILO, so MILO doesn't get the correct memory size, and defaults to 24Mb.

There's a few different solutions, depending on personal preference:

1. pass MEMORY_SIZE=32 to MILO before you boot the kernel

2. boot the kernel directly from SRM console, bypassing MILO completely

3. install ARC console and use MILO as it was intended :-)

Drawbacks and details
---------------------

1. this requires you typing it in every time, as I know of no way to pass
arguments from SRM console to a MILO from a milo.dd.

2. you'd need either an SRM-bootable floppy, or a harddisk with the bootstrap
installed. see the Alpha FAQ's SRM booting info at:

www.azstarnet.com/~axplinux/srm.html

3. the ARC console firmware upgrade is available on gatekeeper.dec.com:

/pub/DEC/Alpha/firmware/interim/multia

Good luck.
