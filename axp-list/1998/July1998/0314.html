Mike \"Ford\" Ditto wrote:
> What's the deal with the UDB's TGA hardware? I've heard
> conflicting explanations of its 8-bit limitation; is is
> hardware or software? Will there ever be an X server that will
> do 800x600x24bpp (or even 16bpp) on the UDB's builtin graphics?

The UDB uses a PC-style ramdac (Bt485) which appears capable of
running in 16- and 24-bit modes. At a quick glance of the 21030
manual, it seems that if acceleration is added to the XF86_TGA
server the 16-bit mode will be unaccelerated.

The changes to add more depths to the 8-plane version are fairly
straightforward, but I can't test them here (only have a 24-plane
TGA and a TGA2).
