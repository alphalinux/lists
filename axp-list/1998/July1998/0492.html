The "one space off" problem only seems to manifest itself if you're
booting from the ARC Console. It doesn't do it if you boot from the SRM.
I believe that it's more closely related to the TGA version of Milo(?).
I found this while I was setting mine up. I went through an
exhaustive set-up, tear-down, excersize, in which I was very involved
with both the SRM, and the ARC. So the long and short of this problem
is:

If you're booting from the ARC, you will see the "one position
off" problem.
If you're booting from the SRM, you won't !

Thus... It's not related to the graphics card "directly". So
there's the challenge. What's causing it? could it be that there is a
problem with the TGA version of Milo? It's only used under ARC...

Just my 2 cents...
