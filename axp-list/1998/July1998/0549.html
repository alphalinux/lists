On Wed, 8 Jul 1998, Michal Jaegermann wrote:

> >
> > Let me get this out first: perhaps I am an idiot for trying to
> > build 2.0.35 (pre-5 w/Alpha patches) using egcs; I've tried 1.0.2 (stock
> > RedHat-5.1) and 1.0.3a RPM. If this is the problem stop me now. :)
>
> >From what I know "rpm" egcs is not the same as the "real" 1.0.3a, but
> maybe I mistaken.

That would really scare me..

> Not directly a solution to your problem, but some data point.
> Just yesterday I run a bit of an experiment - three machines on two
> different networks with Linux/Alpha supplying ethernet aliases on the
> same physical interface and doing masquerading (NOT routing between

Hmmm... masquerading works for you, eh? I don't have the foggiest clue why
it isn't working on my system.

> > Otherwise, I can build a kernel and it boots up fine, but there are
> > a few things which just do not work at all. First, my DE4x5 PCI ethernet
> > card is detected but doesn't work (never gets an interrupt, and it's oddly
> > detected at a different IO address),
>
> Some newer cards of that type want to work only with tulip driver and
> you may want to check CESDIS site for the current versions.

If this is a driver problem, my first guess would be that the newer de4x5
driver is for some reason not working with my card, and the older one is
(they are different in 2.0.33 and 2.0.34+). This is an older de4x5 card.
But what really concerns me is that the IO address is _different_ when I
boot into 2.0.34+ than it is in 2.0.33. I'll try again with
CONFIG_ALPHA_SRM off and see if that's the problem.

> > and second, all ipfwadm commands fail
> > with "setsockopt: invalid argument".
>
> This second would follow from the first.

I don't believe that; the ipfwadm commands should only fail with setsockopt
if the core firewalling code of the kernel is broken, not just because an
interface isn't getting interrupts. You can add IP firewalling commands
for interfaces that aren't even present in the system yet, like ppp0 before
you've connected. I don't have a clue what could be causing this problem.
