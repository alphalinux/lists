Hi Jordan,

Here is some information on interpreting the SRM console device listing.
(sorry if this doesn't come out lined up)

2 4 6
| | |
dka0.0.0.0.0
\/ | | |
1 3 5 7

1. Driver ID The first two digits
Two-letter port or class driver designator
DR-RAID set device
DV-Floppy drive
ER-Ethernet port (EISA)
EW-Ethernet port (PCI)
PK-SCSI port, DK-SCSI disk, MK-SCSI tape
PU-DSSI port, DU-DSSI disk, MU-DSSI tape

2. Storage Adapter ID: One-letter storage adapter designator (A,B,C,...)

3. Device Unit Number: Unique device unit number (MSCP unit number)
SCSI unit numbers are forced to 100 x Node ID

4. Bus Node Number: Bus Node ID

5. Channel Number: Used for multi channel devices.

6. Slot Number: {The following may vary according to architecture, this
is for an Alphaserver 2100}
+ For EISA options-Correspond to EISA card cage slot numbers (1-*)

+ For PCI options-Slot 0 = Ethernet adapter (EWA0)

+ -Slot 1=SCSI controller on standard I/O

+ -Slot 2=EISA to PCI bridge chip

+ -Slot 3-5=Reserved

+ -Slot 6-8=Correspond to PCI card cage slots: PCI0, PCI1, and PCI2

7. Hose Number: 0 PCI_0 (32-bit PCI_); 1 EISA; 2 PCI_1

For those machines with one built in scsi controller it's disks will be
labeled dka0,dka100,dka200,dka300,dka400,... depending on the scsi id
number.
For those machines with two built in scsi controllers, the first will be the
same for one, and the second will be labeled
dkb0,dkb100,dkb200,dkb300,... again depending on the scsi id number.

When you do a show device:
dka0.0.0.1.0 DKA0 RZ26L 00B2
dka200.2.0.1.0 DKA200 RZ28B 0F28
_____________ _________ ______ ____
1 2 3 4

1. Already explained see above.

2. Logical name for device, generally a subset of 1.

3. Device model as reported by the device bios.

4. Device firmware level as reported by device.

Warning: This information pertains only to the SRM console.

Calvin.
