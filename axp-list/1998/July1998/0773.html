> Hello,
>
> I'd like to learn more about finding and fixing unaligned traps on
> Linux/Alpha. I'm an accomplished C programmer, but frankly have very
> little know-how on AXP. I understand the concepts and principles behind
> what an unaligned access is, and I can recognize it in the code if I know
> where to look. My main hurdle now is knowing where to look. I understand
> that the two addresses printed in the printk are 1.) the address improperly
> accessed, and 2.) the address of the accessor. What do the other values
> represent, and how could I use gdb to find the offending line(s) of code?
> I've done some net searches, but turned up very little.

Looking into the kernel sources would have told you something too..

Lets have an example:

X(31263): unaligned trap at 000000012004e50c: 000000012076ddb2 28 1

This means that program called X (pid 31263) has done unaligned
access with Alpha instruction 0x28 (LDL -- 32-bit load) on PC address
0x12004e50c while loading from 0x12076ddb2 into integer register 1.
