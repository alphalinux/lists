On Sun, 5 Jul 1998, Buda Sandor Attila wrote:

> I heard somewhere that the SRM console recognize only bsd style partitions.
> I used the fdisk with with <b> (bsd style).But the redhatinstall didn't
> see anything.I think the install searched for non-bsd partitions or
> something like that.

This is correct. What I did for my install was to first partition using
the PC-style partition table. Write down the cylinder start and end offset
for each partition. Then write the table out, go BACK into the fdisk
"EDIT" screen, and apply the BSD disklabel partitions to exactly match the
PC-style ones. Make sure that you leave some space at the beginnnig of the
drive for the aboot bootloader (in my case, I started my first partition on
cylinder 2 rather than 1). Then continue with the RedHat install and when
you're done, use the swriteboot /dev/sda /boot/bootlx command.

Hope this helps,
