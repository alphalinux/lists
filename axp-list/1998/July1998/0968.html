I'm setting up my UDB as an X-Terminal, and I'm having some trouble.
Seeing as how there /might/ be a bug in the TGA server, let me start...

(Note that a newbie thinking there might be a bug doesn't really carry
that much weight, and it shouldn't...)

I've added the UDB to my xdm-setup on the host (host=real machine),
and on the UDB I have this in my /etc/X11/xdm/Xservers file:

:0 local /usr/X11R6/bin/X -query shrimp (shrimp is the host)

When I start xdm, it loads the X Server, then flashes on the screen the
xlogin thing for "shrimp", then before I can do a thing, flashes back the
the text console like something's gone wrong (which is true) then goes
through the whole cycle again, making the machine unusable until I can
power cycle it (no reset button)...

BUT...When I directly run the server with the -query shrimp, it loads fine
except that after I log in, it doesn't know what to do, shuts down the
server, and starts over again, leaving me at the xlogin prompt again.
I've tried fiddling with my .xsession file to no avail. But the important
thing is that it actually makes the connection to shrimp this time. Before

broken or somehow rejected.

So, two problems, but the first one I really really want to fix!
