I just compiled (with -non_shared) a molecular mechanics package on an
Alpha running Digital Unix (v. 4.0) and transferred it to my local
Alpha/Linux machine. From my preliminary results, the code
runs OK. However, when running (certainly on startup of the program,
but often several times during the execution of the program), I get
several of the following console message:

<sc 49(140ca7de0,40,c)>

and in the syslog the following type of message gets written:

May 25 15:15:05 delta kernel: Cannot read proc file system: 9 - Bad file descriptor.
May 25 15:15:36 delta last message repeated 207562 times
May 25 15:16:36 delta last message repeated 415333 times

A 'file' on the executable gives:

../exec/alpha/charmm: COFF format alpha executable paged not stripped - version
3.11-10

The local (Linux) machine runs RH 5.0 with patches up to beginning
of May, kernel 2.0.33, no modules.

Insights regarding the meaning of this message is appreciated.
(There may well be nonaligned memory accesses in the code, but those
error messages usually are a little bit more informative)

Thanks,

Stefan
