If you use multiple SCSI cards, you also specify
which card, 0,2,0. Since you can query the bus to
find out what each device is, I see no problem with
the way cdrecord under Linux numbers things.
Seems to me the man page is quite clear how to use
it under Linux. I've used it about a year now...

>
> > > Do you know why it's so low?
> >
> > No idea. Actually it seems to be something like cdrecord default.
>
> You're saying that cdrecord specifies what values should be in
> {linux}/include/asm-alpha/shmparam.h ?
>

It would be nice if the SHM limit were settable thru
/proc, similar to file descriptors....
I've run into SHM limits on other UNIX before so
it doesn't surprise me that cdrecord has ways of
changing how much it tries to use. The systems I've
used, usually have a way to up the SHM segment size
limit. I don't off hand see why cdrecord would need
such large buffers for feeding a 2-4x CD recorder.
That's 300-600KB/sec, not exactly fast I/O. I drive
8-10MB/sec tape drives near full speed on networks
with less than a MB of buffer on each end...
