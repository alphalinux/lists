At 03:54 PM 5/22/98 -0700, Wes Nakamura wrote:
>
>I'm trying to get my 4MB Mill. to run in 24-bit mode, X 3.3.2, SVGA
>server, but it won't start, giving the "cannot connect to X server"
>error. It works fine in 16-bit, though, and when I start it (or attempt
>to start it) in 32-bit mode it says it's running with 32 bpp, color
>depth 24, colorweight 888, which looks to me like it's dropped down to
>24-bit mode (which is what I want anyway). So that's how I've been
>running it. Can the Mill. even do 32-bit color? The card is new so I'm
>not too familiar with it. Any help/comments appreciated.

None of the XFree86 servers will run pure 24-bit mode (aka packed) until
someone ports the cfb24 code to the Alpha. There is (when last I looked)
rampant assumption of 32-bit longs in there, not to mention the high
probability of unaligned access. Bottom line: a lot of work for little
payback, IMHO.

However, many of the Xservers, including SVGA for Millenniums, run very
nicely in 32-bit mode, which is officially 24-bit depth, 32 bits per pixel.

So yes, your 4Mb card will not do 1280x1024 at 32bpp, but 1152x900 is close.
Or you could always buy the 4Mb daughtercard... ;-}
