Well, I used to do most of my programming and such on a 110 baud
teletype via modem ("acoustic coupler").

My dad had me beat, though. He used to write some sample programs
for the PDP-5, the predecessor to the -8, though being in sales,
programming wasn't exactly his cup of tea.

The way he did it back then was:

1. Write the assembler code.

2. Assemble the code into machine code, by hand.

3. Transform the machine code into the appropriate
input codes for the paper-tape loader.

4. Convert those codes into the corresponding character codes
(think ASCII, though I guess this predates that).

5. Turn on the teletype's paper-tape punch unit.

6. Start typing in those character codes.

7. Hand-key-in the loader into the machine's core via its console.

8. Feed the paper-tape you just punched into the machine's reader.

9. Hit "START" (or whatever).

All the more painful until he "discovered" that constants could
be "assembled" right into memory just like instructions, instead
of having to be individually built up and stored via longish
sequences of instructions!

And people think Alpha's libc support is troublesome. It is to laugh. :)
