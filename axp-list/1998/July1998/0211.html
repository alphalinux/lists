> Finally got the base install working on my 433a. Thanks for all
> your help. I thought that my next best step would be to install some rpms
> from the update directory on ftp.digital.com. There are quite a few there,
> so I wanted to update the important ones. However, both xrpm and command
> line rpm fail to install any of them. Is there something I missed, or do I
> need some sort of magic rpm that needs to be installed before the rest of

what are the error messages you're getting? it sounds like you're either
getting failed dependencies or or conflicting packages. now, i don't
remeber if or how to do an upgrade to a package. but if rpm says you have
a failed dependency, then you can either find the package that is missing
or, if you already have the distribution installed via different means
(i.e. building and installing from a source tarball), then you can do a
'rpm -i --nodeps <package>.rpm'. The error messages you are getting would
help in diagnosing the problem.
