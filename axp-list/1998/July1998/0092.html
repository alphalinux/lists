Just picked up an AS200 -- having some interesting problems installing
RH5.1:

1) The clock doesn't want to stay set -- usually jumps off to 2018 or
some other year way in the future... Haven't tried replacing the battery
yet.

2) Booting Linux from both ARC & SRM, it begins to start up the kernel,
but it dies after the network code initializes (about the halfway point in
an Intel Linux boot sequence) -- starts dumping out endless pages of
"device_interrupt: unexpected interrupt 0" messages....

3) Digital UNIX 4.0B seems to run OK -- it complains about the date being
way out of whack, but it gets far enough in the install stage to bring up
X and get to the configuration/installation section -- haven't tried going
any further yet.

(I did upgrade the firmware to V6.8)

Interesting tidbit: after I just typed the previous part of this message
in, I tried pulling the battery out of the system for a few minutes. When
I put it back in & turned it on, I got a "1-2-1" error beep sequence --
means either (1) The DROM code detected a realtime clock (BQ3287) failure,
or (2) the realtime clock interrupt did not occur. Cycled the power, now
the clock is staying correct, but it still won't install.

Anybody know enough about the guts of these things to know if this could
be something simple?

Thanks
--Mike
