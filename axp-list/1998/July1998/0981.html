> This is not a good thing. Simply increasing the number of files is NOT a
> good idea. This sounds like it could be some sort of real kernel problem.
> More likely, some other progam is opening files and never closing
> them...

The purpose of increasing file-max and inode-max is to diagnose the
problem. For example, your theory is already disproved. One process
can only have 256 files open maximum. Thus, it takes at least 16
processes with all their files open to fill up 4096 fds, and the
repeating lsof showed that there weren't that many processes out there
with many fds open.

I think this is clearly a kernel problem where fds of closed files are
not reused.

Increasing file-max is an important diagnostic tool. If no one ever
methodically diagnoses the problem, it will continue to be present for
a long time.
