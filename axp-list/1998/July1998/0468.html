On Mon, 6 Jul 1998, Jochen Kuepper wrote:

> >Can anyone confirm success building kernel 2.0.34 with egcs 1.0.2?
>
> egcs-1.0.3 is working for me (You should upgrade anyway)!

I'm not that lucky. My egcs-1.0.3a built kernels are as unstable
as are 1.0.2 built ones. Only gcc produces stable kernels.
Anyway I think RH should release an upgraded egcs package in
case it can be used to compile working kernels at least in
SOME setups. For people who don't want to install contrib
packages, that is.

> But , there are problems with menuconfig/xconfig. Use plain make config,
> and it should work with egcs-1.0.3.

Does anyone know what the exact problems with menuconfig and xconfig
are? They've been broken for ages and many, many kernel releases. I
would love to see them fixed. I mean, really, make config is a pain in
the ass and not the way people want to configure their kernel in the
90s :-) If it's too hard or impossible to fix them, they should be
disabled somehow. Or else we have to read these
frequently-asked-questions from Alpha newbies forever...
