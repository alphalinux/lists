Hello,
sorry if this is a FAQ, but can someone update me on the status
of poll(2) in the 2.0.x kernels? What I see is that:

o It's not implemented as a syscall.

o The glibc emulation of it is broken: stracing an application
calling poll() shows that it attempts to call the syscall 94,
which is not defined in asm/unistd.h, thus getting EPERM.

o The emulation is defined in libc-2.0.7:
% nm /lib/libc-2.0.7.so | grep poll
00000000000922f8 t __emulate_poll
000000000009bfc0 T __syscall_poll
0000000000092220 T poll

Thanks
