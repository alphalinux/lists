On Mon, 06 Jul 1998 mgraffam@mhv.net wrote:
>Hello all..
>
>Can anyone confirm success building kernel 2.0.34 with egcs 1.0.2?

egcs-1.0.3 is working for me (You should upgrade anyway)!

>I'm trying to build a custom kernel for my UDB, configuration and
>compilation go pretty smoothly (there is a CONFIG_TGA option, that
>if set, makes the kernel fail to compile, so I simply don't set it).
>
>In any case, the kernels that get produced don't boot. If I build a
>monolithic kernel, the sound hardware gets initialized (the speaker
>makes a god-awful sound, apparently due to being initialized at to-high
>of a volume) but the system never makes it to a stable environment..
>the whole thing crashes. If I compile with a modular kernel, I don't
>even get to the sound initization part.
>
>In either case, I don't have a heck of a lot to give as far as kernel
>messages or dumps go. I've hooked up a monitor/keyboard to see things
>progress but nothing is given after the kernel is unzipped.. the machine
>just stalls.
>
>I noticed that the stock RH 5.1 kernel that I'm currently using was
>produced with gcc 2.7.2.3, so I'm wondering if switching to gcc would
>help, or if switching to a newer egcs would do the trick.
>
>Is is safe to build a modular kernel? If not, does anyone know how the
>pcmcia-3.* package gets installed?

You can seafely build an modular kernel.

But , there are problems with menuconfig/xconfig. Use plain make config,
and it should work with egcs-1.0.3.
