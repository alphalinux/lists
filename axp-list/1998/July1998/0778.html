Debbie and Mike Meyer wrote:
>
> I'm building RH5.1 on a 164LX with an NE-2000
> compatible PCI network adapter (looks like
> a LinkSys).
>
> OS installs fine, but during network configuration,
> tells me "/bin/insmod not found" regardless of
> the card selected, and then tells me it cannot
> find the card.
>
> So I bypass the network install. When I boot
> the machine, I see it loading the ne2k stuff.
> >From milo prompt, I can also see that
> milo sees a NIC at IO 0x8800 and
> IRQ 17. (The numbers surprised me, but
> we are just starting to use PCI NIC's at my
> POB, so that may be normal).
> However, attempts to manually set the card
> from linuxconf or command line fails as well.
>
> If I understand, the alpha kernel is monolithic
> not modular, so /bin/insmod should NOT be
> available -- so I am puzzled as to why the
> install is even using that command.
>

Don't believe everything you read.

RH5.0 on Alpha does support modules so RH5.1
should too. The numbers look reasonable. Maybe
you need to build a kernel that supports only
the cards you use, not all of them. I always
build a custom kernel for the stuff I'm running
and some modules for stuff I might use but don't
want loaded all the time.

I'm using DEC tulip cards in my systems though
so I can't tell you much about NE-2000 clones.
