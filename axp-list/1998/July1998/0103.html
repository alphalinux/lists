>>> Mike Nowlin said:
>
> 2) Booting Linux from both ARC & SRM, it begins to start up the kernel,
> but it dies after the network code initializes (about the halfway point in
> an Intel Linux boot sequence) -- starts dumping out endless pages of
> "device_interrupt: unexpected interrupt 0" messages....

If you boot from ARC console, boot the xl.gz kernel from MILO; if you boot
from SRM, use the "avanti-s.gz" kernel. Interrupt delivery is different,
depending on which console you are running, and the kernel must match the
delivery type.

> 3) Digital UNIX 4.0B seems to run OK -- it complains about the date being
> way out of whack, but it gets far enough in the install stage to bring up
> X and get to the configuration/installation section -- haven't tried going
> any further yet.

Did you set the date using the SRM console? If you switch between ARC and
SRM, the first thing you should do under the new console is set the date.
The date formats (actually, contents of the TOY chip registers) are kept
differently under the two consoles.

Good luck.
