>Nope, only one file, the binary, is in the .gz file. gunzip by default
>sets the unpacked name to the same as the archive minus the '.gz'
>extention, but you may call it what you wish.
>
>There are four possible problems I can see here.
>1. The binary is not executable. Make it so using:
> chmod 755 xquake-whatever-its-name-is
>
>2. You don't have current dir or the quake dir in your path and try to
> execute it just like
> xquake-whatever-its-name-is
> To execute a binary in the current directory but not in the path:
> ./xquake-whatever-its-name-is
>
>3. Your current dir is not set to your quake directory. I think the 'id1'
> directory must be in your current dir, not necessary in the quake dir.
> If this is the problem, just cd to your quake directory.
>
>4. Your binary is of the binary format a.out (as opposed to ELF which is
> the usual binary format in linux nowadays). Many of the xquake binaries
> out there are so old that they were made when another binary format was
> ruling.
> If this is the case you need support for these a.out binaries in the
> kernel, but I think this comes by default with Red Hat kernels.
> If you miss it, you may need to recompile the kernel or make a module
> for a.out support. You can check the file format of the binary using:
> file xquake-whatever-its-name-is
> If this is your problem, it may be easier to find an ELF binary, there
> are a bunch out there.
>
>It's a little difficult do guess what the problem is when you don't say
>what's happening when you're trying, except that is doesn't work.
