On Tue, Jul 07, 1998 at 11:36:37PM -0500, Jiann-Ming Su wrote:
> Any experience with both? I'm trying to find out if an application would
> run faster as a DU binary or as a Linux/ELF binary. I'm specifically
> thinking about Netscape, Matlab, and Maple. Thanks.

No experience, so far, because I'm still fighting with AS500 ARC/MILO
problems. So just a few thoughts:
em86 has to be slow - because the CPU has to emulate every single CPU
instruction contained in the binary. Only library calls can be handled
natively. (At least they should; I don't know the exact status of the em86
emulator.)
DU binaries, in contrast, can almost be executed natively. Linux understands
the ECOFF format, the CPU understands the Alpha insns and you have to
install the DU libraries anyway.

Note that there is a Linux-Alpha-version of Netscape. At least, I do remeber
having read something in comp.os.linux.alpha. Try dejanews!
