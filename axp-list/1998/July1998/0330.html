Add me to the list of people that are having this problem.

I discovered my problem about a week before going production with the system, and
here's the data I have so far:

1) I believe that at some point I *did* have a stable system. It worked for about
a month and I never noticed these types of problems.
2) I rebuilt my system at the last minute, and that's when I started encountering
these problems.
3) For me, it only seems to happen with programs that I have compiled myself.
Binary RPMS don't display this behavior, but everything I've compiled myself has
done it at least once (apache, imapd, qmail, etc).
4) The memory violation is always in the range of libc, according to ldd output
(not that I know what that means).
5) Fearing that the latest glibc was broken, I re-built my system a third time and
ran a clean 5.0 install without any updates and got the same problem.
6) I've tried the base .30 kernel that comes with redhat, the .32 kernel that came
with my motherboard, a custom .33 kernel, etc. No difference.

At this point, my best guess is that glibc is broken on the Alpha, but I only
received one response when asked about this on the redhat alpha list. Apparently
it just happens more with MySQL so those of us that use it notice it more.

Obviously, it's in our mutual best interests to try to figure this out, so I'm
volunteering to spearhead this effort. If you have an Alpha and you've noticed
this problem, please send me specifics about your system, and any debugging info
you have. If I can get some good stats over the weekend I'll post them on Monday
and try to contact appropriate people about getting it fixed. It's become a pretty
serious problem here, to the point that we will have to abandon the Alpha if we
don't get it fixed fast.

FYI, I've got a Microway screamer 533 (PC 164LX motherboard, I believe) with 128MB
of RAM. I'm also wondering if this problem could be specific to the motherboard,
but I have no stats to confirm this.
