Just wanted to warn people that there seems to be some kind of problem
with the ncr scsi driver in kernel 2.1.108. I was in the process of
trying to do a backup, and the harddrive quit spinning with a clunk,
and my screen started filling up with scsi error messages.

I thought it was my harddrive, which would have really upset me, since I
was in the middle of doing a backup. But after restarting everything
seemed to be fine. But it kept happening during heavy disk usage, and
restarting seemed to set everything back to normal. I'm back to
2.1.107, where I haven't seen the problems - yet. 8-)

My machine is an XL300 with the standard ncr scsi controller.

While I'm on the subject, what tape backups have people used
successfully? I'd like to go as inexpensive as possible (which
probably rules out scsi). I've read that possibly the Travan 5GB
parallel might work, but what are people using? I'd prefer not to go
internal, either (hm, does the XL300 even have IDE...?).

Oh, one other thing. The backup I was doing was onto a jaz, which I
was formatting in dos format so I could read it from a non-linux
machine. I'm using mkdosfs which I got from the redhat contrib
directory, but it doesn't create fs's that linux recognizes. Anyone
know what's wrong or where I could get a working version?

thanks,
-Wes.
