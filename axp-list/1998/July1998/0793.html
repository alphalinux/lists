>>> Secret Asian Man said:
> Is it possible (using the SRM, I think) to boot the Multia off of the
> network (providing I have a "mama" Linux system waiting with bootpd), load
> MILO and load the kernel, then mount root over NFS? Is there a HOWTO
> anywhere?
>
> Has this been done? Am I dreaming?

This is done all the time, and no, you're not dreaming... :-)

1. you can only do this from SRM console; ARC/AlphaBIOS do not do BOOTP.

2. make SURE you configure the kernel from "make config" for SRM boot; ie,
choose "Y" for "Using SRM for boot" question.

3. alpha-sources-2.0.34-0.1 support BOOTP images; do the normal "make boot"
and follow it with "make bootpfile", and look in arch/alpha/boot for
"bootpfile" which is the image.

4. do all the appropriate setup for that machine on the BOOTP server.

5. tell SRM to boot from the Ethernet via something like:

>> boot -proto bootp -file udb -flags root=/dev/sda2 ewa0

Good luck.
