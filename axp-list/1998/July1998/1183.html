Has anybody figured out what went wrong with floppy disk drive support
on the sx164 boards between 2.0.30 and 2.0.34? (For those wondering what
I'm talking about -- apparently something broke floppy support for SOME
Alpha motherboards in those later series, but not for ALL Alpha
motherboards, reports I hear say that floppy drives still work on some).

The symptom is that the drive doesn't read a darned thing. Interrupt count
keeps going up, but no data. "cat /dev/fd0H1440 >/tmp/foo" gets nothing.

I'm about to go diffing through the kernel's floppy sources, but if
somebody has already done that...
