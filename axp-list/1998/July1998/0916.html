I can't seem to get a stable filesystem to work on a DEC PWS600a
with either a QLogic ISP SCSI or a AHA-2940UW board with a DEC RZ1CC-BA
4091MB HD. Here are the specifics:

Heads: 132
Sectors: 62
Cylinders: 1023

Partition:
Device Begin Start End Blocks ID System
/tmp/sda1 1 1 2 8153 83 Linux native
/tmp/sda2 3 3 18 65472 83 Linux native
(NOTE: This is a simple table just for testing. I plan to utilize
the entire disk once I get it working.)

Alpha BIOS: 5.66
Milo: Gatekeeper milo-miata
Kernel: Floppy Gatekeeper miata-2034

Here is the problem:

1. Boot Milo/Kernel/Ramdisk from a floppy.
2. Suspend the RH Linux install program.
3. From a prompt:
# mknod /tmp/sda b 8 0
# dd if=/dev/zero of=/tmp/sda bs=512 count=2 (Clears P. Table)
# fdisk /tmp/sda (Create table above)
# mknod /tmp/sda1 b 8 1
# mknod /tmp/sda2 b 8 2
# mke2fs -c /tmp/sda1
# mke2fs -c /tmp/sda2
# mount /tmp/sda1 /mnt
# mkdir /mnt/foo
# touch /mnt/foo/bar
# umount /mnt
# e2fsck /tmp/sda1
/tmp/sda1: clean, 11/2040 files, 273/8153 blocks
# mount /tmp/sda1 /mnt
# ls /mnt/foo
bar
# umount /mnt
# halt

4. Reboot using the same milo/kernel/ramdisk combination
5. From a prompt:
# mknod /tmp/sda b 8 0
# mknod /tmp/sda1 b 8 1
# mknod /tmp/sda2 b 8 2
# e2fsck /tmp/sda1
Couldn't find ext2 superblock, trying backup blocks...
Attempt to read block fromfilesystem resulted in short
read while tyring to open /tmp/sda1.
Could this be a zero-length partition?
# fdisk
Partition table looks fine!
# ???

6. Huh?

At first I thought (with the expert assistance from Jay at DEC) that it
was the QLogic driver...now that I have reproduced it with the AHA-2940
board I am again lost. Any ideas would be appreciated.
