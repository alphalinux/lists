So far I've had pretty good luck using my UDB with a cable modem. The
cox@home service wants you to use DHCP to go fetch an IP address. However,
it requires your host name to be the one that matches that IP address. On
the other hand, they match names to IP addresses so that the same host name
gets the same IP address every time. When the techie came to "install" the
@home software, the DHCP server was broke. The workaround was(is) to use
the IP address they assign each customer. I'm using that ip for my pcmcia
ethernet card, while the built-in eth0 is the one i gave my internal home
net. With ip-masq and ipfwadm, the other pc's in the house think they're on
the net, too, and I get about 100x the thruput i used to get when I had the
UDB set up to call Netcom via 28.8 modem.
