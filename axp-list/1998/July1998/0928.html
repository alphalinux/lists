Hi Brian !

> On Wed, 29 Jul 1998, WaiSun Chia wrote:
>
> > I'm running a RHL5.1 as a base with kernel upgraded to 2.1.104. As a
> > general rule of thumb, the Alpha generally runs better on the 2.1.*
> > development kernels as all the bugfixes and Alpha-specfic support will
> > be implemented first.

> Barg... I've been trying to boot the 2.1 kernels on my box since I got it.
> Unfortunately, my aboot-0.5 doesn't appear to be capable of it. I'm a bit
> peeved because I went from 2.1.8x on my Intel down to 2.0.3x when I got the
> Alpha. Tough adjustment to make.

This problem is reported very often here...
Maciej W. Rozycki has IMHO a solution for it...
Give the following patch a chance...

diff -u --recursive --new-file linux-2.1.100.macro/arch/alpha/vmlinux.lds linux-2.1.100/arch/alpha/vmlinux.lds
--- linux-2.1.100.macro/arch/alpha/vmlinux.lds Wed May 14 07:41:00 1997
+++ linux-2.1.100/arch/alpha/vmlinux.lds Sat May 9 15:10:00 1998
@@ -1,10 +1,14 @@
OUTPUT_FORMAT("elf64-alpha")
ENTRY(__start)
+PHDRS
+{
+ load PT_LOAD FILEHDR PHDRS;
+}
SECTIONS
{
. = 0xfffffc0000310000;
_text = .;
- .text : { *(.text) }
+ .text : { *(.text) } :load
.text2 : { *(.text2) }
_etext = .;
