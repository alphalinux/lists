brinkmann@nmsu.edu wrote:
>
> > The 'No space left on device' usually means you specified
> > reading a block size smaller than what is on the tape. In
> > particular, I think the default block size for a tar tape
> > is 10240. Try your dd with a block size at least 10240 and
> > it should work. Also, I've found using 'mt -f /dev/st0 setblk 0'
> > helps if you read tapes with variable length blocks. 8mm
> > tape drives are actually variable length block devices.
> > You can put pretty much any size block you want on them up
> > to the system limit. On some AIX systems, I've written up
> > to 256KB-1 block. Think you set that limit when you configure
> > Linux in the SCSI section. It asks about the maximum transfer
> > you want to handle.
