On Wed, 8 Jul 1998, Jon Simantov wrote:

> Anyone know why the clock on my AS200 4/233 keeps jumping to the year
> 2018? Kernel 2.0.34 (though it happened with 2.0.30 as well). Whenever I
> fix the year, it stays okay for a while, and then it just jumps to 2018
> again. What gives?

The time as it is stored inside the memory of the clock has only a byte
for the year. This means that you can't store 1998 in it, but only "the
number of years starting from year X". The problem is that year X is
differend for the different OS'es and Consoles. You can fix this by
using "clock -a" at boot-time. In RedHat you can modify the file
/etc/sysconfig/clock and set ARC=true in it.
