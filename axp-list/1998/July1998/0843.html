Hi,

I tried the new libffm on one of my "realworld" problems on a UDB.
I converted ~100,000 points from UTM coordinates to Lat, Long.
The program is quite small (38K). It reads a record from a file,
converts, then writes a record to a new file.
My system is lightly loaded, and has enough memory (80M) so that it
normally doesn't swap to disk.
I'm using egcs 1.0.3a.

The results:
Consistency:
All resulting output files were exactly the same. I did not add
input data that would have stretched the algorithm to a point where
results could be uncertain.

Times:
Mathlib User System Elapsed CPU
libm 91.50 3.37 1:38.35 96%
92.10 2.69 1:37.34 97%
libfm 97.37 3.76 1:44.32 96% <- strange??
80.43 2.70 1:25 97%
81.98 2.75 1:27 96%
libffm 85.56 3.84 1:33.19 95%
83.5 2.48 1:28.16 97%
87.56 2.82 1:33.05 97%

I'm not sure why the first libfm test showed slow times.
"libfm" seems to be slightly better for the UDB. Comments?

Larry Z.
