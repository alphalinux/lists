Hi Maciej !

> I'll be messing with 'aboot', soon, so I might try to fix this, too.

If you fix this too, you maybe want to implement a secondary
bootstrap chooser too. It should be easy and allow to boot DU and
Linux from a single disk. The srm-howto gives you enough information
for that. (append the DU loader to the aboot-image (check the fields
at offset 488 and 480 of the first sector on disk, this gives you
sector-# and count of blocks for a DU-loader backup.)
The only need is that both loader (aboot with embedded DU-loader)
are loaded by SRM at once. If DU is requested copy the DU loader to
0x20000000 and jump to it...
(Saving the loader to a file must be a tool for end-users)

> The most annoying misfeature of 'aboot' is the inability of booting
> kernels prepared with 'binutils' 2.9.1 and newer ones. This is my main
> concern. Also the inability of following symlinks is somewhat irritative.

I don't think that following symlinks is a main goal. I would simply add the
fact that it doesn't work to the doc's. This approach doesn't break
anything ;-]
