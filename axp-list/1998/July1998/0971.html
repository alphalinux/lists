>>> Stephen Weiss said:
>
> 1. Running DEC Unix 4.0A binaries.
> It seems that my video card, a TGA ZLXp-E3 is not supported by XFree86.

This is now incorrect; there's been a TGA server announced with support for
24-plane TGA controllers, which hopefully will work on your -E3, as its only
been tested on the -E2 version. See previous axp-list postings from Tim Rowley
as to where to find this server.

> I thought, I can run the Dec unix X server.

Not hardly; it requires quite a bit more kernel support than any XFree86
server does, in the form of syscalls/ioctls and such.

> 3. Compiling MILO
> Well, I'm using the MILO image from the xlt-alcor disk. Booting from
> the ARC console (Because the SRM console refuses to boot linux for me, I
> won't even get into that).
>
> The thing works fine, except that between
> when the PALcode is loaded, and the kernel takes over, my screen looks
> like a giant square stoplight.

First off, that MILO was built before there were some fixes for 24-plane
TGA made; I could make one that does have those fixes, if you need it...

Secondly, you should be using the "alcor-s.img" floppy image file for SRM
booting; did you try that? What problems did you run into? Linus boots his
AS600 5/300 that way all the time... ;-}

> 4. Unaligned trap
>
> I read a bunch of seemingly incorrect stuff on this. But then again, my
> machine seems not to conform to any of Redhat's happy FAQs, so I'm not
> surprised. Umm - it says that when you get these warnings, it means
> that the kernel has fixed the problem for you? Well, I don't call a
> segmentation fault a fix. Man doesn't work, for an example.

Man is a known problem, was built incorrectly on the 5.1 release; check the
Errata pages at redhat.com, and look for the update RPMs on their FTP site,
or gatekeeper.dec.com:/pub/linux/redhat/updates/5.1/alpha

> As one last thing, has anyone been able to successfully mount a UFS
> cdrom? If so, how? I'd like to mount my Digital Unix 3.2C CD, so maybe
> I can avoid this whole mess with 2.1.x kernels altogether.

You will need to use a 2.1 xx kernel if you want reasonable UFS support, but
whether it'll work on a CD is another question... :-\
