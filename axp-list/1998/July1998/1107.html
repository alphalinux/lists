>On Sun, 2 Aug 1998, Paul Tomblin wrote:
>
>>You've got the syslogd bug, same as many, many others of us. I bet
if you
>>look in /var/log/messages, you'll see:
>>Jul 16 15:39:02 canoe syslogd: select: Bad file descriptor
>>Jul 16 15:39:33 canoe last message repeated 32511 times

This is NOT a syslogd bug!! Please read the threads about: FSD. What
is happeneing is you are running out of file descriptors! This
usually crashes the machiens in most cases, yet in others syslogd
looks like it is broken. When syslogd starts to report bad fds, you
may also experence problems like programs segfaulting, and/or total
machine lockup.

What needs to be done, to fix this is to up the
/proc/sys/kernel/file-max, and inode-max to a much higher number by
default. I've asked people what they feel is a good number to use, I
can never get any real good feedback on that. Basically anything
about 4x what is default will get you by.
