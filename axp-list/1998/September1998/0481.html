>>> Jan Vicherek said:
>
> That would be great. I've just tried to boot the rawhide-s.img from SRM
> this way :
>
> boot -file vmlinux.gz -flags "ramdisk_load=1 ramdisk_prompt=1" dva0
>
> the result :
>
> kernel loaded and some hardware was detected (most importantly the
> Tulip 10/100MB 21164(I think) network card). But at the end it said
> something like :
>
> VFS: panic, cannot mount FS on device 08:02.

The command line should look like:

boot -file vmlinux.gz -flags "load_ramdisk=1 root=/dev/fd0" dva0

The double-quotes around the whole "flags" argument are necessary!
And have the RAMdisk floppy ready to put in, when the kernel asks for it.

But this will only succeed in getting you into the install; you'll still
have to have some place to install *to*; is there any chance you could
simply remove the DU disks you wish to "preserve", and put a spare one in
that you can install to?
