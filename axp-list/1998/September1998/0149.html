Hello Peter

>> One word of caution, when installing ram into a Digital AS 200 or even XL
>> series. The problem we've seen is they can not run 60ns ram, must use 70ns
>> or have the memory vendor slow down 60ns to 70ns.
>>
>
>Interesting. I've added a lot of 60ns DRAM to AlphaStation 200's with no
>problems. However, in every case, they were mixed with 70ns RAM.
>The manual even says that 60ns RAM is OK. However, I'm sure we're
>not getting any performance benefit from it.

Here's what we've seen. The major vendors, Kingston, Dataram, etc... just
use 60ns parts and slow them down with the jumper setting on the side of the
simm. This way they fit the vendors spec, ie:Digital.

The cheap memory houses ship class "C" ram which really fails at 60ns but
sell it as such. How can an end user tell? You can't.

In addition, Chip Merchant and others use pulls, which are pulled
parts(unsoldered) and reinstalled on what they sell. There is a chip
company up here in New Hampshire that at one time gave me a list of vendors
that do this.

So in essence what the person receives that buys them, is a part that has
been soldered 3 times.

For this reason we don't use cheap no name ram vendors.
