On Tue, 22 Sep 1998, Steph wrote:
> I have a problem to install linux RH 5.1 on my SX164.
> I downloaded all the stuff that is necessary :
> - linload 1.5 and the milo image for sx164
> - I used rawrite to write the images (the one with the kernel and
> the "ramdisk.img" file)

The boot image etc. for the SX164 don't work. The problem is that the
SX164 floppy DMA initialization is broken in the 2.0.34 kernel (see patch
earlier on this list -- I have a link to it on my homepage, unless you're
a Republican, in which case you can still get the link from my home page
as long as you're not easily offended but otherwise you may wish to just
go to the Red Hat archives).

Put linload.exe and the milo image for sx164 onto a MS-DOS formatted
floppy and boot milo as described in the Red Hat documentation. Figure
out what your CD-ROM device name will be (scd0 for a SCSI CD-ROM,
either hdb or hdc for an IDE CDROM). Then instead of following the Red Hat
documentation, do this:

boot scd0:/kernels/sx164.gz root=/dev/scd0

And voila! You will boot the kernel off of the CD-ROM (assuming you have a
boot device supported by MILO) and you will be in the install routine.

Substitute "hdc" or whatever for scd0 if you have an IDE CD-ROM.
