I just set up the environment of running DU binary on the Linux OS.
I followed the description on the FAQ page, and was successful to run both
netscape-v304-export.alpha-dec-osf3.2 and netscape-v304-export.alpha-
-dec-osf2.0 alright. But on Linux, while I tried to run another binary
compiled on DU V4.0C using Digital Fortan with the "-non_shared" option,
the problems occured. One program just died without any warning/error
message, and some other executables produced the following error message:

shmget failed.
shmget failed.: Invalid argument
Segmentation fault

then exited. The hardware is a 164LX with 128MB RAM and NCR53c815a
SCSI/1GB Seagate STN12400 HD. The kernel version is 2.1.124, with
RedHat 5.1 installed. BTW, the manpage could not work either while
I invoked "man anypage" command. It just returned nothing but
"Segmentation fault".

Did I miss something while compiling the statically-linked program,
or while setting up the environment on Linux? Or did I do something wrong
while I install the RedHat? Any suggestion is sincerely appreciated.
