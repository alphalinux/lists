On Fri, Sep 11, 1998 at 12:24:29PM -0400, Greg Lindahl wrote:
> > I wish I could find an authoritative statement of this .got limitation.
>
> The limitation seems to be in .o's, not in executables.

Exactly so. Any one object file is compiled assuming a single
.got, which is hardware-limited to 64k. At link time, different
object files can be given different .got subsegments, but if
you ld -r them together, we loose information about where the
boundaries between object files had been.
