Hello,

I'm attempting to upgrade this system:

UDB 166Mhz 16MB ram
RedHat 4.1 (with a few updated rpm's)

to 5.1 which I ftp'd down to the same machine's
HDD. I ran into a problem during the upgrade
when I tell it the device/path to the RedHat/
directory - it mounts the device and finds the
directory and says it is scanning for rpm files,
and the disk is accessed for a few minutes, but
then it eventually dies - the installation
program receives a signal 10 (SIGUSR1) and it
cleanly exits and tells me I should reboot.

I tried both the ramdisk.img from 5.1 and from
5.0, both had the same result. During the few
minutes prior to the install program dying, the
response on the 2nd v. console grows progressively
worse (I even got a "can't fork" error once, trying
to ls). I only have 16MB memory on this machine,
and thought possibly it's running out (I tried to
run "swapon" from my 4.1 filesystem to enable swap
space, but it wouldn't run). I should be getting
more memory some time soon to see if that helps,
but does anyone have any other ideas/solutions in
the mean time?
