p.s. A recipe to deal with non-partitioned "virigin" disks for
Alphas booting through some variant of ARC and which so far did
not fail me yet once:

- Start with creating a small, formatted, FAT partition using
tools included with ARC. It will be later used for booting.
What is "small" depends on your circumstances. With Linux
only this is 2-3 MB. If you have also NT around make this
big enough for an initial NT to also fit there. (1 MB, i.e.
one cylinder, should be really good enough for Linux but
you may one to keep some "extras" there so a bit more space
is nice - unless you have to be very frugal with a disk space).

Under no circumstances use "Fast NT default" format or similar.
It will screw up badly and you will have to start from scratch.

Even if you do not plan to keep this FAT partition around
start with it anyway. It will make fdisk much happier and
your life simpler. Deleting it is no brainer.
- Continue with your installation and, in an appriopriate moment,
create a full partition table using fdisk (or Disk Druid if
you are very brave).
- After you have a system in place copy to your FAT partition
all files which you need to boot. Use either mcopy or mount it
under Linux and use 'cp' - both ways have the same effect.
You can even keep there a spare boot image (vmlinux.gz) or totally
boot from that partition.
- Adjust your boot menu and milo options to use files on a hard
drive and be happy.

That is all. Works for me on UX as well. :-)
