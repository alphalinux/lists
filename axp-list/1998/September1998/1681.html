Quoting Peter Petrakis (peterpan@wn.net):
>
> Easy fix . Don't use menu or X config, Just use make config. That feature has
> been broken for a while, Don't know why. After that do make dep, make clean,
> and make boot. If you have any modules do make modules then make
> modules_install .When you're all done compiling your kernel go to / and back up
> your old kernel and then cp /usr/local/linux/arch/alpha/boot/vmlinux.gz // and
> reboot. There's no Zimage in Alpha world, that's an intel only thing. ;) .

One little hint - after you've done a regular make config, you can use make
xconfig for little things, although they may not be in the menu. Another
trick I use is to edit the .config file and delete the lines I think are
related to what I want to change, then do a "make oldconfig" which uses your
old config file and only prompts you about the lines you deleted.
