>>> Jan Vicherek said:
>
> I have here an Alpha station from DIGITAL model 4100.
>
> 1. Has anybody heard of this beast running Linux.

Yes. The 4100 is a member of the RAWHIDE family, which also includes the
4000 and the 1200 (the latter aka TINCUP).

> 2. Can this beast run Linux at least in theory ?

Yes. At least, I'm running it on a 1200, even SMP, but I haven't been able
to test a true 4100.

> 3. If so, I would prefer to try RH-5.1 on it. Which floppy images do I
> need ?

You should consult the Hardware Compatibility pages for RH 5.1 at the RH
website; they have recently been updated, and contain the needed info. From
them, you'd know that you need to use the rawhide-s.img for the kernel, and
boot from SRM console.

> 4. I would like to leave the current HDs untouched, and run it over NFS
> (at least until I determine whether it is feasible to run some of the DEC
> binaries). (NFS being the root FS). Anybody has any ideas on that ?

Root on NFS is known to work, but unfortunately, the RH 5.1 RAWHIDE kernel
wasn't built with that option, IIRC.

> 5. I would like to mount the local HDs read-only (the have "AdvFS" on
> them). Anybody has done this ? Is it possible ?

I don't believe there's *any* version of AdvFS under LINUX; anyone know
anything different?

> 6. I would like to try running some binaries found on the above FSs. The
> OS release is DEC unix 4.0 i think, OSF/1.
> `uname` : OSF1 localhost V4.0 878 alpha

You may need to run a development kernel to support all the syscalls that
DU 4.0 support. Fortunately, that RAWHIDE kernel is based on 2.1... :-)

> 7. I think it's got a PCI tulip 10/100 card. Does linux support both 10
> and 100 ?

Yes.

> 8. Some of the FS is on a SCSI RAID. Which RAIDs does Linux support ?

Probably not the one installed there, but I could be wrong. What kind is it?
>From the SRM console prompt (>>>), type in "show config", which will tell you
all about the options installed.

Good luck.
