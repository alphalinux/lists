On 13 Oct 1998, His Holiness the Reverend Doktor Xenophon Fenderson, the Carbon(d)ated wrote:

> I want to give a quick status report on building kernel 1.2.125 on an
> AVANTI-class machine (AlphaStation 200). Note that I'm compiling with
...
> 4. aboot (I boot from SRM) refused to boot any kernel
> generated with binutils-2.9.1.0.6-1, complaining about too
> small a block size (or something like that). Backing down
> to binutils-2.8.1.0.1-1 made aboot happy.
...
> (P.S. If anyone has any suggestions for maintaining two copies of
> binutils, please email me. I'd like to have both the stable and
> development toolchains available, if possible. The compilers co-exist
> peacefully; it would be nice if I could get binutils to do the same.)

Maciej Rozycki posted a patch to this list a while ago, and the URL has
been posted a couple times since. This will allow aboot to boot 2.0 and
2.1 kernels, built with the latest binutils. At least, I'm booting 2.0.35
and 2.1.124 built with egcs-1.1 & binutils-2.9.1.0.13.

ftp://ftp.amg.gda.pl/pub/OS/Linux/people/macro/SRPMS.glibc21/aboot-0.5-6.src.rpm

I've even put a precompiled glibc-2.0 binary RPM of those sources on
ftp://ftp.terran.org/pub/RPMS/alpha-glibc.

cheers,
