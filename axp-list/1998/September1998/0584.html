Let's reword that to:
We will let our (Compaqs) own in-house workstation division get first crack
at the cool new stuff.. So all the guys who "price is no object" this will

Plus, look at the bright side, Steve. The Compaq support folks get to deal
with the bugs and support issues for the "brand-new, just released, version
1.0" problems..

Anyway, having said all that, the fact is I have been told that ALL DP264
boards will ship with SRM.
BUT, and this is a BIG "BUT":
There will be a "crippled" SRM for running NT ( and Linux) and a "full"
version for running DUNIX. This is the way the pre-release DP264 machines
are set up now, so this may likely be true. Apparently writing an AlphaBIOS
for this puppy would be too liliting, since the Microsoft ARC spec for
wrtiting NT loaders for RISC chips does not include support for some of the
needed requirements of this board.
