I just started to setup my AS200 4/233 last night and couldn't get it to
properly recognize my hard drive. I put in a brand new IBM Ultrastar 2ES
DCAS-34330. It's a 4.3GB Fast Narrow Ultra SCSI-II drive. I'm booting from
SRM and I've tried using the boot floppies from redhat 5.0 and 5.1 with the
same results. I also upgraded the firmware to v6.7-2, but this didn't fix
the problem. When I put the boot disk in and type:

boot dva0 -file vmlinux.gz -flags "root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1"

SRM reads in aboot and aboot then starts to boot the kernel. The kernel
starts to boot, but when it gets to the SCSI detection phase it dectects the
card properly. The SCSI card then scans the SCSI bus and detects the hard
drive and cd-rom drive. It prints out the hard drive manufacturer followed
by the model
number, but the model number is incomplete and has a garbage character(an
ANSI graphics character) following it. It detects the cdrom drive without
and problems and can read the filesystem on it.

The kernel then prints out:
sda: Spinning up disk.................................... not responding...
sda: READ CAPACITY failed
sda: status =1, message =0, host =0, drive =28
sda: block size assumed to be 512 bytes, disk size 1GB.

The drive is actually 4.3GB. Does this sound like a bad drive or an
incompatability between the drive and the NCR 810 SCSI controller?
Anybody seen this before?
