Hello. I'm running RedHat 5.1 on a Multia with kernel/utils upgraded
to boot 2.1.122. I've noticed a couple of strange things when booting
with MILO on this machine (same probs with various kernels).

First, there is an external modem connected to the box. If it's powered
on when MILO starts I can't boot. It appears that everything I type
is sent to the modem - like MILO sees something on the serial port and
assumes my console must be there. Not a big deal, just a minor annoyance.
"Console on serial port" is not built into my kernels but must be built
into MILO? Any way to disable it via some environment setting?

Second, the system can't boot if the kernel name is a symbolic link. I
usually keep several kernels and would like to just create a vmlinux.gz
symbolic link to the kernel du jour. When I try however I get:

#attempt to access beyond end of device
08:02 rw=0 want=1768713591, limit=77500
block_read: read failed (after 0 bytes)UNZIP: this is not a zipped file
ext2_bread: read error
####################Bad filehdr magic number 0x0 should be 0x183
MILO: Failed to load the kernel

After several failed attempts with these messages a further attempt
fails with "...Fatal error: not enough memory" and the system is hung
and must be power-cycled.

Anyone seen similar problems and have a solution? Considering these
are my only gripes with Alpha/Linux I'm extremely happy with my Multia
acting as a firewall/router for my small LAN.

-Jason
