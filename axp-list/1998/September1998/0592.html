With regards to your message at 08:33 PM 09-21-98 -0700, Joe Reiter. Where
you stated:
>I accidentally added a line with a typo to my fstab, now the fsck fails on
>bootup. However, when it asks me to enter my root password for maintenance
>mode, it tries to go into maintenance mode but fails saying it can't load
>something from the ncurses library. It then proceeds to shut down.
>
>Anyway, I need to know some way to get in there and edit that fstab to make
>the thing boot. Is there an emergency disk like the intel Redhat versions
>have? Or anything else that might do the job?
>-joe
To boot up in single user mode:
boot sda2:vmlinux.gz root=/dev/sda2 single

The locations and so on are possibly different than your machine setup. I
was only illustrating the use of the "single" command which will avoid
reading your fstab.
You will still have to manually force read-write to the partition with /etc
on it, and edit the fstab, but it will get you by this trap, anyway..
