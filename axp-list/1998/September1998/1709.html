Hi,

I am new to this mailing list and i wonder if there is a list of known
problems and solutions classified by machine type. I also started to
read the entire archive to look for problems and solutions for my alpha
SX164 (kernel patch, clock skew, irq patch...)but for the moment i have
few question:

1)
I have an alpha 21164PC SX and i would like to use his auto-switch-off
feature. I think it is possible because of the standardisation of the
atx cards. Moreover my power button is not a power switch but a logical
one so it would be possible...

But when i activate the apm option in linux, it tells me that it doesn't
know the registers ( it seems to be 80x86 ones ). What should i do?

2)
Does anybody successfully compile xquake on an alpha (SX164 for example)
? it seems to have some problems with a strcmp (segmentation fault) and
some others segv's. after having patched it (testing the ptr before
using strcmp) it works fine but i don't know for how long...
So is there any official patch or something to do with the source code
to make it compile immediatly and cleanly?

3)
I would like to hear about experiences using OpenGl with linux on an
alpha with a FireGl1000 or the new AccelStar II. Is it easyly
compatible? Is there any official support...
