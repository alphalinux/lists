Brian E.W. Wood wrote:
>
> Has anybody tried to compile a kernel on an alpha with egcs 1.1b?

Yes. Doing this all the time. :-)

> gcc -D__KERNEL__ -I/usr/src/linux-2.0.33/include -Wall -Wstrict-prototypes -g
> -O2 -pipe -mno-fp-regs -c -o slhc.o slhc.c
> gcc: Internal compiler error: program cc1 got fatal signal 11

Yes, as well. Drop an optimization on this one particular file to -O
and everything will compile ok.

This specific bug is triggered here by an expansion of an inline function
__ntohs is slhc_uncompress(). Rerun your compiler with -save-temps option
to have a closer look what kind of a fun code it really is. :-)

I filed a bug report about this the first time in June - I think, but
definitely well before 1.1 release - and a few times more on other
occassions. Maybe it is hard. I do not know. Alas I never got a
single line of feedback. It looks like that somebody tried to do
something on the latest snapshot but some other screwup occured.
