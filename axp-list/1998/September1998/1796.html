Andreas Haumer wrote:
> Hi!
> I'm having a _very_ hard time in setting up a Samsung 164UX
> motherboard and I'm looking for help... :-((
> That is, as soon as I install an environment variable set, I
> can't even boot from the MILO floppy!

Suppose you have a msdos partition on "sda1" (including ldmilo.exe,
milo.ful and osfpal.nh) and your vmlinux.gz islocated on the "sda2",
so here is the working environment variable set for the
Samsung UX2/UX4 Mainboards:

ConsoleOut = multi(0)video(0)console(1)
ConsoleIn = multi()key()console(1)
lastknowngood = false
autoload = yes
SystemPartition = scsi(0)disk(0)rdisk(0)partition(1)
LoadIdentifier = Red Hat Linux 5.1 (or different text)
countdown = 10
OsLoadOptions = boot sda2:vmlinux.gz root=/dev/sda2
OsLoader = scsi(0)disk(0)rdisk(0)partition(1)\ldmilo.exe
OsLoadPartition = scsi(0)disk(0)rdisk(0)partition(1)
OsLoadFileName = \
