OK. In short:
Description:
The SX164 has definetely a problem with Linux running mpegtv.
Fault:
When using mpegtv and CPU and memory intensive programs like gcc either
one of them crashes at random with varying errors (seg-fault, sig-11,
illegal instruction, floatingpoint exception)
Cause:
This can only be caused by memory corruption. The astonishing thing is
that a user mode process can corrupt memory used by other user mode
processes. ALERT!
Probable reason:
Maybe kernel, libc or libpthread? (mpegtv uses pthreads).

Currently noone had an idea why this happens. I asked so far
