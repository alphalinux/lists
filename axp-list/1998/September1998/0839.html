For anybody who has experience getting DU binaries working under Redhat, a
few pointers would be greatly appreciated.

I have been diligently following an informal www-based HOWTO to get DU
binaries working. I've certainly made progress, but things aren't working
out quite as they are supposed to. Hopefully somebody can point me in the
right direction for the errors that I'm getting.

So far, I have no trouble getting non-shared binaries running under Alpha
RedHat. But binaries which use shared libraries are still causing
problems. I have RedHat 5.0 with Kernel 2.0.33. I'm using the DU
/sbin/loader and DU libraries which I've copied over into the appropriate
directories. I have ECOFF support compiled into my kernel which is
working since I can run statically linked binaries.

Here's one error that I get:
##### 1 #####
4049:/usr/local/bin/alpha/a.out: /sbin/loader: Fatal Error:
Multiple instances of shared object libc.so loaded as both
/usr/shlib/libc_r.so and /usr/shlib/libc.so
#############
NOTES: The /sbin/loader is the DU binary. The two libraries
/usr/shlib/libc_r.so and /usr/shlib/libc.so were both copied over from the
DU OSF1 system.

With a different DU binary, I get the following error:
##### 2 #####
4175:/mnt/usr-local/bin/a.out: /sbin/loader: Fatal Error:
set_program_attributes failed to set heap start
#############

Hopefully somebody has seen these errors before and knows what
direction(s) I should take. Thanks in advance.
