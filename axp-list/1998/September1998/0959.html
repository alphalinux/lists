mg> Personally, I hope vendors start to get the clue and code their apps
mg> in Java. Granted, for something like Mathematica, the slow down
mg> might not be tolerable .. but for office apps or stuff like that,
mg> one set of Java applets would run well on just about everything

Java isn't necessarily the best language for everything. I could see
the utility of them coding their user interface in java and doing the
kernel in C/C++. The core mathematical routines should stick to a low
or mid level language for speed until there are good Java JIT
implementations around. Since one of the modes of operation is to run
the user interface locally and the kernel on a server, a java user
interface becomes an instant half-port to anything small (NetWinder,
JavaStation, etc...) while the kernel runs on a real machine.

They already have ports to 64 bit machines and ports to Linux. It
shouldn't be a terrible investment to make on for Linux on a 64 bit
machine.
