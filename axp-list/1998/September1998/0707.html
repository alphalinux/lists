I have an Alphastation 500a on which I'm instaling linux.

I can't get a MILO partition created so I can boot to hard disk.

The AlphaBios shows the following disk setup:

QLISP10x0 SCSI ID 0 4091 MB
Parition 1 1004 MB Unrecognized
2 1004 MB "
3 1004 MB "
4 518 MB "
5 510 MB "
6 31 MB "
7 16 MB FAT
unused 4 MB

Disk Druid shows the following on set-up of partitions:

/usr sda1 1004 linux native
/var sda5 1004 linux native
/home sda6 1004 linux native
/ sda7 517 linux native
sda8 509 linux swap
/tmp sda9 31 linux native
sda10 15 DOS 16bit < 32M

The Alpha Installation addendum shows commands to use for ARC, but not
AlphaBios.

AlphaBios seems to not want to recognize partition 7 (the 16 MB FAT
partition from disk-setup)
as a place to enter the "\" for the operating system root directory as
indicated in the adeendum instructions for arc.

Or, most likely, I am confused.

What do I enter in the following command to copy MILO and linload.exe to
the hard disk?

dd if=/dev/fd0 of=/dev/sda? bs=1440k

And, what do I tell Alpha Bios to do to set up a Linux boot selection for
the hard disk?

originally in the following order:

/
swap
/usr
/var

/tmp
