On Mon, 05 Oct 1998, Larry Zimmerman wrote:
>Hi, hope you can help me.
>
>I'm trying to connect my UDB to a GPS receiver using ttyS0.
>I'm running kermit to test the comm. setup. The setup should
>use data-leads only or use CTS/RTS. I've set kermit to ignore
>all flow and handshake signals. I can receive data, but I cannot
>send data. The kermit communication settings seem to be okay, so
>I suspect a problem in the serial driver - like it can't/won't do
>data transmission without RTS/CTS or DSR/DTR signals active.

Does it work okay when you have the handshake signals, with a modem, for
example? How are you booting your machine?

Long ago, I screwed up my tiny little DOS partition with the ARC booting
stuff on it. I booted the machine both with SRM and an emergency boot floppy
(actually, I loaded MILO from each), and both times I was unable to use the
serial port. In my case, characters would come back from the modem, but very
very slowly, like something was timing out. I never did find out what the
problem was, but fixing the DOS partition and booting from ARC again fixed it.

Another thing I discovered is that at least my version of ARC wouldn't look
at any "DOS" floppy that was formatted with Windows 95. I had to go and dig up
a Windows 3.1 machine to create the floppy.
