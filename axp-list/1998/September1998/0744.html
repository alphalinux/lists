Sorry to "me too", but I thought I'd mention that this isn't isolated.

Murray Stokely wrote:

> Has anyone had any difficulty with configuring an interface with dhcp
> under Red Hat 5.1.
> I get the following error message when trying to connect my cable modem.
> % dhcpcd -d eth1
> ioctl SIOCSEFBRDADDR (ifConfig) cannot assign requested address

I get precisely the same error under the same conditions (RH5.1, cable
modem, and eth0 (DE500 PCI 10/100baseT) as the only device in the system.)
If dhcpcd has this problem, I'd imagine pppd might as well, but I don't
have a modem with which to test this.

> I get the same error message with 2.0.x and 2.1.x kernels.

Hmm. dhcpcd DOES work for me on 2.0.35... If you're using eth1, do you
actually have two ethernet devices in your system? If so, try setting
up dhcpcd to use eth0 instead of eth1 (since it works with 2.0.35 on
eth0 on my machine.)
