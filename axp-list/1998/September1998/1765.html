I spoke too soon. It turns out I couldn't complete the install. The SCSI
craps out with that set of disks. I finally found that:

1) The milo disk is as previously outlined; copy the file named avanti
(renamed to milo) to a DOS floppy along with linload.

2) The proper boot image is not the xl as shown in the addendum, it is
the image called avanti and is found in the images directory.

3) Normal ramdisk

With these three, the install completed without any glitsches.

I finally figured out that the instructions in the addendum and all the
HowTo's is wrong(or so it appears). The latest ARC is apparently looking
for files on a fat disk. I simply copied linload.exe and the avanti file
from the cd to a dos disk, renamed avanti to milo, and everything worked
fine.
