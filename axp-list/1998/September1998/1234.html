On Mon, 5 Oct 1998, Dan Morrill wrote:

> Sorry if this is a FAQ, but I couldn't find any information on it.
>
> I have a SCSI ZIP drive that I would like to use on my SX164.
> However, I don't want to put it on the same SCSI controller as
> my UW HDD, because I'm told the SCSI bus would run at whatever
> SCSI mode the ZIP drive runs at (which is definately NOT UW. :)

I specifically asked the tech support at BusLogic about this, and they claim
that *their* interfaces are not limited by this...and that the myth
originated with the combination of old SCSI-2 and SCSI-1 interfaces, and
isn't a problem anymore, since individual devices negotiate their transfer
rate independently.

> So, I'm going to go out and buy Ye Olde Cheape ISA SCSI controller
> exclusively for this ZIP drive. Does anyone have any experience
> with this?

I have my internal (50-pin, SCSI-2 interface) ZIP drive attached to my super
duper fancy BT-958 BusLogic controller. I've found that in order to get it
to work properly I had to add the kernel command line option:
BusLogic=BST:1
Which corresponds to "Bus Settle Time"=1 second. I don't know if other SCSI
controllers need similar parameters, but if your ZIP drive seeks and then
spins down repeatedly, look through the /usr/src/linux/drivers/scsi/{driver}
source file for a similar parameter and try it.

I did, however find that my old ISA Adaptec (1542, 1522) didn't have a
problem with the ZIP drive. (no kernel parameters necessary)
