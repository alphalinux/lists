Here comes the most stupid problem of the day.

If in the ARC boot selection I put:
OsLoadOptions= boot sda5:vmlinux.gz root=/dev/sda5,
ARC loads MILO and then passes it the above line so that the
system boots automatically.

Good. Now I would like ARC to instruct MILO to do two things:
MILO> set MEMORY_SIZE 224
MILO> boot sda5:vmlinux.gz root=/dev/sda5
but I cannot find a way to do it. MILO is kind enough to
allow the semicolon as a separator, so
MILO> set MEMORY_SIZE 224;boot sda5:vmlinux.gz root=/dev/sda5
also works from within MILO, but the idiotic ARC firmware does
not let me input
OsLoadOptions= set MEMORY_SIZE 224;boot sda5:vmlinux.gz root=/dev/sda5
because when I enter this string, it is truncated at the semicolon!!!

Did anyone find a way around this silliness?
