Well, StarOffice-4.0 doesn't work well. Too many segfaults, and all
that stuff. Also much too slow. Easily accepted.

The real tragedy lies in StarOffice-3.1:

After replacing libc (and libm) in /usr/x86/lib to more recent
versions (libc.so.5.4.33, I believe) the setup program works just
fine. (OK, I had to make small changes to the scripts as well, so that
it recognices alpha instead of i*86 as architecture).

'swriter3' itself starts up slowly (it takes about a minute), but once
it's up, the speed is acceptable on my XL-300. I browsed some of the
demo documents, and I found out that the reaction tines on typing and
other things are quite OK. The background spell checker is fast
enough, too.

However, when it comes to saving documnents, it simply fails!
Something like "Error saving documnet, wrong parameter passed"
appears. Investigating further, a similar message appears if I try to
open a document, which is *not* write protected. Everything else looks
like it's working (though I havn't bothered yet trying to print anything).

Is it possible that there is some mismatch between x86 and Alpha
parameters to the file functions? What would be the easiest way to
investigate this? Is there more documentation available to the em86
internals, than what is found in ftp.digital.com:/pub/DEC/Linux-Alpha/em86?
