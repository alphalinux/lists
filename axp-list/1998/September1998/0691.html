>I had a nasty problem arise at the end of installation, when the kernel
>image floppy is requested: the floppy wouldn't mount, with the error 'mount
>failed - invalid argument'.

As Michal pointed out, it is possible to get around this problem by booting
the kernel from floppy with the newly-built filesystem as root. However, I
actually tried doing this when I had the problem, and none of the
configuration files in /etc are set up (including fstab), so unless you are
comfortable with writing your /etc files from scratch then a considerable
amount more work is required to get the system up and running smoothly.
