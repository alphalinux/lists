Mark,

You haven't described what the C program does, but you might want to
try using gdb (GNU Debugger).

A debugger or profiler, like "tprof", will tell you which parts of the
program are taking the longest. This can help you optimize code by
identifying the most inefficient/slowest areas in the source.

Profiling on both the NT side as well the LINUX side may give you a
better insight into actual performance.

Good Luck
