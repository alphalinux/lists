On Tue, Sep 15, 1998 at 02:30:55PM -0500, Rob Rupprath wrote:
> Now when I try to install the new kernel by doing a "swriteboot /dev/sda
> boot1x vmlinux.gz" it suceeds, but the machine will never boot again, not
> even from the floppy (Unable to mount rootfs). I suppose I have the syntax
> wrong somewhere.
>
> I then reinstalled and tried the same thing, but using the floppy method.
> It boots, and brings me to "Now starting the kernel", but never gets any
> further. I do have the SRM boot option enabled in my kernel config.
>
> Are 2.0.x kernels not good for alpha systems? In reading the archives
> many people are running 2.1.x If anyone has any helpful answers on the hd
> boot issue please let me know. I have no problem installing the machine
> again if necessary. I also get a lot of segmentation faults when doing
> simple things like "man ls". In my messages file there are a lot of
> "unaligned trap" errors. Is this normal?

I run Debian (and 2.1.119), so I may be a bit off, but here goes:

I'm guessing that you're using fdisk or cfdisk to partition your disk.
You can't use aboot to boot from the hard drive if you do that. You must
use OSF/1 style partitions, which is a big pain in the ass. The problem
is that SRM stores its pointer to the bootloader right on top of where
your partition table is. If you run swriteboot on an fdisk'd disk,
you'll hose up the partition table.

I'm booting my box with aboot via bootp. I also tried using it on a second
hard disk, which works fine.

If you want to try it, use sdisklabel to partition your drive.

Good luck,

Jeff
