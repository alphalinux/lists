> Just a little problem with postgresql from redhat 5.1/Alpha:
> I just installed the packages and installed a webinterface.
> But whenever the webserver (or someone else using psql) connects to
> the database, the following message is displayed at the console 4 times:
> postgres(3610): unaligned trap at 0000000120116e00: 000000011fffd93a 28 1
> The entries in the dbase are not corrupted or anything.
[...]
This is pretty normal and does not cause any problems. The reason for this
is unaligned memory access which causes a CPU trap. The kernel catches that
trap and automatically aligns the memory request. The only drawback is a
little loss in speed because the trap has to be handled.
You should be able to configure your syslog in a way that those messages do
not show up on your console anymore.
