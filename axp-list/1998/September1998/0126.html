I have an application that I am working on which runs on multiple
platforms including Linux Intel, Solaris, NT, Windoz 9x. When calling
any function in netdb, I get a seg fault on Linux Alpha (Red Hat 5.0
with updated libraries).
The following example code core dumps when compiled as a C++
program, but runs when compiled as a C program:
#if defined (__cplusplus)
extern "C" {
#endif
void setservent(int);
#if defined (__cplusplus)
}
#endif
int main(int argc, char **argv)
{
setservent(1);
}
My gcc(g++) and glibc versions are:
gcc version 2.7.2.3
glibc-devel-2.0.7-19
Kernel:2.0.30
Platform:UDB 233MHz. (EV45 chip).

If anyone can suggest a viable solution, I would like to be able to
continue working on this application on Alpha Linux.
