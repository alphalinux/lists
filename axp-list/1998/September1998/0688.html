Stig Telfer wrote:
>
> with 'data type 9 not recognised' as the last gasp.

'data type 9 not recognised' is a message from an old version of rpm
itself which indeed does not recognize later packages which happen
to use this data type. The only cure I know is to use current rpm
binaries.

> I had a nasty problem arise at the end of installation, when the kernel
> image floppy is requested: the floppy wouldn't mount, with the error 'mount
> failed - invalid argument'.

You can ignore this problem, reboot your installation from a floppy
(do NOT "load_ramdisk" this time and give it as root your real root
partition) and copy on your hard disk any kernel you want; or even
make yourself a custom one.

The above assumes that your installation at this point is basically
healthy.
