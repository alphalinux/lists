Well I need some help.

I am able to completely install the rh distribution via ftp onto my
multia. I am even able to (what I think is properly) properly set up a
swap partition, / partition and /usr partition. I don't need a /var
partition and any other partitions are part of the other filesystems. I
was even able to create a 5 meg dos fat-12 partition as the first
partition on the disk. for reference sake, my partition map might look
something like:

sda1 dos 5M bootable
sda2 linux swap 100M
sda3 linux native 300M /
sda4 linux native 800M /usr

Note, if I create sda3 to be 100M which is what I thought I wanted, the
install can't figure out the mount point and will run out of space
trying to put everything on sda3 instead of using sda4 which is what I
would do if I were an install program... But I assume that is just a
glitch in the install script and I ain't going to go trying to fix it. I
can change the size of the partition to fit my needs later.

My problem is this. How do I properly format the dos partition, and put
milo and linload.exe on it and then what is a good disk config for the
ARC console so the multia will boot everything off the hard disk
partition 1 and then mount the others and go on with the startup...

Currently, I can boot milo off the floppy and then enter the command
string to load vmlinux off the hard drive and that works.

My second problem: the kernel... I want to rebuild the kernel to my
needs. I know there is an xkernelconfig somewhere because I have seen
others use it. And then I need the instructions to load all the modules
and then do a make. The instructions do not cover this at all.
