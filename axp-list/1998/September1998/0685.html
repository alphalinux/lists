On Mon, 21 Sep 1998 23:00:37 METDST, furio ercolessi <furio@spin.it>
said:

> No, this really does not work for me. Today I tried to untar
> a 20MB tar file, and the machine was locked very solidly, with a
> kernel: Kernel panic: VFS: Free block list corrupted

That's useful: it means that kernel memory is being corrupted. The free
block list is not mapped anywhere, and is not touched by dma. It's a
pure kernel-internal data structure which should be touched by nothing
except the standard kernel-mode code.

Of course, it is possible that the data structure was being stomped on
by a stray IO intended for somewhere else, or by the VM if we were
accidentally mapping this page somewhere.
