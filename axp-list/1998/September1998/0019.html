Quoting Shane Steven Sturrock (Shane.Sturrock@ed.ac.uk):
> that worked previously when plugged into my UDB (2.0.34 with PCMCIA,
> PPP etc compiled in) complaining about serial line looped back. Nothing
> suggested in the PPP-HOWTO would fix the problem but minicom would work
> although it always made the connection at 38400 so I figured what the
> hey! Changeing the speed in my ppp script to 38400 I tried again and
> the thing worked! I did setserial /dev/modem spd_vhi and tried higher
> than 38400 but the modem would stop working again.

Correct me if I'm wrong, but I beleive that a PCMCIA modem is like a regular
internal modem - ie. it doesn't use the existing serial ports. So you're not
really using a 16550A UART, you're using whatever is built into the modem.
So stick to 38400 without spd_vhi. Did you try 38400 and spd_hi?
