>>>>> "TH" == Thatcher Hubbard <hubbardj@earthlink.net> writes:

TH> On S/Linux, the text on a console is much smaller, and, in my
TH> opinion, much nicer looking. When I did a 'make config' I saw
TH> an option for TGA_CONSOLE or something like that. Does this
TH> allow you to make your console text size small like that.

A most emphatic *NO*

CONFIG_TGA_CONSOLE (and its cousin CONFIG_FB_TGA) have nothing to do
with the console font. TGA is a kind of video card manufactured by
Digital; on some machines (e.g. some models of the UDB/Multia), the
TGA device is integrated on the motherboard. On others, it is an
adapter card plugged into the PCI bus. It is not compatible with VGA
in any way, shape, or form.

To change the console font, I'd imagine that you'd have to mess around
with some of the options in the frame buffer configuration section.
I'm not certain what all is involved, as I only recently have started
to play around with the frame buffer code.

TH> My other question is how do I make it so I can telnet in as
TH> root?

In order of decreasing badness, here's what you can do:

1. Add every terminal device (only ttys, not ptys) on your system to
/etc/securetty. This is a Very Bad Thing as it exposes the root
account to direct attack, i.e. DON'T DO THIS.

2. Create a normal user account that would be used for day-to-day
tasks (reading email, browsing the web, etc.) and only 'su' to root
as needed. Beware that when 'su'ing to root while connected to a
computer via TELNET, the root password is sent over the network as
plaintext (same as in #1 above).

3. Install some form of secure login system, and use that to gain root
access remotely (whether directly or via 'su'). SSL-enabled
telnet/telnetd is one possibility. SSH is another (and what I use,
if I absolutely cannot get to the machine directly). One-time
password systems such as S/key is a third (and probably the best
solution, when coupled with session data encryption).

If you want to use S/key, you initially need secure (local) access to
the machine to set up the one-time password system. The more I think
about it, the more I like S/key. I've got a box at home with OpenBSD
2.3 on it, which (OpenBSD) supports S/key, and it's pretty neat to
play with.
