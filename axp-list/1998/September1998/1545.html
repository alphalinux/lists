This is going to sound odd but it worked for me. When I told the setup to
check for disk errors during the format (or something like that) it
wouldn't handle problems in the FTP installation very well. In my case, an
HTML RPM wouldn't install. When I stopped telling it to check for errors
during the install it would continue after the RPM failed.

I'm not sure why they're related but it I can reproduce the error at will.
