>> > I had a problem with a Conner 1060 SCSI HDD some while ago. Maybe the
>> > following message from Conner Technical Support helps:

Well, I went home and tried out a different drive, a Quantum 2.1gig SCSI
drive, and the problem persisted. I even mooched an internal CD and set up
everything on the internal chain to make sure there was not a weird
termination issues. It didn't work. Lastly, I installed ONLY the hard
drive and did an FTP install - once again it froze right up in the same
place.. BUT...

The weird thing is that it will always format the SWAP partition! Its only
after doing the swap that it freezes up. And, like I said, it seems to be
random. Sometimes it dies formatting the ext2 partition, sometimes it
succeeds. At the very best, it dies on glibc-?-0.27.

I've tried 3 hard drives, 2 cd's, 2 different SCSI ribbon cables, various
termination methods, and its all the same.

I am wondering now if I have a flakey AS200. I don't know what to do to
determine this, though, since I can't install ANY OS at all at this point.

It seems like SCSI weirdness on the AS200 is a well known problem - for
example the NetBSD home page lists it as a known bug. I have seen it
mentioned elsewhere. YET it remains on the supported lists of all of these
operating systems.. I'm begining to wonder, also begining to be bummed
that I payed money for an OS I can't install.. :(
