> I occasionally want to reboot a machine remotely, and have it use
> a different kernel than the current one. Is there a way to pass an
> argument line to MILO to take affect after the reboot?
>
> e.g.:
> `shutdown -r now "sda3:vmlinux-whatever.gz root=/dev/sda3"`

I didn't pay attention to see if anyone answered this and RedHat has
never put up the October archive (this is quality control? yeah
right), but:

The Debian folks are working on a MILO which reads its starting line
out of a file. That will do what you want. I hacked around this by
simply editing arch/alpha/kernel/setup.c and simply putting a constant
string into the command line. The kernel which is booted is an old one
but that way I can move root around and do other weird things.

In your case, however, the easiest solution is:

mv /vmlinux.gz /vmlinux.gz.old
mv /vmlinux-whatever.gz /vmlinux.gz
reboot

You aren't doing anything weird enough to need a drastic solution.
