> Floating point exceptions in programs like Netscape, where you are
> practically sure that you do not care as this was result of calculating
> some pixel on a border of a display - or something of a similar kind -
> are really trivial to deal with. If compiling with -mieee flag does
> not help (exception may happen in libraries which were not compiled
> with this flag) then install a custom signal handler which just
> returns on SIGFPE. You will get 0 for a, fake, result but most likely

> this is "right".
