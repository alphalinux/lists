Hi Paul !

> Correct me if I'm wrong, but I beleive that a PCMCIA modem is like a regular
> internal modem - ie. it doesn't use the existing serial ports. So you're not
> really using a 16550A UART, you're using whatever is built into the modem.
> So stick to 38400 without spd_vhi. Did you try 38400 and spd_hi?

Most (if not all) internal modem cards do have 16550 UARTs (
some of them (multiport) have a 16554 (=4x 16550) on it....)

The same is true for PCMCIA. Nearly all those beasts do
a COM emulation (including UART).

Here a snipset from the PCMCIA HOWTO (ftfm ;-})

Another tip-off of a conflict is if
the device is reported to be an 8250; most modern PCMCIA modems
should be identified as 16550A UART's. If you think you're seeing
a port conflict, edit /etc/pcmcia/config.opts and exclude the port
range that was allocated for the modem.
