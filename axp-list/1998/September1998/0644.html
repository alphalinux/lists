Update on our UX problem: things are worse than expected, since
we have reduced memory down to 128MB but the machine still
crashes under memory-intensive load, using a 2.0.35 Linux kernel.

This time I used Dave Gilbert's program as a test
( http://archive.redhat.com/axp-list/1998-September/0605.html )
with #define SIZE (31*1024*1024*sizeof(int)) (on purpose large enough
to induce paging activity), obtaining within a few seconds:
Whoops: unlock_buffer: async io complete on unlocked page
The machine had to be rebooted after that.
Disk was running in FAST-5 WIDE mode (10 MB/s) to stay on the safe side.

We are rather desperate.
