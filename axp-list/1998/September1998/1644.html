Hmm. Since you seem to be using the latest glib and gtk+, I can't help
you here. :-(

> The second one concerns Mesa again :
> I finally had to install some packages I found
> with MesaLibs 2.6 and MesaDemos ...
>
> Now when I launch any program I wrote for Mesa,
> it crashes with a beautiful "Floating point exception" !
> I remember I had the same error when I compiled Mesa with
> the -ieee (or something like that) option.

I used to get this when I built Mesa *without* -mieee. Try adding -miee
to the CFLAGS under linux-alpha-elf in the Make-config file. This got
rid of the problem for me.

(You may also have to compile your program with -mieee.)

If that doesn't work, are you using libffm? I was getting underflows in
exp for a while, so I made libffm without exp, and linked -lffm -lfm
-lm.

I hope this helps!
