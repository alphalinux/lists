On Wed, Oct 14, 1998 at 12:44:52AM -0600, Maurice Hilarius wrote:
> >number, but the model number is incomplete and has a garbage character(an
> >ANSI graphics character) following it. It detects the cdrom drive without
> >and problems and can read the filesystem on it.
> The garbage character is harmless. I have seen this before on the DCAS drives.
>
Hi Maurice,

After trying several different combinations of jumpers, I found that the
garbage character went away if the delay auto start jumper was set.

> >The kernel then prints out:
> >sda: Spinning up disk.................................... not responding...
> >sda: READ CAPACITY failed
> >sda: status =1, message =0, host =0, drive =28
> >sda: block size assumed to be 512 bytes, disk size 1GB.
> I think you have the drive jumpered to spin up on command, but this command
> is failing.
> Try setting jumper for this:
> On the back of the drive (opposite end to the SCSI connector) is a jumper
> header block with 24 pins. Holding the drive right side up and looking at
> this block, the pins number 1 and 2 are on the left.
> Counting from the left the 5th pair (numbers 9&10) is for enable/disable
> "auto spin-up".
> Make sure there is no jumper on this pair and it will spin up on power-on.
>
I don't have any jumpers on this pair of pins. I did try it with this
pair enabled once, and the drive spun up when the kernel reported
"Spinning up disk", but even after it spun up the kernel could not detect it.

> To see the diagram for this look at:
> http://www.storage.ibm.com/techsup/hddtech/dcas/dcasjum.htm
>
Thanks, I already had copied this to my local drive for reference.

>
> >The drive is actually 4.3GB. Does this sound like a bad drive or an
> >incompatability between the drive and the NCR 810 SCSI controller?
> >Anybody seen this before?
> We have used many of this model. It is a decent performer, very reliable,
> and not problematic. We have used it on various NCR/Symbios controllers, on
> several models of Alpha.
>
>
It's good to know that the Ultrastar is a solid drive. I chose the IBM
because I have some where I work and they are real quiet compared to the
Seagate drives.

I tried this drive in my 486 running Redhat 4.2. The Future Domain 1680 SCSI
controller immediately reported "Drive not ready" when it went through the
SCSI bus scan at power up. The only jumper pair set was the SCSI id pair.
I then got the "Spinning up disk..." error when
the linux kernel was probing the SCSI bus. I'm starting to think this is a
bad drive.

I've tried removing the internal SCSI cdrom drive in the alpha,
but that didn't
help. I didn't know how to check if the external SCSI bus was terminated,
so I put a external ZIP drive on it that was set to terminate, but that didn't
make any difference. Does anyone know what jumper terminates the external
SCSI bus?


Thanks for all the helpful responses that I've gotten from the list.
