Sigh; here we go again... :-\

There *are* some problems that need fixing via MILO upgrades, most notably
clock "drift" and (rarer) double/half speed. These are/were common on
LX/SX/MIATA, not so on others.

The other problem, also affecting *just* the above machines and PC164, is
totally bogus random dates every once in a while (though it *does* seem to
occur quite frequently during bootup, unfortunately). This one *is* fixed by
a new "clock" binary, as found on gatekeeper. You are afflicted by this
problem if the following results in bogus dates showing up sometimes:

while true; do clock -r -A; done # no -A if booted from SRM

The problem Randy had was that he downloaded the wrong "clock" binary. The
initial fixed version I posted was based on RH 4.2; when one presents such a
file to a RH 5.x system, it *will* think it's not a "command".

PLEASE! Download the appropriate binary from gatekeeper.dec.com:

/pub/DEC/Linux-Alpha/Kernels/clock-pc164-rh50 # for RH 5.x systems
/pub/DEC/Linux-Alpha/Kernels/clock-pc164-rh4.2 # for RH 4.2 systems

And where the heck did you get the "clock.pc164.gz" file anyway?
