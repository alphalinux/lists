On Mon, 12 Oct 1998, Paul Tomblin wrote:
> Quoting Bret Orsburn (borsburn@codonics.com):
> > This is the first I've heard of using the '-s' switch for klogd. What's the
> > rationale for that?
>
> I don't remember, it was just mentioned here once as a "sure-fire cure" which
> of course didn't do anything for making my system more stable.

-s forces klogd to use the syscall interface to kernel messages, instead of
/proc/kmsg. It doesn't solve the problem (of memory corruption due to
missing newlines), but does avoid the "bad file descriptor" symptom.
