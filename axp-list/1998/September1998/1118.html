Err ... I think you should read some documentation about configuring and
setting up TCP/IP networking. Your question shows that you are confusing
many things at once.
If you want to setup routing between interfaces you do not need to fiddle
with the /etc/resolv.conf. This file is only for name resolution i.e.
nameserver lookups and other ways to lookup hostnames.
What you need is a proper interface setup.
The easiest way to do this is to setup to different class C networks. Let's
say for example one network 192.168.1.0 and the second 192.168.2.0
You will then have to configure eth0 to
IP 192.168.1.<some-number> netmask 255.255.255.0
eth1:
IP 192.168.2.<some-number> netmask 255.255.255.0
You should then add two routes to your routing table using
route add -net 192.168.1.0 eth0
route add -net 192.168.2.0 eth1
Now you should be able to ping machines on either network.
If now machines from network 1 should be able to reach machines in network 2
via your machine you will have to add IP forwarding to your kernel
configuration. For 2.0.* kernel this is a kernel compile time option, for
kernels 2.1.* you will have to
echo "1" > /proc/sys/net/ipv4/conf/all/forwarding
which will enable IP forwarding for all interfaces.
