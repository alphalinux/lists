Hi!

I'm having a _very_ hard time in setting up a Samsung 164UX
motherboard and I'm looking for help... :-((

The problem in short: I can not boot the system from hard disk,
because as soon as I create an environment variable set for
Linux, the BIOS quits every attempt to run ldmilo.exe with

"ERROR: Not enough memory to load program."

That is, as soon as I install an environment variable set, I
can't even boot from the MILO floppy!
Booting from the MILO floppy works fine, when there is no
environment variable set installed (using "Run a Program" ->
"a:ldmilo.exe"). That way I initially booted the Red Hat MILO
Floppy.

Some more information: Samsung 164UX MB, 4MB Cache, Alpha 21164CPU
(633 MHz), 256MB RAM, IBM DCAS-34330 UW-SCSI HDD
Installing Red Hat Linux 5.1 initially went without problems,
it's just the last step (setting up the BIOS boot entry) which
doesn't work... :-((

I have no idea what is going on here. Any help would be very much
appreciated!
