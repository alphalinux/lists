Hi,

I wonder if I am the only one who has irregular failures on floppy
drive access on a Alpha system (here DEC AlphaStation 200 4/166)?

It doesn't seem to be hardware related (I already changed the drive
once). The problem occured on any 2.0.x kernel I've tried so far, but
only after uptime of about 2-3 weeks.

If I try to use mtools, I get:

# mdir a:
init: set default params
Cannot initialize 'A:'

Mounting the floppy also fails:

# mount /floppy/
mount: wrong fs type, bad option, bad superblock on /dev/fd0,
or too many mounted file systems

The only workaround to get access to the floppy seems to be a reboot
which is quite unsatisfactory.
