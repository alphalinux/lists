Thanks. Well, first off, I'm one of those people who bought a $300 Alpha
box from the auctioneers. If anyone is curious what kind of machine they
are, and if they should buy one, just follow this thread. I'll let everyone
know the pro's and con's and you should get a pretty good feel for 'em.

I have always wanted an Alpha and finally got a chance. Even if it is only
233...

So far, I've made a floppy using rawrite.exe on the linux CD-ROM. The image
files were avanti.img, ramdisk.img, and resc1440.bin. The workstation comes
with Ultra SCSI 2 onboard, but I have Ultra Wide HD's and wish to disable
the onboard SCSI chip, and use my own PCI Adaptec 2940 UW Controller. I
also have an additional PCI NIC. It is a 3Com 10/100 3C905B TX. This
computer is going to be used primarily for a bridging firewall. The 10
Mbit/sec card included is going to the 4 Mbit/sec internet connection side,
and the 100 Mbit/sec card is on our switched Ethernet side. The video
adapter is a cheezy 1 MB Trident ISA card. My HD is a new unformatted IBM
4.3G UW.

The way it's set up now is, it will boot up to the blue screen with three
pointers. (>>>) I think that's SRM?? I've made it boot to an "aboot"
prompt using the avanti.img disk with the command boot dva0. It will get to
a point where it asks me for the location and name of a kernel. That's
pretty much where I'm stuck.

The firmware still hasn't come in the mail, so I am just fooling around with
the HD connected to a 68-50 pin converter and using the onboard SCSI for
testing. Hopefully, with the arrival of the firmware, I'll be able to add
my SCSI adapter to the list and take off the onboard one.

Thank you for the help and please let me know if you need more specifics,
and I'll try to get them for you.

Steve G.
