Hi.
This is my machine configuration:
Samsung UX/2 with 128 MB SDRAM, Toshiba XM6201B SCSI and IBM 4,5 Wide SCSI HDD.

I would like to install Red Hat Linux 5.1 on this system, so I first
booted from ARCBIOS ldmilo.exe, then inserted the ruffian.img Kernel-Floppy
and then the ramdisk.img from the Red Hat 5.1 CDROM.

So far, so good. All devices (also the CDROM !) are recognized - both
the ARCBIOS and the linux kernel show that everything should be functional.

Then the (infamous) Red Hat CDROM install device selection menu appears -
obviously a critical point in the installation since I remember having had
problems with that in previous releases of Red Hat Linux.
When I coose "Local CDROM" the device is not accessed and an error message
appears "mount failed: invalid argument".
On F4, the following error message can be seen
"CD-ROM I/O Error: dev 0b:00, sector 64
isofs_read_super: bread failed, dev 0b:00 iso_blknum 16 block 32"

Well, I removed the Toshiba SCSI drive and tested it on an Linux/x86 PC -
no errors.
I checked the Red Hat 5.1 installation CD- no errors.

Well, any idea ? Tomorrow I will go and test with another CDROM-drive
(a Pioneer) but my feeling is that this won't work either ...

Does someone have any idea what's going on here ? Thanks !
