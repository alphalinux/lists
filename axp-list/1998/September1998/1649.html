Quoting Jacek Perry (chrobry@yahoo.com):
> I think that I made a mistake in my last question. What I am
> machine, but the nfs still doesn't start automatically. It does start
> however when I go into the /etc/rc.d/init.d and manually run the nfs
> file.
>
> Why is that? What else do I have to change?

Do you have an appropriate symlink in the /etc/rc.d/rc*.d directories? On
mine it's called S15nfsfs, but it might be called something else. But when
booting up the system doesn't look in /etc/rc.d/init.d, it looks in the
appropriate rc?.d directory (if booting the standard runlevel 3, it looks in
/etc/rc.d/rc3.d/) for files begging with a capital S, and executes them in
alphanumeric order with the argument "start". These files are all symlinks to
the scripts in /etc/rc.d/init.d/.
