Steve G wrote:
>
> I thought that you were supposed to un-tar your kernels in the /usr/src dir.
> In my experience, the kernels expand into linux/xxx. In other words, they
> expand into a directory tree starting with linux. The linux symlink refers
> to the actual dir that the kernel is installed in. How can I leave the
> linux dir alone and upgrade the kernel?
>
> Also, does anyone know the RPM file that has the kernel headers in it? I'll
> browse around on the CD, but just in case...
>

The RPM starts with the name "kernel-headers-xxxxx.alpha.rpm" where
xxxxx is a version number depending on which RH distribution
you have.

Yes, the pristine sources start with linux/..... but,
do not let them overwrite your RH installed /usr/src/linux
directory. The post about creating a directory with a
version number is good. I've done that. Also, I've built
kernels in a totally different place. I do builds on a
NFS mounted file system with the source tree so I can cross
mount between all my Linux boxes and use the same tree.
There is no requirement I'm aware of that your kernel
has to be in /usr/src. A side benefit is I keep only one
copy of the kernel source for all my boxes(5 of them).
Those kernels are getting pretty big and if you have a
couple trees around, you see why I don't want them on all
my systems... (I use 100B-TX Enet so NFS works pretty well)
