Jeff Sturm wrote:

> On Wed, 14 Oct 1998, Michal Jaegermann wrote:
> > are really trivial to deal with. If compiling with -mieee flag does
> > not help (exception may happen in libraries which were not compiled
> > with this flag) then install a custom signal handler which just
> > returns on SIGFPE.

-mieee does help. The source currently uses -mieee-with-inexact, but
I've tried with both, and in both cases Mozilla gets past the FPE
but then dies with a segfault somewhere in a system call (I think.)

Of course, my experience with debugging system calls is nonexistant,
so for all I know, this is still related to the FPE, but as it occurs
somewhere in memmove (IIRC), I'm *guessing* it has more to do with
64-bit or memory alignment issues. (Speculation from a tyro...)

> I'd bet that the FPE comes from javascript. I'ts conceivable that
> javascript makes important floating-point calculations sometimes (but not
> on MY web site of course).

The FPE does in fact occur in the javascript code, according to gdb.
If I'm not mistaken, it occurs when Mozilla is examining prefs.js for its
configuration, and gets an FPE during this process. -mieee makes it go
away though, and the subsequent SIGSEGV seems to be unrelated.
