> I stuck in a
> Cirrus Logic PCI card that was lying around. SRM is happy with it.
> I grabbed my avanti-s.img and ramdisk.img floppies and went for it.
> About the time the kernel gets to Probing PCI Hardware, the display
> gets moved elsewhere. I made sure that console was set to graphics,

> Cirrus card. I didn't think this would be a problem, as, AFAIK, the

Sigh; bad choice (Cirrus VGA)... :-\

This card, for whatever reason, does something really weird when you
enable its BIOS ROM: it disables its memory aperture. Major side effect:
exactly the symptoms you describe (the display disappears).

The latest alpha-{sources,patches}-2.0.35-0.1 contain code that detects
the Cirrus and avoids the BIOS ROM enable. Also, the latest AVANTI MILO
on gatekeeper.dec.com:

/pub/DEC/Linux-Alpha/Kernels/Test-Milos-981008

has the fix, though this won't help you if booting from SRM console.

I suggest grabbing the alpha-sources and building your own kernel to run
on that Cirrus-equipped box, or alternatively, putting in a different VGA.

Good luck.
