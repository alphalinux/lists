Steve Frampton wrote:
>
> Hello:
>
> I'm running Linux 2.0.35 on an Alpha XL 233 (yeah, I know :p ). I just
> did some remote administration on it, and upon typing "shutdown -r now", I
> get "The system is going down for reboot NOW !!".
>
> (But then nothing happens, other than a 'switching to runlevel 6' entry
> appear in my /var/log/messages and some daemons restarting).
>
> Has anybody else experienced this problem? How can I get my system to
> restart (other than typing a 'reboot' command!!)?

I've had a similar problem, seemingly connected with the runaway syslogd that so
many posters have mentioned. Speculation: It seems like all of the processes
that want to log anything get hung up indefinitely while syslogd spins.

If you see the problem again, try

kill -KILL <pid-of-syslogd>

and see if things perk up a bit.
