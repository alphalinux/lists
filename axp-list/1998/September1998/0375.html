On Thu, Sep 17, 1998 at 02:16:04PM +0200, Eric@KPN.COM wrote:
> Crosscompiling on SMP PPro

This is your problem.

It is an unfortunate fact that the c preprocessor does not do
arithmetic at the same width as the cross-compiler it was built for.
It is an even more unfortunate fact that this is allowed by ISO C,
making it difficult to convince folks that it needs fixing.

So you'll have to change all of the bits that do this

#if ((~0UL) == 0xffffffff)

and just have the code assume 64-bit data types.

Even beyond this, be warned that cross-compiling from a 32-bit host
will produce _significantly_ worse code that compiling native.
