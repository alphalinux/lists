On Sat, 12 Sep 1998, Paul Tomblin wrote:

> I'm trying to use a parallel port zip drive which I found at work.
> Unfortunately, it doesn't seem to be working. I compiled it as a module, and
> when I insmod it, it says:
>
> #insmod ppa
> ppa: Version 1.42
> ppa: Probing port 0378
> ppa: Probing port 0278
> scsi : 1 host.
> ./ppa.o: init_module: Device or resource busy

Which kernel version are you using? In 2.0.x, you mustn't have lp and ppa
loaded at the same time. If lp is compiled into the kernel, boot with
lp=0 on the kernel command line.

"dmesg" might have some useful log information from ppa.
