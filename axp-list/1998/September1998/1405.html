On Thu, Oct 08, 1998 at 12:50:28PM -0500, Steve G wrote:
> Boot off the LAN... That doesn't sound to horrible. What kind of server do
> you have to run to enable this sort of thing. I know it's a BOOTP sort of
> thing, but what are the in's and out's of configuring it to boot an Alpha
> box? I'm pretty sure Linux can serve as a BOOTP server. I've never had the
> pleasure of setting one up, but maybe now would be a good time to learn.

It's easy. I have an x86 Debian/Linux box running as my bootp server.
You also need tftp enabled.

>From my bootptab:
-- snip --
.e0default::sm=255.255.255.248:gw=mystic:sa=mystic:ht=1
harley:ha=00.00.F8.23.40.FB:bf=/tftpboot/harley-bf:ip=harley:tc=.e0default
-- snip --

Make sure /etc/hosts or DNS is set up for the new box, or else use
IP addresses. Also, your subnet mask is undoubtedly different.

Harley is an AS200 4/233. /tftpboot/harley-bf is a symbolic link pointing
to the kernel image I want harley to boot. I'm using aboot to make the
vmlinuz.gz files net-bootable. Harley is running SRM.

FWIW, Harley does not have a video card or keyboard either. The console
is serial, 9600bps.

Booting is easy:
>>> boot ewa0 -fl "root=/dev/sda3 ro"

You can also set SRM environment variables to allow autobooting and so on.

> I think I did but I don't know what to do with it now that I have it.
> As200_v6_7.exe is the file. I'm not sure how to run an exe file on the
> alpha box.
>
> Is my newness to alpha showing up yet???

I remember having trouble with that too. I think I ended up loading those
files using bootp as well. In fact, I'm sure of it. Just use the EXE file
instead of a kernel image.
