>>> Charl Botha said:
> On Sun, 11 Oct 1998, Jeff DeMaagd wrote:
> >
> > I have a DEC Personal Workstation 500a with a built-in DEC 21143 10/100
> > networking chip, with the bulkhead connector. I am getting frustrated
> > because I've tried the RH 2.0.34 kernel's de4x5 and tulip drivers, and the
> > latest tulip from CEDIS. I just tried a fresh reinstall because my network
> > If I try the tulip driver, it claims that my link is down, which is NOT the
> > case in NT, and in both cases, the link light on the hub is on, indicating
> > a link up.
>
> I had something similar on this alpha which has a DEC 21140 networking
> chip. In my case, the problem was that the card/software didn't autosense
> 100Mbit correctly on every boot. I'd sometimes need to reboot repeatedly
> to get it to sense 100Mbit. Symptons when it didn't sense were identical
> to those documented above.
>
> You can try to compile in the de4x5.c driver, and hardwire it to
> 100Mbit/s. Have a look in the de4x5.c source code -- one has to activate
> a certain define with certain values; hereafter the card should always
> come up in 100Mbit/s mode.

That workaround (hardwire the de4x5 driver to 100MBit) has worked well in
the past. Also meeting with some success is using Donald Becker's latest
tulip driver (0.89H?) and providing a media type argument on the boot
command line, to perform the "hardwiring" at the last minute. Check his
website for details (http://cesdis.gsfc.nasa.gov/linux/drivers/tulip.html).

The builtin 21143 on the MIATA only seems to be a problem at 100Mbit, which,
unfortunately, is becoming more widespread. The biggest problem is that
neither driver writer has had decent access to a MIATA... :-\
