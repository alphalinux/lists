On Mon, 28 Sep 1998, Bob McElrath wrote:

> > For anybody who has experience getting DU binaries working under Redhat, a
> > few pointers would be greatly appreciated.
> >
> > With a different DU binary, I get the following error:
> > ##### 2 #####
> > 4175:/mnt/usr-local/bin/a.out: /sbin/loader: Fatal Error:
> > set_program_attributes failed to set heap start
> > #############
>
> I get the same problem with DU netscape on a RedHat 5.1 system, kernel
> 2.0.34. I also had to install three extra libs not mentioned by that FAQ:
> libots.so, libcxx.so, libexc.so. Have you gotten this working yet?

On suggestion, I downloaded and compiled one of the experimental
kernels: 2.1.122. After that, it was a snap. I am able to run DU
Matlab, Netscape 3.0, and every other binary I try. In fact, to run DU
Netscape and every other DU binary, the only libraries I had to copy were:

% ls /usr/shlib
libICE.so libXext.so libXt.so libdnet_stub.so
libSM.so libXm.so libc.so libm.so
libX11.so libXmu.so libcurses.so

Matlab required some of its own libraries to be copied as well.
If anybody wants to know the exact steps that I took to get this working,
please let me know and I'll try to type something up.
