On 22-Sep-98 Jay.Estabrook@digital.com wrote:
>
> If you have a "fancy" new 53c8xx board in the LX, perhaps the latest MILO
> (using rev 2.6n of the ncr53c8xx driver) can handle it, and the linux kernel
> sources you're using actually have an older driver that can't?

In general a SCSI adapter does not know the difference between an external and
an internal drive, they are electrically on the same buss. The difference
between external and internal devices is mechanical not electrical and I doubt
that any driver would know the difference between them

I would guess this is some sort of cabling/termination problem, as are most
SCSI problems in my experience. Remember that the on=board termination must be
enabled if you are using only internal or only external devices, but disabled
if you are using both (some cards have an "auto" mode).

Also, some wide cards have three connectors (internal narrow, internal wide and
external wide). You may run into trouble if you attempt to use all three of
these ports at once.
