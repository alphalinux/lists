It's been 15 days since I upgraded to RedHat 5.1 and I haven't seen the
syslogd go nuts yet. This is almost a record (I got 35+ days last month
but I was logging remotely). This is:
- A UDB with several busy servers running (mail, news, ftp)
- RedHat 5.1 with all patches and updates applied
- 2.0.35 with the fix that George Gatgul reported about adding a newline to
several kernel messages.
- klogd running with the '-s' flag.

Before doing all those things, I was lucky to get two days without syslogd
running nuts.
