Steve G wrote:
>
> 2: I un-tarr'ed all of the kernel's into the /usr/src dir. There is a
> direcory /usr/src/linux-2.0.34 there. /usr/src/linux is a symbolic link to
> that dir. All kernels unzippped into /usr/src/linux.

Like I wrote before - DON'T DO THAT!!! Change names and links to
keep things separately. In your way you drop sources, and **headers**
from different versions on one big pile. You need some experience
to later sort that out if at all possible.

> setup.c: In function `get_cpuinfo':
> setup.c:753: too many arguments to function `smp_info'
> setup.c:753: warning: return makes integer from pointer without a cast

The above really means, most likely, that you have 'get_cpuinfo()'
functions from one version and a header which declares it from another
and and two interfaces are **different**. Probably you have also
somewhere a correct 'get_cpuinfo()' declaration but a compiler found
a "wrong" one first. Count yourself lucky. If you would find here
a correct declaration but some other file, which is using it, found
a wrong one and if you would managed to compile things anyway then
you would end up with a spectacular kernel crash.
