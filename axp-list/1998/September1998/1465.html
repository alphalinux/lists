>>> Marcelo Tosatti said:
>
> I'm having some problems installing RH 5.1 on an AlphaServer 800 5/333
> The boot image that i'm using is noritake-s-5.

You've a number of problems to overcome:

1. the noritake-s-5 image on RH 5.1 contains an incorrect kernel.

2. the RH 5.1 installation scripts don't handle disks with BSD-style
partition tables.

3. GLIBC won't handle NORITAKE family 5/xxx model machines correctly by
default, so that "clock" and X won't work.

But fear not, there are workarounds for each... :-)

1. grab the latest NORITAKE kernel from gatekeeper.dec.com:

/pub/DEC/Linux-Alpha/Kernels/noritake-s-5-980831.gz

and use it to replace the vmlinux.gz on your boot floppy.

2. use the updated ramdisk.img from RH 5.0 to install with; it works
just fine, it seems, with the 5.1 CD.

3. just *before* the X configuration is attempted, toggle over to VC2
(via Alt-F2), and do:

cd /mnt/etc # to the newly installed HD
ln -s Alcor alpha_systype # GLIBC will look for this
cd /etc # to the RAMdisk
ln -s Alcor alpha_systype # just to make sure

(there may be an error reported, but I think it's harmless).
then continue the install (toggle back via Alt-F1) and all should
finish up correctly.

Good luck.
