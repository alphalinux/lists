Greg Lindahl wrote:
>
> >
> > No, actually we don't (unless this is a lack of a guard against
> > a buffer overflow somewhere where there is no way to control input
> > data).
>
> If syslogd doesn't have a bug, why is it reading from a fd which is
> too large? It must have a bug.

Most likely local variables on stack were clobbered with a garbage
because of a buffer overflow. This will create all kind of funny values
and side-effects. This is a guess, but educated one. :-)

If something assumes that its input satisfies some constraints, and
this input cannot be influenced by a user, then this assumption
is reasonable. When "producers" stop conforming then all bets are off.
Of course this is a bug but you are free to interpret on which side
of an interface and what is the simplest fix.
