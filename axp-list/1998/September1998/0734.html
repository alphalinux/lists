I've been running syslogd with remote logging to an Intel Linux machine, and
I went:

#uptime
9:03pm up 11 days, 6:01, 3 users, load average: 0.81, 2.50, 3.94

without the usual syslogd problem. But it just happened again. The machine
is running out of inodes (files are fine - file-nr is 704), so I increased
inode-max to 4096, and it immediately ran out again. So I increased it to
8192, and it's almost up there again:

#cat /proc/sys/kernel/inode-nr
7840 7611

I can't figure out how to figure out what's chewing up those inodes. Any idea
how I can track that?

I'm not sure if it's significant, but I'm currently getting bombed by newgroup
messages from the HIPCRIME spammers. But I've weathered many of those attacks
in the past without this happening.
