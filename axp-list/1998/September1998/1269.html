Hi,

I make a fast sqrt routine for 21164 which runs as fast as DEC's
libm(a little bit faster). This souce also includes various
exceptional handling(NaN, +-Inf and Subnormal(Not Emulation!, it runs
as fast as glibc's assembler routine)).

This is preliminary source, because of accuracy. Although relative
error is within 2 ulp(same as libffm's sqrt routine), it shows a
little bit WORSE than libffm's. Now I'm trying to improve accuracy.
Does anyone know how to make "correct" table?

If you're interested in, you can find at

(file size is about 4kB)
