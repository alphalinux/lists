That would be great. I've just tried to boot the rawhide-s.img from SRM
this way :

boot -file vmlinux.gz -flags "ramdisk_load=1 ramdisk_prompt=1" dva0

the result :

kernel loaded and some hardware was detected (most importantly the
Tulip 10/100MB 21164(I think) network card). But at the end it said
something like :

VFS: panic, cannot mount FS on device 08:02.

and that was the end of that. I didn't get any further. I think that
this is because the boot cmd didn't hand over to the kernel the
``ramdisk_prompt=1'' parameter, but that's just a guess. When the system
was initializing, it had show on the screen what it was booting, and it
only was :

( boot dva0 -file vmlinux.gz -flags ramdisk_load=1 )

the ``ramdisk_prompt=1'' wasn't shown. Maybe there is some other way to
give it the params. I've also tried
``-flags ramdisk_load=1,ramdisk_prompt=1'', as the "help boot" cmd in the
SRM console suggests that -flags param be of the form
``-flags longword[[,longword],...]''. That seemed to be passed to the
kernel OK, but I guess the kernel parses the flags with space and not
comma, so that wouldn't work either. Any help here would be appreciated
too.

Is there a way (with alpha-like rdev or something) to put a flag onto the
rawhide-s.img boot floppy that it should prompt and expect secondary
(ramdisk) floppy ? That way I wouldn't have to deal with boot params from
SRM console.
