I'm having some problems with an ISA Pro Audio Spectrum 16 card on a DEC
AlphaPC164LX motherboard. I have configured the kernel (2.1.125) to use
both the PAS and its Sound Blaster emulator. (2 audio devices). However,
when I attempt to use the PAS I get the following error:

Oct 16 02:49:45 draal kernel: Sound: DMA (output) timed out - IRQ/DRQ config error?

When I attempt to use the SB emulation of the card I get:

Oct 16 02:52:10 draal kernel: Sound: DMA (output) timed out - IRQ/DRQ config error?
Oct 16 02:52:10 draal kernel: Sound: Error. Audio interrupt (0) after freeing buffers.
Oct 16 02:52:11 draal last message repeated 79 times

The obvious answer is that I've got the IRQ and/or DMA configured
wrong...but I don't (I've checked several times, both the SB and PAS are
configured properly wrt IRQ/DMA). After playing several files,
/proc/interupts reveals:

(0)<root@draal:/root> cat /proc/interrupts
1: 3496 keyboard
4: 2397 + serial
5: 25 soundblaster
8: 670187 timer
10: 0 PAS16
16: 6755 BusLogic BT-958
17: 9279 eth0

Could there be a general problem with ISA interrupts on this board?

thanks,
-- Bob
