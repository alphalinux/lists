I had that problem with the gnome-lib tar balls but not with the srpms. Anyways
I finally got gnome compiled. They're all alpha rpms and they were compiled on
a 533LX. A list of what doesn't work:

background properties dies on startup and continues to eat cpu time, you have
to kill the process.

gnome-libs complains that libgmodule-1.1.so.3 isn't installed but it is under a
slightly differnt suffix, cp the file to the right sufix and you're set or just
--nodeps.

DO NOT INSTALL GTK+-10.0.6 AND GLIB+-10.0.6
They'll screw up your entiire system just do a --nodeps with
glib-1.3 and gtk+1.2. You may be able to make the 1.06 libs from tarballs
abd put the one s with the 1.0.6 extension in by hand but I wouldn't recommend
it. It works fine without it.

gtk+ won't see that glib is installed until you install glibs devel libraries,
weird.

libungif makes everything look like crap. install giflib and cp the approiate
libs to look like they're from libungif. You'll know which ones to fix when rpm
starts to whine.

Balsa runs but crashes when you reply to a message. Don't use it.

gmc starts up with a tool bar and a blank window. It just hangs kill
it

BUT, The panel works flawlesslly you can put in applets and whatever and it
doesn't crash.

I'm going to upload the rpms this week to gnome and I also have a bunch of
rpms for enlightenment that I'll upload too including imlib-1.8 and fnlib.

Does anyone know were the config file is for background properties so I can
set my screensaver etc..

Oh yeah GNOME + E are both wicked fast compared to KDE :).
