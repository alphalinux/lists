Is there anyone else out there who has had or seen problems rebooting
a RedHat 5.1 AlphaStation 200 with SRM (and solved them)?

shutdown -r now halts my system cleanly, but doesn't reboot.
It waits at the SRM console prompt, and the very next SRM command typed,
every one I've tried, gives an error, displays a stream of memory addresses,
and reboots. Like so:

halt code = 5 server to shut down ....
HALT instruction executed
PC = fffffc0000313340
boot failure
>>>

shutdown -h now halts my system cleanly, but hangs,
not even displaying the SRM console, like so:

The system is halted
<0>System halted

It comes up fine at power on, but both shutdowns require a manual
power cycle to reboot, which sucks.

NetBSD on an AXP from the same lot doesn't have this problem.
Anyone know what's up?
