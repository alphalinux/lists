Many thanks to Govindan Chandran for the information on how to install
RH5.1 off CDROM

The machine had an adaptec 2940UW + 4GB + SCSI CDROM

booting off floppy at the Milo prompt caused the ramdisk.img part to
hang the system with the floppy light staying on endlessly.

I tried booting off the CDROM from MILO as follows...
Milo> boot -t iso9660 sr0:kernels/sx164.gz root=/dev/sr0

This loads the kernel off the CDROM and loads of text scroll by. The
last few lines are...
Partition checksda: sda1 sda2 sdb: sdb1
VFS: cannot open root device 00:00
Kernel panic: VFS: Unable to mount root fs on 00:00

Given that Govindan had managed to install using IDE drives I swapped
out the whole lot and installed an IDE CDROM drive and two IDE hard
disks. The command...

Milo> Boot -t iso9660 hdc:kernels/sx164.gz root=/dev/hdc

successfully started the installation process.

Is that likely to cause problems?
