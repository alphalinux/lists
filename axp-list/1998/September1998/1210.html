On Mon, Sep 28, 1998 at 12:55:28AM -0400, Keith Baker wrote:
> even any clues? As I have never touched the kernel I have no idea...

I'm assuming you've configured your kernel for IP masqing tho'. If not,
read the IP Masquerading mini-HOWTO.

The problem with ipautofw is the header files: the glibc and kernel
header files tend to step on each other. The solution is to cut some of
them out.

Replace this block of #include's in ipautofw-2.0.0.c:

#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <netinet/ip.h>
#include <netinet/udp.h>
#include <netinet/ip_icmp.h>
#include <linux/if.h>
#include <linux/timer.h>
#include <linux/ip_fw.h>
#include <netdb.h>

With this one:

#define IFNAMSIZ 16

#include <sys/types.h>
#include <sys/socket.h>
#include <asm/types.h>
#include <linux/timer.h>
#include <linux/ip_fw.h>
#include <netdb.h>
