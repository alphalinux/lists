>
> Hello everyone,
>
> I have a few questions for those of you who've been in this situation.
> Firstly, I have an Alphastation 233.
>
> I'm trying to upgrade my kernel because I want to use my linux box as a
> bridging firewall and also a few other things that I need a new kernel for.
> Namely my 3com 905 NIC I need support for.
>
> I will pull down the .tar.gz file, expand it into /usr/src/linux, and then
> run make mrproper. Then Make menuconfig because I the menu way is faster
> for me. I'll save the file and go on to run make dep.
>
> So far so good.
>

> dir's. Is there something I need to do special before running make boot to
> tell it this is an alpha or something?
>
> I boot off of a floppy from an SRM (>>>) prompt. I need to create a kernel,

> an alpha before, and making the kernel small enough to put on a floppy is
> also worrying me.
>
> HELP ME!!! This thing is driving me crazy. I've tried for or five

I read on this list a while back that "make menuconfig" can cause
problems. Try using "make config". Also you want to "make boot"
instead of "make Zimage". "make boot" will create a vmlinux.gz file
under arch/alpha/boot/
