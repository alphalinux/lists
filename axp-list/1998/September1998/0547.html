The other day, I attempted to build the OpenLDAP package
on my Red Hat 5.1 box. OpenLDAP has threading support, and
the threading library it uses snarfs SIGUSR1 and SIGUSR2.
OpenLDAP itself, however, also requires the availability of two
signals.

OpenLDAP has an #ifdef LINUX around the points where it uses
these signals; however, it uses SIGSTKFLT and SIGUNUSED,
which are present on Linux/Intel but not on Linux/Alpha.

In other words, the authors of OpenLDAP have assumed that
all Linuxes have the same signum.h, whatever platform they're
running on. The Alpha signum.h says:

/*
* Linux/AXP has different signal numbers that Linux/i386: I'm trying
* to make it OSF/1 binary compatible, at least for normal binaries.
*/

While this is a laudable goal, I suppose, it bothers me that this
was done in this particular way: why does Linux/Alpha try to be
compatible with OSF/1 at the expense of compatibility with Linux?

At any rate, what is the feasibility of extending the number of
signals to include all the Linux/Intel signals? (Or at least
all the signals that make sense on the Alpha architecture.)

Granted, this is obscure and pretty arcane, but I think it would
be a good idea to make sure, as much as possible, that when
Joe Programmer says "#ifdef LINUX" it's compatible with ALL
Linuxes.
