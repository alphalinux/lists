On Thu, 1 Oct 1998, Brian McQuay wrote:

> I've got an Avanti AlphaStation 200 4/233
> I don't like using the damn floppy to boot milo. Is it possible to boot
> milo from a small sda partition? How do I get SRM to boot that partition?

I'm not sure I understand exactly what you're trying to do. You must use
aboot if you wish to boot with the SRM console. You must use MILO if you
wish to boot with the ARC console. AFAIK, MILO requires a DOS FAT
filesystem to exist somewhere, either on floppy or on the fixed disk. SRM
does not require any DOS filesystem, but does require the use of BSD
disklabels instead of PC-style partition tables, and some reserved space on
the beginning of the target drive. Richard Payne has compiled a list of
links for AS 200 installation for both consoles. The URL for that is
http://adam.cheshire.net/~rdp/alpha/as200.html. The current RedHat-5.1
aboot RPM should be sufficient for booting 2.0 kernels. If you upgrade to
2.1, you'll need Maciej Rozycki's patched aboot RPM. The source for that
is:

ftp://ftp.amg.gda.pl/pub/OS/Linux/people/macro/SRPMS.glibc21/aboot-0.5-6.src.rpm

I hope this helps,
