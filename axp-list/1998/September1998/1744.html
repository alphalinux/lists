On Fri, 16 Oct 1998, Maurice Hilarius wrote:

> >If you bought the MIDI adaptor (not standard) and chose to use the game
> >port. The game port is going to use I/O ports anyway, whether it's on
> >this board or not.
>
> It was on most of them.. At least the ones I sold.

Sorry, that was not clear. The MIDI was optional; the game port is
standard. However, a game port is going to have the same resource
requirements whether physically located on the motherboard, the PAS, or
another addon card.

> On an X86 based system one often could not afford all the IRQs it wanted.

My PAS used one IRQ. Most modern SBs use two. Under OS/2 it used two,
because I used the PAS for OS/2 native and the SB for Windows. I never
needed to use IRQ based SCSI, and it worked happily (albeit slowly) in
polling mode.

> Also I have seen instances where it and a Buslogic SCSI controller would by
> default both use the same I/O port (300) and a good game of Quake would end
> up trashing your SCSI hard disk. I saw a couple of machines from Micron
> that came configured with this combination, and when the owners installed
> Linux it was quite a disaster.

?? Odd. It is configurable, tho... Too bad that the PAS support in
Linux seems to be on the rather weak side, at least in comparison to what
it can do. Guess things like that happen...
