><div><font face="TTYFont" color="#000000"># Two variables, ARCHI and
>ORG, have to be set to machine-dependent values.<br>
># ARCHI is used to set the cache size to the correct value,<br>
># and thus the block size. The block sizes are included in<br>
># two files blockd.h and blocks.h that give the block size<br>
># for double and single precision, respectively. Additional<br>
># architectures can be included by editing these two files.<br>
>#<br>
># The block size NB is the largest even number such that<br>
>#<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp;&nbsp;&nbsp; 3*(NB**2)*PREC &lt; CS<br>
>#<br>
># where PREC is the number of bytes corresponding to the<br>
># precision (4 bytes using single precision and 8 bytes<br>
># using double precision in IEEE format) and CS the cache size in
>bytes.<br>
>#<br>
># The valid options for ARCHI are the following:<br>
># STANDARD, SPARC10, RS6K32, RS6K64, and DECALPHA.<br>
># In the STANDARD option the block size<br>
># is determined using a cache size equal to 64KB.<br>
>#<br>
># The valid options for ORG are NOTRIADIC or TRIADIC.<br>
># ORG should be set to TRIADIC on<br>
># architectures where the hardware supports triadic operations<br>
># (for example the IBM RS6000 that implements floating-point
>multiply-and-add).<br>
># Moreover, this option should always be used except when you are
>sure<br>
># that the compiler you are using does not generate efficient code
>for<br>
># triadic operations (early SUN compilers for example).<br>
>#<br>
># It is important to select the correct version for the IBM since<br>
># we use different organization of computations<br>
># and timing procedures in these two versions.<br>
># RS6K32 corresponds to an RS6000 using a 32KB data cache (750 or
>SP1)<br>
># and RS6K64 to processors using a 64KB cache (550 or SP2).<br>
>#<br>
># Finally on some DEC Alpha compilers it is necessary to use the
>alternate<br>
># call to the c preprocessor (cpp) since it is not called properly
>from<br>
># cc for this machine.&nbsp; The cc call and its alternate can be
>found near<br>
># the end of the makefile.<br>
>#<br>
># Options for some RISC processors :<br>
># ================================<br>
>#<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - DEC Alpha EV4, CRAY T3D : ARCHI =
>DECALPHA4,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ORG = TRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - DEC Alpha
>EV5&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; :
>ARCHI = DECALPHA5,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ORG =
>TRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - HP&nbsp;
>PA7100&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp;&nbsp;&nbsp; : ARCHI =
>STANDARD,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ORG =
>TRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - IBM
>RS6000&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp;&nbsp;&nbsp; : ARCHI = RS6K32 or RS6K64, ORG =
>TRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - SUN SPARC10 and SPARC20 : ARCHI =
>SPARC10,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ORG =
>NOTRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - MEIKO
>CS2-HA&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp; : ARCHI =
>CS2HA,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp; ORG = NOTRIADIC<br>
>#&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - SGI Power
>Challenge&nbsp;&nbsp;&nbsp;&nbsp; : ARCHI =
>SGI,&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<span
>></span>&nbsp;&nbsp;&nbsp; ORG = TRIADIC<br>
>#<br>
># By default select ARCHI = STANDARD and ORG = TRIADIC.</font></div>
