The bootloader has to be in consecutive blocks, that is correct.
For the rest, the SRM imposes no extra constraints on the position.
It can be in any "gap" between filesystems, (or before the first, or
after the last. The most logical place is IMHO before the first
partition.
IIRC the disklabel has block#2 (byte offset 512-1023) reserved as an "options" block (somebody correct me if Im wrong?).
In my experience with NoName the bootblock et.al can very well
be managed/manipulated using dd. It does require a working system,
but this does not have to be an alpha.
It does require some courage (and precautions) but just dd and cat
and the appropriate milo did it for me.Altering the disklabeled partition table is not difficult either, but
you do have to recalc the checksum after changes.
BTW the checksum is at the same offset as the 55aa magic that RH5.1
writes to the disk when it nukes it. It also nukes the starting sec#/ nsectors
/ flag field, as noted befor.
