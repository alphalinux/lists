There is a problem in ext2fs with large directory entries in filesystems.
I have a directory tree such that I have many files as:

+--+-- ~700 files
| +--+-- ~400 files
| | +-- ~500 files
| | | +-- ~300 files
| | +-- ~150 files
| +-- ~600 files
|-- etc

And of which many are duplicates. So I created a directory above this,
and started doing:

ln */* ../all-dir
ln */*/* ../all-dir
ln */*/*/* ../all-dir

(it wasn't that easy as bash ran out of room for expansion) but needless
to say, at the end of it all, I had a single directory with around 900
files. I left the hard links intact while I tried to sort everything out,
and virtually every time I reboot it complains and forces a disk check,
often losing a few files/links/something (?!?)

So there is an existing problem with either multiple files in a directory,
multiple hard links in a directory, or removing one of these from such a
directory, or something else related to this setup.
