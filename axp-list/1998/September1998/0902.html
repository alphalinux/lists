Quoting Richard Bullington (rbulling@obscure.org):
>
> I'm wondering if anyone out there has successfully run a UDB with more
> than 64MB of RAM. I'm wondering if the pair of SIMMs I'm trying to install
> is bad, or whether the UDB and Linux just don't like memory > 64Mb.

I've got 96Mb in mine. I didn't have to change anything, just throw the two
32Mb SIMMs in (it had two 16s when I bought it) and reboot.

canoe:~> free
total used free shared buffers cached
Mem: 94704 63016 31688 33496 2784 13504
-/+ buffers/cache: 46728 47976
Swap: 82496 19216 63280

Man, it got a *lot* faster when I upgraded, though.

Check out the UDB technical manual (the PDF file is around on various Alpha
web pages), because they may require the two big ones in a specific bank.
