>>> Peter S Galbraith said:
>
> > We have 2 working alpha 164LX running modified versions of RedHat 5.0
> > (C++ compiler and other stuff) and Linux 2.0.33.
> > The machines have NCR53C8XX SCSI cards. The problem is that while external
> > SCSI devices are recognized by the `BIOS' and the first MILO boot, they are
> > not detected during the final Linux boot stage.
>
> I have managed to compile various kernels (2.0.30, 2.0.34; even omitting
> extra stuff like ethernet drivers) and none of them recognize external
> SCSI devices. Deeply weird.

What version of the SCSI driver does MILO use, and which the kernel?

If you have a "fancy" new 53c8xx board in the LX, perhaps the latest MILO
(using rev 2.6n of the ncr53c8xx driver) can handle it, and the linux kernel
sources you're using actually have an older driver that can't?
