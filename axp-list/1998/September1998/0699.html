On Wed, Sep 23, 1998 at 04:53:55PM +0200, Charl Botha wrote:
> When doing xmkmf in the toplevel of vnc, the necessary imake
> AlphaArchitecture variable is not set, so none of the right switches for
> the Makefile get set either.

It sounds like someone is invoking cpp directly, rather than
using gcc -E. The way the preprocessor definitions are created
was rearranged both to be ISO C namespace compliant as well as
allowing ev56 and ev6 to be detected through preprocessor macros.

Likely just replacing /lib/cpp with a script to invoke gcc -E
(as it should have been in the first place) will do the job.
