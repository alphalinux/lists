Rick van der Linde wrote:

> Hi,
>
> At the moment I'm trying to make my new Alphastation 255 to work
> with Linux. I can start MILO now. But I wonder if it is possible to
> boot a ramdisk kernel ala install floppies af Slackware for the
> Intel. I fetched the ramdisk.img file from gatekeeper but this one
> is at first suprisingly a compressed image and secondly starts
> talking about a magic number. Are there any solutions?
>
> I would like to boot a simple kernel ala slackware install and
> partition my drive, format it and install files on it. So I should
> be able to run fdisk, mke2fs and mount.

The ramdisk.img is for a redhat installer, which requires two disks: a
kernel image and a ramdisk. You boot the kernel from milo, and the
kernel asks for the ramdisk floppy and loads it.

In the same directory as the ramdisk.img (gatekeeper.dec.com
/pub/linux/redhat/redhat-5.1/alpha/images), there should be a kernel
image for your machine. (E.g. I use lx164.img for my LX164 alpha.)

Check out the README file in the redhat5.1/alpha directory.
