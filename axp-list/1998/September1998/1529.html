Greg Lindahl wrote:
>
> > unsigned long hidden; /* hidden sectors (unused) */
> > unsigned long total_sect; /* number of sectors (if sectors == 0) */
> > ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
>
> Some days I wonder how anything ever runs on the Alpha!

There is quite a bit of software around written a bit cleaner. :-)
In many cases these cleanup jobs started many years before the first
Alphas showed on a horizon (I know, as I have some personal
contributions into that :-).

Most likely the qouted code is derived from some MS-DOS documentation.
On 16-bit machine if you will declare the above as 'unsigned int'
you will get sizeof 2 and you will come out short (pun intended :-).
A trailing 'unsigned short boot_sign' at the structure end is also not
guaranteed to give you always 16-bits so a signature may end up
in a wrong place even if you have chances of survival on a "little
endian" machine. What is more - without '__atribute__((packed))' you
do not have a shadow of guarantee that a layout of the whole
structure is without holes (C Standard says so and "packed" is GNU-ism)
so you may screw all this up anyway. Linux kernel headers are quite
a bit more careful.

A code with hidden layout assumptions like quoted is a major portability
PITA. Big sigh!

I knew that there was a good reason not to use "mkdosfs" on Alpha
(and not a real need as well :-).
