>>> "North Star@ives (Brendan Louis)" said:
>
> Does anyone out there know if the AlphaServer 2100 ( not 2100A ) is
> supported in RH 5.1 or if anyone out there has actually installed RH 5.1
> successfully on such a machine ?

It is supported in RH 5.1, however if you have a 5/xxx model, you must
grab a working kernel from gatekeeper.dec.com:

/pub/DEC/Linux-Alpha/Kernels/Test-pre34-11b/SABLE-s-5.gz

as both the matching kernel and image file on the RH 5.1 CD are for
the 4/xxx models, due to some mix-up.

> I went through the "supported platforms" section on the RedHat site, and
> didn't see the AS 2100 on the list.

The Hardware Compatibility List for RH 5.1 was recently updated, and does
now list AS2100 under the "SRM Compatible Systems" heading.

> Also, if it is supported, does the base RH 5.1 automatically install the
> kernel to take advantage of it's SMP capabilities or would it have to be
> re-compiled for SMP ?

RH does NOT include SMP kernels on their distributions, Alpha or otherwise,
AFAIK. So yes, you would have to build an SMP kernel. At this time, one can
apply Richard Henderson's d-axp-gen-14.gz patches against pristine 2.1.120
and come up with an SMP kernel that appears to work; although I've not tested
it on SABLE, it seems very stable in UP mode on RAWHIDE and DP264, and a bit
unstable in SMP on those platforms. We are working on it.

Good luck.
