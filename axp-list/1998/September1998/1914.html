Overall, I'm very happy with the way our Alpha-Linux machines are working.
However, booting is a lengthy and very annoying procedure. I'm not sure
what screwed up the way services are started, but it must be fixed soon.
I'm running kernel 2.1.122 on RedHat 5.0 with a lot of updates.

The basic problem: booting takes upwards of 10-15 minutes with several
very lengthy timeouts. The relevant error mesages with comments and a few
modification are shown below:

Please let me know what other relevant machine, software, error
information I should provide. I think the network startup and routing is
happening as it should (i.e. a "route -n" and "ifconfig" seem to show the
expected output immediately after boot: simply LAN).

I'm going to Runlevel 3 because it takes another 10 minutes of delays if
it tries to go to Runlevel 5 directly (many other services timeout).
However, once Runlevel 3 is reached, a "% telinit 5" takes only a few
seconds to switch (I get the "SIOCADDRT: Operation not supported by
device" again).

NOTE: Domainname is set correctly in the rc.local script but I don't think
this is the problem. The machine never used to do this--- on the other
hand, it never tried to do nfs mounts before. I also get the error
message "nfs warning: mount version older than kernel". I installed a
newer version of mount (mount-2.8a2) which changed some of the boottime
error messages, but I still got the nfs warning.
