On Fri, 18 Sep 1998, Secret Asian Man wrote:

> In addition, if you need help I can tell you how to install RedHat over
> NFS too (makes it much easier than manually unpacking RPMS all the

Well, I got to the point where I could really use the help now. I've
booted RH-5.1 install, and (since no SCSI controllers and thus no SCSI
disks were detected, it wouldn't let me proceed with the normal install.
So I went (after setting up networking) trough the "Menu" straight to
"find installatino files" menu item. But that said "cannot find anything
in /tmp/image. So in TTY2 I said :
_____
# mkdir /tmp/rhimage
# mount -t nfs 192.168.232.99:/RedHat /tmp/rhimage
rpc mount: RPC:Timed out
mount failed: Invalid argument
~~~~~
AND IN tty4, i think, the following appeared:

<4> nfs warning : mount version older than kernel
<4> NFS: mount program didn't pass remote address!

so I checked linux version :
# cat /proc/version
Linux 2.1.95

so I guess I need a 2.0.xx kernel for the RH-5.1 to work.
so I guess I need a rawhide-s.img with kernel 2.0.xx.

Does anybody know how can I get one of these ?

Thanx,

JAN

PS : I did check that the NFS server worked and allowed mounts on the
/RedHat dir. Also I could ping the Alpha from the NFS server.
