> The second problem is that some programs, e.g. xdm and WindowMaker, use
> /lib/cpp to parse configuration files. "gcc -E" works on files with the
> expected .{c,cc,C,h} (and others) extensions, but when passed a file with
> an unknown extension, also does not work, but rather exits with an error
> message.
>
> This problem will have to be looked into somewhat more...

Well, gcc *has* a cpp program within it, it's just buried...

It's a bit risky, but you could symlink something like:

/usr/lib/gcc-lib/alpha-redhat-linux/egcs-2.90.27/cpp

to /lib/cpp and it should behave as expected.

[the reason it's risky is that the pathname isn't constant; if you
install a different version of gcc you need to update the symlink
as well...]
