I'm wondering if anyone out there has successfully run a UDB with more
than 64MB of RAM. I'm wondering if the pair of SIMMs I'm trying to install
is bad, or whether the UDB and Linux just don't like memory > 64Mb.

I've been running a UDB with 64 MB of RAM for more than a year. It has 2
8x36 (32MB) SIMMs in it, 60ns I think. I've tried to add some IBM 70ns
4x36 (16MB) SIMMs to it, without success.

The machine boots with the memory in place, but unless I reduce the memory
size to 92MB with a "mem=92M" kernel boot parameter, it will crash as soon
as the machine starts to have its memory filled. Even with the "mem=92M"
parameter, I get strange errors, including processes stuck in the "D"
state, programs that end with sudden segmentation violations, and kernel
oopses like this:
