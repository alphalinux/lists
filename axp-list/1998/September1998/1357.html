> I have a bunch of PC164LX 2.0.35 machines. One, just one, has the
> following feature:
>
> % mkswap /dev/sda2
> VFS: brelse: Trying to free free buffer
> Unable to handle kernel paging request at virtual address
> 0000000000000029
> mkswap(330): Oops 0

I have seen this on a PC164UX, after using ARCINST from the
NT distribution to create the boot partition, as required
by that machine, on a disk which already had Linux installed.
I redefined the Linux partitions within ARCINST using the same
cylinder boundaries, but fdisk appeared later to disagree
about ARCINST's idea of doing things on the disk:

Partition 1 has different physical/logical beginnings (non-Linux?):
phys=(0, 1, 1) logical=(0, 0, 33)
Partition 1 has different physical/logical endings:
phys=(3, 63, 32) logical=(0, 132, 8)
Partition 1 does not end on cylinder boundary:
phys=(3, 63, 32) should be (3, 140, 62)

So I redefined all the Linux partitions again using fdisk,
again without of course changing the boundaries, and wrote out the
new partition table to disk. At this point fdisk was happy but I started
seeing the message that you report on a swap partition which was
previously OK.

I could solve the situation only be reinstalling RH from scratch,
and I learnt the partitioning lesson for the UX: first do boot
partition with ARCINST, then all the others with fdisk, never
use ARCINST a second time otherwise you lose the Linux partitions.
I do not know if this experience correlates with your case.
