I really hate replying on my own email but...

I sent a similar diagnosis to the egcs-bug list, someone did reply but my
procmail put it in a funny place. Following is the script I wrote up to
replace /lib/cpp, + a change advised by Manfred Hollstein
<manfred@s-direktnet.de> to solve the problem when the cpp is passed a
file with a non-program or non-header extension. This works for me (if
there's a more effective way, please enlighten me):

------- SNIP
# /lib/cpp replacement;
# <cpbotha@sun.ac.za>, <manfred@s-direktnet.de>
#!/bin/bash
# check if we have a filename on the commandline
for i in $*; do
if [ -f $i ]; then FNBOOL="true"; break; fi
done

# run gcc with parameters so it'll work on files
# with non-program extensions
if [ -z $FNBOOL ]; then
gcc -x c -E $* -
else
gcc -x c -E $*
fi
------- SNIP
