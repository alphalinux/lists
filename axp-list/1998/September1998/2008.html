I don't know if you found a solution to this yet, but I've been hacking on
the sound code lately, and it looks like it was written by a monkey. There
are lots of problems with it. (ok, so I hope the person that wrote it isn't
on this list! ;)

In particular, there is a definite memory leak in the mixer code (latest 2.1
kernels). You wouldn't happen to be using modules for your sound drivers,
would you? The leak shows up easiest if you insmod and rmmod the driver
several times...it'll crash your kernel eventually.

I think the code in the 2.1 kernels was written by the OSS/Linux people, and
probably has the same bugs. The drivers for OSS/Linux were back-ported to
the 2.0 kernels, (according to the 4front-tech.com pages) so are *probably*
the same...

I'll post patches to linux-kernel and here when (if...) I get it fixed.
