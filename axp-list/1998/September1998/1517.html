> > I'm having a _very_ hard time in setting up a Samsung 164UX

> Yup, have been there and have seen this enlightnening message :-)
> You used Linux tools to transfer the boot files into your

Yes! As I've read through the axp-list archive and found several
articles pointing into that direction, I tried different ways to

the MSDOS-FS at all, and when it can read the files, I get that
"Not enough memory" error...

> Your way out is now to recreate a new boot partition

Uahhh! I don't have NT, and I don't want to buy it just to
be able to create a Linux boot partition... :-((
There _must_ be another way...

> After you have done that, for God's sake leave the boot
> partition alone, and for no reason in the world mount
> it from Linux. Any file upgrade must be done via ARC.

Strange! Me thinks it's time to create a better BIOS version!
