I'm trying to compile apache 1.3.3 with PHP 3.0.5 as an apache module. My
box is an (onsale) AS200 with 64MB RAM and redhat 5.1. Per the PHP install
readme, here are the (summarized) steps:

unpack the apache and PHP source tarballs (I put them in /usr/src)
run the apache configure script: configure --prefix=/usr/bin
run the PHP configure script: configure --with-mysql=[src
path] --with-apache=[src path] --enable-track-vars
in the PHP directory, run make
in the PHP directpry, run make install
run the Apache make script with the --activate-module switch

When I run PHP make I get the error "[language-parser.tab.o] Error 1" I
searched for this error message and the closest thing I found was an error
about "language-parser.tab.c" and the fix was to use the latest version of
bison. I have the latest bison I could find - version 1.25. I don't get any
errors with the apache or PHP configure scripts it just bombs when I try to
make PHP.

PHP has binaries for Linux/i386 and DEC Unix so I know this should work.
