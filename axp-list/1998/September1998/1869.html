At 01:16 AM 10/20/98 -0600, Michal Jaegermann wrote:
>Right! In SRM which comes from VMS/Digital Unix world.

I think I need to switch to decaf. The root= flag was the problem (and HUGE
props to EVERYONE who was cool enough to offer additional advice which I
also used to good effect), I had come across a reference to it in the list
archives but I wasn't typing it within the "..." with the rest of the flags
so it was of course saying that it was invalid.
<cringe>
ANYhoo...got THAT taken care of, nice and predictable, danced thru the
install, got to fdisk, made four partitions and started the first (swap)
partition on cyl 3 to leave room to load aboot in front of the first
partition. Rest of install buzzed along just fine.

Then, according to all the people who were really cool and offered me help
(Jay and Michal were especially thorough in providing add'l documentation),
I need to reboot from the second partition on the hd, okay, fine,
everything is still fine, that works great, I boot on my kernel floppy,
point root at the hard drive, clackety clackety, login and am at the
command prompt. YAYYY! I have an OS!

But NOW I'm supposed to re-fdisk BSD-style, entering the same values for
start and end cyls as I did the first time thru. Then reboot again, and
install aboot onto that little piece of real estate in front of my first
partition. Then I can do 'set' at the SRM prompt in order to set my default
boot string and device and VOILA, I will be able to boot as normal. This
sounds okay to me.

Okay, no prob, get into fdisk, choose b option, supposedly I should delete
the existing partitions now (yes?) and re-enter new BSD diskimage with
starts and stops on identical cylinders to avoid trashing the existing FS
(no prob, I wrote them all down) until I get to the place where I have to
pick a file system type. Should I choose the hex val for BSD 4.2, 4.3,
leave it as undef'd, or just get a really really heavy hammer and bash my
skull in before I do something really stupid?

I left the original partition table as is until I can figure this one out.
Advice??

Also, while sitting around at the prompt (and getting ready to reboot) I
was doing a little checking around and had another disturbing problem.

Typing 'man [whatever]' at the prompt returns a 'segmentation error' if
[whatever] happens to be something that man pages *do* exist for. Simply
typing 'man' or 'man barfoobar' returns the usual 'hey, pick something I
have a man page for, ya big dork' prompt so it appears that the man program
itself is okay, but the man PAGES have a problem...

True? Fixable?
