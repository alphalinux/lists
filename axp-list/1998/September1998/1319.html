I have code that works fine on Intel Linux with the Blackdown JDK port, and
works fine on NT and Solaris with the "real" JDK, but on Alpha it doesn't work
quite right.
> java -version
java version "gat:05/12/98-02:14"

The object in question is a Runnable that the run method loops doing a
ServerSocket.accept(), and then spawns up subthreads to do something with the
incoming connection. I have an RMI method to allow a client program to
shutdown that connection by calling Thread.stop(). But if I try and start it
again (which creates a new instance of Thread with this object, and does a
Thread.start() on that Thread), I get a "java.net.BindException: Address
already in use". This tells me that in spite of the fact that the thread was
stopped, the ServerSocket.accept() is still out there tying up the socket. I
even tried calling Thread.join() after Thread.stop() to make sure that the
thread was really stopped. But no luck.

If you need help tracking this down, I could try writing a simple test
program, but I'm writing about this in case you know of the problem and have a
simple solution.

Other than this, the JDK version I'm using is pretty solid and seems very
compatible with the other JDKs I'm using. Even does Swing fine (after I
realized that the problems I was having with pop-ups was due to how sucky
fvwm2 was and switched to Window Maker)
