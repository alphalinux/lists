> I'm having a _very_ hard time in setting up a Samsung 164UX
> motherboard and I'm looking for help... :-((
>
> The problem in short: I can not boot the system from hard disk,
> because as soon as I create an environment variable set for
> Linux, the BIOS quits every attempt to run ldmilo.exe with
>
> "ERROR: Not enough memory to load program."

Yup, have been there and have seen this enlightnening message :-)
You used Linux tools to transfer the boot files into your
boot DOS partition , didn't you? This is a no-no on the UX.
Your way out is now to recreate a new boot partition
using ARCINST - from the NT distribution - and then copy
the three files from a floppy into it using ARC.
After you have done that, for God's sake leave the boot
partition alone, and for no reason in the world mount
it from Linux. Any file upgrade must be done via ARC.
See also http://archive.redhat.com/axp-list/1998-September/0630.html
You may have to redo the RH install following ARCINST's passage
on the disk.
