You have made a big mess from files from different times, versions,
and so on. It will be difficult to sort it out. It is much better to
clean it. In a directory in which you are compiling your new kernel do
the following:

- set your clock to a sane state
- save a copy of your .config file somewhere else
- run 'make mrproper' ("Mr. Proper" is a brand name of some cleaning
agent :-)
- run 'touch -R .'
- copy back a saved copy of of .config
- run 'make oldconfig'
- follow as usual

I do hope that you did not mix up Red Hat header files installed with
your system with a header files which came with your kernel sources.
If you did, then delete all that mess, use --force to rpm to reinstall
what you had installed previously under rpm control from kernel
sources/headers, change names/links not to clobber these again and
start with new kernel versions from scratch. 'mv' and 'ln -s' are
your friends.

>
> I know that my clock is set wrong, so I understand the "clock skew
detected"

As long as your clock is wrong you can expect all kinds of nasty
surprises. 'make' depends on modification times of things on which
it looks. In general you cannot expect to see a sane compilation
if your system is in shambles.

Michal
