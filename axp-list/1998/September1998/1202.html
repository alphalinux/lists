Okay, Guys, here's the beef:

I'm running my Multia diskless, so I don't really need any drives in
there. So I decided to try putting my semi-aged Millenium I in the little
puppy and try it out. I borrow the PCI riser card from a friend (mine was
one of those "crippled" ONSALE types) and I remove the drives and plug in
the Millenium.

It boots into SRM using the TGA board, however. I haven't figured out how
to make it use the Millenium, so I accept it and boot linux (via bootp)
and I do a "cat /proc/pci" probe of the pci bus. Millenium is there. Good.

I decide to install the XF86_SVGA server that supports the millenium. I
get all the updates from the errata set (XFree, XFree-libs, XFree-SVGA...)
and I configure it. Xconfigurator detects the card all right, and I set it
up for the resolution and all that.

Now, when I start X, it starts (I have to unplug the video from TGA and
into the Millenium) and I get the "meshy" gray background and the X
cursor, which turns in the real cursor. I move the mouse. So far, so good.
BUT when I open the menu (it's running TWM temporarily) with I think
button 1 or 3 (can't remember...the button that opens the menu) I get half
the menu, and the "place window" cursor before the machine completely
FREEZES.

So, can someone help me figure out how to:

1. Get SRM to use the Millenium (ARC locks up as soon as I type "arc" at
the SRM prompt, and besides, I need to netboot.)

2. Compile a Linux Kernel that successfully uses the VGA on the Millenium
instead of the TGA (I tried to configure alpha-sources-2.0.34 but I don't
see a mention of the VGA console unless I say yes to the TGA console
first. Arrgh. And I've been trying to get 2.1.123 to boot, using the
latest aboot, but again I get nowhere.)

3. Get X running? I'm really tired of pseudo-color pixmap switching and
the absolute slowless on the TGA server.

I'd _love_ to love the UDB as a nice small quiet X-Terminal. I'm almost
there!

Thanks to anyone in advance,
