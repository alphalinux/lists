And, similarly, I had a problem with a "genuine" Microsoft mouse on a Linux
box. The problem turned out to be it was a PlugNPlay mouse, which sends a
few bytes on startup to identify itself. That was on RedHat 4.2, with the
mouse plugged into com1. I solved it by getting a Logitech mouse :-(

--Ron Bruck

"James W. Simmons" wrote:

> I'm not sure this is the problem, but I had something similar happen on
> an Intel box running RedHat 5.0 with a Mouse Systems PS/2 mouse. It turns
> out some PS/2 mice need to be initialized. The code to do the
> initialization was off by default in the psaux driver. You could
> change one define and rebuild the kernel and the mouse would work fine.
>
> I decided it wasn't worth the trouble so I took the mouse back and got
> a Logitech one instead -- it apparently doesn't need to be initialized
> because it works with the default kernel.
>
> If I remember correctly, you edit /usr/src/linux/drivers/char/psaux.c
> and uncomment the "#define INITIALIZE_DEVICE" line, then rebuild the
> kernel.
>
> Hope this helps,
>
> Jim
