>>> Jason Orendorf said:
>
> First, there is an external modem connected to the box. If it's powered
> on when MILO starts I can't boot. It appears that everything I type
> is sent to the modem - like MILO sees something on the serial port and
> assumes my console must be there. Not a big deal, just a minor annoyance.
> "Console on serial port" is not built into my kernels but must be built
> into MILO? Any way to disable it via some environment setting?

Most MILOs have been built with serial echo enabled, which allows additional
debugging messages to appear on COM1; this sometimes confuses modems there...

Also, MILO looks to both keyboard and COM1 for command input, so anything a
modem might send would be looked at by MILO as a command.

> Second, the system can't boot if the kernel name is a symbolic link.

This, AFAIK, is a limitation in the implementation of the filesystem code
in MILO, which is *not* a copy of the kernel code, as other parts are.

> Anyone seen similar problems and have a solution? Considering these
> are my only gripes with Alpha/Linux I'm extremely happy with my Multia
> acting as a firewall/router for my small LAN.

Building MILO is not for the faint of heart, but such would allow you to
both turn off serial echoing, *and* have a hack on the FS code to add symbolic
link support... :-) :-)

Go for it!
