> This time milo had to read this 3.7G partition to load my kernel
> (root stays that same as I do not plan to rearrange the whole box,
> at least for now :-). The machine is happily up and running.
>
> The blanket statement in the post to which I originally responded
> was "the current milo does not handle partitions over 2 Gig".

OK, I should have said "does not handle more than the first 2 Gb of a
partition". Which is pretty close - on a big partition one can never be
sure where the files are.

> Maybe there are cases when such partions indeed cause trouble for
> milo. I still have to find one but I would be really interested
> to learn about precise conditions for milo hiccups.

I don't know what the precise condition is; at least when the inode
starts beyond the first 2 Gb, Milo should fail - see the `offset'
variable in ext2_iget() in file fs/ext2.c, which is used to calculate
the byte offset into partition and declared as int.
