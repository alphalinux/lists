>>> Marcelo Tosatti said:
> I successfully installed RH 5.1 on an AS800 5/333.
> But i'm having problems booting Linux.
> >From SRM i executed :
> boot dka100 -file 1/vmlinux.gz -flags "root=/dev/sda1"
> Got :
>
> CPU 0 booting
> (boot dka100.1.0.5.0 -file 1/vmlinux.gz -flags root=/dev/sda1)
> block 0 of dka100.1.0.5.0 is not a valid boot block
> bootstrap failure
>
> Using a bootdisk and "boot dva0 -file vmlinux.gz -fl "root=/dev/sda1""
> everything goes ok.

You must install a bootstrapper to the harddisk, as SRM cannot directly boot
a LINUX kernel (or a Digital UNIX kernel), as it knows nothing about file
systems.

You install the bootstrap using (make sure the "aboot" RPM is installed):

swriteboot /dev/sda /boot/bootlx

BUT!!!

You *MUST* have left space in front of the first partition on the harddisk,
into which the bootstrap can be installed (about 1024 sectors is plenty).

If you didn't do this, be aware that installing the bootstrap will probably
overwrite some of the filesystem in the first partition... :-\
