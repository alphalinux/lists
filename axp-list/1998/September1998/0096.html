Hello!
A few days ago I reported some strange hangs with lastest kernels like
2.1.119. In the meantime I am sure that hardware interrupts are getting
lost, never reach the kernel or the kernel just ignores them.
Now I checked kernel 2.0.33 with alpha-patches-2.0.32-0.1.gz and guess what?
The mysterious hangs are gone!
So the question is now: What has changed on interrupt handling for the SX164
from 2.0.33 (with aboive patch) and recent kernels?
If someone wants to share my exprience, just test this:
compile ans install ldescent, sync, sync, and start it!
It should hang your machine completely just after some seconds.
Also xquake show strange hangups but survves moist of them, ldescent is much
worse.

So please, if someone here on that list is familiar with the SX164 code
could you please give us your comment?

But besides this the memory corruption problem on SX164 reported by several
people still remains in 2.0.33 :(
CU
