On Oct 6, 4:58pm, Jacek Perry wrote:

> I have a curious problem.

It's not as curious as it sounds. The telnet and the ftp "server" deamons are
two seperate programs. So they can miss (partially).

> Any idea why that would be? BTW I CAN ftp from that node to all the
> other ones.

FTPing _from_ a host just means the host has a ftp-client installed. Look at
apropriate /etc/inetd.conf for a line that reads somewhat like this one:

ftp stream tcp nowait root /usr/etc/ftpd ftpd -l

this is the line where the ftpd (the server deamon) is started. If you do not
have line like this (do a "man ftpd") you need to add it. Otherwise you might
just want to check the path that the ftpd is supposed to be.
