>>>>> "BEWW" == Brian E W Wood <beww@intac.com> writes:

BEWW> Where do I get this libbfd?

libbfd is part of binutils. The error message indicates a failed
dependency. The version of ps installed on your system is linked to
the old libbfd. RPM is refusing to upgrade libbfd (and therefore, the
rest of binutils) because doing so may break ps (or any other package
that depends on the old libbfd).

I had no problems with ps (or any other dependent package) after
upgrading binutils, but YMMV. To force RPM to upgrade the system, use
the "--nodeps" flag. You might want to try to install the newer
binutils on top of the old version; you'll need to use "--force" to
accomplish this. All the other binutils programs should be
overwritten, but the version-specific library file will be left
untouched. This is kind of iffy, so you may want to take your chances
with a non-functional ps.
