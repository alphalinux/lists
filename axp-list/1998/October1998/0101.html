Hello,

I was able to solve the problem... by doing 3 things... I don't really
know which one did the job.... but it worked.

First, I noticed that the SCSI ID was the same on both channels... so I
went into the BIOS and set CH-A ID =7 and CH-B ID =6.... also, I turned
on the SCSI BIOS... it was disabled by default. And lastly, I didn't
have any devices on CH-B.. and was suspicious that perhaps termination
wasn't happening correctly.. so I changed host termination on CH-B to
Enable from Automatic.

Anyways,... after this, everything seems to work... except reboot. I
have my system automatically boot into RH 5.2/Alpha with no problems on
a cold boot (well, when it mounts the root disk, it does take a while...
I don't know why, but much slower than my Intel Linux boxes).
The problem is when I shutdown and go back to MILO... if I try to boot
from there... it gives me all sorts of errors... like sound not
initializing, no floppy controller, sometimes the SCSI bus doesn't reset
properly and hangs..... but if I reset the machine (by reset switch) ...
it's back and up with no problems at all.

Software wise, rxvt seems to crash a lot.... and asload (in AfterStep)
crashes with some sort of division by zero? But software problems are
fixable..... so I'm not too worried.

Well,... I hope this will help other people that had my problems... I
will probably write a step-by-step instruction doc for the procedure and
submit it to the alphalinux.org site.... as it took a lot of info
gathering for me to figure out how to get RH 5.2 going on Alpha... my
first Alpha!
