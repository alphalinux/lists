> You might want to ask them for load estimates (memory + cpu) expected
> for such usage. I think rsync can be a memory hog in certain
> situations (such as syncing a large number of files), which could
> cause alpha problems. I've also seen alpha problems when memory usage
> is heavy - programs running regularly that suck up most of memory, or
> programs that write tons & tons really fast, causing the write cache
> to fill up. My guess is that this causes some sort of memory
> fragmentation from which the system sometimes doesn't recover.

Also, I forgot to mention. I've been using rsync to sync up about 1GB
of data several times a day on a large number of linux/alpha boxes
(with 128mb ram), but with only ~10,000 files. I haven't yet noticed
any crashes associated specifically with rsync.
