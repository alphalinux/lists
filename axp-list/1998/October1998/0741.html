Hello;

On Sun, 29 Nov 1998, Michael Schmitz wrote:

> Question 1: Is this normal behavior for an LX164? Maybe the SRM
> isn't fully implemented or tested on LX164 and these problems can be
> ignored. Or maybe I need to replace hardwareg. Anyone know?

I can't comment on the SRM diagnostics, as I have no experience with
them. This reply is more or less just to say, "SRM/aboot on LX164
works for somebody".

We run RH 5.1 (2.0.35 + printk fix for klogd + tulip.c v0.89H) on our
LX164s which all use SRM. I've gotten some machine check failures
during reboot attempts (via "shutdown -r now", "reboot", neither of
which ever work), but that's about it. I *think* we are a bit behind
SRM V5.3-3, though I can't recall for certain. We are using
aboot-0.5-7.

> Question 2: Is there a known working SRM LX164 boot disk image out
> there anywhere? There doesn't appear to be one on gatekeeper.

The ramdisk.img? I *think* there used to be one there (at least
that's where I think I got everything), but I couldn't find it either.

The SRM console doc may be of use:

http://ftp.digital.com/pub/Digital/info/semiconductor/literature/srmcons.pdf

BTW, I wrote up a little doc on what it took to get our LX164s up this
past summer, just so I wouldn't forget. I used the NFS installation
method, and I recall the installation appeared to hang for a few
*minutes* on several packages and litter up the screen a bit. But it
eventually got through it. Anyway, this doc may or may not be useful
to you:

http://www.phys.ufl.edu/~prescott/lx164.html

Good luck,
Craig
