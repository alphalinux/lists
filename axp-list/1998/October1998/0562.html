I've come across these troubles:

gpm doesn't work: complains about unable to get a socket.
xfree doesn't work: same, it goes to the grey background, mouse does
operate, but wm doesn't activate. similar socket error.

I have the /dev/aux driver in the kernel set to yes.
cat /proc/interupts didn't show the PS/2 mouse at IRQ 12 or anywhere.

Also, the network driver doesn't work, I've even dropped in tulip.c
0.89h+Loic driver. It still didn't work. I'm surprised that the latest
experimental kernel is using tulip.c 0.79 or such. the compiler complains
that it uses the 'old' PCI interface. Is this a problem?

It's interesting that you STILL must comment out the SMP=1 line manually.

In contrast, I've come across no bugs or problems in the 2.0.36 kernel.

I am using a MIATA, egcs 1.0.3, RH5.2, gpm 1.14, XFree 3.3.2.

I haven't seriously used any other 2.1.x kernel, or successfully compiled
one either, until now, but I haven't tried for a while.
