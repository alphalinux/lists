>>> Adrian Gartland said:
>
> Hi...we've failed miserably at trying to get 2.1.1[24,25,27]
> to run on one of our UX machines. Managed to get everything
> compiled ok with GCC 2.7.2...but after installing the
> kernel and rooting we get about a page of #s instead of the
> usual 2 lines worth, and then says something like
> "this is not a gzip file" and dumps us back at the milo
> prompt. Is there something about upgrading to 2.1.x on the
> alpha which we don't know about?

This sounds like a size problem; the older UX MILOs were incapable of loading
a large kernel, which most 2.1.xx are, unless done with modules.

Easiest fix is to use the latest MILO and ldmilo.exe for UX/RUFFIAN, found
on gatekeeper.dec.com as:

   /pub/DEC/Linux-Alpha/Kernels/ldmilo-ruffian-981020
   /pub/DEC/Linux-Alpha/Kernels/milo-ruffian-981020

Rename to "ldmilo.exe" and "milo", as appropriate; ie not "milo.ful", as the
new "ldmilo" looks for "milo" now).

Using these, you no longer need an "osfpal.nh" file (it's built into the
MILO image now, just like all the other platforms), and it should work
better for rebooting into MILO instead of the console.

Good luck.
