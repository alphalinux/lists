The 21066 Alpha chip had a number of nice features, but speed was not chief
among them. The on chip local bus between the PCI, Memory and CPU core
chokes up at about 30MB/Sec and for the general case (its a mostly integer
world) you should expect to see performance comparable to a 90MHz Pentium.

Bob Segrest

Here, here... I read the other followup, and he's right, just because the
clock speed is faster doesn't mean necessarily overall throughput of same
tasks will be faster. I think someone suggested a 2:1 ratio for RISC:CISC
clock rates generating "equal" (loosely used, of course) machines.

My UDB/166 with RH5.1 and 64M takes about 45-60 minutes to build the 2.0.35
kernel with only the things I need compiled into the kernel and only the
devices the UDB has built as modules (SCSI, ethernet, etc.) I've been
trying
to compile KDE on this box for the last four days, and I'm still not done.
:)
(It's not really taking that long, but it builds a piece of it every
night--I spend about 2-4 hours a night working on it, and this is as far
as I've got.)

But the Multia is faster than the 486-100 it replaced as a fileserver for
a bunch of old 2G Fast-Narrow SCSI devices, a 4x CDROM, a 2G 4mm DAT, and
my ZIP drive. Imagine this: I (for the first time in my nerdly life)
have a SCSI bus with something at every ID--and it's all external!

The Multia is great for low-volume file-server needs here at home. As soon
as I get a window-manager configuration that doesn't suck and can run under
8bpp, I may even use it as an X-terminal.

Brendan Miller
