On Thu, 19 Nov 1998, Michael Tibor wrote:

> I'm trying to configure my as200 with a dhcp client on an interface,
> but I get what looks like a kernel error when I try it on 2.1.128
> (also 2.1.126) using dhcpcd-0.70 from RH5.2.
>
> However, when I boot into 2.0.35 dhcpcd is able to talk to the dhcp
> server and setup the interface with the correct info.
>
> Is this a know issue with 2.1.x kernels, or maybe did I miss something
> when I did make config?

A very big thanks to Michael Larocque for pointing me in the right
direction on this one. The problem is that dhcpcd-0.70 does not work
with 2.1 kernels, and that the one to use is dhcpcd-1.3.x from:

ftp://phystech.dyn.ml.org/pub/

The latest at this time is 1.3.16.

Unfortunately, when I use this with 2.1.129 (and also 2.1.128) I get
the error "can't locate module net-pf-17" which appears to be
AF_SOCKET or something like that.

Does anyone have any idea how I might fix that?

Thanks
Mike
