On Sun, 22 Nov 1998, Rich Payne wrote:

> Anybody try 2.1.129 on a Multia yet? If I compile w/o sound
> it works fine. With sound it hangs on boot right after displaying:
>
> <MSS audio codec (CS4231)> at 0x534 irq 9 dma 3,3
>
Can't help you with the Multia, but on my AS200, /etc/soundconf looks like
this:
#define CONFIG_ADLIB
#define CONFIG_MSS
#define CONFIG_AUDIO
#define CONFIG_YM3812
#define CONFIG_AD1848
#define CONFIG_SEQUENCER
#define MSS_BASE 0x530
#define MSS_IRQ 9
#define MSS_DMA 0
#define DSP_BUFFSIZE 65536
#define SELECTED_SOUND_OPTIONS 0x00880204

I'm never prompted for a second DMA channel. It compiles fine whether or
not I add it by hand. Why are both of yours set to 3? Obviously the
problem is with the sound setup, so I would start there. Set it up to use
only one DMA, make it 0, unless some oddball requirement of the Multia
sound wants it at 3. That is a most unusual number.
