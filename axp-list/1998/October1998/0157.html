>
> > The problem is that I keep running into runtime arithmetic exceptions
> > (SIGFPE). Using -mieee got rid of a number of these in the actual
> > program, but for some reason I haven't been able to fix a problem that
> > seems to be occuring in the egcs' libI77-sources.

> This looks like libI77 was not itself compiled with -mieee. Don't the
> egcs build scripts default to building libI77 with -mieee?
>

Would make sense to have the libraries built both ways.
That way people who don't want to fix the bugs in their
code can pay the penalty for using -mieee and those willing
to get their code working correctly get the best performance
available by using a non -mieee version of the library.
