With regards to your message at 12:48 PM 11-19-98 -0500, Greg Lindahl.
Where you stated:
>> Ok what is the final word on swap. Is it maximum of 128m as RedHat
>> book suggests or more as I heard some people say. Do I really have to
>> make 8 swap partitions in order to have 1gb of swap space? If I don't
>> then how do I mount the 1gb partition to be a swap only partition.
>
>It depends on page size. 128M for i386, 512M for alpha. If you tested
>you'd quickly find this out.
>
>If you want more, you can make an ext2fs partition and create several
>swap files on it. That avoids eating up gobs of partitions.
>
Yes, but partitions dedicated for swap usually run faster. With an extended
partition and many logicals you pay no real penalty for this, other than a
rather large and confusing fdisk table..
Besides just how much swap can you really use?? If you are using a MB of
swap, something is "fishy"..
