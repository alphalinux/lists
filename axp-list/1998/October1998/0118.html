> I am trying to build sendmail 8.9.1 with Berkeley db 2.5.9 (from
> SleepyCat) on a stock RH 5.2 system (2.0.35, glibc 2.0.7-29,
> egcs 1.0.3).
>
> It apparently works, but I get messages
> kernel: sendmail: unimplemented OSF mmap flags 0201
> in /var/log/messages, apparently every time sendmail starts.
> This does not happen when I use the standard libdb included
> with glibc.

        glibc's locale stuff uses MMAP_INHERIT flag, and
        the kernels *warn* about it. That happens *only*
        when you have LC_CTYPE set to something.

        Comment the printk() away, and be happy.
        (It is already commented away in the lattest 2.1.* kernels.)

        The use of that flag will go off once the glibc is compiled
        against sufficiently new kernel header files, but 2.0.*
        glibc SRPMs use a bit oldish headers with that flag in.

....
> If all this is true, include/asm-alpha/mman.h should be
> modified in the next kernel distributions, removing these
> three definitions.

        They were modified about a year ago.
