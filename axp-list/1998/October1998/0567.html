Jaap Schellekens wrote:
>
> Dear all,
>
> Back in the old days (on my 25Mhz486) I used the -gp option with gcc
> to get profiling info for gprof. For me this worked great. The newer
> version of gcc end egcs for the alpha do not support this. Yes i know
> you should use -fprofile-arcs -ftest-coverage and run gcov.

1. You must use static linkage (`-static') for profiling.
2. The profiling option is `-pg' (not `-gp').
3. For Redhat 4.x, you need two patches (profiling in 5.1 works
   out of the box):

http://www.redhat.com/support/mailing-lists/archives/axp-list/1997-March/0463.html
http://www.redhat.com/support/mailing-lists/archives/axp-list/1997-March/0464.html

> However, this produces a lot of files but not very usefull information
> (for me at least). Does anyone out there knows if egcs can be build to
> support -gp? (other profiling suggestions are welcome as well)
