>
> > Tonight I had a horrible machine crash during a local disk
> > backup using rsync.
>
> The fact that the system crashed while rsync was running (& presumably
> writing substantial system changes) would account for the massive
> filesystem corruption.
>
> The fact that the system crashed at all is probably a kernel bug or a
> hardware problem. There are things which can crash the system (such
> as using up all of memory) which maybe should be considered a lack of
> robustness and not a kernel bug. But in your case, if rsync had a bug
> & ran amok, sucking up all of memory, then this probably would have
> pushed the disk cache onto the disk, thus minimizing file system
> corruption. On the other hand, if rsync was running as root (which I
> assume you were probably doing), it could have started writing all
> over the disk, causing the crash.
>
> So, I'd give you a definite maybe on the rsync bug... :).
