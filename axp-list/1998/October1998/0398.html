----- Forwarded message from Kevin.Brown@asu.edu -----

>PIIs have 512K L2 cache and 32K L1 cache so they will perform even better than
>an overclocked celeron.

Ummm depends on what your doing.

For jobs that mostly access L1 (32k) they will be identical (per Mhz).
For bigger jobs that mostly fit in 128k the celeron will be faster (per Mhz).
For bigger jobs that mostly fit in 512k the PII will be faster (per Mhz).
For bigger mostly memory bound jobs the celeron will be faster (since cache
        misses are faster, and latency to main memory is lower).

It's very hard to know where a particular application will fall without
actually benchmarking it. Benchmarking YOUR application is always highly
apprehended.

As an example a celeron 300A at 450 compiles ssh-1.2.26 is 49 seconds,
a real PII-450 does similiar in 55 seconds. My conclusion is that it's
possible that gcc is probably rather friendly to the celeron's 128k cache
but doesn't have a high hit rate for the L1 cache. I could be wrong,
but I was quite pleased it was close regardless of the actual reason.

A similiar case happened with the alphas, the older alphas (before lx/miata)
with the up to 8 way interleaved, 512 byte wide FPM memory, and large L3
caches were almost always slower then the newer alphas with smaller
caches, 128 bit wide sdram memory. Latency and bandwidth of each
layer of the memory system can have rather complicated interactions with
an application.
