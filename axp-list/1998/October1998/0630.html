Sabrina Minshall wrote:
>
> Hi,
> I recently purchased an AS200 4/166 box from onsale. It comes with the
> standard ARC Console (Alpha Firmware Version 4.45). I've RH5.1.
>
> Here's what I did.
>
> Create a FAT floopy, and copied loadlin.exe to it.

MILO should also be "copied" to this floppy. Do not rawrite either file.

You should also have a boot and ramdisk for your particular computer.
These will be created with rawrite.

> Created a new boot seletion for linux as exaplined in RH5.1.
> When I tell it to boot Linux, I get
> Error: Bad image file type, press any key to continue.

Again, this is probaly due to the creation of the MILO/linload.exe
floppy.
>
> Also, when I boot the machine, it complains,
>
> The firware has found that some information in the machine NVRAM is inconsistant.

If your configuration in the boot menu is looking for MILO and
linload.exe on a floppy this error about NVRAM will be returned. When I
boot I must have the floppy in the drive or it will return the error.

As for my Cabriolet, once I copied MILO and linload.exe to a floppy,
created the boot and ramdisk with rawrite, I found the addendum to be
fairly accurate.

hope this helps
