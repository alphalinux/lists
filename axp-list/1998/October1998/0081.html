On Sat, 14 Nov 1998, Michael wrote:

>
> As some of you know the install program won't read the correct date
> from some Alpha's like my XLT-300. If left unchanged, all files are
> written with a year of 2018. The last time I upgraded (reinstalled) my
> XLT-300 I went in to the shell prompt before the install program started
> copying files and fixed the system date to the current date. This time I
> did a complete reinstall that went very well but I forgot to change the
> date. Does anybody know a easy way to do a global file & directory date
> change from 2018 to 1998. Only the year is affected, the month & date is
> ok.

Interesting - I have some files in a similar state; how about:
 1) Creating a file circa 2017 (either by winding the clock forward or
    using touch with the -t option) (lets call it f2017)

 2) Then find / -newer f2017 -exec touch {} \;

If you wanted to not change anmything but the year you could do something
cleverer than touch just to frob the year.
