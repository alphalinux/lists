> jmd> Any ideas?
>
> It's `only' a performance problem (and disk space, if your logs get
> too big...). Values need to be aligned on certain boundaries for the
> Alpha CPU. If they're not aligned, it generates a trap, reorders
> them, and puts them into the CPU (ie it works around the misalignment
> automatically). To fix it you'll need to get hold of the source code
> for whatever's causing the traps and pad the guilty structures to line
> their members up on the appropriate boundaries.
