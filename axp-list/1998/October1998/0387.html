At 04:31 PM 11/19/98 -0700, you wrote:
<snip>
>I have tried it.
>On Red Hat 5.0 machiine, with 64MB of RAM and 5 32MB swap partitions. Ran a
>bunch of apps and ate up memory.
>Turned on the swap partitions, one at a time, and tried again.
>Then tried it with swap files. The files were giving noticably more
>sluggish response.
>Never did it as a formal benchmark, but you could see the difference..

Interesting.

>I think you are right though, _somebody_ should devise and run some formal
>benchmarks, just to quantify the difference. Any volunteers?

I would suggest that what would be important would be the magnitude of the
difference between the swap files and swap partitions, not the actual
numbers... To that end, it *may* be a valid test to simply create a file
access test that reads and writes to both (swap) partitions and (swap) files,
using a repeatable set of operations, and then comparing the times (actual,
system, other?) to see the difference...

Just a thought...
