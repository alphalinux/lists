On Wed, 18 Nov 1998, Karsten Ballueder wrote:

> Trying to install Linux on our "Digital AlphaStation 250 4/266"
> failed, because I couldn't get it to boot from the install disk.
> I used the "avanti" type boot disk, but never got any boot prompt, not
> even when holding down F2 at boot time.
>
> Instead, it would always boot straight into OSF from the harddisk...
>
> Could anyone enlighten me how to boot from the disk?

I believe the only Alpha's that can't boot from floppy via SRM console are
the Jensen line, so you should be okay. When you get to the >>> prompt,
did you run:

b dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1"

If your SRM is set to autoboot, you would need to interrupt the process
quickly. I think CTRL-C will work.
