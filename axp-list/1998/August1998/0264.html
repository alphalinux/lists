> Did you compile it with -mieee flag? I would expect Mozilla (and Java)
> to do all kind of dirty floating point tricks - like taking square
> roots of really small numbers if not simply dividing by zero.
> These are deadly on Alpha if a generated code is not prepared for them.

Uncle George posted a fix for some FP exceptions in jsapi.c,
jsnum.c, and jsnum.h a while back. Check the archives. See
if your FP exception is in those routines. If it is, apply
the patches. I have the patch if you can't find it...

I still find it hard to believe anyone truly intended to do
something that would cause a FP exception. They ruin your
computations usually.
