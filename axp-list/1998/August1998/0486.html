Installing from RedHat 5.1 that was downloaded from Internet.
Used ARC to run minlabel and built partition on spare drive. Built an
aboot+kernel image on a diskette, using a DOS machine. Used copyboot to copy
this into first 4M partition on spare drive. Switch to SRM, power-restart
Alpha, booted into aboot using "boot -fl "i" dka200". aboot loads. Do a
"0/a root=/dev/fd0 load_ramdisk=1", and stick my ramdisk diskette in drive.
Kernel loads, RedHat installation begins. RedHat ask for stuff, then after
the "Install" or "Upgrade" prompt, ask for install source of CD, FTP, NFS,
or
Disk. I select CD, but it doesn't think the CD that I made (from the RedHat
ftp site) is a valid RedHat CD, so I select FTP install. It then tells me
that the partition on the spare drive is corrupt and will have to be
rebuilt.
If I choose "skip" then that drive is no longer an option, and it goes on
to
the panel, where you choose the root device, and the spare drive is not
shown. If I choose partition the drive, then I am not given an opportunity
to partition the drive; it simply goes on to the panel, where you choose the
root device. The spare drive is shown, but cannot be chosen.Help.
1) I built a CD from the ftp site, so why doesn't RedHat think that it is a
RedHat CD?
2) More importantly, because I can go around the CD using ftp, what could be
wrong with the partition that minlabel built?Thanks.
