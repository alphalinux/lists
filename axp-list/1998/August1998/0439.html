A short description:
We want to share memory between two processes. Typically,
a process waits for info from a socket and prepare data for work.
The other actually process the data. To be able to share the
data ( two areas, one being prepared, one being processed ), we
use pthread or clone.

When we run ~10 instances of the program ( i.e ~ 20 processes ),
we get some faults ( SEGV or Illegal Instruction ) in unrelated
process ( as update for example with a lock of the computer ).

If we replace the use of clone with a fork and share the memory
with a mmap file, everything runs smoothly.

The same process with clone runs correctly on Intel platform ( or
with pthread on SGI and Solaris ).

Has anyone else experienced the same problem ?

I am preparing a small program that will (hopefully) exhibit
this behavior.

Some info on the system:
lulli 1- uname -a
Linux lulli 2.0.33 #3 Thu Mar 19 20:13:24 CET 1998 alpha unknown
( we also tried 2.0.35 from redhat )
/lib/libc-2.0.7.so
gcc version 2.7.2.3
All tests run under a standard user.

Thanks for any hint.
