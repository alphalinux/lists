Not sure of this, but are you sure you should compile in LPT support? I
remember if you're going to use an IOMEGA zip, you specifically DO NOT
compile in LPT support (or compile it as a module and load as needed). If
the qcread software NEEDS LPT support, fine. You might also check to see
whether you have a printer defined that uses the LPT port.

Just as an aside, I've been using Webcam32 to automagically ftp to my UDB
pictures linked be a web page on the alpha. If I could figure out how to
set thresholds for pictures on Webcam32, it would be a poor-mans security
system for my house (since I now have cable modem...:)

Personally, I keep hoping RealNetworks will offer their Linux server/client
for Linux/axp, but that's another story....
