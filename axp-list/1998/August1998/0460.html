On Mon, Aug 10, 1998 at 11:54:32AM -0400, Eric Ding wrote:
> Is this patch broken? It appears that h->root.plt.offset ought to instead
> by h->root.plt_offset. I'm patching against binutils-2.9.1.0.4.

Ah yes. The problem being that the patch was not generated against
binutils 2.9.1, but rather current sources.

There is another problem as well that affects static linking. Use
the following definition instead of the one in the patch.

#define alpha_elf_dynamic_symbol_p(h, info) \
((((info)->shared && !(info)->symbolic) \
|| (((h)->elf_link_hash_flags \
& (ELF_LINK_HASH_DEF_DYNAMIC | ELF_LINK_HASH_REF_REGULAR)) \
== (ELF_LINK_HASH_DEF_DYNAMIC | ELF_LINK_HASH_REF_REGULAR)) \
|| (h)->root.type == bfd_link_hash_undefweak \
|| (h)->root.type == bfd_link_hash_defweak) \
&& (h)->dynindx != -1)

r~
