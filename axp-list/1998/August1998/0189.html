On Mon, 3 Aug 1998, Wes Bauske wrote:

> Christopher Curtis wrote:

> > But the manual is also very Solaris specific, and Solaris does not do the
> > idiot remapping Linux does.
>
> If you use multiple SCSI cards, you also specify
> which card, 0,2,0. Since you can query the bus to
> find out what each device is, I see no problem with
> the way cdrecord under Linux numbers things.

Neither do I. Having significant Solaris experience, that it how I would
expect it. The problem is that *Linux* does idiot SCSi remapping. For
example, right now I have three SCSI devices. A hard drive, tape backup,
and CD burner. The hard drive came with SCSI ID 6. Linux calls this
/dev/sda. When I put the CD-ROM (ID 4) into the chain, the hard drive now
becomes /dev/sdb. Duh? Solaris does it right. Linux does not.

> Seems to me the man page is quite clear how to use
> it under Linux. I've used it about a year now...

I was obviously not expecting any Linux tool to use raw IDs. Solaris uses
them and the man page is very Solaris-ish. I simply expected Linux tools
to act like Linux itself uses devices, not like Solaris does.

> limit. I don't off hand see why cdrecord would need
> such large buffers for feeding a 2-4x CD recorder.

Under load I was able to exhaust a 2MB cache. I'm still using it in
'dummy' mode so I don't know if this would have mangled the CD but it did
empty. With a high-speed tape and a few hard drives on the same chain I
don't see why it wouldn't make sense, especially when blank CDs were
costing ten dollars apiece.
