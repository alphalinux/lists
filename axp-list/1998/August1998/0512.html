Kenneth J. Baker wrote:
>Is there a chance someone would be willing to summarize installing KDE
1.0
>on an Alpha with Redhat 5.1 installed on it?

From what I've seen on the list all i can say is ymmv, but i'll give
you my own
experiences:

Box first: lx164 w/600MHz 21164 (digital board, samsung chip. don't ask
why)
128Mb RAM, qlogic scsi & Matrox MII card. When I put kde on it was a
bog standard
RH5.1 install (2.0.34 kernel). I've since upgraded the kernel to
2.0.35.

I pulled the kde & qt sources down and they compiled quite happily: kde
does throw
up quite a lot of warnings (compiler btw is egcs 1.0.2), most of which
are about
redefinitions and others that i couldn't fathom before they ran off the
top of the screen!
Following the installation instructions pretty much to the letter (from
kde-1.0.install.txt),
with the caveat of the fact that the target was /usr/local/kde rather
than /opt/kde everything
went fine. Rejigged the config of /etc/X11/xinitrc (pretty much straight
from the kde
documentation) and /etc/inittab (to set initdefault to runlevel 5 for
gui logon) and rebooted.
Voila - nice desktop appeared. Memory usage idle is about 30Mb, which is
if anything less
than what i was getting with the stock X precompiled install and
everything seems to be
fine.

I'd seriously recommend getting the source and compiling yourself
rather than using rpm's
'cos i keep finding instablilities in generically coded stuff for
alphas. or maybe it's just 'cos
i like using those cpu cycles....

cheers,
john
