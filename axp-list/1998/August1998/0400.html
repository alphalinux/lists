On Wed, Aug 05, 1998 at 01:46:33PM -0400, Stephen Weiss wrote:
> When I try to load a raw image (I copied vmlinux.nh to
> -, and booted that), it boots fine (after some time (at least 30
> seconds) ), and promptly halts CPU 0 with the following messages:
>
> halted CPU 0
>
> halt code = 5
> HALT instruction executed
> PC = fffffc0000310000
> boot failure
>
> What I'm thinking is, the kernel is panicing when it can't find a
> console.

Nope. What this means is that your raw image did not boot fine.
The indicated PC is the entry point of the kernel -- so the first
instruction it executed was halt, which happens to have opcode 0.
