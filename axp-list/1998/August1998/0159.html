Christopher Curtis writes:
>
> My syntax may be
> off with regards to the "dev" line, but I'm using:
>
> cdrecord -v -V -dummy -speed=4 dev=11,0 iso9660-rr.raw
>
> with version 1.6 final. This is what I'm getting for output:
>
> Cdrecord release 1.6 Copyright (C) 1995-1998 Jvrg Schilling
> TOC Type: 1 = CD-ROM
> cdrecord: Invalid argument. shmget failed

Reduce a bit how much of a shared memory is required. From memory this is
'fs' parameter but check a manual page. 'fs=4000k' should be good enough
and a default is 4M. There is also an environment variable which modifies
this default and which can be used to make xcdroast running. This is also
described on a a manual page (and reading it is a recommended activity).

This is the simplest workaround. Other require hacking kernel to
increase a value of a shared memory segment.
