Replying to myself again -- I got the system fixed today. Thanks to everybody
who gave me help during desperate times! I owe you.

Here's a summary of things, in case someone else has the same thing happen.

It turns out that the RedHat installer's "initialize" (probably actually fdisk,
but I don't know for sure) only screwed up the bootloader, and not the
partition table. On booting the machine with the OSF/1 cd, I found I could
still read the disklabel on the afflicted drive. I did a

# disklabel rz0 -r > /label

to write out the label info to a file. Then I could use

# disklabel -R -r -t ufs rz0 label /mdec/rzboot /mdec/bootrz

to write the label back to the disk, along with the bootloader. On rebooting,
I found that the machine came up normally, and everything was there.

I now know what the word "relief" means.

Thanks to everyone!
-Paul
