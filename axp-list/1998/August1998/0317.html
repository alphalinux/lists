Hello

>> Actually you can not make it headless that I am aware of. Simply
>> because you need milo and other disks to run first. I guess you could
>> use debug mode on some Alpha's and boot it up as if SRM console.
>
>Both SRM and MILO support a serial console on my UDB.

This is true. The older Alpha systems running ARC console can run headless.
What happens is when you don't have a keyboard installed it defaulted I/O to
com1. From there it could be booted.

SRM can do it on SCSI drives only, on older systems. I believe on the newer
LX and SX with SRM installed "should" be able to boot an IDE device.
Loading it however, might be a task requiring a video card first to build
the drive.
