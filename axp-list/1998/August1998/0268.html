On Thu, 6 Aug 1998, Jordan wrote:

> Hi Guys,

Hello. 8)

> I have been told be numerous people that I should use BSD disk labeling to
> enable me to write aboot to the hard disk's boot sector.

If you are using SRM and want to boot off a fixed-disk, you must use
disklabels. Otherwise it's an option you probably don't want to bother
with.

> This all sounds very nice, but I have now idea about BSD disk labeling, and
> looking at fdisk's BSD disk labeling mode confused me even more.

I don't know the details of how the disklabel works (such as where on the
drive it's stored), but I can tell you how to manipulate it. It's pretty
straight-forward. Use the "b" option in fdisk. If you're trying to
install RedHat-5.x, do the install using a regular PC partition table, then
(after the install) create a disklabel using the exact same offsets. The
PC partition table will be obliterated when you install aboot, and you will
never need it again.

For the install I did, here is an example of what my disklabel looks like:

BSD disklabel command (m for help): p

4 partitions:
# size offset fstype [fsize bsize cpg]
a: 131936 8246 swap # (Cyl. 2 - 17)
b: 1030750 140182 ext2 # (Cyl. 18 - 142)
c: 4510562 1170932 ext2 # (Cyl. 143 - 689)
d: 2704688 5681494 ext2 # (Cyl. 690 - 1017)

> should 2 a two meg partition at the start of a disk be enough for an aboot
> boot sector ??

Yes. I believe the smallest amount of space you can reserve is one logical
cylinder (somebody smack me if I'm wrong), and that will vary depending on
the size of your drive. On my Microp. 4.2G, 1 cylinder was 8MB.

> I have also heard a few things about kernel, for example that there are
> generic alpha kernels, and one that don't need to be patched etc. etc. What
> is the best approach when it comes to kernels for an AlphaStation 255 ??

I think Richard Henderson (smack me again?) has been doing some work on
single-system kernels, but I believe that's 2.1 only. In any case, you
should probably start with 2.0.35 and add the alpha patches from
ftp://gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels or from
ftp://ftp.linux.org.uk/pub/linux/alan/DigitalAlpha.

Hope this helps,
