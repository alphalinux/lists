As it was said already few times on this list 'sqrt()' function in
a basic compiler library was replaced by significantly slower one
but one which happens to be correct. The old one had bugs. Until
a faster replacement will show up in a compiler libraries your best
bet is to link with new floating point libraries mentioned a few
postings ago. Such action will also significantly speed up other
elementary functions.
