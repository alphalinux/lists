Larry Creech wrote:
> If I can get the source, I'll try to hack one out myself.. I have a
> different card running, it just seems the 4D40T is too nice to waste, 16meg
> video. Now if if can get the source to XF86_SVGA, I think that the one
> for the Cirrus logic chip. I know the chip is supported in the docs, it just
> returns unknown values when probed. Anyone have any links on where the
> source is?

XFree86 source is available as:

ftp.xfree86.org:/pub/XFree86/3.3.2/source/X332servonly.tgz

Supporting the Cirrus chip that's sitting on the edge of the card
for VGA compatiblity won't get you very far, as I believe it can
only access a small chunk of memory (256k?) dedicated to that
chip. Getting the Cirrus driver working under Linux/axp won't be
trivial, as a fair portion of that driver is written in ix86
assembly.

Supporting the TGA2 would be a useful effort, but the needed
documentation is not available.
