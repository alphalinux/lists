shell. I recently built bash 2.latest and had to change some
default setting that had to do with this. ( It is documented
in the bash README.) If I understand correctly, the `full-featured'
shells like bash aren't appropriate for execution SUID and that
more, uh, rigorous shells like /bin/sh or better yet, perl are more
suited.

If you are going to wrap it and run it suid, you are probably better
off doing it in perl.
