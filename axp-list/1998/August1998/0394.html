Hi,

I was wondering, if someone could tell me, if the "ipfwadm"
util is broken on RH 5.1 systems. I seem to have problems
setting my masquerading settings.

# ipfwadm -F -p deny

So far so good, but...

# ipfwadm -F -a m -S 192.168.1.0/24 -D 0.0.0.0/0
ipfwadm: setsockopt failed: Invalid argument

...doesn't work.
I solved this bug for my system, by using the "ipfwadm"
util from the RH 5.0 distribution. After that, things
work like a charm.

Has anyone else encoutered this problem?
