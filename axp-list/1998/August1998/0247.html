Christopher Curtis wrote:
>
> I guess so, but I was making the case that this could also be done
> (perhaps more properly?) using threads now.

Could be, since threads can share memory access.
Just depends on the problem your trying to
solve I guess. Threads should have similar access
speed.

>
> > because a pipe copies to kernel land and back and
> > SHM is directly in your memory space so there's
> > only one copy to it and the next process can get it.
>
> Shared memory is typically a file, is it not?
>

Can be a file but its not necessary. All the stuff I've
done it's just a temp buffer space for passing data
between processes. It's the high speed local path for
data I usually pass thru sockets. It bypasses the
overhead of the socket calls which perform similar
to pipes. When your contract says you must drive a
device at least 90% of it's native rate across a socket
or locally, you do whatever you can to cut overhead if
you want the contract and to get paid...
