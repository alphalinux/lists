On Mon, 3 Aug 1998 20:05:24 brinkmann@nmsu.edu <brinkmann@nmsu.edu> wrote:
>I've been experimenting with the "poll" system call using the appended
>test program. It works well on our Solaris machines, but fails with an
>"Operation not permitted" error on my Alpha running RH Linux 5.0, both
>using stdin and file pointers. The RH errata makes no mention of any
>problems. Can anyone be of assistance?

You need the latest glibc package from the 5.1 errata, and get the
kernel-source package from the 5.1 errata as well. Rebuild your kernel,
reboot, and you should be fine.

You need a new kernel because all 2.0.x kernels on the alpha return the
wrong error code for an unimplemented system call. glibc needs to see
ENOSYS to do proper emulation, so the kernel must be fixed. You may not
actually need the new glibc for poll() to work, but hey, why not?

These fixes let poll() work fine for xntpd, fwiw.
