> However, it's my understanding that glibc-2.0.x contains some
> assembly code - wouldn't that give a problem compiling it with the
> DEC C compiler ?

Assembly-language source files shouldn't be a problem; gas uses
the same syntax as the DEC assembler.

The main problem is with "asm" directives; these are completely
different (and largely undocumented and unsupported I believe) in
DEC C.

Of course, you can always turn the asm directive into an assembly-language
subroutine in a .s file, at some performance cost.

How common are asm directives in glibc anyway?
