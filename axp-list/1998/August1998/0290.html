>>> Tim Rowley said:
>
> If you don't have the patch mentioned below and want to run on
> the -E2, you'll have to enable advanced fbcon options and turn on
> CONFIG_FBCON_CFB32.

Yes, that works for now, but...

IMHO, I think it better that selecting TGA support either:

1. includes 24-plane as well as 8-plane by default, or
2. asks specifically which you want (and allows both).

Either of these require changes only to drivers/video/Config.in.

I'm currently leaning toward #1; any discussion?

> Actually the 2.1.xxx fbcon TGA driver worked fine on the -E2 as
> written to Geert. I submitted a small patch that fixed some
> quirks (available in the list archive) that has been added to the
> vger tree. I don't believe they've been merged with official
> tree yet.

Uh, do those patches make TUX show up in color on the 24-plane? :-) :-)

Simply change the "visual" type of the 24-plane to DIRECTCOLOR instead
of TRUECOLOR in drivers/video/tgafb.c (the code works, but the terminology
may not be correct).
