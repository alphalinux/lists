I've been using an old version of RH (3.0.3) on my DEC Alpha UDB plus ssh
1.2.17 to access systems at work and have not had any problems for quite
some time. I've used both 28.8 PPP and ISDN router access.

I recently installed RH 5.1 and then recompiled ssh 1.2.17. I notice that
when I use ssh to access a remote system the first several characters are
transmitted okay, but then transmission of them slows down until it
appears locked up. By checking the lights on my ethernet hub, I can tell
that my system in not transmitting as opposed to a problem with the remote
system not responding. When I type short commands that result in
responses from the remote system, it appears fine.

I realize that the problem is a bit strange and I hope that my explanation
is okay. Has anyone else experienced similar problems? Is anyone using
RH 5.1 + ssh 1.2.17 on an Alpha UDB?
