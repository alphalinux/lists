Larry Creech wrote:
>
> I've been trying to compile the 2.1.1xx kernels. I have no problem on the
> Intel box, but trying to compile the same video for linux driver for the
> Alpha I get these errors:
>
> drivers/char/char.a(radio-aztech.o): In function `az_setfreq':
> radio-aztech.c(.text+0x2f0): undefined reference to `__floatdidf'
> radio-aztech.c(.text+0x2f4): undefined reference to `__floatdidf'
....

It looks like that this radio-aztech driver attempts to perform
some floating poing operations when in kernel mode. This is
a big NO-NO as a status of floating point registers is undefined
when in kernel and this is a bug in a driver. If you really
need this driver then you have to fix it (look for numeric constants
with '.' in them - most likely).
