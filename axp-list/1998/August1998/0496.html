> When I do get to the WinNT setup page it does tell me that it has found
> no devices on my system and asks for a manufacture's cdrom. I will try
> that and see where we go. This is NT 4.0 and the card is a new Diamond
> Fireport that may not be supported with this version of NT.

You will get this screen if NT does not find any hard drives. It is asking
for a device driver diskette to load a disk driver so it can continue. If
your hard drives are on the IDE bus, you need to feed it the firmware
diskette. It has a IDE driver (ATAPI.SYS) on it. If the drives are SCSI,
then it looks like you still have a scsi setup problem. The ARC (and NT)
should be seeing your Fireport card as a NCR or Symbios 53Cxxx chipset.
Does the ARC indicate it loaded a driver at the "Initializing Device
Drivers" message? Does the "Display Hardware Config" show any SCSI drives?
