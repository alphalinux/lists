To use XF86Setup you need VGA_16 server - which happens not to exist
on Alpha. This is not a show-stopper yet. One can cheat by
linking this missing server to Mono server which exists (you do not
want to "install" it in rpm sense but only extract a binary using
rpm2cpio and cpio) and you can "--force" XF86Setup when it whines
about dependencies. You will get a black-and-white picture, instead
of a colour one with few colours, but this is minor.
XF86Setup is not in an official distribution but can be found in
contribs.

The bigger problem was that on Alpha Tcl/Tk, used by XF86Setup, used
to screwup badly, especially with file operations, and one could not
write a configuration file after all hard work of setting it up. :-(
Maybe now this situation improved as fairly recent releases of Tcl
were the first ones which were able to run their own self-test on
Alpha without tripping over. I did not try.

I agree that XF86Setup, when it works, is nicer than Xconfigurator.
OTOH things are getting better. Recently I got X running on Linux/Intel
machine in around five minutes from the moment of hooking up a monitor
which I never seen before. No, this particular monitor was not listed
among Xconfigurator choices. Got only one resolution in 16-bit depth
but this happened to be exactly what was desired on that box. :-)
