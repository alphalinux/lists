>>> Jeff DeMaagd said:
>
> The board? I'm not sure how this would work since it's not an AT-stype
> board. The chip is 500MHz 21164a. The board has most if not all the
> trappings of an LX ex. that there are cache slots, possibly a style only
> DEC uses.

MIATA motherboard has 2 64-bit PCI slots in front of a builtin PCI-PCI bridge,
which is where the remaining 3 32-bit PCI slots live.

Core logic is PYXIS, same as on the LX and SX and UX.

Cache is done with a small add-in card, quite proprietary... :-\

Performance can be characterized thusly:

1. for compute bound tasks, the tertiary (L3) cache will help a lot.

2. for tasks which perform lots of bus traffic, ie host memory to PCI bus
or vice versa, the cache actually gets in the way and slows things down
a bit.

Note that neither of these observations are only true on MIATA... ;-}
