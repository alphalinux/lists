Okay, this one's really starting to frustrate me...

I've got an Onsale as200, and I followed the instructions at
http://www.toast.net/~strycher/AS200.html (to upgrade the firmware and
install ARC) and I've gotten into milo just fine, and it sees all 24MB
RAM.

However, the problem I'm running into now is that when I do a 'boot
fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1' with a
boot disk I created using 'dd if=avanti.img of=/dev/fd0' on an Intel
Linux box I get the following:

...
[scsi stuff detected fine]
...
Partition check:
sda:
unknown partition table
(naturally, because sda is a Quantum Maverick 540 out of a Mac)
VFS: disk change detected on device 2/0
...Fatal error: not enough memory

and then it just hangs there with the floppy drive activity light
glowing.

The box has only 24MB in it right now, soon to grow to 96MB when I can
get it home (I have it at work for the time being).

Is 24MB not enough to boot a kernel and start the RH5.1 install
process?

If 24MB *is* adequate, what else might cause that error message?
