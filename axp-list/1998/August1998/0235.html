> Needless to say, the kernel is a bit different. Can anyone point
> out a safe way to track these in a running kernel?

The old-fashioned way 8-)

Start by building a kernel.

Generate a symbol listing for the kernel by issuing the command:

% nm --numeric-sort vmlinux > vmlinux.map

in your top-level build tree.

Now boot your kernel. When you get unaligned accesses, you will get
an "unaligned-access" message containing the PC of the offending
instruction.

The "vmlinux.map" file contains, among other things, the starting
addresses of every routine in the kernel. It's thus a simple matter
to establish the routine where the offending instruction resides, and
the offset into that routine.

You can then go and disassemble the ".o" file containing that
routine to figure out exactly what in the routine is causing the
problem.

Working back from that to source code is an exercise left to the
reader 8-)
