There is no alpha patches in the 2.1.x series kernel because it's all
there first!
Any new support for the Alpha architecture (or for any arch for that
matter) will get implemented into the 2.1.x kernels first! As opposed to
the Intel arch (which is more mature), the Alpha are _TYPICALLY_ (YMMV)
more stable in the 2.1.x kernels (I'm running 2.1.104 for nearing 2
months now without a single hiccup, stable as a rock!).

True, the 2.1.x will have their problems (like the recent framebuffer
thingie between .107 and .111..), if you're like most users, maybe this
won't affect you as perhaps you don't need it!

Therefore, look at the 2.1.x change log for a certain version, and if
the problems stated with that version doesn't affect you then go for it!
This is especially so if the version has a feature that you _MUST_ have!

I realize that the world needs a stable kernel, i.e. the 2.0.x but I
like to live on the fringe... ;-) Try it, maybe you'll like it too! The
bleeding edge welcomes you!
