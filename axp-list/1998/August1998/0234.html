> The 4D40T, 4D50T, 4D60T are using a chipset from Integraph, not Cirrus.

You are both right; the 4D40/50/60 have a (relatively dumb) Cirrus VGA chip
on the board to take care of text mode console, and an accelerator chipset
made by Intergraph, IIRC. The text mode console support is very important,
otherwise you're forced to have framebuffer text rendering code in the
kernel, like we have for the old TGA (TGA2, aka 3D30 or 4D20, uses a Cirrus
for this purpose as well), and now AMIGAs and ATARIs and such.

And Tim is right as well, I think there is only 256K of memory for the
Cirrus to work with.

One could run the XF86_Mono on it, I believe, because that server is not
chipset specific (other than expecting "standard" VGA :-).
