Okay, I got aboot working (yay!) and I'm trying to boot kernel 2.1.113
on it.

When I try to boot the compressed image, I get an error about the block
size being smaller than expected. Same thing happens when I try to load
a plain image. When I try to load a raw image (I copied vmlinux.nh to
-, and booted that), it boots fine (after some time (at least 30
seconds) ), and promptly halts CPU 0 with the following messages:

halted CPU 0

halt code = 5
HALT instruction executed
PC = fffffc0000310000
boot failure

What I'm thinking is, the kernel is panicing when it can't find a
console. There was no option for a TGA console (at least in make
menuconfig), and as such I used a framebuffer device for it, which
people seem to be avoiding at this point (possibly for this reason).
(If anyone cares: AlphaStation 600 5/333 (alcor), ZLXp-E2 TGA video
card, Tulip network card, Qlogic ISP scsi card...) I have no serial
cable to hook it up to another machine with, so I can check the messages
- that was my first thought. My next trick will be to install a VGA
card and see what happens. Unfortunately, this means recompiling the
kernel with support for VGA. If that doesn't work, I'm going to an
earlier 2.1 kernel (say... 2.1.108? I think that had a straightforward
TGA console...) and see how that flies. Anyone with a clue that can
help me out would be greatly appreciated - TIA.
