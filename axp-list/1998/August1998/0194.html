On Tue, 4 Aug 1998, Maurice Hilarius wrote:

> >Neither do I. Having significant Solaris experience, that it how I would
> >expect it. The problem is that *Linux* does idiot SCSi remapping. For
> >example, right now I have three SCSI devices. A hard drive, tape backup,
> >and CD burner. The hard drive came with SCSI ID 6. Linux calls this
> >/dev/sda. When I put the CD-ROM (ID 4) into the chain, the hard drive now
> >becomes /dev/sdb. Duh? Solaris does it right. Linux does not.

> Huh? A CDROM shows up on your system as an SDx device? That is very
> strange, my friend. IT should show up as a SCDx type. All the setups I've
> ever seen number the drives based on sCSI ID, low to high. The only "funny"
> thing I see is how it treats IDE drives as "SCSI" too on many Alphas.

Yes, you're right. I was spouting. What _actually_ happened was that I
was installing Linux on a Sparc with an internal drive set to SCSI id 3.
This was called 'sda'. I then attached an external drive at id 1.
Woohoo, the system now will not boot because it cannot mount the root
filesystem now residing on sdb. I forgot that Linux put CDs and Tapes on
their own s{letter} independent disks. However, I still have the problem
with the sparc (sure I could boot, edit fstab, shutdown, attach the drive
and restart, but the problem is fundamental). The external device did not
have a scsi id toggle externally. Old hardware...
