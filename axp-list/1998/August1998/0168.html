Have you made the ramdisk.img floppy? To boot I did:

boot dva0 -file vmlinux.gz -flags "root=/dev/fd0 load_ramdisk=1"

Or the shorter version:

b dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1"

> > Here is what I did:
> >
> > 1.) Start the RedHat install.
> > 2.) Partition using fdisk, leaving space at the beginning of the drive (1MB
> > should be fine, but you may be forced to waste more due to cylinder-size).
> > Write down the offsets you used for each partition. Save the changes.
> > 3.) *re*-partition, this time using BSD disklabels (the 'b' option in
> > fdisk). Create the same partitions, including the beginning space, using
> > the same offsets. This is to work around a problem in the RedHat install.
> > 4.) Procede with the install.
> > 5.) After install is complete and you've booted off the floppy, run:
> > swriteboot /dev/sda bootlx (assuming your root partition is /dev/sda2).
> > 6.) Reboot.
> > 7.) In the SRM console, you now want to use:
> > b dka0 -fi 2/vmlinux.gz -fl "root=/dev/sda2" (you can set these as
> > default in the SRM console).
