Dear list,

I applied all the modifications that Adam Powell advised (except the
change to dbm/include/hash.h) and added the -mieee switch to the Linux.mk
file in config/. Mozilla then compiles to completion, but when run, a
dialog box stating the following comes up:

"Mozilla was unable to create a network socket connection. There may be
insufficient system resources or the network may be down. (Reason:
XP_GetBuiltinString: 4457 not found) Try connecting again later or try
restarting Mozilla."

The application does function however. (I.e. bookmarks editor works etc.)
This is on lesstif-0.85.2.

The binary kindly made available by Kazushige Goto runs on this machine,
but doesn't seem to be able to resolve DNS queries. It does seem the
rendering of HTML is also very icky. The previous mozilla snapshot that I
compiled on lesstif seemed to give better yield in this respect.

Just thought I'd report as other people might be playing with the same
thing.

Thanks to both Adam Powell and Kazushige Goto...
Charl
