Armin Ollig wrote:

> i try to compile a 2.1.115 and 113 Kernel on a PC164 based Alpha (RH5.1).
> I get an internel compiler error and cannot explain *why*.
> Is there a mistake in the assembly code that egcs generates?

It builds OK with gcc 2.7.2.3.

> page_alloc.c: In function `__free_page':
> page_alloc.c:170: internal error--unrecognizable insn:
> (jump_insn 300 292 301 (return) -1 (nil)
> (nil))
> {standard input}: gcc: Internal compiler error: program cc1 got fatal

That error means that the internals of egcs have determined that it needs
to generate a particular instruction pattern (called 'jump_insn' with
some parameters), but the backend of EGCS hasn't been taught how to produce one.

It means either something is geneating a type of instruction pattern when it
shouldn't, or someone needs to go in and teach the backend about an instruction
type. I suspect its the earler of these two; although I don't know it
that well.

> [root@alpha linux]# gcc --version
> egcs-2.90.27 980315 (egcs-1.0.2 release)

Thats ancient.
