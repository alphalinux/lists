Quoting Larry Creech (lcreech@lonestar.rcclub.org):
> I've noticed very high CPU utilization (~50%)using the precompiled 2.0.34
> from redhat on the AS-200 4/233. Is this a proc problem? If I kill klogd
> and not syslog, things look normal.

You've got the syslogd bug, same as many, many others of us. I bet if you
look in /var/log/messages, you'll see:
Jul 16 15:39:02 canoe syslogd: select: Bad file descriptor
Jul 16 15:39:33 canoe last message repeated 32511 times
Jul 16 15:40:34 canoe last message repeated 79918 times
Jul 16 15:41:35 canoe last message repeated 76453 times
Jul 16 15:42:36 canoe last message repeated 76384 times
Jul 16 15:43:37 canoe last message repeated 81842 times
Jul 16 15:44:38 canoe last message repeated 84954 times

Some people have reported that they could delay the onset of this bug by
increasing the number of file descriptors, but that has *NOT* been the case
for me.
