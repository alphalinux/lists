Michael wrote:
>
> Hello..
>
> I am trying to add a Netgear FA310TX card to my UDB which is running 5.1.
> Upon boot I get the following message;
>
> "Bios32.sio_fixup: wierd device 8086:0484 coming in on slot 7 has no irq
> line"
>
Those id's are in my kernel source (include/linux/pci.h) as an Intel 82378,
what ever that is. Think someone a while back said it is a bridge
but I don't know for sure.

What kernel level are you using?? I've used RH4.2/RH5.0 kernel source
and things were recognized. Also used Richard's generic 2.1.111?
kernel and 2.1.98 and all were working correctly. Sounds like your
kernel is old.

I thought the UDB had an on-board Enet chip? If so, you can
have issues getting it to work and a card at the same time.
Rumblings I heard are that the on-board DEC Enet chips prefer
de4x5.c driver. Haven't used that driver on the FA310TX.
If de4x5.c supports MII, it should work though. Also, I think
you set a boot parm to get both interfaces loaded?? Maybe
using modules allows then to be recognized correctly
without the boot parms. I use modules for tulip. Try
a newer kernel and setup de4x5.c and tulip.c as modules
and try both to see which works.
