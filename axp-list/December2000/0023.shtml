Mike Tibor wrote:
>
> On Fri, 1 Dec 2000, W Bauske wrote:
>
> > Rob Byrnes wrote:
> > >
> > > Mostly for my own information, aren't most desktop systems/workstations
> > > using random small accesses? Shame these drives are so darn expensive
> > >:) would _really_ make Linux fly!
> > >
> > > Rob
> >
> > Not necessarily. The issue is what does a seek cost vs the
> > data transfer costs. If you have a 5ms average seek time,
> > then at 36MB/sec, you could transfer about 180KB in that
> > same time. More likely, DB's would like X15's. I don't do
> > much with DB's so it's not a problem. Programs loaded into
> > memory aren't all that small but they aren't like data files
> > either so they're somewhere inbetween. The Maxtor has an average
> > latency of 14.7ms according to the HDtach chart, about 3x
> > the X15. That's why I say random accesses will love the X15.
> > If your transfer times are dominated by data movement, the
> > X15 is a waste of money. (My files are BIG).
> >
> > Your best bet for a desktop is to buy enough memory to allow
> > Linux to cache your frequently accessed files there.
> > I use 384MB for my desktop. (Athlon 900)
>
> General use file servers, Squid caches, mail servers, and web servers are
> all random I/O sensitive to varying degrees. Naturally, those servers
> have to be running in a relatively busy environment in order for that
> bottleneck to manifest itself.
>
> On the other hand, lots of servers out there combine many of the above
> functions and more, and in such cases there doesn't need to be much load
> to slow such a box to a crawl if it only has one drive installed.
>

Putting more memory on is really the best performance
enhancement. Heck, commodity memory is so cheap right
now, you'd have to be really buudget squeezed not to
put a good chunk in your servers.

Given the choice of paying for Cheetahs or memory, I'd
buy memory.
