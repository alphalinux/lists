> On Sat, 16 Dec 2000, Howard Arnold wrote:
>
> > The disklabel
> > command allowed you to copy the partition table of a disk to a file and
> > modify it. It could than be copied back to the disk.
>
> look at sfdisk, it can read from a file and output to a file.

'sfdisk', which is a short for "scriptable fdisk", works very well, and
for years, for FAT type partition tables. If you will try to use for a
disk with "disklabel" you will see something of that sort:

# sfdisk -l /dev/sda

Disk /dev/sda: 8715 cylinders, 64 heads, 32 sectors/track

sfdisk: ERROR: sector 0 does not have an msdos signature
 /dev/sda: unrecognized partition
No partitions found

I wrote somewhat similar, but simpler and not implemententing full
functionality of 'sfdisk' (main stuff is there), utility which reads
and writes disklabels, only, as they are used by Linux on Alpha. If
there is an interest I may put it up for ftp.

I read once a account of somebody very proud of himself who was driving
in scripts 'fdisk' from stdout. Not very hard to do, if a bit funny and
scary at the same time, but the guy had 'sfdisk' and only was not aware
of that. :-)

Older distros of Linux for Alpha included 'minlabel' utility. I am not
sure how usable it was in scripts, probably it was, but apparently it
had some problems.

  Michal
