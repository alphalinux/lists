> b: 2048256 4097520 fd # (Cyl. 4066 -6097)

> notice the h partition. This is the partition that I boot from. I had to
> do this to keep aboot happy.

It should not be very hard to teach aboot about '0xfd' fstype. You can
likely equate it interally with ext2. 'aboot' is not a very complicated
piece of software.

Of course if you have a partition label to spare you can do the same
what Alvin did. BSD type partitioning explicitely allows for
overlapping partitions but you may have only up to eight of these total.
