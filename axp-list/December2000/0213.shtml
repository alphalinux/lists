This is not likely to change anything as these are Tru64 binaries,
and their supporting libraries, and as such are rather distribution
independent. You may equally well install that on any system with
5.x distro (which I did, by the way, and it works without any problems)
as long as your kernel has support for ECOFF ("AOUT") binaries.

> Nothing that
> has been suggested to Peter has worked for
> me and I have tried many other things as well.

Did you try to set MOZILLA_NO_ASYNC_DNS environmental variable?
This is known to help in certain circumstances.

> Peter can confirm that he has the same problem
> I do by going into netscape and opening
> a web page BY IP ADDRESS.

Which means that your name resolution has problems. One possible
suggestion to try would be to set your machine as a name server
("cache-only" should be good enough and there is a prepared setup
like that in your distro) and point all your name resolution to it.
Maybe responses from a name server you are using right now are so
slow that Netscape is loosing a patience? Or maybe you have some
proxy/firewall in between which is mangling packets?

Other than that you can get tcpdump and start tracing packets to
find out what is going on. 'ethereal' from Powertools may make that
easier but there was a recent security update for it.
