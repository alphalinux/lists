Ron Farrer wrote:
>
> Hello;
>
> I created some partitions on a new disk like so:
> Disk /dev/sdc: 64 heads, 32 sectors, 22130 cylinders
> Units = cylinders of 2048 * 512 bytes
>
> Device Boot Start End Blocks Id System
> /dev/sdc1 1 11 11248 6 FAT16
> /dev/sdc2 12 22130 22649856 5 Extended
> /dev/sdc5 12 2032 2069488 82 Linux swap
> /dev/sdc6 2033 22130 20580336 83 Linux
>
> The FAT16 partition is the only one that works. Milo says something
> along the lines of: device not a directory, when trying to boot from
> it. I created the FS with 'mkfs -t ext2 /dev/sdc6' From milo I can
> type 'ls sdc1:' and it shows the contents of the first partition, but
> when typing 'ls sdc6:' it shows nothing (blank spot where the files
> should be). I can boot off another drive and mount the partition just
> fine... Any ideas?

Possibly sparse super block FS and your milo is too old to
recognize them?? Try man mke2fs and see if yours defaults to
sparse super blocks or not. If it does, use the option to
turn it off.
