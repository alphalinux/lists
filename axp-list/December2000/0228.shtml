On Mon, Dec 18, 2000 at 12:09:11AM -0600, W Bauske wrote:
>
> OK. This then begs the question "how many C compilers shipped with RH7.0
> Alpha?".
> I don't have it so I don't know off hand. Did they do the same thing on Alpha
> as x86, gcc and kgcc? One would assume only one if (2) is really the case.

There are two. The reason is this - it is well known that the Linux
kernel depends on known compiler behavior. While I'm doing fine
running my 2.96-built kernels, there is that distant chance that there
may be some timing bug in some diver that a customer uses that will
pop up and corrupt their data. Now, my feeling is that most of this
is kernel developers feeling uneasy about 'new things'. That's just
my opinion.

I think that the sooner that we can get developers using a new GCC
codebase the better. GCC 3.0 is very important to the Linux and GNU
communities. Hopefully with the fixes in 2.2.18 and the general
fixups in 2.4.x, developers will give 2.96 built kernels a try. If
folks run into problems, we're here to find them and fix them. If the
problem is in the kernel, we'll fix it there. If it's in GCC, we'll
do the same. I personally feel that the GNU/Linux development
platform will be better for it.

Cheers,

Matt
