hi,

i have modified/ported glide2x to linux/alpha.
(and of course modified/ported device3dfx)
i wonder if someone here is interested in the port.

i am playing tux_aqfh on my alphastation255 with a voodoo2 card :)

i have been in contact with daryll from 3dfx but currently he does not
answer ... maybe he is in holiday.
the version i've ported compiles on linux/alpha as well as on linux/i386.
most of the code is like:
#ifndef __64BIT_POINTER
   /* original 32bit code */
#else
   /* my 64bit code */
#endif
currently it is not possible to modify the main source code.
daryll offered me a seperate cvs tree, but we have not talked much
about it... (yet)

i want to give the alpha community some sort of a xmas present.
so i'd like to know if redhat, or the someone of www.linuxalpha.org is
interested in giving me some ftp space...
... i'd really like to share the code, because it makes not much sense
if i can use the voodoo2 on linux/alpha and a lot of other people cannot.

rds, niki

btw: the port i did is just for voodoo2 / glide2x. i have not done anything
with glide3x (but it should be no problem to port that too).
i also did nothing with voodoo1, voodoo3, voodoo4 or voodoo5 code, neither
with DRM, DRI, kernel, X or whatever code.
