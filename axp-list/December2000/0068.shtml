I seems like setting the high water mark for a DATAGRAM socket just does
not take hold. any request for less that 256 bytes ( ie 2 bytes ) gets
uped to 256.

From docs & stevens book, one is setting/getting the max size of the
(data?) in the datagram pkt. Apparently linux has a different viewpoint.
I think setting ( or suggesting) the
high water mark should result in a value <= to that set ( but not larger
! )
