David Singleton wrote:
>
> Wes Bauske wrote:
> > This is off topic but I think a few people will be interested in this.
> > I have a Dell P4 1.4Ghz system for testing and here is it's stream result:
> >
> > Function Rate (MB/s) RMS time Min time Max time
> > Copy: 1600.0000 0.0333 0.0300 0.0400
> > Scale: 1600.0000 0.0344 0.0300 0.0400
> > Add: 1800.0000 0.0443 0.0400 0.0500
> > Triad: 1800.0000 0.0463 0.0400 0.0500
> >
>
> I dont doubt these are roughly right but I think you need larger arrays,
> the timer resolution looks very poor.
>

Yes. That's what you get for cpu resolution on x86. It would take 10x
the memory to reach the recommended 20 clocks per loop. Just use the rms
value to get a better idea of the performance. The rate is from the min
time so (min/rms * rate) is a better measure of the performance.

The test used 67MB of memory on a 128MB machine. I have 256MB of RDRAM
on order to add to this box but given the Christmas crush, I doubt I'll
see it before January. Even adjusted to (1440/1395/1625/1555) it's
still the fastest non-vector box I've seen on stream. I don't rely on
stream to predict preformance though but some people on this list like
it which is why I mentioned it. It would appear Intel are going in the
right direction.
