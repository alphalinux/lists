this present makes it possible on linux/alpha to get
hardware 3d accel on a voodoo2 based (pci) 3dfx card.

it is possible to compile all of the stuff on:
 alpha
 ix86
 linux kernel v2.2
 linux kernel v2.4
(any combinations :) )

to compile the stuff
 1) untar xmas_resent.tar
 2) uncompress / untar Device3Dfx-2.3-6.tar.bz2
 3) uncompress / untar GlideV2-20001221-091251.tar.bz2
 4) compile the 3dfx device (type make in the dir generated by 2))
 5) compile the GlideV2 library:
       cd xmas_present/home/install/GlideV2/glide2x/
       ./niki_make.sh
    some time later the lib should be built.
 6) test the lib:
 6a) plug a voodoo2 pci 3dfx card into your alpha machine
 6b) as root copy the 3dfx device driver somewhere to /lib/modules/<kernel version>/...
 6c) depmod -a; modprobe 3dfx (or insmod 3dfx) (/dev/3dfx should exist on your linux dist)
 6d) cd xmas_present/home/install/GlideV2/glide2x/
     ./run_test00.sh
     then gdb starts with test00.
     when typing
        run
     and pressing <return> when asked
     you should see a blue screen on your monitor with some text.

     if you dont have gdb then modify the line
     "gdb ./cvg/glide/tests/test00 2>&1 | tee out-run"
     to
     "./cvg/glide/tests/test00"
     in 'run_test00.sh'
