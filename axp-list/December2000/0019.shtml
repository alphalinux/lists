Actually, this is how SCSI differentiates itself from IDE. SCSI devices existing
on the same bus can be performing I/Os concurrently, although only one device can
be "on bus" at any one time. This allows dramatically improved performance with
SCSI when you have more than one active device. With IDE, only one device on a
channel can be in the process of performing an I/O.

RE:CPU
If an IDE device is using PIO (instead of DMA) it will use _enormous_ amounts of
CPU. Unfortunately, Alpha systems were slow in the uptake of DMA-enabled
chipsets, but if your system supports DMA (and it is enabled correctly) then
you'll discover that IDE uses the same low CPU utilization (or even less) than a
SCSI system.
