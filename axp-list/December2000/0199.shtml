> I was wondering what is the usually percentage that the
> /var, /usr, /home, and ect partitions should be..

This is going to be fun. This question is about as religious as the one for
the "best editor".

I'll try and give my opinion - but only for servers, not workstations:

a. Don't use percentages - use what you need in certain partitions, leave
the rest for future use.

b. my usual choices:
/ will hardly ever need more than 200 MB

/usr on a server with no X is usually between 200 MB and 500 MB if you
really only install what you need. I usually give it a GB to be safe with a
couple of kernels, sambas and whatever I need to compile.

/var depends on the server's tasks. One of my web servers easily gets a GB
logs daily,
mail spools can be worse, same goes for squid proxies. I usually start with
a GB here unless I know in advance it is going to be more

/root is root's home directory, i.e. new packages to be installed, the
server docs you will of course write ;-) will completing the server
configuration and the like - 300-500 MB will do here

For /tmp I usually give 250-500 MB, depending on the disk size

/home only depends on how many users you have and what you'll allow them. An
rpm install of ftp and http servers will put there data here.

/opt is hardly ever used under linux these days, again unless you have X. I
make it 100 MB to be safe that a package does rather fail upon install than
bust /.

On Intel I would always put a 30 MB /boot at the beginnging of the disk just
to make sure lilo doesn't get in trouble.

I usually leave the rest free so I can use it where I need it when I need
it. (webspace, database, even bigger mail spool, whatever). Others will have
better suggestoin. Listen to them all and find your personal guesstimates.

Good luck!
