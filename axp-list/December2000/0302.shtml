Hi,
  I was trying to get Xine going on Linux/Alpha (Xine is a DVD player not
entirely different from LiVid). Anyway; it is falling over - I think this
is because it is doing a shmat to allocate memory (in the alloc image
buffer code for Xv video stuff)
and a 0 pointer is getting returned from shmat. According to the shmat
man page this should not happen. Neither can I see a reason why it is
happening. Looking through an strace output I see outputs like:

shmget(IPC_PRIVATE, 144000, IPC_CREAT|0x1ff|0666) = 203882558
osf_shmat(0xc27003e, 0, 0) = 2199039279104
shmdt(0) = 0
shmctl(203882558, IPC_RMID, 0) = 0

Why the osf_shmat?

Anyone else seen this ? This is CVS current xine as of about 8pm GMT.
