Does it actually do bit maniuplations within the code and assume an
endianess, or is it trying to read a binary file and assume an endianess in
the file?

We have several standard fortran codes that we use, they don't do any bit
manipulations within the code, but we share binary data files from our
SGI's with them. We compile our alpha codes(using Compaq's Fortran) with
'-convert big_endian'
This will read and write binary files from SGI's fine, but it doesn't
affect bit operations in the code.

Unfortunately, we also have an old fortran code that does bit
maninuplations and for that our only solution has been to stay with SGI's.
