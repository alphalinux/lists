On Sun, Dec 17, 2000 at 10:51:04AM -0500, John Jasen wrote:
>
> There have been multiple reports that it cannot compile kernels. I believe
> them, as I've had a lot of problems with 2.4.0-x kernels that compile fine
> on gcc-2.95-2.
>
> My RH 7 test machine, using gcc-2.96, also croaked on compiling devfsd.

My Alpha XP1000 workstation has been running 2.2.18pre24 compiled with
the GCC 2.96 snapshot for over two weeks with no problems. I decided
to build 2.4.0test12 with 2.96 on Alpha, after a few header patches
for Alpha it built and ran as well, with few problems except:

o de4x5 hasn't been updated to work properly for 2.4.x, oopses the
   kernel in startup in srom_search or somesuch place

o several unresolved symbols in usb modules (I fixed some problems like
   this in 2.2.18pre23, the fixes are in 2.2.18 proper)

o Netscape started segfaulting on startup after about 1 day of use.
   This was different than the root property bug that I've worked
   around in the past.

2.4.0test12 was working well enough for me to use it, except for the
Netscape problem - so I've gone back to my 2.96 snapshot-compiler
2.2.18pre24.

If there were other build problems, like those in devfsd (which I
didn't see in my 2.4.0test12 build), I'd certainly like to know about
it.
