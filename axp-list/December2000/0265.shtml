At 13:26 18/12/2000 -0700, it was written:
>It is somewhat frustrating when various people over time are telling in
>a response to a complaint "It looks like that your DNS is set
>incorrectly" and after that you annouce "Big News! By a long trial and
>error it was established that my DNS is set incorrectly". Indeed.

While this sort of generic response may be fine for experienced Linux
users, raw newbies (such as myself) are often frustrated by these sort of
responses. Yes, I have read the man page, and it was still unclear. Not
until a plain English answer was offered did I know what to change. For
the record, my /etc/resolv.conf file did not have any search line in
it. If I was not informed by Peter Watkinson of what exactly to put into
this file, I would still be none the wiser.

>Let's see what documentation happens to say. From 'man 5 resolver':
>.....
>
> search Search list for host-name lookup. The search list is normally
> determined from the local domain name; by default, it contains
> only the local domain name. This may be changed by listing the
> desired domain search path following the search keyword with
> spaces or tabs separating the names. Most resolver queries
> will

<snip>

This may as well be in French for the most part. I am running a single
box, no "domain", and a randomly assigned IP address. As far as I could
tell, it was not relevant to my case.

>For most dial-up connections it is quite clear that this
>line is just superfluous. Did you notice, conditional, "slow" and
>"a lot of network traffic" and "will time out" above?

It may well be superfluous, but putting it in resolv.conf, and running bind
on the same machine has fixed a lot of problems. Not real good for
security (maybe ... I don't know), but it works.
