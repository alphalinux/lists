1) Thats nice if u care to ( or know of ) xntp. Tell me if xntp sets ur cmos clock
chip.

2) u appear to have a distorted view of ARC/NT. Floppies ? The particular
proceedure is no different than the various other multi-booted Windoz systems. ( or
even the multi booted Linux (RH70) systems )

BTW: simply typing boot at SRM PRESUMES that u set the various other flags to get
this automatic feature going. ARC, on the other hand is more complicated in booting
linux. But once set up, either one is easy to boot from! Some folks are comfortable
with the cryptic SRM, but i suspect that novices are going to me more comfortable
with the GUI.

AND yes, it would be nice if the ARC folks & the ARM folks got together to support
each others supported hardware. That includes SCSI cards ( adaptec ) , on-board IDE
devices, & my nic ( 3com 3c509b ). But u just cant tell what will be supported in
the long run, as adaptec does not give source ( or manuals to program it ) , as
well as compaq for the arc, & srm.
