If you're booting from SRM whay are you using MILO at all? Is the
cheapbytes CD not bootable from SRM?

something like:
boot dqa0 -fi kernels/generic.gz root=/dev/hda should work. Note that the
devices I put in there were IDE, you want to sub in SCSI devices I expect
(a show dev from SRM will tell you what is what).

If you want to use MILO then I'd suggest going over to ARC console for
this. Use the udb-noname.img file from the /milo/images/ directory and you
should be fine.

Once you create that MILO floppy (from the image, using dd) you should be
able to read it as a regular DOS (FAT) floppy on any system. If you can't
then you've got other problems....maybe a bad floppy.

--rdp
