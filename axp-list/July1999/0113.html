A few days ago someone posted (to the V4L list) about whether anyone had
successfully used a Color QuickCam on Linux/Alpha. I tried this several
months ago, asked some questions, was unsuccessful, and moved on to
other things.

But I tried to do it again last night. This is how far I got. Using
kernel 2.2.3, the parport_probe, videodev, and c-qcam modules, I am able
to detect the camera on the parallel port. Using xawtv, I get the following
output:

I got a message before saying that I couldn't use 8bit color (PseudoColor?)
mode--that using the 8-bit greyscale mode (whatever that's called) would
work--so I have started my X server with

   startx -- -cc StaticGray

That raises another question: Does V4L require a particular video card? Maybe
one that supports overlays? I'm looking at the message in my output above
that says "overlay mode disabled". I'm using the builtin TGA (DEC 21030)
video in the Multia.

Any help?
