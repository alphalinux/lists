> However, it is still (I say "still," but for some reason I neglected to
> mention this in my original post - apologies) hanging up on all tty users
> after a 35-40 second interval, be they console, telnet, or root. Logging
> in through an xterm works fine, however. Just to make things more
> interesting, the date command returns a different date every time it is
> invoked. I don't know if this stuff is related to the earlier problem, but
> I do _know_ it all started after I first installed 6.0.

I had this exact same problem when upgrading my Multia UDB from RH5.2 to
RH6.0

I had to change the MILO command line in the bootstrap loader from
boot sda2:vmlinux.gz
to
boot sda2:/boot/vmlinuz

(RH5 Alpha had the kernel in the root directory, RH6 has moved it to /boot
like the i386 version)

HTH,

Brian.
