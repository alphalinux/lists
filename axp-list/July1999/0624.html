Richard Henderson <rth@twiddle.net> writes:

> On Thu, Jul 29, 1999 at 09:24:08AM -0400, Uncle George wrote:
> > BTW as a side issue, there doesn't seem to be any way to get the
> > compiler to do different 'rounding' techniques. Ie chopped, +infinity,
> > -Infinity.
>
> -mfp-rounding-mode={n,m,c,d}
>
> n == round to nearest
> m == round to minus infinity
> c == round to zero
> d == dynamic rounding (mode comes from fpcr)

But doesn't this set the rounding mode at program start? If so, it
means that it can be dangerous because libm might require a particular
rounding mode - switching it could cause libm inaccuracy.
