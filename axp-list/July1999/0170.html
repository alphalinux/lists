After 'nbench' was mentioned on linux-kernel, I wanted to see how my
AS200 would fare. nbench is interesting because it allows you to
compare your CPU and compiler against reference Pentium-90 and K6-233
numbers.

nbench was obtained from http://www.tux.org/~mayer/linux/bmark.html and,
for each benchmark run, was compiled with
'-s -static -O3 -fomit-frame-pointer -funroll-loops -mcpu=ev4'

Platform: AlphaStation 200, 233 Mhz EV45 CPU, 144 MB RAM, RedHat 6.0
Benchmark 1: reference benchmark, using standard RedHat devel toolchain
Benchmark 2: benchmark using CVS-latest egcs, to guage egcs updates
Benchmark 3: CVS-latest egcs and Compaq Fastmath lib

Caveat Benchmark: I did not analyze any of the benchmark code for
bugs. I did not run a statistically significant number of tests. I did
not kill off background daemons. Run the benchmark yourself for real
numbers. Use with adult supervision only. etc. etc.

Summary: CVS-latest egcs seems to add a bit of speed. Compaq Fastmath
library makes a _lot_ of difference.

        Jeff
