> > How would i set this up so that i can assign host names
> > to the internal network that can be accessed from the outside?
>
> You cannot. Only names properly registered and with official
> IP addresses can be visible from "outside" (at least in general).
> Internal (192.168.0.x) adresses by a definition are non-routable.
> If some router is passing them (which happens) then it is broken
> and they will be caught on the next stage.
>
> If you think about virtual hosting, when your machine visible
> on the Internet has multiple addresses/names and redirects IP
> traffic to some internal hosts depending on how a connection came
> then this can be done but you need as many different addresses
> (not names, aliases can be mulitple but will not help) as your
> internal hosts participating in such scheme.
