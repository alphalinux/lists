Lawrence Levinson wrote:
>
> I just installed RH 6.0 on a Multia/UDB. Although I set the root password
> as part of the install process, when I try to login as root after startup,
> I get constant "login incorrect" messages. I thought at first that I might
> have mistyped the password, so I redid the install from scratch. When I
> got to the step in the installer that asks for the root password, I typed
> the password in carefully (making sure the caps lock key wasn't down),
> then went through the rest of the install process. Then I restarted the
> machine as instructed. Result: same problem - I can't log in as root
> because the system doesn't recognize my password.
>
> I've tried booting into single user mode - the machine ignores this and
> dumps me at the login prompt, whether I type "boot dva0 flags -S" or
> "vmlinux.gz -SINGLE", depending on what step of the boot process I'm at.
>
> I've checked the axp-list archives for the past four months, read RH 6.0
> installation FAQs, consulted my local Unix and Linux gurus, wandered
> around www.alphalinux.org and www.linux.org, and done plain old web
> searches for things bearing on root passwords and logins, but I still
> haven't found the answer.
>
> Is there somewhere I should be looking that I'm not? Has anyone
> experienced this problem before?
>
> Thanks in advance.
>

I've seen this happen with flaky keyboards, where sometimes keystrokes
are recorded & sometimes not. THis is likely a long shot however.
Anyway, the standard way to recover from a lost root password is to boot
from media, mount the root partition & manually edit /etc/shadow.

Hope this helps
