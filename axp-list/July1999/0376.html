On Tue, 20 Jul 1999, Todd Henderson wrote:

> At the point of reboot, the document
> http://www.digital.com/alphaserver/linux/install_guide.html
> says to:
>
> set bootdef_dev dka300 (we don't have a dka300 listed so I use the dka0 that is listed)
> set boot_osflags "root=/dev/sda1 bootdevice=sr0"
> boot dva0
>
> It then goes on and says to use swriteboot /dev/sda /boot/bootlx and pico /etc/inittab,
> and now it should go.
>
> I don't get past the boot dva0 above.
>
> Thanks for any help.
> Todd
>

OK, dva0 is the floppy. Do you have an SRM bootable floppy in there? I'm
assuming you do since before you said the kernel did boot. Now what is
your root filesystem? The Compaq doc may list /dev/sda1 but which one did
you use as your root fs?

You see those boot_osflags get passed to the kernel, so the kernel will
try to mount whatever is at /dev/sda1 as your root filesystem. The other
question is which kernel are you booting? I'm guessing your using the
kernel mentioned in the doc (the generic-223 kernel). If you are install
RH 6.0 you can use the kernel that comes with that. Is the kernel
detecting your SCSI controller and disks?

Also I noticed that the Compaq doc states that the max swap size is 128Meg
for 2.2.x kernels, this is of course incorrect. I think the max size for
swap on Alpha is now 128GB.
