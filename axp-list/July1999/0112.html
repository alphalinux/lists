> I'm running a C-shell script via cron that includes
> a "limit stacksize 32M" line, to increase the stacksize
> from the default 8MB. Yet, when the script is run,
> I get a "limit: stacksize: Can't set limit" message.
> Is there any way to do this via cron? This is a box running RH 5.1.
> I have no problem doing this on my Intel 6.0 box.

Only root can raise a limit beyond the hard limit. Perhaps your 6.0
box has different limits? Run "limit -Ha" to find out.

There is an evil way to get init to raise your limits.
