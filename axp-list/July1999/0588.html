>>> Jim Barriault said:
>
> Has anyone been able to get redhat 6.0 or any version for that matter
> installed on an alphaserver 1000 5/xxx which is a mikasa-primo I think..
> When I try to boot the redhat cd from srm aboot finishes loading and the
> kernel image gets decompressed then the system panics and returns to the
> srm prompt. I had this same thing happen with an AS1000 4/266 and got
> around the problem by using the mikasa-debug image from genie.ucd.ie for
> the 4/266 system. But obviously this image won't work correctly for the
> ev5 system. I tried booting the alcor milo image from genie as well and
> this at least gets me to the milo prompt. But this image is not able to
> init the devices. I have tried to build milo from source without success.
> I am using the latest firmware on this system from the 5.4 cd. If anyone
> knows how to build a milo image then the same system options could be used
> but with 21164 op codes instead of the 21064. If anyone could help or has
> any suggestions as to how to get 'any' bootloader to work it would really
> be appreciated.

MIKASA, whether EV4 or EV5, should really be booted only from SRM console.

That doesn't make it any more reliable, however, since the more recent 2.2.x
kernels, in "generic" form at least, seem to ge having problems on a number
of platforms, MIKASA included.

That said, see if you can find a "mikasa-s-5.gz" kernel from off the RH 5.x
CDs, to see if one based on 2.0.x will work better than 2.2.x, and let us
know.

Good luck.
