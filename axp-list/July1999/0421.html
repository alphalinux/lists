FYI,

Thanks to all who responded with suggestions...you *do not* need to rebuild
the kernel on the Multia (or AlphaStation 200's) to get sound. You do need
to edit /etc/conf.modules though. This will allow you to use the stock
kernel
shipped with Red Hat and still get audio. Add this to /etc/conf.modules:

alias char-major-14 ad1848
options ad1848 io=0x530 irq=9 dma=3

Now, if you add dma2=1 it should allow you to use the full duplex features
of this chip. But it looks like something is hosed in ad1848.c and I haven't
had time to take a look. In theory, the Multia should allow audio capture
and
playback at the same time. So for now at least, until this is fixed, leave
off
the dma2=1. There are no MIDI capabilities with this version of MSS, so
that will never be supported.

Also, you'll need to have speakers plugged into the headphone jack. I don't
know why, but the module won't load unless there is something plugged in
there. And I don't get audio through the internal speakers. Curious...
