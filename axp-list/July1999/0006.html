Phil Marsden wrote:
>
> This one is more like a feeling than a bug report, but since upgrading to
> RHL6.0 on my SX users have been experiencing strange mail problems. These
> problems appear, so far, to only relate to mail being forwarded from our
> Netware server.
>
> We are losing, wait for it, only _some_ mail and don't know why. Worked
> fine on RHL5.2, but now it is not. I can't pin it down at all. There is
> nothing in the maillog, when a message fails to arrive, and can't see
> anything on the Netware box either - however I have yet to figure out the
> path which appears to be quite involved going through one NW box possibly
> another Linux box and maybe even a Sun before it gets to my machine. It is
> as if the message has not been sent, but it was!
>
> So my question is... has anyone else seen any problems with RHL6.0 on the
> alpha? Or even the sendmail 8.9.3-10 package?
>

Dunno what the problem is, but remember the following while checking
your logs:

1) Are you running nscd? There are a number of potential problems
that people are encountering running nscd.

2) sendmail-8.9.3 uses smrsh. That means that any local mailer
that are sometimes used by user agents must be found in /etc/smrsh.
A symlink will do ...
