> OK. Finally got it installed, Redhat release 5.2, with the 2.2.3 kernel.
> Now I need to get to 100baseT full duplex on the tulip card. Under the
> bios, show *ewa* indicates that it is set to the correct FastFD, but how do
> I tell Linux to use the faster settings?

"show *ewa*" only affects SRM and Digital Unix, not Linux.

I reported a while ago (on this list) that the de4x5 driver seems to
work better than the tuplip driver on the XP1000. Of course, with
tulip you always have to get the latest and greatest version to have a
hope of autonegotiating correctly. If you can't autonegotiate, the
webpage:

http://cesdis.gsfc.nasa.gov/linux/drivers/tulip.html

explains how to set options at module load time to force a speed.
