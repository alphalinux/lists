If you don't get any low-level IDE messages, it's very unlikely that
the disk goes bad.

Maybe you are only affected by the random corruption, that's
normally a special 2.2.8-2.2.10 kernel feature (hmm, you are using 2.2.5).
Or maybe memory is bad.

So late 2.2.x kernels are a bit risky, I use a 2.2.10 system with 0.90-raid
(RAID-1 + boot support) with nearly no trouble (after disabling quota (!?)).
Also swap to raid-1 seems to work now.

The latest kernel I've tested a lot related to NFS was 2.2.3 (IIRC unpatched).
(If 30 diskless nfs-rooted Alphas hosted by a single one means stress for
you (yes 30x bonnie)) 2.2.7 is also not too bad.

If you have a lot of time, walk through the 3MB (uncompressed) 2.2.10-ac10
patch and apply only things that look sane to you. ;-}
