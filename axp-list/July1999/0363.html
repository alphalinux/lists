On Tue, 20 Jul 1999, Feisal Umar wrote:

According to Jay's MILO table:
ftp://gatekeeper.dec.com/pub/DEC/Linux-Alpha/Kernels/systypes.txt

there is no MILO for the AS4000 (also known as rawhide). Looks like you
are going to have to boot via SRM. I'm not familiar with the 4000, but if
you go into AlphaBIOS, go to CMOS setup, then press F6 for advanced options
you should be able to change the machine to boot into SRM instead of
AlphaBIOS.

You may also want to update your SRM console, I think the latest is 5.4.
It's available from:
ftp://gatekeeper.dec.com/pub/DEC/Alpha/firmware/v5.4/

Once you get this far I'd recomend reading through the SRM Howto:
http://www.alphalinux.org/faq/srm.html

Also I think the RedHat install docs have some into on this also. The big
things to keep in mind are:

1) use fdisk to partition the disk, and put it into BSD Disklabel mode
(the b option).
2) Start the first partition on cylinder 2, leave partition c as the whole
disk.
3) you'll need to install aboot as a secondary boot loader (see the srm
howto for this).

Good Luck,
--rdp

P.S. I seem to remember someone talking about problems with the rawhide
kernel from 5.2, you may either want to check out gatekeeper for a
different kernel or just go with RH 6.0.
