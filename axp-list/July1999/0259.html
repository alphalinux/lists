>>> Flavio Pescuma said:
>
> I'm having some trouble booting a dual processor DS20.
> If I create a boot disk using any of the kernels on the RH6.0 CD(
> kernels/generic.gz or images/generic.gz) the machine goes into a loop
> when setting up scsi devices. If I change the kernel in the boot disk
> to any of ftp://ftp.digita.com/DEC/.../generic-smp.. it boots and
> prompts me for the root disk, I introduce a disk with ramdisk.img (dd to
> disk) from RH60 images/ramdisk.img, the machine starts reading and then
> halts and returns to P00>>
>
> Do you have any suggestions?

Use the generic kernel from ftp.digital.com, but DO NOT use the RAMdisk floppy.

Boot from floppy and set root to the CD via (for SCSI CD):

>>> boot dva0 -file vmlinux.gz -flags "root=/dev/scd0"

and the install should start up normally.

Good luck.
