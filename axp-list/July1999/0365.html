Raxel Chen wrote:
>
> I had already install RedHat 5.1 Successlly and compile 2.0.36 kernel
> When I telnet or ftp from other hosts , it always waiting for much time
> to login . And I ping the hosts from other hosts , if got very nice
> performance . It almost less then 10 ms.
> And when I "telnet localhost" on the hosts , the login prompt display
> very soon. So I suggest that it must be internal problem.
> Does anybody can help me , thank a lot.

I'd suspect a DNS problem. The system tries to do a reverse DNS lookup on
the IP address of the system you're trying to telnet or ftp from so it can
log the name.

As a quick test, try adding one of the "other hosts" to the /etc/hosts
file on the RedHat system and try to telnet or ftp again. If you have
a small local network, you can just add all the machines to /etc/hosts.
If it's large, you probably want a DNS server somewhere.
