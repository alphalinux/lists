> It's a linker thing--an unresolved external--so I don't think adding
> a header file is going to help...

Of course, if inb() is a macro (which it is), leaving out the header
file which contains it (<asm/io.h>, in this case) will result in an
undefined external. When the file is included, the "call" to inb()
is macro-processed into appropriate inline instructions and no function
call is made--and so no undefined external.

Just one example of why the C pre-processor is evil. :-)
