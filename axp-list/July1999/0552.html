> There was a alpha specific bug in the adjtimex syscall that xntp uses,
> which was fixed in 2.2.8. So try a newer kernel and see if that works.

Thanks. I tried 2.2.10, but the behaviour is the same.

However, setting "tickadj 976" (instead of the default 977) gives a
different drift (436.824 instead of -512.000) so it looks like the PLL is
working.

If I understand rightly, tickadj sets the number of microseconds in a clock
tick; since HZ = 1024 this should be (1000000/1024). 977 ought to be closer
to the correct value, but perhaps my motherboard's crystal osc. is slightly
off.

Problem solved, as far as I'm concerned :-)
