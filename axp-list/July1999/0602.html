>>> Jim Barriault said:
>
> > MIKASA, whether EV4 or EV5, should really be booted only from SRM console.
>
> Thats how I always do it. And in the case of the ev4 mikasa's I have
> used I have booted with milo from srm.

Yes, but then you boot a kernel that *expects* to be booted from MILO. The
kernels named mikasa-s and mikasa-s-5 expect to be booted from SRM, and will
most likely *not* work if booted from MILO.

> The kernel at least boots but it panics right after loading the ramdisk
> image. I have tried several different ways of preparing the ramdisk
> image with out any difference. It is that MS-DOS VFS kernel panic which
> i thought was a result of the disk either having or not having a msdos
> format prior to dd'ng the image. I didn't remember which it was so I
> tried both, as well as brand new floppies without success. Do you have
> any other suggestions? I would much rather use milo from srm if it were
> available or could be built for this platform.

The RH 6.0 install uses a RAMdisk floppy in a format different from previous
releases, and so, previous kernels will not work with it.

Try booting the 5.2 kernel, and setting root to the CD, like:

>>> boot dva0 -fi vmlinux.gz -fl "root=/dev/scd0"
