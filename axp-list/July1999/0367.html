Mark Hansel wrote:
>
> >Raxel Chen wrote:
> >>
> >> When I telnet or ftp from other hosts , it always waiting for much time
> >> to login .
>
> Also, make sure /etc/host.conf has the line:
>
> order hosts,bind

Actually pay closer attention what you have in /etc/nsswitch.conf.
This is not so essential with earlier distributions but current (not from
5.1 yet, IIRC) libraries take a contents of this file quite seriously.
If it does not reflect reality than you will slow down not only login
but various other things; for example, tar (a famous side effect :-).
A reason for that is that for every extracted file tar attempts to check
credentials of its owner. If directed to do that over NIS, which happens
not to be there, tar will stumble on every file to extract.

I do not think that the current resolver stuff from glibc consults
an "order" directive but I did not check sources.
