I just received some DLT tapes from another company
that were written on Sun Solaris with approximately
12MB physical block size. I tried to read them with
Linux on my Alpha but Linux fails with the messge

"dd: /dev/st0: Value too large for defined data type"

dmesg shows:

st: failed to enlarge buffer to 11987040 bytes.

It seems Linux is running into a limit on the order of 2MB
from my other testing.

I looked a bit at the code and I think a kmalloc()
for that size failed, or, possibly it tried to get a
DMA region that large. Anyone have any ideas how to
get the tape read?

Any advice is appreciated.
