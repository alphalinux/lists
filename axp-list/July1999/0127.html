Funny thing is, the hard limit for stacksize
is 128MB. When I am running the script interactively,
limit shows the stacksize to be that value. Yet,
when I run from cron, the limit is 8MB, and using
"limit stacksize" to use a greater value fails, unless
root runs the cronjob, which is not desirable in my
case. I'd try and use sudo, but limit and ulimit
don't seem to be valid commands in the /etc/sudoers
file.

Is there a config file that I can change, or, as I suspect,
is this hard cron limit set in the kernel?

Thanks,

Kevin
