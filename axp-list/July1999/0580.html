Larry Sendlosky <larry@scrugs.lkg.dec.com> writes:

> I saw something yesterday that said there was a problem
> with the 2.2.5 kernel on the RH6.0 dist when booting
> on a DS20. There was mention of problems with the NCR scsi
> driver. The solution was build a boot floppy with a newer
> kernel, eg 2.2.10. If you don't have a another Linux/Alpha system
> you may have a problem.

>> Bernhard Gschaider <Bernhard.Gschaider@unileoben.ac.at> wrote:

>> That's my problem.

>> Are there any compiled kernels around that could work?

I got this yesterday, from an internal source:

>The problem with the 6.0 CD installation for the DS20 is that it loads the
>wrong SCSI driver first and tries to use it. It needs to use the Symbios
>driver not the NCR driver. RedHat is aware of the problem, there are other
>issues with the 2.2.5 kernel (which is what is on the CD) as well. You
>should download a copy of the 2.2.10 kernel (I think that their is an
>internal site for the binary form) and make a kernel floppy. Then you need
>to use:

> boot dva0 -file vmlinux.gz -flags "boot_device=/dev/fd0 root=/dev/scd0"

> This will tell it to get the kernel from the floppy but boot the CDROM.

This worked for me on a DS20. I've placed this vmlinux.gz out on our
anonymous server xfer-alf.service.digital.com - once logged in, do a
"cd to_customer" and then you can do a get on " vmlinuz-2.2.10-DS20 "
and that should allow you to boot once you use rawrite (from RedHat 6.0
cd-rom under dosutils) or dd if from UNIX/Linux to put it on a floppy.

I picked up

-rw-r--r-- 1 root root 58767360 Jul 27 16:08 linux-2.2.10.tar

Note the size (whoa!!), 13MB gzipped, but I've not yet been able to build
an image that I have been able to use to boot the DS20. I'm still working
on that. It will let me build the usual stuff, like "make boot" , but I've
still got some problem where it won't let me do a make bzImage, maybe I
don't have bzip or something installed? I'll keep working on that.

I do not have 1GB of memory, but someone replied that they have 2GB
of physical memory, and had no problems getting it installed.

My DS20 is configured with the following :

AlphaServer DS20 500 MHz

KZPCM == Dual Ultra-SCSI / Tulip ethernet combo

Elsa Gloria graphics adapter/ Linux sees as TVP4020 Permedia2, and
this uses the 3Dlabs Xserver.
