>>> "Kevin Wood" said:
>
> Got a question for you. Has anyone run into the problem of upgrading the
> kernel under Linux 6.0 and having it knock out nfsd. I am running a DP264
> with kernel 2.2.10 with the d-asn patch. The system runs great except that
> if you are not running the Redhat Kernel 2.2.5-16/2.2.5-22, nfsd fails. Are
> there any patches that can fix this or am I stuck with having to use the
> stock kernel? Any information would be greatly appreciated. Thanks

If you assume that the default configuration that you get when you build a
RH 2.2.5-based kernel from their sources is exactly what they shipped and
which works for you, then what you need to do is copy the arch/alpha/defconfig
file to the 2.2.10 sources you are using, and try building *that* kernel,
taking all the defaults during "make config".

Good luck, and let RH know if this does NOT work for you... ;-}
