I'm running a firewall on my ruffian system. I've got two intel
eepro100s. I just load the eepro100 module and it detecs and configures
both (you just have to figure out which one is eth0 and which is eth1, but
after that it always loads them in the same order). I then just used
ipchains and it is working fine, aside from not letting a citrix client on
the inside connect to the server (I'm doing network address translation
and I think that it makes citrix unhappy).
        Try just making both ethernet drivers a module and then loading
them in the same order every time. Each driver should take the first
available eth device, so that the first one that you modprobe is eth0 and
the second is eth1.
