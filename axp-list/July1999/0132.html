pancho@galaxy.lca.uevora.pt wrote:
>
> I am running RH-6.0
>
> X starts well. windowmaker icons come up ... I am only able to open a rxvt
> window. Anything else I type the command or use icons does not come up
> into the screen, but a ps uax shows it is running.

I had the same problem last night. Quick look at a strace output and
error messages made clear that somebody forgot to add a line like that:

none /dev/pts devpts defaults 0 0

to /etc/fstab. With this in place everything started to work all of
sudden. :-) Don't forget to mount /dev/pts if you are just editing.
On a reboot this will happen automatically. If you are rebooting
with an older setup where this line is invalid you may see some
complaints but otherwise this should be harmless.

> I mean, say I want a xterm. Type xterm & and nothing comes into the
> screen.

RH 6.0 is set to really use pseudo-terminals. The only trouble that
they have to be there or else... :-)

   Michal

p.s. there are other ways to achieve the same effect as you may mount
devpts file system explicitely somewhere in your startup files
