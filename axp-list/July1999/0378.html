Todd Henderson wrote:
>
> At the point of reboot, the document
> http://www.digital.com/alphaserver/linux/install_guide.html
> says to:
>
> set bootdef_dev dka300

This is an example; use a device you have. Until you plan to
autoboot you can always specify your boot device explicitely in
a boot command so do not worry about that for a while until you
can boot and until your hard disk really has a boot block.

> set boot_osflags "root=/dev/sda1 bootdevice=sr0"
                                            ^^^
                              Here you say that you boot from CD

> boot dva0
        ^^^^
      But here you attempt to boot from a floppy.
I guess that such inconsistencies may create some confusion.

> It then goes on and says to use swriteboot /dev/sda /boot/bootlx and pico /etc/inittab,
> and now it should go.

This is once you installed everything and you want to make your hard
driver palatable to SRM so you do no have to boot from a floppy or CD.

I guess that one set of names used by SRM and a different set of names
by which Linux refers to the same physical devices does not help here
but that's life.
