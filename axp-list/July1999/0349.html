OK, no one disagrees with my last answer on this. So again, it is not enough
to build a new kernel and modules. The help on masquerading within xconfig
tells you:

 The IP masquerading code will only work if IP forwarding is enabled
 in your kernel; you can do this by saying Y to "/proc
 filesystem support" and "Sysctl support" below and then executing a
 line like

   echo "1" > /proc/sys/net/ipv4/ip_forward

 from a boot time script after the /proc filesystem has been mounted.

and after i did this my kernel did forward :)
With the old 2.0.xx kernel it was possible to compile a kernel with
forwarding. With the 2.2.x you need to enable it by hand (or a boot-script)
as mentioned above.
