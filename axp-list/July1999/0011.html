On Thu, 1 Jul 1999, Champigny, Michael wrote:

> FYI, this pertains to a RH 6.0 install on a Multia/UDB:
>
> I'm having trouble booting off my hard disk after a RH install.
> When I get to MILO I need to enter a boot line but it seems
> there are several boot images in /boot including vmlinuz,
> vmlinux-2.2.5-16, etc.
>
> First question, which is the proper one to use when first
> booting?

/boot/vmlinuz-2.2.5-16 should work OK.
 
> Second, I tried all of them and I get through some of the
> kernel messages, but it eventually stalls with:
>
> Kernel Panic: VFS: unable to mount root fs ....
>
> Does anyone have a clue what I'm doing wrong? Is there
> a kernel image update somewhere that fixes this?

Hmmm.... sounds like you're not passing it the right root filesystem. I
can't remember how RedHat sets it up if you choose the default
installations (workstation or server). The kernel by itself will try to
mount /dev/sda2 if you don't tell it otherwise.

What I would try doing is from MILO, use the ls command to look at each
filesystem on the disk (I think MILO is able to tell you how many
partitions are there). So ls sda5: should give you a listing of sda5.
You should be able to find out which is the root fs this way.
