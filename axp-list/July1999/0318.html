David Geary wrote:
>
>
> I'm using RedHat V6.0 and the tulip driver. Everytime that I boot my
> system
> it comes up in full duplex. How do I force the system to come up half
> duplex all the time?

This is from tulip.c:

/* The possible media types that can be set in options[] are: */
static const char * const medianame[] = {
        "10baseT", "10base2", "AUI", "100baseTx",
        "10baseT-FD", "100baseTx-FD", "100baseT4", "100baseFx",
        "100baseFx-FD", "MII 10baseT", "MII 10baseT-FD", "MII",
        "10baseT(forced)", "MII 100baseTx", "MII 100baseTx-FD",
        "MII 100baseT4",
};

and you address that by an option index (not the greatest interface,
I know). It is is the easiest to do that when a driver is compiled
as a module as this allows you to experiment with different options
without rebooting. You put in your /etc/conf.modules a line like
that:

options tulip options=12

which will use "10baseT(forced)" (check with 'dmesg' after a module
was loaded when in doubt).

There are some other tulip options beyond "options" and it is even
documented somewhere but I do not remember at this moment where. Anybody?

Saying that an observed behaviour seems to be that the driver with a
media forced tries to comply but in a case of failure attempts to
renegotiate and switches to whatever it thinks it negotiated. That
means that the best bet is to have such card connected to a hub which
can negotiate properly a desired connection type. The funny thing
is in the past I had troubles with two Tulip cards connected back
to back with a "null" ethernet cable. They were negotiating 100 full
duplex and had problems after that. Yesterday I hooked up that way
a Tulip card with a Via-Rhine on the other null cable end. Cards
decided to go 100-FD and everything worked like it should. :-)
