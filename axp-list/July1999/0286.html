> and added the moduals for ipmasqadm utilities. I run in my rc.firewall all
> the ipmaq ruleset and add these 2 lines to forward the ports to the internal
> network
>
> ipmasqadm ipportfw -f
> ipmasqadm ipportfw -a -P tcp -L XXX.XXX.XXX.XXX 80 -R 192.168.0.2 80
>
> but when i try going to that ip address in the web browser it will not pull
> up the page in 192.168.0.2. I made sure that the web browser was running on
> the .2 machine. I can access that site from the internal network but from
> outside i get nothing. Ive been reading all the documentation from the
> ipmasq site but nothing explains more than what i already have. I also
> looked into the man pages for ipmasqadm but still i cant figure this out.
> can someone help me out. or atleast point me in the right direction. thank

I am not sure if this helps but as i changed to a 2.2.* kernel on an i*86
Machine i noticed that i had to enable forwarding explicitly.
I did a
 
  cat /proc/sys/net/ipv4/ip_forward

and it gave me a 0
so i did a

  echo 1 > /proc/sys/net/ipv4/ip_forward

and viola the machine did forwarding
Hope this helps
