Hello,

em86 will not work with glibc 2.1, because it uses deprecated functions
(such as getdents) which are not in glibc 2.1 anymore.
So it won't work with RedHat 6.0

The source tarball for em86 is not complete, as you only have a .a for some
parts of the emulator, and of course, these .a do not link with glibc 2.1.
Murphy's law, that is.

So, unless someone at Digital recompiles this part on a glibc 2.1 system
and makes it available, em86 can't run under RedHat 6. There are also a few
changes to do in the sources to make it compile with 2.2 kernels, but most of
them are straightforward.
