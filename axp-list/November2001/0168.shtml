On Thu, Dec 06, 2001 at 10:47:52PM +0100, Reinhard Janutta wrote:
> I think the problem is the abnormal end of installation by anaconda.

This was seen on different occasions

> If there is nothing to boot, You cannot boot.

This is indeed the case but with this "abnormal end" you actually have
the whole system to boot, or at least a substantial chunk of it, and
only a boot block is missing.

In some situations when anaconda did not find a space to put a boot
block (the whole disk may be covered by a partition of "unused" type,
which is legitimate but anaconda is not smart enough to know that) this
is indeed a correct reaction. In other this may be a confusion/bugs.
But in such cases you can boot quite trivially although not from a hard
disk. Use an installation CD as a boot device or a floppy made from an
image on that CD. Nothing says that your root file system has to be on a
boot device. Once you booted you can write that boot block yourself,
provided that there is a space for it, and use that hard disk to boot
from that moment on.
