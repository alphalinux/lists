On Thu, Nov 08, 2001 at 01:07:59PM -0700, Michal Jaegermann wrote:
> On Thu, Nov 08, 2001 at 10:07:49AM -0500, Jay Estabrook wrote:
> >
> > Another solution (not useful for installs): replace the Linux QLogic
> > driver with the one ported from *BSD by Matt Jacob, found at:
> >
> > www.feral.com
>
>
> Not useful for installs from scratch but if this is an update, or you
> have another machine running, then one can make a custom kernel to
> replace one on a boot floppy and boot from that. This should work
> even if this is an extra work and booting is slower. The first floppy
> has a regular ext2 file system on it. The other one is a compressed
> image of a ramdisk.

Be sure that the custom kernel has all the same drivers/options, as
the RH install depends on the kernels having certain things or it
won't work.

A couple of other alternate install-from-scratch scenarios occurred to
me as possibilities:

1. do network install, after booting off the CD - so long as the HDDs
   and the CDROM aren't being accessed simultaneously, it might work
   better.

2. try using the 2.2.x based kernel from RH 7.0, booting it from floppy
   and doing a 7.1 CD install from then on.

Good luck.
