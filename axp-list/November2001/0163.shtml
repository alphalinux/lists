On Thu, Dec 06, 2001 at 08:35:12PM -0600, G.L. Grobe came forth with:
> ... and it went all the way through and didn't throw any exceptions near the
> end w/ anaconda. So now I'm back up and running and getting ready to rebuild
> the kernel (w/ NIC support) so that I can get it on my LAN.
>
hmm.. if you're using the built in card, either the de4x5 or the tulip
driver should get it working (sometimes one is unhappy, and you need to use
the other depending on the chip rev).

> #1. Is there anything different about building/installing a new kernel for
> the alpha's that would be any different from an intel box. Is 2.4.x stable
> enough for this box?
>
if it compiles, yes. (A lot of the 2.4 series don't compile straight off).

don't make bzImage though, make vmlinux (or vmlinuz). Then you add the
kernel to /etc/aboot.conf

> #2. Where'd alphalinux.org goto?
>
up and down. linuxalpha seems to be more reliable.

> #3. What's the difference between 500a and 500au? The front of the box says
> 500a, and 500au is on the console screen. Originally I thought they were 2
> different boxes.
>
They were sold as NT or Unix machines. The NT ones had hardware that doesn't
work with VMS/Tru64. I think the NT ones were cheaper as well (but don't
quote me on that).
