Thanks to all who responded this week and last. I started researching
swriteboot but then a funny thing happened. I added the initrd param so my
SRM line looked like this ...

boot dka0 -fi /kernels/generic.gz initrd=/images/ramdisk.img -fl
"root=/dev/hda expert text"

... and it went all the way through and didn't throw any exceptions near the
end w/ anaconda. So now I'm back up and running and getting ready to rebuild
the kernel (w/ NIC support) so that I can get it on my LAN.

Before when I had a 2.2.13 (I think it was ... from 6.1) kernel running, I
had so many problems (much like this time) that I just didn't trust building
a new one for this PWS500a. I build'em all the time for intel boxes, but now
with all this fresh in my mind, I'd like to put the latest on my alpha now.
btw - I installed 6.2 because my local store had this really messed up RH
box w/ an alpha 6.2 (the only alpha version they had) for like ... cheap.

So I have a few questions:

#1. Is there anything different about building/installing a new kernel for
the alpha's that would be any different from an intel box. Is 2.4.x stable
enough for this box?

#2. Where'd alphalinux.org goto?

#3. What's the difference between 500a and 500au? The front of the box says
500a, and 500au is on the console screen. Originally I thought they were 2
different boxes.
