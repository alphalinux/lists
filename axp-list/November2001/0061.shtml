On Thu, 8 Nov 2001, Peter Watkinson wrote:
> And i've read that Itanium will have it's own instruction set to run
> compatible windows 64bit code not compatible with 32bit instructions for IA
> 32 Pentium processors. However IA 64 will be able to run 32bit code if a bit
> slowly.

That's right.

> However i've heard that AMD Hammer will be able to run 32bit code at native
> 64bit speeds thus offering a temporary advantage over Itanium at least until
> 64bit IA-64 software becomes more widely available.

I don't understand what you mean by "run 32bit code at native 64bit
speeds". Hammer does appear to be much more similar to its x86 heritage
than Itanium; it's 64-bit ISA is a straightforward extension of the IA-32
ISA, and from what I hear it won't be crippled when executing IA-32 code.

> However will Hammer be able to run Itanium native 64 bit code? Itanium is an
> EPIC processor and I don't know what Hammer is. Is Windows 64 code
> compatible across both processors? Seems a bit pointless for Hammer to be
> 64bit if it's only going to be running 32bit software.

No. Hammer and Itanium are completely different processors. They will
have nothing in common except for their IA-32 compatibility.

Hammer's 64-bit ISA is completely new, and in order to take advantage of
it we will need support from operating systems, compilers, and commercial
software. AMD is working on these, or the first two at least.

> The same goes for linux I know that Suse is developing a 64bit linux for
> Hammer will Itanium and Hammer 64bit linux code be compatible?

No, the Linux kernel for Itanium is completely different than that for
Hammer.

> Also will 32bit x86 linux code run on Hammer or even Itanium?

Yes, both should support x86 legacy code.

> After all Compaq Alpha NT code was only compatible with Alphas and x86 code
> had to be binary translated using FX32. Alpha NT native code wouldn't be
> compatible with Hammer or Itanium.

Yep. Everybody has their own 64-bit CPU now... Intel, AMD, Sun, IBM,
Compaq, HP, SGI. They are all different. Perhaps the industry really
will converge on IA-64 as they did for x86... then again, perhaps not.
