On Mon, Nov 19, 2001 at 01:43:01PM -0500, Jemiolo, John wrote:
> you cannot span the entire disk with "c"

Yes, you can. Linux does not care. You do _not need_ that partition
with Linux but if it is there there is no harm done (well, you have only
seven partitions left for other purposes). I have many disks in actual
use to show it. You may _want_ that partition if the same disk will be
ever accessed by something else.

> Start it at the end of the
> previous partition, this is NOT like unix where the "c" is the entire
> disk, here it's the root filesystem so your effectivly writing over your
> boot and swap partitions when the install installs packages in /

Note that 'c' partition you from the original message was of a type
"unused" so the above does not apply.

A version of 'fdisk' as supplied by Red Hat has a nasty habit of dumping
a core if there are traces of an old "FAT style" partition table on a
disk and you attempt to write a new "BSD-like" one. This was reported
quite a few times both to Red Hat and utils-linux maintainer, and
reports included proposed patches, but apparently without results (I did
not check for that problem the latest version utils-linux). Maybe there
are also some other troubles of that sort with "leftovers"? If this is
indeed the case you should be able to work around by overwriting first
two, say, sectors with nulls ('dd if=/dev/zero of=/dev/sda count=2' or
something like that) and writing a new disk label on a "clean slate". I
never had troubles with any variant of 'fdisk' after such sweeping.

If you are trying to fix an existing labelling make careful notes about
partition boundaries. You likely want to restore them and not to
reinstall everything from scratch. :-)
