> When I have LABEL=/ in (stock RH7.1)
> fstab, and boot from milo, df tells me /dev/sda2 is mounted as root, despite
> /dev/sdc3 (the correct one from the kernel param "root=/dev/sdc3") is
> actually mounted (!!).

Something like that will really happpen if in that fstab
you are using (you may have many if you have multiple installations
on the same machine) you have a line which starts with

LABEL=/ / ....

Anaconda indeed puts "/" as a label on a root partition while installing
and if you have multiple partitions with the same label the one which
will be scanned first "wins". OTOH nothing says that a label on a root
partition _has_ to be "/". You may put there "MyOwnRoot", or whatever
else you want and make labels unique, or you may mount by UUID (see 'man
mount' and 'man tune2fs' for adding/replacing labels).

If fstab is messed up what gets really mounted and what gets _reported_
are often not the same thing. I have seen that on numerous occasions
when disks got new positions due to hardware changes but /etc/fstab
was not adjusted yet. On the first "single" boot one has to be really
careful to write where you think you are writing.
