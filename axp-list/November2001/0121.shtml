Hello,
a colleque of mine has some problems installing RH 7.1 on our XP 1000
systems:

We have some serious problems running swriteboot after having
installed a RedHat 7.1 system on a Compaq XP 1000 workstation.

The installation itself was uneventful, we had neither problems to
partition or format the hard disk, nor to install all the packets.
So far so good. But then we have to run swriteboot (because RedHat
doesn't), and this is where our problems begin.

swriteboot says, the disklabel is corrupt, and thus, refuses to run.
fdisks verify option segfaults, as does Compaq's JumpStart cd.
Repeating the whole installation procedure does not help.

The BSD disklabel looks like the following:
a: 2 3 boot
b: 4 ... swap
c: 1 ... unused # whole disk
d: ...

Except for c spanning the whole disk, no two partitions overlap.

As a last remark: We have an identical (wrt hardware) system
which is approx. 6 months newer and which we were able to install
without problems. And although both systems have the same hard disk
model, the geometry recognized differs (1106/255/63, 8678/64/32;
I don't know if this is important to our problem, but you might
want to know).

Thanks in advance for your help.

Carsten Luckmann
