Hello all,

    I'm working on installing Red Hat 7.1 on one of my DEC UDB's (166MHz,
160 megs of RAM, 4 gig external SCSI drive, etc etc)...

    I am however running into some problems...

    I'm using the SRM console to start the installation, and it's failing
miserably every time I try. I've tried RH 6.2 and RH 7.1 and both fail at
the same point...

>>> boot dva0 -fl 0
block 0 of dva0.0.0.0.1 is a valid boot block
reading 164 bytes from dva0.0.0.0.1
bootstrap code read in
base = 154000, image_start = 0, image_bytes = 14800
initializing HWRPB at 2000
initializing page table at 146000
initializing machine state
setting affinity to the primary CPU
jumping to bootstrap code
aboot: Linux/Alpha SRM bootloader version 0.7
aboot: switching to OSF/1 PALcode version 1.35
aboot: booting from device 'DVA 1 0 0 0 0 0 0'
aboot: no disklabel found.
aboot: loading compressed vmlinux.gz

    And here it sits forever... The floppy light is on.. It'll sit here for
an hour if I let it...

    Anyone ever seen this before? I've looked around online, and seen a lot
of stuff about it getting past this (ok, now booting kernel or some such),
but mine always stops right here...

    I've also tried booting directly from the SCSI CDROM, but it fails
too... Usually it doesnt even get to "loading compressed vmlinux.gz"...

    Any assistance would be appreciated...

    I used to use ARC to boot the system, but it seemed like a really
kludgy approach (ARC to load linload.exe, which loads milo, which loads the
kernel)... I like the idea of using SRM to load aboot to load linux...
Seems a lot cleaner of a method...

Thanks,
