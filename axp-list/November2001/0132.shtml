The recent 2.4.15-pre kernels (i cant compile the 2.4.14 out of the box, and
I didnt' try 2.4.13) seem to have a problem with MILO on my PC164. A
"shutdown -r now" brings down the machine, but then I get a flickering
screen rapidly switching between a blank screen and the text window. I have
to reset it to reboot. A "shutdown -h now" works, but when I press
"ALT-CTRL-DEL" after halting to reboot, the same problem begins. The 2.4.12
kernels work, with basically the same config (so I dont think I broke the
config), and I faintly remember having seen this on an older kernel. So I
guess something got messed up.
Has anybody seen this and/or have a fix?

BTW: AFAIK, aren't all PC164s EV56 machines?! The kernel config sets the
compiler flag to -mcpu=EV5 for my PC164, so I have to manually edit
arch/alpha/Makefile each time.
