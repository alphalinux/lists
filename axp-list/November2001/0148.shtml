Applixware version 4.41 on RH7.1 on a UP2000 is not able
to find libjava.so. More specifically, in Applix Words,
when trying to import a M$ Word document, Applix runs
/usr/local/lib/applix/axdata/filters/olefilter/olefilter,
and this program dies with a message that it cannot find
shared library libjava.so, "No such file or directory".

When I try to run .../olefilter manually, it tries to
open /lib/ev5/libjava.so, /lib/libjava.so, /usr/lib/libjava.so,
then errors out--accorsing to strace. I would guess the
top-level Applix process sets an environment variable or
gives an argument or option to olefilter to tell it where
Applix is installed (/usr/local).

I have a /usr/local/lib/applix/jre115/lib/alpha/green_threads/libjava.so,
and it appears to be an ELF LSB shared object, mode 755.

Is there an environment variable or some other configuration
issue that should be fixed to allow olefilter to find the
libraries?
