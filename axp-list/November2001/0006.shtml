On Thu, Nov 01, 2001 at 09:42:59AM +0100, Peter Bech wrote:
> More an d more websites now denies to get accessed with our Netscape 4.72
> browser as it is distributed on RH 6.2 on Alpha.
>
> We cannot upgrade to RH 7.1 and get the new ones because the RH 7.1
> installation does not wotk on AS 800.

Netscape rpms are somewhat "special" in that sense that Netscape is
not really a Linux application and therefore does not actually depend
on any libraries outside of those which are in Tru64-compat package.
Therefore any of those rpms will work on any RH distribution on Alpha
including 5.x (this I know from practice :-) and likely earlier if
they are still around.

The only differences are minor location changes (like /usr/share/doc
vs. /usr/doc) and you can provide a link or two.

>
> 2. Downloadable RPM's for RH 6.2 does not work on RH 6.2

There was a number of updates of 'rpm' itself in the meantime and
in particular rpm-3.0.5-9.5 (I hope that I remember the version right
but it was the last last one in 3.0.5 series) still maintains an old
format of a database and accepts both version 3 and version 4 rpm
headers. Wonder of wonders it will still unpack the latest rpms on
an old 5.2 based system. Those who wanted to use up2date service
would have to switch to rpm-4... and probably because new rpms need
to be compatible with up2date they show up in this format.

Besides it is really quite trival to cut an rpm header and feed the
remainder into gunzip or bunzip2 and pipe that to cpio if you really
do not have anything else handy.
