On Wed, Jun 16, 1999 at 03:55:46AM -0400, Uncle George wrote:
> 1) gettimeofday(), which uses timeval, returns 2 ints from the call to the
> glibc routines.

No it doesn't.

> This is done irrespective of whatever the definition of struct
> timeval is. If u define timeval as long, u get bogus looking values. But as
> int, the numbers look more reasonable.

I don't know what you're doing, but it's screwy.

  #include <sys/time.h>
  #include <stdio.h>
  
  main()
  {
    struct timeval tv;
    gettimeofday (&tv, 0);
    printf ("%ld.%ld\n", tv.tv_sec, tv.tv_usec);
    return 0;
  }
  
Produces a sane looking "929823352.596453" with a 16-byte timeval.
