1) gettimeofday(), which uses timeval, returns 2 ints from the call to the
glibc routines. This is done irrespective of whatever the definition of struct
timeval is. If u define timeval as long, u get bogus looking values. But as
int, the numbers look more reasonable.
2) By the same token, getrusage() uses the rusage struct, which uses struct
timeval. If timeval was defined as 2 ints, like gettimeofday(), then
glibc/kernel trashes stack when getrusage() is called.. If timeval is defined
as 2 longs, then I dont get a trashed stack.

So will the real struct timeval please stand up ? :-)
gat
Again this is for RH6.0 & alpha
