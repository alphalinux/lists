Greetings,

Running Redhat 6.0 on a 164LX system for about 3 weeks now. Attempted an
upgrade to the kernel version linux-2.2.5-22 from linux-2.2.5-16.
Basically used the same .config file to build new kernel. There were a
few new questions, but nothing new was selected.

Under the newer kernel, the system stops in the boot process at the line
"Finding module dependencies". Without interaction, it never makes it
beyond this point. Upon hitting CTRL-C, it continues on to boot at
boot-level 3s. We've also noticed that each time we log in after a
reboot, the date is 2019.

Rebooting under the older 2.2.5-16 kernel, it has no problem with the
"Finding module dependencies", nor have we seen this date problem on
reboot.

Has anyone experienced this boot problem? Is there a solution that you
know of?

Thanks,

David
