Hello,

 Time by time, one of our application go in a sleep state with
 messages like :
... kernel: psd: Exception at [<fffffc0000316b48>] (fffffc0000316b4c)
... kernel: psd: Exception at [<fffffc0000316b58>] (fffffc0000316b5c)
... kernel: psd: Exception at [<fffffc0000316b68>] (fffffc0000316b6c)
in the /var/log/messages.
If we attach the process with gdb and try to step, the process
received a SIGSEGV.
According to the system map, the addresses above are somewhere in :
do_sigreturn, do_rt_sigreturn, setup_sigcontext.
I dont understand the correction of the address ( unaligned ? )
and why the process go in a sleep state rather than receiving a signal.

Info on the system:
LX 533Mhz with 512Mb of memory
linux 2.2.4
glibc 2.0.7 ( RedHat 5.2 )

The application use the pthreads library and consists of one main
thread for computation and one thread for communication.

The problem occurs on different machines ( but same system setup ),
so I dont think it is an hardware pb.

Thanks for any hints, doc, modif in kernel/program to get more
 info, etc..
