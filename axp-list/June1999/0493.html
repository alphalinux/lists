On Thu, Jun 24, 1999 at 02:00:46AM +0900, smurf@cosm.co.jp wrote:
> Rich Payne wrote:
> > Linux has a memory limit of 2Gig currently. This is not an Alpha
> > limitation but rather a Linux limitation.
>
> Though, DS-20 still? have limitation of 512MB...I believe.

        This is a FAQ (should be..); the limitation is in between
        the PCI bus interface(s) and the main memory subsystem.

        The mapping logic allows up to half of total 32-bit PCI
        address space (2GB) be accessed from the system side, and
        the PCI devices to access same size memory from the bus
        side. At the same time the other half of PCI address
        space is addressable by the PCI bus devices -- if some
        device wants to access other devices...

        There are ways to access larger memory spaces from the
        PCI, but none such have been supported yet -- it would
        need a IOMMU to be taken into use. (Term IOMMU was
        introduced by Sun, and this is pretty much analogous
        thing, which is present in all Alpha core logic chipsets.
        It is usually called 'scatter gather mapping' in the
        documentation.)

        With 2.2.* series kernels all systems should now be
        configured to support up to 2GB of RAM.

        Proper support of more than that requires major rework
        in common IO subsystems, including *all* device drivers.
        Yes, we may want to do that some day...
