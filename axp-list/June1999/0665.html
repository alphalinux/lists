I have a large simulation code which I compile under digital unix
and then move the executable to an alpha/linux machine to run.
I frequently exceed the stack size, whose hard limit is hardcoded
to 8MB in include/linux/sched.h (look for _STK_LIM). The comments
written there suggest that this ought to be adequate. Perhaps, but
it depends on how it's used -- do the digital compilers use the stack
differently than gcc under linux would?
