Greetings,

I have observed that trying to compile pcmcia-cs-3.0.13 on my 21064 running
2.2.current and glibc-2.1 from RH6 reaches the stage where it tries to do:

make[1]: Leaving directory `/usr/src/pcmcia-cs-3.0.13/debug-tools'
make[1]: Entering directory `/usr/src/pcmcia-cs-3.0.13/man'
make[1]: Nothing to be done for `all'.
make[1]: Leaving directory `/usr/src/pcmcia-cs-3.0.13/man'
make[1]: Entering directory `/usr/src/pcmcia-cs-3.0.13/etc'
set -e ; make -C cis
make[2]: Entering directory `/usr/src/pcmcia-cs-3.0.13/etc/cis'
../../debug-tools/pack_cis -o 3CCFEM556.dat 3CCFEM556.cis

and then sits forever. It is clearly spinning because the load will hold
at 1. If I break into it with the debugger, I can see that it's looping on
some logic that does a lot of floor() operations. Unfortunately, most of
the code is in lex stuff so I can't make heads or tails of it.

Anyone had success compiling this package on Alpha?
