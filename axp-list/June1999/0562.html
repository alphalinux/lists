I'm having problems with the following configuration:
   DEC Alpha 164SX @ 533MHz
   Matrox Millenium I (4MB)
   WinTV Model 404 (BT878)
   3c509 ethernet
   Red Hat 6.0
   Linux 2.2.6 (dmesg output at bottom of this message)
   xawtv 2.46

The kernel locates the card just fine. xawtv seems to start fine. I'm
getting system lockups at random times though. It seems to happen much easier
when I put xawtv in full-screen mode. I'm running at 1152x864x24-bits. My
only choice when it locks up is to do a big-red-button reset.

Any idea what I should try? I tried serching the archives and tried some
random flags like:
   insmod bttv.o triton1=1
   insmod bttv.o pll=0

These had no effect. Sometimes it runs well when I don't touch the machine.
A remote login by ssh seems to increase the likelyhood of a lockup.

Could this be caused by a DMA or IRQ conflict? I'd love any advice you can
offer. Thanks

BK
