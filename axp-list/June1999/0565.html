mgraffam> Does anyone know whether it is possible to network boot a Multia?

I have a spare diskless, floppyless and keyboardless (bad controller?)
UDB with the (in)famous "heat death" problem. I decided to make it a
firewall for my cable modem. I compiled 2.0.35pl2 from DEC site with
NFSRoot support. I setup bootpd on an intel machine (I couldn't get
bootp from RH5.2 to work on an Alpha) and use SRM via a serial cable
to boot

>>> b -p bootp -fl root=/dev/nfs ewa0

I have a very much striped file system (< 5MB) on the firewall with
only a serial login and an sshd running. I think it should be pretty
secure. I miss watching connection refused messages from tcpd :-).
