> On Thu, 17 Jun 1999, Aakin Nipun Patel wrote:
>
> > Has anyone gotten Xemacs to run under 6.0?
> > I'm having difficulty just getting it to compile straight...

> If it still barfs, I suspect that it's the same problem as 20.4 in that it
> still wants to use COFF stuff (uses src/unexalpha.c) rather than ELF
> (src/unexelf.c). In 20.4, the latter still wasn't quite up to par with
> the regular emacs source, so it bombed on me during compile.

Xemacs 21.1 does not need any patches to compile on alpha, but it will
not build with the latest egcs, it requires egcs 1.0.3. I do not know
how it works with gcc.

I have a precompiled xemacs-21.1 available at
http://ibgwww.colorado.edu/~lessem/xemacs but it is linked against a
Debian potato (glibc 2.1.1) system, so I do not believe it works
properly with RedHat 6.0.
