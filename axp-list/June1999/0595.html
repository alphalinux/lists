I don't know if anyone out there is running with a Cybex Commander (older
KVM switch) but I have had the most horrendous problems with it and my
alpha linux boxen (udb/as200/as400).

History:

- Cybex Commander is an 8-way KVM switch. Keeps a 'psuedo feed' on the
ports so the machine thinks something is still connected.

- Have udb/as200/as400 running NT and DU with Cybex, works fine. All
keyboard/mouse/video functions are normal under SRM/ARC and while in OS.

- Attempted installation of RH5.2 with ARC -> MILO method. System boots
fd0/linload.exe fine, but when MILO starts, erratic results from keyboard.
Under 5.2, keyboard mapping is completely shot. Nothing is mapped
correctly, i.e. typing "b" gives "Q" and "O" gives "t". No logic.
Resetting the switch/powering down/unplug/replug does nothing to rectify.

- Attempted install of RH 6.0 with ARC -> MILO method. System boots fine,
but MILO starting gives erratic results, except unplugging KB plug from
unit and replugging ~occasionally~ causes keyboard mapping to work. On one
occasion, holding down the ALT key rendered correct mapping.

- Booting the installation and kernel from SRM allows installation to run
successfully, but unable to boot the system afterwards. I would prefer to
use SRM to be consistent with the DU machines anyhow, but whatever works.

Can someone shed some light here? The problem seems to definitely with
MILO. The problem is consistently erratic on all machines
(udb/as200/as400). The fact that I can get the 6.0 version occasionally
running and the 5.2 version ~never~ tells me something is different (uh-huh).

Thanks.
