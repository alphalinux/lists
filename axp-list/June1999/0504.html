Dan Morrill wrote:
>
> The only command that I can spot to do this, though, is /usr/sbin/vidmode,
> which rpm claims is part of util-linux. However, it's a broken symlink to
> rdev, which doesn't exist anywhere on my box.

rdev is an Intel specific utility which sets some parameters in boot
images. As Alpha boots not the same was on Intel it does not make any
sense on it. 'vidmode' in particular, which happens to be an alias for
'rdev -v', marks in a boot image what video mode is desired while booting.
VGA firmware on Alpha does not suport switching modes so it is not very
useful there.

>
> And now, to the point: :)
>
> * did someone at Red Hat forget to include rdev with the distro (6.0)
> or did I neglect to install the RPM that has it?

It looks like that the only thing they forgot was a removal of a hanging
link in /usr/sbin. Somebody made a minor boo-boo when writing a spec file.

> * is there some other command I can use to tell the console (not
> running X) to return to text mode?

You are in a text mode but from your description it looks like that
you lost your console fonts (or maybe they are now black on black).
Such accidents were pretty popular few years ago with some broken video
cards and two utilties 'savetextmode' and 'restoretextmode' were writen to
deal with such accidents. They are part of 'svgalib' package and I have
no idea if they were ever ported to Alpha and it the would work there.
Never had a need to try on Alpha. Maybe X server got more adept in
workarounds? Unfortunately 'restoretextmode' requires that a state of
your video was saved on file, with a help of 'savetextmode', __before__
you tried to start X server.

Maybe you can try to reload your console fonts with 'setfont'?
This may even work if you are lucky enough - depending on a state
of confusion on your video card. I gather that 'echo ^O' (that is
control-O, one symbol) does not work not mentioning typing, in blind,
'reset' followed immediately by ^J (not return) in case you happen
to still have raw input?

  Michal
