Hmmm while we are on the subject of printing and Ghostscript in RH6.0... I
have been unable to get pdf2ps to generate anything that my LJ4
understands. The postscript views fine in gv, but when I send it to the
printer it tells me there is an error in the "setpagedevice" command.
Unfortunately I have no postscript experience and can't determine what is
wrong. (When you delete the line it just doesn't work - okay it was a
dumb idea)

Anyway, there was an errata list for this version of GS which appeared to
hint at this problem, and a new version announced.. ghostscript-5.50-3. It
compiled straight off from the RPM (amazing) and I tried to convert my,
now standard, pdf file to ps. Did it work?

It did not.

However, I have managed to get pcl conversion working O.K. So I can now
print this document to the printer in pcl. Maybe this shows that some
bits work, but it clearly indicates that there are quite a few bugs in the
current versions.

Of course this is probably out of place on this list as I guess it is a
cross-platform problem.

Do we have any GS experts in the audience?
