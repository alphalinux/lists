On Sat, Jun 19, 1999 at 09:54:24PM -0700, Richard Henderson wrote:
> The mpg123 build process should be adjusted. It only uses the
> math library to initialize some tables. This could be done at
> compile time rather than at run time.
>
Right, but since mpg123 calculates these tables only once upon startup,
the impact on an overall performance is close to zero.

I've made some optimizations of mpg123 for Alphas some time ago,
the patch is at ftp.park.msu.ru:/distrib/misc/mpg123-alpha.patch.gz
(and srpm at
ftp.park.msu.ru:/distrib/RH-5.2/alpha/contrib/SRPMS/mpg123-59q-3.src.rpm)

With this patch CPU load of mp3 playback dropped from 6-7% to 3.5-4%
on SX164/533MHz.
You can also get some extra speedup on alphas with BWX specifying the
cpu model in CFLAGS (-mcpu=ev56 etc.)

Ivan.
