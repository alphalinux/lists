>From MILO you can view the files system. Type help for a list of commands and
syntax. I think is somthing like

ls or list sda1:/.

>From the output you can tell which is the root partition if you forgot. Now
you can do the boot command

boot fd0:vmlinux.gz root=/dev/<partition>.

After it comes up you copy the kernel from the partition to root. I hate the
new layout and location of the kernel. You will have to go in that dir and
change some symlinks I think. I don't remember what but I think it was the
System.map from the kernel build. You will see a warning at boot time if you
upgrade your kernel.

You can then use the old boot line with the proper root partition specified.
