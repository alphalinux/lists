> > available from BladeEnc's homepage. Looks like we'll finally be able

> Although, in the meantime, lame (music only) has gotten much better
> encoders, so that might do what you're looking for. Back when Blade
> was first asked about, lame had only the default (bad) iso encoder.
Right!
The latest lame version 3.13 produces absolutely OK mp3 music files even at
128kbps! Neither Blade or any other ("free") encoder can compare to it ...
and believe me, I know them all ;)
Besides this, lame is pretty fast. With V3.13 I get about 1.35x speed on my
533MHz SX164 using Compaq's fast math lib (i.e. 1.35 times the speed than
realtime!).
