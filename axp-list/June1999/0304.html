Atsuko Crum wrote:
>
> Hi All,
>
> I'm trying to install Red Hat 6.0 on Personal Workstation 433a. I'm having
> a problem with booting.
>
> 1. When I typed
>
> boot dva0 -file vmlinux.gz -flags "root=dev/fd0 load_ramdisk=1"

Is this SRM or ARC. I am not sure if the 433a can do this from SRM.

> I got
>
> block 0 of dva0.0.0.1 is not a valid boot block
> bootstrap failure
>
> 2. Also "Alpha Addendum Installation Guide" says "Before you can boot your
> MILO diskette, you'll need to add a new boot selection to your ARC
> console." How can I get the boot menu?

type arc at the >>>.
Then there is a prompt to enter the Arc/AlphaBIOS Setup.
In there, is a menu choice that spawns another menu for the boot menu. Sorry,
I don't remember the name now. That is where you put the MILO commands.
If you entered it wrong and MILO started you will be kicked back to the MILO
prompt.
>From there you can type:
boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1

I might have missed a few steps but that is it in a nut shell. I have one of
these running myself.
BTW for sound recompile the recent compiler or 2.2.9 kernel w/ac2 patch. Build
in Soundblaster compatable/ESS: 220 5 1 5 330 -1. For network it would be best
to use tulip drivers. Get the latest from
http://www.bmen.tulane.edu/~siekas/tulip.html. The stock kernel fails to bring
up some DEC cards.
