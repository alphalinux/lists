glenn murphy wrote:
>
> I'm sorry if any of my questions are FAQs. I'm just learning to program,
> and I get a lot of floating point exceptions at runtime with code that
> runs fine on x86 systems. I'm using a 21064 (cabriolet) with RedHat 5.2.
> Is this something specific to the 21064 that I need a compiler option for?
> Is this a problem with egcs 1.0.3 that comes with RHL 5.2? If I upgrade
> egcs, do I need to upgrade all the libs as well? Any help would be
> greatly appreciated.
>

Well, there are a couple things to do. First, run the program
under gdb and see why you get the FP exception. It is quite
likely that you are accessing something you shouldn't be. The
Intels don't show the problem because they force the bad numbers
to zero or FLOATMAX, and happily continue on with the calculation.
Now, sometimes this fixup is quite reasonable to do and so you
let it go but Alphas are a bit more picky in that they expect all
FP values to be normalized. So, it you think the results of your gdb
run/examination are acceptable (meaning you don't want to fix the
code to eliminate the problem) then you might try -mieee to force
the Alphas to slow down and handle the oddball numbers in SW rather
than HW. This still might not work if your FP exception is in a
library routine though, like exp().
