On Thu, 10 Jun 1999, RaymondInFinland wrote:
>Hi,
>
>I was wondering if it is possible to get a TV card working in a
>multia. The kernel supports the BT848 chipset through video4linux so
>the only problem could be the non-standard videocard.
>
>Anyone ever tried this (or has reason to believe that it should or
>should not work?). Any pointers, links, tips and/or help is welcome

The TV card (Hauppauge) works fine in my LX but I use the orginal
bttv-drivers instead of the Kernel ones as that was recommended in the
Kernel documentation. Since it has a very low cpu overhead (viewing,
havn't tried grabbing yet) it should work performance wise in a Multi.
The only problem could be the PCI subsytem. I believe (but I can check
tonight at home) that the Hauppauge ones require PCI 2.1 - I do not know,
which revision the Multia has, probably someone else can comment on that.
