The answer to the comment's question is no. Pre-ev56, loading an
unaligned 16-bit quantity will silently fail if addr & 7 == 7. There
will be no unaligned trap to fix up.

With an ev56, we'll use the ldwu instruction, which does trap,
which is how I noticed.
