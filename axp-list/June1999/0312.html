> > I've been exploring migration over to linux from Digital UNIX. Does linux
> > have something analagous to Digital UNIX's mfs (Memory File System)? Our
> > application makes use of this to avoid I/O and we need something similar.
> >
> > Chris

Actually, I looked into this before some time ago, and the answer is no, if I
read what you are doing correctly. SunOS (and presumably Solaris) have a
similar filesystem called "tmpfs", which isn't really a filesystem, but is a
virtual filesystem that lives solely in memory. The thing about it that's
different from Linux's ramdisk, is that (under SunOS) it shares space with
swap. Linux's ramdisk doesn't work this way. Typically /tmp is mounted with
this filesystem type so it's cleared when the system reboots, and running out
of /tmp space also could crash your machine depending on how well it handles
running out of swap. Naturally, /tmp can also be swapped to disk (since it is
swap), and can be resized with swapon and swapoff. There was a project to do
this under Linux (called swapfs, I think) but it has since been abandoned.
I know of no efforts to resurrect it.
