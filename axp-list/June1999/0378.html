My problem with 2.2.7 (on RH5.2) appeared only when I had postfix and
squid running concurrently. postfix is a high-performance MTA, inducing a
certain amount of forking activity. The well known squid caching software
does it with a single process which takes a lot of memory and
keeps many TCP connections open. Only after I had these two apps
running simultaneously I observed forking problems.
Before, the machine appeared nicely stable...
