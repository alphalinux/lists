> Hey folks...
Hello!

> Anyone have any particularly fast MP3 encoders for linux/alpha that you'd
> suggest? I get about 1.4x realtime encoding with the 8hz-mp3 encoder compiled
> against libcpml...
Hmm ...
MP3 encoding is a question itself because there are currently no real good
"free" encoders (and free is not really true for _all_ of them because if
those redivulous patends held by FHG :(
If you want a fast nearly pure ISO based encoder you should try
  ftp://wopr.campus.luth.se:/pub/mpeg_layer_3/
This one is really fast but only offers the ISO based features (with a start
in joint stereo in the pre releases). And to be more on-topic, Andreas who
did this encoder uses an Alpha for his development ;)
If you want more decent encoder with more features than the ISO based ones
(and 8Hz is one of those) then you should have a look at
  http://www.sulaco.org/mp3/
This is the home of lame which is IMHO currently the best "free" encoder
around. It can handle 128kbps JS mp3 quite well but still by far not as good
as FHG's encoder (you might want to try their Alpha OSF demo version for
comparison).
