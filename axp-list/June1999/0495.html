I'm finally got my spare UDB up and running headless, diskless and
floppyless. I boot linux (2.0.35) from SRM serial console with bootp
(however, I couldn't get RH 5.2 alpha bootpd to work and had to use an
i386 bootpd.) It mounts NFS root OK and inittab starts a serial login
along with sshd. It runs nicely with 16MB memory (no NFS swap). It
will become a firewall for my cable modem.

I get one strange problem, it can't mount another directory (such as
/usr.) I found that rpcinfo didn't return anything, although it works
fine from an Intel machine (The NFS server is another UDB) and works
fine locally. So, it is not an Alpha nor a server setting issue. All
services and exports (protected by tcpd) are open to the whole subnet.

My headless UDB file system is striped to < 10 MB. Maybe I took out
something rpcinfo was looking for? However, strace didn't tell me
that is that case. My network setting seems to be OK. I can ssh in
and telnet out. It didn't makes any difference after I manually added
lo interface and routing.
