On Tue, Jun 29, 1999 at 09:19:42AM -0700, Linus Torvalds wrote:

[upgradable read-locks]

> There can only be one writer (whether he's a "reader-upgrader" or a real
> writer) holding the lock at a time.

Yup, IMHO the critical point that there can be many who _want_ to gain
the write lock (waiting) but only one who _wants_ to "upgrade" at the
same time. This makes the thing extremly dangerous.
Even if there is only on "upgrade" code-line it has to be protected
in some way (including previous the read-lock) if the function is
reentrant. Can this be really useful in some way ?
