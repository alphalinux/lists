Jeff Lessem wrote:
>
> Xemacs 21.1 does not need any patches to compile on alpha, but it will
> not build with the latest egcs,

Out of a sheer curiosity - did you try with '-fno-strict-alias' by any
chance.

I recently tried to recompile on Alpha and with egcs-1.1.2 the latest
version of LyX. It turned out that _one_ of files has to be recompiled
with an optimization off or results are unusable. The problem is
known and noted in INSTALL. A slight tweak on included specs does that
automatically and things are after that you can do even 'rpm -ta ...'
and everything seem to be fine.

No, I did not try yet what an effect an inclusion of '-fno-strict-alias'
would have or if droping down to '-O1' would be enough; '-O2' was
clearly too much. The code is written in C++ and it is really hard
to guess what a compiler may be doing with it.
