>>> Rich Payne said:
> On Sat, 5 Jun 1999, Jeff Bolle wrote:
>
> > This may be a dumb question, but do all the graphics cards that work under
> > x86 XFree86 also work under the alpha version of XFree, and are there any
> > true performance standouts, or does performance pretty much stay to what it
> > is under x86/win98? Thanks.
>
> The issue usually isn't will the card work under XFree86. It's will the
> card get initialized properly by the x86 BIOS emulator in the Alpha
> systems. Chances are that if that part works, then it should be OK in X.

Well, yes, that *is* an important first step... ;-}

However, once the card is initialized, the kernel will only treat it as a
generic VGA, unlike the Xservers which will attempt to use the acceleration
features of the device. And it's the Xserver code which may, or may *not*,
have been ported to Alpha (usually the int/long/pointer stuff, but register
access is also critical).

Now, from the RELNOTES of the 3.3.2 release of XFree86 (28 February 1998),
we have:

The following X servers are available for Alpha hardware:

        Mach 64 server
        Mono server (generic driver only)
        P9000 server
        DEC 21030 (TGA) server
        S3 server
        old S3 ViRGE server (please use SVGA server)
        SVGA server (Matrox Millennium driver only)

Unfortunately, later releases don't have the same RELNOTES format, and this
information is missing. It is also now out of date, as a number of other
servers do work on Alpha just fine, I believe:

        SVGA on Alpha now supports Matrox Millennium G200, Trident(?), Tseng(?)
        Number Nine Imagine 128 (I,II, and IV, maybe Revolution 3D?)
        3DLabs Permedia 2 and 2v

Red Hat 6.0 includes the following RPMs:

XFree86-3DLabs-3.3.3.1-49.alpha.rpm
XFree86-FBDev-3.3.3.1-49.alpha.rpm
XFree86-Mach64-3.3.3.1-49.alpha.rpm
XFree86-Mono-3.3.3.1-49.alpha.rpm
XFree86-P9000-3.3.3.1-49.alpha.rpm
XFree86-S3-3.3.3.1-49.alpha.rpm
XFree86-S3V-3.3.3.1-49.alpha.rpm
XFree86-SVGA-3.3.3.1-49.alpha.rpm
XFree86-TGA-3.3.3.1-49.alpha.rpm

And a SuSE 6.1 beta has:

xi128.rpm: Server for Number Nine Imagine 128 graphic cards
xmach64.rpm: Mach64 Server
xmono.rpm: X-monochrome server
xp9k.rpm: Accelerated server for P9000-based cards
xs3.rpm: server for S3-based cards (excluding ViRGE and ViRGE/VX)
xs3v.rpm: server for S3 (ViRGE and ViRGE/VX)-based cards
xsvga.rpm: server for svga cards
xbanshee.rpm XFCom Server for Voodoo Banshee based cards
xfbdev.rpm: X server for the vesafb device in recent Linux kernels (2.1.119 or
             newer)
xglint.rpm: Accelerated server for GLINT/PERMEDIA/PERMEDIA-2 based cards

(Sorry, I couldn't figure out where corresponding info was for DEBIAN).

The only things I'm not sure of are:

1. the set of ATI cards that are supported by the Mach64 server, as these
   may all work on the Alpha because they started with a known working base.

2. the additional cards supported under the SVGA server that work on Alpha.

Perhaps someone else with more direct experience in these areas can help?
