> The Tsunami\Typhoon method of
> using a segmented chipset of CIA (System controller), DSW (Data slice) and
> PCI chipsets means better flexibility for system board design but jacks up
> the price quite a bit and makes the implementation at lot more costly.

However, AMD is moving to precisely that model with Hype[rd]Transport,
and API just announced a chip that links to Hype[rd]Transport. So
apparently the AMD folks think they can control the cost, and API is
interested in doing that. What you will get on the low end is an open
question, though, since your low-cost wishlist has some features that
low-cost x86 boxes typically don't have.

> EV7 should
> integrate more of the system chipset on die (I think this is limited to the
> memory controller, which I've heard is being "strategically" redisigned for
> support of DDR SDRAM, not just the DRDRAM/RDRAM that they had originally
> planned), but I'm not certain if that will include I/O bus support, like the
> high speed serial Infiniband (though it may).

Both Infiniband and Hype[rd]Transport are as suitable for putting on
the cpu die as the RAMBUS controllers that Compaq has been promising
for EV7.
