After managing a mostly-successful install of redhat 7.0 on a 433au system
(getting X running required doing a little homework), a co-worker of mine
has been using it quite happily. I noticed that it seemed a bit pokey,
though. I then found that the Alpha System Type was set to generic.
I'm assuming that the correct type should be "Miata"? However, upon
reboot into a kernel compiled for miata, I get a

"Keyboard timeout - AT keyboard not present"

and

"hda: lost interrupt"

warnings. I then have to ssh in from another machine to reboot back to
the generic kernel as the system won't respond to the keyboard. I'm not
sure what is going on here. 433au is a miata, right? I've got a feeling
I've missed something pretty obvious... The system is certainly usable
under the generic configuration (and I'm not the one using it anyway),
but I'm just curious as to what's going on here.
