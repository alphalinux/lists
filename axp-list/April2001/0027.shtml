Tony,

You've stated my original "prospective" requirements in summary very well.
Like me, you included the desired support for OpenVMS and Tru64.
Unfortunately, the previously mentioned issues of supported hardware is a
major hurdle to building a system that could support Tru64 and OpenVMS as
easily as Linux and even xBSD.

>some PCI slots to run dual monitors, SCSI, etc. ( is 66MHz, 64bit PCI out
of
>the question? )

Apparently, the AMD 761 Iron Gate can't do 66 MHz PCI (at least not from
what I've read), though I suppose you could implement an external PCI bridge
that could do the 64 bit/66 MHz. IBM makes an excellent PCI set just for
the purpose, but it would be a bit messy and cost more than our target
$1,500-$1,900 price range would allow. Dual peer PCI buses would be very
nice with full 64 bit/66 (or faster support), but its only going to happen
if someone like VIA comes up with a single chip low cost chipset that has
dual PCI bus and/or 66MHz support built in. The Tsunami\Typhoon method of
using a segmented chipset of CIA (System controller), DSW (Data slice) and
PCI chipsets means better flexibility for system board design but jacks up
the price quite a bit and makes the implementation at lot more costly.
Alcor/Alcor II and Pyxis both integrated the PCI and memory controllers to
reduce costs but only supported a single 33 MHz PCI bus. EV7 should
integrate more of the system chipset on die (I think this is limited to the
memory controller, which I've heard is being "strategically" redisigned for
support of DDR SDRAM, not just the DRDRAM/RDRAM that they had originally
planned), but I'm not certain if that will include I/O bus support, like the
high speed serial Infiniband (though it may).
