> Thanks for all the help on my password problem. Setting the os flags to
> single worked to get me to an aboot prompt. I still have not made the
disks
> I need to boot from aboot and change the root password but am confident
that
> after doing so this problem will have been resolved. I'm sure I will have
> more questions around optimization of this server for database processing
> after I have a chance to poke around in it and see what my options are.
>
if you can get to aboot, you don't need to make disks. From aboot you can
specify what kernel you want to boot, and the options. check the help, but
I think list will give the list of kernels, and then boot /boot/vmlinuz
root=/dev/sda1 ro 1
(change /boot/vmlinuz to the right kernel, and root to the right root
device).

You can basically take one of the predone lines and add 1 to the end (to get
single user mode). Then edit /etc/shadow :)
