On 2001-04-18 19:10 Uncle George wrote:
> I'm sorry that u believe this but i believe that the results from
>
> -O0 == -O1 == -O2 == -O3 == -O4
>
> should all be the same.

Well, I am not sure if I do right rebutting this claim but the simplest
proof of your misunderstanding of the problem is the following program:
#include <stdio.h>
int main()
{
  float f = 1.0/3.0;
  printf("%20.10f\n", f+f+f);
  return 0;
}

compiled with
gcc t.c
and
gcc -O t.c

(actually, the compiler should not matter). Straightforward compilation
without compilation will produce result different from 1 due to rounding
errors; gcc -O will detect that it can get rid of the addition and the
result will be equal to 1 - as expected(?).

The code is correct when it follows the standard, not matter how it is
interpreted. if the code depends on the interpretation, the code _is_
broken. I have seen quite many broken Fortran programs written by
chemists that believed they knew the language so I think I know what I
am talking about. if you don't care about standards, don't be suprized
that you waste your time finding errors in compilers.

just my 0.02zlp
