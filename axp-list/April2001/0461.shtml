Dear axp list:
        I've recently migrated from MILO to SRM on my LX164 system. I was
please to find out that I didn't need to backup all my disk and reformat
using BSD disklabes, but rather through a posting on one of the axp lists,
I was able to change the labels to be BSDish.

        My problem now is getting SRM to work with my recently configured
and compiled 2.4.2 kernel. I'm able to boot up easily using SRM and the
stock 2.2.17-14 kernel supplied with RH7. However, when I try my 2.4.2
kernel, I end up with a message like (going from memory):

sda: unknown partition types

        When I was using MILO, the 2.4.2 kernel booted fine. So, I know
that there is no problem with the kernel per se. I suspect that I'm not
setting a particular kernel option.

        Here are the relevant options I'm using. I can submit my .config
file if someone thinks this is more useful. I don't include it now simply
for length.

        CONFIG_ALPHA_LX164=y (I know generic may be good here too)
        CONFIG_ALPHA_SRM=y
        CONFIG_BSD_DISKLABEL=y (do I need UFS support too -- not sure why)
        CONFIG_ALPHA_LEGACY_START_ADDRESS is not set

        Are there other options that I should be setting? Again, the
2.4.2 kernel worked under MILO and the 2.2.17-14 kernel boots from SRM.
So, I feel I'm safe to assume the BSD disk labeling is correct and that
the 2.4.2 is at least "good" to run.

        Any help is appreciated.

Sincerely...Tony Leclerc
