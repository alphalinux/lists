Sorry, thats all that i know. I tried to use ccc, but was not able to get it
to go without coredumping. That is until I installed the bin-utils, which was
ment to fix something in cxx . Then things worked.

If u still have missing symbols, maybe one of the *.o 's from the GNU crowd is
around. I think u should also link using ccc/cxx to make sure that that the
'proper' ccc libs are included.

I would reinstall the pieces ( with -U --force ) ( libots for the proper cpu
( ev5 should work everywhere ), libcpml, ccc, cxx, and then bin utils ).
rebuild ur src from scratch. There is not much more that I can help with.
I'd suggest the 'help' from the university of NH, but their website has been
moved, or changed. :-{
