> Getting the rsh commands to work with my other
> machines always seems to be a problem after an upgrade.

This distribution started to use xinetd instead of inetd. Various
servers are configured in /etc/xinetd.d/ and by default are turned _off_
for security. You want something on then you have explicitely edit a
corresponding configuration file. Try to type 'chkconfig --list' and at
the bottom you will see a list of services which go through xinetd and
their status. Type 'man xinetd.conf' if things are not obvious.
