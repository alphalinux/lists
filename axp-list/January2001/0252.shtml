This is a general question, but it is running on one of my alphas :)
I run a script that checks for people over quota. Generally I'm looking
for mailbombs and the like. (repquota -a | grep +)

The script turns up a user "1328" which doesn't really exist. There is no
directory in the user directory that is owned by anyone with that user
number. Doing a bit of digging, it appears that there was a user 1328 but
this user was deleted some time ago (before I admined the box even) So
there is no way to find the user name.

Thinking that it was a dir that wasn't deleted when the user account was
removed I have scoured the user dirs to no avail. Wrote a perl script
that did a du -s on every user directory and list those over quota. It
matches the repquota EXCEPT for mysterious user 1328. Wrote another
script that searched the /home/users directory for anyfile that was
owned by a uid of 1328, nothing.

So how do I get that darn thing out of my quota report? Do I just ignore
it or is a bug with repquota? If it's a repquota bug who do I report it
to.
