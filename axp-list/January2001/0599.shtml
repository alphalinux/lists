> >
> > You miss the point, Mike: It's increasinly frustrating, that
> > most ports to
> > Linux goes to the Intel only world.
>
> I don't think I do. I've been down this road before.
> FX!32 was an awesome engineering effort. There's
> no arguement there. I had many friends that worked
> on it. HOWEVER, it kept Microsoft and others from
> porting natively to Alpha. Why bother when "I think
> it'll run under FX!32?". Instead, all the energy that
> could have gone to helping PORT apps, went into making
> FX!32 run them.
>
> I see that as unfortunate. Linux gives Alpha the chance

<flamebait>
Oh, quit complaining fellas.

Soon enough Intel will make REAL 64 bit CPUs, and code will get written
finally.
Then we can stop fooling around and get with the program and run Windows
software like God intended us to!

</flamebait>

OK, so the above was a joke, right?

No, actually it wasn't.
If you want "mainstream" software then buy "mainstream" hardware.

You didn't buy an Alpha to run Word on, nor to play Quake on. You bought it
for some very specific purposes.

So, if it does not do things outside of that list of specific purposes, is
it such a big problem?

After all, you can certainly buy a cheap PC and run everything else on
that, right?
And as nearly none of that "other" software will gain much if anything from
the superior floating point and memory bandwidth, you will not need an
Alpha for it, right?
Think of an Athlon, P3, or P4 computer as a "compatibility emulator device
for Linux Alpha" and it makes it a much different proposition. At around
$1000 to $2000 it is not even proportionally expensive, is it?

You don't buy a Ferrari to pick up groceries in.
And you don't complain that it does not seat a family of 6..

Just my $.02 worth, anyway..
