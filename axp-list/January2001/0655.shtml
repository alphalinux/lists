> On Wed, 14 Feb 2001, Michal Jaegermann wrote:

> > gcj is great but it is not written in Java. :-)

> You want the compiler written in Java? Interesting idea.

No, I would like to see some serious, really useful, code
written in Java and performing in a decent manner. Not necessarily
compiler. I only said that gcj cannot serve as a shining example
of a Java code as it is nothing of that kind.

I do not negate that some interesting stuff did show up in Java,
although Boehm garbage collector was available for Objective-C,
and succesfully used, long before Java showed up. Also Perl, Python
and Ruby manage memory pretty well - thank you very much.
I was quite interested in Java at the very beginng, and even bought
some books about it, :-) but I am still waiting for Java to come
close to its promisses, and waiting, and waiting ...

As for Objective-C then at least the whole NeXTSTEP interface was
written in it. Not a huge commercial success but quite sizeable
chunk of a real code which mostly worked as advertised.
