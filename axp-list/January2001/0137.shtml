On Tue, Jan 09, 2001 at 12:19:09PM -0500, SteveCraft wrote:
>
> I went to a rawhide distro site, and grabbed
>
> rpm-4.0-4.4.alpha.rpm
> rpm-build-4.0-4.4.alpha.rpm
> rpm-devel-4.0-4.4.alpha.rpm
> rpm-python-4.0-4.4.alpha.rpm
> rpm2html-1.5-2.alpha.rpm
> rpmfind-1.6-2.alpha.rpm
> rpmlint-0.25-1.noarch.rpm
>
> but when I did a rpm -U rpm-4.0-4.4.alpha.rpm My RH 6.2 install said

DON'T DO THAT unless you are updating a _number_ of other things at the
same time and you _really_ know what you are doing. You have here a
potential to screw up your rpm database badly enough so it will be
recoverable only from a backup.

> only packages with major numbers <= 3 are supported by this version of RPM

To handle also version 4 packages update to the latest version of
rpm from updates to 6.2 and _leave it there_! Just make sure that
you run 'rpm --rebuilddb' right away after you run an update. It is
supposed to that in '%post' action but just in case; it will do no
harm.

BTW - there is also 'up2date' in updates to 6.2 and a version of rpm
able to handle version 4 packages in updates to 5.x.

> .. so how do I update my RPM?

See above. RPM is trying hard to prevent you from shooting yourself
in a foot but you can still do it if you really want to.

> There are more than a few treats I want to
> try out..

Keep in mind that some stuff binary stuff may have dependencies
on libraries which you do not have. Quite often 'rpm --rebuild ...'
is good enough.
