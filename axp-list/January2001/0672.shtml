Aaron,

>My PC164 keeps spitting messages out like the following

>"eth0: Transmit timed out: status 0050 0c00 at xxx/xxx command 000c0000."

Check your hub/switch, cabling and if possible use a loopback to test tone.
Failing that, inspect your Ethernet controller for metal shavings, loose
connectors or foreign materials in the RJ45 socket (sounds dumb, but you'd
be surprised).

What model of the Intel EtherExpress Pro 100 do you have? The chip will
have a designation (much like the Digital network products had "21040,
21140, 21141, 21143x), these are as follows for 10/100 products: 82556 (EE
100A) 82557 (EE 100B) 82558
(EE 100C) 822559 (EE 100+)

The first Intel EtherExpress 10/100, the "100A" has a firmware upgrade which
may fix your problem, if you can find it still. It was in the driver update
disk, so you may need a PC to load it. For later models, the firmware is
either not upgradeable or not user upgradeable (no flash tool is supplied
and no EEPROM image updates are provided, except for some of the SMART and
Server NICs which have different board layout and ASIC engines).

>ALSO as an aside, I'm looking for 2 sticks of 32mb true parity ram for the
>same machine. Anyone have any used ram that I could take off their hands?

Go to pricewatch.com and pick up some 72pin lead/tin modules for about $25
each. Note that you will need at LEAST 4 modules per bank to build the
128bit memory bus in order to boot. It's much more rewarding to fill both
banks of 4 with 8 identical modules and turn on 2 way bank interleave for a
256bit bus. It's noticeably faster (and a lot more fun). This will also put
you at 256MB of RAM using 32MB modules, not a bad workstation.
