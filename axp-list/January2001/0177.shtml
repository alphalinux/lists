>I was hoping that you might lead me in the right direction on a little
>problem. Without getting into the details of why, I was wondering if you
>knew of a way to run a command like pwconv on a file other than
>/etc/passwd (and the same for grpconv). I'm looking to convert a password
>file into a shadow password file, but want this done in the background
>(rather than disturbing /etc/passwd) until it's completely finished (to make
>sure that no one has access to the (cleartext) file until it is done and
>shadowed).

In the worst case, you could run pwconv in a chroot sandbox. Afaik, here's
what you need to place in the sandbox to do that:

   etc/
      group
      passwd
   lib/
      libcrypt.so.1, libc.so.6, and ld-linux.so.2,
      or whatever is reported by ldd /usr/sbin/pwconv.
   usr/
      sbin/
         pwconv
         grpconv

Then you can do:

   chroot /path/to/sandbox usr/sbin/pwconv
   chroot /path/to/sandbox usr/sbin/grpconv

        --Robby
