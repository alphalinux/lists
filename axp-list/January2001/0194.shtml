I'm trying to get Squeak (Smalltalk) to run on Alpha Linux, and
it's doing some strange things, so I'm seeking suggestions.

------------

In one sandbox, I'm trying to build from sources. After working
around a few issues in the source, and after following Michal
Jaegermann's excellent advice on how to get a "ld" that takes
-taso (and how to recover from a big mistake), I have "ld"
building a binary. However, "ld" gives a warning:

    ld: warning: cannot find entry symbol __start; defaulting to 0000000012013ae0

Then, when I try to run the binary, I get "command not found"
errors. The file exists and has the 'x' bit set. However, even
gdb refuses to run it.

What is this missing __start? Any suggestions to make the binary
actually run?

------------

In another sandbox, I'm trying to install the OSF binary, for
alphaev56-dec-osf4.0e, and it almost works. (I have the Tru64
compatibility package.) When I run the binary, the window comes
up, but the fonts are mangled, and there seem to be off-by-one
errors in pixel addressing (slivers left behind when moving
windows, etc.).

I have a UP2000 (ev67). Should this be able to run the ev56 OSF
binary? Any suggestions for fixing the font and pixel addressing
issues?

------------

I may build a third sandbox, trying to run the X86-Linux binary
under em86. I imagine that will be fun. :-)
