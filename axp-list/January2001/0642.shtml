Ugh. Please don't associate the Java language with crapplets. Nobody
takes buggy browser VMs and bloated applets seriously anymore.
Netscape/Microsoft succeeded in killing that off.

> > I don't think anybody would
> > suggest the you just recompile/rewrite Java applications because the
> > will run faster in native C code.
>
> Very frequently infinitely faster and that is where the fun begins... :-)

Not so. GCC now includes a Java frontend, gcj, that shares a compiler
backend and optimizers with GNU C/C++. Quality of the generated code is
comparable to other compilers. You can run most non-GUI Java programs
fully compiled for Alpha, much, much faster than with the JDK.
