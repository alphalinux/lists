Hello !
On Wed, Jan 31, 2001 at 10:26:46PM -0500, John Jasen wrote:
> On Wed, 31 Jan 2001, Speedracer wrote:
>
> > Whats the best way to do this in SRM? I currently have my SRM set up so
> > all I have to do is type boot. Any one else had any upgrade expereince
> > with 2.4.1?
>
> P00>>boot -fi partition#/path/to/vmlinuz-2.4.1 -fl "any boot flags that
> need to be added"
>
> for example:
>
> P00>>boot -fi 1/boot/vmlinuz-2.4.0 -fl "root=/dev/sda1 max_scsi_luns=16"

It might be easier to add this line once to your
/etc/aboot.conf
So you can easily "create" a safe boot option (your current one,
probably already 0: in /etc/aboot.conf) as well as several
experimental ones (like the 2.4.1). Of course you can still boot
explicitly as shown above.

See the docs on www.alphalinux.org for details about aboot and
aboot.conf.
