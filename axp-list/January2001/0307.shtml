Now, I'm new to incorporating command-line arguments into my programs,
but this looks bad. With the updated GCC (gcc-2.96-69), when I try the
following simple program:

/* --- */
#include <stdio.h>

void main(int argc, char *argv[])
{
  int i;
  for(i = 1; i < 20 ; i++)
     printf("%s\n",argv[i]);
}
/* --- */

I get the following output:

--- This has been edited, but it looks like the info is correct. ---
(null)
PWD=

LESSOPEN=
KDEDIR=

malicious user.

also lists HOME=, TERM=, and PATH= before segfaulting), or is this kind
of bad?
