> Use "boot dqa0 -fl 0 "
> The command your using is for 6.2, it changed for 7.0

It really does not matter. '-fl 0' means only that if you are booting
from SRM then options from a line labelled with '0' in /etc/aboot.conf
on this floppy will be used. Nothing prevents you from typing them
explicitely although '-fl 0' gives you less opportunity for mistyping. :-)

As a matter of fact, if you need, you may take this floppy, mount it on
any system which can handle floppies with ext2 file system on them and
with an editor add up to nine other "private" boot commands to this
/etc/aboot.conf (these commands are labelled by single digits hence the
limit). Then you can boot "b dqa0 -fl 3", for example. Labels do not
have to be consecutive or in order. They are just one character labels.

If you boot "b dqa0 -fl i" then you will have an opportunity to review
all defined boot lines before choosing one.
