Hi
   I am cuurently responding to a tendor to supply a number of high
performance systems to a government agency in the UK.One of the manditory
requirments of the OR is
 to provide a means of initiating a telnet session from a unix host and
tying that session down to a pty port at system startup.This is to allow
data collection from remotelab equipment,connected to serial ports on telnet
based terminal servers.Currently this reverse telnet functionality is not
included with TRU64,as Red Hat Linux is supported on Alpha I'm wonering if
this functionality can be provided on Linux.

origional request :-

                                "We have had a few abortive attempts in the
past to replicate the DGUX
                                "pseudo terminal connectivity" functions on
AIX and Digital Unix.
                                On the DGUX hosts this allows the system to
be configured to make a "reverse
                                telnet call" from the host to the IP address
of a port on a tcpip terminal
                                server i.e. the reverse of the normal
process where a terminal server makes
                                the connection to the IP address of the
host. This is usually established at
                                boot time or application startup and then
is used as if it is a serial
                                device connected to the hosts e.g.
/dev/ptty/01. This provides 2 way serial
                                communication to any device connected to the
terminal server port, usually a
                                laboratory analyser.
                                For various reasons we would like to attempt
to do this again. Howvever
                                ,given past history where the non DG host
purchased for the lab application
                                had to be removed and a DG host put in its
place we need to be sure that any
                                new proposal works. Most of the existing
lab hosts are DG 88K hsost which
                                need to be replaced in the near future and
so there is an opportunity for
                                those who can demonstrate such a solution."
