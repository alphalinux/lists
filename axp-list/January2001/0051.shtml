X starts up and works as expected at boot time
on my UP1000 running RedHat 7.0 with all recent
updates and with a patched
2.2.17 kernel. However, if X is brought down,
it cannot be restarted. XConfigurator works at
this point and brings up the "can you see me" display,
but still X cannot be restarted with either "init 5"
or "startx".

Error messages include something about gpm
being mysteriously murdered, and when gpm is
removed:

Could not init font path element unix/:7100, removing from list!

Fatal server error:
could not open default font 'fixed'

This only happens on my UP1000 running a Permedia 2 video card.
Intel and PC164 machines appear to be fine with a variety
of other video cards.

Has anyone else seen this?
