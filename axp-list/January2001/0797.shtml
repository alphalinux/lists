On Mon, 26 Feb 2001, Helge Kreutzmann wrote:

: > #include<math.h>
: > int main(void)
: > {
: > double x;
: > x=exp(-744.769);
: > x/=2.0;
: > printf("%g\n",x);
: > return(0);
: > }
: >
: > Compilation with gcc and linking with libm from glibc-2.2.1 gives a
: > floating point exception immediately. With compaq math library or
: > compaq C compiler it is not so easy but it is also possible to get a
: > FPE, one has to play with numbers. Why is it so?
: >

How was this compiled? With gcc you have to specify -mieee explicitly both
when compiling and linking. With ccc you use -ieee, but -mieee is mapped
to -ieee.
