On Sun, Feb 04, 2001 at 09:00:56PM -0500, John Jasen wrote:
>
> gcc-2.96 is not suited for kernel, nor development work. The C++ APIs will
> not be forward or backward compatible.

Furthermore, libstdc++ bootstrapped on *any* system that is running
glibc 2.2 will not be compatible with a system running glibc 2.1, even
if the compiler version is the same. The ABI will also be forward
compatbile, as applictions built on Red Hat Linux 7.x will continue to
run on Red Hat Linux 8.x through compatibility libraries.
