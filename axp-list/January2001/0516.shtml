Maurice Hilarius wrote:

I wrote:

>>1. Has anyone done any kind of performance measurements like this?
>> I mean in both raw number crunching capability, and disk I/O
>> capacity? I know that buffered I/O in *NIX depends a lot on CPU
>> performance, but some seems smarter at it than others.
>Yes, what specific test results do you want to see?

Well, our tasks are primarily large FFTs andsolving of very large
matrices. In between, there will be access to lots of data on disk.
It's typical of what a lot of people who need supercomputers for.

>>2. Would it be easy to run Tru64 UNIX on one of the UPXX00 or CS20
>> platforms?
>Very, if you have some understanding and experience with the hardware.
>Certain part selections must be strictly adhered to for Tru64
>compatibility. SRM version is important too.

Well, given the limited expandability of these systems, especially the
CS20, I'll probably be using the hardware that comes with it, and maybe
adding some kind of fast network interface. We need to be able to deal
with virtual addresses space of greater than 2GB, and my impression of
LINUX is that the big memory feature is not straighforward to achieve
nor stable. I hope that Tru64 UNIX can work with whatever API puts
into these systems.
