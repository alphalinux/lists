On Wed, 14 Feb 2001, Michal Jaegermann wrote:
> > Ugh. Please don't associate the Java language with crapplets. Nobody
> > takes buggy browser VMs and bloated applets seriously anymore.
>
> Maybe. I am waiting to get convinced. In words of Linus: "Show me the
> code".

http://gcc.gnu.org/cgi-bin/cvsweb.cgi/gcc/gcc/java/

Is that really what Linus meant? ;)

> Over many years I see from Java proponents mostly hot air and
> legal squables. From the very beginning it looked to me that Java is
> just a badly rewritten Objective-C, mostly to make a false impression on
> C++ programmers that they know what is going on, plus tons of a
> marketting crap.

If you s/Java/Sun/g I mostly agree. I don't care much about the business
side, and I don't care about Sun. Luckily they can't fully control
Java. Now that they've dealt with Microsoft, there are various free
implementations against which they have no recourse. Some of these have
improved on Sun's mistakes.

Objective-C was a good language too, but doomed to obscurity. Java
actually improved on Obj-C in a number of ways, for example strings as
objects, automatic memory management, and multithread support. I'm a
convert on the first two... still undecided about threads :\

Consider bytecode. OK, portable execution and interpreted runtimes are
not such a great idea. But bytecode also provides something the C++ guys
should be envious of: the equivalent of precompiled headers. You can
think of jar archives as large repositories of type information. For
that, Java bytecode is a big win.

> gcj is great but it is not written in Java. :-)

You want the compiler written in Java? Interesting idea. Actually its a
nice language for abstract data types, and there are plenty of those in a
compiler.

But the FSF has a tradition of using plain old C for free software. Plus
you have a bootstrap problem if the compiler is Java. Not unsolvable, but
what do you gain? (I've heard that part of GNAT, the GNU Ada compiler,
is written in Ada. So there is a precedent for it.)

> It is not that I do not appreciate efforts of Blackdown guys, and Uncle
> George in particular. Far from it. But troubles he got in, and mostly
> not of a technical nature, did nothing to elevate Java's image in my eyes.

Again, s/Java/Sun/ and I agree.

branched this week (so it'll be released "real soon now") and will
completely support Alpha/Linux. In the meantime anyone can try it by
downloading a snapshot dated 2/1 or later. (I could post binaries
somewhere too.)
