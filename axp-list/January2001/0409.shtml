the qlogicfc.o module is in the 2.2.16-3 /lib/modules tree, so I'd guess
that it would be recognised on install -- if not, after you install to
another hdd (which you'd almost have to, since SRM doesn't see the
device!), you can modprobe -v qlogicfc, and you may/may not have to cd
/dev; ./MAKEDEV

alternatively, adding 'alias scsi_host_adapter0 qlogicfc' to
/etc/modules.conf should autolaod it.

I think I have the syntax right ... I depend rather heavily on the fibre
stuff at work, and thus it gets added into the kernel.
