On Thu, 18 Jan 2001, John McGrath wrote:

> Hello:
>
> I am the newest of newbies and am in the throes of installing Alpha
> Deluxe on my XL300, I am still trying to decipher the documentation.

A nice little machine, I have one myself.
 
> Here are the assumptions I have made based on my computer, please
> correct or advise as you see fit.
> -devices are: fr0 = floppy;

fd0 = floppy 0

sr0 = scsi CD; sda(1,2,3,5) Harddrive with 3
> fat volumes (why 4 sda volumes?)

sda is the entire disk, sda1 would be the first partition on the disk.

> -Because I have an XLT machine milo is the only way to boot to linux. No
> Arc nor SRM!

Correct.

> -RH don't support the XLT installation.
> -I can not boot off Disk 1 of the Alpha Deluxe package

Umm.....why not? Is this RedHat 6.2 or 7.0?

> Here's what I have run into so far
> When I boot using linload into milo I can egt to the milo prompts and
> get all the device info. When I ask to boot the kernel on the CD using
> something like
>
> boot sr0:/kernels/vmlinuz.gz root=/dev/sr0

If this is 7.0 then try:
boot sr0:/kernels/vmlinux.gz initrd=/images/initrd.img

7.0 uses an initial ramdisk (initrd image).

> My intent is to start the Kernel from the CD using the CD's boot blocks.
> I get an error quite a way into the boot with the following error:
> "kernel panic VFS unable to mount root fs on 00:00" (what does this
> mean in lay terms? what does it want from me?)
> I tried using a couple of different milo's some generic others specific
> to XLT.

This isn't a MILO problem.
