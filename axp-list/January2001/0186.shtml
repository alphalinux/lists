On Wed, 10 Jan 2001, Michal Jaegermann wrote:

> > On the Tru64 alpha system, I get great timings for both user and system, with
> > very few page faults:
> >
> > 257.027u 3.176s 4:29.85 96.4% 0+1336k 101+18143io 94pf+0w
> >
> > On the alpha Linux system, though, the user times are comparable, but
> > system and wall are miserable:
> >
> > 256.284u 209.641s 7:46.35 99.9% 0+0k 0+0io 54820pf+0w
>
> Which Linux kernel version is that, how much of virtual memory you
> are using and what percent of a real memory is that? All of these
> can be significant. From what you show you have really high
> page fault count under Linux on your Alpha. This is pretty costly
> and your whole performance difference is in a system time.
>
> Michal
>

This machine has 2 GB of real memory and I am using about 80 MB for this
calculation. This behavior is exhibited under both 2.2.17 and 2.4.0
kernels.

I do not trust the system time at all, as the PS from my original post
indicated. In my experience with other OS's, a high system always leads
to poor % utilization with these kinds of codes. Another poster here has
indicated that system time is not accounted correctly on alpha linux systems.
In addition, I note that the same calculation on the Intel Linux system
(whose times I gave in the original post) also had a large number of page
faults, but the associated system time (and wall time) were reasonable.
Hence, I do not believe that the large number of page faults and the
reported system time on the alpha Linux system are necessarily linked.
