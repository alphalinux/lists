2.2.10 and my crond keeps dying.

http://www.gnu.org/software/mailman/mailman.html

Below is what I've pieced together from the logs. The trend seems to be
that a python script is run from mailman's crontab, an unaligned trap
shows up in the message log and crond dies. I've since removed the
gate_news entry from mailman's crontab but that didn't fix the problem.
Crond seems to die at random every ~24 - ~48 hours. I also dont
understand why a failure in something called by crond would cause it to
die...
