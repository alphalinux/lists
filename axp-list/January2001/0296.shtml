I encountered the same problem when I tried to install it on the
Linux-Alpha machine. Since we have an ancient DEC 3000 running DU
4.0A tucked away in our network closet cranking out some of our network
services, I installed it on that first, then copied the whole
/usr/local/matlab6 directory to our Linux box. For the record, both the
install program and MATLAB 6 script complain about the OS being too old on
the DU box as well, and the whole program dies on startup with similar
errors about unresolved symbols.

It appears to me that MATLAB 6 requires the Tru64 5 libraries to run,
which apparently are not the ones provided with the Tru64-compat
package. Perhaps one day Mathworks will decide to build Linux binaries of
MATLAB for architectures other than x86.
