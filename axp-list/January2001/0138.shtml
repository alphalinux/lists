On Tue, Jan 09, 2001 at 10:49:41AM +0100, Georg Wittig wrote:
> jeff.sturm@appnet.com wrote:
>
>
> > Does anyone know if the following is valid C:
> >
> > extern int a;
> > extern int b;
> >
> > void f(int x) {
> > (x ? a : b) = 1;
> > }
>
>
> (OT, because this question has nothing to do with axp)
>
> Don't know if it's valid C.

No, it is not. gcc has a number of extentsions. In particular it
will accept expressions as "lvalues" ("lvalue", in short, is something
which can show up on the left hand side of an assignment).
'info gcc' and try to find 'Lvalues' under 'C Extensions'.

> But remember you can always write this as
> * (x? &a : &b) = 1;

This should have the same problem as the above but if you will expand
it to
        int *p = (x? &a : &b) ; *p = 1;
which is really the same thing (and probably will compile to the
same code as the original '(x ? a : b) = 1;'), and is valid C.

What is wrong with

void f(int x) {
    if (0 != x)
        a = 1;
    else
        b = 1;
}

You can even squish that into a one line if you really want to. :-)
BTW, 'if(x)' will earn you moanings from gcc; you would have to write
'if((x))' if you are (mis)saving on an explicit test.
