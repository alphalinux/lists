I don't know if this will help much, but I experienced much the same
problems when installing RH 6.2 on an AS 255 machine with an S3 Trio64
(86c764/65 chipset). Xconfigurator test would work fine, but the screen
would go blank and lock up the machine whenever an X session was
started. At the advice of a post found through seaching the axp-list
archives, I added the lines:
        Option "no_pci_disconnect"
        Option "sw_cursor"
under the "Device" section in XF86Config, and the X server worked ever
since. This may or may not work for the S3 968, I don't know. Another
option suggested by the man pages and such for S3 lockups is
"nolinear", and "nomemaccess" if all else fails. Good luck.
