I do hope that you are not considering that seriously as this is an
obvious voodoo junk which happens to work for some by an accident.
Working around faulty DNS configurations with a help of such ruses may,
or may not, work depending on how fast and where it will get rejected
by nameservers but surely it generates a lot of spurious net traffic.
Probably putting there "search my.funny.invention" would at least
be funny. As far as I can see "netscapeonline.com" domain does not
exist.

BTW - my /etc/resolv.conf file, which is in reality written by my ISP
via dhcp client (it rewrites is every time when it starts), does not have
any bogus search lines like the one above and Netscape works like it
should.
