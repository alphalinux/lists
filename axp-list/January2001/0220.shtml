Externalize the temporary memory array to allow the application to
allocate it and save/destroy it as needed. Other high performance
libraries I've used do this. That way it's the app-writer's problem
to figure out whether to malloc() it on each call or malloc() it
once and use it repeatedly. Just need to document how much memory
is required based on the input sizes.
