On Wed, Feb 07, 2001 at 10:43:32AM +0100, Stefan Schroepfer wrote:
> lello wrote:
> >
> > ..., but when I pass to milo the
> > command line
> >
> > boot -t iso9660 hdb:kernels/vmlinux.gz root=/dev/hdb
> >
> > Warning: unable to open an initial console
> > Kernel panic: No init found. Try to pass init= option to kernel.
>
> Booting from CD isn't supported from within
> Milo. Create boot floppies instead.

Well, it is "not supported" but it works. :-) The problem is that
that command line is wrong. Either you have to load initrd (only some
recent milos will work with that option) or to use 'load_ramdisk=1'
appended to the command above with 'root=/dev/f0' for a good measure.
In the later case you need a floppy made from 'ramdisk.img' and you
will be prompted for it.

BTW - '-t iso9660' usually you can skip; most likely a file system
type will be recognized anyway.
