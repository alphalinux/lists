Hey all -

        I have an old 533mhz LX164 system and I need some help. It came
from DCG with 2 IBM drive (4.5g and 9.1g), and I have recently added a 18g
WD Enterprise for more space and an OS upgrade. However, the WD drive
does not like my alpha. I'm running the latest AlphaBIOS I can get my hands
on (5.60), the latest BIOS for my SCSI Controller (Symbios 4.18.00), and
the latest MILO I can get my hands on (2.2-17). Drive is configured in
SE mode, along with all the other devices. There is an active
terminator on the bus. This is the same bus and cable with the two
other drives on it. I have command tag queueing turned off for that
host in the SCSI bios. Now, here's the interesting part: the BIOS sees,
formats, and passes the drive for bad block checks (with no bad blocks
found). The AlphaBIOS sees the drive. MILO sees the drive, but when it
does a READ CAPACITY - it times out, then returns the correct capacity.
Then when it reads the partition table, it times out and then returns
the partition table. After MILO boots Linux, the kernel just keeps
resetting the SCSI controller until the end of time. Never getting past
READ CAPACITY. If I take the same controller, same drives, and same
SCSI cable and put them in a x86 based PC - everything works fine with
no timeouts. Thus is my delemma. Anyone have any ideas on how I can go
about getting this to work???
