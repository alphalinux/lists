It seems that we have a similar setup for installing RedHat Alpha on Multias. I do
use Slackware on my x86, though :-)

Anyhow, the way I was able to net-install RH (5.2, 6.0 and 6.2) on the Multia was
through using the x86 machine with an apache server pointing to the CDROM with
RedHat on. When attempting to use ftp I always got the error you describe, but at
least in 5.2 and 6.0 it was also related to a bad tulip driver when using the
10Base2 port (BNC). Now I finally got an inverted cable to use on the 10BaseT port
(RJ45) so I don't know if it is still the case for 6.2 (a transmitter on the
AUI port, converting to BNC was what I used then).

Also, you should make sure that the httpd server (apache?) is actually working and
providing access to the CDROM where you expect it. This might seem as an insult to
you, but something I at some times didn't check when I should have and it didn't
actually work (having only one monitor and switching back and forth helps with the
confusion:-)

And finally, do you know that during the RedHat installation you can switch
virtual terminals (F1 through F4 I believe) and from there see better information
about what is going on during the installation. That will help you debug your
installproblems.

I don't know if any of this was news, but those were what came to my mind when I
read your email.

Have fun,

Peter
