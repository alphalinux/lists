"T. Daniel Crawford" <crawdad@vt.edu> writes:

> Iprobe: The RPMs available at the Compaq site are geared to RH6.0. When I
> run the iprobe binary, it fails and reports an ioctl error. I also tried
> to compile the source myself, but there are numerous header errors and
> incompatibilities to current Linux distributions. From past discussion of
> iprobe on this list, it seems to be the method of choice, if only I could
> get it to work.

Are you trying to compile iprobe under 2.4.0? In that case, you at
least have to change the ipr_fops structure in
./driver/src/linux/driver_linux.c as the file_operations structure has
changed sometime between 2.2.x and 2.4.0 (the owner field has been
added to the beginning of the structure). I tried iprobe with an early
2.4.0-test kernel and changed the ipr_fops structure to:

struct file_operations ipr_fops = {
    NULL, /* owner */
    NULL, /* llseek */
    NULL, /* read */
    NULL, /* write */
    NULL, /* readdir */
    NULL, /* Poll */
    ipr_ioctl, /* ioctl */
    NULL, /* mmap */
    ipr_open, /* just a selector for the real open */
    NULL, /* Flush */
    ipr_close, /* release */
    NULL, /* fsync */
    NULL, /* fasync */
    NULL, /* lock */
    NULL, /* readv */
    NULL, /* writev */
};

(maybe the owner should be set to something more sensible)

IIRC this was the only change I made, to get iprobe working on an
upgraded RH6.1 system.
