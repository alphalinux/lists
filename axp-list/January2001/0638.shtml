The idea of running a binary for a foreign architecture on a system is
not not new and is in use all over the place.
SCO and Linux support old xenix binaries.
Java runs in a virtual machine.

In the case of the x86 there is a built in emulator so there a
reasonable level of performance.

Java is targeted to a psudocode machine. nobody seems to have a problem
with emulating the java virtual machine. I don't think anybody would
suggest the you just recompile/rewrite Java applications because the
will run faster in native C code.

There are 2 general approaches to solving the problem of how to run a
binary on a foreign system. Hardware emulation or recompilation.

Hardware emulation is in some ways the easiest and most reliable but
suffers greatly from performance issues. The X86 strange register
configuration and instruction set are difficult to emulate in a 64 bit
RISC CPU. Well at least efficiently.

Recompilation can be far more efficient since there is an opportunity to
evaluate the code and to optimize it so that it does not require a full
register emulation of the original hardware. This is a far from trivial
task and can in some cases become impossible if you have self modifying
code.
About 20 years ago I was involved in a research project to evaluate the
possibility of recompiling 8086 code to run on a mainframe class
computer system.
In other areas work appears to be underway to recompile java binaries so
that they can run natively instead of having to blindly run an
instruction code interpreter.

There are some systems that will do a combination of interpretation and
recompilation or incremental compilation. Transmeta's new CPU seems to
be one of these along with a number of Java runtimes.

FX!32 from what I have read is more of a recompilation system using this
technology it should be possible to take an x86 binary and convert it
into a binary that will run natively on the Alpha.

It would be interesting to see if we can beat up on Compaq to release
the FX!32 core technology into the Linux community. It would be
interesting to take the FX!32 core software and build a wine like
windows converter and being able to recompile X86 linux binaries would
be useful.
