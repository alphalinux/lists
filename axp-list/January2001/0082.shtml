Hi all,

I have a UP2000 (dual EV67) with RH7.0 installed.
No problem compiling the linux-2.4.0 kernel apart from the
usual warning messages. The compiler is gcc-2.96.

However, it systematically refuses to boot! The kernel was configured for the
DP264 system since there is no special choice for UP??00.
(The reason I chose DP264 is because I was pointed out to do so
in a previous discussion of the list.)

The SRM does indeed load the kernel but after a while the system crashes
without responding at all! What I get eventually (if I recall correctly)
is a panic message: kernell not being able to kill init. (???)
Furthermore, I tested it for both legacy and 'new' start addresses.

The same message I used to get for all linux-2.4.0-test* kernels,
but not for 2.2.x ones.

Could this be because of the SCSI controllers? I have two:

Adaptec AHA-2940U2/W / 7890
and
Symbios Logic Inc. (formerly NCR) 53c896

Any help appreciated.

Nikos
