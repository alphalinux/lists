SteveCraft wrote:
>
> I know my filesystems are fragmented. Is there a preferred AXP tool out
> there to re-order the disk?

In an ext2 filesystem "fragmentation" is normal. It is not the same
thing as the phenomenon with the same name seen on cruder platforms
such as Windows.

The ext2 filesystem is designed to automatically keep the blocks of
a file near each other. It will never become fragmented the way a
Windows disk will unless you defeat the mechanism by setting minfree
to zero and running the disk near full. A "fragment" is a subset of
a filesystem block that is allocated at the very end of a file. The
fragmentation percentage reported by fsck is talking about these ends
of files and does not indicate a performance problem.

The Windows filesystem allows blocks of a file to be allocated all
over the disk, resulting in bad performance. Since the filesystem is
too dumb to order file blocks sensibly, third-party defrag utilities
have arisen to patch over the problem.

Filesystem fragmentation, like shared DLL's, executable email, and
the "registry" are Microsoft design flaws. Microsoft refuses to even
acknowledge these flaws. This is one of the reasons I use Unix rather
than Windows.
