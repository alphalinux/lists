Peter Bech wrote:
> This is the Alpha list. We, the Alpha users, are missing a lot of applications
> that are solely available for Intel Linux. I'm for instance searching for a CAD
> package.

So port one. Can't find any free CAD software? Build it. No time? Buy it.
Can't buy it for Alpha? Forget it.

Whenever you buy software priced for volume shipment (i.e. anything with fewer
than six zeros), you are on the vendor's terms. You buy the software on the
platforms they support.

It sucks, but that's part of the reason free software is picking up steam:
Users want to do whatever they wish with the code.

> Therfore wine cold be an option, but even wine is not able to run on Alpha
> Linux because it doesnt emulate the x86 code, it emulates the windows run-time
> environment.

FX!32 might be the best emulator ever, but it still sucks. There's no
substitute for native execution.

And don't forget that your commercial software vendor won't support you if you
run an emulator.

> We are a lot of serious businesses, that are very willing to pay for good
> applications. We did not enter the Linux world because of the free software.

Then why exactly do you use Linux?

> We see the problem as "the hesitance of porting to Linux because the Linux
> community resists to pay for software". If a market doesn't provide a business,
> there is no market.

I see the problem as normal market tendencies. Even Linux on Intel doesn't have
critical mass. Why do you think Quake for Linux was pulled from retail
channels?

Of course, if Windows came with the full complement of software found on just
about any Linux distribution, even that market wouldn't be nearly what it is
today. Imagine if Linux distributions came with a kernel, shell and window
system, and everything else was "sold separately".
