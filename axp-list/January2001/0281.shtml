On Thu, Jan 18, 2001 at 10:16:45PM -0500, John McGrath wrote:
> Rich Payne wrote:
> > If this is 7.0 then try:
> > boot sr0:/kernels/vmlinux.gz initrd=/images/initrd.img
> >
> > 7.0 uses an initial ramdisk (initrd image).
>
> I had the same result as before the "kernel panic VFS as mentioned previously.

The above boot command will work, or not, depending on a version
of a milo you have. If it does not work then make a ramdisk floppy
by "bit-banging" initrd.img to a floppy (dd on a Unix/Linux machine,
or rawrite on MS-DOS and derivatives) and try to boot like this

  boot sr0:/kernels/vmlinux.gz root=/dev/fd0 load_ramdisk=1

or

  boot sr0:/kernels/vmlinux.gz root=/dev/fd0 load_ramdisk=1 text

if a graphics interface gives you troubles. At least on one Alpha
I did install RH 7 using a command like above. You will be explicitely
asked for floppy with initrd by a message on your screen.

  Michal
