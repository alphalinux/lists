Yes, this is normal emacs behavior. Emacs uses the mv command first and
then creates the new file. This does change ownership, and protections
as well as kill off links.

To change it, in your ".emacs" file add:

        (setq backup-by-copying 1)

or eliminate the creation of <filename>~ backup files add:
        (set make-backup-files nil)

Note that in Tru64 5.1 (maybe even in 5.0A, I forget) this default behavior
of emacs was changed to deal with the Context Dependent Symbolic Links
(CDSL) found in TruCluster 5.1. They also switched from using GNU emacs
to using Xemacs.
