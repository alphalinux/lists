Is there a system call that fills a variable of type
struct hwrpb_struct (asm/hwrpb.h) so that I can find
the processor frequency from the user-land?

I do know that cat /proc/cpuinfo prints out, among
other things, the processor frequency. I do not feel
very comfortable grep'ing for the string
"cycle frequency [Hz]" and picking up the value after
the ":" simply because the string "cycle frequency [Hz]"
could change to something else in future. There is a
lesser chance of the structure member change in
hwrpb_struct.
