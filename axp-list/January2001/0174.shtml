Thanks to some advice from posters to this list, I have ported some
computationally intensive Fortran code to my alpha Linux system (XP-1000,
RH7.0). While the code gives correct output, my recent benchmarks indicate
a substantial difference in the code's performance on alpha Linux vs. Tru64
systems with nearly identical hardware (both using the same optimization
flags with Compaq compilers). I would like to profile the code using
gprof or iprobe, but neither of these seems to work:

Gprof: I can compile the code with -pg, and a gmon.out is produced,
but the subsequent execution of "gprof <binary>" gives the error: gprof:
gmon.out file is missing call-graph data. I know from my perusal of the
archives at alphalinux.org that gprof has had serious problems in the past,
but most of the discussion appears to be out-of-date.

Iprobe: The RPMs available at the Compaq site are geared to RH6.0. When I
run the iprobe binary, it fails and reports an ioctl error. I also tried
to compile the source myself, but there are numerous header errors and
incompatibilities to current Linux distributions. From past discussion of
iprobe on this list, it seems to be the method of choice, if only I could
get it to work.

If anyone has dealt with these problems and had more success than I, I
would appreciate your input.

Thanks,

-Daniel

P.S. Can anyone explain the current state of system time accounting on
alpha systems? Part of my problem involves very large system times
relative to reasonable user times, and the elapsed time is pretty darned
close to the sum of the two, with nearly 99% utilization. This is rather
confusing.
