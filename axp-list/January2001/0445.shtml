Most likely from ftp://gatekeeper.dec.com/pub/DEC/Linux-Alpha/Milo
but this was years ago so I do not remember. If the size is the
same then 'ldmilo' is probably also the same. The date may got
reset in some moment. 'md5sum' gives b7147b6a28d6e2005e2a9eca2f06f508
on it.

> The milos and ldmilo from RH5.1 work fine with that and earlier
> distributions but for some reason (something to do with the size of
> the kernel) won't install RH6.2 and later.

I thought that I have seen quite a few things on Alphas but maybe
not enough. :-) Something does not compute here, in my experience.

If what you are saying is correct then use any kernel you can load to
install any distribution you want and later make yourself a custom
kernel, small enough, to work with your loader. Replacing a kernel on a
boot floppy is quite simple operation if you have an access to any
machine which can write on floppies with ext2 file system. Even some
'Windows' boxes, with proper utilities installed, fit the bill.

Besides is it quite likely that with 5.x root/boot pair of floppies you
can install any 6.x distro (not RH 7, though, as 'rpm' has different
underlying formats). I was installing "wrong" versions that way
although I do not remember now exactly which particular ones.
