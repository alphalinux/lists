Thanks Stefan, (and others replying earlyer)

I get an error message '...rev. 16 FAT' and 'Unknown file system'. 'read
fd0: 0' gives me all zero's; I would have expected at least a few strings in
sector 0.

I'm beginning to think that it is purely MILO doing strange things. I have
to boot MILO under ARC 4.59, that goes fine both from floppy and dos
partition on sda. The MILO from the RH 7.0 Alpha CD 1 announces itself as
Compiled with gcc 2.96. experimental and references to a SuSE 2.2.16 kernel
(from a RH CD!!!!!). I have several other MILO's but there is no difference
in behaviour.

Under MILO I can 'read' and 'ls' my sda partitions (also the DOS partition
om sda8 which holds MILO and LINLOAD.EXE) but not the floppy. It does not
matter if I put in a DOS floppy or a floppy which was mkfs-ed under RH 6.2
on the same XL266

As soon as I am have RH 6.2 (that still works fine) loaded I can mount the
floppy without any problem regardless if there is a DOS or ext2 floppy in
the drive.

I do see that the floppy is recognized as a 2.88 Mb device. Might that be
the problem? Maybe MILO cannot handle that? I use 1.44 formatted floppies
ofcause. But again, both ARC and Linux 2.2.14 (RH 6.2 Zoot) have no problems
with it.

Any idea what can go wrong here?
