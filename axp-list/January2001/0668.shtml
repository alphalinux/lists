   I had wrong dma/dma16 settings (dma=0 & dma16=4?) but
   build-in ES1887 still worked! I just started to wonder why
   sound woes when there was some heavy/sudden load
   on machine; obvious cause was wrong dma-settings, but
   it was still interesting that ES1887 worked at all?!!
 
   Now I have this line in "conf.modules"
 
      options sb io=0x220 irq=5 dma=1 dma16=5 mpu_io=0x330
 
   Although "dma16" isn't stricktly necessary (if "dma"'s
   value is right? look at line ~82 on file "sb_card.c")
   I still added it as an "warning example".
   
   Also "/proc/dma" seems to have right values now:
 
     1: SoundBlaster8
     4: cascade
     5: SoundBlaster16
