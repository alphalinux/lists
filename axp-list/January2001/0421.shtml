Actually they'll be moving up to the next level of integration for this
system, so
rather than building up from the chip level as they did with the Alpha
microprocessor
in the T3D and T3E, they'll use the API NetWorks CS20 server as the basic
building
block for the Cray SuperCluster. You can think of the SuperCluster as a T3E
reimplemented with commercial off-the-she (COTS) technologies. So rather
than
develop their own network fabric and switches as they did with the T3E,
they'll
use Myrinet-2000 -- similarly with other components of the system.

In addition, to selecting and integrating best of breed components which are
available
on the market and in the open source community, they'll also be bringing
over pieces
of functionality from their UNICOS/mk system for things like global
checkpoint & restart
(allows you to save that 512 processor job that's been running for 2 months
so you can
use the machine for something else), job scheduling, resource management,
etc.
