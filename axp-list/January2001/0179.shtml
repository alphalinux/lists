I am hitting a lot more page faults under Linux system than under DU, but
it's not clear to me what the impact of this should be on the final timings.
On the Tru64 alpha system, I get great timings for both user and system, with
very few page faults:

257.027u 3.176s 4:29.85 96.4% 0+1336k 101+18143io 94pf+0w

On the alpha Linux system, though, the user times are comparable, but
system and wall are miserable:

256.284u 209.641s 7:46.35 99.9% 0+0k 0+0io 54820pf+0w

For completeness, here's the timing on my intel Linux system (2xPIII 733),
where I still get lots of page faults, but totally reasonable system times:

692.650u 15.360s 12:11.23 96.8% 0+0k 0+0io 37477pf+0w

I am using Compaq's math library, and this code is *heavily* dependent on
the BLAS DGEMM. Perhaps I should try to build the ATLAS project's BLAS on
this alpha?
