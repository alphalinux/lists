>
> my scsi cdrom drive apparently doesnt like cd-r media.

It looks that way. This is not unheard of although scsi is not
a factor here. :-)

>
> im actually using redhat 6.2, which may be part of the problem.

Not likely but did you check updates? Boot images in particular?

> last night i seemed to read a few fairly nasty things about the redhat
> 6.x series on alpha, they all seem to have suggested the best redhat
> for alpha is 5.2.

There are folks, usually without a trace of a clue, which will say
nasty things about mostly anything. Actually 5.2 is obsolete and
6.2 + updates is very solid on Alpha. 6.0 was a junk but this is
a different story.

> on top of that, redhat seems to be fine with seeing the
> scsi hard drive, so it really should be able to dump the entire cd 10
> or 11 times onto the thing if it needed.

If it can read that. This seems to be a problem. There are drives
really sensitive to media and I have seen batches of CD-R media which
were simply bad. We (i.e. Hard Data) write quite a bit of that stuff
so I know the trouble first hand. :-)

> i had noticed a few odd errors while booting any sort of floppy on the
> multia. last night i also read that the flimsy little floppy cable on
> these machines tends to produce a lot of noise.

Possible. I did a number of installs on multias but this was a long
time ago. I expect that their hardware does not improve with age.
They run pretty hot inside of this small case. A heat corrosion on
a connector providing a power to mobo was quite popular incident.
