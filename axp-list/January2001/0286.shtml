Hello all,

This year, Mathworks has generously provided us with their latest greatest
version of MATLAB (R12, AKA MATLAB 6.0) with our license renewal. Our
primary number-crunching beast is a PC164-LX system with the standard
21164 Alpha processor, running Red Hat 7.0 and the 2.2.18 kernel.

Up to this point, MATLAB 5.3 has been running great, using the
Tru64-compat package from Red Hat. However, MATLAB 6.0 does
not run at all! The exact errors are shown below (run with the '-alpha
-nojvm' options):
14611:/usr/local/matlab6/bin/alpha/matlab: : Error: Unresolved symbol in
/usr/local/matlab6/bin/alpha/matlab: __cxx_call_static_dtors
14611:/usr/local/matlab6/bin/alpha/matlab: : Fatal Error: this executable
has unresolvable symbols

The installation guide says that the Alpha binaries for MATLAB 6 requires
Tru64 UNIX 5.0 (4.0f from the Mathworks website), which I suspect is the
problem. We are using the Tru64-compat-1.0-6 package -- which versions of
Tru64 does this provide compatiblility for? -- and that particular symbol
doesn't seem to appear in any of the supplied libraries.

Has anyone had any success getting MATLAB 6 to work with Linux-Alpha? If
so, are the libraries from Tru64 5.0 required (ie, do we need to purchase
a Tru64 UNIX 5.0 license to run MATLAB on Linux), or is there some trick
to getting it to work with the current compatibility libraries? Any
thoughts or advice will be greatly appreciated.
