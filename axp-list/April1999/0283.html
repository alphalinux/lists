On Sun, 11 Apr 1999, Wai-Sun Chia wrote:

> I got this too on my RH5.2 machine (Alcor) and my Debian 2.1 machine
> (Avanti). Same telnet problem, dies with 1st. character! This is
> immediately after upgrading to 2.2.5-ac6. Basically the entire session
> hangs after that.

This problem is here since 2.2.5-ac2. A quick solution is to 'un-patch'
the file linux/drivers/char/n_tty.c (that means either remove the patch
for this file from the original patch-2.2.5-ac* before patching the kernel
source tree or extract it and do a 'patch -R' after).

I guess the right solution would be correcting the 'down_trylock' function
in linux/include/asm-alpha/semaphore.h (the patch for this has already
been suggested on the kernel list, but is yet untested).
