JE> WRT the Mach64 Xserver not working on JENSEN: this is because JENSEN needs
JE> special code in order for an Xserver to access its framebuffer memory, and
JE> that code has only been provided for the S3 server. If curious, take a look
JE> at the S3-related code in any of the latest XFree86 releases. [...]

>> I was unable to boot it up with this card. As I was saying I don't even
>> see the srm console promp, because the screen "running" if the card is
>> in the machine. I don't know if it is Jensen specific or not, but I can
>> use it with an old 256K vga card.

In fact, the Jensen seems to run fine with most old 8 Bit VGA boards.

JE> This problem is because the JENSEN console (at least the ones I'm familiar
JE> with) did not have a BIOS emulator, but had specific code for the (small
JE> number of) supported video options, among which, obviously, was NOT the
JE> Mach64. Upgrading to the latest SRM firmware *may* help with the card, but
JE> may break other things, making it impossible to boot LINUX. At least, this
JE> is what I remember from axp-list postings; anyone care to comment?

At least, the initialisation code of later SRM firmware versions than 1.7 does
not work with the ELSA Winner 1000 (S3 928) in EISA mode, ISA mode seems to
work, as well as both of them with ARC and NT.

There might be a small chance of getting this card to run, but I doubt that it
would be more useful than any other VGA as long as nobody does the necessary
modifications of the Mach64 X server.
