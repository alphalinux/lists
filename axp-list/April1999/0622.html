> > What does it mean: "Jensen only supports S3 based card under X?" AFAIK,

WRT the Mach64 Xserver not working on JENSEN: this is because JENSEN needs
special code in order for an Xserver to access its framebuffer memory, and
that code has only been provided for the S3 server. If curious, take a look
at the S3-related code in any of the latest XFree86 releases.

(For the technically-minded, it's because JENSEN has no DENSE access to
bus memory, and must use *only* SPARSE accesses, which cannot be done
transparently).

> I was unable to boot it up with this card. As I was saying I don't even
> see the srm console promp, because the screen "running" if the card is
> in the machine. I don't know if it is Jensen specific or not, but I can
> use it with an old 256K vga card. And again as I know X only supports S3
> based cards in the Jensen because of the EISA bus. Correct me if I'm
> wrong.

This problem is because the JENSEN console (at least the ones I'm familiar
with) did not have a BIOS emulator, but had specific code for the (small
number of) supported video options, among which, obviously, was NOT the
Mach64. Upgrading to the latest SRM firmware *may* help with the card, but
may break other things, making it impossible to boot LINUX. At least, this
is what I remember from axp-list postings; anyone care to comment?
