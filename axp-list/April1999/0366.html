On Tue, 13 Apr 1999, Shaw Terwilliger wrote:

> As far as I know, the state of the Mozilla code is so bad that it
> was never even close to being 64-bit clean, and all Digital Unix
> binaries were hacks from Digital's compiler run with switches to use
> 32-bit data types inside.

Sure, the old Mozilla was, but I've been building and running the new Mozilla
code (i.e. NGLayout and "XP" new world order) for months, with nary a hitch.
Not even an unaligned trap. Remember that this is the codebase which is also
going to have to run on UltraSPARC 64 and probably Merced, so they need it
64-bit clean before then anyway.

They've actually done a fairly decent job of keeping things cross-platform.
