On Thu, 15 Apr 1999, Tom Browder wrote:

> It would be nice if one or more of the following were true regarding
> the instructions that come with a kernel tar ball:
>
> 1. Tell alpha users what make command to use ("make boot" versus "make
> vmlinux").
> 2. Have the configuration script set Makefile so the same kernel-making
> command is used for all (the config script identifies the system).

 Actually `make boot' works both for Alpha and ia32 resulting in an
appropriate image, i.e. vmlinux.gz for Alpha and zImage for ia32. I
haven't checked other architectures but I suppose the same is also true
for them.
