As far as I know, the state of the Mozilla code is so bad that it
was never even close to being 64-bit clean, and all Digital Unix
binaries were hacks from Digital's compiler run with switches to use
32-bit data types inside. Those options would still not conceivably
accept "different" assembly; it's possible they did this assembly
hack (and many, many more like it for many, many more platforms) but
filed it away in a proprietary branch they weren't willing to open.
