J> [...] What is on-the-fly-tar? I am assuming that it is a
J> combination of tar and rsh, but don't really know how to do it.

Nope, some FTPds will automatically tar a directory up if you get the
notional tar file, eg if you want to get a directory called /dev as a
tar file, cd to the directory immediately above, and `get dev.tar'.
YOu should see a file in your download directory called `dev.tar',
which will have permissions, maj/minor device nos, etc. preserved.
This may or may not work with an imaginary .gz extension as well --
some FTPds support auto-compression too. Usually there's a README
file somewhere on the FTP server which explains what it supports.
