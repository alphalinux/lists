> > > I am having a hard time compiling a new kernel for my Multia.(RH5.2 W/
> > > 2.0.35)... As per my other 2 Intel machines, make Xconfig, make dep and make
> > > clean work as they should. But when I make zdisk or make zImage it returns a
> > > single line with "*stop" at the end (I am not close enough to the machine to
> > > give a verbose listing)
> >
> > First off, don't use make xconfig on this kernel on Alpha. It
> > doesn't work correctly. It will generate a kernel configuration, but
> > the kernel from that configuration likely won't boot. Use the old,
> > ugly (but reliable) make config. Some tips: Use the DE4x5 ethernet
> > driver for the Multia's ethernet card, not the tulip driver. Use the
> > MSS driver for the Multia's built-in sound - base 0x530, IRQ 9, DMA 3.
> >
> > Then make dep; make clean; make boot will get you an image you can
> > boot from milo.
