Tom Browder wrote:
>
> It would be nice if one or more of the following were true regarding
> the instructions that come with a kernel tar ball:
>
> 1. Tell alpha users what make command to use ("make boot" versus "make vmlinux").

Actually this help message says 'make zImage' and 'make boot'
is really equivalent to that on i386 architecture and you still
have to run 'make bzImage' separately if you need that one.
'make boot' in reality it the the only architecturally independent
kernel compile command in Linux sources. It runs a default target in
arch/<your_configured_architecture>/boot/Makefile - whatever that target
happens to be. You can aquire that bit of wisdom with a help of a quick
look at the top level Makefile in linux sources. :-)

Quite a while ago I submitted to a relevant maintainer patches both for
this help message and for a top level README file to take into account
all other non-Intel architectures. We even had some e-mail exchange
about the issues. I cannot tell you why changes are coming so slowly.

> 2. Have the configuration script set Makefile so the same kernel-making command is
> used for all (the config script identifies the system).

It can be done trivially with Makefile includes - if so desired. There
is no need for an extra manipulation. And actually is not required
unless you want to prevent scaring some Intel users by a change
from 'make zImage' to 'make boot'.

Complaining about that on this list is not likely to help very much. :-)
