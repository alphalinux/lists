On Thu, 15 Apr 1999, Hoover, Tony wrote:

> >From what I've heard, RISC machine code and Alpha in particular is best left
> to the compiler. If you can understand what the piece of X86 asm is doing,
> and then re-write it in C/C++ things should work out for the best.

This can't be left to the compiler. The asm in question is responsible for
supporting arbitrary function calls at runtime; i.e. preparing the stack,
copying the arguments, preparing registers, etc. The compiler does this at
compile-time, which is not useful when you need this stuff on the fly.

The Mozilla developers responsible for introducing this requirement have said
that if there were any other way to do this, they would gladly do it, but
unfortunately they're stuck.
