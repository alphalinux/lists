I thought they processed HTML into a word list for each
document. Probably use a sorted binary search tree against all
words, and also maybe eliminate uninteresting words to
reduce size.

Once you find the words of interest, have the tree point
to a list of HTML pages that contain it in another set of
tables. merge those tables to create a list of pages to check.

Maintaining the tables pointing to HTML pages/URL's would be
the hard part but you'd do that in the background.

Then all you need is fast disk and lots of memory.

Just some thoughts.
