On Fri, 16 Apr 1999, Simon Shapiro wrote:

> I am SURE this was asked and answered before. Especially on this list:
>
> Aside from setting the kernel option for serial console, what else
> needs to be done? I tried anything I could think of with little
> success. Other O/S boot with serial console on the same hardware, so
> it is me who screws up. But how?
>

Always append serial to the boot line unless you've set it in aboot.conf.
Make sure you have a correct /dev/console set up- char 5,1 I believe, and
make sure inittab invokes a getty for it. Oh yeah, make sure you've
configured a kernel that has serial console enabled- some of the RH
distributions don't have them enabled.
