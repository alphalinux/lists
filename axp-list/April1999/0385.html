Hi Y'll

Been struggling with these all day. Metrox Millennium II with 4/8MB
of RAM:

X11:

* The SVGA driver blacks out the screen and goes south on just about
   any setup. Very same card, very same code work fine in an LX164.

* Tried the Metrolink one. $39.00 later - Same result; the config
   works, the server hangs.

* Tried to compile the XFBdev driver; No luck. Way too broken for my
   ultra-tight schedule.

* Found a server from Suse; Has bizzare shlib links. most are
   resolvable, but one: What the hack is libgz? Can find neither hair
   nor hind of it anywhere.

* Configuring the kernel for the frame buffer is all but simple. the
   working combination here is everything is compiled into the kernel
   except for the matroxfb driver. This one must be a module, or else
   your kernel will hang/freeze real good at boot time. First thing
   after it tells you how it was compiled. I can nicely have the
   display at 1600x1200x32, but the type is on the small side :-) Anyone
   with giant fonts that work?

Shutdown:

* Seems like the PCI subsystem preserves some states that it tries to
   be nice to the SRM and restore. Problem is it is too eager and pulls
   the rug from under the drivers' feet. The result is that either the
   drivers (typically SCSI) complain bitterly, or the kernel blows its
   top with illegal instructions and whathaveyou.

* If that is survived (typically it leaves the shutdown scripts
   hanging), the SRM is not at all happy with the results. Me thinks
   be better resetting the darn machine.

* MD is a bit fragile; It tends to rebuild the arrays too much.
   Especially after a crash. Eventually it gets confused and causes
   mount & co. to oops the kernel. It should probably learn to
   distinguish abrupt shutdown from a crash and handle writing through
   its buffers more intelligently. Just a thought.

* Building a kernel from clean is about 2-3 minutes with -j 8. Much
   more than 8 and gmake looses it, emitting nonsense errors, etc.

Any help on these matters (including re-directs :-) will be much
appreciated.
