On Thu, 8 Apr 1999, Rusty Lingenfelter wrote:

> I did a rather informal poll, and it seemed that many people were using
> the ATI successfully. I was wondering if you needed to change a jumper or
> an ARC/SRM setting in order to disable onboard video in order to use a PCI
> video card.

Interesting, most people I have seen who had success with the UDB have
been using an S3 Virge (myself included). No jumpers or ARC/SRM settings
needed, the machine should just start up in ARC using the new video card,
MILO will see it but the stock RedHat kernel doesn't so you need to switch
back to the TGA port while the machine boots. Starting X will switch you
back to the PCI card but you really need to recompile the kernel without
TGA support.

I wonder why it is that MILO is compiled such that it recognises the PCI
video card rather than the TGA but the stock kernel doesn't. Is it
possible to compile a kernel which will default to the PCI card if
available but still work with TGA if none is present?
