> I have an alpha running redhat 5.2 on a UX2 board with an ensoniq
> audioPCI sound card. It plays cds fine, but when I try to run _any_
> mp3 app, it hard locks the system. has anyone else experienced this
> and/or has any solutions?

The latest (2.2.6) kernels are supposed to have some fixes for the
ES1371 driver. Alas, my LX164 with AudioPCI still locks up in exactly
the same way as your UX2: the mixer (CD, etc) works, but any attempt
at PCM in/out causes an immediate hard freeze.

My solution was to punt the AudioPCI and get an ISA card: a Turtle
Beach Malibu. It was cheap, and so far it hasn't had any problems.
It uses the CS4232 driver.
