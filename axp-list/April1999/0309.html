Christoffer Walther wrote:
>
> it fails on partition verify.
>
> I dont know the AlphaBios well, but I configured the boot like this:
....
>
> osloader on scsi(0)disk(6)rdisk(0)\linload.exe
                             ^^^^^^^^^^^^

Simple, if it fails it means that is likely not correct. This should be:

   scsi(0)disk(6)rdisk(0)partition(1)\linload.exe

Instead of trying to memorize gory details of AlphaBIOS it is
much saner to create first a default entry, as close to what you
really want as it will let you, and edit a resulted stanza
to right values. Much better chances for a success. :-)

Note that AlphaBIOS asks for a partition to load OS from and not
for a file to load (ARC BIOS is slightly different) so do not put
'milo' there. :-) Last, but not least, various entries may not verify
but still be ok; we are not really loading NT.
