Yes, setting up a font server on the Tru64 UNIX side
is indeed a better idea

I've started the "/usr/bin/X11/xfs &" daemon (after
adding /usr/lib/X11/fonts/cde to the catqlogue list
of the /usr/lib/X11/fs/config file) on the Tru64 side
and added the entry

        FontPath "tcp/<fs-Server>:7100"

to the /etc/X11/XF86Config file on the Linux side.

This made away with the afore mentioned error messages.

The Program "/usr/dt/bin/dtterm -display linux:0"
core-dumps now but that's dtterm's problem now I suppose.
