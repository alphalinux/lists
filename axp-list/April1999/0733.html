I've done some experimenting with mpg123 and stuff and found rebuilding
it and linking it with -libcpml makes it alittle more robust but strange
things still happen. Besides the regular FPE I get from mpg123 I also
see artifacts in my X session. Like parts of the screen have been lifted
off and will not refresh. Other things that happen include X crashing ,
X locking the machine (reboot), and spontainious program death. With
program death A good example is I could be installing an rpm while
playing an mp3 and the install fails or compiling something, anything
and it will die at random places with a SIG 11 or something.

My config

533LX
SCSI U2 ncr53c895
FireGL 1000 pro 8 meg.
SB AWE 64 value , type 4 using 4-front sound drivers
Kernel 2.2.3

I've tried to use more recent kernels but then my oss drivers stop
working. I've also tried configuring it using the kernel drivers and the
isapnp tools. I can configure the card well and
build the kernel with the right drivers according to the docs but no
sound. I have similar trouble with esound. It just dies with a FPE and
then E gets an FPE and then dies. Any thoughts?
