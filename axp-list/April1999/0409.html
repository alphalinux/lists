> Now, the 2.2 kernels changed to a depth-then-breadth technique (at least
> that's what it looks like to me), in which all directory levels are
> activated at the same time (unless limited by a max number of processes to
> spawn, ie -j 8), with one compile at each level. This results, with just -j
> and no max limit, in a veritable flood of processes/compiles starting up,
> and it seems to overwhelm some part of the system.

OIC. I have hit a perhaps identical bug: I have some gizmos which do
things in parallel on all the machines in my cluster, which means 290
forks in a row. At times this is not reliable on AlphaLinux ("resource
temporarily unavailable"). But my test program didn't succeed in
breaking things, so I gave up trying to characterize it. Apparently
the process you fork has to actually do something, and then the
planets have to be in alignment. I've never seen that problem in Intel
Linux.
