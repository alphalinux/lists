2.2.0-pre5 CONFIG_RTC on AXP missing mouse interrupts

This existed in pre4 as well, maybe more. With CONFIG_RTC enabled, the
mouse *often* does not switch windows when it is moved, as it should. X
seems to always swap to the correct cursor, but the window often doesn't
get focus. The only other difference I can see is that vmstat reports far
fewer interrupts per second. Without CONFIG_RTC I get all the timer
interrupts included (1024 Hz) and with RTC I get 20 or so per sec.

Also without RTC hwclock doesn't work, of course.

AlphaPC164LX, egcs-1.1.1, XFree86 3.3.3
