Dave Beck wrote:
>
> I must be silly or something, but I'm at a loss why anyone would
> use the mkdosfs tools if you can simply dd a milo disk image.

If you have, say, 5 Mb (or 3, or whatever) disk partition then you
just reduced it in size to 1440k (not even 1.44 Meg). Maybe you
do not care, but somebody might. Partition did not shrink, but
a file system on it will believe in what it says in a header.

If your original partition was smaller than 1440k then you are opening
nice possibility that in some moment you will scrible on the next
one (no, not necessarily right away). Have a nice time recovering
from a "badly damaged file system" disaster.

> It seems like a lot of extra work

'mkdosfs' is indeed a longer program name than 'dd'. But you
will gain on arguments length. :-)
