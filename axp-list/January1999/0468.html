Ron Farrer wrote:
>
>
> The stock RH5.1 kernel gives a "attempt to access beyond end of device"
> error when using sda2, I remembered (somehow :) many months ago someone
> else had the same problem and the workaround was to use sda5.

I think that you are really talking about a problem which milo has
(I do not know if all versions) when trying to read a kernel which
is "too far" on a device, i.e. beyond 2 Gig boundary. This one is
totally independent of partition numbers and a workaround is to put
a kernel on a partition which is smaller than "a magic limit"
(milo may load fine from a big partition - a kernel **position** and
not a total partition size is important). If you have small FAT
partition you can put your boot image, i.e. vmlinux.gz, for example
there. Milo will not have any problems with that and your root
partition can, and in this case should :-), be somewhere else.

> The kernel I built also gives this error when using sda2, and gives the
> error below for sda5.

Do you have file systems which can serve as a root file system on both
sda2 and sda5?

> > > Partition check:
> > > VFS: Cannot open root device 08:05
> > > Kernel panic: VFS: Unable to mount root fs on 08:05

This is a typical error when you are trying to mount something
which is not a valid root file systems (like your swap partition,
for example :-). The other possibility would be that you forgot
to configure in your kernel a support for ext2 file system.
