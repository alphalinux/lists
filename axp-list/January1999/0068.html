>>> Dan Smely said:
>
> executing startx it ends up with the error "xf86EnableIOPorts: Failed to
> set IOPL for I/O".
>
> Just in case you need to know it, I'm using:
> Alpha Ruffian
> RedHat 5.1
> Kernel 2.0.36
> XFree86 3.3.3-1
> Video: Matrox Millennium on 64bit PCI
> Ramdac: tvp3026

The stock 5.1 GLIBC doesn't recognize the RUFFIAN system type (the update
does, I believe, as does the 5.2 version).

So, do exactly the following as root:

        ln -s Miata /etc/alpha_systype

and try X again.
