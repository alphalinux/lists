Hi,
  I have just run blencmarks 1 & 2 on my system - here are the results:

The system:

  DEC LX motherboard with DEC 533MHz 21164a (2MB cache fitted) running
Linux 2.2.0-pre5 - 320MB of RAM

The graphics card:

  Matrox Millenium 2 (4MB)

The following benchmarks were taken with X in 16 bpp mode:

   blenchmark 1 draw: 98, draw+z: 413
   blenchmark 2 Time: 00:05.48 (0.26)

(I think that number 2 is the record according to your web page! And this
is NOT the fastest Alpha you can get)

Running it in 8 bit graphics we get:
  blenchmark 1 draw: 71, draw+Z 263

(I didnt do b2 in 8 bit mode).

It is interesting that the blenchmark 2 is so damn fast - but blenchmark 1
is quite slow in the scheme of things.

Perhaps this is due to the Matrox being slow - or does it indicate a
problem with the speed of X/ some Alpha compilation issue ?

I build Mesa-3.0 with egcs-19990103 using -O6 -mcpu=21164a
(except one file which had to be built with an older one due to a crash -
but that wasn't in the lib)

*** -O6 gives a factor of 3-4 speed wise in the Mesa trispd tests
improvement.
I'd like to know if it makes any difference to have the blender binary
itself built for the 21164a and with a -O6 for luck (if it isn't already).
