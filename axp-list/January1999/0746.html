Vladimir Kamenov wrote:
>
> cdrecord: Invalid argument. shmget failed

Look for CDR_FIFOSIZE environment variable in cdrecord documentation.
A value of 4000k, instead of a default 4Meg, is fine for Alpha and
cdrecord works fine. Another option you have is to recompile your
kernel with an increased amount of a shared memory (PITA, as you
will have to remember to change that every time you experiment with
a new kernel). There is also documented command line option which
has the same effect as CDR_FIFOSIZE.

BTW - xcdroast 0.96e is out for a while. I do not remember its long
URL now but it is easy to find via index on http://freshmeat.net
Look for "x-cd-roast" string. 0.96e comes with source rpm which
recompiles immediately on Linux/Alpha. Here is my script,
located in /usr/local/bin/xcdroast, which I use to avoid problems
with shmget:

#!/bin/sh
#
# A minor hack which seems to be required by cdrecord on Alpha
# A default 4m value for CDR_FIFOSIZE causes errors in shmget;
# somewhat lower 4000k works fine at the moment
#
CDR_FIFOSIZE=4000k ; export CDR_FIFOSIZE
exec /usr/bin/xcdroast
