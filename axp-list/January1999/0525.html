Ric Tibbetts wrote:
> Or, you can modify the SAMBA Server end to accept the encrypted
> passwords from the Windoze box. Again, requiring hacking files, and making more
> modifications. And again, requiring more steps to rebuild a blown server.

Hacking files? Making modifications to smb.conf is a standard part
of an installation process as a default smb.conf is just that and
nearly never an exact fit. It is very hard to "blow server" that
way as everything which is required not to do that is a backup file
and a text editor.

If you feel overwhelmed buy "Samba" boot by John Blair. Recommended
any time you are using Samba in a production environment.

> Do any of
> you really want to do all of that for 500+ boxes?

Which 500+ boxes? You mean hacking registry on 500+ boxes? One has
to be crazy to do that (but apparently some people are). So you need
a password server. What is the problem with such scale? BTW - Samba
2.0.0 beta, soon to be 2.0.0 release can do that or you can use
NT in that role.

Maintaining any big network, does not matter of what kind,
requires assorted resorces; documentation and books, in use,
is one of these.
