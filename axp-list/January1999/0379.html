On Thu, 7 Jan 1999, Larry Snyder wrote:

> > I'm also having a problem mounting
> > filesystems as a user. It says only root can mount /dev/* on /mnt/* but
> > according to filerunner (chmod) all users have r-w-x privledges on both
> > the devices in question and the mount points. I even change the owner
> > on said points from root to me (peterpan) with no effect. I'm stumped.
> > TIA.
>
> mount won't let anyone but root do anything active. Users can run it to
> see what's mounted where, but only root can mount/umount things. If
> you have a need beyond that, check out sudo. We use it at work.

This is not completely accurate. You can set up /etc/fstab to help with
this sort of thing.

/dev/cdrom /cdrom iso9660 ro,user,noauto,unhide 1 1

Any user can mount /dev/cdrom on /cdrom simply by typing "mount /cdrom"
.. I have this command tied to F6 under the window managers that I use.
This way, all my friends who are not penguin-friendly don't have to
mess with bash while using Netscape or Acrobat when sitting at one of
my X terminals. Keeps complaints to a minimum :)

However, sudo is a great tool. I heartily recommend it.
