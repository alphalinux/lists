If this is a Digital Personal Workstation 500a,... you'll have to give
up using the onboard ethernet if you want two ethernet interfaces. The
problem you describe is well known... and there is hint that it may be
fixed in a firmware update... so try that.

But no, compiling the tulip drivers as module or in kernel does not
help. The problem is supposedly related to the way the onboard ethernet
chip was implemented on the board.... Digital (Compaq) has officially
recognized this problem.. or so I recall from a email response from
them. This problem is SPECIFIC to the PWS 500a/au systems.. I guess
600a/au too.. but I've only confirmed with people that have the 500a.

I have the PWS 500a, and I've tried a million different things to try to
work with the onboard ethernet and another tulip card I installed. I
also tried installing a 3com card... but still the same problem. I
eventually gave up and installed both the tulip card and 3com card and
disabled the onboard ethernet chip. You're probably noticing that the
onboard ethernet starts spitting packets out like mad... and then dying.
I've tried all combinations of compiling the drivers as modules and in
the kernel... no luck. I've also tried several versions of the tulip
drivers... but after I noticed the same problem with the 3c59x driver, I
realized it wasn't a driver problem... there is just something funky
about that onboard 21143 tulip chip.

Sorry, this isn't good news. But at least you know.... however, there
are hints in the latest firmware update that might fix this.... so give
that a try. I haven't yet tried it,.. as on my first attempt, it failed
to update the firmware.... if you have better luck, please let us know.

D.
