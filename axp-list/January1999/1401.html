> On Mon, 25 Jan 1999, Designer 7 wrote:

> > #define __ELF__ 1
> > #define unix 1
> >
> > is this it? or am I missing anything? is __alpha suppose to show up here
> > too?
>
> It can be (and should be really...at least for now until everyone catches

I looked around and older versions of egcs, and gcc, have more predefines.
For example, for egcs-2.90.29 there is something like that in specs
file (a long line folded for mail):

*predefines:
-D__alpha -D__alpha__ -D__linux__ -D__linux -D_LONGLONG -Dlinux -Dunix
-Asystem(linux) -Acpu(alpha) -Amachine(alpha) -D__ELF__

For egcs-2.91.60 (a.k.a. "1.1.1 release") it is only

*predefines:
-Dlinux -Dunix -Asystem(linux) -D_LONGLONG -D__alpha__ -D__ELF__

I have no idea why this change as there are various sources around which
rely on a presence of defines from a longer list. X is only one example,
although pretty big one. :-) You can, obviously, hack your specs file
and Red Hat apparently did in what you can find in rpm format.

Anybody wants to file a bug report to egcs-bugs@cygnus.com?
