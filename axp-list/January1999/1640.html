>
> One task I have not figured out is how to automatically create the initial
> disk label. I know how to manually do this with fdisk, but I don't want to
> do this several hundred times.

You can find on net archives 'sfdisk'. This one can be driven
from a file with commands and data. It is actually more flexible and
capable than fdisk but its interface is a bit cryptic. :-)

I have no idea if 'sfdisk' is still maintained, if its code if 64 bit
clean and if it knows now anything about disk labels. Last time I used
it it was definitely writing only partition tables. If these things
are missing it likely should not be very hard to update its code
by "borrowing" from the current fdisk.

> Does anyone know of a tool that can write the disklabel without manual
> intervention ???

If you are really talking about disklabel then did you check 'minlabel'?
Its counterpart from DU definitely can do what you ask.
