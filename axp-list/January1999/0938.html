> 3) My disk has crashed, and the root partition has damage. How can I do disk
> maintenance?
>
> Solution for RedHat users: start the installer from the boot disk and
> ramdisk. Proceed until it asks whether you want to install or upgrade, and
> answer upgrade. After that, hit 'Alt-F2' and you will be at the ash shell
> prompt. A few tools will be present, including mount and e2fsck, however
> mount only accepts the -t option. To use e2fsck, therefore, you have to do
> "mount -t ext2 hda2 /mnt ; umount /mnt". This action will create /tmp/hda2
> which you use with e2fsck.
>
> Any errors?

An ommision actually. Add a half sentence above that explains that you
need 'umount' because you really do not want to run e2fsck on a mounted
file system - in particular when it is not healthy; and that you can
repeat this 'mount ... ; umount' stuff for all partitions which need
a check.

As an aside - if you cannot avoid a run of fsck on a mounted partition at
least remount it as "read only". On Red Hat systems you can always force
fsck during a **normal startup** by adding a file /fsckoptions with any
options for fsck you care to add there. See file /etc/rc.d/rs.sysinit
for details (and you can always add such mechanism if not present on
your system).

For the last week, or so, on tsx-11.mit.edu you can find version
1.14 of e2fsprogs (you likely have now 1.12 or earlier). Source rpm
which is there recompiles on Alpha without any fuss. The new version,
in particular, has an improved interaction with 'badblocks' program;
important if your disk is acting funny on a media level.

Last, but not least, starting X automatically in startup files of
any account, but 'root' in particular, may land you in a hot water.
Using different run level and initab is really a better idea (root can
boot in any run level s/he wants). Red Hat supplied /etc/inittab even
has an entry for xdm on run level 5 so a simple action of changing '3'
to '5' (a default run level in /etc/inittab) and rebooting will start
X windows for you; this will also provide a remote X-login window from
machines running 'chooser'. Before you change defaults make sure
that X is correctly configured for your card/monitor combo.
