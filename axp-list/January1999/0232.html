On Mon, 4 Jan 1999, Designer 7 wrote:

> I was having trouble running DU binaries on my Alpha/Linux machine until
> I compiled the FP Completion code option into the kernel. Up until now,
> I've always ignored that option since I didn't really know what it was
> (still don't). Now that I know I need it to run DU binaries, I want to
> know what does that option do? If I wasn't smoking anything, I think I
> recall that there is no help for that option... so I'm totally in the
> dark. Can someone please explain?

in order to streamline the alpha floating point hardware, the designers
opted not to include full support for IEEE 754 (this would include
things like denormals). the logic behind this decision is that
it takes a lot of extra hardware (meaning extra complexity and slower
operation) and in the common case, you don't need such things anyway.
therefore, fp code can be compiled either to ignore the condition or to
raise an exception. in the latter case, the operating system can then
complete the fp calculation in software. this is essentially what that
kernel option does. without it, all the os can do is send a floating
exception to the process, killing it.

note that fp completion in software is extremely slow. if you have an
executable that needs it, it's a lifesaver. but the best solution is to
change the code so that it doesnt produce such exceptions in the first
place.

it's been awhile since i looked at all this in detail, so if i've gotten
any of it wrong, please don't hesitate to correct me :)
