On 12-Jan-99 Konrad Podloucky wrote:
> Hello, you 64-bit penguins out there!
>
> I'm having a _very_ strange problem with my clock on my SX164. Sometimes,
> when
> I boot my machine and look into the CMOS settings, the clock is ticking WAY
> too
> fast (about 2 times as fast as it should be). Note,that this only happens
> sometimes, not always.
> But when it happens, milo and the kernel both give weird bmips values when
> computing the delay loop (about half as much as it should be).
> I know that there have been some discussions about clock problems, which were
> software-related but this also seems to be a hardware bug.
> After browsing through my motherboard manual I've discovered that there are
> some switches, which set the system clock frequency. Could it be that they
> aren't correctly configured and are creating this symptoms?
> If anybody has an idea please tell me. I'd also to know if somebody else has
> had (still has) a similar problem (I could need some sympathy now ;) )

This is a fairly common problem with the SX board. There is a fixed MILO around
someplace that corrected it for me. Interestingly I did not have the problem
when I installed RH 5.0 but later kernels showed the problem up. Since using
the modified MILO I've had no trouble.
