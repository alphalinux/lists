> Just noticed a discrepancy in RH 5.2 in the way 'struct timeval' is
> defined in /usr/include/timebits.H:

All non-Linux OSes seem to have a long for those fields. Linux has
been confused about it for a while, and I am unhappy to see that the
confusion is continuing. Doesn't anybody own a copy of the standard?

> This means that code that previously did this:
> gettimeofday( &p, &q ) ;
> s = ctime( &p.tv_sec ) ;
>
> now needs to do this:
>
> time_t tmp ;
> gettimeofday( &p, &q ) ;
> tmp = p.tv_sec ;
> s = ctime( &tmp ) ;

That is already required since I don't think that tv_sec is supposed
to be a time_t... a bug in the standard. But we have to follow the
bugs, too.
