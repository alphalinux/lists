> Please remember that I CAN successfully boot the kernel from the RH 5.2
> Alpha release. That boots and talks to my IDE disk just fine. My problem
> is COMPILING a kernel that does the same thing. I don't think I have a
> firmware or a hardware problem due to the fact that the RH 5.2 kernel
> works perfectly.
>
> -Joe Chott

The only thing to check if your firmware is up to date is to look in the ARC
console under display system configuration. The IDE disk should appear there.
If this is the case, try to compile a kernel without built in sound support,
but as a module. I can't recall exactly but I had the same problem and I think
I solved it by using a sound module instead of a built-in driver.If it does
not work, try to modify the -m21164 flag that is passed to egcs to -m21064. It
solved all my compilation problems.

Good luck,
Nicolas.
