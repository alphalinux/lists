Ok, sorry if I wasn't clear...

It is OK, and would be my strong recommendation, to rebuild on your own
system the SRPMs... and not to use the prebuilt RPMs. I get the feeling
that the system RedHat used to build those RPMs is full of bugs... and
did not properly build those RPMs.

Also, if you are wise, it is always good to take a peek at the source
code, the spec file, and Makefiles in the source before you build your
own RPMs and install. I typically change the spec files in the build
section to include flags for my system... an example would be:

%build
CFLAGS="${RPM_OPT_FLAGS} -finline-functions" ./configure
make

You should really check the source Makefiles to see what you need to
set, sometimes I also have to use CXXFLAGS when there are C++ code.

My point is, build your own code either from source tarballs or SRPMs..
do NOT use the prebuilt RPMs from rawhide... at least, that is my
recommendation, from my experiences... which could just be an isolated
issue.... but I've heard of many others having trouble with RPMs in
rawhide.. especially for Alpha.

Also, I'm using RH5.2/Alpha,... and I've noticed that some packages in
the distribution are broken... examples are:

ipfwadm
dhcpcd (so someone claimed.. don't know first hand)
gdbm (found out about this one when I tried building something that
linked to this lib... a rebuild and reinstall from SRPMs fixed it)

Good Luck!
D.
