Bruce Trull wrote:
>
> the next screen deviates from the 5.2 manual. Instead of prompting to
> create the boot diskette,

Manual has an obvious Intel slant. There is no "boot diskette" in
an Intel sense, apart of that one you already used to start the whole
installation, as there is no such BIOS as on Intel machines (so no lilo,
no 'make zlilo', etc...)

> I install the boot disk and get a "mount failed: invalid argument." I
> can't go anywhere from this
> screen.

This is really a minor annoyance at this point. You have a bootable
kernel on a floppy from which you started your installation. It should
be mountable but sometimes installer indeed seems to get confused in
this point. It really does not even matter what type of a filesystem
is on it as long as your kernel supports it.

Do this - switch to the second console and type 'sync; sync <Return>'.
Reboot your machine - hitting even power switch if a need will arise.
Once back at milo prompt (you were there before), and assuming that
your root file system is in /dev/sda2 (adjust to taste) boot with
the following command
   boot fd0:vmlinux.gz root=/dev/sda2
Your alpha should boot after running fsck on your file systems.
Mount your floppy on anything convenient, copy 'vmlinux.gz' to /
and proceed to boot reading in the future your kernel from a hard drive.
That is all what is to it. Set OSLOADOPTIONS on your console
accordingly.

> can anyone shed some light???

Installation was supposed to make this copy step automatically but
it tripped. Actually when updating I do not let this copy to happen -
no matter what. I may have my own, customized, version of 'vmlinux.gz'
and do not want it to be clobbered by some "installation kernel"
from Red Hat.
