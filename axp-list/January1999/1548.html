>
> I'd like to link a .o file from our vendor (compiled for Digital UNIX) with some
> .o files of our own (compiled for Alpha Linux) and the Alpha Linux standard
> libraries and run the resulting program under Alpha Linux.
>
> I don't want to use any of Digital's code.
>

But, because they were compiled on a DEC Unix box, the
compiler may force you to use DEC libraries. For example
if you used DEC FORTRAN, you probably have to link with DEC
FORTRAN RTL. It gets complicated real quick.

Anyone know if the *.o files used to link an a.out format
binary are different from the *.o to link an ELF binary?
If they are the same, you at least have a shot if your
library does not use any DEC RTL's. That's not very likely
though.
