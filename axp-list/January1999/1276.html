Andreas Czerniak <cognac@toppoint.de> asks:
> Hello,
>
> i have a apc164ux4 and install rh 5.2 with images
> (milo,kernel,ramdisk) from alphalinux.org, but i have an emergency
> message: 'unimplemented OSF mmap flags 0402'.
>
> This message come from arch/alpha/kernel/osf_sys.c, but what is
> problem ?

        In 2.0 kernel headers some MMAP flags inherited from
        DEC UNIX were still present, and locale code (LC_CTYPE,
        et.al. environment variables) uses them if they are
        known. In 2.2 those flags are 1) changed to other names
        so that when glibc is compliled against 2.2, they won't
        get used in the first place, and 2) kernel ignores them
        silently.
