Ok, this is not alpha-specific, but my multia is the only system i have...

I recently got my hands on this old Pinnacle OHD-1300 MO-drive.
Using 2.2.0-final, i get

scsi : aborting command due to timeout : pid 11, scsi0, channel 0, id 6, lun 0 I
nquiry 00 00 00 ff 00
ncr53c8xx_abort: pid=11 serial_number=14 serial_number_at_timeout=14
ncr53c810-0: abort ccb=fffffc000056e820 (cancel)
SCSI host 0 abort (pid 11) timed out - resetting
SCSI bus is being reset for host 0 channel 0.
ncr53c8xx_reset: pid=11 reset_flags=2 serial_number=14 serial_number_at_timeout=
14
ncr53c810-0: resetting, command processing suspended for 10 seconds
ncr53c810-0: restart (scsi reset).
ncr53c810-0: command processing resumed

on bootup.
This repeats, and goes on forever...

Strangely enough, MILO recognizes this beast nicely as an SLOW-SCSI-device.
