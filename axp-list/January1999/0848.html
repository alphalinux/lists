> > > Just a reminder (since this hasn't been discussed here for a while):
> > > mkdosfs is not 64-bit clean, and won't always produce a DOS partition
> > > that will be recognized by the firmware (ARC or AlphaBIOS).
> >
> > >From what I know the problem was not a code itself (which is "64-bit
> > clean") but a header which used wrong types in a layout for a sector

> > a totally bogus file system.
> >
> > > It worked

> >
> > This likely depends on a version. I tried recently mkdosfs from 5.2
> > distribution on UX; it created a file system which was OK for ARCS
> > to boot from so the problem seems to take care of. There was no
> > NT on this machine so I cannot tell you how happy it would be with
> > that partition.
> >
> > OTOH if you happen to have a dual boot box you will be well advised
> > to use NT tools to format said partition. A presence of published
> > standards was never a good reason for M$ software to follow them
> > and there could be some quirks.
