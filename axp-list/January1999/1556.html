Quoting Jonathan L Dubois (dubois@bec.physics.udel.edu):
> I keep getting errors mailed from the cron daemon like the following:
>
> Date: Fri, 22 Jan 1999 15:01:00 -0500
> From: Cron Daemon <root@bec.physics.udel.edu>
> To: root@bec.physics.udel.edu
> Subject: Cron <> Subject: Cron <root@bec> root run-parts /etc/cron.hourly
>
> /bin/bash: root: command not found

I'm guessing that you made the same mistake I made when I first moved to a
system with vixie-cron, and you installed the crontab. You don't install that
crontab that's sitting in /etc/crontab, it's run automatically. And it has
slightly different syntax than the sorts of crontabs you install using the
"crontab" command. The important difference is that "root" there - it tells
cron which userid to run it as. But that's only valid when the file is
sitting in /etc/crontab, not when it's installed.

Confused yet? Ok, do this:

crontab -r

Then everything should be fine.
