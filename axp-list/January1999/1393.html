"Maciej W. Rozycki" <macro@ds2.pg.gda.pl> writes:

> On 7 Jan 1999, Alexander L. Belikoff wrote:
>
> > aboot: loading compressed vmlinux-22p5.gz...
> > aboot: block-size smaller than anticipated!
> >
> > unzip: failure
> > aboot: loading plain vmlinux-22p5.gz...
> > aboot: unexpected object file format 143631
> > Enter kernel name and arguments:
> > aboot>
> [...]
> > PS. Aboot is aboot-0.5-7
>
> You must be using an older release of the aboot RPM package, as I
> completely ripped off the "block-size smaller than anticipated!" message
> from aboot-0.5-5. Check that you reinstalled the second stage boot loader
> after upgrading aboot -- replacing of /boot/bootlx is not enough. You
> would use swriteboot for this purpose, usually.
>

Maciej, thanks for your kind advice. Apparently, while the upgrade
process did upgrade the aboot package, it didn't bother to upgrade the
boot loader itself. Running:

        /sbin/swriteboot /dev/sda /boot/bootlx

fixed the problem.
