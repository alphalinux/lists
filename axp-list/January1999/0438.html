Larry Snyder wrote:
>
> This is from ROOT_DEV in the top-level Makefile. Default is CURRENT.
> This can really bite you if you build a kernel for machine A on
> machine B. It can be changed. Doc is in the Makefile.

This is indeed that case for Intel, but I gather that we are talking
about Linux on Alpha here. Things are a bit different and, in
particular, 'rdev' utility on Alpha does not exist. It is easy to check
that ROOT_DEV is really used only by arch/i386/boot/Makefile while
making boot images. The whole boot process is not identical
for these two architectures.
