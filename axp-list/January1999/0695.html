Designer 7 wrote:
>
> Ah.. cable modems... now, maybe I understand the situation a little bit
> more.... you see, the @HOME cable modem system uses an odd DHCP
> system... requires the passing of a hostname for the DHCP server to
> lease an address (or something like that)... The RoadRunner cable modem
> services uses a more standard DHCP....

How odd is this DHCP system this seems to be a "local property"; i.e.
literal instructions from one place do not work for another location
even if both are serviced by some @HOME subsidiary.

Actually this "new" dhcp-0.70 has an option to pass a hostname (which
you have to know in the first place and a local provider does not tell
you that). The only trouble is that it did not work but dhcp-0.65,
from Red Hat 5.1 distro, did. I heard that 0.70 does work for some so
my first hypothesis that we have a situation similar to that one with
ipfwadm or an older xntpd (a type mixup once again) does not seem to
be valid. It sounds like a time to investigate some other dhcp clients.

Before somebody will ask - no, I did not try dhcp server from 5.2.
