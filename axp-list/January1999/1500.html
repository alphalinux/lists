> > This is a bit confusing. Already for some time, the "official" libm on
> > AXP/Linux **IS** based on fdlibdm !!

do you know how to measure the accuracy of your sqrt routine? I would be
curious to know how the routines compare in this regard.

from libffm README:
Ideal function routines should have a *relative* error of less than 0.5
ulp (ulp = units of the last place), this means that the result (mantissa)
is always a perfectly rounded, 52 bit version of the infinite precision,
theoretical result.

The new fast sqrt is a stripped down, rescheduled version of the libm one,
that does no final rounding of the last bit. Error is guaranteed to be
within -1 ulp < err < +1.0625 ulp (according to libdm documentation). This
routine is now as fast as Crays sqrt, but much more accurate.
