Hi, just my little experience with fortran codes on x86 and axp.
As already pointed out, it strongly depends on the code.
With a sqrt() based code I have, a PII-400 and a LX-660 run head to
head. (Thanx to this discussion I finally understood the reason)
With a matrix multiplication based code, the LX-600 is almost twice
faster.
So, I bought a PII for to run the first code, and I will stay happily
on the alpha with the second code.

However, I think that the relevance of the compiler hasn't been stressed
out properly. My experience is that fortran from portland group on the x86
is damned faster than g77. The speed up (my short experience) can be in
the 20-50% range (not bad). I never tried g77 on the alpha, I always used
DEC compiled executables. However, as far as I know g77 is very slower
than DEC fortran. (Is it true?)

Comparing a g77 axp run with a portland x86 run is not fair. I think this
is another case of comparing apples and oranges.
The same holds for a DEC fortran run compared to a g77 x86 run.
BTW, also a g77 axp vs g77 x86 is not a correct benchmark. IMHO, the
proper comparison should be done with executables compiled with the best
compiler available on each platform. (I know it is obvious!)

A bad compiler can punish a worthy hardware.
