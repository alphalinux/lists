If both vmlinux.gz and vmlinux.backup.gz are in the same directory ( / )
on /dev/sda5 and,

boot sda5:vmlinux.backup.gz root=/dev/sda5 works and,

boot sda5:vmlinux.gz root=/dev/sda5 does not work, my guess would be a
problem with the new kernel.

In doing an ls -al command in / root do you find the files to be similar
or close? right date and time of compile? maybe the wrong file got
copied? Check those things out before recompiling.
