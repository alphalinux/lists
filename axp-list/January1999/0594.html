> That really depends on the nature of `core()',

core() is actually core(n). It only depends on
n. no other variables. so it calculates exactly
the same for every run of my testprogram.

the core is mostly unconditional, but i had a test
to the end of the core which would skip some 300
instructions if a certain condition is true.

i took this out, so now the core has no
conditional code at all. i would expect, that
it takes the same amount of ops every time.
(calculations like "c=a+b" take up the same amount
of time, no matter what the values of a and b are.
right?)

I'm not shure, if the behavior of my program really
changed (maybe it's just that my computer was less
busy), but i hat to run my speedtest about 30 times
to get some runs which were slower. Mostly the
