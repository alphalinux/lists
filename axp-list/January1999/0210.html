This is an AlphaPC164LX, 2.2.0pre4, egcs-1.1.1.

With CONFIG_RTC set:
        * 'vmstat 1' reports around 30 interrupts per second
        * moving the mouse between windows *often* fails
        to switch windows

When I disable CONFIG_RTC,
        * 'vmstat 1' reports over 1000 interrupts per second
        * moving the mouse always switches windows correctly
        * hwclock doesn't work (needs /dev/rtc)

Another problem: when I was running with CONFIG_RTC set, I rebooted
via ctrl-alt-del to the kernel without RTC, and got:
Jan 4 15:36:19 bhalpha1 kernel: Keyboard timeout[2]
Jan 4 15:36:19 bhalpha1 kernel: hda: IRQ probe failed (0)
Jan 4 15:36:19 bhalpha1 kernel: hda: lost interrupt
Jan 4 15:36:19 bhalpha1 kernel: floppy0: no floppy controllers found

I don't know if this is 'cause RTC left the system in a funny state or
what. I hit reset and the system booted normally (without RTC).
