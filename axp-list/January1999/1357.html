> is this it? or am I missing anything? is __alpha suppose to show up here

 The C preprocessor program has an internal list of predefines. You
probably has your `lib/cpp' set up as a symlink to the real program
located under '/usr/lib/gcc-lib' directory tree. This is not the best
available solution as the internal list of `cpp' is fixed -- it cannot be
changed without recompilation. A much better solution is to use `gcc -E'
which also invokes the C preprocessor but as this one uses the `gcc'
driver, the 'specs' file is being taken into account where additional
predefines as well as other parameters may be set (the file is plain
text).

 Unfortunately, one may not just replace the `/lib/cpp' symlink with a
shell script which execs `gcc -E' as the command line syntax of these two
is slightly different. There was an example `/lib/cpp' replacemet script
posted on one of egcs-related mailing lists some time ago which solved
some of these compatibility issues. You may check that one -- I use it
for a few months now and I see no troubles.
