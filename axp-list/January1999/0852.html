Hi,

I've got an PC164UX-2 Motherboard (from Samsung) with
a 21164a with 533 Mhz.

My Problem is that it seems I don't get any IRQs from
the parallel (printer) port (0x378, IRQ 7).

I've asked this before, but I did some reality checks
in the meantime.

What I did was to connect a PC (i386) box to the alpha
machine, because I wanted to transfer data over an
parallel laplink cable. It didn't work.

When I looked in /proc/interrupts I saw the IRQ 7 was
there, but with a count of 0.

I'm quite sure that the cable is OK (it just works,
when I connect it to another i86).

I tried with 2.0.35 and 2.2.0-pre7/6 Kernels.

(I compiled with a "Ruffian" setup. When I tried to compile
 with "Generic" the kernel would hang in the boot procedure.)

I also had a look at the plip source code of 2.0.35 and I'm quite
convinced that it SHOULD work. (It seems interrupts
are turned on correctly, the handler is installed correctly
...)

So at the moment I suspect that my Motherboard has a defect.
(Some Pin broken or something like that).

But before I go to my vendor and complain I would like
to know if anyone else got it working. (Which means has
someone checked if there are interrupts arriving ?
If you got a printer and a 2.2.0-pre7 it's easy to
check, because you only have to provide an IRQ to
parport_pc and it will use an interrupt driven handler.)

Please, if someone got a PC164UX board could you check
and tell me ? (I know this is not really a serious problem,
but it really makes me nervous to not know what's going on.)

so long
  Ingo
