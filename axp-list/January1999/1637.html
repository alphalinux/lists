Random set of things to try:

1) try another shell (tcsh, csh, zsh) since bash gives the error, or do
        something like "perl -e 'system daaxp_linux'".
2) re-build
3) copy executable to a different directory (test to make sure you don't
        have a filesystem error)
4) Is your disk full, and the executable not written out by the compiler in
        its entirity?
5) Look at the file with less. Notice anything odd? the "No such file or
        directory" is also given by text scripts starting with "#!/bin/perl"
        and /bin/perl isn't found (or other interpreter etc...) Though the
        "file" output you quote below should rule this out.
6) Try a different kernel version and/or compiler version.
