I have a peculiar problem on a Alpha Personal Workstation 433a, RH5.2
56KFlex Modem, all software ver that come with rh5.2

Under Winnt I can talk to the modem using hyperterminal at
38400,57600,115200 no problems and with ppp get downloads speeds in the 5.2k
range.

The same machine booted to linux using minicom I can talk to the modem at
38400 ok, 57600 works sometimes, 115200 not at all. Under pppd at 38400
works fine with transfers at about 3.5k, but any speed above this I get the
error messages below (ie serial line is looped)when the ppp0 int tries to
initiate. I have tried RH4.1,5.1, and now using RH5.2, all give the same
results.

I have tried using setserial, numerous modem init strings/configs, different
modems.

Any help would be greatly appreciated.
Is there anymore info I can give?

Bernard
