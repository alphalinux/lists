The case of the bogus BogoMIPS ...

I have a 533 MHz Ruffian that reports 29930.55 BogoMIPS during the boot
sequence. Unfortunately, this affects the clock behaviour and a "sleep
10" takes quite a bit longer than 10 seconds :-(

This is a system (without a video card) booted using
ldmilo-ruffian-981020 and milo-ruffian-981020 from gatekeeper.dec.com.

It happens with 2.0.36, 2.2.0, and 2.2.1 kernels built on Red Hat Linux
5.2 for Alpha using the stock compiler: egcs-2.90.29 980515 (egcs-1.0.3
release).

Any clues as to how I can restore my Ruffian to civilised behaviour?
