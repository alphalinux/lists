Well using pkzip on a Win95 machine and spanning the 2.0.35 patched kernel
source from gatekeeper to 5 disks. Copying each disk to /tmp/temp3 as
lk1.zip, lk2.zip and so on. Doing 'cat lk1.zip lk2.zip lk3.zip lk4.zip
lk5.zip > lk.zip' then 'zip -F lk.zip' and lastly 'unzip lk.zip' I was
able to copy the kernel source to my noname.

I then compiled the kernel as normal and booted it. No more kernel panics,
errors or getting stuck at VFS: mounting root FS etc.

Now I can finally get back to the network card. Well I followed the
instructions at the end of tulip.c file and then did 'insmod tulip.o' and
the dirty little bugger tells me:
"
tulip.o: Kernel-module version mismatch
        tulip.o was compiled for kernel version 2.0.34
        while this kernel is version 2.0.35.
"

Can anyone tell how to correct this?
