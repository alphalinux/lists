I am using egcs-1.1.1 which I build myself from sources. It works
without doing any such strange things. The only problem so far is
that a fix to combine.c, which Richard Henderson produced in October,
is NOT included in the release. Without it an attempt to compile
with shlc.c using -O2 will bomb out (optimizer goes into an infinite
recursion). Dropping optimization to -O on this particular kernel
source file, or applying the fix mentioned above, resolves the matter.

I believe that fixed combine.c is also included in the current
"snapshots" but I did not check that yet. Probably can be just
substituted if you are compiling your own compiler.
