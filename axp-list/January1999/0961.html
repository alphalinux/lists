>>> Ray Schultz said:
> Here is the message my boot floppy disk gives me
> while trying to boot my Cabriolet:
>
> <3>BusLogic: FlashPoint Host Adapter detected at PCI Bus 0 Device 5
> >3>BusLogic: I/O Address 0x8000 PCI Address 0x19000000, but FlashPoint
> <3>BusLogic: support was omitted in this kernel configuration.
> scsi : 0 hosts.
> scsi : detected total.
>
> Ok, somebody help me translate this into American.
>
> Doesn't this say that the Linux kernel that I am booting recognizes
> the SCSI adapter ( which is the correct adapter ) but that FlashPoint
> support was omitted in this kernel configuration?
>
> Doesn't this mean by inference that there could be a kernel configuration
> with FlashPoint support around somewhere or one could be built?

At one time, if you *did* configure the Flashpoint support, the compiler would
crash trying to compile that file. SO most kernels were built without... ;-}

This is no longer true of (at least) the 2.2.0 kernels; ie it may be true of
later 2.0.3x kernels, but since I've never even seen such a card, I have not
bothered with it.

Note that the MILOs I built for RH 5.2 *do* have the Flashpoint option; but,
from the config files that RH shipped, apparently all their normal kernels
have Flashpoint omitted. However, if you're willing to try it, the "generic"
kernel on the 5.2 CD *does* have Flashpoint included, so you could try that...

Good luck.
