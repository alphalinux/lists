On Thu, Jan 14, 1999 at 12:46:39PM -0500, Greg Lindahl wrote:
> How about the suggested abuse of -Tbss, -Tdata, and -Ttext...

Linker scripts, please. Those options only work properly for a.out.
Other file formats are more complex than that.

> ... coupled
> with a kernel hack that moves the start addresses of the stack and
> heap?

That would be setting EF_ALPHA_32BIT in the e_flags field of the
ELF header -- em86 uses this.

> Wouldn't that get the desired result with minimal effort?

It doesn't get you 32-bit pointers -- sizeof(void*) is still 8.
But it does make you tollerant of casting pointers through ints.
I suppose there might be some application subset that would be
happy with this.

> Or is there other stuff mapped down there?

No, nothing is mapped below the stack (top at 6GB) by default.
