On Mon, 04 Jan 1999 15:58:53 +1000 (EST), Bruce Williams <b.williams@gu.edu.au> said:
> I'm running a pretty simple 2.0.36 kernel, and when I attempt to use
> the second serial port (either /dev/ttyS1 or /dev/cua1), the whole
> machine hangs requiring a reboot. I am quite happily running a getty
> on cua1 at 9600bps so I can get console access if I need it.

This is is a long known problem. (It's even listed in the FAQ ;-) I
don't know any solution to this. But it is reported that the port
works with NT, so it should be solvable in software.

> Any ideas?

I helped myself with an additional serial card. You should be able to
pick one up for about $10, which would also having the advantage to
feature a FIFO 16550 chip that supports much higher rates.
