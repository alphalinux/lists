Peter Petrakis wrote:
>
> Have you documented your work? A ramdisk howto would be a great addition
> to the alphalinux FAQ's.

Have you ever looked at /usr/src/linux/Documentation/ramdisk.txt?
It is a pretty detailed "HOWTO" with examples. There is nothing
processor specific in it.

Actually mounting ram disk images with '-o loop' option is often
more convenient.

> Bob Segrest wrote:
> >
> > I have
> > modified the file selection and would like to create a new image disk from
> > /mnt/ram1. Any suggestions ???

$ dd if=/dev/zero of=new_image count=<ram_disk_size> bs=1k
$ mke2fs -m 0 new_image

Mount 'new_image' with '-o loop' on any convenient directory
or on one of /dev/ram? and blast away. You may start from deleting
'lost+found' there as it will be definitely not needed.

Remember that using ram disk of a size different that a default
4096k requires either booting with a kernel parameter specifying
your new size or booting with a modified kernel with a default changed.
