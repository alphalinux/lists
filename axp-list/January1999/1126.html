First I want to thank Oleg Gusev who sent me this tip. Here's how you do
it and all you need is the libs presently on your system to run DU
Netscape.

'ln -sf /usr/shlib/libc.so /usr/shlib/libpthread.so'

'ln -sf /usr/shlib/libc.so /usr/shlib/libpthreads.so'

This will do it. It starts up fast as hell too! Could someone please add
this to the alphalinux FAQ's. I've included Oleg's message to me below.
I'm downloading 2.2-9 right now and once I build and boot it I'll
download the DU JDK-1.1.7 and see if it works.

If you'd dig in archives, it's possible to find how to run DU Acroread.
In short, you don't WANT to install libpthread.so library.
use simply 'ln -sf /usr/shlib/libc.so /usr/shlib/libpthread.so'
and acrobat would work.
The way how pthreads implemeted in DU makes it very difficult to emulate
on linux.
There are also some unimplemented DU syscalls ( 247 == pathconf, etc. )
and partially implemented ( getsysinfo/setsysinfo ).
If they are really needed it's not very difficult to implement them.
For example i have a backup program that uses getfsstat() syscall,
so now i'm working on it's implementation in alpha-linux.
So, don't give up.
