All my attempts to install RH 5.2 on my EB164 (Vobis) failed. Here's
what I tried:

Milo floppy from redhat-5.2/alpha/milo/images/eb164.img
md5sum = f73b3ea04f40161e7397e304bd4a458a
Worked fine for me.

Boot floppy from redhat-5.2/alpha/images/eb164.img
md5sum = ff51f11d6f276fa00ca775b4e49c59ef
Worked fine for me.

Ramdisk floppy from updates/5.2/images/alpha/ramdisk.img
md5sum = ae91ccdce4505ed0ada0a50973c3f7fc
Worked fine for me.

The RH52 install script starts fine. But as soon as the rpms are
installed (i.e. copied to my harddisk) the system freezes completely;
I have to push the reset button. This happens when I chose "Upgrade"
as well as when I chose "Install".

Also, there's no difference when I boot the installer OS like this
MILO> boot sr0:redhat-5.2/alpha/kernels/eb164.gz root=/dev/fd0 load_ramdisk=1
So a bad copy of the boot floppy cannot be the culprit. (I also double
checked the md5sum of die floppy images.)

Why I'm sure my processor is an eb164? Well, it's a Vobis PC and my
current RH50 /proc/sysinfo says:
cpu : Alpha
cpu model : EV5
cpu variation : 0
cpu revision : 0
cpu serial number : Linux_is_Great!
system type : EB164
system variation : 0
system revision : 0
system serial number : MILO-0000
cycle frequency [Hz] : 0
timer frequency [Hz] : 1024.00
page size [bytes] : 8192
phys. address bits : 40
max. addr. space # : 127
BogoMIPS : 497.02
kernel unaligned acc : 1 (pc=fffffc00003fdcc0,va=fffffc00000944dd)
user unaligned acc : 182 (pc=28,va=1206a4682)

So what am I doing wrong? Thanks for any help.
