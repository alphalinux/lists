To anyone who cares, and especially to Peter P.

I finally found out why the Mesa-3.0 src.rpm doesn't build right. There
is no problem compiling (except for a LOT of arguments of incompatible
pointers ...scary!). The problem is that during the install section, the
spec file tries to copy the whole lib directory. Unfortunately, the lib
directory contains symbolic links. The problem occurs when one tries to
copy a symbolic link BEFORE copying the actual file that the link is
linked to. Hence, 'cp' craps out, and the rpm build terminates. The way
I got around this was to copy the files one by one first, then manually
create the links... make these changes in the spec file, and the build
went without any problems, ... except, I'm still scared by the countless
incompatible pointers...
