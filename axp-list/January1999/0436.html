Ron Farrer wrote:
>
> but when the kernel boots it
> says
>
> "
> Partition check:
> VFS: Cannot open root device 08:05
> Kernel panic: VFS: Unable to mount root fs on 08:05
> "

This is, in other words, /dev/sda5 (fifth partition on the first
scsci disk). As a default system partition happens to be /dev/sda2
then it looks like that you told your Linux to look for root there
(by passing 'root=/dev/sda5' in a boot command, for example).
Do you have a file system there which can serve as a root partition?
If not, then tell you kernel where it really is.
