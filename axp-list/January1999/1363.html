First spend time with setseral and minicom/seyon until you get the
com port modem talking to each other. This can take patience. It
can also be sensitive and different kernels builds can support
different max speeds for the same hardware, knernel versions, and
ports (I suspect depending on whether the serial code (and called
code) all fits within the CPU cache and or crosses a page
boundary).

Once you have that, using minicom/seyon and the PPP HOW-TO, work up
a little chatscript and pppd options file that looks good (turn
debug on everywhere and watch syslog to see what is happening).
This step can be a bit of a black art. It gets pretty easy once
you've done it a few times, but mostly because you've learnt about
the gotcha's.

If you'd like a working example of the various files, drop me a note
at claw@kanga.nu amd I'll send mine over.
