On Thu, Jan 14, 1999 at 10:57:19PM -0500, Sanjay Lal wrote:
> I am not sure what addresses to use for the various segments?

If you did go with Greg's suggestion -- modulo the comments
I just added in another message -- I'd suggest copying the
standard linker script from /usr/lib/ldscripts/elf64alpha.x
and modifying

  . = 0x120000000 + SIZEOF_HEADERS;

at the top to read

  . = 0x100000 + SIZEOF_HEADERS;

which gives you a program base of 1MB. That makes sure that you
can still trap NULL pointer errors. In addition, some versions
of binutils chose 1MB as the maximum alpha architectural page
size and will pad the binary appropriately. (The correct number
is 64k, and has been fixed in, i believe, binutils 2.9.0.)
