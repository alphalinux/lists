For a short time, I ran a Multia with 16 MB that served as a firewall/gateway
machine at home, running DNS, Sendmail and POP, all non-essential services
disabled. It was right on the edge, would swap at a login session, but
otherwise seemed to run okay. I wouldn't recommend compiling anything on it,
however.

For the record, the machine originally had 128 MB, which was removed to install
in an Alphastation 200 and replaced with 16 MB. Redhat was installed prior to
the memory swap, so I cannot vouch for a successful installation with only 16
MB in the machine.
