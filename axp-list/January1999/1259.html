Designer 7 wrote:
>
> I also noticed some conflicts with a package called xserver-wrapper???

This is a small program which grabs priviledged resources, starts your
Xserver and "drops dead". That way your X server, which is a big program,
complicated, and hard to scan for security problems does not have to
run suid root; only wrapper program needs that. This is a good thing.

'xserver-wrapper' is an old name. In 5.2 RH relase this is called
Xwrapper and /usr/X11R6/bin/X is really a link to Xwrapper
(but /etc/X11/X is a link to a server you are actually using).

Even if you are installing a commercial server running it through
Xwrapper is a good idea. You will have to adjust permits and links
for that.
