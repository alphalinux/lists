Rich Payne <rdp@talisman.alphalinux.org> writes:

> On Wed, 8 Dec 1999, Themis Lazaridis wrote:
>
> > Finding software for alpha-linux is its biggest drawback.
> > I wanted to run Acrobat reader, someone said try the web-based
> > version, but that needed java, and there was no java for alpha-linux.
>
> You can run either the Tru64 version or the Intel version, or use
> something else like xpdf.
>
> > Why is it so difficult to organize alpha-linux so that
> > it can automatically run OSF executables? What does Compaq
> > say about this?
>
> It already does run Tru64 executable, but you need Tru64 libraries to go
> along with that ($99 non-commecial or $1200 commercial).

Well, you should consider the support of OSF binaries on Linux/Alpha
beta. At least if your binaries uses threads (libpthread). Some
applications like Acrobat reader and others is linked with libpthreads
but can easy be fooled with a small fix (it's in the FAQ). Other
OSF applications really needs threads and does not work with
Linux/Alpha.

Some people says that libpthread checks the kernel for a license and
thats why it is incompatible with the Linux. Can anyone comment on
this?
