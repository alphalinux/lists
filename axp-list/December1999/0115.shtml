Hi Shrikant!

The the reason for this is quite simple: the generic boot image is not
that generic. To be precise it will not work with the jensen. (BTW same
problem with the suse generic kernel) For this reason you have to use
one of the two jensen specific kernels that are floating around (have a
look at tom's place, or at
ftp.lur.rwth-aachen.de/pub/jensen/jensen3.img. The last one is
monolithic and contains all drivers for nic's I that should fit
physically into the eisa slots). Be sure to use the updated ramdisk.img
as well.
Once the system is installed freshly, the monolithic kernel will get you
past the missind modules warning, but the broken awk problem will be
still there. You will have to boot in single-user mode (in aboot: '0/a
root=/dev/sdXY', X has to be replaced by the SCSI-ID of your boot-hd, Y
by the number of the bsd partition (counted sequentially)) and replace
awk by a symbolic link for mawk. Mawk has to be installed obviously.

Good luck
