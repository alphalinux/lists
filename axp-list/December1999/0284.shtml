> > >From what I've seen on my server here, ram and disk play a much larger
> > role than cpu power. I've only got 128MB in the machine that my archive
> > lives on, and the archive is big enough now that when I rebuild the entire
> > database the disks are starting to thrash a bit due to swapping. Also,
> > with sufficient memory the server won't have to swap parts of the database
> > to disk, making searches faster.
>
> Now how much disk is this taking you? For the actual archive and the
> index? I'm not worried about CPU or MEM on the new hardware, but disk has
> been an unknown.

The archive itself is about 85MB, and the database is about 125MB (this
would now also include the alphalinux.org lists, however).

Also, while it's indexing the archive you can tell htdig to append ".work"
onto the end of it's db filenames--this way people can continue to do
searches against the existing database. Then when htdig is finished
indexing it rm's the old db files and mv's the db*.work files to the
correct filenames. If you go this route then you should allow for 2x the
normal database size.
