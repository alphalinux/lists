are you on an internal network that doesn't have reverse-lookups on the IP
addresses?

sometimes if you don't have reverse lookups on the IP addresses, or a name
server set up to handle those reverse lookups, it can be very slow initially
as it tries to resolve name to IP. I've seen this on my internal network
with telnet on almost every linux distrib until I put refrences to all my IP
addresses in /etc/hosts
