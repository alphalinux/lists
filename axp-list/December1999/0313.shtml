michal@ellpspace.math.ualberta.ca said:
> rpm --rebuild binutils-2.9.5.0.22-1.src.rpm (this should make after
> a while binutils-2.9.5.0.22-1.alpha.rpm which you can then install as
> usual but you better do all of this from a 'root' login).
                                      ^^^^^^^^^^^^^^^^^^^

Depends on your paranoia level. I always try to build things from a normal
account. This prevents the build process from installing a fraction of the
files to their "true" final destination (instead of the build-root) and thus
create incomplete binary packages. But this requires well behaved makefiles,
and some packages (I think pam comes to mind) do not have well behaved
makefiles and won't build (without further patches) unless you build them as
root. The build process usually fails when the makefile tries to install SUID
programs.
