Shane Sturrock wrote:
>
> I don't see why, every integer app I have run on my Alpha outperforms the
> Intel boxes we have around here quite handily. ccc makes a big difference

If Alphas would outperform Intels in every integer based
application then Alphas would have far superiour SpecINT95
results. But why isn't it the case? Generally? Don't know.
One would have to analyse each and every application, which
doesn't run faster on Alphas than on Intels.

One possible reason is that the three level cache architecture
of 21164 based machines, together with the very small L1 caches
(you are takling about a 164LX) can hurt performance in cases
where linear memory access isn't possible.

> admittedly, it is quite possible with egcs for code to run slower, we are
> seeing upwards of 50% speedup for our codes.

I'm not talking about gcc/egcs here.
