Matthew Monacelli wrote:
>
> As another reply states, dhcpcd can handle the bootp requests, plus in
> RedHat 6.0/6.1 dhcpcd was replaced with pump.

Last time I looked at 'pump' code it had in it some very doubtful hacks;
but I did not investigate that very carefuly. I have seen some patches
floating around and maybe they are good enough. I cannot tell.

OTOH dhcpcd-1.3.18-pl3 is now essentially "64-bit friendly". :-)
Last night I removed from it what it seems to be the final
"unaligned access" and things look, so far, good. If there is
an interest I can put patched up src.rpm on ftp to grab.

One of things I like about that daemon is that it saves the current
resolv.conf in /etc/resolv.conf.sv, instead of clobbering in mindlessly
like some other similar programs do, and restores it on exit.
