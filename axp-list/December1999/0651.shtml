I've got a basic AlphaServer 4000 (5/300 2M CPU, 256M RAM) that I'm trying
to install Linux on. I've had no end of machine check problems, but I know
the hardware is fine since it ran Windows NT for nearly three years without
a problem.

I've got the machine torn down to the simplest possible configuration: there
are only two cards in the slots (Number Nine 764 video that came with the
machine, and a 3Com 3C590 10/100 Ethernet card (I've also tried the DE500-AA
that came with the machine)). There are two devices on the internal NCR
53c810 SCSI host adapter: the DEC supplied CD-ROM, and a 4G Western Digital
SCSI hard drive. That's it.

I've gotten RH 6.0 installed, and the machine is very stable as long I don't
perform any significant network accesses. As a test, I loaded up all the 6.0
SRPMS, and started a complete rebuild. Two days later the machine was still
cranking along, so I'm pretty sure the core hardware and the SCSI hard drive
are just fine. However, I can't get network access to work reliably at all.
I've tried both the 3C590 and the DE500 in PCI buses 0 and 1, tried using
"SRM PCI Setup" and not using it... I'm using kernel 2.2.13 configured for a
Rawhide (not generic).

Most of the time, the machine will boot up just fine, then I can log in and
do whatever I want. Regardless if it's been up for two minutes or two hours,
though, if I try any significant network activity (FTP large files, or copy
a directory over from an NFS mount), the system will machine check.

Anyone have any ideas for additional troubleshooting? I know there are
people running these systems with Linux out there and they work just fine...
but mine's being irritable. Thanks in advance.
