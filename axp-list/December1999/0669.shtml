On Tue, Dec 21, 1999 at 01:56:20AM -0500, Michael Rich wrote:
> I selected to install the anonymous FTP server inside the RH 6.1 setup, and
> all appears to be well, i can ping the machine from my W2K machine (<10ms
> ping), but it simply refuses to allow me to connect to the FTP server
> running on the Alpha for some reason. Are the some sort of parameters I
> just didn't set that would cause it to timeout?
>
> FWIW, it does appear that it can be connected to because i can use the
> CuteFTP program to connect to the server and after a few minutes, it will
> finally connect, and will have a very fast connection to the server. Why
> would it take so long to connect to? Any ideas?

One possibility is that you are behind a firewall which drops
(instead of rejecting) ident (port 113) packets sent by the
server to the client. This in general implying waiting for a
timeout and then continuing, but some recent wu-ftpd's distributed
by RedHat have a timer bug which makes them hang altogether (see bugzilla).
If you are in a such a situation, rejecting instead of dropping
the ident packets will solve the problem.
