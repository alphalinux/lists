Eric Brown wrote:
>
> I have my avanti running as a firewall/router (connecting the rest of my
> network to my DSL line). Recently, it has been silently dropping all
> network connectivity to the outside world. Nothing shows up in
> /var/log/messages or /var/log/dhcpcd.log.

You may want to investigate the latest dhcpcd-1.3.18-pl3 from
ftp://ftp.phystech.com/pub/. I only looked at the code yesterday
and did not try it yet but it seems to be substantially 64-bit cleaner
than older versions. I do not have a smoking gun but I have some
doubts if dhcp lease renewals are done correctly on Alpha with
older versions.

> Manually restarting dhcpcd or the
> ipchains script doesn't seem to restore connectivity; only a reboot will do.

If your ethernet driver is in a form of a module you can try to take
this interface totally down and 'rmmod' for a few minutes before trying
to restart. Maybe DHCP server is spooked with garbage and you have
to "hide" MAC address of your card for some time to clean that up.
I still believe that new dhcpcd should improve things.
