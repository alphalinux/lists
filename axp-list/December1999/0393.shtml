Oh, dang, not that one again... :-\

I'm assuming that you are not booting from serial console, right? I'm not
sure if that could cause this problem, but it might cause other ones later
during the installation.

> What I've done is used:
> P0>>>boot dva0 -file vmlinux.gz -flags "root=/dev/scd0"
>
> Kernel boots, cdrom gets mounted as root file system, but then the
> error...

Sigh; I never seem to be able to remember what causes *or* solves that one.

Things to try:

1. use the updated RAMdisk floppy instead of directly from the CD.
2. don't use the floppy at all; boot the kernel directly from the CD
   with -fi kernels/generic.gz
3. grab the 2.2.12 kernel floppy image from gatekeeper.dec.com at
   /pub/DEC/Linux-Alpha/Kernels.

Anyone else with ideas? I *know* we've all see reports of this go by on this
list, but I can never remember what we did to solve it...
