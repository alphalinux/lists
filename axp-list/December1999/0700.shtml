Michael Rich (istan@alltel.net) wrote:

> Appears from everyone's comments this was a harsh thing to do, but as of yet
> i don't know the full suite of commands that are available to me under
> Linux, so I didn't know how to kill a specific process, i only knew that
> reboot would solve my problem the old fashion way.

In case it happens again, you want to:
telnet in
su root
ps aux|grep X
kill -9 PID

[PID = X's Process ID number]

Example:
# ps aux|grep X
root 308 0.0 0.1 6120 120 ? S Dec18 0:00
/usr/bin/X11/xfs
root 392 2.4 19.4 38152 11816 ? S Dec18 98:17
/usr/bin/X11/X vt7 -auth /var/lib/xdm/authdir/authfiles/A:0-PaOtWb
rbf 467 0.0 3.5 22208 2160 ? S Dec18 2:55
/usr/bin/X11/WindowMaker
rbf 494 0.0 0.2 13152 152 ? S Dec18 0:00
/usr/bin/ssh-agent /usr/bin/X11/fvwm95
rbf 25164 0.0 1.3 2968 824 pts/0 S 10:22 0:00 grep X

# kill -9 392

[also note that if you are running X from a display manager it will now
be restarted. If you don't want this you'll need to kill the display
manager instead. I use xdm.]

# ps aux|grep xdm
root 396 0.0 0.0 16824 0 ? SW Dec18 0:00 [xdm]
rbf 25166 3.0 1.3 2968 824 pts/0 S 10:29 0:00 grep xdm

# kill -9 396

[then to start X again you would type 'xdm' or whatever the name of your
display manager is.]

If someone finds a mistake in any of that, feel free to correct me!
