Helge Kreutzmann wrote:
>
> If those disks are SCSI, you should be able to just hook all disks
> parallel in the already installed system and
> cp -a / /mnt/newdisk1

If they are SCSI or IDE it does not matter but the command above will
attempt to make a copy of things like /proc/kcore and /mnt/newdisk1
to /mnt/newdisk1/mnt/newdisk1 - which you really DO NOT want to do.
That is why I mentioned 'cp -dpRx' in another message in this thread.
