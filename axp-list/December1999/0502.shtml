I seem to always be having problems with crypto
software on Alpha. I don't get it...

example1: RSA md5 ref code... compiles on x86, sparc,
alpha... but, md5 hash on alpha is always wrong.

example2: SSLeay-0.9.0b... again, compiles on x86,
sparc, alpha... but, on alpha, most of the 'tests'
fail.

example3: OpenSSL ... same story...

compiler is gcc-2.95.2 and also tried CCC-6.2. I
remember having the same problems back when I was
using egcs-1.1.2.... and egcs-1.0.3...

WHY? If anyone can explain this, please do! For
example, the GNU tool md5sum compiled by gcc-2.95.2
seems to always generate the correct hash. What makes
it so different? the RSA md5 ref code works fine on
Intel and SPARC... just not Alpha, and that's the
exact same C code...!!
