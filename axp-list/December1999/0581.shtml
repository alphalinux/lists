SuSE 6.3 CDs (I have seen so far only beta, but a release should show up
any day now) have an arrangement when they can load a contents of a
ramdisk (a big one, which may create problems on multia ) from CD. It
is true that with a box on which I tried that on (definitely not multia
:-) I had to hack my own kernel to boot at all, so this one had to go on
a floppy, but hopefuly in a release these wrinkles will vanish.
In such situation you would not need a floppy at all.

With another running machine around there is also a "screwdriver option"
when one moves a disk physically and installs an initial setup
(by copy or by putting in a few chosen rpms - the later can be done
even on x86 box with Linux) to be moved to the rightful owner once
this is "good enough".
