Well, Sun's making noise about releasing a Linux version of the Java 1.2.2 JDK.
Of course, the implicit assumption is that Linux == Linux/Intel.

FWIW, they have a comment email address at: j2se-linux-comments@sun.com
If anyone else would be as interested as I am in seeing some official support
for Java on Linux/Alpha, that address might interest you.

For now, Uncle George's excellent JDK 1.1.8 port is serving me quite well
(pun intended -- it's running Apache JServ :), but due to the difficulties
he's had with licensing and so forth, my understanding is he won't be able
to continue that port, so we need a different solution.

We can ask Sun for a Linux/Alpha port, but honestly I doubt it'll happen;
porting to a 64-bit architecture is, like, WORK; if we see anything it'll
probably be PowerPC or SPARC or one of the other 32-bit blackdown ports.

So, maybe we can cajole Compaq into it. :) The SCSL is relatively liberal
(for a business license) and I'm sure Compaq could get a redistribution
license from Sun with minimal pain. If Compaq's serious about Linux as
driving Alpha sales, they need a Java port.

I don't miss Shockwave and all that, and Mozilla's looking pretty decent, but
I can't live without Java. :( It doesn't have to be incredibly fast, but it
does have to exist. (But what happened to the guy I talked to at JavaOne who
was touting the 64-bit Java on Tru64? Let's get that ported!! :)

Does anyone know if Compaq's got any plans for something like this, or is
there someone we can make a request to?
