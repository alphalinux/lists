On Wed, 8 Dec 1999, Hoover, Tony wrote:

> the shared libraries need to be taken from a "live" system. The CD's are
> not readable via standard means. At least that's how the DUnix 3.2G and
> 4.0A media is, perhaps that was changed for TRU64 5.0.

This is not true. All Tru64 installation CDs I've come accross, were in
standard UFS format. You can just mount them under Linux with the "-t ufs"
mount option. The CDs contain a root and /usr directory tree which contain
all files required for the installation process. It may be possible, that
a certain library is not under /usr/shlib on the CD, then it can at least
be extracted from the appropriate "setld" archive files which are plain
cpio or tar archives (can't remember which one).
