On Sun, 20 Feb 2000, James McCollough wrote:

>
> I have attempted several times to compile glibc-2.1.2 on my ruffian, but
> each time it exits catching signal 11 after trying to compile
> nsswirch.c.while compiling this file, gcc takes up more and more cpu and
> memory resources, until the machine is completely unusable. It only
> happens with this file. Has anyone else had this problem, and is there
> a solution? I read the docs carefully, and the faq doesn't address
> this.

Having compiled glibc about 100 times in the past few months, I can
honestly say that compiling that particular file (nsswitch.c) consumes an
enormous amount of resources. I have 128MB RAM on my SX and another 512MB
of swap and it uses quite a sizeable portion of that. Most likely, you
exhausted your available memory. How much RAM do you have and how big is
your swap?

It is a possibility that you're hitting a compiler bug as well,
though. Try removing optimisation when compiling that file (or reduce it
to -O rather than -O2). That usually fixes ICE's.
