Need advise on adding a 4th PCI card:
In any normal machine - just plug it in.
Not so with a PWS433 *#*?*!!!!

5 slots (2 64bit)
4 cards
Qlogic scsi(original), Matrox Millenium, Dlink 100mb ethernet, adaptec 2940U2W(the 4th card)

Using SRM- firmware wont allow anything other than the qlogic in a 64 bit slot but then the kernel
(2.2.13) hangs at 'probing PCI'
Using AlphaBios with this arrangement the firmware refuses to boot - 'illegal PCI config'

AlphaBios does however allow the matrox in a 64bit slot and boots (using a previous system on
another disk) and accepts all of the the cards.

However I had hoped to stay with SRM which is what my current kernel config is set up for.

So I thought before I wear out the edge connectors on all the cards which I have tried in
just about all combinations I would ask if a firmware update would help - or have I missed
something simpler? Yes I could remove the old scsi card but it's just annoying being stuck
with 2 unusable wasted PCI slots (assuming no one ever writes an XFree86 TGA2 driver which
is what originally lived in one of the 64bit slots).

The firmware is all original as of about 2.5 years ago.
Will an SRM firmware upgrade be more liberal about what can go in a 64bit slot?
and if so - and this is where the nervousness sets in -
does the upgrade described for DU on ftp.digital.com/pub/Digital/Alpha/firmware/readmes/dpw433.html
work on linux (or does mkbootfirm require DU to run)
and will it -
1. trash the current AlphaBios firmware
2. leave it untouched
3. update both
he asks, hoping the answer is not 1.
and if I do successfully create the update floppy and boot from it how safe is it
- just looking for some reassurance here :-) not wanting to find that due to
some other incompatibility I end up with no firmware at all.

Any advise/reassurance gratefully accepted.

Mark.
