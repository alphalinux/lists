Since the Amaya 2.3 binary you linked to on rpmfind.net was built here, I guess
I should probably respond. :)

I've not had opportunity to build Amaya 2.4. 2.3 works here and I heard of
some problems with 2.4 under Alpha Linux so I decided to skip a version and
wait for the next.

Mozilla M11 is the last release of Mozilla to work properly (as properly as
any Mozilla milestone) on Redhat 5.x. Starting just before M12, all my recent
tries build, then bomb out any time the user clicks a text entry box. Since the
"official" builds work on my Multia with RH6, I'm guessing that it's something
in RH5.x that's causing the problem. (Same version of gtk/glib etc. on both
machines.) Mozilla.org also warns of the perils of using recent versions of
Mozilla on an older system. :) I've got Nozilla M11 and the 100898
lesstif-based release of Mozilla on my site. I also have a binary of the
text-based "links" web browser as well. This one does tables, and responds to
mouse clicks in an xterm or on the console if you're running gpm. Pretty neat.
I use this one a lot.

I've also built gzilla/armadillo, but IMO these aren't worth distributing
binaries of as they don't work very well.

There is another Java-based browser called (I believe) Skate that works well
with Uncle George's JDK. Can't recall the URL for that one offhand, though.
