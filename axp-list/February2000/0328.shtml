Larry Sendlosky wrote:
>
> > When installing redhat 6.x on an au500 I got the following error:
> >
> > qlogicisp: new isp1020 revision ID (5) qlogicisp: i/o region 0x9000-0x90ff
> > already in use ..
> >
> This is because RedHat "turns on" the "Generic PCI IDE support"
> and the "DMA support" under it. Basically, this option gets the PCI
> addresses needed by the Qlogic card (before the scsi card is
> init'd).
>
> Copy this kernel to your RH6.0 boot disk (I believe Jay made this
> kernel without the above options enabled.)
>
> ftp://ftp.digital.com/pub/DEC/Linux-Alpha/Kernels/generic-up-2.2.12.gz
>
> Once you get RH6.0 installed, you'll have to use this kernel to
> boot and then rebuild the RedHat kernel source and "turn off" the
> options I mentioned above. (Actually I believe it's just the DMA
> support option - Does anybody do things that require these options
> to be enabled ???)

I use E-IDE devices on 12 Alphas. The new E-IDE disks
are quite fast, 13-25MB/sec on the 40GB Maxtors, with
the DMA patch.

> It would be real nice if RedHat didn't enable these config options
> until somebody gets a chance to fix it. Lots of Alpha/NT Miatas
> are being turned into Linux boxes and this problem happens alot.
>

Actually, I'm surprised Qlogicisp is using the same spot as
E-IDE DMA. They should have provided alternate address ranges.
I thought most PCI cards have multiple options for that sort
of thing? Or, is this just a driver issue?

> Also, if you use the built-in ethernet adaptor, use the de4x5
> driver/module. (Hoping to save you some grief... 8^) )
>

If this is the same chipset as XP1000/DS10 (21142/3), use the newer
tulip.c (0.91 or so) and it should work fine. The default one in the
kernel sources doesn't work right with these chipsets. (0.89 or so)
