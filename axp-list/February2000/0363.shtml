On 17 Feb 2000, Ron Farrer wrote:
>
> A lot of people are tired of the x86 architecture. Creating an Alpha
> interest in these people should help. Some of these people are moving to
> Mac and SPARC systems. Not many are coming over to Alpha. Why? Because

Unfortunately I think that IA64 is likely to grab a lot of this "people
tired of x86 architecture" group. A lot of people (at least in my neck of
the woods) are very impressed with this new architecture and they are
going to buy it over other 64bit archs because it will run there 32bit
code too.

This is where em86/fx32 could come in. It appears there are quite a few
people interested in developing this project, but as I understand it, the
code is still quite closed. Isn't fx!32 more advanced than em86 too? One
for the open source commiunity..

> there are no inexpensive developer/hobbiest systems. Three grand for a
> DS10 is not acceptable when you can get Ultra SPARCs for less then two
> and Mac's for less then one.

This is one of the most disappointing areas. At UNSW it looks like we are
more likely to get a lab of 30 Ultra Sparcs rather than Alphas (the other
contender) due to price. Even if Alphas perform better....

It is even more disappointing to see 164 + UX and similar cheaper
configurations go EOL when they are reaching very nice pricing.

It would also be nice to see SRM open up a bit. Sure there are probably
propriety driver code in there, is there any chance we can get a source
code release minus these parts (ala mozilla)? I'd like to see a Voodoo
running on my multia (for example:). Another for the Alpha open source
community to make into an even better product (an easy way to support
older systems that Compaq dont want to support)..

> All IMO,

 like wise..
