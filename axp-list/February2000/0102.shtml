I have an AlphaStation 200 (Avanti) that I'm using as a gateway/firewall
machine. It currently is running ARC bios, and booting RedHat 5.1 with a
2.2.12 kernel.

It works great, but it currently needs a keyboard/monitor to get properly
booted, and I don't want to dedicate a port on my keyboard/monitor switch to
this machine (since its only real job in life is to sit around and forward
packets). What I'd like to do is have it run 'headless' - i.e., without a
keyboard or mouse, and either telnet in or log in via one of the serial
ports.

I tried powering the machine on w/o a keyboard/mouse connected, and I got 4
beeps at startup (1 long, 3 short) which I *assume* means 'hey, you have no
keyboard attached'.

So my first question is: How do I get ARC to autoboot under these
conditions? (I'm not *real* keen on installing SRM, especially as I don't
have any particular desire to reinstall Linux & the rest of my system.)

After I get the system to boot, then MILO seems to dump its boot info to the
first serial port; what do I need to do to get the kernel to do the same
thing (and enable logins on the first serial port)?
