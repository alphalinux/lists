mgraffam@idsi.net wrote:
>
> On Wed, 9 Feb 2000, Jurgen Kramer wrote:
>
> > It seems that there is a beta version of RH6.2 up on various ftp sites.
> > The even have a Alpha AXP version up both in RPM and
> > ISO image version. So for the dare devils among us...
>
> Hmm.. RH 6.1 came out just a little while ago.
>
> I'm a UNIX geek -- always have been. My first PC (at home) was built
> to run 386BSD; prior to that I used oddball hardware -- low end
> workstation stuff, and Motorola VME boards with real-time OS's.
>
> So, I can't say that I moved to Linux because I wanted to escape
> Microsoft. I moved to Linux because I hoped (and it looks like I'm right)
> that it would provide the UNIX dream: compatibility. I wanted to be able
> to run the same software on all my machines without headaches.
>
> Along the way, I gained an appreciation for Linux's staying power. I
> don't like having to wipe my system out every 6 months because RH
> decides to make another release. I like being able to upgrade a few parts
> here and there -- want a new kernel's features? Upgrade the kernel, not
> the OS.
>
> I have a K6-2 downstairs running stock RH 5.2. Nothing is upgraded, and
> that box performs fine. It runs all the commercial packages that I throw
> at it, too.
>
> I upgrade my OS when I want a) a new libc; b) when I upgrade the hard
> drive.
>
> It seems to me that all of the "new functionality" present in each new
> RH release could just as easily be achieved through loads of package
> updates.
>
> I understand RH needs to release product so that they can make money. More
> power to them. But, sometimes I wonder whether all these constant
> distribution upgrades do more harm than good (introduce incompatibilities)
> and whether or not they are taking advantage of the consumer's tendancy
> (probably due to the mentality of the Windows crowd of "always upgrade
> so that we don't Blue Screen") to always want the last release of any
> given software package -- even when they don't need it.
>
> Hey.. I'm a geek, and I'm all for lots of toys and tricked out systems ..
> but I wonder how people can get work done if they are always upgrading
> the OS.
>

Don't know how you run your systems but I can upgrade a
lot of computers in a day. One day every 6 months to a
year is not a big deal. I never put valuable data on anything
but servers so wiping any one of them clean is no big deal,
even the server since I put the OS on it's own disk/partition
separate from my own stuff. This of course assumes the distribution
is itself stable. Sometimes RH makes me wonder so I wait for
other folks to try it and see what they have to say first.
I'd say a beta falls into the "wait and see" category.

YMMV

Wes
