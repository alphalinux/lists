While people are talking about manpower and coding, I'd like
to note that virtually no support exists for us programmers without
intimate knowledge of the linux kernel. This beast is complicated
and difficult to learn for someone who only sporadically has
time for non-paid stuff. If there would be some kind of list with
tasks like "grab this and this packages, put your code here which
must do this and this and send it back to <...>" I'd be able to
be MUCH more useful.

Otherwise everyone who is new to the kernel is faced with
uphill struggle to learn a lot (what is already explored by others,
who are now hovewer looking for newcomers for help) before
he's going to be useful contributor.

I, for one, came from a recently cited here "hardcode number
crunching" group who went with Alpha because of its unbeaten
FP performance. We know programming and axp specifics
down to axp asm, but don't know the kernel (or other packages
which need help for that matter) :-(

Is there anyone up to such a guiding help ?
