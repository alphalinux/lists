it is, if you setup the disk correctly,

Start the first partition at Cyl 2, This will leave room at the beginning of
the disk for aboot.

Make the first partition about 6M in size with a DOS FS This would be used
for milo.

Make the second partition your ext2 root.

and a third partition as swap.
after your done, write aboot with swriteboot.

setup alphabios/milo to boot from partition 1.

You can then boot the drive from either Alphabios or SRM.

In one word: no, not easy.

Well, Linux installation is almost the same - there are another issues
to deal with.

For example, you may have to backup all your HDDs and reformat
them in order to be able to boot from SRM if currently your system
boots from AlphaBIOS.

Another problem is, SRM and AlphaBIOS generally do not support
the same hardware like SCSI controllers, so it well might be that your
hardware is not supported by SRM.

>
> Is it possible to switch back to AlphaBios ?

This is possible. Real question is how long it will take :-) Sure if one
will try it several times, one will be able to decrease time of such
operation down to minutes <grin>
