No, the problem with AlphaBIOS not staying has NOTHING to do with
Milo.

The AlphaPC 164lx and 164sx systems were designed to only support
SRM or AlphaBIOS being loaded. Not both at the same time. In order to
switch from SRM to AlphaBIOS it is necessary to write the AlphaBIOS
image into the flash memory. In order to do this you need to load
AlphaBIOS and then use AlphaBIOS to update the flash memory.

The most reliable way to do this is to create a floppy with the AlphaBios
ROM image and use that to perform a "failsafe boot" off of the floppy.
As soon as AlphaBIOS comes up enter the AlphaBIOS setup utility and
select "AlphaBIOS Upgrade..." using the image on the floppy. This
second update writes the AlphaBIOS image to the flash memory. Then
power the system down, remove the floppy, and start the system. You
 should come up into AlphaBIOS and be able to boot into Milo. (I went
 through this exercise with an AlphaPC 164SX a year or so ago.)
Good luck.

Eric Ebinger
