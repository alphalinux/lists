I'm having trouble getting xxgdb to work well with C. If there are
solutions to these problems, I'm sure many of you have come across them
before.

One problem is that xxgdb won't recognize many simple library functions,
such as printf, sqrt, and pow. It will execute these functions
correctly as long as I don't step to or break in lines that contain
these functions, but if I do step through lines with these functions
xxgdb says that the function is not defined (or something like that). Is
there a problem with where it's looking for the libraries, such as in
some configuration file? If so, how can I fix it?

Another problem is that I can't get xxgdb to recognize global macro
definitions included from other files. Say I were to have the line

#define NUMBER 3

in a file that is included at the head of another file. When I list a
function in the second file and request the value of anything that refers
to NUMBER, such as an array element y[NUMBER], the debugger says that
NUMBER is not in the current context. How can I fix this?

Your suggestions are appreciated.
