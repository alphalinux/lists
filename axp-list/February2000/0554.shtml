I have no idea if this is related, but as it seems to [me to] be TCP related
as well, together with other similarities, I thought I'd give a shout to this
forum of experts :-)

I have a UDB/Multia running mainly as my Internet gateway. I presently have
it with 2.2.14 with IP masquerading and ipchains for firewall as well. I also
use pppd (2.3.11) in demand mode (with persist,ktune,idle and holdoff) and a
number of other machines on an ethernet running netscape and/or lynx as
web browsers.

It has happened a couple of times while pppd was actively connected to my ISP
(modem dialup) and I had logged into the UDB to check how the connection was
going that I was suddenly diconnected and netscape crashed. The machine that
happened on incidentally is an i386 PC with kernel 2.3.47. All systems stayed
up and no reboot was necessary.

My question would be if anyone is aware of any problems with using RedHat 6.0
Alpha, ipchains and kernel compiled with IP masquearing, which is what seems
to be common in these two reports?

Thanks for any input,
