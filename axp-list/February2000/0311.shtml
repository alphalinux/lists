Hello,

> I'm running Linux/Alpha on an LX164 board. The problem is that after random
> uptime the system hangs. The machine is still running and the monitor still
> detects a VGA-signal, however the screen is black and no action can be taken
> anymore.

I had the same problems in an LX164, too. Using the up arrow under less in text
mode would always cause the machine to hang with a PYXIS machine check, which
was 99% of the time not visible because the screen went black, with the cursor
in the lower right edge.

After trying never to use the up arrow under less for almost a year, I found
that the problem was related to the video board. It was a 1Mb s3 trio extendend
to 2Mb, but the extension simms were 50ns whereas the original simms were 70ns.
Since I've changed the video board, I've never had this problem again.

> So far I replaced the power supply, RAM DIMMS, motherboard with an SX164
> board (21164PC 533mhz), SCSI controller with an Intraserver and 3COM adapter
> with a DEC Tulip.
Try the video board, then...

Hope this helps,
