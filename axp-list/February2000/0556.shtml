I have 3 alpha computer A,B,C(all are SRM firmware),their configs are:
A:PC164,no floppy,no cdrom,no OS;
B:LX164,RedHat5.2,SRM+aboot,Kernel is 2.2.14
C:LX164,Digital Unix4.0

I booted A over the network(Server is C) from SRM, It succeeded.

The problem is I want to boot A over the network(Server is B) from SRM,but I failed:
A reported "failed open file bootp/ewa0"
B received requests from A but ignored it:
bootpd:info(6):recvd pkt from IP addr 0.0.0.0
bootpd:info(6):bootptab mtime:......
bootpd:info(6):ignoring request for server XX.XX.XX.XX(IP of B) from client at ethernet address XX.XX.XX.XX.XX.XX(A's ethernet address)

Why B ignore A's request?
Why bootpd at B report A'IP addr is 0.0.0.0?
I use 'bootptest' test bootp and tftp,it's OK.
