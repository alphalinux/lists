On Thu, 17 Feb 2000 Jay.Estabrook@compaq.com wrote:
->1. SRM sets an address into the PCI base register used by IDE DMA.
->2. LINUX kernel (on Alpha, anyway) doesn't change any of the IDE's
-> PCI base registers, initially for fear of changing the wrong ones.
->3. LINUX kernel (on Alpha) allocates new addresses for other PCI devices.
->4. QLogic ends up with a new address that conflicts with the (unchanged)
-> IDE DMA address.
->
->Fixed in 2.3.x; could be fixed in 2.2.x, just need some code that will
->ignore changing the first 3 IDE PCI base registers and only change those
->that come after 3 (this is how it's done in 2.3.x, or close, IIRC).

Is there a chance to tell SRM to use other base register? And could
Tru64 Unix work with that changed one?
