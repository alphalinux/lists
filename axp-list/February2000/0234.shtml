Charles E. Taylor IV wrote:
>
>
> I've not had opportunity to build Amaya 2.4. 2.3 works here and I heard of
> some problems with 2.4 under Alpha Linux so I decided to skip a version and
> wait for the next.

$ rpm -q amaya
amaya-2.4-1hd

and it works fine. Changes, beyond some minor fixes, are mostly to make
Amaya to use system libraries like zlib or libjpeg instead of it
substituting them with static versions of their own (often with older
versions).

Printing on Alpha is broken and it always was - so this is nothing new.
As long as attempts are made to pass "FILE *" objects as "int" argument
to functions this is not going to fly. This time, at last, my one more
bug report caused some reaction and acknowledgment so maybe there is
some hope for the future.

OTOH Amaya is really a fine HTML editor/validator which is doing also
some browsing on a side so various functionality one would expect from
a browser is not there.
