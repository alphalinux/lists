Well, I just started developing for Linux Alpha in October when I took
a job at LLNL. So far I've found that there really isn't a centrally
organized group working on the Linux Alpha kernel. I have absolutely
no idea who the main contact(s) are for submitting patches or how to
find patches for the latest kernels in regards to the Alpha platform.
It always seems to lag behind the Intel code by several versions. In
this regards, I can see where people would want to throw up their arms
and say "this sucks".

I feel like there needs to be more organization behind keeping current
with the latest kernels and hardware variations. As it is, the
linux-alpha mailing list is VERY low volume. Compaq doesn't seem to
know what policy to make in regards to Linux or the Alpha platform in
general.

On the other hand, API seems to be trying real hard to get things
moving. They seem like a likely candidate for throwing a little more
resources and/or effort into organizing things better. Maybe setting
up a sourceforge account for Alpha Linux related development would be
good. Along with encouraging Alpha developers to post their patches
with a core set of people who maintain the CVS tree.

BAPper
