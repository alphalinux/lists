In reply to Speedracer's letter who wrote on 15 Feb:
>
> When using suse, I got used to using a program called SU1 which allows
> certain users to execute certian programs (simillair to sudo but without
> the need to enter a password) I can't get it to compile on any of my
> machines. My multia with a stock RH 5.2 or my other Multia with a stock
> RH 6.1 I keep getting an the error
>
> su1.c:2530: warning: implicit declaration of function `getsid'
>
> Everything looks ok in the include files. Anyone have any ideas?
> Checked it out on my RH 6.1 intel machine and got the same thing.
>

Hi Aaron,

Why not use sudo? If you set NOPASSWD in your sudoers file you
shouldn't be prompted for one.

Here is an excerpt of my /etc/sudoers file:

User_Alias FULLTIME=username1,username2

root ALL=(ALL) ALL
FULLTIME ALL=(ALL) NOPASSWD: ALL

Works okay for me.
