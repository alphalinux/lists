>>> "Andrei A. Dergatchev" said:
> >
> > There are two (now) important things missing from the EBSDK PALcode sets:
> >
> > 1. facilitating access to the HW performance counters in the CPU
> > 2. handling pre-fetch code as produced by CCC
>
> Concerning your note about pre-fetch code handling -
> I'm just curious, are you saying that pre-fetch code is
> not working under AlphaBIOS ? Or part of it - what
> would "as produced by CCC" mean ?

CCC (as in Compaq CC) produces particular code sequences for the purpose of
prefetching stuff into the cache for performance purposes. These sequences
actually cause a trap into PALcode, which under SRM is quickly ignored, but
under the current EBSDK PALcode looks like a problem, and may be reported as
such by delivery of a machine check. This would be counter-productive... ;-}
