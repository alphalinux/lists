>>> Clemens Vonrhein said:
>
> I want to set up a Digital Personal Workstation 500au as a dual-boot
> machine via SRM (Digital Unix 4.0e and Redhat 6.1). So far I installed
> RedHat 6.1 on a second disk and can boot it (using the
> generic-up-2.2.12.img kernel ... no other would work).
>
> Since the machine has a Powerstorm 4D60T (unsupported under X) - which
> I still want to use when booting into Digital Unix - I would like to
> put a cheap PCI graphics card into one of the free PCI slots and
> switch the monitor between these two graphics card depending on the OS
> I'm booting.
>
> I found in the documentation, that there is a 32bit PCI (slot 1) and
> two PCI/ISA slots (2 & 3) still free. Which one should I use? Are
> there any restrictions for the PCI graphics card regarding the SRM
> firmware (of course there are restrictions in XFree86)?

All the 32-bit slots are behind the builtin bridge, but should work OK.

> So far I put a SiS 6326 into any of the free PCI or PCI/ISA busses: it
> is visible in SRM (always at bus 01, but in different slots). But if I
> attach the monitor to it the console just stays black during booting
> (DU or Linux). I did the genvmunix/doconfig/reboot under DU (although
> that shouldn't make any difference for Linux anyway, right?).
>
> I was wondering if
>
> 1. the SiS 6326 is just useless in a PWS 500au

It's possible that the BIOS emulator in the SRM console is not up to the
task of running the SiS code; you can find this out by removing the 4D60T
and leaving only the SiS, or, IIRC, by disabling VGA on the 4D60T via the
little DIP switch on the card, then see if SRM comes up on the SiS.

> 2. keeping two graphics card in the box won't work

Possibly, but it's more probable that the issue is that there are two VGAs
competing for the same I/O and memory space ranges. By disabling the VGA on
the 4D60T (if indeed that is possible), you will be left with the SiS VGA
and the 4D60T accelerator, which do not compete for the same ranges.

> 3. some jumper have to be set

As above, if there is a VGA disable on the 4D60T.

> 4. some SRM variables determine which PCI card/slot is used as
> console

Not necessary, nor is it possible, AFAIK. I believe that SRM will choose
according to some criteria, if there are multiple instances of cards that
don't compete for the same ranges, such as lowest slot number, or fastest
card, or some such. But that's not really the issue here.

> 5. the console has to be in a specific PCI slot

Not as far as I know.

The only solution that I can envision is:

        SiS (or any other commodity VGA) as the only VGA present
        VGA disabled on the 4D60T so only accelerator is active

I believe that SRM console will do the right thing in this instance, ie
choose the VGA to do console output on.

Critical to the final solution is whether the DU Xserver will function (or
can be made to function) with an unknown VGA as the replacement "front-end"
and still recognize and work with the 4D60T accelerator back-end. Sorry, I
just don't know the asnwer to this question, you'll have to experiment on
your box to see.

Good luck, and please let the list know if this solution works, or if you
find another one.
