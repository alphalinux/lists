Brian Pomerantz wrote:
>
> Is that system an EV56? This is a known problem, there was talk on
> the XFree86 development list on this problem but as far as I know,
> no one has come up with a fix. It has something to do with sparse
> addressing.
>
Rah !
Yes, it has. The fix just hasn't made it into the official distribution
since no-one could agree on the proper way to handle it.
I made one patch which parses /proc/cpuinfo to find out the machine it
is running on and selects the addressing scheme accordingly.
An improved version of that patch relied on the pciconfig_iobase
syscall, added by rth for 2.3.99-preXX, but this also never got anywhere
since it didn't quite fit into the scheme the powers that be had in
mind.
Oh well. Meanwhile I posted the first patch in the previous post; just
contact me if you're interested in the second version.
