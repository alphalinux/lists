TCP/IP print servers are available nearly everywhere for ~$100 USD that will
handle one parallel printer. We have one here on campus that suffers from a
time-out problem, so I wrote a script that will ping the printer every 10
minutes as a keep-alive mechanism.
Every external print server that I have seen emulates lpd in the device, so
printing from Linux, Unix, or NT is straight forward. You just have to
configure the printer in the OS with the destination being the IP address of
your print server.

Yes, Samba can share ANY print queue that you have defined on your Linux or
Unix box, ( remote or attached via parallel or serial cable )
