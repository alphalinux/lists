Harry:

- The DS10/DS10L uses an ALI 1543C SouthBridge/SuperIO chip, with a
Dallas
Semiconductor 12887 RTC/CMOS RAM/Integrated battery hanging off the
1543C's
XBUS.

- The XP1000 uses a Cypress Semiconductor 693U SouthBridge which has
integrated RTC/CMOS RAM with an external coin cell battery.

- In both cases, the RTCs provide a 32KHZ square wave which is the time
base
for the software TOY clock.

- Both the ALI 1543C and the Cypress 693U have integrated a Timer Unit
that
is compatible with 8254 register addressing and description.

hth!

Does anyone know much about the motherboards of various Compaq Alpha
machines? Specifically DS10 compared with XP1000. We're planning on
using RTLinux on Alpha for an upcoming project, ideally on Compaq
XP1000's.

We know that RedHat 6.1 (and 6.2?) runs on these machines, but RTLinux
relies on the RTC. The Alpha port of RTLinux (see www.rtlinux.com) is
being developed on a DS10, which has this configuration, but no-one
seems to know if the XP1000 does also.

I've tried asking various people within Compaq, but can't get any
firm answers, and the RTLinux developers can only speak for the DS10.

According to one of the RTLinux developers, the thing we need to
find out is;

> Key thing to check is wheether the xP1000 follows the standard
> Compaq design: an 8253 that is generally not used and a RTC hooked
> to the timer pin. This design is convenient for the port since we
> have seen the 8253 before and since Linux does not use the 8253s at all.

Any help would be appreciated.
