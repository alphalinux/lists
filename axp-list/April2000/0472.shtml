Here's a pretty nasty hack I've used in the past (though I don't remember
the exact details). In the "Linux Device Drivers" book, there's a section
that describes how to tell the kernel to use less of the physical
memory than is actually available. For instance, if you have 128 MB of
RAM you give the boot flag MEM=126. Now you can use these upper 2MB as
you please since it is not managed by the kernel. The only trick is to
address the memory with it's physical address rather than a virtual
address.

Yes, it's kind of nasty and requires special bootflags, but it helped me
avoid this DMA problem long enough to decide I didn't want to write a
driver any more! :)
