> > We also netboot these nodes using the SRM so I probably do need to take
> note of the
> > ewa0_mode,
> > is auto-negotiate safe to use with Linux or should I set it to FastFD ?
> >
> SRM can do some sort of network boot and the ewa0_mode settings will
> be in affect. However, if you load any software that "drives" the
> NIC itself (i.e. initializes it, etc) I don't think the SRM
> ewa0_mode settings will do any good (unless of course your software
> interrogates the values of the variables).
>
> I always use FastFD for Tru64 booting (and in general because 'auto'
> makes reloading SRM that much slower). And like I said before,
> Linux drivers ignore the values. I always force what I want in
> /etc/conf.modules. eg. for tulips
>
> alias eth0 tulip
> options tulip options=5 debug=1
>
> 'options=5' will forces 100Mb full duplex. 3 is 100Mb half duplex.
> 'debug=1' just give more info when module is loaded.
