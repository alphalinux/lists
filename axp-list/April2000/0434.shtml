>
> I need to increase the maximum stack size available to the users of the
> cluster. I've tried the obvious things:
>
> - tweaking PAM

The only way I know in which that could be changed with a normal boot
sequence is by taking and editor to /etc/security/limits.conf . If that
does not work that would mean a bug in PAM libraries. Did you check if
there are not updates for a version you have? There were some.

Setting limits is documented in stuff you will find in /usr/doc/pam-...

> - changing the default in the kernel and recompiling

That would indicate that PAM is actually setting things.
Unless a stack value is not hardwired in a version you are using
this would sort of contradict, to a degree, a guess above.

> - Passing the kernel 'init=/bin/sh' starts bash, which sees the new
> stack size correctly.

This is not that surprising. You are not running here any usual
init stuff so whatever defaults you compiled in they will stay.
