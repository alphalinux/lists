One of my UP2K's seems to have a problem. It locked up
and I had to power cycle it yesterday. When it came back
up, it took me a while to notice performance was bad and
I looked around to find out the kernel only brought up a
single processor instead of two! So, since I had some code
running, I waited until today to reboot. I shut it down,
popped the cover and reseated the processors in case there
was a problem there and then started it back up. At that
point, SRM using "show config" identified both processors
as there. When I booted though, I got a message something like:

CPU 1 Unable to complete console transition mode 1
CPU 1 start=some_big_number end=another_big_number difference=the_difference

CPU 1 Unable to complete console transition mode 2
CPU 1 (different big numbers)

Then, it finally booted and Linux showed:

SMP: 2 CPUs probed -- cpu_present_mask = 3

and

SMP starting up secondaries.
SMP: Processor 1 failed to start.
SMP: Only one lonely processor alive.

So, it seems the second processor was not a happy camper.

I did swap processors between slots and it did boot OK
without the errors but a couple hours ago it hung again
and I had to swap the processors back before I got a blue
screen again and it gives the same errors as above. Any
ideas whether it's really the cpu or maybe the motherboard
has problems??

Anyone else seen this??

Any info appreciated.

Wes
