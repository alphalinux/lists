Thanks for the link ... I downloaded Donald's latest tulip.c driver and
pci-scan.c code and tried them but they made no difference to my problem.
So then I moved on to the other Tulip drivers provided under RH6.1 namely
old_tulip and de4x5.
old_tulip made no difference but de4x5 fixed it!
I now get;

51200000 bytes received in 6.18 secs (8.1e+03 Kbytes/sec)

thtough my Intel switch with this device driver.

The cards I'm using identify themselves as;

  Bus 1, device 9, function 0:
    Ethernet controller: DEC DC21142 (rev 48).
      Medium devsel. Fast back-to-back capable. IRQ 39. Master Capable.
Latency=255. Min Gnt=20.Max Lat=40.
      I/O at 0x10000a000 [0x10000a001].
      Non-prefetchable 32 bit memory at 0x109002000 [0x109002000].

This is plugged into a dual EV67 UP2000.

Anyone know why there are 3 different device drivers for the Tulip cards and why
de4x5 works for this card ?

Addy.
