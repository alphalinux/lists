On Sun, 9 Apr 2000, Mathieu Millet wrote:

> Hi everyone,
>
> I have troubles with cdrecord. I get an error saying that "shmget failed".
> I use RH 6.1. My kernel is 2.2.14. my scsi card is a DigitalScape NetStorm 885
> based on a Sym53C885.
>
> the cdrecord version is 1.8a5-1 .

The shared memory setting under linux/alphas is really set low to where it
really isn't usable. You can modify shared memory by doing the following:

echo 16777216 > /proc/sys/kernel/shmmax

Which would allocate shared memory (the maximum, not saying that it's
going to waste 16 MB because it frees unused memory to the system when the
application is done with it) to 16 MB.
