Hi all -
        Please pardon if this has been discussed recently, but I didn't
find anything in the archives....

        Have any of you experienced problems with RH6.2 not correctly
writing bootstrap? I'm setting up a few (18) DS20Es - nothing unusual (Dual
667s, 512MB, dual 9GB SCSI). The setup runs fine (text or X) but when the
system goes to run autobootconf I get an error that says "could not find
aboot on disk /dev/sda". At this point I tried to manually write bootlx
with swriteboot and got "existing disk label is corrupt" "couldn't get a
valid disk label, exiting".
 
        The obvious solution was to re-install, use FDISK to partition and
add the BSD label, and run swriteboot manually before rebooting. My
question is - this wasn't necessary with 6.1 - what happened?
