> On Wed, 26 Apr 2000, Daniel Potts wrote:
> > I'm trying to get the DP264 debugger supplied with the Alpha sdk4.0 cd to
> > boot on a DS20 system. So far I haven't had any luck. Does anybody have an
> > idea whether it runs on this system or not.
> >
> > Are there any differences between Compaqs dp264 and ds20 ?
>
> DP264 and DS20 are essentialy the same thing. How are you trying to load
> it? It should be in flash, but I don't know the jumper setting off hand to
> start it (instead of SRM/AB).

I do the following to create a bootable floppy:
objstrip -v -p dp264dbm.cmp mboot
cat mboot dp264dbm.cmp > /dev/fd0

Then boot from SRM with:
boot dva0

As far as I recall, this is how I boot it on other (non ev6) Alphas.
SRM is loading the image successfully, but once jumping to it, just
freezes.

I'd prefer not to put it in flash. What is the normal way to test new
images before flashing them?
