Hello Alpha Linux World!

I have a Alpha 2000 5/3xxx box. I was unable to install rh 6.2 on this box
due to rh not having support for my SCSI cards in the installation kernel.
I got around this by putting an external SCSI disk drive onto the box and
installing rh onto it.

The box boots fine and stayed up all weekend on it's own. I need to compile
a kernel for my internal SCSI cards.

Following rh's directions I do a
        cd /usr/src/linux
        make mrproper
        make config
        make dep
        make clean

The next step in the directions say do a "make bzImage"... It doesn't work I
get the following
        make: *** No rule to make target 'bzImage'. Stop

I scanned the archives and there is talk about just doing a "make" and then
doing a gzip on the resulting vmlinux file. I am not getting the "make" to
make anything. <g>

I'm not making a module kernel for this box. Is this what is called a
monolithic kernel?

I'm not using lilo nor milo the current kernel boots from the p00>>> prompt
from the external scsi drive.
