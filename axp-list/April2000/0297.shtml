On 22 Apr 2000, David Huggins-Daines wrote:

> Rich Payne <rdp@talisman.alphalinux.org> writes:
>
> > LX can only hold one firmware image at a time, either SRM or
> > AlphaBIOS. You can set the os_type variable to whatever you like, it'll
> > still come up to AlphaBIOS.
> >
> > So, you can either learn to install and run from SRM (this would be my
> > suggestion) or reflash to AlphaBIOS.
>
> I'm a bit curious how he managed to start the Red Hat installer from
> AlphaBIOS and then end up with SRM in his flash ROM, seeing as the
> only ways you can reflash are from the debug monitor, by running the
> AlphaBIOS firmware update utility, or by running the firmware
> management utility from MILO...

If he started AlphaBIOS, did the 'upgrade', afterwards does AlphaBIOS
force you to reboot to start the new code? or can you continue on doing as
you wish? I've been using AB so little lately that I can't remember.

> Besides, if the LX is anything like the SX, replacing AlphaBIOS with
> SRM is decidedly non-trivial, and you would probably know if you had
> managed to do it successfully :)

I would disagree, just put the lx164srm.rom file on a FAT floppy,
choose the 'Upgrade AlphaBIOS' option, not read what it really says
because 'I know what I'm doing' and voila...you're in SRM (been there).
