Hi,

    Forgive me if this is a naive suggestion and your question is actually
deeper, but here goes:

    I had a similar problem with a "too small" stack on an IBM SP2 (ugly
statically allocated FORTRAN code), and although it took me a long time to
figure it out (the docs were sketchy), it turned out that the maximum stack
size could be set at link-time with a simple compiler flag (something like
-bx:maxstack=<whatever>). Now, it may be totally different in Linux, and I
will eventually have to port said ugly FORTRAN code there, so I have more
than a passing interest in this issue.

    Please note that I am by no means an expert ... and the only reason I
even TOUCHED FORTRAN code is because I have to .. its nasty legacy code that
I have no time to rewrite. Hope this is your "magic bullet" (please let me
know if it is)

Dave Moore
