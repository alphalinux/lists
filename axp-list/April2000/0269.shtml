> wrote down:

> 0:1/vmlinuz-2.2.14-6.0 root=/dev/sdb1

This will be '0:1/boot/vmlinuz-2.2.14-6.0 root=/dev/sdb1'

assuming that you are installing something like Red Hat 6.2
(I do not remember you ever said that explicitely).

Actually an installer probably wrote for you a correct /etc/aboot.conf
but you neet a bootblock written with 'swriteblock'. Also you should
use 'abootconf' to configure which partition holds your 'aboot.conf'
file. Then, after you set in SRM 'boot_dev', you boot like that "b -fl
0" or simply "b" if your 'boot_flags' are empty or zero.

You can always boot for the first time using your floppy and/or CD.
Once you have a system running it is much easier to sort all these
details.
