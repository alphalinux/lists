When I boot one of my UP2000s I get the following message displayed.

Apr  8 20:20:15 linux3 kernel: Floppy drive(s): fd0 is 2.88M

So is this really a 2.88MB floppy unit ?

I've looked at floppy.c in the kernel and there copied the floppy_struct info
from there into /etc/fdprm and then done;

setfdprm -p /dev/fd0 2880/2880

But when I fdformat it it says;

fdformat /dev/fd0
Double-sided, 80 tracks, 36 sec/track. Total capacity 2880 kB.
Formatting ... done
Verifying ... Read: : Input/output error
Problem reading cylinder 0, expected 36864, read -1

Do I need to buy 2.88MB floppy media for this to work ?
