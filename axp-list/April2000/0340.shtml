On Mon, 24 Apr 2000, Michael Hennecke wrote:

> Hello.
>
> I'm building a firewall on a RH 6.1 System using the
> Firewall-HOWTO.
>
> So far I compiled a new kernel and tried to enable and
> disable ip_forward
> (echo x > /proc/sys/net/ipv4/ip_forward).
> But the echo 0 command doesn't work. I can always ping
> the external network card from a computer on the
> inside.
> Do you have an explanation for this behavior ?

Reaching an address that is part of the router/firewall is not considered
routing because the packet does not traverse the forwarding chain (and it
does not leave or even enter the destination NIC's transmit routine).
You'll see that you cannot access hosts _beyond_ the firewall (try it), but
if you have allowed access to one interface, you have allowed access to
all, unless you explicitly set up an input chain for that
address/interface.

In short: the packet isn't really going _through_ the router, it's going
_to_ it.

BTW, this question is better suited for linux-kernel@vger.rutgers.edu, or
any of the firewall mailing lists (the addresses of which I do not have on
hand).
