Hi,

I upgraded my LX164 to RedHat 6.2 the last _days_.
As Rob mentioned the installation process works very well, but the boot
configuration of RH really sucks!

I changed my firmware to SRM as described in the SRM-HOWTO and it works just
fine. Then I run the installations program from CD through SRM, fine too!
But after the the system was installed and I returned to SRM I could not boot
the system.
After playing a few hours with the system and booting the CD generic kernel
with my new root-partition I figured out, that my aboot isn't able to boot
from 2/boot/.
But if I copy the kernel to / and add "1:2/vmlinux root=/dev/sda2" to
aboot.conf it works! Does anybody know why I can't boot from a dir under / ?
I also updated aboot to 0.6b but no luck!

Another really boring problem is, that the kernel-sources in RH are
possibly broken (I think).
I compiled a new kernel and added SCSI-cdrom support as module. But if boot my
new kernel and use the cdrom-drive linux dies!
I was just able to read one or two files and then "zap" I was back in SRM! No
error messages or hints in syslog!
I downloaded the kernel sources from kernel.org and compiled the kernel with
the same configuration and now the system works really great!

cu Lukas

PS Is it possible that the system now "feels" a lot faster? Is this because of
SRM or RH 6.2?
