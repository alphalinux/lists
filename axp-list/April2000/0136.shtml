        Well, I can probably shed some light on this. If you go to
www.alpha-processor.com and look at the specs for the UP2000 there
is a significant differnce in performance under Tru64 vs. linux
partially I suspect this is due to a better compiler. The rest
I suspect are better device drivers(linux is intentionally fairly
generic to make support for all hardware types easier.) A good
point to start if you want to narrow the gap might be the tricks
that need to be done to support IEEE floating point. Other possibilities
I've been unable to guarantee are that if you run multiple SETIathomes
on a single box they may average out in such a way that if each one takes
4 hours and you have 4 CPUS you get near 1h each. The best I seem to
get on my 750MHz 21264a is 3:31. This is ofcourse with the EV5 binary.
So another option to improve performance would be to get an EV67 port
done. I suspect some improvement.

        Please anyoen and everyone correct me where you know I'm wrong.

                        Hope this helps,
                        Chris
