--- "B. James Phillippe" <bryan@terran.org> wrote:
> On Mon, 24 Apr 2000, Michael Hennecke wrote:
>
> > Hello.
> >
> > I'm building a firewall on a RH 6.1 System using
> the
> > Firewall-HOWTO.
> >
> > So far I compiled a new kernel and tried to enable
> and
> > disable ip_forward
> > (echo x > /proc/sys/net/ipv4/ip_forward).
> > But the echo 0 command doesn't work. I can always
> ping
> > the external network card from a computer on the
> > inside.
> > Do you have an explanation for this behavior ?
>
> Reaching an address that is part of the
> router/firewall is not considered
> routing because the packet does not traverse the
> forwarding chain (and it
> does not leave or even enter the destination NIC's
> transmit routine).
> You'll see that you cannot access hosts _beyond_ the
> firewall (try it)
...
No, it works. I can ping a host beyond the firewall,
even the ip_forward has been turned of (echo 0 >).

In my test environment I can ping from the inside the
internal network interface of the firewall
(192.96.87.110, 255.255.192.0), the external interface
(192.168.1.1, 255.255.255.0) and a test-host in that
network (192.168.1.5).

On the other hand I can't reach any host in the
192.96.87.0 net from 192.168.1.5 except 192.96.87.110
and 192.168.1.1. It doesn't matter if I turned
ip_forward on or off.

There must be another reason.
