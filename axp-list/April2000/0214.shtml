Unfortunately there is a rather nasty combination you need to look for
between CPU revisions and chipset revisions. Some chipsets will work
while others won't, including various revisions of the same chipsets.
It sounded like a mess to have to deal with and no one seemed to come
up with something that handled all the corner cases so nothing was
excepted. If I have time next week, I'll have to try the patch that
was sent here. On my system at home (Athlon with a G400), XFree86 4.0
was probably 2-3 times faster than the 3.3.x code. It would be nice
to be able speed up my Miata at work. The powers that be are those
that work on the Alpha code in the XFree86 project.
