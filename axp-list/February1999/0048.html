This is probably not an alpha specific problem.? I was wondering if
anybody has a pop3 daemon working properly on their alpha box. I
installed "imap-4.4-4" under redhat 4.0 and verified that /etc/services
and /etc/inetd.conf were setup properly:

***
/etc/services:
pop-2 109/tcp # PostOffice V.2
pop-3 110/tcp # PostOffice V.3
pop 110/tcp # PostOffice V.3
***
***
/etc/inetd.conf:
#
# Pop and imap mail services et al
#
pop-2 stream tcp nowait root /usr/sbin/tcpd ipop2d
pop-3 stream tcp nowait root /usr/sbin/tcpd ipop3d
imap stream tcp nowait root /usr/sbin/tcpd imapd
***

However, when I try a fetchmail command "% fetchmail machine", I get
errors such as (with the user, machine, and password entered correctly):

fetchmail: Authorization failure on fetchmail: Authorization failure on user@machine
fetchmail: socket error while fetching from machine
fetchmail: terminated with signal 13

The authorization message points to a problem with the pam module
"/etc/pam.d/imap". However, I have tried plugging in various fixes to the
file without success. The default one is:

***
/etc/pam.d/imap
#%PAM-1.0
auth required /lib/security/pam_pwdb.so shadow nullok
account required /lib/security/pam_pwdb.so
***

My question is: does anybody have this working? If so, what is
different on your system?
