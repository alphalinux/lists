Uncle George wrote:
>
> This little example caused a (un-expected ) pointer failure in one of
> the font routines in my java 1.2 port on linux/alpha.

Do you mean that you are getting different results when you will do
         a = (int *)a +(x2-x1-x3)*P + (y2-y1-y3)*S;
and
         a = (int *)a +((x2-x1-x3)*P + (y2-y1-y3)*S);

Ordering and promotion rules got you. :-)

> It was expected to
> add a -1760 ( -1760 * sizeof(int)) but does not.

Eh??? Both P and S are unsigned int. Whatever a result it
definitely will be not -1760.

> Ugg. what is a 64bitter to do.

Exactly what you asked for. :-) Print that way and things will be
clear

        a = (int *)a + (x2-x1-x3)*P;
        printf("%p %x\n", a, (x2-x1-x3)*P);
        a = (int *)a + (y2-y1-y3)*S;
        printf("%p %x\n", a, (y2-y1-y3)*S);

If you want signed quantities then do not use unsigned. Stupid tricks
with types and their representations will eventually bite your back.
BTW: " a = NULL ; printf ("%p\n", (int *)a - 1760); " will print
0xffffffffffffe480. Is this what you really wanted in the first place?
