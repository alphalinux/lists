<QUESTION TYPE="STUPID">
*sigh* I ruined my uptime. (14 days 22 hours on 2.2.1. Oh well. :)

I forkbombed my Alpha to see what would happen.

By "forkbomb" I mean:

#include <unistd.h>
#include <stdio.h>
int main( int argc, char** argv ) {
    int i = 0;
    while (fork())
        printf( "Hi! %d\n", i++ );
    return 0;
}

(Yes, I have too much free time.)

No problemo; I just did a:

while true; do ps aux | grep -v grep | grep morrildl | awk '{ print $2; }' | \
xargs kill -KILL; done

...as root and morrildl went bye-bye.

Well, except for one last unkillable zombie process. I was under the
impression that no process should be unkillable to SIGKILL. Since this SIG is
intercepted by the OS and not the process, is this a bug in the kernel, or a
"feature" I don't know about? Is this maybe an Alpha-specific thing? (I don't
recall ever seeing this happen on Intel.)
</QUESTION>

(I lost my uptime because I rebooted to get rid of this zombie process; even
though it wasn't burning any cycles, looking at unkillable zombie processes
in ps makes me nervous. :)
