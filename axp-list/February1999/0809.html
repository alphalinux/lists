> Pardon the possible FAQ, but is this the "retrofit" patch that merely gets the
> kernel to build or does it bring Alpha semaphores into parity with Intel's?
> I'm confused as to which this is.
        Richard Henderson created a 10k or so patch filling the missing
semaphores that caused problems with 2.2.2pre5 and above. My retrofit
patch (and yes, this is a FAQ) was meant to be a _temporary_ solution
so kernel developers could compile the most recent kernel while working
on the semaphore code.
        If you look at the patch, you will see it fills several gaps that
were present in earlier incarnations of include/asm-alpha/. A few people,
myself included, were laying plans to work on the semaphore issue over the
weekend/next week. I am glad Richard outdid himself and saved everyone
else from hard work :-)
