On Tue, 2 Feb 1999, Michael Schmitz wrote:

> > On Fri, Jan 29, 1999 at 08:06:06PM +0900, Tom Holroyd wrote:
> > > CIA machine check: processor detected hard error
> > > vector=0x670 la_ptr=0xfffffc00002085a0 pc=0xfffffc0000420db0
> >
> > IIRC, vector 0x670 is bad or missing memory.
> >
> > Jay Estabrook has answered this question several times in the
> > moderately recent past -- check the archives to be sure.
> >
> It seems that 660 indicates bad main memory, 670 (at least in my
> case) seemed to be related to the L2 cache chips. Motherboard
> had to be replaced.

I believe it is also quite possible that you can see this with a broken
kernel, as well. I asked Jay about this a while back and he suggested it
would be possible, since (IIRC) 670 could also be caused by IO access to a
non-existant location. In my case, I was seeing this every few days on
2.1.131, but I've been running 2.2 now for 11 days and have not seen the
problem yet. Fingers crossed..
