By the way, I observed recently a strange behaviour of Linux-native XEmacs
on my UDB. I compiled xemacs-20.4-3 from SRPM at www.xemacs.org under RH
5.1, and it immediately seemed to me that it was considerably slower than
the xemacs-20.4-2 version (from binary RPM from www.xemacs.org) I had
installed before on my RH 5.0 installation. So I tried to install the self
compiled xemacs-20.4-3 RPM also on my old RH 5.0 and upgraded all shared
libraries to which xemacs dynamically links to the same versions that I
have in 5.1. It turns out it indeed runs considerably faster under 5.0 than
under 5.1. How is it possible that the same program using the same shared
libraries runs with a different speed under 5.0 and 5.1 ? Any idea ?
