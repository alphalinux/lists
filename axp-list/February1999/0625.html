Kid wrote:
>
> Hey all. I finally got my Multia setup to install 5.2. Since I don't
> have a CDROM drive on the box, I setup my x86 NT box for FTP services
> and put the 5.2 CDROM in the NT machine and shared the root dir of the
> CDROM. I made the 3 required AXP-Linux boot floppies and chose FTP
> as my installation source.
>
> Things went fine until the system had to get files from the CDROM/FTP.
> Linux was looking for long file named files, but everything on the
> CDROM was in the ISSO9660 "8.3" format. What happened and how can
> I get the install going?

I am having the exact same problem with OS/2, Win95, and Win98 using the
original RedHat CD. It seems that none of these operating systems is
capable of reading RockRidge extensions which contain the long file
names. I assume NT has the same limitation. The funny thing is I have an
old slackware CD with RockRidge extensions and OS/2 can see the LFNs
just fine. The funny thing is that the RedHat manual says that the FTP
or NFS install functions can be used to install from Win95. Apparently
this is only if you forego the CD download all the files yourself. :(
