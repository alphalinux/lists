Hi Kerry -
It looks like your modem is never getting out of command mode. If
you invoke chat with a -v it'll tell you what's going on. You
shouldn't start any LCP activity until the physical connection is
established. BTW, I had to set mine up on the 2nd serial port as the
the kernel was echoing console messages to the first (this on the
Multia I use to manage demand dialup and NAT). Another possibility is
that you're hanging up immediately after getting the first ConfReq.
This would put you back in command mode and echo all you send right back
to you. Note that near the top of your log you start receiving exactly
what you were sending. That's how pppd senses the loopback condition.
HTH,
-ls-
