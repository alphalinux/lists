Jaap Schellekens wrote:
>
> Andreas Magnus writes:
> > Hello,
> > Is it possible to compile sources on the Alpha and get Intel-Binarys as result ?
> >
> > I have an old Intel 486 System which is verry slow when compiling the Kernel.
> > So it would be nice to compile it on my fast Alpha.
>
> I know that the other way around gives problems (32-64bit issues, has
> to do with gcc's desing).

To use 32-bit Intel in order to crosscompile 64-bit Alpha code would
be a major overhaul of gcc. The problem is that various compiler
stages had to be able to do arithmetic for a target and gcc does not
have provisions to make this arithmetic wider than that of its host.
To go the other way around would be entirely possible. In the past I
used 32-bit machine (I was running that on NeXT but other folks had an
array of different platforms) to crosscompile 16-bit code for Atari ST.

Still to make such cross-compilation possible you would need not only
this compiler itself but a bunch of other tools to provide the whole
enviroment (two assemblers, linker, library archiving tool, assorted
auxilliary programs running on Alpha but geared towards i386 output).
All doable; but unless you look for an interesting exercise in a practice
of compilation then upgrading your 486, for example, would be more
cost-effective. :-)
