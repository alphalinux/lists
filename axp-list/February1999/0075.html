        Just want to point out that Intel is making the privacy "feature"
disabler software, but only for Windows (and only NT, I believe). One of
the interesting consequences of the embedded id implementation is that
we may see an extension of the "Windows-only website" trend - indeed,
"Wintel" sites could become more commonplace than you now think. Let's
say a company developes a streaming media player which makes extensive
use of the Katmai instruction set. To ensure that only the target audience
(x86 w/Katmai instructions) can benefit from the software, the developer
coded a nice routine that checks for a valid PentiumIII id.
        This could even be extended to the dark side of computing - let's say
that a malicious bug in silico in the first batch of P3 processors is
discovered which causes a processor freeze. Future virus authors now have the
benefit of a hardwired identification code to provide their creations with as
a means to target specific machines - if a ex-sysadmin knows that administrator
X and Y's machines have a specific id, they can release a trojan onto the
network with a sort of seek-and-destroy warhead.

Once again, it's a great time to use AlphaLinux.
