At 20:16 17/02/99 -0500, Brian Beuning wrote:
>This question might be better answered on the squid mailing list.
>But it sounds straight forward enough to have two boxes running
>squid and have them be squid ICP siblings of each other. If you
>don't know what an ICP sibling is, check out
> http://squid.nlanr.net/Squid/Hierarchy-Tutorial/

Yes, but the problem is that the user browser is set up to use
proxy.latrobe.edu.au:8080 (in our case) to get both their settings and
data. If you have a sibling (say proxy1.latrobe.edu.au) then if proxy
fails, everything does. You can either change DNS entries or get the users
to temporarily update their browser settings.

I seemed to recall that there were some features in linux that allowed an
automatic fail over of the DNS name to a second system. This is possible
(but not trivial :-) under Digital Unix (err Tru64 Unix) with TruCluster
availability server (our production SAP environment is set up this way). I
was trying to avoid the costs in setting up another TruCluster for squid
and trying to promote Linux at the same time.
