>jeff wrote:
>>
>> if they are the same type of harddisk, why not use dd to copy the first to
>> the others?

>This sounds like a pretty risky proposition. Various the same type
>disks may be not really exactly the same because a production process
>is not perfect. Few sectors here, few sectors there....
>
>Also running the first time mke2fs, or mkswap, without -c flag will
>likely work nearly always - untill you will **really** need that
>disk right away. :-)
>

I also answered this guy ( in private mail) sugegsting dd as a solution,
BUT I told him to make sure they are identical disks, including the ROM
version ( what you see on startup messages) and run a low-lvel format and
check from the SCSI controller to refresh the bad sectors table, then
maybe, after that run badblocks on the disks. This will pretty well
eliminate any bad blocks/sectors from the disks before you dd.
dd can not override the SCSI bad table, btw. Nothing can override this,
short of a "mode sense, mode alter" command set referred to the right mode
page, etc. - on the actual SCSI device.
