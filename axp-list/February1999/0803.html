On Fri, Feb 26, 1999 at 12:49:21PM -0500, Steve Frampton wrote:
> Hello:
> I am moving an Alpha XL 266 to 5.2. I created the XL MILO diskette as
> well as the XL boot diskette. The system boots fine, but when I get to
> the "Insert RAM diskette and press any key", my supplementary diskette
> craps out and I get a "kernel panic: cannot mount VFS root".

Please use the updates ramdisk.

> I checked my images -- I can mount the MILO and boot images on both the
> floppies as well as using the loopback device. However, I cannot do
> *either* with the 'ramdisk.img' file. I've tried all of minix, msdos, and
> ext2 types, but mount doesn't like whatever file is there.

The ramdisk is a compressed image. So if you want to mount it, you first must
uncompress it:
gunzip -S .img ramdisk.img
That'll give you a file names "ramdisk".
You should be able to mount this.
So you could check whether your ramdisk is broken or not.
