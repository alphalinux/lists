The AlphaPC 164LX motherboard has incorporated within it two EIDE ports. It
is my understanding that SCSI is a faster disk access bus on a Linux system
than EIDE because each of the SCSI devices can function autonomously thus
reducing the work load of the CPU. However it is my reasoning that if only
one device such as a CD-ROM is on the EIDE bus while the hard disk(s) reside
on the SCSI bus then the CD-ROM and hard disks could still function in
parallel without bogging down the CPU. EIDE CD-ROM drives with write/rewrite
capability are significantly cheaper than their SCSI versions. If there
is no penalty in having only one CD-ROM drive on the EIDE bus then purchasing
an EIDE CD-ROM drive makes sense.

Has anyone else thought about this? Please criticize my facts and logic if
anyone can see errors in it.
