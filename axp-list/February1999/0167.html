Marian Stagarescu wrote:
>
> Another part of the hint was to boot from CD but I have the following
> problem:
>
> milo> boot -t iso9660 scd0:kernels/pc164.gz root=/dev/scd0
                                                 ^^^^^^
It is not very likely that you have on scd0 at the same time
both a root file system for Linux and that one with 'kernels'
subdirectory. Not impossible, mind you, but Linux setup expects
to be able to write there during installation and that would be
a bit hard on CD; so it is not done that way. :-)

> is booting the kernel but after that:
> VFS: mounting read-only file system
> Unable to open a console.

It is not terribly surprising; there is indeed no console device around.
Try this:

milo> boot -t iso9660 scd0:kernels/pc164.gz root=/dev/fd0 load_ramdisk=1

You need obviously a ram disk image to load and your kernel is set to
load that from a floppy.

Red Hat in November last year published some errata for boot images
and for ram disk in particular. Check mirrors with updates.

  Michal

p.s. small FAT partition you mentioned in your message is needed later
for you to be able to boot from a hard drive and not all the time from
a floppy.
