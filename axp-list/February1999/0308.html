 Hello Chris!

 I tried the previous snapshot on my SX164, kernel 2.2.1, RH 5.2 with updates.
USB is on the Cypress CY82C693U-NC chip on the motherboard. I am getting si-
milar error messages, but the first problem seems to be that the USB control-
ler does not have an IRQ assigned (there is no IRQ even in /proc/pci).

 I remember I had to use

#define ioremap_nocache(X,Y) ioremap(X,Y)

in the file drivers/uusbd/ohci/driver.c to get the patch to compile. I don't
know enough about the hardware and the kernel to know if this is not dangerous
- I just saw the same trick in the matrox frame buffer driver.

 But reading the docs, it looks like the driver for non-Intel USB controllers,
i. e. ohci is still incomplete:

>From drivers/uusbd/INSTALL:

                        much], on other platforms [i.e., Compaqs,
                        non-Intel ...]. Fine controllers.

 Anybody can comment?
