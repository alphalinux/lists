Hannu Karhu wrote:
>
> Hi,
> I have a problem with Digital Alpha XL-266, trying
> to install RedHat 5.1 or 5.2.
>
> Milos (with show command), and kernels give this error
> message (below). RedHat 4.x works just fine, so I guess
> hardware is ok.
>
> Is there any fix or workaround?
> Or is it possible to simply install RH5.2 g77 rpm's to RH 4.x?
>
> ncr53c8xx: at PCI bus 0, device 6, function 0
> ncr53c8xx: ncr53c810 detected
> ncr53c810-0: rev=0x02, base=4001000, io_port=0x8800, irq=11
> ncr53c810-0: ID 7, Fast-10, Parity Checking
> CACHE TEST FAILED: host wrote 1, ncr read -65535.
> CACHE TEST FAILED: ncr wrote 2, host read 0.
> CACHE TEST FAILED: ncr wrote 2, read back -16129.
> CACHE INCORRECTLY CONFIGURED.
> ncr53c810-0: detaching...
>
> I noticed updated kernel source for RH5.2 with new
> NCR drivers is available. I'd like to use
> pre-built images though, as I don't know enough
> about SCSI/Alpha specific MILO/kernel issues.
>

Make sure that the scsi termination is set correctly for your
configuration. I had the same problem with an AS250 with external
drives, SCSI term was set to internal instead of external. Once it was
set correctly milo stopped reporting a cache error.

-Jim
