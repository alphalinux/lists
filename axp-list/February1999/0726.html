On Wednesday, February 24, 1999 12:25 PM, Tom Browder
[SMTP:tom2@fwb.asi.srs.com] wrote:
> I can successfully ftp FROM my alpha, but I can't ftp TO it.
> I have modified the inet.conf file to change the default setup
> so the -a argument is not there, but still I get access denied
> (I don't get a password request--just a message saying the
> ftp server has disconnected).
>

I've had this before with RH5.0 & 5.1. The problem will almost
certainly be that /etc/inetd.conf has a line like
ftp stream tcp nowait root /usr/sbin/tcpd in.ftpd -l -a
and if you look in /usr/sbin you'll probably not have an in.ftpd,
but instead an in.wuftpd or similar. make a symlink between
the two or edit the file and you should be okay...
