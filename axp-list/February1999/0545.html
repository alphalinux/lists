I may have the ideal way of introducing Linux (most probably on alpha) into
work if there is a good Linux solution to the following problem. Like most
organizations we have a squid proxy server which (perhaps unlike most)
_must_ be used by all our users to access WWW content outside of the
University. All browsers are configured to use this proxy server (which
runs squid on top of Digital Unix).

Naturally this is a single point of failure which just happened :-) To add
excitement, the two members of staff who could have fixed this relatively
easily (someone kicked out the SCSI cable) were on a training course and as
a result there was no web access for more than 12 hours (local pages
weren't really accessible as browsers tried to contact the squid server).

This problem can be solved by purchasing a new Digital Unix box and a Cisco
IP redirector for an all up cost of lots (say AUD$40K). Now I seem to
recall that this can be done totally in software with a linux box and if
this is the case we can probably do it for far less (AUD$6K perhaps). Has
anyone got any pointers or experience with this sort of thing? It would be
especially neat if Linux/Alpha could be used (we'd save more money on
maintenance on the current box and could buy the other one out of savings).
This should be the thin end of the wedge...
