Larry Snyder wrote:
> dump_i365.c(.text+0x28): undefined reference to `outb_p'
> dump_i365.c(.text+0x40): undefined reference to `inb_p'

'grep' is your friend, use 'grep'. In conjuction with 'find', like
this:
  find /usr/include -type f -follow | xargs grep outb_p
That reveals immediately that outb_p (and inb_p) is a macro defined
in <asm/io.h>. Looks like that the header was not included, so
include it.

> Feb 20 20:52:33 killer cardmgr[261]: +
> /lib/modules/2.2.0/pcmcia/ide_cs.o: unresolved symbol ide_unregister

Same thing. Either this symbol was not provided because something
which provides it compiles conditionally and some constants were not
defined (grep for this again) or it has to be exported by a kernel (in
linux/arch/alpha/kernel/ksyms.c) and is missing. If the later case then
add it.

What symbols are actually exported it is easy to check with
   cat /proc/ksyms | less
