> Well, except for one last unkillable zombie process. I was under the
> impression that no process should be unkillable to SIGKILL. Since this SIG is
> intercepted by the OS and not the process, is this a bug in the kernel, or a
> "feature" I don't know about? Is this maybe an Alpha-specific thing? (I don't
> recall ever seeing this happen on Intel.)

a zombie is a process which has exited, but whose exit status has not yet
been collected. in general these only occur if the parent process is
still alive, since otherwise init (pid 1) inherits the process and wait()s
on it to allow the zombie to go away. so a zombie occurs when the parent
is still alive but for some reason is not wait()ing to collect the exit
status of the deceased child.

since a zombie is already dead, it won't hurt anything. it takes up only
minimal resources ... so you probably didnt need to reboot and lose your
uptime.
