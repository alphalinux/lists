> SIMMs from MCT)
> "The Thing" is stable running it with WinNT Wkst.
> I have set it up with a 4.3Gb IDE (Seagate) disc + Pioneer CD and using the
> MILO image from an original RH5.2 CD, I can boot, setup and run RH5.2 .....
> for about 3 - 5 hours doing various jobs.
>
> Then i start getting messages like these in /var/log/messages:
>
> 11:45:12 ...009234949.... Cleaned Cache of 0 RRs
>
> 11:51:34 Kernel: bash Memory Violation: 0093874849b3a ..... // ...
> 0000000008 Bad address
>
> And if I keep it running it slows down to a crawl and finally dies with:
>
> 13:01:28 Infinite loop in Interrupt handler......
>
> I have tried compiling Kernel 2.0.36 but it does the same thing.
>

I've had my PC164 for a couple of years now, and the only time I had
_any_ stability problems was when I had a 2940UW in it. That was
replace by an Intraserver 3140 and it's been stable ever since. I've
got two suggestions:

1. Make sure you have all the updates from the errata pages.
2. Try kernel 2.2.1. It's been rock solid for me (current uptime >3
   weeks, under heavy load)
