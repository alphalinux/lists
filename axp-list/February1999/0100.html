I have recently been working on some benchmarks comparing the 164's(LX
motherboard at 533) and the new 264, thanks to one of the vendors on the
list and
have not found the speed up I expected going to the 264. So I thought
I'd toss out
a few questions for the gurus.

First some background, the code is a computational fluid dynamics code
and was
compiled under a DEC Unix box, I believe an older 3?? mhz box if that
matters.
Anyway, after compiling I've moved the code to several linux boxes and
tested it,
along with a comparison on a SGI Octane that we routinely run on. Here
are
the results:

Octane:225mhz 37 min. 47 sec.
Alpha164:533mhz(-O4) 46 min. 38 sec.
Alpha164:533mhz (-O5) 39 min. 03 sec.
Alpha 264 500 mhz 23 min. 40 sec.

The -O4/-O5 is the optimization level under DEC fortran. The code run
on the
264 was the -O5 compiled version.

I've seen SpecFP95 numbers that indicate a factor of 3 speed up(for what

a standardized benchmark means) so I have a couple of questions.

1) Is there a newer DEC fortran that 'knows' about the 264? Would this
possibly
     improve the results?

2) Has anyone else done benchmarks of their own code? How did they
turn out?

3) We've tweaked with the options for compiling on SGI's, but this was
a
     quick and dirty compile under DEC Unix, i.e. use one of the global
-O
     flags and go, does anyone have any suggestions for particular flags
to
     look at under DEC's fortran that may help it on the alpha cpu's and
on the
     264 in particular.

I'm very pleased with the 164 performance compared to the Octane,
considering
that it is probably less than 20% the cost, and on a cost comparison the
264 is
twice as fast for what is probably similar prices so I'm pretty happy
with that.
I was just expecting a little more.

Any information would be greatly appreciatted.
Thanks,
Todd
