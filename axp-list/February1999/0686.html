The above is the pretty typical.....

I find it a real shame that there are patches done, and we get kernels to
work, but these patches never seem to "make it" past one revision of the
code. For instance it seems that only "every other" kernel version since
1.0.30 has sound correct for the multia, and thats just straight MSS.

With the latest 2.2 kernels out there, it seems a VERY flustrating task to
compile when none or little of the previous alpha patches make it into the
source code for release. Of course I can goto alphalinux.org and get
various patches, but wouldn't it be nice if this code made it into the
source tree??

I don't know, perhaps I'm too visionary, getting kernel sources that
compile AND work at LEAST as good as the kernels that came before.
