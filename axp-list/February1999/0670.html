This weekend I was working on the one of the LX164s at work trying to
install a couple of 16GB IBM IDE drives. It wasn't working right, so I
decided to upgrade the BIOS (5.60). Well, the new BIOS (5.66-1) didn't work
with the old version of MILO (2.0.30 build 8), so I upgraded it (2.0.32pre)
which worked even less, so I backed off on MILO (2.0.30 build 88) which
didn't work in yet another way, so I backed off on the BIOS (5.66) which
seemed to work (didn't fix the HD problem, it turned out that using fdisk to
force the drives into LBA fixed that (I can't understand it, the machine
seemed to have a problem with an IDE drive with 32760 cylinders :-) )).

With the old BIOS/MILO combo (5.60/2.0.30-8) the machine would never do a
soft reboot. It would return to MILO from the kernel, countdown from 30,
then shit itself. If you hit escape, it would immediately shit itself.
With the new combo (5.66/2.0.30-88) soft reboots act odd as well. It
returns to MILO, but the countdown freezes at 29. If you hit escape, it
goes ahead and boots, but is very flaky until a) it crashes, or b) the
operator gets anoyed and does a hard reset.

What is the best LX164 AlphaBIOS and MILO?.

I've read about shooting MILO directly into the flash chip, but could find
no mention of it working on a LX164. Is it possible? Would it work better
than using ARC to start MILO?

I've never actually seen an Alpha do an unattended reboot, is it just a
pipe-dream, or does it actually happen for some? Admittedly, I've only ever
worked with LX164s.

Hopefully, I'll be putting a PC64 together this weekend, all the same
questions apply to it.

Thanks for your time,
