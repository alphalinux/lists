Uncle George wrote:
>
> This little example caused a (un-expected ) pointer failure in one of
> the font routines in my java 1.2 port on linux/alpha. It was expected to
> add a -1760 ( -1760 * sizeof(int)) but does not.

> a = (int *)a +(x2-x1-x3)*P + (y2-y1-y3)*S;

in expressions of the form

int * unsigned
                
The int gets promoted to the unsigned type.
The value therefore is _not_ -1760, it 0xFFFF~~~~
When promoted to a 64bit type (e.g. (type*)) as it's unsigned it will become 0x00000000FFFF~~~~
and then scaled by sizeof(type)

It's doing exactly what was programmed, the programmer was in error.
If you want the difference between pointers, why not use ptrdiff_t?
That _is_ what it was invented for.
