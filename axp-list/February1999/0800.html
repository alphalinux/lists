> I was trying all the things I could but no succes.....

Was it a generic kernel, or a NORITAKE specific? If the latter, try the
former, as some folks (other boxes?) have had better luck with the generic.

> I've just discovered that the kernel still crashes on procedure (?)
> conf_read (found in System.map)

This sounds like a configuration problem; your box has an EV4 CPU, so if
the kernel is configured for an EV5, it may very well show this symptom.
Do NOT configure for a PRIMO, that means EV5 CPU.

> anyway... I've tried to compile kernel 2.0.36 and systype NORITAKE,
> kernel booted, but stuck on setting SCSI... (also image noritake-s.img
> from RH5.2 booted with the same problem... but booted the machine..)

Both the RH 5.2 NORITAKE kernels are broken; they were not configured for
SRM boot.

> So I'm running now kernel from RH5.1, mikasa-s.img.... It's ok.. but I
> need to compile with firewall option, 2ethercards and so.. :(

Please try bulding with the alpha-sources-2.0.35-0.2 found on
gatekeeper.dec.com at:

        /pub/DEC/Linux-Alpha/Kernels

Or try building a "generic" kernel with 2.2.1.

Good luck.
