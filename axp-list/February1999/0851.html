On Sat, Feb 27, 1999 at 01:51:46PM +0100, Stefan Schroepfer wrote:
> This was discussed not too long ago, but I cant find
> the answer in the archive:
>
> What is the maximum file size in ext2 filesystems?
> Is there any difference between 32 and 64 bit systems?

As per 'struct stat', file size (st_size) is of type 'off_t',
which is typedef'd from 'long int'.

On a 32-bit system, (2^(32-1))=(2*(1024^3)) or 2 gigabytes.
On a 64-bit system, (2^(64-1))=(8*(1024^6)).
So, an alpha's ?stat _can_ return an enormous file, but I
don't believe ext2 supports it, ( sizes are __u32 ).
You _might_ be able to squeeze out a 4gb file (duh, I'm not sure).
Maybe I'll give it a try (if I can find that blank 4gb drive).

> Is there some filesystem implemented in Linux/Alpha,
> which can handle files bigger than 2GB?

Though >2gb files might be nice, I don't think its very practical
(with today's hardware). Even on the e4000's arrays at work, when
oracle wants something bigger that 2gb, we toss it a raw device
( even for 1gb spaces on fibre, performance is measurably better
on a raw device than in filesystem (at least on sunos) )

If you're dealing with huge amounts of contiguous data, its usually
best to let it live on its own spindle. :)
