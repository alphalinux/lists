green threads, with Linux 2.0.x serieas just cant be allowed to fully IDLE. This
is a problem not with green threads but with the OS's failure to inform the jdk (
through SIGIO's ) that I/O is possible. If this was reliable, then the
green-threads can just pause() until some interrupt. This is a prob with network
connections.
With 2.2.x, the lost SIGIO's prob has seemed to disappear. I couldn't get my
simple yet abusive 'C' program to fail like it did on the Linux 2.0.x series.
There is still a prob of PIPES/FIFO and lost sigio's
