wsb@paralleldata.com said:
> That's odd. I have a 400Mhz PII and an overclocked Celeron from 333Mhz
> to 400Mhz and they both run fine with the standard heatsink/fan.

I have found with other processors that a marginal cooling situation
can be mitigated by the operating system running on it. My specific
experience is with the i960, which has a HALT instruction. The i960RP,
if run in a tight loop that does not access memory, will heat up to the
point of collapse unless given extra cooling. However, when sitting halted
and servicing interrupts (or blocking on memory) the same processor will
not melt itself.

A loop such as:

        for (register unsigned idx = 1 ; idx > -1 ; idx += 1) ;

will probably warm up the processor a bit.

With that sort of experience, I can easily believe that processor X might
fail while running O/S Y and application Q, while it works fine running
Q on Z. That may mean that Linux can break some processors because it
puts it to better use, or "fixes" others because it doesn't waste cycles.

By the same token, I suspect a carefully crafted application can also
break a marginal combination.
