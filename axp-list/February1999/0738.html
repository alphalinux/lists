Michal Jaegermann wrote:

> Paul Tomblin wrote:
> >
> > Quoting mlarocque@home.com (mlarocque@home.com):
> > > monitor at 1600x1200 (rock solid) with 65K colours (I haven't been
> > > able to coax the SVGA Server (on alpha) to do 24 bit properly).
> >
> > I've found that several cards on Intel Linux only do 32 bit colour properly,
> > not 24 bit.
>
> This is really a question of naming. You tell the card to run in 32-bit
> mode and it runs in 24-bit mode anyway but with '-bpp 24' the server
> crashes. I do not know why this but it happens. Try '-bpp 32'.
> Of course one has to define a proper subsection in a config file.

Side-note: 24 bpp works in the (at least Matrox) kernel framebuffer, but getting
accels to work in XF68_FBdev is very tricky (haven't tried yet, but I've heard).
Of course, 8 MB VRAM will get you up to 1600x1200 with 32 bpp, but for those of
us with 4 MB, it would be nice if 24-bit could work- even if it only gets us from
1152x864 to 1280x1024. :-)
