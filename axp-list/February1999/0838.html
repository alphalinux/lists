Ron Farrer wrote:
>
> It was just a idea I had while trying to go to sleep.... Maybe RH should
> look into making a program to upgrade a RH system without rebooting for
> future version some day?!? I mean everything except the kernel (of course)
> can be shutdown, upgraded and restarted on a RPM based system without a
> reboot, right?

No. Not really. If you are bumping up a major version of shared
libraries, as oposed to changing only a minor one, you would be faced
with a nice "chicken-and-egg" problem. I guess that this can be worked
around if enough of "basic necessities" you are running are linked
static. In any case you are facing a nice exercise in a graph theory. :-)

Your kernel is totally independent of your libraries and can be switched
now or later - at your convenience. The fact that some version comes
in a a particular distribution is somewhat coincidental.

> Although on a Alpha, everything is much faster then on that
> other platform that you could probably reboot, perform a upgrade and be
> done with it without many noticing.

I strongly suspect that on any platform it would be faster to upgrade
and reboot than to carefuly plan such upheaval. If you are concerned
with a downtime spent doing such changes then 'rpm -Uvh' as many
sets as feasible from a new distro, without shutting down, and leave
what is left to a "normal upgrade". Whatever was replaced already
will be not replaced again thus cutting downtime considerably.

Note that if you have long running processes using "deleted in an upgrade"
copies of libraries they will hold both to memory and a disk space.
Whatever is in use will be purged only when **all** references to
it will be gone. Sometimes you can restart things (syslogd, for
example) but sometimes not.
