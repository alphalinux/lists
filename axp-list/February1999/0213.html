I have an AS200 4/233. When I installed 5.2 about a month ago,
I noticed the date/time stamps on some files showed 2018 for the year.
I checked my clock using the date command, and checked the ARC console.
Both showed 1998. Using steps I found in the list archives, I boot
into single user and did the following:

#touch testfile
#find / -newer testfile -exec touch {} \;

Most of the stamps got updated execpt for some links and directories.
When I tried to compile the 2.2.1 kernel the other night, I got the error
message:

*****clock skew detected: your build may not be complete*****

Everything seemed to compile ok, and I'm up with the new kernel, but is
there something else I need to fix the time/date stamp problem once and for all?
