You wrote:

Another part of the hint was to boot from CD but I have the following
problem:

milo> boot -t iso9660 scd0:kernels/pc164.gz root=/dev/scd0

is booting the kernel but after that:
VFS: mounting read-only file system
Unable to open a console.

I also hoped that appending " load_ramdisk=1" at the boot command
will solve it but no luck.

If somebody can help I'll appreciate a lot.

Thanks,
Marian

This looks like what I had when trying to install from a copied CD.
It turned out that the CD was copied using MS software, which uses
"joliet" filesystem extentions , instead of RockRidge.
This causes for the /dev directory to be empty , the device special files
do not exist, so /dev/console is not there....

Check if you have files in the /dev/ on the cdrom.
