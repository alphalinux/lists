> Would the 08 be a SCSI ID and the 02 be the partition number? If so, that is
> my problem. my second drive is at ID 8 and there is no root partition there!
> I am specifying where the root file system is but it still goes to 08:02!
        It's going there because arch/alpha/kernel/setup.c is telling it
to use /dev/sda2 as the default root fs. 08:02 just refers to the
major:minor device specification:
brw-rw---- 1 root disk 8, 2 May 5 1998 /dev/sda2
  
See line 286 of setup.c:
                /* Default root filesystem to sda2. */
                ROOT_DEV = to_kdev_t(0x0802);

        What does your partition table look like? It's strange that
the kernel is not respecting the parameters you're passing through
Milo.

My system looks like this:
   Device Boot Start End Blocks Id System
/dev/sda1 1 2 2032 6 DOS 16-bit >=32M
/dev/sda2 3 130 131072 82 Linux swap
/dev/sda3 131 4340 4311040 83 Linux native

        and I am able to boot sda3:/vmlinux.gz root=/dev/sda3
with no problems at all now.
 
> Is there some bug with the Fireport 40 SCSI Controller?
        Maybe. I don't know enough about that controller to make an
educated diagnosis.

> SCSI IDs greater than 7?
        In all likelihood, no.
Have you tried also posting to the kernel mailing lists? If it is a
driver bug, maybe someone has suggestions on how to fix it.
