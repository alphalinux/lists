On Mon, Feb 15, 1999 at 11:58:40PM -0500, Ross A. Knepper wrote:
>
> a zombie is a process which has exited, but whose exit status has not yet
> been collected. in general these only occur if the parent process is
> still alive, since otherwise init (pid 1) inherits the process and wait()s
> on it to allow the zombie to go away. so a zombie occurs when the parent
> is still alive but for some reason is not wait()ing to collect the exit
> status of the deceased child.

Often, you can do a

        ps aufx

to find the parent and kill it (assuming that it's no init). More
often than not, this will get rid of a zombie.
