dr john halewood writes:

> not used it on a udb, but did come across this snippet of advice about
> speeding kde up & fixing the compile problem, particularly if you're
> using
> gcc 2.8 or egcs 1.1. when running the configure program, set the
> CFLAGS and CXXFLAGS variables to -fno-exceptions, i.e.
> # CFLAGS=-fno-exceptions CXXFLAGS=-fno-exceptions ./configure
> Exception handling is enabled by default but neither qt nor kde actually
> use them *except* for the ksirc program, which needs them (editing
> the makefile to remove those flags & I think the -fno-rtti flags got it
> to
> compile quite happily).
> Rebuilding qt & kde this way (in qt edit configs/linux-g++-shared to set
> the flag) reduced the memory usage by anything up to 30% in some
> apps.
>
> cheers
> john

Could you specify where did you find these hints, in case there's perhaps
something more to find out ?
