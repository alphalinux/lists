>>> "Dumb Kid" said:
>
> With single host adapter, everything's fine.
>
> disk1(id0) scsi(0) -- boot drive
> disk2(id1) scsi(0) -- data drive
>
> Then, I added a second host adapter:
>
> disk1(id0) scsi(0) -- boot drive
> disk2(id1) scsi(1) -- data drive
>
> The AlphaBIOS sets the boot partition to be
>
> disk1(id0) scsi(0) -- which works fine.
> options: root sda:/vmlinux.gz ... -- no good (see below)
>
> Somehow Milo reconigze :
>
> sda --> disk2(id1) scsi(1)
> sdb --> disk1(id0) scsi(0)
>
> it will not boot since "sda" now is pointing to the data
> drive.
>
> Any idea how to manually set the sd[a-z] to point to a
> specific device in Milo options?

The stock MILOs won't understand any options to do this.

I suggest you swap the drives to the other controllers, and reset the
OS Boot Selection to boot from the correct drive.

Or you could boot single user, and edit /etc/fstab as approp.
