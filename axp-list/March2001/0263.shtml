is there anything else on the SCSI chain with the drive (i.e. CDRom)? I
solved my MILO problem by switching to SRM. You may want to try removing all
other devices from the scsi controller and seeing if that helps.

My problems were also on a SymBIOS 53c895 based controller. I'm not sure
what controller the AS200's use, but if it's SymBIOS based, it may be a
similar problem.

Kurt

Hi,
I recently bought a Seagate Barracuda ST15150N and am having trouble getting
Linux onto it. My system is an AlphaStation 200 4/233. I set the drive's
SCSI id to 0, the termination to active, and the termination power set to be
supplied by the drive. However when try to boot from MILO I either get a
series of periods after "Please wait while setting up devices" with MILO
2.0.35-c7b or the boot process stops after expanding the kernel and doing
some SCSI probing with an older MILO.

Am I doing anything blatantly wrong in what I describe above? Does the
ST15150N work with the SCSI controller on the AS 200? What would you suggest
my doing?

I read the archives of the list and found a similar problem, but found no
applicable answers in that thread. However, that person with a similar
problem was trying to add a SCSI disk to an already set system, whereas I
would like this disk to be my main (and only) disk on my system.
