On Sun, Mar 18, 2001 at 11:21:58AM +0900, J. Hart wrote:
>
> I am trying to install RedHat 6.2 on an AlphaPC164 I was given
> recently. My cd is located at dka200. I boot the install process
> from the Redhat alpha linux cd as follows :
>
> boot dka200 -file kernels/generic.gz -flags root=/dev/scd0

....

First try to boot

  b dka200 -fl 0

or

  b dka200 -fi kernels/generic.gz -fl 0

If this still gives you troubles then try the following instead:
  - make a floppy from images/ramdisk.img on this CD (write this "raw"
    to a floppy, i.e. no file system)
  - boot as follows:

  b dka200 -fi kernels/generic.gz -fl "root=/dev/fd0 load_ramdisk=1"

You will be asked, hopefuly, for a ramdisk floppy at some moment.

If X is a problem then use '-fl "root=/dev/fd0 load_ramdisk=1 text"'
instead. If none of the above works then make a boot floppy from
an image in images/ directory on CD and boot:

   b dva0 -fl 0

with CD still in a drive (some boards are reluctant to boot from CD).

One of these should work unless your CD is really corrupted.

  Michal
