        There is work going on with the LinuxBIOS project
        http://www.acl.lanl.gov/linuxbios/

        They've gotten it working on DS10's. It's a far
        cry from replacing SRM/AlphaBIOS on all Alpha's.

        Also, I presented for API Networks an overview
        of some work we are doing at the Extreme Linux
        conference a few weeks ago. More about that will
        be forthcoming over the next few weeks. There has
        been mention of it at some tradeshows where it was
        referred to as "Linux in the ROM". The trademarked
        name is DBLX(tm) Direct Boot Linux. In a nutshell,
        the Linux kernel and, if you choose, a small initrd,
        is loaded in the flash rom. SROM called a bootloader
        that instead of starting SRM, starts the Linux kernel.
        
        In theory, you could boot from any device or filesystem
        that is supported in the kernel. Also, all naming
        conventions are in Linux device format. (no more
        PKA0 = /dev/whatever)

        Note that API NetWorks is -NOT- targetting previous
        generation (LX/UX, etc) for any work that we are
        doing. Thankfully, alot of work is or will be
        Open Sourced and it's hoped by many that the
        community will support these platforms.
