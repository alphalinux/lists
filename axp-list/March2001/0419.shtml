I have two ES40s and I am trying to find a stable software set for
them.

node 1: 8GB, 4 CPUs
node 2: 2GB, 2 CPUs

Other common equipment: DE600, Myrinet Lanai 7, Qlogic QLA2200 FC.

I started with RedHat 6.2 and then upgraded the kenrels and other software
packages listed in /usr/src/linux/documentation/changes.txt.

I have been fighting with 2.2.19 and 2.4.2. I have heard that the
PCI bus support is much better in 2.4.2 for SMP alpha boxes. I would
like to stick with 2.2.19 though as all the other alpha linux boxes
are running 2.2.18 (and soon 2.2.19). Moving to 2.4.XX is not an option
at this time for the other boxes.

I have been working with both boxes with the DE600 card. This has been
a nightmare. I cannot get the ethernet card to reliably work. Under
heavy loads (dd of a file over NFS to an SGI server) I get messages like
the following:

     e01 kernel: eth0: Transmit timed out: status 0050 0c00 at 39554015
           command 000c0000

I have read on this list and on the eepro list that this just happens
with the eepro100 cards. I have not yet found a solution. I have tried
a 3com 3c905B card and this problem goes away. However, I want SRM support
so this is not a long term solution.

I have tried the drivers with the kernels, I have tried Becker's latest
driver (v1.13) and I have tried the Intel driver with Soohoon Lee's patch.
Nothing helps fix the above problem.

Besides this I am having stability problems with both kernels. I have
been running some multi-threaded programs. They are openmp versions
of the NAS Parallel benchmarks using the openmp compiler from rwcp.
I am going to try some non-threaded applications, but still max out
all the CPUs to see how things work.

I just would like to know what kind of setups users with ES40s are using
with Linux?

Thanks,
Craig
