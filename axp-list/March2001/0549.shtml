Hello Alphans,

Now that my UP2000 is humming along pretty well for the most part, I
started to run some benchmarks on it. The CPU benchmark that we use
most often is running FFTs. More specifically, we use the FFTW package
from MIT. It seems to run quite well on all platforms, except on this
Alpha system. We've tried building it with both GCC and CCC, and the
results were similar; the FFTs run at about the same rate as on a Sun
Blade 1000 (750 MHz). If one were to compare SPEC results between
these two systems, one might expect the Alpha (833 MHz) to be almost
100% faster than the UltraSPARC-III. So the problem must be in the
code generated by the compilers; perhaps they simply are not very well
optimized (either CCC or GCC). Are there any special flags (besides
-fast) that will generate better code?

I'll try CXML next.
