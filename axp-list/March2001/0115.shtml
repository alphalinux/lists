For some of us we multiboot into Tru64, OpenVMS, and Alphalinux our Alphas, thus we do not need an Alpha for every O/S or we can test on different hardware platforms. There is only one console that allows this, SRM. Any other option becomes more troublesome to deal with. Esp. if you start talking about flashing some other code into firmware, thus replacing SRM. I believe others like me would prefer effort to be concentrated on enhancing and improving SRM. It has already come a long way from the early days.
