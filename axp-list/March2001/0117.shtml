Richard Polton wrote:
>
> Hi,
>
> While this is not an Alpha-arch question, it is a problem I am
> having on my Alpha so here goes.
>
> I have a SCSI disk which the adapter cannot find (it used to work fine)
> >From the Ctrl-A BIOS screen, I get when trying to access this disk:
>
> Unexpected SCSI command failure
>
> Target SCSI Id: 1
> SCSI CDB Sent: 03 00 00 00 0E 00 70 00 02 00
> Host adapter status: 00h - no host adapter error
> Target status: 02h - error condition
> Sense key: 02h - Not ready
> +Sense code: 40h
> +Sense code qualifier: 80h
>
> Can anyone offer any advice? I have tried fiddling with the cable, the
> card, socket, terminator (active) etc. to no avail.
>
> It is AHA-2940 Ultra/Ultra-W if it makes any difference.
>

If you have some additional equipment, try put the device on another
system and see if it works there. Or, if you have another device, try
it on your Alpha and see if it works. May be a dead drive. Also,
set the drive jumpers so it spins up. Could be it just isn't spinning
up. I think the bios has an option to send a start command on boot
if you can't set a jumper to do it.
