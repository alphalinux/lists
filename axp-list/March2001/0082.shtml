  I just had the same symptom loading 7.0 on an XP1000, the fix was to
  add "-fl 0" to the boot line...

     "b dqa0 -fl 0"

  my default was 1, which ended up using a different aboot config line
  and messing up the cd access. You can fool with this by using -fl 99,
  which will cause aboot to drop into interactive mode so you can look
  at the aboot.conf that is on the cd

  hope this helps
