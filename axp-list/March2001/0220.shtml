On Fri, Mar 16, 2001 at 01:42:58AM +0100, Andrei A. Dergatchev wrote:
>
> I don't have a clear idea of what is going to happen when
> a drive finds a bad block. From one viewpoint, with automatic
> block reassignment of the drive turned ON, the filesystem should
> expect that the drive will deal with the problem - reassign the bad
> block i.e. - and the present logical address should be fine (?).
> From another viewpoint, I'm stuck in the situation when
> a drive does not do reassignment at all, or at least does not do
> reassignment for a few "special" bad blocks, so I'd prefer
> the file system to mark them as used and never refer to them.

These are two different levels. When a drive/device driver is
doing remapping your file system should not see that bad block at
all (you may loose it contents if errors happened on read but usually
you will have first problems with writes). Utilities like 'e2fsck -c'
will just tell a file system that some disk pieces are "used" so
they are not available. 'badblocks' will produce a list for a device
but it is up to a given file system to do something with it. If
you have such block in swap, for example, then it may turn out
to be a real PITA and bad reads of a swapped out stuff may provide an
interesting way to crash. :-)
