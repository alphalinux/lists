I think there are a couple of problems that we are looking at here.

Only Compaq and API are making Alpha mother boards. There are other
manufactures of special products like Force for special applications but
these are low volume and high price.(I could be wrong here but I have
looked and had little luck so I am sure that this is the case)

Of these 2 Compaq is interested in selling only complete systems and
seem to have dropped the old DEC OEM stuff. They also only sell the
Alphas through the server channel which is designed for lower volumes
higher value add and higher prices. It would be nice to be able to buy
an alpha workstation directly off the Compaq web site.

I am not quite sure what API's target market is. I think they are trying
to stick with the higher margin multi-processor and cluster market. the
UP1100 is now end of life and although the 1500 is in the works there is
no assurances from API that they will be selling the 1500 or any single
processor board in the near future.

The cost of developing, producing and selling a motherboard can be quite
high and in the case of the Alpha where the volumes are low the problems
are compounded.

At one time API announced that a deal that would the AMD processor using
the same motherboard interface as the Alpha. I had hoped that this would
mean that there would be a reason for companies to build motherboards
that would support both the K7 and the Alpha. To date I think only API
has a mother board that will do this(UP2000). The CPU's alone for this
board run over $2000 each.

The problem as I see it is not that there are no inexpensive systems
based on the Alpha but no inexpensive motherboards based on the Alpha.

These are only my opinions and observations. Anybody please feel free to
correct me.
