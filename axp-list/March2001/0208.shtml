Peter Petrakis wrote:
> Basicly Tru64 kernel's threading archtitecure is fully POSIX compliant
> where Linux's kernel threading system only implements part of the POSIX
> standard.

Why add functions to the kernel that can be done adequately in user
space?

> I don't think we'll ever see full POSIX threads support in the
> kernel, Linus has said himself that he hates POSIX threads and they suck.
> Linux's kernel threading system act more like lightmweight processes than
> anything else.

Pthreads is an API specification, it isn't required to be implemented in
the kernel.

The major shortcoming of POSIX threads may be that the standard is too
unrestrictive, and not adequate for real applications. Most threaded
applications I have worked on depended on behavior that is not specified
by POSIX, and therefore system-dependent. That's not just a Linux
feature; one library I use has separate sources for Irix, Solaris, Linux
and HP-UX threads. They all have subtle differences in behavior. In
that sense, pthreads has been a failure.
