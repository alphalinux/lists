On Sun, 8 Apr 2001, Miles Bower wrote:

> Rich I have a digital 7300 that I picked up at a liquidation. It has redhat
> 6.2 installed and the password the folks gave me is invalid. I don't see
> any means of getting a prompt that offers an option like single where I can
> reset the passwd. I have been attempting to do boot from the srm console
> but when I do that it comes back to console secure - parameters not allowed.

AH...so you don't know the SRM password? That's a different
story....there's a way to clear it. I'll have to look it up on Monday
(something to do with the HALT button if I remember correctly, anyone??).

> I have also tried setting the bootdef_dev to the cdrom but I get a double
> exception when I go this route. Is there a way to boot from the cdrom and
> get past the system console so I can rescue the passwd?

If you can change the bootdef_dev variable then the console isn't locked.
You should then be able to add 'single' to the boot_osflage, and erase the
root password from the /etc/passwd file
