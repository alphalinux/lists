Hi,

I managed to isolate tha problem, a little ... So, it looks like netscape
works just fine for local users, but, it can not read NIS maps (for the
passwd map) and has a problem with NFS (for nfs mounted home directories).

I did the following test: created a a user with the passwd entry in
/etc/passwd and an nfs mounted home dir. And the NIS problem is confirmed
(for a NIS user it didn't start at all) ! Netscape starts, but cannot
display anything. It hangs up within "Looking up host". If I delete
.netscape/preferences.js, it starts and it works wonderful. However, at
the second run, where the .netscape/preferences.js file already exists,
the same story. Hangs up at "Looking up ...". Stop, Esc, nothing helps.

I have tried all the possible combinations of nsswitch.conf, resolv.conf,
svc.vonf . Nothing. However, I am open to any suggestion.
