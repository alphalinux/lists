Yes, this is correct. The problem is that Linux and SRM use different
methods for numbering devices. SRM thinks of the devices in terms of bus and
hose, but Linux remaps each hose sequentially to a bus in /proc/pci. In
addition, each Alpha system has its own quirks about how it numbers devices.
PWS (Miata) machines will remap IDE devices to SCSI if os_type is set to
UNIX. Machines earlier than EV6 ignore non-PCI buses when assigning PCI bus
numbers, but EV6 and later machines will assign a non-PCI bus a PCI bus
number, and then skip that bus number on all the hoses on the system.

So to figure out what device is attached to what bus and what hose,
JumpStart has to do a lot of guesswork. And since we no longer have any
functioning AS/1000 machines, it would come as no surprise to me to find out
that JumpStart was guessing wrong on something and getting all mixed up.
However, this should not cause the machine to lock up.
