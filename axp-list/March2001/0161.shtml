Alvin,

>Is there any reason that an open source boot loader could not boot
>True64 or OpenVMS?

PALcode, pure and simple. Linux is booting using ancient PALcode from the
earliest SDKs DEC made available, but it works. I have often wondered if
Compaq decided Linux was too much of a threat if it couldn't quash the whole
effort by challenging the use of licensed code all Alphalinux distributions
require to operate (whether MILO, raw or aboot bootstrapped).
