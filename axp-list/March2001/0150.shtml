Peter Petrakis wrote:
> EXT2 support in SRM is nice but whats stopping SRM from loading linux
> off a FAT fs of which it already understands?

When did SRM start supporting FAT?? Is that new?

I strongly agree that SRM could benefit from some sort of filesystem.
That paves the way to boot-time loadable extensions. As it is right now
SRM is like a monolithic executable.

I disagree that Compaq should open source SRM (hardly anyone would want
to touch it) but if they were to open it up a little, bring back the SDK
and maybe introduce an extension mechanism, that could only be a good
thing, no?
