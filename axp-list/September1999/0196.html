Hi,

I'm helping a friend with a dual-boot pcLX-164. The
Alpha BIOS has stopped initialising the cd-rom drive,
and the new ide drive. It still initialises the scsi
drives(DEC and Syquest)fine. It also now fails to run
Linload.exe though it finds it during the Operating
System Selection Setup. It doesn't seem to matter if
the cd-rom drive or the Quantum fireball KA Ultra66 is
connected to the motherboard or the Promise33
controller.
It will boot NT, but since the BIOS doesn't initialise
the cd-rom drive and the ide drive, the NT drivers for
them fail to load. Both linux and NT are on the scsi
hard drive. I don't know if the following info from
the Alpha BIOS will help.

Display System Configuration>Integrated Peripheral
State:
IDE Chnl 0: Enabled [Port 0x1f0, IRQ 14]
IDE Chnl 1: Enabled [Port 0x170, IRQ 15]

Display System Configuration>PCI Configuration
(The Promise Ultra33 controller)
Device Name: VENID=105A, DEVID=4d33
Device Type: Mass Storage
Revision: 1
Physical Slot: 2 (I also tried Slot 0)
(The NCR scsi controller)
Device Name: NCRC810
Device Type: SCSI
REvision: 2
Physical Slot: 3

Any ideas or suggestions will be appreciated.
Thanks in advance.
