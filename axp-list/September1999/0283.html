I've been trying to install a jensen machine too. I am able to use the
ARC utils to parition my drive and copyboot to copy the image
"bootdisk.1.3.72.jensen.srm" to the harddrive. I can then boot the
drive from the OSF console it "boot -fl install dka200". The loader
starts and (after detecting hardware and stuff) asks for the root disk
to load into the RAM disk.

At this point, pretty much whatever image I copy onto a floppy causes
the machine to read the floppy for a while and then die with a pile of
disk read errors.

The image "psuedo-bootdisk" is different. It reads for a while (tells
me it found an ext2 file system at block 0) and then says

"VFS mounted root (ext2 file system) readonly"

and then stops. It sits there with the cursor blinking and does seem to
accept any keyboard input.

The files I used are all from
http://www.alphalinux.org/ftpsite/Linux-Alpha/JENSEN_INSTALL.

My firmware is version 1.9 (should I downgrade to 1.7?)

Anyone have any ideas?
