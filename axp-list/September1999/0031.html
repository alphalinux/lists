Xiaozhou Qiu wrote:
>
> I think you have to type in the entire boot command every time you switch
> kernels. There is no such a boot menu for SRM. You can only save one boot
> option.

While this is roughly correct with a little help of /etc/aboot.conf
(described in /usr/doc/srm.txt) it is possible to have ten "canned"
boot entries with labels from '0' to '9'. You boot them by typing
at SRM prompt commands like 'b -fl 3'. If you are willing to type
things like 'b -fl "2:3"' then you can multiply ten by a number of
ext2 partitions on your boot drive. :-) 'b -fl h' is also available.

Write yourself /etc/aboot.conf and run 'abootconf' utility to tell
'aboot' on which partition it resides (or you can use an explicit
partition number in boot flags). Set in SRM 'boot_osflags' to 0 and
'boot_file' to an empty string (variables names from memory so check
them as here they may be slightly wrong). Then typing 'b' at SRM prompt
will boot this line which has a label '0' in yours aboot.conf file.
Even with SRM set to autoboot switching your default kernel and boot
options is then as simple as editing one line of a text file.

Xiaozhou Qiu, I think that you should add something like that to your
writeups.
