On Wed, Sep 08, 1999 at 01:23:13PM +0200, Christian Iseli wrote:
> Ok. So, if I get this straight, init is using libc itself. Thus what
> happens is that after init quits, the last link to the deleted libc is
> severed, the file is unlinked completely, but at this point nothing
> performs a sync.

Something like that. I remember a detailed explanation of the
problem came up on one of the libc lists.

I think the actual problem is more like (1) init keeps the refcount
of the old libc non-zero, (2) the remount ro of / with open deleted
files forced the dirty bit set (3) init dies.

> Guess I could avoid this by keeping a link to libc in the filesystem until
> after the reboot (something like "ln /lib/libc.so.6.1 /lib/libc.old" before
> upgrading the library) ?

That plus a bit in /etc/rc.d/rc.sysinit or something to delete the
old library would probably be ideal. My / is only 150M and takes
near zero time to fsck, so I just don't bother.
