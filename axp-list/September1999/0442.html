On Tue, 28 Sep 1999, Jakma, Paul wrote:

> Lame also actually compiles on Alpha.. latest bladeenc didn't for me. Which
> means you can try out Compaq libcpml.a and ccc to gain some speed.
>
> So bladeenc users, search for lame on freshmeat, and give it a try.

I sill use both. To get the latest bladeenc to run you must patch
the source. In this case a long int is used in the wav header stuff,
change to int and it work.

Both can be compiled with ccc and this gives a _HUGE_ speedup. On my
Xl300 a 4 min song (bitrate 192) used to take 14 min to encode using
bladeenc. With ccc it now takes 6 minutes. However, lame is a lot
faster, it takes only 3 min. So yes go for lame although i do not agree
that bladeenc is so bad, it is just slow.

Sorry for keeping this tread alive, maybe this is something
for the redesinged FAQ?
