Hello Janina,

   The only method I've ever found to work,
(after installing on several AS200's), is to go ahead
and partition the disk via fdisk (_NOT_ BSD Style),
recording the partition offsets and sizes as I go.

   (BTW, if you're going to be booting from SRM,
remember to leave a few cyl's in front of the drive
for 'swrite boot')

   Then, prior to rebooting the system, I call up
an alt-console, then I do the
'fdisk -b' to the drive, effectively relabeling
the drive BSD style. remember to reset all of your
partition names/types (don't forget swap).
    Then setup the boot with swrite before exiting the
alt console.

    This is loosely documented in the RH6.0 alpha
install addendum. Also, if you peruse the axp-archives,
you'll find an excellent little guide provided to
me by Phillipe (sorry, can't remember the last name
and I'm on a plane), on how to boot/install from SRM,
which very well details the exact steps to follow...
   I _still_ have the hard copy of that email, and
use it as a crib-sheet.

Luck,

Rob
