The original question Ric Tibbetts posed was:
 
> This is another new one.
> On my Alpha (AlphaStation 4/233 w/ Redhat 6.0), If I start up in KDE,
> if I open a terminal, it crashes xfs, and then I'm hosed (can't log out,
> can't do anything but telnet into it, and re-boot it. My current
> work-around is to just use WindowMaker, but I'd like to get the "real"
> problem fixed. It just recently started this.

I/we noticed that xfs crashes when I execute xterm from a
gnome terminal. It looked like any client that failed to
find the fonts resulted in the termination of
xfs. Consequently no X application worked unless you pulled
the machine down to init level 3 and set it back to run at
init level 5 (which meant among other things, shutdown the X
server and bring back X server, xfs, xdm etc.)

I was able to fix this problem by commenting out the line

       FontPath "unix/:-1"

in /etc/X11/XF86Config.

I have not had time to investigate deeper into this but some day....

Comments?
