I take it that you mean you are running the de4x5 driver.

You can stress the system without the network by building the kernel
repeated, or by using the "test" command at the SRM prompt. That
should tell you if your memory is bad, or you have some other hardware
problem unrelated to networking.

For networking, you can download netperf and run some big tests using
it.
