On Sun, 19 Sep 1999, Paul Jakma wrote:

>
> I have xfstt running on an ia32 machine serving fonts to 3 other
> machines (ia32/alpha), users running windowmaker, gnome and kde, and
> it works fine.
>
> Problem is probably jsut specific to RH6/alpha. xfs is straight from
> the X11R6 codebase, so it's not linux specific.
>

I encountered the same troubles, and ended up nuking xfs and placing
FontPath statements in XF86Config. I don't think this xfs problem is
limited to Alpha, though. We were having trouble with Netscape
crashing when trying to run JavaScript under KDE on RH 6.0 on a PII.
I read (on this list I believe) that the xfs problem can affect Motif
applications, so I applied the same "fix" to the PII and now Netscape
is stable.
