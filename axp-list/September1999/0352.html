>Installing RHL 6.0/Alpha. And this is probably a FAQ.
With presumably a lot of new Multia users this is going to become an FAQ :-)

>I'm having Install problems. Here's what I did:
>
>downloaded from alpha/milo
>udb-noname.img -> created MILO disk
>
>downloaded from alpha/images
>generic.img -> created Kernel Floppy
>ramdisk.img -> created Ramdisk Floppy
>
>Replaced Battery in the Multia!
>Switched to arc console from SRM console
>Setup boot options for MILO as explained in RHL Alpha Install Guide
>Booted MILO Disk just fine
Til here it looks OK.

>
>=> Problem started here.. The following did not work (from install guide)
>MILO> boot floppy
What made you try this?

>
>Then I tried:
>MILO> boot fd0:vmlinux.gz root=/dev/fd0 loadramdisk=1
And why this? Incidentally, did you read the information on TheLinuxStore's
web-site about the Multia? It does specify that it is for RedHat 5.2.

Anyway, the correct boot command line is this:

MILO> boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1

Try that out,

Peter
