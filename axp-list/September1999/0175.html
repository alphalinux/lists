On Thu, 9 Sep 1999, Greg Lindahl wrote:

> Against my better judgment I'm using the de4x5 driver, because I have
> some bizarre problem that goes away when I use it instead of the tulip
> driver. However, after a lot of activity, it has an annoying bug where
> the timer used to expire TCP sessions seems to stop, and the machine
> gradually runs out of TCP ports. At that point it's pretty FUBAR.
>
> Has anyone seen this? Any fix?
>

What kernel version and what version of the de4x5 driver do you use?

I use the de4x5 driver on quite al lot of machines and have never had any
problems with it. Also, I think that the timers of TCP sessions live on a
different level in the kernel. To me it sounds more like a 'generic' TCP
bug that you happen to trigger.
