Did you check if this actually is a proper block dev? From 'ls -l'
you should get:

brw-rw---- 1 root disk 8, 0 Jun 9 16:16 sda

i.e. it should be (8,0) for major/minor dev numbers.

Otherwise, the fdisk -b should work unless something's badly broken.
