> > > Is it possible to remote boot the Linux with SRM Console?

> > There is a section (3.7) in the SRM-HOWTO that describes how to boot
> > Linux/AXP via bootp using the SRM console. One divergence is that in my
...
> There is so many environments about bootp on SRM console.
> For example ewa0_arp_tries, ewa0_bootp_file, ewa0_bootp_server,

Don't need to dork around with most of that unless you want to set
defaults. Set ewa0_protocol to bootp, then use:

>>>b ewa0 -fi remote-file

There is also an srmcons.pdf file which explains most of the SRM console
features and operations in better detail. Try:
http://ftp.digital.com/pub/Digital/info/semiconductor/literature/srmcons.pdf
