Dear list,

I built a egcs (1.1.2) cross-compiler to run on my i686 and build binaries
for my multia. It compiles simple programs (like hello world) which run
correctly on the target machine.

My ultimate goal was to be able to compile my kernels on this machine. The
kernel (2.2.12) configuration I am testing with compiles and executes
correctly on the alpha itself.

But when I compile it with the cross-compiler the kernel compiles
correctly. It seems to boot ok. But when you get to the login screen
something strange happens. You can type and login but your password is
displayed (rather than hidden), it then logs in and you can see a prompt,
but it then immediately logs you out again. httpd works, and I can ping
the machine, but I cannot telnet to the machine.

Any suggestions?

Thanks in advance,
Ken
