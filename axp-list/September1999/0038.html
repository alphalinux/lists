Jeff DeMaagd wrote:
>
> Is (x|g|k)dm running somewhere? It could be running and trying to
> spawn a new X server when one is already running.
>
> It may be in the init directories somewhere.
>

This sounds likely given the error message. To fix it,
edit /etc/inittab the way you want it and then do
"telinit q" to have init reread the file. In particular,
look for the statement at the top of /etc/inittab like:

id:3:initdefault:

A '3' means start with a normal text console. Use '5'
if you really want to run xdm or it's relatives.

You can also just manually switch run levels with telinit.
For example, "telinit 3" puts your system in run level 3
which will disable xdm.
