Larry Snyder wrote:
> > On my Alpha (AlphaStation 4/233 w/ Redhat 6.0), If I start up in KDE,
> > if I open a terminal, it crashes xfs, and then I'm hosed (can't log out,
>
> This has come up several times in the last month or two. The current
> workaround IIRC, is to nuke xfs and hard-code the fontpaths in
> XF86Config. I know that worked for me.

Yeah... I thought of that one already, I was just hoping for a more
solid answer. I'd like to use the Alpha as a font server for other
machines (I was actually happy to see font service included!). Then I
can set up all the fancy fonts, TT fonts, etc. on one machine, and use
them from all of them (I've done this on large production systems under
AIX, & HP-UX, and it works out great!). But it seems that the Linux
implementation of xfs just isn't quite there yet.

What I find interesting is that it seems stable under other
WindowManagers. WindowMaker, for example, seems perfectly stable. Could
I dare to suggest that the breakage is in KDE?
