On Tue, 7 Sep 1999, Christian Groessler wrote:

> I've got this program:

[...]

> If I try to compile it I get:
>
> --------------
> [chris@alpha:~]$ ccc -c ccc-bug.c
> cc: Error: ccc-bug.c, line 4: Missing ";". (nosemi)
> XXX(0ea) { return; }
> ^
> cc: Error: ccc-bug.c, line 4: The type of the tentatively-defined variable
> "bla0e" is incomplete at the end of the compilation unit.
> (incomptent)XXX(0ea) { return; }
> ^
> [chris@alpha:~]$
> --------------

This does not happen under Digital Unix (Tru64) 4.0E:

csh> cat l.c
#define XXX(xxx) void bla##xxx(void)

XXX(08a) { return; }
XXX(0ea) { return; }
csh> cc -c l.c
csh> cc -V | head -1
DEC C V5.8-011 on Digital UNIX V4.0 (Rev. 1091)
csh> cc -E l.c
# 1 "l.c"

void bla08a(void) { return; }
void bla0ea(void) { return; }

So I think you should file a bug report.
