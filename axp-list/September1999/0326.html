    My Jensen install is moving along slowly, but I seem to have
troubles getting aboot to boot my kernel image.

    I did the following:

1) Burned floppies of generic.img and root.img from the redhat 6 alpha
install set

        dd if = generic.img of=/dev/fd0 bs=8192
2) Downgraded to the 1.7 firmware from 1.9
3) Set my floppy environment variable to "1"
4) Used minlabel to label my disk
        There is a 4MB partition (type 8-ext2, not type 9-boot) and the
of the disk is a big ext2 parition
5) Used copyboot to copy generic.img off the floppy to /dev/sda
6) Switched to OSF console
7) Booted with "boot dka200"
8) Typed "1/vmlinux.gz load_ramdisk=1 root=/dev/fd0H1440"

aboot told me that it was reading the kernel image (vmlinux.gz) but then
stopped with

"unzip: attempted to read past eof"

Can anyone offer any ideas?
