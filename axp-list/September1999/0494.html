The following is not specific to Alphas, but I wouldn't
know that if I wouldn't have tried it on a x86, too. ;-)

I compiled PVM 3.4.2 using gcc-2.95 on a PC164 (2.0.36)
and on a Celery (2.0.35) with more or less modified RH 5.1
installations. These two systems are the onlz ones in the
network, whcih otherwise works flawlessly (for me, it's
my home network).
If I try to add "the other" system at the PVM console
(the problem is completely symmetrical) the console
seems to hang for some minutes, the system to be added
to the virtual machine never gets actually added.
The PVM-Logfile of the called sytem shows lines like

netinput() bogus pkt from [inadport_decimal(&osad)]

Has anyone else had this problem? Is there a solution
or a workaround?
