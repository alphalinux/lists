> When I used the RH4.2 on AlphaPc 164LX before, the date changed from time to
> time after rebooting. Sometimes it is back to about the beginning of the
> century. Mostly it goes to the future time like 2019, 2039, etc.
> I really don't like it. After installing the RH6.0, the problem seems still
> there. Can anyone let me know how to fix it?
>

  Try this:
 
  Edit /etc/sysconfig/clock and make sure UTC and ARC both "=" false.
  Set the correct time with 'date'.
  From root, do a 'clock -w'. This will set the CMOS clock to the
  correct time.
  
  This may only work is you are using SRM console - not
  sure. Anybody??
