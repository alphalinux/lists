On 10 Sep 1999, Eric Ding wrote:

> are there any glibc 2.0/2.1 compatibility issues of which I should be aware
> when compiling a C/C++ application on Linux/Alpha? On the x86 platform,
> we've been able to compile with egcs 1.1.2/glibc 2.0 and run succesfully on
> RedHat 6.0 and other similar glibc 2.1 platforms.
>
> Will we be able to do the same on AlphaLinux? Will stuff compiled on RH
> run on the Debian (or other) flavors?

There seems to be only one issue that I can find between Debian and RH
(__setjmp symbol availability). Other than that, I frequently test-run RH
software on my Debian system. I'm looking more into the symbol problem
now and, if it's a Debian "problem", then I'll correct it (I compile the
glibc, binutils, and gcc packages for Debian on Alphas).

I would be happy to test out anything you'd like on a Debian Alpha if
you'd like (NDA is ok, if needed).
