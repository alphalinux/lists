> > Anyone knows if more than 2GB memory is already supported or if not, when
> > will it be?
>
> Current constraint of 2GB memory is because the PCI bus-master
> device drivers usable at Alpha are not yet converted to use
> bounce-buffers (or whatever) in case the physical memory
> address is not within the first 2 GB of main memory.
>
> To do it will require some non-trivial work.
> (and people with machines with lots and lots of memory..)

That's absolutely ridiculous. All the machines in question have both
scatter/gather maps as well as multiple direct map dma windows- both can
address more than 2GB of primary memory. It's a software issue of how you
generate DMA mappings that's an issue. I can't believe somebody has
mentioned 'bounce buffers' for this. Tchaa.
