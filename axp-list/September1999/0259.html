On 14 Sep, Michal Jaegermann wrote:
>> So i changed my conf.modules to
>>
>> options de4x5 args='eth0:fdx autosense=100Mb'
>>
>> But i get no connection :-( (ifconfig -a shows eth0 up but a ping hangs)
>> When using the old options it connects fine but with only 10Mbit
>>
>> Any Ideas ?
>
> Yes. Don't use de4x5 driver. I have never seen it working above
> 10Mbit. Depending on your card you may need to use some current
> version of a 'tulip' driver (an address in sources). 0.89h included
> with a kernels does not cut for many chips either.
>
On 15 Sep, Daniel Potts wrote:
> On Tue, 14 Sep 1999, Max Weninger wrote:
>
>> Hi,
>>
>> My system is a Ruffian UX 600MHz
>> kernel 2.2.11 (de4x5 module version 0.544)
>>
>> I use the de4x5 ethernet module and it works fine since the last months
>> on our 10Mbit network with the following options in conf.modules
>
> You should be using the tulip driver is you want 10/100Mbit (21143
> driver).

I tried it with the tuilp 0.89h it works but "sometimes" i get timeout
or the complete net connection breaks :-(

I tried it also with the "latest" version 0.91g but this version does
not work at all (no connection).

Does anyone know which version of the tulip driver work for the chip
used by Ruffian UX motherboards (or which options to use)
