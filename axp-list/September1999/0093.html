I've had success with:
Netscape Commmunator 4.x
Adobe Acrobat Reader 3.x and 4.x
CVSup

Using DU 4.0 D libs.
With acroread you have to move the existing pthread.so and pthreads.so
out of the way and instead link those names to libc.so. Not doing so
will cause acroread to die with a failure to call DECthreads (in the
kernel?). I once helped a gentelmen try and get DU Mathematica up and
running though I lost touch with him and I'm not sure if he succeeded.
We got the Mathematica kernel working but the front end would die. I
don't remember the error.

I suggested installing the i386 Linux version of Mathematica using em86.
In this case the front end works but the kernel dies when it starts. So
you could run it so you had the emulated frontend and the Alpha native
backend. I suggested this and then never heard from him again. Maybe it
worked , I don't know. I really haven't had access to other DU
applications to test them.
