On Wed, Aug 09, 2000 at 03:23:20PM +0200, Vock Reinhard wrote:
> Hi folks!
>
> I was wondering if there is something like a tool to produce bootable
> backups under linux? I am refereng to something like ignite/UX from HP;
> build an image of the machine, wirte it to tape, boot from tape and either
> just boot or rebuild the machine?
>
> Thanks,
> Reinhard
>

I too would like the same thing. That way, when a hard disk crashes, I can
replace the hard disk, boot from tape, and have the system recreate itself.

Yes, I know, the traditional Linux approach to this problem is to boot from
floppy, load a vanilla Linux from CD-ROM, and then restore a tar archive from
tape. Alternatively, the tape carries a dd archive. In that case, I boot from
floppy, and un-dd the tape onto disk.

The problem with the tar approach is that I have to re-partition the disk
before I can de-tar. I have to somehow remember the sizes of the partitions,
or extract that information from the tar tape.

At a previous job in the Old Days, we used dd to 9-tracks to save and restore
partitions on BSD systems on Vaxen. There are several problems with the dd
approach. I have to replace the hard disk with a similar hard disk. If I
install, say, a larger hard disk onto the system, I'm not sure if the dd will
work. At the least, part of the new disk will be unuseable without
re-partitioning. And I'm very sure that if I install a smaller hard disk,
whatever data was at the end of the last disk partition won't be restored, and
the partition probably won't even fsck correctly. Also, if there's an error on
a dd tape, you potentially can lose much more than just a file, which is what
you'll likely lose with an error on a tar tape.

Also, I'm less likely to mis-place the backup tapes than I am to mis-place the
emergency boot floppy and Linux CD-ROM. Why? I make backups more frequently
than I restore from floppy. So I have handled the backup tapes more recently
than the floppy. So I'm more likely to know where the backup tapes are. If I
can reboot from those backup tapes, I save myself the step of having to search
for the boot floppy and the Linux distribution CD-ROM.

I do know about having a spare root partition from which to boot. We used the
same trick on Vaxen, 15 years ago. We referred to these as shadow roots. I'm
reluctant to have to allocate space for, and maintain, a shadow root partition
on each system I oversee. It also means that I have to have a second disk
drive on each system.
