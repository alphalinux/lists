Helge Kreutzmann wrote:
>
> As I understand the docs, I install aboot one time using swriteboot and
> then I can configure all bootable installations in /etc/aboot.conf, so
> if I have Linux in A: and B: I can boot both with appropriate lines in
> /etc/aboot.conf on A: ?

The easiest way to tell aboot on which partition it should look for its
configuration file is with a help of 'abootconf'. Run the later without
any arguments and you will get all documentation you need. :-) In
'aboot.conf' on each label line you can specify explicitely, if
applicable, from which partition (and a location on it) of a boot device
to load your kernel and on which device/partition (it does not even has
to be the same as your boot device and even less the same partition from
which you loaded your kernel) is located your root file system.
Keep in mind that all required kernel modules will be expected to
reside on your root fs and not necessarily with a kernel you load.

There is this "if applicable" above because your boot device can be
a floppy or CD, even if a root file system is on a hard disk, and then
you do not have any partitions present at all.

In practice with SRM and aboot I have usually things set up that way
('boot_flags' in SRM set to 0) that single 'b' at SRM prompt will use a
boot command specified with a '0:' label in 'aboot.conf' and I may have
a few other boot lines ('b -fl i' if I forgot what precisely they are).
