On Mon, 28 Aug 2000, Jeff Sturm wrote:

> > [~$] ldd /usr/lib/netscape/netscape-navigator
> > libBrokenLocale.so.1 => /lib/libBrokenLocale.so.1 (0x40021000)
> ^^^^^^^^
> Those library addresses are 32-bit... are you sure that's not an x86 binary
> running under emulation?

 What you point at is actually not a virtual address but a value the
shared object got relocated comparing to the base address stored in the
ELF file. In other words it's a difference between these two addresses,
which doesn't tell you whether the virtual address the object gets
mmap()ped at fits in 32 bits or not.
