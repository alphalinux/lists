Larry Snyder wrote:
>
> What hacks did you try? I tried building flash on DU4.0(no letter).
> The build broke big time.

You mean your compiler barfed on sources or results were not acceptable
to Netscape? The first should be fixable with a help of any editor. :-)
For the second I would expect that similar flags (-taso?, -xtaso?)
which were used while building Netscape would be also needed for plugins
to ensure that interfaces and memory models agree. This is a guess.

I am not sure if the current binutils for Linux, which have -taso
(or similar) switch see memory the same way as DU binaries compiled
with -taso. If not then attempts to produce on Linux object files
in COFF format, instead of ELF, may run into some snags.
