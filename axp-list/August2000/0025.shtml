> Something that gives the option of direct I/O (no file
> system caching) like XFS would be very good to have.

The cost of filesystem caching can be an extra copy. The Alpha has a rather
great copy speed.

> I'd hate to see
> something like, for example, connecting a FCAL RAID system capable of
> over 100 MB/s transfer rate to my new Alpha Linux box and see no more
> than 20-30 MB/s of effective throughput due to file system overhead.

I have seen 50 MB/s to ext2 filesystems over a single FCAL link, for large
writes. That's not as good as it should be, but one extra copy isn't the
guilty party.

> One thing I was looking at is GFS (Global File System), which offers
> direct concurrent access to the same file systems by multiple hosts.
> This feature is highly desirable as well, but I have no information
> on how efficient it is. It also only works with certain hardware with
> support for certain SCSI extensions.

GFS can be installed a variety of ways; you don't have to have the special
hardware.

We're using CVFS (CentraVision FileSystem) at FSL, but unfortunately they
have yet to send us a full-speed client. On small SGIs, they get up to 170
MB/s for a single client reading over 2 FCAL links. There's no reason why
the AlphaLinux client should be any slower, but the work isn't done yet,
alas.
