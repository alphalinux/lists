  We've been running 100bt fdx on XP1000s, which I think are very similar to
  DS-10s.

  I had all sorts of troubles with the tulip driver, various versions,
  but finally found a stable configuration with the de4x5 driver. The
  tulip driver, IIRC, would start out OK, then decide to drop back to
  10bt after a while.

  Anyway, the details:

     2.2.13 kernel
     XP1000 with 512 meg ram

     in /etc/conf.modules

        alias eth1 de4x5
        ## 100bt fdx
        options de4x5 args='eth1:fdx autosense=100Mb'.

     eth1 (from lspci) is "Digital Equipment Corporation DECchip 21142/43 (rev 41)"
