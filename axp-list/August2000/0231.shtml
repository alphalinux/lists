Alan Keane wrote:
>
> Hi,
>
> When I've used a tulip driver in the past I've noticed that it has pretty
> lousy
> performance if you use a switch as opposed to a normal ethernet hub.

Tulip does have problems with the built-in chips on XP1000
and DS10 although I did get it to work in half duplex mode.
de4x5 also has it's problems as mentioned previously in
this thread.

> You may want to try that.
>
> We found that switching to the de4x5 driver fixed the switch problem,
> are you setting the card to auto-negotiate at the SRM?
>

This might have some effect on the XP1000/DS10 but wouldn't
do anything for the other dozens of cards that tulip runs since
SRM can't do anything with them. The Netgear fa310tx for example.
It shouldn't ever make a difference if the drivers init the
card correctly without assuming something was already done for
them to the card.

The main problem I see using de4x5 is it typically can't put an
XP1000 into 100Mb mode on the first try. Most likely it doesn't
set some bits correctly when initializing the card. Reloading
the de4x5 module corrects the problem.
