Jemiolo, John wrote:
>
> True64 (Unix) and Linux, have incompatible partition requirements, although
> both use bsd type labels.

For example??? I really would like to know. I failed to notice
anything the last time I tried. Linux has more leeway with
partitioning but these are not requirements.

> Put them on Different disks.

I agree. But this is a question of rather limited capabilities of a
boot loaders then partitioning. That one from Tru64 seems to have
really thin options about what and from where to load. 'aboot' used
with Linux can easily load few different installation of Linux from the
same disk (see 'aboot' docs for details and in particular how to specify
a partition on which a boot image resides) but I doubt that it can load
Tru64. So, if you would have both Tru64 and Linux on the same drive
then you would have to leave a boot block to Tru64 loader and you would
have to start 'aboot' from a different media (floppy, CD or ... another
hard disk).

I have no idea if one can make 'aboot' to load FreeBSD. Not
"out-of-the-box" but maybe some hacking is feasible?
