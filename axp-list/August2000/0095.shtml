for the "new" raidsystem it should be raidhotadd <md dev> <dev to be added>

for the old raid stuff (aka what is in a vanilla 2.2.x kernel) look at ckraid.

Mike Tibor wrote:

> On Sat, 5 Aug 2000, Franco Tassone wrote:
>
> > Hi,
> > today our raid 5 on our UX ruffian went out of sync, due to an H.d.
> > overheat.
> > The raid 5 was built using only three linux fd partitions (sda3, sdb2,
> > sdc2), and linux kernel signalled sda3 fault; it continued working using
> > only sdb2 and sdc2.
> > A new H.D. was provided, now how can I resync the whole raid, without
> > loosing data ?
>
> I believe you need to use the command "raidhotadd <raid dev>". For
> example, if the array with the failed disk was /dev/md0, you would install
> the replacement disk, fdisk and set the partition type to fd as you did
> with the rest, then type "raidhotadd /dev/md0".
>
> You might want to take a look at the linux-raid howto and mail archives
> just to make sure though.
>
> Mike
