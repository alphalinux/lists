Hello AXP/Linux people,

I was wondering about file system performance on Alpha Linux systems.
Having no experience with such configurations (will get one soon), I
have no idea how efficient (or not) the "standard" Linux file system
is on Alpha.

I read that SGI has ported XFS to their Linux systems, but it seems to
be very specific to their systems. Is there something equivalent for
Alpha Linux? Something that gives the option of direct I/O (no file
system caching) like XFS would be very good to have. I'd hate to see
something like, for example, connecting a FCAL RAID system capable of
over 100 MB/s transfer rate to my new Alpha Linux box and see no more
than 20-30 MB/s of effective throughput due to file system overhead.

One thing I was looking at is GFS (Global File System), which offers
direct concurrent access to the same file systems by multiple hosts.
This feature is highly desirable as well, but I have no information
on how efficient it is. It also only works with certain hardware with
support for certain SCSI extensions.

thanks for any information you can provide.
eyc
