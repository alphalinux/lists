I have not researched this yet, but from what I gather they are replacing the firmware image with their own? If this is the case it will not reduce the code base needed to be maintained for booting linux. Many of us multi boot, esp. in the alpha linux world, getting rid of SRM/aboot will not help us. So this just means another boot form that is maintained. It sounds like it has its advantages, but do not think this will diminish the code base.
