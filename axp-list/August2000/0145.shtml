Michael Quinn wrote:
>
> The cool way of doing it would be to retain partition info for volumes.

'sfdisk' from util-linux will write this information in a form which
is suitable for feeding it back to 'sfdisk' to repartition drives but
it does that on a disk-by-disk basis. True, it assumes that a partition
table used is of "FAT type".

I wrote a somewhat similar tool, called 'bdsfdisk', (less complicated
and capable than 'sfdisk') which writes from stdin data "BSD type"
labels of a kind used by Linux on Alpha. The later means, among other
things, no support for "BSD subpartitions" hidden in "FAT partitions"
like used by various BSD* cousins as this was not needed.

The above does not store all this info on a tape which means that you
would have to keep that, say, on a floppy and to know where it is after
a disaster. :-) Probably together with a copy of fstab and other
relevant information. You would have to boot to a ramdisk and
repartition your disks using this info before restoring. OTOH this is
likely such infrequent event that I am not sure if more effort in making
such tools would be really warranted. But if somebody feels
otherwise then fire up your editors.
