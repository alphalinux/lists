On Fri, 25 Aug 2000, Ben Rusiecki wrote:

> To all:
>
> I'm setting up a 3 node cluster and I seem to be having a problem with the
> hosts.allow and hosts.deny file.
>
> Basically, I want to allow only certain services to certain hosts.
>
> My hosts.deny file looks like this:
>
> ALL: ALL
>
> My hosts.allow file looks like this:
>
> in.telnetd: master, compute1, compute2

Remove those commas and you should be okay. I usually use IPs in mine
though, and consolidate it to just the hosts.allow file list this:

portmap: 192.168.1. 127. : allow
ipop3d: 192.168.1. 127. : allow
imapd: 192.168.1. 127. : allow
in.telnetd: 192.168.1. 127. : allow
in.ftpd: 192.168.1. 127. : allow
rsync: 192.168.1. 127. : allow
ALL : ALL : deny

For more info do a "man 5 hosts.allow".
