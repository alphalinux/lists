I'm porting to Alpha Linux. I have a development
machine (Personal Workstation 500a aka "Miata") and a
test machine (AlphaServer 400 aka "Avanti"). The
Miata is running like gangbusters. The Avanti keeps
wigging out. Both running RH6.0

I've had three different set of symptoms all leading
to the same solution: re-install. Here are the
problems:

1) I deleted my app and found that two directories
were totally wrong. That is, the permissions looked
something like "?-rw-xrw-rwT" and the file sizes were
astronomical (like 4 terabytes). I couldn't delete
them or do anything to them. I rebooted so I could do
an fsck and found so many errors that it finally gave
up. Re-install.

2) I was running along and suddenly my app crashed.
Looking at /var/log/messages I saw there was a
filesystem corruption message being posted there
(unfortunately I forget the message). Looking around
the Internet I found some references to this message
but no conclusive solutions. I resolved to try a
newer kernel. Rebooting to do an fsck ended the same
as above: Too many errors, re-install.

When I compiled my own kernel (2.2.16), I wasn't able
to get it to boot. It's gets to "OK, starting the
kernel" and then just stops. Go back to 2.2.5 kernel
from install.

3) Running along, app crashes. Can't kill process.
Reboot. fsck on bootup finds so many errors I give up
and decide to reinstall.

Is this a known issue? Is it my hardware? Is it the
kernel version? If so, how to roll my own?
