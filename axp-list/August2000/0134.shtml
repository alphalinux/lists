> I am still puzzled why you are averse to using SRM?
> SRM is available for most if not all the platforms now, and at no cost.
> It seemingly supports all the options and features that people need.
>
> I agree that with NT out of the picture ARC and AlphaBIOS are now superfluous.
> SRM works well for everything else..

There is not a single reasion that I dislike SRM. I do not like having to
use BSD disklabels and partitioning. I know there are better tools but it
is that I have a long memory of having to set down and calculate
cylinders/heads/sector numbers. I also like the ability to use the same
set of tools to partition and manage disks that I use on X86. Using SRM
requires a second level boot loader and I just feel that there should not
be a need for that given the amount of flash memory on these systems now.

It is just an issue of personal preferance.
