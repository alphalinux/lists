Steven N. Hirsch wrote:
>
> On Sat, 5 Aug 2000, Speedracer wrote:
>
> >
> > This arises from the fact that the no-names and multias they have the 2.8
> > mb drives.

No, generally they don't. The only thing is that firmware does not
check or reports floppy drive capacities so they end up described as a
possible maximum 2.8 but this is really immaterial. Formatted media
have a density recorded anyway so this may be only essential when you
are formatting a "really blank" disk. Then it is likely better to use
'fdformat /dev/fd0H1440' instead of 'fdformat /dev/fd0'. Otherwise this
should not make any difference.

The current 2.2.17pre kernels incorporate a fix which writes an extra
byte to a floppy controller (in 'arch/alpha/kernel/smc37c669.c'
there is an extra 'outb(0xc, 0x3f2);' just before
'SMC37c669_disable_device( IDE_0 );', but this was supposedly for
a benefit of UP2000 which otherwise sometimes "forgets" that a floppy
is there).

> Also, this doesn't explain why the serial
> ports also refuse to work.

No, it does not. But what 'setserial' has to say?
