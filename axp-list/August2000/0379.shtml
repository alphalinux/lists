> The Tru64 binaries, to my knowledge, use a flag to the
> Compaq compiler to generate 32 bit code. From what I've been told,
> this switch doesn't work on the Linux port of the compiler, so we're
> stuck with Tru64 binaries until Communicator 6.0 (Mozilla).

There are a couple of different switches in question, and it's worth
pointing out that the -taso switch is supported by recent versions of gnu
ld. This lets applications that occasionally truncate pointers to 32 bits
run successfully -- pointers are still 64 bits, but the top 32 bits are
always zero. I've used this switch to run some grungy old Fortran programs
that assume that pointers fit into a default-sized integer.
