On Mon, Aug 07, 2000 at 05:04:14AM -0700, David Webb wrote:

> I'm running RH6.2 on a pc164lx. I wanted to try
> using the Compaq compiler to compile the
> 2.2.16 kernel that I downloaded from
> alphalinux.org.

I'm afraid, that's not a good idea. The Linux kernel can, at the moment,
only be compiled by GCC. But that's not as bad as it might seem -- the
linux kernel does not contain much performance-intensive code, so
optimizing the linux kernel would not result in a big speedup of your
system anyway.

Try ccc on some CPU-intensive application program (scientific programs
are usually best for this...), and you will have more success.

As for the specific instructions on how to use ccc instead of GCC:

> Will changing "gcc" to "ccc" in the two following
> lines in /usr/src/linux-2.2.16/Makefile invoke
> the ccc compiler instead of the gcc compiler?
>
> "HOSTCC =gcc"
>
> "CC =$(CROSS_COMPILE)gcc -D__KERNEL__ -I$(HPATH)"

Yes, it would. Other programs could probably work in a similar way.

> Will "make" pass options such as "- fast" to the
> compiler ( make boot - fast)?

No. You have to change the CFLAGS variable for that.
