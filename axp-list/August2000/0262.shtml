On Mon, 21 August 2000, Ron West wrote:
> <P><FONT SIZE=2 FACE="Arial">We are porting our applications over from SCO Unix to Linux.  Our user applications are not written to take advantage of the GUI, but are all Borne or Korn shell scripts executing from the command line.  One command that executes from the .profile is the SCO setcolor command which changes the forground and background colors of the user's screen.  I (a new Linux admin) have looked everywhere for an equivalent command, but have found nothing, no referrences to anything other than DIR_COLOR which isn't what we want.  Is there a way to do this?  I simply want to color the screen at the Borne shell blue with white characters.</FONT></P>

I'm guessing here, but I suspect it will be tricky.

Does DIR_COLOR work for setting colours for an ls?
If so, look at the source for ls, and see how it changes the colours. It's probably something in the termcap, i.e. terminal-type specific escape characters. Maybe you could find the excape sequences used in the termcap file if you know your terminal type is "linux" or similar?

Are you never expecting people to run these from within the X windowing system? If not (i.e. they might use X) why do you want to override the users' own color preferences. As a colour-deficient computer user I find that other people's choices can be nighmarish.
