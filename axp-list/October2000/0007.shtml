Hello,
This is kind of an off the wall question because I don't know much of
the specifics, plus I am new to sys admin, especially with the alpha.
Here goes - We have an alpha machine running RH 6.2 w/
kernel 2.2.16. We have a fortran program that writes out a binary data
file (DMP file) for each timestep of calculation, plus
a number of temporary (TMP) files. We also have a number of
GUI programs written with C and motif. Everything works fine on our
alpha machine (dual 667 MHz). We have a customer who is running the
same operating system and kernel, and the only obvious difference is his

is a dual 750 I believe. He can write out the TMP files, but not the
DMP file. It seg faults as soon as the program opens the DMP file to
write. Also, the GUI programs seg fault when he's accessing the disk.
One wierd bug is if he uses one of the GUI programs compiled for 1 GB
Ram (he has 2 GB), it crashes if he does a specific function, but if I
compile it for 128 MB Ram, it doesn't crash. Obviously this is no good,
he needs to be able to use a lot of Ram.
My question is this: what should I look into on his machine to setup
the memory correctly? Usually we don't
support our customers hardware problems, but we suggested to him that he

buy the alpha so we feel a little responsible.
Thanks,
