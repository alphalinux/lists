I've been using /etc/initscript for years with 2.0 and 2.2 to raise
stacksize, and it works on both x86 and Alpha. I'm not sure what bug you
mention, but it isn't really relevant; if you change the stacksize as root,
it only affects that one process, and not all user processes. Most people
who want to raise the limit want to do it for everyone. This script works
because it raises the limit for the "init" process, and then every other
process inherits the raised limit from init.

$ more /etc/initscript
ulimit -Hs 2048000
ulimit -s 2048000

eval exec "$4"
