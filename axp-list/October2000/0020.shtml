Greg Lindahl wrote:
>
> > I've got a fairly large program that I've ported to
> > Linux (Intel and Alpha). Running some stress tests on
> > the Alpha I found that I was leaking memory (enough to
> > bring the machine down, eventually).
>
> RedHat 6.0 is ancient. Even if you discovered where the leaks are in, say,
> libc, they're probably subsequently fixed. You should really upgrade before
> you waste too much time chasing fixed bugs.

Greg's right... the glibc in RH6.0 had a lot of problems. Upgrading might help
your problem, and eliminate some unaligned traps as well.

But if you are serious about a leak detector for Alpha, why not try the
Boehm-Demers-Weiser garbage collector at
<uri:http://www.hpl.hp.com/personal/Hans_Boehm/gc/index.html>? It works well on
Alpha. It's main purpose is not as a leak detector of course, but it can serve
that purpose. See <uri:http://www.hpl.hp.com/personal/Hans_Boehm/gc/leak.html>.
