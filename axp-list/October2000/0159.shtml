Another thing that you may want to try is compiling a specific kernel
(2.2.13 or later... 2.2.12 had a problem w/ dropping the keyboard and mouse
on the server when it booted). I only had a problem with things dropping
when I tried running RHat's stock kernels, and not the ones that I compiled
myself. When I had the same problems, I looked in the obvious places too
(like /var/log/messages) and found nothing to indicate what was going wrong.

I've had uptimes in months (until I decide to re-organize my room and reboot
the server).... but I haven't had any problems since I've upgraded the
kernel (telnet, ftp, etc).

Subject: DEC Server 3300 inexplicably locks up

I've got a digital Server 3300 running Red Hat 6.2. The install went
perfectly. The machine has been idle all Summer (basically running Seti@Home
until I had time to work with it).

In the first few months of non-use, I had one or two incidents where it
would lock up for seemingly no reason at all. It never did it while I was
working on the console, but occasionally just telnetting to it would lock it
up.

Now I'm trying to bring it online as a production box. I've been getting
more lock ups as I use it more. I can't find a pattern to what causes them
or why. This week I FTPed a bunch of files (500MB) to it from home and tried
to FTP those same files to a local machine at work. When FTPing from the
3300 I watched the transfer rate drop quickly from hundreds of kb/sec to
nothing -- the machine locked up. It was an orderly loss of transfer speed,
about 5-10 kps every second.

It just died on me again mid-SSH-session. I wasn't doing anything but
creating dirs, etc...

When left alone, it runs Seti@Home for months at a time (also running
Apache/PHP/MySQL daemons, but with NO traffic).

Any clues about 1) what could be going on or 2) where I could look for more
info about what could be going on (nothing to see in /var/log/messages). I'm
using the tulip driver for ethernet (it's got built-in DEC Ethernet).

Thanks!
