The Alpha support in the development compilers was one of the main
reasons we choose to use it.

Compaq was demoing povray rendering races of gcc versus ccc in our
(Red Hat's) booth during ALS. I didn't really like how much ccc was
beating gcc (being a fan of Free Software). So I asked Jim, "Which
gcc did you build that with?" He said "egcs 1.1.2." I said, "I'd
like to try rebuilding povray with the 2.96 snapshot." So we built it
with "-O6 -ffast-math -mieee -mcpu-ev6". ccc no longer hands us our
hat when it comes to the povray render race. There were a few scenes
where the gcc 2.96 built povray *beat* ccc. Yes, the Compaq compiler
guy scrambled over to our booth from the Compaq booth to see what was
going on...

Currently our mysql package builds with it. As far as kernel builds
go, we had decided long ago to use egcs 1.1.2 to build the kernel, so
making gcc 2.96 snapshot build one was never a release criteria. That
was probably a mistake.

During development of 7, I was running 2.96 snapshot-built kernels for
weeks at a time.

Cheers,

Matt
