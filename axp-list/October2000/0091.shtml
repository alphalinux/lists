As usual the RH release ending in a "0" has some nasty problems. Not sure
if anyone has tried it yet, but my advice is to wait.

The GCC installed was a snap shot, and not a stable version. Although
this version of GCC is supposed to be better suited to Alphas.

See the mailing list link for more details.
http://gcc.gnu.org/ml/gcc-announce/2000/msg00003.html

I was unable to compile a decent kernel or much else (mysql) with it (oh
BTW did I mention to compile the kerenl you use kgcc with RH 7.0).
*crash*crash*crash* Was able to compile very simple kernel, but by scsi
adapter and ethernet cards (same as my multia) caused the kernel compile
to fail.

This was on a x86. Alpha users may wish to wait for something more
stable and with a compiler that works. The new X server worked fine
though, I guess that was reason enough to bring out this update?? 7.0 is
NOT ready for a production environment IMHO. Just a heads up before the
Alpha version hits.
 
YMMV

If you do decide to use RH7.0 on your alpha please report back to the
list, I would be curious on your results.

PS a new 3.0 GCC will be out probably by January, KDE2.0 should be out in
Nov or Dec, as well as the new 2.4 kerenl about the same time frame
(Maybe later now just read Linus's update :( ). This is an exciting time
to be a Linux user in general, but on a production machine it may behoove
people to wait and test a bit before moving up.
