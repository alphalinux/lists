James Fowler wrote:
> Shouldn't the Sparc code be 64-bit clean (not that it is any help to us).
> Admittedly I haven't a clue. But I was up for a guess this afternoon.

With Solaris 7 on a UltraSPARC-II, the default is still to build 32-bit
executables. I've seen very few SPARC programs that are 64-bit.

DEC similarly could've made -xtaso the default on Alpha, but they didn't chicken
out the way Sun did...

It's a bit disingenuous of Sun to claim Solaris is a 64-bit OS.

P.S. Anybody on this list ever benchmarked programs built with -taso vs.
without? I'd be curious to know if it helps performance, and how much.
Executables should be somewhat smaller with -taso.
