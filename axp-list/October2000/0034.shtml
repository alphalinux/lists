On Tue, 03 Oct 2000, you wrote:
> Hello,
> Thank you also for your reply. We compile everything on our ALPHA machine, for
> our use and our customer's use. We thought maybe compiling it on his machine
> would help, but we still had the same problems after doing that. The limits are
> as follows:
>
> cputime unlimited
> filesize unlimited
> datasize unlimited
> stacksize 8192 kbytes
> coredumpsize 0 kbytes
> memoryuse unlimited
> descriptors 1024
> memorylocked unlimited
> maxproc 256
> openfiles 1024
>
> This is the same setup as on our machine. Any other thoughts?

The setup seems ok to me. Has the code been written so that parallel
execution could be exploited? In other words, does the code include HPF or
MPI directives?
If yes, is the SMP of the problematic machine working properly?

If not, then maybe you should check the glibc and gcc versions as well. Are
they the same?
