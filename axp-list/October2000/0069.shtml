hi,

i am currently porting glidev2.
there are also a lot of troubles with pointer <-> int conversions.
i defined
  __64BIT_POINTER
and
  __64BIT_LONG
for the compilation on alpha.
(you have to pass -D__64BIT_POINTER and -D__64BIT_LONG to gcc).

then i compiled - checked where the warnings came from and inserted
#ifndef __64BIT_POINTER
   /* old (32bit) code */
#else
   /* new (64bit) code */
#endif
this way the programm compiles normal on 32bit cpus and with the new code
on 64bit cpus.
the __64BIT_LONG is defined because for example on sparc cpus
long is 32 bit and 'long long' is 64bit.
on alpha: long = 64bit and 'long long' is also 64bit.

hope this helps, niki
