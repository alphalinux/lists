On Tue, Oct 24, 2000 at 03:10:27PM +0000, Ken Siersma wrote:
> Hello,
> Have two Alpha processors, and did not have RTC support in the kernel
> previously. Now that I do, the date gets set to today's date 2052 on
> boot. How do I correct this?

If you do not need RTC support maybe the simplest way would be to
configure your kernel without RTC? If you do need that for some
other reasons then typing 'hwclock --help' brings, among other things,
this:

Options:
....
  --directisa access the ISA bus directly instead of /dev/rtc
  --badyear ignore rtc's year because the bios is broken
  --date specifies the time to which to set the hardware clock
  --epoch=year specifies the year which is the beginning of the
                hardware clock's epoch value
  --jensen, --arc, --srm, --funky-toy
                tell hwclock the type of alpha you have (see hwclock(8))

Maybe something from here, like '--badyear', would solve your problem?

  Michal
