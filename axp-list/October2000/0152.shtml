I'm not sure of the cause of the lock up, BUT I've had a lot of problems
with WU-FTPD running under INET. It usually crashes or brings the machine
to it's knees when my webtrends box tried to FTP reports over to it.
(this was caused because web trends opened a seperate FTP session for each
file, or so it seems) My first solutions was to run WU-FTPD under TCP
server. This fixed my FTP problem, but ecause of the REALLY bad security
in WU-FTPD I switched to pro-ftpd. It gave me the EASY ability to use a
"home dir jail" without resorting to futzing with me /etc/passwd file.

Also there was a kernel update for 6.2. I noticed a slight improvement
when I updated, althought I did not have any "lock up" problems, the box
did seem to functiona bit better. I'm currently trying to run some of the
newer kernels with the API patches. Haven't finished compi.ing the
kernel, had a hard drive failure, but I'll let you know what I found out.

(7 year old Quantum 1 gig sci drive. Finally died!)
