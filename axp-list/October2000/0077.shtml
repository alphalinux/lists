A few months ago I submitted some patches to the development gcc
that implemented an -mtaso switch for alpha (architecture-specific options
in gcc begin with -m); this relied, of course, on -taso support in -ld.
For me, it was a way to have a lisp heap take 1/2 the space it would
otherwise take, so it wasn't quite "a hack which provides a workaround for
broken code".

It worked, but was ugly. It was so ugly that it wasn't incorporated into
the sources. Perhaps someone else would like to try to improve it.
