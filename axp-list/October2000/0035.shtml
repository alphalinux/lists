On Sun, 1 Oct 2000, Ken Siersma wrote:

: He can write out the TMP files, but not the DMP file. It seg faults
: as soon as the program opens the DMP file to write. Also, the GUI
: programs seg fault when he's accessing the disk.

: One wierd bug is if he uses one of the GUI programs compiled for 1 GB
: Ram (he has 2 GB), it crashes if he does a specific function, but if I
: compile it for 128 MB Ram, it doesn't crash.

Could this be a case of flaky hardware, most probably memory? Bad memory
can be disguised as a harddisk problems via the file buffer.

A reasonably sure sign of bad memory is when the nigtly cron jobs like
slocate are segfaulting. Does something like "find / -print >& /dev/null"
produce a segfault on your clients system? Do you have a way of testing
memory directly?

Reinier.
