On Thu, Oct 05, 2000 at 06:29:28PM +0100, Nikita Schmidt wrote:
> I just upgraded SRM to V5.8-4 on XP1000 and discovered that the wtint
> PALcall had disappeared (this is PALcode V1.76-22). Wtint definitely was
> present in the previous version -- the one that the machine came with.
>
Are you sure 'wtint' was working in previous version? Months ago I looked
in the palcode from SRM 5.7 for lx164. There was some code at address
0x2f80 (wtint entry point). I don't remember details now, but that code
had _absolutely_ nothing to do with "wait for interrupt".

> Does anybody have any idea how this could happen? Wtint is architected
> and not optional, and besides, 21264 can clock itself down while waiting
> for interrups, IIRC, so a working wtint could make a lot of sense.
>
I strongly suspect that 'wtint' was dropped (or significantly changed)
some (long?) time ago, but documentation was not updated...

Ivan.
