Tom Morris wrote:
> > Can anyone tell me if they will give any performance boost on ev56
> machines?
>
> The stuff which is conditionalized for ev6 you won't see at all. If you
> were
> to build with it and run on an ev56 you'd take an illegal instruction fault
> (i.e. crash) when you hit an instruction which was added to the instruction
> set after the ev56 was designed.

That's not strictly true. Certain instructions like wh64 don't trap on ev56, it
is essentially a nop. Also prefetch instructions should be available, though
reportedly they don't benefit anything prior to ev6.

Speaking of which, has anybody observed any benefit from wh64 in their code? I
would have liked to use it in some heap libraries that allocate memory on
block-size (64-byte) boundaries. Unfortunately I couldn't find any measureable
benefit whatsoever. I don't have the code handy, but it was something like:

  struct cb {
    long l1;
    long l2;
    long l3;
    long l4;
    long l5;
    long l6;
    long l7;
    long l8;
  } *p;

  ...

  wh64(p);
  p->l1 = 0;
  p->l2 = 0;
  p->l3 = 0;
  p->l4 = 0;
  p->l5 = 0;
  p->l6 = 0;
  p->l7 = 0;
  p->l8 = 0;

where the bottom six bits of "p" are zero. It actually ran slower with wh64
than without. Figuring I would need a delay between wh64 and the store, I tried
wh64(p+1) and other variations to no avail.
