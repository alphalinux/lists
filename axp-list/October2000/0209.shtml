> My Alpha (RH 6.2) has locked up. One of the users tried to Ctrl-C a
> fortran program, and while it was trying to close the program, whenever
> you tried to do anything (like ps -a so the program could be killed) it
> would tell us:
>
> hurricane:~/ls: Too many open files in the system

I had a similar problem caused by having an inconsistant set of rpms for the
Compaq compilers. Apparently something weird in the runtime interacted with
a kernel bug in the 2.2 series. That kernel bug hasn't been fixed, but
apparently doesn't show up in other circumstances.

So, if you're using the compaq compilers, I would suggest going to the
Compaq download area and making sure you have exactly the set of rpms they
have there.
