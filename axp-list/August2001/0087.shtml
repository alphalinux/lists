On Tue, Aug 14, 2001 at 11:17:35AM +0200, Peter Mittermayer wrote:
> Seems not to be so easy. Everytime I want to create a new non-BSD
> partition I get a segmentation fault ...
>
> I'm doing the following:
> I delete the BSD partition, write the disklabel, change to DOS disklabel
> ('r') and press 'n' to create a new partition
>
> But when pressing n I get the segmentation fault ...

This is a bug in fdisk which I reported to the author, and to Red Hat,
already a number of times. I even provided some patches (and a list
of steps to cause segmentation fault with a 100% success rate :-).

To workaround that "wipe" an initial portion of the disk writing to it
nulls from /dev/zero with a help of dd. One or two K should be enough.

There is no "magic" in using either kind of partitioning if information
you put there is correct. Both work just fine. BSD-type disklabel
is limited to eight partitions total. With "FAT-style" you may have
more but, for a change, it is so hacky in places and has so many "dark
corners" that you really do not want to know details. :-)

  Michal
