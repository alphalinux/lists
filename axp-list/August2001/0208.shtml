On Tue, Sep 04, 2001 at 10:28:39PM +1100, Luke Brennan wrote:
>
> Now with 7.1, this old milo doesn't work.

Ok, one more time. A default kernel supplied with RH 7.1 for Alpha
is configured to boot also "Wildfire" class machines
(LEGACY_START_ADDRESS is off). This will _not_ work with milo; although
Stefan Reinauer was working recently on new milos, making those based
on 2.2.18 obsolete, and maybe he has some tricks about which I do not
know.

OTOH there is absolutely nothing which would prevent you from using with
RH 7.1 a custom kernel which can be booted from milo (that includes also
kernels from 2.2.x series which work with RH 7.1 distro just fine).
This applies in an installation/upgrade phase as well because you can
use a kernel already present on your hard disk and/or replace one on
supplied floppy images. Yes, this is somewhat painful if you are doing
an absolutely new installation and there are no other Alphas running
Linux in vicinity but I gather that this is usually not the case. Even
if it would there are ways out - starting with an older version and
upgrading it, for example.
