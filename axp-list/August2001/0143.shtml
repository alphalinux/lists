Hello !
I build and booted 2.4.8 without problems on my LX. All hardware
supported in 2.2.19 (bttv, AWE32, SCSI, ppp) works "almost" fine. I
noticed that I get delays on all ends except disk benchmarks, which
show throughput larger then possible (i.e. on IDE). After some poking
around (I can describe the details if necessary) I found the cause:

My clock requires approx. 5 seconds for a second, i.e. a watch
proceeds five seconds while at the same time my system clock proceeds
only one second. I never experienced this under any 2.2.x kernel I
used before (and my machine is at rest relative to me) so I belive it
might be some new setting in 2.4.x.

Did anyone experience something simmilar or knows what to turn on/off
or to patch ?

If necessary I can provied a detailed description of the "quirks"
(which I think are all clock related), a dmesg output or some
benchmark results (bonnie).

Thanks
