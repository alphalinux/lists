Peter Mittermayer <p.mittermayer@astro.univie.ac.at> writes:

> I created a nsd disklabel (partition c only) taalking default values
> and switching type to 8 (ext2). Then I rebooted. The disk was
> recognized by the kernel and the partition as hda3. I used mke2fs to
> format the disk and (unfortunately) moved data from the old disk to
> the new one. After the next reboot however the disk was no longer
> accessible with the message 'corrupt partition table'.

Partition c is by convention used to denote the whole disk. It is not
for putting a file system on, since it spans *really* the *whole*
disk, including the partition table. You should make additional
partitions to put filesystems on, even if you want only a single
partition. Remember to leave ~5MB empty before the start of the first
"real" partition to have space for aboot.

> What happened? What can I do to repair the partition table? How can
> I get my data back??

Try mounting the device itself, e. g. sda, instead of sda3.
