I had a problem with the AXP-PCI boards also. I replaced the clock
chips and that did not seem to make much of a differance. I tracked the
problem down to a MILO start up issue. It seemed that if the memory size
was set to some unreasonable size the system took forever to boot. I
burned a new MILO with a fixed memory size initalization. Going back to
SRM also made the problem go away.

A few thoughts with our friend with the PC64.
1) try plugging in a serial console.
2) run up a failsafe boot and reinstall SRM.
