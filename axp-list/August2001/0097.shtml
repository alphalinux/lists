> The whole-disk "c" partition is for Tru64 alias DU alias ...
> For Linux, you should never use the whole-disk partition. In
> fact, if doing just Linux, you don't need a whole-disk partition.
> However, if you have one, you must mark it unused and actually

        If in any case no partitions are needed /dev/[h|s]d[a-z] serves
as an entry point to the whole disk (actually even if there is a
disklabel of any sort, /dev/hda will still give you "raw-access" to the
first ide disk). Linux needs no partition
to reach the disk. It is perfectly possible to establish an filesystem
directly on the raw disk (tested under x86, alpha may differ). I
would be amazed if SRM/aboot ARC/milo will boot from such devices...
