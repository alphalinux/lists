On Mon, Aug 20, 2001 at 09:35:10AM +0200, kjetil.bugge@borregaard.com wrote:

> Backup runs but crasj (stops) when coming to proc/kcore
> Have to kill the process.
> I check the file and it seems extremely large, like 4,3 TB (my HD is only 2
> GB).
> Look at the size of the file (printscreen picture in attached file)

Don't backup /proc. It's not a file system on a disc, but it's some
information the kernel provides you via a filesystem as an interface.
/proc/kcore is a means to access the physical memory of your machine,
and as such, it surprises me that the file is 4.3 TB. Maybe you have a
problem with "ls" wrongly interpreting the value?

Anyway, just exclude /proc from being backupped. Then you're all set.
