[Goswin Brederlow writes with my subscription:]

Helge Kreutzmann <kreutzm@itp.uni-hannover.de> writes:

> Once again I try to get our Ruffian stable. We now moved him into a
> colder room but he remains quite instable, many things simply cause
> memory faults, i.e. some grep, cp, ...
>
> I noticed that the part just below the memory sticks gets very hot, even
> if the machine was turned off for several hours.
>
> It is a Ruffian (UX) with 2*256 MB Ram. I tried to boot with a single
> memory stick but it did not boot.

Sticks have to be inserted pairwise in any case.

> Does anyone have a suggestion to single out the faulty part ? Would
> it be sensbile to add extra coolers (i.e. for the memory) or is
> there some "tricky" programm to single out the "true" memory problem
> (if it is such).

There's a FAQ on this at alphalinux.org:
http://www.alphalinux.org/faq/FAQ-11.html, which recommends using
silicon paste. It also would be better to keep the case closed and to
add a fan blowing out at the back and one blowing in at the front (my
case has fittings for such fans at the front together with the speaker
and at the back above the power supply). If you have many or hot hard
discs, you might also want to use a fan carrying frame for them.

Did you have any success with the onboard network? The tulip driver is
flaky and the newer ones (2.4.7-ac4) don't work at all. The de4x5
seems to work, but only without full duplex.
