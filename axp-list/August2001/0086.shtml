On Tue, Aug 14, 2001 at 08:58:19AM +0200, Gregor Maier wrote:
>
> Because I know of the fdisk bug (I think this is corrected now) that shows to
> much sectors in fdisk while in BSD disklabel mode - so I left the last sector
> free.

Yes, indeed. There was historically such bug but you have to have a
rather old fdisk to be bitten. A default end value for the last
partition was off by one and you were offered a partition reaching one
sector past a total capacity of your disk (displayed a little bit
earlier on your screen and this value was correct). If you would accept
that bad value this could have nasty consequences but you would have
actually write something into that "wrap-around" area (not that you have
much control where physically you are really writing).

I do not recall when this was fixed but a considerable time ago.
