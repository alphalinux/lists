On Wed, Aug 01, 2001 at 02:35:21PM +0100, James Hallam wrote:
>
> trying to install RH7.1 on the same alpha as before
> using milo 2.0.3x, (only one that will work)

Are you SURE? Have you tried the latest 2.2.18-based ones, found at:

        www.suse.de/~stepan/

> from the milo prompt i type
> "boot -t iso9660 sr0:kernels/vmlinux.gz root=/dev/fd0
> load_ramdisk=1 text"
>
> it loads the kernel,
> then resets itselfs and will keep doing that till i interupt it.
>
> any ideas ?????

This is because the generic 2.4 kernel built by RH does NOT use the
legacy kernel address option, and the old 2.0.3x-based MILOs do NOT
know how to load and run it.

If the new MILO still doesn't work for you, try booting the 2.2 kernel
from the RH 7.0 CD from a floppy, and loading the 7.1 ramdisk.img from
off the floppy as well.

Good luck.
