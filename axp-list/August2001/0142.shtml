Danilo Godec wrote:
>
> Hello, folks!
>
> I have a client that requires 'PC-NFS'...
>
> Just what is PC-NFS, what are the differencies to 'normal' NFS and is
> there an OpenSource implementation of the deamon available (a Linux
> src.rpm would be even better)?

PC-NFS is a proprietary NFS client implementation for Windows. We used
to use it here but never were really happy with it. It was a Sun
product,
the last time I looked.

If you know that the requirement is for NFS on the client, the easiest
thing to do is to ge Microsoft's Unix Interoperability package, which
includes an NFS client. There's another NFS client for Windows called
"Disk Access", or something like that. It's reported to work well. Do
a Google search to find out who makes it.

If the requirement is for access to NFS servers from Windows, the latest
version of Samba fills this bill nicely without any "touches" on the
Windows side. The NFS server simply looks like another NT fileserver
to Windows. If you have more than a few Windows clients, this approach
is a lot less total cost than buying and installing NFS on all the
clients. The old version of Samba that came with RH 6.2 works OK, but
the latest version adds support for NT acl's. On our Sun servers with
acl's in the filesystem, this is cool. Haven't tried it on a Linux
server.
