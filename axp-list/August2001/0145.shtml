On 28-Aug-2001 Davies, Huw wrote:
> Hi,
>
> I'm attempting to port some software (a computer simulator) which runs under
> Intel Linux to my Alpha. (The fastest Intel box I have is 166MHz so I
> suspect the alpha will be a little quicker :-). I can confirm that the
> software runs on Intel but "works" significantly differently on Alpha (works
> implies the user interface seems to start but the application loops rather
> than runs).
>
> I have done some simple "eye" checks on the code and nothing obvious appears
> wrong. I have tried using both gcc and ccc and get more warnings with ccc
> (mainly casts). Are there any pointers to information about solving this - I
> suspect either an initialisation or possibly a data alignment assumption.
> I'm trying to avoid understanding all the code, but if that's the next step
> so be it!
>
Mayb this will help.

the problem could be the int datatype in C. The width of an int (number of
bits) in C is equal to the processor's register width (i.e. 32 on Intel and 64
and alpha)

maybe there's some similiar problem with pointers (i.e. storing pointers in 32
bit integers combined with some fancy typecasts, ...)
