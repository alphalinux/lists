On Tue, May 22, 2001 at 11:07:01AM +0200, lello wrote:
>
> I have installed the 4.0.2 XFree86 server on my alpha ev56,
> it seems to work fine, but it uses all the memory, using
> top I get the following result:
>
> %MEM COMMAND
> 8359.6 X
>
> I am using a gnome-session as window manager.
> This cause to me a lot of problems, how can I solve it?

You can't, except by getting more memory... ;-}

This (Xserver using ~20GB of address space) is required on machines
that need SPARSE addressing to get at PCI bus memory and I/O space.

[By "required", I really mean "as currently implemented in XFree86";
 doing it this way has allowed graphics driver writers to NOT have to
 worry explcitly about SPARSE access on Alpha; otherwise, IMHO,
 there'd be far fewer servers that run on Alpha.]

Some EV56 machines don't need SPARSE (MIATA, LX, SX), but some do
(PC164, AS500 5/500); the server knows and does appropriately.

It's not that the Xserver actually uses all that address space that
takes up the memory, it's the page tables to manage that space that
are the memory hog: 20GB of address space requires ~2500 pages to hold
the PTEs, or ~20MB.

Moving to a different (read: more economical :-) window manager instead
of GNOME or KDE may help somewhat, but more memory is the real solution.
