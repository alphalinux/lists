On Tuesday 29 May 2001 03:05, you wrote:
> New to the list, so forgive me it this has been covered. I'm having a bit
> of a problem with the Texas Inst. 8meg Permidia 2 based card in my DEC PWS
> 500a. Running RH 7.0 and XFree 4.0.1-1. X starts and runs fine 1280x1024
> 16bit, but has video corruption when moving windows, and in particuliar
> with Netscape. Netscape causes X to lock, requiring me to ssh in and
> reboot (killing all X related processes does nothing).
>
> I'm trying to get an idea is this a a hardware (video card) problem so I
> can replace the card, or if it is a problem with my alpha in general.
>
> Thanks in advance.

The same for me but also for xpdf, netscape and kde window closing.
I used with the same results XFree 3.3.6 and XFree 4.0.x.
My computer is a DEC PWau433 with a RedHat 7.0 and a Matrox Card (10xx, just
the one before the G200 )

It seems that there two main solutions:
- Some accelerating options must be removed one by one from XF86Config until
all is ok (but for me, with no option it always randomly freeze);
- It exists a patch for XFree 4.0.3
        gatekeeper.dec.com:/pub/DEC/Linux-Alpha/X11/diffs-403-01apr2150.gz.
but I think you must recompile your X server. Good luck!

For some people, all the problems come from a bad implementation of the
mouse/keyboard devices.

Anyway, I hope that this problem will be resolved in the next XFree.
