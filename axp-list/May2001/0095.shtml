I have a program doing JPEG compression using libjpeg that I am trying to
profile. The program is a mix of C & C++ with the best performance if I
compile the C portions with ccc (the Compaq C compiler) and the C++ portion
with g++ (GNU C++ compiler). The problem I'm having is that the times in
the profiling with gprof are attributed to the wrong symbols. I'm using
floating point DCT in JPEG and routines that are only executed in the
integer DCT are being attributed. Everything works fine if I use all gnu
tools. The problem only occurs with ccc.

The symbols from cxx appear fine as long as I use the --style=compaq with
gprof.

Current versions:
RedHat Linux 6.1 for Alpha
gcc/g++ egcs-2.91.66
ccc 6.2.9.504-2
cxx 6.3.9.5
gprof 2.9.1

What compiler flags should be used with ccc to get it to generate gnu
compatible symbols? I need this with optimization on. We are currently
compiling ccc with:

-g2 -fast -ev67 -O3 -speculate all -non_shared -lsys5 +GemFB

The documentation I can find doesn't really talk about profiling on the
Linux platform. It is mostly recycled Digital UNIX documentation.
