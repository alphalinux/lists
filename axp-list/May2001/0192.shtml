Hi,
I'm new to Linux but I had what sounds like the same problem with my
PWS 500 au with the same video board and RH 7.0. I could alter the when
the problem arose but not eliminated it by playing with resolution
settings. When I switched to a 4 meg Matrox Millineum board this solved
the problem. Unfortunately I wanted to dual Boot with VMS so the Matrox
was not a permanent fix. But I reloaded Red Hat with the matrox board
in. This allowed me to use the graphical installation which had
previously always crashed (at the mouse selection window) with the
permedia board. Once I had Linux up and running smoothly I swapped
boards and reconfigured for the new board. Magically things worked this
time. I think I have my configuration set for 1280 x1024 (16 bit) as
well. Now I can dual boot.
If you don't want to run VMS I would just switch to a different board.
You don't even need anything fancy.
SRM itself should not have a problem with the board.

Good Luck
