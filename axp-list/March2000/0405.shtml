Robert Williams wrote:

> My 6.2 on a PC164 freezes just where the startup
> sound starts to play.

No freeze occurs as long as the KDE Control Center
System Sounds panel is not set to enable system sounds,
or to "Test" any of the available wav sound files.

I can play any of these wav files thus:

Play /usr/share/sounds/KDE_Startup.wav

with no problems, but if this same sound
file "/usr/share/sounds/KDE_Startup.wav"
is played with the KDE Control Center "Test"
button, the system freezes completely toward the
end of the play, no network, no keyboard...
It looks like a KDE problem to me right now.
