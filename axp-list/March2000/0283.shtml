On Tuesday, March 07, 2000 3:02 AM, AlphaAS@netcourrier.com
[SMTP:AlphaAS@netcourrier.com] wrote:

> I've read on the digital site (some weeks now) that the Alpha 21164PC
> got mpeg player capacity.
>
> Quote: "include MVI (motion video instructions)"
> "enabling mpeg 2 decode without additional hadware assistance"
> "Host based DVD/Mpeg2 playback"
> "MPEG 1 real time authoring"
>
> So i ask myself a question, Is there an apps that use this capability ?
> I do not find any that mention that it use these options.

I'm not aware of any Linux apps using this capability (there are some things
on NT like NetMeeting, NetShow, & ActiveMovie), but MVI isn't really that
big a help for playback anyway. It can improve some things like color space
conversion, but most graphics cards these days do that anyway, making it
moot. The big win for MVI is doing video compression since it can save a
lot of time on motion estimation which is a very expensive part of the
algorithm.

> If not why not begin to create one ?
> I don't find any apps that decode mpeg-video at a correct rate, full
> screen with audio.

I haven't looked into this in detail on Linux, but I've done a lot of work
on both Tru64 and NT and I bet that MVI doesn't have much to do with
reaching the goal of full size, full frame rate playback. A good basic
starting point is the display subsystem (X + graphics card). If you can't
display 720x480 YUV 4:2:2 frames at 30 fps (or even better 60 fields/sec)
using a minimal amount of CPU (<20%?), then you aren't going to have enough
CPU left over to do the rest of the work.
