On Thu, Mar 16, 2000 at 10:45:40PM -0800, Brian Pomerantz wrote:
> I ended up just using the symbios driver...

The symbios driver is intended for the newer cards.

> I think I had to change a few
> other things as well, mincore was not defined anywhere and sys_madvise
> was not defined either.

I have no idea what sources you're using then. They're there
in the snap I pulled off ftp.kernel.org.

> This is the first kernel I've used after the IRQ rewrite, but it seemed like
> response time is a little slower when there is heavy disk activity.
> Also, is there a reason that there is never an IRQ handled on CPU 3
> (the last CPU)?

I figured that might be the case. Andrea's last set of irq patches
delivered an interrupt to all cpus, only one of which would eventually
handle it. The rest of them wasted time.

I just wrote a little widget to statically distribute the PCI interrupts
across the active processors. That should help a bit.
