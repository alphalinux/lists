On Wed, 22 Mar 2000, Ivan Kokshaysky wrote:

> I have no idea what the code below was intended to do, but it
> effectively freezes XT-PIC and all devices on it (keyboard,
> mouse, IDE etc.) with the first parport interrupt.

 It looks like a spurious 8259A interrupt detection, i.e. it checks if bit
7 of ISR is set. An 8259A feeds a default IRQ 7 to the host whenever no
IRQ is active upon receipt of an interrupt acknowledge. No ISR bit is set
in this case.

 The code looks right provided the 8259A is configured to return the
contents of ISR (Interrupt Service Register) on read by default. If it's
configured for IRR (Interrupt Request Register), then the code needs to be
fixed. Basically you need to outb(0x0b, 0x20) in the 8259A setup code to
switch to ISR or outb(0x0a, 0x20) to switch to IRR.

 I can't see any of these statements in the current Alpha code. So the
8259A retains whatever setup the console left over. I think an 8259A
defaults to IRR after initialization (but I'll have to lookup the
datasheet to be sure).
