On Wed, 15 Mar 2000 kraj@ac.housing.berkeley.edu wrote:

> I know that /dev/nst0 won't rewind the tape everytime I write to it,
> but how do I know that it's correctly point to st0?

That's just one of those things you can safely assume. :-)

> > You might want to try something like "dd if=/dev/zero of=/dev/nst0
> > bs=512k" and you should be able to see how much data gets written to the
>
> What will this command write to the tape?

Just a bunch of zeros in blocks of 512 KB. From the man page: "...reads
from [/dev/]zero always return \0 characters."

One thing I should probably point out, is that this list deals with Linux
on the Alpha platform, and these issues are not specific to Alpha.
