That's just plain strange. The autonegotiation for 100bTx has two
situations where you can achieve 100/FDX:

1. Both the switch/hub *and* the host are forced there.
2. Both are set to auto.

If one end advertises autoneg=no and the other is auto, the best it
should be able to do is 100/HDX on the auto end. This will of course
be a mismatch if the hard-coded end is set to 100/FDX. It works, but
not real well. If the driver's working per the standard it shouldn't
work that well :-)

Sun has a doc out there that describes it. If you're interested I can
try to find it.
