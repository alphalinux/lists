glindahl@hpti.com said:
> Ergo, one way to work around this is to split your source into more
> files...

Not as easy as it sounds. The C++ code is machine generated from a Verilog
input file. And those input files can get pretty big. I guess it's not clear
to me whether static symbols make a difference or not. If the .got section
only stores addresses of global symbols, then that is something I can work
on. I have a lot of member functions that C++ always makes external. I can
replace them with ordinary, and static, functions if that would help.

rth@twiddle.net said:
> Each object file may reference only one of these subsections, thus
> each object file is limited to 64k.

Does you really mean that the *object file* is limited to 64K, or that the
.got subsection for each object file is limited to 64K. You must mean the
.got subsection is limited.

What things go into the .got subsection? External symbols? If I make some
of the symbols static, will that keep them out of the .got subsection? As
I said, if this is the case, I can arrange for my symbols to be made local
to the file.
