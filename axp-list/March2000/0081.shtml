On Fri, 3 Mar 2000, Greg Lindahl wrote:

> > DOS partition tables are really brain-damaged (the 32-bit fields in
> > them are unaligned, extended partitions are a mess, etc), so, except
> > for the fact that the fdisk support for BSD disklabels as they're done
> > on Alpha is somewhat broken at the moment, you might consider it a
> > blessing rather than a curse :)
>
> All this may be true, but I would prefer it if all Linux implementations
> could use the same partition table... just like all RedHat distributions now
> use the same /boot/vmlinuz convention.

The problem is that it's not just up to aboot, SRM would also have to know
about this, so you'd have to convince Q that's it's in their best
interest to add this stuff to SRM. However, this makes more difficult for
the disributions right? At the moment if they see SRM they are safe to
setup BSD and go on, if they see MILO in the serial number string then
they need to use PC partitions.

>From my personal view, the last thing we need right now is to add
something else that is going to make use/installation of Alpha any more
difficult.
