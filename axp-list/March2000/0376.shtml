kraj@ac.housing.berkeley.edu wrote:
>
> > (FYI, you'll probably always want to use /dev/nst0 instead of /dev/st0)
>

Depends what you use tapes for. I don't bother with
more than one file on a tape most of the time. Working
now on a set of 141-150 out of 180 3590's.

> I know that /dev/nst0 won't rewind the tape everytime I write to it,
> but how do I know that it's correctly point to st0?

By convention it should unless your kernel is screwy.

> >
> > You might want to try something like "dd if=/dev/zero of=/dev/nst0
> > bs=512k" and you should be able to see how much data gets written to the
>
> What will this command write to the tape?
>

Zeros, as a test. Might be better to just use your own
data/files as the test case to understand your average
compression rate.

> The way it stands, the density on my tape drive hasn't been set by linux
> yet, and I was wondering if that's a problem while doing writes.
>

Most drives default to their highest density setting
and compression, if available, at power on. Also,
mt doesn't show the true density setting until you
put a tape in the drive and it's loaded.
