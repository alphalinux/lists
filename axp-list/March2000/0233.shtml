tibor@lib.uaa.alaska.edu wrote:

> Awhile back I got a PC164 barebones system (board, 500MHz cpu and 400w ps)
> via ebay from Sysmerica Technologies (seen on Ebay as
> DaveD@sysmerica.com).

I have three of the same.

> My initial problem back then... That board had a
> 50MHz crystal.
>
> I just got a replacement board and cpu, and while I don't see any machine
> check errors, it does lock up after a bit (or the RH 6.x install dumps
> with a python error.

The python error is a known problem with RH 6.1 for alpha.
My solution was to do a full install on a new disk.

I have been getting lockups on the one sysmerica board that I've
installed so far, and on the 433 MHz system it is replacing.
I've narrowed the problem down to three possibilities:
1. Operating system or kernel.
2. The network card, an early NetGear FA310TX with an authentic
Digital tulip chip. I've tested the system twice now without the
board installed and no freeze occured. It has always frozen
with the board inserted (network wire does not have to be attached).
(I just start up a build of Mozilla14 and let it run.)
3. Memory. I've put all of the simms on a tester and they
check out with the extended test, but now I'm rotating the chips
out in 128 bit path mode.

> ...I'm trying to deal with now is that even though it's a 500MHz cpu, it
> shows up in SRM as 366 regardless of the clock multiplier jumper setting
> (this new board has a 36.6MHz crystal).

To get 500 MHz from this board, you must have the 50 MHz crystal
installed (my boxes from sysmerica came with a pack of crystals.)
and no jumpers installed for speed selection.
