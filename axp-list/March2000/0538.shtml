On Sun, Mar 26, 2000 at 09:41:08AM -0800, Stephen Williams wrote:
> I don't understand what the .got section is, why it is limited to 64K
> and why my code so thoroughly overflows that section.

The .got section stores addresses of symbols. It consists of a
number of subsections, each limited to 64k, because that's how
much one can address with one memory instruction. Each object
file may reference only one of these subsections, thus each object
file is limited to 64k.

As to why your code requires so many symbol address loads, one
would have to examine the code to be sure.

Note that there are several techniques that can be applied to
reduce the number of symbol addresses needed by a program. GCC
does not employ all of these, mostly because GAS lacks the ability
to emit all of the relocations. Since CCC emits object files
directly, it does not suffer from this restriction. You may have
better success with that. Of course, Compaq's C++ compiler isn't
released yet, but I recently heard noises about "soon".

There are also .got reduction strategies that can be done at link
time. You might try getting the most recent binutils (2.9.5.0.31?)
and using `g++ -Wl,-relax'.
