OK. What is the machine specifically. (alpha server 2000)
are you trying to install from cdrom?
If you are when I did the install on an old alpha server 2000 using
redhat 6.0
I created the disks from the disk images on the cd,using a win machine
and rawrite, using the generic images.
from the >>> prompt
you type show boot* and it shows all your devices.
set boot_file vmlinuz.gz (or vmlinuz can't remember which now)
set boot_osflags "boot=/dev/fd0 root=/dev/fd0 load_ramdisk=1
prompt_ramdisk=1"
set boot_device_def dva0 (for the floopy drive).
type show boot* to see if its correct.
boot
it should prompt you for the ramdisk and after loading that it should
bring asking you what media to install from. If its a cdrom select that.

Note hard disk partioning is irrelavant because you do this during the
install.
It must be BSD format.
You can't boot from the hard drive until you first boot the linux
system, then you use writeboot or what ever its called.
You can't install to hard drives that are installed on a raid controller
(easily) like a dac960.
The kernel will never find them.

Mostly state specifically what machine model it is, then someone with
that machine might be able to tell you how they did it..
Post a config (type "show config) so we can see what devices you have.
