I have installed kernel versions 2.2.12 and 2.2.14 compiled for SMP on a General Micro Systems C2P3 compact PCI system with dual PIII 600 MHz processors.
 
The system boots fine with a uniprocessor kernel, but when running an SMP kernel, only one processor is detected and the system locks during the boot process with errors related to the IO-APIC.
 
Using the noapic command at the LILO boot prompt the system will boot, but runs with only a single processor.
 
If anyone can offer me any suggestions as to how to remedy this situation I would greatly appreciate it. Please reply to dnkst7+@pitt.edu

Thank you

Damon Kuskie
