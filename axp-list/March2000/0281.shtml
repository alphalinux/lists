On Sun, 12 Mar 2000, Ryan A. Krenzischek wrote:

} Any ideas as to why this machine dies after laoding stage2.img (loading
} was successful, transferring to the machine from the intranet server and
} executing the mount command to get stage2.img available to the system)

I have a similar system [AS 4/166] and had similar problems during the
install [my box also has one HD, and it does RAID just like you displayed
as well]. One thing that =might= be causing the problem is SCSI
termination. Make sure your chain is properly terminated.

When I setup my 4/166, I just had 24MB of RAM but it didn't seem to matter
much [though the couple times I ran X, I was swappin' left and right!].

FWIW, I couldn't get an FTP install to work, as I'd get a network timeout
after unpacking a large package [i used download.sourceforge.net].
