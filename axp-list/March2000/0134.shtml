Hello
I have a stock AS3300 running RH Linux 6.1, and I have run into quite a few
interesting problems

I upgraded to kernel 2.2.14 to try and solve some problems that I am having
with the machine, namely:
-you have to compile the kernel as generic to get the keyboard / mouse to
actually work. Compiling straight away as noritake-primo makes the system
drop the keyboard / mouse as soon as the kernel boots

-the machine just locks up for no aparent reason. The only way that I could
get to the machine was by telnet, and then it would just stop responding.
The machine would not go back down to SRM when it locked. The syslogs show
nothing about why the lock occurred

-changed the NIC from the DE500 (21143) that was in it to a 21140 based card
(Netgear) to see if it was the NIC that was giving me problems. The original
nic (DE500) refused to jump to FDX on my hub, but the same card on a FreeBSD
box (my 164SX which I pulled the netgear card from) runs it fine and makes
the swap to FDX without a problem. If I force that particular port on the
hub to FDX, the NIC re-aligns and works at FDX....

anyone have any ideas as to why any of this is happening?
I'm tempted to install FreeBSD on it to see if it locks up then. (I still
have like 2 weeks to return it if it's defective)

the AS3300 has 64MB of RAM in it,
2 18.2G 10k UW compaq drives
1 4.3G 10k UW compaq drive

Thanks
Kurt Ludwig
