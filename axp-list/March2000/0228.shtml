Awhile back I got a PC164 barebones system (board, 500MHz cpu and 400w ps)
via ebay from Sysmerica Technologies (seen on Ebay as
DaveD@sysmerica.com).

We had a small thread going back then here as a result, dealing with clock
speeds, jumper settings for the clock multiplier, and different speed
crystals.

My initial problem back then was correctable machine checks streaming
across the screen after a few minutes, or it would just lock up. I
installed a new battery, tried all combinations of IRQ[0-3] jumper
settings, but never was able to get it to work reliably. That board had a
50MHz crystal.

I just got a replacement board and cpu, and while I don't see any machine
check errors, it does lock up after a bit (or the RH 6.x install dumps
with a python error. Installing FreeBSD 3.4-REL it locks up shortly after
installing the ports tree trying to reset the scsi bus). The main thing
I'm trying to deal with now is that even though it's a 500MHz cpu, it
shows up in SRM as 366 regardless of the clock multiplier jumper setting
(this new board has a 36.6MHz crystal).

I did flash it with the most recent version of SRM, but I've been
wondering--if the motherboard jumper was set for a 366MHz cpu speed when
I flashed in SRM, could that cause my problem? I wouldn't think so, but I
can't think of *any* logical reason why the thing would hold onto 366MHz
regardless of how I set the jumper.

Is there something I'm missing?
