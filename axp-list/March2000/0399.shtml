On Wed, 15 Mar 2000, B. James Phillippe wrote:

> Greetings,

> I am running a RedHat-6.0/6.1 (partial upgrade) on an Alpha with ZLXp-E2
> graphics board and XFree86-3.3.5. The server is in 24bpp depth.
> Everything else works fine, but I have a problem with xv. It runs, but the
> images are always solid black. I suspect this might be some kind of color
> depth or color-map problem for xv in 24bpp mode, or perhaps (maybe more
> likely?) a 64bit problem with color depth in xv?

Which "images" are you referring to? The image that xv displays, or the
little images on xv's buttons?

If you are referring to the latter, it is a problem with hatch-pattern
acceleration. Turning off acceleration will fix this problem; of course,
video will then be very slow.
