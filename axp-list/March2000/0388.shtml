Anyone have any input on the following problems?

I'm running XFree 4.0 on XP1000s with two Elsa Gloria (Permidia 2,
3dLabs) cards.

 . In 8 pseudo color mode the first screen is fine, the second (and
    third if configured) have streaking that flickers on and off, always
    at the same x offset but and a variety of vertical locations. The
    frequency of striping can be affected by the mode line values, but
    can't be eliminated. The mouse cursor does not seem to be effected
    by this.

  . The /var/log/XFree86.x.log files indicate errors running the
    libint10.a module.

       (II) Loading sub module "int10"
       (II) LoadModule: "int10"
       (II) Loading /usr/X11R6/lib/modules/linux/libint10.a
       (EE) LoadModule: Module int10 does not have a int10ModuleData data object.
       (II) UnloadModule: "int10"
       (II) Unloading /usr/X11R6/lib/modules/linux/libint10.a
       (EE) GLINT: Failed to load module "int10" (invalid module, 0)

  . In /var/log/XFree86.x.log the second card is mis-identified as
    having only 4 meg of ram (forcing it to 8 in the XF86Config seems
    to work)

  . In 24 bit mode the DAC speed is cut down to a small number (80?). I
    can see where this is done in the source code, but I don't know
    why. When forced back to 220mhz in the XF86Config it seems OK.

  . In 24 bit color mode, the first screen is fine, the second appears to
    have three shifted images of the xterm displayed there, with the
    shifted ones about 100 pixels off to the right of the prior
    image. The shifted images are not solid, but are streaking and
    flickering just like the artifacts on the 8 bit display second
    screen. Again the mouse pointer appears unaffected by this.

  
I suspect all these problems are a result of int10 not running to
initialize the second card.
  
Does anyone know if int10 is even expected to work on an Alpha? I've
started to look at the code for this, but I don't know if I'm looking
for a bug or an unimplemented section of code.

Any help would be appreciated.
