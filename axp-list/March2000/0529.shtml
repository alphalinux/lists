I'm working on a Verilog compiler that generates C++ code. This generally
works pretty well, but I'm having troubles with some larger Verilog source
files. To wit:

icarus.com % sh go.sh
+ ./ivl -P a.pf -N a.net -fVPI_MODULE_PATH=vpi -sm16x16 -Fcprop -Fnodangle -tvvm -ofoo.cc foo.vl
+ g++ -g -O -rdynamic -fno-exceptions -Ivvm -Ivpi -Lvvm foo.cc -lvvm -ldl
/usr/bin/ld: /tmp/ccIBXz4j.o: .got subsegment exceeds 64K (size 138800)
/usr/bin/ld: failed to set dynamic section sizes: No such file or directory
collect2: ld returned 1 exit status

I'm only having this problem on my alpha. People on Linux/ix86, Solaris
etc. are not having these difficulties. I don't understand what the .got
section is, why it is limited to 64K and why my code so thoroughly over-
flows that section.

This is a real impediment to using my alpha for this sort of thing, so I
need to find a way to get past this. Real hardware simulations are real
big, so I really need to find a way to cope. It has been suggested to me
that reducing symbols helps, and I have done some of that, but the problem
seems to have actually become *worse*.

gcc version egcs-2.91.66 19990314/Linux (egcs-1.1.2 release)
