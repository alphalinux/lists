It is discouraging that there isn't 3D support for Alpha yet. But, as I
was reading through the documentation at xfree.org I saw a couple of
interesting things. Some quotes:

<snip>
"Unlike XFree86 3.3.x where there are multiple X server binaries, each
of which drive different hardware, XFree86 4.0 has a single X server
binary called XFree86. This binary can either have one or more video
drivers linked in statically, or, more usually, dynamically load the
video drivers and other modules that are needed. . . .

"The XFree86 X server has a built-in run-time loader, donated by Metro
Link."
</snip>

I thought it was really strange that Metro Link would give this work to
an open source competitor. Then I realized what they want out of this
and what this has given the X community. With modules Metro Link and
other X driver programmers can use XFree as the core server and then
just write better modules that they can sell to users as they need
them. No need to have a long complex install of entire X
distribution, just download the small module and run the configuration
script and you have support for your wiz bang new video card.

Metro X has shown support for Alpha in the past. And I would be very
happy to spend $50 to $100 for an accelerated 3D module. And this
seems to be where everyone will be going. You just go to
metrolink.com, enter your credit card number, and download the
module. Compaq, API, or the graphics card manufactures can also write
their own modules and have them available for download. It just seems
that module writing will be much more simple than X-Server writing.

I might even lower the bar (in programming skills) so that more people
can contribute to open source modules. Just my opinion. Anyone else
think this makes sense?
