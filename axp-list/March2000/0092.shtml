On Sat, Mar 04, 2000 at 03:55:45PM -0500, David Huggins-Daines wrote:
> I like Exim's RBL support more than postfix's because you can reject
> based on some RBL lists and just filter (i.e. add an X-RBL-Warning
> header) on others. I can reject mail from hosts in the RBL and filter
> from hosts in ORBS.
>
> Of course postfix may have gained this feature recently, so I could be
> wrong there :)

This feature is not currently in postfix. From what I understand,
the development line is to implement per-user filtering at the MTA level,
rather than delegating it to the end user. In other words, hooks are
being made to link filter sets to recipient addresses, via the usual
map mechanism. Of course the ISP can then build some web-tool
to allow the user to modify the filter set. In this sense, Exim
solution may be handier for places with a technically skilled
user population.

Jozsef Kadlecsik already implemented Postfix patches along these lines
( http://www.kfki.hu/~kadlec/sw/postfix_patch.html ), although
I have the impression that Wietse is revising that area.
