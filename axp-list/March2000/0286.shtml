For anyone who's not already participating in one of the other distributed
computing challenges (Seti@home, RC5, etc), have a look at Rob Harley's
ECDL7 crack challenge.

  http://cristal.inria.fr/~harley/ecdl7/

I've done a little work on optimizing his program which cranks Alpha
performance up a bit to 450+ Kops/sec (about double the best posted Athlon
or PIII scores). Check out the relative speeds at
http://cristal.inria.fr/~xleroy/ecdl/speed.html

The challenge is more than 50% complete, so the probability of finding a
solution is increasing relatively quickly now. Even with the thousands and
thousands of machines participating, Alpha's speed advantage means that even
if we only marshalled 100 machines to contribute, we'd have a 1 in 10 chance
of being the ones to find a solution. Currently, the API team is #25, even
though it's a team of one right now. With a few more folks participating,
we could drive that up a lot. Join the team!

The note below contains a pointer to a Tru64 binary which is also the
fastest to use on Linux if you are licensed for Tru64 libraries on your
system. If you want to compile your own from the sources, use

        ccc -fast -inline speed -DPROD=3

I'm don't think that the sources have been updated with the additional
inline hints that I suggested, so if you want to add those, add the inline
keyword to the following functions: GFProduct56x56, product, inverse, popc,
changeBias, squareNtimes.

Have fun. Let me know if you have any questions.

Tom
