-lm is for libm ... which should be part of glibc and
typically resides in /lib. Just to make sure the
routine stuff is done, check the file /etc/ld.so.conf
and make sure there is a line with '/lib' in it. If
not, add it (for some reason, my ld.so.conf didn't
have it). After that, run 'ldconfig -v' .. the -v is
for verbose.. so you can see what is getting put into
the ld.so.cache ... you may want to grep or more it to
see if you get libm in there.

If the file(s) libm.so.* is completely missing, then
you have a bigger problem.
