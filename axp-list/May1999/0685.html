clc31@cornell.edu wrote:
>
> About using ldd on a DU executable,

'ldd' is a shell script in which an execution starts somewhat like this
(could be slightly different with a different library versions):

RTLD=/lib/ld-linux.so.2
RELOCS=

while test $# -gt 0; do
  case "$1" in
  --v | --ve | --ver | --vers | --versi | --versio | --version)
    echo 'ldd (GNU libc) 2.0.7
.....

It is not likely to do you very good on "alien" executables in ECOFF
format. :-)

> do I need a new version of ldd

No.

> to get a list of shlibs used by the executable

Use 'strings' and filter out an output a bit. 'grep lib | sort | uniq'
should be good enough; or grep for '\.so'.
