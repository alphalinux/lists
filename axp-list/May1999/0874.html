I use a Multia for a f/w myself. It provides NAT and demand isdn ppp
management very nicely. Assuming you want to run diskless, you're
obviously booting from another box. If it's Linux, try minicom.
It it's Sun, tip works. The Multias have both arc and srm
firmware, and I'd recommend using a native console with them until you
get the box up and running. Mine runs headless, with all services
turned off except ssh. If it dies (to the point of not being
reachable over the net), I just plug in a kbd & monitor from another
box to get in and fix it.
