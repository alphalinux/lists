Well, according to the Adaptec docs, 7 has highest prio then 6,5,4...
Not sure about the higher numbers, thought it was 8,9,10,11,... 7 still being the highest prio. Being the bus, UW still has 7 ( by default) as the host adapter adress.
0 was for bootable device, as we all now, I put my CDW on six, just to make sure it can have a higher priority.

The reason I was opting the scsi id 2 for a tape drive is that some unix systems only can install a complete system when tapedrive is on 2. ( Sun Sparc for instance)

> No, that would fundamentally go against what SCSI is all about -> a
> homogenous multi-device, multi-access bus.. The only set device in the
> SCSI standard(s) is device 0 and device 7(15 for UW). Boot devices are
> at 0 and the scsi adapter is at 7(15). The exception to this 'standard'
> are scsi cards that are able to boot from different scsi id's via a
> jumper mod on teh card or a scsi card BIOS manipulation.
>
> I have tape drive (travan, and DAT) in various machines at varying SCSI
> id's in the 4,5,6 range. AFAIK, the scsi bus prioritizes i/o requests
> based on scsi id (please correct me if I am wrong here guys/gals) with 0
> receiving a higher priority than 1, 1 than 2 and so on. So on a mixed
> bus of hard drives and tape units, it is 'better'(but not required) to
> have lower priority system devices such as tapes and cdroms sitting at
> higher numbers (such as 4,5 or 6).
