99% certain you can copy the passwords from one to the other. crypt()
should be the same on both. It uses DES after all, which is a govt
Standard (the s in des), and crypt() is also governed by the X/OPEN
unix standard as using DES.

Perhaps you can even copy the complete password file over. Just
delete the system accounts and append it to the fresh linux one.
Linux's password is just a standard UNIX password format. the only
catch is variable lengths - so you'd need to check it carefully. (eg
DU allows much larger values for UID than linux)
