>
> Hello,
>
> When writing an image file to my JVC XR-W2010 writer under kernel 2.2.5 using cdrecord1.8 I get the following error: cdrecord-1.8: Invalid argument. shmget failed.
>
> Does anyone have experience with this ??

Ok.. I'll bite..

Yes.. I've run across this.. it's a result of cdrecord trying to fetch
a block of shared memory that is slightly larger than the kernel will
allow.

The quick fix is to set the CDR_FIFOSIZE shell environment variable to
something smaller before running any cdrecord commands (including
xcdroast). Assuming you run csh or tcsh, you can do this by typing the
following:

setenv CDR_FIFOSIZE 4000k

Once that variable is set, cdrecord should work fine.
