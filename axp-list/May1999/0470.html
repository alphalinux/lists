>>> Mark Lin said:
>
> I'm trying to install Red Hat 6.0 on a DS20. I'm using the default images
> from the distribution. I have the ncr53c875 scsi controller on this
> machine. Every time I boot with the installation disk, the "ncr53c8xx"
> driver is used. However, this driver seems to timeout after resetting
> the scsi bus. I have this same card on my pc but I'm using the "sym53c8xx"
> driver and I've had no problems.
>
> My question is: how do I get the installation setup to use the "sym53c8xx"
> driver instead of the "ncr53c8xx" driver?

You don't, as it was not built into the DP264 aka DS20 kernel on RH 5.2.
In fact, it wasn't even available then; check the kernel output, I think
you'll see it was based on 2.1.120 or some such, over 6 months old.

But I think you have two options:

1. that old kernel didn't work well with that SCSI card on the second PCI
   bus, so you could simply move it into one of the top slots and try again.

2. run with the 2.2.3 kernel found on getekeeper.dec.com at:

        /pub/DEC/Linux-Alpha/Images/generic-up-223.img # floppy image file

Good luck.
