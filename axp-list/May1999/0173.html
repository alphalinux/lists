Hello,

Well, I don't know if this matters at all, but I was
recompiling XFree86-3.3.3.1-49 for my RH 5.2 w/ glibc
2.0.7.

Everytime I've attempted to compile XFree86, It always
failed if I apply the patch named something like
"xfs-redhat.patch". Everything compiles fine without
that patch, and in fact, I'm running that version of X
now that I recompiled, and it works fine. (I'm using
both xfs and xfstt at the same time) The thing is, I
don't know if the problem with that patch is Alpha
specific. They've been including that patch for a long
time now, and everytime I have to comment it out
before I compile. I'm guessing it probably works great
in the i386 world, and hence Redhat continues to use
that patch.

The other thing is, xterm in RH 6.0 is linked with
-lutempter or something like that
(this is new in RH6.0) and that library requires glibc
2.1... from what I can gather, it allows xterm to work
without suid...?? Can someone ellaborate on this?..
just for intellectual curiosity, as I didn't really
understand this part...?

But yes, for me to compile XFree86 3.3.3.1-49 on my
RH5.2 (hence glibc-2.0.7 and no libutempter), I had to
remove the xterm-suid patch.

OK, so how much of this was actually relevant?

D.
