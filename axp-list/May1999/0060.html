"Ray Kiesbrink (DSN)" wrote:

> I don't use the framebuffer (matrox millenium) just the rest of what is
> offered by linux/Xfree86_SVGA
>
> should I use the framebuffer?

Reasons to use the frame buffer (in approx. decreasing utility order):

   * No VGA emulation in video card (is this why TGAfb is used on Multias?)
   * Run full-screen apps that don't use X
   * Run XF68_fbdev more securely than normal X (shouldn't need suid)
   * 24 bpp works! (can't make it work in XF86 on my Millennium II; have to
     go to 32 bpp for truecolor)
   * Tux at startup
   * High res -> more chars in console screen
   * Fancy fonts

Reasons not to use the frame buffer:

   * Accels in XF68_fbdev nontrivial (unless you're an X hacker)

Any others? Does anyone know whether XF68_fbdev runs in less memory? Anyone
with an RPM? (With Matrox accels? :-)
