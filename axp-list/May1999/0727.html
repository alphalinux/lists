Few - big question....

Gnome is a collection of tools which form a common looking and feeling user
interface. It is built on a collection on libraries (including GTK+ and
many others) which do a lot of the common work of the applications.

For Gnome to draw things or react to the user it communicates through the
host operating systems I/O system. On Unix it normally is built ontop of X.
It also can be used under MS Windows.
When I say 'built ontop of X' - I mean it uses a number of libraries (e.g.
libX11) which
talk the X11 protocols to the X server.

Window managers are the bit of X which allows you to move/size etc windows
- they
are also applications which talk to the X server; and they will use the X11
libraries
and can also use the GNOME libraries and understand how to talk to GNOME
applications.
(Enlightenment and Window manager both know about GNOME).

OK - so say you have a GNOME application running which has a button in it -
when you press
that button:

  1) The OS tells the X server about your button press.
  2) The X server talks X protocol to the application owning the button.
  3) The application receives that button press in its lowest X library
(libX11 probably)
  4) libX11 probably calls another routine which is part of the GNOME
libraries
  5) The GNOME library calls the part of the application which the
programmer asked to be
     run when the button is pressed.

(Simplified in bits!)
