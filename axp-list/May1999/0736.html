> I then try to do a make zImage (as the README file says), im then told
> that there are no rules for making zImage,I also try bzImage and zdisk
> and end up with same result. I end up doing just make, which apparently
> does some compilation, I then end up with a kernel of approx 2.5Mb(thats
> the vmlinux file in the /usr/src/linux directory), which
> seems a bit big for a compressed kernel. And as far as I have understood,
> the kernel is supposed to be compressed, so how can I end up with a
> compressed kernel(or do I allready have one). And how can i choose which
> kernels to boot, when I finally have got a new kernel.

"make boot" is the typical command you'd type to make a kernel on an
Alpha. And don't forget make modules;make modules_install if you want
to use kernel modules. By the way, be careful of make menuconfig and
make xconfig on Alphas. They may give you a broken configuration (the
last time I compiled a kernel - both 2.0 series and 2.2 series, the
only way I could get a workable configuration was to use the old make
config - your experience may differ)

How you actually *boot* the new kernel depends, of course, on how
you're booting the system in the first place.
