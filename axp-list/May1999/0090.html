Personally, I wouldn't upgrade from 5.2 to 6.0...
especially if you have a lot of stuff that you've
compiled yourself and linked with glibc 2.0.

If you want the latest stuff from RH 6.0 (except glibc
2.1)... then you can download the SRPMs and compile
them yourself... this is what I've been doing... it
requires a few tweaks with some code that is
specifically for glibc 2.1... but I've gotten a lot of
the packages from RH6.0 to work fine.

If you don't really have a reason to stay with Rh5.2,
I would just wipe the disk and install a fresh
RH6.0.... I've only tried RH 6.0 on Intel so far,...
and it works well... except getting SSH2 to compile
was a little tricky.

Or, if it just makes you feel good to "think" you have
RH6.0... just update the /etc/redhat-release file.

D.
