On Mon, 17 May 1999, M. Nicklaus wrote:

> We had to power down our Alphas for a couple of days, and now one of our
> RPX164-2 Ruffians (RH 4.2, 2.0.30) doesn't come up any more. Nothing at
> all was changed between power down and power up.
>
> The system displays the following error messages at the ARCS BIOS level:
>
> Verify Environment Variables
> ERROR : Invalid Program Path
> OsLoader = scsi(0)disk(1)rdisk(0)partition(1)\os\linux\ldmilo.exe
                                ^^^ ^^^
Are you sure this is OK?

I have MILO on the first Partition on a hard drive with SCSI ID 0:
OsLoader = scsi(0)disk(0)rdisk(0)partition(1)\ldmilo.exe

> ERROR : Invalid operating system loader path
> OsLoader=scsi(0)disk(1)rdisk(0)partition(1)\os\linux\ldmilo.exe
> ERROR : Invalid partition for system files
> SystemPartition=scsi(0)disk(1)rdisk(0)partition(1)
> ERROR : Invalid partition for operating partition
> OsLoadPartition=scsi(0)disk(1)rdisk(0)partition(1)
>
> Does this inevitably point to a hardware problem, such as the SCSI disk
> gone bad, or could it be related to something else, (hopefully) more on
> the software side?
