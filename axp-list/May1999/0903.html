I'm assuming that updates are probably going to come along to fix my
problem, but I'm anxious to see if anybody has a quick fix in the mean
time. I apologize in advance for any stupidity when it comes to X which I
consider a magical thing that either works or is broken.

*Situation: Upgraded a 600 MHz LX164 machine from RedHat 5.0 to Redhat
6.0. The upgrade went smooth and everything seems to be working for the
most part. The generic kernel that came with 6.0 does not seem to bring
up any network support on the machine (requires tulip support), but I was
able to use my previous 2.1.122 kernel just fine.

*Problem: X does not seem to work at all. When I "telinit 5", the
correct services are started etc, but no GUI screen appear. When I try a
"startx" or "xinit" or "X", I get the error immediately:

/usr/X11R6/bin/Xwrapper: error in loading shared libraries:
/usr/X11R6/bin/Xwrapper: undefined symbol: __syscall

strace output does not seem to yield any obvious clues to me. The correct
video server is the Mach64. If I try to run this directly, I get the
following output: (I tried this with the new XF86Config file built by 6.0
