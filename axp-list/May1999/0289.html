On Wed, May 12, 1999 at 12:02:32AM -0400, allen@suberic.net wrote:

> I did have
>
> CONFIG_BSD_DISKLABEL=y
>
> set in my .config file when I compiled my kernel, so disklabel support _should_
> be in there. Of course, I don't know of any way of testing that without
> actually disklabeling another disk...
>
> An interesting follow up: while fdisk followed by 'b' (to edit the bsd
> disklabel) responded with the message
>
> 'There is no *BSD partition on /dev/sdb.'

Same here. I created a BSD disklabel with fdisk "b" on an DP264
and fdisk on a i386 ("b") did not dedect the label.

Fdisk does _not_ depend on CONFIG_BSD_DISKLABEL only the kernel should
do. It also does not depend on the fdisk version (that creates the label)
because the labels created with v2.4 or v2.8 are exactly identical.
