Howdy -

I'm having trouble getting things (meaning Redhat Linux) going on a
Jensen DECpc 150. I started (and ended) by reading the install guides
from AlphaLinux.org and from Digital; specifically, the document at
http://www.alphalinux.org/docs/jensen-install.shtml which describes an
install using 'aboot.' I've been through this thing 100 times.

I did do the firmware upgrade to 1.7 like the docs say. That went
fine. But the problem is making the boot image disks and doing the
copyboot thing...

Every time I run copyboot it starts up and copies for a while, but then
it stops with a read error and gives a read error number "8". Here's
what I've tried:

>From a DOS prompt on an NT workstation, I did a "copy /b lxboot.aboot
/b + lxboot.aboot /b + bootlx.aboot /b + vmlinux.gz image" which
supposedly stuck those files into the file named image (duh). I then
used rawrite to copy "image" to a diskette. Tried to copyboot that
diskette, got the error.

>From an Intel Linux box, I tried a "cat lxboot.aboot lxboot.aboot
bootlx.aboot vmlinux.gz > /dev/fd0" and then took that diskette over to
the Jensen. Did the copyboot and again the error message.

I've tried using different diskettes, tried formatting the floppy under
Linux using "fdformat /dev/fd0H1440" and then re-doing the above steps.
 Doesn't seem to make a difference.

I did see what would happen if I ignored the error 8 and booted the
system into OSF, and tried the ">>> BOOT -FL INSTALL DKA000". Also
tried ">>> BOOT -FL "i" DKA000" and several other permutations. No
good.

I have gone through the Jensen-HOWTO doc available at
ftp://ftp.digital.com/pub/DEC/Linux-Alpha/JENSEN_INSTALL/Jensen-HOWTO.
Followed these instructions several times, too. These instructions had
me make an image of a file named bootdisk.1.3.72.jensen.srm.gz and
follow similar instructions as the AlphaLinux doc. When I went through
these instructions, the copyboot was successful. But when I go into
OSF and do the ">>> BOOT -FL INSTALL DKA000", it fails. Actually it
the screen blips black and it looks like it might try to do something,
but then it stops. No error message but a single line of goofy
characters that don't mean anything to me.

The Jensen has two hard drives running off an Adaptec controller.
Drive 0 has two partitions -- a 4 MB boot partition and a 390-or-so MB
ext2 partition. Drive 1 has two partition, too -- a 150 MB swap
partition and a 2 GB+ ext2 partition.

So that's it -- not sure which docs and files I should be using, but
heck, neither one of them work. What should I try now ('stick with the
Intel Linux box' is not the answer I'm looking for)?

thanks,
