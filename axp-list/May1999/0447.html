So, latest scoop in my adventure to get this Alpha with the DAC960
controller running linux...

I load RH 6.0, and notice the DAC960 is compiled in as a module from the
start. Thats cool I think... but upon trying to insmod it or modprobe it it
bombs saying there's an error in some internal function...

So, I say, RedHat must not have something right... I go out, find the DAC960
site, pull down the lates .h/.c files and the 2.2.4 patch file. I pull down
the matching 2.2.4 kernel from kernel.org as well. I place the .h/,c files
in /usr/src/linux/drivers/block, apply the patch from /usr/src, and begin my
compile with the DAC960 option compiled in, not as a module.

The compile breaks while compiling DAC960 related stuff as you can see from
the error I grabbed posted below. Has anyone run into this? Does anyone
have any recomendations on this one? Much appreciation for any help.
