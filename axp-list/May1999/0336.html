Otis Gospodnetic wrote:
>
> Hello,
>
> Linux on x86 has the limit on the size of the file - the max size of a file
> is 2GB.
> (if you attempt to write/append to a file that's already 2GB in size you get
> errors about trying to access beyond the end of the device, etc.)
>
> Now, what would happen if I mounted a partition/disk that is really on an
> Alpha machine (running Digital UNIX) onto a Red Hat Linux running on an
> Intel (x86) machine and then attempted to write to a 2GB file on that disk,
> but from the Linux side?
>
> Would Linux complain or would it work because Alpha doesn't have the
> 2GB/file limit?
>

You need NFS V3 to do that and it's not generally available
for Linux. That gives you the access to files GT 2GB. Assuming
Linux gets a proper NFS V3 client, you'd have a shot at it.
As it is now, it won't work.
