Speedracer wrote:
>
> I have a file that I cannot delete,
....
>
> [root@sys]# ls -la factor
> b-wSr-s--T 1 16457 847 0, 3 Aug 16 1971 factor

Something got scrambled in a file system. This is not really a file
(well, it claims to be a block device :-).

Use 'debugfs' to empty inode(s), sync, remount its file system
"read only", run fsck on it, reboot. After that this should be
a file of length 0 which can be deleted. If still not, then use
debugfs once again to delete its inode and repeat the procedure
above.
