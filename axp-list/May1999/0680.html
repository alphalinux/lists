I am only seeing ~2.5MB/s throughput for buffered disk reads (hdparm -t)
on a IDE hard drive on my Ruffian RedHat6.0 system. I am in the process
of trying various 2.2.x kernels and patches. I see 8-10MB/s throughput
on Linux pentium systems.

Current hdparm settings:
/dev/hda:
 multcount = 16 (on)
 I/O support = 0 (default 16-bit)
 unmaskirq = 0 (off)
 using_dma = 1 (on)
 keepsettings = 0 (off)
 nowerr = 0 (off)
 readonly = 0 (off)
 readahead = 8 (on)
 geometry = 8940/15/63, sectors = 8448300, start = 0

1) Has anybody been able to get better IDE disk performance on a
Ruffian (UX) and if so, what kernel, patches, hdparm settings, etc were
used?

2) Is there any documentation on the patches that are available? It
seems that the "ac" and "uniform-ide" patches are referenced often.

Thanks,
Cliff
