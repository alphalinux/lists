Hi:

I have been trying for many days to get Linux running on my DEC
personal workstation 600au. I have verified that a Q Logic ISP1040b
scsi
controller is in the box. In
all cases during installation (redhat 6.0 and Debian 2.1) when I try to
initialize a swap partition or create an ext2 partition, the machine
hangs and messages such as this are displayed to the console:

<4> scsi0 channel 0 : resetting for second half of retries.
<4> SCSI bus is being reset for host 0 channel 0.

I have looked at the qlogicisp driver documentation, and there does not
seem to be any parameters I might fiddle with to improve the situation.
Does there exist any newer versions of the drivers that might fix this
problem?

I finally found a web site which describes a problem similar to this:

http://members.brabant.chello.nl/~s.vandereijk/miata.html

It refers to a patched kernel which does not have this problem.

I have been able to compile this patched kernel on another personal
workstation running DEC unix. I then transfer the compressed vmlinux
file to a floppy and boot
up using this kernel:

MILO> boot fd0:/vmlinux.gz root=/dev/fd0 load_ramdisk=1

All this works fine using the Debian 2.1 install disks (redhat 6.0 does
not work, I suspect a problem with the root disk being designed for a
2.2 kernel, the patched kernel I compiled being 2.0) but I still hang
when trying to initialize a swap partition or format an ext2 partition.

Help! I am almost at the end of my rope!

Regards,
