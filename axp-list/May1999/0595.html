>>> Brian Rerecich said:
>
> Maybe someone will know this. When booting to the SRM console on DP264, if
> I type "show dev" it will not show any devices hooked up to the onboard
> SCSI controllers. Is this supposed to happen? When I boot linux from the
> floppy, it gets into this loop saying there is a loop in the free list.
> This occurs on the bus that has no devices hooked up to it. Has this
> happened to anyone else? If so, what did you do. My main concern is the sho
> dev at SRM console prompt issue. should it show my SCSI device there?

SRM doesn't have Adaptec drivers, so no, it won't show any devices on them.

If you get hung up in the LINUX Adaptec driver probing, you need to run some
funky little .exe (7895up.exe?) under AlphaBIOS, that inits the Adaptec BIOS
settings.

Or you could build a kernel without Adaptec support... ;-}

This last isn't so hair-brained as it sounds; there's a good reason why we
don't ship DS20s for NT (or any OS) using the builtin Adaptecs - you can't
run both channels "safely", as there's problems with the 7895 chip, or the
way it was implemented on the motherboard, so I'm told.
