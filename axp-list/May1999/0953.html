On Fri, 28 May 1999, Brian Kurotsuchi wrote:

> The doc at http://www.alphalinux.org/docs/kernel.shtml says that menuconfig
> and xconfig are broken. Is this still true for the 2.2.9 kernel?

Nah - this comes to you from a 2.2.9 LX box configured using make xconfig

> I'm having trouble booting any kernels I make. I get the dashed line and the
> "Linux version" line that it prints as it begins to boot, which appears at the
> bottom of the screen. Then it stops and nothing else happens.

Hmm - not good!

Some things to check:

    1) Which version of the compiler have you got installed (gcc
--version)

    2) Have you got a very old MILO (I've not had to upgrade my MILO but
there have been stories about people who have)

    3) Are you setting the machine type in the kernel correctly? Try
'generic' and also try the one specific to your machine.

    4) Make sure that the console stuff is setup appropriatly for your
machine. LXs and the like probably want to be setup with VGA Text
Console, if you have a TGA card instead then you'll have to do something
else.

    5) You are trying to boot the file which arrives as
linux/arch/alpha/boot/vmlinux.gz aren't you? Dont decompress or anything
else.

Good luck,
