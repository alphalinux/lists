Hello Linux/Alpha community,

We've been having some problems with RedHat 6.0 and MILO lately on our
AlphaServer 1000 4/233. Our 4 GB SCSI disk has a small (4 MB) DOS partition
with the following on it:
  loadlin.exe (from the RedHat 6.0 distribution tree)
  mikasa (a file containing MILO from the RedHat 6.0 distribution tree)
  mikasa2 (the latest MILO from genie.ucd.ie)
  milo (the latest MILO in the 'mikasa' directory on gatekeeper.dec.com)
In short, none of the MILOes do what we want them to.

'mikasa' and 'mikasa2' have several problems. Perhaps most confusing is that
their syntax seems to be slightly different from the syntax described in the
MILO Howto - the variables are altered, disk names must always be specified
with a colon suffix (e.g. sda2: - it is unclear how the 'boot sda2 linux' as
described in the Redhat 6.0 Alpha Installation Addendum is supposed to work),
and so on. Furthermore, if we try to boot the system with a path or from the
boot: prompt as follows:

boot: linux
boot: boot/vmlinuz
boot: boot/vmlinuz-2.2.5-16
MILO> boot sda2:boot/vmlinuz
MILO> boot sda2:boot/vmlinuz-2.2.5-16

the screen blinks briefly and MILO just reboots. What DOES work is to
specify the uncompressed kernel image, as in

MILO> boot sda2:boot/vmlinux-2.2.5-16

but we'd really rather it boot the compressed image specified in
/etc/milo.conf (this file is not modified - it is RedHat's own) so that
kernel updates are quick and painless.

'milo' has fewer problems, in that it will boot the linux image described in
/etc/milo conf and seems to match the syntax described in the MILO Howto.
The one thing it will not do is boot that image automatically when passed
the argument 'boot sda2 linux' by the ARC console or from the prompt. It
instead sits at the boot: prompt after telling us that 'linux' is one of the
available images.

My questions are:
  How can I stop the new versions of MILO from rebooting themselves?
  How can I get both versions of MILO to boot the image specified as 'linux'
    in /dev/sda2's /etc/lilo.conf automatically?

Thanks a lot,
--Tom
