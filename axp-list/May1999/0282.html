> > So the question is: has anyone ever accessed an Alpha Linux disklabeled
> > disk under an x86 system? If so, how? Should it have worked just with

>
> You forgot enable bsd disklabel support in your x86 kernel config.
>

I did have
 
  CONFIG_BSD_DISKLABEL=y

set in my .config file when I compiled my kernel, so disklabel support _should_
be in there. Of course, I don't know of any way of testing that without
actually disklabeling another disk...

An interesting follow up: while fdisk followed by 'b' (to edit the bsd
disklabel) responded with the message

'There is no *BSD partition on /dev/sdb.'

I decided also to try minlabel. minlabel showed my disk's partition table
without a problem. Which makes me think that there's definitely some difference
between a standard BSD disklabel and the one that OSF/Linux-Alpha uses.

Now I suppose I get to figure out what the difference between the two is,
and then to get my kernel to recognize this one.

Or maybe I should just buy one of those new 264's.... :)
