Are you booting via SRM? If so: Have you tried setting os_type to VMS
rather than to Unix ? There's a bug in the SRM console which
does not initialize all 1GB of memory when os_type is set to UNIX.
I've stumbled across this when installing Digital Unix 4.0E on
a DP264 with 1GB of RAM. The kernel would not boot until I set
os_type to VMS.
