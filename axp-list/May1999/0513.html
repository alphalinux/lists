On Sun, 16 May 1999, Peter Petrakis wrote:

> > What I have put away with first was the local font server
> > with the reconfiguration of the XF86Config to search
> > for the fonts locally. This improved things a bit but still,
>
> Wow, That made a difference.
> 1) NS 4.51 doens't blow up when java is started.
> 2) NS doesn't hang anymore when I try to save something
> 3) acroread is working fine now
> 4) I swear X is faster...
>
> All this from disabling the font server. Jeez...
> Could someone explain to me the benifits of running a font server?

Often the font server is compiled with code to render TrueType fonts. I
have found (through lots of pain) that this is a very good way to hang X.
Any X app that requests a TrueType font can hang X quite easily -- try it
with xfontsel.

So for now, TrueType fonts are broken for alpha. Eventually this will be
incorporated into the usual XFree86 distro (i think RedHat adds TrueType
support to their distributed, compiled rpm's), at which time I hope it gets
fixed...
