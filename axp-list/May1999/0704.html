At 00:20 20/05/99 +0100, Paul Jakma wrote:
>On Wed, 19 May 1999, Cliff Brake wrote:
>
> I am only seeing ~2.5MB/s throughput for buffered disk reads (hdparm -t)
> on a IDE hard drive on my Ruffian RedHat6.0 system. I am in the process
> of trying various 2.2.x kernels and patches. I see 8-10MB/s throughput
> on Linux pentium systems.
>
> 1) Has anybody been able to get better IDE disk performance on a
> Ruffian (UX) and if so, what kernel, patches, hdparm settings, etc were
> used?
>
>try hdparm -a128A1c1d1m32u1 /dev/hda

Now that is what I call a bizaar command line :-)

>the ide controller on the Ruffian is done by an Intel 82371SB PIIX3,
>exact same as on older socket7 VX (and FX/HX?) chipset intel
>machines. and it's being driven by the exact same Linux code. so it
>shouldn't be slower than the equivalent intel pc.

That isn't so - it seems like non-Intel processors (including some x86's)
do very badly on PIO IDE transfers.

If you can get the DMA going on the Alpha; which should be possible - maybe
with selecting some kernel build options - then you should be able to get
decent speeds.

>On what kind of intel pc did you get 8-10MB/s, and what kind of
>disk?

On my Alpha LX board it was giving around 2MB/s with PIO - I got
around 6MB/s when I got the DMA going. That is a different IDE chip and
needed a particular rune to get the DMA going when building the kernel.
That is with a 2 or 3 year old 3.5GB IDE disc - I can quite imagine that
a modern IDE disc could shift 8-10.
