All that you need to do is to edit your /etc/aboot.conf file using pico or
another text editor. You need to assign a number to your boot command, e.g.

0:4/vmlinux.gz root=/dev/sda4 load_ramdisk=/dev/sda7

If you want other boot options then put them in as the next number.
1: boot line
2: other boot line

etc., etc.
If you are booting from the SRM, which I assume that you are since you are
using aboot, you will still have to type the command: boot -fl [line number]
For example I boot with line 0 so I type: boot -fl 0
Not too bad really and then you don't have to keep remembering all of the
stuff. I don't beleive that aboot will boot any more automated than that,
i.e. at power on. You still have to type the boot command. Someone correct
me if I am mistaken about this.
