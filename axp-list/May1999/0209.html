Duccio Medini <Duccio.Medini@PG.infn.it> asked:
> The difference that I experience between fortran code compiled
> under DU - and - under Linux with g77 AND libffm or libcpml is still
> remarkable.
> I saw at alphalinux a detailed set of instructions about running DU
> Netscape under Linux. Is there anything similar about running DEC-f77
> compiler under Linux (a compiler should be, in some way, a binary like
> the others)?

        For various reasons running the DEC Tru64 *compiler* at Linux is
        likely doomed to fail. Linux doesn't supply license management
        calls for example, which seem to be needed.

        However, running compiler at DEC Tru64, producing *static*
        binary, and then running that binary at Linux is very likely
        to work, and even (barely) within license rules.

        (Copying shared libraries to a machine without DEC Tru64 license
         isn't strictly speaking allowed...)

> N.B.: I have a PC164 500 MHz under Linux RH 5.2 and a 164LX 400
> MHz under DU v4.0
