On Tue, May 18, 1999 at 05:12:47PM -0500, Bharadwaj Yadavalli wrote:
>
> I get the following error message when I try to
> install mozilla RPM available from alphalinux.org.
>
> # rpm -ivh mozilla-5.1-M5.alpha.all.rpm
> failed dependencies:
> libstdc++.so.2.9 is needed by mozilla-5.1-M5
>
> The latest alpha RPM for libstdc++ is 2.8 on
> rpmfind.net. Has anybody tried installing Mozilla
> M5 RPMS? how can I get aroud this problem?

The RH-6.0 package "libstdc++-2.9.0-12" installs (among other things)
the file '/usr/lib/libstdc++-libc6.1-1.so.2' and makes a symlink
to it ('/usr/lib//usr/lib/libstdc++.so.2.9').
Hence, you have the required libs (if you mirrored rh6.0-axp).

However, "rpm -q --provides libstdc++" only lists the following:
- libg++.so.2.7.2
- libstdc++-libc6.1-1.so.2
- libstdc++.so.2.7.2
- libstdc++.so.2.8
(it probably _should_ have "libstdc++.so.2.9" listed)

So, either:
1. grab its SRPM, add the 2.9 provides line, build and install -or-
2. install mozilla with an "rpm -i --nodeps <packagename>"

In any event, refer to rpm(8).
