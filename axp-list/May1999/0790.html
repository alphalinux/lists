I'm not the person who wrote the precedent e-mail but I experienced the
same problem. I'm using Window-Maker, with Xfree86 3.3.3.1 on a
modified RH 5.0 alpha machine and I can not start applix from the
mouse menu.
Here is a sample a my menu definition file:
        "Editors" MENU
                "Applix" EXEC applix
                "Emacs" EXEC emacs
                "VI" EXEC xterm -e vi
Emacs and vi start normally but I can't start applix.
On the other hand, if I just put
        applix &
in a xterm, everything works fine.
As the other person said, it is a minor inconvenience, Applix
itself works well and I'm very happy with it (the only major thing
I would like to see is include suupport for Mpeg and Animated Gif
formats in Present, that would be really nice)
