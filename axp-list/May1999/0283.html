Brian Rerecich wrote:

> CD I made isn't a "bootable" CD, so I am trying to boot off of the floppy

> boot dva0 -file vmlinux.gz -flags "root=/dev/scd0 bootdevice=fd0
> bootfile=vmlinux.gz"

> Warning: Unable to open an initial console.
> Kernel panic: No init found. Try passing init= option to kernel.

Since the cd is not bootable, you probably have the layout wrong as
well. You may need to change the root filesystem to something else.
The message about init is simply the system looking for a way to start
booting. The message about the console is probably a missing
/dev/console.

Try something more like:

boot dva0 -file vmlinux.gz -flags "root=/dev/scd0 init=/bin/install2"

I'm not terribly familiar with the other parameters. What you may also
want to try doing is booting with a boot floppy, then from there loading
an (installation) root floppy. It will then ask you from where to
install and you can specify any directory on the CD.
