I was wondering if anyone has had similar problems and if a fix was found for them. I am currently running an SX based system with IDE drives. I have RedHat 5.2 installed with a SuSE 2.2.3 kernel. This kernel is the only one that will boot. I tried creating a new kernel from the source for 2.2.7 found on AlphaLinux.org site. I applied the Uniform IDE patch to it as well. The error I get is as follows when trying to boot with the new kernel.

PCI_IDE: unknown IDE controller on PCI bus 00 device 41, VID=1080, DID=c693
PCI_IDE: not 100% native mode: will probe irqs later
PCI_IDE: unknown IDE controller on PCI bus 00 device 42, VID=1080, DID=c693
PCI_IDE: not 100% native mode: will probe irqs later
PCI_IDE: port 0x01f0 already claimed by ide0
PCI_IDE: port 0x0170 already claimed by ide1
PCI_IDE: Neither IDE port enabled (BIOS)

At this point the system hangs and thats it. Thanks in advance for any help.

Robert Jakiel
