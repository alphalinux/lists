Thanks to all who answered.

Two consideration arose:

- SGI is little endian, while AlphaLinux is big endian but ...

- Although shifting/redefining/setting the ut_time variable in ru_utmp
  structure of AlphaLinux, the result is only garbled dates.
  Various attempt were made, building structures as suggested from you
  all or shifting the data; the attempts were made both in IRIX box and
  AlphaLinux box; no success, at the moment.

Solution: Re-wrote the original file in the SGI box substituting the
          original value (ut_date as long) with the ascii value
          returned by ctime(3).
          Then re-converted to ut_time, on AlphaLinux, using
          strptime and mktime.

Thanks to you all.
