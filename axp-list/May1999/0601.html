> 1) RedHat 6.0 general kernel image unzips fine. When it tries to run,
> the screen continuously scrolls messages like:
> +40 FFFFFFc00007a868e 0
> +50 FFFFFFFFF8070f740 1f1e161515020100
> etc.
> and then sometimes stops.

This is a machine check. Bad news, usually.

> I don't know why the RedHat6.0 distribution images
> directory does not contain precompiled kernels for Ruffian like the
> RedHat5.2 distribution!!

Because you shouldn't need one -- the generic.gz kernel is supposed
to run on all supported hardware.

Not that it helps you any, but I just installed 6.0 with that very
kernel on a Ruffian here. It would be interesting to see the first
part of that machine check message -- it may indicate some sort of
hardware fault. What sort of scsi and eth devices are you using?
