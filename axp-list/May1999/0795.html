On Fri, 21 May 1999, Sebastien Merkel wrote:

> same problem. I'm using Window-Maker, with Xfree86 3.3.3.1 on a
> modified RH 5.0 alpha machine and I can not start applix from the
> mouse menu.
> Here is a sample a my menu definition file:
> "Editors" MENU
> "Applix" EXEC applix
> "Emacs" EXEC emacs
> "VI" EXEC xterm -e vi
> Emacs and vi start normally but I can't start applix.
> On the other hand, if I just put
> applix &
> in a xterm, everything works fine.

Interesting - If I had this problem I'd replace the menu entry for
applixware with a call to a script which ran applix while logging any
errors into a file.

The problem may be that it is:
  1) Running it in a different directory
  2) Running it with different path
  3) Running it with different environment variables etc

I'm going to guess it will work in the script; however I suggest that if
you normally use bash or sh that you put #!/bin/csh at the top of the file
so that it doesn't pick up your bashrc (similarly if your a csh user put
#!/bin/sh at the top) and see if it still does it.

If you can get it to fail in the script then it is easy to print out the
environment, path etc there and compare it with that in the shell.
