....

>
> >[root@frigate /root]# mount /dev/sdc2
> >mount: /dev/sdc2 already mounted or /usr/pest/test2/1 busy
>
> I haven't tried this number of mounts myself - but as some suggestions:
>
> 1) Is there a limit on the maximum number of mounts?

    That's what I thought, but I couldn't find it anywhere in the /proc area -
even super-max (in /proc/sys/fs) was set to 256, which is the default. I'm
going to poke around the source tree a bit to see what I can see. I wonder if
it's related to the (unsuccessful) change I made to up the number of swap
partitions to 32.

 2) There used to be an issue with the number of minor device numbers for

> SCSI discs
> - I can't remember how big the problem was, but I wodner if you are
> hitting that?

    I shouldn't be. Somewhere along the line in the 2.1 series I remember
Linus sneaking in a whole bunch of new entries into the devices.txt file
outlining major/minor pairings for up to 128 SCSI disks. Obviously you're
supposed to be able to do this, although maybe the intent was to use md. Too
bad that's probably not an option... :(

>

> Incidentally - what the hell are you doing with 40 discs :-)

    Think really, really big benchmarking. Unfortunately, due to the fact that
it's an internally hacked kit, I don't know that I'll be able to release the
numbers. Maybe if I say that they're not official blah blah blah...who knows.
Suffice it to say this could be cause for some serious interest up the ladder -
IF I can get this to work...Oh...and 40 disks might not be all. Our resident
filesystems performance engineer told me that we would (at least, should)
_still_ be IO bound at 40 disks. This system, technically, can address 4 PCI
busses, which means 16 controllers (minus two - ethernet and video), so we have
12 slots times at least 4 disks per controller. _IF_ we get that far.

 - Pete
