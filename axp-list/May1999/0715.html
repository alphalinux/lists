As to the message:
[root@amanda /root]# rpm -ivh --nodeps mozilla-5.1-M5.alpha.all.rpm
error: file /usr/lib/libxmltok.so from install of mozilla-5.1-M5 conflicts
with file from package w3c-libwww-devel-5.2.8-3

There must be a file conflict between the two.
try the following

rpm -ql -p mozilla-5.1-M5.alpha.all.rpm > /tmp/a.out
 rpm -ql w3c-libwww-devel >> /tmp/a.out
sort /tmp/a.out | uniq -c | grep " 2"

This will tell you ALL of the conflicting files.

Now from the error messsage we know that there is a last one:

/usr/lib/libxmltok.so

Which is the xml tokinizer. The question is does w3c-libwww-devel
use the same tokinizer? If so than install by saying

 rpm -ivh --nodeps --force mozilla-5.1-M5.alpha.all.rpm

NOTE: check to see if there are any other conflicts first. If
you could send me information about where w3c-libwww came from
I will look into it this weekend.
