libm is the math library
it is contained in the glibc package
the file you want should be /lib/libm.so.6.1 which should be
a symbolic link to /lib/libm-2.1.1.so (for redhat 6)
I don't know what's causing your particular problem, but sometimes
these error messages occur when you are trying to compile something
but haven't explicitly included the path to a library you are using.
I get this error message for -lX11 something, and it can be fixed by
including -L/usr/X11R6/lib as an option along with -lX11.
It may work for you if you use -L/lib
