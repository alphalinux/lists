On Fri, 14 May 1999, Bharadwaj Yadavalli wrote:

>
> Does anyone know why the time stamp on some of the system
> files has the year 2019? I installed glibc-2.1.1-7 and it
> updated /usr/share/zoneinfo and now I have date command
> returning a random date in the year 1970 each time I invoke
> it. Is this time problem known? I can not seem to convince
> utime to work correctly either. How can I convince my system
> that it should believe in a time I give it is indeed right
> and not generate one randomly?

Hi,
  The 2019 problem is due to the way in which you've set up the clock
in Linux; sometime during installation it asked you whether you were
using an 'Arc Console' - you gave the wrong answer.

  Look in /etc/sysconfig/clock and you'll see a line which presently says:

ARC=false

replace the false with true.

  The reasoning is that there are two standards for how the hardware clock
is setup; in one the base date is 20 years in front of the other.

The 1970 stuff I'm not sure about; one thing that could be up is an old
version of the 'clock' binary. Try doing

/sbin/clock -r -A

about 20 times (in a loop say) - if you get them occasionally being way
out then you have an old version of the 'clock' binary.
