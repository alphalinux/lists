James Peret wrote:
>
> Good question...
>
> what do i need to enable in the 2.2 kernel?

Support for a.out / ECOFF binaries (same thing).

> i need to build milo to match the 2.2.7 kernel, true or not true?

Nope, The milo you have will do.

> how can i get my current kernel configuration (i don't have a .config, i
> am using the generic kernel)?

a .config is created in /usr/src/linux after a successfull make config
or make menuconfig

> how do i get milo to recognize the backup kernel?

Just pass it the name of that kernel in the boot string.

Example

boot sda7:/boot/vmlinux.gz root=/dev/sda7
        This is my normal boot kernel.

  
When i want to test a new kernel I usally drop it in / and name it
vmtest.gz.
So now my boot string would be.

boot sda7:vmtest.gz root=/dev/sda7
