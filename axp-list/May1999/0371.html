We have just installed RH 5.2 on our Jensen last week, and after reading
and trying both methods, I prefer the aboot; it work just fine for us.

 
> Sounds like the Jensen floppy has reverted to being a 2.88 drive. I had a
> problem with copyboot and the solution was to tell SRM about the drive.
> Note that this involves going SET OS NT (or similar), entering SRM,
> changing and saving the floppy config and reselecting OSF boot method.

We made the disks (ram, pseudo, etc) using both an intel based linux and a
pc w/windows boxes; all the disks work fine.

>
> Good luck!
> Giuliano.
>

The only trouble we found was with the pseudo boot disk used to copy the
kernel to the hard disk: the kernel hangs at boot. To solve this, we copy
the jensen kernel found under the kernels directory on the CD to the psudo
boot disk (an ext2 floppy), redo the instalation and this time all work
just fine.
