If you can tolerate the re-install, do it. If you've
got lots of customized stuff, you're in for a lot of
work any way you look at it. RH6.0 seems to be using
glibc2.1 and RH5.2 uses 2.0.7 or so. A bunch of stuff
I built just segv's under RH6.0 so I'm having to rebuild
my binaries to get it working under RH6.0. Also, I'm
getting an odd segv building egcs1.1.2. When it tries
to run gengenrtl in the gcc subdirectory, it bails
out. Nothing special I can see in the code that indicates
what's wrong. I added a printf() as the very first statement
in main() and it doesn't print so I'm wondering if there
is a binutils problem on my system. I did build my own
binutils at one point but thought I deleted it all after
this upgrade. (egcs1.1.2 is the default compiler for RH6.0
so I really don't need to build it but I use it as a test
of the system)
