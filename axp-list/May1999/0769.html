On Thu, May 20, 1999 at 11:06:30AM +0100, T.Hval wrote:
>
> Robert Bowles wrote:
> > The RH-6.0 package "libstdc++-2.9.0-12" installs (among other things)
> > the file '/usr/lib/libstdc++-libc6.1-1.so.2' and makes a symlink
> > to it ('/usr/lib//usr/lib/libstdc++.so.2.9').
> > Hence, you have the required libs (if you mirrored rh6.0-axp).
> >
> > However, "rpm -q --provides libstdc++" only lists the following:
> > - libg++.so.2.7.2
> > - libstdc++-libc6.1-1.so.2
> > - libstdc++.so.2.7.2
> > - libstdc++.so.2.8
> > (it probably _should_ have "libstdc++.so.2.9" listed)
> >
> > So, either:
> > 1. grab its SRPM, add the 2.9 provides line, build and install -or-
> > 2. install mozilla with an "rpm -i --nodeps <packagename>"
> >
> > In any event, refer to rpm(8).
> Finally got my box running with RH 6.0, I tried to install mozilla
> following the seckond option. I then ended up with the following error
>
> [root@amanda /root]# rpm -ivh --nodeps mozilla-5.1-M5.alpha.all.rpm
> error: file /usr/lib/libxmltok.so from install of mozilla-5.1-M5 conflicts
> with file from package w3c-libwww-devel-5.2.8-3
>
> What to do next to get it to work?

If you do an 'rpm -i --force mozilla-5.1-M5.alpha.all.rpm',
the package will install, overwriting '/usr/lib/libxmltok.so'.
I didn't have any of the 'w3c-libwww-*' packages installed, so
this problem didn't effect me.

What I actually did to install mozilla was run the command:
# rpm -i --badreloc \
        --relocate /usr/lib/mozilla=/opt/mozilla \
        --relocate /usr/lib=/opt/mozilla/lib \
        mozilla-5.1-M5.alpha.all.rpm

Then I wrote a tiny script '/usr/local/bin/mozilla':
-> #!/bin/ksh
->
-> LD_LIBRARY_PATH=/opt/mozilla/lib
-> MOZILLA_FIVE_HOME=/opt/mozilla
-> export LD_LIBRARY_PATH MOZILLA_FIVE_HOME
-> /opt/mozilla/bin/apprunner ${*}
