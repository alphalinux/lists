At first it did not work like expected, milo on TGA , text console on TGA
but I was able to start X on
the S3.
Then I build a kernel with only VGA and no TGA support , and that gave me
both text-console and X on the
S3 card , but milo would still default to the TGA. Never solved that , I
dont know if there are special
versions of Milo available that do this.
You will probably also see the firmware / ARC still on the TGA ....

In short , you have to change a lot of things to get the multia to use
the VGA as default.

I did not find any jumpers to disable the onboard TGA.
