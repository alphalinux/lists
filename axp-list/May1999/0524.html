> The SCSI commands do not have any priority based on SCSI ID. It is the
> arbitration process that uses the SCSI ID to decide who will get the bus
> when multiple masters want it. Also the prority of arbitration is 0 ==
> lowest 7 == higest. This is why the host is typicaly given ID 7. On some
> systems the boot disk is placed at ID 6. Which makes sense since you would
> want to have higher priority given to your disks over tapes and other
> devices.
>
>
        I would not give my hd the highest prio on the bus as some devices are more picky about data: CDR, Scanners. These device require an uninterupted data transfer if possible or at least a minimum in bandwidth. Giving them higher prio assures you this is more likely.

> So. Technicaly the way most SCSI systems are configured is wrong.
>
        SCSI configuration is fairly forgiving, if you keep the basics in check.

        cu, ray
