> anything)
> Then I tried to create a boot disk (by dd'ing the /images/generic.img to a

> When I simply boot from this floppy on my alpha 1000 I get the aboot prompt.

I had that too when I tried the actual 6.0 images. I made a
new floppy and tried it again and it worked. Also, the ramdisk
image is a gzip'ed romfs now, where as on RH5.2 it was ext2 so
if you want to build your own kernel to install with because of
unique HW, then you need to include romfs support. Also, the
RH6.0 generic image appears to have problems initializing my
Symbios 875J SCSI so I had to unplug it to get it to work. My
2.2.3 kernel has no trouble with it. Looks like Symbios drivers
are going thru changes lately that are causing once stable
systems to have troubles!!
