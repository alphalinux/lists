Hello,

We have been trying to update our kernel from the one on the RedHat 5.1
software distribution 2.0.34 to 2.0.36. We used practically all the same
.config parameters in building under the "make config" command, with the
exception of things we don't use like appletalk and ipx. The system boots
with the new kernel, but after a few minutes it locks up and the only way
to get response is to hit restart. BTW, on some occasions it has operated
for nearly 30 minutes, while at other times, it operated for barely a
minute. It's probably most often around the five minute timeframe when it
locks up. Does not matter whether X windows is started or not.

We are finding no error messages indicating what the problem might be.
Perhaps there is other places you can suggest we look other than the
message log.

System info:

AlphaPC 164LX running Redhat 5.1. Kernel currently 2.0.34. Trying to
move to 2.0.36. Had similar problems when trying to move to 2.0.35.

uname -a gives:

Linux xx.xx.xx.xx 2.0.34 #1 Fri May 29 17:36:08 EDT 1998 alpha unknown
