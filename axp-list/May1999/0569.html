Dan O'Reilly wrote:

> I just got RH 6.0 up on my UDB system (standard video card). How do I

Standard? Do you mean the onboard TGA controller?

> change resolution from what looks to be a standard 600x480 to something
> more useable like 1024x768? I'm using GNOME, by the way.

Using GNOME means nothing -- the X server is what you want to change.
I've attached my XF86Config file for you to use as a model. I run at
1280x1024 so you may want to make it smaller. I'm using RH 5.2 but the
file should be compatible.

The TGA X server does not support on the fly mode switching so it's a
complete trial and error process. It may just be easier to change the
resolution manually with vi (or in your case, maybe pico would be
better).

The file lives as /etc/X11/XF86Config

Basically, change this section to what you want:

# The accelerated servers (S3, Mach32, Mach8, 8514, P9000, AGX, W32,
Mach64
# I128, and S3V)
Section "Screen"
    Driver "accel"
    Device "TGA"
    Monitor "NoName"
    Subsection "Display"
        Depth 8
# Modes "640x480" "800x600" "1024x768" "1280x1024"
        Modes "1280x1024" "1024x768" "800x600" "640x480"
        ViewPort 0 0
        Virtual 1280 1024
    EndSubsection
EndSection

Multiple modes are unnecessary, and you may want to up the color depth
with something like:

Subsection "Display"
 Depth 16
 Modes "1024x768"
EndSubSection

I prefer resolution to color depth. You can have multiple subsections
for each Screen, and toggle among them with "startx -- -bpp 8" (default)
or "startx -- -bpp 16". You can make 16 the default by using either an
alias or by editing the startx script.
