Greg Lindahl <lindahl@cs.virginia.edu> writes:

> > > [nupgup@jbslab1 nupgup]$ /bin/xquake-1.06-alpha-bin
> > > bash: /bin/xquake-1.06-alpha-bin: No such file or directory
> >
> > The program loader isn't present. In this case /lib/ld-linux.so.1.
> > Which came with glibc 1.99, which was distributed with RH 4.2.
>
> While we're at it, could we PLEEEEEEASE fix bash to print a more
> useful error message? Yes, I realize exec() came back with ENOFILE,
> but it's worth checking access() and printing a useful message.

First of all, you'd have to fix all shells, not just bash. Then, I
think this solution is more like a hack.

A proper solution would be to add another errno value like ENOLOADER
and make exec*(2) spit it in case of such problems. However, I am not
sure, whether POSIX actually permits adding new error codes to the
"standard" system calls...
