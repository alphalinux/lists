There's no boot.img or boot.cat so the "-b" and "-c" parameters don't apply.
Also, the alpha distribution tree contains many symlinks that point into
limbo so add the "-a" parameter to mkisofs or else they will be omitted in
the image.

Along with the isomarkboot item mentioned below check the /dev directory
somewhere down in the /misc tree. In my case, there were some character
devices that didn't copy correctly, so I had to create them manually.

It works for me.
