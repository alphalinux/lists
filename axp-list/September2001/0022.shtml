Wes Bauske wrote:

>Consider what a Hyperthreaded 3.5Ghz P4 could do in terms of
>workload. Also, I think EV8 had SMT, Symetric Multi-Threading.
>Sounds similar which would be a good reason to get Alpha
>engineers.

I'm still considering what a single threaded application would do on my
1.7 GHz P4 with a full compliment of memory. For those who forgot, I'm
still wrestling with my P4 systems that hang on the benchFFT program.

Following a tip from Thorsten Koch (thanks, Thorsten), I removed half
the RDRAM from my systems (down to 1GB), and 3 of the 4 systems that
were hanging before would now finish the program (after about 6.5
hours). One would still hang. On the systems that ran, I have tested
again with full memory, and they would hang again, seemingly at the
same place into the program. Leaving in the other half of the memory
that was removed before, the systems would run through. The other
system that still hung, still hangs.

On all of them, the BIOS logs have registered numerous single-bit ECC
errors, and a few multi-bit ECC errors. I've always thought that ECC
will correct single bits, and detect multi-bit errors, which are fatal.

Intel is going to swap out the memory for us, but I don't know if that
would really help in light of our test results.
