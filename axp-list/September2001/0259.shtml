On Thu, Oct 25, 2001 at 02:03:41PM -0400, Shoshana Rosenthal wrote:
>
> We are running RH7.1 and
> OpenSSH_2.5.2p2, SSH protocols 1.5/2.0, OpenSSL 0x0090600f

The current version of openssh for 7.x, and this includes Alpha as well,
is openssh-2.9p2-8.7. There were some security updates. You can find
more on http://lwn.net/ and on bugtraq. Check also Red Hat
annoucements on www.redhat.com.

> ssh and sftp work fine from nodes on the same domain. However,
> when one tries to connect via ssh or sftp from a remote node
> on a different domain the connection is refused returning the following
> message
> ssh_exchange_identification connection refused.

Bad values in "known_hosts" files? Messed up reverse DNS? Blocking
needed ports on a firewall? Wrong routing? Some other misconfiguration?

You can find out more what happens in a setup phase if you will run
'ssh -v ....'. Actually you may have up to three v's and
'ssh -v -v -v ...' will print _a lot_ of details on your screen. You
may want to use 'script' to capture that all.
