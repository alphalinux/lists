Netscape you are using on Alpha is not a Linux application but DU
binaries in ECOFF, not ELF, format running thanks to compatibility layer
in your kernel and extra libraries from Tru64-compat package. Try
'file /usr/lib/netscape/netscape-navigator' or whatever is installed on
your system. I would be _really_ surprised if that could load Linux
binary format modules and/or dynamic libraries.

OTOH if you will recompile that plugin on Tru64 then resulting stuff
may even work on Linux. I have no idea if anybody tried. Probably
not. This is for somebody with an access to a required OS. Maybe
shockwave plugin for DU already exists somewhere for a download?

IIRC Netscape binaries for DU are compiled with a help of '-taso',
or '-xtaso', switch which limits it address space to 32 bits. Any plugin
would have to use the same address space - obviously enough. Here you
have the next reason why you are getting runtime errors after a "naive"
recompilation.
