Hi, all,

I am migrating one of my Alpha workstations from Linux (RH7.0) to Tru64 5.1.
However, the installation fails when it tries to create a correct disklabel
on the disk where I had originally installed Linux. For example, if I try
to write a default disklabel:

# disklabel -w /dev/rdisk/dsk0a
disklabel: ioctl DIOCWDINFO: Open partition would move or shrink
Use alternate partition

Or if I try to zero the disklabel:

# disklabel -z /dev/rdisk/dsk0a
Disk is unlabeled or, dsk0a is not in block 0 of the disk

Can anyone here tell me how to reset the disklabel so that Tru64 will know
what to do with it?

Thanks in advance!
