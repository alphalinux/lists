Alvin Starr wrote:

>Greg Lindahl wrote:
>>
>> On Thu, Sep 06, 2001 at 01:14:07PM -0700, Eugene Chu wrote:
>>
>> > On all of them, the BIOS logs have registered numerous single-bit ECC
>> > errors, and a few multi-bit ECC errors. I've always thought that ECC
>> > will correct single bits, and detect multi-bit errors, which are fatal.
>>
>> And you're wondering why they hang? You should only get an occasional
>> single bit ECC error on a correctly working system.
>
>I would worry if a system generated a single bit error more than once a
>month or so. If I remember correctly the ECC correction process will
>correct single bit errors flag double bit errors and if there are more
>bits bad then you are out of luck.

Indeed, it would make me worry too.

>Depending where the ECC detection is being done you may be having memory
>buss problems more than bad bit ram problems. If taking 1/2 the ram out
>makes the problem better there may be a buss loading issue there.

Well, this seems to be consistent with what I'm seeing and what
Thorsten told me. Moving the memory around doesn't seem to change
behavior. So I'm worried about my symptoms being from a basic design
problem; the bus simply can not handle a full memory confiugration. I
wonder if power supply voltage would affect this? I can try to take
some measurements on the board, but I'm not sure if the modern power
supply has any fine adjustments like the older styles did.
