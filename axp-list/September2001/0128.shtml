Hi,

I just installed the new openssh-2.9p2-8.7.apha.rpm on my xp1000 under
rh71. But openssh doesn't work correctly for me. The command

        ssh -n machine1 "command1" | ssh machine2 "command2"

(which is part of a crontab job) used to work fine several hundred
times a day since months under openssh-2.5.2p2 which came with rh71
and under a hand-built ssh-1.2.31.

Now with openssh-2.9p2 that command says
        Write failed flushing stdout buffer.
every now and then. Not always, just when it likes to. In these cases
stdout of machine1 which is transferred to machine2 is incomplete.

I checked groups.google.com, but I couldn't find a solution to my
problem. You can find there several problems with the scp command and
that error message, but not with the ssh command. My shell
initialization files (.profile etc) definitely don't write anything to
stdout - or else it wouldn't have worked perfectly under
openssh-2.5.2p2 and ssh-1.2.31, wouldn't it? Also, my problem is with
the ssh command, not with the scp command.

Does anyone know, if there is a bug in that openssh-2.9p2 rpm, or did
I configure something wrong which accidently worked up to now? Is
there a work-around besides downgrading to openssh-2.5.2p2?

Thank for your help,
