On Mon, 15 Oct 2001, Emmanuel Seyman wrote:

> I'm running RedHat 6.2 on an AlphaPC 164LX. The machine is a web
> server (Apache, PHP and MySQL). Somebody's installed HtDig
> and now I have these error messages (plus an average uptime of 2.3):
>
> Oct 15 10:32:22 beta kernel: htdig(25372): unaligned trap at 0000000120072aa0: 00000200000a5acb 28 1
> Oct 15 10:32:22 beta kernel: htdig(25372): unaligned trap at 00000001200675b8: 00000200000a5ac7 28 3
> Oct 15 10:32:22 beta last message repeated 6 times
>
> Does anybody see where the problem is?

The problem lies in htdig. I use htdig to index my archive of this mailing
list. However, I maintain this on an x86 box partly because the author of
htdig seems unwilling to put much effort into supporting Alpha.

I don't know if he dislikes 64 bit architectures in general, or if he just
doesn't like Alpha that much, or if I misunderstood him entirely, but he
did respond rather coldly to me when I asked him about unaligned traps in
one of the 3.2.0 betas (something you'd think he would be interested in
getting bug reports about).
