Coincidentally, I just installed RH7.1 this week, with ext2
partitions _only_, and everything boots fine as long as I use the
numbered entries in aboot.conf.

One thing I have heard (on this list, IIRC) is the kernel must be
in the first 2GB of the partition on which it resides, which
basically means the partition (/boot) on which the kernel lives
should be smaller than 2GB. All my /boot partitions are 1GB.

With RH6.1 and RH7.0, I used the SRM variables to control exactly
which kernel would be booted. I have yet to try the <n>/ prefix
the more modern aboot versions seem to want. With luck, that
will allow me to use the SRM variables to control booting instead
of having to muddle around with aboot.conf.
