[Applied to i386 dist as well!]
Message in /etc/rc.d/rc.modules is either confusing or just plain
wrong:-
# Check for /sbin/insmod
if [ ! -f /sbin/insmod ]; then
echo "iBCS: /sbin/insmod missing !"
exit 0
fi
The "iBCS:" in the message should be changed (to "rc.modules:"?).

[More general one on packaging]
The initscripts rpm installs init stuff for various things that are
installed as separate packages (ie lpd, sendmail, nfs)cd ..
. Surely the packages should include these themselves??
Philosophical point - this makes the packages more RHCL specific, but
not dramatically so.

bootptest core dumps! (in NetKit-A)
>From previous experience on bootp work it may not be happy with a
64bit base, I'll come back on this one. I only want client side
bootp so haven't looked at the server.
bootpc does not do anything useful (certainly it doesn't work!).
[again a packaging question - NetKit-A seems too big and too full of
disparate items]
A check of this shows that both bootpc and bootp declare the bootp
XID (in bootp.h) as (effectively) a long - 64 bits - rather than 32
bits. Fixing this solves most things. However the IP/Ether layer
does not correctly handle IP layer broadcasts.
