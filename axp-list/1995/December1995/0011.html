I thought that I would just for fun try the Byte benchmarks to
compare the box to a standard pentium.

Imagine my suprise when I found that apparently the beast does
infinite dhrystones per second.... some system!

/usr/bin/time was returning 0 for sys and elapsed time under all
circumstances - user time appeared OK.
Recompiling (ie installing the SRPM and building it), fixed that.
Presumably something in the kernel has changed???
I am using a 1.3.46 kernel, built here.

Also, with the byte benchmarks, the pipe context switch test won't
work. I finally tracked this down to a bit of code that (roughly)
does this:-

unsigned long iter
int check
...
write(p1[0], (char *)&iter, sizeof(iter))
...
[in different process]
read(p1[0], (char *)&check, sizeof(check))

The moral being, read into the same variable type as you wrote.. or
don't assume sizeof(int) == sizeof(long)

[FYI, the raw speed benchmarks are better than a good pentium, the OS
dependant ones are not so good - in fact in some cases really lousy.
Hopefully optimisation of the system should improve these. I can
post these when I have rerun them]
