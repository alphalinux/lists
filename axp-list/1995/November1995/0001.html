OK, here's another one (a test reply, actually... :-)

I grabbed those kernels from off that FTP site, and the MULTIA one works
fine. I'll try the EB66 and MULTIA ones at home sometime in the next few
days.

One thing I *have* noticed from the latest (42-43) kernels: the audio is
*very* choppy (on MULTIA, at least) unless you use the mixer to set the
"line3" value to 0 ("mixer line3 0"). Then it seems to work fine, both
simple /dev/{dsp,audio} as well as the MOD player.

I tried sound on a PAS16 on the EB66 and got the same choppy/static in
the output there as well, with a 1.3.42 kernel. Didn't think to adjust
"line3" (or whatever) to see if it helped.

Must be something Hannu changed since the late 30's kernels, as it wasn't
nearly so bad then (although still present)...

--Jay++

NOTE: the "mixer" is the applet found in sndkit-x.x packages as simply
mixer.c in the top directory.
