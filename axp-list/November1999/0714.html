Red Hat 6 has some significant errata (eg putting up an xterm screws the
system), but the updates are good. Apply the updates and you should be OK.
For kernel we use standard 2.2.13 and it seems to work.

If you're new to installing Linux on Alphas (and the SRM console firmware),
there are some catches about how you make your system bootable. If you
haven't done this before, I can give you the gory details.

Red Hat 6 adds an extra catch, because the BSD partition table section of
fdisk offers to make all your partitions one unit bigger than the size of
the disk. This killed our DP264 system so completely, it practically left a
crater in the machine room floor :-)

With the exception of this caveat, the DP264 has worked well and been solid
for about eight months. But I work for a company that sells them...

There are some white box Alpha vendors in the UK, you might like to try
CompuSys.
