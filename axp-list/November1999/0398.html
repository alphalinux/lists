I want to update everyone in the same vein...

I've achieved new heights of stability by statically linking the sound
drivers and *not* using modules. I still have to use the plug and play
driver, which complains of irq conflicts on boot, but my sound card wouldn't
be properly initialized without it.

I've been up for 3 1/2 days with constant sound use, and I'm listening to
mp3's right now as I type this!

Why is the system so unstable when the sound drivers are built as modules???
I don't know. But if this keeps up, I'll be very happy!

Kurt, I'm curious... Were you filling all your RAM slots when you were using
384MB? It's possible that if you weren't and had your motherboard jumpered
for 256-bit memory accesses, that your system was screwing up.
I'm pretty sure that the 256-bit memory access can only be used if you fill
*all* the RAM slots. I could be wrong, but it's a thought.
I have 64MB ( 4 X 4MBx36 ) and I have my board jumpered for 128-bit accesses.

-Mark
