        I hope it's a lot better than the 6.1 release for IA. I was not at
all impressed (by 6.1) given that RH now has shareholders to answer to and a
large staff with plenty of time to build & test the dist. I was amazed to
see that many packages were not selected for install - sure, it was all
minor stuff like: bzip2, tar, find, less, rpm, grep, etc. The list was
quite long......
        The install itself was shipped broken - you need an updated boot
floppy or to specify "updates" (and still have the floppy) in many cases
(mount perms issues). Seems like RH spent too much time on the happy
X-based GUI install and not enough time on the content behind the pretty
window.
        Another thing - they have you configure X before a single package is
installed. Sure, you can skip the step, but I can just see all the newbies
that "follow the instructions" and after spending 2 hours selecting packages
for install they go to configure X and the system hangs because they've
botched something and they have to start from ground zero. I also had
strange behavior out of fdisk, but who can say if that's a RH thing (I
loaded 14 Netfinity 3000 boxes simultaneously).
        FWIW - the X based install on my laptop (Compaq Armada 1750) was
questionable. If you're attempting it you might want to configure X after
the install.....

        I purchased Linux-Mandrake 6.1 about 3 months ago, and other than
the broken PCMCIA code (easily fixed) it's a far better dist. IMHO. Whish
they did Alpha.
