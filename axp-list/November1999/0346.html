Greetings,

I recently acquired an (apparently) new an unused Multia from a co-worker.
After many weeks, I had finally obtained enough components to get it
working (memory, kbd, mouse, floppy, etc). However, to my dismay, it
appears it is fatally inoperable. I'm hoping someone can suggest
something.

I am already intermediately familiar with several Alpha systems, and I am
acquainted with some of the peculiarities of the UDB (including the battery
problem).

My system has 2 pair of working 8MB x36 SIMMS (pulled from my AS200) and a
new battery. It fails to boot. Removing the battery/re-arranging the
memory makes no difference. This is a VX40. The symptom is that the
system will power up (fans, CPU gets hot over time, etc), but the console
never initializes. The SROM LED on the back flashes hex E, indicating a
memory configuration error - or mainboard failure. Since no memory
combination I try (including no memory present) does not change this error
code, I am dreading the worst. Is there any hope? If not, where can I get
a replacement? For the cost of the unit, I'd rather replace the entire
thing (perhaps for a better Multia) than try to get a new mainboard.

Suggestions?
