>>> "Jakma, Paul" said:
> > From what I learned recently "features" in core chipsets on existing
> > boards (read "logic bugs") makes it very unlikely that this can be
> > doable on anything below ev6. Machines which currently are showing
> > up should be fine but LXes or UXes are out. None of these
> > older designs
> > was actually ever officially qualified to run with more than 1.5 GB
> > of RAM.
> >
>
> perhaps those boards aren't qualified. but the chipset, 21172, 21174
> certainly is. IIRC the same same chipsets are used in DEC ev5 servers which
> are qualified for 2GB+.

If you're thinking RAWHIDE (AS4000 or AS4100) they did not use CIA,
but their own chipset (MCPCIA); same with SABLE/GAMMA/LYNX (AS2000,
AS2100, AS2100A), which used the T2/T4 chipset.

It's possible that AS800/AS600A/AS1000A (NORITAKE family) which used CIA-2
(21172A, IIRC) were qualified at more than 2GB, but I have no knowledge.

And actually, it's not so much the core logic chipset that is the
determining factor, but the SROM code, which does the memory sizing
and configuration during initial power-up. If the SROM has appropriate
code for handling the large DIMMs, you win, otherwise you (may) lose.
