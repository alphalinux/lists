>
> Don't know much about ISA on an alpha, but I remember somthing
> about the ISA system on x86 and DMA transfers. There is this nice
> little option in the sound section to grab a chunk of mem for DMA and
> hold it (but only when compiling the card in, no modules). Somthing
> about ISA DMA transfers and memory locations above 16MB causing
> strange things to happen.

Speaking of which.. Most modern alpha's (at least 21172 chipset and up iirc)
support programmable scatter-gather bus -> phys address space windows. Using
these would solve the ISA 16MB address space problem, ie we could map the
16MB of contiguous ISA space to anywhere in physical space. We could even do
it chunks, so the mapping wouldn't have to be contiguous. Same argument for
the rest of PCI addr space.

Also might make 2GB+ RAM on alpha possible. The standard answer to "how do i
get more than 1GB to work with my PC server?" was always "go get a proper
64bit machine like an alpha".. Yet now x86 supports 4GB+ RAM and alpha
doesn't!!!

Is anybody working on scatter-gather windows for Alpha? I guess this'd need
some kind of uniform kernel scatter-gather support in addition to the alpha
hardware specific stuff though... which is perhaps an issue for
linux-kernel.
