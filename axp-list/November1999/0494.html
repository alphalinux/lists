steve@icarus.com said:
> After a week, the SCSI chain finally gave me trouble again. When it
> failed, all the devices on the chain started reporting errors, the the
> bus was reset, and that's the last message on the screen.

I can be more specific, now. Linux/alpha 2.1.13 w/ an AHA2940UW and
two hard disks. I managed to hang the machine a dozen or so times,
and every time, it was a timeout (and bus reset) doing *writes* to
the disks. It's always writes that timeout, and it's always errors
more or less the same time, going to *both* the disks. I've never
seen an isolated error.

It seems to be able to read forever without any trouble.

I've disabled tagged queuing to both disks, no improvement.

I have ordered a new Intraserver ITI3140U. If that fixes it, then the
Adaptec card gets abandoned. If there are any tests anyone wants to see
done, I'll be willing to try them up until the day my new card arrives.
