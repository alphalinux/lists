        I've noticed the font server causing instability problems for me.

        I ran the RedHat-supplied xfs for a long time, but my X server
wasn't configured to use it (I used the config I had from back w/RH5.2,
which I'd hand-tuned to run at 110Hz refresh rate). Once I noticed this, I
turned it on because I wanted TrueType support. Prior to this, I never had
any relevant problems.

        It crashed. Repeatedly. It often caused X to not function when it
died. Since X uses direct hardware access, I sometimes had to reboot to fix
the problems. X itself was also unstable in ways that should not
theoretically be related.

        I reconfigured /etc/X11/XF86Config to not use xfs, and all those
problems went away again. I then turned off xfs, and told chkconfig not to
start it on boot (chkconfig --del xfs).

        These aren't exactly the problems you're reporting, but they're
similar enough that I offer up my experience as something to try.

G'luck!
- Bob
