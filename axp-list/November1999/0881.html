Chris Kloiber wrote:
>
> I think that between 6.0 and this
> Rawhide, I have noticed a subtle difference in the way ext2 partitions
> are formatted, with much fewer copies of the superblock being created
> now. This seems to have broken both milo and aboot,

Are you using e2fsprogs-1.17 and partition from which you are booting
is bigger than 1/2 Meg in size? IIRC (i.e. size) correctly this version
of e2fsprogs will format such partitions by default with 4K blocks
and sparse superblocks but there are options which allow to override
these settings. I am not aware of any version of Alpha boot utilities
which would be able to handle a file system formatted that way.

If you are using milo and you have a FAT partition on your hard drive
you should be able to boot with a kernel on it.
