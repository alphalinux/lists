Maurice Hilarius wrote:
>
> A good example:
> The LX motherboard, equipped with 2MB cache and a 633 CPU got a spec95 fp
> of 21.4
> Stick the same CPU in an LX with 4MB cache and you get 26.8

Where did you find these numbers? It doesn't appear to be on the
SPEC web site near as I can tell. Only up to 600MHZ is listed.

> So, a 2MB cache difference makes this machine 25% faster? No, I really
> don't think so..
>

So, then why bother with cache at all if it doesn't really
improve performance?? (just kidding)

It's entirely possible depending on the working set size of
the problem you're testing. All you're doing is putting less
burden on the memory subsystem. (this does say the LX
memory subsystem is not the processor's equal)

It all depends on your inner loops memory consumption.
Vendors have used caches for a long time for exactly that
reason. I'd consider IBM to have the best understanding of
cache subsystems, but others are catching up. Same thing
would help other cpu's, if they are hammering memory.

Fast large caches do work, even in the "real world". Ask
yourself how many processors you know of that don't use
some amount of cache. I know of only one or two.
