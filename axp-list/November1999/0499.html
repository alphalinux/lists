I have a problem with my SX164 where on a "warm reboot" the options
which get passed to MILO seem to be corrupted such that it fails to
reboot. This may be the problem the original message is referring to.

For example, if I boot from AlphaBIOS my boot string might be something
like:
boot sda10:boot/vmlinuz-2.2.13 root=/dev/sda10
on a warm reboot, MILO will attempt to boot with a
boot string along the lines of (can't remember exactly, sorry):
boot sd10:linuz-2.2.13 root=/dev/sda10, which of course fails.

Entering the correct boot string at the MILO prompt works OK, as does a
cold boot from AlphaBIOS.

I haven't tried every available MILO, because it isn't really a problem
for me, but it would be nice to know that a warm reboot would result in
the machine coming back up again without intervention.
