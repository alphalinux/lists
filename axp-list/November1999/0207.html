I think it would be wise to assume.

1. If you have an NT-only (33xx,53xx,73xx,xlt....) system, or an older oem
system for which there is no SRM available, you will not be able to run the
$99 TRu64 unix. You can switch the 33,53,73 to srm console, but they will
still not boot unix or vms.

2. If you have a Standard AlphaSever, or a PWSXXXau (note "au") you should
be ok, most PWSXXXa's should be ok, as most did not ship with an Adaptec
disk controller as primary, though it would be wise to check first, and
install the latest firmware.

3. However.. if you added an Adaptec disk controller and are currently
booting linux from AlphaBios. You will not be able to boot from SRM, which
you need to do to run TRU64, as currently SRM does not support the Adaptec
as a boot device.
