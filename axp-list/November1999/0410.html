Bill Broadley wrote:
>
> Netperf is a great tool, I've used it to diagnose various strangeness
> that occur when an 8 node beowulf starts running heavy MPI code and
> similiar.
>

I've used it but don't usually bother with it. I just
rsh to the machine I want a file from and launch "dd"
and either pipe it into a program that will process it
or ">" it into a local file.

For example:

rsh remote_machine dd if=/remote/file/name bs=64k > /local/file/name

This is the fastest way I've found to move data around.

Of course if you use rsh, make sure things are secure.

YMMV
