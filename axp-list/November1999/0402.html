> As a point of reference, I use various tulip based 10/100 cards
> and they typically sustain 7-9MB/sec on large file transfers.
Interesting. Is both the client and server running Linux?

Hardware configuration of the FBSD and RH boxes is identical,
all operating systems are "out of the box" without black majic
optimizations.
Simple experiment using a single 100M dummy file created with
dd and /dev/zero, transfers are done using scp:

Server Client avg xfer rate(kb/sec)
Solaris 2.6/sun4u RH 6.0/x86 70
FreeBSD 3.3/x86 RH 6.0 43
Solaris 2.6 FreeBSD 3.3 350
RH 6.0 FreeBSD 3.3 475

I would blame this on ssh, but the transfer rates are mirrored
(+/- 10%) by ftp.
