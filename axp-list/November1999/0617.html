On Thu, 18 Nov 1999, Dr. Andrew Almond wrote:

> Hi,
>
> I'm running redhat 6.0 on a UP2000 machine (2x ev67 CPUs), and have
> noticed that when I run kernel 2.2.13 with SMP and rtc support then
> top,ps etc. are broken i.e. they only show statistics for one of the
> CPUs. However, if I recompile without rtc then top shows the correct
> information, but then the system date is erratic. Has anyone else
> observed this behaviour?

Yes, don't compile in rtc support! You can also see this for more info:
http://www.alphalinux.org/archives/axp-list/September1999/0474.html
 
> Also "show config" from the SRM only lists the floppy drive, and not the
> hard-drives or cdrom which are coupled to the on-board SCSI controller.
> Hence I'm currently booting from a floppy. Is this normal, or am I
> missing something stupid? Its a new machine so I presume it has a the
> newest version of the firmware.

Correct, there is currently no driver in SRM for the onboard SCSI chip (an
adaptec 7891).
