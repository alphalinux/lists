There are jumper settings for the Multia's video that SRM uses. Here's some info I
found somewhere:

VX40 VIDEO JUMPER SETTINGS

A number of UDB owners have discovered that the default SRM video console mode
"Digital 1024 x 768" set by jumpers W3 and W5 being present
on their system is not suitable for the monitor they use to boot their systems, the
following illustration provides clarification of the jumper locations and settings.

Jumpers W6 to W3 for all models. 0 indicates that the jumper is installed, 1
indicates no jumper present.

W W W W Vertical Pixel
6 5 4 3 Video Standard Resolution Refresh Freq (MHz)

0 0 0 0 VGA 640x480 59.940 21.175
0 0 0 1 ERGO VGA 640x480 72.790 31.500
0 0 1 0 SVGA 800x600 60.320 40.000
0 0 1 1 ERGO SVGA 800x600 72.190 50.000
0 1 0 0 VESA 1024x768 1024x768 70.070 75.000
0 1 0 1 Digital 1024x768 1024x768 72.033 74.367
0 1 1 0 Digital 1024x864 1024x864 60.000 70.000
0 1 1 1 1280x1024 60Hz 1280x1024 60.100 110.150
         S3 86C928
1 0 0 0 1280x1024 66Hz 1280x1024 66.510 119.843
1 0 0 1 1280x1024 72Hz 1280x1024 72.556 130.808
1 0 1 0 1280x1024 72Hz 1280x1024 72.050 130.080
         S3 86C928
1 0 1 1 1280x512 Stereo 1280x512 139.336 130.804

My multia starts out in SRM mode for a few seconds then switches to ARC. While in
SRM mode, all I see is a few rows of text that's been skewed by mismatched refresh
rates. The text is not recognizable as text. I know it is because on another
monitor that does support the default setting, it displays the SRM screen.

If your monitor doesn't support the default, try another setting that it does
support.
