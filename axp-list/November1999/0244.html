Hello all,

I'm trying to get a DS-10, Redhat-6.0, and Mathematica-4.0 working.

I followed various notes on getting Digital Unix/Tru64 binary compatibility
working, and have had a fair amount of success. For instance CPLEX works:
        http://www.cplex.com/

For instance:
[root@volume cplex65]# file ./cplex
./cplex: COFF format alpha executable paged stripped - version 3.11-10

It runs fine, (for those interested the 466 Mhz 264 runs about twice as
fast as a PIII-500 on our cplex problems).

I searched the archived of this list and found a few references to
Mathematica-3.0 working, and the 4.0 Kernel (not the gui) working, which
is great.

So I couldn't managed to get the install to work:
You can specify the type of computer system on the MathInstaller
command line as -sys <sys> where <sys> is the Mathematica
$SystemID specification for a computer system type.

I wasn't sure which to use for -sys.

So I installed on a DU 4.0 box, and then copied it back to my linux box.

Volume = ds-10 running linux.

Morgan = OSF1 morgan.itd.ucdavis.edu V4.0 878 alpha
