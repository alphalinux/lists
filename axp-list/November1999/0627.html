Konrad Podloucky wrote:

> Hi there!
>
> The latest gnumeric (0.41) gives me a floating point exception when
> trying to have a look at the print preview of a non-empty spreadsheet.
>
> Unfortunately I can't provide a gdb-backtrace, because I have no
> debugging information compiled in. The only thing I know is where the
> exception occurs:
> Program received signal SIGFPE, Arithmetic exception.
> 0x20000941bf0 in art_bpath_affine_transform ()
>
> I really haven't got the time right now to do some debugging myself so
> I just wanted to know, if that's a known problem and if there's some
> kind of patch available.

Sounds like a problem with libart, the canvas rendering engine, which is
in the gnome-libs package. If you file a bug report at bugs.gnome.org
(basically email submit@bugs.gnome.org with the first two lines "Package:"
and "Version:"), you should get a pretty quick reply.
