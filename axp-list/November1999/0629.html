> And now, the showstopper. I wasn't able to enter text into any textbox into
> Mozilla. The URL bar wouldn't accept text input. Okay - I'm used to that (I
> usually browse with Mozilla 100898). Unfortunately, "Open URL" wouldn't take
> any input either. The only way I could specify a URL was to do so on the
> command line. Speaking of text boxes, I wasn't able to enter any text in
> HTML
> forms either. Or in any text box in the preferences dialogs. Or - well,
> anywhere in Mozilla. This makes using it as a web browser rather -
> challenging. :)

I at least figured *this* one out. Apparently, there are a couple of settings
in the default preferences file that *must* be in there for the text input
boxes to work:

user_pref("nglayout.compatibility.mode", 2);
user_pref("nglayout.widget.mode", 1);

Might be an RTFM error on my part or an installation hiccup. At any rate, I
can enter text into the dialog boxes and such now, so I can start putting
Mozilla through its paces as a web browser.
