Adam C Powell IV wrote:
>
> Wes Bauske wrote:
>
> > If one could get the compiler to issue AMD's multimedia instructions,
> > the Athlon would have a peak performance of 2GFLOPS at 500MHZ, Alpha
> > 21264 is 1GFLOP.
>
> REALLY? Is this double-precision? I know G4 500 MHz is claiming 1 GFlop
> sustained, but that's only single-precision...
>

I have an interest in G4 too but the coding style is cumbersome
at this point since the compilers don't generate Altivec
instructions automatically. I ain't going back to that
kind of code. I did something similar 10-15 years ago with array
processors.

Single precision FP at 2GFLOPS. It's only 2x the clock rate for
64bit FP for 1GFLOP peak, not sustained. I've had my Athlon sustain
300-400MFLOPS in tests but the compiler breaks on some so I can't
get a full view of it's performance (both egcs1.1.2 and gcc2.95.1).
I can drive Alpha 21264's at 800+MFLOPS, sustained.

That code SEGV's on Athlon when it initializes.

Many problems are solvable in single precision. 99% of mine
are so the higher multiplier (4x vs 2x) is useful to me.

YMMV.
