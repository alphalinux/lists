On Sunday, October 31, 1999 9:56 AM, Maurice Hilarius
[SMTP:maurice@harddata.com] wrote:

> Fastest shipping chips readily available are 667MHz.

Not true. There are plenty of 750 MHz chips available, but
we have very few chip level customers, so modules are
more interesting.

> 750MHz is easily done but:

I don't know about easily, but we have done it.

> with current chipsets a chip at 750MHz has a 75MHz memory bus
> clock.
> To get a reasonable gain in performance and overcome this memory
> speed issue:
> a bigger cache is needed 8 MB ideally.
> There is a problem with 8MB cache running reliably in the
> current CPU packages

There is no reliability problem with the 8 MB cache. The problem
has been getting a predictable supply of parts, but now that we've
switched vendors I think that problem is solved. Having said that, we
have had 750 MHz 8 MB Slot Bs in customer's hands for some time,
just not as many of them as we'd like...

> Cache on SlotB runs at CPU clockrate, I believe.

The cache speed is variable and set by a divisor. I can
guarantee that it's not running at 750 MHz with today's
technology!

>
> In simple terms: If API or Compaq shipped a 750 with the same cache
> and
> other design details as the 667, it would not be much ( if any)
> faster,
> depending on which benchmark you ran.

This is misleading. Of course everything depends on the application,
(as does everything else in the real world), but a 750 MHz UP2000 is
faster than a 667 MHz UP2000 for most tasks.

As is typical, for this type of thing, there are three cases:

1. Application fits in cache for both systems - Performance
scales with processor speed.

2. Application working set fits in 8 MB, but not 4 MB - Application
scales superlinearly, benefiting from not only increased processor
speed, but also faster memory access from L2 instead of
main memory.

3. Application working set is much bigger than 8 MB - Performance
is hurt by slower memory bus (to the degree that this isn't
masked by larger L2 and out-of-order execution)

As with everything else, you need to pick the configuration which
is appropriate to your application, but there are many applications
which will benefit from 750 MHz UP2000s.

> We are now being told that nothing faster than 667 will ship in the
> near future.

You should check back with your reseller after the conference
next week.
