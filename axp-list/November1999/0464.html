>
>
> Speaking of which.. Most modern alpha's (at least 21172 chipset and up iirc)
> support programmable scatter-gather bus -> phys address space windows.

From what I learned recently "features" in core chipsets on existing
boards (read "logic bugs") makes it very unlikely that this can be
doable on anything below ev6. Machines which currently are showing
up should be fine but LXes or UXes are out. None of these older designs
was actually ever officially qualified to run with more than 1.5 GB
of RAM.

>
> Also might make 2GB+ RAM on alpha possible. The standard answer to "how do i
> get more than 1GB to work with my PC server?" was always "go get a proper
> 64bit machine like an alpha".. Yet now x86 supports 4GB+ RAM and alpha
> doesn't!!!

Well, it does and an alpha machine with over 4GB of memory running Linux
already exists. But this work (not mine :-) is still in early stages
and a driver support is rather limited. Still things will happen RSN.
I dabbled a bit in a driver conversion really trying to understand what
is required; it can be "fun" and still doable with that knowing a driver
really well would be of a great help.

   Michal
