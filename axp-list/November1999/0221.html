I've been running AlphaLinux for years, but always with Milo from the Arc
console. A friend of mine is an Alpha BSD developer. Apparently alpha
BSD can only be launched from the SRM console and he claims a performance
benefit from NOT using the arc console, apparently having to do with the
pal code and what is passed to the kernel.

This seems a bit suspicous to me, but I know NOTHING about the internal
workings of the consoles and how PAL code works. What are the differences
between the consoles? and how does the pal code works? Are there any
bottle necks we need to be aware of?

It seems that we (the alpha community) eventually are all going to be
going to an SRM console pretty soon as the NT console looks to no longer
be supported. Or am I mistaken?
