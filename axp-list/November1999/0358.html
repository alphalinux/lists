Hi folks,

I'm having some SCSI funnies, and was wondering if anyone can shed some light
on these for me...

I have an AlphaPC164LX, with one Adaptec 2940UW, and one Symbios 53c875 board.

$ uname -a
Linux ludwig-alpha 2.2.13ac1 #4 Tue Nov 2 15:10:02 CET 1999 alpha unknown
                         ^^^
It says ac1, but it should say ac2...

Anyway, the Symbios board is connected to an internal Seagate Cheetah drive,
and an external Jaz 1Gb.

My system disk (an old Barracuda) is connected to the Adaptec, and seems happy.

Every now and then, when I use the Cheetah drive while compiling things (I
have /usr/src mounted there), I see this in the dmesg log:

sym53c875-0:1: ERROR (0:18) (8-28-0) (f/9d) @ (script be8:18000400).
sym53c875-0: script cmd = 18000000
sym53c875-0: regdump: da 10 80 9d 47 0f 01 0e 26 08 81 28 80 00 00 08.
sym53c875-0: resetting, command processing suspended for 2 seconds
sym53c875-0: restart (scsi reset).
sym53c875-0: enabling clock multiplier
sym53c875-0: Downloading SCSI SCRIPTS.
sym53c875-0: command processing resumed
sym53c875-0-<1,*>: WIDE SCSI (16 bit) enabled.
sym53c875-0-<1,*>: FAST-20 WIDE SCSI 40.0 MB/s (50 ns, offset 15)

1 is the Cheetah's id.

These come in short bursts. Nothing really bad seems to happen, but I'm still
puzzled (and a tad worried).

The regdump line varies a bit from one message to the other:
sym53c875-0: regdump: da 10 80 9d 47 0f 01 0e 62 08 81 28 80 00 00 08.

Can anyone please translate this message to something more "human readable",
or point me in the right direction.

When I use the Jaz drive, I do not see those messages appearing.

I also tried to use the ncr53x8xx driver, and obtained similar messages.

If you need more info, please tell me.

TIA,
                                        Christian
