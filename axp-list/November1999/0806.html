I've been trying to compile a few apps with CCC.
However I seem to have a consistent problem and was
hoping someone knowledgable on this list could help
me.

It seems that whenever I have multiple source (*.c)
files that include <signal.h>, I end up getting
linking errors. For example:

source0.c - which uses <signal.h>
source1.c - which also uses <signal.h>

ccc -O2 -fast -c -o source0.o source0.c
ccc -O2 -fast -c -o source1.o source1.c
ccc -o program source0.o source1.o

at the linking stage I keep getting "multiple function
definition errors".. and more specifically, the
functions are:

__sigismember
__sigaddset
__sigdelset

which are functions declared in sigset.h which is
included by signal.h.

here is a sample output from an actual compile:

dialog.o: In function `__sigismember':
dialog.o(.text+0x0): multiple definition of
`__sigismember'
defaults.o(.text+0x0): first defined here
dialog.o: In function `__sigaddset':
dialog.o(.text+0x30): multiple definition of
`__sigaddset'
defaults.o(.text+0x30): first defined here
dialog.o: In function `__sigdelset':
dialog.o(.text+0x60): multiple definition of
`__sigdelset'
defaults.o(.text+0x60): first defined here

I'm not really a programmer, so my ignorance is
keeping me from understanding why this is happening.
(BTW, this all works without any problems using
GCC/EGCS)

I've seen the same exact errors when compiling the
source code of other things too. Turning off the -fast
doesn't help either.

Other information:
CCC 6.2
glibc 2.0.x
RH 5.2 with many updates.
