Playing tag?

I have an x86 Solaris box (Solaris 7) that I have had as home
server for quite some time (since it was Solaris 2.4). I have
had absolutely NO complaints from it or about it - except that
is was a royal pain to configure. NONE of the default programs
that you need to do decent webservice were included, nor was
even a rudimentary compiler. Myself, I used it because it was
so difficult and I wanted a thourough knowledge of the programs
and configs. It operates flawlessly under load, with uptimes
of several months being extremely common (the only time I have
to reboot it is when the internal modem goes nuts - part of the
x86 trash that I haven't had the gumption to replace).

If I was (still) in a work environment with the appropriate software
and hardware support, I would take a Sun Sparc over an x86
anything anytime. But that's not real life. Sun software costs. A lot.
I have not had the same results with linux. When a sparc hoses
a mainboard, you just ate a sparc. When my linux box eats a
mainboard, I fork over a good $90 for a decent motherboard,
install, reboot and let the kernel figure out the new settings.

As part of it's job description, a typical web server is constantly
undergoing updates and revisions. The ability to drop in the
latest P-III 600mhz into a previous P-133 and have it work is
a real strong arguement for a flexible OS with a hardware and
software upgrade path. And as much as I like Solaris, it is not
that OS. Linux is.

- James
