Just a general note, you need many trials to eliminate noise from
downloads from a possibly busy www/ftp site across the ever changing
internet.

For those who want to beat on many different aspects of a local network
I'd suggest netperf:
http://www.netperf.org/netperf/NetperfPage.html

From the www:
Netperf is a benchmark that can be used to measure the performance of many different types of networking. It provides tests for both
unidirecitonal throughput, and end-to-end latency. The environments currently measureable by netperf include:

     TCP and UDP via BSD Sockets
     DLPI
     Unix Domain Sockets
     Fore ATM API
     HP HiPPI Link Level Access

I've often seen close to wirespeed (within 10%) over various 10 mbit
and 100 mbit networks with linux. It doesn't surprise me that *bsd and
linux have slightly different handling of falling back/throttling to
make the most of a slow 70k/sec connection.

Netperf is a great tool, I've used it to diagnose various strangeness
that occur when an 8 node beowulf starts running heavy MPI code and
similiar.
