Mozilla M11 built successfully for me on my first try. This is approximately
where my success with M11 ended. :) It runs (asking me to create a user
profile). However, I immediately notice that I can't type text into the text
box asking me to name my profile. I can, however, click the button to accept
the default and get out of the Wizard, so I do so. A bit later, the Mozilla
window opens and begins to load one of the pages off of mozilla.org.

One thing I notice quickly is the dynamic page rendering. Nice trick, but it
seems to kill performance during a page load (eats 100% CPU during the entire
page load). The stop button, as cool as it looks, doesn't seem to respond
during this loading. This behavior has been mentioned by Intel users too, so
I suppose it isn't an Alpha=ism. The other thing I immediately notice is the
sidebar on the left. Shrinking it does strange things to the Mozilla window.
Luckily, it can be turned off via the menus.

Ahh yes, the menus. I'm not sure what the problem is (others have also
reported this behavior), but trying to bring up the menus takes an eternity.
Despite the apparent speed of the rendering engine (which seems to render the
HTML I've run through it nicely), the UI is slow. Extremely slow. Unusably
slow if you actually want to use the menus. Menus shouldn't pop up several
seconds after you click them.

And now, the showstopper. I wasn't able to enter text into any textbox into
Mozilla. The URL bar wouldn't accept text input. Okay - I'm used to that (I
usually browse with Mozilla 100898). Unfortunately, "Open URL" wouldn't take
any input either. The only way I could specify a URL was to do so on the
command line. Speaking of text boxes, I wasn't able to enter any text in HTML
forms either. Or in any text box in the preferences dialogs. Or - well,
anywhere in Mozilla. This makes using it as a web browser rather -
challenging. :)

On the upside, I was able to get viewer to accept text input. It had the same
problem with forms until I switched (menu option) gadget rendering to "native".
 Viewer is almost usable as a browser now. It just seems to be missing the
"stop" button to keep it from sucking CPU if one image (say from an ad server)
is keeping a page load from completing and sucking up CPU.

In a nutshell:
Upsides-
* Nice HTML rendering. Much better than the previous milestones and 100898
* Nice looking (and theoretically changable - haven't experimented) interface
* You at least get part of a page while waiting for page loading
* Preferences dialog now exists
* Viewer is looking better and better. :)

Downsides-
* Sluggish. Lots of debug code?
* The nice looking interface doesn't work so well.
* Can't enter text into anything.

The test system is an AS200 4/233 with 96 megs of RAM. OS is RH 5.1 + various
updates. GTK version is 1.2.6. I expected some Mozilla instability since I'm
not running glibc 2.1. I'm not certain if that was related to my other
problems with M11. I've not had time to research these issues much farther.
