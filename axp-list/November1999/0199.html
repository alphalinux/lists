> Shane Sturrock wrote:
> >
> > > > an x86 gets to 1Ghz before an Alpha. My current integer benchmarks using
> > ^^^^^^^
> > Note this, it will come in useful later.

> Yes, lets do that.
>
> > > the Athlon would have a peak performance of 2GFLOPS at 500MHZ, Alpha
> >
> > Since I was comparing integer performance with x86 chips I don't see why
> > you should be bringing up fp performance.
>
> If it's integer performance, Athlons will do better against
> Alpha than they do with FP. Specint95_base on an Athlon 600 is 27.5.
> Specint95 on Alpha 21164 600 is 18.4. There is no 600MHZ 21264
> so I can't quote it's performance. The 466MHZ 21264 Specint95 is 24.6
> and the 667MHZ 21264 is 37.5. Note this is all done using compilers,
> no hand coded ASM allowed.

No hand coded ASM here, I'm just using a basic C program written
originally on my old 486/33 without any modifications for the Alpha, using
egcs the code is twice as fast as the same code running on a PII. It is
my understanding that the Athlon is faster than a PII but not by a factor
of two. With ccc the code performs even better, I suggest you try running
some actual real world programs you might be using for comparison. I
have yet to use an x86 system that didn't feel sluggish by comparison to
an Alpha. It may be that the benchmark codes are doing the Alpha a
disservice, not unheard of. As I said, if you really want performance
then you need to look at the architecture and code for it. Just writing
sloppy C and expecting the compiler to fix any problems is not the right
attitude.

> > can be expected to be faster again clock for clock, I heard 2-2.5x faster
> > than a '164.
>
> The Athlon is capable of nine operations per clock, 3 integer,
> 3 address, and 3 FP. The Specint95 indicates they did something
> right.

Possibly, but I still find it unlikely that the Athlon will be faster than
a '164 clock for clock, let alone a '264 based on my experience. I simply
don't believe benchmarks, the best benchmark is your own code.

> > will perform very well there maybe with a little massaging to fit the code
> > to the architecture a bit better.
>
> Must be highly unusual code. I suppose you're doing cryptography
> or some such if it's all integer based work.

Biological sequence analysis code. One version written in simple C (64
bit clean) which compiles and runs on anything, the other is written
specifically to the 64 bit architecture and needs true 64 bit longs to
work and that goes like mad.

> > pair) you would need 20 x86 boxes of similar clock speed. Looks to me

> > > Anyway, this a bit off topic so thats it for me on this topic.

Why? People are claiming Athlons are fantastically fast when what is
actually the case is that they are fantastically fast for x86 processors.
If you're going to badmouth Alpha you need to know that there are cases
which allow an Alpha to far outstrip an x86 processor. If this was just
about Athlons then I would agree with you.
