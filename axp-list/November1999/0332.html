Not sure if anyone else has noticed this problem, but tcpdump
under Linux/axp will not correctly read pcap captures gathered
on Linux/x86, FreeBSD, or Solaris (and probably others):
tcpdump: pcap_loop: bogus savefile header
If the captures are collected under Linux/axp, tcpdump will read
them back without complaint.

When I contacted Vern earlier this year, I mistakenly thought
tcpdump was the source of the problems. Breaking out the
debugger, it is apparent that the fault lay with libpcap.

I have attached a simple debugging session showing where the
error occurs; if anyone wants to help attack this, please let me
know. Better still, pass along patches if you've already fixed this.
Surely I am not the only person here that uses tcpdump ;-)
