RH6.0 on everything. I've compiled my own versions of
the kernels. I always use Becker's latest tulip.c.

> Hardware configuration of the FBSD and RH boxes is identical,
> all operating systems are "out of the box" without black majic
> optimizations.

I haven't sacrificed anything lately...

> Simple experiment using a single 100M dummy file created with
> dd and /dev/zero, transfers are done using scp:
>
> Server Client avg xfer rate(kb/sec)
> Solaris 2.6/sun4u RH 6.0/x86 70
> FreeBSD 3.3/x86 RH 6.0 43
> Solaris 2.6 FreeBSD 3.3 350
> RH 6.0 FreeBSD 3.3 475

> I would blame this on ssh, but the transfer rates are mirrored

>

This is on a local net??

Even on 10Mb Enet, I've never had that slow of
transfer, all the way back to when I ran coax.

ftp is a reasonable result to cross reference against.

> > This, whether in Intels or Alphas. It can easily be limited by
> > your HD speed if you're using older HW. The size of files I
> > muck with can easily exceed 10GB on my Alphas.
>
> > You may have a config problem on your Linux box.
> Unlikely, unless its a hardware problem with the interface;
> I've been tinkering with Linux since 1993, and have
> basic setup and configuration down pretty well ;)

Not necessarily your SW config. It could just be a
card problem or a switch/hub/wire problem. You know
if your connections are good if you don't get many errors
showing with "netstat -i".

For example, my XP1000 is:

Iface MTU Met RX-OK RX-ERR RX-DRP RX-OVR TX-OK TX-ERR TX-DRP TX-OVR Flg
eth0 1500 0 399087558 36 0 0 705149584 96 0 0 BRU
lo 8020 0 38371 0 0 0 38371 0 0 0 LRU

My PC164LX is:

Iface MTU Met RX-OK RX-ERR RX-DRP RX-OVR TX-OK TX-ERR TX-DRP TX-OVR Flg
eth0 1500 0 511161625 0 0 0 1287531594 0 0 0 BRU
lo 8020 0 12 0 0 0 12 0 0 0 LRU

Also watch for network errors with dmesg.

For reference, I sent a 1.4GB file from the XP1000 to /dev/null
on the PC164LX at 9.1MB/sec using my normal file transfer:

rsh xp1000 dd if=/the/file bs=64k > /dev/null

Same thing using my Athlon as client. From the Athlon to
the XP1000 is about 7MB/sec. The Athlon is currently running
an EIDE disk so it's a slower data source than the XP1000's SCSI
disk and the file won't fit in memory.
