geerten kuiper wrote:

> - aboot seems to get in a funny state if you pass the wrong parameters. It
> seems to go on to interactive mode, but everything I try after that will
> fail. Hit reset and try again.

That one got me too. A ? will list the contents of /etc/aboot.conf.
Booting the kernel takes the form n/filename [options] where n is the
partition number where the filename resides. This is from some documentation I
found at alphalinux.org.

>
> And some questions:
> - should I ever want to return to ARC, where can I get the firmware image ?

ftp.digital.com/pub/DEC/Alpha/firmware/

OpenVMS Hobbyist seems to run fine but needs DEC supported stuff. I use a
Qlogic 1040b scsi conroller, a null modem cable (no graphics card) plugged into
my PC, and a DEC NIC.
