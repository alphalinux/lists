>
> On Wed, 3 Nov 1999, Wes Bauske wrote:
>

> ^^^^^^^

Yes, lets do that.

> > the Athlon would have a peak performance of 2GFLOPS at 500MHZ, Alpha
>
> Since I was comparing integer performance with x86 chips I don't see why
> you should be bringing up fp performance.
>

If it's integer performance, Athlons will do better against
Alpha than they do with FP. Specint95_base on an Athlon 600 is 27.5.
Specint95 on Alpha 21164 600 is 18.4. There is no 600MHZ 21264
so I can't quote it's performance. The 466MHZ 21264 Specint95 is 24.6
and the 667MHZ 21264 is 37.5. Note this is all done using compilers,
no hand coded ASM allowed.

> The point is that at the present moment in time a basic dual pipeline
> 21164A processor at 600 Mhz is more than twice as quick as a 400Mhz PII
> running purely integer code compiled with egcs on both machines. Using
> ccc makes the Alpha execute the same code around 30% quicker again. The
> '264 has four integer pipelines and features to make it better able to use
> those to full effect (out of order execution for instance) and as such it
> can be expected to be faster again clock for clock, I heard 2-2.5x faster
> than a '164.

The Athlon is capable of nine operations per clock, 3 integer,
3 address, and 3 FP. The Specint95 indicates they did something
right.

>
> For the work I am involved in the extra benefits of the long word make a
> '164 10x quicker than an x86 processor clock for clock which makes them
> very economical. I haven't tried the code on a '264 but I expect the code
> will perform very well there maybe with a little massaging to fit the code
> to the architecture a bit better.
>

Must be highly unusual code. I suppose you're doing cryptography
or some such if it's all integer based work.

> What is different about the way we code here is that pour over the data
> sheets on the architecture and write our code accordingly, this behaviour
> has gained us a performance benefit of a factor of three from when we
> first moved our code from SGI R10000 to Alpha. We are now working on an
> MPI version of the code and running test jobs on a pair of '164s we have
> sitting here and the results are very encouraging. To get the same
> performance as these two '164 boxes (that cost us around $4000US for the
> pair) you would need 20 x86 boxes of similar clock speed. Looks to me
> like the Alpha is very economical, it all depends on what you are doing
> with them.
>

99.9% of the people using computers would find the Athlon
a much better buy for integer work. If your code is not
that way, first I'd ask myself why, and second if there is
absolutely no way to get Athlon to do the job, stick to
Alphas.

> > Anyway, this a bit off topic so thats it for me on this topic.
>
> Seems to be perfectly on topic to me.

Discussing Athlons in an Alpha forum is off topic.
