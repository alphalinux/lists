On Thu, 18 Nov 1999, Mike Foley wrote:

>
> As I understood it, the -taso options call
> functions only available in Tru64. It's part
> of the OS.

        If you're talking about allocating memory chunks in the lower
31-bits of the address space, Linux does support this. That's what the EM86
options do. It just creates a separate Personality in the kernel, and
programs like em86 change the current (i.e., the program's) personality to
that special one.

[prophet: alpha]$ grep -i personality /usr/include/asm/*
/usr/include/asm/a.out.h: ((current->personality==PER_LINUX_32BIT) ? (0x80000000) : (0x00120000000UL))
...

        And seeing as it's in a.out.h, the DU-style support for it appears
to be there. Has to be, since I'm running DU-netscape on my machine (which
was supposedly compiled that way).
