Adam Last wrote:
>
> This is WRONG. It doesn't work. Please somebody organise proper boot disks.

Assuming you have an Alpha already running, you can
use whatever kernel you want on the boot disk. It's
just an ext2fs. Just build what you want and copy it
to the floppy after removing the old one. I always
build generic kernels now and I have 4 different Alpha
types the kernels work on. (PC164/PC164LX/XP1000/DS10)

>
> I think one of the problems is some of the systems don't recognise the file
> system being used. I don't have a linux box running at the moment so I
> can't even see what's on the Red Hat boot disk.
>

You can't use a RH5.2 kernel for RH6.0 due to the ramdisk
being a romfs now instead of an ext2fs. If you went out
hunting for a model dependent kernel, you most likely
found a RH5.2 based one and that's why it didn't work for
you.
