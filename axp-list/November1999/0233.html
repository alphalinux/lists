>
> To load the boot disk from the SRM console, i use the following command :
> >>> boot dva0

This would work only if you set correctly beforehand various variables
in SRM. From results it is pretty clear that you did not so boot
process used defaults and ended with starting up 'aboot'.

Try something like that instead:

boot dva0 -file "generic.gz" -flags "root=/dev/fd0 load_ramdisk=1"

The above assumes, of course, that you indeed have a file like
"generic.gz" at the top of your floppy and not, for example,
"images/generic.gz" or "kernels/generic.gz", or something of that
sort.

>
> The 3 disks (boot (generic.img) , ramdisk (ramdisk.img) and kernel
> (generic.gz)) have been made with rawrite under a DOS system.

I am afraid that this tells me nothing about their contents. This
"Alpha1000A Linux Installation Help" may be describing some boot disk
and you got yours from another source and it is not exactly the same.
Some writeup with that floppy should describe proper commands to boot
from it. If you have any other Linux machine around, a hardware
platform immaterial, mount this boot floppy on it and have a peek.
