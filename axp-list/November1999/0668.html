hey all, me again. the slow-moving saga of me vs. my as200's sound config
continues. the problem- i cannot play mp3s- mpg123 says it can't reset
audio every time i try to play a file with it. i've been assuming it was
a problem of incorrect irq/dma assignment in the card config, so after
going through the suggested combinations (e.g. 7,3 9,0 9,3) without
success, i just started trying all of them- 7,0 7,1 7,3 9,0 9,1 and so on.
some of the resulting kernels wouldn't boot, but all the ones that have
produced the same effect- mpg123 doesn't work. my /dev/sndstat currently
says this:
[root@grubby lene_marlin]# cat /dev/sndstat
Sound Driver:3.5.4-960630 (Fri Nov 19 00:08:16 EST 1999 root,
Linux grubby 2.0.35 #1 Sun Oct 17 22:08:31 EDT 1999 alpha unknown)
Kernel: Linux grubby 2.0.35 #1 Fri Nov 19 00:09:42 EST 1999 alpha
Config options: 0

Installed drivers:
Type 10: MS Sound System
Type 24: MS Sound System (AXP)
Type 27: Compaq Deskpro XL

Card config:
MS Sound System at 0x530 irq 10 drq 1

Audio devices:
0: MSS audio codec (CS4231A)

Synth devices:

Midi devices: NOT ENABLED IN CONFIG

Timers:
0: System clock
1: MSS audio codec (CS4231A)

Mixers:
0: MSS audio codec (CS4231A)
[root@grubby lene_marlin]#

when i cat /dev/audio, i get a stream of characters, like it's streaming
to stdout.
when i cat /dev/audio1, i get device not configured (not sure if that's
right or not).
when i cat /dev/dsp, i get nothing.
and when i cat /dev/dsp1, i get device not configured (again not sure if
that's right).

but then tonight i discover something, in looking through the man pages
for mpg123. i called it with -a /dev/audio and i got sound- no 'can't
reset audio' or 'no supported rate found!', though the sound quality was
very rough. apparently /dev/dsp is the default device for my system, and
i have a feeling that it is supposed to be that way. but is this hope for
me? is it indeed an incorrect irq/dma config that's preventing /dev/dsp
from working? any ideas? does output using /dev/audio result in poorer
sound quality than with /dev/dsp? and accordingly, if i call it with both
-a /dev/audio and -w, will i get a worse quality .wav file?
it's all still a mystery to me. can anyone shed any light?

as always, i thank you all very much for your help.
