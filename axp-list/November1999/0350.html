Your email address was registered as downloading the beta 2 version
of Compaq C for Linux Alpha, ccc-6.2.0-8.alpha.rpm. If this address
was registered in error, or you do not want to be notified of problems
or new releases, please reply to this message with "remove" as the first
line in the body of the message.

While the beta 2 package installs properly on "vanilla" Red Hat 5.2,
6.0, and SuSE 6.1 distributions, a problem was reported where the
list of include file and library directory search options stored
in the comp.config file was corrupted when installed on a Red Hat
6.0 system to which updates had been applied. Similar problems
could easily occur on other distributions. Visible symptoms
of the problem would be failure to find include files or libraries,
but depending on the compilation you might not notice a problem.

To check for a bad comp.config file, run:

egrep '[^ ]-(SysIncDir|L/)'
/usr/lib/compaq/ccc-6.2/alpha-linux/bin/comp.config

The package on the download page will be updated to ccc-6.2.0-9.alpha.rpm,
but below is a simple script to fix the problem if you have already
installed the 6.2.0-8 package. Run uudecode on the following to
create a shell script named patch-ccc-8-to-9.sh in the current
directory, and then run that shell script. The script patches
the create-comp-config.sh script, and then runs it to generate
a new comp.config file, so it must be run as root.

 -Compaq C/C++ Team
