>
> >>> Ryan Kirkpatrick said:
> >
> > I am having an odd problem with an IBM 2GB Fast SCSI disk on my
> > XL366 alpha. This disk is being used as a system disk and is connected to
> > the onboard NCR53C810 controller. The system is running Debian 2.1 and
> > kernel 2.2.7.
....
> > Only, when the system goes to get partition maps for the disks and then
> > later load init off of this disk (since it is the system disk), I start
> > getting SCSI timeouts on this disk for the onboard scsi chain.
>
> This *may* be the same problem others have seen WRT compiling recent kernels
> with EGCS 1.1.2 or later. Try patching the following way, rebuild, and see

        After being completely too busy for too many days (college can be
such fun at times!), I finally got around to applying the patch and
recompiling a 2.2.12 kernel. Booted the system up with the new kernel and
the exact same problem occured. The 2GB Fast SCSI IBM disk was setup for
Fast 10MHz SCSI-II and then promptly timed out on partition check.
        So, what is the next idea? Is there some way to force a disk to
not use fast scsi-II and instead use async (without doing so for all other
disks)? Not great, but would at least allow me to run a 2.2 kernel, and
get my CD-R working again. I.e an acceptable solution for the mid-term
(several moths).
        I guess I could consider replacing that disk (I can find another
home for the current one). It is slow and small, but I like it because it
runs cool and buried deep in the XLT366 case (above the 3.5" floppy drive,
under the 5.25" drive bays) it doesn't get much cooling. That, and trying
to locate a fast (no need for ultra on the internal controller) 2-4G
narrow disk these days appears to be nearly impossible. :(
        Thanks for any help anyone can give me!
