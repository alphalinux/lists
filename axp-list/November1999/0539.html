At 09:01 PM 3/11/99 -0900, Michael Tibor wrote:
>Ugh... I hate to reply to my own message, but it appears my system is
>not an SX164, it's a plain PC164. The machine was sold to me as an SX,
>and I didn't think to verify that.
>
>I was able to switch to SRM by getting the files from the correct
>directory on Gatekeeper, copying fwupdate.exe and the .rom file to a
>fat-formatted floppy, and selecting "Update Firmware" in the ARC
>console.
>
Well, that sounded easy enough, so I thought "Why not give it a try ?"
(Mine's a PC164 too.) It was easy.

Discovering how to boot from an aboot-floppy, mounting / from harddisk, was
the next step. Easy too.

Converting the disklabel to BSD-format, while retaining the existing
partitions. I got myself into trouble on this one (at one time I was
running from a disk that had no label at all, and fdisk suddenly decided on
a different geometry....) but pulled it off.

Installed aboot to boot from the harddisk. OK.

Some issues I ran into:
- aboot seems to get in a funny state if you pass the wrong parameters. It
seems to go on to interactive mode, but everything I try after that will
fail. Hit reset and try again.
- SRM (and aboot) enumerate the PCI-bus in reverse order (compared to ARC &
kernel) which results in a switch of the harddisks. Once I found out what
was going on this wasn't a problem.
- In BSD-style partition tables use type 8 for ext2, not 0x83. Your kernel
won't mind, but aboot will.

And some questions:
- should I ever want to return to ARC, where can I get the firmware image ?
PC164 will not hold both, so setting OS_TYPE and restarting will not be
sufficient. OTOH, SRM seems to have a command to start ARC from floppy. Any
experiences ?
- next step would be to try and run TruDU and/or OpenVMS. Anyone been
there, done that ? Hardware is as follows:
Aspen Durango mainboard (= PC164)
sym810a + sym875 SCSI adapters
IBM discs (2GB fast + 4.3GB ultra)
S3 Trio64V+ graphics card
NEC 3x CDROM
3Com 509B 10Mb ISA + 905B 100Mb PCI network.
- and does anyone know when noncommercial TruDU will be available for the
rest of the world (which is the part I belong to ?)
