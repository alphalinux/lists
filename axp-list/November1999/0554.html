Paul Jakma wrote:

> On Fri, 12 Nov 1999, Chance Reschke wrote:
>
> 2) We'd like to use 4GB of physical RAM on one of these machines, but the
> current Linux kernel only supports 2GB. The i386 BIGMEM patch doesn't
> appear to address our problem either. Is anyone working on native support
> for big memory on Alphas?
>
> the problem is with the way pci/isa address space is mapped into
> physical address space. Currently linux maps the first 2gb of bus
> addr space to 2-4gb physical space.
>
> the correct solution is to implement scatter-gather bus -> phys
> translation which most alpha's support. (see other posts). But this
> involves a lot of changes to the driver interface.
>

This is true. However, we might be able to get away (until the "correct
solution" is in place) with just adapting to the zoned allocator that mingo is
working on. This is a fairly simple solution to get us moving into the
direction we all want to go, that is, true full support for our entire address
space. I'm already looking into the implications this mod will have in terms
of NUMA (it basically doesn't do anything NUMA-ish yet), but overall it looks
pretty doable.
