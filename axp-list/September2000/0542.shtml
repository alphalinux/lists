Just in case anyone is still interested (Kurt, maybe?)...

I found out what was preventing my auto.homes nis map from being processed
by the automounter: The man page for auto.master says that the second field
in an auto.master entry should be of the form maptype:mapname, so that for
my NIS auto.homes map it should be

/homes yp:auto.homes

If you put it in in this form, you get an entry in the syslog

Sep 27 13:52:36 racoon automount[447]: starting automounter version 3.1.4,
path
= /homes, maptype = yp:auto.homes, mapname = none

i.e. the maptype and mapname are not set correctly. If however you put the
entry in like this

/homes yp auto.homes

it works!!

I guess the man page needs a bit of re-editing...
