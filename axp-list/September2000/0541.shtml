On Tue, Sep 26, 2000 at 01:25:47PM -0400, Larry Sendlosky wrote:
> MAGIC is an unsigned int (32bits) with value 0xc00fc00f. The compiler
> should generate 0x00000000c00fc00f for the register value. This is
> "sign extending" an unsigned int (there is no sign bit or it's
> "inferred" to be zero).

Incorrect. The calling conventions specify that bit 31 of a 32-bit
value should be replicated up. We choose this form because this is
what the 32-bit arithmetic and memory load instructions do, so oft
times we need perform no extra work.

Basicly, one should choose *some* canonical form for 32-bit data in
your 64-bit registers. Sign extending happens to be easiest on Alpha.
Failure to do so results in catastrophic failure in the face of missing
or incorrect prototypes -- all too common in source code even today.

Anyway, the gcc problem was brought to my attention by Steve Hobbes
from the Compaq compiler group back in January. It's been fixed since,
though I guess that's well after 2.95 was released.
