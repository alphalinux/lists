OK.... The mini-mini (rushed) howto ;-)

get the Glide3-64bit branch out of CVS from the the cvs tree at sourceforge.

export CVSROOT=':pserver:anonymous@cvs.glide.sourceforge.net:/cvsroot/glide'

cvs login (no password)

cvs -z3 checkout -rGlide3-64bit Glide3

That will get you the right branch of Glide.

Read the INSTALL file for details on building. need help? email me.

You then need to get the tdfx-2-1-branch from the DRI project's CVS tree.
If you still have the source tree from XF-4.01 lying around you should be
able to update to that branch.

The DRI tree...

export CVSROOT=':pserver:anonymous@cvs.dri.sourceforge.net:/cvsroot/dri'

cvs login

cvs -z3 checkout -rtdfx-2-1-branch xc

and that will get you the proper DRI enable XFree branch.
You should be able to 'update' your exising Xfree source tree the same way.

Finally grab a 2.4 series kernel, for example I'm running 2..4.0-test8 on my
LX, and compiling in the DRI module for tdfx.

and remember to configure your XF86Config file to load the dri and glx
modules. There's a user's guide at the DRI website that details this.

http://dri.sourceforge.net/DRIuserguide.html

I didn't say it was painless :-)
