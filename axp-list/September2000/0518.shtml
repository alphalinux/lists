I have ~15 PWS 500a workstations running RH5.2 that can all rsh/rlogin back
and forth in root account without prompting for a password. I have
configured a new PWS 500a workstation running RH6.2 so that I can
rsh/rlogin to the older (RH5.2) workstations via the root account without
being prompted for a password. For some reason I cannot rsh/rlogin from
the older(RH5.2) nodes to the newer(RH6.2) node. I get the following error
"rcmd: newernode.sub.net: Connection reset by peer"
I have checked the hosts, hosts.allow, hosts.deny, hosts.equiv in the /etc
directory and also the /root/.rhosts files on both nodes and they all look
OK! Does anyone have any idea what I am missing here?
