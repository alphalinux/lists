It took a little fiddling but here is how it works.
1) RH6.2 out of the box does not know how to load up OSF partitioned disks
   as part of the autoconfigure sequence. I have a patch to fix
   genhd.c(really minor) and will post it when I figure out the best place
   to have it included in the kernel. This saves having to have an initrd
   image to start up the raid disks. I expect that a pristine kernel will
   have the same problems but I have not checked.

2) aboot does not know about file type 'fd' so I configured partition 8(h)
   to overlay my root raid partition and labeled it as ext2.

Now I have a complete booting root raid system.

P.S. This is only raid-1.
