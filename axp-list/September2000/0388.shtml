Peter Petrakis wrote:
>
> If you're running redhat-6.2 you already have the required libraries ,
> if not go and get the Tru64-compat rpm from your favorite mirror.

Tru64-compat rpm is really distribution version (or even distribution
maker) independent so you can install that does not matter what you
happen to have already on your box. It may require recent enough
version of rpm to install/unpack but this is another issue. OTOH
Netscape rpms, also from Tru64, which also does not really depends on RH
6.2, contain binaries which are using system calls which were NOT
emulated in 2.0 kernels so something from 2.2 series is a requirement.

As Netscape indirectly implies that a machine is exposed to Internet
then your kernel has to be at least 2.2.16 for security reasons.
An upatched 2.2.16 does not run on Alpha so if you use "official"
kernels you may start from 2.2.17 instead. :-) Also use Netscape
4.75 from RH 6.2 updates; security reasons once again.

> Next
> get acrobat reader for Digital Unix aka Tru64. Version 3.0 works fine.

For this I do not know if 2.0 emulations are good enough. I use
xpdf anyway. :-) As far as Netscape goes the difference is that PDF
documents open up in a separate window (after a minor Netscape
configuration adjustment, of course) and this is not the case with
plugin.
