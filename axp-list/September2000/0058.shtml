On Mon, Sep 04, 2000 at 02:13:58AM -0500, Jiann-Ming Su wrote:
> I've been having strange problems syncing time on my SX164 computer.
> I use ntpdate to sync time. The time appears to be set properly when
> check it with 'date'. My TZ variable is set to America/New_York, which
> is correct. But, after a few hours, the time drifts ahead by about
> 8 hours. I've always had problems with keeping proper time with
> this computer. It is a home computer so I don't have xntpd running
> all the time. However, it makes no sense why the time would just
> drift off. Any clues on how to set the time properly? Thanks for
> any help.

        i've seen a couple of Alphas have this problem, both under OSF (where
i used to work) and Linux (although not nearly as badly). i think it's just a
problem with the clock hardware on the motherboard.
        Since you are not connected full time to the Internet, might i suggest
you get another computer that can keep more accurate time (maybe an Intel
box?) and set up the Alpha to use it as an xntpd server. Also, if you keep
xntpd running, even without a server, i believe it should help keep a check on
the drift.
