Hi.
I've found that compiling GMT3.3.5 on a Compaq Alphaserver succeeds but many
of the programs bomb with seg faults when I try to use them. When compiled
with gcc there isn't any problem. In the past I've found that this occurs
due to the stringency of the Compaq (nee DEC) compilers which are quite good
but intolerant of weak typing. Does anyone have any suggestions for
debugging this problem? Are there some compiler options I should be using
in particular? Anyone interested in helping solve this puzzle? I'm afraid
that it permeates the code but I'm hoping that some kind of switch will
overcome this and let me take advantage of the full precision of this
machine for GMT.
