Hans J. Johnson wrote:
>
> For some reason the maximum shared memory is set too small. A quick and
> easy fix is as follows:
>
> Enter the following command as root
>
> #This is needed to increase the size of the shared memory
> #Set by Hans J. Johnson this fixes problem shmget() used by cdrecord
> echo 16777216 > /proc/sys/kernel/shmmax

As far as 'cdrecord' goes setting fifo size to 4000k, instead of a
default 4M, works just fine without messing with a size of a shared
memory segment. The later is kind of inconvenient if you still run
some kernel from a 2.0 series.

Edit your '/etc/cdrecord.conf' to accomplish that feat, for example, but
there are other ways to do it. 'man cdrecord' will provide details.
