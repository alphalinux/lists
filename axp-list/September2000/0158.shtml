A 9/7/2000 12:03 PM, Vanco, Donald em VANCOD@PIOS.com escreveu:

> I'm confused - I thought the purpose of "make oldconfig" was to dump
> the characteristics of your currently running kernel into a .config file.
> Why do you need to have a "known good" .config in place when it will simply
> be overwritten? Am I missing something?

make oldconfig will use the settings of a ../linux/.config, prompt you to
answer to any changes in options that were included in the kernel since that
.config was created, and create a new ../linux/.config file.

Take into consideration that if you issue a make mrproper, then you will
lose the .config file in ../linux/. Make a backup of the file before issuing
such command.
