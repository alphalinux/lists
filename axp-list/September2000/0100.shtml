Alvin Starr wrote:
> The Alpha and othre RISC CPU's trade off instruction complexity and code
> size against instruction speed. In the xx064 the cpu would execute an
> instruction on each clock cycle so that MHZ == MIPS. In the xx164 the
> number of instruction was pushed to 2 and with the latest incarnation the
> xx264 MHZ == 4MIPS. So an 600MHZ Alpha will actually run at 2400MIPS.

That's misleading. For "typical" apps nowadays the memory bus is the
computing bottleneck. A CPU can stall for hundreds of clock cycles
during a cache miss. This phenomenon is largely responsible for
narrowing the performance gap between x86 and Alpha-- both are crippled
by memory speed.

> An issue that is often overlooked is the 32 vs 64 bit issues. With 32 bits
> the ammount of memory that a system can have access to is limited to
> 4Gbytes and in a number of OS's that number is more like 2GB. Under
> linux(without kernel hacks) the limit is more like 1.5GB per process. A 64
> bit address space gives you a MUCH bigger area to play in. Another point
> often overlooked is encyption where the fundimantal processes are based on
> multiplying large numbers(1024 bit) a 64 bit multiply in a 32 bit system
> requires 10 times as many instructions as a 64 bit multiply in a 64 bit
> system.

Actually, recent Linux kernels have this "hack" in place and allow for
up to 3GB of user space on x86.
