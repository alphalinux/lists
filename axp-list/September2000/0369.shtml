When I installed Gnome recently I had to remove all of the old Gnome
packages first. I had to actually go into /usr/share and delete by hand a
couple of Gnome files that kept getting left behind even when I would
remove them all using rpm. My problem wasn't that Helix wouldn't start.
My problem was that both gnome and helix code kept running together. And
they didn't like each other enough. (applets wouldn't all work, terminal
windows would seg if you tried to use them. . .etc.).
After I removed all of the gnome stuff and then installed the Helix
packages in 3 series (1,2, &3 not that this had any effect). It seems to
work fine now.
James
