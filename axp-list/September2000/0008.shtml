On Fri, 1 Sep 2000, Peter Petrakis wrote:

> looks like the window manager will SEGV whenever a KDE application
> starts. I've down a remote
> debugging session and also have recompiled kdebase with debugging
> symbols. here's what i've found.

Netwm.cpp writes over allocated memory on 64-bit platforms and causes all
sorts of weird problems. I have solved this for Tru64 Unix.

Here's the source for my modified netwm.cpp:

   ftp://finwds01.tu-graz.ac.at/pub/kde2/develop/netwm.cpp

Beware, though, that the 64-bit compatible sections have an #ifdef __osf__
around. You need to change this to:

#if defined(__osf__) || defined(__LINUXAXP__)

or something similar.

> so it looks like the culprit is qt.

No. It's netwm.cpp!
