I suggest that you peruse a copy of ieee754-1985, the ieee floating
point specification for a detailed description of why some of these
things can and do happen. You're working on the edge cases of IEEE,
and those are fairly tricky. A quick rule-of-thumb is that it is fair
to assume that any floating point value can vary by an amount equal to
the value of the least significant bit for any operation.

                                /-- 0x7fffffff fffffffe
        0x7fffffff ffffffff
                                \-- 0x80000000 00000000

                                /-- 0x3fefffff ffffffff
        0x3ff00000 00000000
                                \-- 0x3ff00000 00000001

                Input Input + 0.0e0

There are other things which affect the result, such as an odd-even
when to add, and when not to.
Another thing you can look at is paranoia.c (from someplace on the net)
which I think has commentary in the code about what's happening.
Yeah, sometimes it's counter-intuitive.
