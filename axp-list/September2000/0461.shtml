> On Thu, Sep 21, 2000 at 12:23:34PM +0100, David Rhodes wrote:
> > do you know how to reference different kernels from the SRM 
> console?  At the
> > moment all I do to boot is issue 'boot dkc0 -fl 0'.  Is 
> there a flag option
> > I could use to specify an alternate kernel, perhaps?
> 
> Edit
> /etc/aboot.conf
> each line is one configuration. Simply edit the name of the 
> kernel (and
> add/modify some flags) to your liking. Each line has to have a unique
> number. So it could look like this:
> 
> 0:6/vmlinuz-2.2.17 root=/dev/sda4
> 1:6/vmlinuz-2.2.16 root=/dev/sda4
> 
> With -fl 0 I would boot 2.2.17, with -fl 1 I would get 
> 2.2.16. I am not
> sure what the maximum number of entries is though.
