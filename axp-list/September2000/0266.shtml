Steve Rogers replied:

> > We did some recent testing here, between a 1 GHz Athlon "Thunderbird"
> > box and a 500 MHz 21264. The Athlon had 256 KB of full-speed L2 to the
> > 4 MB of 2/5-speed L2 cache on the Alpha; on the other hand the Athlon
> > had PC100 memory (theoretical bandwidth 800 MB/s) as opposed to our
> > Alpha's 2656 MB/s theoretical memory bandwidth.
>
> PC133 would have made for a better comparison. Hopwever if you used a board
> with the AMD chipset you would have problems.

Yes, I'd like to have tested a PC133 (1064 MB/s) box; in fact, we have one
on order but it hasn't arrived yet. Still, the code in question (and the
codes we use that Alpha for) are typically memory intensive, and I expect
would still run faster on our Alpha.

> > Athlon motherboards with PC2600 (DDR 166) memory should close the
> > bandwidth gap, but those won't be available for a while yet. In the
> > meantime, we're very happy with our 21264, and we've been able to do
> > real work on it for quite some time. :)
>
> There are several DDR boards that will ship in October. One board maker was
> claiming September (iWill).

Hopefully yes. My understanding is that those will be PC2100 (DDR 133)
boards with the PC2600 boards showing up next year, but even so that'd
be a *big* improvement. And yes, hopefully the Alpha manufacturers will
also be able to take advantage of this technology. As someone (Wes?)
pointed out, the Alpha we're using gets 2656 MB/s per channel by having
4-way interleaved memory running at 83 MHz. If the board could take
PC2600 memory instead, the theoretical bandwidth works out to 10624 MB/s!

And, we're not waiting for those boards. We've had our current Alpha
server for something like a year, and done good work on it.

This isn't to harsh on the Athlon; the motherboard we tested it on is
not as capable, frankly, and that's going to have an impact on how
well the processor can shine. After all, we're certainly finding ways
and places to put Athlons to work, as well as Alphas. :)
  
  -- Steve Bonneville
