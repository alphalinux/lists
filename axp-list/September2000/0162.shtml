Vanco, Donald wrote:
>
> I'm confused - I thought the purpose of "make oldconfig" was to dump
> the characteristics of your currently running kernel into a .config file.

No. 'make oldconfig' is like 'make config' but it is using all "known"
options as defaults without asking any questions. So, if you want to
change somethning in an existing configuration just delete all lines
which refer to what you intend to change, does not matter if lines
start with a '#' sign or not, and run 'make oldconfig'. What you are
trying to change will appear as "NEW".

> Why do you need to have a "known good" .config in place when it will simply
> be overwritten? Am I missing something?

When you are updating a kernel then options may be not the same.
Running 'make oldconfig' with .config from your old kernel ensures that
new .config will be updated to the current set of options.
