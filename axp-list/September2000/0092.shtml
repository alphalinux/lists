To clarify a point that nobody has made directly. Clock speed is not the
same as instructions per second. I have not looked at the Athlon so I
cannot comment on specifics and I have not looked a the insturction timing
in the Intel products for a while but I think my general points still
will hold true.

the X86 and all its clones are complex instruction set machines and this
implys that the CPU wll take several cycles to decode an instruction and
in a number of cases the instructions will also take a number of cycles to
run. In the days of the 386(last itme I looked carefully at instruction
timing) an instruction would take between 4 and 100 cycles to complete.
Most of the real instructions would take somewhere between 10-20 cycles to
complete. With newer processors and lots of transistors there are a number
of techniques to improve performance but still the number of clock cycles
will be greater then the number of instructions so a 1Ghz clock will not
equal a 1000MIP CPU.

The Alpha and othre RISC CPU's trade off instruction complexity and code
size against instruction speed. In the xx064 the cpu would execute an
instruction on each clock cycle so that MHZ == MIPS. In the xx164 the
number of instruction was pushed to 2 and with the latest incarnation the
xx264 MHZ == 4MIPS. So an 600MHZ Alpha will actually run at 2400MIPS.

An issue that is often overlooked is the 32 vs 64 bit issues. With 32 bits
the ammount of memory that a system can have access to is limited to
4Gbytes and in a number of OS's that number is more like 2GB. Under
linux(without kernel hacks) the limit is more like 1.5GB per process. A 64
bit address space gives you a MUCH bigger area to play in. Another point
often overlooked is encyption where the fundimantal processes are based on
multiplying large numbers(1024 bit) a 64 bit multiply in a 32 bit system
requires 10 times as many instructions as a 64 bit multiply in a 64 bit
system.

Of course I have not addressed the issues around cache size and memory
speed and these go a long way to set a real systems performance beyond the
raw MIPS or MHz of a CPU.
