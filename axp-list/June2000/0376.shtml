Greetings!

I've got a digital Server 3300R on which I just installed Red Hat 6.2. I've
figured out enough about SRM and aboot to get it going. The stuff in
/etc/aboot.com is simple:

0:1/vmlinux-2.2.14-6.0 root=/dev/sda5

So, from SRM, I just type:

>>>boot dka0 -fl 0

...and it goes!

PROBLEM:
The kernel starts loading, the partitions are fscked and it starts setting
up the hardware. When it gets to "Setting up eth0" it just stops there 4
times out of 5. Every once in a while it will continue and boot normally.
I've gotten it to successfully boot only twice since last night.

It has the DE500 NIC that came standard with it. I remember reading on the
list over the past few months about very spotty behaior by DEC NICs.

Can anyone shine any light on this??
