I am running 6.1 on a 533 MHz SX, 128MB ram, and Elsa Gloria Synergy
(8mb) video card. I had exactly the same problem with KDE, ultimately
removing the Gnome entry as below, but -- get this -- keeping all the
Gnome stuff! Now it still locks up periodically and then I usually can
log in from the HP9000 next to it and kill the session on the Alpha.

I suspect that KDE is not 64 bit clean or was not carefully debugged
for Alpha because it's rock solid on Pentiums (also RH 6.1). I also
suspect that RedHat's installation program (which sure tries to FORCE
the unwitting user into Gnome) may well be at fault. In fact, the
install is usually the single buggiest thing Red Hat has supplied
over the years...
