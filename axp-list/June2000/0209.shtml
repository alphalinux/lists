ok I found a way to do dynamic bootp

using the stock RH6.2 dhcpd.

it's not explained how to do this in the man pages, so I figured I'd post
it to the AXP list in case anyone else is doing similar things (booting
kernels w/ bootp in SRM).

this is an example of the /etc/dhcpd.conf file:
#DHCPD.CONF
#Manages the internal subnet
#K. Ludwig 6/14/00
subnet 192.168.0.0 netmask 255.255.255.0 {
  range dynamic-bootp 192.168.0.10 192.168.0.250;
  default-lease-time 6000;
  max-lease-time 7200;
  option subnet-mask 255.255.255.0;
  option routers 192.168.0.1;
  option domain-name-servers 192.233.60.2;
  option domain-name "isobutane.net";
  option broadcast-address 192.168.0.255;
  allow bootp;
  allow booting;
  filename "/tftpboot/bootp-2.2.14up2k";
  server-name "192.168.0.1";
# host tester1 {
# hardware ethernet 00:08:C7:CB:D4:7F;
# fixed-address 192.168.0.5;
#
# }
}
   subnet 192.233.60.0 netmask 255.255.255.0 {
  deny unknown-clients;
}
   
 
The machine that I am running this from is dual homed (so I can pull the
shared CD-Rom from the NFS server on the 'real' network.

it's a bit of a pain, but it works and keeps a lot of traffic off the main
network.

oh also..... if ur using NFS roots w/ the RH installer, you need to 'hack'
the installer so that it doesn't go messing w/ the IP addy info in the
kernel when you use kickstart disks/files. Otherwise, it drops the NFS root
info and just sits there dumbfounded.

Course, if you don't use kickstart, the NFS root stuff runs fine.
