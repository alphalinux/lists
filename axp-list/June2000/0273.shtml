  OK. The problem is the following.

  When running 6.0 I had the well-known problem with the DU binaries (Netscape/Maple)
  
  Warning: Cannot convert string
  "-*-Menu-Medium-R-Normal--*-120-*-*-P-*-ISO8859-1" to type FontStruct

  I managed to fix that problem by following Michal Jaegermann's advice in the archives
  and editing fonts.dir/fonts.scale in /usr/share/fonts/default/Type1/fonts.dir to include
  n019003l.pfb -urw-menu-medium-r-normal--0-0-0-0-p-0-iso8859-1
  and altering the font count at the beginning of the file.

  Now, after the upgrade, the same problems appear. So I go back to the directory
  /usr/share/fonts/default/Type1/
  and sure enough the upgrade has replaced my fonts.dir with a file with
  0

  So I copy fonts.dir.rpmnew to fonts.dir, add the line
  n019003l.pfb -urw-menu-medium-r-normal--0-0-0-0-p-0-iso8859-1
  and restart xfs. Now when I start Maple I get the messages

Warning: Cannot convert string "-*-Menu-Medium-R-Normal--*-120-*-*-P-*-ISO8859-1" to type FontStruct
Warning: Cannot convert string "-adobe-helvetica-medium-r-normal--12-*" to type FontStruct
Warning: Cannot convert string "-adobe-helvetica-bold-r-normal--12-*" to type FontStruct

  I go back to /usr/share/fonts/default/Type1/ and, sure enough, my fonts.dir file has
  been completely munged and replaced with a file containing only 0

  Any help gratefully received.
