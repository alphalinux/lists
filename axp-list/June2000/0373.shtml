On Thu, 29 Jun 2000, Speedracer wrote:

> My question is do I have to wipe and reload 6.2? or is there an easier way
> to go back to regular old fashioned crypt?

 You may mix password types. You may copy old passwords as is and only
modify /etc/pam.d/passwd (and files for other services that may ask for
password change, e.g. /etc/pam.d/login) so that it contains a line similar
to the following:

password required /lib/security/pam_pwdb.so md5 shadow use_authtok nullok

instead of the default one. This way new passwords will be generated
using md5 while old ones will still work.
