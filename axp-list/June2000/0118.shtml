Hiya list...
trying something somewhat odd here, but I'm trying to get it to work

I want to completely boot a kickstarted installation off a network

I'm using a DP264 to do this with w/ a DE500 based NIC in it... on RH6.2

kernel configs:
kernel 2.2.14 compiled w/ DE500 support, kernel IP auto-config (so it can
get an IP address) and NFS root support, platform specific to DP264

the commands I am passing it are this:

boot ewa0 -fl "root=/dev/nfs nfsroot=<IPADDY>:<PATH> ip=bootp
ks=nfs:<IPADDY>:<PATH>"

the kernel boots fine from bootp/tftp, the kernel gets the same IP address
from the bootp server and root mounts the nfs share fine.

When the install starts, a message comes up saying 'determining IP address'
and the systems stops. there is no kernel panic, lock, or halt.

upon checking tty4, I get a message saying the NFS server has timed out,
still trying.

note.... this install WORKS w/o using the kickstart. (if I go through and
use all the menus by hand, heck even X works right).

I have tried this also on kernels 2.2.15 and 2.2.16

anyone have any ideas?

I have tried setting the IP address in ks.cfg to static, and specifying it,
dhcp, and bootp. Whatever way I set it, it still goes to the determining IP
address screen.

I have also tried setting the install path to cdrom and the same nfs share
as before, to no result.

if ANYONE can help, please do so.
