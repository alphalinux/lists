Larry Snyder wrote:
>
> I'm speaking from a perspective of a 2.2.5 box and xcdroast 0.96e.
>
> You should be able to see what the kernel believes is shmmax by:
> cat /proc/sys/kernel/shmmax
>
> I did a quick hack to the xcdroast source to add something like
> '-fs=3072k' where it calls cdrecord, to set the fifo size below the
> 4M max. This is perhaps the most inelegant way to fix it, but I've
> never had a problem with it.
> -ls-
>

You can also reset shmmax thru the same interface.
Just "echo 8388608 > /proc/sys/kernel/shmmax" for
8MB as root.

Also, I think there's an environment variable for it,
CDR_FIFOSIZE.
