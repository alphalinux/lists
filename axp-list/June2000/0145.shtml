Jay Estabrook wrote:
>
[..]
> HOWEVER (and this is a *big* however), the patches/binary will most
> likely ONLY work on EV6 boxes, as there is some pretty funky code in
> there that is going to require some work to run on boxes that must
> use SPARSE addressing (PYXIS-based boxes should be OK, with further
> infrastructure changes, ie GLIBC).
>
Can you elaborate a bit further ? This sounds suspiciously like the MGA
problems on PYXIS we had (and still have, sadly) with XF4.0.
I did the patch which led to the inclusion of pciconfig_iobase syscall
in later kernels, but the actual patch to XF4.0 has been rejected due to
the missing global framework (And, truth to be told, the syscall was not
_that_ helpful, for we first had to figure out how to access the bus (ie
BWX or Sparse addressing), before we could use the syscall, which would
have told us this information ... So it would be better if we had a
global variable where exactly this information is stored, without having
to explicitely referencing the bus. After all, the access method is not
likely to change. ).
If that's a step in this direction please keep me posted.
