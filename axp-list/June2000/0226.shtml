Richard Henderson wrote:
>
> On Fri, Jun 16, 2000 at 04:38:25PM +0100, Hannes Reinecke wrote:
> > What I _really_ meant with that statement is that the mapping scheme is
> > independent from the individual cards, and so in theory it should be
> > possible to figure out the mapping scheme beforehand.
>
> And it is. Use bus 0, device 0. I thought this was
> documented in the patch.
>

Hmm. It doesn't exactly say that. arch/alpha/kernel/pci.c:515 reads:

        /* Special hook for ISA access. */
        if (bus == 0 && dfn == 0) {
                hose = pci_isa_hose;

A bit more digging revealed that indeed the PCI hose and the ISA hose is
always the same (for those machines with only one hose anyway), which is
not immediately obvious from the context above. But then, maybe it is
and I should go back to coding 'Hello world.'. You never know.
