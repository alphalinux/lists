I'm going through this phase with my Red Hat Linux cluster (10 va linux
boxes) where I can't tell if I have enough of them or not. Ever felt that
way? I have no real way of determining the over all bottle necks or seeing
if are indeed ANY bottle necks. Btw, these are used as web servers running
apache w/mod_perl.

So, in disgust of this feeling of being in a black open space and not
knowing jack, I'm on a small quest to figure this stuff out and accurately
predict the resource utilization and bottlenecks for present and future so
that I can know things like "Hey, in a month or so you may need to add
another 1GB of memory to your front-end boxes and add 2 more machines".
Where as now it's more like "Well, none of the servers crashed this week,
guess they are doing okay".

So the Q's:
- Where can I find TFM on this stuff. I want to learn!!! Yeah, I know...
there is no manual. But any (subtle or not so subtle) clues would be nice.
- Software or even hardware packages which can help with this?
- Out Source this quest to a company that masters it, are they out there?
Can I learn from them (as this task of planning for the future will probably
be a regular thing, unless I die soon)

A Somewhat DIFFERENT Note:
I hacked up a system monitoring system that gives me graphs for whatever I
tell it (CPU Idle time, Memory usage, httpd children, total proc's, swap,
etc). It looks great but I have this fear in the back of my mind concerning
the CPU Idle % numbers...
I grab a CPU Idle% number every 5 minutes by grabbing the output from:
vmstat 60 2

Is that accurate enough to plot over time and look for spikes?
Example: every day or so I'll see a drop from 98% CPU Idle to 75%. Should I
be concerned about this?

Thanks for the help... oh, and please email me and I'll be sure to write a
summery on this issue.
