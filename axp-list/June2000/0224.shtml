Right. Let me clarify it:
(Interesting to find out what I really meant with that posting ...)

What I _really_ meant with that statement is that the mapping scheme is
independent from the individual cards, and so in theory it should be
possible to figure out the mapping scheme beforehand. This would enable
us to set a global flag somewhere in the depths of XFree86, and we could
use that information to set up the framebuffer mapping.
(As it is now, the Bus/Card information is not passed down the the
appropriate function (os-support/linux/lnx_video.c:xf86OsInitVidMem()
for those who want to know) and we would have to change the calling
parameter to obtain it.)

Und _this_ is not possible with the current syscall, since it wants to
have the Card information.

Sorry for the confusion.
