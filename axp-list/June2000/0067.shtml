Graham Van Epps <gvanepps@gankish.net> writes:

> i dont suppose anyone got the linux frame buffer working with one of
> these. i personally use console a lot, and the fb device makes it much
> more usable, and it also looks pretty sweet:) right now i hear the fb
> support for the matrox cards is "sketchy". any thoughts?

_fbdev_ works fine (for XF86_FBDev, GGI, or what have you). However,
_fbcon_ (the graphical console) does not work with any driver other
than tgafb as far as I can tell (I've had no luck with matroxfb on
Millennium I, nor with pm2fb on ELSA GLoria Synergy).

I'd be interested to know if anyone has any idea why ... I would also
really like to get framebuffer text consoles working.

My best guess would be either (a) 64-bit issues, or (b) missing wmb()
in various places (the code probably uses eieio() instead, which is a
no-op on anything except PowerPC; I seem to remember reading an
inconclusive flamewar on this subject).
