On Tue, Jun 13, 2000 at 04:04:16PM +0100, Mathieu MILLET wrote:
>
> I have just seen in a complete Alpha configuration (based on a
> UP1000) a Nvidia Quadro-based video card.
>
> I thought that Nvidia video cards aren't supported on the Alpha
> plateform (because of 32 bits i386 operations at startup that the
> firmware can't handle).
>
> Am I wrong ?
> If I am, what is the state of the support ?

It seems that the latest SRM firmware release, ie v5.7 (which apparently
is used on the UP1000), does have the ability to handle the BIOS code on
some of the nVidia cards.

I've not tried anything except the ELSA Synergy II (Riva TNT2-based),
nor have I tried it in anything except EV6-based boxes, but it seems
to work fine in a DS20 and a DS10.

I've even been able to coerce the XFree86 4.0 "nv" server to run (again
on the DS20), and will feedback patches to XFree86, hopefully in time
for their next update. In the meantime, I can make the patches and a
binary available for anyone wishing to try this out.

HOWEVER (and this is a *big* however), the patches/binary will most
likely ONLY work on EV6 boxes, as there is some pretty funky code in
there that is going to require some work to run on boxes that must
use SPARSE addressing (PYXIS-based boxes should be OK, with further
infrastructure changes, ie GLIBC).
