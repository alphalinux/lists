Wolfram> using in alpha-libc, but if it's GNU malloc or (worse)
Wolfram> derived from the old BSD code, there is a lot to gain from
Wolfram> using Doug's version. It is by far the most
Wolfram> space-efficient malloc implementation that I know of. It

like to see it in glibc if it really does improve things.
