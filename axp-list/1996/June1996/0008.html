On Sat, 1 Jun 1996, Richard Henderson wrote:
>
> > While pokeing around and fixing the specfile for emacs-19.31 (thanx for
> > pointing out the error David!) I found out that 'strip' on linux/x86
> > preserves permissions (like suid group), while strip on linux/alpha
> > removes them.
>
> The sugid bits are supposed to go away when a file is modified.
> It is a security measure for which the kernel is responsible.
>
> I wouldn't know why they would be perserved on i386. Is the
> strip used on i386 binutils based? Does H.J. Lu have a special
> patch in place to magically preserve those mode bits?

Umm.. What was the kernel version on the x86 setup? There is a problem
with the code that strips the suid/sgid bits off an inode with 1.2.x and
1.3.x (x < 80 or something), so the code didn't actually work 100%. So it
_may_ be that you're using a new kernel on the alpha, and an older one on
the x86, so...
