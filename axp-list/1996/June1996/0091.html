On Thu, 6 Jun 1996, Jay Estabrook - Alpha Migration Tools - LINUX Project wrote:
>
> Now, as I remember, you're booting via SRM; I don't believe the kernel
> actually exits back to the SRM console correctly, and thus you're left
> with an (essentially) dead machine upon shutdown.
>
> I don't know how hard this would be to fix; anyone else?

If I remember correctly, Linux just does a "call_pal halt" to return to
the SRM console.

The problem seems to be that the SRM console does not correctly
re-initialize the hardware - I suspect it gets confused by the fact that
Linux has reprogrammed the keyboard and/or the interrupt controller. So
while the SRM console shows the nice blue screen, you can't actually do
anything with it.

At least that's what happens on my Jensen.

I'd consider it a SRM bug, but maybe one that could be worked around..
