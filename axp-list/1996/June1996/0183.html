OK, I bit the bullet and upgraded to procps-1.01. There is now a new
patch at:

ftp://ftp.azstarnet.com/pub/linux/axp/glibc/procps-1.01axp.patch.gz

It should have all the old procps fixes plus some fixes by Kazushige
Goto and by Harald Koenig (but I was in a hurry, so more testing will
be needed). I modified psupdate so it can accept either a.out or
ELF-style binaries or if these don't work, it falls back to using BFD
(as long as BFD-support has been configured in, which is the default
for Linux/Alpha).

Oh, there are also some other minor fixes the get things working with
glibc (the utmp entry there allows a hostname of up to 256 characters,
which resulted in a funny behavior in w).

Charles, would you mind integrating these patches into your source
tree? None of the patches should break anything for the x86 (I wasn't
able to test elfupdate and aoutupdate, but I was careful not to break
anything, so I hope they will still work just fine).

Harald, now I have a bug report for you (or any X-server hacker, I
suppose): xcpustate prints the bar-chart all in black when running the
latest XF86_S3 server in 16bpp mode. In 8bpp mode, the stippling
works just fine. Bug or feature?

cheers,
--david
