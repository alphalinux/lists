This _sounds_ like the infamous termio vs. termios problem. I'm not
very familiar with curses/ncurses, but if a program attempts to mix
termio and termios structures, it is calling for trouble. Due to
OSF/1 compatibility (which, in turn is compatible with some old
version of SysV), the termio structure for Linux/Alph is not
compatible with termios. The best solution is to use termios only,
IMHO. So, I'd suggest to grep for "termio" and check whether the
program might declare "struct termio" and then use termios calls (or
vice versa). Also, it's wrong to include <termio.h> and then use
"struct termios" (the defines for the V* macros are different).

Hmmh, independent of curses, this is a FAQ, I guess...
