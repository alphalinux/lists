MILO only tries to uncompress kernel immages which file names end
with ".gz" or ".GZ" which can be *bad*:

trying a new kernel version I moved my old one from /vmlinux.gz to /vmlinux.gz.old
and installed a new /vmlinux.gz which (unfortunately) didn't boot.
trying to boot my "safe" vmlinux.gz.old didn't work.

also, MILO doesn't support or use RockRidge extensions. when I tried
to load a new boot kernel from the Craftswork CDROM the first ".gz"
hit me again: the kernel images in /cdrom/boot are

-rwxr-xr-x 1 root root 559971 Apr 23 01:37 vmlinux.cabriolet.gz*
-rwxr-xr-x 1 root root 559729 Apr 22 19:48 vmlinux.eb164.gz*
-rwxr-xr-x 1 root root 560973 Apr 22 18:55 vmlinux.noname.gz*
-rwxr-xr-x 1 root root 561883 Apr 22 18:52 vmlinux.udb.gz*

which gives "vmlinux.cab" etc. in MILO, again not possible to bot them :-(

I really think MILO should load the first block of the boot image and
check it's magic number than rely on the filename.

Harald
