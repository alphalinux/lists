On Tue, 4 Jun 1996, Berndt Josef Wulf wrote:

> >
> > I'm considering getting a fairly low end alpha to run linux/axp on (red
> > hat specifically) and was kind of curious as to the performance attributes
> > of some of the low end linux machines. The one machine that I really like
> > is a "UDB" or "noname" I think you guys call it, based on either a 166 MHz
> > or 233 MHz 21066 CPU with 256kb cache. With linux/axp in it's current
> > state, and gcc/alpha in it's current state, does anyone have an idea of
> > what a 166 21066 or 233 21066 would compare to performance wise to a
> > linux/intel machine?
> >
> >
>
> I rate the 166Mhz Intel to be superior to a AXPpci33 166Mhz or
> AXPpci33+ 233Mhz and by far cheaper ;)
>
> cheerio Berndt

Steve Gaudet at DCG says he figures a 166Mhz. UDB to be about equal
to a 75 Mhz. Pentium (correct me if I'm wrong).

I purchased a 233 UDB "clocked up" to 266 (and please let's not start
THAT thread again).

Comparing it to my previous 150Mhz. Pentium I find:

X seems to run significantly faster, this in spite of ther fact that I
was running AcceleratedX's server on the Pentium which was not available
for the Alpha - and AccelX did speed up the i586 greatly.

Compiles seem a little faster, this may well be due to faster disk I/O
(fast-scsi-2 as opposed to IDE on the pentium) and more RAM (56 MB
on the UDB as opposed to 48MB on the i586)

Code is larger due to no shared libraries (yet), not a problem if you
have a lot of disk space and RAM.

All in all I think the 266 Mhx UDB is a great system, but once you add
in the large external Hard drive, an external tape drive, an external
CDROM and some extra RAM the $3K machine starts pushing $5K :-)

Anyway, just my $0.02
