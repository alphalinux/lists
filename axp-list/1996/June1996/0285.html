This is not a bug. This is a silly limitation on the part of
the COFF object file format that ECOFF did not bother to correct:
no section may have more than 65535 relocation entries.

Unfortunately for you, cpu.c has a huge table with 65536 function
entries, each of which needs one relocation entry. (There are
other data bits that bring the total up to 0x100c5).

One thing you can do for now is replace the Nil function with NULL
and test for NULL in execute(). That should bring the total down
under the limit.

Or you can wait for ELF; we've got no silly 16-bit (or even 32-bit)
restrictions on relocations there.
