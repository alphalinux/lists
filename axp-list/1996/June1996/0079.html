>I've been down that road. I continue to get errors in binfmt_elf.c
>saying that there is no such member 'edx' in structure
>regs. It's found in the function load_elf_binary and accepts a parameter
>of pointer to structure type pt_regs and, in fact, after looking through
>the relevant headers and finding the declaration of pt_regs there is no
>member variable named 'edx.'
>This kernel version is 1.3.97 (also, when I change the arch to alpha, I
>can only make vmlinux, but not vmlinux.gz or zImage).
>Any thoughts?

You need some kernel patches in order to be able to run ELF
binaries.
I can send out the patches to anyone who wants to try, but you need the ELF
compiler, ELF binutils, ELF libc and ELF gdb before it would really be of
much use.

If it weren't for the fact that we were close to 2.0, I would have
sent the kernel patches to Linus. They affect a bit more than just the
Alpha, since I had to generalize and abstract a few things so that
binfmt_elf
could work for any architecture.

FWIW, I think the kernel end of things is fairly stable right now.
There are still some userland issues to be solved before it is really
ready for prime time. We haven't really tried ELF kernels yet - it is a
lot easier to shake things down in userland than it would be to try and
debug a linker by testing kernels.
