gcc has the inverse switch: -traditional. gcc by default operates in
ANSI mode, cc in K&R mode. Back in the old days (umh, is it really
_that_ long?? ;-), C folks where to lazy to declare prototypes. As a
result, the compilers had to be conservative and always pass arguments
in the widest possible form (floats as doubles, char, short, and int
as long). Similarly, all expression evaluation was done in the widest
type (long & double, again). In other words, "float" used to make
sense only for the purpose of saving memory.

Maybe DEC can be convinced that the time has come to make -std1 the
default?

In case of doubt, objdump -S will tell it all... ;-)
