Hello everbody,

in the last few days I played a little with "samba". My system is based
on Red Hat 3.0.3axp. One of the first things I noticed is, that the
"nmbd" in the binary distribution is of version "1.9.13" while the
packages claims to be "1.9.15".
I investigated the "SPECS"-File and noticed a typo (?) :

mv /usr/bin/nmbd /usr/sbin/nmbda

and in the "Files"-section:
/usr/sbin/nmbd

This is why the nmbd is not an original 1.9.15-Binary. Anyway, the
delivered package works mostly as expected. Browsing on WfWg doesn't
work correctly but mounting etc. is OK.
Another thing that doesn't work is a command mentioned in the
"/usr/doc/samba-1.9.15p8-2/DIAGNOSIS.txt"-File.
"nmblookup -B <hostname> __SAMBA__"
should return the IP-Adresse of the host. It doesn't: ("couldn't find
name __SAMBA__").

Then I tried to recompile the whole package (I had to do it anyway,
because "nmblookup" is missing in the rpm-package). After applying the
patch, the compilation works out of the box. But the resulting "nmbd"
doesn't work at all with "Windows Clients". They report "unexpected
network error". The previously not working:
"nmblookup -B <hostname> __SAMBA__"
now reports "0.0.0.0" as IP adress.

Is this a known bug in respect to "nmbd" on the Linux Alpha platform?

Markus
