> > Is it possible to get 'gated' working as a bridge between two different
> > subnets, and how ?
> > Thanks for the help.

It seems that you don't have IP forwarding configured into your kernel... You
will have to recompile it.

>
> I'm just learning this stuff myself, but I don't think you need to run
> gated to turn your machine into a network bridge. All you have to do is
> 'ifconfig' the network interfaces correctly and then set up a couple of
> routes by hand with /sbin/route. Check out the Linux NAG for details.

After recompiling the kernel with IP forwarding enabled, here is what you
need in your rc.inet1 or equivalent :

ifconfig lo 127.0.0.1
route add -net 127.0.0.0

ifconfig eth0 $MYADDR_ON_ETH0 netmask $NETMASK_ON_ETH0 broadcast \
$BROADCAST_ON_ETH0
route add -net ${NETWORK_ADDR_ETH0} netmask ${NETMASK_ON_ETH0}

ifconfig eth1 $MYADDR_ON_ETH1 netmask $NETMASK_ON_ETH1 broadcast \
$BROADCAST_ON_ETH1
route add -net ${NETWORK_ADDR_ETH1} netmask ${NETMASK_ON_ETH1}

route add default gw ${GATEWAY} metric 1

In short, just start with the classical Ethernet setup and duplicate the
commands involving your local network. Your gateway for the default route
(${GATEWAY}) remains unchanged; and your adresses on the two subnets must of
course be chosen in the good range.

I don't think this thread belongs to that list, anyway...

>
> It's quite possible that I'm dead wrong, but I don't think you need
> gated to set up a simple bridge.

True. Gated/routed are only useful when your routing scheme is more
complicated than the kernel built-in one -- which is not the case for a bridge.
