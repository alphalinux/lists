>env_init: No environment present

It simply means that MILO found no environment that *it* could recognize,
ie one that *it* created. Go ahead and get MILO to create one for you, as
I believe that all works just fine on the NONAME. You can set some variables
so that you need only type "boot" to get it to go, such as:

set BOOT_FILE vmlinux.gz
set BOOT_STRING root=/dev/sda2

Look in the MILO docs for more info...

RE: the scsi problems, is your CD at unit #3? Try disconnecting it and see
if things go better. I know, that's not a solution, but at least will try
to identify the problem more explicitly. Is this a stock kernel from off of
a RedHat CD, or a custom built one? If custom, did you have probblems with
a stock one? If no, try rebuilding, specifying *none* of the faster options
to the NCR53c810 configuration questions...

Good luck.
