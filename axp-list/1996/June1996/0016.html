I have been running a SQUID cache on our alpha box running RHCL/AXP
2.1.

The new version broke, badly!

This was due to a line like this:-

if (inet_addr(ip) != -1) {

which never failed (ie never returned -1). This is becuase in
/usr/include/arpa/inet.h there was a definition like this...
unsigned long inet_addr __P((const char *));

Its the "good old lets shove an ip address in a long because they are
always 32 bits" mindset problem! I don't have 3.0.3 so don't know if
its now fixed!
