Hello again,

David Mosberger told me the follwing about the nmbd-problem:

> Internet (IPv4) addresses are 32 bits long. It's better to use
> u_int32_t instead of unsigned long.

I then was quite daring and used the brute force-method:

cd /usr/src/redhat/SOURCES/samba-1.9.15p8

for i in *.c # (and *.h)
do
mv $i $i.orig
cat $i.orig | sed -e 's/unsigned long/u_int32_t/g' >$i
done

make

And my _very_ unexpected result so far is: everything works like a
charme :-)) I'm really happy about this.
