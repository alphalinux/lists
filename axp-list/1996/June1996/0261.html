Hello Michael!

> I just tried to compile STonX 0.6.5 (a free Atari ST emulator). When
> compiling cpu.c, after a short time, a huge (24MB) 'as' is started,
> and after some time, this process does not seem to do anything.
When compiling uae (the unix amiga emulator) I had the same problem.
'as' seems to have problems with files that are longer than 64K lines!

> Is this 'normal' behaviour? Any way to get around this?
There does exist a simple solution to the problem:

1. Compile the file with gcc and all necessary options. Do only generate
an assembler file, no object file!
2. Split the assembler file into pieces that have less than 64K lines each
and send them through 'as'/'gas'.
3. Link all pieces one after each other.

That was the only possiblity to compile the large function pointer table of
the uae 68K emulator!

I hope this solution is applicable to STonX, too.
