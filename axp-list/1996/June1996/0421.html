Hi netters,

I cannot access a Exabyte 8200 on a UDB (Multia) running 3.0.3 and kernel
1.3.89. The kernel seems to recognize the device :

>[root@hal log]# cat /proc/scsi/scsi
>Attached devices:
>Host: scsi0 Channel: 00 Id: 00 Lun: 00
> Vendor: DEC Model: RZ26F (C) DEC Rev: 1Q0A
> Type: Direct-Access ANSI SCSI revision: 02
>Host: scsi0 Channel: 00 Id: 01 Lun: 00
> Vendor: CONNER Model: CFP2105S 2.14GB Rev: 2D4D
> Type: Direct-Access ANSI SCSI revision: 02
>Host: scsi0 Channel: 00 Id: 03 Lun: 00
> Vendor: FUJITSU Model: M2934S-512 Rev: 0110
> Type: Direct-Access ANSI SCSI revision: 02
>Host: scsi0 Channel: 00 Id: 04 Lun: 00
> Vendor: DEC Model: RRD42 (C) DEC Rev: 4.5d
> Type: CD-ROM ANSI SCSI revision: 02
>Host: scsi0 Channel: 00 Id: 05 Lun: 00
> Vendor: EXABYTE Model: EXB-8200 Rev: 425A
> Type: Sequential-Access ANSI SCSI revision: 01

Note the last lines. When the kernel boots it shows a

>Jun 28 12:18:08 hal kernel: scsi0 : setting target 1 to period 100ns
>offset 8 10.00MHz FAST SCSI-II
>Jun 28 12:18:08 hal kernel: Type: Direct-Access
>ANSI SCSI revision: 02
>Jun 28 12:18:08 hal kernel: scsi0 : target 3 accepting period 100ns
>offset 8 10.00MHz FAST SCSI-II
>Jun 28 12:18:08 hal kernel: scsi0 : setting target 3 to period 100ns
>offset 8 10.00MHz FAST SCSI-II
>Jun 28 12:18:08 hal kernel: Type: Direct-Access
>ANSI SCSI revision: 02
>Jun 28 12:18:08 hal kernel: scsi0 : setting target 4 to asynchronous SCSI
>Jun 28 12:18:08 hal kernel: Type: CD-ROM
>ANSI SCSI revision: 02
>Jun 28 12:18:08 hal kernel: scsi0 : setting target 5 to asynchronous SCSI
>Jun 28 12:18:08 hal kernel: Type: Sequential-Access
>ANSI SCSI revision: 01
>Jun 28 12:18:08 hal kernel: scsi : detected 1 SCSI cdrom 3 SCSI disks total.
>Jun 28 12:18:08 hal kernel: SCSI device sda: hdwr sector= 512 bytes.
>Sectors= 2050860 [1001 MB] [1.0 GB]
>Jun 28 12:18:08 hal kernel: SCSI device sdb: hdwr sector= 512 bytes.
>Sectors= 4194304 [2048 MB] [2.0 GB]
>Jun 28 12:18:08 hal kernel: SCSI device sdc: hdwr sector= 512 bytes.
>Sectors= 8506782 [4153 MB] [4.2 GB]

where the target 5 corresponds to the SCSI ID of 5 from the Exabyte.
The problem occurs when I try to access the device:

>[root@hal log]# ls -alg /dev/st*
>crw-rw---- 1 root disk 9, 0 Jun 28 20:15 /dev/st0
>crw-rw---- 1 root root 9, 1 Oct 4 1995 /dev/st1
>lrwxrwxrwx 1 root root 17 Jun 28 2016 /dev/stderr ->
>../proc/self/fd/2
>lrwxrwxrwx 1 root root 17 Jun 28 2016 /dev/stdin ->
>../proc/self/fd/1
>lrwxrwxrwx 1 root root 17 Jun 28 2016 /dev/stdout ->
>../proc/self/fd/0
>[root@hal log]# ls -alg /dev/nst*
>crw-rw---- 1 root disk 9, 128 Jun 28 20:15 /dev/nst0
>crw-rw---- 1 root root 9, 129 Oct 4 1995 /dev/nst1

it seems to have the right major number (and I have only 1 tape, so st0
should it be according to the SCSI-HOWTO).

>[root@hal log]# tar vtf /dev/st0
>tar: Cannot open /dev/st0: No such device

On any device (st0,st1,nst0,nst1) the same answer. The tape is online and
can be accessed without any problems on the same computer from dUnix and NT.

Can anyone help me?
