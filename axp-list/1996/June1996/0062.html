If that's the version I picked up May 25 at 2:00 AM EDT, then it
does have some problems. Trying to use "next" to step through
code, I've had it just keep going until it hit some breakpoint
farther _up_ the stack (suggesting it didn't properly place the
temporary breakpoint for the "next"). I've also had it crash
on me when trying to print a member of a structure.

I've assumed these are known problems, but, if not, please let
me know what I should do next time I run into this. Just emailing
in a debug session probably isn't much use, since I use Emacs' gud
mode, so there's no source code visible, and the executable I'm
usually debugging is around 9MB long.

BTW, the latest gdb _does_ fix the "p $f10" problem I reported,
where f10 had a denorm in it! I've suggested to the gdb developers
that they change "info reg" to _first_ print the raw hex values,
in case gdb has trouble with the FP representation -- that way
you'd at least see the hex value before gdb gave up!
