Hello,

I just installed RedHat 3.03 on my Jensen. Everything went fine, except
partitioning:

I connected a new 1Gig (Quantum Fireball) drive and used fdisk in BSD disklabel
mode to create space for aboot, swap and root fs. During the installation, when
mke2fs was executed, the system crashed with '1742: asynchronous event'
messages scrolling through (and a SCSI reset occured each time). The only thing
I could do was power down and reboot.

After reformatting the disk changed nothing in this behaviour, I used
sdiskbale instead - and voila, it seems that fdisk had created a partition
which extened beyond the end of the disk.

Now it seems that there are two problems:
- fdisk creating this wrong disklabel and
- the low-level 1742 driver not handling error codes correctly.

Have there been any changes between the fdisk in RedHat 2.x (which I used)
and the current one?
