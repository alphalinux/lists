On Sat, 29 Jun 1996, Harald Koenig wrote:

> I've seen such problems with early versions of RPM2 too.
> it would be really nice if RPMs of one version would all be compatible
> or, if this is not possible, you would add strict version checking for
> the rpm revision first and output at least a warning, better a fatal error.

The only other time I've seen this problem is with corrupted packages, which
RPM handles very badly.

The new format has the following fixes:

1) all packages include an md5 checksum for detecting corruption
2) file sizes are recorded, and all operations on a package file
cause that to be checked (cheap signature)
3) the major version number of the package file format will change

The package that got into the wild was built with a preliminary version of
RPM. It hadn't had the package version number change yet and has never been
released.

We are trying to make these occurences go away, and the recent changes to the
package format should make that easier.
