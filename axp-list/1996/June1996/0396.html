Works nice with my Emerald Q525 drive. Pity it doesnt like my Matshita
CDROM drive, to the point that if I try to access it, the system locks up.
The Matshita drive seems to be refusing SYNC negotiation by protocol
violation. MILO complains about it when I 'show' devices.

The killer though it what it does to a Seagate ST3283N, unless I turn sync
off the file system gets hoplessly corrupted. Trouble is corruption occurs
before the boot script gets to mounting /proc and echoing a setsync command
to the driver.

Ill have to switch back to the old driver for the sanity of my news spool.
