Just uploaded a patch for the latest NFS server. Since it's only known
to compile against glibc, I put it in that directory:

ftp://ftp.azstarnet.com/pub/linux/axp/glibc/nfs-server-2.2beta16axp.patch.gz

This patch is a lot cleaner than the earlier ones for the old libc (it
actually takes advantage of 64 bit ints and should work reliably even
with very large device and/or inode numbers).

While I haven't tried it, it should compile with the old libc as well.

--david

PS: If you want to try this with glibc, you may want to wait a day or
two---the NFS server exposed a bug in realpath() that the "official"
snapshot hasn't fixed yet.
