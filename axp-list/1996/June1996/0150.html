Hey guys, I thought you might be interested in some cross platform results.

I had to change the timer routine somewhat to get it happy under NT, but otherwise
it's the same code. Machine tested is a Dell Dimension XPS Pro200, 200 MHz Pentium
Pro, 256kb L2 cache, 32 MB 2-way interleaved 60 ns FPM dram. I noticed that it
seems to be operating on an array of floats - meaning if you have a 2 MB L3 cache
alpha system this test almost never gets outside the cache- and the cache on the
pentium pro is escaped pretty quickly, is only 256kb in size anyway, and has no L3
cache to support the system outside that.

System was NT 3.51 Workstation, Compiler was Intel Reference C Compiler 2.3.2 beta
(and I got nearly identical numbers from MSVC 2.0, which is somewhat
suspicious...)

I figure the pentium pro's L2 cache should be good to about the 50,000 size array
(if they are indeed floats, haven't looked at that closely yet) so it's kind of
interesting that performance drops off so rapidly compared to the small array
sizes. Perhaps some strange unaligned access problem? Dunno. Anyway, take those
numbers with a grain of salt, but thought you guys might be interested.

Oh yes, compiler options were:
-O2 -Qip -Qmem -G6
