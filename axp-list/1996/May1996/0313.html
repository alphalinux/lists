Yes, old binaries *may* be broken. It depends on the specifics
of their section sizes and how they got aligned.

To be frank, between the two bugs brought to light with your strip
report, I'm surprised anything worked before at all.

> but why is it possible to use "quickstrip" on the "broken" programs with
> no problem? does quickstrip fix the real problem which was caused by ld ?

As Jim Pardis explained, quickstrip simply truncates the file and
twiddles some bits. Strip, on the other hand, is actually objcopy
with a few defaults. It performs a complete translation of the
input file to the output file with a filter in between.

In the time-before-last problem (one liner with _PDATA), the linker
did not have the right flags for the segment so it did not align
it properly. When strip read the section, it got the flags set
properly, but when it went to write the section back out again,
the changed now-correct flags caused the alignment to change. This
alignment change breaks programs.

I hope this is somewhat clear.
