About the new Xservers based on 3.1.2E, available now for the Alpha:

1. unless Harald used the same "magic" as Erik Troan of RedHat, those
servers will *ONLY* run on the EV4-based machines, much as my original
set based on 3.1.1 and 3.1.2 releases.

2. the "magic" Erik used is available via the RedHat 3.0.3 LIBC; I am working
the appropriate patches for 3.1.2E to make the *next* set of beta servers
able to run on EV4 *and* EV5 machines, same binary executable. The same
facility (mostly) will be available via the GNU LIBC that David M-T is still
working on, release date sometime not too far away.

Note that the S3 and Mach64 Xservers on RedHat LINUX 3.0.3 for AXP, *will*
run on either EV4 or EV5 machines (although the Mach64 was quite unstable on the
EV5 I tried it on).

Thanks to Harald and others of the XFree86 org, the 3.1.2E servers on the
Alpha have fixed that annoying font-related problem, as well as having gotten
rid of all (?) the unaligned accesses in the S3 server, among *many* other
general improvements.

NICE JOB, folks! :-)

Now, on a somewhat different topic, is anyone:

1. interested in patches for the XFree86 Mach32 server to run on Alpha.

2. working on/interested in having, the SVGA server running on the Alpha.

The first is (mostly) working, simply 'cuz I had a Mach32 and the time... :-)

The second is a concern, because there's an Alpha notebook made by Tadpole of
the UK, which needs the SVGA server (its got a WD90C24A (?) onboard).
