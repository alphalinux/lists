> We'd want to install Linux on an Alpha 600 5/266, but we don't know
> nearly anything about the Alpha Linux's distribution.

Since you found your way onto the RedHat Linux/AXP mailing list,
you've already located the source of one distribution... :-)

> First of all we'd like to know if it's possible to install that
> distribution on our machine,

I don't believe it's possible; even though there's EV5 (21164 CPU) support
in the latest LINUX sources, and kernels for the EB164 in the RH 3.0.3
distribution, you'll find it won't work on your box.

Your box is known as an ALCOR; it has the EV5 CPU, plus PCI and EISA busses.
There's a TGA in there, which does have LINUX console and X support. There's
also a COMBO card as well, with a DEC TULIP Ethernet controller on it and a
pair of QLogic 1020ISP SCSI F/W controllers on it.

Now, there is a driver for the QLOGIC in LINUX pre-2.0.8, but my Alpha
patches for it aren't in there (yet). The TULIP works quite nicely on
that COMBO card (I've had it in both an XL and an EB164).

But the biggest problems are:

1. EISA bus - of all the supported Alpha platforms, only JENSEN has EISA,
and JENSEN is such an odd duck, that doesn't help at all. All the others
are PCI and ISA, and so we've no experience with the PCI/EISA bridge chip
on that MB.

2. Because of the EISA bridge, I think that our MILO will not run on an
ALCOR. Without MILO, you must depend on SRM, and the kernel's SRM support,
while fine for some platforms, seems woefully deficient for others (I'm
currently playing with a MIKASA (AlphaServer 1000) with PCI/EISA and SRM,
and having a good number of problems).

So, bottom line is, I think you're (currently) out of luck... :-(

> and what's the stability of the SO on it. Where to find the latest
> distribution, how to install it, how
> to make a backup of the flashrom in case we get in trouble, etc...

Stability, for *supported* platforms, is quite good, now. No shared libraries
(yet), but they're coming...

Backing up FLASH isn't really needed unless you intend to install MILO
there; but I believe this isn't possible right now, anyway. Or maybe
possible, but not advisable, given that total support for the platform
is not present in LINUX nor MILO...

If you don't have an Alpha platform that currently runs LINUX, you'd best
wait for someone else to do get the ALCOR going. If such is available to
you, then use that as a build machine, and have a go at the ALCOR... :-)
