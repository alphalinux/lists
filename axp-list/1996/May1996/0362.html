I believe a gdb linked against glibc would avoid this problem. This is
because (AFAIK), all fp I/O in glibc is done through multiprecision (integer)
arithmetic. So printing exceptional numbers such as NaNs and Infs should not
be a problem. But to be honest, I haven't had time yet to try this. I have
been planning on building binutils and gdb against glibc in the next few
days (hopefully starting today). I'll test this and if it works, send you
a note.

The alternative would be to rebuild the entire libc with -mieee, but that would
be painful.
