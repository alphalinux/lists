> problems:
> - I cannot read the realtimeclock! It always says 1.1.1970 or 12.31.2016!

I asked this question a while ago and think that I got the answer now:
In the 'jensen.h' file it says, that on a jensen, the clock is on a
different io-port. (0x70 instead of 0x170, I think). While the kernel
adjusts to this, the clock programm does not. Well, I was too lazy to
recompile clock for the jensen, but use 'date -s "-2 hours"' in the
rc.sysinit file and adjust the real time clock within the console
instead. (The kernel read's the correct time upon startup -- 2
hours account for the difference between local ARC console time and
GMT.)

Bye, Martin

One more thing: If you use an old firmware, subtract another 20
years. (e.i. 'date -s "-20 years -2 hours"')
