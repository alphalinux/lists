> this has bothered me before, too. It appears as if the list server
> inserts a "Reply-To: axp-list@.." line, if no Reply-To line exists in
> the original posting (mail). This is annoying, since I have to
> manually change the "to:" line of my answer in case I'd like to send a
> personal reply. And I also have often forgotten it, since I expect the
> "Reply"-Command of my mail program to send a reply to the author
> instead of a follow-up to the list (and the author). The "Follow-up"
> command gets useless, as it's now going to send two copies (To: and
> CC:) to the mailing list. BTW, this is the only list I am on that
> shows this behaviour.
>
> I send this to list instead of the list maintainer, because I'd like
> others to comment on this issue.

This is the intended behavior of all the redhat run mailing lists.
In general, responses should always go back to the list. Whether you
guys know it or not, there are tons of "lurkers" on every list. They
won't post, but they do read everything and get tons of useful information
from the questions and replies. Every time someone suggests changing
our setup, those "lurkers" email *us* (the redhat maintainers) asking
that we *not* change it.

Most questions that are asked here need to be answered here. In the
cases where they are off topic or the author would rather summarize
responses, the author should set his Reply-to: in his header appropriately.
Our server will not override one that has been set. If the author
hasn't set one, it is still up to the person responding where to send
the reply. In general, it should go to the list. If that isn't desired,
the responder should change it manually.

I tend to get *really* annoyed when someone changes the Reply-To
to point at themselves for no apparent reason. Just doesn't make
sense...mailing lists exist for *everyone*, not just for each person
to ask questions and get answers. Even the dumbest of questions
should be answered *once* on the mailing list. Why? Because those
responses get archived and it might save someone from asking *again*
if they can find the answer themselves.

This is the way it works and it isn't likely to change any time soon.
(Doesn't mean we aren't open to discussion, but this comes up from time
to time...there are those on the side of the "Reply-To should be
the author", but there are many more on the other side of the argument.)
