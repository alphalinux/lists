running DEC Unix images using shared libs shows a problem with argv[0].
this is important for "virtex" which tries to open `argv[0]`.fmt as
TeX format file.

-------------------------------------------------------------------------------
main(int argc, char *argv[])
{
for (;argc--;) printf("'%s'\n",*argv++);
}
-------------------------------------------------------------------------------

when this program is compiled using DEC cc and run on Linux I get:

# ./a.out
'/sbin/loader'
