Well, hello all,

There seems to be a number of very interesting and related issues
floating around, so I thought I might offer my 2 cents (will total 6 cents
when finished).

Is DEC an open, friendly, and helpful company?

We have been designing small Alphas for industrial applications for
over two years here Singapore and I have never! had stronger more open
support from a company before, both locally and from overseas locations,
in terms of information, help, samples, problem solving, and good old
fashioned good will. From what I have seen on this list I would say
the same is true of software issues (such as Linux Alpha). Enough said.

DEC UNIX/Linux binary (library & object code) compatibility

Oops, I didn't even think to see if DEC UNIX binaries would run under
Linux, but as developers of lots of high performance industrial software we
have found it extremely useful for object code and library things to be
cross platform compatible, both between our own (mini) OS and OSF and now
between Linux and the first two. This has been somewhat for convienence's
sake but mostly from the standpoint of performance enhancement. Which
leads to....

Would performance enhancement for GCC and its (Linux??) be popularpular?

For several (and honorable) reasons GCC and related software are not
nearly as well tuned for performance as DEC's software. Although Alpha is
a beautiful hardware architecture, I think most people have (or want) an
Alpha for the sake of performance. It might seem a natural time to see
if the (friendly, but probably overworked) Digital people and the GNU
community might begin to look in to some of these performace issues. I
for one would be glad to help if I could.

Allan
