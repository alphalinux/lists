Subject: Cabriolet configuration problems RH 2.1

We have a Cab. w/ 32M, NCR SCSI, NE2000 (FO or BNC, 2 cards), and 2.2 G HD.
Trying to configure with RedHat 2.1 has been a REAL PAIN due to the following:
1. FDISK did not properly handle the 2.2 G HD. We had to fdisk it on
a P5 LInux box.
2. Install scripts did not see the P5-made linux native and swap partions,
I had to manuall mke2fs and mkswap, mount root on /mnt, and swapon, all
from the ramdisk.
3. I had to use the install software to mount the linux partitions prior to
install, but this did not actually mount (I had to use ALT-F2 and manually
mount them as well).
4. Install software did not check freespace on the root (/mnt) disk when
selecting packages. It let me select all then ran out of space on the
root disk, then tried to copy the kernel to the root disk (and, of course
failed as it was full). I had to reformat (mke2fs) and re-load a subset
of the packages.
5. Even after remaking the kernel several times, it still does not see my
NE2000 (and I get no status messages on the ethernet connection at boot
time or via dmesg). Ifconfig does not see eth0.

Questions:
1. What should I update (RPM's, and other packages)?
2. Should I use a more recent kernel (I have 1.99.9 on a P5/120). If so,
are there any tricks to using a stock kernel "out of the box"?
3. Any ideas on the network card (without it, my box is kind of useless to me
as it is in the lab and I hope to use it as a compute server)

Thanks
