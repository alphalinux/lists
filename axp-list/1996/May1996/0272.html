> Harald> yesterday I run a "virtex" binary from a DEC Unix machine on
> Harald> Linux

> Harald> GCC binary : 37 seconds
> Harald> DEC-cc & shared : 22 seconds
>
> You have to be a little careful with making conclusions here. First,
> the DEC libraries are generally well optimized. Libc (and dare I say
> glibc...) has gotten a lot better, but if TeX happens to use a
> particular function that hasn't been optimized for the Alpha yet, it
> may easily explain the difference.

I've tried to get some more information what's the difference between
those binaries (first idea was to link DEC-CC *.o files with Linux libs
or vice versa but this failed) so I rebuild virtex from the same source
tree using both gcc and DEC cc.
to my surprise I wasn't able to build a "slow" linux binary!
both the Linux/gcc and DEC/cc binaries needed ~22-23 sec for the texbook
using

-rw-r--r-- 1 root root 1801432 Apr 2 23:39 gccaxp-2.7.1-3.axp.rpm
-rw-r--r-- 1 root root 519185 Apr 9 21:34 libcaxp-0.43-1.axp.rpm
and
-rw-r--r-- 1 root root 10355690 Apr 9 22:36 tetex-0.3.3-10.src.rpm

I've extracted the virtex binary (and plain.fmt) from both Red Hat
AXP 2.1 and 3.0.3 CDs

-rwxr-xr-x 1 root users 622592 Dec 20 22:20 virtex-2.1*
-rwxr-xr-x 1 root root 630784 Apr 10 18:38 virtex-3.0.3*

and both are slow (~37 secs). does anyone know how this binary was built
and which libc version was used ?
