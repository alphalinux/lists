>I used the libcaxp-0.43-1.axp.rpm from RedHat 3.0.3 so it's includes all

Ah, but did you make ther same patches as Erik for the BUS_BASE stuff in
os-support/linux/lnx_video.c and os-support/shared/bios_devmem.c???

The LIBC not only offers inb/outb for the appropriate CPU/platform, but
a new function, "_bus_base()", which allows the xf86MapVidMem() routine
to pass the appropriate bus base address for mapping the frame buffers.
The patches are truly trivial, but very necessary for correct mapping to
take place...

As for JENSEN, I believe that the inb/outb's are as they should be, but I
forget whether Erik came up with a solution for the "_bus_base()", since
JENSEN doesn't have DENSE memory space, only SPARSE. We talked about having
"_bus_base()" return 0 (zero) for JENSEN, thus flagging what platform was
actually there, and allowing (forcing) the programmer to make the appropriate
adjustments...

>and are there other architectures out there aside from EV4/EV5/Jenson which
>do need any other special support ? it would be nice to be able to have
>real support for all platforms before the next XFree86 version is finalized...

Well, I think we've handled all the architectures we *know* about; I've tried
to get some specs from DeskStation re: their boards, but so far nothing. Their
boards don't use the DEC-based bus/memory interface chips (APECS/LCA/CIA), and
thus the correct inb/outb and bus memory mappings are, as yet, unknown...

I'll proceed looking into the SVGA server, then, and let you know...
