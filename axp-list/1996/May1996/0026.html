Harald Koenig wrote:

> the following example shows that even these 20% are by
> far not enough: here are cpu times for this fortran
> program (~20k lines) using f2c to convert it to C (~40k
> lines) and then using either gcc-2.7.1 or the DEC cc from
> DEC UNIX 3.2 or similar (I have to check this again).
> both (static) binaries are then run on my XL266 on Linux.

> gcc -O3 1556.99 secs
> cc -O3 824.56 secs

> this almost a factor of TWO:-(

Could you elaborate on the sort of Fortran code at hand here ?
There are known shortcomings in the gcc loop optimisation routines
for "Fortran type code" (whether compiled with f2c + gcc or g77)
that might be obstructing performance here. If your example does a
lot of (relatively simple floating point - add, subtract, multiply,
divide) arithmetic in nested do loops, it might help to try out the
optimisations g77-0.5.18 offers [ the optimisations can also be
applied to f2c + gcc compilations ].
