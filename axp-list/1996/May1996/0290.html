On Sat, 18 May 1996, Smeet Eday wrote:
>
> I use 1.3.100 AXP on EB164 with 128Mbyte memory
> and with 3com's 3c509 ehternet board.
>
> 1.3.100 can't recognize 3c509.

I have no idea if the 3c509 driver works at all with the alpha. Has it
ever worked for you?

The only network driver I've personally used on the alpha is the de4x5,
but some others have had success reports (ne2000 clones at least).

> it says "ALCOR machine check" * 35 times,

This is the PCI probes, I think - it should be totally harmless. You can
just uncomment the message if it bothers you.

> and
> "fcntl_setlk() called by process 71 with broken flock() emulation"

This happens due to the current Linux/alpha libraries. Again, it's
harmless, and will print out a maximum of five times, then the kernel
will shut up about it.

> and
> "kernel: analigned trap at ffffffc0000400868: fffffc00018381f6 28 3"
> and so on.

One more harmless debugging message. These tend to happen with msdos
filesystems and/or some cases with sendmail (which gives unaligned
arguments to the kernel in some cases). The kernel fixes up the unaligned
accesses, so things still work correctly. It prints out the messages to
tell the user it did this, because the trap overhead of a unaligned fault
is noticeable if it happens very often.

In short, you shouldn't be worried about this unless it happens more than
a hundred times a second ;-)
