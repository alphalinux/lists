Howdy!

Just got a new hd, and I didn't want to wait any longer on the ftprelease of
redhat-3.0.3/axp, so I took the plunge and decided to install the devel
tree which I hoped would work fine, and be close to the the 3.0.3 release.

Everything worked almost perfectly, but some small glitches..

* There was a couple of perl files missing on the "base-set", which made
it impossible to install. It wasn't so hard to find the files on my
old redhat install though. (I wanted to do a complete new install, instead
of upgrade, because I had made some major changes to my system and wanted
to see how the new release looked like...)

* All the packages had a size of "0.0Mb" during the install, but maybe that
was an error that I manage to create when I installed the perlfiles
from the old install?

* localtime link is wrong, there is an ".." missing in the relative path,
which makes the link go to ../../etc/localtime (which evaluates to
/usr/etc/localtime), instead of the desired ../../../etc/localtime, which
becomes /etc/localtime, which are the desired one.

* And why use relative paths anyways? I first installed all /usr on its own
partition, but noticed that after a complete install, the X11 dir was a bit
big, so I decided to move that to its own partition, but found out now that
the /usr/lib/X11 link pointed to nowhere..

* The most important one, insmod doesn't work at all. it segv's constantly
with all kernels I compiled (91,99,pre2.0.3), like this:

> insmod nfs.o
insmod: memory violation at pc=12000177c rp=120001770 (bad address = 00000008)
Segmentation fault (core dumped)

Other than that, great work!

/Mikael
