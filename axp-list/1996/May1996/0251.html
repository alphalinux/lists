> ~3 seconds and on my i486 "gcc -O2" takes ~9 seconds.

There is a bit of over-aggressiveness in the combiner as it tries
to simplify the entire function as a set of nested if-then-else
expressions to see if the return value is constant.

Unfortunately, this creates an expression tree that leads to
exponential run-times later on.

I would suspect this nastiness to be triggered on any target that
supports conditional moves. Which explains why the problem does
not show up on the i386 target.

Though this is not a particularly good solution, it will let you
get on with your work while a more permanent fix is explored.
Expect a bit of fuzz on the patch.

r~
