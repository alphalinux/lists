With libc 0.43, I've found these two lines in /usr/include/sys/utsname.h:

#ifndef __linux__
#include <linux/utsname.h>

That looks like a typo to me, since there is no domainname
element except in the new_utsname in <linux/utsname.h>, and
especially since if __linux__ isn't defined, it doesn't make
sense for <linux/*> to exist.

If it's not a typo, there should probably be a comment there explaining
why not.

If I'm just behind on libraries, please tell me. I'm just using a
machine installed by someone else, and haven't been following library
versions recently because I haven't had access to an alpha until
recently...
