Whoo-kay... we've got Linux up and creaking on a early-prototype Tadpole
laptop, and now I'm trying to get PCMCIA support working on it. Did
this before on the UDB, so I figured it would be un piece de gateu,
n'est-ce pas?

After squashing a buggie in cardmgr I finally got the thing compiled,
installed, and starting to come up. However, when I try to install
anything "insmod" seg-faults. Upon closer examination, what is happening
is that insmod's call to "bfd_make_empty_symbol()" is returning NULL.

The funny thing is, when I try to trace this in gdb, the call to
"bfd_make_empty_symbol()" dumps me into "_bfd_ecoff_get_symtab()".
I thought it would map to "_bfd_ecoff_make_empty_symbol()"!

Now I *could* track this down all by my lonesome, but if there are any
bfd experts out there with ready answers I'd like to hear them!

The funny thing is, this all worked a few weeks ago... back when I
was developing this on 1.3.89. I'm now doing pcmcia-cs-2.8.13 on
1.3.98, using Red Hat 3.0.3. Any ideas?
