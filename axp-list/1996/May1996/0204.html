Hi folks!
Since kernel V2.0 is coming up I think it's time to talk about my old Jensen
problem again. It should be solved at least for final V2.0.
The problem is that I can access ttyS0 as usual, i.e. I can do
stty 19200 < /dev/ttyS0
and an
echo ATZ > /dev/ttyS0
and everything is fine.
If I do the same thing with ttyS1 the machine crashes sompletely!!!
Some hundred messages pass the screen too fast to be remebered and finally
the kernel dies with something like
unable to handle kernel paging request at .....
For me this seems like a kernel bug. Linus mentioned some time ago that the
Jensen's IO subsystem is some kind of strange. May that be the reason for
this complete failure?
Everything else seems to be fine :) (at least for me)
CU
nils
