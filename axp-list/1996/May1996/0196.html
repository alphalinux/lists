> I'm trying to figure out if my Aspen Timberline EV5 300mhz system is a
> "PC164" or an "EB164". I was told by someone at Aspen that I should
> be able to install redhat on my machine. All efforts to get milo
> working have failed. I was told by someone else that this machine is
> a PC164. The obvious dilemma here is finding out who is right....
> I would appreciate it if someone could shed light on the situation.

The Aspen Timberline is Aspen's own board design. It's essentially an
EB164 with the addition of a PCI-PCI bridge to give it the seven PCI
slots. It has had problems running MILO and Linux. I haven't gotten
my hands on one yet to debug any of this, but you *might* try getting
the very latest MILO from ftp.digital.com:/pub/DEC/Linux-Alpha/Miniloader
and trying again.

The reason I say this is that we solved a similar problem with the
Aspen Telluride. This is an EB64+ with the PCI-PCI bridge.
