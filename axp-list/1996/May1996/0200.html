} Linux 1.3.94 on my standard UDB box displays many of the
} following error lines when I invoke 'shutdown -t 3 -r now' and/or
} do a C-ALT-DEL:
}
} INIT: error reading initrequest

There were changes to named pipes back around 1.3.90 which broke
SysVinit.
Version 2.60 works OK on intel, and there is an intel rpm in contrib
on the redhat ftp site. Assuming there is a source rpm, try getting
that, seeing if there are any alpha specific patches in the supplied
redhat source rpm which need applying, and rebuild for the AXP.

} Also, my 'ps' command outputs a strange looking TTY column:
}
} PID TTY STAT TIME COMMAND
} 152 pQ8 1 N 0:00 /sbin/getty tty3 VC linux
} 153 ps7 1 0:02 -bash
} 154 v38 1 0:01 -bash

Thats a kernel change to the format of a /proc file. There is a
patch to procps to fix it.

The best place to look for this sort of information (assuming you
don't want to read 100 messages or more a day on the kernel list), is
"Software Victims of the 1.3 Kernel Development" at:-
http://www2.nether.net/~jared/victim.html

Nigel.
