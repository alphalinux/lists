Hi,
there seems to be a problem with XServer for TGA on UDB which could not
resolve yet. Short description:
1) I start the Server via xdm from /etc/rc.d/rc.local
2) the login window is displayed and users can log into the system
3) ending the session by quitting fvwm (this is the only window manager I've
tried so far) seems to reset the screen to some mode i.e. the screen
is blacked out and a cursor block appears at the upper left corner - then
xdm restarts but the text cursor stays on the (X-style) screen.
4) if the next session is ended the system seem to hang with black screen
and a blinking cursor block.

Additional:
It is still possible in this situation to log into the system form the
network using XDMCP

If X ist started from a tty logged in as root (using startx) then quitting
fvwm displays a message on the teletype saying that e.g a xterm process
has been stopped because the XServer terminated and everything goes fine.
Is it this type of message that corrupts the screen? Does anyone know a
work-around or fix connected to xdm?
