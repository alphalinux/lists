> I would like to be able to cross compile the kernel and milo from
> Digital Unix (Since I don't have linux yet). I was told that there is
> supposedly a suite of sources or some such which are configured to get
> this accomplished... If that's not the case, are there any clear
> concise instructions that will tell me what software I need to get and
> what options I need to compile it with in order for me to correctly
> compile the kernel and milo under Digital Unix 3.2c? Thanks for your
> time.

Lessee... (how I did it..)

From RedHat AXP development sources, pick:
- binutils
- gccaxp

You propably need to install some of the RPM tools too to
get those packages open -- rpm2cpio SHOULD be sufficient.
There is a bit of manual job to get compilable binary from
the base source + patches, though...

Then:

./configure --target=alpha-linux --prefix=.... --with-gas

About same for both the binutils, and gccaxp.

Then have a path-prefix to access these linux-targetted
binaries takeing preference over possible other ones.

When configuring kernel, and MILO, define

CROSS_COMPILE=alpha-linux-

for the use of the cross-compile tools.

You need at first to compile the kernel, then you
can proceed with MILO.
