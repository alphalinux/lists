>> This seems the "standard" value. It's the same value that glibc
>> uses. I don't think there is anything in POSIX requires CLK_TCK
>> to equal HZ. And since OSF/1 uses 60 as well, it's probably
>> better to stick with that value.
>>
>> IMHO: avoid times() if you can...

Harald> but why is times() no real system call to sys_times() in
Harald> linux/kernel/sys.c ?

If memory serves, that system call wasn't enabled for the Alpha until
work on glibc started. OSF/1 doesn't have that syscall so it probably
was felt that Linux/Alpha wouldn't need it either (which is true,
strictly speaking).

Here is the situation with glibc: it uses the real system call, but it
currently has CLK_TCK defined as 60. This is wrong so I fixed it to
use HZ instead. I hope the patch will show in the regular glibc
sources soon.

So, CLK_TCK on the Alpha will be 1024, not 60 as under OSF/1. But
that is OK, since OSF/1 doesn't use the times syscall to implement
times().

As I said: avoid times if you can... :-)
