dvips (dvipsk 5.58f) from tetex-dvips-0.3.3-10.axp.rpm has a nasty bug
on alpha: the options "-p" and "-l" to specify page limits don't work:

# dvips xf86-notes -o /dev/null
This is dvipsk 5.58f Copyright 1986, 1994 Radical Eye Software
' TeX output 1996.05.20:2246' -> /dev/null
<texc.pro>. [0] [2] [2] [3] [3]
# dvips -p 3 xf86-notes -o /dev/null
This is dvipsk 5.58f Copyright 1986, 1994 Radical Eye Software
' TeX output 1996.05.20:2246' -> /dev/null
<texc.pro>. [0] [2] [2] [3] [3]
# dvips -p =3 xf86-notes -o /dev/null
This is dvipsk 5.58f Copyright 1986, 1994 Radical Eye Software
' TeX output 1996.05.20:2246' -> /dev/null
dvips: ! End of document before first specified page
