On Tue, 21 May 1996, Bryan W. Headley wrote:

[deleted stuff about commercial apps]

> Also, let's think this through. If I wanted commercial apps on AXP, and were
> a Linux user, wouldn't I actually be better served buying Windows NT and going
> that route? I'm talking straight word processors, spreadsheets, etc.

*yikes!* Here's where I have a quite different attitude/approach. We run
several commercial OSs (mostly UNICES but some vaxen) here, and we
consider vendor supported versions of Linux to be commercial. In point
of fact, I have started a push to migrate all of our 2nd line servers and
some front line servers over to RedHat Linux 3.0.3 (Caldera Network Desktop)
and 2.1 AXP.
I(we) support Linux not because it's "free" (we don't run any
downloaded versions - only what we buy with support from a vendor) but
because of the spirit in which it was created and in which it continues
to be developed. I have made several changes to the kernel (such as
putting in a cryptographic file system with file/dir/inode level
encryption and challenge/pwd) and like being able to "steer" and add to
the continual evolution of this great OS.
However, we have need to run applications that are not currently
available free for download nor would probably ever be free. With the
Internet Office Suite from Caldera, you get your commercial spreadsheets,
word processors, etc, and I find working in a Unix or Unix-esque
environment much more enjoyable (and productive) than jumping on an NT or
other windows machine.
Linux ain't just for the `hackers and slackers' anymore. It's being
found in more and more production environments in roles such as primary
Internet Servers to Frame Relay routers to personal workstations. It's
gotten more respect and is being seen as a more viable alternative as
time progresses to other more costly OSs - not because it's
free/cheaper, but because of the ability to change things if you don't
like them, widely supported native applications (AXP is getting there,
but would still like to see OSF/COFF support just because I have >5 DECs
running 3.2C - 4.0 and would like to see some compatibility between the
twain), and the fact that you can reach almost at any time someone that
can help you with a problem you've run into with Linux.
I certainly want to see Linux continue in the spirit in which it was
created, but "free" does not, IMHO, mean the OS, it's applications, and
it's documentation must always remain available without cost (it *always*
costs somebody something) and anyone who puts alot of sweat into an
application, or prints up documentation and binds it nicely, etc and
wants to see a return so that they can, say, feed their kids or put food
in their belly, then I fully support it. If somone put their
documentation up in HTML/SGML format or included it in a package that I
often use, I say good. But if they also offer it nicely bound and printed,
then I'd gladly pay $50 bucks for it - especially if I knew I was helping out
someone who was struggling and was trying to contribute something to our
community. My primary concern is that the knowledge of *how* to code
commercial grade applications on a particular platform doesn't stay only
with those that can afford to pay to know. So far, DEC has been very
good to us in this respect.

> Actually, I'd like some books on Alpha assembler. Where do you send off for
> this type of literature?

I bought the entire documentation package from our vendor and it included
lots o' GREAT books ranging from how to write a well behaved device
driver to network administration to API programming.
