On Fri, 17 May 1996, Harald Koenig wrote:
>
> running DEC Unix images using shared libs shows a problem with argv[0].
> this is important for "virtex" which tries to open `argv[0]`.fmt as
> TeX format file.

Yes, I've seen this problem. I think it used to work corretly at some
point, because I used a lot of OSF/1 shared binaries in the early days of
Linux/alpha. However, I think it broke subtly when the loader was
rewritten. I don't use OSF/1 binaries any more, and most importantly I
don't have a OSF/1 system that I can use easily for testing, so I'd
really prefer it if somebody else (hint hint) tried to look into this.

I might as well warn you that all the OSF/1 binary compatibility stuff
has been gotten by trial and error, and especially the stack layout for
the /sbin/loader call is very much empirical. I don't even know what it's
supposed to look like, I only know that the linux loader works (I got the
stack layout by writing a dummy /sbin/loader and looking what OSF/1 put
on the stack by using execve_with_loader() or whatver the OSF/1 system
call is called that allows you to supply your own loader binary path)

Linus
