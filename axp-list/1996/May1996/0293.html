Ok. I took a look at the 3c509 driver, and it _looks_ ok. It only uses
programmed IO, no shared memory as far as I can tell, and as such it
should work on the alpha without modifications. Does anybody on the alpha
list know if somebody else has ever tried this? Maybe there is just some
trick to it..

One thing that has bitten the 3c509 even on x86 is that linux doesn't
necessarily like the plug-and-pray features, so people usually need to
disable those with the 3c509 setup program in order to get Linux to
recognize the card. On the alpha, I don't know how that could be done.

You can also build the 3c509 as a module (or give a bootup kernel command
line), and specify which IO address and interrupt it should use if you know
those (do they show up in the hardware info given by the ARC console, for
example?). That _might_ work, if the problem is detection rather than the
actual use of the driver.
