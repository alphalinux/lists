Hello all!

As regular reader may know... I've been having serious trouble with the
keyboard on my Jensen and I've come up with the following idea for a
workaround...

Since to all intents and purposes I can't work _at_ the jensen how about
setting it up as a 'displayless' server? I mean after all X is a
cient/server, network-based windowing system. The 'topology' would be as
follows.

Linux on the Alpha set up as a server. No keyboard/Monitor etc. Just a
VT220 off the serial port for maintanence and booting.

My current 486DX/33 with a network card set up as a X-terminal. No
disks/CD-ROMS etc. All of these are in the Alpha.

Allow the 486 to boot off the alpha somehow. Options Linux or
Windows 3.1 (for those things like CorelDRAW....). Filesystems on the
Alpha.

Since I'd be displaying on the 'X-Terminal', I'd be free to boot the right
X-Server for my ET4000 card skipping all the AXP X-server/chipset hassle.

Possibilities to put X-terminals according to this scheme all over the
house seem only limited to cashflow ;-)

Questions:

1. Is this booting 'hack' possible?

2. How much memory do the machines need in this config.

3. What sort of networkcard would I have stick into the 486 to get good
performance?

All input is greatly appreciated!
