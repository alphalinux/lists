> I'm getting an odd error message when booting my UDB which I have been
> receiving since I built the 2.0.24 kernel.
>
> When the system starts the syslogd daemon, I get an flock() emulation
> error message.

That is due to an old libc and a change in preferred system calls
to do locking. It is a warning, nothing more. Everything still
works. (Though I really thought this warning went in about 1.3.8x
or before; what kernel had you been using?)

> I noticed that when I compiled the kernel I said "NO" to
> mandatory locking. Is this related?

No. The solution set contains

-- Upgrade the rest of your system to ELF.
-- Update the ECOFF libc, then recompile syslogd.
-- Comment out the message in the kernel.
