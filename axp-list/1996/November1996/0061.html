>
> When I try to do "make menuconfig" or "make xconfig" or "make modules"
> or almost anything.... I get a message that says:
>
> no display name and no $DISPLAY environment variable
> make: *** [xconfig] Error 1

I'm sure you get this only from make xconfig.
make xconfig requires that you start it from X-Window. If you can't get X
running use make config.
Answer all questions in the right way. (This is the hardest part).
Dont' use modules, please. First start with a kernel with all driver you
need in it.

Then do a

make dep clean boot

If the kernel compiles clean do following:

cd /
cp vmlinux.gz vmlinux.old.gz # its for save, so you can boot from
# it later by changing the kernelname
# for milo
cp /usr/src/linux/arch/alpha/boot/vmlinux.gz /

Thats it. Reboot and look.
