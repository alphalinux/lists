>OK, one more nitpicky question - suppose you have
>a DEC site-license for the compilers, like at a university.
>Is it OK to use the Digital-compiled binaries
>on the linux machine under those circumstances?

I understand that this is complicated. After sixteen years of explaining
this, I am just getting a little tired of it. I really don't like it any
more than you do, which is ONE of the reasons I like Linux so much. I will
try to make this as plain as humanly possible:

If there is licensed code operating on a non-licensed system,
that is illegal.

So, if you have a site license for the DEC compilers, you would normally be
able to use those compilers on a Linux/Alpha system EXCEPT that the libraries
that are used to run those compilers are not site licensed (as people run to
check their CSLG license forms....). CSLG states that each DUNIX system has
to have an initial license (i.e. you are paying those royalties, folks), so
you can not run the compilers on Alpha Linux, UNLESS you link them with
royalty-free libraries. If you DID link them against royalty-free libraries,
then you could run the compilers on any system, as long as it was covered by
the campus-wide license.

What about the output of the compilers? Sure, you can take the objects
generated by the compilers (no matter WHAT license you have or don't have)
and link them against royalty-free libraries and run them.

md

P.S. If you think this is nit-picky, try explaining GPL to the unitiated - md
