> >>> Dreyer said:
> >
> > I have Number nine Graphixc cards (Model 332) witch has a S3 Virge Chip
> > on Board.
> >
> > The Chip has also the letters H1C3BB and 86C325 and 9612 BA0677.1 on his
> > body.
> >
> > Can anybody tell em witch Xserver to take. In my understanding the Virge
> > Chip is a new version of the Trio64+.

it's not compatible to Trio64 (well, S3 claims that it's pin-compatible;
don't know about that but the graphics engine etc. is *completely* different)

> The new release of XFree86, ie 3.2, is available; there's a server for the
> S3 ViRGE chip included, which has been ported to the Alpha.

it wasn't really "ported" but only complied. I didn't have the time
to run or test it on AXP before XFree86 3.2 was finalized and guess what,
it doesn't work at all on AXP, sorry :-(

> Last I heard,
> however, was that the released server did not work well, so unless the
> problems have been fixed and a new binary generated, I'd not count on the
> one that's available at ftp.xfree86.org (and some mirrors).
>
> Harald, sorry if this is inaccurate...

absolutely correct. right now I have a version of the ViRGE server
running on AXP and there are some minor problems left (text fonts may get
corrupted; and switching to other virtual consoles and back to X11
still might hang the server; hope to find/fix this RSN).
for "X-only" operation it looks better (but I didn't run any tests yet).

hope there will be a new binary within some time (1-2 weeks?)
but I first have to ask how we handle such fixes for XFree86
(I'd guess there will be some "patchlevel N" server binaries..)
