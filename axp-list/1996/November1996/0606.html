Robin,

The basic steps to do this would be :

1. Close down as much as possible to allow you to turn off the
swapping. I believe that rebooting in single user will also have
it running at minimum usage, but could be wrong.

The alternative is to create a reasonably large swap FILE in
your main filesystem somewhere, and then to start this.

2. turn off the swap :

swapoff /dev/sda2 (this is assuming that it is sda2)

3. Use fdisk to delete the swap partition, then to replace it
with a smaller one, and the DOS partition. I would recommend
keeping the swap partition as the first of these, that way the
/etc/fstab is still correct; but that is easy to change if needed.

4. Use mkswap on the resized swap partition, and run swapon

5. Use either mkdosfs or dd depending on preference to create
the DOS section.

This is all left vague to save me time (sorry) but do read the
man pages before you try this, it is always a good idea.
