Greetings, I have an eb164 running RedHat Linux, and I have been plagued with
problems with this combination from the first installation (when it crashed
several times before I got a clean install of 3.0.3)

the eb164 has 128 MB of 60 ns RAM, 2 4GB Quantum Atlas drives, a symbios ncr
53c810 scsi controller, and a Dec 21140 based ethernet card -- of course, a
trio64 s3 stealth 64 (all PCI). It also has the SRM console.
I tested the RAM myself, it all tests fine at 50ns on a RAM tester, of course,
they are 36 bit parity simms.
The disks are labelled with BSD-style labelling.

occaisionally, the SRM console refuses to initialize, it keeps showing the
video BIOS screen, going black, then repeating. If I power down and unplug the
keyboard, the SRM will boot to the serial console just fine. As if by magic,
this problem can clear up for days, then manifest iteself again.

The longest uptime I have had with RedHat was 1 week.

the clock no longer sets upon boot-up. (at one point, this worked fine)

X windows no longer works (Ditto...)

Here is the error message for the clock:

Setting clock: clock: unable to get I/O port access
Fri Nov 4 11:49:56 CST 2016

For X-Windows:

giving up.

xinit: No such process (errno 3): Server error.

...or the whole computer will hopelessly crash. Also, I see oopses like this:

that I am dealing with an invalid address. ie:

(gdb) disassemble 0xfffffc0000312c18
No function contains specified address.

interestingly, xdm will work fine on OTHER machines, there is a Sparc IPC that
has used xdm on this eb164 as its display manager.

Okay, other than having a /var/log/messages full of errors similar to the above
example, this eb164 will stay up for a while, at least, but I suspect that

a) redhat Linux is awful on an Alpha
or
b) this eb164 has some hardware problems.

I would appreciate any opinions on what might be causing these problems.
If they are a standard requirement with using RedHat - AXP, then I am heartily
dissapointed. If I have a hardware problem, I'd really like to know, so that I
can get a motherboard swap.
