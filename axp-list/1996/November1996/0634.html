> 1. I've loaded MILO from
> ftp://gatekeeper.dec.com/pub/Digital/Linux-Alpha/Miniloader/images/pc164/
> and it behaves as follows:
>
> MILO> O> O> O> O> O> O> O> O> ...etc... for more than 90 "O>" repeats
>
> When I type a command, such as "help", I get:
>
> MILO: unknown command, try typing Help
> MILO> O> O> O> O> O> O> O> ...etc...
>
> Has anyone seen this, and what is the solution?
>

Yes they have and what's more it's described in the MILO HOWTO. It's
COM1 receiving input from someplace. By default MILO uses COM1 so that (a) headless
systems can run and (b) problems can be found.

>
> 2. I see that this is a MILO for kernal 2.0.12.
> If the above problem can be solved, can this MILO
> be used to boot and install RedHat Linux 4.0
> as it now stands? Is there a MILO at ftp.redhat.com
> that might be more appropriate?

Yes, this MILO should work. MILO executables are not particularly related to
the Linux kernel versions except to build. Do you rebuild 'ls' everytime you
upgrade your kernel?
