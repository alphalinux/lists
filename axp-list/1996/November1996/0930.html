I just tried to access a remote SCSI tape from Linux/AXP for the first time
using

# rpm -qf /bin/tar
tar-1.11.8-8
# export TAPE=turtle:/dev/tape
# rpm -qf /bin/mt
mt-st-0.4-2
# mt stat
turtle:/dev/tape: No such file or directory
# tar tvRi
tar: memory violation at pc=155556c96e0 rp=15555703c04 (bad address = 00000000)
Segmentation fault (core dumped)

so "mt" doesn't support rmt at all and tar dumps core :-(
"rdump" and "rrestore" work fine (but don't use environment variable "TAPE")

here is the tail of strace output (10.2.170.191 is "turtle"):
