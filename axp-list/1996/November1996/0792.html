OK. I see there is now a MILO for the alcor (AlphaStation 500/600) family
at
ftp://gatekeeper.dec.com/pub/Digital/Linux-Alpha/Miniloader/test-images/alcor/

I downloaded the milo.dd, and sure 'nuf, >>>boot sda3 popped me straight
into the MILO> prompt! So I pillaged a SCSI disk from a perfectly
functional Linux AlphaStation 200 4/166 (Mustang) and installed it into this
AlphaStation 600 5/266 (Alcor). Mistakenly assuming that once I got past
MILO Alpha Linux is machine independent, I did:

MILO> boot sda3:vmlinux.gz root=/dev/sda3
and got:
...
partition check sda1 sda2 sda3
Boot line is boot device=sda3 bootfile=vmlinux.gz root=/dev/sda3

Bootstrap complete, kernel @ 0x31000, sp@0x304000
... turning on virtual addressing and jumping to the linux kernel
-------------------------------------------------------------------

and then the machine just hangs :(

I suppose that the avanti kernel does not work in an alcor machine, right?
Does anybody have an alcor kernel?

I didn't see one in:
ftp://gatekeeper.dec.com/pub/Digital/Linux-Alpha/Miniloader/kernels/
