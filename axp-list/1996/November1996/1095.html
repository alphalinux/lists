It could be an old version of MILO that is to blame. There were versions
that were very picky about floppies; only new ones would do.

A Side Note:

The filesystems supported by MILO are ported code rather than the real
filesystem code from Linux. I suppose that a slightly better approach
would be to implement the VFS layer in MILO and maybe parts of the buffer
cache. That way, any filesystem supported by Linux would be supported by
MILO. Unfortunately, MILO would also be bigger.
