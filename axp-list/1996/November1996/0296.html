D. Steven Daniel wrote:
>
> On Tue, 5 Nov 1996, Jay A Estabrook wrote:
>
> > At 04:06 PM 11/5/96 -0500, you wrote:
> > >According to D. Steven Daniel:
> > >
> > >> What do the jumper settings on the UDB motherboard really control?
> > >
> > [We are talking about the rectangular group of 4 jumpers on the UDB MB]
> >
> > They are used for the same thing as the DIP switches on a TGA PCI card;
> > they indicate to the *SRM* console (and subsequently, to the DU TGA server)
> > what resolution/refresh rate you wish to run console (and X server) at.
> >
> > They are ignored by ARC console (AlphaBIOS as well, I think), and MILO.
>
> Are they ignored by the Linux console mode? Can the linux console mode be
> changed similar to using SVGAtextmode under Linux-Intel?

Yes, probably, but you have to hack a "litle bit" in tga.c in the
drivers/char
in the kernel source. 8-)
I was attempting to get the local (iso1) character set working for the
TGA.
No problem to get it to work with static char tables, but to support
dynamic
loading of char tables (ala setfont) is a litle bit more complicated.
But I will start the dynamic support next. 8-) So litle hacking is all
you need 8-). I have not looked at the resolution code so much but
there
are some variables in tga.c that have direct correspondance to
resolution.
So the actual support of dynamic changin of resolution might not be so
difficult to implement.
Don't forget that TGA is not VGA and not even close 8-).
