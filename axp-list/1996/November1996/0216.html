It was working for a while and then it stopped again. I learned a few
more things.

(By the way, you'all can jump in anytime when you are possessed of an
inspired solution.)

- If I initialize lo before eth0, it seems to always work.
- What is hanging everything up is a DNS query on
127.0.0.1
which appears to be a mistaken attempt to resolve the explicit IP
address into ... another IP address.
- The verison of ifconfig on x86 does not make this query. I checked
the source of this one and it is

NetKit-A-0.10/net-tools-1.2.0/ifconfig.c

It makes a different query on

1.0.0.127.in-addr.arpa
- I seem to be able to work around the prolem by renaming the two
ifcfg files ifcfg-0lo and ifcfg-1eth0.
