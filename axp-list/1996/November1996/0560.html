Has anybody seen the patched ncr538xx driver working on an alpha with 2.0.25 ?
At my avanti box, the driver issues the following at boot:

ncr53c8xx: at PCI bus 0, device 6, function 0
ncr53c8xx: unit=0 chip=810 rev=0x1 base=0x1802000, io_port=0xb800, irq=11
ncr53c810-0: using memory mapped IO at virtual address 0xfffffe0000082000
Unable to handle kernel paging request at virtual address ffffbc0201040060
swapper(1): Oops 0

and the machine stops.

this looks like the driver is not working at all, i think.

the driver was compiled with the (default) settings:

CONFIG_SCSI_NCR53C8XX=y
# CONFIG_SCSI_NCR53C8XX_TAGGED_QUEUE is not set
# CONFIG_SCSI_NCR53C8XX_IOMAPPED is not set
CONFIG_SCSI_NCR53C8XX_MAX_TAGS=4
CONFIG_SCSI_NCR53C8XX_SYNC=5
# CONFIG_SCSI_NCR53C8XX_NO_DISCONNECT is not set

The same kernel works with the old version of the driver, boot message:

ncr53c8xx : at PCI bus 0, device 6, function 0
ncr_attach: unit=0 chip=810 base=1802000, io_port=b800, irq=11
ncr53c810-0: initial value of SCNTL3 = 13, final = 13
ncr53c810-0: requesting shared irq 11 (dev_id=0xfffffc0009ff00a8)
ncr53c810-0: restart (scsi reset).
scsi0 : ncr53c8xx (rel 1.12c)
