	bash: /usr/bin/control-panel: No such file or directory


This typically means that the executable is not a recognized format. Since
control-panel is a shell script that calls wishx, I thought that might be the
issue. Well, it comes up at the command line, so I'm not sure.


The bug shows up on both Intel 2.1.13 and Alpha. Sadly I went from 2.1.10
straight to 2.1.13, so I don't know which kernel broke this.
