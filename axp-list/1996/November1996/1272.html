>
> I've upgraded my RH 3.0.3 to RH 4.0, but now the anonymous ftp account is
> unable to execute the "dir" command.
> The result of the command is simply empty.
> Is there any workaround to restore the old (right) situation ?
> Thanks in advance.
It is a bug. The reason is the new annoftp rpm package is dynamic linked.(
(It shd be static linked.) So the ftpd can not execute dir command since it
can not find those dynamic linking library.
You can either wait for bug-fixed version or get the annoftp package for
3.0.3 and install it.
