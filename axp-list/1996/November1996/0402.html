> 1) Can I run Linux/x86 binaries on a Linux/Alpha box?

Not yet. Probably not for a while.

> 2) Can I run Digital Unix binaries on a Linux/Alpha box?
> 2a) If so, can I use Digital Unix shared libraries? (Motif?!?)

Yes. You can run statically linked Digital Unix binaries. If you have
the shared lib, and it looks like you do, you can also run dynamically
linked binaries. I don't think the Linux/Alpha linker can deal with
the Digital Unix shared libs, so if you want to compile and link against
the shared libs, you will have to do it under Digital Unix. Of course,
with enough patience, you can probably figure out how to run the DU
compiler from under linux.

You asked for a project to work on. I have one that I would like, and that
would probably help you a lot, but its not one that intrests most people
who do not have Digital Unix and Linux running. Linux has some support for
reading UFS filesystems in it, however it can not read the Digital Unix
UFS filesystem. The aboot loader, which is used to boot linux from the
SRM console can read Digital Unix file systems, and the sources for
aboot are small and available. It would be really nice to fix up the
UFS support in the kernel so that we could mount Digital Unix file
systems. The sparclinux people have done a LOT of work on the UFS file
system, but I don't know if its in the mainstream kernel yet.
