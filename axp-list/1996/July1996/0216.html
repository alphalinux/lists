Rudolf Gabler <rug@usm.uni-muenchen.de> writes:
> 5.) after rpc.mountd starts (in my 3.0.3 redhat distr. setup)
> the system hangs for any kernel version. This is not the case with the
> 1.3.90 kernel.

I had the same problem, which was solved by editing
/etc/sysconfig/network-scripts/ifup-lo. Replace the line

route add -net ${IPADDR}

with

route add -net 127.0.0.0

I suspect that Red Hat has already put this fix into a newer version of
the initscripts RPM, but I haven't looked.
