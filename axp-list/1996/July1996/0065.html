This sounds like deja vu all over again. I think the mailing list
archives should have pointers to how to fix this. Basically, the
problem is that void * != int. The author of mgetty knows about this
problem and I hope the latest version has it fixed, though I don't
know for sure. If memory serves, the easiest fix is to change the int
field in the structure that's causing the problem to a long, and then
cast the pointers to long. And no, there are not enough braces there.
It's an array of structures, so it needs nested braces.

To summarize: gcc is doing perfectly fine (for once, at least! ;-).
It's the program that's broken.

I hope this doesn't sounds too confusing, but if you take a look at
the source, you'll quickly notice what's going on.
