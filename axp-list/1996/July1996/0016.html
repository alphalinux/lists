There is kernel profiling. Haven't tried it myself, but if it doesn't
work already, it should be straight forward to get working.

At one point I implemented kernel-level instruction tracing. It
wasn't perfect, but it was certainly good enough to capture the
execution trace all the way from the bottom half of the network system
to the point at which the packet got delivered to a socket. Together
with cycle-level execution timings and a CPU simulator (a la
ev4sched), you can get some interesting numbers out of this. It
certainly could help optimizing stuff, though more on a global level
than at the driver-level. Getting the instruction tracing to work
requires the kgdb patches and some hacking. If interested, let me
know.
