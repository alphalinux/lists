I think some of the confusion about CPU performance comes from 1) running
software that doesn't specifically test just the CPUs performance and 2)
comparing Alpha with intel using software that has been optimized for intel.
The vast difference between RISC and CISC can lead to misleading results.

Much of the signal processing software we write is highly optimized for the
Alpha and a couple of examples might be interesting. The first is rotating a
350x350 box in a frame buffer 500x500 (floats). Results are:

Pentium-120 512. ms.
Alpha 21064-300 11.3 ms.
Alpha 21164-333 4.5 ms.

If this had optimized it for intel/pentium the results would be very
different, perhaps the Alpha would only be 3-4 times faster, but that's the
point. When comparing two different architectures, the code should
be optimized for each.

The other example is a small benchmark program designed to evaluate Alpha
performance. Its a basket of 6 different computationally intensive
loops:

Pentium-120 36.0 sec.
Alpha 21064-300 1.69 sec.
Alpha 21164-333 0.87 sec.

Allan
