> > "FAT" binary for linux (i86/axp/whatever else) like with NextStep
> > (quad-FAT binaries are quite large :)

> No. What use would they be?

The same use they'd be on NeXTstep and the Macintosh -- you have a single
binary that'll run correctly on all platforms.

One use is for distributing commercial packages, but the use I'm most
interested in is for programs I install in ~/bin that are in a home directory
NFS-mounted on both Intel and Alpha Linux boxes.

It's not important, but it's vaguely neat, so I expect someone will eventually
get around to doing something similar.
