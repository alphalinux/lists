On Tue, 16 Jul 1996, Robert Allen Barker wrote:

>
>
> I think some of the confusion about CPU performance comes from 1) running
> software that doesn't specifically test just the CPUs performance and 2)
> comparing Alpha with intel using software that has been optimized for intel.
> The vast difference between RISC and CISC can lead to misleading results.
>

But why does it take such a long time (about 35 minutes) compiling a
kernel on my noname box (21066/233, 256KB L2-cache)? (it's equipped
with 64Megs RAM and runs the actual RedHat distribution.)
The native 2.0 kernel on a 488/66 takes approximately the same time,
most pentium boxes do this job between 10 minutes and 25 minutes, PPro
machines below 6 minutes. If I compare the calculation power e.g. with
an (simple) dhrystone-test the axpPCI system beats the intel boxes as
well as an SGI with R4600/100. My impression of the performance working
with the noname machine is that the old 488/66 is much slower.

First I thought it has something to do with the fact that it's a more
difficult thing making code for a RISC-CPU. But recently I've build a
cross compiler for a Linux/m68k system and the difference in time for
the same 2.0 kernel (approx. with the same number of drivers) was very
small (1 minute).

I know that the noname box is compared to other available alpha-systems
a low-end machine. But why is the performance for this special job so
bad ?
