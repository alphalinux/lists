>>>>> On Sat, 13 Jul 1996 23:18:06 -0500, "J. Joseph Max Katz" <jkatz@in.net> said:

Joseph> That's weird! Try using only 2 of the 16M SIMMs at a time,
Joseph> maybe one of them is bad, and that could be throwing the
Joseph> whole machine off...

That doesn't work. The AlphaPC64 has a 128 bit memory system which you
have to upgrade 4 SIMMs at a time.

The AlphaPC64 positively needs parity SIMMs (with 36 bits of data, and
_not_ logic/computed parity). I upgraded my system a couple of months
ago and had no problems at all: just plugged in the 4 SIMMs, booted
the machine and all was fine. Except possibly for the MILO
MEMORY_SIZE environment variable, there is nothing that needs to be
done to get the new memory recognized by the system. While I bought
70ns SIMMs, the chips all ended up being 60ns in reality (seems to be
a common thing these days). I'm certain that the Cabriolets in our
department have 70ns chips, though, so I don't think that explains the
difference.

Actually, one thing I haven't figured: do any of the Alpha systems
take advantage of faster RAM chips? Can the speed of the chips be
autodetected? If so, how? It seems to me that a 70->60ns upgrade of
memory should give a nice speedup in memory system performance
(>10%!), but I haven't bothered to actually try to measure this
(besides, most of the memory in the Cabriolet is 70ns, so it wouldn't
be that easy to get reliable numbers; well, it's not that hard either,
but there are just too many other fun things to do...).
