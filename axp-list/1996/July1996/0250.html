I am attempting to NFS install RedHat 3.0.3/axp from my Intel linux box to
my new UDB (connected across ethernet). I ftp'ed from the redhat ftp server
the necessary files, stored them on the intel box, and got nfs going on the
intel box (the intel can nfs mount itself). I also stuck the IP address I'm
using for the udb into the named database on the intel box and tossed in a
name just in case something cared about looking it up.

After booting the UDB with the three floppy boot set, and setting up the
network parameters I can ping the udb from the intel box and the udb
responds. However, when I tell the udb to nfs mount the intel box by IP and
path I get back the following error message on the udb:

mount clntudp_create: RPC: Program not registered

My logs on the intel show absolutely nothing, even though I made sure to
load the rcp.* drivers for nfs with their respective -debug flags to dump as
much as the will to the logs.

I can also nfs mount the same path elsewhere on the intel box from the intel
box, so as far as I can tell, nfs on the intel is working.

Anyone have any ideas what might be wrong?
