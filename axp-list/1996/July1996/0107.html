Okay, I've jumpered the motherboard as suggested, and the 233MHz UDB is now
talking to the plain old VGA monitor as I needed. It came up in SRM, and I
switched it over to ARC because that's what all the documentation talks about.

So, I downloaded lots of disk images, from redhat and from DEC. I'm able to
bring up MILO, and I'm able to boot arbitrary kernel images. The only kernel
image I've found that does anything useful, though, was on one of the "noname"
disk images from redhat's site. It start's to boot, but that's as far as it
gets.

Unfortunately, it dies a horrible death upon attempting to mount the
filesystem. I've got a floppy disk with a root filesystem written to it (also
copied from the redhat site), but the system never prompted for it -- I'd
assumed it'd load it on to a ramdisk and then let me install the rest of the
system via FTP (this is what I've done with Intel-based Debian systems).

All I've got at this site is a floppy drive and a T1, no CD-ROMs. Also, this
box appears to have *no* operating system (or even filesystem or partition
table) on the internal disk, so I hope that ramdisk has a partitioning
program...

So, am I doing something wrong? Should I be passing the kernel some boot
parameters to tell it to load a floppy drive as the root filesystem? Or if I
need a different kernel image, where should I get it?

By the way, the inside of this box is gorgeous. Everything is really tiny --
it almost looks like a modified laptop design. The thing has 24M of RAM, in
the form of two *12M* SIMMs, which I'd never heard of before opening this box.
Does anyone know if SIMMs have to be added in banks of two? The thing has two
empty slots, and I'd like to eventually pump this system up beyond 24M.
