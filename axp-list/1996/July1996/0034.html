On Fri, 12 Jul 1996, Harald Koenig wrote:

> > I just look into io.h.
> > I did not understand a lot all that stuff.
> > Address ffffbc0..... seems weird (to an ignorant as I am about Alpha).
> > Are you sure that __reab() is expecting addresses that have been remapped
> > by vremap() ?
>
> no, these strange addresses are almost correct; this is the PAGE_OFFSET
> (asm-alpha/page.h) or IDENT_ADDR (asm-alpha/io.h) and part of the address
> for PCI/ISA bus mapping. from the Xserver sources I only know that the
> ISA/PCI bus is mapped to BUS_BASE / APECS_DENSE_MEM = 0xfffffc0300000000UL,
> so this really should be 0xfffffc0300000000 + 0x44000000 = 0xfffffc0344000000
> AFAIK.

I have understand something like that too. I think that this address
calculation allows to acces the video memory directly, without calling
some readx() function. However, it seems that our virtual address
0xfffffe0000042000 has been processed by __readb() or __readw().
The result is something that donnot look like a valid address.
With i386 architecture we need to remap high physical addresses
corresponding to bus addresses in the kernel virtual space in order to be
able to do memory mapped IO from the kernel.
My opinion is that Alpha architecture does not need such recipe and that
the readx() functions are not expecting addresses with IDENT_ADDR offset.

> or for kernel symbols:

> 0xfffffc0000310f98 <init>: 27bb0016 ldah gp, 1441792(t12)
>
> > Perhaps we should use directly the bus base address (44000000).
> > It is just an idea. I do not reply to the list because I am very probably
> > wrong.
>
> I've tried this too (one never knows;) but it really didn't work

Well.
IMHO, mmio seems to be unnecessary under Alpha. io functions are
implemented with memory instructions and readb() and readw() is too
large and may decrease performances against inx() (x=b,w,l) functions.

I just try to find some code in the kernel tree which try to use readb()
or readl() from bus addresses > 16 MB (our is 68 MB) and I did not found
anything.
Have readb() or readw() ever worked for addresses > 16MB?
