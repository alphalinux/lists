Donnie Barnes wrote:

Well, MILO has filesystem access. It can boot darned near anything
on the disk that's bootable. It can even access CD-ROMs. Just
keep the kernels independently named and boot them. If I recall,
the MILO syntax is something like:

MILO> boot sda2:vmlinux.gz

David A Rusling wrote:

> To set Milo up for multiple images is easy if you are using WNT ARC.
> Otherwise it is not so easy. With ARC simply define several boot
> lines, one for each kernel. Without, you'll have to do it by hand, but
> shutdown -r now will boot the last kernel that you booted. One of the
> reasons for Milo using filesystems and not boot blocks was to make it
> easy for it to boot any old images. I keep all mine in a /kernels
> sub-directory. On some systems this is a seperate partition. I keep
> up with the kernels as they come out and this works fine. Just like
> my intel system, I keep a known good one handy.

OK, so you're saying that MILO can't give you a menu like LILO can
to pick your kernel, but the same effect can be achieved by configuring
the WNT ARC to call MILO with different arguements?
