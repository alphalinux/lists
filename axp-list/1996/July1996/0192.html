> libm.so.6 => /usr/lib/libm.so.6 (0x0000015555556000)
> libc.so.6 => /lib/libc.so.6 (0x0000015555682000)
> /lib/ld.so.1 => /lib/ld.so.1 (0x0000000000000000)

Even without -g, there is some amount of debugging information generated.
This minimal amount is there so that you can link some files with -g, some
without, and still be able to debug the -g files.

Stripping the binary will get rid of the info. For a simple hello program
the unstripped size is 12865 and the stripped is 4768.
