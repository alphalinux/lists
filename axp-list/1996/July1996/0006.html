Allan, here are some more answers to your questions.

>>
>>We have been developing some image processing systems based on the EB164
but
now we can't get them and need to move to the PC164. Could you answer some
questions about that board? The EB164 we have is currently running Linux,DEC
UNIX 3.2,NT3.51, and NT4.0beta2. (happily)
>>
>>
>> 1. Aside from the PC164 milo and a new HAL layer for NT would we need
>> anything else to just swap motherboards in the system and get things
>> to work?
>>

>> You will need a new PC164 Linux kernel too. The interrupt handling
is
>> different (again). The PC164 is in the kernel, so build one is not
a
>> problem.

>>
>> 2. Some IO and memory related things in libc initially didn't work
>> when linux first went on the EB164. Will there be another breaking
>> in period for the PC164?
>>

>> No, the old EB164 I/O is the same as the PC164. Same memory map.

>> 3. Do we need a new XFree86?
>>

>> Again, no.

>> 4. Probably just me, but I could never get the svgalib in redhat to
>> work properly. I finally got the latest sources from sunsite and
>> added a couple of lines about ALCOR and got that to work. Do you
>> think we need to adjust things there again?
>>

>> No, PC164 uses Alcor II, the next generation.

>> 5. Can we use the same standard DRAMS that we are currently using with
>> the EB164, or do we need edo?
>>
Standard Page Mode DRAMS are required. EDO won't be any better & may
not work.

>> 6. The DEC distributor here says he can't find any part numbers for a
>> cache module and suspects the cache is now included with the mother
>> board. Is it, or is it cacheless, or he just can't find the part?
>>

The cache is soldered onto the board, 1MB, synchronous SRAM third
level cache.

The hardware user guide for the PC164 is now on our website as a PDF
file. URL is:
http://ftp.digital.com/pub/Digital/info/semiconductor


>> 7. Last but quite important. DEC semiconductor in Hong Kong, who is
>> tying to support us, told us that older 21164 and 21164NT were
>> actually the same chip and for our own use just use the NT chip
>> for UNIX. If we get the PC164 and the NT version of the 21164A
>> will we be able to run UNIX? I know that's possibly sensitive but
>> the person we get the info from in Hong Kong will be away for a
>> while nad we are trying to make some purchasing decisions.
>>

You won't be licenced to run Unix on the WNT version Support would
also be difficult. There could well be technical reasons why Digital
Unix won't work due to some subtle internal register things.

>> Sorry for so many questions, any help would be useful. If you think
>> these are more appropriate for axp-list either you or I could send
it >>on there.
>>(except for question 7, I would think).
>>
>>Again, many thanks,
>>
