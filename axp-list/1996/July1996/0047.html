> > I finally after many trials got Linux working on my Multia.
> > However the system is swamped with the messages:
> > lca: machine check
> > reason: correctable ECC error (short frame, retryable)
> > This message never appeared when the Multia was running Windows NT
> > Do I have a hardware memory problem??

These look similar to messages one gets using older revs of MILO and the
kernels, like from BLADE, as opposed to the latest CDROM distributions.
The most up-to-date MILO (and associated linload.exe) can be had from
gatekeeper.dec.com:pub/DEC/Linux-Alpha/Miniloader directories, and an UDB
kernel can be got from ftp.redhat.com in the 3.0.3AXP tree, I think.

Try the latest of each that you can find, and see if it still happens.
And not that it *is* important to have the latest linload.exe to go with
the latest MILOs.

> I have the same problem with my Multia "Easy Web Server," (VX40E-F3)
> though I also get a line that says something like "b-cache error."
> Someone in France reported a similar problem.
>
> The service guy replaced everything _but_ the SIMMs (it looked like a
> cache heating error as it took a half hour of uptime to start with the
> mess); supposedly he'll have them in this week. Still haven't gotten the
> thing to run usably for longer than a half hour, and I'm seriously
> considering returning the thing (which I got for practically nothing) &amp;
> buying a Pentium if I get no joy with the SIMMs.
>
> I'll let interested parties know if it was _not_ the SIMMs, as that might
> imply some incompatability with certain Multias.

Um, did you *add* some SIMMs to the machine? If so, were they "by-36" type,
or the "by-32" type (as in 2Mx36 or 4Mx32, respectively)? Most of our Alphas
really require the "by-36" kind of SIMM, and won't work well or at all with
the "by-32" type.
