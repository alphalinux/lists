> So I decided to try this first. So the XServer starts without an error
> and hangs there, doing nothing (output appended).
> So I guess rebuilding the libc and rebuilding the XServer won't help.
> (BTW, I tried to symlink all valid names)

> (--) Mach64: card type: ISA

At least it knows its ISA...

> (--) Mach64: Aperture mapped to 0x04000000

Now *this* REALLY worries me, as an ISA bus only addresses to 16Mb,
ie 0x01000000, so it may *not* be able to touch the frame buffer memory
directly with that mapping.

Try specifying an explicit "MemBase" option in XF86Config, say:

MemBase 0x00c00000

and see if that works...

> and there it hangs... screens black. Actually, the server seems be running,
> since i can start applications that seem to connect to it. It's just that
> the screen stays black.

That's why the following line worries me:

> (--) Mach64: Ramdac is Unknown (87)

Is there a separate RAMDAC on the board? If so, try setting the "Ramdac"
option in the XF86Config file to see (better check the Mach64 info in the
XFree docs for any help, first) if explicit setting does any better...

Let me know how you get on...
