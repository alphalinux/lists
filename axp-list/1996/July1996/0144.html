The "excellent ELF shard library stuff" really is more due to Eric
Youngdale's and Richard Henderson's excellent work.

But yes: ELF is a good thing. Not only do shared libraries save
disk-space, but the dlopen() etc interface allows building
applications that couldn't be built before. For example, GNU libc now
has full support for SVR4-style /etc/nsswitch.conf to select how the
various Unix databases (/etc/passwd, /etc/group, /etc/hosts, ...) get
accessed (file, DNS, NIS, etc). This is implemented by dynamically
loading the name services that are in use (currently, file and DNS are
supported). Also, perl can now dynamically load extensions and xdm
can dynamically load the "greeter library" (is there anything more
fancy than the default greeter, though? ;-) I hear some databases also
require the existence of the dlopen() interface---those should be easy
to port now. Finally, it's just wonderful that if we fix a libc bug
now, we no longer have to recompile each and every binary in response.
