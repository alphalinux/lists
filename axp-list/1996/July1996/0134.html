On Wed, 17 Jul 1996, Erik Troan wrote:
> On Wed, 17 Jul 1996, David Andrew Bushnell wrote:
>
> > I was wondering if it was possible to run Digital Unix binaries
> > under Linux for alphas. If so what do you need to do to get a program
> > to run. The reason I ask this is I tried to run denzo (a crystallography
> > program) compiled on digital unix under linux and I got the error command
> > not found. I made sure denzo was executable, in my path and the file
> > type was alpha (denzo: ECOFF alpha demand paged not stripped - version 2864.)
> > but I still got the command not found error. I am running Red Hat Linux
> > version 3.0.3 on a Multia.
>
> You need all of the shared libraries installed as well. I've never used
> Digital Unix, so I can't tell you how.

The shared libs is a start, but you need to make sure you have a
filesystem that looks like Digital UNIX binaries expect too. The error
above does seem to imply you're missing the shared libs (and the loader:
/sbin/loader), but just in case you do have a license for the libs, you
_also_ need at least a /usr/bin/sh (just link it to /bin/sh), and some
special files in /etc/ that Digital UNIX binaries tend to assume.

To get a working Digital UNIX binary environment, get the shared libs,
and then use "strace" to see what files etc the binary tries to open.
It's not hard, but there is no ready-made package for this as the shared
libs are obviously not freely available..
