Your message on: Fri, 19 Jul 1996 17:23:15 EDT
> Hi, I'm consulting for a company that has recently purchased a
>pair of EB164 based Alphas. I'm not new to linux, but I'm very new to
>Alphas (well linux on alphas - I've been working w/ Digital Unix for a
>while). My client has Redhat 3.0.3. I'd like to move to using the 2.0.X
>kernels. What is there to do beyond grabing the kernel source (which I've
>already done - 2.0.7)? I've seen mention of a axp-diffs* file. Where
>might that be found and what exactly is it's purpose?

I'm not sure on this one, but others will pipe up.

> Also, how does one go about setting up milo for multiple kernel
>images? I'm a bit nervous about screwing up the compile and being left w/
>a non-bootable machine. I'd like to make sure I have access to the old
>kernel. I've looked through the milo docs (I'll look again), but I don't
>remember any mention of that issue.

Well, MILO has filesystem access. It can boot darned near anything
on the disk that's bootable. It can even access CD-ROMs. Just
keep the kernels independently named and boot them. If I recall,
the MILO syntax is something like:

MILO> boot sda2:vmlinux.gz
