I've got a 233MHz UDB sitting in front of me, and another in a box 5' away.
The intent is to install Linux on these and set them up as servers.

Unfortunately, at this site the only monitor I've got too hook up to the thing
is an old IBM VGA monitor. It can't do SVGA resolutions at all -- it really
maxes out at 640x480.

I tried plugging it into the UDB and booting. I got a couple of test
patterns at first, and then I seemed to get the colorful snow that one gets
when one has a messed up XFree86 configuration file. I'm guessing that the
UDB is immediately trying to go to more than 640x480 resolution.

Is this what's going on, or is it more likely that the monitor is faulty?
This monitor appears to work just fine with an Intel NT box at 640x480x256.

If the UDB is trying something outside this monitor's specs, is there a way to
ask it not to? Perhaps a jumper on the motherboard?
