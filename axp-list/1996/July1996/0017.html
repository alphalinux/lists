Przemek,

>There have been posts on this list [1], announcing that in the future
>there will two physically different CPU versions. The NT/Linux version
>will have some registers disabled, which, as far as I understood,
>affects the top-end memory mapping capabilities of the CPU. Linux can
>work around those limitations; NT doesn't care, 'cause it is 32-bit
>anyway. Digital Unix won't run, and so won't VMS (although it also is
>32-bit; don't ask, I don't understand it either).

As of V7.0 of Alpha VMS applications can create and use memory in the region
above P0/P1. They call it P2 and it requires 64 bit pointers. Much of the
operating system is now 64 bit with a lot of 32 bit stuff for compatibility.
One major change for VMS was to take the page tables from the shared S0 into PT
space (noshared).
