Hi all,
A couple of small bug reports. I wanted to install NIS on my
alphastation, so I got the yp-clients SRPM and after removing the hard
coded -m486 options to the compiler, it compiled easily. Unfortunatly, I did
not get much further than that.

When I try to run the domainname program, it gives me an
"Operation not permitted" error message. I ran domainname in the debugger,
and the setdomainname() system call is returning an error, and setting
errno to 1. I looked at the kernel source, and the only way this should
happen is if your are not root, and I was root. I assume there is a problem
somewhere in libc which causes this. I am using the libc and compiler
and binutils that Erik put in the update directory earlier this week.

Another thing I noticed is that if you single step through domainname
with gdb, gdb dies with a segmentation violation. You should be able to
reproduce this by typing:

cd domainname
make clean
make CC='gcc -g'
gdb domainname
b main
r rtp
n
n
n
n

Jim
