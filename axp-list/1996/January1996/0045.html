All,
there's a new version of the Milo sources on
gatekeeper (milo-1.3.52-011096.tgz). This contains fixes in
the timer code that make the floppy drive work on the EB64+
(and clones) and on the EB164. These were kindly donated
by Linus and are somewhat of an improvement over the original
code (how did it work?). The code also now allows linload.exe
(v1.2) to pass the real memory size to Milo - this means that
Milo _will_ get the memory size right when booting via ARC.

I've put new Milo images with these "features" into the
UDB (Multia) and EB164 test-image areas. I'll add in new images
over the next week or so. Meanwhile, the images that are there
should work and you can always build your own.

I'm not planning to add any more new features into
Milo at the moment (I _still_ have to write the flash update
utility for AlphaPC64, EB66+ and EB164) but I do plan to track
kernel releases and I'd like to sync with 1.4 when it comes
along. Of course, if someone wants a feature and adds it in
then I'm happy to take patch files. I'm also planning to add
support for more boards and systems.

There are a few things that Milo users can help with.
Firstly, if I know when someone is planning to take a snapshot of
the Miniloader area on gatekeeper as part of a CD release, then I
can make sure that the images are all up to date. Secondly, I'm
still not sure exactly which clones there are and what they're
called. I'd like to have softlinks from clone names to Digital
board names so that users can easily find the right Milo. Thirdly,
if there are boards that are slightly different from Digital boards
then I'm happy to take patches but I'd also like some mechanism
of knowing that the images I build are good ones.
