I do not think this message ever made it out of vger, so I'll try here
if that's ok. Apologies if it did actually get forwarded by vger and there
just haven't been any replies.

The latest milo will not recognize my Number Nine I-128 PCI card.
An older milo will, but only if I load it from floppy.

I got the [then] latest milo from gatekeeper.
-r--r--r-- 1 0 0 533648 Nov 20 12:37 milo.bios
-r--r--r-- 1 0 0 466080 Nov 20 12:37 milo.bios.rom

Here are the results from booting:
BIOS32 0.03
device/vendor id [it recognizes the I-128]
ROM base address+1 = c0001
vga_bios_init: starting emulation...
halted
vga_bios_init: emulation done...
x86_bios.int10 unknown function AH=0 BL=0xe8
mem_read address 0xffff out of range!
halted...
NVRAM: not supported on this system [Aha!]
MILO 1.3.34
[BIOS Emu, IDE, SCSI, Floppy, 64 MB RAM]

I also have a version of MILO that came with the machine:
Linux/Alpha Miniloader (MILO) #81 Thu Aug 3 15:30:31 EDT 1995
(It also came with a 1.3.35 kernel.)

This sets up the I-128 properly when booted from floppy (flboot scsi.apc),
but when I load the rom version and flash it,
flload scsi.rom
flash 200000 8
bootopt 131
the card is not set up--no signals go out to the monitor.

Let me know if you have any suggestions; for now I will use the older milo.
