There's two parts to this, the first is the ability to hit the reset
button and the second is how to soft reboot a system as in

bash$ shutdown -r now

The first problem is board specific. There are two jumpers, one that
should be wired to the reset button and the second that can be wired
to a lockable switch _or_ just jumpered. The second jumper enables
the reset button to work. Having said that, I've not got my Noname
wired this way 'cos I didn't think of it until after I put the disk in.

The second problem is somewhat more difficult. The nub of the problem is
that the system is booted in a two stage boot. First the SROM code
gets going and then it loads the console/loader. After that, I believe
that you can do a soft reset by calling the PAL reset entry point. The
problem is that the PALcode has been swapped to and so it will re-call
the Linux kernel, which isn't ready for that. It must be possible 'cos
you can do it on Window's NT, but I haven't spent a lot of time thinking
about it - I've spent more time worrying about how to boot the system and
keep it going...
