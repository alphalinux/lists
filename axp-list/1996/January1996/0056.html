> Ive found the redhat reboot command doesnt work at all for me. I copied my
>inittab and rc files off my Slackware based x86 system, and they are sublty
>incompatable with the way RedHat setup init and friends to behave a sys V
>init.
> Specificly, reboot doesnt kill any running programs, the only programs
>that die are the ones started by init, as a result the umount always fails.
> I havent been annoyed enough yet to grab the source and set them to work

Ugh. We see this all the time in Intel land when converted Slackware
users first find out about our init. Take time to get used to it...it
is different, but it *is* a better way to do things than the BSD style
init. Trust me...I used to love the Slackware way and hated changing.
Now I'm glad I did.

Also, I think more recent versions of Slackware use SysV init as well.

Get used to it....it's better. :-)
