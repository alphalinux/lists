All,
there are now a whole lot of new Milo images on gatekeeper.
These have the following features:

(1) Self expanding Milo; that is, Milo is mostly gzip'd on load and
it unpacks itself and moves itself to the right location. This
means that the noname flash image has all the latest features (ISO9660 etc).

(2) Milo now contains both TGA and BIOS video code. It works out what's
there and uses it. BIOS first, TGA second. It still doesn't care too
much if there isn't a video device.

(3) Support for NVRAM (ie environment variables) on the EB164, AlphaPC64
and EB66+ based systems. (Just for you, Jay).

(4) Flash Management Utility that sets up environment variables and
allows you to flash an image. Like Milo, this works on either TGA or
BIOS video.

All the images are in test-images even though all the ones that I've
tested work (AlphaPC64, Noname, Avanti, EB164). The latest source
is milo-1.3.57-012696.tgz and it's more resiliant to different kernel
sources so it works against 1.3.48 and upwards. There's also David
Mosberger-Tangs milo tools in the source directory - I've not yet merged
them into the source tree.

Dave

------ howto use the flash management utility ------

(1) put fmu.gz onto an ext2 formatted floppy,

(2) boot milo and use the run command:

MILO> run fd:fmu.gz

(3) FMU will come up and, like, Milo, offer you a prompt. Typing
help shows the available commands:

list - show what's in flash
program - put the image that's embedded into fmu into flash. You will
be given a choice of where to put it and allowed a last chance to
not do this.
bootopt - allows you to change to boot options
help - obvious.
