On Sat, 13 Jan 1996, Matthew Geier wrote:

> Specificly, reboot doesnt kill any running programs, the only programs
> that die are the ones started by init, as a result the umount always fails.

Please read the Red Hat FAQ (http://www.redhat.com) at some point for
more information on this.

The crux is that Red Hat uses a newer version of SysV init then Slackware
which actually works like the SVR4's init. The right way to shutdown a
system is "shutdown -h now" where -h means "halt". If you use "-r", it
should reboot (and does on Intel machines), but the Linux kernel doesn't
know how to reboot itself yet.
