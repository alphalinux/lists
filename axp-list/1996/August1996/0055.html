Hello Sean

>I just installed the TGA server on my Dec Alpha UDB (dated Mar 2 1996)
>and I'm having trouble getting it to work right. I followed all the
>instructions in the README/XFree86Config. I can "runx" and get to twm
>however this will last ony 1 to 5 minutes before it crashes and I get the
>error "Broken pipe." Is this a problem with my X configuration (such as
>wrong values for the monitor) or is this a problem with the Server
>relative to my kernel (redhat 3.0.3)? I'm not setting Bandwidth, I
>noticed it was commented out in the sample config so I didn't set it.
>Could this be the source of my troubles? I also noticed there was
>another TGA server dated Feb 96 and named with an "EV5" in the file
>name.

Try the following:

Configuring the X TGA server

Preperation:
Delete the old Xserver(selected during the redhat install)
This is located in the dir: /usr/X11R6/bin/ usually called either XF86_S3
or XF86_MACH64

obtain the latest Xserver and README/Config file from either:

ftp.digital.com/pub/Digital/Linux-Alpha/X11

or

on the Red Hat CD \X

The files are:
axpbin-Xtga-BETA-0.2.gz ;Xserver
axpbin-Xtga-BETA-0.2.README ;README/Config

>From the CD do the following:

root]# cd /
root]# mount /dev/scd0 mnt
root]# cd mnt
root]# ls (should see the CD Rom)

gunzip the Xserver
copy/rename it to the dir: /usr/X11R6/bin/ renamed to XF86_TGA

chmod 4711 XF86_TGA

copy/rename the README to: /etc/X11 renamed to XF86Config
edit it with vi comment out all the instructions in the beginning with #

(This is at the top with notes from Jay Estabrook. You have to comment )
(out the notes or the X server will not start properly)

from the root issue:
ln -sf /usr/X11R6/bin/XF86_TGA /etc/X11/X

this relinks the server to X windows

type startx

===========================================================================

This should get the X server up and running.:->
