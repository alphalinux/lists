Hello Niel

>I am having trouble installing RedHat AXP on my UDB, which has 24 MB of
>memory, the 2.88 floppy drive, and an external SCSI box with an HP DAT
>drive, CD ROM Drive, and 2.0 GB drive as /dev/sdb1. At the MILO prompt,
>when booting off of a floppy boot disk, Linux reports all the SCSI device
>information correctly, and then lists the following error message:
>
>#lca: machine check (la 0xfffffc00002084d0, pc=0xfffffc000025894)
>Reason: access to non-existent memory (long frame):
>reson: 100000084 exc_addr: fffffc000025894 dc_stat: 7 car: 20a01141
>A write error to b-cache occurred at address 18090d0
>Attempted to access non-existent memory.
>
>I am using MILO from the NT ARC Firmware, and have set the MEMORY_SIZE
>variable to 24 MB, but the error still persists. Despite this error, I am
>still able to load the ramdisk okay and get the RedHat installation prompt,
>but I thought I should try to clean up the above error message before I went
>any further. Any suggestions or help would be greatly appreciated. Thanks.

I have possibly loaded about 300+ UDBs and they all say the same message.
Its due to less than 32mb ram on the system. It only comes up during boot
up and will not cause any problems. So my only suggestion is ignore it. I
also believe that this will be fixed in newer versions on the kernel.
