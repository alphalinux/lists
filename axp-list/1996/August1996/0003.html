On Tue, 30 Jul 1996, Richard Henderson wrote:
>
> Thanks for the tester. I compared four versions of memcpy; the one
> from glibc, Linus', Martin's, and my own.

If you're comparing different versions, you should make sure that you compare
them with the same physical memory layout, otherwise cache effects can hide
any results (unlucky placement of pages resulting in cache trashing on some
runs, not on others). So you should either use the exact same binary that
tests all the versions on the same memory area (with cache flushes in between
tests), _or_ you'd have to do multiple runs and average them (or take the
minimum value from the runs).

If you're _really_ serious about this (very nice), you should probably unroll
the inner loop a bit (that can help hide load latencies), and you can do
stuff like pre-fetch with a "ldq_u $31,64($srcreg)" etc.
