I think what Tim means is, that he used "minlabel" to setup the partition
table of his disk, and then did an install to it. He continues using the
floppy to boot MILO since the ARC console will *NOT* recognize a "minlabel"
style partition table.

Sigh; I'm afraid (someone please correct me if I'm wrong) that if you try
to write an "fdisk" style partition table, that unless you're both extremely
careful *and* lucky, the existing partitions will not be compatible with
the new partition table.

If you want to boot linload/MILO from the harddisk, it MUST be an "fdisk"
style partition table in order to be palatable to the ARC console. My normal
format is:

partition 1: ~2Mb DOS format, to hold linload/MILO
partition 2: whatever is left after partitions 1 and 3, EXT2 format
partition 3: enough SWAP for the memory/usage expected

It's a pain to do it this way, but then the root FS is /dev/sda2 which is the
kernel default and you don't have to remember to tell it something different
*every* time...

So, do what I originally suggested: use "minlabel" ONLY so far as the get the
H/C/S values, then use "fdisk" expert mode to set those parameters and normal
mode to build the desired partition table.

Good luck...
