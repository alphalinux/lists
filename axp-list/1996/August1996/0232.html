Hi axp guys,

I new to this list so my questions are probably basic but I have a major
deadline to make so any help would be appreciated.

I just took delivery of of an Alpha 500/333 machine. It is an EB164
mother board. Linux installation is a bit trick the first time but I had
it up and running in under an hour. It has 128 megs, a DEC Ethernet card
and a fast wide NCR53c825 SCSI card with a Quantum 4Gig drive. It's job
is to be a fast database server.

I have msql running fine but would like to get Postgres95 running. I got
hold of the source and started the port but soon found that it depends
on dynamic libraries, specifically the availability of libdld. The
Makefile gives me a choice but libld is tied to ELF format binaries and
Alpha Linux seems to use ECOFF. I've seen COFF before but not ECOFF.

So here goes:

What is the latest on dynamic libraries/linking for Alpha Linux?

Will we switch to ELF or stay with ECOFF?

What is ECOFF? Perhaps compared to ELF and AOUT...

Where should I be looking for info rather than asking already answered
questions here?

Any other Axp'ers in the San Jose area?

Has anybody else ported Postgres95 somehow.

I actually managed to do a complete build by short circuiting al of the
dynamic loading stuff, hoping that it might only be needed for user
written 'C' functions but alas initdb needs it. One other problem I ran
across was the spin lock function. It needs an atomic Test And Set
instruction for a semaphore type operation. I won't say how I hacked
around that. Does anybody know how to do this in a 21164 chip? I don't
have the manuals yet. Seems like a kernel semaphore should work but the
Linux x86 port of postgres used an xchg instruction.

I think I might write a clearer install doc when I have some time, at
least for the EB164 board.

Thanks in advance for any info

Mike Cruse
CTI Limited.
