> I finally tracked the problem down to this
>
> time (&t);
> printf ("%s\n ", ctime (&t) + 4);
>
> Does anybody have a clue as to why this is happening.

I'd bet a beer you don't include time.h and so get a prototype for ctime().

With no prototype, the compiler, by law, must assume ctime returns an int,
and so the 64-bit pointer gets truncated to 32 bits. gcc -Wall should
warn you about using an integral type where a pointer is expected in the
printf format.
