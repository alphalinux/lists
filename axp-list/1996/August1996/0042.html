On Thu, 1 Aug 1996, Brett McCormick <brett@speakeasy.org> wrote:
>
> doesn't gcc need the heads & libs for making the cross compiler?

No. It needs headers and libraries for the system the cross compiler
runs on, not for the system it generates code for.

Of course, you'll need the later for cross-compiling applications,
but the Linux kernel (and I would guess most others) is self-contained.

But as I noted earlier, the code quality resulting from cross-compiling
from a 32-bit system is poor, so you'd only want to do this while
developing a kernel for new, unsupported hardware. Once you are booted,
you can recompile the kernel and build your applications natively.
