The time command as distributed with Red Hat 3.0.3 does not count page
faults properly (they're always zero). Turns out this is due to a bug
in configure (the same problem exists on the x86). The easiest way to
fix this:

- run "autoconf" in time-1.6 directory (must have a recent
version of autoconf; 2.10 is known to be good)

- create "install-sh" file (an empty file is OK as long as
you don't run "make install"

- run ./configure

- build with "make CFLAGS=-O2"

- install new "time" binary

This works fine at least when using GNU libc---I haven't tried the old
libc.

Note that there will still be a few stats that are always zero
(avgtext, avgdata, maxresident, inputs, outputs), but that's because
the Linux kernel doesn't keep track of those values.
