>>> Jim Nance said:
> I tried to install Linux on an Alcor this morning, and it did not go so
> well. First I built a 2.0.11 kernel, cross compiling from Digital Unix.
> I included SRM and TGA support, as well as de4x5 and tulip networking,
> and NCR and qlogic SCSI support.

Um, you don't want both de4x5 *and* tulip drivers, they handle the same thing;
maybe they're fighting over who gets control of the controller... :-( :-(

I don't think you need the NCR810 support; both SCSI controllers are QLogics,
and they're on a combo board along with the TULIP behind a PCI-PCI bridge so
they only take up one PCI slot on the primary bus. One QLogic to handle all
the internal devices, the other is attached to the external port on the back.

> When I try to boot the kernel, I get
> about a hunderd messages like:
>
> CIA machine check NOT expected

The messages aren't a good sign; I think I changed the testing in the
machine check routine so if the MC *was* expected there's no message (during
PCI bus probing, you look at *all* slots to find where the devices are, and
take machine checks for the slots that are empty).

> and the the machine locks up. I edited the .config file and removed the
> CONFIG_ALPHA_CIA entry, and did a:

Uh, oh; gotta have that, it's the glue between the CPU and memory/PCI busses.
And another flavor won't help, they're quite different...

> Am I doing anything stupid, or is this kernel not supposed to
> work on the Alcor yet? I think I remember booting a kernel from the late
> 1.3.xx series on this machine, and getting up to the insert ramdisk prompt.

It better be working on the ALCOR, as David M-T and Linus both have one as
their main machines...

If you want, either David M-T or I could put a kernel out for FTP (David's
would be faster, ie no firewall... :-).
