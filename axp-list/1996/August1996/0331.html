} I'd sure like to implement a shadow password file scheme in the new
} Red Hat Linux/Alpha 3.0.3 installation on my Digital UDB. Does anyone
} have some good success (or failure) stories to tell me about the
} Shadow Suite? Is the Alpha port ready and stable? Does a precompiled
} version exist? What else will I have to compile and replace on my
} system?

Rather than putting a load of work into shadow on AXP (although I
hear that Craftworks have done it), it might be better to look at
implementing PAM (Pluggable Authentication Modules) on AXP. This
might require ELF (or at least dynamically loadable libs). For
details of this see
http://www.redhat.com/pam/

The Red Hat Rembrandt beta release (for i386) uses PAM - see
http://www.redhat.com/redhat/rembrandt/

PAM promises to allow "normal" unix passwds, shadow, MD5 etc all by
changing a single config file (and having the appropriate modules
around to load).
