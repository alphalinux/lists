Hi fellow Alphenhacken,
I am trying to do an install of Linux from Redhat 3.03 on an EB164 ( EBP41-AN ).
I grabbed the kernel from gatekeeper kernels ( the one called
EB164-IDE-VGA-2.0.10.gz) and the Milo from test-images for the EB164 ( the
milo from Aug. 13th).
Using the above kernel on my number 2 floppy ( I copied it to the floppy
using Linux on my Intel box's Linux as the floppy is E2FS format ) I can now
boot, run fdisk and it sees the architecture of my 2.1GB Quantum Atlas
properly ( Yahoo, no more mindisk to get heads and sectors !)
Now, I try and let the Redhat install do a mkswap -c on /dev/sda1 ( I made
the first partition my swap) and it gave a segmentation fault and died.
Boot again, tried it with mkswap -c /dev/sda1 524450 (Yes, _that_ big a swap!)
One thing I should probably mention is that when I tried previously on old
kernel as per the images from the Redhat CD, it allowed me to do fdisk with
X mode changes to input the heads, cyls and sectors. It also allowed me to
get throught the mkswap on the install, however it failed on mkfs for the
root partition. Another thing I should mention is that this box has 256MB of
RAM, so I am building swap at about 512MB (Going by the old minimum swap =
at least 2x real memory theory).
So, what do you suggest. It's a bit of a "chicken-and-egg" thing to try a
2.1.12 kernel here on it...
I can run mkswap without -c and it doesn't bitch. Also when I did mke2fs on
the root partition it goes to "writing inode tables: 41/ 193CIA machine
check NOT expected".
Now we are hung.

More "Spys are watching" stuff, I suspect, but what to try next?

A subtle, polite, humbled, but desperate "Help" !!!
