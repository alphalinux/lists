Yes. libc is dead, long live libc!

Note that glibc is supposed to compile under ECOFF too (it did last
time I tried). That is, it's not a question of ECOFF or ELF. But
upgrading to ECOFF/glibc requires about as many changes as upgrading
to ELF/glibc, so it's not really an interesting option.

As for feature testing: just do:

#if __GNU_LIBRARY__ - 0 < 6
# error This is libc-4.3.26
#else
# error This is glibc with _bus_base_sparse
#endif

As Richard mentioned, I plan on making a new release this weekend
(along with an updated kernel and aboot).

If you need anything else in glibc for the X server, please let me
know asap. I added _bus_base_sparse() to make svgalib work on all
Alpha PCs with sparse and dense memory. I have never looked at the X
servers, so I'm not familiar with what it needs.
