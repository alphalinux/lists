davidm@AZStarNet.com said:
} As Craig (I believe) pointed out, another option would be to use the
} ncr53c800 driver.

I used this driver, in FAST mode with various other things enabled on
2 nonames with kernel 2.0.8 using a set of Quantum Grand Prix drives.
It worked very well - at least it worked on one machine for a couple
of weeks (so far), the other went belly up with SCSI errors which I
think were real disk problems rather than driver related. The author
said that it was untested on Alphas but one other list person has
implied they are using it.

NB I enabled tagged queueing operation.

The surviving machine is a web cache - lots of disk traffic!

As ever, your milage may be different, but I thought I'd give this as
a datapoint.
