Hello *,

} there has been quite a lot of talk about the pros and cons of the
} newly introduced NCR-SCSI-driver in the Linux kernel.
} So I decided to give it a try, but "make boot" fails:
} ncr53c8xx.c: In function `ncr_host_info':
} ncr53c8xx.c:8146: structure has no member named `use_mmio'
} ncr53c8xx.c:8148: structure has no member named `reg_remapped'
} Any hints?

Yes, not only a hint, but the solution (I encountered the
same problem):

In the file ~linux/drivers/scsi/ncr53c8xx.c there the
NCR_IOMAPPED is defined when:

a. SCSI_NCR_MAPPED is defined (only when explicitly done in configuration)

b. it is an alpha (they need that)

In the file all occurances of SCSI_IO_MAPPED are changed
to NCR_IOMAPPED, but one. Change that one misser too, and
your compilation problem is solved.
