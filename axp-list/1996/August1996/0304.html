Ok, so I compiled 2.0.12 kernel on Alpha (Cabriolet, EV4). It even
boots and runs and now I try to load some modules. CONFIG_MODVERSIONS
is not set.

With insmod from an updated RedHat distribution
(modules-1.2.8-8.axp.rpm) I am getting this:

$ insmod /lib/modules/2.0.12/net/bsd_comp.o
insmod:error: The module's `kernel_version' (2.0.12) differs from the current
kernel version (pha). Verify the module is up-to-date for
kernel version pha, then try again after recompiling the module
with the current kernel's header files.

So I tried to recompile the latest modules-2.0.0, found on
ftp.azstarnet.com. "As-is" this does not compile due to problems both
within a code itself and header files (.a_trsize, .a_drsize and
.a_syms structure fields have to be replaced by macros, also in
<linux/a.out.h>; the code needs some fixes and I turned elf support
functions roughly into no-ops). After modifications it compiled, but
now I see

$ ./insmod /lib/modules/2.0.12/net/bsd_comp.o
kernel_version needed, but can't be found

for a change. Does somebody has a solution for this problem?
The catch is that some stuff I need is available in a module form
only.

Thanks,
Michal
