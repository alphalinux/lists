>>>>> On Fri, 16 Aug 1996 13:43:37 -0400 (EDT), Janos J Krezgy <janos@ns.ncsa.com> said:

Janos> I'm forever getting "warning: cast to integer from pointer of
Janos> different size" around malloc calls... things _seem_ to be
Janos> ok, but I've tried casting things all manner of ways to no
Janos> avail...

Pointers are 64 bits on the Alpha and C defines that implicitly
declared functions default to return an `int', which is 32 bits only.
Just include <stdlib.h> and the warnings will disappear (no casts at
all are needed with a __STDC__ compiler).

It is always a good idea to compile with gcc -Wall---it will help you
discover many bugs. In the above case, gcc would have printed a
warning that there was no declaration for malloc() etc.
