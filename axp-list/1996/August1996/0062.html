On Fri, 2 Aug 1996, Martin Cooke wrote:

>
> Hi,
> Can someone point me to a good backup program that will work on
> our alpha (multia). It is running the 2.0.8 kernel and the tape drive is a
> conner python 2 gig dat. I tried useing tar, but it choked:
>
> [root@beam linux]# tar -cvW -X/etc/tar-backup/exclude -f /dev/nst0
> tar: Read error on /dev/nst0: I/O error
> tar: Read error on /dev/nst0: I/O error
> [you get the idea..]
> tar: Too many errors, quitting
>
Hmmm... I am using a Conner CTMS-3200 on my UDB, running kernel 1.3.89.
I simply type something like:

tar cvzf /dev/st0 /usr

and everything works fine.

Not that I think it makes a difference but my scsi tape support is
loaded as a module (st.o).

You might check your scsi buss and make sure you are using good quality
cables etc. as well as be sure you are using an active (forced perfect)
terminator. I used to have some strange problems with external scsi devices
before I took the advice of the scsi HOWTO and purchased an active
terminator (about $50).

Wish I could be of more help.
