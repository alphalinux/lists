I was trying to port a program to my Noname and I kept getting a run time
memory violation error (it compiled fine). I have successfully complied
and ran this on my Intel Linux and Sun Sparc Solaris with no problems.

I finally tracked the problem down to this

time (&t);
printf ("%s\n ", ctime (&t) + 4);

The "+ 4" should move pointer so the output is looks like "Aug 6 08:43:16"
instead of
"Tue Aug 6 08:43:16". Even though it compiles fine it will produce a
memory violation when it gets run. I have never seen this happen on any
other platform.

Does anybody have a clue as to why this is happening.
