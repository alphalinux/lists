jestabro@amt.tay1.dec.com spake unto me, saying:
>
>
> >>> ethan_shayne@mail.amsinc.com said:
> > I tried running xf86quake (yes, I know it has some problems, but I
> > wanted to give it a try anyway) and ran into the following error
> > immediately:
> >
> > Xlib: extension "XFree86-DGA" missing on display ":0.0"
> > Error: DirectVideo could not be setup
>
> Yes; it means that the TGA server was built with XFree86 3.1.2 release, and
> DGA did not become part of XFree86 until the 3.1.2D (?) betas.

> Note that on the machines I've tried xf86quake on, it runs *dog* slow on
> a 275MHz CABRIO with S3-764 (like 1 fps), and it hangs an EB164 with S3-968.
>
> Steer clear of it.

Well, yes, but call me an adventurous fool -- I'd like to give it a
try anyway. MY S3 server, however, reports the same "XFree86-DGA"
error. I grabbed the newest S3 server I could find at RedHat but no
dice. So where can I find such a 3.1.2E server, or do I have to
compile it myself?

Alternatively, it it possible to configure either X or xquake into a
higher screen area mode? Playing Quake in a tiny rectangle in the
middle of my screen gets tiring -- and that's with X in 640x480... I
tried switching X into 320x200, but the video system hangs up (Diamond
Stealth VRAM PCI). I can only presume that I have a config glitch
somewhere when I try that. I have heard that Quake can be switched to
a higher res mode, but I haven't figured out how (is this even
supported in the Linux/Alpha version?)

And as another data point: I, too, experience the stack overflow
error when running xquake. It happens in more than one location,
though -- I have encountered a number of different places where it
happens -- some repeatabley, some not.

But it's still a great thing to have. Lot's o' fun. Now if we can
just convince the Descent people to port Descent II... _That_ would
be heaven...
