These changes were needed because you dropped David M-T's patches on top of
the straight source tree; those patches add large file support, and MILO's
routines reflect the original kernel read/write arguments, not the large ones.

> Ok, now for a new problem :)
> I set the system clock to GMT. After booting via arc->milo i get a totally
> wrong system time like
> x@force:[0] $ date
> Thu Jan 1 01:03:47 MET 1970
>
> booting via srm->milo gives me the correct date.

ARC console maintains the clock format differently from SRM, so you have to
change:

/etc/sysconfig/clock

to be:

CLOCKMODE="ARC"

instead of the "LOCAL" it prolly is now. Thus, when "clock" gets called during
processing of /etc/rc.d/rc.sysinit, it'll get the "-A" flag which indicates
that the hardware clock is being kept in "ARC" mode...
