>>> ethan_shayne@mail.amsinc.com said:
> I tried running xf86quake (yes, I know it has some problems, but I
> wanted to give it a try anyway) and ran into the following error
> immediately:
>
> Xlib: extension "XFree86-DGA" missing on display ":0.0"
> Error: DirectVideo could not be setup
>
> I'm running on a Multia, TGA video, RedHat 3.0.3, 1.3.90 kernel, and
> the beta TGA X Server.
>
> Since this happened before the game even started up, I don't think
> this is the error ddt was talking about. Anyone know what it means?

Yes; it means that the TGA server was built with XFree86 3.1.2 release, and
DGA did not become part of XFree86 until the 3.1.2D (?) betas.

The TGA server *will* be updated sometime in the future (September/October,
I hope), once the ELF/shared libs and the XFree86 3.2.0 release are stable,
*and* I get some time... :-)

Note that on the machines I've tried xf86quake on, it runs *dog* slow on
a 275MHz CABRIO with S3-764 (like 1 fps), and it hangs an EB164 with S3-968.

Steer clear of it.
