> situation. Do you have any other ideas? Thanks.

I had the same problem. The problem is that on bootup, init (or rather
rc.sysinit) looks at the file /etc/sysconfig/clock to see how to configure
the clock. The default I belive is CLOCKMODE="local". Change that to
CLOCKMODE="ARC" and reboot, and it *should* set the clock to the ARC's
time.
