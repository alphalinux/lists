I tried to install Linux on an Alcor this morning, and it did not go so
well. First I built a 2.0.11 kernel, cross compiling from Digital Unix.
I included SRM and TGA support, as well as de4x5 and tulip networking,
and NCR and qlogic SCSI support. When I try to boot the kernel, I get
about a hunderd messages like:

CIA machine check NOT expected

and the the machine locks up. I edited the .config file and removed the
CONFIG_ALPHA_CIA entry, and did a:

make ARCH=alpha CONFIG_SHELL=bash oldconfig
make ARCH=alpha CONFIG_SHELL=bash clean dep
make ARCH=alpha CONFIG_SHELL=bash -j

After I strip, compress, install, and boot this kernel, the messages about
the CIA machine checks are gone, but it eventually dies after complaining
about several SCSI timeouts. In case it matters, this machine has 2 SCSI
controllers. Am I doing anything stupid, or is this kernel not supposed to
work on the Alcor yet? I think I remember booting a kernel from the late
1.3.xx series on this machine, and getting up to the insert ramdisk prompt.

On an unrelated note, I have a SCSI zip drive connected to this machine.
Does anyone know what the command is under Digital Unix to write a disklabel
onto this drive. I hate trying to use disklabel, because I dont do it
enough to remember how, and it is non-obvious how to put a label on an
unlabeled disk, especially when the disk does not have an entry in
/etc/disktab.

Thanks,

Jim

PS I really do realize that the CIA in the message does not refer to the
US government agency.
