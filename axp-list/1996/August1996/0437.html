Tim,

>Tim Waugh <tmw20@cyberelk.demon.co.uk> wrote:
>
> I've installed RedHat on my UDB, and I used minlabel to partition the disk
> because fdisk didn't know how many h/c/s the disk had. I want to be able
> to put MILO on a small MSDOS partition of the hard disk, but I can't
> pursuade minlabel to label it as MSDOS. Should I run fdisk to set h/c/s
> and then rewrite the partition? Will this destroy the installation? Is
> there another way of loading MILO from my hard disk rather than from
> floppy?

Destroy what? Why don't you tell what you have in that drive?

Do you have any OS installed in that drive? If you have one, use it
to copy milo and linload.exe to exist MSDOS partition. If none, use
fdisk to set CHS of that drive and create MSDOS partition to hold
them.
