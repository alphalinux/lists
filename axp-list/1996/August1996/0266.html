> SUN uses 512-byte sectors. I was able to use a SUN CD-ROM on a PC running
> Linux without problems. I don't know sure what SGI uses, and what the SRM
> or MILO do about differing sector sizes.

SGI used (still uses?) 512-byte sectors too and their CDROM firmware
even set the SCSI device type to "fixed disk" (hard disk?) so that it
really looks like a normal hard disk which the old boxes' boot roms
can boot from.

the Toshiba 3401 and newer have two "solder pads" where you can cut
two lines specifying the firmware version; one is "SUN compatible",
one "SGI comp". and at least one setting is for normal 2048-byte-sector
CDROM behaviour (but I can't find the exact information anymore; this
is only from memory and my Toshiba 3201 doesn't have even this feature :-(
