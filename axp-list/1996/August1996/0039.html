On Thu, 1 Aug 1996, Brett McCormick <brett@speakeasy.org> wrote:
>
> So what target should I configure for then, if any?
> And can I install the patched gcc to compile the kernel?

You should configure both binutils and gcc with "alpha-linuxecoff"
to get the standard ECOFF compiler. If you want to do ELF you'll
need a completely different set of patches for gcc. But if you're
just trying to cross-compile the kernel, ELF doesn't buy you anything.

In a separate message, he continues:
>
> I've run into a few problems.. first off, where can I get the include
> files and libraries for linux alpha (include libgcc1.a, if i need it)

You don't need any headers or libraries for compiling the kernel.
As for libgcc1.a, just touch it before you start the gcc build.

> make LANGUAGES=c CFLAGS+=/usr/local/src/linux/kernel/include/asm-alpha
>
> the output is:
>
> _muldi3
> /tmp/cca21820.s: Assembler messages:
> /tmp/cca21820.s:1: Error: Expected comma after name "noreorder"
> /tmp/cca21820.s:2: Error: Expected comma after name "volatile"
> /tmp/cca21820.s:3: Error: Expected comma after name "noat"

Your new gcc isn't finding the Alpha assembler. Either get that
properly installed first, or symbolicly link it into the gcc top-level
directory.
