After installing redhat-linux 3.0.3 on an as255 I have problems to get
networking up. I tried both the tulip and the de4x5 driver to use
with the onboard 21040 ethernet controller. It seems there is
something wrong with the PCI setup. While booting linux 2.0.10 the
following messages are displayed:

bios32.sio_fixup: weird, found device 1011:0002 in non-existent slot
14!!
bios32.sio_fixup: weird, device 8086:0484 coming in on slot 7 has no
irq line!!
:
:
ncr53c8xx : at PCI bus 0, device 6, function 0
:
:
eth0: DE434/5 at b000 (PCI bus 0, device 14), h/w address
00:00:f8:22:25:c8 and requires IRQ10 (provided by PCI BIOS).
de4x5.c: v0.43 96/6/21 davies@wanton.lkg.dec.com

The kernel is 2.0.10, for "avanti" and was compiled on this machine.
/proc/interrupts says

1: 3642 keyboard
8: 0 + rtc
10: 0 DE434/5 (eth0)
11 3883 53c8xx

and /proc/pci:

PCI devices found:
Bus 0, device 14, function 0:
Ethernet controller: DEC DC21040 (rev 36).
Medium devsel. Fast back-to-back capable. Master capable.
Latency=255.

I/O at 0xb000
Non-prefetchable 32 bit memory at 0x1000000.
Bus 0, device 13, function 0:
VGA compatible controller: S3 Inc. Trio32/Trio 64 (rev 0).
Medium devsel.
Non-prefetchable 32 bit memory at 0x1800000.
Bus 0, device 7, function 0:
Non-VGA device: Intel 32378IB (rev 132).
Medium devsel. Master Capable. No bursts.
Bus 0, device 6, function 0:
SCSI storage controller: NCR 53c810 (rev 17).
Medium devsel. IRQ11. Master Capable. Latency=255. Min Gnt=8.
Max Lat=64.
I/O at 0xb400.
Non-prefetchable 32 bit memory at 0x2000000.


The ethernet controller does not recognize any interrupt, but the
controller LEDs flash.
Has anybody an as255 and network running?
