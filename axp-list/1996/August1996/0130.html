sp0t the w0nder slug wrote:
>
> I've got a 3Com Etherlink III that i'm using on my EB164, I notice that
> according to the list of compatible hardware this particular chipset
> isn't supported.. I was wondering if anyone had any luck in getting one
> of these cards to work under Alpha-Linux.
I have my 3c509 working on a noname. I have, instead of fixing
insl/outsl
system routines, fix the 3c509b driver instead. Much easier to remember
that the 3c509 doesnt work, rather than some other routines that maybe
harder to find.
The interrupt setting for the card is somewhat harder to do. My 3c509b
is
set at 10 ( as are all of my 3c509 cards ). Although the 3c509b driver
is
happy, and apparently does the setup correctly, my noname with the
ARC/NT
console, continuously interrupts, rendering a frozen system. I had
traced
this disabaling feature to the alpha/bios32 routines. This has not been
fixed in 3.0.3/alpha/linux, nor in the 2.0.8 version of linux ( you
should
note that the has been enhanced/modified, but the offending pair of I/O
calls are still there, although not in the same location!)
On my noname SLOW_DOWN_IO in io.h should be defined as an outb
instruction
( like that in the i386 version of io.h ). The while loop that is
substituted
for SLOW_DOWN_IO in some of the later releases just dont make it (cant
read the
eprom!)
Plz let me/us ( this mail group ) know if u need specific changes, as
every note thusfar, including this one, has only vague howto's.
