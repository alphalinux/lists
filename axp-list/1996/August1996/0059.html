>>>>> On Fri, 2 Aug 1996 13:52:06 +0200 (MET DST), Miquel van Smoorenburg <miquels@cistron.nl> said:

Mike> You (David Mosberger) wrote:
>> The regular kernel sources do not contain Alpha ELF support
>> (yet). You'll need the patch at:
>>
>> ftp.azstarnet.com:/pub/linux/axp/sys/axp-diffs-*.gz
>>
>> The latest patch is for 2.0.7 (I think), but it probably applies
>> cleanly on top of 2.0.10 as well. I'll update the patch soon
>> (hopefully no later than this weekend). Linus plans on adding
>> Alpha ELF support to the 2.0 kernel, so this problem will be gone
>> soon.

Mike> Is that just the ELF support, or the 64-bit patches (as in
Mike> your patch) as well? And will this make any difference for
Mike> GNU-libc?

It's ELF support, large-file support, header fixes and random bag of
other bug fixes that, for one reason or another, won't make it into
2.0. The ELF support will be in the kernel soon, so you might just
want to wait a little longer if you don't want to mess with the other
patches.

Mike> Hmm this is not the topic of this mailinglist but I see glibc
Mike> is getting ready for a 1.93 release. Any chance they will
Mike> integrate your alpha patches? Or do we have to keep patching
Mike> ourself each time?

We have been working closely with the glibc maintainers and lots
(most) of Alpha/64-bit-related patches are in glibc already. The ones
that aren't there yet are (a) dependent on kernel-header fixes that
need to be applied on top of 2.0, (b) of a copyright status that FSF
doesn't like (sqrt, is the only one, I think), or (c) gross
Linux-hacks that haven't been resolved cleanly yet (determining the
controlling tty and a clean fix for Linux syslogd come to mind here).

It will be some time before glibc will build out of the box, but it
will happen, eventually.
