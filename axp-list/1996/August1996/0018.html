I'm installing Red Hat Linux/Alpha 3.0.3 from the CD-ROM onto a
Digital "Easy Web Server" (Multia/UDB/noname).

After finishing the install, I tried booting the Linux kernel off the
hard drive from MILO with "boot sda3:vmlinux.gz" and it loaded the
kernel OK and finds all the hardware, but then it gets hung on:

Kernel panic: Unable to mount root fs on 08:02

The standalone shell in the install has no trouble mounting /dev/sda3
(or /dev/sda5 or /dev/sda6), and all the files I expect to see are in
those partitions. So, why won't it mount the root partition at
boot time?

Some background - I had a tough time getting the drive initialized and
partitioned the way I want, but I finally got (according to fdisk):

/dev/sda1 2MB DOS <32MB partition (for loadlin.exe and MILO)
/dev/sda2 128MB swap (Linux swap)
(Unit has 32MB RAM now, but will have 64 MB soon)
/dev/sda3 64 MB / (Linux native)
/dev/sda4 Extended partition which includes:
/dev/sda5 128MB /var (Linux native)
/dev/sda6 Rest-of-the-disk /usr (Linux native)
(/home will be put on a separate disk when it arrives)

I had to wrestle with fdisk a few times, and at one point I did a
"mke2fs /dev/sda", which wiped out the head/sector/cylinder
information, which I conveniently forget to write down beforehand :-P.

This machine has an RZ26F-E (1.0GB) disk, and my Digital UNIX
machines' disktab files don't have an entry for it, but all other RZ26
drives listed say "ns#57:nt#14:nc#2570" which I take to mean 57
sectors, 14 heads, and 2570 cylinders. So that's what I used in
fdisk. But I *do* remember that when it was set up for NT, it had
1001 cylinders, so I don't know what to believe.

Can anyone give me a clue as do what to do next? Thanks!
