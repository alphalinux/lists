On Wed, 7 Aug 1996, Johan Sarkinen wrote:
> Here's my wanted funktionality: I want to have my Linux machine, hereafter called
> MY-GW, working as a Internet gateway for my local network, i.e. Internet completely
> available for all machines on the local network.
> MY-GW connects to my Internet operator via PPP, dynamical IP, and everything works
> fine on it (ping, DNS, ftp, www, ...).

In order to do what you want, you have to have control of the routing
information on the *other* side of the PPP link as well as on your own side.
If your provider won't help you with this, you have to do it some other way,
like via masqerading.

My home network has my Alpha, a desktop PC, a laptop PC, and a Mac SE/30, and
I connect it all to the net with masqerading through my Alpha. It works
(except for FTP/RealAudio, because I can't get modules to load, but that's
minor).

> But now:
> 1. Isn't there some way to use route(8) to set up proper network routes or
> must I use routed/gated.

You don't need a daemon if you've got the help of your provider, and if you
don't have that help, it won't work even with a daemon.

> 4. Is there any other trick one can use to get this up and running?
> (I've read about the IP Masquarade, but the ipfw(8) command isn't supported
> on my axp with kernel 1.3.90 - at least what I've read in net-tools packages
> found by me.)

The ipfw command isn't supported, but ipfwadm works.

I had to explicitly masquerade individual machines instead of my local net
(ie. "ipfwadm -F -a masq -S 192.168.1.0/24 -D 0.0.0.0/0 -p all" didn't work,
but "ipfwadm -F -a masq -S 192.168.1.3 -D 0.0.0.0/0 -p all" does work).

I have the ip-up script that PPP calls turn on masqerading, and ip-down turns
it off. I also run a cacheing nameserver on the firewalling box. All of my
other hosts are told that box is their nameserver and gateway.
