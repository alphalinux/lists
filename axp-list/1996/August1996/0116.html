At this point, I don't know what could cause this. All I can say is
that things seem to work fine for me (on Noname, Cabrio, and Alcor
though the Alcor is definitively the one that gets beaten up the
most).

One suggestion: you could try stock 2.0.11 and then apply the glibc
header patches only (i.e., omit all the patches that involve int ->
long changes). An updated axp-diffs-2.0.11.gz is in the usual place
(ftp://ftp.azstarnet.com/pub/linux/axp/sys). The glibc patches should
be safe. If anything is likely to cause problems, it's probably the
bigfile patches---in particular if you happen to use drivers that I
don't (watch for "incompatible pointer" warning messages).
