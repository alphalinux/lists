Hello David and Jay,
I finally got my Alphastation to work. It turned out to be a firmware
problem. After upgrading from the 3.7 to the 5.0-13 SRM console, everything
is working fine.

I got my instrustions for doing the firmware update from the URL:

ftp://ftp.digital.com/pub/Digital/Alpha/firmware/readme.html

in case anyone else wants to do this. I used the V3.5 firmware instead of
the V3.6 firmware, because 3.6 is not guaranteed to work with the version
of Digital Unix that usually runs on the Alphastation, but I assume that
using the latest would also work if you were only running Linux, or Linux
and Digital Unix 4.0.

I am sort of confused by this. I thought the console firmware was only
used to load the kernel image into memory. I assume that this is not
true since I get SCSI errors with one version, and everything works
with a later version. Is the PAL code in the firmware also? Anyway,
I think many people might be interested in this if one of you has the
time to explain it on the mailing list.

Thanks for all the help,

Jim
