The line should read vmlinux.gz with a 'X'. I'm used to Intel which uses
the other naming convention with a 'Z'.

To get somewhere with the other error, I moved the drive to a dos machine
and formatted it there. Fdisk could then get the parameters right.
I think you'll still have problems as you have to label the partitions
too which I think is something harking back to NetBSD type stuff. You'll
be able to make you're partitions but you have to then label them before
you put anything on them as it seems to screw things up a bit if you do
it later. If you don't label them milo won't see them and you wont be
able to specify which partition you boot off of. There's a command in
fdisk that does this but it's not obvious and I can't remember it.

This is my interpretation of the problems I had. I still haven't got
milo to work because of this label thing but boot fine using a floppy
which is automatically booted with the correct flags set in SRM.

Check out the sdisklabel and e2writelabel bits in milotools but beware
the zero option as it wipes the partition table and the labels. I think
David M-T might be able to clarify my ramblings, I'm sorry I gave up
before I got a result. :)
