> Can someone tell me if the 21164 uses virtual or
> physical tags in its data caches? I'm trying to
> design some tests to address Linus' concerns but
> realized I don't know how the cache is tagged.

Quote from "Alpha Implementations and Architecture":

The instruction cache (Icache) is an 8KB direct mapped, virtual cache.

The data cache (Dcache) is a dual ported, 8KB write thru, read allocate,
direct mapped, virtually indexed, physically tagged cache with 32-byte
blocks. The access time is 2 cycles and the bandwidth is 16 bytes per
cycle.

The on-chip second-level cache (Scache) is a 96KB, 3-way set associative,
physical, write back, write allocate, combined data and instruction
cache. It is fully pipelined and supports both 32-byte and 64-byte blocks.
Access time is 8-cycles and bandwidth is 16 bytes per cycle.

What else you wanted to know ?
