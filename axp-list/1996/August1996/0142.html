No, I'm pretty sure your announcement made it. I just didn't
realize/remember that you fixed the configuration bug too, so I never
bothered to download it. Next time I'll check more carefully.

Stephen> Anyway, the only fix needed is to explicitly tell the make
Stephen> process that the kernel supports WAIT3. The old autoconf
Stephen> was not detecting this for the very good reason that the
Stephen> kernel fails to count context switches in the process info
Stephen> structure. Compiling with -DHAVE_WAIT3 is sufficient both
Stephen> to correct the HZ problem and to get page fault accounting
Stephen> working.

The newer autconf checks a few other variables in the resource
structure, so it will correctly detect the presence of wait3().
