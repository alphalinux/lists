On Fri, 16 Aug 1996, Tim Barthel wrote:

> Jeffrey M. Odom wrote:
> >
> > I had the same problem. The problem is that on bootup, init (or rather
> > rc.sysinit) looks at the file /etc/sysconfig/clock to see how to configure
> > the clock. The default I belive is CLOCKMODE="local". Change that to
> > CLOCKMODE="ARC" and reboot, and it *should* set the clock to the ARC's
> > time.
>
> Thanks Jeffrey, that's the ticket. Works like a charm now. And thanks
> to all those who replied.

Has anyone come up with a good way to keep the clock set
accurately? If I use 'clock -Aw' I have to reset the clock in the ARC
next time I reboot. I'd like to have some automated (or semi-automated)
method of keeping the system time accurate.
I've been running 'getdate' as a cron job. It has an '-adjust'
option that is supposed to slowly adjust the clock so as not to freak ou
the system. It never seems to catch up to the time creep.

BTW - I have set "ARC" in /etc/sysconfig/clock.
