I'm porting an application to Linux/AXP that needs direct access to Ethernet.
On BSD-derived systems it uses the BPF, which is not available on Linux. So
I looked to see what tcpdump does. It turns out that it uses libpcap, and
the Red Hat tcpdump source RPM builds libpcap as an intermediate step in
building tcpdump, but it doesn't install it.

I'd like to suggest that either there be a separate RPM (and SRPM) for libpcap
to support applications like mine, or that the tcpdump RPM install libpcap.
I have provided below a patch for tcpdump-3.0.2.spec that does the latter.

Cheers,
Eric

*** tcpdump-3.0.2.spec.orig Thu Mar 7 21:26:22 1996
--- tcpdump-3.0.2.spec Wed Aug 7 16:39:29 1996
***************
*** 22,27 ****
--- 22,28 ----
make

%install
+ (cd libpcap-0.0.6; install -m644 libpcap.a /usr/lib; install -m644 pcap.h /usr/include; install -m644 pcap.3 /usr/man/man3)
cd tcpdump-3.0.2
strip tcpdump
install -m755 tcpdump /usr/sbin
***************
*** 30,32 ****
--- 31,36 ----
%files
/usr/sbin/tcpdump
/usr/man/man8/tcpdump.8
+ /usr/lib/libpcap.a
+ /usr/include/pcap.h
+ /usr/man/man3/pcap.3
