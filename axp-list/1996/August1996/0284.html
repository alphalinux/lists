The settings are as they should be; if you'd had them different, you'd
*ONLY* have gotten the beeps, as the sound chip wouldn't have worked.

I don't know the reason that *no* sound is produced after you disconnected
the internal speaker; maybe you ought to reconnect it and deal with the two
problems a little more rationally... :-)

First off, there's a very helpful mini-HOWTO that talks about ways to quiet
down the beeping via the speaker. It involves a number of things, like
telling bash that you want beep-style set to none or some such, along with
editing termcap/terminfo to remove/replace the beep-generating codes. I
don't remember all the details, but it *was* effective; that internal
speaker nearly drove *me* nuts at first, too... ;-)

The second problem, the "hiss", can be reduced/eliminated by adjusting one
of the "line" values using "aumix" (which has to be run in text mode, not
X), or the "mixer" command that comes in "snd-util-3.5.tar.gz". When you run
"aumix", you'll see some vertical bars labelled "line1", "line2", and
"line3", I think. By reducing one of them (I can't remember which, but
think it may be "line3"), you'll have the hiss reduced, along with a good
bit of distortion. Reduce it too much, and it'll reduce the overall volume
to too small a range. It needs to be fiddled with. Use of "mixer" is made
for rc.local on UDB's, I believe, as it provides a nice command line
interface to these parameters, that can be run automatically during boot.

See what you can come up with, and share the details, as I *know* there are
others being aurally bombarded out there... :-)
