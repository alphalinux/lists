>>> Sean McDirmid said:
> I just installed the TGA server on my Dec Alpha UDB (dated Mar 2 1996)
> and I'm having trouble getting it to work right. I followed all the
> instructions in the README/XFree86Config. I can "runx" and get to twm
> however this will last ony 1 to 5 minutes before it crashes and I get the
> error "Broken pipe." Is this a problem with my X configuration (such as
> wrong values for the monitor) or is this a problem with the Server
> relative to my kernel (redhat 3.0.3)? I'm not setting Bandwidth, I
> noticed it was commented out in the sample config so I didn't set it.
> Could this be the source of my troubles? I also noticed there was
> another TGA server dated Feb 96 and named with an "EV5" in the file
> name.

I don't believe the crashes are due to the monitor settings, but you need
to be careful, as the wrong settings could allow damage to the monitor.

The only crashes of the TGA server I've heard of were Netscape-related, ie
running Netscape on another box and pointing the display at the TGA. The only
other cause is toggling back and forth between X and virtual consoles, and
the README has a large warning about that...

What kind of error messages are you getting when it crashes?

BTW, I normally start X via "startx", and have seen very few crashes, either
on TGA or S3 or Mach64.
