I'm trying to install RedHat 2.1 on a Multia. I make a DOS-format disk
with linload.exe and a milo image on, and set up ARC to use it - that all
seems to work fine. I made a boot disk using dd, and it has vmlinuz.gz on
it. I also made a ramdisk (and corrected a few errors about which part of
the CDROM the install program wanted to find things on).

I got as far as the install program, and everything was working fine, but
I couldn't use fdisk on my Seagate Hawk (ST31051N) - pressing return on
the `fdisk' menu option just returned me to the same menu.

I tried using fdisk by hand, and it told me to set the number of sectors,
cylinders and heads. That scared me a little so I retreated to my PC to
look at a HOWTO. (As the two computers are sharing a power lead I had to
turn off the Alpha.)

Once I was confident enough to try tackling the problem, I tried to get
back to the same stage with the Alpha, but this time I couldn't get past
MILO. I tried `boot fd0:vmlinuz.gz root=/dev/fd0 load_ramdisk=1' just as
before, and it told me that it failed to load the kernel. Yet `ls fd0'
revealed that the kernel was there.

What can I do?

Tim.
