I also tried xf86quake, since that's the only version that supports the mouse,
and I can't play games like this with the keyboard. I had the same problem.

On Wed, 14 Aug 1996 ethan_shayne@mail.amsinc.com wrote:
> Xlib: extension "XFree86-DGA" missing on display ":0.0"
> Error: DirectVideo could not be setup

There's apparently an extension to the X protocol named "XFree86-DGA". This
version of Quake requires it. The Xtga server doesn't provide it. My guess
is that either Xtga is old enough that it predates the extension. It'd be
nice to get a version that supports it, so I could use that mouse...

By the way, I tested Alpha Linux Quake against Intel DOS Quake running under
W95 (via the "Q95" batch file). They can play against each other over TCP/IP
(though performance sucks on a 486, naturally).
