I have a Toshiba SCSI CD-ROM XM3701B, that gives exactly
this same error, since the AXP beginning. I then shortly
looked into it, and found that the error is in the photo-cd
code, and since I never entered a photo-cd in the drive,
I didn't bother. Mounting works fine, and reading the CD
is ok too.

Now, I have a multisession photo-cd, and obviously, it
doesn't work. I have been digging in the code again, but
there is something intrinsically wrong^H^H^H^H^Hnot fully
done yet. I have sofar understood that the verification of
the virtual memory segment always fails on an Alpha (which
is what the error message is about), because there is no
such segmentation. This segmentation should (one way or
the other) be emulated, or so (getfs/getds/setfs/setds
stuff); I haven't figured it out yet (still studying
David's response to my question). Several other things
are also wrong in the sr.c driver, notably the numerous
assumptions that a long is only four bytes large :-).

If anyone can help us out, I'd appreciate it.
