> On Wed, 14 Aug 1996, Tim Barthel wrote:
>
> > > > I just received a UDB. After I finally tracked down the way to set the
> > > > timezone, I set about setting the time using the control panel.
> > > > Unfortunately, my time setting is lost when I reboot the machine. I
> > > > have RH 3.03. Thanks.

> > >
> >
> > Well, thanks for the idea but this command didn't seem to help the
> > situation. Do you have any other ideas? Thanks.
>
> I had the same problem. The problem is that on bootup, init (or rather
> rc.sysinit) looks at the file /etc/sysconfig/clock to see how to configure
> the clock. The default I belive is CLOCKMODE="local". Change that to
> CLOCKMODE="ARC" and reboot, and it *should* set the clock to the ARC's
> time.
>
> Jeffrey M. Odom
>

Thanks Jeffrey, that's the ticket. Works like a charm now. And thanks
to all those who replied.
