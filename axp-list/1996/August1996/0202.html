I recently got a UDB that had RedHat 3.0.3 (Kernel 1.3.90) on it. I've
remade the kernel with sound support easily enough. I tried the
pcmcia...srpms on the Redhat CD but they didn't compile. I then went to
hyper.stanford.edu to get "the latest" at version 2.8.19. They compiled and
installed but at bootup when the modules went to load I'd get dozens of
either references undefined or symbols not found. So, methinks, I'll get new
kernel source, too. Downloaded 2.0.10 source tree, made from scratch
including support for modules, user device drivers, etc., etc. Went fine.
Donwloaded 2.8.20 PCMCIA sources. Compiled them fine using kernel source
tree for options and the -fomit-frame-pointer -fno-strength-reduce and
-mno-fp-regs flags that I saw used during kernel make. Compiled fine with
only a few warnings about variables not used. When I rebooted the ds module
and pcmcia.core modules load but i82365.o fails with a message: 'In function
irq_scan function referenced but not defined: _bus_inb'

What else do I need to do to get PCMCIA working? (I tried makeing the PCMCIA
tree with a -D__alpha__ flag because I saw that in the i82365.c source and
thought I needed to set it). Are there settings I need to make in
/etc/sysconfig/pcmcia or do I need to update something else?

Thanks for your help.
