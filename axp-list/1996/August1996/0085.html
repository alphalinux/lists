Ok, after playing a few days with several milos, kernels and kernel options,
and lots of support from Jay and David this is the status:

The problem with milo "hanging" when initializing the devices was because
i had srm-support enabled in the kernel tree i was using to build milo with.
So if you're compiling milo make sure there's no srm-support in the kernel
source tree you're using!
Disableing srm-support also let me build milo almost out of the box. Only
had to change these function definitions in devices.c:

-int block_read(struct inode * inode, struct file * filp, char * buf, int count)
+long block_read(struct inode * inode, struct file * filp, char * buf, unsigned long count)

-int block_write(struct inode * inode, struct file *file, const char *buffer, int a)
+long block_write(struct inode * inode, struct file *file, const char *buffer, unsigned long a)

No more problems with undefined symbols.

Ok, now for a new problem :)
I set the system clock to GMT. After booting via arc->milo i get a totally
wrong system time like
x@force:[0] $ date
Thu Jan 1 01:03:47 MET 1970

booting via srm->milo gives me the correct date.

any ideas?
