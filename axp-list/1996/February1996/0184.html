I should have said but this is what happens.

(1) boot Milo/Linux everything is just fine and dandy,
(2) reboot back to Milo:
a) milo uncompresses itself in memory
b) milo attempts to work out the memory size from the
impure area (a data structure shared between PALcode and
Milo) but fails,
c) Milo defaults the memory size to 24Mbytes
d) all hell breaks loose

be fine. I _will_ fix this, but it will be a few weeks...
