On Sun, 25 Feb 1996, Matthew Geier wrote:

> It now seems happy to serve my xterminal a bootp reply. Unfortunatly my
> Xterminal and in.tftpd disagree on some thing...

Linux in.tftpd requires the full path for a file. If you have it configured
in inetd.conf as "in.tftpd /home/tftp", it will only allow you to get
files of the form "/home/tftp/file". The file name *must* be prefixed with
"/home/tftp".

I've changed this in the latest NetKit-B RPM in the devel tree. It will need
RPM 2.0 to compile and build though. I'll try and port this to the Alpha
when I get a chance.

The patch is trivial though if you look at tftpd.c. It took me about five
minutes to fix this. Without the fix, a SPARC couldn't netboot from a Linux
box.
