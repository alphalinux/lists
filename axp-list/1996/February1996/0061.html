Hi everybody,

I've got a problem with a new Noname/233MHz board. It's the same
thing that Bev (tsunami@imsa.edu) reported in an earlier posting
to linux-alpha@vger.rutgers.edu:

- Screen stays black. Removed graphics card. Hooked terminal to
COM1 -> No output at all.

System configuration is: No cache (so far), 8 MB of DRAM (70ns),
no slot cards, SCSI harddisk and CDROM connected, floppy connected.

The LED Display Circuit as described in the OEM design guide shows
the tests and stops with 1010 0000 which reports 'no cache found'.

Switching the boot select jumpers to 'Boot service console' doesn't
help anything. Switching to 'Boot from floppy' doesn't cause the
floppy to move (I'm not sure if it is connected and jumpered
correctly).
I purchased the board with ARC console firmware.

Any suggestions are highly welcome!
