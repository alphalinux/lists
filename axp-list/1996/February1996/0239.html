I am a BLADE_0.3 user; on this system getegid(2) does not behave like on
Linux/i386 when the calling program file has the setgid bit set: it
returns the gid of user invoking the program, not the one of the file.
Is this a known problem (if so, is it a libraries/headers problem or a kernel
one ?).
As a matter of consequence, `elm' compiled from `elm2.4' (without Linux/Alpha
patches, if any) fails to read incoming mail(s) unless chowning to o+w the
/var/spool/mail directory.
