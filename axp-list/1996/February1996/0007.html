I think gcc for the alpha needs a lot of work. As you have noticed,
it takes a very long time to compile a kernel. I have a 275Mhz
AlphaStation 2100 with 2G of ram (running OSF/1), and a P90 w/ 32M will
easily beat it compiling the linux kernel. Now, I do not know how much
of this difference is due to the fact that it is harder to generate
optimized axp code, or due to the fact that less time has been spent
optimizing the axp code generation process. Hopefully it is the latter.

If you compare gcc built executables to ones produced by DECs own compilers,
you will usually (but not always) find that gcc produces slower code. It
also can not generate debugging code which contains enough information for
gdb to locate offending statments which produce floating point exceptions.

The bright side of this is that it is all fixable, given someone who has
enough time and knowlege. This could also be quite a rewarding project,
since it should be possible to achieve some significant improvements.
Now all we have to do is wait for this person to show up :-) :-).
