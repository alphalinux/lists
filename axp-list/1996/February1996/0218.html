> m4 sources). However, I'd be interested in seeing the source files that

It should be in the source RPM for sendmail.

> to sendmail/mail on Unix/Linux is there an easy way to marry user names
> with pseudo user names? For example, my external mail address is
>
> david.rusling@reo.mts.dec.com
>
> and reo.mts.dec.com is a gateway that knows that david.rusling is really
> rusling@rdgeng.reo.dec.com. What software would I need to accomplish
> the same thing on Linux?

Sendmail. On reo.mts.dec.com, you create an /etc/aliases with a line:

david.rusling: rusling@rdgeng.reo.dec.com

When you've done that, run "newaliases" to create /etc/aliases.db from
/etc/aliases. Then the sendmail on reo.mts.dec.com will forward mail to
david.rusling to rusling@rdgeng.reo.dec.com.
