> I am attempting to install axp-beta onto an Alphastation 400. I have

> errors from the floppy drive, usually resulting in a kernel panic. It is
> my assumption that is caused by the root partition not being moved into
> ram and thus running off of the floppy.

> MILO: started with "boot fd0:/vmlinux.gz root=/dev/fd0 ramdisk=1440"

My guess is that you are using a kernel built after 1.3.5X, when the
ramdisk format changed. Try this instead:

boot fd0:/vmlinux.gz root=/dev/fd0 load_ramdisk=1
