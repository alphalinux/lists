OK, since Linus integrated pretty much all of my patches into 1.3.65
(thanks! & I hope you aren't going regret it... :-), I made available
a new patch and 1.3.65 kernels for the Cabrio and the Noname. They
both seem to work fine.

The patch is still necessary since there are some unresolved issues as
to how to link modules (the BFD insmod *needs* option -dc whereas x86
users seemed to complain about big files when using that option).
Also, there is a small patch to tga.c that's needed to compile the TGA
code. Linus, maybe I accidentally deleted that patch from the file I
sent you?

Other than that, it's the by now infamous patch to the NCR SCSI
controller that turns off OPTION_DISCONNECT. Yes, bad things can
happen in a system with multiple SCSI devices if that option is on.

The rest is just KGDB patches which you may or may not want, but they
don't hurt in any case.

The files are all in ftp://ftp.azstarnet.com/pub/linux/axp:

axp-diff-1.3.65.gz # patch
vmlinux-1.3.65-cabrio.gz # pre-compiled kernel for Cabriolet
vmlinux-1.3.65-noname.gz # pre-compiled kernel for Noname

Oh, in case you own a Kenetics board: simply select "Platform2000" as
the "Alpha system type" and you should be in business.

Enjoy,
--david
