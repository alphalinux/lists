Had trouble last week with 128 MB RAM on my EV5 system. When
booting from ARC linload gave messages but none from milo
appeared. The problem seemed to be linload.exe. Therefore I
tried to bypass linload by putting milo in ROM.

When I flashed milo into the NVRAM the system booted very nicely
in 128 MB (vmlinux-1.3.48). Milo loads environment variables
(defined in milo) for the memory size, boot partition and file.

At first after I defined the boot variables I left out the
autoboot variable and found that milo could not easily be rerun
in "manual" mode to modify the environment since milo booted to
the HD partition. After erasing milo from the NVRAM from the
debugging monitor I still had the same problem: the environment
variables are stored in a different block than the milo code
and the NVRAM seems to be consulted for all versions of milo
(whether in NVRAM or on floppy). Finally I erased the variable
block and was able to re-flash milo with corrections to the
environment variables.

Now with AUTOBOOT and AUTOBOOT_TIMEOUT defined, I can stop the
automatic boot if I need to change the environment. MEMORY_SIZE
needs to be set since milo (1.3.57) finds only 64 MB instead of
128.

Quite a nice system. My simulator that took 5 minutes or so to
compile on a 486 DX4/100 compiles in about 45 seconds.

Thanks to Dave Rusling and Jay Estabrook!
