} >Personally I think that the Alpha machine (a standard noname in our case)

} Even compared to a simple Pentium Box ?

OK, I stressed the "feel" because that is all I am basing my
statements on.
A Kernel compile takes around twice as long on a 32MB alpha 133
noname, than on a Pentium 75 8MB (and the speed of the pentium
effectively doubles with 16MB). These "figures" are finger in the
air guesstimates. Also I tend to compile using batch and leave the
system to it, so my measurements are rather dependant on me seeing
incoming mail!

When the compiler gets better this will improve a lot.

A while back I tried the byte benchmarks. They took some hacking to
compile (the standard long == 32 bits problem, with a new twist), but
when I did get them going the results were basically that raw compute
power (dhrystones) is good, and the stuff that was OS dependant was
very slow. Overall it came well down the pentium range.

Now there is a bright side to this for people like thee and me that
want to use the things as servers. If it will do the job now, in 6
months time when compiler and libraries get better and the kernel
optomisation is better (look at the i386 string ops for example - all
optomised assembler), the machine will get faster. Your hardware
investment should keep going longer (the flip side is that you are
crippling it at the start).

} As far as web server, do you think is better a forking or not forking server?
} My own software is quite small and doesn't use external libs and so no
} problems.

I would expect apache, with the pre-forking model, to be as good as
anything available for web serving on the alpha.

} >Also, for our application, the inability to reboot is an absolute killer.
} A system that goes down and stays down is not good!
}
} What do you mean? I can't issue a reboot command? Or what?

The system goes through the shutdown procedure OK, but then cannot
bootstrap itself back up again. A manual reset is currently
required. This must be fixable - they have done it for NetBSD, but
then their loader is rather different I imagine.

A side effect of this will be that a crash will hang the system, not
reboot.

} I'm going to order the noname these days...what firmware I've to ask to run
} Linux (Red Hat or Blade distribution!?!??)

Have a look at the FAQ - its linked from the redhat alpha web pages.
I think (from memory - and its not my field), that either will work fine, so you should choose the one with the lowest licensing costs! [NT firmware will work with NetBSD if you want the choice]
