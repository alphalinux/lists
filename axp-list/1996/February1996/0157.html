Hi,

The Token Ring driver has been ported to Axp platform. The driver
was more or less completely rewritten to make memory accesses (mmio
and shared RAM) work. Now we are able to have full TCP IP support over
Token Ring Network. The problem is that a few applications act strangely
(ftp and tin), but our system seems to lack stability (for example if we
CTRL-C we get lots of memory access violations, signals doesn't seem to
work very fine, ... ). So we have a few questions :

- Are jiffies the same as i386 ? (10 ms)
- Are the problems we descibed about stability common ? (We only
have an Alpha Linux since 2 days - kernel 1.3.57)

As we still have these problems, we will wait a few time before
releasing the patch publicly but if someone wants to Alpha test :) we'll
provide the driver.

Many thanks to those who allowed this port :)
