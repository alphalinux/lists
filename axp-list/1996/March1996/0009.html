I believe there is an RPM for gdb 4.15. For some reason it got left
off the final CD-ROM, but it should work. Otherwise, there are
patches on ftp.azstarnet.com.

The things you want to be careful about:

- link with -lg since libc 4.6.27 has no debugging symbols;
gdb will crash if you try to step or break in a function
without debugging symbols (this will be fixed, eventually...)

- don't use -O2 when debugging; due to instruction scheduling
and register allocation, debugging such a binary is a little
tricky (it works, but gdb will jump around in the sources
quite widely and trying to print a local variable will often
result in "no such variable" messages); using gcc options
"-g -O" works well, though.

If you notice any other problems, let me know and I'll add it to my
TODO list.
