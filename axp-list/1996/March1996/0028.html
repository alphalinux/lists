On Mon, 4 Mar 1996, Nigel Metheringham wrote:

> Have you tried (and succeeded) in booting off a CDROM yet - ie
> something like
>
> MILO> boot scd0:vmlinux.gz root=/dev/scd0

It works about 80% of the time, and fails abysmally the other 20% of the
time. You actually need to do "root=0b00" though, as the CD devices aren't
in the kernel's command line parsing.

There doesn't seem to be any rhyme nor reason to this. It works fine on
my DEC Cabriolet box, and fails on my NekoTech one. Of course:

1) Both have 32 MB of RAM
2) Same kernel
3) Same CD ROM
4) Same NCR SCSI

I did use different MILO's, but the kernel boots fine and panics when it
tries to mount the CD on the NekoTech box, so I doubt it's Milo. Jay and
David MT have booted the CD fine on Cabriolet's as well with the same Milo
I use on the broken box.

That said, if you boot off the CD as the root fs, it installs just fine.
