All,
I've updated the Milo stuff on gatekeeper.dec.com.
The new sources are 1.3.76 and _all_ the images in /test-images
are updated (and tested). The new feature is full support for
reboot in all cases. Milo builds a reboot block (as part of
the HWRPB structure) which saves information across boots. The
result of typing "shutdown -r now" is that control returns to
Milo and Milo will start to reboot _exactly_ the same thing
in the same way as the last time Linux booted. There's a 30s
timer that you can stop by hitting any key.

There's an irritating bug that means that the show
comamnd gets the names of the IDE disks/partitions wrong which
I will fix next week. I'll also update the HOWTO next week.

Dave
