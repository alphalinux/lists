Bert,

Sigh. I'll bet this is a relatively new box, or new TGA, right?

I've just seen this problem crop up. If you do "cat /proc/pci", I'll bet it
describes the TGA like:

Display Controller: DEC DC21030 (Rev 3).

The current TGA server code has a problem with the Rev 3 TGA chip; something
changed in there between Rev 2 nd Rev 3, that I haven't figured out yet.

With the *first* BETA server (0.0), this crops up a lot, as the cursor
is done via software emulation, and apparently those functions encounter
the difference in the chips.

There's a second BETA server (0.1) on gatekeeper now, that uses HARDWARE
cursor support, so the problems are not quite so obvious or frequent. You
can still encounter the problem in other ways, "xmag" being a common one.

Make sure to change your XF86Config file to *remove* the 'Option "sw_cursor"'
line.

I'm actively working this problem, so hopefully, I'll have a fix soon.
