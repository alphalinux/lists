I've just built new GCC and libc packages for the Alpha.

They are available from ftp.redhat.com in /pub/devel/axp/RedHat/RPMS as:

gccaxp-2.7.1-2.axp.rpm
gccaxp-c++-2.7.1-2.axp.rpm
gccaxp-objc-2.7.1-2.axp.rpm
libcaxp-0.40.3-1.axp.rpm
libcaxp-debug-0.40.3-1.axp.rpm
libcaxp-include-0.40.3-1.axp.rpm
libcaxp-profile-0.40.3-1.axp.rpm

They contain the following changes:

1) limits.h in /usr/lib/gcc now defines LONG_MAX as a long
instead of a (long long). I don't know which is correct,
but the (long long) way causes things like GNU textutils
to fail to build.
2) removed BSD curses and termcap from libc. You should use
ncurses as the BSD curses replacement, and the libtermcap
package for termcap (it is available from the same place)
3) cleaned up stdlib.h so you can build things with __OPTIMIZE
and __BSD turned on without gcc failing

David MT, if you want to do a new libc release, feel free to base it
on the libcaxp source package at ftp.redhat.com:/pub/devel/SRPMS. If
you don't want to make one, will you make my tarball available on
azstarnet?
