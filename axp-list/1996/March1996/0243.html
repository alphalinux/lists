We had two distinct problems getting RedHat to run on the EB164, neither
related to RedHat. After fixing these everything was suprisingly easy,
including X Windows.

1. Memory requirements: the memory modules MUST be true parity, not the
simulated parity type (known here as BP types). This is a requirement
for the hardware, not MILO or Linux. We also found that "PD bits" on
the modules had to be set. These tell a system the size (=3 bits)
and speed (=2 bits) of the modules, and are set by small surface
mount resistors. Most modules don't seem to have them installed,
although most modules DO have the pads for them.
These resistors can be added by anyone with good eyes and a
steady hand. (I suspect that 8MB modules will work without them
because that is the proper bit setting for that size
module.

2. The MILO that came on REDHAT 2.1 (axp) would not boot. I obtained
the latest test image from
gatekeeper.dec.com/pub/DEC/Linux-Alpha/milo/test-images, and that
seemed to do the trick.

3. RedHat works quite well though you will find that many things
related to graphics and the video card don't work on the EB164
though they do appear to work on other systems. The source of the
problem is that the EB164 handles its IO somewhat differently from
most older 21064 boards, and is due (I think) to chip architecture.
Svgalib, or programs built with it, can't seem to find the video card.
(I am still looking for a version of svgalib for the
EB164) Along the same lines there is a special verion of X Windows
required for the EB164. It also can be found in the above Linux-Alpha
directory.
