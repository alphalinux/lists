OK, since the older sysklogd (version 1.2) had some problems, I
thought I'd try the new release and it does indeed seem to work much
better. Specifically, whenever I'd dialed out via pppd, all other
syslog activity blocked. E.g., an "su" at the same time would block
until pppd finished execution. Thus, if pppd ever got stuck, the "su"
would never complete unless syslogd was restarted. I seem to recall
other people seeing similar problems. If you did, please try
sysklogd-1.3 after applying the following patch. So far, I haven't
seen any problems. The patch is at:

ftp://ftp.azstarnet.com/pub/linux/axp/axpsrc-sysklogd-1.3.patch.gz

Also, it looks like my earlier klogd patch never made it into the Red
Hat distribution, the above patch will also make klogd work.

I cc'd this mail to Greg Wettstein since I thought it would be nice if
the above patch could be integrated into the next release of sysklogd.
Except for the change that adds `-linux' to the klogd linker-command,
none of the patches should affect the x86 platform negatively. In
fact, there is a bug fix that's also needed for the x86.

Enjoy,
--david
