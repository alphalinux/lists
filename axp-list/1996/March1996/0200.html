>
> Ok, We're having the following problems with X and networking and we think
> that they're related. First, X refuses to come up even though the Xconfig
> file is correct, all we get is a blank screen. We can kill the X terminal
> with ^+Alt+backspace as usual.

> Any suggestions? We're working on an alphastation 200 with the ZLXp-E1
> video card, we have the correct xtga Xserver (the beta0.2 version) and
> we're using the Xconfig file that came with it.

OK, two things:

1. Did you check the XF86Config file "MemBase" address for correctness?
That is, is that address the same as you get for the 21030 device when
doing "cat /proc/pci"? THEY MUST MATCH!!!

2. Did you check that the monitor you're using can do the selected resolution?
I believe the config file comes set-up for 1024x768 @70Hz, or close to it.
If your monitor won't do that resolution/refresh, it'll be blank; I know,
I've done it... :-(
