Its me again. Just uploaded the following patch (relative to 1.3.71):

ftp://ftp.azstarnet.com/pub/linux/axp/axp-diff-1.3.71.gz

It contains:

- Fix to arch/alpha/kernel/time.c to adjust for lost timer
ticks. A better solution is of course to write drivers in such a
way that this won't happen in the first place, but since IDE isn't
gonna go away any time soon, I think we need to be able to tolerate
them. There is a very small window during which jiffies still might
not be updated correctly, but the probability of this happening
should be *really* small. And, if you disable interrupts
longer than say, 10-20 seconds or so it won't work either... :-)
There is a related fix to include/asm-alpha/param.h. The HZ
constant was defined incorrectly for the EB66 and EB66P (mea
culpa---one day I'll learn not to trust comments...)

- A fixed driver for the IBM Token Ring. This patch is by
Yann Doussot <doussot@via.ecp.fr>, Pascal Andre <andre@via.ecp.fr>,
and Michel Lespinasse <walken@via.ecp.fr>. The driver reportedly
works fine both on Alphas and x86 platforms. Please send
questions/bugfixes etc directly to them since I don't have such
an adapter and can't do any testing (not that I wanted to :-)

Oh, the precompiled kernels are back as well: same directory as above,
files vmlinux-1.3.71-cabrio.gz and vmlinux-1.3.71-noname.gz.

Enjoy,

--david
