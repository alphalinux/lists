Matti, and all,

There are (or will be, later today), a newer MILO and kernel for true XL's
on gatekeeper.dec.com, in pub/DEC/Linux-Alpha/X11, as:

milo-1.3.76.xl /* MILO */
xl-1.3.77.gz /* kernel */

These should be used instead of the earlier ones based on 1.3.75. They raise
the memory limitation to 96MB. This limitation is strictly in support of
floppy use (actually, generic ISA bus DMA access to main memory); if you can
live without floppy, you could run a stock AVANTI kernel (with an XL-related
keyboard patch), and put in as much memory as you can afford... :-)

Note that you *can* use "set MEMORY_SIZE 96" to MILO, with more than 96MB
installed, and it will work appropriately. I've successfully built kernels
with 80MB in my XL-233, and it used them all... :-)

I'll examine this limitation in detail in following mail, and request some
input from the net at large for possible work-around schemes.

Thanks in advance.
