matthew@sleeper.apana.org.au said:
} There is a patch mentioned in the doco about OSF/1 that needs to
} applied, or it cant resolve the names in the access control files to
} addresses. Its a 32/64bit int problem ( again...). Some one decided
} that IP addresses are the same size as int's.

IP addresses *are* the same size as ints on all the systems I use at
present - its people making assumptions that they are the same size
as longs that causes trouble. Use one of the bittypes definitions
(uint32), or the defs in the network include files.
