Ok, there is (or soon will be) an even *newer* TGA Xserver on gatekeeper,
found as:

pub/DEC/Linux-Alpha/X11/axpbin-Xtga-BETA-0.2.gz

This has a fix for the problems those with Rev 3 TGA boards have been seeing.
It has been tested on both Rev 3 and the older Rev 2 TGA chips (Rev 2 is in
all UDB's I've seen).

Some other problems have been cleaned up since the BETA-0.0 version, but you
still should avoid toggling between X and VC's; it's *MUCH* safer to simply
exit from X and then restart it if you really need to get to a VC...

Also, please *do* remember to remove the 'Option "sw_cursor"' line from your
XF86Config file.

Note also, that it's possible that different kernels (actually the PCI BIOS
code therein) place the TGA memory at *different* PCI bus addresses; make
sure to check via "cat /proc/pci" and adjust the "Membase" line in the
XF86Config file. If you start X, and it freezes/crashes the machine while
coming up, it's prolly this problem...

Good luck, and again, let me know of any problems...
