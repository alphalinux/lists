> I have an Aspen Alpine 275 w/ 4GIG HD, 64 Megs RAM, #9 pci video, dec
> 'tulip' pci ethernet (TP).
>
OK, that looks close enough to a CABRIOLET to be possible, so...

> I have the install tree NFS mountable, and I am using the 1.3.57
> boot image, as the 1.4.43 does not talk to the ethernet properly.
>
> Anyways, I get to the point that the install wants me to put the boot
> disk back in, and as soon as I remove the disk I get all sorts of errors,
> and the system eventually hangs. (due to no root disk)
>
> I *think* that for some reason, the ramdisk is not being used.
>
> Any suggestions??
>
> I only have this box as an evaluation, and I need to get it going
> TODAY!!
>
> Many thanks in advance.
>
> PS - I am using milo and the stated boot line (from the docs)
>
> boot fd0:vmlinux.gz root=/dev/fd0 ramdisk=1440

Note that the RAMdisk activation *CHANGED* with 1.3.48 or thereabouts;
you need to say:

boot fd0:vmlinux.gz load_ramdisk=1 root=/dev/fd0

instead...

Let us know if it works!!!
