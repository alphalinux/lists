On Mon, 25 Mar 1996, David Mosberger-Tang wrote:

> There seems to be an instruction scheduling bug in that version of
> gcc. The bug shows if -O2 is used. For example, the X11 library
> function XGetRGBColormaps() triggers this problem (the `property'
> argument passed to XGetWindowProperty will be wrong because of this).
> The result is that a "Bad Atom" error is reported when trying to call
> this function. I'll report this bug to the gcc maintainer. In the
> meantime, it may be safer to use -O only.

Of course, -O2 won't work if you're building something that requires
inline functions :-)
