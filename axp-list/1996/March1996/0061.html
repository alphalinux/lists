>
> However, if the community feels that:
>
> o it is better to get started now with a partial standard, and evolve
> later
> o Linux should be used as a "prototype" for the standard (a use for
> Linux of which I heartily approve)
>
> then I will ask the Digital Engineering group to help locate a copy of the

> After all, if Linux can handle both a.out and ELF, how hard can it be to handle

Actually that's not a problem. Binary "personalities" are the way Linux
handles a multitude of formats for iBCS on Intel, the most recent kernel
patches have done the same thing (for the most part) with scripts. It all
comes down tlike you said, to magic :-) (in recognizing the file, that is).

I know I'm new voice, but one of the reasons I got an Alpha was to play around
with shared libs and bin formats. I've been reading the kernel source and
had a little experence with both BFD and COFF (on bare m68k boards), so
I offer a willing and (hopefully) capable warm body to help code the stuff
if you can find the spec.
