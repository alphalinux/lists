Hello all,

Any EB164 people have any experience or ideas on the following?

I have an EB164 running with Redhat 2.1 and the latest
milo test image from gatekeeper.dec.com. I am trying to do some
programming and am seeing a trend.

1. Software that has been specifically ported to the EB164 runs
and can talk to the hardware. This includes MILO, the EB164
kernel, and the EB164 X Windows port.

2. Programs that don't talk directly to the hardware run too.

3. Other axp software in redhat 2.1 that needs hardware access
doesn't run and causes memory violations. This includes
the graphics libraries, programs compiled with the graphics
libraries, and even the io portion of libc.a (with _inb() etc).

4. This is due to the libraries not yet being ported to the
EB164 hardware environment?????


Any help greatly appreciated.

Allan
