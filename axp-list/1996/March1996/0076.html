Federico,

>How can I use the serial console instead of usual vga/keyboard on a NoNmae!??!?
>
>What are the setting of the serial port?? (a or b???)
>
>I want to capture the boot of my linux to catch all the errors which make it
>sopping!

I have a set of patches, relative to 1.3.69, that I ported to the Alpha
from patches made available by Dominik Kubla <dominik.kubla@uni-mainz.de>,
that implement serial console. With these patches you configure serial console,
and *ALL* normal kernel printk() and init() messages come out to the serial
port (COM1, or ttyS0, although this can be configured as well). These are
not just echoes of messages, like the SERIAL_ECHO option. They truly make
the serial port
the system console. You just have to remember to configure a "getty" for the
serial port before you re-boot... :-) You can even use the VGA VC's, even X
if you do some magic ("-vt" command line option... :-), that is, if you do
*have* a VGA... :-)

Let me know if you want these made available. I *did* send this set of patches
to Linus, but don't know their current status, ie if they're scheduled to go
in to a later revision, or if they've been rejected.
