>>> Aaron said:
> I'm not sure what I did, but when I updated to the latest TGA X-server
> (for the UDB) I was unable to run X windows unless I was root?
>
> That seems rather strange. The only files I changed were the actual X
> server and the /etc/XF86Config file. What permissions do I need to set to
> allow me to run X windows from any-body that is logged in. (Like it used
> to be!!)
>
> Also does any one have the mode lines that will work with a NEC
> MultySynx 3v. I can get 640x480 and 800x600, but no resolution above
> that. I tried the mode lines that were given for a sony, but to no avail.

The XF86_TGA binary must be setuid root; ie "chmod 4755 XF86_TGA".

Did you try activating each of the 1024x768 Modelines, starting with the one
with slowest refresh rate?

Did you update the XF86Config file with *accurate* information about that
monitor, ie the horizontal and vertical refresh rate limits? This is critical,
not only for correct operation, but to protect your monitor.
