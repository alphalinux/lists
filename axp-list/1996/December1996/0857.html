This utility is used by various sample desktop configuration files
but is nowhere in sight. Even worse, even sources - which used to
reside on tsx-11.mit.edu disappeared. :-) To be fair, you could
have possibly problems in compilation as they used to hack directly
a kernel space.

To remedy the problem I put xload-1.36p.tar.gz on sunsite.unc.edu
and two rpm archives (xload-1.36p-0.src.rpm, xload-1.36p-0.axp.rpm)
on ftp.redhat.com. They are modified to read /proc/loadavg.
xload-1.36p-0.axp.rpm is compiled for axp/ELF. Use spec file
from xload-1.36p-0.src.rpm and rpm to recompile this on other
Linux platforms.

Have fun,
