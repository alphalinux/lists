>
> "Configuration alpha-unknown-linux (is) not supported" with gcc2.7.2.

Well, this is not exactly true. :-) A number of Linux-axp machines
running using this compiler provide a counterexample. Likely a mismatch
is some name strings.

> 1) Is it correct that g77 is not ready for Linux-Alpha yet?

Looks that way so far. But people are working on it.

> 2) Can fort77 optimize fortran codes and how is it's performance?

Sure it can. This is just a front end to f2c/gcc combo and many
optimizations can be performed. The problem is with libraries, although
the stuff which comes with RH 4.0 is reputedly much better. I do not
have a personal experience with them.

> 4) Would NT 4.0 be a better choice for PVM&fortran combination ?

If you are willing spent money on a Fortran compiler then check
www.microway.com (and bug www.absoft.com).

> 5) Is Linux/Alpha binary compatible with OSF1/Alpha ?

Yes, but this will not help you very much as you will run into
licensing and shared libraries issues. Anyway, UDBs are not, by today
standards, high performance machines - does not matter which OS.

> Btw. I have installed RedHat 3.0.3. and could not make ftpd
> let anybody log in( yes I have edited ftpaccess file).

You probably miss '/bin/sh' in /etc/shells. ftp will not work witout
it.

> Furthermore I keep getting the message
>

This is a warning, but it slows you down.

> Worst part is that I am not exactly involved with Linux and C :-).
