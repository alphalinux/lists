There is now a beta version of a linker capable of multiple got
subsections, which allows for much larger applications.

Getting set up to test this sucker is not for the faint of heart,
so don't just everyone rush to install it. You have been warned.

The problem is that, in fixing a long-known bug in the linker, I
found one in the assembler. Meaning that all static object files
have to be recompiled. Along with the binutils binaries, I provide
new versions of crt*.o and libgcc.a, but this does mean that any
other .a library is tainted. However, as long as you stick with
existing shared libraries, everything will be fine.

So, hanging out in ftp://ftp.redhat.com/pub/alphabits/ you'll find:

binutils-961205-axp-bin.tar.gz
binutils-extra-961205-axp-bin.tar.gz

The installation should begin, of course, by backing up existing
programs:

(tar ztf binutils-961205-axp-bin.tar.gz ; \
tar ztf binutils-extra-961205-axp-bin.tar.gz ; \
cd /usr; echo lib/gcc-lib/alpha-linux/2.7.2/crt*.o ) | \
grep -v '/$' | \
(cd /usr; tar -cvf /tmp/backup.tar -T - )

Then install the new files under /usr, delete the crt*.o files
that came with gcc (they are now part of libc as with the intel
boxes), and be happy.
