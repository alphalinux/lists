When I install Redhat 3.0.3 from scratch on my 233mhz Multia, I sometimes
get the following error message after typing from the milo prompt:

boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1

milo queries the system and prints out various diagnostic info about my
machine. And then it reports:

#lca machine check (la = xxxxxxxxx, pc = xxxxxxxxx)
reason: access to non-existant memory (long frame)
reason: load xxxxxxxxx exc_addr xxxxxxxxx dc_start xxxxxxxxx
car xxxxxxxxx

write error to b_cache occurred at xxxxxxxxx

(the xxxxxx's are addresses that i didn't have time to jot down)

The installation appears to continue without problems but I'm curious if
this suggests that there might be something wrong with my 512K cache. Is
there any utility under linux that I can use to determine if the cache is
defective? The ARC console reports that the 512K cache is installed, but
I do not think that it does too much error-checking. Anyone encounter
this problem during installation?
