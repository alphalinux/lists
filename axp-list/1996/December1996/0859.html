RH 4.0 uses /etc/man.config to define search path for 'man', but 'xman',
at least for axp, still relies on $MANPATH. Would be nice to have that
mentioned in Errata, or something like that.

A tip: if you have current /etc/man.config, then put in your
/etc/profile
MANPATH=`man -w` ; export MANPATH
to keep both automatically in sync. Without that xman is unable
to find its own documentation as it searches only subdirectories
of /usr/man. :-) There can be other programs which want to see
well defined MANPATH as well.
