On Fri, 29 Nov 1996, Miguel Avillez wrote:

>>
>> hello
>>
>> Have this question:
>>
>> What is the fastest OS installed in a UDB with a 233 MHz cpu?
>> RH-4.0 or Digital unix 4.0?

> Digital Unix, by far. Linux is still a "baby" OS when it
> comes to the Alpha archetecture. It's only been out for a
> year or so now. GCC's compiler and libs are not nearly as
> optimized as DUNIX's libraries and compiler is. If you can
> afford the steep price for DUNIX, though.
> Linux's advantage is that it's free. 8< )}

I would agree if we were not talking about a UDB. One of the
advantages of Linux is that it takes FAR less ram
to run than Digital Unix. If I remember correctly, one of the
features of DU 4.0 is that it will install on a machine with
only 24M of ram if you dont use it as an NFS server and don't
do several other things. I don't know how much ram a UDB has,
but if its not much, you may be better off with Linux. The UDB
also has rather small caches, which would also tend to make
something that used less memory run faster.

Perhaps the fastest combination would be to compile the
executables under Digital Unix and run them under Linux. This
way you get the well optimized compilers and libraries from
Digital Unix, and you get the small fast kernel from Linux.
