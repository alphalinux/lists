Michael,

>bogomips is *totally* bogus to measure machine throughput in actual use.

But it is a very accurate measure of how fast a machine is at doing BogoMIPS !!

>It is a meaningless number, heavily influenced by instruction scheduling
>in a tight loop. The results are very machine dependent.

Because of this tight loop, and the internal cache on Alpha (or most other)
CPUs, you might expect the BM value to be directly related to the
clockspeed, when the CPUs are the same type (21164, EV5 in both cases). If
executing the code has to wait for some off-chip event, it is possible that
the faster CPU loses more cycles, breaking the linear relationship, but I
cannot see that happening in the case of the BogoMIPS-loop.

>
>A 486DX2/80 shows 39.??, while my P133 512k cache 64MB machine shows in
>the high 50s. Is it less than twice as fast? I'll let you make a guess
>as to the answer. My UDBs (266 MHz, 56MB, 512k cache) show 261.62, and
>they are much slower than the P133.
>

Did you notice that even on your 21066 BogoMIPS is very close to MHz. As
long as you don't access memory even a UDB is a very fast machine ! However,
as you said, this will apply to very few real-life applications.
