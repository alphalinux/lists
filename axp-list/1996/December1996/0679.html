>>> John Banghart said:
> I have an AS255 running Redhat 3.0.3. Currently, it boots of a floppy
> which contains the kernal and aboot.
>
> This is fine except that SRM console won't boot this automatically,
> which means I can't do a remote reboot.
>
> How do I get SRM to boot from the floppy with human intervention? Also,
> is there any reason why I should use MILO instead of this system?

Sorry, but you can't do a remote reboot on *any* of our platforms using SRM
console. If this is a necessity, you must use ARC/MILO to boot the kernel.
This leaves (at the moment) any platform using AlphaBIOS out of luck. It may
be possible to boot SRM->MILO->LINUX, and have LINUX return gracefully into
MILO, which (may be able to) reboot successfully. Worth a try.

Cold reboot, after power failure or power cycle, *can* be setup to
automatically boot into LINUX, via ARC or SRM.

Under SRM, for auto-booting off of floppy, setup the environment variables:

set bootdef_dev dva0
set boot_file vmlinux.gz
set auto_action BOOT #I think this is right; anyone?

Then simply type "boot", and it'll go.

Consult your ARC or SRM console docs for the last word... :-)
