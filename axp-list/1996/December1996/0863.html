I recently upgraded my video as well. I am using a Diamond Stealth 64 Video
VRAM 2M (3240), and am getting 16-bit color @ 1024x768. It is a pretty fast
card, but barely fits in the Multia case. I had to remove the floppy
momentarily to install the card in the slot. If the card was 1 mm longer, it
couldn't fit.

Using the XFree 3.2 S3 driver, the image is nice and stable. I was getting
some strange color problems, mainly in the shadow of 3d widgets with the 3.1.2
driver, so using XFree 3.2 is recommended. The server is so stable, I have had
no problems running Netscape 3.01 via remote X session from my Intel Linux
box. This has been problematic with earlier TGA server, although the latest
one on Digital's site is pretty stable with Netscape as well.

The Multia ARC console and MILO recognize the card at boot. The main thing to
prepare for is to compile a kernel image that has TGA support turned off prior
to installing the card. Otherwise, output will go to the TGA device rather
than VGA when MILO boots the Linux kernel, resulting in no console output.

>From what I could see, any other S3 alternative was a DRAM-based card, which is
a bit of a step-down in performance compared with the on-board TGA. In fact,
even the S3 VRAM board at 16-bit color is a stepdown in speed compared to the
TGA at 8-bit.
