On Fri, 13 Dec 1996, Eric Jones wrote:

> Milo works fine. When the kernel boots, nothing is sent to the vga card but
> I can hear the HD hammering away and loading linux. The question I have is:
> Are there any command line arguments I need to pass to the kernel at boot
> in order to get it to use the VGA card?

You need to build a kernel w/o TGA support. The kernel w/ support for both
uses the TGA if it can find one (as it's much easier to look for a TGA
on the PCI bus then a VGA god knows where).
