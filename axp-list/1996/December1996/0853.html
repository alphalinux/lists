Michal Jaegermann <michal@ellpspace.math.ualberta.ca> wrote:
> For that use "mount -n". You do not **want** to update /etc/mtab
> on read only file system. "cat /proc/mounts", which you should have,
> is a good source for the same information.

/etc/mtab is a never-ending source of frustration for me. I don't understand
why it exists at all. Surely the mount command doesn't really need it, because
the mount system call will reject improper mounts (or at least it should).
And the umount command shouldn't need it either; it should just do what the
heck I tell it.

I suppose it wouldn't bother me nearly so much if the mount and umount
commands simply made a "best effort" to keep /etc/mtab up to date, but didn't
actually depend on it. This would satisfy people who don't use /proc/mounts
for whatever reason. Or at least if the commands had options to ignore
/etc/mtab, so I would be able to get out of sticky situations. I thought
part of the "Unix Philosophy" was that it provided "enough rope".

Cheers,
Eric
