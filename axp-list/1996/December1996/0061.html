I have my new UDB running RedHat 4.0 with kernel 2.0.27, built using
axp-diffs-2.0.21.gz.

Everything looks fine, apart from two things:

1) Running ping from any machine to any other machine on the network
yields three duplicate responses. This is not the case when the UDB
is down, or one of the machines *is* the UDB.
Traceroute's are also confused when the UDB is on the net.
In case it is relevant, I have not configured multicast into the
kernel. There are no duplicate IP addresses on the local net.

2) The network card seems to be cycling through all its interfaces
(AUI, BNC, 10BaseT) all the time.

Any ideas ?
