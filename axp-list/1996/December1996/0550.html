Hi, I've been trying to install Linux on an Alphastation 500/333. I've
been having some difficulty. I can get MILO installed without too much
problem (booting off floppy) and when I come to use the 'alcor' kernel
boot disk it fails.

I'm booting from the SRM console
| 'Digital AlphaStation 500/333 Console V5.0-13'

Milo loads, but gives an error message:

| dev_config: don't know how to set hostid of SCSI device (1077:1020)

and then reports system configuration:-

| MILO:
| Built against Linux version 2.0.22 (rusling@linux.reo.dec.com) gcc
| version 2.7.2) #11 Fri Oct 18 18:01:31 GMT 1996
| Video is Digital BIOS Emulation (VGA), Zlxp (TGA)
| Device drivers are IDE, SCSI (NCR 810, Qlogic ISP, Buslogic),
| Floppy
| Memory size is 32MBytes

The actual memory size is 128Mb

I then try booting the kernel boot disk with

| boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1

This seems to work until the kernel has finished and reports:-

| Bootstrap complete, kernel @ 0x310000, sp @ 0x304000
| ...turning on virtual addressing and jumping to the Linux Kernel

Where upon it hangs, and does nothing...

Am I doing anything wrong?
