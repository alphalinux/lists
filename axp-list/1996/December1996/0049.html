> What files needs to go boot disk, and what files needs to go root disk.

Read the Red Hat installation guide on the Red Hat CD but you need:

(a) a ramdisk image,
(b) a MILO floppy with MILO + linload.exe on it
(c) a floppy containing your kernel file named as vmlinux.gz, Red Hat
will copy this to your root disk during the installation

> Is there way to boot off of CD rom once I'm in Milo?
>

Yes, read the MILO HOWTO, its the

MILO> boot -t iso9660 scd0:/

command.

> In the boot disk, only thing I have is linload.exe and milo, but how can I
> boot with boot command to execute vmlinux.gz?(it's not hidden file is it?)

Read the Red Hat installation guide but basically you need to put
vmlinux.gz on another floppy that MILO and the Red Hat installation can
read. MS DOS or EXT2 is fine.

> and this bad magic has no FAQ or explanation in any Alpha-Linux homepage,

> appropriate, please re-post this). Thank you every much.

I beg to differ, the problem solving section of the MILO HOWTO has the
following to say:

MILO complains that the kernel image has the wrong magic number

Older versions of MILO did not support the ELF object file format and
so could not recognise an ELF image and this might be your problem.
If this is reported, upgrade to the latest MILO that you can find.
All 2.0.20 and beyond MILOs support ELF. On the other hand it could
be that the image is indeed damaged. You should also note that MILO
does not yet automatically distinquish between GZIP'd and non-GZIP'd
images; you need to add the ".gz" suffix to the file name.
