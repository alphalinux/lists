>hi, guys, I am running redhat4.0 on an udb, today I use gcc to
compile
>a program and got such error messsge:
>gcc: Internal compiler error: program cc1 got fatal signal 6
>
>What does it mean? It is my own problem or gcc's bug? What shd I do?
>Thanks in advance.

The compiler itself is at fault. It is gcc's bug. I've run into this now and then compiling using g++. So far I've been able to work around this problem by turning off the optimization flags for the objects that it has this problem with, but a more correct answer would probably be to get and compile a newer version of the compiler.
