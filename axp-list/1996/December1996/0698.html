When I use the login command, from root, I get a the message:

login: memory violation at pc=155558d2944 rp=15555b6ccc4 (bad
address=0000000)

The "login:" is actually part of the message, not the login prompt itself
(which is also there,preceeding the message, so it actually starts
login: login: ...). After I hit enter, the login prompt appears again, and
I can login with my user account.

Is this a known bug?

I'm running a freshly-installed RH 4.0 on a UDB; kernel is at 2.0.18, and
rpm -qf /bin/login reports util-linux-2.5-26.

The RH web page
http://www.redhat.com/support/docs/rhl/rh40-errata-general.html lists
errata for "pam, util-linux, NetKit-B", including a URL to a
util-linux-2.5-28.axp.rpm, but none of the symptoms listed seem to match
this. There's nothing like it on the alpha errata page, either.

Possibly related symptom: under X window, when I use login, the shell I
execute the command from simply disappears. This is what really bugs me,
as it prevents me from logging in as two users at once. I'm guessing (but
just guessing) that this is a different symptom of the same problem.
