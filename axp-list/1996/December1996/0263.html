>>> Iwao Jack Makino said:
>
> Is there any better XSERver for TGA? can't use virtual desktop and can't
> change the resolution is kind of a pain in X with small monitors.

Virtual desktop and resolution changing are non-trivial on the TGA. Virtual
desktop may be too hard to do because of hardware features/limitations, and
resolution changing on-the-fly requires virtual desktop before it becomes
possible. But have a go at doing it, as there are sources available for the
XFree86 3.2 release's TGA server, and docs available on the TGA (aka 21030)
at:

ftp://ftp.netbsd.org/pub/NetBSD/misc/dec-docs

Source code is *not* available for the server based on XFree86 3.1.2. Less
highly-optimized code was given to XFree86 by DEC, but is not actually used
in the 3.2 release, I believe. The current 3.2 code relies on the server CFB
routines and treats the TGA as (mostly) a dumb framebuffer, so there is much
that could be done to improve its performance. Anyone interested, get in
touch with XFree86 via xfree86@xfree86.org, and volunteer for this effort.

As mentioned above, the 3.1.2-based versions use some highly optimized code
for wringing the most out of the TGA hardware. If speed is a concern, and
you don't mind not having sources to look at/play with, choose these. As an
apples vs oranges comparison, I got an Xmark of 6 on a UDB-166 with a 3.1.2
based TGA server, and on a 275MHz CABRIO (EV45) with S3-968 VGA under 3.1.2,
an Xmark of only 4.2 (the 3.2 S3 server would do better than this, I
believe). And the 3.1.2-based TGA server improved along with CPU
horsepower: on an XL-266 (EV45), an Xmark of 11, and a 266MHz EB164 (EV5),
the Xmark was over 16.

All that said, the currently available TGA servers are:

1. XF86_TGA from the XFree86 3.2 release, available on ftp.xfree86.org and
its mirrors. This is available in both ELF and ECOFF formats, I
believe. It has a bug which seems to appear most frequently when viewing
certain web sites via Netscape, and can crash the server. This server
will run on *all* Alpha platforms.

[The below are available on gatekeeper.dec.com in pub/DEC/Linux-Alpha/X11,
and possibly on mirrors andd elsewhere.]

2. axpbin-Xtga-BETA-0.2.gz and axpbin-Xtga-BETA-0.2-EV5.gz are ECOFF versions
based on the XFree86 3.1.2 release. These have the "netscape" bug, and some
font rendering problems. The EV5 version is for Alpha platforms using the
EV5 or EV56 CPU (like EB164, AS500, AS600, PC164, XL-300/366), and should
*not* be used by platforms with EV4, EV45 or LCA (like the UDB, CABRIO,
AVANTI, XL-233/266, NONAME).

3. Xtga-ELF-EV4-test.gz and Xtga-ELF-EV5-test.gz are ELF format only, and
are based on XFree86 3.1.2 plus some font-related bugfixes. These still
exhibit the "netscape" problem. As above, pick the EV4 or EV5 version as
is appropriate for your platform. These appear also on the RedHat 4.0 and
Craftworks (?) distributions, I believe. They are superceded by the servers
listed below, and their use is deprecated.

4. XF86_TGA-3.2-bugfix.gz is based on the XFree86 3.2 release, but with
the "netscape" problem crashes fixed (there is still an occasional site
rendering problem, but I've seen no server crashes). Available in ELF
format only, and works on all Alphas.

5. XF86_TGA-3.1.2-bugfix-EV4.gz and XF86_TGA-3.1.2-bugfix-EV5.gz are based
on the XFree86 3.1.2 release, plus font-rendering and "netscape" bugfixes
(the site rendering problem is still present). Available in ELF format
only, and pick the version appropriate for your platform's CPU type.

Summary:

If you are running an ECOFF system, ie RedHat 3.0.3 or Craftworks 2.1, either
#1 or #2 from above. Sorry, at the moment there is no version available in
ECOFF format that fixes the "netscape" bug.

If you are running an ELF system, #4 or #5.

As always, let me know of any problems with any versions.
