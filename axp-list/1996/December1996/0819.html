>
> We just got a PC164 which I've successfully installed linux on. There
> are a few problems. It crashes quite often, I usually can't finish a
> kernel build without a lockup. I've tried to build two kernels for it
> so far, one 2.0.21 and the other 2.1.1. When I tried to boot from
> either of them, the system would hang right after milo got finished
> printing the # marks. They were both gzipped. I tried to boot the
> 2.1.1 kernel uncompressed, and it gave me an error message along the
> lines of 'got 2 program headers, expected 1'.

Use 'make boot' and take either the stripped image (vmlinux) or the
gzip'd stripped image (vmlinux.gz) and boot with those. By the by, the
latest stable MILO (2.0.27) on gatekeeper will work out if it is zipped
or not and fixes a bug that wouldn't boot unzipped images.

> When I tried to rebuild

> Should I be using the 2.0 or the 2.1 kernels? What can I possibly do
