I threw a Diamond Stealth 3D 3000 into my UDB last nite (tight fit!) and had
a problem when booting into both Red Hat 4.0 and NT Server 4.0. The initial
firmware screen comes up fine, but when the o/s is moving into a different
graphics mode (Linux - at kernel boot / NT after the inital "4.0 loading..."
message) the monitor does squat. For the hell of it, I plugged the VGA
cable back into the TGA port and both everything was on screen as expected.

So.. it looks like as soon as each o/s tries to pop into a graphics mode, it
throws its output to the TGA card instead of the Diamond Stealth.

Has anyone else run into this?
