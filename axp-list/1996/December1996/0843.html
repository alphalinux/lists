> I am growing quite frustrated after battling for an entire evening to
> produce a minimal root diskette for emergency purposes.

Although it would be nice to have a real emergency set if you have Red
Hat 4.0 installation CD (it could be from "Archives" set, not from the
whole package) you can boot from it typing in milo something like that:
boot -t iso9660 scd0:kernels/noname.gz root=/dev/scd0
Start an upgrade and when media are chosen switch to the second console.

Admitedly when I needed that I was able to change to a troubled disk
in question (even "chroot bash /mnt" was possible). I do not know how
would I fare with only what is available on this CD.

> - Something during the root-mount phase of the boot complains about
> "can't create /etc/mtab~, read-only filesystem..". (???)

For that use "mount -n". You do not **want** to update /etc/mtab
on read only file system. "cat /proc/mounts", which you should have,
is a good source for the same information.
