> I'm installing Linux on a new UDB, which I haven't powered up yet, since
> I've just gotten the disk, RAM and other components recently. I'm putting
> up RedHat 4.0
>
> Can someone please give me the quick tour of what SRM, ARC and MILO are,
> and the distinctions between them?
>
> Here's my (mis)conceptions so far; corrections are solicited:
>
> 1. SRM is the default console;

Nope, SRM is just a console. It is needed to boot VMS or Digital Unix and Linux
will run over it too. It is Digital proprietory and eats around 5Mbytes of memory
(it stays memory resident after linux has booted waiting for linux to make callbacks
that it never does).

> 2. ARC is another alternate console, but the documentation I've seen
> leads me to believe that it's ordinarily used only with Windows NT, and
> isn't applicable to Linux.

It is only used with WNT. The PALcode (sort of microcode) that it exports is
completely unapplicable to Linux - it runs in a different addressing mode and
provides different services. WNT ARC is Microsoft proprietory. It also stays
memory resident after booting WNT so that WNT can make callbacks.

> 3. MILO is a loader comparable to LILO, but that also loads some sort of
> microcode. It runs in conjunction with either SRM or ARC, but not in
> place of them.

Actually it takes complete control of the system and so does linux when it
runs. It takes back all of the memory that was occupied by either WNT ARC or
the SRM and it regards both of those as just a way of getting control.

It does, however, bequeath correct PALcode to Linux when it is running.
If you drew a block diagram, you'd have a wide thin block called palcode and
above that a tiny block called milo and a big block next to it called Linux.
Linux does not use any console services once it has booted and linux is kept in
memory only as a gzip'd file. When you type 'shutdown -r now' linux transfers
control back to MILO which ungzips itself and starts all over again.

MILO stays memory resident in a zipped form and takes around .5Mbytes.

I prefer to think of MILO as LILO with knobs on, a kind of freeware console.
There again I'm biased, I wrote it.

> 4. SRM or ARC is an either-or thing: you run either SRM or ARC, but not
> both (can you even install both?).
>

Some systems have enough flash for both and choose between them at boot time
using some information someplace. On some systems MILO can be blown into
flash too.

> Like I said, what I've written above is probably at least partially
> inaccurate, but I figuer it's better to set out my misconceptions to be
> specifically addressed rather than a blank "what are these?" kind of
> question.
>

No problem.

> Thanks. Sorry to be the latest pain-in-the-ass UDB guy.

Your questions were structured and showed that you had thought through some of
this stuff yourself before asking.
