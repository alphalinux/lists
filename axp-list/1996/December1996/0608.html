On 14 Dec 96 at 16:58, David A Rusling wrote:

...
> MILO stays memory resident in a zipped form and takes around .5Mbytes.
>
> I prefer to think of MILO as LILO with knobs on, a kind of freeware console.
> There again I'm biased, I wrote it.
>
> > 4. SRM or ARC is an either-or thing: you run either SRM or ARC, but not
> > both (can you even install both?).
> >
>
> Some systems have enough flash for both and choose between them at boot time
> using some information someplace. On some systems MILO can be blown into
> flash too.
...

This brings up a question in my mind:

If you put MILO into flash (assuming you have the space and don't
need SRM or ARC) can you reclaim that .5M of RAM while Linux is
running? I *GUESS* it's not a big deal, especially if you have 64MB
or so, but for me Linux has been about getting as much as you
possibly can out a machine...

This also leads me to believe that if I want as much memory as
possible I can create my own MILO with the minimal set of drivers
right? (Which I'd probably do with the kernel as well...)

Just a thought! :-)
