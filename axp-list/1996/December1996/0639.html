Hi!

I'm about to join a project at the university (bringing lburg/lcc to Alpha
platforms). The primary goal at the moment is to get lcc to life on
digital unix. -- Unfortunately I don't have a digital unix license for my
box at home, but i _do_ have Linux/Alpha ;-) -- Of course I'd like to be
able to do some (or even most) of the work on my home box.

The problem with lcc is that I've been told that it is not completely
64bit clean at the moment. -- The very first version for the Alpha should
therefore use a digital unix feature that allows you to place all your
text and data into the first 4GB of a process's address space.

So I'm facing the following problems:

1) Is it somehow possible to do the same thing under Linux/Alpha (i.e.
squeezing everything in the lowest 4GB of the address space.)?
We've tried to fool around with the ld options "-Tbss -Tdata" and
"-Ttext" to force everything into the low 4GB but all we've got
was an executable that immediately got a "Signal 11" (and even
killed the shell it was started from :-( ).

From what we've tried I've got the expression that it _should_
be possible to do that, but it does not work at the moment. - Is
this correct?

2) How about the compatibility of the gcc-assembler and the
digital unix assembler? -- Is it possible to get the same assembler
code compiled by gcc and the digital unix assembler as well? -- If not,
are the differences small enough to be eliminated by preprocessing
the assembler listing with a small perl script or such?

(begin off topic)
3) If all else fails -- How much is a digital unix license
for a student? ;-) -- Would it comfortably run on a 233MHz UDB
w. 56MB RAM and 1GB HD? (Typical workload: one user running X-Windows,
emacs, a few xterms and tools like cc, lex, yacc... every now and
then.) Or would it be possible (and cheaper..) to get a license that is
only valid for a certain period of time e.g. one that expires after 6
months or so?
(end off topic)

Any help on these topics would be greatly appreciated.

Thanks in advance!
