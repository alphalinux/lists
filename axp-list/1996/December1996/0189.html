> Are you sure you got them all? The only thing left in /usr/lib should be
> some libnss_*.so symlinks pointing to files under /lib.
>
> I agree that this really shouldn't been found in testing, but it _is_ an
> easy mistake to make. Bash doesn't directly load those files itself ("ldd
> /bin/bash" won't list them); the C library loads them on behalf of bash
> when doing lookups in some of the files in /etc (passwd, hosts, etc.).
> This is a glibc feature, and so the same exact code works fine on x86
> RedHat 4.0, which doesn't use glibc. Since the scripts are identical
> across platforms, I can imagine that this particular combination (RedHat
> 4.0, on an Alpha, with /usr and / on different partitions) was not tested.
