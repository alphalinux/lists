On Sat, 30 Nov 1996, bruce a. campbell wrote:

> > I entered a nameserver address in /etc/resolv.conf and a route to it. But
> > when this nameserver is not reachable (e.g. down), my X server won't start
> > (it terminates immediately after initializing the screen).
>
> Anyway, since the resolver is ignoring /etc/hosts, you go to your nameserver
> and get a "not found". The work arounds possible are :

you need a file /etc/nsswitch.conf, here is mine
# does not use any naming service.
#
# "hosts:" and "services:" in this file are used only if the
# /etc/netconfig file has a "-" for nametoaddr_libs of "inet" transports.

passwd: files
group: files
hosts: files dns
networks: files
protocols: files
rpc: files
ethers: files
netmasks: files
bootparams: files
publickey: nisplus
netgroup: nisplus
automount: files
aliases: files
services: files

The important line is
hosts: files dns
This works for me.
