>>> Robert Allen Barker said:
> Are any of the following 3D graphics cards supported under redhat linux, and
> therefor have documentation available for writing drivers?
>
> 1. Matrox Millennium
> 2. Diamond Monster 3D (3D co-processor card)
> 3. Diamond Stealth 3D (Virge VX chip)
> 4. Velocity 3D (Virge VX chip)

There *is* Virge and Millenium support for INTEL in the 3.2 XFree86 release,
but it had not been made to work on the Alphas at the time of the release
(October 26).

Since then, I've ported the Millenium code to the Alpha, and Harald Koenig
has gotten the Virge code to work, I believe, but I don't know the details
about *which* Virge chips are supported, nor where any servers might be
located. Harald?

I'll be putting a Millenium server out onto gatekeeper.dec.com later today,
based on the 3.2 release. Look for it in:

/pub/DEC/Linux-Alpha/X11/XF86_SVGA-3.2-test.gz

A patch set is available for any who might want it, but be warned, the MGA
support is changing in a major way in the beta development now going on.

IMPORTANT!!! IMPORTANT!!! IMPORTANT!!! IMPORTANT!!! IMPORTANT!!! IMPORTANT!!!

The Millenium server REQUIRES the use of kernels 2.0.26 and above, or 2.1.13
and above, as there's new system calls that need to be in place for the
server to use.
