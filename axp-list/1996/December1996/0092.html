> "make menuconfig" on alpha hides certain configuration options
> that "make config" does not (eg. CONFIG_SCSI_NCR53C8XX).
>
> Is this intentional or a bug ?

it's intentional (read linux/drivers/scsi/Config.in).
CONFIG_SCSI_NCR53C8XX can only be selected if CONFIG_SCSI_NCR53C7xx is
*not* selected; it's not possible to use both drivers at once.
try to deselect CONFIG_SCSI_NCR53C7xx ...

with "make xconfig" this is a bit more obvious since it still shows
the new NCR driver but you also can't select it as long as the old one
is selected.
