On Sun, 22 Dec 1996, Alan P. Kennedy, Sr wrote:

> Is it okay to blow away the NT firmware with milo on the UDB. I WAS

Don't do this unless you have a really good reason. Loading MILO from
ARC is a perfectly reasonable thing to do (my primary build does this)
and works just fine. You don't gain anything by flashing MIL on
a UDB.

> Another question should I build my own milo or use the prebuilt ones.
> The faq hinted that you should have the same version of milo with the
> kernel. It's running here without any problems milo 2.0.22 and kernel

The FAQ meant to hint that you need to match the kernel source to the milo
source when you build milo. The kernel doesn't care which version of MILO
boots it.

> If I could not flash milo I was thinking about putting a small dos
> partition on the scsi disk and using the NT firmware to load milo
> without having to use the floppy drive.

Perfect. I normally "dd if=/dev/fd0 of=/dev/sda1" and it works like a
champ.
