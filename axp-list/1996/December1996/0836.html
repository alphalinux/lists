>
> I set the
> time using the NT arc console but that time does not appear in Linux.

Make sure that you have a line
CLOCKMODE="ARC"
in your /etc/sysconfig/clock (if you boot through ARC console).
Also /etc/rc.d/rc.sysinit file distributed by Red Hat has a long
standing bug. Instead of a line 'clock -A -a' they have
'clock -A -a -u' (which would work if your ARC console would be
set to UTC - so maybe this is a feature :-).

> What's the proper way to setup the time on UDB systems.

'man -k clock' brings these:

adjtimex (2) - tune kernel clock
alarm (2) - set an alarm clock for delivery of a signal
clock (3) - Determine processor time
clock (8) - manipulate the CMOS clock
clock (n) - Obtain and manipulate time

'rdate -s' is also handy if you are on a network.

> Does anyone know what the best
> blocksize to use for NFS transfers on a 10BaseT or 100BaseT network?

They depend on your cards (on both sides of a line), but a default 1k
is no good for anything beyond of not crashing some really broken
ethernet interfaces. Try 4k and 8k.
