Hardware config: (my apologies for not including it before)

from arc console system info:

processor id: 21164
processor revision: 7.1
system revision: 0x1
processor speed: 433.08 MHz
Physical Memory: 128mb
Backup Cache Size: 1MB

Extended Firmware Info:
Version: 4.49-4 960806.1339

my info:

NCR 810 scsi card (without bios chip)
128MB ram
DE500-XA ether card
unsure about video card

It is running kernel 2.0.18 (standard pc164 installation kernel). It
is the only kernel I've run on the machine (I haven't gotten the
others to boot). Various weirdo symptoms include:

time (in upper right) flashing in arch console
(flashing=rapidly changing from one value to another, impossible to
read)

linload.exe halting (no message, no keyboard lights)

milo halting after being reboot (right before boot prompt, same as ^).

OS halting, usually during a kernel build (usually with no messages,
sometimes with a memory dump)

When linking object files, sometimes the process will just sit
around indefintely.

Amazingly enough, I was just able to compile kernel 2.0.27 (without it
crashing, with make boot). I installed it, along with the latest milo
from gatekeeper. The first time I tried to boot it, I got a hex dump
(memory dump?) which looked like the following:

0000 0000000000000000 0000000000000000

and so on... most of it was null bytes.. Unfortunately, I was not
looking at the screen before it started scrolling down, so I missed
where it started.

It appears to be slightly more stable now. (It doesn't crash quite as
often). It just crashed with the hex dump shown above scrolling down
screen.

It seems like a hardware problem to me as well (although I know very
little). If any more info will help, I'll be glad to oblige.

thanks very much!
