I'm not sure what I did, but when I updated to the latest TGA X-server
(for the UDB) I was unable to run X windows unless I was root?

That seems rather strange. The only files I changed were the actual X
server and the /etc/XF86Config file. What permissions do I need to set to
allow me to run X windows from any-body that is logged in. (Like it used
to be!!)

X server should not be run by regular users; rather, it should be run on
startup by 'init' via the xdm process, and present a login box. X has to
run with high priviledge level and making it SUID root is insecure.

Changing the setup so that xdm is run automatically on bootup is easy:
edit file /etc/inittab, and change

id:3:initdefault:
to
id:5:initdefault:
