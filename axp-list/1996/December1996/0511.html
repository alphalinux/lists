When booting from ARC console (via MILO), one wants the clock command
executed during bootup to be:

clock -A -a

which tells the code to expect the year to be as maintained by the ARC
firmware.

The /etc/sysconfig/clock script file should contain:

CLOCKMODE="ARC"

And finally, the /etc/rc.d/rc.sysinit script file should have clock-related
stuff looking like:

> # Set the system clock.
> echo -n "Setting clock: "
>
> if [ -f /etc/sysconfig/clock ]; then
> . /etc/sysconfig/clock
> if [ ${CLOCKMODE} = "GMT" ]; then
> clock -u -a
> else
> if [ ${CLOCKMODE} = "ARC" ]; then
> clock -A -a
> else
> clock -a
> fi
> fi
> else
> clock -a
> fi
>
> date

Note especially the intentionally missing "-u" on the "clock -A -a" line. I
think this was a problem with the stock RedHat 4.0 script?

If you run from SRM, then CLOCKMODE would be LOCAL or GMT, I guess, and you'd
not want the "-u" on anything *except* when "GMT", as above.

Make sure, if you update the time while running LINUX, that you write it
back to the hardware with the appropriate switches; ie, if booting via ARC,
use "clock -A -w".

Would someone please put this in a FAQ somewhere?
