Hello

>> i'm wondering: can i set the clock speed on the 233MHz
>> machine to something a bit faster, say 266MHz? has anyone
>> out there tried this, and care to share success or failure
>> stories? how about the 166MHz machine; how fast can it
>> go, without too much chance of damage?

The 166Mhz can't be clocked up or it will burn up. To start with its
already running hot. Second, we tried it and because the 166Mhz was the
first of the 21066 processors I don't believe it was ever stable running at
higher speeds. At least we never found it to be. Running NT stress test
you'd see errors in the cache on the processor.

>I have read people working on 233 upto 300mhz(set is, however actual
>output is somewhere around ~285mhz) with some tweaking. 266mhz is kinda
>standard clockup for many companies.
>I read this tip in some Linux-Alpha HP, and was doing ok for over 3 weeks,
>except you need really good ventilation.

No this we had success with only at 266Mhz. The rule we've always used is
nothing over 10% and stay on 33Mhz clock frequencies, ie:166, 200, 233, 266,
300, 333, 366, 400, etc...
