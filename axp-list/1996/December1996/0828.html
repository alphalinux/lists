Hi all,

I have uploaded to

ftp://linux.dcs.ed.ac.uk/pub/linux/kernel/

the two files:

ser-cons-2.0.27-b.diff
and ser-cons-2.1.17-b.diff

These contain my patches to Miquel van Smoorenburg's excellent serial
console patches for linux-2.0 and 2.1.

Briefly, the serial-console patch lets you send a serial BRK signal to
a designated console serial line to interrupt the kernel, dropping
into a tiny monitor prompt which allows you to do minimal status
reporting or to reboot the machine. The monitor prompt also appears
on kernel panic (unless reboot-on-panic is selected) or system halt.

The serial console patch is perfect for environments where remote
reboots are required even in the event of system hangs or crashes,
although it won't save you if the kernel is locked into an
interrupt-disabled loop (you need a hardware watchdog timer card for
that).

With the serial-console patches and SysVinit-2.64 or later installed,
all console messages, including kernel printk's and the system startup
messages, will be redirected to the selected serial port.

The changes since Miquel's v0.1 patch include:

* Fixed to work on both Alpha and Intel platforms
* Kernel command line option parsing fixed
* Updated for 2.1 kernels
* Now, sending a serial BRK drops into the monitor even if
the tty is software-selected to ignore BRK
* When you exit the monitor, the default is NOT to propogate the
BRK signal to the underlying tty. A new monitor option allows
you to explicitly send the BRK. (Type "?" for a quick list of
monitor commands available)
* It is no longer possible to return from the halt system call.
That is just going to get init too confused (correct me if
I'm wrong...) You can still perform a hard reboot from the
halt> monitor prompt, though.
* Default baud rate is 9600, not 2400 (I have yet to come across
a machine which uses other than 9600 for console)

This release numbers itself v0.1b to distinguish itself from Miquel's
original. The patch will add a new file to the kernel source tree,
"Documentation/serial-console.txt". Please read it, even if you don't
need to. :)

Cheers,
Stephen.
