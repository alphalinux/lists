> /usr/include/timespec.h:1: linux/timespecbits.h: No such file or directory
> make: *** [scripts/mkdep] Error 1
OK. RedHat 4.0 installs the glibc package which are the new shared libraries
for AXPs. Since glibc != Linux-libc it uses different header files. RedHat
has merged some patche into the 2.0.18 Kernel sources so 2.0.18 compiles
cleanly. For everything else, I use the following:
Install only 2.0.18 headers in some directory and make a symbolic link from
/usr/include/linux -> <to_some_place>
and
/usr/include/asm -> <to_some_place>
and you should be fine and able to compile normal user level programs (such
as Milo).
For the latest kernels to compile you have to change the main kernel
Makefile in
/usr/src/linux/Makefile
and comment out the include directive from the line
HOSTCC=gcc #<something_I_have_forgotten>
Not very nice, but it works.
