On Wed, 4 Dec 1996, Randy Lee Carpenter wrote:

> On the UDB running Redhat4.0, the system is never able to umount /usr no
> matter what drive it is on. Why is this? I am able to do so only in single
> user. It gives the message "Device busy".

Do the following:

mv /usr/lib/libnss* /lib

The libnss* files are dynamically loaded by bash (well, actually glibc
does it, but bash is what is causing it), and since bash is executing the
shutdown script, it can't unmount /usr.

> How do I format a partition MSDOS? I want to make a ZIP disk to boot form,
> and want to include milo/linload on the disk. Is there another way to do
> so without having to format msdos. If not, how do I do it? There doesnt
> seem to be a 'mtool' that will work with hard disks.

There is a "mkdosfs" command to make MS-DOS filesystems on a partition.
You should have it on your system unless for some reason you decided not
to install that particular package (mkdosfs-ygg-0.3b-4).
