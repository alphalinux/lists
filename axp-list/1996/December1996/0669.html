If you're tired of random segfaults in amd, sendmail, and other
programs due to the silly bug in memchr() that I mentioned a while
ago, here are a few RPMs that have this problem fixed. It's all
in ftp://ftp.azstarnet.com/pub/linux/axp/misc:

1187703 Dec 17 09:25 glibc-0.960906-1-fixed-memchr.axp.rpm
843053 Dec 17 09:25 glibc-profile-0.960906-1-fixed-memchr.axp.rpm
693191 Dec 17 09:25 glibc-static-0.960906-1-fixed-memchr.axp.rpm

Note that the RPM version numbers are unchanged from the ones that
come with RH4.0 (I'm sorry about that, but that's how it is). This
means you'll have to use the --force option to install the thing.
E.g., something like:

rpm --force -Uvh glibc-*

worked for me.

The Alpha at work has run with these RPMs for over a week now and no
segfaults so far.

Enjoy,
--david
