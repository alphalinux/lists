Does anyone know what the format of __ivaliduser is? It seems that under
RH 4.0, lpd calls __ivaliduser to check if a host is in either
/etc/hosts.equiv or /etc/hosts.lpr. The way it calls it is:

__ivaliduser(fd, hostname, DUMMY, DUMMY, baselen)

where fd is the file descriptor of either file, hostname is the remote
machine's full name, and baselen is the length of the machine's short name
(i.e. "foo" in foo.bar.gov).

It looks like lpd expects __ivaliduser to return 0 if the hostname is
found in one of the files. However, when called, all it gives is a line
that goes something like:

memory violation at pc=155556bfa20 rp=155556beba0 (bad address = 00000000)

and then dumps a core. I can bypass this by putting in my own routine, of
course, but that's not exactly the best fix. I'd really like to know what
the fields are that DUMMY fills in, too...
