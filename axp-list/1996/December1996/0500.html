Once upon a time, Mike Holling wrote
> > On Wed, 11 Dec 1996, Randy Lee Carpenter wrote:
> > > WHy is it that Linux believes the date to be 20 years later than what the
> > > firware is set to? if I set the firmware to 1996, Linux sez 2016. Is it
> > > safe to set the firmware to 1976 so Linux is right?
> > I've got the same problem on my EB164... I've just set the date manually
> > and haven't rebooted. =)
> Same on my UDB. It's really annoying, anyone know what's going on?

I guess the Alpha systems store the date differently internally.
According to clock(8), there is a "-A" option to tell Linux that your
clock is a UDB clock. The annoying thing about this is that during
install, RedHat 4.0 asks you if your system stores the date as local,
UTC, or UDB time. However, the system assumes that if you have it on
UDB time that you are using UTC, not local, time. So, the startup files
have "clock -A -a -u" in them for the UDB. I think these should be
separate options (as some other, not to be named on this list, operating
systems prefer the time in local time).
