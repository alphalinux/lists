rrussell@dps.com writes:
> I'd like to set up XDMCP chooser panel on my Linux Alpha box
> (redhat 4.0, Colgate) using a local font server but choosing an
> XDM for itself, a SPARC or an Intel. I've done this previously with
> something like X -query, X -indirect or X -broadcast (under R4 ?).
> I've read about but didn't quite get the idea of Xaccess, etc... in the X
> man pages.

You need to remove any entries from /usr/X11R6/lib/X11/xdm/Xservers,
because otherwise xdm will start an Xserver by itself which is
"hard-wired" to it. You have to start your local display with an entry
in in '/etc/inittab', or something similar, I haven't done that on the
Alpha (its /etc/ttytab on SunOS). For testing, 'X -indirect localhost'
on a console should do.

For a start you need two entries in /usr/X11R6/lib/X11/xdm/Xaccess:

* #any host can get a login window
* CHOOSER BROADCAST #any indirect host can get a chooser

But I believe those are the defaults, anyway.

> Has anyone tested this under Linux Alpha ? My Alpha currently enters the
> run level for XDM on boot, I'd like to change the appropraite stuff in
> X configs and rc if necessary. Any hints on telling the Solaris 2.5 box
> to manage other Xservers (ie respond to braodcasts), seting up the above

It's just the same on Solaris, except that it is in
/usr/openwin/lib/xdm. Just change Xaccess as above.
