Hi,

first of all: YES, I read through all the FAQs and HOWTOs :)

Ok, I'm trying to install RedHat Linux for Alpha on a Jensen for more than
a month now, but it either hangs or runs into a HALT.

My system is a Jensen with ROM 1.7, 32MB RAM, an EISA Adaptec SCSI controller and a
Diamond Stealth S3 video card. For some reason the video-card doesn't display
anything so I'm using a DEC VT240 terminal at the serial port to work, but that
can't be the problem, can it? The harddisk has been partitioned using minlabel.exe
and I'm using copyboot to write the bootdisk images on it, that also should
be OK, the only thing I'm not exactly sure about is, whether it could be a
wrong partitioning that causes the problems...

I tried RedHat 3.0.3 (the original RedHat distribution) and RedHat 4.0 bootdisks
from different sources, but the result is virtually the same whatever I do:

(this was done with a 2.0.x kernel image from ftp.azstarnet.com, but it runs
somehow the same using any other image)

>>> BOOT -FL "i" DKA000 (I also tried several variations)

aboot: Linux/Alpha SRM bootloader version 0.5
aboot: switching to OSF/1 PALcode version 1.35
aboot: valid disklabel found: 3 partitions.
Enter kernel name and arguments:
aboot> 0/a root=/dev/fd0 load_ramdisk=1
aboot: booted_dev='scsi 1 1 0 0 0 0 jens-io', guessing boot_device='scd0'
aboot: loading plain a...
aboot: unexpected object file format 137165
aboot: loading compressed a...
aboot: ok, now starting the kernel...

(and at this point the machine hangs). I also tried it using an uncompressed
kernel image and didn't get the "unexpected object format" error, a "loading
uncompressed a" or something like that and the machine hung again.

I also tried the "old" way without aboot, but the result also was a
"now starting the kernel" and a hanging machine. Is the kernel switching to
the VGA screen (which does not work on my machine, as mentioned above) or
something?

Cul8er, Ray
