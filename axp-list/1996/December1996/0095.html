sorry to say this, but i haven't seen any really stable linux at my original
DEC AS/400 alpha server yet. The longest time without crash has been
about 4 days ( with 2.0.26) , and i tried nearly all kernel-versions from
2.0.0 upwards.

The latest crash just happened with 2.0.27 under quite heavy nfs-usage,
giving 2 messages to the console, one about misaligned access and the
next one about 'bad kernel misaligned ...`, total system lockup, nothing
written to the logs.

The 2.0.27 kernel was compiled with gcc 2.7.2 , axp-diffs 2.0.21
and the ncr-scsi-driver 1.14c update.

i'm quite sure the ncr-scsi-driver 1.12 has caused many crashes, because
it uses 10MByte/second, if the scsi-devices report they can handle this speed.
Using the default configuration, my system crashed whenever the disks were
heavily used, sometimes logging scsi-errors.
Switching to 5MB increased stability a lot, making it possible to compile large
applications or the kernel 'in one go', so i think was merely a problem of an
overstressed scsi-cable.

Because i haven't seen complaints like this on this list, and i know linux is
really stable at intel systems, i'm beginning to think my alpha hardware
must be somewhat broken. Maybe one of the memory chips occasionally
faults or the MMU gets confused.

Are there known problems with AS/400 hardware, or am i just too impatient ?
