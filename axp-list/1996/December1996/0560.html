Hi,

I'd really like to experiment with eliminating milo -- that is, compiling an
SRM bootable kernel for use on my eb164. I have not yet solved the 64 MB RAM
barrier (a new SROM is on the way soon), and I'd like to at least eliminate
the SRM as a possible fix for this problem. The SRM always sees the correct
amount of RAM, whereas milo is completely dependent upon the MEMORY_SIZE
variable stored in NVRAM.

Anyway, the reason that I am mentioning this here, rather than simply building
the kernel that I want, is this: the eb164 typically does not come with SRM,
and so, the kernel configuration program seems to think that if eb164
has been selected as the machine type, that SRM booting is not an option.
Now, I'd simply select another machine type, if I thought that this would
be a fix, but I fear that something about hardware specifics might interfere
with this, and give me a non-working kernel...

Perhaps there is a way that I can hand-edit the Makefile?

Thanks for any help.

Rolf Karlstad
