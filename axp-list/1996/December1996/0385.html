Sure enough, the problem got much better when I turned the clock rate
down on the SCSI card. I hacked aix7xxx.c to deny devices from running
faster than 5Mhz. Now I can run 5 big dd's on the disk at once. I was
still able to panic the kernel by doing more of these, but this behaving
much better.

The only devices are an internal CDROM and an internal fast SCSI
disk. The termination appears to be set correctly on the Adaptec card
according to the output when the card boots.

So, I don't think this is a real solution, but it is a better work
around. The oops appears to be occurring at the same location. The only
thing different is the virtual page.

Any other ideas?
