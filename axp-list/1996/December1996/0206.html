As if all the other comments were not enough on this one, I have had this
problem with my 233. I found that for me specifically I thought it was a
gateway problem. The gateway name could not be resolved, but the ip was
fine. So the true problem was the nameserver was setup wrong in
/etc/resolv.conf. Maybe this is simular to your problem.

Anyway, if you leave it for 15 - 20 minutes it will time out and boot up
without a hitch. In my case nameservice did not work though. Ther my 2
cents worth.
