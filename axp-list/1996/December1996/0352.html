On 9 Dec 1996, Jens Lautenbacher wrote:

>
> Hi,
>
> I've just got a AXP 164/300Mhz box running RedHat 4.0 and have soon
> run into major problems with it. PLEASE can anybody help, as we
> _really_ need this machine working...
>
> 1) How am I supposed to make new kernels? Fetching 2.0.27 from the net
> and just trying to compile ended with a not found include file
> timespecbits.h.

Get the axp-diffs: axp-diffs-2.0.21.gz
Look into ftp.tu-graz.ac.at:/pub/Linux/azstarnet/sys
Apply it with:
cd /usr/src
zcat /blahfasel/axp-diffs-2.0.21.gz|patch -s -p1
Then do a "make xconfig" and set your System type correct. Set all other
properties.

after installing do a "cp /vmlinux.gz /vmlinux.old.gz"
then cp the kernel to / (cp /usr/src/linux/arch/alpha/boot/vmlinux.gz /)

>
> 2) I have this box for 4 days now and had 3 major breakdowns with it,
> always ending in filesystem corruption. It simply broke during
> extracting of some big tar.gz's like (two times) the linux kernel
> sources.
We had the same on machines with bad RAM or EB164 boards with more than
64MB RAM. Please, give details of your machine.

>
> 3) I need to have g77 running so I fetched gcc-2.7.2 and g77-<newest>
> and tried to compile this which ended with "alpha-unknown-linux is
> not supported" from configure. This seems quite funny, as I _have_
> a working gcc 2.7.2 from RedHat (I need to recompile just for g77)
> How did they produce their compiler? How did they produce their
> kernels?

Don't know about that, I think you have to make a own specfile.
