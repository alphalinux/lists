Hello,

>> But, if I try to run mcopy, or any mtool, or cdplay I get
>>
>> mcopy: memory violation at pc=#address# rp=#address (bad address = address)
>> segmentation fault (core dumped)

Yes, since "segmentation fault" is not always enough to find out what
the problem is, and because AXP is still a fairly young platform (where
you might not even be able to run a debugger before something segfaults
:-), the kernel explains memory violations to the user by printk'ing
infos. This does not mean that your kernel is broken, so upgrading it will
do no good : the bugs come from mtools and cdplay themselves.
