On Sat, 30 Nov 1996, bruce a. campbell wrote:

> > I entered a nameserver address in /etc/resolv.conf and a route to it. But
> > when this nameserver is not reachable (e.g. down), my X server won't start
> > (it terminates immediately after initializing the screen).
> >
> > If I remove the route entry, everything is fine.
> >
> > Removing the X server authorization didn't help; neither did I get any
> > hints by reading the xdm-errors file.
> >
> > What is the X server trying to look up?
>
> The server is trying to look up your local hostname by way of DNS. Not finding
> it, it barfs. The next question is why does it not find it in the local
> /etc/hosts? It appears that even when everything is set up right,
> (/etc/host.conf is setup to say "order hosts,bind\nmulti on"), the resolver
> does not seem to be looking at /etc/hosts. It this a bug or a feature?
>
> Anyway, since the resolver is ignoring /etc/hosts, you go to your nameserver
> and get a "not found". The work arounds possible are :
>
> 1. Add your host name and localhost to your nameservers map.
>
> 2. Add your IP address and 127.0.0.1 to /etc/X0.hosts. This way the X server
> will know someone to talk to, and possibly not give up on life.
>
> 3. Name /etc/resolv.conf to /etc/resolv.conf.good and only set a symbolic
> link to it when you need DNS, after the X server has started.
>
>

check /etc/nsswitch.conf and make sure you have a line

hosts: files dns
