You need netatalk (presumably better than CAP, but the later also an
option) for that. None is Linux specific but they work - at least
with Linux on i386.

> In the end,
> the only REAL problem I had was that I could not make the mac run on ppp
> and ethernet at the same time, so I was unable to use the mac for a
> gateway,

The machine which does that has to have IP_FORWARDING capability.
I do not if and how you can do that on Mac. Linux can be easily
configured to do that.

> although, if I set up the modem on the UDB, it was fairly simple
> to turn it into a gateway for the Mac.

If you have only one "official" IP number for you connection then
IP_MASQUERADING can be used with internal unroutable IP addresses
from a set reserved for such purposes on your ethernet interfaces.
