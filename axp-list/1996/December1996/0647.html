|
| zsh as provided by zsh-3.0pre6-1.axp.rpm is broken. If any program exits
| with a signal (is killed), zsh will die with a memory violation. When I
| tried to recompile zsh, I ran across problems due to the way that zsh's
| configure script expects the header files to behave: it parses signal.h
| and a few others to find the SIGXXX to number mappings. Since glibc's
| SIGXXX defines are in asm/sig(something).h, which is included from signal.h,
| the configure script breaks. I fixed it by telling the configure script to
| look in the correct file; I had to hand-edit the configure script.
|
| There was another problem having to do with the LIMXXX macros, which (if I
| recall correctly) I had to edit a system header file to get working.
|
| I'm hesitant to make RPMs up, since I don't have much experience with that
| kind of stuff, but I can make a zsh executable available via http or ftp if
| anyone is interested.
|
| I'll also try to get better notes and patches for what I did to make it
| compile. There were just the two fixes, so it shouldn't be too hard to
| replicate.
