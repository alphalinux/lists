The problem:

Have UDB with RH3.0.3 pre-installed. Have added new Seagate ST43400NB
2.9GB drive externally. Partitioned it (using fdisk). Used minlabel
which wrote label to it. (Before minlabel labelled it mke2fs failed
disastrously, after minlabel mke2fs worked a treat...is this
significant?)

Start RH4.0 install procedure. Eventually figure out which MILO
linload.exe etc to copy to floppy and manage to create an ARC bootable
floppy.

Created boot disk from noname.img on the CD, ramdisk from ramdisk.img.

Boot and start an NFS install (BTW had to use IP address for the NFS
host, using DNS name kept complaining).

Everything is fine until we get to partitioning the disk. Originally
had swap, root, usr, extended (including 4 logicals). No soap. Messed
around repartitioning to a) remove the extended (no difference) b) put
the root first and swap next (no difference) and a few other attempts.

Firstly the install only finds the swap partition on /dev/sda but not
the one we created (as type 82) on /dev/sdb. OK no biggie.

But...when asking about which partition we want to mount where, sdba
is detected OK but the process only finds 1 partition on sdb,
/dev/sdb2 (!) start 16384 stop 56........ for a size of 2.8GB. Which
is completely incorrect (there are 4 partitions, sdb1 is 30M, sdb2 is
30M, sdb3 is 1GB and sdb4 is the rest).

Does anybody have any idea what is going on here? Have we screwed up
the minlabel aspect of this?

All help gratefully received.
