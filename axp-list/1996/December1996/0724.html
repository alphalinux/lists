>>> David Rudder said:
>
> I'm just setting up my AlphaServer 1000. I've got the latest
> Craftwork Linux installed and I'm trying to get X up. It has two cards,
> one on board (currently jumpered off) and an EISA video card. I believe
> the EISA video card is a Compaq, but I'm not sure which one. The on
> board is a Cirrus Logic that is supported on Intel by the SVGA X server.
> Is there a server for either of these cards? On the DEC ftp site
> I was able to find a SVGA server labeled as a test, but I can't seem to
> get it to run. Is there an SVGA server out there that works? Can I run
> either of these cards? If not, can I run the EISA card in my ISA slots
> on my Pentium as a simple text driver (my Pentium doesn't even have a
> monitor. I just use my DEC monitor to get it up) and put the S3 I have
> in my Pentium on my Alpha?

First, the EISA COMPAQ card is prolly the so-called QVision, and there's
unsupported and untested code for it in the latest XFree86 release, 3.2.

Second, there's no Alpha specific server for *either* of those cards. Either
*might* be run using a monochrome server based on XFree86 3.2 that I have
patches for, but no guarantees. Let me know if you want to try it.

The XF86_SVGA on gatekeeper is *ONLY* for the Matrox Millenium, nothing else,
and *requires* kernels 2.0.26 or 2.1.14 and above.

An EISA card won't work in ISA slots, I believe.

An S3-based PCI/EISA/ISA card ought to be fine in the Alpha, and there's good
support for most of them under XFree86 on the Alpha.

Best solution: find a really old/cheap VGA for the Pentium and move the S3
over to the Alpha.
