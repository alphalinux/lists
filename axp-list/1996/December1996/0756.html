Hi all,

I picked up a UDB a couple of months ago and have on and off
been trying to get 2.0.26 to compile cleanly.

I started with 2.0.21 and applied the axp-diff-2.0.21 patch.
Then I patched up to 2.0.26 (making sure that all new files
got put in there proper places before going on the the next
patch.)

I've been getting warning like this from the networking code:
de4x5.c: In function `de4x5_alloc_rx_buff':
de4x5.c:2697: warning: `p' might be used uninitialized in this function
de4x5.c:2728: warning: `skb' might be used uninitialized in this function
de4x5.c:2730: warning: `tlen' might be used uninitialized in this function
de4x5.c:2731: warning: `skb' might be used uninitialized in this function
de4x5.c:2732: warning: `skb' might be used uninitialized in this function

and I end up with a kernel that is ~8 Meg uncompressed!!!!

Is this a known problem? Anyone have any work arounds or know of
a kernel source version that compiles cleanly?

Thanks in advance,

-John
