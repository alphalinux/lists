Hrmph...it has become obvious to me, after trying every possible known
setup, that the RH4.0 networking is completely different from that of
RH3.0.3 (with which I never had problems). I feel that the nsswitch.conf
file (mentioned by Linus) may be what I need, but I have no idea how it
should be configured, or what it does for that matter.

For the record, something is definitely screwy with the /etc/hosts. I only
use PPP (no ethernet) and have a static IP. With 3.0.3, I simply had it as
follows:

127.0.0.1 localhost
208.201.52.120 nd4spd.collider.com nd4spd

For some reason, using this same setup in 4.0 results in a host not found
when I try to ping nd4spd, and nd4spd.collider.com (when I'm connected!).
Just to see what would happen, I tried 0.0.0.0 nd4spd.collider.com nd4spd,
but it didn't change anything.

My hostname is properly set at nd4spd.

On a side note, I *tried* to set up my system using netcfg in the
control-panel. For some reason, RedHat seems to think the hostname is the
full address. If I just put my hostname in hostname, then sendmail hangs.
Odd. Also, RedHat apparently messed around with the rc.sysinit files so
that it now uses its own network files, then moves those settings to
/etc/HOSTNAME. WHY!? Anyway, I # out those lines and returned the rc.sys
back to the way the 3.0.3 vers worked (and worked properly!).

Thanks to the 'new' setup, I no longer understand how to properly configure
my network. I can't even refer to the HOWTOs since they are based on the
'correct' setup.

Thanks again,

I really hope this has to do with my network probs!
