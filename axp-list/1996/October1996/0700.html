> I'm trying to get linux installed on my alpha (a 233MHz multia). I
> can load milo fine, but when I boot linux, it loads linux, and then
> hangs. BTW, this is a headless multia. The following is a transcript
> of what happens:
>
>
> MILO> boot fd0:vmlinux.gz root=/dev/fd0 load_ramdisk=1
> #########################floppy0: timeout handler died: do fd request
> ###############################################################################################done.
> Boot line is bootdevice=fd0 bootfile=vmlinux.gz root=/dev/fd0 load_ramdisk=1
> Bootstrap complete, kernel @ 0x310000, sp @ 0x304000
> ...turning on virtual addressing and jumping to the Linux Kernel
>
> ------------------------------------------------------------
>
> Any idea why it just hangs here and what to do to fix it? Also, what's
> the floppy error message means? Thanks.
>
The floppy error is what it says, a floppy request timed out. The FD will retry.
It looks to me like you loaded a good .gz EXT2/COFF image that was either (a) the
wrong kernel image or (b) doesn't echo to the serial port...
