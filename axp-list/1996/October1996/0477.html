Hello,

after installing a small RedHat 4.0 system on my Alpha (Jensen), I tried to
compile a 2.0.21/22 kernel on the new system. This works fine, but after
booting the kernel detects only two of the three partitions on the first
(and only) SCSI disk - it seems that the first partition, which is marked as
type 'unused', is omitted (this partition contains aboot + install kernel).

When I change the root device accordingly (sda2 instead of sda3), the
machine can mount the root fs, but I get lots of errors from ll_rw_... that
1k blocks are not supported (it says it can only do 8k), and the system
crashes then.

My old 2.0.14-kernel, which was built on the old ECOFF system, works fine,
and a 2.0.14 kernel which I just built on the new ELF system works fine,
too.

What was changed between 2.0.14 and 2.0.21 that makes the kernel not work
anymore? What should I check?

All kernels were built using David M-T's axp-diffs, 2.0.22 was built using
the 2.0.21-diffs.
