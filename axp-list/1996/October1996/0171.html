OK, a long time ago, there were some pretty serious problems in
getting the keyboard initialized properly. I believe Jim Paradis took
a good hard look at the problem and rewrote the keyboard
initialization code and since then everything has been working great.
I think in the process of this rewrite, the explicit command to select
scancode set number 2 has been dropped (because the keyboard-reset
will already select this scancode set, so there obviously isn't any
need to send this command explicitly). Well, all worked fine until my
department purchased a Cybex commander (console concentrator) so not
each of our Alphas has to have its own space-consuming keyboard and
monitor. Anyhow, to make a long story short, the Cybex firmware seems
to have a bug in that a keyboard reset command is not interpreted as a
switch to scan code 2. This has the not so funny effect of giving a
keyboard that works mostly except for minor keys such as Alt, Ctrl, or
ESC.

To fix this problem, I'd like to change the keyboard initialization
code to send the explicit "switch to scancode 2" command in addition
to the keyboard reset. I _think_ there should be no problem at all in
doing this, but then, there seem to be all kinds of buggy keyboards
out there so I'd like to double-check to make sure there are no known
problems with this. So, if you know of any problem with this command,
please speak up.

--david
