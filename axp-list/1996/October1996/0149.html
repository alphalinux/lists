Heya!

These problems I will describe might be bugs in the x86 version also, but
I haven't tested that yet. Some of the bugs might be because I screwed up,
so please correct me if you see anything strange...

System: noname,newest milo from 'test-images' ...

I did a *ftp* install from scratch

Install problems: * name-resolving isn't working (I tried 2
different ns's, which I *know* was working at
the time being, using ip-nr's worked ok)

* I tried installing from ftp.ibp.fr, which is a
very fast mirror for me, but I noticed that the
install *needs* passive transfer mode, which
their server doesn't support. This 'error'
made the whole install abort and the machine
rebooted..Hmmm, maybe needs some error-recovery?

Ok, damn, just for finding out I wasn't doing anything wrong, I started to
install directly from ftp.redhat.com (just wanted to see that the
installation started, dl the whole from that server would have taken
days..). Ok started correctly, formatted the partions, got some small
files but then:

* Got an ftp-timeout (I guess to many ppl
wants the 4.0 release :-) and the installation
aborted and yet another time the machine rebooted...

Ok, started again, and I'm using the ftp.gwdg.de mirror now.

* Because I formatted the partitions when I
tried installing directly from ftp.redhat.com
I thought I didn't need to do it again so I
skipped it... Bad move, because the previous
install had kept a bad rpm database, which
this new install didn't remove first, and
ofcourse the machine rebooted yet another
time...

* I selected *all* sets, except "print server" and
"news server", but the installationen only
installed 128 packages. Just the very basic
packages got installed, nothing fancy like X
got installed... (I can send you the install.log
if you want to)

Well, the install now finishes, the machine reboots and everything works
nicely :-) Except for all the missing packages that is, but I just ftp'd
all the packages manually and installed them by hand with rpm.

Ok, now some bugs in the system:

* xload exists in both X11R6-contrib and procps-X11
* Several manpages exists in both libgr-devel and netpbm
* /usr/bin/play exists in both sox and exmh
* /var/log/wtmp is missing (just a simple touch fixes it)
* depmod is 'missing', but is called from rc.sysinit
* fdisk's disklabel support is bad, it insist on all partitions
starting on cylinder 1
* finger(d) output is to wide (no need to print the whole 32char
field)
* joe segv's due to an global non-initialed var. This is *not* an
bug in joe, it should really work (and it works on x86, and
ecoff/axp) so something is broken in elf (ld.so probably). I
think Andreas Johansson sent a mail about this a while ago.

Oh, and the note about 'disklabel-type' partitiontables in the
RH-4.0-Manual is wrong. It says that the first partition should normally
be swap. This is not the case, the bsd style says that partion 1 (a)
should be the rootpartition, 2 (b) should be swap, (c) should be the whole
disk, and from here you can have whatever partitions you like...

And at last some suggestions for the install..

* Have some selectable list of ftp-sites to install from (ofcourse
there should be a 'custom' entry where you can put in whatever
you like).
* Maybe a simple 'ftp-browser' when you use the 'custom ftp-site'
because it can sometimes to be hard to remember the exact
path..
* Include minlabel on the rootfs because of the crappy fdisk
disklabel support
* The possibility to abort the installation

Thats it!

Now I have to play with all the 380 packages of goodies :-) Keep up the
good work!

/Mikael
