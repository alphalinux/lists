I have a standard RedHat 3.0.3 installed on my UDB and
is it running without any problems right now, so I thought
it is a good time to test if I can get my Zyxel Omni 288S
attached to it and use the UDB as Dial-in Server.

I followed the Serial-Howto and tried dial-out and it worked.
Then I added the following line to my inittab
S1:2345:respawn:/sbin/uugetty ttyS1 38400 vt100

Before I did this I checked the gettydefs and set the spd_hi on cua1

I stored the following at commands in my Zyxel:
at e1 q0 v1 s0=0 &c1 &s0

When I restart init ( I set debug = 010 in /etc/conf.uugetty.ttyS1)
i get the following error message in my log-file:

Oct 31 13:25:08 twix uugetty[1206]: warning: INIT sequence failed on /dev/cua1

I cannot figure out if I made a mistake in my modem initstring or
if this is an AXP Problem. Since the init does not work the modem does
not answer any calls.

I thought it might be a problem since I used an selfmade Y-cable for
the second serial port, but it does not work on /cua0 either.

Any ideas?
