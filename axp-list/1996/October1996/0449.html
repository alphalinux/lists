>>>>> "TW" == Thomas Weber <x@AEC.at> writes:

>> The archives for this list are still unavailable and I
>> continue to have trouble with my system clock.

TW> Depends on what you use to boot from, ARC or SRM. You should
TW> give the -A option to clock if you use ARC. Check
TW> /etc/rc.d/rc.sysinit (on a RedHat system) to see what i
TW> mean. To change the current setting see /etc/sysconfig/clock.

Um, is /etc/rc.d/rc.sysinit supposed to do the following when I'm
using the ARC clock?

if [ -f /etc/sysconfig/clock ]; then
. /etc/sysconfig/clock
if [ ${CLOCKMODE} = "GMT" ]; then
clock -u -a
else
if [ ${CLOCKMODE} = "ARC" ]; then
clock -A -a -u
else
clock -a
fi
fi
else
clock -a
fi

Won't the '-a -u' trailing the '-A' just reset it to GMT and not ARC
time? (I'm having problems with time on my UDB, too.)
