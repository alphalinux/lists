>>> Daniel Simmons said:
>
> Now for a quick question: I find two versions of ELF TGA X server on
> the redhat site. One with EV4 in the name and the other with EV5 in
> it. Which one should I use on my UDB? Both seem to work...

Now that *is* interesting; I'll bet if you had an EV5-based machine it
wouldn't be true...

The UDB is an EV4-based machine (actually LCA4), so use the EV4 version.
I think you'll also find it a bit faster than the EV5 version, also.

> I did, however, encounter at least one spontaneous drop out of X using
> the EV5 version, so I'm using the EV4 one for now in hopes that it
> makes a difference. Back under 3.0.3, my X server would drop on a
> fairly regular basis. It seems that Netscape (run from another server
> just displaying locally) would cause the problem more than anything
> else. I saw reports from others on the list of similar behaviour, but
> the reports seemed to indicate that the problem went away with the ELF
> version of the server so I was hoping beyond hope that I would be over
> it now that I upgraded. Doesn't look like I was so lucky.

Sigh; no, the "Netscape" problem is *not* fixed; it's only an ELF version
of the old server, with the font-related problems fixed by back-patches
from XFree86 3.1.2E to the released 3.1.2 sources.

I hope to get the time to fix that problem, but at the moment am rather
busy attempting to port the new Matrox Millenium XFree86 server code to
the Alpha, get the 2940UW driver working (almost there, but crashes when
attempting to run at ULTRA speed), get a working MILO and kernel for the
Alpha XL, aka XL-300 and XL-366 (almost there, the kernel and MILO work
but there's still a problem between AlphaBIOS and linload), so it may be a
while, yet. I *am* aware of how many people out there are using the TGA
server, so it *is* on my TODO list.

Oh, BTW, there will be a different TGA server, with sources, coming out
in the XFree86 3.2 release sometime soon. This may very well not have the
"Netscape" bug, but could be slower, so you'll have to make a decision
when the time comes... :-)
