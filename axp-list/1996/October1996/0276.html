Since the patch applied cleanly, you must have applied it to the bfd
insmod (which is the Right Thing to do). Getting the "regular" insmod
working (the one that comes with the modules package) requires a
non-trivial amount of work (at the very least, relocation needs to be
adapted for the Alpha, but I think there will be some other nastiness
such as creating a .got section). I don't think it's very difficult,
but it takes somebody with some time at hand and the necessary
knowledge of both ELF and Alpha. Until then, bfd-insmod is the way to
go. The only reason I want to go away from bfd-insmod is that there
is a cyclic dependency that makes it impossible to guarantee that
insmod will work for all cases (in practice it should work fine for
all but the most contorted test cases, but still...). The cyclic
dependency is due to the fact that in order to create the final
object, you need to know its destination address, but to know the
address, you need to know the size of the final object. Of course,
one influences the other, hence there is a cyclic dependency.
