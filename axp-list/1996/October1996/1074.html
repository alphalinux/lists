reporting bugs here is one more (already reported some time ago)
but without fix:

rpm -qif /bin/bash
Name : bash Distribution: Red Hat Linux Colgate
Version : 1.14.7 Vendor: Red Hat Software
Release : 1 Build Date: Wed Sep 4 04:44:15 1996
Install date: Fri Sep 20 14:58:01 1996 Build Host: judy.redhat.com
Group : Shells Source RPM: bash-1.14.7-1.src.rpm
Size : 749383

this bash version, like all others I tested so far for Linux/AXP
doesn't work correctly when the terminal size changes.

just run bash in a local xterm or use telnet in a remote xterm and
then log into an AXP system using telnet or rlogin and then resize
the window.

when the xterm doesn't have the original width when bash was started
all command line editing fails horribly when the line is longer than
the original terminal line width...
