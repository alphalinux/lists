On Wed, 16 Oct 1996, Marc Singer wrote:

> > Server is an DEC Alpha 1000
> > clients are:
> > Linux 2.0.18
> > OpenBSD 1.2
> > (both are Dell P120 with 32 meg of ram and 3C509 network
> > cards - The only different is one is running BSD the other
> > Linux)
> >
> > File size: 175630534
> > time cp access.log /dev/null
> >
> > Linux bytes/ps: 470,224 in 390 seconds
> > OpenBSD bytes/ps: 1,021,226 in 171.98
> >
> > Am I missing something. I hope so!!! This is about the same
> > speed that I saw with early version (1.9 etc) of Linux.
> >
> > Is there any thing I can do to help NFS writes? Writes is about 27,000
> > bytes/ps.
>
> I have experienced similarly 'slow' NFS performance on reading. It is
> possible that the OpenBSD version had cached more (or all) of the
> source file. Still, I'm skeptical of this 1MB/s transfer rate since
> that is greater the theoretical throughput limit of 10BaseT.

No, OpenBSD didn't cache any of the source file. The file
was large almost 200 meg. The theoretical throughput of ethernet
is 10 Mbit/s serial synchronous, which is 1.25 MByte/s, this still
leave 18.4% of the bandwidth remaining for protocol overhead and
acks.

Linus suggestion to increase the buffers to 8k really helps. It went
from 470KBytes/s to ~940KBytes/s on reads and this was during the
day with others using the LAN.
