The *ONLY* valid Modelines are the ones that appear in the sample XF86Config.
That is because the TGA is *not* programmed like the VGA, WRT video parameters.
The only meaningful parts of the Modeline to the TGA code are the name and the
dotclock rate; of the remaining parameters, the generic XFree86 code checks
to be sure that the given resolution/refresh can be handled by the monitor.

And the safest way to get the server to choose the desired Modeline is to
have only one uncommented. I just can't remember if the code simply takes
the first one it finds, or the last... :-)

The TGA code only knows how to program the clock to the fixed number of
resolution/refresh rates. Unfortunately, you must find the best one for your
monitor and live with it (or get a better monitor... :-).

The TGA server that will be available as source in the next XFree86 release
still has this limitation, but at least will be open for others to enhance.
