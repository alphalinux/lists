> Well, ftp sites have MILO sources numbered the same as kernel sources.

> To quote the MILO How-to:
>
> "you must first configure and build the kernel that matches with Milo that

> want to build. Mostly this means building the kernel with the same version
> number. So, milo-1.3.76.tgz will build against linux-1.3.76.tar.gz. Milo may
> build against a higher version of the kernel, but there again it may not."
>
> So either you are mistaken, or the HOW-TO is wrong. Which is it?

The key word above is "build"; MILO requires many pieces from a kernel build,
and when the kernel changes the way things work, MILO has to change as well,
in order to be able to use those pieces. So the MILO version is meant to
correspond with a kernel version against which it *builds* successfully.

Once MILO is built, there is no dependency on kernel version.

However, older MILOs may *not* have the feature set necessary to boot later
kernels, the example being an old MILO without ELF support being asked to
boot a kernel in ELF format.
