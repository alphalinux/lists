Redhat V 3.03 for Dec Alpha

Configuring and building a new kernel was easy (with xconfig), but when I
reboot with the new kernel I get the followig error:-

scsi-ncr53c7,8xx : at PCI bus 0, device 6, function 0
scsi-ncr53c7,8xx : NCR code relocated to 0x40054694 (virt 0xfffffc0000054694)
scsi0 : burst length 8
scsi0 : test 1 started
scsi0 : driver test 1 timed out due to lost interrupt
Please verify that the correct IRQ is being used for your
board,
and that the motherboard IRQ jumpering mactches the PCI setup on
PCI systems.
If you are using a NCR53c810 board in a PCI system, you should
also verify that the board is jumpered to use PCI INTA, since
most PCI motherboards lack support for INTB, INTC and INTD.

I know this has nothing to do with the configuration of my SCSI card, as
when I boot with the supplied kernel on RedHat CDROM, I get:-

scsi-ncr53c7,8xx : at PCI bus 0, device 6, function 0
scsi-ncr53c7,8xx : NCR53c810 at memory 0x4001000, io 0xb400, irq 11
scsi0 : burst length 8
scsi0 : NCR code relocated to 0x40058694 (virt 0xfffffc0000058694)
scsi0 : test 1 started
scsi0 : NCR53c{7,8}xx (rel 17)

(You don't get these problems when you rebuild a SunOS or Solaris kernel).

Has anyone any idea how I set the SCSI driver to the correct IRQ (11) ?

Thanks in advance for any help.
