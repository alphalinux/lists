I tried out the new milo and fmu.gz images from
gatekeeper on my pc164. The fmu now recognizes
NVRAM and seems to program it and the environment
block OK. Milo stores and loads environment
variables correctly. Both milo and fmu seem
to be able to set bootopt (as the value stays
after a powerdown/up) but the value of bootopt
seems not to affect which NVRAM image boots.
It is always the Win NT ARC monitor.

The first flash image shown by fmu's "list"
command seems to be NT, implying that the MILO
image might be image #2, so I tried bootopt set
to 130 and then also to 131, with no success.
I also tried other values such as 2, 3, 67, etc.
but Win NT ARC always booted.

Evidently though milo and fmu both read and
write "bootopt" correctly, the system is not
correctly interpreting the value or is reading
from a different address.

Does anyone have an idea about how to get
a different boot image than Win NT ARC to be
recognized ?
