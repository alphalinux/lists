Now that I have RH 4.0 running (actually more like limping, but that's another
story), I'm trying to build Milo 2.0.18.

The first problem I ran into was that x86_sysenv.c wouldn't compile. Somewhere
down deep in the nested headers some things aren't getting defined. I spent
half an hour trying to track this down, but I finally gave up because (1) it
really looked like the stuff *was* getting defined, and (2) I don't need any
x86 BIOS emulation of any kind, so it seemed like a waste of time to keep
studying headers that are only causing problems with that one file.

So I hacked the makefile and video.c to add a new NO_BIOSEMU option. I had
to copy the malloc function (which calls vmalloc) from xf86_sysenv.c to
another file because the DOS file system uses it. And now it all compiles.

But the link is giving hundreds of "truncated relocation" errors. From what
I've seen on the list previously, this happens when linking COFF and ELF
modules together. I'm suppose that the COFF osfpal.o is the problem.
Is the version of gas in binutils-2.7.0.2 able to build palcode yet?

And can linload.exe handle an ELF Milo anyhow? Perhaps I need to figure
out how to build a COFF Milo? What does it take to build COFF stuff on
a predominantly ELF system?

Cheers,
Eric
