On Sat, 19 Oct 1996, David A Rusling wrote:
> It sounds like MILO is not getting interrupts anymore. Normally this should
> timeout but even that is not happening. Does the WNT ARC firmware see the
> disk properly?

The WNT ARC firmware sees the disk just fine when my external drive is NOT
connected.

I've determined that the problem probably ISN'T Milo. First off, if I
have my external drive running (I've got /usr mounted on it) it takes
about twice as long for the system test to complete the test on the
NCRC810 driver. Invariably, at this point, when the normal boot of Linux
takes place, I get the nice red error telling me that there is a bad
device or partition. *If*, on the off chance I can get it to boot through
that part, I get a whole bunch of scsi reset errors, and then it just
bombs. No dice.

On the other hand, if I disconnect the external drive entirely from the
box, I can almost always boot up. But I still see a few scsi reset errors
and the like. It's not the cabling - I've switched that around, a few
times. I'm starting to wonder if the SCSI2 connector on the back of
this box is weird, or maybe the scsi bus itself is fried somehow. I've
cracked open the box and tried re-seating the connectors, but they were
not loose.

Anyway, I'm getting the impression that it's a hardware problem. And
since this is most likely the case, anyone know where to start to try and
get the darn thing fixed?
