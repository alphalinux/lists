> The program contains almost no floating point arithmetic (it is a cross
> assembler for a variety of microcontrollers like the 8051, HC11...), but
> it contains a lot of string operations..

Do you call <string.h> routines to do all the work or do you play with
the bytes yourself? If the former, this might be entirely a library
issue. The ECOFF libc did most of its string routines in C, and most
of them byte-by-byte. I've gotten 8 to 20 fold increases rewriting
them in assembly.

You might try using the assembly string routines added to the 2.1.[45]
kernel and see if that helps the cause. The only important one not present
is str*cmp. I'd been meaning to finish that one up; perhaps this will
give me a prod.
