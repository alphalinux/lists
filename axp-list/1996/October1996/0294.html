This is a little off-topic for axp-list, but I will post it anyway,
in the that someone might benefit from the explanation and/or
will correct my misconceptions.

The role of the .login file is to set up your environment; it should
be shell-independent, and doesn't have to be idempotent (you are
supposed to use it only once in a session). The setup might involve
setting environment variables such as PATH, MANPATH, etc; choosing
terminal characteristics ('stty', TERM) based on where you are logging
in from, setting up mail notification ('biff') etc.

.xsession is run on login by xdm, and is treated as a main login
porcess, i.e. xdm treats the session as alive as long as the .xsession
is running. Typically, it will set up some environment variables,
start up some X window clients and exec a main login process (most
often the window manager, but some people prefer an Xterm). The exit
of the exec'ed process results in logging out and restarting the login
process. When your .xsession says '#!/bin/bash', bash simply runs but
is not given any commands to execute, exits and you get logged out.

You could simply put the stuff from your .login file into .xsession; or, if
you want to maintain a common .login for X and non-X logins, you could
keep your setup in .login and 'source $HOME/.login' in this file.
