> The archives for this list are still unavailable and I continue to have
> trouble with my system clock.
>
> I have an AS255, running 2.0.18. Although I have set the date using the
> 'date' command and rebooted a number of times, the date continues to
> reset to the year 2000.

Depends on what you use to boot from, ARC or SRM. You should give the
-A option to clock if you use ARC.
Check /etc/rc.d/rc.sysinit (on a RedHat system) to see what i mean. To change
the current setting see /etc/sysconfig/clock.
