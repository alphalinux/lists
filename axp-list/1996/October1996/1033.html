> I've been told, that the trick with the add library in /lib instead of
> /usr/lib works only if this library is really moved instead of only
> copied (what I did).

Copying definitively works, but you probably have to rerun ldconfig
afterwards (don't remember exactly since I did this a while ago). As
long as you get something like this:

$ ldconfig -p|grep nss
68 - ELF libnss_files.so.1 => /lib/libnss_files.so.1
69 - ELF libnss_dns.so.1 => /lib/libnss_dns.so.1
70 - ELF libnss_db.so.1 => /lib/libnss_db.so.1

you should be fine. Moving will save you some diskspace, but I'm just
very cautious when dealing with such fundamental shared libraries,
that's why I suggested copying.
