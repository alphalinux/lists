If this is the case than either your name resolution is not setup
properly or this is yet another manifestation of problems in
resolver libraries. It should work with a name.

BTW /etc/hosts.equiv, which looks roughly the same, should not work
for 'root' and if it does not, then this is a good thing. :-)

The file .rhosts has lines which look like this
<machine_id> [<optional account name>]
...
You may have different account name on different machines, so you may
tell here about it.
