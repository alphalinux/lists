>>> Eric Smith said:
> Now that I have RH 4.0 running (actually more like limping, but that's
> another story), I'm trying to build Milo 2.0.18.

WARNING WARNING WARNING WARNING WARNING WARNING WARNING WARNING WARNING

!!!!!!!!!! YOU CAN NOT BUILD MILO UNDER AN "ELF" SYSTEM !!!!!!!!!!!!!!

Well, maybe you *can*, but it's bound to be a *lot* of trouble.

The DEC BIOS emulation library is ECOFF, thus will *not* link with ELF
objects.

The same is true of the PALcode objects, unless you completely rebuild them.
I don't know if this is possible (yet) under the ELF compilation system.

For the time being, use the canned MILO's available from gatekeeper.dec.com.

Yes, I know there are some problems with the serial echoing that is built
into all of them; if you scream loud enough, maybe someone will rebuild them
without such...

Sorry; there's always some hidden tradeoffs; you didn't think ELF was *free*,
did you??? :-) :-) :-)
