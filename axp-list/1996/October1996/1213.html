Hi netters,

the network setup of RH 4.0 seems to have a weekness which comes because
the standard ethernet interface is evaluated before the localloop:

In my configuration:

router ---------| <--- local net
UDB

the default route goes over the gateway (router). Nameserving (besides
the hosts on my local net which are in /etc/hosts) goes over the gateway too.

My /etc/nsswitch.conf
hosts: files dns

My resolv.conf lists my nameservers which are not on the local net.

In the ifup script eth0 comes first, with the default route going over
eth0 and using gate router. When now lo should be configured the
situation is that

a) all routes to my local net go over eth0 and
b) all others (esp. 127.0.0.1) goes over the gateway

when my router is switched off this doesn't work (and I can't reboot my
Alpha without the router on).

My solution is easy but patches files which should go conform with the
distribution (GATEWAY=number to ifcfg-lo and use the "broken" line in
ifup but with the correct form route add default gw ${GATEWAY}).

I suggest therefore to change ifup in a way, that lo is processed first
(this should include PCMCIAs too); maybe independent of the other network
(a localhost is needed even if ethernet is not connected).
