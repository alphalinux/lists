On Sat, 26 Oct 1996, David Richards wrote:

> Redhat V 3.03 for Dec Alpha
>
> Configuring and building a new kernel was easy (with xconfig), but when I
> reboot with the new kernel I get the followig error:-
>
>
> scsi-ncr53c7,8xx : at PCI bus 0, device 6, function 0
> scsi-ncr53c7,8xx : NCR code relocated to 0x40054694 (virt 0xfffffc0000054694)
> scsi0 : burst length 8
> scsi0 : test 1 started
> scsi0 : driver test 1 timed out due to lost interrupt
> Please verify that the correct IRQ is being used for your
> board,

Did you perhaps build in the MSS sound driver? I believe it defaults to
irq 11 when you configure it, but it really needs to be set to irq 9. I've
never used xconfig so I'm not sure if it even gives you the option of
setting the sound irq. You may have to run the good 'ole "make config"
instead.
