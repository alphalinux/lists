There is one more issue with performance of Fortran code which
I did not recall seeing raised.

The problem is that a basic 'integer' type, at least in 3.0.3 Red Hat
distribution for axp, is typedefed to long. This was probably done
so f2c will compile correctly on MS-DOS machines. On axp it has this
side-effect that various Fortran types, which are of width 4 on other
platforms, are 8 bytes wide. This itself may have some impact on a
performance - one way, or another. I do not really know but anybody
measured that?

Another nasty gotcha from that is that real*4 and integer*4 on Linux
axp aren't. This is a straight bug which affects adversly ports of
"dusty decks". Also various flags to f2c and fort77 do not perform
as advertised. Linking with functions written in C is also affected
(one can work around, but my physicists friends are lost).

Fixing that will make object files compiled with f2c "before" and "after"
incompatible. Sigh!
