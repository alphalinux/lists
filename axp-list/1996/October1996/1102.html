On Sat, 26 Oct 1996, Chris Owen wrote:
> On Sat, 26 Oct 1996, Scott Serr wrote:
> > I've been trying to install 4.0 via ftp from my mirror (Intel 3.0.3),
> > with no success. I've also tried install from NFS with no success.
> > Next week, I will attempt to mount a big scsi disk on one UDB (3.0.3)
> > put the files on the ext2 fs and then take the disk over to the
> > (soon to be 4.0) UDB for a hard drive install.

> I went the HD route. Seems like a much cleaner solution, especially if
> you have more than one machine (why FTP it twice). Also I wanted to make
> some modifications to the install and could just start over. Wouldn't
> want to do that via FTP.

I went the FTP route. It was a pain in the ass to download everything to
my dUNIX box, but the actual FTP install went swiftly, smoothly and
without a hitch. What sort of problems were you having with the FTP
install?

> > If someone could please tell me if they have had ANY success with
> > an FTP or NFS install on the Alpha I would appreciate it.
> >
> > Also, is all this worth it? The 3.0.3 isn't on the RedHat site anymore,
> > should I try searching for someone that has this archived? I'm
> > worried that I will be fighting these "baddies" for months...
>
> I upgraded both of my machines and have had zero problems, but then again
> I'm a luck machine ;-]

Agreed. No problems at all, except a day or two when SCSI went crazy due
to a bad connection inside my external case, but that had nothing to do
with Redhat whatsoever.

I probably should add that I didn't do an upgrade, rather I did a complete
re-install via FTP, which might account for my success; I have heard
problems occuring during upgrade. But, if you have any questions I'd
be happy to answer them if I can.
