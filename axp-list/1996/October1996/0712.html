> added in the first place? Apart from the problems I have, you get inconsistent
> drive numbering between kernel/aboot/fdisk.

This is not right, either - I get eight partitions, where only the first
three are valid (but at least these do work). The following works as
expected:

*** genhd.c.old Thu Oct 17 17:13:27 1996
--- genhd.c Fri Oct 18 16:41:16 1996
***************
*** 233,239 ****
if ((current_minor & mask) >= (4 + hd->max_p))
break;

! if (p->p_fstype != BSD_FS_UNUSED) {
add_partition(hd, current_minor, first_sector + p->p_offset, p->p_size);
current_minor++;
}
--- 233,239 ----
if ((current_minor & mask) >= (4 + hd->max_p))
break;

! if (p->p_size != 0) {
add_partition(hd, current_minor, first_sector + p->p_offset, p->p_size);
current_minor++;
}
