The driver is probably configured for 10 MHz scsi synchronous negotiation.
After boot up, you can increase the sync period or set asynchronous
transfer mode for your hard disk.

The command that allows to change synchronous parameters for a device is:

echo "setsync #target period" >/proc/scsi/ncr53c8xx/0

- echo "setsync 0 255" >... renegotiate asynchronous transfer with target 0
- echo "setsync 0 50" >... 5 MHz synchronous transfer
- echo "setsync 0 25" >... 10 MHz synchronous transfer (max speed)

If it is a hardware problem, you will have better chance with
asynchronous transfer (setsync 0 255).

I donnot know which sync parameters are used by NT4. In my opinion, it
should be "asynchronous" or at most "5 Mhz sync transfer".
