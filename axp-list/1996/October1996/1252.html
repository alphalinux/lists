Hi all,

Sorry if I'm overlooking something obvious, but how do I get f2c to
map INTEGER*4 to 4-byte integers on Linux/AXP?

>From the f2c man page:

-I2 Render INTEGER and LOGICAL as short, INTEGER*4 as long
int. Assume the default libF77 and libI77: allow only
INTEGER*4 (and no LOGICAL) variables in INQUIREs.
Option `-I4' confirms the default rendering of INTEGER
as long int.

Also, f2c.h has

typedef long int integer;
typedef short int shortint;

Under Linux/AXP, short is 2 bytes, int is 4 bytes, and long is 8
bytes, right?
