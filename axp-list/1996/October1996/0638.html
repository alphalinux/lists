I wrote earlier:
>I don't know if this is related, but I have a double-precision FFT
>fortran code that only seems to produce single-precision results (as
>compared to output from SGI and Digital Unix f77 compilers) using f2c
>on Linux-x86. I haven't tried to track it down.

Just a follow-up that may be of interest to f2c users. The fortran
code

IMPLICIT DOUBLE PRECISION (A - H)
c72=0.30901699437494742

yields the following C code under f2c:

static doublereal c72;
c72 =(float).30901699437494742;

Why "float"? However, the following fortran source fixes the problem:

c72=DBLE(0.30901699437494742)
