I posted a couple of hours ago about 4.0 oddities. I now realize that
some of the #define conflicts came from not having the latest axp diffs
applied to the kernel source. Doing so cleaned up some of the noise level
messages.

Also, I just found that SIGCLD is not defined anymore in
/usr/include/signal.h; it was defined in the 3.0.3 i386 release. I was
compiling the original source on Vixie cron and found that it refers to
SIGCLD, not SIGCHLD. Should this be put back in?
