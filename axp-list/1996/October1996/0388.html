On Sun, 13 Oct 1996, Michael Schwingen wrote:

> The kernel reads the root disk, the install starts up and everything is fine
> - until after installation of the packages, the install wants to copy the
> boot (kernel) disk - what boot disk do I ask?

You need to make a disk with the Jensen kernel on it (well, and vmlinux.gz
will do).

> So after creating one, it copies a (useless) kernel to disk, and - hangs.

That's not good :-(

> - can I sync and reboot, or would the install do anything important after
> this step which I am missing? The machine seems to boot fine, but I don't
> want to risk problems later due to incorrect rpm databases or whatever
> else might be wrong.

You'll be fine with a sync/reboot. The kernel is installed last as lilo is
one of the trickier things to do on Intel's. Seems like copying a kernel is
tricky on an Alpha as well :-(
