Yeah, but.....

If you apply the axp-diffs (2.0.21 to 2.0.23 source kernel, for example)
then the pcmcia modules WONT COMPILE!

I've been piddling with trying to get pcmcia working again after upgrading
to RH4. There are a set of 'backwards compatability' items in the 2.0.23
kernel source include files (like in termios.h and some other places) that
the pcmcia sources want to see.

However, if you don't apply the axp-diffs, you get blown out of the water
during the config step in the kernel make. A catch-22 (or catch-23 in this
case....:)

Anybody have a nice solution yet?
