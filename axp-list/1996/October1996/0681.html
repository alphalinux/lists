I have not found docs on nsswitch, so I resorted to editing the
host.conf file. I changed the order to be hosts exclusively. And,
the machine boots colgate.

Now, It is clear that there is something fishy in the way that route
and ifconfig do name resolution since nslookup DOES work. If I boot
the machine in single user mode, I can use nslookup to resolve the
same address that route fails to resolve. Once I changed the hosts
file, I was able to use successfully the route command.

So, what gives? Have you at RH tested the resolver library? It seems
preposterous that I am the first to find this trouble since DNS is so
fundamental.

We know that the interface works because I can ping. We know that the
name server will respond to thumper. We know that route works because
it is OK when not using DNS. Hmm..

I did an strace on nslookup to see what it is sending to the named on
florence. The data in the packet does not look the same as the data
that was sent by route's poll. How can we verify that the libresolv
is OK?
