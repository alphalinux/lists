Oops. The 2.1.4 axp exceptions patches had a silly bug in <asm/segment.h> for
the alpha: the "strncpy_from_user()" macro has a "{" for the if-test, but no
corresponding "}".

It looks something like this:

if (__access_ok(((long)__sfu_from),__sfu_len,__access_mask)) {

and you should just delete the "{" (or add a closing brace after the call
to __strncpy_from_user()).

I didn't get to test the patches on an alpha before I released 2.1.5, so the
clean 2.1.5 sources simply won't compile fs/namei.c without the fix. I only
now tried to compile it all on my ev5 - I haven't rebooted to make sure it
actually works, but at least the above fixes the compile. Oops,

Linus
