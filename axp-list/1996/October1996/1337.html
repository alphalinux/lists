The supplied binary of tracker dumps core whenever you run it. This seems
to be because of an incompatibility with glibc; when building, it
complains during the final link:

[root@paddington tracker]# make
gcc -o tracker -funroll-loops main.o Linux/./audio.o st_read.o commands.o
audio.o automaton.o player.o getopt.o open.o tools.o dump_song.o
setup_audio.o notes.o display.o Unix/./ui.o prefs.o tags.o autoinit.o -lm
ld: Warning: size of symbol `error' changed from 4 to 444 in
/lib/libc.so.6
[root@paddington tracker]#

If the binary is statically linked, it works correctly.
