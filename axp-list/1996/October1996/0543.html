On Wed, 16 Oct 1996, Andrew J. Hutton wrote:

> Nope, the FTP install for i386 is buggered too. Seems like they install from
> CD or NFS at RedHat for testing. Kinda embarrasing for them though, when it
> works it works great, but not everyone can figure it out. I hope they didn't
> print too many first revision CD disks.

Very few ftp users get their install images from CD -- this bug isn't much
of an issue for CD users at all. It also doesn't affect Intel or SPARC
users.

It is pretty embarrsing though. We actually do ftp installs regularly (they
are faster then NFS ones most of the time) but we didn't test that one
case. This bug also doesn't affect the SPARC or Alpha ports (due to slightly
different ramdisk layouts) which made it a little trickier to catch (we
tend to switch from Intel to SPARC/Alpha testing near the end of our cycles
and this bug was introduced right around that switch).

At any rate, intel ftp users can find new supp.img files on ftp.redhat.com
that fixes this problem.
