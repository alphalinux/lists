Most likely this is due to insufficient debug info in libc.
Unfortunately, gdb on Alpha does not handle this situation well (and,
even in the best case, there is only so much gdb can do without
debugging info---the DEC Unix calling convention is highly optimized
and just looking at the code etc will not give you all the information
to, say, backtrack in the stack).

I can guarantee you that gdb works just fine on malloc---I've spent
many long hours tracking a bug or two in it and it definitively works.

On pre-4.0 systems, you can link against -lg (instead of -lc) to get
debug info for the standard libc. I'm not sure there is an easy way
of doing this on 4.0 (short of recompiling libc). Maybe the next
release of RH will include debug info for all shared libraries. The
only cost of doing this is that the shared libs take up some extra
space, but given the current diskprices, I think it's well worth it.

As for compilation problems due to the 8MB stack limit: all I can say
is that I have built gcc and lots of other stuff many times without
having seen any problems like that. Not to say that the 8MB stack
limit isn't too small, but maybe there is something else that makes
the problem more acute.
