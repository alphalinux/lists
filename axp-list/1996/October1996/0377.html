>
> After reinstalling Red Hat 3.03, and trying to upgrade via 4.0 Boot
> diskette, the following has been observed:
>
> 1. Red Hat 4.0 will not detect any of my partitions on my SCSI drive.
> 2. Red Hat 3.03 has no problem with this.
> 3. Has anyone seen this error before, or have had this problem before?

Yes, see my rembrant posts on this mailing list. They should be in the
mail-list archive. If you dont find them I can forward them to you. I
should have them somewhere.
The problem is that the /tmp/sda device file entry
is removed somwhere in the install2 program and is not recreated.
There are a few unlink calls in the hd.c file that might be wrong but
I have not had time to look deep into it. This hd.c file can be found
in ftp://ftp.redhat.com/pub/redhat/redhat-4.0/axp/misc/src/install/
as well as the rest of the install program code (at least I think it
is the whole install program code).

Maybe we should look for some common factor between our hardware. This
seems to be a rare problem. Any way my hardware:

166Mhz UDB
24MB Ram in two 12MB simms.
520MB 2.5" Hard disk from Toshiba (Don't remember the model, but can
find out 8-))
I think (more) standard is 340MB disk, so this could be part
of the problem.
SCSI controler is NCR810, i.e. the standard controler for UDB.
"standard" TGA vidoe card, that comes with the UDB.
