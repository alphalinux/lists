Probably a better idea to do make mrproper first then make menuconfig
otherwise you will erase what you configured. Sorry for the mistake.

On Tue, 22 Oct 1996, D. Steven Daniel wrote:

> The procedure I use is :
>
> make menuconfig (a curses based fs config editor)
> make mrproper
> make dep
> make clean
> make
>
> After the kernel compiles , I run make boot. This places the compressed
> kernel in /usr/src/linux/arch/alpha/boot , I then copy the kernel to /
> using an alternate name (not vmlinux.gz) to keep the old kernel in case I
> screw up. I then reboot the machine and from the Milo prompt I type boot
> sdxx:yy.gz root=/dev/sdxx where yy is the name you gave new kernel and xx
> is the partition where / resides. If anyone has a better way (which I am
> sure there is) please let me know. Hope this helps.
