> So after this long introduction, is there a way to change the fonts
> in the TGA so that iso-1 encoded fonts will by displayd correctly?

The TGA has no hardware character generator, like VGA; the driver must do
the rendering of the characters from bitmap data compiled into the kernel.

For the initial release of the TGA support in the kernel, I picked the
default 8x16 font from the kbd-092 (?) distribution, and built the approp.
table into the driver.

There's nothing that would prevent that table from being dynamically
replaced, as VGA fonts are, but the rendering code is pretty particular
(read "hardcoded" :-) about the bitmap format and character size.

If all you need to do is fix up certain characters, feel free to have a go
at the table in <drivers/char/tga.c>. If you want/need help in any more
ambitious project in this area, let me know.

Good luck.
