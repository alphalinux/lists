> > 0: 0/vmlinux.gz root=/dev/sda2
> > 1: 2/vmlinux.gz root=/dev/sda2
>
> I think it should work if you do the following:
>
> set boot_file "1/vmlinux.gz"
> set boot_osflags "root=/dev/sda2"
> set bootdef_dev DKA0
> init

Or - if you are using a Jensen:
set boot_osflags "1:3"
set bbotdef_dev DKA0

to get entry '3' from aboot.conf, where aboot.conf is located on partition
'1'.

> Also, note that you can't have a "0 partition"; both minlabel and fdisk start
> numbering partitions with "1".

Right, but aboot uses the '0' to mean a kernel that is written directly
behind the aboot code, without any partitioning of filesystem being
necessary for this to work.
