On Fri, 18 Oct 1996, Przemek Klosowski wrote:

> f2c (using it by itself and with the various drivers) gives me bad results
> with rather simple programs. I was using a simple, well known Runge-Kutta
> diff eq solver (rkf45.f at netlib) which works fine on my HP palmtop,
> various Suns, OSF/1 boxen and etc, but it won't work on Linux-AXP or
>
> I have often seen Fortran code that fails on f2c because of the
> missing SAVE's, i.e. the code assuming that subroutine local variables
> are preserved across different calls to this subroutine. This
> assumption is not only invalid as per Fortran 77, but is ...

f2c allocates all local variables static _unless_ you specifically tell
it with the -a switch to make local variables automatic. By default, f2c
plays it safe when it comes to SAVE - SAVE's are not required.
