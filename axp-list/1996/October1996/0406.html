I found the problem with the Alpha port of ssh; I suspected the nature of
the problem, so tracking it down wasn't too hard. The problem is in
rsaref2. The file ~/rsaref2/source/global.h defines UINT4 as follows:

/* UINT4 defines a four byte word */
typedef unsigned long int UINT4;

typedef unsigned int UINT4; /* fixed*/

On the Alpha, a long is 64 bits, which broke a routine in digit.c.
Changing the typedef to remove the long fixed the problem.

I realize that rsaref2 is not part of ssh, and I don't know what it would
take to change the code. I have also sent this to the ssh mailing list.
This should only affect US users, as rsaref2 is apparently not needed
outside the US (patent licensing problems, I think).
