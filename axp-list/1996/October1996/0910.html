I was installing a network card and was curious what effect it would
have on NFS and raw throughput. Also took the opportunity to
see actually how fast the linux nfs is thesedays. Figured I'd share
the results:

Alpha UDB client, standard ethernet, 24 mb ram, 340 mb EIDE, redhat-4.0,
kernel 2.0.18

Dell p6-200 server, NE2000, 64 mb ram, 2.5GB disk, redhat-4.0, kernel 2.0.22

Bonnie run on UDB client. Disk mounted from Dell server 1k blocks.

teleost:/mnt on /mnt type nfs (rw,addr=168.150.200.3)
-------Sequential Output-------- ---Sequential Input-- --Random--
-Per Char- --Block--- -Rewrite-- -Per Char- --Block--- --Seeks---
Machine MB K/sec %CPU K/sec %CPU K/sec %CPU K/sec %CPU K/sec %CPU /sec %CPU
72 329 25.6 354 20.3 167 21.5 317 27.9 340 23.2 52.3 23.7

Same as above with 8 k blocks.

teleost:/mnt on /mnt type nfs (rw,rsize=8192,wsize=8192,addr=168.150.200.3)
-------Sequential Output-------- ---Sequential Input-- --Random--
-Per Char- --Block--- -Rewrite-- -Per Char- --Block--- --Seeks---
Machine MB K/sec %CPU K/sec %CPU K/sec %CPU K/sec %CPU K/sec %CPU /sec %CPU
72 701 26.4 756 21.2 382 23.8 728 33.9 875 21.5 100.8 14.8

[root@paladin netperf-2.1pl1]# ./netperf -H teleost -l 15 -t TCP_STREAM
TCP STREAM TEST to teleost
Recv Send Send
Socket Socket Message Elapsed
Size Size Size Time Throughput
bytes bytes bytes secs. 10^6bits/sec

65535 65535 65535 15.02 7.89
