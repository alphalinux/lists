>>> Bill Scanga said:
>
> The problem is that when I get to the "Choose A Card" option menu of
> Xconfigurator, the program consistently aborts no matter which video card
> option I choose. An error message results resembling:
>
> Server doesn't exist, can't continue.
> tried to use /usr/X11R6/vin/x86_P9000
>
> The filename changes depending upon my choice in Xconfigurator.
>
> Does somebody know where I can find the right server for my particular TGA
> video adapter? How can I figger out exactly what adapter is installed?
> dmesg doesn't seem to tell me. All I know for sure is that upon bootup,
> MILO reports:
>
> Video is Digital BIOS Emulation (VGA), Zlxp(TGA)

I've found the easiest thing to do is simply tell Xconfigurator that there's
a Mach64 installed. It'll ask a few questions about clocks and video memory
and such; it doesn't really matter how you answer, you're going to throw away
the /etc/X11/XF86Config file it builds for you anyway... :-)

!!!!!!!!! DO NOT UNDER ANY CIRCUMSTANCES LET IT PROBE FOR YOU !!!!!!!!

Tell it to skip the Autoprobe part, otherwise you'll prolly have to reboot,
as it *will* hang the machine.

Then, when the install is done, reboot and remount the CD, go to the X/
directory on the CD and read the README. All the files you need for doing
the install of the TGA server and XF86Config file are there.

Good luck.
