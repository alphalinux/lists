>>> "Scott D. Webster" said:
>
> I've been trying to get a watchdog timer board running in a EB164
> to no avail. I was wondering if I could entice any of you system
> programmer guru types to take a peak at the code and see if you find
> anything that would suggest that it just wont work on an alpha. The
> driver code is in linux/drivers/char/wdt.c.

Since it just uses inb/outb, it should be fine. Did you check the card
configuration versus the driver configuration? There's some comments in
the code that lead me to believe you have to manually configure the driver
for the jumper settings (?) on the card, ie IRQ and IO base.

> Linux just doesn't seem to know that either the driver or the card
> exists. I'd think I should at least get an error message if the driver
> initialization was crapping out on boot up.

You're correct, the driver ought to print something out if it gets called
to init, ie wdt_init(), even if the hardware isn't present, as noted by the
comment:

* You must set these - there is no sane way to probe for this board.

So, did you try to configure it as a MODULE? Or, did the "make boot" actually
compile a wdt.o?
