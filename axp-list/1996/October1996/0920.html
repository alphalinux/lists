David A Rusling <rusling@linux.reo.dec.com> wrote:

>MILO does not have to be the same version as the kernel to load
>it. Just as libc doesn't - it's just a tool. Do you upgrade
>LILO every time you boot a new kernel - I don't.

Well, ftp sites have MILO sources numbered the same as kernel sources.

SEE: ftp://ftp.digital.com/pub/DEC/Linux-Alpha/Miniloader/src

To quote the MILO How-to:

"you must first configure and build the kernel that matches with Milo that you
want to build. Mostly this means building the kernel with the same version
number. So, milo-1.3.76.tgz will build against linux-1.3.76.tar.gz. Milo may
build against a higher version of the kernel, but there again it may not."

So either you are mistaken, or the HOW-TO is wrong. Which is it?
