I put a 2.9G externel drive on my 233/multia and set it up as
follows:

[cmr@iroha cmr]$ df -h
Filesystem Size Used Avail Capacity Mounted on
/dev/sda1 105M 17M 82M 17% / ( a large root)
/dev/sdb3 704M 13K 668M 0% /export1 (export dir)
/dev/sdb1 991M 48M 892M 5% /home (1G of ext drive)
/dev/sdb2 991M 381M 559M 41% /usr (1G of ext drive)
/dev/sda3 292M 8.3M 269M 3% /var (Main part of int dr)
/dev/scd0 626M 626M 0 100% /cdrom (1st RH cd)
koibito:/cdrom 283M 283M 0 100% /cdrom_net (2nd RH cd)

and 100M for swap on the internal drive.

"Oct 28 16:49:15 iroha kernel: Adding Swap: 110584k swap-space"

As you can see, the internal drive is used for the root, var, and
swap partitions. If we disregard the /home , we can see that the initial
install used 406M plus swap. I guess you could put everything on a
540 drive if you used a small swap partition, but the 340M drive can
clearly only take a partial install.
