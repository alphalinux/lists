I hate to do this, but I can't compile any kernel except for the
kernel-source.RPM included on the RH 4.0 CD. In fact, I downloaded
the *same* 2.0.18 kernel from Linus' site, but it failed to work!
The problem has to do with a missing 'timespecbits.h' in the include/linux
dir along with a few other odd problems.
I tried stealing the file from the RH kernel-source RPM, but the kernel
still failed to build. Is it impossible to compile an axp-diff'less kernel??
This is quite frustrating since I'm trying to compile a non-AXP-DIFF'ed
kernel (because I'm using an ECOFF server). None will work. I've tried them
all past 2.0.18 with the same problem. Obviously some files are missing from
RH 4.0. The interesting thing is that with RH 3.0.3, I easily compiled 2.0.18
kernels, without ever having these errors.
If I can't compile any other kernels, is anyone planning on making an
Xtga-ELF server for the UDB??
