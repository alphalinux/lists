Ok, 2.1.1 is available at the usual places. This kernel fixes:

- vm86 mode hopefully works, so dosemu people please try it out again..
- initrd should compile. Whether it works needs to be tested by somebody
else.
- eata SCSI driver updates for 2.1 mm handling
- fdomain SCSI driver update for 2.1 mm handling
- SCSI tape bugfix
- UltraStor 14F/34F SCSI driver update for 2.1 mm handling
- sound should work
- file locking update
- NCP filesystem should work again.
- UFS update
- umsdos updates
- asm-i386/unistd.h error detection update
- arp NULL device fixes

NOTE! The problem with module loading is _not_ a bug in the kernel, it's a
problem with the error detection in user space. Don't send me kernel patches
for this: you need to recompile insmod with patches that have been floating
around here on the kernel mailing list.

The arp NULL pointer fixes and the SCSI tape bugfix is also in my personal
2.0.x tree for future release (probably 2.0.22 early next week).

Note that the SCSI driver changes should _also_ mean that the drivers now
work on Linux/alpha. If there are people who have any of the SCSI cards
listed above, I'd be very interested in hearing about it (and I'm sure the
people keeping up the alpha FAQ would also want to know about new supported
hardware). Maybe someone could try out a card for quick testing?

Linus
