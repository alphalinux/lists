Sorry about the short first reply; I don't do Eudora often enough... :-(

Combinations of kernel/Xserver that will run XQuake:

1. Stock LINUX kernels (*NOT* with the latest axp-diffs) and ECOFF Xservers.

2. LINUX kernels with the latest axp-diffs and ELF Xservers.

The GLIBC used by the ELF Xservers has a mismatched data structure with
the stock LINUX kernels, in the area of shared memory, as used in the
MIT_SHM extension to X.

Note that the Red Hat 4.0 kernels do (I think) have axp-diffs added to the
/usr/src/linux tree (among other things), but it *may* be earlier than when
the appropriate patch appeared from David M-T. Please correct me if I'm wrong.
