/test-images and not /images.

/images are for fully tested (cross my heart) and _stable_ MILOs not
neccessarily built against the latest kernel.

/test-images contain the latest images, some tested, some not.

In a little while (when I've tested the Adaptec 2940 UW in Milo) I'll
release all new /test images which will be built against 2.0.22 and
will be ELF/COFF and _all_ tested. From then on /test-images will
contain only ELF-built 2.1.x MILOs.
