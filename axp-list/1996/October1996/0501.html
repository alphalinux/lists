> Other problems: for some reason, my hostname is not recognized. I can't
> ping it, but yet it is set in the environment variable. All /etc network
> files are set per the ISP and PPP HOWTO's (identical to my RH 3.0.3 setup,
> which worked without a hitch). Am I missing something??

If you are having trouble with networking, the first place to check in
ifconfig. If ifconfig reports that the ethernet interface is properly
set-up, then check the routes with route -n. If route reports that
everything is fine then you either have a bogus firewall, or the
physical layer is not connected. Usually, this kind of thing is easy
to track when you check those three things: physical, ifconfig, route.
