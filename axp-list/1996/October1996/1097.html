>>> David Richards said:
> Redhat V 3.03 for Dec Alpha
>
> Configuring and building a new kernel was easy (with xconfig), but when I
> reboot with the new kernel I get the followig error:-
>
>
> scsi-ncr53c7,8xx : at PCI bus 0, device 6, function 0
> scsi-ncr53c7,8xx : NCR code relocated to 0x40054694 (virt 0xfffffc0000054
> 694)
> scsi0 : burst length 8
> scsi0 : test 1 started
> scsi0 : driver test 1 timed out due to lost interrupt

Is it possible that you took the default IRQ for the MSS sound card, which
might have been 11? If so, this could be the problem.

Note that the MSS sound card in the UDB must be configured as:

IO base 530
IRQ 9
DMA 3
