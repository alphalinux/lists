I've had problems with printing too come to think of it.

I can do "cat blahfile > /dev/lp0" and it prints

When I set up a printer for /dev/lp0 using the "printtool"
its NO GO!

Also irritating is the way you cant specify a filter for a network printer -
only a local printer. One thing about slackware was at least I could put in
a filter to get the damn HP printers to stop doing the "staircase" routine -
I've had no such luck with redhat. I've put obvious things in the filter
like "echo HEY THE FILTER IS RUNNING > /dev/console" and verified that for
some reason the filter is not being executed.

While I'm complaining - here are some more gripes:

- snmp tools are broked on the AXP distribution. They complain about
"unable to form packet" or something. Works fine on the intel version.

- If you partitioned your system with /usr and others rather than just one /
partition - it complains on shutdown about "unable to unmount /usr is busy".
I worked around this by adding a command to remount /usr read only in the
/etc/rc.d/rc0.d/S00halt file

- The way the RedHat/sets and RedHat/by-group directories are set up on the
FTP site is brain-dead. It takes a long time to retrieve 1000 symbolic
links, each one a seperate data connection. Especially when sets.tar.gz and
by-group.tar.gz compressed down to about 20kB after I had them downloaded.
