That it, there are bug, not that there is some sort of jalopy involved.

This release won't install from an arbitrary directory on a mounted
hard drive. I put the AXP install files on a hard drive because I
don't have a CD rom that I can move. I selected the 'install from
hard drive partition' option, but it wouldn't allow me to select which
directory on that partition, so I had to do an NFS install.

It was confused about the partitioning I had done for rh 3.0.3. I
made three partitions for the last release: a DOS one for MILO, a swap
partition, and a linux ext2 partition. It ignored the DOS partition
when counting sda's and tried to install to the swap partition which
was only 32MB. Fine. I repartitioned, but I don't know where I'm
gonna put MILO.

It did install the second time, but the machine hangs after the kernel
configures the network adaptor. I can ping the box, but the console
does not start a getty, nor can I log in via the network. Because RH
doesn't ship a rescue disk (ala Slackware) the machine is dead where
it lays.

It took me nearly a week to install 3.0.3 and I had hoped that this
version would be easier. Am I alone in having so much trouble getting
v4.0 to install?
