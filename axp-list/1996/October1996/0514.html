On Tue, 15 Oct 1996, J. Zimmitti wrote:
>
> Now I've seen everything. I managed to track down the older Quake 1.01
> files and lha'ed them. I also grabbed both versions of XQuake (orig and ELF).
> Both core dump upon boot right after attempting to load the Quake console.
> I've never had so many core dump problems with RH 3.0.3 as I am now. In fact,
> I can only recall two dumps ever in 3.0.3 and Quake ran beautifully. Is
> anyone running Quake w/ RH 4.0 successfully??

Oh, there is a bug in the quake hostname stuff for networking: it uses
"gethostbyname()" on the local hostname, and never checks the return value
for NULL.

So if gethostbyname() fails, you're dead.

> Other problems: for some reason, my hostname is not recognized. I can't
> ping it, but yet it is set in the environment variable. All /etc network
> files are set per the ISP and PPP HOWTO's (identical to my RH 3.0.3 setup,
> which worked without a hitch). Am I missing something??

Ok, if you fix this problem, your quake will probably also magically start
working.

The problem is _probably_ just one of the host configuration files: a missing
/etc/nsswitch.conf or a missing entry in /etc/hosts. I have a pre-RedHat 4.0
install (not the final one), and it works for me, but I saw the same symptoms
when my name server had gone away (it didn't look things up in the hosts
file, so if the nameserver was down nothing worked).
