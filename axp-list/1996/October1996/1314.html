> Am I right in assuming that I only need these if I want to run ELF
> applications? Or are they for compiling ELF executables? Or are they
> something to do with glibc?

The axp-diffs does: adds kernel-gdb option, adds the needed kernel-
headers for compiling against glibc, adds support for "long" filesystem,
some small bug-fixes usually exists there also until Linus integrates
them into the main kernel.

David-MT, if you read this, could you please try to convince linus
to alteast integrate the glibc patches asap in the 2.1 tree? Its a
hell to use the 2.1.1 axp-diff to patch against say like 2.1.5 kernel
source, tons of rejects that is. If he wants to remain compatible
with linux/libc-ecoff I wouldn't mind some ifdef's in there..

> My system seems to be OK running 2.0.22 (standard kernel as
> distributed by Linus); in other respects, it's a standard 3.0.3
> distribution.

Then you have no need (if you don't want to migrate to glibc/elf)
to use axp-diffs if you don't want to use the above 'features'.
