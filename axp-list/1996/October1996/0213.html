> Right now I have the ARC firmware passing the kernel boot command to
> MILO so that I can just sit back and have Linux boot automagically (I
> have a UDB). The problem is, there is a count down from 10 seconds
> before control is passed to MILO. Is there an option in WNT ARC to
> speed up the delay. I thinks it's a good idea to have it, but not 10
> seconds...5 seconds would be more than enough time to abort the normal
> boot sequence.
>
> Any clues on what, if anything, I can do to speed this up?
In the ARC console, go the the "Supplementary Menu...."
Then go to "Setup System", and "Edit Environment Variables". The "COUNTDOWN" variable is the one you want to change. I suggest that you don't make it so small that you paint yourself in a corner.
