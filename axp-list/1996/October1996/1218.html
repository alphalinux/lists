I'm having problem with my 8X external SCSI-II CD-ROM on the 166Mhz
UDB. It works without a hitch about 50% of the time. However, the
other 50% of the time, it won't allow me to mount it (ie. CD-ROM I/O
ERROR, bad superblock or some-such).

I'm beginning to suspect I have an old kernel (it's 1.3.90). It's not
the hardware, as I've already brought one of these back and still have
the same problem.

I tried rebuilding 1.3.90 (I'm using RH 3.0.3) but quite frankly, the
config and build documentation *SUCKS*!!!!!! I tried:

make zImage

No dice, so I looked in the makefile and used vmlinux as the target.

Then I got compile errors in the /drivers/blocks/ area and it gave
me some message about set_vesa_blanking.

Mailing to Red Hat for support is about as useful as sending mail
to /dev/null.

Can someone let me know what the latest ECOFF kernel sources are that
are stable??? Also, can someone provide a nice, easy to follow, idiot-
proof cookbook method of building an ECOFF kernel on the UDB???

A UDB web site is sorely needed...if I had more info I'd do it myself.
