j brasher <brasherj@zoology.washington.edu> wrote:
> has anyone else had trouble constructing a root floppy from the
> ramdisk.img file in the redhat-4.0 axp distribution? the 'file'
> command reports it as being 'gzip compressed data, deflated, ...',
> and a floppy built from the thing won't fsck worth a hoot.

The root floppy is compressed, as file reports. The kernel decompresses
it to load the ramdisk. This is a new feature since late in the 1.3.xx
kernel development.
