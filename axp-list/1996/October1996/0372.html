I'm seeing a few quirks in RH Alpha 4.0. This is upgraded from a CD 3.0.3
install. I downloaded the 4.0 ~/RedHat tree and have it installed on a
local disk to upgrade. This is against linux-2.1.3.

The vixie-cron*.rpm and zlib*.rpm files in axp/RPMS appear to be bad.
'rpm -e' indicates that these are not installed. Then 'rpm -i' indicates
that it is already installed (which it is not). 'rpm -U --force' then
gives the message "unpacking of archive failed". This is with rpm 2.2.5.

Any thoughts on the the bad .rpm files? I got these from a mirror and
they were bad there also (expected).

A few include file warnings:

/usr/include/signum.h calls for asm/signum.h, with does not exist.
Touching asm/signum.h is enough to get things to proceed.

There are some redefinition warnings between sigaction.h and asm/signal.h
over some SA_ definitions.

signal.h and asm/signal.h both define sigset_t and __sighandler_t.

linux/time.h and sys/select.h both define a few FD_* preprocessor symbols.
