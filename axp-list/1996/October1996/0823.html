Hi,

I have some problem with my previous configuration of
/etc/syslog.conf: it isn't working any more.

The problem could be tracked down to this line:

# See something hapening on the xconsole
# The named pipe /dev/xconsole is for the `xconsole' utility. To use it,
# you must invoke `xconsole' with the `-file' option:
auth.*;daemon.*;mail.*;*.=debug;*.=info;*.=notice;cron.none |/dev/xconsole

As you can see, "/dev/xconsole" is a named pipe. This configuration
works without any problem on my Intel box and it did also with Red Hat
3.0.3axp.

After the upgrade (worked like a charm with the new ramdisk, thanks
Eric) I couldn't log into the system and the only way to get access to
my system was the single user mode. After disabling the above setting
in /etc/syslog.conf everything works again, but now I don't see
anything happening to the system on the xconsole.

Once I had some problems with this setting when my /etc/hosts-file was
corrupted, but this isn't the case this time.

Any hints?
Thanks, Markus
