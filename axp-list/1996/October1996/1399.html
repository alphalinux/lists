>>> Bill Scanga said:
> I have the AlphaStation 600 5/266 described at:
> http://www.workstation.digital.com/products/a600.html
>
> This machine is currently running Digital Unix. Of course, I'd prefer to
> run Redhat Linux :-) I tried using the same boot floppy which I use to
> successfully boot the AS 200 4/166, but in the 600 it only gets as far as:
>
> >>> boot dva0
> ( boot dva 0.0.0.0.1 -flags a)
> block 0 of dva0.0.0.0.1 is a valid boot block
> reading 717 blocks from dva0.0.0.0.1
> bootstrap code read in
> base = 138000, image_start=0, image_bytes=59a00
> initializing HWRPB at 2000
> initializing page table at 12a000
> initializing machine state
> setting affinity to the primary CPU
> juming to bootstrap code.
>
> and then it hangs. Should I be using a different MILO image? -Or else is
> it possible to boot Linux directly from this AlphaStation's SRM?

That's exactly what you *do* want to do, boot via SRM, bypassing MILO, as
we (currently) have no MILO for the ALCOR (AS500/600 family).

Read the SRM Firmware HOWTO at:

http://www.azstarnet.com/~axplinux/srm.html

Good luck!
