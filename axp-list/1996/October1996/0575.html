I cannot speak for the earlier alpha CPUs, which might have very different
sensitivities to compiler optimization, but I have done some benchmarking
on my 300MHz EB164 that shows some very interesting results.

I have a set of benchmarks that use FFTs to test the speed of floating-point
processing. They are written in fortran and I have run them on several other
computer for which I have quoted numbers for SPECfp92. Here are comparisons
of some typical results:

system SPECfp92 my benchmark
------ -------- ------------
200MHz PPro 255 13.9
125MHz HP-735 203.8 12.7
99MHz HP-735 167.9 10.4
200MHz SGI Indigo 2 131 8.93
80MHz HP-712/80 121.8 7.97

A regression curve based on the HP workstations gives

SPECfp92 = 15.8 x (my benchmark).

My Alpha is said to have a SPECfp92 rating of 503, which would suggest that
it should achieve about 32 on my benchmark, If the HP scaling law holds.

I was able to get the manufacturer to run my benchmark prior to purchasing
the unit, but only on a similar system running DEC UNIX and DEC fortran. The
result of that test was 24.2, low but respectably close to the HP curve.

When I run my benchmark on my Alpha running Linux and using f2c/gcc the result
is 8.2 (only 34% of the result obtained with DEC software), very dissapointing.

I conclude that, at least for the 21164, compiler optimization is critical to
floating-point performance.
