On 25 Oct 96 at 12:36, Bryan Thompson wrote:

> Hello,
>
> I am looking at buying an Alpha UDB, but I am concerned about the performance of C/C++ under Linux. The benchmarks that I have seen (DGC, Inc.):
>
> > 166Mhz running NT equal to a P-75, running 64-bit Linux equal to a P-100(maybe)
> > 233Mhz running NT equal to a P-133, running 64-bit Linux equal to a P-150
>
> suggest that all is not well. Can you enlighten me concerning gcc for Alpha? I would be running principally custom built neural-network simulators in C and C++. The code is floating-point intensi
>
> Thanks,

Well, when I looked at my UDB (166Mhz) I was a little dissapointed
that the SPECInt92 was similar to a P-75, and SPECInt was similart to
a P-100. But keep in mind that this is the LOW END of the Alpha
series... This probably doesn't relate much at all to the C/C++
compiler optimizations since SPEC's are base on CPU performance. And
I've seen/heard rumblings about using glibc for better FP performance
for things like sqrt(), etc..

AND, if the ads are right, the 500MHz Alpha is 20X faster than the
166MHz UDB. All I can say is *W*O*W*
