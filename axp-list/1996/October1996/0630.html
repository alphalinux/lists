On Thu, 17 Oct 1996, Linus Torvalds wrote:
>
> Note that we can indeed do disassemblies: it doesn't even need to be fast. It
> can be slow as molasses, and nobody would care, because the exceptions never
> happen for good code. So we can have an _extremely_ stupid disassembly
> function that takes ages to do it.

Uhh, I actually forgot the _real_ problem with disassemblies, which is to
make sure it is a _valid_ exception. The kernel exception code needs to know
some way whether a bad kernel memory access was due to a kernel bug, or due
to a bad user pointer to a system call. That's why we have the current
"ex->count" stuff - to limit the area in which exceptions can be accepted
(anywhere else the kernel will do a panic when it encounters a wild pointer
access).

Other than that problem disassembling is quite reasonable (although I'd still
not want to do this on x86 for the first few versions of the exception code
due to the strange instruction length calculations ;)
