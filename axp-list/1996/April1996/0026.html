On Tue, 2 Apr 1996, Martin Ostermann wrote:

> * `Top' shows (almost) all processes with a process priority
> of 15 instead of zero. After playing with nice and renice, I
> assume that the function, which determines the current
> process priority, returns the wrong value.

Yep :-( The process priority numbers were changing for a while. I'll try
and get this right in the next release.

> * Multi-volume tar's don't work. `tar xvMf /dev/fd0' hangs
> after reading for only some seconds. Probably it only reads
> one track.

Too bad :-( I'll try and look at it.

> * `Clock' doesn't work. It hangs a while, then returns "Jan 1,
> 1970."If `clock' is not called after booting, the system
> time is off by +20 Years and 1 Hour relative to the time
> established by Windows NT. (timezone is GMT+1), Easily fixed
> with `date -s "-20 years -1 hour"', but how do I change the
> real time clock?

Did you set CLOCKMODE="ARC" in /etc/sysconfig/clock? On an ARC booted
machine, you should always run clock with "-A".

> * C-c never works in `xterm', and C-z only sometimes. Both
> signals work as usual in `rxvt'. I read the man page of
> xterm, but found nothing useful. Am I missing something?

Type "stty sane" in the xterm. Know, I don't know why, but this has
been a bug forever.

> * It should be noted in the aboot readme, that aboot can not
> handle partitions greater than 1MB. When trying Redhat
> first, I only used 1 big partition to keep things simple --
> that was a mistake.

Really? I didn't know that myself.
