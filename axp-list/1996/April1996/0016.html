All,
I have updated the Miniloader release area on gatekeeper.
More than that, I've had a big tidy up ready for Red Hat's upcoming
release. Here are the high points:

(1) There are no images in /test-images (there are no test images),
(2) There are all new images in /images, these have all been tested
by me except for the Platform2000 images and I'm pretty confident that
that should work. The latest linload.exe (1.3) is required if
you have more than 128Mbytes in your system.
(3) There is only one milo image for all circumstances (called "milo").
This will boot via the WNT ARC, the SRM console and via Debug Monitor
and it uses either Digital's BIOS emulation or TGA or serial port.
(4) The Milo HOWTO is up to date bar the section on booting from
SRM. ("A thumping good read - The Times").
(5) The irritating bugs with the "show" command have been fixed.
IDE disks are now shown properly as are second, third and so on SCSI
disks.
(6) Rebooting now works correctly (where correctly is defined to mean
"does what I think that it should do"). So, "shutdown -r now" will
reboot exactly the last way that you booted.
(7) MSDOS support is complete. Incidentally, I installed Red Hat
yesterday onto a new system and used the same MSDOS floppy to hold
linload, milo and the kernel and the Red Hat installation was happy
with it,
(8) The XL image works on both variants of the 21064 based XL systems
(Avanti and XL).
(9) The EB164 image now sets up the system number in the HWRPB.
(10) The sources build against pretty much all of the kernel sources
from 1.3.57 to 1.3.81 (and beyond I hope).
