I sent a previous message regarding ppp not working with chap , but I just
solved the problem . My problem now is the following : I am trying to
automate pppd to connect whenever the connection fails , so I added a line in
the inittab file, and a script that calls pppd , with chat option for a file
that has all the commands for init. the modem and calling the number . All
that works fine , and as soon as it fails , it reconnects again . The problem
is when the server is shutdown , and started up again, then the script
doesn't work as it cannot init the modem . So the problem is in chat . the
onlyl way around it is to manually comment the ppp line in inittab, run
minicom , then exit from minicom ,uncomment the ppp line again , and every
thing works fine again. I tried to copy all the init strings in minicom
config options , but still it doesn't work for the first time after the modem
or the server shutdown . An example of the ppp script and the chat script
follows :

/usr/lib/ppp/pppd connect "/usr/lib/ppp/chat -v -t 60 -f /usr/lib/ppp/mychat"
/d ev/cua2 115200 -detach crtscts modem defaultroute debug

~^MATZ^M~ OK ATQ0E1 OK ATDT5884410 CONNECT
