I'm not familiar with the locatime() implementation in the old libc,
but the GNU libc had a function that used an iterative algorithm which
took forever when passed 64 bit values (values that don't fit into 32
bits). That's fixed now in glibc. I seem to remember Bob Manson
mentioning problems similar to what I saw in glibc when he originally
started the libc work. So it wouldn't surprise me if the old libc
uses the same broken algorithm. In any case, if localtime() takes 28
seconds on a 32 bit value, there definitively is something broken (in
case that wasn't clear... :-).
