Greetings,
I recently purchased a noname alpha with 233MHZ processer. I
installed MILO with no problems, but when I boot Linux, I get this message
about my hard drive.

Partition check:
hda: Unaligned trap at fffffc00002edf4: fffffc00002c5ca 28 1
Unaligned trap at fffffc00002edfc: fffffc00002c5c6 28 18
Unaligned trap at fffffc00002eel0: fffffc00002c5ca 28 19
hda1Unaligned trap at fffffc00002edf4: fffffc00002c5da 28 1
hda2, hda3, hda4

My hard drive is an IDE Maxtor 71626A, 1554MB, 64KB cache, LBA,
CHS=3158/16/63
I have used it for sometime in my intel 486 linux box and never had any
problems.

I was still able to install Linux, but after it's running (and while it's
loading), if the OS to seeks the hard drive, it will thrashe for a couple
seconds and print something like this:
<sc 256(2d,11ffffec0,100)><sc 257(e,11ffffec0,101)>

Everything appears to work, though I can't get anything accomplished with
this stuff popping up all the time.

I tried another Maxtor hard drive, this time a 540MB, and got the same
results.
I have reformated, and repartitioned both drives with no results.

Has anyone seen this before? Is this a problem with Maxtor's and Alpha?
I would be very grateful if someone could clear this up.

I am using MILO version 1.3.34
Linux kernel version 1.3.43
