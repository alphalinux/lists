I'm trying to install Red Hat AXP 2.1 from a CD-ROM on my Multia (which are now
supposed to be called UDB, but mine says Multia right on the front and runs at
233 MHz, so it can't really be a UDB, can it?).

Anyhow, since no one was ever able to tell me how to disable the internal SCSI
termination (on the PCI riser card), I can't get my external CD-ROM to work
reliably. The 220/330 ohm surface mount resistor pack for the termination is
very obviously present on the card, but there is no jumper in evidence.
Someone claimed that the Multia had automatic termination, but I don't believe
it. With any SCSI device hooked up via a quality cable (that works on my
Pentium box), the Multia takes a *loooong* time to boot, and the external
device isn't accessible by the firmware, Milo, or Windows NT. I'm tempted to
forcibly remove the resistor pack from the board. I don't understand why
there is an external SCSI connector at all, if they couldn't be bothered to
provide a way to turn off the termination.

So my next attempt was to plug a Lomas Data Products PCI SCSI host adapter
based on the NCR/Symbios 53C825 chip into the Multia's PCI slot, and run the
CD-ROM drive from it.

The Multia ARC console firmware initializes the '825 just fine, finds the
CD-ROM drive, and NT accesses it with no problem. So obviously my SCSI
cable, CD-ROM drive, and external terminator all work fine.

The version of Milo for the UDB which came with Red Hat doesn't like it
though. It recognizes the '825 just fine, but when it tries to test it,
it complains about lost interrupts. As a result, the "show" command
claims that my only disk device is fd0.

So I tried a newer Milo. I got the latest Milo image from gatekeeper,
version 1.3.76. It doesn't even get as far as the old version did! It
hangs right after the "NCR code relocated" message for scsi1, and never
outputs the "test 1 started" message for scsi1.

Has anyone else been able to install from a CD-ROM on a Multia or UDB?

Eric
