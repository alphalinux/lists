I picked up the clinpack stuff from the ftp site cosc1 mentioned in the
previous post and ran it on the 21164 with the DEC C compiler.

The best set of compiler flags I could find (-O4 -float -tune ev5 -assume
noaccuracy_sensitive -ifo -migrate -DFAST_MATH -fp_reorder) give results
showing speeds of 40MFLOPS/46MFLOPS for the two matrix alignments from the
clinpack (odd or even doubleword) with unrolled loops (-DUNROLL). The numbers
for loops without unrolling are 37MFLOPS/38MFLOPS.

The same routine in Fortran (Dongarra's standard fortran linpack) using OSF f77
with the best set of compiler flags gives 99MFLOPS/102MFLOPS.

These results were done on a 300mhz 21164 running DEC OSF1 v3.2 (I will be
putting Linux on this box soon...).

Note that the fortran version linpack speeds are way up there compared to C,
more than double, not just a 10% difference...

Note that I did not change the C linpack routine to reorder the arrays, the
code treats all arrays passed in to the matrix functions as pointers with
offsets used to access the matrix elements. Does anyone here know the details
of declaring a dynamically dimensioned array in C? This might allow the
compiler to speed up the loops....
