I'm trying to install Linux (eventually Red Hat 2.1) for the first time on a
DECpc 150 using the instructions from
ftp.digital.com/pub/DEC/Linux-Alpha/JENSENINSTALL. I don't have the resources
to try the work-around by Michael Schwingen and the aboot loader (namely a
Linux/x86 System with SCSI controller). My system has a new Seagate 4.2GB drive
which I'm using solely for Linux and I followed the instructions in the
JENSENINSTALL-Howto:

Formatted the drive into 3 partitions with minlabel.exe: 1st partition: 4MB boot
fs; 3rd partition: 100MB swap fs; 2nd partition: everything else in a ex2 fs.

Used copyboot.exe to copy the bootdisk (from same ftp site) to the first
partition.

Switched to SRM console and typed (as according to instructions):
BOOT -FL INSTALL DKA200

I got the message:
INIT-S-CPU...
AUDIT-BOOT-STARTS...
AUDIT_CHECKSUM_GOOD
AUDIT_LOAD_BEGINS
AUDIT_LOAD_DONE

Linux/AXP bootloader for Linux 1.3.72
Switching to OSF PAL-code .. Ok (rev20123)
Loading vmlinux ... Ok
Now booting the kernel

And then the system sits forever. It is supposed to (at some point) prompt for
the ramdisk but it just stops after the above with a blinking cursor. I had
heard that this may take awhile but after 6 hours, my patience ran thin and I
shut it down and booted NT to download another bootdisk image and try it again.
It is so far showing the same results.

Any idea how to fix this? Is there another way to setup Linux or has anyone
succeded with this?

Any help appreciated,
