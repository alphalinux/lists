Jumping through hoops here with Linux installation. I have a
AlphaPC64 motherboard, SCSI HD and CD-ROM, ISA NE200 clone (cheesy,
yes, but it was cheap, and it worked in a x86 Linux box). Initial
software install was BLADE 0.3, just because I had it handy. I want
to install RedHat, but have been having some difficulties. The
kernel I have (the newest one I could find on the net) doesn't
support networking, and I would rather get RedHat by FTP if I can.
So I got the newest kernel sources I could (1.3.81 at the time) and
tried to compile a new kernel. No go. It got to the final link and
complained of undefined symbols (memcpy and memset) in
arch/alpha/kernel.o. I couldn't figure out if it was a compiler
problem (gcc 2.7.0) or a kernel source problem (although I suspect
not, as I tried 3 different revisions and had the same error on each,
while no one on the list seems to have experienced any of these
difficulties). So I borrowed a friend's SCSI tape drive and data
line and used his machine to drop RedHat to tape. _Then_ I discover
that my kernel doesn't support SCSI tape drives either. So now I'm
stuck. I was hoping that someone out there could either point me to
a pre-compiled kernel or be kind enough to compile one for me with
the following features:

AlphaPC64 motherboard
NCR SCSI controller
SCSI HD, CD-ROM, and tape support
networking support w/ ne2000 ethernet card

Any help is greatly appreciated.
