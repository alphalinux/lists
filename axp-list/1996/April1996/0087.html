To test glibc and the c++ library I compiled the vrweb VRML browser.
It turns out that I was really testing g++ (it doesn't compile some of
the files right with -O2). But after compiling the user-interface
with -g only, vrweb seems to work fine and speed is not too crappy
either (since the rendering engine still gets compiled with -O2).

And the demo factor of vrweb is *definitively* close to the max (right
up there with the non-existing netscape... :-)

If anybody wants it, I can bundle the binary with a few sample files
and make it available on azstarnet.

Oh, and since you might be wondering: yes, I think glibc etc. is
*quite* close to a state where a test release can be made. But there
are still a few things missing. I hope I'll have a little time this
weekend to put in missing parts, but I can't really promise when it
will become available.
