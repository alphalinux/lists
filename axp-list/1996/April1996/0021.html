> * `Clock' doesn't work. It hangs a while, then returns "Jan 1,
> 1970."If `clock' is not called after booting, the system
> time is off by +20 Years and 1 Hour relative to the time
> established by Windows NT. (timezone is GMT+1), Easily fixed
> with `date -s "-20 years -1 hour"', but how do I change the
> real time clock?

If booting MILO from ARC console, set the date correctly using ARC, then
make sure that the "clock" command is called with the "-A" option. That is,
when setting the system clock from the HW clock, do "clock -s -A". This
adjusts for ARC console format hardware clock settings. Under RedHat install,
there's /etc/sysconfig/clock which should contain "CLOCKMODE=ARC" if you
are booting from the ARC console. This file is included by one of the
/etc/rc.d scripts that sets the system clock.
