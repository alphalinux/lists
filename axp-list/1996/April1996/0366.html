> Hi,
> I am trying to compile the program Internet Movie Database v3.2d and I get:
> gcc: Internal compiler error: program cc1 got fatal signal 6
>
> What is signal 6? I read the FAQ (03/11/95) and there is signal 4 and 11 but
> not 6. If I remove the "-O2", I am able to compile. I have kernel 1.3.95 and
> running Red Hat 2.1/AXP.

I ran into this porting the XFree86 servers a few times, so there's something
you might try.

I found that short<->{float,double} conversions were the problem in a number
of instances; try looking for anything like that, and force a short<->int
conversion *FIRST*, then the int<->{float,double} should work OK.

This was with an earlier version of GCC, but I think it's the one shipped
with RH 2.1/AXP. I don't know if it has been fixed in the latest GCC...???
