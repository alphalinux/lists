Hi,

just making my first steps on Linux AXP (Red Hat 2.1) on a DEC XL266
and here are a first few questions:

- is there a version of "strace" for linux-alpha or any other tool
to monitor system calls?

- does anyone have a working implementation of /usr/bin/time ?
the Red Hat binary doesn't show reasobale user/system nor elapsed time:-(
the time in tcsh works fine though.

- is there a linux tool to view/change the (ARC?) console boot loader
parameters? I'd like to specify another kernel image and/or options
(OSLOADOPTIONS parameter) but I can't the cursor keys of my keyboard
in this boot menu (they work fine in linux afterwards; this is a
pretty old DEC LK250 keyboard for PCs).

any idea or hint where I should start looking for the problems with
my cursor keys in this boot loader ? any chance

- is a version of f2c available somewhere for AXP ?

- is it possible (using software) to find out there is a 21064 or 21164 CPU
in a machine ? /proc/cpuinfo for my XL266 only says

cpu : Alpha
cpu model : Unknown
cpu variation : 0
cpu revision : 0
cpu serial number : Linux_is_Great!
system type : Mustang

where can I find the most recent version of tools like time etc.
and where is the most recent version of libc available ?

any idea when a first alpha version of shared libs might be available?

right now there is gcc-2.7.1 on this machine.
is it wise to get gcc-2.7.2 or any other alpha test version ?

are there known problems with networking in genel or with the DE434/5 card
(DEC DC21040 rev 34 chip according to /proc/pci) ?

I get lots of "timeouts" (waiting several 0.1-5 seconds) and complete lockups
with telnet and ftp to a linux box using either TP (null-modem-TP) or thinwire
with both 1.3.74 and 1.3.82.

AXP network adapter initialisation:
eth0: DE434/5 at 10000 (PCI bus 0, device 12), h/w address 08:00:2b:e2:1c:14,
and requires IRQ9 (provided by PCI BIOS).
de4x5.c:v0.41 96/3/21 davies@wanton.lkg.dec.com

i486-Linux network adaptor:
lance.c: PCI bios is present, checking for devices...
eth0: PCnet/ISA 79C960 at 0x300, 02 07 01 16 f9 a3, probed IRQ 9, DMA 5.
lance.c:v1.08.02 Mar 17 1996 tsbogend@bigbug.franken.de

but this might be a problem with the lance network adapter in the 486 box too
since I never used this card in my PC before (but tested it in our institute).

Thanks,
