I've had no problems with the 810 based SCSI and synchronous operation
at 10 MB/sec and the disks. Disconnect must be off, or all hell can
break loose. I managed to hose a sector or two on my 210 MB disk on which
NT now sits somehow when I was testing with disconnect turned on.
But I'm not sure why, really. I suspect a reformat would fix the sectors,
but I haven't bothered yet.

However, I was unable to get my Archive Python DDS-2 DAT drive to function
reliably unless disconnect was turned on. So, if I have to use the DAT,
I boot single user and only access a single disk. :-(
