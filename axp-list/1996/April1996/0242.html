One thing I've seen from personal experience with pop servers is that sometimes
file streams aren't flushed very well... When a user has a mailbox open the pop
server usually copies it to a tmp file, And a problem comes up when the user
gets new mail during the time the mailbox is opened (almost always caused by
pop clients that use the leave on server commands and thus leaves the mailbox
opened for extended periods of time) and the pop server goes to merge the now
two mailboxes together...

For debugging fun try attaching a debugger to a pop server process while this
is happening and watch the streams when it tries to merge new mail with an
exhisting mailbox...
