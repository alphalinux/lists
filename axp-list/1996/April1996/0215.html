Eric,

>The problem is that the extra terminator that shouldn't be present is the
>one built into the PCI riser card, and there is no obvious way to disable
>it, short of desoldering the resistor pack. I've heard numerous claims
>(but not from Digital) that it is automatic. If someone would care to
>explain how a resistor pack can be automatic, maybe I'd be more inclined
>to believe it.

OK, I got curious about this, having been inside a UDB a number of times,
I've even got a few extra risers with various configurations (more later).
I've *many* times used external CDROM and disk off the external SCSI port,
and *never* experienced a problem. Can't tell you how many test RH installs
I've done on UDB's with external CDROM... :-)

Anyway, I popped open one box: no terminators on the riser. I looked at a few
extra risers: no termination. My belief is that termination is normally done
*not* with resistors on the riser, but with the external SCSI connector itself;
ie it terminates if nothing is plugged into it, and doesn't if something *is*.
Isn't this typical of this type of setup nowadays?

Now, Eric, I believe you must have some strange variation of riser card; I've
seen a number of different ones, and none have had termination like you
describe. I think you/we ought to see about getting a replacement for it.
It almost sounds like a proto, as I'm sure that production models wouldn't
be set up this way.

I've seen a number of these risers, some without the PCI bus slot on them, so
that a 3.5 inch disk would fit inside (I've got a 233MHz model, 1MB cache,
with a SCSI Fireball inside); that one goes along with a small plate that
gets suspended inside and holds the drive. There are lots of little widgets,
ie risers with variations, like one with no PCI slot, only external SCSI and
internal power feed, so that one could hook up an internal IDE disk. I don't
even know if all these widgets even have separate part numbers, maybe someone
else knows.

Anyway, Eric, I think you've got a dud riser; lets see what we can do about it.

--Jay++
