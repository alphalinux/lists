Erik> On Tue, 2 Apr 1996, Martin Ostermann wrote:

>> * It should be noted in the aboot readme, that aboot can not
>> handle partitions greater than 1MB. When trying Redhat first, I
>> only used 1 big partition to keep things simple -- that was a
>> mistake.

Erik> Really? I didn't know that myself.

Just for the record: aboot itself has no such limit. It turns out
that the ext2 code used by aboot currently has a hardcoded limit of
1GB (like one *giga*byte, not megabyte). The corresponding MILO
sources now have a limit of 2GB so making aboot do the same is trivial
(just up a constant in the sources). The real solution is probably to
make it a little more dynamic so there isn't a hardcoded limit.

Regarding aboot and FAT: the filesystem interface in aboot is almost
identical to that of MILO, so merging the MILO FAT sources into aboot
should be straight forward. Maybe it would even be possible to make
MILO and aboot share the same filesystem support sources.
Unfortunately, I won't be able to do this any time soon. But I'm
always glad to accept patches...
