>>>>> On Wed, 3 Apr 1996 23:28:27 +0200 (MET DST), koenig@tat.physik.uni-tuebingen.de (Harald Koenig) said:

Harald> - is it possible (using software) to find out there is a
Harald> 21064 or 21164 CPU in a machine ? /proc/cpuinfo for my
Harald> XL266 only says

Harald> cpu : Alpha
Harald> cpu model : Unknown
Harald> cpu variation : 0 cpu

The kernel prints whatever it finds in the so-called "hardware restart
parameter block". That block is setup by the firmware (MILO, SRM, or
ARC) and is documented in the 2nd edition of the Alpha Architecture
reference manual. MILO does not currently initialize the "cpu model"
field. E.g., on my MILO-booted Cabrio at home I get "cpu model:
Unknown" but on the SRM-booted Cabrios at work I get "cpu model: EV45"
(which translates into 21064A). Adding the missing code to MILO would
be trivial *provided* you know how to find out what CPU you're dealing
with (short of hardcoding it). Neal, if you're listening: in the ev56
announcement I saw mention of an IMPLVERS instruction or some such
that allegedly allows to determine the CPU type (and/or its features).
Is this implemented in all CPUs and documented some place? It would
be handy, occasionally.
