> Do any of the other SCSI devices have termination built in? Most disks
> seem to have termination that can be enabled or disabled. Too much
> termination would also cause similar problems.

Yes, too many terminators is exactly what the problem is. Here's a diagram:

----------------------------------------------------|
| Multia/UDB |
| |
| ----------- --------------------- |
| | Toshiba | fine | PCI riser card | std. |
| | disk | pitch | | pitch |
| | hdr| ribbon |hdr hdr| ribbon | -------
| | --------------------+--------+------------------------ |term.|
| | | | | | | | fine| | ---|---
| | | | | | | | pitch| |delta |
| | ------- | | -------- ------- | ext| |ribbon |
| | |term.| | | |53C810| |term.| | conn| |conn. |
| | ------- | | ---|---- ------- | | ---|-------|----
| | | | | | | | | std. | |
| ----------- | ---|-------- | | | |pitch | |
| | | PCI slot | | | | |ribbon | |
| | ---|-------- | | | ----+---- |
| | | | | | | hdr |
| -----|--------------- | | ----|----- |
| | | | | CD-ROM | |
| --------------- | | |no term.| |
| | motherboard | | | ---------- |
| --------------- | | |
| | | external box |
----------------------------------------------------- ----------------

The problem is that the extra terminator that shouldn't be present is the
one built into the PCI riser card, and there is no obvious way to disable
it, short of desoldering the resistor pack. I've heard numerous claims
(but not from Digital) that it is automatic. If someone would care to
explain how a resistor pack can be automatic, maybe I'd be more inclined
to believe it.

Note that I am only guessing that the Toshiba disk is terminated. There
are no obvious jumpers on the drive, even for the SCSI ID.

If the Toshiba wasn't terminated, then there would only be one terminator on
the bus when the external connector wasn't used. This would cause problems.
And even when the external connector was in use, the Toshiba drive and the
fine pitch ribbon would constitute a long stub, which is also a violation of
SCSI specs, and would cause problems.

My conclusion, having failed thus far to obtain any response from Digital, is
that the design is brain-damaged and can't work reliably with external
devices. I would really like to have an engineer from Digital explain why I
am wrong, and how I can get this working without hacking up the riser and
voiding the warranty on my machine.

(This is why computer manufacturers are *supposed* to provide or sell
technical manuals for their products.)
