>
> Hi,
> I am trying to compile the program Internet Movie Database v3.2d and I get:
> gcc: Internal compiler error: program cc1 got fatal signal 6
>
> What is signal 6? I read the FAQ (03/11/95) and there is signal 4 and 11 but
> not 6. If I remove the "-O2", I am able to compile. I have kernel 1.3.95 and
> running Red Hat 2.1/AXP.

This means you have a bug in gcc.

Signal 6 is "SIGABRT". It is what gcc uses when it panics. This often
occurs when there is a bug in the GCC machine description file such that
it generates an insn sequence that the optimizer can't handle.

The alpha code gnerator for GCC is a tad-bit fragile, needless to say...
