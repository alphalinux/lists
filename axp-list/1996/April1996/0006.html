The unaligned trap messages are exactly that. When the DOS partitioin
checking code runs it makes unaligned data accesses that generate those
traps. The traps fix up the accesses. This is not too bad (except thqat
it fills up your log file). There are two solutions, one is to remove
the printk(), the second is to fix up the DOS partition checking code.
I know which one I prefer and I know which one Linus prefers. Unfortunately
they're not the same one.

The other messages are not good. My noname has an IDE disk that
got significantly better after 1.3.7x...
