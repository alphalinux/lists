Nils> Hi... while I compiled a new Kernel (1.3.80) on my Jensen, I
Nils> got the message, that something in memcpy and memset is wrong
Nils> (.weakext!!). So I changed the compiler to 2.7.1 and compiled
Nils> the newest binutil (2.6.0.10 ?) I found.

Nils> But now I get the message, that gas have the problems with
Nils> const > 32bit (or somthing simular), when I compile the
Nils> Kernel.

Nils> Can anybody give me a hint, what's wrong?

bintuils 2.6.0.10 is not the right thing to use. I'd suggest to get
the latest Alpha binutils RPM on Red Hat's site. Even if you don't
want to use RPM (which I think is a jump well worth taking), the
source RPM contains all the relevant patches. And "rpm --rebuild"
beats having to apply those patches manually any time of the
day... :-) The same is true for gcc.

FYI, binutils 2.6.0.10 is an interim release for the x86---it has no
Alpha related fixes. But don't worry---things *are* converging. The
next official binutils release should be much more usable under
Linux/Alpha.

--david
