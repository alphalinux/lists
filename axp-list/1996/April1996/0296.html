The code works. Speed tests indicate that it is faster for oodles
of short jobs and about the same for larger ones. Which is just
about what I'd expect: large jobs fill up the cache on their own
while small jobs left room in the cache for others.

job imb code asn code
cvs co linux 1:02 0:37.52
make dep 0:30.53 0:24.57
make vmlinux modules 6:05 6:09

All trials have the buffer cache primed with "tar cfv /dev/null .".
