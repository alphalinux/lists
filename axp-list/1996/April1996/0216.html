> I also read that even gcc and/or libc have to be built with "-mieee" too.
> is this correct? and if yes, what about the latest RPMs ? Do I have to rebuild
> them in order to use f2c ?

No, you don't have to do this.

The idea is that any code that *might* generate floating-point exceptions
must be compiled -mieee; otherwise, the kernel exception-handling code
can't deal with it (read the section in the Alpha Architecture Reference
Manual on floating-point exceptions for the gory details).

gcc won't generate fp exceptions, so you don't have to rebuild it.

Most library functions (except maybe what's in libm) won't generate fp
exceptions either, so that's not so much of a problem.

FORTRAN code in general, and the stuff put out by f2c in particular, may
generate FP exceptions, so it's a good idea to use -mieee with it. Of
course, if your FORTRAN code is all-integer, you needn't worry about
FP traps.

I'm hip-deep into the f2c and -mieee issue right now in fact... I still
seem to be getting some problems (SIGFPE if I don't use -mieee, seg-fault
if I do), and I'm tracking them down.
