Uh, maybe we ought to put an explanation of these in the FAQ... David MT?

Unaligned trap error messages are *STRICTLY* *WARNINGS*!!!!!!!

They are they kernel exception handler informing the user that certain
instructions were executed attempted to access data not on a "natural"
boundary, The execption handler subsequently does a "fixup", loading
(or storing) the appropraite data in the appropriate place.

Note that on the older Alpha CPUs, *ALL* memory accesses are done in
either 32 or 64 bit quantities, *AND* that the standard LOAD/STORE
instructions require the memory address to be on the appropriate boundary,
ie mod 4 for 32-bit, mod 8 for 64-bit.

Now, unfortunately, both the MSDOS partition table (set by fdisk) and the
MSDOS FS have "int" fields that are *not* on the appropriate boundaries.
So you'll see these messages *EVERY* time, unless you modify the kernel
to remove the "printk" that outputs them. This will *not* harm anything.

> I was still able to install Linux, but after it's running (and while it's
> loading), if the OS to seeks the hard drive, it will thrashe for a couple
> seconds and print something like this:
> <sc 256(2d,11ffffec0,100)><sc 257(e,11ffffec0,101)>

This indicates you are running a kernel older than the user utilities; the
messages mean that syscalls 256 and 257 were not handled by the kernel.
LIBC system calls depend on having a kernel that knows about those system
calls, so try to upgrade your kernel to a leter version.
