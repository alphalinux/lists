<tag/Unaligned accesses:/
If the kernel or an application performs an unaligned memory
access, the Alpha CPU traps into the kernel. The kernel then
fixes up this unaligned access, so that the faulting process
can continue execution. Thus, unaligned access are much slower than
aligned ones, but they do <em/work correctly/. To indicate that
there is a potential performance problem, the kernel prints
a warning message. Typically, such messages look like this:

<tscreen><verb>
Unaligned trap at fffffc00002edfc: fffffc00002c5c6 28 18
</verb></tscreen>

You do not need to be overly alarmed when seeing such a message.
The program will work <em/correctly/.
Eventually, all unaligned accesses will be fixed, but
in the meantime, just ignore these messages (if you're a programmer,
please take a minute and fix the source of the unaligned access
instead...).
