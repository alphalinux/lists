> I've just experimented a problem with the boot floppy during the very first
> installation of Linux/Alpha BLADE 0.3 on a XL 266.
> The 'milo' image for XL downloaded from ftp.digital.com seems to work, but wh
en
> I try to ...
>
> boot fd0:vmlinux.gz root=/dev/fd0 ramdisk=1440
>
> ... none of the boot floppies works.
> The following message is sent after a line of "###########": length-error,
> halting.
>
>

You're going to have to use an XL kernel that's on gatekeeper as well:

pub/DEC/Linux-Alpha/X11/xl-1.3.77-27mar1525.gz

The latest MILO for XL should work fine, but you need this kernel!!!

Also, note that RAMdisk activation changed around 1.3.48; you need to say:

boot fd0:vmlinux.gz load_ramdisk=1 root=/dev/fd0

now, instead of the old way...

Good luck.
