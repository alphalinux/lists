Hi,

I noticed a thread here about Fortran on Linux-Alpha.

This is a concern of mine as I use Fortran for fluid dynamics codes. Most of
these are still in Fortran. My experience is that a good f77 compiler runs
numeric codes much faster than the same code written in C. On a 300mhz 21164
running OSF with the DEC C and f77 compilers the difference is even higher, a
Fortran matrix solver I have runs at about 105 Mflops or so with best
optimization, the same thing coded in C runs at 45 Mflops with best
optimization. This expands on some of the comments that were made on relative
speed of different compiled languages. My understanding of why echoes the
earlier comments about optimizing memory fetches for Fortran, where the
structure of the arrays can be used in code optimization.

RE f77 compilers for Linux-Alpha:

I have been dealing with Microway, in Kingston, MA, on their Alpha Fortran
compiler for Linux. I bought a copy of the early version that runs under Linux
(on the Alpha) and produces about 60 Mflops with the same Fortran matrix solver
that I mentioned above. Microway was not happy with the optimizer and has been
making some changes to speed up its code. I have not tested the newest
changes. Note that this is a full-featured f77 compiler that runs native
under Linux, not free, but it works and is pretty damn fast now, maybe even
faster later. Note also that Microway has optimizing C and C++ compilers as
well, although I have no experience with them and don't know the status of any
ports to Linux-Alpha. It would be interesting to use a good optimizing
compiler (rather than the currently not-so-optimizing gcc) to compile the
kernel and system...
