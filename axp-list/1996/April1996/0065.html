The struct statfs.f_type slot is too small to hold many of
the SUPER values in linux/*_fs.h. While a change of the
size of this field would break binary compatibility, simply
changing the type to unsigned will suffice for all current
filesystem types except hpfs and xia.

r~

--- statfs.h.orig Tue Mar 26 13:47:42 1996
+++ statfs.h Wed Apr 3 19:18:07 1996

* match the beginning, at least.
*/
struct statfs {
- short f_type;
+ unsigned short f_type;
short f_flags;
int f_fsize;
int f_bsize;
