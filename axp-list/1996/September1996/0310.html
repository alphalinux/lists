is there an easier way to fix the fp problems for the two examples below
for a clean RH303 system other than installing a recent ELF gcc
(which would require new gcc, libc, ELF kernel support, milo for ELF kernel etc.) ?

the two small programs below give a SIGFPE for gcc-2.7.1 + libc-0.43 (rh303)
but work fine with recent ELF devutils.

-------------------------------------------------------------------------------
#include <stdio.h>
#include <stdlib.h>
#include <math.h>

void main()
{
char *arg = "1.e-305";

printf("%s -> %g\n", arg, atof(arg));
}
-------------------------------------------------------------------------------
#include <stdio.h>
#include <stdlib.h>
#include <math.h>

void main()
{
double x,y,z;
char *arg = "12.34";

x = atof(arg);
y = modf(x, &z);
printf("%g -> %g + %g\n", x,z,y);
}
-------------------------------------------------------------------------------
