I'm having a couple of strange problems with my UDB:

When I run fdisk on /dev/sda (my UDB has the TOSHIBA MK2326FB internal) I get
the following errors:

%fdisk

Kernel: unaligned trap at fffffc00004145b8: fffffc000004c1d6 28 3
Kernel: unaligned trap at fffffc00004145bc: fffffc000004c1da 28 2

fdisk starts however, and when I type 'p'

fdisk(31) unaligned trap at 000000012000389c: 0000000140007fe6 28 2
fdisk(31) unaligned trap at 00000001200039a8: 0000000140007fe6 28 6
fdisk(31) unaligned trap at 0000000120003a0c: 0000000140007fe6 28 6
fdisk(31) unaligned trap at 0000000120003a1c: 0000000140007fe6 28 8

I can install fine (RedHat via NFS), but when I boot vmlinux from sda1 I get

Kernel: unaligned trap at fffffc0000387b28: fffffc000005c809 29 4
Kernel: unaligned trap at fffffc0000387c50: fffffc000005c811 29 2
Kernel: unaligned trap at fffffc0000387cfc: fffffc000005c811 29 2
Kernel: unaligned trap at fffffc0000387fe4: fffffc000005c811 29 1

and finally

Kernel panic: VFS: Unable to mount root fs on 08:02.

If I boot from floppy and mount sda1 (NOT as root partitiion), fschk runs fine,
and the files/directories all seem to be ok. If I try to boot from floppy and
mount sda1 as root it fails as above.

I'm using MILO from the SRM console. MILO says linux version 2.0.1
The kernel says version 1.3.90

Any thoughts? Disk problem, SCSI problem, MILO/linux version mismatch, memory
problem?
