David Lott <dalott@diablo.cisco.com> wrote:

>You mention that you got better performance using the Digital Unix compiler.
>I can undestand that as I would assume that Digital would know it's
>processors better than GCC. Would it be possible/recommended to compile the
>Linux kernal and take advantage of this optimization for the kernal?
>
Theoretically, yes. You would need to set up a kernel build with
the relevant cross compiler, and rework all of the modules that perform
in-line assembly inside of C routines.

>You mention that the UDB was never intended for performance work. That's
>ok, cause I just want a web server to act as a virtual web host.
>...
> Do you feel that this would be an acceptable use of a UDB 266?

It depends upon the load that you throw at it; what you consider 'acceptable'
might not be 'acceptable' to others. Not having ever implemented a web
server on a UDB, I don't believe I'm qualified to address this issue.

>Would you compare Linux and Digital Unix? I'm looking for things like
>performance, ease of administration, etc.

The compilers that come with Digital UNIX are constantly being worked on,
and generate rather good code.

>If I have the GCC libs, can I use the Digital compiler "out of the box"?

The Digital 'cc' compiler comes with it's own set of libraries that are
mated to it. So, while the externally visible entry points to libc
may match between gcc and Digital's cc, they may not interoperate well.

>How is the alignment requirement handle?

There is a utility available on Digital UNIX to permit misaligned
code to function correctly. Personally, I think it is a bad idea,
as it encourages the continuation of horrible programming practices.

>Does the Digital compiler correct alignment errors or at least notify me
>during compile?

How could it do either?
char * fooptr;
int data;

data = (int) *fooptr;

Is perfectly valid, if fooptr happens to be 0 mod 4 aligned.
So the answer is 'No, not that I know of'.
