> > > Remove the securetty setting from /etc/pam.conf. If you don't have one,
> > > install the pamconfig package and then reinstall NetKit-B.
> >
> > thanks, that fixed it. what does "securetty" mean for a rsh or is this
> > a bug in /lib/security/pam_securetty.so ?
> > I've included ttyp[0-9a-f] in /etc/securetty but this didn't help.
>
> It always fails for rsh. That's kind of gross, but we decided too much security
> is better then too little. As rcp/rsh are big security issues that should
> be regulated every bit as closely as rlogin, we decided to make people think
> about what they are doing.

agreed (for rsh and security), but I couldn't find any docs for it yet.

not now that rlogin/rsh work I've two more problems:

- without having a ~root/.rhosts file rsh/rlogin succeeds for root
when I have a /etc/hosts.equiv entry for the source host which should
not be (for all other systems I know /etc/hosts.equiv doesn't work for root).

- with the following line in /etc/pam.conf

rlogin auth required /lib/security/pam_securetty.so

the first root rlogin query for a password doesn't work; login name and
password have to be entred again (no problem for non-root users).
without that line, entring the root password works for the first time.
