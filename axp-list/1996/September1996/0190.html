Sounds like a DNS problem. What's your /etc/resolv.conf and
/etc/nsswitch.conf say? I think glibc defaults to using DNS only if
/etc/nsswitch.conf isn't present. If so, try the file below (glibc
uses /etc/nsswitch.conf instead of /etc/host.conf).

--david

--
#
# /etc/nsswitch.files:
#
# An example file that could be copied over to /etc/nsswitch.conf; it
# does not use any naming service.

# "hosts:" and "services:" in this file are used only if the /etc/netconfig 
# file contains "switch.so" as a nametoaddr library for "inet" transports.
