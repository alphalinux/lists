> 1. What does " passing arg1 of xxxx from incompatible pointer type" mean
> exactly, the way I understand it, it has to do with a 32 bit number?????

It means you're passing an actual argument to a formal parameter and
the type of the actual is a pointer that is not compatible with the
type of the formal argument. Say, passing a pointer to an "int" where
a pointer to a "long" is expected. You'll be well-served by tracking
and fixing such problems--they usually spell trouble.

> 2. What is the best way to figure out this memory violation?
> diald: memory violation at pc=155556be3cc rp=120009770 (bad address =
> 00000000)
> Segmentation fault (core dumped)

> I've gotten a few of these core dumps since moving to the 081096 rpms for
> ELF, I don't know f this has anything to do with it or not since I didn't
> try this source before the change.

Compile the program with -g, fire up gdb on that program, then do:

list *0x155556be3cc

and
list *0x120009770

This will show you the source line of the call that caused the
fault (first list) and the call-site of the function that caused
the fault (second list).
