Hello Brian,
I think I can answer to your questions:

> 1. What does " passing arg1 of xxxx from incompatible pointer type" mean
> exactly, the way I understand it, it has to do with a 32 bit number?????
> It should occur in something like:
/* 1.a */
int *a;
extern int xxxx(char *b);
main() { xxxx(a) }; /* here [int != char] */
Note that function xxxx might have been defined by a system header file
included
by your program. E.g.:
/* 1.b */
#include <time.h> /* this header defines time_t and time() */
long *tp;
main() { time(tp); } /* here [time_t != long] */

> 2. What is the best way to figure out this memory violation?
> diald: memory violation at pc=155556be3cc rp=120009770 (bad address =
> 00000000)
> Segmentation fault (core dumped)

A way is to compile your application with the option "-g", then to run it
under a debugger such as gdb. Do "man gdb" for details. The example 1.a of
source code given above is likely to abort with a "Segmentation fault",
because pointer a was not initialized.
