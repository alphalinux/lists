> > if you add tty to your /etc/securetty, it doesn't fail.
>
> Have you tried that? With rsh stdin isn't a tty at all, so I wouldn't expect
> it to ever work.

yes I've tried it and it works for both my rsh and rlogin problems.

# grep tty /etc/pam.conf
login auth required /lib/security/pam_securetty.so
rexec auth required /lib/security/pam_securetty.so
rlogin auth required /lib/security/pam_securetty.so
rsh auth required /lib/security/pam_securetty.so

allows incoming rsh as root based on ~/.rhosts and rlogin succeeds
when entering the password for the first time.

as far as I can see adding "tty" to /etc/securetty has exactly the same
effect as commenting out the pam_securetty lines for rsh&rlogin
