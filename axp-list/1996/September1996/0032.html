>>> Max Hadersbeck said:
> Hi
> After I set the time on my LINUX UDB (Multia) Boxes with the graphical
> time program available in the Configuration Toolset the time and date is
> accepted in LUNIX. After I shutdown the machine and powered it off
> during the next power on booting stops in the ARC and says:
> "Wrong System Time. Must be setted before booting."
> Then I have to run through the machine setup, set the time. Now I can
> continue booting.
> The time I set during machine setup is never taken from LINUX, it is
> always the default time 1970 ... So I have to use the Linux Time
> program.
> What can I do ?

Complain to Red Hat... :-)

The graphical timetool's script does *not* take into account the possibility
that the CMOS clock needs to be kept in ARC console format (ie
/etc/sysconfig/clock was set to: CLOCKMODE="ARC" during installation). This
possibility *is* tested for in the normal bootup script
/etc/rc.d/rc.sysconfig. But /etc/sysconfig/clock *must* be set correctly
for the bootup setting to be read correctly.

Workaround until this is fixed: don't use the graphical tool, but simply
use the ARC console's facility to change time/date. Then, make sure that
/etc/sysconfig/clock contains: CLOCKMODE="ARC", and during bootup the clock
will be read correctly.
