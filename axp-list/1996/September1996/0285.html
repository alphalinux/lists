>The above is not really 1280x1024" just an old label. I have two fixed

*Theoretically*, the TGA could be made to do a lot more than what is
currently available in the sample XF86Config file. However, I don't have
the specs for programming the clock chip that is present on the board, to
any *arbitrary* setting. And the code for the currently available TGA server
is *not* in the public domain; the XFree86 organization has been given some
that is, but they've not had the time or manpower to get one working with it.

The "Modelines" are really just place-holders for the resolution/refresh
rate pairs that I *do* have the programming data for; the only necessary
information (for the TGA hardware code) in a "Modeline" is the name and
dotclock rate. Other parts *are* used by the XFree86 code to validate that
the monitor can do what the Modeline requires, but those parts aren't used
by the current TGA code.

So, the bottom line is: if one of the Modelines doesn't describe what your
monitor needs in terms of a fixed frequency, you're out of luck. If you have
a monitor that claims to be able to do 1280x1024 at 66Hz, for example, try
the Modeline that corresponds; it just won't work if it's not right, and I
don't think it will hurt it, but I could be wrong.

Sorry I can't be more help...
