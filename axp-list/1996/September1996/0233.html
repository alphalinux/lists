Harald Koenig wrote:
> [I wrote]
> > Might it be possible to run executables
> > compiled on an OSF/DEC Unix Alpha,
> > or to use libraries (such as DXML)
> > under axp Linux?
>
> yes for most static binaries this works fine and even using the
> dynamic bianries using the DEC Unix shared libs and /sbin/loader
> works for most programs (of course the latter is only legal if you
> have a DEC Unix license for your Linux/AXP box...)

Both former and latter are illegal without DEC licensing.

To summarize this thread of discussion:

Even statically linked binaries require DEC licensing
if they are linked against a licensed product, ie.
any DEC library. So, to legally run DEC Unix binaries on
a Linux machine without a DEC license, you
must do all of your linking against free Linux
libraries.

Jon Hall (maddog) wrote:
>bob@bob.usuf2.usuhs.mil said:
>>DEC has done an outstanding job of optimizing
>>the Matrix routines. For matrix intense calculations
>>this can make a huge difference. You wouldn't want
>>to buy a 433 MHz Alpha board and not run an optimized
>>matrix multiply routine in Gaussian 94, would you?

>I take this as a compliment, and representing Digital,
>I thank you.

Yes, it was meant as a complement. I tried to
optimize some of the BLAS but couldn't come close
to DEC's version. The difference makes it almost
imperative (at this time)
to have commercially optimized linear
algebra routines on a Linux system doing
big BLAS bound jobs.
