Have got a PC164 motherboard (32 MB RAM) and am trying to
flash milo into NVRAM. Am using the latest milo (2.0.17)
and I get the following message when milo comes up:

Flash device is an Intel 28f008SA
16 segments, each of 0x10000 (65536) bytes
ERROR: failed to find the NVRAM
No environment present

When I use milo's run command to run fmu.gz, I get
on the console the loading message for fmu.gz (####### etc.):
followed by "done", followed by:

> CIA machine check NOT expected

On the serial port, I get:

> Failed determine memory size---using size of 32MB
> Valid Milo reboot block found
> Setting memory size to 32 Mbytes from reboot parameter block
> Initializing timers
> Alpha PCI BIOS32 revision 0.03
> Probing PCI hardware.
> Milo Stub: V1.1
> Unzipping Milo into position @ 0x500000
> Allocating memory for unzip
> ##########################################done.
> Jumping to Milo...
> hack alert! Initialising the floppy
> Flash device is an Intel 28f008SA
> 16 segments, each of 0x10000 (65536) bytes
> ERROR: failed to find the NVRAM
> No environment present
> Failed determine memory size---using size of 32MB

which repeats on the serial port.

Evidently the flash is being recognized to some extent
but fmu can't recognize it as valid, so it returns
back to milo to retry.

The NVRAM is working since I can store boot sequences
through the ARC console, but milo can't use it.

By now I've got the thing to boot from the ARC boot
OK, but really milo loaded into NVRAM is much faster
and cleaner.

Anyone have an idea what the problem might be?

Thanks,

Rob Smith
