Harald Koenig writes:
>> >- with the following line in /etc/pam.conf
>> >
>> > rlogin auth required /lib/security/pam_securetty.so
>> >
>> >the first root rlogin query for a password doesn't work; login name and
>> >password have to be entred again (no problem for non-root users).
>> >without that line, entring the root password works for the first time.
>>
>> It's probably falling through to login. Have you disabled securetty
>> checking for login?
>
>to accept the first rlogin password only the rlogin pam_securetty setting is significant
>(or adding "tty" to /etc/securetty). it doesn't matter if login pam_securetty
>is commented out or not.

Unless you put tty in /etc/securetty, root login should fail. With rlogin,
if authentication fails, it falls through to *login*, and login authenticates
you. Without tty in your /etc/securetty, the securetty line in /etc/pam.conf
should keept root from loging in at that point, too.

If you've put tty in /etc/securetty, then unless rhosts *also* allows the
authentication, you will be prompted for your password by the rlogin
authentication, but it will fail anyway. *Then* login, which doesn't do
rhosts authentication, will allow your login to succeed.
