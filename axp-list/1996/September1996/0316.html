At 08:32 AM 09-10-96 EDT, you wrote:
>
>Well Jay,
>
>We finally tried to boot milo on a Alphaserver 4100. This is where I'm at:
>
>On the Alphaserver 4000 series, the WinNT Arc is different . We found the
>place to setup a different os, but our only options to fill out were load
>identifier,osloader partition and name, ospath partition and path, and os
>boot options.
>
>We could not find anyplace to specify the OSLOADFILENAME. When it tried to
>boot the selection it choked because it showed the OSLOADFILENAME= whatever
>we set the ospath partition to.
>
>After trying every combination we could think of, and looking for a place to
>change the osloadfilename, we gave up.
Why fight the defaults?
I find it is simpler to change the name of linload to osloader.exe and
create the floppy with the paths that ARC expects for WINNT.
For example, on an EB164 I just set up use the following (the defaults):
/os/winnt351/osloader.exe
Therefore I created a directory on the floppy /os/winnt351
and copied (with rename) linload.exe to /os/winnt351/osloader.exe
Then copied milo to /winnt
copy (with rename) milo to /winnt
So what if we are lying and saying that milo is winnt, the system doesn't
care, and I don't believe ( unless, maybe you are selling it) that it's
wrong to call milo winnt ( it is not offended, anyway!).
We saw the same problem in the past where the OSLOADER and other variables
might be changable, but the changes seemed to not work for some reason. So,
to "go with the flow" works, however...
