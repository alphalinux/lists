hmm, it is a little strange isn't it. Well here goes:

[1] ARC was supposed to be a general purpose, machine independent boot environment.
Basically a mini-OS that could load/run OS loader programs. That sort of fell
through and now Microsoft owns it.

[2] Linload is an OS loader and it uses the WNT ARC services (eg open/read file and
the built in device drivers) to load MILO. Linload.exe is built using the WNT ARC
firmware tree + Microsoft compilers/linkers and is NOT free license.

[3] Milo is a mini-console/BIOS that uses Linux device drivers to load and boot
Linux itself. Additionally it provides the Unix/Linux PALcode that Linux requires.
Note that the WNT PALcode is completely different (it runs the CPU in a different
addressing mode) and useless to Linux.

Ideally, Linload.exe and Milo would be merged but in all the experimenting
that I did I couldn't get that to work (including building a dummy WNT ARC executable
header onto the front of Milo and other stuff). On systems where Milo can be blown
into flash/ROM then you avoid this "maze of twisty little passages".

Oh, just remembered, Linload passes information to Milo, including how much
memory there is in the system. So, you shouldn't need to use SET MEMORY_SIZE...
