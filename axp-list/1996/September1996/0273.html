> There's two ways that you _could_ go with emulation, solving two distinct
> problems:
>
> [1] Run windows and dos apps on Linux AXP
> - a possible solution is Wine + x86 emulation

With Bochs you don't need Wine. It provides a complete virtual
machine including a Hercules video driver in an X window.

> [2] Run Linux Intel applications on Linux AXP
> - a possible solution is emulation libraries + x86 emulation

Well, again, with Bochs you have a virtual machine. As soon as
386 emulation is complete you *should* be able to boot Intel/Linux
natively. Then write a VGA driver and run VGA X windows inside of
an X window and run your apps there. Sick...sick thing to do.

Bochs has potential, but the biggest drawback is that it isn't
GPL'ed...it's shareware. That and it's still *way* slow. :-)
