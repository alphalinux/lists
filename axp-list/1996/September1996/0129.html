I am one of the unfortunate souls who recently purchaced a DEC 166MHz UDB from
"The Computer Guys" aka "StarShip Computer Company" aka "American Business
Service Corp.". I regret the purchase, since I am having various hardware
problems, and have been unsuccessful in contacting the company to correct them.

My main problem is that I cannot run Linux (RedHat 3.0.3) with more than 32M of
memory installed on the machine. I get memory errors upon booting Linux, when
accessing the drives through the SCSI interface. It looks to me that the
address of the SCSI device does not relocate above the installed physical
memory when I have more than 32M on the machine. This is using perfectly good
Xx36 parity SIMMS. The SRM and ARC consoles properly show the installed
memory. I also have a problem with the system not detecting the parallel
interface at all (even on the ARC console). Does this machine have a PCI BIOS,
and if so, is it possible that it is defective?

As long as the installed memory is 32M or below, the machine seems to work fine
(except for no parallel port).

I am interested in information on any support options available to get the
machine fixed (at my expense). Does (will) DEC repair these machines? Does
any other company offer repair facilities? Is it just cheaper to take the loss
and buy a new machine?

Perhaps there is a simple solution. Does anyone have experience on problems
with the UDB with more than 32M of RAM?
