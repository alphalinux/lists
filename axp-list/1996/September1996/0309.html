and on one machine I didn't pay attention that in /etc/inittab
/sbin/mingetty was used which I don't have, resulting in the following
kernel messages when booting

init: cannot execute "/sbin/mingetty"
init: Id "1" respawning too fast: disabled for 5 minutes
init: Id "2" respawning too fast: disabled for 5 minutes
init: Id "3" respawning too fast: disabled for 5 minutes
init: Id "4" respawning too fast: disabled for 5 minutes
init: Id "5" respawning too fast: disabled for 5 minutes
init: Id "6" respawning too fast: disabled for 5 minutes
init: no more processes left in this runlevel
kernel: init: memory violation at pc=00000000 rp=00000000 (bad address = 00000000)
init: PANIC: segmentation violation! giving up..

note the PANIC. init is restarted fron the kernel but it's not
possible to change the run level, reboot etc. anymore (network login
etc. works fine).

while I understand that it resulted from an easy-to-fix configuration
error I think the PANIC etc. shouldn't happen [tm] ?!

repeat by [only if you're *really* know what you're typing]:

mv /sbin/mingetty{,.not_there}
reboot
