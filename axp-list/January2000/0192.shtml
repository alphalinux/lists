On Wed, 12 Jan 2000, Ivo Clarysse <soggie@starlab.net> wrote:
: On Tue, 11 Jan 2000, Steve Craft wrote:
:
: >
: > I need to do some FTP installs of Linux, but my FTP server with a CDROM
: > drive is NT.... And since NT can't read the Rock Ridge-formatted RH CDROM I
: > have, I was wondering if anyone out there could direct me to a place that
: > has all of the CDROM contents located on ISO9660 media.
:
: ftp.redhat.com:/pub/redhat/redhat-6.1/iso/6.1-alpha.iso
:
: This is a bootable ISO9660 image of RedHat 6.1/Alpha.

It is also, if I'm not mistaken, a Rock Ridge image (since Rock Ridge is
just a convention for ISO9660-formatted disks to represent Unix-style
files and directories). So I think what Steve is asking for is a disk
that, when mounted under WinNT, has all the long filenames and file
permissions that a Unix system would see. This could be a Joliet
disk (WinNT's extended ISO9660 format) or a disk mounted with a Rock
Ridge-compatible WinNT filesystem utility, or a Linux utility that
can write ISO9660 with Joliet extensions.

If you look at a Rock Ridge disk from an ISO9660 perspective, files all
have DOS-style 8.3 names, but there are special files that allow these
to be translated into Unix-style names, permissions, and directories.

Joliet is similar, except it has added Microsoft attributes, lacks some
Unix attributes, and is completely (and unnecessarily) incompatible with
Rock Ridge. It, too, uses ISO9660.

One other solution that I'd really like to see is a bootable CDROM
that has a RAM-based Linux system set up as a ready-to-run FTP server.
(Are you listening, RedHat?) This would allow people in Steve's
situation to simply boot such a disk on a free WinBox without the need
to do a local install, leaving the WInBox's hard disks untouched.
