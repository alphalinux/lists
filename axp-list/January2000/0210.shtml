Thomas Weyergraf wrote:
>
> > Michal Jaegermann wrote:
> > [ ... stuff deleted ... ]
> > To add my $.02.
> >
> > Ask yourself how many businesses would buy N boxes based
> > on looking at a game? I certainly wouldn't. See how odd
> > that sounds??? A business selecting a box because it can
> > waste an employee's time?? I know you say it means performance
> > in other areas are good too but that is exactly the point.
> > You sell based on the other areas to businesses, not games.
>
> Probably no one would buy a system for serious use by just
> looking at games. But looking over the "fence" towards SGI
> helps. On every computer show, they ususally show off some
> fancy, colourful moving/flipping/rotating graphics demo.
> In effect, that gave them the reputation of being *the* graphics
> company.

Note here you said graphics demo, not a game. I have no
problems with a graphics demo. I've used graphics myself to
catch a passer-by's eye at shows.

> Now, Imagine, one would use a recent x86-world 3D games
> accelerator on an alpha. Make people aware, that
> the grahpics used on the machine is some top-dog PC
> graphics ( like Nvidia's GeForce256 with DDR Ram ).
> Now port Quake to it. Use a GL, that makes use of all the
> funny 264 features: fully pipelined FP, NVI instructions etc.
>

I'd rather all that effort be spent on a real application.

> Put that system right *next* to a top-end x86 System using the
> *same peripherals* ( important - same graphics ). Run the
> timedemo on both and see the Alpha fly.
>

That's a good idea only if the demo requires heavy FPU
usage. Otherwise, the Intel/Athlon just might win.

> Now what ? I admit, that this demo will probably not sell a
> *single* system to a coorperate customer, but it will generate
> the perception of alpha being a superior platform among the
> game-kidies. magazines will love to see a system like that
> being some sort of top-end-games-reference machine.
> Give it as a free loaner for testing to the "pc-mega-games"
> magazine ( or whatever they are called ).
>

I agree with the idea of creating perceptions. That's
what marketeers do.

> You catch a trend here - the *only* reason, the average user
> is buying a faster x86 machine is for games. No one with some
> brains would claim, that you need a 30+ SPECint95 800Mhz
> Athlon to run Word or Excel.

Depending on the spreadsheet, it might actually be useful.
You'd be amazed at what technical types put into them.

The only reason I buy fast x86's is to see where they're
at relative to my other HW based on my own codes. So far, I'm
not happy with their performance so I relegate them to chores
like file servers/firewalls/print servers/X stations. If I had
that level of performance on Intels's two years ago, I'd
probably be running on them but they lag state of the art
performance. Alphas are, at this point state of the art, near
as I can tell. At least when considering price/performance/
space.
