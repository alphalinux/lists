Mike Foley wrote:
>
>
> All this talk of passing "pci=reverse" in the boot
> flags has me wondering what other interesting things
> you can pass to the kernel in the boot flags.

Checking available documentation would help to resolve at least part of
this "mystery". Start with 'BootPrompt-HOWTO' which is most likely on
your disk. Files like 'kernel-parameters.txt', and other, from a
subdirectory 'Documentation' of kernel sources will have some essential
information as well.

OTOH it is true that pieces of this puzzle are often scattered all over
the place. For example, a very detailed description of various
configuration parameters for ncr53c8xx and sym53c8xx drivers can be
found in drivers/scsi/README.ncr53c8xx. Similar with BusLogic, Adaptec
and other SCSI controllers. For other parameters probably only kernel
sources are the only, and an ultimate, information trove. I think,
although I am not sure, that this is the case, for example again, with a
meaning and values of various parameters for tulip driver.

Volunteers which would like to contribute some cleanup and expansion in
the area would likely be very welcome. :-)
