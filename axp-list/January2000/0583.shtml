>>From talking to a Q on another Alpha list, he indicated that there will be
compiles for Alpha Linux, Tru64, Alpha NT and OpenVMS. They are waiting for
the SETI@home people to release the source kit to the "porters".

Compaq did come up with better performing code, but it was not used in the
1.x packages. They are not sure if it will be used in 2.x. SGI is
apparently not being allowed to use their custom code either. Before you
get in a tizzy over that statement, the SETI@home people are trying really
hard to make sure every client gives the same results from the same code
base. While it would be nice to be able to process the blocks faster,
I for one do not want to miss the intergalatic boat. :) Hopefully, the
speedy code can go through some validation/verification process and be
used in the future.
