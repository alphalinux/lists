Thanks to Rich Payne and Vanco Donald for the answer. My problem is solved by
build a new kernel (nothing to do with "guessing the device")
I did try
boot dqb0 -fi 2/boot/vmlinux -fl "root=/dev/hdc2 bootdevice=/dev/hdc"
and it did eliminate the line:
aboot: booted_dev = 'ide 0 13 1 0 0 0 0', guessing boot_dev = 'scd0'

Everything works fine now.
