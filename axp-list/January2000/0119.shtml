Andy Infante wrote:
>
> The issue I have is with an Intel
> EtherPro (10MB) not being recognised.

10MB EtherPros are hopelessly broken by design. This is NOT the case
with similarly named 100MB cards. So probably the first reasonable
step would be to use that card as the first handy doorstop and replace
it with something sane or you are asking for constant network
troubles. You can buy a not bad ethernet card in $20-$25 range.

> Perhaps I am not going through all the
> steps needed.

It this Plug&Pray variety? Most likely. In that case
you have to install 'isapnptools' package, read documentation,
use 'pnpdump' to write a skeleton configuration file and edit
results to fit your card. Such config has to used **before**
you make any attempt to load a card so the easiest way to achieve
that is make a card driver a module and run P&P configuration
in your startup scripts before you attempt to start network.
I believe that Red Hat standard init will do that for you.

If you insist on a monolithic kernel then you have boot with
a help of initrd and run your configuration from there before
anything else.

Current 2.2 kernels have an option to configure P&P in kernels and
Configure.help says:

Plug and Play support
CONFIG_PNP
  Plug and Play support allows the kernel to automatically configure
  some peripheral devices. Say Y to enable PnP.

What "some" really means your guess is as good as mine.

Saying all of this see above again about your particular card.
