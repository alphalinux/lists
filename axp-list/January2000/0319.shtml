> FWIW, some people have reported problems with the libraries from 5.0
> (Netscape users report DNS doesn't work etc..) that didn't exist with the
> 4.0x libraries.

More precisely, it can't stat /etc/svc.conf. It is not even able
to resolve 'localhost'. As suggested on comp.os.linux.alpha, I
made it work by installing junkbuster as a proxy http at
127.0.0.1. (thanks to the problem above, putting localhost does
not work.) In this way http works. ftp, or file, protocols don't.

I have to say, I had this problem with the 4.0F libraries as
well. (I'm running 2.2.13 on a 164SX.)
