What I did was to start with a fresh /etc/isapnp.conf created with pnpdump.
Set everything the way you like it so it matches the way your soundcard is
configured. Change the following line:

(CONFLICT (IO FATAL)(IRQ FATAL)(DMA FATAL)(MEM FATAL)) # or WARNING

to read:

(CONFLICT (IO WARNING)(IRQ WARNING)(DMA WARNING)(MEM WARNING)) # or WARNING

This is so isapnp doesn't abort when it hits a "conflict".
Make sure your /etc/conf.modules isn't loading any old sound modules, since
you linked them into the kernel.
Do a "pnpdump --reset". Then a "isapnp /etc/isapnp.conf".
Do a "reboot". Everything should now be ok, provided your settings don't
conflict with any other devices. (I'm assuming they don't.)

Enjoy sound. Enjoy system stability. Curse linux's DMA code for being broken.

The only caveat is if you hit reset, the soundcard may get confused again.
If so, go back to running "pnpdump --reset", etc. etc.

If it helps, I'm using the settings:
ioports 0x220, 0x330, 0x388
irq 5
dma 1 and 3

I get an error message on bootup that my 16-bit dma channel is wrong, but I
can live with it. Doesn't seem to hurt anything.

Good luck,
-Mark
