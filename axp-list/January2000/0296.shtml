Hello,

I just I would drop everyone a note to let you know where em86 is
currently at.

The linker problem has been solved 2 ways. First I have a quick patch
to ld.so that removes an assertion but  Ruediger Oertel pointed out
the real problem was a symbol in the emulator library.

Second: I have traced the last segfault in the program to the
interaction of the emulator library and the glibc 2.1 stdio functions. My
guess is a recompile of the emulator library will fix this and I am hoping
real soon now to get the NDA for that.

Third: With a patch to em86 and to ld.so I can get em86 to run under
glibc 2.1 any application that does not use stdio. The only application
so far that fits this bill is ldconfig86. sigh.
Fourth: As soon as I sleep (been up way too long on a really cool
porting project for the alpha) I am goin to post the latest sources to
my web site and upload them later to ftp.alphalinux.org (when their site
is working again) The only real change is some additional comments and it
can correctly detect which version of glibc is installed and it looks
for /usr/lib/libcpml.a to link against.

As soon as I get the sources to the emulator I will tell everyone and
squash the last couple of bugs.

My future plans involve trying to relink em86 with the shareware booch
emulator so that works. (This is a lot of work as the CPU interface has
totally changed)

Sorry about it being so long between updates but there has been no real
news.

Leslie Donaldson
