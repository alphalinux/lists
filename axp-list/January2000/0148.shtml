Adam C Powell IV wrote:
>
> Phillip Ezolt wrote:
>
> > One thing I would love to see is a rocking Alpha/Linux game/demo page.
> >
> > We have source to games (Quake, descent, doom, heretic).
> > We have source to the Voodoo chipset drivers.
>
> And to Mesa...

You would likely need for that working 3d graphics cards (which would be
useful not only for games :-) which drops, in a sense, a ball on a field
of firmware which has to be able to initialize these cards or you will
not even boot.

> I'd like to see an installer which chooses which binary or lib to install
> based on the CPU or mobo. The Debian package format includes a postinst
> script which can do this, can RPM do the same?

Yes, and actually quite a few packages you installed has such
postinstall scripts. An example which probably does the most of
"serious work" in postinstall scripts is likely provided by rpms of
Compaq Fortran and C compiler libraries; all shared versions of
libraries are created by postinstall scripts.

Try 'rpm -q --scripts ...' on few packages. :-)
