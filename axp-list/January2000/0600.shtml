Robert Williams wrote:
>
> Does anyone have a Promise Ultra66 running
> on an alpha, kernels 2.2.14 or 2.3.xx?
> I've applied the patch for 2.2.14 at
> http://www.kernel.org/pub/linux/kernel/people/hedrick
> but the boot process freezes shortly after recognizing
> the Promise card. The disk is a 40G Maxtor.
> I've read http://www.moisty.org/~brion/linux/Ultra-DMA.html
> Is there something else I need to do?
>

I don't have the Promise card but I have put a 40GB drive
on both DS10's and UP2000's on-board E-IDE connections.
Things seem a bit touchy at the moment. The DS10 doesn't
like seeing a cdrom and the 40GB Maxtor on the same channel.
Gives errors with the patch you mention above on 2.2.14
using DMA. The UP2000 doesn't want to see anything above 32GB.
I'm using an E-IDE CDROM there also so it could be tied to
that device being attached too. Haven't done all permutations.

There appear to be issues with mixing device types/speeds.
Haven't tried a 2.3.x kernel yet.

On Intel, if I put two 40GB Maxtors on a channel, it seems
to behave fine but this depends on the motherboard. Some
motherboards are unhappy even doing that. The Supermicro
PIISCD with Intel 820 chipset does not like large drives
particularly well. VIA Apollo Pro+ chipset boards are more
tolerant(PC133 memory/133FSB). Tried two of those and they
are both happy. Possibly it's a BIOS issue but then I thought
Linux didn't use the BIOS except during boot so that leads
back to a driver problem.

Great price and performance though for those drives. I see
13-24MB/sec on Intel using DMA depending where you're at
on the platters. I filled the whole drive with 20GB files
to check them out.
