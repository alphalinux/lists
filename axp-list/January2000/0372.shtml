>>> Christoph Dworzak said:
>
> I just upgraded my PC164 a little bit. I put in
> a second SCSI-Adapter and put some of the HDs on
> it. Now there are 4 HDs on the first and 3 HD plus
> one CDROM on the second Adapter.
>
> Now the problem was, that MILO and Linux don't
> agree on the scan direction, MILO puts the 3 HD-Chain
> first and the 4 HDs behind it. Linux does it the
> other way round (where Linux=Redhat6.1 with
> Kernel 2.2.13).

Try specifying "pci=reverse" on the boot commandline to the kernel.

Note that this usually is only effective if the controllers are the
same type, which it is in this case (NCR875s). This parameter only affects
the list order of the devices, and not the order in which the different
types of SCSI adapters are scanned, which AFAIK is always fixed.
