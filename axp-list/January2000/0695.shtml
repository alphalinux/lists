2.3.42 was released while I was in the process of generating
the diffs, but I don't think there's any overlap.

The only change to generic code is to get bootmem to allocate
memory more aligned than a page. Apparently there's some HW
bug in CIA and/or PYXIS that requires iommu page tables to be
32K aligned instead of 8K aligned.

I got lazy toward the end. MCPCIA is stubbed out so that the
rest of the kernel can compile. I left those docs over in
Dave's machine room some time ago and keep forgetting to go
collect them.

Tested on ds20 and pc164, though there aren't many devices yet
in the main tree that can take advantage of the iommu.
