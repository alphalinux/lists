Dan,
        Don't get suckered into thinking that games are irrelavent.

1) Some games require a working, optimized 3D enviornment. Many
scientists would benefit from being able to use this to look at their
climate data, genetic data, financial data in new ways.

2) Most games demand full resources, and work the system hard. Any
program that stresses many elements of the system (network, graphics,
sound) will find bugs that affect programs in other areas.

3) Most games require real-time like response. Any program that requires
such a reponse will benefit from problems fixed by working on a game.

Games are a fun way to make sure of all the above works.

As for the demo, I think that Alpha has the potential to wow the
crowd. A nice quick, graphical program (such as a game) would make
it really obvious that it is a fast little box. ;-)

> I've been using Un*x for a decade, Linux for 7 years, and have a keen
> dislike for most graphics.. so what I'm about to say might be biased.
>
> When I decided to go with Alphas, the first thing that came to mind
> was not "Gee, I hope this thing runs [game] fast". The majority of
> organizations and individuals using Linux on Alpha for real work

> game performance under Linux/axp tells me you'd like to sucker in
> people whose use for a computer could be better served by a game
> console. Don't get me wrong.. I think it's a nice afterthought
> to have some slick games under Linux, but the having the
> first impression of "Wow, it runs Quake." is probably not constructive.
