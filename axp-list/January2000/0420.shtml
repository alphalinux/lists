Since I last wrote, I've learned a bit more about our Alpha/Linux system. Hopefully this extran info will lead to a solution.

I should have stated before that one SCSI controller is embedded on the motherboard. So here's the sequence of events that take place at boot up:

- BIOS points to file on sda1 (sda1 is connected to SCSI controller embedded on motherboard)

- System then initializes all other "stuff" on cpu
      - Recognizes Adaptec 2940

- System then looks for hard drives
     - Finds external drive FIRST and calls it sda
  
     - Finds internal drive next and calls it sdb

Therefore the system tries to use the external drive and halts.

I've thought about copying the contents of the internal drive to the external - but have yet to try it. Let me know what you think. Thanks again!
