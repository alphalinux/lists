> I have a non-practical question for everyone.
The same question we (ALO) were asked by API last year.
Funny how everything comes around in time ;-)

> What is your dream of the Alpha/Linux future?
Clusters for scientific computing and rendering farms,
dedicated systems for networking, telecomm (telephony
 under Linux is really taking off!) and signal processing,
general-purpose scientific workstations...
 
> One thing I would love to see is a rocking Alpha/Linux game/demo page.
Not me. Besides the hack value, what would it accomplish?
 
> We have source to games (Quake, descent, doom, heretic).
> We have source to the Voodoo chipset drivers.
>
> I would love to have a knock-your-socks off graphic demo that I could
> use to show off the alpha.

I've been using Un*x for a decade, Linux for 7 years, and have a keen
dislike for most graphics.. so what I'm about to say might be biased.

When I decided to go with Alphas, the first thing that came to mind
was not "Gee, I hope this thing runs [game] fast". The majority of
organizations and individuals using Linux on Alpha for real work
probably only care about the performance of one or two applications
critical for day-to-day operations. Focusing energy on improving
game performance under Linux/axp tells me you'd like to sucker in
people whose use for a computer could be better served by a game
console. Don't get me wrong.. I think it's a nice afterthought
to have some slick games under Linux, but the having the
first impression of "Wow, it runs Quake." is probably not constructive.

If Compaq or API plan to introduce a low-end Alpha box with fast
graphics and a < $1500USD price tag targeted at home users and use
suped-up video games to market it, more power to them. I just feel
that showing off video games on a $5k+ system is a bit silly ;-)
