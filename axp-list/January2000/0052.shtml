>>> Designer Seven said:
>
> I just wanted to confirm if anyone else is having this
> problem... or if they could try to verify it.
>
> All kernels between 2.2.14pre9 and 2.2.14pre17 have
> had this problem on my Miata.
>
> The problem is, if I specify the amount of RAM at
> MILO:
>
> boot sda3:boot/vmlinux.gz mem="256M"
>
> the kernel boots and decides that there is 0MB of RAM.
> If I leave out the mem= parameter, it recognizes the
> 256MB or RAM and I can boot fine.

Syntax should simply be:

    boot sda3:boot/vmlinux.gz mem=256M

I tested this on an LX booting a generic 2.2.14 kernel from SRM, and it
worked fine. Note that earlier 2.2.14-preX kernels might have had this
broken, which may be what you see. Or possibly it's something that MILO
is actually doing something with, and getting it wrong.

Please try the released 2.2.14 and see if it works better for you.

Just out of curiosity, why bother to specify what it already knows?

The functionality was added for various reasons, one of which is to
"reserve" some physically contiguous memory that won't be managed by
the kernel; can you say DMA memory for 3D accelerators? ;-}
