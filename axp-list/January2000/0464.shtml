I tried some experiments with RH 6.1 installer. In particular I passed
an option "expert" when starting the whole thing so I got "Use fdisk"
button. So far so good.

Note that you have to be an "expert" in order to perform a basic
installation from SRM console, as fdisk is required, but
'etc/aboot.conf' on CD does not include that option so in principle if
you are newbie then you are screwed (and in any case do not get me
started on this other abomination which pretends that it can be used to
partition disks).

Ok, so I pick up 'fdisk' and continue; anaconda opens a big white window
on my screen with an 'fdisk' title, a cursor starts blinking and that is
about it. The only operational button is "Help" which toggles on and
off an emtpy help window. That means that we not crashed yet but we are
totally stuck. There are some text consoles around but there are no
error messages or anything else of any help. Yes, I can start ok
another copy of fdisk from a shell prompt (with that small catch that
for that one has to know that it is possible to switch there and that a
device node to work on can be found in /tmp) but I am in this game
for a while.

This is on UX which I happen to have handy. I just wonder if this
piece works for somebody on some other Alpha or this is broken in
general?

There are installation options which are actually available only
if you add "expert text" to your boot command. Oh, well...
