On Friday, 14 Jan, Themis Lazaridis wrote:
>
> I'm trying to get two alpha linux machines to share the same
> mailbox, so I NFS mounted the mailbox of A onto B and linked
> the /var/spool/mail directory of B to the mounted directory.
> Problem is that I cannot send mail to B. It gives me the
> error
> lockd: couldn't bind to server ...

To do fcntl locking over NFS in Linux both machines require:
- kernel at least 2.2.14 with _kernel_ NFS server (compiled in or modular);
- activated kernel lockd (through rpc.lockd from NFS-kernel-utils or
  something like that).
Make sure you can see [lockd] in the ps ax listing on both machines.

(There is currently no lockd support in the userland NFS server.)
