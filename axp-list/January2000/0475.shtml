Ron Farrer wrote:
>
> Larry Snyder (larrys@lexis-nexis.com) wrote:
>
> > Like Michal said, a newbie wouldn't stand a snowball's chance in
> > Hell getting an axp up with the current distros.

I did not say that. Actually I can imagine a hardware/firmware
configuration on which "point-and-click-aren't these pictures
nice" installation would work.

> I wouldn't say all the current distro are screwed. I've done several
> installs of Debian GNU/Linux 2.1 (slink) from scratch, each one worked
> on the first try.

Various things will work, in different places, "on the first try".
I did some succesful installations of RH 6.1 and even tried to
"play dumb" - although with the later I was less lucky. At least
it does not crash on a broken 'librpm' which is a progress. :-)

The more disturbing is a general trend in Linux installation utilities
for better and better hiding tools which you may need when things will
stray from a prescribed scenario. They are still there but if you do
not know where to look then you may find out yourself facing greyed out
both "Next" and "Previous" arrows and now what? This is not a property
of a particular platform or distribution. Yesterday "Linux Today" had a
link to a review in which its author was smitten by an installation
utility from Mandrake 7.0. A comment from a reader pointed out that on
his machine an attempt to use this installer become a minor nightmare, a
big headache and a huge screwup. YYMV.

I am getting also more and more careful about possible loss of data
("express install" anyone? :-). I was playing with "Update
installation" a little bit and it "updated" ___with the same versions___
some 180 MB of perfectly installed packages modifying various
configuration files in the process. That after I walked manually a list
of packages to update and verifying that none is chosen. Sigh!

> Although I didn't try booting from the CD, just made
> the 3 floppy disks and used the CD as the source...

It did boot where I tried. With SRM firmware you can get away
with no floppies at all. This may lull you in a false sense of
security, but this is another story. :-)
