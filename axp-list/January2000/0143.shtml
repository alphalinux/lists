On Mon, Jan 10, 2000 at 06:43:57AM -0500, Craig P Prescott wrote:
> I ran into the same problem when I grabbed the gcc-2.95.2-3 source rpm

These rpms are broken.

> You can solve this problem by building gcc-2.95.2 from the tarball.
> When you build gcc-2.95 from the tarball, it puts cpp in
> "--prefix"/bin and everything just works. And you can make a link

Actually, there are two different cpp executables - one in
"--prefix"/lib/gcc-lib/${ARCH}-${VENDOR}-linux/2.95.2 and another
in "--prefix"/bin, which is the one what we need and which is missing
in rpms. The former is intended for gcc internal use only.

Fix is quite easy - just add %{GCC_PREFIX}/bin/cpp in %files section
for cpp package in the gcc.spec and point the link /lib/cpp to correct
executable, although I think this link isn't really needed - I happily
live without it last few months.

Or grab working gcc-2.95.2 rpms I built for myself from
ftp://ftp.park.msu.ru/ink/rh6.1-rpms (BTW, rth's float-complex patch
is included there, so you can compile glibc with it, although
only with -O2).
