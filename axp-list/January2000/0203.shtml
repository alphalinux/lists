Attribute that to increased photon emissions or something.
The machine check errors have been discussed at length several
times - browse around http://www.lib.uaa.alaska.edu/axp-list to
get the full story.

> Jan 12 13:24:05 hera kernel: TSUNAMI machine check:
> vector=0x630

Processor Correctable correctable cache and TLB errors, detected by
processor (to quote Jay).

> pc=0xfffffc0000469408
Program counter. Means nothing (to quote Greg).

> I know this has been discussed recently, but I can't remember seeing a
> definitive respone. The question I have is does this imply a hardware
> (memory or otherwise) failure, and should I seek a fix while the machine
> is under warranty before more serious problems occur? Or is it simply

Maybe a DEC or API entity can back me up here.... but this sounds like
a defective memory module(s). A machine check is thrown when the
kernel tries to access invalid memory regions.

> current kernel incompatabilty with UP2000? We do not experience similar

If the kernel were incompatible with the UP2k architecture, you would
not boot, let alone see errors to report.
