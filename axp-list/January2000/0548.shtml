On Wed, 26 Jan 2000, Hoover, Tony wrote:

> Doesn't the SRM need all of sector one of the hard drive for boot code?
> Sector one is where DOS style partition tables are kept.

 AFAIR, SRM uses 24 bytes at the end of the starting block of a device.
The bytes form three longs of which one is a number of the first block
to read, the second one gives a number of consecutive blocks to read and
the third one is a magic value to confirm the two other ones are valid.

 As a result, only two DOS-style partition entries can be created and also
there will be a rubbish definition of a fourth partition, which can be
considered valid by the kernel with all sorts of weirdness resulting.
