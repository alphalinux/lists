Eric Jones wrote:
>
> The problem is this. I want to set up masquerading on this box so when I
> go adsl in a week, the machines on my lan can get out. That means that I
> need to have two ethernet cards in here, but I've only got the one is
> ne2000.

No, the conclusion does not follow from premisses. You can actually
set up masquerading, and a firewall, using one card and IP aliases.
There are considerations, like a performance and an insulation, which
make two physical cards preferable but they are of less importance
on a "home network" of few machines hooked up via adsl. Most likely
you will not run in your setup any servers for an outside world anyway
so, additionally, you can block all incoming traffic on an "exposed"
(virtual) interface in /etc/hosts.allow(.deny) and that should do it.

There is also, a relatively minor in most cases, problem that you
cannot shut down your main network interface without stopping aliases
as well. If your "outside" connection requires main IP (DHCP, for
example) then this precludes your "home connection" running without
"outside" beeing active unless you reconfigure (switching a link, or
two, does that so this is not end of the world either).

> ... or do I go out and buy a pci
> ethernet card?

As this solution will cost you something of an order of $20, or less,
then it does not seem to be that hard. I thought that you are more
concerned about loosing your last PCI slot.
