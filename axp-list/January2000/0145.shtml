Phillip Ezolt wrote:

> One thing I would love to see is a rocking Alpha/Linux game/demo page.
>
> We have source to games (Quake, descent, doom, heretic).
> We have source to the Voodoo chipset drivers.

And to Mesa... CPU-specific compiler flags help performance a lot (I got a
several-x spedup in the timed test which comes with Mesa by using -O6
-mcpu=ev56), but why not "vectorize" some things, e.g. using Goto's vectorized
sqrt/sin/cos in libffm, or some of his assembler BLAS? Seems it could help a
lot.

I'd like to see an installer which chooses which binary or lib to install
based on the CPU or mobo. The Debian package format includes a postinst
script which can do this, can RPM do the same?

Also, we have the source to Quake, Voodoo drivers, etc., but making
them 64-bit clean could require a lot of effort...

My .02,

-Adam P.
