On Mon, 17 Jan 2000 putthipong@sisthai.com wrote:

>
>
> Dear all,
> I have a problem with installing Linux on DS10 IDE harddisk machine. After
> I installed Linux and used "swriteboot" command, I reboot (on SRM)
> boot dqb0 -fi 2/boot/vmlinux -fl "root=/dev/hdc2"
> it said:
> aboot: booted_dev = 'ide 0 13 1 0 0 0 0', guessing boot_dev = 'scd0'
> Then it did load and start vmlinux but could not continue and returned to SRM
> again.

If it loaded the kernel but then returned to SRM then chances are you are
passing it the wrong root filesystem. The best way to see this it to
attach a serial console to the box, watch it boot and you'll see the
unable to mount root fs message.

Is hdc2 the root filesystem and did you compile the correct filessytems
into your kernel (ext2)?

> How can I force this guessing from scd0 to hdc?

I don't think that's your problem.
