On Wed, 12 Jan 2000, Darin Walters wrote:

> We just upgraded a server running RH 5.x to RH 6.0. This server runs
> sendmail for our site. After the upgrade there is a noticeable lag
> sending mail from a client workstation using a program like Netscape or
> Eudora.

Most often this delay is due to dns problems. By far the best thing you
can do when running a mail server is to also run bind (dns server) on it.
This doesn't mean it has to be a primary or secondary dns for your domain,
and it can even be what's called a "caching" dns server--it's not
authoritative for any domains, and it caches the responses it gets from
other name servers.

You may try adding an entry in /etc/hosts for a client machine and see if
that fixes the problem for that machine. If so, you know sendmail has
been trying to do dns lookups on the client machines as they connect to
the server to send out their mail.

BTW, this is a bit off-topic for this list, as this issue doesn't seem to
deal with Alpha Linux. For issues dealing with servers running Linux, I
really like server-linux@netspace.org.
