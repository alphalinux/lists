Dear all,

I want to set up a Digital Personal Workstation 500au as a dual-boot
machine via SRM (Digital Unix 4.0e and Redhat 6.1). So far I installed
RedHat 6.1 on a second disk and can boot it (using the
generic-up-2.2.12.img kernel ... no other would work).

Since the machine has a Powerstorm 4D60T (unsupported under X) - which
I still want to use when booting into Digital Unix - I would like to
put a cheap PCI graphics card into one of the free PCI slots and
switch the monitor between these two graphics card depending on the OS
I'm booting.

I found in the documentation, that there is a 32bit PCI (slot 1) and
two PCI/ISA slots (2 & 3) still free. Which one should I use? Are
there any restrictions for the PCI graphics card regarding the SRM
firmware (of course there are restrictions in XFree86)?

So far I put a SiS 6326 into any of the free PCI or PCI/ISA busses: it
is visible in SRM (always at bus 01, but in different slots). But if I
attach the monitor to it the console just stays black during booting
(DU or Linux). I did the genvmunix/doconfig/reboot under DU (although
that shouldn't make any difference for Linux anyway, right?).

I was wondering if

  1. the SiS 6326 is just useless in a PWS 500au
  2. keeping two graphics card in the box won't work
  3. some jumper have to be set
  4. some SRM variables determine which PCI card/slot is used as
     console
  5. the console has to be in a specific PCI slot

On another issue: has anyone out there the .config file for the
generic-up-2.2.12.img? Compiling any of the RedHat 6.1 kernel, 2.2.12,
2.2.13 or 2.3.40 didn't work so far ...

Any help or hints would be apreciated!

Cheers
