On 17-Jul-00 bryan@ak.net wrote:
> Michal Jaegermann wrote in
> <200007161508.JAA22698@ellpspace.math.ualberta.ca>:
>>
>> > The MILO HOWTO suggests using the same MILO version as your kernel.
>>
>> This is not a suggestion but a requirement when you are _compiling_
>> your milo from sources. Versions of sources HAVE TO agree.
>
> Okay. So apparently, whoever builds the 'prebuilt' milo images
> has a special way of doing it that doesn't depend on the kernel
> version...?

This is not strictly needed, but was a silent convention for
some time now. As Milo links against Linux kernel objects, it depends on
some symbols (function names etc.) of those objects. Function names are
alike to change from kernel version to kernel version, so it's probably not
possible to use a kernel version for building (!) Milo except the one(s) a
particular milo release was written for.
In most cases the changes needed to use a more/less recent kernel version to
build Milo are very small.
