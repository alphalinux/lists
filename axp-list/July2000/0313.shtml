On Wed, 26 Jul 2000, Michael A. Smith wrote:

> > I'm recompiling apache from scratch on a Stock RH6.2 system. Since I've
> > been using the stock Apache that came with RH I'd like to keep the
> > directory structure the same (it's got a lot of virtual sites on it. I'm
> > wondering if anyone has the configure options used so that when I
> > recompile I don't have to change and file locations etc.
> >
> > i.e. ./configure --sysconfdir /etc/httpd/conf/ .....etc.
>
> Check the INSTALL file in the Apache dir. There's a config flag
> (--with-layout=XXXX) that lets you easily set the whole Apache directory
> layout. I thought that there used to be a Red Hat compatible layout
> (e.g. --with-layout-redhat or somthing) but I see no reference to it in the
> current INSTALL file.
>
> You can make your own custom layout files, also. Maybe someone has a Red Hat
> comaptible layouyt file.
>
> >From INSTALL:
> <SNIP>
> Use the --with-layout=[F:]ID option to select a particular installation
> path base-layout. You always _HAVE_ to select a base-layout. There are
> currently two layouts pre-defined in the file config.layout: `Apache' for
> the classical Apache path layout and `GNU' for a path layout conforming
> to the GNU `standards' document. When you want to use your own custom
> layout FOO, either add a corresponding "<Layout FOO>...</Layout>" section
> to config.layout and use --with-layout=FOO or place it into your own
> file, say config.mypaths, and use --with-layout=config.mypaths:FOO.
>
> Use the --show-layout option to check the final installation path layout
> while fiddling with the options above.
> </SNIP>
