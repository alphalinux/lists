Frank... are you refering to the boot up of a linux distribution, or are
you refering to the boot up of the JumpStart! CD?

If you are refering to the boot up of the JumpStart! CD, then you want to
first try a 'boot <cdrom> -fl 0'. If that doesn't work, then the correct
parameters to pass to aboot through SRM are 'boot <cdrom> -fi vmlinuz -fl
"root=/dev/ram initrd=initrd.img"'

In reguards to the jensen question. I tested this morning on a jensen, and
the JumpStart! kernel doesn't boot since the generic kernel doesn't work
on a jensen. We'll definately look into this.
As far as booting a linux distro cd so it works on a jensen,
you probaly need to find a distro that supplies the correct kernel.
