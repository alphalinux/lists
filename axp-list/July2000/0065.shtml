Wynne Anderson Hexamer wrote:
>
> Thank you all for the suggestions on my AS800 installation, especially Jay++
> for the hint to check the other terminals. After much consertnation, it
> appears that the problem is with the cdrom timing out. Since both SCSI
> cd drives I have are the same model, I'm suspicious of them. Therefore I'd like
> to try an nfs install, but haven't managed to get the syntax down from SRM.
>
> I've got a DHCP server with the Alpha cd exported on /mnt/cdrom. I've
> managed to mount the cd from another workstation on the LAN. The Alpha does
> allow configuration of the network card through alphabios, and when booted from
> the floopy or (sometimes) the cd using dva0 or dka300 it appears the network
> card is detected.
>
> Does anyone know the correct syntax to do an NFS install from SRM, using
> perhaps the -flags parameter? I tried nfsroot=nfsserver:/mnt/cdrom but that
> was just a guess. The x86 disk contains a different boot image... is there a
> nfs boot image for the as800?
>

Do you mean an NFS install to local disk or using an
NFS root file system? Assuming you mean an install to
local disk, just boot using two floppies, boot/ramdisk
and when it asks where to install from, select the NFS
option and setup your network card and tell it where to
mount from and it just works, at least it does for me with
RH6.1 on XP1000's and UP2K's. I don't install CDROM's in
machines anymore and always use this method. Also, make sure
you're allowing the IP address to mount from the NFS server.
If it can't mount it, it will complain and ask you to try
again.
