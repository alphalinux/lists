Bryan J. Smith wrote:
>
> Max Weninger wrote:
> > rpm 3.0.5 has a compatibility mode and can use 4.0 rpms!
> > You can get 3.0.5 e.g. from www.rpm.org
>
> First off, thanx dude!
>
> Secondly, where did you find this info?

On those few occasions when I wanted to peek iniside of some RPM
archives from rawhide, and using an "old" rpm, changing \004 to
\003 in an rpm header (the fifth byte from a start) worked well
enough. :-) It is easy to hack something in a few lines of perl,
for example, which will do this replacement in bulk - if you need
that. It is also possible to hack rpm 3.0.4 (say) sources to make
this new rpm version acceptable to it.

The above may not always work as there is a possibility that you will
find new tags from version 4 which are not recognized in a version 3.
So far this was not a problem in practice.

Careful with using an rpm from 4 series to install anything.
A database format did change (again!).
