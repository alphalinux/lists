On Sat, Jul 22, 2000 at 09:48:16PM -0800, bryan@ak.net wrote:
> Okay, here's the .config that generated the kernel that oopses
> on boot. 2.2.16-3, LX164.

> # CONFIG_ALPHA_SRM is not set

Booting via MILO, right? Otherwise, this should be "Y".

> #
> # Console drivers
> #
> CONFIG_VGA_CONSOLE=y
> CONFIG_FB=y
> CONFIG_PCI_CONSOLE=y
> CONFIG_DUMMY_CONSOLE=y
> # CONFIG_FB_PM2 is not set
> # CONFIG_FB_TGA is not set
> # CONFIG_FB_MATROX is not set
> CONFIG_FB_ATY=y
> # CONFIG_FB_VIRTUAL is not set
> # CONFIG_FBCON_ADVANCED is not set
> CONFIG_FBCON_CFB8=y
> CONFIG_FBCON_CFB16=y
> CONFIG_FBCON_CFB24=y
> CONFIG_FBCON_CFB32=y
> # CONFIG_FBCON_FONTWIDTH8_ONLY is not set
> # CONFIG_FBCON_FONTS is not set
> CONFIG_FONT_8x8=y
> CONFIG_FONT_8x16=y

You'va an ATI VGA card, and want to run FB console? It may be broken;
I just fixed some FB stuff in 2.2.17.pre12, though I think it was only
for Rage 128 and Permedia 2.

Does the kernel switch over to FB console from VGA console before the
crash? If not, try leaving that option out and try again.
