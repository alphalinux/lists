On Thu, 20 Jul 2000, Mike Barnes wrote:

> Now correct me if I'm wrong, please - but as I understand it, the 3000
> is not much more than a Decstation with an Alpha processor instead of a
> MIPS. Same basic hardware, right?

 Not completely, but there are similar to I/O ASIC based DECstations, like
DS 5000/240. DEC 3000 AXP is documented well, fortunately.

> So am I dreaming when I think it might be possible to drag the
> Turbochannel drivers and other support out of the Decstation patches and
> compile for Alpha? Is anyone working on this? I've been crawling over
> the web and found the occasional discontinued project, but nothing
> current.

 Support for TC devices should work for DEC 3000 out of the box as drivers
are platform-independent. Glue code is for the core chipset is missing,
though and it needs to be updated and/or fixed. Also certain onboard
devices, like the Fast SCSI controller might not work. Craig P Prescott
<prescott@phys.ufl.edu> is working on DEC 3000 -- you may consider
contacting him.

  Maciej
