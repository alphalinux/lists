I have an alpha LX164 with a Matrox Mill II 8 Meg board. Since
upgrading to Red Hat 6.2 and Gnome X is freezing so often it is
almost unusable. The kernel seems OK as I can telnet to it and my
cron job is getting my mail while X is frozen.
This has happened at various points:

   1. While a screen saver is running (I have random selected)
   2. At logout (various points)
   3. After typing exit in an nxterm. The xterm was still visible.
   4. Clicking on a button on the control center->themes screen
   5. Clicking on a button on the control center->screensaver button.
   6. Just a freeze during normal activity.

All of the above leads me to believe that the server is at fault. I
didn't have *any* of this happen running the updated XFree binaries
for 5.2.

So, the question is--what should I do? Is this a buggy fbdev? Can I
comment out the section containing fbdev in my XF86Config file?

Any advice greatfully accepted.
