On Sat, 22 Jul 2000, Richard June wrote:

> has anyone successfully gotten the Voodoo3 2000 PCI to work in an
> Alpha? I've got an XL 366 running rh6.2 (I know it isn't supported
> hardware), it(Xconfigurator) recognized the card and everything seemed to
> be cool until I actually tried to start X then the screen went black, I
> could hear my monitor switching video modes, but X never came up proper. I
> tried X 4.0 as well, however the drivers don't compile.
>

I think your modelines might be incorrect. I have never seen Xconfigurator
work well with modern video cards and good monitors. (It happens in PC's
too, i haven't done an in-depth analisis of the problem but methinks
Xconfigurator overrates some monitors, or some info in the monitors
database is plain wrong)

i would ...

1) try xf86config . I know the interface sucks , but it ussualy builds
a handy XF86.conf that is a good starting point.

2) then edit the file manually (it is in /etc/X11/XF86config)

Have the handbook of your monitor by hand when you do this.

E mail me directly if you need help with the modelines
