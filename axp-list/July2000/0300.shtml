We've been seeing thread problems as well here at LLNL. None of us
have had time to look into it more. We're working on a parallel
file system and when we ran with pthreads, it would cause the server to
hang shortly after starting (it uses a work pile for handling
requests). Some researchers over at Compaq's SRC took the pthread
code and strip a bunch of code out of it to make a thread library that
wouldn't hang. I was told that they sent a message to Xavier Leroy
about the problem, but I don't know for sure that anything was done
about it. From what I understand, there are missing memory barriers
in the locking routines. I've seen condition variables fall through
when another thread has already grabbed the mutex (more than one
thread holding a mutex at once). They seemed to think the problem
existed on Intel as well but it seems to be a little harder to
trigger.
