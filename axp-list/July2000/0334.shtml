I asked:

> Are there any FCAL PCI cards supported on Alpha Linux with IP as well
> as SCSI protocols?

Thanks to the following for their responses:

-------- Matthew Jacob <mjacob@feral.com>

>Interphase- don't know if works on alphan

-------- Greg Lindahl <glindahl@hpti.com>

>We're using the Qlogic driver at FSL on Linux Alpha. I'm not sure if it's
>the UNM one or the Qlogic one. It's supposed to have IP support, I haven't
>used it.

Is that University of New Mexico? They have a driver for Qlogic FCAL
HBAs? Would the IP feature be difficult to test?

-------- Peter Rival <frival@zk3.dec.com>

>> I am currently running EMULEX LPFC 8000 on Solaris,
>> which does have both protocols, but they do not currently have Linux
>> support. They plan to release a driver for Linux/PC soom, but said
>> nothing about Linux/Alpha.
>
>No, actually their press release specifically states Alpha as well as
>Intel. At least, it had better...I've got a GS160 full of Emulex adapters
>just waiting for support...

I have not gotten any firm dates on when EMULEX will be providing this.
Do you know for sure that it will happen? Hope you don't have to wait
too long to get your GS160 working.

>> The other possibility is to adapt the
>> Qlogic driver code to Linux/Alpha. (Anyone done that yet?)
>
>That's used all the time by our CSS (I think they're still called that
>now...) folks here. We even have an orderable part number for it. Dunno
>about IP over FC support in that driver tho'...

It looks like the Qlogic QLA2300 supports IP as well as VIA, with 2 Gb/s
bandwidth capacity.

The reason I ask about IP capability is that this would allow me to
have a very fast network over the same medium that's carrying my disk
traffic. Is that bad? It depends on how my applications use data from
disk and pass data among each other. VIA support makes this more
interesting, as this gives the ability to have highly efficient data
exchanges among systems.
