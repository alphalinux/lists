Olle Hansson wrote:
>
> I have run AlphaBIOS and Milo before, and now have i changed my SCSI card to a
> NCRxxx so i can run SRM.
>
> And to get aboot boot from a harddrive must i partition my harddrive
> whith BSD disklabel ?

Yes for a device which will be read by SRM. If you have other disks
they may be partitioned one way or another.

> Must i delete all my current partitions and remake them whith BSD disklabel ?

Not really. But if you keep your current partitions then you have to
 - leave a space on a disk start where a bootblock will reside (unless
   you are booting, say, from a floppy); this is easy if you had there
   before a small FAT partition which was used by milo and which can
   be "recycled"
 - you have to write a disklabel with *exactly* the same partition
   boundaries on a *sector* level; if you have extended partitions they
   will be, obviously, gone and eight is a maximum number of partitions
   allowed by BSD disklabel.

Writing a "bad" partition table, in whatever style, does not change
anything in your data so, in this sense, it is a safe operation.
If you will mess up something then you can try it again but boot
first from a floppy, or CD, and make careful notes. Even better
save the original FAT partition table on a floppy so it can be
restored in an emergency. 'sfdisk' will do that for you.

Remeber that if you will "drop" an old FAT partition, and/or some
extended partitions were deleted, then your partition numbers will shift
so your fstab will require small edits.
