Michael A. Smith wrote:
>
> You can't share *.so files between progs compiled on different compilers,

Oh? Is this really the case? Can you explain details? If this is so
then we cannot share Compaq math libraries between ccc and gcc and this
flies in the face of an everyday experience. Incompatibilities between
g++ and cxx come from something else.

Regardless - these are other tools, and not a compiler, responsible
for collecting object files into a shared library. The fact that a
front end provides a unified interface for doing so can be easily
"unmasked" if you will use '-v' flag and look close what really
happens.

> if you have a suite of apps that you compile together (e.g. PHP & Apache),
> *both* need to get compiled with CCC to work together.

I did not try these particular examples but I would not bet on it
unless C++ jumps somewhere in the picture.

> I'm not a guru on kernel compiling. After reading the article, I wondered if
> the kernel could be compiled with CCC...

No, it cannot. Linux kernel is using many different constructs,
starting from an inline assembler, which are gcc specific so it is not
written in standard C. Kernels have to work in places very close to a
"bare metal" and they are not "everyday programs".

Don't try to learn compilation specifics, or running benchmarks, from
that article you mention. You will be disappointed.

  Michal
