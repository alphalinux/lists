Hi,

I discovered a very strange problem using shared libs that seems to be
specific to AlphaLinux.

We are building an application that uses a set of shared libs.
When quiting the application I get a core dump with the following
backtrace.
...
Program received signal SIGSEGV, Segmentation fault.
0x0 in ?? ()
(gdb) bt
#0 0x0 in ?? ()

I discovered that one specific shared lib is reponsible for this core
dump. When building this library as a static lib ( archive ) instead
everthing works fine.

I first thought it has something to do with global variable
initialisation or any other obvoius problems specific to shared libs but
the exact same shared lib works on other platforms like IntelLinux and
Solaris.

Environment:
binutils-2.9.5.0.22
gcc-2.95.3-0.20000323
glibc-2.1.3-14

I think it has something to do with the size of the shared lib because
with smaller shared libs ( size of the *.so file ). I never had such
problems.

Is this behaviour/bug known ?

I read about linker problems specific on AlphaLinux in this newsgroup a time ago.
Can this problem have similiar reasons ?

The next thing I will try is to split the relevant shared lib into
smaller parts and see if that helps.

Any other help, ideas or feedback is welcome

Regards

Max
