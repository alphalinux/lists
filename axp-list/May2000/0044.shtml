Blake Carlson <blake-carlson@uiowa.edu> writes:

> I had a bit of scientific code handed down to me which I've been trying
> to understand and optimize for the last few weeks. With GCC (and the
> best combination of flags I could find) the runtime was about 40 minutes.
> With CXX and the '-fast' flag, the runtime was reduced to 3 minutes on the
> same machine (DP264) yielding nearly identical results!

> So, I got lucky and CXX picked up something GCC missed. In some of our
> other well written programs were not seeing significant performance
> differences.

A recent Slashdot article is about a review of the Compaq Alpha DS-10
running Linux. The review includes several different benchmarks that
had some very surprising results when run compiled with GCC vs. the
Compaq C compiler available. At the time of the review, GCC was NOT
optimizing code well to run on the Alpha chip; it did not take
advantage of the plethora of registers available, for example. The
Compaq compiler *did* take advantage of all available registers
yielding much better run times in some cases.

The review is on aceshardware.com
