Aric Coady wrote:
> I want to load a third-party .so file that is in ecoff format on alpha-linux.
> It was built on an alpha-dec. I have access to both platforms; so I can
> successfully build another executable that uses it, and run the executable on
> alpha-linux. But what I really want is to load it explicitly.
>
> So, /sbin/loader works on ecoff, but dlopen() doesn't. It just returns a
> non-ELF format error. Can dlopen be patched to open ecoff? Is there an
> ecoff->elf file converter for linux? Any other options?

I don't believe it is possible to mix ELF and ECOFF objects into one
process image. For one thing, they must link with different libc
implementations. A .so isn't really any easier than a .o in this
regard.

Perhaps you could build some RPC wrapper around the ECOFF .so in Tru64
and execute it under Linux. There probably isn't an easy solution.
