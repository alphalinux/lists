Have you been able to use the modem for any connections? If not...

You may need to use the "setserial" command to gain access to the modem.

            example: setserial /dev/ttyS3 uart 16550a irq 10 spd_vhi
where /dev/ttyS* is the serial port where /dev/modem is linked.
All the modems I've used have had jumpers for port & irq settings and
have been ISA cards.

            setserial /dev/modem
shows current state of the serial port.

You can also try "dip -t" to manually check the modem/serial port. From
the DIP> prompt, try using the "port /dev/ttyS*" and "reset" each
followed by <enter> This should hangup/reset the modem. "quit" and
"help" are also useful.

Once the modem is dailing and connecting, you'll have to work out the
ppp specific issues. At least thats been my experience.
