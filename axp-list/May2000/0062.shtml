Carl Aron (carl@maiatech.com) wrote:

> What I hope for:
> Hardware: at least 256 MB RAM, at least two drives, preferably SCSI, with
> software mirroring (RAID?), so that I can just swap drives in the event
> of a failure. Tape backup, too. CPU fast enough to stand up to multiple
> domains using Perl, PHP, and SQL.
>
> Software I want to run: Apache web server and Redhat or other SSL web
> server, Perl, PHP, MySQL, PostgreSQL, POP, IMAP, Sendmail or similar,
> FTP, Secure Shell, PGP. (ASP if I can become convinced it won't choke my
> machine or open security holes).

All of the above sounds fine to me. It's all pretty generic (non-Alpha)
questions. Don't run ASP.. unless you have to. It has more security
holes then I have fingers and toes. ;-)

> DEC4000 model 610 -- Big monster with tons of RAM, 6 SCSI drives and bays
> for 10 more drives, DAT and DLT tapes. Attractive because it is very
> complete with lot's of room to grow, but possibly an ancient model,
> either slow or or not usable with linux?

These do not currently run Linux. They will run NetBSD (to some degree).
These systems are the oldest Alphas and do not use PCI or ISA bus.
Basically they are very old and slow, but still hard working machines.

> Alpha Server 2000 P233 -- A complete system, with drives and tape backup,
> but I don't know how fast it is or if it will boot linux.

If it is a AS2000 233, it is 233MHz. These will run Linux, but it is
pretty old. It would probably be about the same as a 200MHz Pentium at
Integer and about a 300MHz Pentium at Floating point.

> Alpha 21164SX 533 MHz Server -- A nice complete system, but has a single
> EIDE drive rather than multiple SCSI. Can SCSI be added for a reasonable
> price?

The 164SX (uses 21164PC processor) is a stripped version of the standard
21164. While a desent system it has little room to grow. More ideal for
a personal workstation.

> AlphaServer 3300R -- This looks like a real nice one. No Hard drives and
> not enough RAM, but that can be fixed.

The "R" at the end of the model number means it is the rackmount model.
Out of the systems you listed I would probably go with this one. I must
point out that this is "white box" system, which was originally meant to
run NT. There have been a few people recently asking how to install Linux
on these. Try checking the list archives (available from alphalinux.org)
to make sure it is fully Linux compatible before buying.

Hope that helps.
