Hi,

I am trying to compile the ALSA (Adcanced Linux Sound Architecture)
code (Version 0.5.7) and get a few warnings like:

mixer.c:1128: warning: passing arg 1 of `swpipl' makes integer from pointer without a cast

rme9652.c:347: warning: cast from pointer to integer of different size

The latter (which I would like very much to work), is due to this code
fragment:

{
        u32 pb;
        u32 cb;

        cb = snd_malloc_pages (RME9652_DMA_AREA_BYTES(rme9652), &pg, 1);
        pb = snd_malloc_pages (RME9652_DMA_AREA_BYTES(rme9652), &pg, 1);

[ ...]
}

and so on. If I replace u32 for u64, the warnings get eliminated as
one would expect from the comments in the alpha-linux FAQ. Now, what
is the preferred canonical way or The Right Thing to code this in a
portable way, that means a portable int <-> void* cast ?

Another thing, the drivers use autoconf / configure and it is
necessary to add the -mcpu=ev6 flag to the gcc options. What has to be
changed there correctly ?

many thanks in advance,

Johannes
