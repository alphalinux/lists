>>> "warp@xs4all.nl" said:
>
> I get shiploads of 'CIA recoverable ECC error on 0x630',
> even when doing the most simple things, like 'ls'.

You didn't say, but this is a PC164? Doesn't really matter, the following
are true for all platforms:

Code Reason Example or Common Cause
==== ====== =======================

620 System Correctable correctable errors in the memory subsystem

630 Processor Correctable correctable cache and TLB errors

660 System Uncorrectable unrecoverable memory errors

670 Processor Uncorrectable unrecoverable cache or TLB errors, or
                                  read of a non-existent I/O space location

> Everything does continue to work but after a while the
> whole machine dies, continuously printing 'CIA unrecover-
> able ECC error on 0X21E'... this varies from 2 minutes
> (heavy network load) to about an hour or 2 (system idle).

Well, 630 is correctable (single bit ECC error), but then it becomes
uncorrectable (double bit ECC error). Bad news... :-\

> This box was running for about 2.5+ years continously
> (till yesterday), no changes whatsoever... I just found
> it in this state when i turned on my monitor :(
>
> It doesn't seem to be bad RAM, running diagnostics
> from AlphaBIOS passes all tests just fine and so do
> all other systemtests (trough CPU module jumpers).

AlphaBIOS turns off machine checks, so it won't see the problem... ;-}

> Can this behaviour be caused by a broken DEChip 21171
> chipset (which is on Alpha XLT's CPU module)?

Not a core logic problem; it's most likely either:

1. bad memory (try reseating the SIMMs, or try running with less memory to
   try to narrow down the culprit).

2. a power supply gone marginal/bad can produce the same symproms.
