Michael Tibor wrote:
>
> I would probably do something like this:
>
> mke2fs -m1 -b 4096 /dev/<superhugedisk>
>
> That only reserves 1% instead of 5% space for the superuser,

Watch out. This space is reserved not only for root but also
is needed during "dynamic optimization". So with a "normal usage"
I would not drop that percentage. This space is not wasted anyway
and, if really required, you can change that with tune2fs anyway.

> and uses 4K
> blocks (very important for a single filesystem that large if you don't
> want to waste half the day when you do a fsck).

Block sizes also affect a maximum partition size although I do not
have numbers handy.

A while ago somebody reported on linux-kernel list that he created
ext2fs partition of something like 800 GB in size (on two RAID farms put
together).
