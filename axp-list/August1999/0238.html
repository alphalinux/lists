There are a couple reasons, all related to the allocation of memory and
virtual-to-physical mapping. It essentially boils down to T64 using some page
colouring schemes along with some tweaks and Linux doesn't. This can cause a
couple different problem: cache thrashing,

 possible cache aliasing(in the 21264 it could be possible to have alias in the
l1 D-cache because to virtual bits are used in the index. To get around this,
the 21264 only allows 1 cache line of a possible aliased set of cache lines
into the D-cache), tlb issues, etc.

AFAIR, these issues have been brought up with linus, but he wasn't willing to
put in the page coloring. I don't know if Phil Ezolt is on this list, but he
would probably have more insight into the issue.
