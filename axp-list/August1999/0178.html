You don't need the milo disk if you use SRM to boot.
First use generic.img, then use ramdisk.img when prompted.
If you still get kernel panic, try downloading the updated ramdisk
image from
ftp.cc.gatech.edu/pub/Linux/distributions/redhat/updates/6.0/images/alpha
If it still doesn't work, you can also try specify CD-ROM as root.

boot dva0 -fi vmlinux.gz -fl "root=/dev/scd0 load_ramdisk=1".

You may come to my website for more information about alpha Linux
installation.
