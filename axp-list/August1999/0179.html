On Mon, 9 Aug 1999, David Reitz wrote:

>
> hello everyone,
>
> i have a multia with redhat 6.0 on it, and i would like to know if it is
> possible to run it w/o a monitor or keyboard? if so, how do i go about
> doing this?

        If you do not need access to the console (i.e. Linux already
installed and boots automatically), then just pull the monitor and
keyboard cables, place it where you want, connect power and network (will
be quite useless without at least these two), and turn it on. After a few
minutes it should appear on the network.
        Now, if you want access to the linux console, the 2.2.x kernels
have support for console on a serial port, activated via a kernel
parameter (i.e. from milo). Don't know specifics of this (if you get a
shell there or not or one needs to set up a getty there). This is also
possible with earlier kernels, but not quite so straight forward or clean.
        The trickest part is getting access to SRM and ARC w/o
monitor+keyboard. It is possible, as when the system boots, it will detect
that there is not a keyboard and do I/O through the first serial port (std
pins on the 25pin serial connector). As far as I know the srm console is
completely accessible this way. The ARC on the other hand, while it will
display its initial menu, can not be interfaced with as arrow keys from
about any terminal emulation I have tried are not recongized correctly. In
other words, you can see the ARC loader, and if set to auto-boot, it will
boot, but you can not choose any menu options other than the default one.
:(
        This is really bad in my case where my keyboard and monitor ports
are toast (my UDB has seen too many battles), and only the serial port
works, and the CMOS battery died, so the NVRAM is invalid, and I can't
boot the system at all. :(
        Anyway, hope this clears things up for you. If you find a way to
interact with ARC via a serial console, I would be very interested to hear
how you did it! TTYL.
