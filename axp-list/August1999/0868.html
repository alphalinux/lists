>
> As a temporary solution set the FORGOT_THE_NAME_SEE_MANPAGE environment
> variable to have cdrecord use less than 2Mb of this stuff.

Why "temporary"? First of all 4000k, as opposed to a default 4M, works
just dandy and a difference in buffers size is insignificant. Calling
cdrecord via a script wrapper which sets an environment and execs "the
real thing" is a total no-brainer. Works with things like xcdroast as
well. Or you can use a flag provided by cdrecord expresly for that
purpose and clearly documented.

Hacking SHM_MAX with every kernel version you want to experiment
with is a major PITA and totally superflous. If something of that
sort I would rather modify cdrecord sources to drop its default a bit. :-)
It can be actually higher than 4000k without bumping into a kernel
default limits on alpha. OTOH 4000k is such easy number and I do not
remember precise values.
