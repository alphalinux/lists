| 1. Obvious problem: Crashing font server. But symptoms seem to indicate
| more extensive problems. I'll try Peter Petrakis' suggestion (20May 1999)
| and put the font path directly into the XF86config file.

Works well that way. The main problem I had was the xfs would cause X's cpu
utillization go to 100% when a motif app called for fonts. That and random
lockups and crashes were all due to the fontserver.

| Right now, only AfterStep will not run at all. startx fails.
| xdm logins fail. Only gdm and kdm logins work and are moderately
| stable. I have never gotten xterm to work. I prefer using startx
| on this machine.

There are a multitude of updates for redhat 6.0 already. They include fixes
to Xfree, the xfs (which still does not fix my problem), xterm, rxvt, ping,
xscreensaver, and the list goes on. Do not install the dev update or be
prepared to fix the permissions on all your devices. Xscreensaver still does
not function properly after the update due to an X authourity problem. To
fix it insert the line "xhost +localhost" at the end of your
/usr/X11R6/lib/X11/xdm/Xsetup_0 . Since I disabled the font server and did
all the updates I haven't had a problem with X since. I also use
gdm/enlightenment.
