On Mon, Aug 16, 1999 at 03:24:38AM -0400, Craig P Prescott wrote:
> #define tbia() \
> ({ \
> register unsigned long __r16 __asm__("$16") = -2; \
> __asm__ __volatile__( \
> "call_pal 0x0033" \
> : \
> : \
> :"$1", "$16", "$17", "$22", "$23", "$24", "$25"); \

This doesn't make sense; __r16 is no longer used by the asm,
so the assignment should be optimized away, so $16 probably
now contains garbage.

> 2) why would the original tbia() fail on a DEC 3000 but work on
> everything else?

I have no idea. It doesn't make any sense for the PALcall
to fault at all.

> __tbi is generalized beyond the immediate needs of
> tbia() and tbiap(); could this be causing some problem?

I very much doubt it.

> P.S. This is a bit off-topic, but if someone could spell out in full detail
> __tbi above, I'd be grateful, as I could learn more from this
> example. I.e. what do the #__tbi and ##arg1 do?

Standard argument substitution from the macro -- # is stringification,
## is concatenation.

> And I learned from a
> gcc doc what the "=r" and "0" mean, but what about "i"?

Immediate.

> If one wanted to learn a bit of Alpha Assembly -
> with no prior knowledge of any assembly language - what would one do?
