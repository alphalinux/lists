Hi,

I'm wondering what might cause this, or if anyone else
has seen it. After running 2.2.11 for a couple of
days, I noticed that X kept grinding away and my CPU
utilization by X kept going up. (SVGA server 3.3.4) As
of yesterday, X was taking about 11% CPU when I was
doing nothing. (just top)

The odd thing was that if I started Netscape (and only
netscape)... and waited until Netscape finish loading
and stop doing everything, X cpu utilization goes back
down to 4% which is how it use to behave (2.2.7). If I
close Netscape and wait... X goes back up to 11%.

Last night, I went back to 2.2.7 and this behavior was
gone. Just very odd....

The machine is a PWS500a,Matrox Mill II... and 2.2.11
was compiled with gcc-2.95.

Anyone have a clue?
