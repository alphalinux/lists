The problems are still there. With a stock gdb 4.17 and egcs you will
get "strange" results using "next" and "step" commands. Strange means
that e.g. next runs till the end.

The problem is that egcs generates incorrect information about function
prologues. This problem has been dicussed on the gdb mailing list.

I use gdb 4.17.0.11 + a special patch (more a hack than a patch :-) )
for using gdb together with egcs (and gcc-2.95).

If needed i can provide the patch.

BTW gdb 4.18 does not work either out of the tgz (e.g. when calling a
function it crashes)

Another problem is reading core dumps generated with 2.2.x kernels. With
a stock 4.17 it is not possible. Some patches where communicated on the
gdb mailing list but i never tried one of them,
