libawt from sun requires a motif library. If u dont need libawt.so , then
there is no need for motif's libXm.so.
So whether motif comes from commercial sources ( ie $$$ ) or from lesstif, is
up to u.

I think what may be confusing u is that I use libXm.so to build libawt.so (
rather than the static libXm.a ). But because libXm.a is static, u cannot fix
any of the (lesstif) bugs without getting a whole new jdk. If there is a fix
for Motif, then for the shared libs, all one has to do is replace the libXm.so.

gat.

BTW I usually get the latest ( current ) daily snapshot from www.lesstif.org to
keep up, and see what bugs have been introduced, or more probably fixed.
