>
> I was considering what I thought was called 'striping', where the data
> is split across two drives, but not the redundancy, somewhat like the
> RAM interleaving to widen the data path. I am not sure which RAID this
> is called.

This is RAID-0. See /usr/doc/HOWTO/mini/Software-RAID. On your
disk you should also have /usr/doc/HOWTO/mini/DPT-Hardware-RAID
and /usr/doc/HOWTO/Root-RAID-HOWTO. The first document provides
also this list:

       <*> Generic RAID overview:
          <http://www.dpt.com/uraiddoc.html>.

       <*> General Linux RAID options:
          <http://linas.org/linux/raid.html>.

       <*> Latest version of this document:
          <http://linas.org/linux/Software-RAID/Software-RAID.html>.

       <*> Linux-RAID mailing list archive:
          <http://www.linuxhq.com/lnxlists/>.

       <*> Linux Software RAID Home Page:
          <http://luthien.nuclecu.unam.mx/~miguel/raid>.

       <*> Linux Software RAID tools:
          <ftp://linux.kernel.org/pub/linux/daemons/raid/>.

       <*> How to setting up linear/stripped Software RAID:
          <http://www.ssc.com/lg/issue17/raid.html>.

       <*> Bootable RAID mini-HOWTO:
          <ftp://ftp.bizsystems.com/pub/raid/bootable-raid>.

       <*> Root RAID HOWTO:
          <ftp://ftp.bizsystems.com/pub/raid/Root-RAID-HOWTO>.

       <*> Linux RAID-Geschichten:
          <http://www.infodrom.north.de/~joey/Linux/raid/>.

Probably not very up-to-date (www.linuxhq.com - for example).

If you do striping you better keep excellent backups. Your chances of a
hardware failure are much higher and hopes of recovering **anything**
from a failed striped pair non-existent.

If you start with a software RAID now you better use 0.90 - which is
NOT included in "standard" 2.2 kernels. This require a fitting set
of raid tools.
