On Tue, 31 Aug 1999, Jaap Schellekens wrote:

> If i remember correctly, you must increase SHMMAX in
> include/asm-alpha/shmparam.h
> to a more correct value. It's written in some readme of cdrecord.
  
  As a temporary solution set the FORGOT_THE_NAME_SEE_MANPAGE environment
  variable to have cdrecord use less than 2Mb of this stuff.

shmmax is run-time configurable on newer kernels, eg:

echo 33554432 > /proc/sys/kernel/shmmax
  
  Jaap

And a large buffer is a good safeguard against dud cdr's. So it's
definitely worth setting it.
