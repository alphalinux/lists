> Hi
>
> I compiled linux-2.2.1-ac3. While running a
> program of mine (which uses lots of cpu-time
> and also high nfs-usage) I got the following
> message once:
>
> > kmem_alloc: Bad poison (name=size-256)
>
> while i was greping trough the kernel-sources
> (and found the msg in mm/slab.c) the computer
> just froze. Only reset could start it again.
>
> What does the msg mean?

Alan Cox made a slab poisoner that would automatically check memory
allocation/deallocation (from the slab allocator) by marking pages.
It found a number of device drivers were being naughty with freed
pages. Could it be you're getting bitten by this?

>

> it never worked stable.

Not necessarily. I had my alpha very stable till 2.2.6, then it
segfaulted like crazy. Yesterday I put 2.2.11 on it and its just done
the 24 hour kernel compile tortue test with no segfaults. Try 2.2.11,
I say.
