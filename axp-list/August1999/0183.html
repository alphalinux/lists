problem. But it seems that there is something wrong with the website
recently(?).
another reference is SRM HOWTO!!!

> I'm trying to put RH6.0 on a Digital AlphaServer 400 4/166 (Avanti).
>I had created 3 disks:
> - a MILO image (from /milo/images/avanti.img)
> - a GENERIC disk (from /images/generic.img)
> - a RAMDISK disk (from /images/ramdisk.img)
>
> I can't understand what to do with it! I'm trying to boot the install
>software from the MILO disk with the command via SRM:

The MILO use ARC mode ..,

>
>>>boot dva0 -file vmlinux.gz -flags "root=/dev/fd0 loadramdisk=1"
>

set boot_file vmlinux.gz
set bootdef_dev dva0
set boot_osflags "root=/dev/fd0 load_ramdisk=1 prompt_ramdisk=1
bootdevice=/dev/fd0"
--insert boot-imag
boot dva0

>but it didn't work. It work on the _generic_ floppy, but it stops asking
>for the _root disk_ and if I put any one of them I get a Kernel Panic
>message! There is some docs where I can get this thing up? I had already
>installed Linux on countless Intels, and some Sparc machines. Can someone
>point me the right direction?
