On Tue, 31 Aug 1999, Michal Jaegermann wrote:

> B. James Phillippe wrote:
> >
> > Anyone have any advice for a (free) very reliable, compress-capable
> > text-mode tape backup program that runs on RH6/AXP?
>
> afio with a -Z option compresses on a file-by-file bases which is much
> better (reliability) than trying to compress the whole archive on a
> tape; the later is a crapshot proposition does not matter what serves
> as a base.

Right.

> If you want afio repackaged in .src.rpm format, to be recompiled with
> 'rpm --rebuild ...', this can be arranged. I have version 2.4.4 on
> hands.

Hmmm, it's been a few years since I used afio. When I used it last (on
Linux/x86), I had bad experiences with it. The multi-volume code was
broken and I had to fix it (and several other glaring bugs) before I could
consider it stable. If the package is still being maintained, I might give
it a shot again and hope those bugs are all washed out.

I would consider purchasing a backup program (for Linux/AXP) if it promised
high reliability.
