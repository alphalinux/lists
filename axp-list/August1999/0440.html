On Wed, Aug 18, 1999 at 11:29:12AM -0400, Peter Howell Jr wrote:
>
> libm.so.6.1(GLIBC_2.0) is needed by umb-scheme-3.2-9
>
> Ok. Not to confusing. It looks like I need to update glibc. I ran:
>
> rpm -ivh glibc-2.0.7-29.alpha.rpm

 
> and get :
>
> package glibc-2.0.7-29 is already installed
> error: glibc-2.0.7-29.alpha.rpm cannot be installed

This is because this is the package you already have. You need a
newer one for the upgrade to do anything.
 
>
> I then tried the -U, instead of the -i switch, in the vain hope it
> was that simple, but I got the same response.

Well, I always use U, because that installs if the package isn't
there, and upgades if an earlier version is.

>
> Am I looking in the wrong place for GLIBC_2.1 and GLIBC_2.0?
>
> How do I figure out what files glibc-2.0.7-29.alpha.rpm provides? I think
> I'm having trouble getting the syntax right.

You do rpm -qpl glibc-2.0.7-29.alpha.rpm
you would do rpm -ql glibc to get the same info for a package that has
been installed already. Do rpm -qi glibc to get general info about
the glibc package (or rpm -qpi for the one you haven't installed yet)

Having said that, a note of caution. Upgrading your libc is somewhat
tricky and not for the faint of heart. SImply upgrading the package
could break all finds of things. If you need glibc 2.1, you may be
better off upgrading your box to a distro that uses it (eg RH 6.0).
This would also include all the gnome packages too.

Otherwise, find versions of the packages that were built for glibc 2.0.
