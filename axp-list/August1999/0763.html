Ric Tibbetts wrote:
>
> Want an easy way?
>
> rpm -qa | grep amd

With only this catch that the package recently was renamed to am-utils
so this may not work.

> When it lists the amd pkg...

rpm -q amd

will do the same if 'amd' is really its name.

This one will work does not matter what is the current name:

  rpm -qf /etc/rc.d/init.d/amd

Run 'rpm -qif ....' if you want a short description what all of this
is about and 'rpm -qivlf ...' to find the whole contents and basic
information about a package to which belongs a given file.

>
> rpm -e <amd pkg>
> It'll rempove itself from all the run-levels.
>
> then: ps -ef | grep amd (to get the PID)
> kill <pid> (to kill the daemon.
>
> there's a longer harder way to do this..

/etc/rc.d/init.d/amd stop

**before** removal strikes me as a simpler, shorter and more reliable
way which will not leave some dead lock files if they were indeed
created. There is also 'pidof' utility if you are after pid of
something.
