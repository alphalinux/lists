> > Actually, it's possible right now. Create linker script with
> > `ld --verbose', edit it like this
> > - . = 0x120000000 + SIZEOF_HEADERS;
> > + . = 0x10000 + SIZEOF_HEADERS;
> > and link with it. Then you have to set 1 (EF_ALPHA_32BIT)
> > at file offset 0x30 (Elf64_Ehdr.e_flags) in the executable.
> > Any 2.2.x kernel supports this flag.
> >
> > (Thanks to Richard Henderson who pointed that out at the gcc list)
>
> This is great news! Can we:

Have you ever tried to build Netscape 4.X on a new platform? (or even an old
one?). Monumental pain...

Feel free to ask, but I'd guess you'll be told: help with the
mozilla effort on alpha Linux (btw, various mozilla versions do in fact
run on Alpha Linux, last I checked)... Mozilla is at least something
mere mortals can deal with, from what I've seen (I've tried both early
mozilla bits, which were mostly Netscape 4.x bits, and later mozilla bits,
with new toolkit and build stuff in it, and it is night and day; they've
gone from essentially unbuildable unless you are a fulltime Netscape
employee to something mere mortals have a chance of building and getting
running).

This is where work should go in my humble opinion, as we have other ways
of running the DU netscape on Alpha Linux already. (and yes, there is
one person in Compaq who has already been working on mozilla on alpha linux:
but more help and testers are clearly a good idea).

This is not to say that -taso isn't useful: would be nice to get ld to
do it without such obscure scripts for other apps, but mozilla isn't one
of those apps which need it....
