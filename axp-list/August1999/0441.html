On Wed, Aug 18, 1999 at 10:23:49PM +0100, Sean Hunter wrote:
> The trouble is, you need to use setsysinfo to get your program to do
> this, which is fine for stuff you're writing, but a pain for other
> programs. What I propose is to set up a sysctl so that it will
> automatically set this behaviour as desired. This means that normally
> you just leave this as is, bit if you feel like doing a spot o'
> debugging you can echo "some_magic" >> /proc/sys/kernel/uac_control...

The DU syscall allows to set the system default too --

#define SSIN_UACSYS 4 /* set system printing on/off */
#define SSIN_UACPARNT 5 /* set parent proc on/off */
#define SSIN_UACPROC 6 /* set current proc on/off */

I only implemented the SSIN_UACPROC case. If you'd like to
implement the rest, be my guest. ;-)

I don't remember at the moment, but I believe that the value is
(or should be) inherited across exec. (Guessed from SSIN_UACPARNT --
you run a program which changes your shell's default, then run
another program which inherits the value.)
