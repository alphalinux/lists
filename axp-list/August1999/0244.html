Background Information
----------------------
I'm trying to install Linux (RedHat 6.0) on a AlphaStation 200
which doesn't have a floppy disk drive (see my previous posting).
I have successfully upgraded the firmware to version 7.0-9 using
as200_v7_0.exe from the Digital site and using bootp. I have also
managed to get MILO to load using bootp. The version of MILO
I am using is V2.0.35-c5.5. I took the hard disk out and mounted
it in a x86 linux box and formatted the disk with an ext2 partition.
I then copied the alpha install files onto the disk and then put
the disk back in the Alpha! (We only have the one Alpha box.)

The Problem
-----------
I have tried a number of kernel's and am having a similar problem
with two of them - namely xl.gz (off the DEC site) and generic.gz
(from the redhat distribution).

I have tried boot commands (from MILO) like:

boot sda1:/kernels/generic.gz "root=/dev/sda1"

and

boot sda1:/kernels/xl.gz "root=/dev/sda1 init=/bin/sh"

The boot procedes fairly normally - it finds all the scsi devices
and the network card but then gives the following error:

Warning: unable to open an initial console.
Kernel panic: No init found. Try passing init= option to kernel.

I have tried using init=/bin/sh, init=/bin/init, init=/bin/install
(all these files exist on the file system), but I get the same
error in each case.

One hint is that as the kernel is booting the following message
appears: "tga_console_init: TGA not found!! :-("
Is this relevant?

The avanti-s.gz kernel goes into an infinite loop which prints the
message "device_interrupt: unexpected interrupt 0" so I gave up
on this kernel.

Does anyone have any suggestions??? Two and a half days to get
here... I can't give up now!!!!

Andrew.
