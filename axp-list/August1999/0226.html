Todd,

You can't expect same result from Tru64/Fortran to Linux, even Fortran
for Linux.
OS is still Linux and not Tru64 where the difference is few hundred dollars.

This doesn't answer your question, but please also take a look at our benchmarks.
http://www.hpc.co.jp/Bench/CPUbench2.htm

Todd Henderson wrote:
>
> We are currently building a system from XP1000's and have been
> benchmarking our code and playing around with the new Compaq fortran
> compiler for Linux. It turned out that the beta Linux compiler produced
> a slower code than the tru64 compiler did on tru64. Well, since it is
> in beta this seemed reasonable. So we tried running the tru64 compiled
> code on the Linux machine since they are compatible. I expected the
> performance under Linux with the tru64 compiled code to be about the
> same as the tru64 compiled code on a tru64 machine. Turns out, we
> didn't get that. Does anyone have any ideas why an executable moved
> from the DEC tru64 machine to a Linux machine would be slower? Is there
> some overhead in Linux or something? Here are some of the numbers for
> the CFD code:
>
> (OS/Compiler used) time
> tru64/tru64 27.5 minutes
> Linux/tru64 34.6 minutes
> Linux/Linux 34.4 minutes
