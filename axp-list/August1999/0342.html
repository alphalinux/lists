On Fri, 13 Aug 1999, Bill Broadley wrote:

> >Anyone out there have a similiar setup? Exactly which boot/root
> >did you use. All my attempts end with:
> > halted CPU 0
>
> to be more precise, following exactly the docs compaq has
> for linux on the DS-10 I get:
> initializing HWRPB at 2000
> initializing page table at 1ff900000
> initializing machine state
> setting affinity to the primary cpu
> jumping to bootstrap code
> aboot: Linux/Alpha SRM bootloader version 0.5
> aboot: switching to OSF/1 PALcode version 1.50
> aboot: no disklabel found.
> aboot: loading compressed vmlinux.gz...
> aboot: ok now starting the kernel...
>
> halted CPU 0
>
> halt code=5
> HALT instruction executred
> PC = FFFFFc00000310088

This usually happend when the kernel can't load the root filesystem you
have specified. First make sure that when you boot you pass it the
load_ramdisk flag. Something like:

b dva0 -fi vmlinux.gz -fl "root=/dev/fd0 load_ramdisk=1"

Second, I think that starting with 6.0 the ramdisk uses a different format
and I remember somebody telling me that ROMfs had to be in the kernel. I'm
not sure if Jay's 2.2.3 image has this or not.

Why are you not using the 6.0 kernel? NCR SCSI controller?
