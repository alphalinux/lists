> Netscape requires the following Tru64 shared libraries:
> libX11.so libXmu.so libc_r.so libots.so
> libXext.so libXt.so libdnet_stub.so
> libXm.so libc.so libm.so
>
> Would it be possible to run Netscape with the GPL equivalents of these
> libraries, all compiled under Tru64 unix. If so, then this
> would seem like an
> effort that would be much appreciated by many people.

wouldn't make any difference. The point is that GNU libs are not binary
compatible with DEC unix libs. never mind which platform it was compiled on.
You'd have to recompile netscape.

And that hasn't been done because, as explained to me, netscape isn't 64bit
clean. The DEC Unix compiler has a switch to compile with 32bit pointers and
32bit addresses instead of 64bit, which allows netscape to compile. gcc
doesn't have such a switch, so netscape for linux.

maybe if/when Compaq release their C compiler for linux, and provided they
include the 32bit compatibility feature, we might see a native netscape for
linux/alpha.
