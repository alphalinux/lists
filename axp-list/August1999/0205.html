                First check, what chipset this card is based on. Then see,
if there is an
                X server for this chipset (hint: a lot of different chipsets
are covered
                by XF86_SVGA server). Check http://www.xfree.org
<http://www.xfree.org>

                If there is none, you can still use a frame buffer based X
server
                (XF86_FBDev). It's probably not the optimal solution, but it
might get
                your card working. Check out
                http://metalab.unc.edu/pub/Linux/docs/HOWTO/mini/Vesafb
<http://metalab.unc.edu/pub/Linux/docs/HOWTO/mini/Vesafb>

                In worst case, use XF86_VGA16. It's ugly, but it really
should work. :)
