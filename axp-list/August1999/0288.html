Hello,

I'm trying to enable the linuxconf html-access on my machine.
It's an alpha running Redhat 6.0, with Linuxconf 1.16r2.1.
(the original one shipped with RedHat 6.0 has the same
behaviour). The issue is netaccess.

I've configured netaccess as follows:

[netaccess]
htmlaccess.from 127.0.0.1
htmlaccess.from 192.168.1.12 255.255.255.0
htmlaccess.dolog 1
htmlaccess.enable 1

When i point netscape (or lynx, it doesn't matter) to
the machine I get the following response:

500 access denied: Check networking/linuxconf network access

The /var/log/messages shows the following:

Aug 12 19:24:08 cgmd77035 linuxconf[16066]: IP 192.168.1.10 do not match
127.0.0.1/4294967295.255.2192.168.1.10
Aug 12 19:24:08 cgmd77035 linuxconf[16066]: IP 192.168.1.10 do not match
4294967232.168.14294967295.255.2192.168.1.10/4294967295.255.2192.168.1.10

On my other computer (an Intel) I've installed the same version of
linuxconf (they were compiled from the same SRPMS), and here it
works without errors...

Do you have any idea's what might be causing the problems? It could
be an architecture related issue...
