he has to recreate the console in /dev to reflect the major minor device
code of "5, 1" to allow the kernel to automatically default to the console
device present (Graphic's is always first choice) and if the kernel has
serial console support built in , it will automatically switch to serial
port 1 if no graphics is present.

The problem is the stock /dev either has /dev/console defined as 4 0 (tty0)
first virtual device or symbolically linked to tty0. 4 0
