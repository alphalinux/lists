Hello all!
Last night I tried to set the 164SX's parallel port to bidirectional mode
but very obviously failed. The only value I get back from it is 0x04. That's
for sure one of the values I might want to read but on the long run very
monotone ;)
What I did was the normal way, I set bit 5 of the control register (base + 2)
to enable "PS/2 byte mode". The bit is set properly, i.e. I can read it
back, but the port remains output only.
So my question is, is the 164SX parport indeed just a SPP port? If not how
can I set it to bidirectional ECP mode?
Just FYI I will have to do this by hand because this will be incorporated
into a small application that will do the data transfer. The application
already exists and works on Linux Intel with ECP enabled in BIOS.
Thanks for any tips, hints and clues ;)
