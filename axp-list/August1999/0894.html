On Tue, 31 Aug 1999, Nils Faerber wrote:

> > I am thinking about shelling out 100 bucks and buying a
> > Elsa Gloria Synergy card from a PC parts seller and hoping
> > that it works...
> That's exactly the way to go. The ELSA Gloria is the cheapest card around
> AFAIK to get 16bpp. It is supported with OSF 4.0d and the latest Open3D
> package and runs with vanilla OSF 4.0e. I can only recommend 4.0e because I
> did not manage to get 4.0d and Open3D to properly work with the Gloria card.
> Maybe someone else has a tip for this.

There's only one possibly trap using the Elsa GLoria Synergy: If you have
bad luck, the SRM firmware doesn't like it. This is no problem with Alphas
for which you can get the latest and greatest SRM firmware version. But if
you have a Multia, for example, there is a good chance that the SRM
firmware has problems with it. In this case, maybe it's possible to run
the console on a serial terminal and still configure the card in the
kernel and have the X server use it. I can think of no reason why this
shouldn't work. Another crazy idea is to plug in two graphics cards.
One for the console and one for the X server.
