David Webb wrote:
>
> Hi,
>
> I'm using an lx164 with the 2.2.11 kernel.
> I'm going to be installing a Quantum fireball plus
> Ultra66/33 hard drive in the system. I know that I'll
> need a Ultra66 controller card, cable, and kernel
> patch if I want the drive to transfer data at it's
> highest speed. What I don't know is weather the
> on-board eide controller will work as an Ultra33
> controller. If it will, then I may not bother with the
> Ultra66 card and cable. The motherboard manual calls
> the on-board controller an eide controller. I don't
> know if this means that it is not an Ultra33
> controller. Is there a HOWTO or web site that someone
> can point me in the direction of that talks about
> this?

A PC164LX uses a CMD646 e-ide chipset.

The newer kernels support it in DMA mode if you turn on that
option.

Keep in mind you need special cables for the newest
ultra-ide connections. 80 pin vs 40 pin or something
like that. When you get it running, try a "hdparm -Tt /dev/hdx"
on it.
