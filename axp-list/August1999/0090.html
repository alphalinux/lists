On Thu, 5 Aug 1999, Wes Nakamura wrote:

> Has anyone else seen problems with nfs being very slow in one direction?
>
> I can copy from an nfs mount to a local drive, and it transfers at about
> the speed I expect it to. Going the other way, though, from a local
> drive to an nfs mount, is many (many) times slower.

Perfectly normal behaviour for NFS I'm afraid. The story is that under
the semantics defined by NFS when a server receives a write request it
must be 100% sure it is on the disc before it acks to the client. The
result is that it is quite slow.

For a start you can add the 'async' flag when mounting which basically
means don't force it to wait for it to be on the disc before carrying on.
(Some could argue that this is less afe).

Also play about with the block sizes by using rsize=8192,wsize=8192 on the
parameters and see what happens.
