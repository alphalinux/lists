The programming stuff is way beyond me, so please don't berate for my
ignorance.

--- Sean Hunter <sean@uncarved.co.uk> wrote:
<snip>
> For those who don't know, its possible (as Michal told me, and rth's
> code snippet demonstrated) to exercise a small degree of control over
> what the kernel does when an app encounters an unaligned access. In
> particular, you can turn off the message, tell the kernel not to fix
> the access (this sounds like a bad idea), or (more usefully) get the
> kernel to send the app a SIGBUS, causing it to dump core. You can
> then plug the coredump into your favourite debugger and fix the
> access.

So this means for the "regular" user (assuming an alpha linux system) that any
time there was this unaligned trap, the program would core dump? Therefore a
non-programmer-type person (like me) would be unable to continue using many
programs intended for x86 which were compiled on his/her alpha because of this.
 That seems a little counter-productive to me, but might in fact spur some
programmer-types to finally fix other programs ;)
