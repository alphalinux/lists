On Sat, Aug 14, 1999 at 10:43:17PM -0400, Jay.Estabrook@digital.com wrote:
>
> >>> Rich Payne said:
> >
> > Second, I think that starting with 6.0 the ramdisk uses a different format
> > and I remember somebody telling me that ROMfs had to be in the kernel. I'm
> > not sure if Jay's 2.2.3 image has this or not.

Ahh, sounds good. I guess the Compaq Linux install page should
either mention that the directions are for Redhat-5.2, or a newer
kernel should be put up at:
ftp://ftp.digital.com/pub/DEC/Linux-Alpha/Images/

> Correct; that 2.2.3 image from off gatekeeper does NOT have the ROMFS
> needed for the 6.0 RAMdisk to work. You could use either the 5.2 RAMdisk
> or you could eschew RAMdisk altogether, via (assuming SCSI CD):
>
> >>> b dva0 -fi vmlinux.gz -fl "root=/dev/scd0"
>
> This should work on DS10, and is worth a try on other boxes when needing to
> use that 2.2.3 kernel to install 6.0.

Okay so I tried the above:
        generic-up-223.img
        Boxed redhat-6.0 cd
        b dva0 -fi vmlinux.gz -fl "root=/dev/scd0"

And got:
aboot: ok, now starting the kernel...
[boot happens, ethenret/scsi recognized, then flash to blue and ..]
halted CPU 0

halt code=5

HALT instruction executed
PC = fffffc0000310088

Probably a result of redhat's new ramdisk.img implying the one on the
cd has problems.

So I tried:
generic-up-223.img
redhat-5.2 ramdisk.img
b dva0 -fi vmlinux.gz -fl "root=/dev/scd0 load_ramdisk=1 bootdevice=/dev/fd0""

It works, I get to the install and it hangs during the fdisk with:
<4>scsi : aborting command due to timeout : pid 362, scsi0, channel 0, id 0, lun 0 Write (6) 14 51 3c f4 00
<4>ncr53c8xx_abort: pid=362 serial_number=712 serial_number_at_timeout=712

So currently I'm going to boot redhat-6.0 on the IDE disk I dropped in
and see if the newer kernel's/drivers to handle the controller better.
I'm open to suggestions though.
