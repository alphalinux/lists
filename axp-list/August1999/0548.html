On Sun, Aug 22, 1999 at 10:51:39PM -0700, Mitchell Blank Jr wrote:
>
> Also: 2.2.12final3 works fine and 2.3.15pre2 is still broke
>
> > So the cy82c693 gets detected fine (albeit at a different IO range - I
> > assume that's just a change in the PCI code or something),
>

Interesting. Recent 2.3.x kernels somehow break settings of the Cypress chip,
so IDE addresses are bogus (something like 0xfff0). As a temporary workaround
try this hack (tested with 2.3.15pre2), it set IDE i/o back to 0x1f0/0x170.

Ivan.

--- linux/drivers/block/cy82c693.c.orig Thu Jun 3 09:21:51 1999
+++ linux/drivers/block/cy82c693.c Tue Aug 24 22:15:46 1999
@@ -383,6 +383,8 @@
         /* write info about this verion of the driver */
         printk (KERN_INFO CY82_VERSION "\n");
 
+ pci_write_config_dword(dev, 0x10, 0x1f0);
+
 #ifdef CY82C693_SETDMA_CLOCK
        /* okay let's set the DMA clock speed */
