>On Sun, 29 Aug 1999, Eric Jones wrote:
>
> Afaik, the ONLY two video cards that have kernel framebuffer support for
> alphas are matrox and tga cards. I don't know if this is being worked on
> at all.
>
>I can't get matroxfb to work on my alpha with Matrox MGA2164W. I know
>about the boot commandline in the FAQ, but it didn't work either.
>Anybody have it working? And how?
>

for kernels <= 2.3.6, you have to specify a text mode, like
video=matrox:vesa:0x10c
(i'm not sure for this value, check the documentation in
linux/Documentation).
This works for me on my SX164/Matrox. But it seems that only 2.3.15 allow
you to boot in
graphic mode, because i saw the linux logo on the top of my screen at boot
time...
The frame buffer access is buggy and not totally operationnal for older
kernels.
2.3.15 allow me to try the GGI library with target framebuffer, and some
stuff appear
on the screen; ugly and buggy, but it works. It seems that GGI need some
alpha-check...

Well, in fact i havent checked kernel 2.3.7 to 2.3.14, so what apply to
2.3.15 may
apply for those.
