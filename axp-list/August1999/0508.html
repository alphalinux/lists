On Sun, 22 Aug 1999, Dave Gilbert wrote:

> On Sun, 22 Aug 1999, Rich Payne wrote:
>
> > Good luck. With NT on Alpha going away, I would guess AlphaBIOS will also
> > go the same way.
>
> What - since when did that get decided?

AlphaBIOS really exists for one purpose, to boot NT. Linux, OpenVMS and
Tru64 can all be booted from SRM. I beleive it is Compaq's plans to have
new Alpha hardware boot Linux from SRM. So that begs the question, why
would AlphaBIOS still be developed?

Granted, there may be a few more revisions for vital bug fixes etc....but
I would guess it's almost come to an end.

> > There is a slightly newer version at www.alpha-processor.com for the LX
> > boards. I think it's 5.68-1, it might help with some of the problems
> > you've seen.
>
> Thanks - the history definitly doesn't suggest that - but there is a
> history hole between 5.66 and 5.68 - hmm.

5.66 was (I think) the last version that Dec/Compaq did for the LX. After
that production was done by Samsung in Korea and they did a new release of
the AlphaBIOS (5.68-1).
