This is interesting; the problem you're reporting sounds a lot like
what I saw with the LX164 and a 53c875 card in some of the 2.2.11pre
releases. This got fixed (after I reported it to Linux-Kernel), and
I'm running 2.2.11 just fine. The problem had to do with mismatching
patches--one set assumed the PCI windows were set up one way, while
another assumed differently. It could be that the fix for the LX164
either wasn't extended to the PC164 or somehow broke the latter.

It's actually not a problem with the driver, but is rather a kernel
resource management issue that doesn't affect most drivers.

Has anyone reported this? Is there anyone running 2.2.11 on a PC164
where this driver is working?

                -Ed
