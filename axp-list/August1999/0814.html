OK, finally got some DS10's to test with. Ended up with
a SCSI controller and everything installed and boots
fine, after using my own kernel to fix the Symbios timeout
problem. Need to try Richard's udelay() fix at some point
to see if that is the cause of the Symbios timeouts.

Now, I'm a bit curious about what the kernel (2.2.11) is
trying to tell me about the EIDE ports:

PCI_IDE: unknown IDE controller on PCI bus 00 device 68, VID=10b9, DID=5229
PCI_IDE: not 100% native mode: will probe irqs later
PCI_IDE: simplex device: DMA disabled
ide0: PCI_IDE Bus-Master DMA disabled (BIOS)
PCI_IDE: simplex device: DMA disabled
ide1: PCI_IDE Bus-Master DMA disabled (BIOS)
hdc: COMPAQ CDR-8435, ATAPI CDROM drive

It is a bit worrisome that DMA is disabled. Anyone
running EIDE on a DS10? If so, what is your disk
performance?

The cheetah SCSI is quite fast so that's not an issue
on the machines I'm testing but I'd like more input on
how EIDE does.

A 9GB cheetah:

/dev/sda:
 Timing buffer-cache reads: 64 MB in 0.35 seconds =182.15 MB/sec
 Timing buffered disk reads: 32 MB in 1.84 seconds =17.37 MB/sec

Note the cheetah is substantially slower than my 50GB barracudas
which run around 23MB/sec.

Any additional info appreciated.

Wes
