On Tue, 31 Aug 1999, nm wrote:

> Elsa Gloria huh? I will see if I can find one. I wish I could find
> out what part number it is so that I can be sure that the card I buy
> is the one that will work.

It might not be that easy, though. Apparently there's a DEC OEM'd version
of the card which is more expensive than the original ELSA version.
However I have contratictory reports about these two versions: Some people
have claimed that the original ELSA version did not work with DU, others
(namely a DEC/CPQ employee in comp.sys.dec or was it comp.os.unix.osf1?)
said that the DEC OEM'd version is not modified in any way. Eventual
backdoors to the situation are: DU allows to specify the PCI device/vendor
ID to kernel driver mapping in /etc/sysconfigtab. This could be modified
if only the device/vendor ID is different. Another idea is to use the
utility from the Elsa web page to copy the on-board bios from the DEC
OEM'd version to an original Elsa version and hope that it works, provided
that this is legal, of course. Third, but much less exciting way is to
buy an original DEC OEM'd version :-)

> Any ideas if it should be the 8 meg pci
> card?

Yes, definately.

> any idea what color depth/resolutions I will be able to run
> at?

Up to 1280x1024 @ 24 bit depth at 80Hz or something.
