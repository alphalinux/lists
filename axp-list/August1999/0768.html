(experiences, FWIW)

I've found too that this hasn't worked well for me with any of my Alpha
machines (Rawhide, AlphaPC164, Multia) - naturally enough.

What I've found so far that if I want serial console support with a kernel
any where near current, I need to

a) configure in serial console support

b) enable a line in /etc/inittab that will run a mingetty on either ttyS0
or /dev/console (bad things happen if you define running lines for both
ttyv1 and console and you have a graphics head. Be warned.)

c) The best state of affairs so far for me is to take 2.3.13 and apply
Richard's two patch sets *and* (a modified) d-sercon patch (to enable the
serial console early). This has various breakages in it (files that had to
be modified to compile), and NFS seems to be broken in it, but it's at
least a start on the road to recovery.
