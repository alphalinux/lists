I just got a DS-10 from one of the alpha vendors. The machine is
very nice but looks like it got some real abuse. So far I've found:
1. The case lock was broken, needed major surgery to get the case open.
        The center pin was jammed up so far it was touching one of the
        edges (making it impossible to insert a key)
2. The front face plate had it's top/center screw stripped and in crocked,
        had to force it out. (cheap labor with power screwdrivers I bet)
3. The reset botten in front was broken, had to remove the gray part
        to allow the small white part to move freely in/out without getting
        jammed on the case. Now requires a pen or similiar to hit reset,
        not altogether a bad idea.
4. The 64 bit dual U2W pci card was in a 32 bit pci slot.

Otherwise the machine seems to be in good condition. It was installed
with RedHat 5.2, it ran some of our code for 12 hours then crashed
hard. I might have some error messages to post if I can get it running
again.

I tried installing RedHat-6.0, the vendor claims that redhat-6.0, the
DS-10, Intraserver dual channel U2W (SYM53c896 or similiar), and the
compiler/kernel is incompatible so they downgraded to Redhat-5.2. But
shipped a redhat-6.0 boxed set which was nice.

Anyone out there have a similiar setup? Exactly which boot/root
did you use. All my attempts end with:
        halted CPU 0

Just after loading the ramdisk.img
