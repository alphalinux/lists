Ryan Kirkpatrick wrote:
>
> The trickest part is getting access to SRM and ARC w/o
> monitor+keyboard. It is possible, as when the system boots, it will detect
> that there is not a keyboard and do I/O through the first serial port (std
> pins on the 25pin serial connector). As far as I know the srm console is
> completely accessible this way. The ARC on the other hand, while it will
> display its initial menu, can not be interfaced with as arrow keys from
> about any terminal emulation I have tried are not recongized correctly. In
> other words, you can see the ARC loader, and if set to auto-boot, it will
> boot, but you can not choose any menu options other than the default one.
> :(

If you set autostart to boot to milo prompt (say 'show' as the only
boot option) then you should be able to boot any configuration available
from there. When combined with /etc/milo.conf this can be even made
simple by a usage of labels. OTOH in this way you are loosing autoboot
as 'timeout', even if it can be specified in /etc/milo.conf, does not
work so far with any milo which I have seen.

'milo.conf' manpage is on ftp://genie.ucd.ie/pub/alpha/milo

> This is really bad in my case where my keyboard and monitor ports are
> toast (my UDB has seen too many battles), and only the serial port
> works, and the CMOS battery died, so the NVRAM is invalid, and I can't
> boot the system at all. :(

This makes it kind of hard to change boot menu options, I suppose. :-)
