Make sure you are including 'rsize=8192,wsize=8192' as mount options
for nfs mounts. default read/write size is 1024 bytes.

>
>
> Has anyone else seen problems with nfs being very slow in one direction?
>
> I can copy from an nfs mount to a local drive, and it transfers at about
> the speed I expect it to. Going the other way, though, from a local
> drive to an nfs mount, is many (many) times slower.
>
> This is with a stock redhat 6.0 installation. I don't seem to remember
> this problem with the 5.x series.
>
> Everything else about nfs seems to be working all right, and the
> regular network stuff is working fine...
