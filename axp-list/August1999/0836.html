Linux version 2.2.12 (root@castle.home.front)

This, and few other machines, run from the same "generic" kernel for a
while and I did not see a single crash yet. Date indicates, obviously,
that this is one of "2.2.12pre" kernels and not the release but still.
I would say that you are likely looking for reasons behind your troubles
in the wrong place (unless you have a real evidence of compiler bugs).

This particular machine is my "hack box" - which implies that is
rebooted quite often for various experiments and the current uptime on
it is only four days. What "crash/reboot regularly" means?

BTW - this is a compiler I recompiled myself directly from sources and
not precompiled binaries.
