Jay.Estabrook@digital.com wrote:
>
> >>> Frank Tiels said:
> >
> > I just tried to install a redhat 6.0 on a personal workstation 500au.
> >
> > Everything seems to work except for the video card.
> > This card is a powerstorm 4D30T with a S3trio64v2 GX chipset.
> > This chipset is in the Xconfigurator, but doesn't seem to work.
> > Anyone who has some experiences with this card?
>
> I can't find *any* current DEC internal references to a 4D30T. Does it have
> any other disinguishing numbers on it, like PB2GA or PBXGK or ??? Look for
> stickers, or silk screening, or even embossed numbers next to the VGA plug.
>
> And are you sure the TRIO isn't just the frontend for a 3rd party
> accelerator card? This is true of the TGA2s, which have a CIRRUS VGA for
> console mode text support (otherwise you have to have some kind of character
> mode rendering code in the consoles and the kernels, as per the original
> TGA). Look on the board for any other "large" chips which might have some
> identification on them.

I have docs on that particular Trio chipset, which is definitely a
mainline gfx accel chip, though I wasn't aware of its usage in Digital
equipment.

Taking a quick glance at the XFree86 S3 code, this chipset doesn't seem
to be specifically supported, though the framework definitely exists.
What does the X server do/report? And have you tried both the XF86_SVGA
server and XF86_S3?
