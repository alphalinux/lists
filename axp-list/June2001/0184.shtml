On Thu, 21 Jun 2001, Uncle George wrote:

> all u have done is suggest that the 'application layer' is inefficient. I would think
> that if someone would want to prove that the device layer was inefficient, would want to
> try to flood the application layer with packets
> 2 FTP's running at the same time would have a better chance of flooding all 7 layers.

Perhaps. But there is no certainty that the device driver and kernel
interfaces will handle concurrent device access efficiently.

These sorts of benchmarks are comparing the quality of the driver
implementation as much as the hardware.

If you wished to experiment you could run a performance tool such as
iprobe during the test to see where kernel cycles are going.
