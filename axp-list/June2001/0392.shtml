On Sun, 1 Jul 2001, Al Pha wrote:
> does anyone know who to compile snort to avoid this? or do i need some sort
> of update?

(Isn't there an unaligned trap faq somewhere? Alphalinux.org seems to be
down...)

> ul 1 08:09:03 thezorb kernel: snort(719): unaligned trap at
> 000000012000c044: 0000000120164afe 28 3

The first number is the PC of the faulting instruction. In this
case it lies in the main executable, so a library is probably not to
blame.

Assuming you built snort with debugging info, use addr2line to find the
code responsible for the trap:

addr2line -e snort 0x12000c044

The fix depends on how it is used in the application. You may be able to
force alignment somehow, otherwise use a macro to copy the value to/from
an aligned field.
