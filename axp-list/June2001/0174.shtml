  I just fought this battle with a set of XP1000s and DPW433, 500s and 600s.

  A few of the DPWs would play sounds at half speed. It seems to be
  related to the rev level of the motherboard. For our cases, if the DPW
  had an external SCSI adapter, it would produce bad sound. Those with
  on-the-motherboard SCSI were OK.

  I fix my problem by using sox to downsample the sound files to a
  sampling rate all the machine could handle the same way.

    /usr/bin/sox -t wav WAV_FILE_NAME -r 16000 -t wav - resample | /usr/bin/play -t wav -

  You might be able to get a better rate than 16k...it seems to me it
  worked up to 19k.
