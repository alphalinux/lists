Problem: keyboard and mouse become unresponsive (driver dies?)
         when I invoke xawtv, using the bttv driver

Hardware: UP2000, one processor, bt-878 TV card, TNT2 16MB
          graphics card, rocketport 4-port serial card, a few
          other cards

Software: RH 7.0, kernel 2.2.19-7.0.1, xawtv version 3.18-2,
          rocketport driver version 1.21-HardData

When xawtv is invoked/started, which loads the bttv module, then
the cursor moves around (to bring up the xawtv "options" menu),
the keyboard and mouse drivers seem to die. X continues to
display an xclock ticking away. The mouse cursor stops moving,
the mouse buttons don't work, and the keyboard doesn't work. The
"Num Lock" key doesn't even make the LED light up. Restarting
gpm mouse services does not help.

I understand the bttv driver is probably old, as it apparently
has a bug when starting in composite input mode. I'm using xawtv
instead of kwintv, because kwintv refuses to even compile, let
alone run.

This problem used to be sporadic, only happening when I tried to
bring up the xawtv options menu too soon after starting xawtv.
Now, suddenly, it happens much more often (three times in a row
today). The only thing I remember doing to the system between
before and now is arranging for mgetty to run on two of the ports
on the rocketport card.

Any suggestions to either prevent the keyboard and mouse driver
from going comatose, or recovering use of the keyboard and mouse
without a full reboot?

Thanks.
