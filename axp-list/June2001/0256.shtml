Hi,

Here's a problem I having after installing kernel 2.4.2.

Disks exported from my Linux kernel 2.4.2 system running
NFS v3 can not be mounted by my solaris systems or my
digital unix systems. Adding "--no-nfs-version 3" makes
the problem go away.

Here's the failure messages:
----------------------------
>>From a Digital Unix system, Mount failure
# /sbin/mount nanna:/mount/s-nanna /mnt/disk
nanna on /mnt/disk: Error 10001 occurred.

>>From a Solaris system, Mount failure
# /sbin/mount nanna:/mount/s-nanna /mnt/disk
nfs mount: mount: /mnt/disk: Stale NFS file handle

Here's some version numbers
---------------------------
Red Hat Linux release 6.2 (Zoot)
Linux version 2.4.2smp
nfs-utils-0.3.1-0.6.x.1

Here's some config settings from the exporting system
-----------------------------------------------------
CONFIG_NFS_FS=y
CONFIG_NFS_V3=y
CONFIG_NFSD=y
CONFIG_NFSD_V3=y
CONFIG_SUNRPC=y

The export file has these kind of settings
------------------------------------------
(rw,no_root_squash,insecure)

Any comments will be appreciated.

Thanks in advance, --Lew
