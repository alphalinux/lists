> I agree,
> It was my hobby like thing, reporting bugs and fixes but not anymore.

> after long period of time. I would rather report any bugs and fixes to him
>
> Is it true that any kind of kernel debugger is not included in linux because
> linus doesn't like kdb?

That's pretty much true. Linus doesn't like kernel debuggers in general
and the rest of the "kernel elite" feel that we should all be able to
write bug-free code the first time (which, apparently, not even one of
them can do...at least *I* haven't seen them do so). The subtext of that
stance is that they don't want "just anyone" writing kernel
drivers/patches....you have to be accepted into their cabal to be able to
do that...

Part of the practical reasons for not including one, though, is the
possible size that it would add to the kernel source. Granted, it
wouldn't be that much compared to the benefits, but it's somewhat
understandable. The kernel source is getting quite large now. I have had
some ideas about how to deal with this from the point of distributing the
kernel source, but have been hesitant to bring it up to anyone since
nobody that matters wants to listen to anyone but themselves.
