> I have installed RH 7.0 on a dual UP2000 system. During the installation, I
> have to choose which resolution I want to use because after the installation
> has finished I can not run Xconfigurator to change it. When I do run
> Xconfigurator, I get to the part where I choose which monitor I have and
> then it dumps me back out to a Linux prompt and says "Segmentation fault
> (core dumped)".
>
> Any ideas on why I'm getting this error?

I had a similar experience with my one-processor UP2000, in case
it's relevant to your case.

The XFree version 4 servers would segment fault (and would not
even dump core) for me. The folks at Harddata diagnosed it to a
bug in the RH 7.0 default kernel, 2.2.17-4. The XFree 4 server
tries to see how many PCI buses there are on the machine. It
attempts to access PCI bus number 2, which does not exist. The
kernel is suposed to return an error code, but 2.2.17-4 does
something different that results in a segment fault by the X
server.

In my case, the 2.2.19 kernel works fine. I'm currently using
2.2.19-7.0.1, which was the latest one available as of a few
weeks ago.

An alternative solution for me was to use the version 3.3.6 X
servers, and use X itself to create the configuration file.
IIRC, I used something like "Xconfigurator --preferxf3" to get
the XFree 3.3.6 server.

With the new kernel, you can configure X by doing "XFree86
-configure" instead of Xconfigurator. In some ways, XFree86 is a
better configurator, if you are willing to do a little
experimenting.

Good luck.
