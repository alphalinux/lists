On Tue, Jul 03, 2001 at 09:50:15AM -0400, Christopher C. Chimelis wrote:
> >
> > Symbol xf86SetDDCproperties from module
> /usr/X11R6/lib/modules/drivers/atimisc_drv.o is unresolved!
>
> Looks like they modules got stripped. Try copying unstripped versions of
> the drivers to /usr/X11R6/lib/modules/drivers and see if that fixes
> things...

I've had this happen a few times in the past, usually with a CVS
tree which wasn't quite there yet. I've been able to solve it several
times by putting some explicit "Load" statements in the XF86Config-4
file, for the missing modules (in this case, both DDC and vbe would be
indicated).

That's not to say that those extra "Load" statements should be needed,
just that they work around some problem that seems to be associated
with the newly-built drivers/modules *not* automatically loading some
other modules that they usually do.
