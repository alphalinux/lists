On Tue, Jun 12, 2001 at 12:12:44PM +0200, Peter Bech wrote:
>
> When I recompiled my AS800 I found a paper on RH site before trying. But I
> cannot find it anymore. It specifically told, step for step, how to configure
> with "make xconfig" and how to recompile with "makes".

In the top directory of kernel sources there is a file named README
which describes required steps in a quite detail. Various particulars
may be a bit behind times in places but general principles are valid.
You are particularly interested in a section entitled "CONFIGURING the
kernel:".

There is a catch though. The next section "COMPILING the kernel:"
is very x86 specific and, in particular, talks about commands like
"make zImage" and "make bzImage". These do work _only_ for that
specific architecture (targets are defined in arch/i386/boot/Makefile).
The one you want to use everywhere else is "make boot" (which works
also on x86 and is equivalent there to "make zImage" - which nowadays
is not good enough in most situations for x86).

The one you really want to use is "make boot modules", although it
can be split into separate steps.

In the same location there is another important file which deserves your
close attention and which has a title REPORTING-BUGS. Guess what this
is one is about. :-)

> I myself had to figure out how to set the kernel in production by copying to
> the boot dir with the samne name as the SRM boot name.

This is not really a way to do it. Check documentation included with
your 'aboot' package. If SRM.txt, or equivalent, is missing in your
version then there should be around SRM-HOWTO. Check www.alphalinux.org
if not among docs you should have on hands. Also examples of
'etc/aboot.conf' should be on your installation CD and floppy.
