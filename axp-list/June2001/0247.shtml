hi,

i have an alphastation255 (avanti) with srm.
when i use
 isacfg -slot 1 -dev 0 -mk -handle PCXBJ -etyp 1 -enadev 1 -iobase0
530 -iobase1 388 -dma0 0 -dma1 1 -irq0 9
to enter the sounddevice then isacfg tells me that something is not correct.
 isacfg -slot 1 -dev 0 -mk -handle PCXBJ -etyp 1 -enadev 1 -iobase0
530 -iobase1 388 -irq0 9
(whithout the dma stuff) does work.

when i use
 isacfg -slot 1 -dev 0 -mod -dma0 0 -dma1 1
after the above command to modify the dma then srm says that it's just
possible to enable/disable the device (-enadev ... 'isacfg -slot 1 -dev
0 -mod -enadev 0' does work for example).

if i use
 iconfig
to modify the isa table then i can enter a dma channel - but just one!
it is also not possible to enter 2 iobase addr there.

very strange.
is this a srm bug?
do i have to use sthg else then '-dma0' in the isacfg comand for
modifying/entering dma channels?
