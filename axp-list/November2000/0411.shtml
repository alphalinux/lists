On Mon, 20 Nov 2000, SteveCraft wrote:

>
> Have BackupExec 8.0 running under Windoze NT.
> Is there source for an agent that will compile under RH6.2 for Alpha?
> I know there is an x86 Linux agent, but their web site is so bad I can't
> find the binary for it, let alone src.

I spoke with one of Veritas' engineers at a presentation done at a local
networking user group and specifically asked him about the possibility of
getting two things with respect to BackupExec:

1. A version of BE to run under x86 Linux

  (this one was personal--I'm going to be setting up a backup server
  using Amanda on an Athlon system and was curious what he would say)

2. Support for Alpha Linux

He told me that at this point they had no plans to develop a Linux version
(x86 or alpha) of their backup server software. He also was clearly under
the impression that Linux on the Alpha was some kind of "geek toy" and
struck an almost condecending attitude at that point in the conversation,
despite the fairly impressive list of sites using it which I pulled off
the top of my head from my time on this list. I really wish I could
remember the man's name... :-(

Oh well, personally I'm not too concerned--I've been far happier using
Amanda than I was with BE 7.x. Steve, I might suggest that you move the
backup server functions to a Linux box and use Amanda. You'll be able to
backup all your win32 boxes as well as any unix box that can compile the
Amanda client parts. I do know that RH 7.0 includes amanda, and I think
all the BSDs do as well.
