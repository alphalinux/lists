I only have experience with 6.4. The only bug that's held me
back is that Utah-GLX doesn't work in direct rendering mode, but
that's to be expected since it's not supported on Alpha anyway.

A big plus with SuSE is that it runs on older Alpha's (ie. Multia
and Jensen). RH 7.0 doesn't support older Alpha's, as far as I know.
At least the beta versions of 7.0 didn't. Maybe that has been fixed
for the production release.

As you say, it's personal taste and depends entirely on what you use
your machines for. I can only offer my informed opinion, having run
RH, Debian, SuSE, and Blade Alpha boxes since 1995.

-Michael

Apparently the best distro depends on the release and the specific
setting, and also on personal taste.

I for instance had mixed experiences with SuSE 6.1 and 6.3:
- 6.1 had no working bootp or dhcp. The programs in the distribution
  dumped core. Similar situation with some other daemon, forgot which.
  Maybe this was the reason that there was a voucher in 6.1
  for a free 6.3 package.
- I was not able to boot the Alpha according to the confusing and
  incomplete description in the SuSE documents, which apparently were
  written for a different type of machine and bios.
  I finally succeeded with the Redhat instructions and some document
  mentioned therein.
When it finally ran, I had no more problems.
And also no problems with SuSE on x86.

Gernot
