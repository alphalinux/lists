I'm under the impression that when you install a new release, you get what you
asked for (or deserve, depending how you see it).

Usually, you have spent quite some time setting up your systems, and you know
their quirks, and how they run. Just installing a new release is like opening
a whole new can of worms. I try to avoid it.

All my systems still display "RedHat release 5.2" (or even 5.0), but have
evolved over time, when there was some need. They now tend to run a 2.4.0
kernel, with glibc 2.2, for example.

I think you are better off, in the long run, to reuse your current setup, and
update the necessary pieces. This way, it won't be such a black box.

Setting up a new system is not a problem either: just boot it using some CD
that came with one of those colorful Linux magazine, and do a minimal install
in a small partition that will become the swap partition of the new system.
Then boot this minimal setup, copy "your" current setup from another of your
machines, edit some /etc files, and you are all set, with no nasty surprises.

I guess I'm a bit old-fashioned, but that's the way I like it. Needless to
say, I tend to want to compile all my stuff from source (and tweak the SRPMS
too, of course :)
