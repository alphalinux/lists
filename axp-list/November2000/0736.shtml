I think I figured out part of my performance problem... In my
host.def file, I set the compiler switches on DefaultGcc2i386Opt
                  instead of DefaultGcc2iAxpOpt

Doh!

So, I only built with generic instructions. I have rebuilt the
server and while it seems faster, I'm seeing some drawing artifacts.
It looks like some misoptimization of OR or XOR code. I get jaggy
lines around the edges of icons and window frames and the bar cursor
leaves a ghost of itself on the letters as I type. Interestingly,
the garbage goes away when I hit enter. It appears to redraw the
line and after the redraw the line looks ok...

Also while looking through linux.cf, I noticed a switch called
UseCompaqMathLibrary. If this is set, then it will link with the
CPML before the standard math library. I've set that too. I don't
think that's causing this problem, because I seem to recall something
similar when I've compiled with -mcpu=ev56 before.

On the plus side, the server doesn't crash anymore after I close
the last Netscape window. :)

Alan
