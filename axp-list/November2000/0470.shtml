On Tue, 21 Nov 2000 kwood@atipa.com wrote:

> I reloaded the machine myself just to make sure, but it still does it.
> If you log into the machine and run free you will get an amount of used
> and free ram. Compare this to the amount you get after running X and
> then exiting and you get a different number. Now log out and in and you
> get yet a different number. All lower than the one you started with.
> Now is this a good example of a memory leak or am I thinking
> incorrectly. Shouldn't you have the same amount of ram before and after
> running a task and then exiting?

Not really. In almost every case, some/most of the memory that was used
by the applications will remain "used" as cache or shared mem (look at the
right columns in free's output). It will be reclaimed and reallocated as
needed, though, by the VM. To be honest, 128MB RAM is not a lot in a RISC
machine, but it should be adequate for most tasks (I've been running with
the same amount of RAM for years now). He may hit swap every so often,
though, especially when trying to run Netscape plus a few other X apps
simultaneously.

> His concern is that when he runs xinit and then netscape, his machine
> locks up hard. It seems to be a memory problem as he can't telnet in
> and he can't move his mouse (which is usually hardware).
>
> If you have any idea about what I am saying and possibly a way of fixing
> or explaining why this happens, could you let me know. I'd appreciate
> any help I can get. Thanks

What kernel is he running? It sounds like either the VM is falling over
(not surprising), he's got a problem when exceeding physical RAM (problem
with swap...or the VM again), or he's got some defective RAM (doubtful
since there would be some other warning signs). If it's either of the
first two, though, it is probably kernel related, hence the question...
