>
> On Wed, 29 Nov 2000 reig@dcs.gla.ac.uk wrote:

> before getting into a prolonged discussion, i think it might be worth

I'm trying to port the SML/NJ compiler
(http://cm.bell-labs.com/cm/cs/what/smlnj/) to alpha-linux. It only
runs on alpha OSF or DUNIX now.

The runtime system is written in C, and I need to compile this for
alpha linux first. I got this from one of the authors of SML/NJ:

> > > The main issue is that you need to create a 32-bit executable during
> > > the linking process. I've been told the following URL explains how
> > > to do this:
> > >
> > > http://www.alphalinux.org/archives/debian-alpha/September1999/0078.html

I'm afraid I don't know the exact reason why I must produce 32-bit
binaries for the runtime, but I suspect that SML/NJ's code generator
only emits 32 bit code, and that has to be linked with the
runtime. All I know at this point is that this is required. For ex,
they do this for the DUNIX port:

CC = cc -std
LDFLAGS = -non_shared -T 20000000 -D 40000000

So assuming I *really* need 32-bit binaries, does anyone know of an
easy way to do it?
