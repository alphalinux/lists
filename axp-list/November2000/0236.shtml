Hello!

    We have a cluster of Alpha DS10 with a DS20 as front-end and I
cannot change the stack size limit on any of them as a normal user even
when the hard limit is unlimited. I have RedHat Linux 6.0 with Kernel
version 2.2.12 running on the cluster

    With the bash command ulimit and csh limit I can do it as root, but
it is impossible if you are a normal user. You get a message like this:

     ulimit: cannot raise limit: Operation not permitted

    Do someone know how can I solve this problem? The users do need
greater stack_size limit that the default value: 8192 kbytes.

    On the other hand, the virtual memory size is abnormally big as
default:

    hard limit (kbytes): 18014398509481982

    soft limit (kbytes): 9007199254749183

and as I get from the information available, this value cannot be
changed with ulimit or limit. How can it be changed then?

    Thanks

    Best wishes

        Elena
