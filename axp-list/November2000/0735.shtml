> "ML values are represented as 32-bit quantities, so all code and data
> has to live in the first 2^32 bytes of address space, or things break.
> The run-time system is actually designed to support a 64-bit
> implementation, but the compiler needs substantial reworking to do
> so."

This is exactly what -taso fixes. However, -taso isn't actually called -taso
in gnu ld, so see the manpage. I was under the impression that ld 2.9.1 has
it, but the archive for axplist should have a note from Richard Henderson
with the details. He's the one who provided both the clues initially and the
later implementation. (I'd go look myself except I'm on an airplane...)
