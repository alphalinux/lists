Just a couple points I've noticed about RH7.0
running on Intel. RH appear to be moving toward
the MS model of SW. First there was kudzu, which
I always disable. Second, they've added a daemon
called rhnsd which appears to be designed to
automatically retrieve SW updates for you, again,
I immediately disabled it. Third, they've totally
screwed up inetd configuration by switching to xinetd
which looks nothing like inetd and in fact is broken.

I'm sure there are other things I haven't noticed yet.
If you expect RH to install your devices the way they
see fit and you expect them to update your SW for you
(consider how this could effect a real server) then by
all means install RH7.0.

They've finally crossed the point where I'm actively
looking for another distribution to replace them.
