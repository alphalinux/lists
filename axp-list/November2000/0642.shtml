Absolutely. In fact, 2.2.18 (when released) will build and boot with
2.96 as shipped in Red Hat LInux 7 for Intel (the biggest problem was
a Makefile bug that was calling cpp in -traditional mode when using
GNU extensions to macro arguments).

I've built 2.2.x kernels with the 2.96 snapshot as shipped in Red Hat
Linux 7/Alpha and ran on them for weeks on end. I built and booted
2.2.x kernels on Intel after fixing the cpp arguments and some other
random bits and booted without problems. The last time I saw a
booting problem on Intel was before we fixed the slew of sibling call
optimizations that were screwing the stack.
