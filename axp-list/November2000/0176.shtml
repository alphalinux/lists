While I realize it's not exactly what you are looking for, but I have
observed a speedup by compiling the jpeg library with the Compaq C
compiler and using a modified makefile setup for interflow optimizations.
This was on a 533Mhz 21164a, so I know it did not generate the SIMD
instructions. I don't know if the compiler is smart enough to generate
them if you to compile for a 21164PC or later. I'm pretty sure the use
of the byte extensions and greater inlining of the code made a
difference for me. If you're interested I can post the modified
makefile to the list or the alphalinux.org site.
