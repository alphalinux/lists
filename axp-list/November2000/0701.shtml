On Wed, 29 Nov 2000, Maurice Hilarius wrote:

> With regards to your message at 04:45 PM 11/29/00, Rich Payne. Where you
> stated:
>
> >Correct me if I'm wrong, but doesn't Linux already use all the available
> >swap partitions in a RAID like sceme? In other words it balances them out
> >instead of waiting for one to fill, then starting on the next.

> Yes, but in a single sequential stream.
> Ultimately we hit the wall with the bottleneck on the head/media interface.

Given the following from fstab on one of my systems:

/dev/sda1 swap swap defaults,pri=0 0 0
/dev/sdb1 swap swap defaults,pri=0 0 0
/dev/sdc1 swap swap defaults,pri=0 0 0
/dev/sdd1 swap swap defaults,pri=0 0 0
/dev/sde1 swap swap defaults,pri=0 0 0
/dev/sdf1 swap swap defaults,pri=0 0 0

Are you saying that swapping is *not* striped across all those partitions
similar to RAID0, given that they're at the same priority?
