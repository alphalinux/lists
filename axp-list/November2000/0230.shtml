Greg Lindahl wrote:
>
> > 4 DIMM's supply 2.6GB/sec peak bandwidth.
>
> Not in any real sense. Reasoning about memory systems is much less useful
> than measuring them.
>

Just quoting the marketing info. You and I both know
the only way to know how fast a machine will be is to
run one's own codes on it and see. STREAM is not
representative of any real applications I use.

> > If you want more bandwidth, buy a DS20E.
>
> Why? The stream benchmark is only slightly different.
>

Spec wise, the DS20E should be faster. I don't buy
them so I don't have any measurements. They aren't worth
the extra $$$ to me. They do however have twice the
memory bus width. Whether STREAM can use it is hard to
say. I consider STREAM pretty naive code so it's not a
useful measurement to me.

> If your codes run better on the DS20E, it's probably due to DDR cache or
> bigger cache.
>

Cache of course can influence performance, depending on
your problem. On some problems, a larger cache does nothing
and on others, it helps. Cache in general can hurt the
performance of certain classes of problems so, again, always
run your own codes on a system you're considering to see how
it behaves.
