With regards to your message at 12:00 PM 11/11/00, Rich Payne. Where you
stated:
> > Similar behaviour has been observed on the UP1100.
> > If turned off at the wrong time during reboot ALL set variables are lost.
> > It has been reported.
>
>This is somthing different. The UP1100 problem only occurs if you power of
>the board at a certain time, at any other time all the vars stay as
>intended. This gentleman is saying that whenever he powers the board off
>it goes back to defaults.
>
>--rdp
Absolutely true.
However, what is common between these is the "most recent SRM".
We have observed similar behaviours, in 5.6.x SRM on various machines.
On LX, for example, the exact way to trigger the "forgetfullness" is not
precisely determined yet, but we have seen it occur on a few occasions.
I suspect there is an underlying common element in this problem, in current
SRM code, which affects a number ( if not all) of platforms.
If it is any consolation we have not seen this problem on UP2000 platform yet!
