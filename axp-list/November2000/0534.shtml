On Fri, 24 Nov 2000, W Bauske wrote:

> Just a couple points I've noticed about RH7.0
> running on Intel. RH appear to be moving toward
> the MS model of SW. First there was kudzu, which
> I always disable. Second, they've added a daemon

I always disable KUDZU myself. The fact that it comes with the distro
doesn't bother me. It takes less than 5 seconds to disable.

The very FIRST time it runs it IS interesting to see what it thinks is in
the system. It correctly idetified a mismarked ethernet card for me once.
If it hadn't been for Kudzu I would have either had to

        a. replace the card with a different one
        b. pull the card out and realized that the chipset numbers are
          different than the label

I'm not a fan of Kudzu, but that one instance saved me serious time in
trouble shooting a problem. I consider it another tool that I keep in the
toolbox. Right next to the Big Hammer.

> called rhnsd which appears to be designed to
> automatically retrieve SW updates for you, again,
> I immediately disabled it. Third, they've totally

Again 5 seconds to disable it. For the people trying to get Linux "to the
desktop" this idea makes sense. Just not for you and me or other l33t
haxorz

> screwed up inetd configuration by switching to xinetd
> which looks nothing like inetd and in fact is broken.

inetd itself is such a pice of crap I'm suprised you still use it!
Personally I started using Dan Bernstiens TCPserver a LONG time ago when
SEVERAL boxes couldn't handle various services (POP3 FTP) under heavy
load. Indetd would take a dump. (x86 linux, alpha linux, Solaris 2.4,2.5
all had the same problem) Never played with xinetd, but almost ANYTHING is
an improvement over inetd with a seriously LOADED server.

> I'm sure there are other things I haven't noticed yet.
> If you expect RH to install your devices the way they
> see fit and you expect them to update your SW for you
> (consider how this could effect a real server) then by
> all means install RH7.0.

Again great for people that are trying to get into Linux for the first
time. Great when you ship a broken package with your distro and need to
upgrade so that "ping" works.

> They've finally crossed the point where I'm actively
> looking for another distribution to replace them.

I've been playing this game a LONG time. My first Alpha Linux distro was
"craftworks" (anyone remember that??) I think in the end for advanced
users no distro is perfect. For me I read the lists, and use the most
"stable" version if RH. I do a text install, choose my packages. Install
things from source that I want, ssh, pro-ftp (WU-ftp bleach) and go from
there. Generally a painless procedure once I compile a kernel for that
machine I'm done.

Upgrades are generally done a package at a time and only whats needed.
This puts me agreement with the person that reccomends only upgrading the
packages you need when you need. That tends to really work for me, YMMV

RH as a company is doing a LOT to promote Linux, and I've never had a
"show stopper" from them. So I continue to "remain faithful". I feel the
same way about Solaris. Never been a fan of a Solaris install either.

I have always been happy with RH Each users mileage varies. I'm not going
to stop using a distro because they are trying to make things easier for
the un-initiated. As long as I can remove or disable the things I don't
like (or not install them) I'm happy. Perhaps, maybe someday RH will
change their INSTALL procedure. All we have to do is ask?
