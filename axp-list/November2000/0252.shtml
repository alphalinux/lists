The RH default RHat kernel is generally OK. I've found that if you compile
the kernel specific for the platform especially on the AS800 / DS3300
(Noritake Primo) things tend to be more stable. On my LX and SX, the generic
kernels run just fine though.

RH compiles the kernels as generic to support as many platforms as possible.
It makes dealing w/ all the different platforms out there a heck of a lot
easier instead of having a seperate kernel for each platform on the CD.

The same thing goes for the modules. It's easier to make as many drivers
into modules as possible and then detect if they exist in the system when
you're dealing w/ multiple platforms of varying hardware.
