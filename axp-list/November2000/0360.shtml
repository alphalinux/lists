On Thu, 16 Nov 2000, Robert Williams wrote:

> I've been trying to get software raid working
> on an UP1000. With kernels 2.14 and 2.16,
> I get the following. What does this indicate
> and is there a way around?

Are you using the stock Red Hat kernel, or one that you built from
pristine sources? If not the stock RH kernel, one of the raid patches
from http://people.redhat.com/mingo/raid-patches/ should be applied.

If you've already done the above, you might try doing a "raidstart
/dev/md0". You shouldn't have to do that since you're using persistent
superblocks, but oddly enough that was the only thing that worked for me
in a similar situation using pristine 2.2.16+raid-2.2.16-A0 on a RH 6.1
box.
