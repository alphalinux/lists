> Be a little wary of associating peak disk performance and swap
> performance. Paging is done in (I think) 8KB pages on Linux Alpha -
> that is not the way to get good IO performance.
>
> I have done a number of tests hoping to use paging for good ol'
> fashioned job-swapping (like suspend a 5GB job and start another 5GB
> job when you only have 6GB of RAM).

Patient: Doctor, it hurts when I do this.
Doctor: Don't do that.

Traditional Unixes have both a pager and a swapper. Linux only has a pager.
This means that it doesn't do what you want efficiently. There is some
attention to clustering pageouts, but it's nowhere as efficient as a real
swapper. If you need a real swapper, you'll have to write one.
