Well, I just completed a up to date rebuild and install of
XFree 4.0.1. It's up and running and it appears to be
doing DRI. I'm seeing two immediate issues...

1) my colors seem to be off slightly. What was grey, now
seems to be a grey with a bit of brown so it looks kinda
tan. I'm not sure if I don't have everything symlinked up
right, so it may be an issue with what I built.

2) In skimming the build log, not everything was built
with the compiler switches I specified in host def. For
example, I specified -mcpu=ev56 in host.def; but many
command lines did not have the switch. For now I guess
this means it's not as fast as it could be.

For the curious, here's what I did from my notes. I have
also attached a server startup log to the end of this message.

Alan

I'm using GCC 2.95.2 on a updated SuSE 6.3 system. It has
various updates including Glibc 2.1.3. It is running
a slightly patched 2.2.16 kernel. My system is a LX164.
It is running AlphaBIOS 5.70. It has a Voodoo 3000 PCI
card.

1) Did a CVS checkout/update of dri and Glide-64

2) Build Glide

   a) edited swlibs/include/make/3dfx.linux.mak and
      makefile.autoconf.bottom and changed the -mcpu=ev6
      to -mcpu=ev56 -Wa,-m21164a -DBWIO_ENABLED

   b) (might be specific to me) copied acinclude.m4 to
      the glide3x directory

   c) created a symlink in glide3x for swlibs. depending
      on how you checked out glide you may not need to
      do this.

   d) configured glide with (note: this is all one line)
     ./chores.3dfx --clean --generate --configure='--disable-fx-debug
--enable-fx-glide-hw=h3 --enable-fx-dri-build
--enable-build-architecture=alpha'

      If you are using a Voodoo5, then change h3 to h5.

   e) built glide with ./chores.3dfx --build

3) Install Glide

   a) backed up my old glide headers and libraries

   b) installed updated glide by running ./build3dfx install
      from the glide3x/build directory

   c) ran ldconfig as root

4) Build Xfree 4.0.1e

   a) configure build directory per instructions on the DRI webpage

   b) edited host.def. Specifics...

   #define DefaultGcc2i386Opt -O2 -mcpu=ev56 -Wa,-m21164a -DBWIO_ENABLED
   #define LibraryCDebugFlags -O2 -mcpu=ev56 -Wa,-m21164a -DBWIO_ENABLED
   #define BuildServersOnly YES
   #define XF86CardDrivers vga tdfx glint
   #define LinuxDistribution LinuxRedHat
   #define GccWarningOptions -Wall -Wpointer-arith -Wstrict-prototypes \
                             -Wmissing-prototypes -Wmissing-declarations \
                             -Wnested-externs
   #define DefaultCCOptions -ansi GccWarningOptions -pipe -g
   #define NormalLibGlx NO

   #define BuildXF86DRI YES
   #define HasGlide3 YES

   /* Optionally turn this on to change the place where you install the build
*/
   #define ProjectRoot /usr/X11R6-DRI

   /* Optionally turn this on to force the kernel modules to build */
   #define BuildXF86DRM YES

   #define XnestServer NO
   #define XVirtualFramebufferServer NO
   #define XprtServer NO

   c) changed to the dri-cvs/build/xc directory and entered

          make World &> World.log.txt

      and then went off to watch Red Dwarf. :)
      The build took about 1.5 to 2 hours.

5) Install Xfree 4.0.1e

   a) shutdown Xserver

   b) backed up current /usr/X11R6-DRI directory just in case...

   c) made a new /usr/X11R6-DRI directory

   d) cd /lib/modules/<kernel_version (2.2.16 for me)/misc
      mv tdfx.o tdfx.o.bak

   d) cd'ed to dri-cvs/build/xc and did a

         make install &> install.log.txt

   e) copied
     
dri-cvs/build/xc/programs/Xserver/hw/XFree86/os-support/linux/drm/kernel/tdfx.o
         to
      /lib/modules/<kernel_version>/misc

   f) did some clean up in the include and lib directories.
      I had to add some symlinks in my standard X11R6 directories
      to point to the X11R6-DRI equivalents. This is because most
      stuff looks in X11R6 for libraries and headers and does not
      know about the DRI directories.

      I also had to add a symlink from /usr/X11R6-DRI/lib/libOSMesa.so.3
      to libMesaGL.so.3.

   f) run ldconfig. NOTE: your ld configuration needs to be changed to
      have /usr/X11R6-DRI/lib BEFORE /usr/X11R6/lib!!!

   g) sync for luck

   h) started Xserver...

   i) Once X was up, I tried some of the Mesa demos. A good one to
      run is glinfo. It should provide some output like:

      GL_VERSION: 1.2 Mesa 3.4
      GL_EXTENSIONS: GL_ARB_multitexture GL_ARB_tranpose_matrix
GL_EXT_abgr GL_EXT_clip_volume_hint GL_EXT_compiled_vertex_array
GL_EXT_histogram GL_EXT_packed_pixels GL_EXT_paletted_texture
GL_EXT_point_parameters GL_EXT_polygon_offset GL_EXT_rescale_normal

GL_MESA_resize_buffers GL_NV_texgen_reflection GL_PGI_misc_hints
GL_SGI_color_matrix GL_SGI_color_table GL_SGIS_pixel_texture
GL_SGIS_texture_edge_clamp GL_SGIX_pixel_texture
     GL_RENDERER: Mesa DRI Voodoo3 20000821
