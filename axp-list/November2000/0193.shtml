On Wed, Nov 08, 2000 at 02:02:14PM -0000, David Addison wrote:
> This was working fine until I merged in the API EV6 2.2.17 assembler patch
> and it then broke everything in that init seems to fail to parse command
> line options correctly.

That means that the new ev6 code is not being careful with
the .prologue directives. Those indicate if and how the
incoming r27 value is used.

> I just tried adding the -relax flag to CFLAGS as '-Wa,-relax' and now the
> arch/alpha/lib/copy_user.S code fails to compile;
[...]
> lda $31, $exitin-99b($31);

The -relax option as given to the assembler tries to leave
enough relocations in the object file that the linker could
delete instructions out of the middle of a function. (The
linker doesn't do that at the moment, merely nops them out.)

We do not have a 16-bit pc-relative relocation, which means
that we cannot represent this instruction, thus the error.
