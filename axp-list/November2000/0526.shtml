W Bauske (wsb@paralleldata.com) wrote:

> Just a couple points I've noticed about RH7.0
> running on Intel. RH appear to be moving toward
> the MS model of SW. First there was kudzu, which
> I always disable. Second, they've added a daemon
> called rhnsd which appears to be designed to
> automatically retrieve SW updates for you, again,
> I immediately disabled it. Third, they've totally
> screwed up inetd configuration by switching to xinetd
> which looks nothing like inetd and in fact is broken.

Sad, but true.. I really dislike how they do all this without giving you
a choice over the matter.

> I'm sure there are other things I haven't noticed yet.
> If you expect RH to install your devices the way they
> see fit and you expect them to update your SW for you
> (consider how this could effect a real server) then by
> all means install RH7.0.

Exactly how I see it.

> They've finally crossed the point where I'm actively
> looking for another distribution to replace them.

I went to Debian from RedHat some years ago and haven't looked back
since. One of the final straws, for me at least, was having
configuration files overwriten with new "stock" ones. Having your apache
(just an example) config replaced with a generic one is completely
unacceptable! Plus you don't know what state software is in when you
install it (is it known to be stable or unstable). I've OFTEN found
Debian's unstable tree to be more stable to any of RH's .0 releases.
RH's fix to security problems (as far as I've ever seen) has been to
release the latest version of the said software (often a beta version),
where with Debian most security fixes are back ported to a known stable
version...

JMO,
Ron
