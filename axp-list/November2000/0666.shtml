I'm using vixie-cron-3.0.1-40 on a stock rh 6.2 distribution running kernel
2.2.10 and my crond keeps dying.
It seems to have started when I installed mailman-2.0rc1
http://www.gnu.org/software/mailman/mailman.html

Below is what I've pieced together from the logs. The trend seems to be
that the python script gate_news is run from mailman's crontab, an
unaligned trap shows up in the message log and crond dies. I've since
removed the gate_news entry from mailman's crontab and I'm waiting to see
if that fixes the problem. The funny thing is that the gate_news process
is scheduled to run every 5 minutes and does so without any problems but
crond seems to die at random every ~24 hours. I also dont understand why
a failure in something called by crond should cause it to die...
