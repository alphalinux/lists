On Sat, 25 Nov 2000, Robert Williams wrote:

> Sure some config files are overwritten,
> but reports of those changes are stored in the update
> log file, and the old configuration file is saved.

Key word: SOMETIMES. I've run into situations on RH servers that I
inherited where config files where just overwritten without notice and
without a backup. It may have been a packaging error or the fault of an
older version of RPM, but it was still enough to leave a bad taste in my
mouth towards future upgrading on a RH system.

> The upside of this is that you must pay attention
> to these changes right away.
> At worst, you may have three or four config files
> to "port". The format of some config files changes
> radically from time to time (apache for example).
> You are given a choice in this case... You can
> overwrite the new config file with your rpmsave
> file, or transpose the important parameters into
> the new config.

Again, SOMETIMES. I'll be the first to admit that Debian's system isn't
perfect WRT config files either, but even recent experience has shown it
to work better for me than RH/RPM's. I think the key thing is, when I do
massive updates, I'd rather know right away that a config file needs to be
modified than have to go back and look at a log (meaning some services may
not have restarted properly). On a running server, I just don't have time
to have a service or two down for that long....

> Stable fixes can usually be installed in older systems, one way
> or another. However, it is certainly true that Debian is a good
> distribution.
