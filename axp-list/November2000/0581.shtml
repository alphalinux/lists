I changed from AlphaBios to SRM
this weekend, also installing a new
40G Maxtor Ultra ATA 100 to make
a fresh Linux install, as recommended.
The Linux install was eventless and the
system comes up on the Red Hat
install 2.2.14 kernel. This kernel
does not support dma, so I wasn't
concerned to see that the new disk
was running at about 2 MB/s according
to hdparm -t. However, after building
a new 2.2.16 kernel with dma support,
the system will not boot to completion
on this kernel.
Upon recognizing the ide devices, the
boot process reports
"hda lost interrupt"
repetedly until I stop it. The new disk
is attached to the primary ide.

This same 2.2.16 kernel runs well when
booted from one of the other ATA66 disks
in the system.

Is this a problem with the ATA 100 drive
and Linux drivers?
