Greg Lindahl wrote:
>
> > Spec wise, the DS20E should be faster.
>
> But it isn't, because STREAM is The Right Way to measure memory peak form
> the memory system.
>

Just for grins, if you like STREAM so much, run two simultaneous
tests on a DS20E and a UP2K and see what the aggregate bandwidth
is. I would suspect with a switched architecture, a CPU talks to one
bank at a time. So, unless you run a problem that forces use of all
memory banks, it probably makes no difference compared to a UP2K.
Once you cross that line, I suspect you'll see a difference. Pick
the STREAM total data size to be, say 45% of your memory so with
two active, you use 90% of real memory. Note STREAM uses something
like 3 arrays so one array needs to be 15% of memory.

That should give the memory systems a good workout. Make sure you
populate all memory banks on both systems to keep it consistent.
If that doesn't show the difference, then it probably means the
cross bar switch is falling down on the job or there is an OS problem.

In the end, the only truly good way to measure a system is with your
own codes and with your configuration. At least then you see what the
system does with your environment.
