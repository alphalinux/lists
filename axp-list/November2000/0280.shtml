On Sun, 12 Nov 2000, Michal Jaegermann wrote:

> On Sun, Nov 12, 2000 at 12:55:49PM -0500, John Jasen wrote:
>
> > In practise, we've manipulated 20TB arrays at work for testing,
>
> Does that mean that you had working filesystems of that size?
> If not then what really this number describes? Raw access
> devices?

We're doing funny things, so its probably more accurate to call them raw
access devices.

I don't know to what limits ext2fs can be effectively pushed, versus
theoretically.

> One can push filesystem limits by using bigger blocks and this
> surely makes sense with big filesystems anyway; but I am not sure
> if there are not some limits in the current Linux implementation
> of a block device. In any case we are talking about numbers
> in many terabytes range.

The current limits are in VFS, with, as I think I recall, are for
filesize: 2^^(CPU bits - 1) and filesystem size: blocksize^^(CPU bits -
1).

This is why x86 has the 2GB file size limitations, BTW, at least under
kernels before 2.4.x.
