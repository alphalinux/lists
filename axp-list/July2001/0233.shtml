> Alpha. The problem
> with the Qlogic driver is that it assumes that sizeof(dma_addr_t)
> == BITS_PER_LONG.
> On the Alpha, this is not the case.
>
> If someone wants the patch I can send it out.

I have developed a patch for the same symptom. I'd like to see
what you have done, Please send what you have to me. I've also
noticed that even after this has bee enabled. proc & lspci
still report the card's PCI addressing as 32 bit non-prefectable...
I fear that the damn thing will need it's bios initialzed or
at the very least it's NVRAM twiddled. Furthermore :-), The
64 bit addressing support is coded for linux 2.4 only.

Another issue that I've realized working with 64 bit capable PCI
cards latley is that the drivers make the assumption that your boxen
may only have 64 bit PCI slots if the BITS_PER_LONG > 32. That
simply isnt the case anymore on other platforms like IA32.
IA32 SMP server clase boxen regularly come with 64 bit PCI expansion
these days. It's not OUR problem but it's something that should be
addressed. The less #ifdef's the better.
