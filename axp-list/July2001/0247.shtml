On Sat, 21 Jul 2001, Robert M. Riches Jr. wrote:

>Date: Sat, 21 Jul 2001 18:43:30 -0700
>From: Robert M. Riches Jr. <richesr1@inetarena.com>
>To: axp-list@redhat.com
>List-Id: Linux and Red Hat on Alpha processors <axp-list.redhat.com>
>Subject: any way to restore keyboard function
>
>Is there any known way to restore normal keyboard function
>after the keyboard has been rendered inoperative by bttv
>and/or xawtv?
>
>The platform is a UP2000 running RH 7.0 with XFree 4.0.3-9
>(to try to solve problems with 4.0.1, etc.).
>
>Shortly after starting up xawtv, usually when I try to use
>the option menu to actually select the composite input
>(which the xawtv developer says is a very, very old bttv
>driver bug), the mouse and keyboard become entirely
>unresponsive. Even the Num Lock light does not toggle.
>
>Some days, this will happen three times in a row. Other
>times, I can go a week (starting xawtv once a day) without
>seeing the problem.
>
>So far, the best solution is to log in to a serial
>terminal and do "shutdown -r now".
>
>Is there any known way to restore the keyboard and mouse
>drivers/interrupts/etc. to a functional condition?

On x86 with PC keyboard I do this by hitting "2" "F12" "=" in
rapid succession. You must press and hold each key without
release, and may have to do it several times to get it to
generate the proper scancodes. If done correctly on a VC with a
prompt (visible or not), it will generate CTRL-C "X", ENTER,
which will start up a dummy X server. Usually I find C-A-BS
fixes things after that.

Dunno if this works similarly on AXP or not though..
