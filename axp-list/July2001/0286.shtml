On Mon, Jul 23, 2001 at 02:55:15PM -0400, Christopher C. Northrop wrote:
>
> I read all the replies to this issue then went and made a 150m /boot
> partition, and the thing it worked.
>
> My question is, how come RH 6.2 - 7.0 did not have this issue? I have a
> Alpha with RH7.0 running right now with a 3.5g root directory - / . This
> machine never had a booting issue.

This is not a question of a partition size but of a position of your
boot image in it. If it is all below "magic limit" you win, otherwise
you loose. The simplest way to ensure that you always win is to
boot from a partition small enough so your boot image will never be
"too far".
