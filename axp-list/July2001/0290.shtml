Hi there,

I've a little Problem. I have a AlphaStation 200 with Redhat 7.1 installed. I
use an additional Promise Ultra100Tx2 IDE/ATA100 PCI Controller and I am
booting through the SRM console.

With the original Redhat Kernel (2.4.3) the Promise Ultra100 did work fine. I
then tried compiling the new 2.4.7 kernel and the Promise Card isn't recognized
any more. I've build the IDE/ATE support directly into the kernel (not a
module). I used the kernel config from the original Redhat (make oldconfig in
the Redhat kernel src-dir), copied this file to my 2.4.7 kernel and adjusted
it. I deselected all low-level IDE interfaces except my Promise PD... The
optione for UDMA by deault, IRQ sharing etc. are the enabled (as in the
original RH kernel).

I can compile and install the kernel and modules. When I boot the new kernel
I don't see the lines where the Promise Card is initialized. But booting goes
on normally until the initscripts try to fsck my ide-disk (mounted on /home).

My question: Has someone any ideas what kernel config options are necesary so
that my Promise card will work with my custom kernel.
