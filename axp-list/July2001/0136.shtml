On Mon, Jul 16, 2001 at 08:59:47AM +0200, Linus Gasser wrote:
> All I did was to 'install' the kernel-source from RH7.1 on my RH6.2-system,
> and then recompile this kernel.

Kernel is really _the least important_ part of the whole deal. You can
run 2.4 kernels on RH6.2 based systems for "always". Mind you - a
number of packages should be updated. In particular modutils (and they
are still fine after that to run 2.2 kernels). There are requirement
listed in 'Documentation/Changes' file in kernel sources. You may
ignore them at your own risk but don't complain later about nasty
surprises.

> - One of the things you gotta take care of is to turn on the option
> CONFIG_ALPHA_LEGACY_START_ADDRESS
> if not you get the kernel in a wrong adress space (that is, for MILO).

That is actually quite important (but this has to be off for some
"high end" machines which are not using milo :-).

> And, after all, I have to admit that I don't run a 2.4-kernel anymore. I
> switched back to good ol' 2.2.19, as it's the only one that runs stable on my
> system.

And even if updated the whole software setup to RH 7.1 you are not
really obliged to run, by the same token, a kernel from 2.4 series
(you may need to install a "generic" version of glibc instead of a
processor-optimized one - depending on your board). The version of
a kernel packaged with RH 7.1 is not a very good one in my experience.
I even used an expression "total turkey" in some my older postings
on this list. :-) There are other options and 2.2.19 is one of these.

Once again - a kernel version is nearly coincidental in a given
distribution. These are libraries and the whole "user space" which
makes that difference.
