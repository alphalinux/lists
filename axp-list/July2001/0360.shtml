Hi -- occasionally, when I compile somthing on my alpha, I
get this error:

In file included from /usr/src/linux/include/asm/semaphore.h:11,
                 from /usr/src/linux/include/linux/fs.h:173,
                 from fakeaudio.c:25:
/usr/src/linux/include/asm/current.h:4: global register variable follows a function definition
/usr/src/linux/include/asm/current.h:4: warning: call-clobbered register used for global register variable
make: *** [fakeaudio.o] Error 1

I mentioned this on the list last September, and Richard June
said the workaround was to add '#include <asm/current.h>' before
anything else. It worked at the time, and I tried the same thing
this time. While the program compiled, I still got this warning:

In file included from fakeaudio.c:21:
/usr/src/linux/include/asm/current.h:4: warning: call-clobbered register used for global register variable

I'm using kernel 2.2.19 now, which is of course an upgrade from
whatever I was using last year. Is there someone on this list
that may be able to address this issue in the kernel?

I'm not familiar with the details of kernel innards or alpha
architechture, I just run into this bug sometimes while compiling
software.
