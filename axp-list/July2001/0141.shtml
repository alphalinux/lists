    Hi

    I was trying to upgrade a Ruffian (ARC BIOS) Alpha from
    RedHat 7.0 to 7.1; and I have found several problems.

    It seems MILO is the only boot loader I can use on this hardware,
    right?
    Well it seems it uses kernel start address 0x310000 (hardcoded)
    while kernels 2.4.x use 0x810000 unless you use
    CONFIG_ALPHA_LEGACY_START_ADDRESS when compiling the kernel.

    I'm afraid this means you can't boot from RH 7.1 kernels in order
    to install from scratch...

    If you try to load the kernel from generic.img using installed
    (old) milo, it will fail and return to MILO prompt.

    If you use milo from milo/ruffian.img on RH 7.1 distribution,
    there are two problems:

    - First: it can't find MILO and prompts for RUFF.MILO diskette.
    It seems the problem is (according to `strings linload.exe`)
    it looks for "milo.ful" while on that image the file is named
    "milo":

| Load MILO Utility 1.0 (Copyright DeskStation Technology, Inc. 1997)
| Searching for MILO files in current partition...
| osfpal.nh
| milo.ful
| Unable to find MILO files in current partition.
| multi(0)disk(0)fdisk(0)\
| Please insert floppy disk RUFF.MILO #1 and press a key...

    - Second: well, renaming the file starts working, but when milo
    gets loaded, you don't get the prompt; it hangs forever at:

| Loading PAL File: multi(0)disk(0)fdisk(0)\osfpal.nh
| PAL File with 49088 bytes loaded at 80200000
| Loading MILO File: multi(0)disk(0)fdisk(0)\milo.ful
| PAL File with 565400 bytes loaded at 80500000

    So, the questions are:

    - Is it possible to build a milo from sources which will be
    able to load 2.4.x no-CONFIG_ALPHA_LEGACY_START_ADDRESS kernels?

    - There is another way to upgrade a Ruffian host from 7.0
    (kernel 2.2.19) to 7.1?

    - Will dri/"NoAccel" "off" work for a "Matrox|MGA G200" with
    the kernel/X11 included on 7.1?
