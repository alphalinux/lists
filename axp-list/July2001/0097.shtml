On Fri, Jul 13, 2001 at 11:45:34PM +0200, Thorsten Koch wrote:
>
> One big problem for us are applications without a Linux/AXP version.
> One possiblity is to copy the shared libraries from Tru64 to Linux
> but that seems to help only for some of the programs.
> (e.g. I got ILOG CPLEX to run on Linux, but only the sequential not the
> multi-threaded version)

Not all DU system calls are emulated by Linux. Some can be just
ignored. Other may put in a monkey wrench. Things which depend just on
libraries should run - as long as you are able to provide required
libraries. Tru64-compat package from Red Hat distribution has only some
(basically what you need to run Netscape from DU but this is enough for
quite a few other programs). Other than that Linux runs DU binaries
provided 'a.out' (i.e. ECOFF) support is compiled in (it can be in a
form of a loadable module).

If you feel like hacking sources to add more OSF system calls -
go ahead.

> The second problem are developer libraries without a Linux version,
> since the last time I tried it was not possible to link Tru64
> libraries with Linux (ECOFF) object files.

Linux object files and executables are currently not in ECOFF but in ELF
format. This is Tru64 which is using ECOFF. Linux linker actually may
link ECOFF but trying to mix-and-match both is not likely to succeed.
