Linux does not know how to mount an NTFS partition. I have heard that there
are file systems that will allow you to read, but not write NTFS, but they
are still experimental.

DEC Unix ( or OSF/1 or TRU64 ) also do not know about NTFS.

I believe that you should have used FDISK, because any drive that was used
with DEC UNIX should have a BSD disklabel. FDISK can write either BSD or M$
type disklabels ( partition tables ), but Disk Druid only knows about M$
type.

Disk Druid MAY have screwed your BSD disklabel to the point were it will
take some SERIOUS hacking to get your data.
