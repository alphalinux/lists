Beeing called explicitely to a blackboard I can only say that
I already "shared some experiences" although in a private mail. :-)

Althought most of our experiences comes from running 4.0.3 (various
cards) with a server recompiled by yours truly with libraries
from RH 6.2 I do not think that the later is a significant factor.
I do have also RH 7.1 running but a range of tried cards is smaller.

> I believe many improvements came forth with these patches for
> Alpha.

Indeed this is the case although dealing with a video does not seem to
be always a hard science. There could be a deep reason for a Voodoo
name for graphics cards. :-) I think that above 4.0.3-15 version we did
not observe any major gotchas. These which showed up in RH 7.1 for
Alpha are 4.0.3-21. The problem Robert has that these versions with
binaries compatible with libraries from RH 7.0 (which he is using and
not updating for some reasons) are hard to find. I do not think that
they were ever officially released and "rawhide" tends to change quite
fast in places. Later stuff may assume that some updates were already
made. Mike, maybe you still have these binaries stashed somewhere? Or
4.0.3-21 will still work on RH 7.0? I never tried. Well, you will
definitely need to update Mesa but this will have to be done anyway.
If you do have an Alpha with an AGP slot then having around 'agpgart.o'
module and allowing it to load can be a grave mistake.

Jay was also promissing more Alpha patches for 4.1 (hint, hint :-).
