On Fri, Jul 20, 2001 at 08:27:42AM -0400, Jim Simmons wrote:
> I'm having a strange problem with a PC164LX machine and RedHat 7.1.
>
> This machine was running 7.0 using SRM without any problems. I merged
> the contents of the two 7.1 install CDs, applied the latest updates, re-ran
> genhdlist, then did booted from floppy and did an NFS install of 7.1.
>
> The install completed without any problems, but when I attempt to boot
> the system from the scsi disk it looks ok until I get:
>
> aboot: booting from device 'SCSI 0 5 0 0 0 0 0'
> *** no timer interrupts on CPU 0 ***

Not sure about this one...

> aboot: valid disklabel found: 4 partitions.
> aboot: loading uncompressed boot/vmlinuz-2.4.3-12...
> aboot: loading compressed boot/vmlinuz-2.4.3-12...
> failed to send Read to dka0.0.0.5.0
> failed to send Read to dka0.0.0.5.0

Now THIS one I've seen... :-\

Doesn't happen on all platforms, but frequently enough to be a nuisance,
but not (as of yet) a big enough nuisance to figure out why. Must be
something in aboot... ???

Workaround is to copy that kernel to /vmlinux.gz, and modify aboot.conf
accordingly. This has worked every time for me, but of course, YMMV... ;-}

Good luck.
