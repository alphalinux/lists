On Tue, Jul 17, 2001 at 08:25:52AM -0600, Scott Serr wrote:
>
> I have an old Multia that I have run RH 5.2 on for 3 or 4 years. I used
> to boot with ARC. My firmware is 3.8-3 if I remember right. My 340MB
> internal SCSI drive died along time ago. My floppy died a couple years
> ago. But they weren't used much. It has 80MB of RAM.
>
> I decided to install 7.1 and found that the smart/supported way to do it
> was with SRM. (my first experience with SRM... it's alot cooler than ARC
> in my opinion)
>
> PROBLEM:
> I'm booting off a SCSI CD (boot dka1). It has a little trouble
> with SCSI resets but manages to get aboot going and then bootstraps the
> Linux kernel. The X install won't work. I get an error in /tmp/X.log
> can't remember what it is exactly.

Same as all the others get with TGA console: the latest 4.0.3 TGA
server requires a "BusID" option in the XF86Config-4 file, and RH's
installer doesn't put one there... :-\

> Anyway, I rather do the text mode
> install. In text mode I answer the standard questions. I let it
> automatically format a wiped disk. I get to the point that it wants to
> set the root user password. After I type one in and hit enter, then it
> (anaconda?) hangs! I can still switch VCs and type in the bash prompt,
> but the install has just stopped.
>
> My md5sums are ok and I've burned several Disc 1s just to make sure it
> wasn't a bad CD. Has anyone installed successfully with text mode? I have
> a feeling that if I could get into X, that install would be more debugged.

The hang is evidence of a problem that afflicts JENSEN SRM as well as
UDBs running SRM: a bug that prevents correct handling of fault
processing of an unsupported (by EV4 CPU) floating point instruction.

There have been various patches available on the net to solve this,
but unfortunately a comprehensive solution has not become part of the
official kernel sources (yet).

I have some patches which I will be submitting to 2.2.20 and 2.4.7;
if you want a copy, please Email me directly.

Once equippped with those patches, you'll need to build an appropriate
kernel for booting from floppy, and then use the RAMdisk/CD normal
install technique, which I think should work.
