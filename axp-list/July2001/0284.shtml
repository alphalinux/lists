Subject: Milo 2.2-18 released.

Hello world,

it's been a while since the last release of Milo-2.2, the Alpha-Linux
boot loader for ARC/ARCS/AlphaBIOS machines.

This release, Milo 2.2-18, has some new features and has a fixed Makefile
so it should work on machines that hung after "swapping to palcode @"

Some of the changes are:

  * Added support for putting MILO into the DEC UDB flash rom.
  * Created new utility (makeflash) that will write a UDB compatible
    flash image suitable for burning into flash using the SRM "update"
    command.
  * Added configuration options to remove support for some filesystems
    to make milo smaller so that it can fit into the "arc" flash position
    (256k). (you need at least one filesystem to boot your kernel from)
  * Init video and keyboard only if MINI_SERIAL_IO is not defined.
  * Fixed fmu for AlphaPC LX164
  * Fixed bogomips calculation to use loops_per_jiffy instead of
    loops_per_sec
  * Added reiserfs (3.6.x) support.

I'd also like some feedback on what Milo images are proving useful.
Testing all different flavours is not possible due to the limited
range of Milo machines I can access (LX164 and Ruffian at the moment)

Thanks to Bill Carr, Will Woods, Frederic. R. Roussel, Kazushige Goto,
Vladimir Saveljev, and others for their patches.

Thanks also for the (unfortunately only few) bug reports.
