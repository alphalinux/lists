> When I look into the SCSI configuration it reports
> ATAPI #0, SCSI ID 255, SCSI BUS 0
> SCSI ID DEVICE DESCRIPTION
> 0 CD-ROM HP CD-Writer 2700 3.01

I'm assuming that SCSI configuration also shows your disk? (Presumably
as 'Disk 0' on SCSI BUS 1?)
 
> The result of all this seems to be that the osloader thinks the
> cd-writer
> is the drive to boot from and I cant find a way to change it.

Given the above assumption, you should be able to go into the os load
options (Utilities... Os Selection Setup...) and just save them again
(F10). Most likely you were booting off 'Disk 0' (which used to be on
scsi 0) and AlphaBIOS just needs to update its idea of what 'Disk 0' is
in the boot selection.

If that doesn't do it, you can go one step further and edit the boot
selectino changing the boot devices manually to match what the disk is
currently called.
