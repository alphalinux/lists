Craig Tierney wrote:

On Fri, Jul 20, 2001 at 11:46:59AM -0700, Eugene Chu wrote:
> Hi Andrei,
>
> Thanks for the information.  I guess that it was not QLOGIC that
> developed the QL2XXX drivers for Alpha Linux?  It's good to know that
> the support is built in.  Now I have to figure out how to get funding
> for the new FCAL cards.

Qlogic did not develop the driver that is in the kernel.  However, there
is a driver from Qlogic that works with Linux.  However, "Linux" in this
sense means i386.  However, you can get the driver to work under Alpha.  The problem
with the Qlogic driver is that it assumes that sizeof(dma_addr_t) == BITS_PER_LONG.
On the Alpha, this is not the case.

If someone wants the patch I can send it out.


Humm sounds like something tbat should be useful in the kernel. Was this patch never accepted upstream? or am I going to horrified to find out it's not GPL code that we can merge with the kernel?
 

I am running the Qlogic-4.27beta driver on an XP1000 quite successfully right
now.  It has been running for the last 6 weeks in production without problems.
I am exporting the filesystem over NFS, so I am not always pushing the filesystem
as hard as if it were local.  With NFS and gigabit Ethernet I am seeing 45 MB/s for
writes and reads (Local performance on writes saturates the FC card).  I never
had too much luck with the driver in the kernel.  The last one I tried was the one in
linux-2.4.3.  Maybe it just didn't like something it saw from my SAN (Data Direct).
