On Wed, 18 Jul 2001, John Doe wrote:

> Hello List,
>
> I am trying to write to a tape on a remote linux
> box by using the following command:
>
> gtar cPvf - $FILELIST | rsh -l $username \
> $remote_host /bin/buffer -o $TAPE_DEV

Others have already commented on using the norewind tape device, and
that's what I'd recommend too. One other thing I'd recommend is that if
the backup is being done over 10Mbit ethernet, you could probably use
compression depending on the cpu power of the host being backed up.
For example:

rsh $remote_host "mt compression 0"
gtar zcPvf - $FILELIST | rsh -l $username \
   $remote_host /bin/buffer -o $TAPE_DEV

This would be more appropriate for relatively recent cpus over slower
ethernet.

Another way to shrink your backup window if you are stuck with a
relatively slow tape drive, is to output the data to files in a holding
area on disk and then flush them to tape using cat or dd. This way your
tape drive is less of a bottleneck.
