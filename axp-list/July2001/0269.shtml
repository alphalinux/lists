Jay,

  Thanks for the speedy reply. I have in fact backed out out the X
  server from 7.0 and am using a 3.3.6 X server for single head systems
  and a 4.0 server with a few patches for dual head systems. I had
  assumed that the problems I saw with the stock X server were the
  result of bugs in our code that were made visible by the newer X
  version.

  In all the X servers I have tried, up to 4.0.3, I have never seen
  really good support for the second ELSA Gloria card. The second head
  works OK in 8 bit color mode, really terrible in 24 bit color mode. I
  looks to me like the second card in not fully initialized. I suspect
  the problem is related to the fact that the second card has VGA mode
  disabled.

  The symptoms are the same on XP1000s and DPW600s under kernel 2.2.13
  or 2.2.17. Do you know of any workarounds for dual headed systems
  using Elsa Gloria cards?
