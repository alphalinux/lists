On Mon, Jul 09, 2001 at 04:07:46PM -0400, Mike A. Harris wrote:
> On Mon, 9 Jul 2001, jared.cook wrote:
>
> >I have a 4 cpu 4100 Rawhide systems that is very unstable with RH 7.1 I
> >have installed RH 6.2 and it is very stable. 7.1 is completely
> >unuseable. Anyone else having trouble with stablity? Is kernel 2.4
> >just not stable enough?
>
> Rawhide is development releases, and is not intended to be used
> on stable production systems.

Misunderstanding. He is talking about "Rawhide" machine type (a kind
of Alpha board) and you, habitually I guess, :-) are mixing that up
with "rawhide" Red Hat development stuff. Jared installed 7.1 release.

Actually from my experience 7.1 is really stable as far as software
goes. There are some relatively minor gotchas; like 'eject' from
a release is quite broken but that one from rawhide (the second sense :-)
is ok. You will find few other things like that and there are already
some updates.

> Our Quad Alpha runs 2.4.5-9 and seems to work fine.
                       ^^^^^

And this seems to be the key element of your success. The RH kernel
2.4.3-12, which is packaged with 7.1 distribution, is in my experience
a total turkey (yes, I wrote about that on bugzilla quite a while ago).
I never run it long enough to see it crash but it is clear that it has
serious virtual memory issues and I would not be surprised to see it
blow up right away on an SMP machine. OTOH there is really nothing
which would force anybody to use that particular kernel apart of that
that it comes precompiled on a CD and this is a truly minor reason. One
can use also something from 2.2 series as well if it works well for your
purposes.

  Michal
