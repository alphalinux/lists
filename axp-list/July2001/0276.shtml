On Mon, Jul 23, 2001 at 01:41:33AM -0700, my name wrote:

> However I know
> there is a e2writeboot utility to write on ext2
> partitions. I think however this only is meant to be
> used on floppies

Right. 'e2writeboot' is meant for floppies where you do NOT
have a partition table. Use 'swriteboot' if you write on
your hard disk. There is also 'isomarkboot' for making bootable
CD images with iso9660 file systems on them. The first two
come with manual pages.

> and it still ruins your filetable.

No, not really, if you mean 'swriteboot' with '-f' flag and
a "disk label" type partition table. It will ruin your data though
if you will force writing over a partition with a file system
on it. This was discussed a number of times in the past so
check archives of this list if you want to know more.
