If it's already working well in Linux you don't need to mess with
isacfg at all. It's not like you can bootp from this card in SRM
so there's no need to go through the effort. isacfg is usefull for
some isa sndcards that need their resources defined. Besides that
it's the arcane tool that you know is there but don't want to know
exactly how to use it ;-)
