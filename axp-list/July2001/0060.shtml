Hi,

I accidently updated an UP2000+ with the Firmware Disk that was
with the Tru64 package. (The machine is running Linux, but I wanted
to make some test :-( )

The result it won't boot anymore but will beep 1-2-3 the same as if
flupdate from the Debug Monitor is called. There is no video signal.

If I install the jumper 1-2 for the flash recovery the beep will go away
but nothing will happen. If I press reset, the Alpha Debug Monitor
will start. (This will work only if the Jumper is installed)

>>From there I could start the SRM Console with "romboot 2" and even
succeeded to reflash it with the correct version from api.

Having started SRM I could boot linux, the usual way.

I set os_type to unix and console to graphic

But if I remove the jumper 1-2 it won't boot. Obviously it directly
starts flupdate, since if I put a Disk with AlphaBIOS in, it will start
the AlphaBIOS from the disk.

Has anyone an idea? Or can point me to a decent documentation on the
UP2000 boot system. The documentation provided by API do not even
mention the Debug Monitor. Also I found a "failsave Bootloader" in the
romlist.
