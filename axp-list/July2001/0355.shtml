> The following message is a courtesy copy of an article
> that has been posted to comp.os.linux.alpha as well.
>
> I have been running DU/Tru64 for many years on an Alpha Station
> 255/300. Wanting to change to Linux (of course!), I bought a second
> hard drive (/dev/sdb) onto which I have done an initial install using
> Alpha JumpStart! (v1.4.1). Digital Unix (v4.0D) still resides (and is
> usable) on the original smaller, hard disk (/dev/sda).
>
> Doing some initial testing (before the mandatory second install ;-)),
> I have noticed *much* slower performance than either DU or other Intel
> Linux boxes. As an example, when 'fsck' ran during the initial boot,
> it showed the "progress bar" as usual. However, even though the disk
> is only ~35% used, the disk checking -- unlike other systems I'm using
> -- continued at the same rate over the entire drive. In my
> experience, 'fsck' runs slower at first, then moves rapidly over the
> empty portions of the drive.
>
> Another thing that seems odd is the length of time for 'updatedb' to
> run. It takes forever! Starting up applications also seems to take
> longer than I'd expect on a 64-bit, 300MHz machine.
>
> During the AJS! phase, I selected the 'ev45' kernel. It was the
> choice highlighted, so I went ahead and used it even though I'm not
> sure the 255/300, in fact, corresponds to 'ev45'.
>
> Another factor that might be relevant is RAM. The machine currently
> has 192MB of RAM. Also, I am using a Permedia2 video board with 8MB
> of memory (should I use the "Permedia generic" option during X
> configuration?).
>
> I would be interested in anyone's comments about my observations. I
> plan to do a second install this afternoon, so please let me know if
> there is something I should do differently.
>
> Maybe I am wrong assuming that AlphaLinux will be faster -- or at
> least as fast -- as DU/Tru64. I hope not!
>
