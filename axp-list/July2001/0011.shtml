On Sat, Jul 07, 2001 at 12:21:18PM -0700, James Fowler wrote:
> I only have my normal kernel that I could
> copy to the floppy. I am trying that. Will that cause problems with the
> installer?

It should not, with this exception that if booting your system with this
kernel depends on some modules then this will likely fail; but this also
depends on your definition of "normal". :-) If you failed to start a
network, or some other thing - like a printer, because some modules are
missing then this minor and you can fix up things later. But if you
cannot access your main disk then this will create a bit inconvenient
situation.:-)

The reason in suggesting another kernel is that it looks like that what
is provided has some disagreements with your CD device. You should file
a report on http://bugzilla.redhat.com/bugzilla/ describing your
troubles (once you know that md5sum for your CD is the correct one).
Include precise model of your CD drive and attach the whole boot log you
posted here.

> Oh, I did check the CDROM on two systems and two OS. It mounts fine on my
> alpha under linux and on and x86 windows machine.

Yes, but can you also read it? 'cat * > /dev/null' in a directory
with suspect files, is not a bad way to check that under Linux.
If it is silent then this is good.

> The boot command I give for floppy is 'boot dva0' this works for graphical
> installer.

This will indeed use a line labelled with "0" from your aboot.conf
_provided_ your 'boot_osflags' (I do not swear that I remember the name
correctly but it easy to check from SRM prompt) is set to 0 or not to
set all. If there is something else there then who knows what will
happen? Therefore 'boot dva0 -fl 0' in general avoids surprises.

Indeed 'boot dva0 -fl 3' will start a text installer with a floppy from
7.1. This is known to be a good thing is some cases. :-) Sounds like
not good enough for you. Bummer.
