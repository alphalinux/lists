You are not seeing compile errors; as stated, these are warnings
spit out by the compiler or assembler. If this were a perfect world
(which it is not) and Linux were perfect software (which it is not),
warnings might be something to cause alarm.

> This continued on for several other functions, until it successfully
> produced a vmlinux file. Also, it did stop completely with a segfault at
> one point, which I then just ran 'make' again, and it completed. Could I
> possibly have a memory problem? I do have the latest AlphaBIOS installed.
Probably not a memory problem. Which kernel is currently loaded when
you are compiling 2.2.13pre? A few releases back, vfork(?) had
some issues which caused things like gmake to segfault readily.
Think David Miller patched that up.
 
> Should I be concerned about these warnings?
Other than being annoying, the warnings should cause no concern.
Unless of course you compile with -Werror and you see them....
