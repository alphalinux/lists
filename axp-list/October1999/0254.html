I have the Alpha here running on a Cable modem, an ATM 25 (1.6mb download
speed) network NEC box on
Hong Kong telecoms 'ITV' video on demand service. The Alpha has two network
cards
one goes to the Cable Box, the other goes to a hub and 3 other PC's (another
linux and two
Micro$oft machines) As the Cable network only allows one machine to connect,
I use IP masquerading
and IP Chains on the Alpha, and can share the connection with all the
machines.

ITV uses DHCP, but I alway get the same IP address.

IT all works great for me, and never ran windows on the Alpha.

I have a small perl script running, monitoring a directory shared out under
samba, when it spots
a file with the extension '.con', it opens the connection to the cable box,
and when it is deleted
it closes the connection.

This way I can go on and of line from any machine on the network just by
'touching' a file in the
connect directory. I can also do the same from cron or other scripts. If
each machine or application
creates a file with a unique name and the .con extension, then I can connect
then have other machines
write and delete a .con file without cutting each other off.

It's been going great here for ages.

Just my 2 bob's worth.
