On Sun, Oct 24, 1999 at 10:05:44AM +0000, Mark L. Holloway wrote:
> I was just curious.. If a .COM company wanted to have a cluster of Linux
> boxes and was looking at DS10 with Apache, what is the additional software
> requirements to make the cluster happen on Alpha? I'm not too familiar
> with clustering..
  [price comparisons deleted]
> Is Beowulf the only way to cluster Alphas?
> Does it work for .COM/Internet type of stuff?

  I am getting an impression that you really are looking for a WEB-server-
  cluster implementation method, not a computing cluster.

  As such you propably want at least two network interfaces for each box,
  one for externally incoming traffic load, other for backend communication.
  (Which method isn't exactly Beowulf one..)

  Your nodes are likely going to be fairly separate entities, and unless
  you need to run database interactions for each outgoing page, one way
  to do it is to have identical WEB-server setups at each of server
  "cluster" (or "farm") nodes along with content replicated to all cluster
  node disks.

  To get more read IOPS from the system, have 2-to-4 way RAID-0 (mirror)
  of the content disk area.

  A different approach to things would be needed for database stored
  content - a layer of HTTP servers talking to high performance database
  server; such server must likely be distributed to a cluster in itself.
