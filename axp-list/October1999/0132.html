Hi,

even though RH 6.1 is not Alpha-ready yet, I thought I'd share 2
points with the list concerning ix86 version, hoping this issues
will be fixed in Alpha release, whenever it comes up. Here's hoping
people from RedHat are listening.

1) /etc/init.d/rc.d/autofs script was modified so that if local
/etc/auto.master file exists, it never looks at NIS map. It could
be considered a (ill-advised) feature, however man pages don't reflect
the change, so I'm calling it a bug

2) During the installation, if there's a partition which is marked
as Linux but doesn't contain ext2 filesystem on it, the error is
issued, and then the system hangs up. A cold boot is required
to try again ( otherwise kernel panic occurs during the next CDROM
boot ). After I mkfs'ed that partition, the installation process
went smoothly.

Thank you

Igor

P.S. I went to http://developer.redhat.com/bugzilla but didn't find a proper
place there to report the 2nd bug. Besides, from personal experience with
emacs and mozilla, I know that bugs filed automatically never get
fixed. For me, only direct email contact with developers/maintainers
has been fruitfull in the past.
