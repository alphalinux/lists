Christopher T. Lansdown wrote:
>
> So, has anyone tried out 2.2.13?

$ uname -a
Linux toaster.home.front 2.2.13 #1 Thu Oct 21 15:15:32 MDT 1999 alpha unknown
$ uptime
  8:32am up 15:04, 5 users, load average: 0.00, 0.00, 0.00

Actually the above lies a little bit as this is the second version; this
one with a patch to support IDE disks bigger than 32 GB. This code is
not really used, as I do not have such disk, but I wanted to check if
there are any adverse side-effects.

> Anyone have any experiences with it?

It is not out long enough to collect much experience with it :-)
but I was using various "late" 13pre in a networking environment
and during quite heavy compilation activity and did not run in any
nasty surprises. So far so good. YMMV.

BTW - this particular box is UX.

If you have plans which include knfsd and/or RAID you should have
a look at 2.2.13ac1. Yes, there is one already. It includes, among
other things, some alpha specific patches which did not make it
into 2.2.13. OTOH you may want to wait a bit for a higher "ac" number
unless you plan to help with testing. 2.2.13ac1 __may__ turn out
to be a big buggy.
