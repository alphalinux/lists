I'm still having random system freezes related to sound.
I have a PC164 with an original SoundBlaster 16 ISA card.
I've tried various kernels... 2.2.5, 2.2.7, 2.2.9, 2.2.10, 2.2.12, 2.3.1, 2.3.10
and so on. They all have the same problem. Most of them freeze the machine
either when I try to play a sound, or soon afterwards.
2.2.9 seems to be the most stable for some reason when it comes to sound, that's
what I'm running currently. Though it still hangs, just not as often.
I'm using the PNP driver, and the sound drivers compiled as modules.
They load up fine, no errors, and sound works flawlessly, except for the
random hanging. I'm using SRM, if that matters, and I've tried both "generic"
alpha support, as well as PC164 specific in the General Setup section.
Does sound just magically work fine for everyone else with a PC164?
Could there be a problem with the Saturn I/O driver? I don't have any other
ISA cards, so I can't compare. Could it be an ISA DMA problem????

Can someone please offer some suggestions?
