On Mon, Oct 25, 1999 at 06:19:13AM -0400, Uncle George wrote:
> yup, but it sppears that the 'sender' of the parameter has to know what
> type it is to get this picture correct.

Kindof. The caller is supposed to sign-extend all 32-bit integers.
Note that this is without respect to whether the variable is declared
as `unsigned int' in the source.

Values smaller than `int' are promoted by K&R rules to `int', so eg
`unsigned short' winds up zero extended since bit 31 is never set.
