On Wed, 20 Oct 1999, Champigny, Michael wrote:

> I am using a local network addres of 192.168.0.0.
> I think my netmask is 255.255.255.0 which is what
> it should be for a Class C network. I notice most
> people use 192.168.1.0. I wonder if maybe something
> is "different" when I use 0 as the third byte? Hmmm...

Nope, shouldn't be any difference at all.

> Also I notice that the route command is nowhere to
> be found in the the network scripts. Is ifconfig
> the "new" way to configure ethernet connections now?

In 2.2.x kernels, when you ifconfig up an interface, it automatically
adds a route for that network, so in this case doing a 'route add -net
192.168.0.0 netmask 255.255.255.0 eth0' would be redundant.

> I haven't attempted to get a LAN up since the RH 4.x days. :-(
> Much has changed.

I would try using the de4x5 driver, as someone else suggested. I
remember trying to use the tulip driver once on my multia and was never
able to make it work. However, that was over a year ago (and I
currently use the tulip driver without any problems on my as200--0.89H,
I think is the version).
