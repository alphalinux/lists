Shane Sturrock wrote:
>
> I don't see what all this talk of Intel processors and clock speed has to
> do with the actual available performance of Alphas? I mean, who cares if
> an x86 gets to 1Ghz before an Alpha. My current integer benchmarks using
> ccc on a 21164A at 600Mhz show that you would need an x86 of well over
> 1Ghz to get similar performance. Given that the 21264 should be around
> twice as quick again I don't see that Alpha has anything to worry about.
> If you are not seeing much difference in performance from your code on
> Intel and Alpha then you should probably look at modifying the code to
> make it perform better on the Alpha. At least now with ccc we are seeing
> the sort of performance we expect from Alpha. Look at what Intel et al
> are having to do to get performance out of their chips, supercooling and
> .18 micron processes and the things are still slower than a .35 micron
> Alpha at room temperature.
>

I've tuned code a long time. An Athlon is approxmately 0.3-0.55 of a
21264, even using Compaq's FORTRAN on the 21264 at the same MHZ.
If you see a bigger difference, it is probably library related codes.
(sin/cos/etc) The SPEC benchmarks also agree with what I've seen so
I'm fairly certain it's a reasonable ratio. If I can buy a 1GHZ
Athlon for $1.5K or so, that's better price performance than Alphas.
If one could get the compiler to issue AMD's multimedia instructions,
the Athlon would have a peak performance of 2GFLOPS at 500MHZ, Alpha
21264 is 1GHZ. Unfortunately, that doesn't work with egcs/gcc, at least
not yet. I hope someone is thinking about it. The main problem with
Athlon is it's memory bandwidth I believe, and perhaps using DDR-SDRAM
would help there but finding that kind of memory seems impossible.
I'd buy a 128MB stick if I could find it.

Anyway, this a bit off topic so thats it for me on this topic.
