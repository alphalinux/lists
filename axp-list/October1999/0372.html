maurice@harddata.com said:
> I think I agree with the suggestions to try some different memory, if
> possible. Generally look in any machine check errors for lines
> mentioning "630" or especially"670" which are often memory errors.

This damn PC164 system board takes expensive SIMMS, and takes them in
large banks, so trying different memory is not convenient. However, the
memory is relatively recent, and I had similar problems even before
changing it.

Where would I find machine check errors? none seem to find their way
into any logs I have.

I also had a hang up just now, with the 2.2.12 memory leak patch applied.
It is true, though, that I've been running memory-intensive programs,
namely egcs-c++ on some bison generated C++ files. (These tend to generate
lots of disk activity too.)

*sigh* I'd be ready to blame the Adaptec board, but at least one other
person on this list has reported similar problems with a non-adaptec
(QLog, I think) SCSI card. And I've heard that the 2940UW is reasonably
well supported in Linux.
