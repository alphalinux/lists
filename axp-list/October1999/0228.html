Phil Carmody wrote:
>
> I'm confused, I'm used to RH5.2 (admittedly on x86), and when I export /,
> I export all the mounted partitions below that too as one virtual file
> system. However, under RH6.0 (alpha, original CD no patches) I export only
> / and no submounts.

If you are using knfsd (kernel side NFS daemon) then yes, it behaves
that way (and has some other nasty habits/bugs). You have to export
every file system separately. BTW - if you are using this then you
should likely update to the latest one from 1.14 series (not 1.15
though) and patch kernel accordingly. knfsd will likely be interated
"for real" only in a future 2.2.14 (there is no 2.2.13 yet).

> Personally I go with RH5.2, I prefer the virtual file system view.

There is really nothing which would prevent you from using unfsd as
you did before. You will have to recompile it yourself for 6.0 though.
Grab the latest .src.rpm from 5.2 updates and 'rpm --rebuild ...'.
You will probably loose some performance but you will gain a piece
of mind. :-) Pay attention to locking issues, especially if you do
things like mounting mail spool over NFS or similar.
