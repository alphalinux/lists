I've just built 34 XP1000's using a kernel I compiled. Each one should be
identical because I used dd to create the main disk of each one from one I had
originally setup the way I wanted it.

Now, I randomly get the following error when I do a rsh command, i.e.

rsh n30 ls

poll: protocol failure in circuit setup

Once I've gotten the error, I can retry the command and I get:

n30.cluster.fltsci: Connection refused

However, I can do:

rsh n30

and it'll log me right in. After waiting awhile, everything seems to go back to
normal and I can execute commands with rsh.

Anyone else seen this, or have any ideas?
