Ron Farrer wrote:
>
>
> After switching back to slink and reinstalling the Tru64 4.0d libs NS 4.7 doesn't work! It was working fine on slink with 4.0 (note: no 'd') before I upgraded to potato. What happens is it will spit out:
>
> Warning: Cannot convert string "-*-Menu-Medium-R-Normal--*-120-*-*-P-*-ISO8859-1" to type FontStruct

This is truly a minor annoyance. It can be trivially fixed by adding
a line like that to 'fonts.dir' file in some directory with scalable
fonts in your FontPath:

n019003l.pfb -urw-menu-medium-r-normal--0-0-0-0-p-0-iso8859-1

This assumes, of course, that you have readable n019003l.pfb from URW
fonts. If not then use something else handy. Do not forget after that
to bump by one a font count at the top of 'fonts.dir' and to copy
results into 'fonts.scale'. The fact that you already likely have a
line of that sort

n019003l.pfb -urw-helvetica-medium-r-normal--0-0-0-0-p-0-iso8859-1

is not important here.

> but no window ever comes up.

Maybe you simply do no have any suitable fonts installed, or they are
not in your FontPath, or you are using broken xfs? Even with this
"Cannot convert ..." things should start using some defaults.

> P.S. Sorry for the cross post, but I *think* its a generic issue.

Probably you are right.
