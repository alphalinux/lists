On Sat, Oct 23, 1999 at 09:48:19AM -0700, Mark Holloway wrote:
>
> After successfully installing Red Hat 6.0 on my 500a, I want to install it
> on an "ALPHAstation 255/300mhz". It's a desktop style case (looks similar
> to a DS10) and is an older 21064/300mhz machine.

I run two of these where I work, both run Red Hat. Nice little machines.

> I don't know what the nickname is so I can't get the right MILO
> image.

As someone else said, you can very likely just boot the generic kernel
from the RedHat CD. But.. for the record.. the AS255 is an Avanti
machine (regardless of what other documentation might indicate).

> Also, this machine had
> Digital Unix on it and if I type "nt" in the SRM I can't get to the ARC
> firmware.

Why would you want to do that? SRM is a much better way to
go.

> It should exist because this machine had NT on it before Digital
> Unix. Could someone have changed something in the SRM command line options?
> Any thoughts?

Well, if you insist on running ARC.. I believe that the AS255 only has
enough space to hold one console image at a time. So.. chances are
good that if it used to run ARC, and it now runs SRM, someone flashed
the SRM in over the ARC and it's gone until you figure out how to
re-flash it. Alternately, you can try typing 'arc' and see if that
does anything.. or set the os_type variable to NT and power cycle the
machine.

Personally.. I think the SRM console is better for loading
Linux.. but.. that's just my preference, take it for what it's worth.
