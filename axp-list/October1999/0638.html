Mark Holloway wrote:
>
> Is it better to run Netscape for Tru64 or Netscape
> x86 with em86?

Define "better".

The first and foremost obstable for using DU stuff on Linux is a
possible lack of a license for required libriaries and their loader
**for a CPU on which you intend to use these things**; otherwise you
would be doing that illegally. Various folks actually have such
licenses aquired in different ways.

As for Netscape over em86 startup times are of such order that they
can put you to sleep. Once it started it is not bad if you are using
version 3.x of Netscape (it has to be compiled for libc5). When I
tried some variants of 4.x they did run but they were definitely on
a big fat and slow pig side. Also you should turn OFF Java and Java
Script or 'scrape will be crashing all the time. The last thing
can be construed as a feature. :-)

If you insists on using glibc2.1 then em86 does not work, so far,
with it. Leslie Donaldson was looking at that but he run into
problems - rather lack of an information and access to some things
than truly technical obstacles. Also 'math-emu' stuff in all kernels
above 2.2.11 kills em86. Jim Paradis, who modified math-emu, promissed
to look at this and I do not know what happened. A workaround is to
take a contents of 'math-emu' directory from 2.2.11 and replaced with
it the same thing from, say, 2.2.13 and recompile your kernel.
This works fine.

Other than that there is nothing special to running Netscape. :-)
It works.
