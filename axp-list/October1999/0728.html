I had some issues with 3Com 3C905b on both x86 and Alpha, so I searched
http://support.redhat.com (knowledge base) you'll see that there are some
issues with this card. Even thought it's in the kernel it bahves strangely
and you need to try different PCI slots.

A real easy way to test it (I'll probably get slammed for this) is to use
linuxconf and add it as a second ethernet card. Configure the appropriate
IP address etc.. When you reboot see if "eth1" comes up. If so, log in as
root and ping the IP you assigned to this card. If it works, the card is
successfully installed on the local machine. Next, go to another machine
and ping the same address.

Today there was a guy at work who tried installing Red Hat 6.0 with a 3c905b
card and had an incident where it didn't work under 6.0, so he re-installed
5.2 and it detects the card fine. I strongly suggest you try my above
method for a basic, no-brainer approach. Red the support.redhat.com pages
on 3c905b.
