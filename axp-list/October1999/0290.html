Pounding repeateadly on ^C like a madman actually works; even if this
is not the most elegant action. :-) DU machines have the same problem.
But with 'aboot' and Linux there is a friendlier solution.
Set up a file /etc/aboot.conf, run 'abootconf' to make sure that
aboot will look this table on a correct partition and change SRM
settings above like that:
     set boot_file ""
     set boot_osflags 0
With this an autoboot will use whatever parameters are given
on a line with a label "0" in /etc/aboot.conf and you may edit this
file any time to your heart content.

Believe it or not but all of this is described in a great detail in
a documentation which comes with 'aboot'. You can even include
a partition in boot_osflags if you do not trust 'abootconf'.
Have a peek there.

A timeout in SRM which allows you to intervene and supply your own boot
parameters, something along lines of 'milo', surely would be nice. To
boot in an emergency with the above into a single user mode you are
still left with a trusty ^C key.
