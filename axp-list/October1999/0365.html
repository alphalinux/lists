Hi, I just bought an SIIG UW-SCSI Pro card with the Initio chipset because
it was reasonably priced and in the hardware compatibility list. However,
I can't get it to work. I have trawled the news group archives and
mailing list archives and every source of information I can find and come
up with nothing.

The card initiallises fine under AlphaBIOS 5.66 on my 164LX, but linux
(kernel 2.2.5 or kernel 2.2.12) doesn't recognise the card no matter how I
compile the initio driver. All I end up with is the following:

[root@beta /root]# insmod initio
i91u: PCI Base=0x8800, IRQ=19, BIOS=0xFF000, SCSI ID=7
i91u: Reset SCSI Bus ...

scsi0 : Initio INI-9X00U/UW SCSI device driver; Revision: 1.03g
scsi : 1 host.

scsi : aborting command due to timeout : pid 0, scsi0, channel 0, id 0,
lun 0 Test Unit Ready 00 00 00 00 00

This happens when there is a drive attached or now, with terminating, on
off auto, everything I can think of.

The card is reported in /proc/pci as:

  Bus 0, device 9, function 0:
    SCSI storage controller: Initio Corp 360 P (rev 2).
      Medium devsel. IRQ 19. Master Capable. Latency=32.
      I/O at 0x8800 [0x8801].
      Non-prefetchable 32 bit memory at 0x19002000 [0x19002000].

I've tried it in both 64 and 32 bit slots, same result each time. I've
e-mailed the driver author but not had any reply yet and I need to get
something done about it soon otherwise the shop won't take the card back,
although I'm a little doubtful that they will anyway.....

Has anyone been able to get this card to work and how?
