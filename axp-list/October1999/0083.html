>>> "Kirk R. Erichsen" said:
> >
> >I would like Compaq to add DOS disklabel support to SRM though, BSD
> >disklabeling is a pain.
> >
> >Chris Kloiber

Does DOS-style disklabel allow specification of where a bootstrap
lives and how big it is? BSD-style does, and this gets used by SRM
console to load in the bootstrap. Not sure of the exact reasons, it
may be tradition, but it allows bootstraps of varying size and/or
varying location to "just work", which is good, since "aboot" and
"osfboot" are quite different in both respects... ;-}

> SRM's lack of filesystem support is a pain, it makes abstraction of your
> config almost impossible. However, as much as this would do for my
> "fuzziness" effect, we'll be lucky to have an SRM for the masses.

What do you need FS support in SRM for?

The FS support is in the bootstrapper, ie "aboot", and you can put
whatever FS code in there that you want, along with whatever additional
functionality might be required (initrd comes to mind). It is open source.
