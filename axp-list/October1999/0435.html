All,

I have noticed a problem where the statistics about cpu-usage for the
processes are invalid on my Compaq DS20 machine with Linux 2.2.9 SMP. The
statistics will only take time used on the first processor but not on the
second. Am I the only one to experience this?

Quick test: Make sure both CPU's have been working. Do cat /proc/stats.
This gives a printout that looks something like:
cpu 2718 0 8127 5046643
cpu0 2718 0 8127 2517899
cpu1 0 0 0 2528744
...

The first line corresponds to total cpu usage and the columns to user,
nice and system time. As you can see not a single "time-slice" have been
accounted for cpu1. The same problem applies to per process statistics.

I am not very well qualified in kernel debugging but I managed to find the
following clues:

* The macro smp_processor_id() (which is defined as current->processor)
always returns zero in function
smp_percpu_timer_interrupt (in file arch/alpha/kernel/smp.c) which is the
function that should update the statistics.

* The map between processor numbers and id's are okey. (and the inverse
mapping too)

* in the function handle_ipi the smp_processor_id() returns okey (both 0
and 1)

Any ideas what I should check next?

Grateful for any thoughts or ideas on this,
Lars
