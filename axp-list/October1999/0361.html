Greg Johnson wrote:
>
> It also seems less likely if I
> boot from AlphaBIOS/Milo rater than SRM/aboot. When it hangs from the console
> (not X) I sometimes get dumped back into SRM. It says "exit code 7. Machine
> check while in PALmode." (from memory). I have played with examining the
> registers, but see no clear pattern.
>

This is perhaps a strong clue. Maybe you have a HW problem.
One would assume a machine check is HW related but I don't
have any docs to indicate what they mean. Finding that info
might be of help.

> Hardware: 164LX2/533MHz. Elsa GLoria Synergy 8MB (Permedia2). 4GB Segate
> (ST34321A) IDE disk. Built in CMD646 controller. Booting through SRM/aboot.
>
> I have tried moving my hard drive and video card to another identical machine
> and had the same problems. I've also tried using an S3 Virge Trio64 card with
> the same behavior. This seems to be independant of which kernel version I'm
> running. I've tried 2.0.36 and 2.2.[7-12]. Currently I'm running
> 2.2.13pre14, and it's been up for four days...
>
> I'm at a loss. I know other people are running on similar hardware. We have
> 3 other machines like this one and 16 164LX4/600's which are headless
> (Beowulf). I have also had these machines lock up on me, and I expect it's
> the same problem.
>
> So, does anyone have any help to offer?
>

How about swapping memory DIMM's around? Looks like you've swapped
about everything but that.

I have a 164LX/533 with 256MB memory, ATI Rage II video,
a 2GB Seagate ide disk, ide cdrom, 2 symbios 810 controllers
with a 9GB IBM disk, a DLT7000, and an Exabyte Eliant attached
that hasn't gone down without my taking it down this year.
Currently up 81 days. Running RH6.0's kernel off their CD.
