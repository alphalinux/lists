I know this will seem stupid to some, and I apologize..but if I
compile/install a program [Squid] rather than using RPM, how do I uninstall
it?

I ran into an issue where I downloaded the Squid sourc, compiled, and realized
I needed a patch to replace one of the original C source files. Well, once I
replaced the source with the new one, I know I can do a "./configure, make
all, make install" again over the existing code, but it got me to wondering
"Should I have un-installed squid and re-installed?" - then I got to thinking,
if I'm not using RPM, is there a way to uninstall other than just deleting
files? In FreeBSD I was able to use "pkg_info" and "pkg_del" even if I didn't
use "pkg_add" to install the files to begin with..
