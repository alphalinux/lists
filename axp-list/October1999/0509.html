>>> Brad Lucier said:
>
> but I was also
> wondering if the components I have have known hardware problems (beyond
> the known problems for the Adaptec controllers on the motherboard),
> or whether I chose the right kernel options for the hardware I have, etc.

I would definitely use the SYM53C8XX driver with the 895, as that one is
being more actively enhanced, IIRC.

I'd also stay away from the Adaptecs, though with the recent mobos, they
should work OK.

Also, you may run into a SCSI-timeout-related problem using that compiler
(EGCS 1.1.2) with those sources (2.2.13-pre13 and later), as there is a
"delay" bug that can crop up in those circumstances. I'll include the
patch which should apply to most of the latest sources at the end, which
should fix the compiler-related problem.

> It takes a few hours of my time to try something, and then if it doesn't
> work, it takes a few hours again a few days later. I'd like to try to
> get several problems (if there are several problems) solved at each
> go-around.

And yes, definitely try to specify/use one less cylinder than reported.
