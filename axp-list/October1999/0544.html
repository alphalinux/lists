So far, you're correct. However, the -taso and -xtaso flags aren't needed by
Mozilla (supposedly they've fixed the 32bit assumptions). As it stands right
now, using ccc doesn't get too far in Mozilla - ld chokes on an object file
format last I looked. Of course, that's probably because the standard RH6.0
compiler is not what they test against.

As for M10, I did manage to get it to build on my Miata with standard gcc, but it
"doesn't quite work". As in the GUI starts up, and the I get some type of GTK+
message about a pointer not being NULL and then it quits. Maybe someone with GTK
experience has more of a clue.
