Christian Iseli wrote:
>
> nicolasw@hal.famaf.unc.edu.ar said:
> > Hi! I was wondering if I can attach an IDE CDR to our LX164 (ev56
> > 600Mhz, ncr53C8xx SCSI adaptor, 256Mb Ram, rh6). Is there any problem
> > with the ide-scsi.o (SCSI emulation) and the "native" SCSI drivers?
> > Will I have many troubles?
>
> This will not work (AFAIK), and migth damage your hardware. From the kernel
> doc, the SCSI emulation:
> "will provide SCSI host adapter emulation for IDE ATAPI devices,
> and will allow you to use a SCSI device driver instead of a native
> ATAPI driver.
> This is useful if you have an ATAPI device for which no native
> driver has been written (for example, an ATAPI PD-CD or CDR drive)"
>
> All this is software. Not something that will make an IDE drive hardware
> compatible with an SCSI controler.
>

The LX164 board has an IDE interface built in. One can
attach the CDR/CDROM to it and all works as expected. You would
use SCSI emulation since cdrecord only talks to the SCSI generic
device driver. I recall some postings on making that work
a while back. I think it does work but can't remember any
details. I use a SCSI CDR/CDRW on my Alpha without problems.
