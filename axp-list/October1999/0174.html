Jay.Estabrook@digital.com wrote:
>
> These kinds of problems used to happen if there was a mismatch between
> the linload.exe and MILO, as MILO expects the linload.exe to pass it
> info about the memory size. If it does NOT get what it expects, it will
> default to 24MB, which was the default minimum config of UDBs.
>
> Either upgrade to an appropriate pair of linload/MILO (preferred),
> or try doing:
>
> set MEMORY_SIZE 32
>
> in MILO, and then booting the kernel.

In doing a quick check, this did work. I'll have to look into MILO a
little more. I'm still learning, but having fun banging on some hardware
that's completely new to me.
