On Tue, 19 Oct 1999, Wes Bauske wrote:

> Try "xhost +your_host_name"
>
> X authentification is stopping the window from appearing.
> Only the person who starts X has authority to put a window
> on the screen. I assume you're using xdm because if a user
> logs in and starts X themselves, they should not see this.
> There are probably config files somewhere that will fix
> this. Try looking at the man pages for X and xdm.

 It breaks security badly, of course. Better see how to use X server's
"-auth" option (`man Xserver') and generate an authentication key within
`startx' using `xauth'. You may use MIT-MAGIC-COOKIE-1 for a start
(`mcookie' will create a random cookie for you).
