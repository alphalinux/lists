Red,

I'm affraid that isn't possible for two reasons:

1) UP2000 is a Slot B motherboard (even though it is using the Tsunami
chipset, minus a few D chips) shipping with AlphaBIOS firmware console and
only AlphaBIOS code.

2) UP2000 and Slot B in general still have no SRM firmware and no
associated PALcode (plus, you need certification of hardware to run Tru 64).

UP2000 was released by API prior to the big "news" of Alpha/NT
discontinuance. This is an "NT Only" system running AlphaBIOS 5.7 and
supplied with NT HAL and Palcode. You would be guessing right if you
realized that you need SRM firmware to support Tru64 (or OpenVMS, or BSD
UNIX, or Linux using Aboot code). There is not SRM available for any Slot
B motherboard as yet, I have brought this up to Compaq (who own the code)
and API (who build the boards). Right now all the API Slot B boards are
running AlphaBIOS which has reached the same status as NT (discontinued,
the team has been re-assigned or laid off). No future revs of AlphaBIOS
will be released (unless a bug fix for the 64 bit NT development team is
needed, 2 engineers are left to do this) or SRM replaces AlphaBIOS in all
new Alpha systems. We are in a time where it is far from clear which one of
these is going to happen. Compaq has seen fit not to consider this before
making their final moves.

--Kirk
