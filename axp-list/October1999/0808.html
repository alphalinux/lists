Hi Stephen!

Stephen Williams wrote:

> and I'm not finding any SCSI errors in any logs.

If your system disk is connected to the same adapter it is not very
likely to get things written to the logs. Maybe dumping all messages
(*.* in syslog.conf) to a virtual console might help. If the
SCSI-adapter goes down you obviously won't be able to write to the disk
but messages to a console should work. (When my systemdisk head-crashed
I could see all the scsi madness on the console. 'T was an intel
though.) If switching consoles does not work in the locked-up state, it
might help to start the back-up and switch to the log vt before going
home. Just my thoughts though, and I am by no means an expert in this
field. Anybody please correct me, if I am wrong.

Good luck...
