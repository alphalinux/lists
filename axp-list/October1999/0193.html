Elliot Lee wrote:
>
> On Sun, 10 Oct 1999, Michal Jaegermann wrote:
>
> > Recently I submitted an "official" bug report
> > (after a font change from a theme selector in CC the whole X locks up;
> > a power switch time if you cannot telnet in).
>
> This is a bug in either the X font server (xfs) or egcs which manifests
> itself when xfs is compiled with optimization. As a result of the xfs
> crash, the X server locks up.

This particular thing seems to be hardly applicable as I am not using
xfs, at least not explicitely, and have all font directories listed
in my XF86Config.

> The latest XFree86 packages in Rawhide build xfs without optimization on
> the Alpha.

Which means that I would probably have to recompile the whole X myself
as binary archives from Rawhide are likely for glibc2.1 and I am
using glibc2. I guess that I will edit font strings "by-hand" for
the moment. :-) This works just fine.

>
> Please make sure you're using the latest Gnome stuff from ftp.gnome.org -

I do not want to go into all details on a list which is not really a
gnome development list but sources I am using are fairly recent and
often the most recent available. Assorted problems still remain but
there is a huge qualitative difference between what is available now
and what was distributed as "Gnome for Alpha" in RH 6.0 set. The later
could be described politely as "a proof of concept", if one would try to
be really kind; the current version is actually quite nice to use even
if various wrinkles still show up. This, of course, means that
Gnome is currently moving ahead in quite brisk pace.

There are some things I miss. For example - if I will position a window
in such way that is title bar sticks over a top screen edge (it happens
accidentally or some programs open a window which is too high) then it
does not seem to be a way to reposition that window; there is no "Window
Operations" menu, no key combinations to shift it down a bit (you can
send it to another desktop but a relative position will not change), no
option to move windows in a pager. Edge resistance is way too small for
my taste and does not seem to be a way to change it. And my mouse is
way too overworked. All these seem to be deficiencies in enlightment
rather than in Gnome itself but this is still a default window manager.
