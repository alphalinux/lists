carlos@torroja.dmt.upm.es wrote:
>
> Hi, Wes
>
> We are having more problems installing RH6.0 on our DS10's, particularly
> on NFS...
>
> We know that main problem is to get a good kernel. Could you please tell
> us how or where to get it?. At this moment we are using a 2.2.12 boot
> image from Compaq.
>
> Thank you very much
>

As was posted, make sure you have the latest ramdisk image.
Also, you should get the newest tulip.c from Donald Becker's
site to drive the 21143's on the DS10. I installed from the
local CDROM so I didn't need to update the tulip.o module
for the initial install. One option is to build a monolithic
kernel with the latest tulip.c built into it and place it on
the boot floppy so that hopefully RH6.0 will use it but the
installer still might complain when it tries to insert it's
own tulip.o module. I haven't sorted out how to actually update
the RH6.0 ramdisk since it uses a romfs which I've never built
one of. I expect there should be docs somewhere that describe
how one can create a romfs. Basically you probably do something
similar to an iso9660 cdrom image but I can't guarantee that's
how it works.
