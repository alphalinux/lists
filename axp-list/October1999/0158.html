Phil and Kirk,
    Pal code was virtually unchanged. I believe I heard the comment "we
didn't
change a single line of pal code". I seem to recall that first port of
OpenBoot
was "trivial" (the engineer was VERY good). The issues which took time
involved
some difficiencies in the specification of the of the console environment in
the EBSDK.
Once overcome, the issues were customization for the sophisticated server
environment
developed by the customer. Some weaknesses in SRM such as looping back
through the
memory test n-times could than be addressed, and boot options added. The
ability to shape the boot environment, control boot diagnostics, provide
support
for proprietary or new devices, are only some of the OpenBoot advantages (I
sound like a salesman for OpenBoot; I'm not, but I tried to sell it
internally).

Mike Breen
