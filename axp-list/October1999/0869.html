ks@hirvi.net said:
> It runs Red Hat 5.1 and I need glibc for the rc5 proxy server, thus I
> have either to install glibc or update to 6.0.

I managed to go from glibc-2.0 (RH5.X) to glibc-2.1 (RH6.X) on my
alpha without breaking anything. You'll need to update the compiler
and C++ libraries too. Frankly, I was amazed that it worked.

What I did to test it before committing was to install the rpm for the
C library in a separate root, then with some config magic I ran some
old programs with the new C library to see that it would work. It did,
so I went ahead and did an rpm -Uvh of the new library.

Note that upgrading your C library is not to be done lightly, YMMV,
it's not my fault, keep backups, etc.
