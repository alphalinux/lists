Last night I installed Red Hat 6.0 on my DEC Alpha 500a Personal Workstation.
I'll try and be crystal clear on my configuration..I really think I'm just
overlooking something silly!

After making 3 floppies (my first disk is MILO/Miata) I successfully loaded
[from the ARC firmware] MILO floppy -> Generic floppy -> Root floppy. The Red
Hat installer found my IDE CD-Rom and installed Red Hat Linux 6.0, including
X. I was prompted for a reboot and when MILO> came back up I had to:

MILO> boot fd0:vmlinux.gz root=/dev/sda5 load_ramdisk=1

and that would load Red Hat 6 from my floppy then hard drive. Then I cold
booted my Alpha back into ARC firmware to see what my partitions look like. I
pressed F2 for setup -> Hard Disk Setup it sees my partitions as:

SCSI ID 0 4087MB

Partition 1 8MB Unrecognized File System
Partition 2 4008MB Unrecognized File System
Partition 3 71MB Unrecognized File System

My problem is that I want to be able to load Linux entirely from the hard
drive and not rely on floppies. In MILO I do a "show" and have:

sda: sda1 sda2 sda5 sda6
hda:

According to Red Hat, I should modify my ARC OS Setup so instead of pointing
linload.exe from A: it should come from the first partition of the first hard
drive (which is correct for my system) and this partition should be FAT.
Should I format "Partition 1" as FAT? Is that what it's there for? Or should
I have reserved a seperate FAT partition from ARC before I ever launched Red
Hat 6 Install..? Their docs also say to do a:

dd if=/dev/fd0 of=/dev/sda1 bs=1440k

I'm confused if I should format the 8MB as FAT, if I should have pre-made a
FAT before Red Hat Installation (but doesn't Red Hat wipe the disk anyway?),
and how do I get the files from floppy to the correct partition (sda1? sda2?)
so I can point the ARC firmware to load linload.exe from the hard drive.

I appreciate any support, help, and feedback..
