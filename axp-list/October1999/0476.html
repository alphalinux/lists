"Champigny, Michael" wrote:

> I'm trying to get a Multia talking to an AlphaStation. Both have built in
> ethernet and right now I just have the two hooked up via a (working)
> hub and (working) cat-5 cables.
>
> Both are using modules for tulip on eth0.
>
> It looks my software configuration is OK. I used the netconf tool and
> double checked the effected configuration files to be sure they were
> doing the right thing.
>
> Problem is I get host unreachable messages from ping.
>
> Also, I get a SIOCADDRT message when running the route command.
> And even though I have the /etc/networks file in place, I still get IP
> numbers from "netstat -r" and not the aliased network names.
>
> Can anyone shed some light on this setup? It's just a basic two machine
> hookup via a hub and I know the hardware is fine. I must be missing
> something
> very basic.
>

Try seeing if the de4x5 driver works for the ethernet adapters. In some
releases, tulip works, in others it doesn't. You might also want to try the
latest driver from Don Becker's site
(http://cesdis.gsfc.nasa.gov/linux/drivers/tulip.html). My first
thoughts...
