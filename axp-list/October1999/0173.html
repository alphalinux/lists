I am trying to recompile KDE with glibc2 (not 2.1) libraries.
Discounting an obvious brain-damage in sources, where ltconfig files
assume that if this is Linux then executables/libraries will be of an
ELF-32 kind, things seem to compile more or less (some Makefile fixes
required).

Trouble starts when I attempt to start freshly recompiled KDE.
I am flooded with messages which repeat over and over
QFont::setPointSize: Point size <= 0 (0)
On the last try in a very short time I collected in a log
file 18 MB+ of that junk (some four and a half __millions__ lines).

This smells like a bug in qt library. The fact that the behavior is the
same with older "1.1.1" series of kde srpms and also with recent "1.1.2"
ones seem to support this assumption. Switching compilers between
gcc-2.95.1 and egcs-1.1.2 does not seem to change anything. This is
with version 1.44 of Qt library. Unfortunately even if a version 2 of
Qt is available then KDE sources do not compile with it as various
functions used by KDE are missing.

Does anybody encountered this bug before and has a fix for that?
Debugging through a twisted maze of somebodys else C++ code is not my
idea of fun.

BTW - the current version of Gnome, although not totally problem free,
actually works rather well in the same settings which send KDE bonkers.

Another "aside" - kdebase cannot be recompiled with gcc-2.95 or
gcc-2.95.1; an "internal compiler error". egcs-1.1.2 works but see
above.
