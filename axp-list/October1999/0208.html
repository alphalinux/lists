> One thing I haven't found one the web is a detailed description of
> kernel bugs and which patches should be applied to fix them. Maybe
        Alpha-specific kernel bugs? For a while there, we were seeing 2-3
compile "bugs" per 2.2.x release on the Alpha. Mostly due to someone
using a structure foo in the code's "neutral land" which only had
defs in the header files for architecture bar. There seems to be a dearth
of people willing to keep up with the changes and work on the Alpha
portions of the kernel. In recent times, its mainly been Rich Henderson
and Jay Estabrook contributing Alpha code. I kept up with fixes for
Alan's -ac and the main kernel for a while, and started up again with
2.2.10. One of the big problems is that folks like Linus and Alan
either don't have the time for or do not want an Alpha system for kernel
development. Several people have offered them free systems; I'm not sure
how/if Linus has put that donated DS20 to use.

> I'm just not looking in the right place, but short of collating a lot
> of stuff from various email archives, I haven't see this info
> available online. Since the 2.2 series has been nightmarish (IMHO) on
> the alpha, I think a web page summarizing kernel problems and
> solutions would be quite helpful.
        I laid the foundation for something like this months ago but it
never got off the ground. It was called the... "Alpha Linux Hack Shack",
and would've incorporated a comments section, per-release kernel summary,
and an unofficial patches section.
I'll try to dig up the images/html I made for that, and stick that back on
the project list.
