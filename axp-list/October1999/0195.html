I have an XP1000 500 mhz set up just the way I want it. We have just
gotten some XP1000 667 mhz boxes in. My plan was to pull the disk from
the faster system and do a complete copy from the working 500 mhz system
using dd, then put the disk back in the 667 mhz systems and run them.

The problem is, the 667's want boot from the drive. They get past the
loading of the compressed vmlinux.gz file, and to:

aboot: ok, now starting the kernel...

And it seems to freeze there.

In order to check that disk, I booted the original 500 mhz box from it
without any trouble. Is there some reason a kernel compiled on/for a
500 mhz XP1000 shouldn't boot the newer faster ones? I've gone through
the SRM params and haven't found anything.

Any thoughts or advice would be appreciated.
