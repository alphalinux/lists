On Tue, 26 Oct 1999 mgraffam@idsi.net wrote:

> I disagree. I think that desktop icons can (if implemented correctly)
> be very consistent with the UNIX philosophy. I like the way KDE has its
> desktop set up. The desktop is a directory and you can copy files to/from
> the desktop like any other directory. Rings of the UNIX maxim "everything
> is a file" .. I like it.

Hmmmm, I suppose it could look like that. I generally have little on my
desktop because most stuff I start from the CL (which is the UNIX way as
far as I am concerned!).

> I agree that windows sometimes get in the way, but a full screen window
> hides your root (and thus root menu) too .. its the same fundamental
> problem. A dock like Afterstep's Wharf can help if it always floats above

Not really, you can set windows so they won't cover the dock or icons in
full screen mode, not that I use that much anyway. All you need is a tiny
little piece of the root window visible for the menus to work anyway, but
with desktop icons you end up shoving stuff about to reveal the
appropriate icon and I have never found the process of selecting
icons/folders for moving around to be quicker than just jumping in there
with the CL so I wouldn't use them anyway.

> apps (I hate that), and a toolbar like KDE's can help since it can limit
> effective screen size and make apps occupy less screen area (I hate that
> too)..

True again, but still reminds me too much of the beast that is MS :-)

I still stand by WindowMaker for its speed anyway, an Alpha is a terrible
thing to waste and WindowMaker doesn't bog the machine down like KDE or
Gnome do. Besides, if people come to Linux and use the KDE or GNOME
desktops is it any wonder they feel it is slow compared with Windows?
WindowMaker is nippy, good to look at and extremely functional. If you
want you can use GNOME with it but frankly I tried it and dumped it.

Still, we haven't solved the browser problem yet, and kfm isn't really
much of a substitute for a proper browser.
