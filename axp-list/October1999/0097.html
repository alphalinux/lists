I finally got an IDE disk for my Multia and succeded in installing RedHat 5.2.

I found some things that aren't obvious to me how to deal with:

The mouse is on a PS/2 connector like the keyboard. I got a cheap 3-button
Genious mouse, but haven't been successful with configuring it and neither
can the RedHat setup program. Any hints to how to deal with this would be nice.

When booting from the ARC console I now use a MILO floppy, which is an MSDOS
type filesystem. I therefore wanted to create a DOS partition on the IDE
disk so I could boot without the floppy, but mkfs.msdos doesn't exist with
RedHat Alpha, or so it seems, so I wonder what other people do to boot the
Multia without using the floppy?

Another thing I am curious about regarding the Multia, my first and so far
only Alpha experience, is the delay there always seems to be when starting up
things, eg. login. After entering the login name it always takes some time
before password: appears, at least when it is done when the disk may be in
some kind of idle mode. I have never seen the same amount of delay on any other
i386 Linux (on machines from 386SX, 485 and Pentium) and I wonder if it is
hardware or software (RedHat? Alpha Linux in general?) related?

Finally, does anyone have a good place to study the SysV boot system? I have
until now always used Slackware and find BSD boot easy to use, but the RedHat
SysV style is a little puzzling still. Especially because I want to have
minute control of everything that happens on bootup :-)

Thank you very much,

Peter
