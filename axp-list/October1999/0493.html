It means that you have a very busy disk drive. It looks like the drive is
reallocating memory for more commands as it finds that its current queue
depth is insufficient. A queue size of 16 would likely be quickly filled on
a relatively busy system. You might want to look at seeing if you could
edit a configuration file somewhere to allow a larger default. The maximum
supported by SCSI is typically 254, though a value of 32 or 64 is likely to
be sufficient.
One other possibility is that the drive is unable to complete commands for
some reason, and this means that more and more commands are accepted for
processing. I would discount this possibility if these messages are only
encountered during furious disk activity.
