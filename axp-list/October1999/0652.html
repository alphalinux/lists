Mark Holloway wrote:
>
> The default install of Red Hat 6.0 creates a GNOME/Enlightenemnt environment
> so when I "startx" it launches everything for me.. If I decide I don't want
> this, ie. I want to load a different Window Manager and not use Gnome
> either, what file do I edit?

If you are using gdm-2 to provide a graphic front end to a login then
the answer is "none". You pick up the one you want from a menu
and you can add entries to a menu by simply dropping new scripts
to /etc/X11/gdm/Sessions/ directory. That is one of reasons I was
recommending its use.

> On BSDI and FreeBSD I always changed the .xinitrc file in my home directory
> but that file doesn't exist in Linux my system.

But noting prevents you from writing yourself one. Actually a better
idea is to have ${HOME}/.Xclients (this has to be an executable script).
You can start in it whichever clients you desire, do some other
preparatory work (keybord and fonts, for example) and 'exec' any window
manager you fancy as the last action (for Gnome 'exec gnome-session',
for KDE 'exec startkde'). Look at your /etc/X11/xdm/Xsession file.
You will find in it lines which say:
....
elif [ -x "$HOME/.Xclients" ]; then
    exec "$HOME/.Xclients"
....

By the same token ${HOME}/.xsession will do as well. If you start
your X windows with 'startx' then you need something like the above.

Some Red Hat distributions provide 'switchdesk' utility but it may
not cover the fact that you want to check how 'sawmill' window manager
works for you, for example. :-)
