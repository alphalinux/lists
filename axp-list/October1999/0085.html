>>> "Jemiolo, John" said:
> I though there was an SRM OEM developers kit available. Thats where the PAL
> code for inclusion in Milo came from, if I remember correctly? Jay, is that
> right?

The EBSDK CD does contain SRM binaries, but not SRM code. It contains
source of "sample" PALcode, called EBSDK PALcode, which is NOT the
same as OSF PALcode, and it is this source PALcode that is used in
MILO.

> But I believe Compaq just sold the OEM group to another company.

Which was just bought by another; who can keep track anymore? ;-}

> By the way, I've seen conflicting documentation on what SRM stands for, is
> it: System Resource Monitor or System Reference Manual? The DEC and Compaq
> websites have both listed, which is a fellow to believe?

System Reference Manual.

The console provides an interface which follows the spec present in
that manual. Note that most of it is published as the Alpha
Architecture Reference Manual, but there are appendices in the SRM
that are company confidential because they contain information about
specific implementations.
