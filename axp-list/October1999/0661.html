On Tue, 26 Oct 1999, Shane Sturrock wrote:

> get covered by open windows. I use WindowMaker and keep it simple.
> Desktop icons are a stupid idea, why should linux have to have such a
> stupid idea just because Redmond think it is a good idea?

I disagree. I think that desktop icons can (if implemented correctly)
be very consistent with the UNIX philosophy. I like the way KDE has its
desktop set up. The desktop is a directory and you can copy files to/from
the desktop like any other directory. Rings of the UNIX maxim "everything
is a file" .. I like it.

I agree that windows sometimes get in the way, but a full screen window
hides your root (and thus root menu) too .. its the same fundamental
problem. A dock like Afterstep's Wharf can help if it always floats above
apps (I hate that), and a toolbar like KDE's can help since it can limit
effective screen size and make apps occupy less screen area (I hate that
too)..
