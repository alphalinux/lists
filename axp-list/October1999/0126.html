Jo Schambach wrote:
>
> I also replaced the "killproc" function in this
> file, since it also seems to hang, with the following:
>
> killproc() { # kill the named process(es)
> pid=`/bin/ps -e |
> /bin/grep -w $1 |
> /bin/sed -e 's/^ *//' -e 's/ .*//'`
> [ "$pid" != "" ] && kill $pid
> }

If you do not filter out, say with 'grep -v', processes which
are doing 'grep' then you have a race and sometimes this will work
and sometimes you will pick up this process which does here
'/bin/grep -w $1'. You may also delete 'grep' in this 'sed'
expression but something should do it.

Besides something like
         pid="`/sbin/pidof $1`"
is simpler and likely more reliable.
