> We have trouble getting SuSE 7.1 installed on a DS20E: After booting from
> CD using the standard aboot configuration (/etc/vm_full.gz), the kernel is
> loaded but then stops with the message "please insert root floppy disk and
> press enter" (although the ramdisk had already been loaded by aboot and
> the cd-rom is properly detected). On our XP1000, the installation
> procedure just continues with the message "RAMDISK: ..." loads linuxrc.
> Using the 2.4 kernel (CD1:/etc/vm_24.gz kernels) results in a kernel panic
> before linuxrc is started :(
>
> Booting from floppy disks (aboot_xx, install_xx) does not work either but
> stops when reading the root disk (ext2 read error). The same disks work on
> two different XP1000s though.
>
> Our configuration: DS20E, 2x 21264 CPUs, 256MB + 2GB RAM, 18GB HD,
> Firmware SRM-6.0 (standard Compaq configuration).
>
> What's going wrong here?

Aboot loads the initrd at the very end of phys. RAM, but the kernel only
sees 2GB, so it can't read the initrd anymore (on machines >2GB).
The solution is to use the image 'install' (not install_xx) and boot:

>>>b <boot-device> -fi /etc/vm_full.gz -fl "root=/dev/fd0 fake_initrd"
