Hi!

I had great trouble to get X running with SuSE 7.0 and compiled XFree myself which
messed around a lot inside the X dirs, but finally resulted in a working XFree 4.0.2.

To clean up, i did a complete new install of 7.1 on my Ruffian system.
Worked fine, but X didn't work either (ATI Xpert@work card).
3.3.6 says: None of the configured devices could be found or anything like that.
IIRC, recently there was a thread of someone else, who has the exact same problem.

sax2 doesn't like my PS/2 mouse (couldn't get it to a functional state).
So i made my own XF86Config for XFree 4.0.?. and
that worked o.k. (sax2 deactivates the mouse immediately on startup, while it
works in X - strange...)

But my real problem is with ISDN.
While updating anyway i switched from kernel 2.2.19 to 2.4.4-suse.
I entered the Nameserver of my ISP (T-online) and if i do a
ping www.t-online.de it lasts a while, the connection comes up
and then it says www.t-online.de host name not found (or anything like that).
If i immediately restart the same command, it works like a charm.
This problem is with any machine on the internet.
In SuSE 7.0 and kernel 2.2.19 it worked o.k.

Of course i activated the dynamic-ip patch in the rc.config file.
cat /proc/sys/net/ipv4/ip_dynaddr returns 7.

Any ideas?
