On July 2, 2001 08:26 am, Stefan Muehlemann wrote:
> Hi,
>
> my machine is a 2100 w. 1 21064, 256MB RAM, DAC960.
>
> I have installed SuSE 7.1 w. kernel 2.2.19. Thats the only way i can
> boot w. Linux.
> The other kernels do not work (hang during boot).
>
> Regardless which kernel i compile (2.2.19 with same .config like the
> running kernel, 2.4.1, 2.4.5) i can not boot. Either the machine hangs
> (2.2.19) or the NCR53810 has an endless time-out...
>
> Any suggestions??

Are you using ARC or SRM? That sounds exactly like the problem I was having.
It would crash on something like:

Spinning up sda...

The only way I could boot in ARC is if I did the following:

- - In ARC go to the 'Supplementary Menu' and select 'Display Hardware
Configuration'
- - Once in MILO type 'ls <device>' where device is the device you are booting
from, eg: 'scd0'

It was very strange and all of my problems were fixed by using SRM instead. I
have all of my notes at:

http://arthur.funktronics.ca/arthur/AXPInstallation

In the end I installed with ARC so that LVM would be installed, then I
recreated my first disk (it was non-LVM) with BSD disklabels to that SRM
could boot it.

Hope that helps.
