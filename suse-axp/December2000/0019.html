Hello all,

the weekend is over and I installed the new SuSE Linux 7.0 on my Multia. I
upgraded the firmware of the box to the latest level that was ever
released, switched to the SRM console, booted from the Compaq Jumpstart CD
and prepared the Disk for installation. After fiddling around with boot
parameters for a short time I managed to boot from the SuSE CD and
installed the system. The installed system booted from SRM without
problems. Never before I managed to set up an Alpha with Linux so fast!

Unfortunately ther is one small problem left:

SRM sends all output to the onboard TGA adapter and seems not to initialize
the additional ViRGE/DX PCI adapter. Before I updated the system, all
output went to the ViRGE/DX board. Is there any possibility to disable the
onboard TGA and use the PCI-VGA adapter instead? Do I have to go back to
the ARC firmware? Did I miss something during the firmware upgrade?

I would be happy about any hint!

best regards
