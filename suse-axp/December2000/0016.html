Hi all,

i have a problem with the installation of suse 7.0 on my aspen alpine board
with arc-console version 4.35 and ncr53c810 on-board scsi controller.
I'm not able to boot any kernel file. The net says i would have to use the
EB64p image to create a boot disk.

1. try:
  Using Milo 2.2:
  I created a boot disk from image CD1/Disks/EB64P
  I do NOT get the milo prompt. Before that milo hangs.
  There is a message on the screen telling me something about CABRIO

2. try:
  Fallback using Milo 2.0.35:
  I created a boot disk from CD1/Unsorted/Disks_2.0/EB64_
  I get the milo-prompt but when vmlinux.gz gets loaded i get
  the error message "unexpected interrupt 38" which scrolls
  through the screen.

3. try:
  I tried the CD1/Disks/Cabriole image and wrote it to disk.
  I get the milo prompt, but when i try to boot vm_full.gz from the CD
  the system hangs when reseting the SCSI bus:

" scsi bus is being reset for host 0 channel 0
  ncr53c8xx_reset : pid0 resetflags=2 serial_number=xx
serial_number_at_timeout = yy"

this message scrolls through the screen with incrementing xx and yy
serialnumbers.
It seems for other SCSI controllers the problems occurs too
(see page 300 : suse 7.0 alpha edition handbook: option: aic7xxx=no_reset)

So my simple question: how do i tell the ncr53c810 driver not to reset the
SCSI bus.

thank you !
