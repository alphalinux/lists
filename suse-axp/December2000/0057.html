Hi!
There seem to be problems using X in SuSE 7.0 and Ruffian boards.

I tried it with my old ATI 3D Rage Pro card and yesterday with a SiS 6326.
The ATI card used to work perfectly on SuSE 6.3.

Both X servers (Mach64 and SVGA, XFree 3.3.6) produced the same error messages:
None of the configured devices could be found, but the found card is exactly
the configured one (X puts these in (--) ...)

A mail of Andreas Czerniak on the suse-axp mailing list indicates,
that he has the same problems with his Ruffian system.

So: Is there anybody out here, who has a running X on a Ruffian board
in SuSE 7.0?

Do the newer RedHat distributions have the same problems?

Thanks for any hint,
Martin.

PS: I tried the old milo, the 2.2.18 kernel and a kernel compile
    with -O, but nothing helped.
    Of course, i tried downgrading to the X system of SuSE 6.3, too -
    but i did not succeed with it.
    Perhaps i should downgrade libc, too?
    Other components should not affect X when started by 'X'.
