On Sun, 27 Jun 1999, Andreas Czerniak wrote:

> This is the mail from Ruediger at the beginning of june.
yes, the CDs are being pressed and packaged at the moment. The german book
is finished and the english manual is almost finished.

[...]
As I wrote back then:
> Not all that bad. It's just: I want a kernel that runs solid at least
> on the machines I have around....
> The Distribution itself is mostly finished.
> Candidate for the release kernel is a 2.2.9 with most recent
> patch-set from alphalinux.org/patches plus some other updates
> (basically same as on SuSE-x86). This kernel runs fine (even as SMP),
> but gives some trouble (unaligned traps for random applications) on
> Ruffian machines.
Yes, this problem turned out to be far worse: something invented in
the changes in 2.2.8 caused this (mis)behaviour (not only on the
Ruffians) and presently you've got the choice between two bandaids: the
one I'm using costs a little bit of performance (though I don't think its
even a full percent) and the other one is turning off the vfork system
call.

As far as I know, a clean solution has not been found yet, but both
above versions result in a steble machine. (The kernel on the CDs have
the first version which use an extra pal-call on every process switch).
