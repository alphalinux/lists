On Mon, 20 Mar 2000, G. Hugh Song wrote:

>
> Today, I was told a very interesting story.
> On an SRM-console machine, "aboot" process is specific to
> each installation.
>
> Well. I really don't understand what it means. Let me tell you
> my situation. The vendor installed RedHat-6.1 on an LVD hard disk
> connected to the single-ended SCSI connector of UP2000 motherboard.
> He should have connected it to the LVD SCSI connector.
>
> He told me that because he created aboot while he was installing
> RedHat-6.1 with the connector hooked to the single-ended SCSI connector,
> simply switching the SCSI cable to the correct LVD SCSI connector
> would not let me use the existing "aboot". So he said the boot
> will fail.
>
> Yes, I failed to boot the machine when I experimented with the
> SCSI cable connection.

Then someting else is wrong, the UP2000 only has 1 SCSI channel, so moving
the connector from the SE port to the LVD port should have no effect.
 
> I thought that AlphaBIOS is a lot more flexible than SRM-console.
> SRM-console is like Windows95, in which whenever the system is
> moving between hard disks one should reinstall everything from
> scratch.

Well, AlphaBIOS is going away, and while there is a MILO for the
DP264/UP2000 it's not very reliable, you are in much better shape using
SRM console.

Now, do you know which version of SRM is on this machine? It's printed out
when SRM starts and it will be either V5.5-82 or A5.5-82 (unless somebody
else has already changed it). The difference is that A5.5-82 will see and
be able to use the onboard SCSI, V5.5-82 will not.
