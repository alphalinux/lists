> > here's the script called find_stalls
> > it simply looks for float registers that appear as target on one line
> > and anywhere on the next in the assembler code. A clear indication of
> > pipeline stalls. I then identified the guilty parts of the C code and
> > changed it around (e.g. using temporaries) until the stalls were gone.
> > And performance was way up (by a factor of about 5 :-)
> >
> > #!/usr/bin/perl -wp
> > # usage:
> > #objdump -d myobject.o | find_stalls|more
> > #then search for UGLY!
> >
> > use strict;
> > use vars qw/$t/;
> >
> > $t && /\Q$t,/ && s/$/\t\t#UGLY!!/;
> > $t = (/(?:mult|subt|addt|ldt).*(\$f\d+)/)[0];
> > #target is last floating register arg
