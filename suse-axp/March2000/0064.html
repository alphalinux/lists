Hi,

it seems to be a kernel problem. This is one of the machines only intended
for Win NT, so I don't know if SRM really works fine (but it did on Multia
also !).

Try a generic kernel from the CD or try a kernel for an AS800 (it seems to be
a small AS800 without support for Digital UNIX). Don't forget to run the ECU
(EISA config utility) after the firmware update !!!

I got similar problems while booting from harddisk because of missing
built-in SCSI-Support on an AS400.
