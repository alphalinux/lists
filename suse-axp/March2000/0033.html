> I have a Digital 3305R Server 500Mhz, 256Mb RAM (NT White Box Server)
>
> I have setup my boot options via SRM I get the follow error below:
>
> The boot disk loads Ok and the ramdisk starts to load but then I get the
> following error:
>
> Halted CPU 0
> Halt Code = 5
> Halt instruction executed

at console prompt. It helped some people.

: There are rumours that if you switch the 5300 to SRM,
: edit the nvram and add a line

: 10 set srm_boot true

: That when you boot the 5300 the SRM will choose NOT to
: 'poison' the HWRPB and will identify itself with the system
: type of an AS 1200. This is, of course, not supported.
: However, it is the way I run Hobbyist OpenVMS on my Digital Server
: 3305. Not that the system MUST be set to reset on boot, otherwise
: the environment variable doesn't get set quite right and it
: doesn't work.

: Eric Ebinger
