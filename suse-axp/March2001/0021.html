Hi! I am trying to install 7.0 on the PWS500 and am having a problem. I
am booting from the CD through SRM to begin the install. I get to the
point where Yast restarts (after partitioning, package install, and
network setup) and is supposed to ask for me to set the root password and
a user account. BUT YAST SKIPS THESE STEPS, moving straight to modem and
mouse configuration. Therefore, I am left with a system that has no root
password, so it's basically useless.

Am I doing something wrong to cause the root password and user account
steps to be skipped? Has anybody else seen this problem?

Other items that might shed light on the problem:

- MILO 2.2 in AlphaBIOS does not work on my system. MILO 2.0.35 in
AlphaBIOS does, but gives me the same problem as the SRM/CDROM approach.

- I have 2 SCSI hard drives on my system. I am only partitioning and
formatting one, however. During installation, I select the "Whole Disk
Partitioning" option for that disk.

- I have an IDE CDROM drive. However, my system recognizes it (during
bootup, at least) using some sort of SCSI emulation, so the drive shows up
as SCSI within SRM and AlphaBIOS. This has never caused a problem in the
past, however.

Your help is appreciated.
