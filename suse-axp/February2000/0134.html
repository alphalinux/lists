Trying to install 6.3 onto a PC164lx w a 9 GB Seagate SCSI drive hosted
by a Buslogic 958 controller. It loads Milo, but no joy getting a linux
install to boot. Milo regularly reports cant mount /sda2

Got SuSE 6.3 beta installed and running using a boot disk which
contained the Buslogic host controller driver. Then hard drive
crashed. RMA'd drive; got a new 9 GB Cheetah; now trying to install 6.3
using same install diskette and the 4 6.3 CD's. All seems fine thru
selecting standard AXP kernel (I was told the std kernel of the regular
6.3 release contained the buslogic driver) and installing it. Then
install process asks to reboot and when it tries, screen dissolves and
returns with underscore cursor on a black screen. No response to
keyboard; no alternate consoles available.

Tried to find a way to copy or dd the replacement kernel I got from SuSE
onto sda2; but have not been able to do so.

I have tried to use the "boot installed system" option when I reboot to
run the install. But, I get only a repeat of the black screen with
underscore; no response to keyboard; no consoles available.

Do I need to build my own monolithic kernel including Buslogic drivers
and install that kernel in sda2? Suggestions? I haven't figured out
yet how to copy to sda2; machine shows it symlinked to a cdrom file and
won't accept copying.

Drive (1106 sectors) configured
   sda1 6-7 MB dos partition /dos for linload.exe and milo [I went
outside the install process and used an alternate console to copy these
to /sda1]
   
   sda2 500 MB ext2fs partition for / (including /boot)

Alphabios 5.66-1.
