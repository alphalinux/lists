Two questions arising from trying to RTFM:

1. Section 2.1.7 tells me I "must change the block size of the
partitions [to be mounted as / and /boot] to 1024 Bytes, and it advises
me that more information on this point can be found in Section 3.3.9.
But, I find no mention of block size choices (as opposed to sizing
partitions themselves) in 3.3.9. Near the end of Section 3.3.10, there
is an italicized heading "Block size," but the adjacent text addresses
other variables reached by using the "F5 Expert option" but does not
address the "block size" variable. It looks as though someone intended
to write some info, but forgot.

2. In Section 2.1.7, there is a graphical rectangle drawn around a
section of text and a warning triangle posted in the margin. The text
in the rectangle advises that default block size in fdisk is currently
4096 bytes, a size not supported by MILO or ABOOT. However, I find this
confusing because:

  a. Either the fdisk whose default block size is 4096 bytes has
already been corrected by the SuSE install process (because the F5
screen shows the default Block size for every ext2fs file system set up
in the install process is already 1024 bytes)--in which case, the needed
change has already been made and why the emphatic warning that making
this change is "absolutely essential"; or,

  b. Perhaps the Install manual text should refer to some other
variable, such as inode density, which the SuSE install does set to 4096
bytes.

What _should_ the manual say?

BTW, in Section 2.1.7, the sentence containing the statement, "the block
size of the partitions in / and /boot (in our example, /dev/sda2 and
/dev/sda4)" seems to list the partitions backwards rather than
respectively and seems wrong as to /dev/sda2. Or, am I looking at the
wrong example? The only example I found was a list of partition names on
the prior page, where /boot is listed first and / is listed fourth.
Wouldn't that make / and /boot /dev/sda4 and /dev/sda1, respectively?

Maybe I'm relying too much on the manual here, but I find it less than
helpful.
