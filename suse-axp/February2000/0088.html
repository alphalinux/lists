I would count on that once rpc.nfsd changes his uid for to the user uid, it
can't 'execute' directory = access to the is denied. Change the mode to 777.

Additionally there is an nfsd bug with user squashing which might
additionally affect you. Be sure to use no_all_squash to avoid it:

http://sdb.suse.de/sdb/en/html/kukuk_nfs.html
