Hi list,

I was finally able to get SuSE AXP 6.3 up and running on my AS250 4/266
thanks to the new TGA kernels that R"udiger supplied last night.

However, in the installation process the device names for my two
harddrives somehow got switched around. What used to be sda became sdb
and vice versa. Since I didn't pay enough attention I partitioned and
formatted the wrong drive and thus overrode what was there originally
together with my previous 6.1 installation. Well, that should not have
been a big deal since I was planning a complete fresh install after some
initial testing anyway. Unfortunately, what has happened in the process
now is that SRM doesn't recognize block 0 of dka0 as a valid boot block
anymore. After noticing my error I repartitioned the drive manually and
wrote the BSD disklabels to it leaving the first couple of blocks
unassigned. Thus the disklabels on /dev/sdb (which used to be
/dev/sda) now look like this:

# size offset type fsize bsize cpg

a: 18117 14091 ext2 # (Cyl. 8 - 16)
b: 263703 32208 swap # (Cyl. 17 - 147)
c: 2049234 0 unused # (Cyl. 1 - 1018)
d: 1753323 295911 ext2 # (Cyl. 148 - 1018)

Trying to boot with the command
>>> boot dka0 -fl 0
produces the following error:
block 0 of dka0.0.0.6.0 is not a valid boot block
bootstrap failure

Looks like my boot block on dka0 got trashed and I need to repair it
somehow. Can someone please give me pointers as to how to do that?

Mind you that SRM still recognizes /dev/sdb as dka0 while /dev/sda is
recognized as dkb0.

TIA, Alex.
