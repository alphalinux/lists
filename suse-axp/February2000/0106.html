On Mon, 7 Feb 2000, Brian Pomerantz wrote:

> I'm in the process of adding an exit-handler and dynamic system call
> API to the kernel and have gotten permission from LLNL to release the
> code under GPL. I'm hoping to get these changes put back into the
> main kernel tree but if that doesn't happen, we'll probably make our
> own tree available for others to download. I've been hoping to find
> out more about who is maintaining the Alpha branch of the kernel, so
> far I haven't seen a very well organized bunch. One of the things we
> are going to be needing in the near future is a better interrupt
> handling system. We're hoping to show that we can match or better
> True64 on a large cluster (128 nodes, >= 2 CPUs each), that may be
> difficult if interrupts are only scheduled on CPU 0.

This sounds very interesting. You might want to get in contact with Andrea
Arcangeli <andrea@suse.de>, he is putting some serious effort into the
Alpha Kernel development right now.
