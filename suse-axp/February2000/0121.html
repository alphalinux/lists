On Sat, 12 Feb 2000, Maz Spork wrote:

> Hi list,
>
> I have that same problem - though on a PC164 with EB firmware.
> For a long time, I thought the problem related to hardware, as
> I believe freezes are typically caused by unacknowledged
> interrupts. Then I thought the problem to be related to
> masquerading and ipchains. But in the end, the SuSE kernel
> (2.2.13) simply locks up. This happens between five and 30
> minutes from boot. I do suspect some I/O or amount of I/O to
> be the problem, but cannot come closer. It is a "byzantine"
> problem in that it cannot be reproduced just by creating the
> same circumstances.
>
> Using /etc/full_vm.gz as a replacement just makes the problem
> worse (i.e., locks up almost instantaneuosly).
>
> However, I had an old 2.2.3 kernel from a red hat distribution
> compiled by myself. This does not lock up the machine - but
> compiling the SuSE kernel 2.2.13 with the same options (and any
> new ones turned off) does not solve the problem.
>
> So I conclude that something has happened between 2.2.3 and
> 2.2.13 that makes the latter unusable on PC164 systems.
Have you tried building and running a monolothic kernel already,
especially with the network-driver compiled into the kernel
and not as a module ?
So far with all my tests here, I could always get a stable
machine by compiling the network driver into the kernel.
I looks like a real kernel problem (bug?), but noone could
trace the symptom to its real cause yet.. :(
