On Thu, 3 Feb 2000 home@jephill.com wrote:

> Three observations, two questions re SuSE 6.3 AXP:
>
> 1. I had installed the SuSE 6.3 AXP demo/beta on one pc164lx, so I
> thought I was ready to install the released version of SuSE 6.3 AXP on
> an identical machine. In sec. 2.1.1 (p.16) of SuSE's printed manual, I
> found the sentence, "This 'initdisk' is contained on the floppy disk
> provided in the box." There was no floppy in my shrink-wrapped SuSE 6.3
> AXP distro box. Was my shrink-wrapped package incomplete; or, is the
> manual wrong? [I have already d/l'd an install disk image from the SuSE
> ftp site and made the disk.]
No, unfortunately, die "install" disk is not part of the package.
>
> 2. In Sec. 2.1.2, under the heading "MILO: Booting via ARC Firmware,"
> the SuSE manual describes briefly some boot steps which may be relevant
> to a ruffian box, but bear no relationship to the screens seen or lines
> and variables required to be filled out while booting a pc164lx. Has
> anyone seen a page or so of _correct_info_ about filling in the blanks
> on the pc164lx's crochety firmware?
What are the variable names on lx164 ? I'll be glad to fix this soon.
>
> 3. In Sec. 2.1.7 (p. 23), the SuSE manual mentions the need to change
> the block size of the DOS partition to be created for milo. It says
> this can be done using "Expert Menu." But, when I highlight the DOS
> partition line in the linuxrc screen, the "F5=Expert" option is grayed
> out. Neither is it activated when I go into "Installation Using Expert
> mode."
> Best workaround I found was to create the partition first as a type 83,
> then use Expert menu to reset block size to 1024, then go to Console and
> use fdisk to change the partition type to 6, then reboot. Is this a
> glitch in linuxrc or a glitch in the manual?
If it's really like this in the manual, the manual is wrong. The 1024
blocksize is meant for an ext2-partition if MILO wants to load the kernel
from ext2 (some of the MILOs do have a bug that prevents loadin a kernel
from a DOS partition on a harddisk...)
