Hello Michael,

Well, here is how i did it. Setup is: APXpci33 with scsi harddisk and cdrom.

- create the two floppys (noname and install)
- start the boot from the floppy (without options in the bios)
- insert cd1 in cdrom drive
- after the milo prompt type 'boot scd0:/etc/vm_full.gz root=/dev/fd0'
- continue as usual, when it comes to partitioning i did the following:
     sda1: /boot ext2 10Mb (see NOTE !!!!)
     sda2: /dos msdos 10Mb
     sda3: swap 128 Mb (same as installed memory)
     sda4: / ext2 (the rest that is left)
     NOTE: sda1: has to be modified to use blocks of 1024 bytes otherwise
loading
     the kernel during boot will fail !!!!
- install the rest as normal
- when it comes to choose the kernel choose whatever you want. System will
now proberly hang.
- reset computer and start from floppy again until you get to the MILO prompt.
- type 'boot scd0:/etc/vm_full.gz root=/dev/sda4'
- system will now boot. Now the last few things can be done.
- from the noname floppy copy linload.exe and milo to the /dos partition
(mcopy).
- from the cd1 copy the /etc/vm_full.gz kernel to the /boot partition.
- halt linux and restart the computer.
- add setings to bios with as options 'boot sda1:vm_full.gz root=/dev/sda4'
and referring to linload.exe and milo in the second partition (partition(2)).
- ready

Hope it helps,

Richard Schiffelers

The thing that caused the biggest problem for me was the fact that the ext2
partition with the kernel needs to be made out of 1024 bytes blocks. I read
over that part in the manual. Did however eventualy find it.
