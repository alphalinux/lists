Hello together,

at home i have a PC164 Board (500 MHZ 21164A) with ARC-console from
*obis with the following Hardware configuration:

- 256 MB RAM
- Adaptec 2940UW SCSI-Controller (PCI)
- Matrox Millenium I 8MB (PCI)
- Soundblaster 16 ASP (ISA-Bus)
- Teles 16.3 ISDN-Card (ISA-Bus)

When i am running Linux then the machine unexpectedly hangs completely
and forever (no Sysrq helps out). This happens when doing backups or
when going online with ISDN. There seems no deterministic behaviour when
the machine crashes.
I changed the memory modules back to the original 64MB Ram but nothing
changes.

When going online with lynx from Text-console all works fine for some
time. When under X with kfm the machine locks up after a short time. I
assumed the X-Server as the failing component and therefore went to
XFree 3.3.5 but that doesn't change anything :-( and so i went back to
the original Xfree86 shipped with 6.1.
On the machine i also have a running NT 4.0 System and it runs without
problems (and hangs) ...

Has anybody experienced such a strange behaviour on that kind of alpha
hardware ?

Sometimes it would be nice if could write a dump or go into a
kernel-debugger but this is not implemented yet :-|
