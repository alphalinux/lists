> After installing SuSE I copied the whole /var/mysql from my current
> SuSE (Intel) to /var/mysql on the Alpha (of course expecting the
> /var/mysql/mysql !) - now I tried mysql <database> .. but what is
> that?
>
> Didn't find any fields in table <Table>
>
> The old mysql-Version is 3.22.19b, the Alpha uses 3.22.21
>
> Copying the tables has always worked fine - so I'm a little confused
> now :-(

I am pretty sure, that mySQL tables are not binary compatible between
Intel and AXP. However, mySQL on Linux/AXP seems to be broken anyway.
We´re waiting for a patch.

> How can I import the old databases?

Use mysqldump:

Intel# mysqldump database > database.out

(copy database to AXP)

AXP# mysqladmin create database
AXP# mysql database < database.out

That´s it!
