I've finally managed to install SuSE on my Avanti (AS 200 4/166) using SRM
and BSD partitions.

1) (in reply to Kai Schmidt) I must stress that you should boot using the
"aboot" disk image when booting from SRM.
2) (in contrast to the guide
http://www.digital.com/alphaserver/linux/ds_linux_install_guide.pdf) I did
NOT start the installation by manually creating BSD partitions. Instead, I
used YaST to set up ordinary partitions. After the installation, before
reboot, I manually wiped the DOS partition table and created matching BSD
partitions.

The DOS partitions I used were

1: A 2-cylinder dummy FAT partition on cylinders 1-2 (unmounted)
2: A giant Linux native (ext2) partition starting at cylinder 3, mounted at
/.
3: A swap partition at the last 256MB of the disk.

This permitted YaST to complete the installation. (Don't forget to include
the "aboot" package!)

When the installation of alla packages was completed, I wiped the DOS
partition table (using the dd command, as per the hint in the Appendix E of
the guide), and created BSD partitions matching the DOS table.

I could now reboot and finish the installation exactly as described in the
guide, after which I loaded Rüdiger's updated YaST.
