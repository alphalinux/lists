Are you specifiying the type of machine in the config file or just using
Generic? The other thing that lots of people miss is the 'Boot from SRM?'
setting, as it is ONLY available if you select something other than generic.
If you try booting from SRM without that switch, you will get a behavior
similar to what you are describing. The other thing is if you select the
wrong machine type you also get a similar result. There should be some
information in the boot log (boot with the bad kernel, then the good one) to
help you isolate the problem. The other issue is that alot of the stuff for
a base SuSE kernel is modules, so you might need a ramdisk, though I have
never used one on Alpha.
