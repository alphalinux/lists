I'm still struggling to compile a working kernel on my alpha. I've managed
to do it on my x86s', so I hope I've at least got some inkling of what to
do. But perhaps I'm missing some crucial step on my Alphas.

My problem: When I boot my newly compiled kernel, the system either crashes,
halts, or the screen goes blank and I can't hear any activity on the disk.
In all cases I have to reboot.

What I do:

make menuconfig : Trying to guess appropriate values...
make dep
make modules
make modules_install
make boot
cp /usr/src/linux-2.2.9/arch/alpha/boot/vmlinuz.gz /boot/vmlinuz (quoting
path from memory)
cp /usr/src/linux/System.map /boot

(I've previously save the original vmlinuz and System.map so I've got a
chance to boot something that works...)

Reboot (from SRM), which fails.

Am I doing something basically wrong here? I've tried reinstalling the
default kernel (to get a .config) to start with, but even so I'm unable to
compile a working kernel.

I've tried this both on my Avanti, which boots from SRM, and on my Multia,
which boots from ARC, with equally bad results. Both systems have an
otherwise working SuSE 6.1 installed, but I want my Windows Sound System
cards to work, which apparently requires a kernel recompilation. At least
the oss installation seems to require it.
