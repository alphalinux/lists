> Interesting.
> I do not believe that there is such a thing as "-O4" for ccc

There should be. According to the man page and to my experiments there is.
bladeenc creates mp3s at 1.23X with gcc, 0.33X ccc + no optimization,
1.40X ccc + -O3 and 1.63X ccc + -O4
 
> Probably what it is doing is defaulting to "no optimization" because you are
> passing it a bad parameter.

Without any optimization the ccc creates very slow binaries, but with
optimizations it creates as fast or faster binaries. I'm not quite sure if
the speed boost is the maximum accomplishable but the ccc binary sure is a
lot cleaner and nicer than the gcc one. One can set -O4 on all the files
with ccc but with gcc certain files must be -O1. Way to go!
 
> If you would like I can have the compiler team spin up an explanation and
> reply to this thread.

I'd sure like some tips on how to create faster binaries. The faster the
better! Go ahead and ask them...

/Mattias
