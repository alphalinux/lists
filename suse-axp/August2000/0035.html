"William H. Magill" wrote:

> One issue might be with the startup.
>
> Netscape needs an environment under the user's home directory before it can
> run. (especially for java.) It's supposed to create it, but sometimes
> cannot. That leads to unpredictable behaviour. NIS users might fall into
> this category.
>
> Tru64 uses the following script as /usr/bin/X11/netscape to get around the
> problem of the missing setup, installed as ugo excutable, it checks the
> environment and then launches the "real netscape."
>
> The variable "MOZILLA_HOME" might be the real key.

Thank you for the answer. There is a similar (a little bit more sophisticated)
script for starting netscape in SuSE 6.4 distribution. But it seems that this
script sets at least MOZILLA_HOME correctly in the case of nis user as well
as in the case of local user and creates proper ~/.netscape directory if it's
necessary.

Besides, I've tried to launch netscape directly as well as using the script.
For local users it works in both cases (at least netscape window appears), for nis
users
it doesn't work.
