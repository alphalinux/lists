Hi,

I would like to take the chance to comment upon the 7.1 release
candidate. First of all I think it was a nice idea to release it to the
public. I was quite amazed how stable this thing already is. Well to be
more precise, I found it to no more unstable than 6.2 to 7.0.
        Some of my issues with 7.1 were already there in 6.4 and 7.0:
My hardware:

AlphaServer 1000 5/300 (21164)
192 MB
ncr53c810 scsi controller
SRM 5.4-104
matrox millenium II 8MB
SuSE-2.2.19 kernel

1. midnight commander:
        If mc is run from a console (no X) users beside root just get a black
screen. This is onlý curable by killing the offending mc process. It works
fine though if users run it inside xterms or konsoles. (I did not yet
take the time to investigate this any further though...)

2. KDE 2:
        KDE 2 starts up fine but will sooner or later lock up. The screen
freezes to some extent, but the mouse cursor still is movable (seen under
X4). The keyboard seems dead, but is not really. Ctrl-alt-backspace does
not work. The magic system request combination alt-sysreq-s (sync disks),
alt-sysreq-u (unmount disks), alt-sysreq-b (reboot, issued 20 second
after the unmount command) reboots the system cleanly.
        This seems to be KDE specific, as some preliminary tests with
windowmaker did not result in lock ups, but it is to early to tell.

Ahoi
        Sebastian Moeller
