Hi friends,

my XP1000 (EV6, DP264, Tsunami, Monet) is equiped with a HP C5686A DDS4
(20/40GB) tape drive that is connected to a QLOGIC ISP SCSI controller using
ID 4 (disks have IDs 0 to 3). While doing a tar to /dev/st0 - no matter which
file or dir is archived - the system hangs after 2 to 4 seconds (reset is the
only escape). All mt commands work well and compression is jumpered. I'm
using SuSE 7.1 AXP running kernel version 2.4.4 and GNU file utils that come
with SuSE, kernel tape support is activated. No additional modifications have
been done to the system, because I'm not very familiar with tape drives.

Output of "mt -f /dev/st0 status"
