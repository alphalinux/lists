I am just trying to install SuSE 6.3 AXP on a ds-20.
My Problem is:

I can boot the installed system by booting the CD-ROM and then telling the
installation-program to boot the installed system. That works.

So I correctet the /etc/aboot.conf with the line

0:2/boot/vmlinuz ro root=/dev/sda2

Then I did a swriteboot /dev/sda /boot/bootlx

This all worked!

I have following partitions:

sda 1GB swap
sdb 1GB ext2 mounted on /
sdc The whole disk, unused
sdd rest of the disk (ext2) mounted on /usr

The disk has 16 GB.

Now, going to SRM und typing boot dka0 0 , the system starts to boot until
the lines:

XY82C693: not 100% native mode: will probe irq's later
    ide0: BM-DMA at 0x1200-0x1207, Bios settings: hda:pio,hdb:pio
    ide0: BM-DMA at 0x1208-0x120f, Bios settings: hdc:pio,hdd:pio

There, the system freezes!

It has to be a aboot problem, because i can boot the installed system with
using the first cd.
