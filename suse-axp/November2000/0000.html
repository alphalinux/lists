Hi,

I looked through the mailing list archives about this issue I am having and was able to find someone with the same issue but no resolution posted. Hopefully somone will be able to help me out here.

I have an Alpha XL 266 running ARC version 4.57. I boot the linux installation by booting from milo image for the Alpha XL from the SUSE 6.3 CD.
I use the following boot options:
boot scd0:/etc/vmlinux.gz root=/dev/fd0 fake_initrd

booting is ok. and it loads the ramdisk floppy just fine. Installation starts ok. I set up the following partitions on my hard drive:
/boot - first 10Mb of disk
/dos - next 10MB of disk formatted dos not umsdos
64Mb swap partition
the rest I assign to /

I'm using an NCR SCSI adaptor, Toshiba CD-ROM, Seagate ST32550N SCSI hdd.
The default files are installed onto the disk and runs through all 4 SUSE cd-roms just fine. It then asks me about the network and setting the time issuing the warning that x has not been configured yet. I choose the default AXP kernel and set sendmail to a single user machine with no other connectivity.
After this it tells me the installation must be started so that SUSE may continue and then it goes to a blank screen with a cursor in the top left hand corner and does nothing. No disk activity, nothing...

rebooting the system finds the installation unfinished...

Any help will be much appreciated.

Thanks,

William Moore.
