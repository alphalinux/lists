Iwao MAKINO (makino@hpc.co.jp) wrote:
> hi list-members,

Hi,

> I have following questions;
> On UP2000, we are running SuSE6.3 and
> When I use 2.2.14 as SMP w/ Xosview or TaskManager, it shows 2CPU working.
> But when I switch into 2.2.15 SMP kernel, it will only show as UNI CPU.
> However, system is working as SMP as I tested w/ several test tools.
> Mainly, gemm bench, and mpich benches.
> Anyone knows about this symptoms?

well, this a known problem. You have to use the "RTC light" kernel patch
from Andrea A.

You can find it here:
 ftp://ftp.kernel.org/pub/linux/kernel/people/andrea/kernels/v2.2/2.2.15aa2.bz2

Just patch it againt a plain 2.2.15 kernel.

Activate the new option in the kernel config:

        Character Devices -> Enhanced Real Time Clock Support
          -> Use only lightweight version (no interrupts)

Compile and boot...

This config works here on a UP2000/750 and a DS20E like a charm.

BTW, the original SuSE kernel contains this patch.
