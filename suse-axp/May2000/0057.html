Hi Lennart [cc:] and all others !

Lennart Börjeson wrote:
>
> Now that Compaq has made their own Alpha-optimised C and C++ compilers
> available, I'm sure more Alpha-users than myself have toyed with the idea
of
> building a complete Linux with these tools. I'd guess most of the
> applications and much of the kernel would compile without changes, but
> ofcourse, that's unknown until tested. I know there are assembler parts
and
> inline code which probably would not compile, but these might possibly be
> left to Gnu tools.

I am just taking C code in the other direction:
from Alpha-Tru64 (Compaq compiler) to Alpha-Linux (gcc),
works immediately (admitted, it does not use compiler-specific features
and was written for portability, large parts are tool-generated).
However, there are problems when I combine it with assembler code:
a combination that works on Tru64 fails on Linux.

I have not yet done a complete analysis, but my first suspicion is
that Compaq C (at least on Tru64) might generate structure layouts
which differ from gcc's (even though both are on/for Alpha hardware).

If that should prove true, I fear your idea is hard to realize
because it might require an all-or-nothing decision.

A "compatibility check" of these compilers might help:
- size and alignment of types
- structure layouts, padding etc.
- function calling and parameter passing conventions
- C++ : name mangling
- ...
Is there any source for that kind of info ?
