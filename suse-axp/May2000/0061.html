Well, I have an old multia hanging around at work at that I thought I would
try to do something with. I'm having the hardest time loading suse on it.
First of all, is there any way to boot straight off the cdrom without a
floppy drive? When I try

        boot dka600 -fl 0

it starts to boot up fine but hangs because it's trying to read the root
off /dev/fd0. So, since I don't have the annoying little cable to connect
a floppy drive directly to the machine I decided to go hook up a scsi floppy
(on scsi id 4). I dd'ed disks/aboot_xx onto a floppy and booted with the
command

        boot dka600 -fi /etc/vm_full.gz -fl "rw root=/dev/sdb
         initrd=/etc/initdisk.gz"

this is essentially what's going on in etc/aboot.conf except I changed the
boot image from /dev/fd0 to /dev/sdb (where my floppy drive is). In this
case aboot loads the kernel and recognizes root is on /dev/sdb (and the drive
light goes a couple of times) and then it locks solid with aboot being the last
thing on the screen, specifically:

        aboot: starting kernel /etc/vm_full.gz with arguments rw root=/dev/sdb

Since it did the same thing when I tried to load Mandrake AXP recently I
figure I must be doing something stupid. Can I get some ideas before I push
this poor thing into the back of the closet and ignore it?

thanks
