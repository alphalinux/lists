Hi,
I downloaded ccc from Compaq's ftp site and installed it in my 6.4 beta
machine.

First of all the installation of ccc-6.2.9.002-2.alpha.rpm had a
unsatisfied dependency:

        egcs >= 1.0.3a

there is no such package, so I used the --nodeps option. Next, the
installed files have the following broken links:

/usr/lib/compaq/ccc-6.2.9.002-2/alpha-linux/bin/crtbegin.o ->
/usr/lib/crtbegin.o
/usr/lib/compaq/ccc-6.2.9.002-2/alpha-linux/bin/crtend.o ->
/usr/lib/crtend.o
/usr/lib/compaq/ccc-6.2.9.002-2/alpha-linux/bin/ld ->
/usr/lib/gcc-lib/alpha-redhat-linux/egcs-2.90.29/ld

I replaced those to links to where I found those files:

/usr/lib/gcc-lib/alpha-suse-linux/2.95.2/crtbegin.o
/usr/lib/gcc-lib/alpha-suse-linux/2.95.2/crtend.o
/usr/bin/ld

I tested the compiler with this trivial program:

#include <stdio.h>
main()
{
  printf("Hello\n");
}

I got:

$ ccc main.c -L /usr/lib/gcc-lib/alpha-suse-linux/2.95.2
cc: Severe:
/usr/lib/compaq/ccc-6.2.9.002-2/alpha-linux/include//stddef.h, line 23:
Cannot find file <stddef.h> specified in #include directive.
(noinclfilef)
#include_next <stddef.h>
-^

If I remove the #include statement, it compiles and runs.

The question is, what's broken here? Thanks for any help.
I posted this quetion before but got no answer. 

-- 
Rafael

P.S. The -L option was given so the linker could find libgcc.a
