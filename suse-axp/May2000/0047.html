Hi to all !

I just was caught by big surprise:

In a directory containing files with small and CAPITAL letters
(for example, random order: "ABC" "CEF" "ade" "ccc" ),
the bash expands some patterns from small to capital letters:
    [abc]* -> ade ccc
    [a-c]* -> ABC ade ccc CEF
IMHO, the latter expansion is wrong because the character range
from "a" to "c" (small) does not include "A" or "C" (capital).

I use bash. The effect happens with both "ls" and "echo" ('type echo'
yields "echo is a shell builtin"), so I attribute it to the bash.

Any comments, hints, corrections of my understanding ?
Any environment variable involved (I saw no obvious one) ?

The system is an "Alpha Station 200 4/166" running SuSE 6.1 AXP :
 kernel 2.2.9
 bash 2.02.1(1)-release (alphaev56-unknown-linux-gnu)

Thank you for hints,
