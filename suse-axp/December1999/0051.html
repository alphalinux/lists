I am trying to install SuSE's Beta of its 6.3 release of linux for Alpha
on the PCI64LX. I would prefer to use SuSE's 6.3 AXP distro, but it
seems to lack a proper SCSI module for my BusLogic BT-958. Where's the
SCSI module; or, how do I install SuSE 6.3 AXP w/o either the module or
a different installation image? (As noted below, the image I used is
/milo/LX164/vmlinux.gz [= /disks/LX164] from the 6.3 Beta CD)

Sorry to bother all with this, but SuSE saved a ton of $ by not
bothering to translate the short README.AXP into English (or maybe it
would not have helped anyway).

THE APPARENT PROBLEM:
>From the Linuxrc v0.91 (Kernel 2.2.13) console screen which includes
options "Settings," "System Information," "Kernel Modules," "Start
installation/system," and "End/Reboot" I choose the "Kernel Modules"
option and get a detailed screen for loading/unloading various modules.
However, when I click "Load SCSI module" to load a SCSI module (for my
BusLogic BT-958 host controller), the install program makes no response.
[I am familiar with how it works, because I have installed SuSE versions
6.3 and earlier for Intel many times] If,however, I click either "Load
network card module" or "Autoload of modules" or "Show loaded modules"
or "Unload modules," the install program responds. Clicking "Load
network card module" or "Autoload modules" loads a network card module
(for my DEC Tulip 10/100 NIC), but no SCSI module. {I don't see anything
even vaguely labeled as a modules disk on any part of the CDROM and
README.AXP scheint nur auf Deutsch.}

If I ignore this, choose the "Back" button and choose Start
installation/ system" then choose "Start installation" and "CDROM" the
cdrom is mounted, YaST 1.02C starts and gives me 4 choices, including
"Install linux from scratch." Choosing "Install linux from scratch," I
get a screen displaying the choice to partition or not or set up LVM.
If I choose not to partition, I get a following screen which bears the
blue-highlighted message "No partitions available." If I choose to
partition, I get a red screen with yellow letters saying "NO HARD DISK
FOUND No hard drives were found on your system. Maybe your hardware
was not correctly recognized by the Linux Kernel.

MY BOOT PROCESS

I created floppies from the SuSE 6.3 Beta CD

    from /disks/LX164---------------> rawrite3 to floppy 1
    from /milo/LX164/loadlin------||
                    /milo---------||rawrite or copy to floppy 2
                    /vmlinux.gz---||
    from /etc/initdisk.gz-----------> dd'd to floppy
3

Using either floppy 1 or 2 to boot, I then insert floppy 3 when the
machine calls for insertion of a ramdisk, and the machine goes to the
linuxrc installation console, from which YaST is launched.

OS Load options set up:

Boot Name: Milo
Boot File: CD [rarely allows an "A:"] Milo\Lx164\Linload
OS Path: CD [it will not allow an "A"] Milo\Lx164\Milo

>From the Milo prompt, I issue the following boot command:

  boot fd0:vmlinux.gz root=/dev/fd0

Then, when a ramdisk floppy is requested, I insert the floppy to which I
dd'd /etc/initdisk.gz. The ramdisk at /etc/initrd.gz is recognized as a
ramdisk and expanded, but then results in a repeated message, "signal
11!"

I have looked for, but not found, a file which would give me another
kernel or ramdisk image which might either be used to make a "modules"
disk or a alternate boot image which would recognize my BusLogic BT-958
SCSI host controller.

OTHER POSSIBILITIES EXHAUSTED

Under YaST's "Start installation/system" screen, I have tried several
options other than "Start installation":
  
  Boot installed system--->result: Requires a valid root partition, but
there isn't any visible, nor any available on the SuSE CD

  Start rescue system--->result: Requires a valid root partition, but
none is available

  Start live file system--->result: no live file system on CDROM, none
on reachable HD, none on floppy.
