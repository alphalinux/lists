On Fri, 5 Nov 1999, Lennart Börjeson wrote:

> I trying to install SuSE on my AS200. (Having previously made a successful
> installation on my Multia.)
[...]
>
> Even after partitioning and rebooting, Yast can't find the partitions. (In
> the screen "Creating filesystems" I only see my existing /dev/sda
> partitions.) If I enter fdisk again, it complains about "neither DOS nor Sun
> disklabel found", but when I enter "b" then "p", my prevoiusly (during
> previous boot) created partition table looks OK.
>
> Also, during boot I can see "Checking partitions, sda: sda1 sda2 sda3 sda4,
> sdb: sdb1 sdb2 sdb3", so I must have done *something* right.

ok, that is the old annoying bug: yast would not check for BSD partitions
on the second disk if it found a DOS-partition table on the first dist.
There should be an updated YaST package on ftp.suse.com, but that is hardly
usable at this step of installation.
(If you have enough space on another machine to copy CD1 to a disk and do a
nfs-based install, I could tell you how to make use of the update package).

>
> Anyway, I'm unable to get past the Creating Filesystems step in Yast. What
> to do?
>
> BTW: The disk is a 9GB IBM Deskstar. This is the partition table I've
> created:
>
> 3 partitions:
> # size offset fstype [fsize bsize cpg]
> a: 4096 0 boot # (Cyl. 1 - 2)
> b: 524288 4096 swap # (Cyl. 3 - 258)
> c: 17387520 528384 ext2 # (Cyl. 259 - 8748)
>
> (The AS200 has 128MB RAM. The number 8748 matches the number reported (1
> less than default maximum), as per the guidelines mentioned above.)
