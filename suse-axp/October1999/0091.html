I have a Multia on which I have installed Suse V6.0 which was given to me
by a friend. The manual is in German, which I
am unable to read.

I have a 2 GB IDE drive in this Multia onto which I have successfully
installed Suse. I can boot via SRM from
a boot floppy, and the system runs beautifully.

I'd like to be able to boot from the IDE drive, but when I try to do so,
early in the boot process, I get an error message

VFS Kernel Panic unable to mount file system.

I have the following partions on the IDE drive: hda1, hda2, hda3.
hda1 is msdos type and has bootlx, vmlinuz, system.map. and mounts at /boot
hda2 is swap
hda3 is ext2 mounted at /

I set up the ARC boot selection so that:

OSLOADER=multi(0)disk(0)rdisk(0)partition(1)\LINLOAD
OSLOADOPTIONS=boot hda1:vmlinuz root=/dev/hda3
OSLOADFILENAME=\milo

I copied linload and noname from the \milo directory on CD1 to a msdos
floppy, and then copied them into /boot
after mounting the floppy via a mount -t msdos /dev/fd0 /mnt command.

I also copied the file vmlinux.gz into /boot from my boot floppy which
works from SRM.

I get the same error message whether I try to boot vmlinuz or vmlinux.gz.

I have made RedHat systems boot from the same type of IDE drive in a
Multia, but that was a long time ago, and I'd rather
be running Suse.

When I verify the boot selection in ARC, it checks OK, and when I boot this
boot selection from ARC,
linload then Milo starts, then MILO starts loading the kernel from the IDE
drive, then I get the kernel panic.

What am I doing wrong??

Thanks and regards in advance
