I feel your pain.
I just endured the first two chapters of the SuSE manual
and it is worse, by far, than any Microsoft documentation around.
That I'm brand new (two weeks old) to UNIX/Linux served to improve
neither the installation process nor my disposition a whit.
If you'll consider a newbie's input, here's what I did on my XP1000 (and it
worked!):

See if aboot is installed
# rpm -q aboot

If it's not installed, you'll need to install it.
Get to the configuration screen where you get to add/change
your configuration. See Section 3.12.3 in the gawdawful manual.
Highlight the 'a' series of software and hit return.
You may notice that aboot is not selected. Imagine that.
Select it and hit F10 a couple times to get back to the
screen where you can install what you've chosen.
Let 'er rip and aboot should get installed.
Once you've verified that aboot is installed,

# swriteboot /dev/sda /boot/bootlx

and, assuming you partitioned the drive as specified

>>> b dka0 -fi 2/boot/vmlinuz -fl root=/dev/sda2

does the job.
That "boot dkb0 0" didn't do squat for me either,
although I had made the requisite entry into /etc/aboot.conf
I'm prolly missing something, but it does boot, so how bad could it be?
Any refinement upon /correction to these instructions would be appreciated.
