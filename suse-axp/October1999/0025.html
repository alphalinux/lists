On Fri, 8 Oct 1999, Izaak Bozof wrote:

> I have used the recently-posted XF86Config file which was stated as
> working, but--
>
> when I run (by 'startx') the system gives the following error:
>
> The file XF86_xxxx (binary of X-server) doesn't seem to be a binary file.
> Please check it now / or reinstall a new X-server-binary.
> I'm aborting now.
>
> I checked that the XF86_TGA was installed and it was, so I re-installed the
> package. Same results.
>
> I executed a "file" on the XF86_TGA file and this is the reply:
>
> XF86_TGA: ELF 64-bit LSB executable, Alpha (unofficial), version 1.
>
> Help!
usually /usr is (possibly) readonly, so we need a way to
link /usr/X11R6/bin/X -> /usr/X11R6/bin/$REAL_SERVER
we use: /usr/X11R6/bin/X -> /var/X11R6/bin/X (this should already exist).
Now add: /var/X11R6/bin/X -> /usr/X11R6/bin/XF86_TGA
and you should be ready to go.
