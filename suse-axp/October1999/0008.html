Could someone please advice on how to set the ARC boot selections on a
Multia?

I've booted from floppy (using the exact boot selection specified in the
SuSE manual) and installed the minimum configuration on my third HD, sdc.
I've partitioned the whole disk for Linux, so I've got sdc1: (FAT, boot),
sdc2: (swap) and sdc3: (root)

I've managed to boot this system only by booting from floppy, skipping the
install floppy and telling MILO to "boot sdc1:vmlinuz root=/dev/sdc3".

The boot selection I've tried is:

LOADIDENTIFIER=Boot Linux from HD
SYSTEMPARTITION=scsi(0)disk(6)rdisk(0)partition(1)
OSLOADER=scsi(0)disk(6)rdisk(0)partition(1)\linload.exe
OSLOADPARTITION=scsi(0)disk(6)rdisk(0)partition(1)
OSLOADFILENAME=\milo
OSLOADOPTIONS=boot sdc1:vmlinuz root=/dev/sdc3

This gives an error, "error in device name", so ARC can't find linload
and/or milo. I can't either from my floppy-booted system: I tried "ls -R / |
grep linload", which didn't find a thing.
