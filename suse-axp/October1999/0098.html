Not using the 'C' partion and setting it to the hole disk is the standard
(if you want your disk to work with other OS's...) but as far as I know
there is no requirement saying it must be that way.

In most cases these are not multi-os machines so you should be fine using it
for anything. Also you can skip partion letters, however Linux ignores that
and does a direct letter to number mapping. So:

> a: 4096 0 boot # (Cyl. 1 - 2)
> b: 524288 4096 swap # (Cyl. 3 -
258)
> d: 17387520 528384 ext2 # (Cyl. 259 -
8748)
   ^^^
is
        sdb1, sdb2, sbd4

Jon
