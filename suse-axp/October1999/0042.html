I just received my SUSE Alpha AXP distribution CD's 1 - 4 with printed
moderately incomplete manual, ordered online at SUSE web site.

The issue with CD 1 missing items is verified here also.
Yes, I'm stuck looking at the contents of these CD's on Win98SE box until
Linux AXP installs on my Miata 500a & 600a.

CD1:\dosutils does not exist.
CD1:\disks does not have any kernels as mentioned in the manual also.
CD1:\disks does have Milo images for various Alpha platforms along with
Install image.
CD1:\setup.exe, not.

Boot instructions are incorrect, although manual says stuff about it being
for certain configurations in this section.

So, Rawrite milo and install disks using a DOS machine with RedHat copy of
rawrite.exe (or download)

Note to Compaq or ARC app developers:
Run Maintenance Program is available of ARC & AlphaBIOS machines. I realize
your focus on Linux pre-built systems are SRM-based, but that also has a way
to run Alpha code. It seems extremely odd that your DS/ES - Linux install
book has (paraphrased) "Use Intel DOS or NT to create boot and root/install
floppies for installing Linux on Alpha." I'm surprised that's not on Ms's
"Let's poke a little fun at Linux" site that just went up. <big smile>

Once I get this install process down to where it actually works most of the
time, I'll drop a short note back here with trimmed-down steps.
