The SIA stands for "Security Integration Architecture," in Tru64. This is
part of the C2 security environment.

"The matrix.conf file provides the matrix that selects the appropriate
 installed security mechanism when a security-sensitive command is executed."

It actually defines which binaries are executed.

"The siainit command is the SIA's initialization command and is executed
 only at a reboot. The siainit command informs the configured security
 mechanisms of a reboot and allows each security mechanism to do its own
 reboot initialization."

Both of these files must be included in ~ftp/etc/sia if you want to
implement anonymous FTP under C2.

siainitgood is actually a zero length file which is created at boot time by
        /sbin/rc3.d/S27sia

For FTP one normally simply uses "touch" to create the file.

As I said, it's part of the Tru64 Unix SIA/C2 Security infrastructure, so
I can understand why Netscape might use it -- since it's the same code as
from Tru64. However, I don't know that there is anything else in Alpha
Linux that has even a vague notion of what security means, let alone C2.

The matirx.conf file is group/world readable.
siainitgood is readable and owned only by root under Tru64.
However, I would guess that without the SIA libraries and the kernel hooks,
that it might need to be world and group readable for Netscape to use.
/etc/sia is world and group read/execute.
