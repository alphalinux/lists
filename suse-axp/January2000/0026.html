On Mon, 17 Jan 2000, Eugen R. Neuber wrote:

> Hi to all!
>
> I tried to update my AS200 4/233,NCR810 system from 6.1beta to 6.3 last
> weekend. No success! I had no trouble to get into update procedure (after
> loading the ncr53c8xx driver via menu option) but it was not possible to
> mount my /home partition due to a reported error in a superblock. It was
> not possible to 'fix' this error (6.1beta worked without problems with
> this partition -- I did a backup just before, and I tried later once
> more). So I decided to make a complete installation. This worked as
> expected -- until the warm start...
>
> now yast tries to load/start the precompiled kernel and failes!
>
> A reboot with the kernel from the Avanti-disk shows:
>
> there is no support for NCR810 with this(+) precompiled kernel!
>
> so my AS200 does not boot anymore.
>
> (+) I installed generic kernel since I have no use SMP or TGA
>
> (I will try to compile a new kernel on another Linux/AXP 6.1)
>
> SuSE: maybe you could add a kernel with NCR810 support as you did for TGA?
It's there:

The kernel with SCSI-Adapters compiled in is installed to /boot/vmlinuz

If you can't access this, try:
 - use the bootdisk like for installation
   and add the kernel-parameter "fake_initrd" (as in the manual)
 - insert the install-disk, load the scsi-module and then
   choose "boot-installed-system"

Now you can copy /boot/vmlinuz to whereever you need it.

> Rest: DON'T LOOSE your bootable 6.1 kernels on AS200/NCR810!
