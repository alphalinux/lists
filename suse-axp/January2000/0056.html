Hi folks, my apologies in advance for the following spew, and if I'm
missing something obvious please let me know, but I've been chasing
this for a week and...

<rant>
I've had similar experiences w/ both RH 6.1 & SuSE 6.3. Both of these
seem to lock you out of using linux fdisk and ptn the disk as they
see fit. There's no *need* to create 5 secondary ptns, but these
guys seem to insist on taking that decision away from the admin
who's creating the box. They also don't appear to modify their build
scripts very well (from the intel) to accommodate the differences in
bootloader installation. This is not a problem if you have control
over how the disk is cut up, and wasn't a problem in RH5.x (dunno about
SuSE), but it's a pretty thorough frustration with the current
releases. I'd like to do something simple like:

sda1 = 8M msdos (linload, milo, kernel if necessary, etc)
sda2 = 2048M ext2 /
sda3 = 160M swap
sda4 = 2000M ext2 /opt or /home (symlinkable to whatever)

Additionally, the SuSE-6.3 default kernel doesn't support the 53c810
scsi host. Having milo load /etc/vm_full.gz from the cd gets past that,
but it's one more thing that isn't documented well. I suppose I could load
up an earlier RH and ptn the disk then do the install, but that really
shouldn't be necessary.
</rant>
