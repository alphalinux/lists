For those that want to run
Tru64 binaries (like Netscape.exe) on SuSE Linux, you must rebuild your
kernel.

I need to send the enclosed information to someone at SuSE who is
responsible for
building the kernel that it distributed for SuSE Linux for Alpha-AXP.
Could someone little-reply to me as to who to send such requests to?

Thanks in advance.
__________________________________________________

Dear SuSE representative, who builds the kernels for distribution of SuSE
Linux for Alpha-AXP releases:

It is essential that SuSE Linux support A.OUT binaries by default in the
kernels distributed for Alpha-AXP.
The source configuration distributed with SuSE Linux 6.1 and 6.3Beta for
Alpha,
indicates that "Kernel support for A.OUT binaries" is enabled.
However, it turns out that the source does NOT match the kernel that is
distributed with 6.1 and 6.3Beta
(a rather annoying problem).

More importantly, this feature needs to be part of the default Alpha
configuration.
Binaries built on Alpha Tru64 Unix platforms are in A.OUT (AKA COFF) format
and Alpha Linux platforms really must support these binaries by default.
Red Hat Linux distributions supports these binaries by default.
It is essential that SuSE Linux also support A.OUT (COFF) format binaries.

>From a quality perspective it would also be wise for SuSE Linux to
distribute a kernel that
matches the source configuration as distributed in /usr/src/linux.

Please let me know if you have any questions about this.
Also, when can we expect to see this fixed in a SuSE Linux release?

Thank you in advance for your attention in fixing this situation.

Regards,

John Goshdigian
