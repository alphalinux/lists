On Mon, 31 Jan 2000, Erik Arneson wrote:
> > > init complains first about not being able to run the /etc/rc.d scripts,
> > > and then about not being able to run /sbin/mingetty. I can't even boot
> > > to single-user mode. An 'ls' command from the MILO> prompt shows that
> > > /sbin/mingetty and the /etc/rc.d directory stuff is all there. What
> > > could be causing this problem?
> >
> > Could it be that the permissions isn't correctly set up. I haven't seen
> > this problem myself. Can you boot with the "Boot installed system" in the
> > bootdisk YaST?
>
> I can't install with the "Boot installed system" option, either. One of
> my coworkers suggested that perhaps the SCSI driver isn't being loaded
> correctly, or something like that. How would I be able to check or
> specify which modules to load at boot time?

I doubt that it's a SCSI or disk problem, otherwise the system would not know
about the /etc/rc.d scripts. But as you are saying it could not run
/sbin/mingetty, it could somehow only be missing libc. Did you install
the package shlibs?
If it is fully installed, there should be a /lib/libc.so.6.1
