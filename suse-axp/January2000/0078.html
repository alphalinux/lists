"Izaak Bozof (Lucent NPS)" <izaak.bozof@ins.com> writes:

> I have an AlphaPC164 with the latest SRM build (from
> gatekeeper.digital.com) and SuSE 6.1 axp.
>
> Occasionally, I've had problems with the OS freezing up which I think was
> due to some things I was running.

Similar freezes were reported over at the RH AXP list. There seems to
be an issue with recent kernel (i.e. 2.2.13 as used in RH6.1 or
2.2.14), SRM AND sound cards. Are you using any sound cards? If so,
try without for a while or try compiling the sound drivers into the
kernel. I had bad luck with a number of combinations myself.

> However, on HALTing the system, and then rebooting all the boot* settings
> in the SRM are cleared!!
>
> What's up with this? Can anyone offer insight into what is going on? It's
> a realllll pain to have to retype the incantation to get it back up..

Are you using aboot as descried in SRM-HOWTO on alphalinux.org? If
not, try using it. You can set all sorts of parameters in
/etc/aboot.conf so that an 'boot dk??' should be sufficient. At least
my install didn't require any more settings at the SRM.
