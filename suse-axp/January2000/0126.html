Last night (or more precisely sometime around 0200 this morning...) I
finally overcome my AS200 installation problems.

Since I have had problems installing, even when I used the procedure which
worked for Justin et al. (i.e. boot via ARC from floppies, select NCR810
scsi modules), and the symptoms seemed to indicate some kind of CD read
error, I hooked up my little laptop beside the AS200, booted from the
floppies and proceeded to do an FTP installation, which worked. The only
problem was I had to restart yast several times since I was not prompted to
switch CD. (I understand why not...)

I then booted the installed system and copied vm_full.gz from the CD, and
rebooted. I was not surprised when this boot failed (milo said something
about "length error" when decompressing the kernel), so I once again booted
from floppies and copied vm_full.gz via FTP from my laptop. This time the
decompression of the kernel worked and I can boot directly from disc.

I have also sucessfully upgraded the boot block and can now boot from both
ARC and SRM.

Now, I would dearly like to investigate the CD read problems further. Since
I did not get any errors when I copied vm_full.gz from CD but it neverheless
failed to decompress, I need some other simple test method. Any suggestions?

I would also like to state that, IMOHO, the sections in the manual covering
ARC and SRM boot setup are woefully inadequate.

1) There is an entry in the SDB covering ARC setup, which ought to be in the
manual, but this too fails to mention that you must yourself copy milo and
linload.exe to disc.

2) The SRM section in the manual should mention swriteboot.

Enough said... Viel Spaß!
