containing: SRM 7.2-1; ARC console 5.70; OpenVMS PALcode 1.20-16; Tru64

>What is the OS?

I'm running SuSE Linux 7.0 for Alpha, using the 2.2.16 kernel and I boot it
with aboot from the SRM console.

>What are you using to install packages?

I use the standard YaST setup tool.

What happens is: Almost every time I try to install some packages, my
screen simply turns blue, showing the message cpu halted error code 7
machine check while in pal mode pc = 18400 kindly it does offer a prompt to
type init at, but still, at other times more garble rolls over the screen
ending with a message to press the HALT key (whatever that may be). Most of
the time it occurs while YaST is updating the configuration files, but
during the first time installation it also happened while copying packages.
Therefore I had to install the default system, and afterwards I added X and
the lot one at a time (often having to retry several times before it got
installed). But things like configuring a soundcard (using SaX) have also
made the error occur. During normal operation however this problem hasn't
occured ?yet?.
