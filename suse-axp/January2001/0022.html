First, there is no xserver for the TGA2 card, as it uses a Cirrus Logic
chip with is not supported.

> For some reason while trying to get X to work something got trashed.
> When I boot the disk from SRM>> "boot dkb0 -fl 0" the system boots
> up but I'm still in SRM. The last message I see is SRM telling me
> it's booting the particular kernel. The machine boots all the way
> up to run level 2. I can blindly login and shutdown but no linux console
> is displayed. What have I done. If I boot from the 7.0 CD #1 it works
> ok and I can get to the point of booting an installed kernel in yast
> and it works ok from there. I've actually done a complete re-install on
> a
> different disk. Same thing. What could have happened?

If you start YaST from CD and then start the installed system, You're
still running the kernel from CD, so I think the TGA-Kernel is wrong.
(AFAIR you're booting with ...-fl 0 which starts the default kernel w/o
TGA-support)
The only solution I can see is to use another VGA-adapter (or no X).

The best experiences I made with G100/200 or Permedia 2.
