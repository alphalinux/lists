Hi!

I cc'ed the xfree list, since this mail contains some suggestions
for them, too.

Please cc me, because i am not subscribed to xpert@XFree86.Org,
thanks!

Some time ago i reported in the suse-axp list,
that my newly installed SuSE 7.0 system (Linux alpha, ruffian) stopped
working with X. (In 6.1 and 6.3 my card worked without any problems).
I made a SuSE support call, but they gave up.
I even bought a new SiS 6326 card and replaced the ATI Xpert.
This setup didn't work either and produced nearly identical error messages
(None of the configured devices were found - but there is a line
in the error output of X, that shows, that the card was detected).
All this was with XFree 3.3.6.

Now i tried a different approach:
I downloaded XFree 4.0.2, but the binary version contains the sis driver
only for intel architecture, but not for Alphas (the same holds for ati).
So i gave the german telecom a few bucks and downloaded the sources, added
the sis driver to XF86CardDrivers in the file
xc/config/cf/xfree86.cf (otherwise sis will not be built) and did a make World
(after that i gave the coffee industries a lot of bucks).

After struggling with some installation problems (make installs bombs, if
the app-defaults isn't a directory but a symbolic link like it is in SuSE),
i got a nearly usable X.
I would suggest to replace 'if [ -d foo ]' with 'if [ -d foo -a -L foo ]'
or something more senceful in the Makefile (install section) in programs/bitmaps iirc)

The remaining problems:
- i can only use 8 bit color depth otherwise i somehow don't get any fonts.
- i cannot use hardware acceleration, because if i do so, X comes up and
  lookes really nice, but if i move a window, it looses its complete interior's
  content (i.e. its interior becomes completely white - which differs
  from my bg color).

This X setup is usable but incredible slow.

My questions:
- Has the sis driver on x86 the same problems?
- If it has, why isn't it built on Alpha systems, too?
- Could the xfree experts at SuSE provide a XFree 4.0.2 rpm update?

Thanks for any help an advance,
Martin.
