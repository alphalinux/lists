This seems to happen quite often on AlphaServer machines. Could you send me
your .config file?
I suggest to use a generic kernel. This is the more stable and more tested way
and it should not be slower at all since it uses a vector table, the machine
vector, to do I/O and other machine specific stuff.

> - - During boot I get a "Hard Link failed, Symbolic Link used instead" when
> it tries to set the timezone using tiz in "sbin/init.d/boot". When I try to
> run "tiz" manually (using the "Europe/Berlin" Timezone) I get the same
> message!

This happens, when /usr and /etc are not on the same disk. Generally zic
(time zone compiler) tries to link /usr/share/zoneinfo/YOURTIMEZONE to
/etc/localtime using a hard link. In this case a hard link is not possible,
since the two files are on different disks/partitions. So the boot script just
sets up a soft link instead.

> - - I cannot make the machine use Software RAID-0!
[..]
Have you run an mdcreate before the mdadd stuff?
       /sbin/mdcreate -c8k raid0 /dev/md0 /dev/sda8 /dev/sdb1 /dev/sdc1
If you did that and reboot the machine, it should register the md device
with mdadd and mdrun automatically when booting the machine.
(Sorry, if you already did that, but you didn't tell :-) ..)
If you don't do this, the devices might work but do unexpected things like
data loss etc...
