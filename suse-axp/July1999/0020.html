Hi,

today I installed SuSE 6.1 on a AS 800 (5/500) and ran into several Problems. I really hope that
somebody here can help me:

(I installed using "aboot", since there is no MILO-Image for the AS 800)

- - From time to time (today twice) the machine dies completely. One time it happened after I
opened "fdisk /dev/sdb", selected b and had it print the disklabels. Suddenly no
Keyboard-Entry was possible, a PING from another machine didn't get any response from the
server and so on - I had to reset.

- - I wanted to compile an optimum Kernel for this machine. I selected "Noritake" as the type and
configured everything not to use Modules, then, after re-checking the config, performed a
make dep ; make clean ; make boot (btw: one time the machine hang completely (see above)
during the "make boot" part). After that I moved the "vmlinux.gz" from the .../arch/... Directory to
the /boot. Then restarted with "boot dka0 -fi /boot/vmlinux.gz -fl root=/dev/sda1". Then I get the
usual aboot-Messages, then "Loading compressed Image", "OK, now booting the Kernel" -
then the screen flashed for short (got a short black-flash, then immediately got blue again) and
I got a "Halted CPU 0 / halt code = 5 / HALT instrauction executed / PC=fffffc0000310088". When
I instead boot with the -fi /boot/vmlinuz (to boot the standard-kernel that came with SuSE 6.1)
the machine boots 1a (btw: I use the beta-qlogicisp Driver, as the Mainboard uses a Qlogic ISP
1020a Controller).

- - During boot I get a "Hard Link failed, Symbolic Link used instead" when it tries to set the
timezone using tiz in "sbin/init.d/boot". When I try to run "tiz" manually (using the
"Europe/Berlin" Timezone) I get the same message!

- - I cannot make the machine use Software RAID-0!
I set up three Disklabels, each of the same size, on thee disks. These are the "h" Disklabel on
/dev/sda and the "a" Disklabel on /dev/sdb and /dev/sdc (unfortunately only one controller).
Reboot. Then I performed a "mdadd /dev/md0 /dev/sda8 /dev/sdb1 /dev/sdc1", then a "mdrun
- -p0 /dev/md0", then "mkfs -t ext2 /dev/md0", then mounted the partition, the umounted again,
"mdstop /dev/md0" and then rebooted. Then tried to "mdadd ..." again, but got an error for
/dev/sdb1 and /dev/sdc1. Tried several times, then went into "fdisk", selected "b" -- and got a
message, that there were *no* disklabels on /dev/sdb and /dev/sdc. It seems, the RAID-0 -Set
mysteriously erases the Disklables off the two disks (but not off the /dev/sda)!

I just realized that I started the "a"-Disklabels on /dev/sdb and /dev/sdc @ Cyl. 1 (length of 290
Cyls) - while I started at some bigger Cylinder @ /dev/sda! May this cause the problem (may I
not start a used disklabel @ Cyl. 1 but at some bigger Cyl.? (I do have a "c" Label on all Drives
that spans the whole drive!))??

Would really be great if somebody could help me with one or the other of the Problems I have!

Thanx in advance,
_ralf_
