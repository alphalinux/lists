On 18-Jul-99 Ralf Folkerts wrote:

> I will install SuSE 6.1/AXP on an AlphaServer 800. I can "only" use SRM/aboot
> on that machine
> and so have to use BSD-Disklabels...
>
> ... on the other hand I'd like to set-up a Software-RAID-0 set on two or
> three disks.
>
> Will the Software-RAID work with these Disklabels or are DOS-Style Partitions
> mandatory??

The RAID-Code should work OK with any disklabel. As far as I know the SOFT-RAID
code is widely independant from lower access layers of the kernel. You should
be able to put any block devices together to a RAID. (I think I heard of
someone having a raid1 with /dev/ram and /dev/fd0 to copy floppy disks in an
exclusive manner.
I don't know whether it's possible to boot from a Soft-RAID-Drive though.
This might work with RAID1 but it's unlikely to have a RAID0 or RAID1 boot
device while booting directly from SRM. but this can solved with a small
partition containing the kernel and aboot configuration stuff etc...
