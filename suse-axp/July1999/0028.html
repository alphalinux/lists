Stefan Reinauer wrote:
>
> On 26-Jul-99 Ralf Folkerts wrote:
>
> > Is there a more recent version that doesn't cause
> > these problems (where the data and instructions are aligned on
> > Alpha-Word-Boundaries?)?
>
> no, afaik. But since it's some work to track down each and every unaligned
> access without bringing a big win on the other hand, it's in my opinion more
> like something with very low priority on the todo list.

First of all: I know nothing about squid (please excuse
my ignorance).

But: Shouldn't it be sufficient to recompile squid with
`-mcpu=ev56' (or higher), assumed that the target machine
has the BWX extension?
Even better: To compile a recent gcc (was egcs) snapshot
on the target machine first, which then automatically
will build binaries for the target machine without the
need for `-mcpu=xxx' options.
