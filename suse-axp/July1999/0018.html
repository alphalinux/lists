I also recently installed Linux on a Alphaserver 2100 model 4/233. The biggest
problem is
to get it to boot. On this type of machine you have to use the SRM console and
aboot.
I used the Debian distribution since I still don't know where to get the SuSE
AXP stuff.
But nevertheless the boot procedure should be pretty much the same. If you want
to boot
from the hard disk you have to use BSD disklabel style partitioning
Here is a rough procedure (probably incomplete but hopefully gives you an idea)

-first go into the SRM console (press the HALT button and power up or reset the
machine)
-now use your boot floppy with boot dva0 ...
-when you are done with the installation (remember use BSD disklabel style
partitions)
  you have to configure aboot
-after that you go into the SRM console and set the following variables like
set auto_action BOOT
set boot_osflags 2 #corresponds to the number in your aboot.conf
set os_type UNIX
--------------------------> press reset to test this

Hope that helps a little.
