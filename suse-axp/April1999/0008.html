Hi,

I´ve tested the bttv driver with a WinTV card (WinTV radio Model 405
with old BT848, but also with MSP3400). It works out of the box with
kernel 2.2.5. For module loading via kernel mod you have to add the
following entries to conf.modules:

alias char-major-81 bttv
options bttv card=2 pll=0 radio=1
pre-install bttv modprobe -k tuner; modprobe -k msp3400
post-remove bttv modprobe -r msp3400; modprobe -r tuner

The bttv options have to be adjusted for your card. Examples are
in /usr/doc/packages/bttv/README.SuSE, if you install the package
bttv from gra1, but they are only ok for newer WinTV cards.

The tv program xtvscreen from this package makes problems. Sometimes
the tv windows appears somewhere on the screen or the tv screen
flickers or the X server crashs. But it´s the same on Intel.

I try to compile and run xawtv from Gerd Knorr. It´s very stable on
Intel. But there are known problems with xawtv and glibc-2.1
