ok, this is interesting (maybe a little, anyway). so i've got these AS250's.
one i'd been playing around with suse 6.3 and one with tru64 unix on it. when
i tried to install 6.4 on the one i had 6.3 on it failed to boot (cdrom or
floppy) with the error in the earlier email (attached). here is the
interesting part. i'd written a BSD disk label to the disk with a partition
like:

a /dev/sda1 boot (1-2)
b /dev/sda2 /boot (48MB)
c /dev/sda3 swap (128MB)
d /dev/sda4 / (rest of disk).

i was able to get 6.3 install using the 6.1 aboot and install diskettes but it
was a little tedious. now 6.4 fails to install on this disk. sooo, i built
some debian floppys, booted debian, rebuilt the disk using fdisk leaving
cylinders 1-2 free. debian booted fine from aboot. next i tried to boot the
suse 6.4 CD using the debian formated disk, boots fine. at the partition
screen i dropped into a shell, and rebuilt the about partition table using
fdisk and now 6.4 fails to install again on this machine. sooo i grabbed a
fresh, tru64 unix, AS250 out of the rack, 6.4 cd boots fine but fails to boot
after i build a BSD disklabel.

so what is the correct way to partition out the disk for booting from SRM
console?
